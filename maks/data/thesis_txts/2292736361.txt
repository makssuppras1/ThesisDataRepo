Summary (English)
The number of individuals aﬀected by type 2 diabetes is rapidly increasing.
The goal of this thesis is to investigate if type 2 diabetes can be predicted more
accurately from genome-wide association data using machine learning methods
opposed to traditional statistical methods. A variable selection process using
random forest has been performed and the variables in the genome, called Sin-
gle Nucleotide Polymorphisms (SNPs), showing the highest importance for the
prediction of type 2 diabetes have been identiﬁed. It has then been considered
if including these SNPs in the models over only using clinical variables or previ-
ously univariately identiﬁed SNPs will improve the performance. Furthermore,
the possible improvement by using random forest over logistic regression has
been considered.
The analysis has resulted in identifying genes through the SNPs that are related
to biological functions related to type 2 diabetes. This includes genes which have
not been directly associated with the disease. These are interesting for future
study. However, the results show little to no improvement in prediction perfor-
mance over models using only clinical variables suggesting that the signal for
type 2 diabetes in the genome-wide association dataset is weak. Similarly, there
is no improvement from using random forest over logistic regression for the ﬁnal
models suggesting that the linear signal in the genome data is much stronger
than any non-linear signal.

Chapter 1
Introduction
Type 2 diabetes is a common disease; the number of individuals aﬀected by it
is rapidly increasing and is projected to reach 552 million worldwide by 2030
[WGWS11]. It is a metabolic disorder characterised by insulin resistance and
relative insulin deﬁciency and has both environmental and genetic determinants
[WPD+14]. It is among other factors associated with over-nutrition and a seden-
tary lifestyle and can be delayed or prevented through lifestyle choices (i.e. a
change in diet, an increase in exercise etc.). It is therefore important to identify
high risk individuals early, such that they can get help and direction towards
preventative behavioural changes. This is traditionally done using clinical risk
factors, such as the ones mentioned above, but many of these appear late fol-
lowing years of metabolic dysfunction. Since a propensity for diabetes might be
present from birth, it is hoped that high risk individuals can be identiﬁed earlier
in accordance with their genetics.
In the past, the genetic risk for type 2 diabetes has been calculated by identifying
genetic associations univariately and then combining these additively into a
genetic risk score [WPD+14]. That is to say, each associated genetic variable
is weighted by their eﬀect size and summed. In this project, we will consider a
genetic risk score that takes interactions and correlations into account, and, in
that way, learn more about the condition and understand better how it arises.
We will do this by using machine learning methods. The goal of machine learning
methods is to automatically detect patterns in data and use these to predict
==PAGE:14==
2
Introduction
future data and outcomes [Mur12]. The ﬁeld is thereby closely related to the
ﬁelds of statistics and data mining. Machine learning can be divided into two
types: supervised and unsupervised learning. In supervised learning, the aim
is to establish a mapping from input to output given a dataset consisting of
labelled pairs of input and output (e.g. the classiﬁcation of diabetes cases given a
GWAS dataset with individuals labelled as cases and controls). In unsupervised
learning there are no pairs of data, but instead only data in which we attempt
to ﬁnd some kind of structure; this would, however, not be applicable to this
setting. If we consider the output variable in supervised learning it can either
be categorical, resulting in a classiﬁcation problem, or real-valued, resulting in a
regression problem. For the diabetes cases, we have a classiﬁcation problem as
described above. The machine learning methods can also be used for selecting
the variables that are most important for our predication, and this is what we
need to identify important genetic variables in order to compose a genetic risk
score.
The machine learning method used is the random forest algorithm [Bre01].
It has been chosen because of its ability to handle large datasets with many
variables and few observations. Moreover, because it can identify important
variables and estimate interactions. The performance of random forest will be
compared to logistic regression in order to consider if it obtained a signiﬁcant
improvement.
We want to ﬁnd out if type 2 diabetes can be predicted more accurately from
GWAS data using machine learning methods as opposed to traditional statistical
methods. Furthermore, we want to identify risk genes and ﬁnd out if this can
improve on the prediction from clinical variables.
1.1
Background
This section brieﬂy explains the genetic terminology needed to understand the
dataset, the analysis and the results obtained.
The genome is organised in 23 chromosome pairs (22 + two sex-chromosomes)
half from each parent. The DNA is composed of four building blocks, called
nucleotides. Each nucleotide is composed of a nitrogen-containing nucleobase
- either guanine (G), adenine (A), thymine (T), or cytosine (C) - as well as a
monosaccharide sugar called deoxyribose and a phosphate group. Two nucle-
obases form a base-pair. There are not two people (except monozygotic twins)
which have the exact same DNA and many diﬀerent forms of genetic variation
exist. The most common form are changes in a single base-pair at a given po-

pronounced "snip") [BM12]. These genetic variants are studied in genome-wide
association studies (GWAS’s). There are typically two, but sometimes more,
diﬀerent base-pairs involved in a SNP. The least common base-pair is called the
minor allele, and is used as the reference point.
If the minor allele is in neither of the two chromosomes in a pair, we display 0

A SNP is a diﬀerence at one base-pair in the DNA sequence. These diﬀerences
can be located within or near a gene, which is a range of base-pairs in the DNA.
The genes are coding parts the DNA, often coding for a protein. A range of
base-pairs before and after the gene is associated with the expression of the
gene and hence a SNP can be associated with a gene without being located in
it. This also means that SNPs can be associated with more than one gene and
genes can be associated with more than one SNP. An illustration of these two
concepts can be seen in Figure 1.2.

1.2
Related Work
Machine learning methods have previously been applied to GWAS datasets.
Here, some related work is presented.
A genetic risk score (GRS) comprised linearly of 62 SNPs has been shown to
predict type 2 diabetes on the Framingham Oﬀspring Study dataset by Walford
et al. using logistic regression [WPD+14]. They have further shown that the
62-SNP GRS provides complementary information to metabolite biomarkers.
However, in the article they also consider prediction using traditional clinical
risk factors, e.g.
BMI and fasting blood glucose, and this was only slightly
improved by adding the 62-SNP GRS to the model. They found that the GRS
added more information for younger versus older individuals.
Vassy et al. considered the Framingham Oﬀspring and the Coronary Artery Risk
Development in Young Adults (CARDIA) studies [VHP+14]. They found that a
genetic risk score composed of 62 previously identiﬁed SNPs could predict type 2
diabetes in both European and African populations. However, the improvement
was weak compared to traditional clinical risk factors. The 62 SNPs are from 65
independent type 2 diabetes loci identiﬁed using mainly European individuals
by The DIAbetes Genetics Replication And Meta-analysis (DIAGRAM) [MV12]
and the risk score is a weighted sum of the number of risk alleles (zero, one or
two) at each loci. Lastly, Vassy et al. suggest that genetic risk scores which do
not assume independence of the loci or additivity of their eﬀects may improve
the prediction performance.
==PAGE:17==
1.2 Related Work
5
A way to include dependence between loci is to use machine learning methods.
In 2012 Kruppa et al. made a review of machine learning methods used on asso-
ciations between genetic variants and phenotypes [KZK12]. They went through
the steps needed to evaluate and validate a rule of classiﬁcation or probability
estimation for a speciﬁc phenotype. They further did a systematic literature re-
view, in which they found that there have only been few applications of machine
learning on GWAS data and that these have not fully exploited the possibilities
of the machine learning methods.
The most frequently used machine learning method in the context of genomic
data analysis is random forests. Chen et al. reviewed the applications of the
method including prediction, classiﬁcation, and variable selection [CI12]. For
variable selection they consider diﬀerent variable importance measures, which
can be used to rank and select variables.
For GWAS data they argue that
random forests are appealing because they can prioritize SNPs both according
to marginal and interaction eﬀects. Further they suggest that one rank using
permutation importance in GWA studies, though one should correct for bias
from linkage disequilibrium in small-scale studies. This is not necessary in large-
scale screening studies.
The ﬁrst application of random forests on a GWAS dataset was by Goldstein
et al. in 2010 [GHCB10]. They found that random forests were feasible to use
on GWAS data but that the standard default parameters of the algorithm was
not a good choice for GWAS data.
They further found that the number of
trees to grow depended heavily on the strength of the signal in the input. They
considered variable importance in random forests, more precisely, permutation
importance. They discussed that this way of determining variable importance
could give problems, since two SNPs in perfect linkage disequilibrium would be
chosen only half the time each and hence get lower importance.
Kang et al. ﬁltered their GWAS dataset using linkage disequilibrium pruning
[KYY13]. This was done by keeping only the most signiﬁcant SNP in LD with
other SNPs in the region in the dataset. They then selected SNP combinations
according to the Bonferroni threshold in order to further reduce the complexity
of the method. Finally, they performed variable selection using random forests.
They found that combinations of not only signiﬁcant SNPs but also with SNPs
with low statistical powers had lower error rates.

2.1.2
Random Forests
Random forests is an ensemble method that uses multiple decision trees for
regression and classiﬁcation [Bre01]. A decision tree works by splitting the input
space recursively and making a local model for each part. This is represented
by a tree as the one shown in Figure 2.1. At the root the variable x1 is split on
whether it is less than a constant c1. Each new node will then recursively be
split as seen in the ﬁgure. This results in a local model for each part that the
input space is divided into. The procedure continues this way until a certain
purity is reached, e.g. all observations at a leaf is in the same class, or a set
number of maximum nodes.

forests use a greedy method that ﬁnds the locally optimal maximum likelihood
estimation. This means that it chooses a variable and the best threshold to
split on for that variable by minimizing a given cost. This can be formalized as

2.1.2.1
Variable Importance
The importance of a variable for the accuracy of the model can be estimated
using random forest [Bre01].
This can be done in multiple ways.
We have
chosen permutation importance because it was suggested by Chen et al. for
large GWAS datasets [CI12]. The importance is found by permuting the values
of each variable and then measuring how much the permutation decreases the
accuracy of the model.
More precisely, suppose there are p input variables.
For each forest that is constructed, the values of the jth variable in the out-
of-bag observations are randomly permuted and the out-of-bag examples are
run down the trees. The classiﬁcation of the out-of-bag observation is saved
for all j = 1 . . . p variables. In the end, all the out-of-bag class votes for an
observation with the jth variable permuted are compared to the true class label
of the observation. This gives a misclassiﬁcation rate. The output is then the
increase in misclassiﬁcation as compared to the out-of-bag rate with all variables
==PAGE:23==
2.2 Performance Evaluation
11
unpermuted. This output is what is known as the mean decrease accuracy or
permutation importance. We will refer to this simply as the variable importance.
When considering the variable importance it is important to be aware that if
two variables are highly correlated the variable importance can be misleading.
This is because they carry similar information and hence if one variable is chosen
then the second will not be and hence has a lower importance. Furthermore,
it should be noted that permutation importance can be misleading if variables
vary in scale of measurement or number of categories [Qi12]. This is however
not a problem for GWAS data since all the variables are of the same type.

2.2.1
ROC Curve
In binary classiﬁcation a prediction between 0 and 1 is ﬁrst obtained. Thereafter,
a threshold τ between 0 and 1 is set that deﬁnes the classiﬁcation boundary
between the two classes and thereby deﬁnes the decision rule. When the decision
rule with a speciﬁc τ is applied the predicted classes are obtained, which can
either be the true classes or not. If the true classes have not been predicted, then
there are errors and these can either be false positives (FP) or false negatives
(FN). In our case a false positive is one predicted to get diabetes but does not
and vice versa. If predicted correctly, the results are true negatives (TN) or
true positives (TP). These numbers can be combined into a confusion matrix,
an example is shown in Table 2.1.

2.2.1.1
Area Under the Curve (AUC)
The area under the curve (AUC) is the area under the ROC curve. It ranges
between
1
2 for a completely random model to 1 for a model that separates
the classes perfectly, consequently a higher AUC is better. It has been shown
[Mur12] that the AUC equal to the probability that a randomly drawn positive
observation has a higher score than a randomly drawn negative observation,
thus the AUC measures the ability to rank items correctly.
The AUC summarizes the ROC curve into a single value and therefore not
contain as much information as the curve itself but it simpliﬁes performance
comparisons. Note that AUC have been criticized for making improvements in
performances diﬃcult to see [FHT01], and should therefore be supplemented by
other measures such as ROC and error rate.
2.2.2

A simpler way to consider the errors is to look at the overall error rate,
Error Rate = FP + FN

not distinguish between classes in the errors and therefore gives less speciﬁc
information than the ROC. It can, however, provide another perspective on the
performance.
2.2.3
Cross Validation
In order to estimate a value for the diﬀerent performance measures a test set is
needed. There are three standard methods to create one. The ﬁrst is holdout
where the dataset is separated into two sets: a training and a test set. The
second is cross validation where the dataset is split into k folds and the model
is trained on k −1 folds and tested on 1 folds. This is then repeated k times,
once for each fold. The last is bootstrap where the training set is created from a
dataset of size n by sampling n observations uniformly with replacement. The
test set contains the observations that are left out.
Here the method of cross validation is explained [FHT01]. In k-fold cross valida-
tion one splits the dataset in k roughly equally sized subsets. These are random

samples without replacement such all observations are in exactly one subset.
We then use k −1 of the subsets as a training set and 1 as the test set, as
illustrated in Figure 2.3. This is repeated k times once for each subset 1, . . . , k
and the estimated prediction error from each iteration are then combined, e.g.
by estimating their mean and standard deviation.

Stratiﬁcation can be used to make a better estimation of the prediction error.
That is, each of the subsets is not chosen completely random, but random within
in each class, i.e. the ratio of the classes are maintained within the subsets.
Kohavi has reviewed the three diﬀerent standard methods for accuracy estima-
tion and model selection [Koh95]. The latter two, cross validation and bootstrap,
were tested on a range of datasets and they found that stratiﬁcation is gener-
ally to prefer and that bootstrap has very high bias for some problems. They
recommend that one uses ten-fold stratiﬁed cross validation

3.2
Modelling Strategies
Two kinds of models were set up: Logistic regression and random forest. Logistic
regression only took the linear eﬀects into account whereas random forest also
considered interactions between variables. Random forest was selected because
it was good at dealing with small samples and large number of variables, which
was the case for GWAS data [Qi12]. Both types of models were set up in order
to consider if there were any interaction eﬀects which could signiﬁcantly improve
the prediction over the standard logistic regression.
The models were set up on diﬀerent subsets of the data. All the subsets had
their missing values imputed, see Section 3.2.1. First, the models were set up
using only clinical variables. These were included in three levels: age and sex;
age, sex, and bmi; and all phenotype variables listed in Table 3.1. Models with
only phenotype variables were made for comparison so we could evaluate the
improvement in prediction obtained by including SNPs. The three levels are
chosen because bmi is known to be a strong predictor of type 2 diabetes.
Two diﬀerent sets of SNPs were included into the ﬁnal models. The ﬁrst set is
the previously identiﬁed SNPs listed in Table 3.2. The other set is the SNPs
which were identiﬁed using the variable importance from random forest models

3.2.1
Imputation
The missing values were imputed in two diﬀerent ways. They were either im-
puted by setting all the missing values to zero, since that is the most common
value, or by using the R function rfImpute found in the randomForest package
[LW02]. The function rfImpute ﬁrst imputes with the mean and then run the
random forest algorithm. The proximity matrix from the random forest is used
to update the imputed values using a weighted average where the proximities
are the weights. The process is repeated ﬁve times. All models used for ﬁnal
prediction were trained and tested on data imputed using rfImpute. However,
the large random forests used in the variable selection process, see Section 3.3,
were built on data imputed with zeros. Speciﬁcally, the large number of forests
built in parallel in the variable selection process and all forests containing more
than 1,000 variables. This compromise was chosen due to high computation
time of the rfImpute imputation algorithm.

schemes (referred to as method 3) is explained here. For an example of the R

Correlation
As discussed in the introduction, Section 1, not all SNPs are independent, but
some SNPs are in linkage disequilibrium and hence correlated. The correlation
matrix of the 239 SNPs chosen in at least one fold was therefore considered. The
correlation matrix ordered according to the order in Figure 4.4 can be seen in
Figure 4.6. In this ﬁgure the positive correlation is green, negative is red, none
is white, and a correlation of 1 (i.e. with themselves) is grey. Most of the SNPs
were uncorrelated but there were some SNPs that were strongly correlated.
The correlation matrix was reordered according to the correlations to further
explore these correlation groups, see Figure 4.8. There were small clusters of
SNPs which were highly correlated and thus groups of SNPs expected to carry
the same or similar information. From the plot it is seen that the SNPs were
either strongly correlated or almost uncorrelated. A histogram of the highest
absolute correlation with another SNP for each SNP was created to further
examine the correlations. That is, a histogram of the following,

Chapter 5
Discussion
In an eﬀort to see if machine learning methods could predict type 2 diabetes
more accurately from GWAS data than traditional statistical methods, we have
performed classiﬁcation using random forest and logistic regression. Further-
more, random forest was used to identify SNPs that were important for the
classiﬁcation.
5.1
Previously Identiﬁed SNPs
The ﬁrst results using the previously identiﬁed SNPs show that including these
SNPs gives an improvement over only predicting diabetes from age and sex. This
means the SNPs help predict type 2 diabetes, which was expected because the
SNPs have previously been associated with the disease. Both random forest and
logistic regression were tested. We saw that there was no signiﬁcant diﬀerence
between the performances of these, meaning that random forest did not utilize
any interactions between the previously identiﬁed SNPs. Even though there is
no signiﬁcant diﬀerence, we see that the AUC is slightly higher for the logistic
regression.
This can possibly be explained by the way the SNPs have been
identiﬁed. All these SNPs were univariately identiﬁed, and hence may favour
the additive nature in which the eﬀects are considered in logistic regression.
==PAGE:60==
48
Discussion
5.2
Variable Selection
Variable selection using random forests was performed. Many of the variables
were selected in many of the folds, and were consistently important for the
prediction. However, there was also a large set of variables where the variables
were only selected once. The latter type consists of two types of variables: The
ones that are mostly noise and the ones that carry similar information as other
SNPs, which are then also not chosen in all folds. This is because the variables
chosen at each split in the trees are chosen given previous splits and hence
similar variables will not be chosen in the same tree. This will then lead to a
decrease in the overall importance for all SNPs despite the eﬀorts of averaging
out the trees. This must however be seen in the light of logistic regression, were
similar variables are also not chosen, since they do not provide complementary
information. But in logistic regression there is no averaging, and the similar
variables would never be chosen.
In order to describe the similarities of the selected variables, they were grouped
into groups of strongly correlated SNPs. When considering these groups, see
Figure 4.9, we see that the SNPs within the groups do not have consistently high
importance. This means that some of the SNPs, that have low importance and
seemed like noise in the selection, are important but simply not chosen because
of the similarities to other SNPs.
The groups with the total highest importance when considering all SNPs in the
groups were mapped to genes. The correlation groups generally corresponded
well with genes, see Table 4.2. This means that the groups, that were found,
have biological meaning.
One of the genes that were identiﬁed, TCF7L2, is
coding for a transcription factor 7-like2. This gene is known to be associated
with type 2 diabetes [MV12], and hence a gene we hoped to identify because it
shows that the method ﬁnds genes associated with type 2 diabetes. This gene
is mapped from the SNPs in the second most important group, which is the
large group seen in the right side in Figure 4.9. The SNPs in this group all
have consistently high importance and hence are all shown to be important for
the prediction. Furthermore, if one consider the univariate p-values for each
of these SNPs, see supplementary Table C.1, it can be seen that these are also
genome wide signiﬁcant, i.e. have a p-value lower than 5×10−8. This is also the
only one of the newly identiﬁed genes that were also identiﬁed by the previously
identiﬁed SNPs.
We further notice the genes RUNX1 and HLCS. RUNX1 is suspected to be as-
sociated with type 2 diabetes [ZTZ+13], and HLCS encodes for an enzyme that
catalyses the binding of biotin to carboxylases and histones, which is associated
with type 2 diabetes [LdlVV+12]. The fact that these are identiﬁed here gives
==PAGE:61==
5.2 Variable Selection
49
additional indication that they are interesting for further studies.
The gene
RUNX1 is identiﬁed by the SNP rs3132538, which is in the sixth most impor-
tant group. The univariate p-value for this SNP is 1.62 × 10−5 and hence it is
not genome-wide signiﬁcant and would not be identiﬁed by traditional methods.
The other gene, HLCS, was identiﬁed by three diﬀerent SNPs also in the sixth
group. Two of these SNPs have univariate p-values in the order of 10−6 and on
the order of 10−5. This means that none of these are genome-wide signiﬁcant
but two of them are close. The SNP with the highest importance, rs10739592,
is not known to be associated with type 2 diabetes but is here suggested for
future study, as it has also previously been based on analysis of the GENEVA
dataset [GSA13].
This approach to identifying new associated SNPs is a longer process than uni-
variate identiﬁcation, however it has shown to give results that would not have
been seen using traditional methods while also identifying all the SNPs that
are univariately signiﬁcant in the dataset. This corresponds well with previous
studies which have shown that random forest should be better at ranking true
disease associated SNPs [LHSVE04]. In order to further investigate the signiﬁ-
cance of the identiﬁed SNPs one could reconsider the gene mapping as it is not
fully understood; whether some of the genes interact in relevant gene networks
and thus take more prior biological knowledge into account; or ﬁnally replication
of the results in another GWAS dataset.
The results of slightly diﬀerent variable selection strategies were considered. We
changed the number and size of the random forests used iteratively in the vari-
able selection process. All strategies gave similar performance results and hence
did not ﬁnd any interactions to improve the performance. When reconsidering
the selection method, one reason for this might be the initial blocks and not the
following number of random forest iterations. The data was split into blocks of
1,000 SNPs randomly and thus the probability that two interacting SNPs are
in the same block is very low. Consequently, there was no or little interaction
in the initial round of selection and hence the SNPs might interact with other
SNPs which do not have an importance high enough to be selected to carry
on to later steps in the selection process. An alternative approach to forming
the initial blocks could be to form the blocks according to physical placement
of the SNPs in the genome, hoping that physically close SNPs interact more
and thereby being able to aﬀect to importance in the initial round. One could
thereby hope to ﬁnd an interaction signal. This idea is related to a suggestion
for an extension to the random forest algorithm by Botta et al. [BLGW14].
They suggest to use sub-trees on blocks of physically close SNPs instead of bi-
nary splits at each node, such that correlation between physically close SNPs
can be exploited.

5.3
Improvement On Clinical Variables
The prediction performance of the newly identiﬁed SNPs was then considered.
They gave a signiﬁcant improvement in performance over including just age and
sex, as also seen for including the previously identiﬁed SNPs. This means that
these newly identiﬁed SNPs also carry information regarding type 2 diabetes.
Notice that random forest performs better with the newly identiﬁed SNPs than
with previously identiﬁed SNPs and that logistic regression has the opposite
behaviour. This might be attributed to the diﬀerence in the selection processes.
However, none of the diﬀerences in AUC are greater than the standard deviations
and we cannot conclude that there is a signiﬁcant diﬀerence. Models including
more phenotype variables were also considered to investigate whether including
the SNPs gave en improvement in the prediction performance. It was found
that the SNPs do not give a signiﬁcant improvement and this was true both
for newly and previously identiﬁed SNPs, and hence it could not be shown that
the SNPs provided complementary information to the phenotype variables, as it
was shown by Walford et al. [WPD+14]. It is again noticed that random forest
performs a little better for the newly identiﬁed SNPs, though not signiﬁcant. It
does, however, suggest that there might be some interaction eﬀects that have
a very weak signal but we cannot conclude that either since the improvements
are not signiﬁcant.
The conclusion is that random forest does not perform better than the logistic
regression and hence does not utilize any interaction signal for better perfor-
mance. This could be a consequence of the data structure. The dataset has
many more variables than observations, meaning that even if the interactions
exist we simply cannot estimate them due to the low number of observations.
It stands in contrast to what is commonly known about random forest, as it has
been shown to not suﬀer from the curse of dimensionality but rather improve
in accuracy as it grows in complexity [Ho98]. However, Arevalillo et al. showed
in 2009 that bivariate interaction signals were lost when the number of noisy
predictors increased in high dimensional data [AN09]. This is exactly the case
for GWAS data as only very few of the variables contains information relevant
for the classiﬁcation, here about 50 out of ∼650,000 were expected to. Further-
more, each of the predictors have a very weak signal for the disease as both
they and the disease are common in the population, making prediction a hard
problem.
However, the random forest algorithm is able to ﬁnd interactions between vari-
ables, but if the signal is very weak it is simply not be possible to register. It
will therefore perform better than logistic regression if a stronger interaction
signal is present and hence future applications of the algorithm are appropriate
if such interactions are suspected.

Chapter 6
Conclusion
Classiﬁcation using random forest and logistic regression have been performed.
This was done in attempt to predict type 2 diabetes more accurately from
genome-wide association study data using machine learning methods opposed
to traditional statistical methods.
Variable selection was further performed
to identify the genetic variables, SNPs, which were most important for the
prediction.
The previously identiﬁed SNPs, known to be associated with type 2 diabetes,
gave an improvement over just predicting from sex and age of the individuals.
However, no improvement from using random forest over logistic regression was
seen.
The variable selection process identiﬁed a set of SNPs most important for the
prediction. These were sorted into groups based on their correlation as some
were very strongly correlated. One of the SNPs in the most important groups
of SNPs were associated with a gene, TCF7L2, known to be associated with
type 2 diabetes [MV12]. Furthermore, two genes, RUNX1 and HLCS, which
are suspected to be associated with type 2 diabetes, were identiﬁed. The se-
lection method was shown to give results which would not have been identiﬁed
using traditional methods while also identifying all SNPs that would have been
univariately signiﬁcant.
==PAGE:64==
52
Conclusion
The inclusion of the newly identiﬁed SNPs into models including only age and
sex gave an improvement in performance, similarly to the one found by including
previously identiﬁed SNPs. This means they were shown to contain information
about type 2 diabetes. However, when including more clinical variables such as
bmi, no improvement was seen by including either previously or newly identiﬁed
SNPs. That is, the SNPs were not shown to hold complementary information.
In the same way, no improvement was seen by using random forest over logistic
regression, meaning no interaction signal was utilized to improve prediction
performance. This does not imply that there was no interaction signal but only
that any non-linear signal in the data was much weaker that the linear signal.
