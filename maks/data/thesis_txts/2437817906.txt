Abstract
Buildings oŌen have a performance gap between calculated and actual energy con-
sumpƟon, which can occur due to insuﬃcient post occupancy monitoring of the
building services. To bridge this gap, building commissioning can be a useful tool
to ensure opƟmal operaƟon of the building services via monitoring and analysis of
building data.
This project has invesƟgated the use of IoT sensors for fault detecƟon and opƟmisa-
Ɵon of building services, through data collecƟon and analysis.
15 wireless sensors were set up at DTU SMART Library to monitor speciﬁc parts of the
building installaƟons. Sensors were set up in the venƟlaƟon system, heaƟng system,
as well as to monitor the indoor climate and lighƟng in the building. The sensors
operated on the Sigfox Network, using radio transmission to send data to the cloud.
The IoT sensors monitored the parameters; Temperature, humidity, light, CO2 con-
centraƟon and movement in speciﬁc places in order to evaluate the operaƟon of the
systems. A custom heaƟng pipe sensor was created by 3D prinƟng a new casing for
a Sens’it sensor board.
Assessment of the system operaƟon was done through Ɵme series analysis of the
collected data, by recognising paƩerns and rule-based comparison of the diﬀerent
parameters.
The results showed great potenƟal for collecƟon of detailed Ɵme series data about
the systems, in a cost-eﬀecƟve and user friendly way. The data could be used to au-
tomate demand control tesƟng in the funcƟonal tesƟng in BR18, by long term moni-
toring. Data analysis showed that sensor data could be used to monitor; Room tem-
perature, CO2 concentraƟon in room, light in room, supply air temperature, supply
and return temperature of heaƟng pipes, temperatures in the AHU, damper posiƟon
and movement of the AHU (on/oﬀindicaƟon).
PotenƟal for predicƟon of future behaviour of systems via neural networks was in-
vesƟgated. It was not possible to obtain useful results from the simulaƟons, but with
further detailing of the networks in future work, there could be potenƟal for fore-
casƟng using neural networks.

4.7.1
Neural Network predicƟons
As research on predicƟons of Ɵme series data, neural networks were tested. A neu-
ral network is a machine learning tool based on historical data (Li, Ding, Zhao, Yi, &
Zhang, 2017). A neural network is trained based on a set of training data, and can
be used to predict future values based on said training data. For Ɵme series data,
the applicaƟon of neural networks shows potenƟal in predicƟng behavior of building
systems, based on forecasted values. For this project a simple example with two vari-
ables was used. The neural network was tested to predict future values of the room
temperature, y(t) based on a forecast of outdoor temperature values x(t). This, done
by ﬁrst training the network to a set of x(t) and y(t) historical values.

5.5
Sources of errors
There were several sources of error in the sensor measurements. As determined
by the calibraƟons, there is a large uncertainty on the Sens’it temperature sensor.
However, the measurements seem to be consistently approximately 1.6oC too high,
so if measurements are used for comparison of higher/lower temperatures of Sens’it,
this is not an issue.
The light sensors were very unstable and no reliable measurements of lux values were
possible. For the sensors to be useful, the plasƟc casing should be taken oﬀand a
proper calibraƟon should be done. However, the sensors can be used for indicaƟon
of whether the light is on/oﬀ. AddiƟonally it should be noted that both sensors were
placed on top of work surfaces, which could lead to occupants moving the sensors
or accidentally blocking them. The table on which one sensor was located was used
for group work and would frequently be moved around within the zone.
The frequency of which the vibrate mode sensor was sending messages was unstable,
which could be because of the limited number of messages per day, as well as poor
connecƟvity (despite using a repeater). The signal could be opƟmised by ﬁnding a
beƩer locaƟon for the sensor or the repeater. AddiƟonally, it was diﬃcult to ﬁnd a
place on the AHU where there was enough vibraƟon for the Sens’it to register it. This
could be improved by placing the sensor inside the actual AHU and not outside on
the duct.
CO2 measurements from the Hummbox Air Indoor became unstable over Ɵme, which

5.7
Overall discussion
Generally, there were uncertainƟes in the sensor measurements, which could be re-
duced by a proper calibraƟon of all sensors before use. Most of the collected data
was accurate and could be used for the intended purposes. However, some of the
data was not as detailed as predicted, for example the light sensor. The light mea-
surements could only be used as an indicaƟon of the lighƟng in the room.
There were some uncertainƟes with the magnet sensor, since it was unknown how
sensiƟve the sensor was to movement. For future similar setups it could be useful
to use an 3-axis accelerometer instead, to monitor the exact locaƟon of the damper
and not only open/closed indicaƟon.
The way the heaƟng system is designed made it hard to control during occupancy
hours, since the library has many users who could adjust the seƫngs on the radiators
throughout the day.
It should be taken into consideraƟon that the invesƟgated zone was part of a large
open plan room, which could easily inﬂuence the systems based on condiƟons in the
surrounding zones.
Lastly, the project period was in an overlap of heaƟng and cooling season, which
meant it was possible to test all systems. However, for on-going commissioning based
on sensor data to be eﬃcient, data from at least one full year should be obtained to
include all seasons.
5.7.1
Future work
The next step in automaƟon of commissioning through IoT sensors would be to de-
velop algorithms or rule-based diagnosƟcs of the collected data. Unfortunately this
project did not have the Ɵme frame to derive algorithms for automaƟc detecƟon of
faulty operaƟon.
Ideally, in the future the sensors would be able to communicate back to the sys-
tems and control the operaƟon through advanced data collecƟon and forecasƟng.
In the future the sensors could potenƟally replace tradiƟonal building management
systems, but the technology does not seem to be present currently.

Conclusions
From the ﬁndings in this project, it can be concluded that it possible to obtain detailed
Ɵme series data about building services from low-cost wireless sensors. By using IoT
sensors it was possible to obtain data about room temperature, CO2 concentraƟon,
air supply temperatures, heaƟng pipe temperatures, indicaƟon of damper operaƟon,
indicaƟon of AHU on/oﬀand light levels. CollecƟon of data through sensors showed
to be possible in a cost-eﬀecƟve and user friendly.
It can be concluded that the venƟlaƟon system is in operaƟon majority of the day,
also at night, which is ineﬃcient. From the heaƟng pipe measurements it can be
concluded that the supply and return temperature are close, which indicates eﬃcient
operaƟon. However, this might look diﬀerent in a Ɵme period with a larger heaƟng
need.
FuncƟonal tesƟng of building services can parƟally be automated by using low-cost
wireless sensors. The parts that have been proven potenƟal for automaƟon are
mainly the passive tests for the demand control, which requires data over a certain
period of Ɵme.
From the ﬁndings of this project it can be concluded that opƟmisaƟon of operaƟon
based on sensor data is possible, if further data analysis are carried out.
TesƟng of predicƟon with Neural Network gave no useful results, however if further
invesƟgaƟons were made, the use of neural networks could show potenƟal.
