Abstract
This work aims to develop convolutional neural network models for the purpose of image denoising in
the frequency domain. Preliminarily it is detailed how images can be transformed from the spatial
domain in which the pixel values reside into the frequency domain using a frequency transform. The
frequency transformed used are the Hartley transform and the discrete cosine transform which are both
real-to-real transforms making them simpler to use on images. We find that the Hartley transform is
closely related to the Fourier transform so that the latter can be used to calculate the former. The
noise to be removed in the denoising process is additive Gaussian noise. This is added to images to
create synthetic noisy image data for the denoising experiments. The images used for the denoising
experiments come from the BSD500 dataset. The developed denoiser model is finally used to denoise
images with noise in both the frequency domain and the spatial domain to compare the performance in
each domain. The performance is evaluated using the common image quality metrics PSNR and SSIM.

CHAPTER 1
Introduction
Image processing is an essential part in the technology which plays a major role in many industries
and the daily life of most people. So many tasks and activities hinge on the processing of acquired
and transmitted image data. However, in the acquisition/transmission process the true image data is
guaranteed to be affected by random noise inherent in digital systems. Regardless of the behaviour of
the noise, which depends on the digital system in question, the main concern is the fact that the image
data becomes contaminated to an extent. This is problematic for the image processing that we desire
to perform. Here the field of image restoration finds its importance.
Image restoration covers a several low-level computer vision tasks including image denoising, image
deblurring, image super-resolution, image decmosaicing, etc. In [Zha+20] they use a trained image
denoiser as an image prior in their image restoration pipeline. In this work, the aim is to develop
an image denoiser which operates in the frequency domain that is to be used in a similar restoration
pipeline.
The most widely used neural networks for operating on images are convolutional neural networks
(CNNs). These networks typically operate on the image pixels, that is in the spatial domain. This work
however attempts to create networks that operate in the frequency domain. This means that the image
input shall be transformed from the spatial domain (image pixels) to the frequency domain (image
frequency contributions) using some appropriate frequency transform. The most common frequency
transform is the Fourier transform. This transform maps an input in the spatial domain to the frequency
domain using the complex exponential function. That is it involves complex numbers. In [LYR18] they
used a transform called the Hartley transform which avoids complex numbers. However, they derived
the Hartley transform from the Fourier transform. Thus they effectively compute the Fourier transform
and therefore do use complex arithmetic in the spatial-to-frequency domain transformation. However
this issue is minor as fast and eﬀicient implementation of the Fourier transform are readily available.
Still, [Xue+20] decided to instead compute the Hartley transform directly and thus avoided the use of
complex numbers in their computations. Another frequency transform is the discrete cosine transform
(DCT) which does not use complex numbers. This work is going to utilize the Hartley transform and
the DCT and compare the denoising results in each frequency domain.
With a developed denoiser neural network model the aim is to produce denoising results that are
close to what contemporary models achieve.

2.1 Signals and images

2.1
Signals and images
Here, we establish the definition of a one-dimensional signal. Then we cover the definition of a digital
image. Lastly we introduce two-dimensional signals, and the latter’s interpretation as an image. These
definitions become relevant when we introduce the concept of a frequency transform for one-dimensional
signals and we introduce an extension of the defined transform which applies to two-dimensional signals.
2.1.1
A one-dimensional signal
In its most basic form, the data which is handled in this work consists of a signal which varies with the
change of one or more parameters. In this work we denote the signal f. If the signal is one-dimensional,
it is to be interpreted as a time-varying signal, and so varies with the parameter t. For a given value
t, the signal assumes the value f(t). f can either be continuous or discrete. In this work our interest
lies on signals that are discrete and have finite length, that is t = 0, . . . , N −1. For further details see
[AD14], [AP14], and [BN14].
2.1.2
A digital image, a discrete two-dimensional signal
We use the definition of a (gray-scale) digital image as is presented in [PM17]. To briefly summarize it
here, a gray-scale image is a 2D array of height H and width W. The array represents a rectangular
region of pixels. Each pixel can attain a value in a certain range, e.g. in an 8-bit image each pixel has
a value between 0 and 28 = 255. This array of pixels can be represented by a matrix I ∈RH×W , where
H and W denote the height and width of the image, respectively.
Another representation which will be useful when discussing frequency transforms, is interpreting
the image pixels as values of a discrete function f. The function values depend on two parameters
h = 0, . . . , H −1 and w = 0, . . . , W −1 which represent the pixel location in the image. Note that
this representation is equivalent to extending the one-dimensional discrete signal with finite length to
two dimensions. As the parameter values represent a location in the image (in space), we say that the
signal varies spatially. The image coordinate system is said to be 0-based and the origin is located in
the top-left corner of the image.
For now we have touched on images where each pixel contains a single value which is a measure
of the intensity at the pixel’s location in the image. But images can contain multiple channels, such
that each pixel consists of multiple values which can be represented by a vector. This can be used to
represent color in images, for instance in RGB-images.

3.1.2.2
Connections between the Hartley and Fourier transforms
Now let us compare the two-dimensional discrete Hartley transform in (3.13) to the two-dimensional
discrete Fourier transform in (3.8). The main difference lies in the choice of the basis function: the
Fourier transform uses the complex exponential 2.2, while the Hartley transform uses cas(). Therefore
the Hartley transform definition does not include the imaginary unit, i. Additionally, unlike the Hartley
transform, the Fourier transform exhibits different signs in its basis function between itself and its inverse.
This is why it is not an involution like the Hartley transform. If a two-dimensional input f consists
entirely of real values, that is f ∈RH×W , then the Hartley transform of the signal will also consist
entirely of real values, i.e. H(f) ∈RH×W . This is in contrast to the Fourier transform, the output of
which will always be complex, i.e. F(f) ∈CH×W . This property of the Hartley transform is brought up
in [LYR18] as a reason to use the Hartley transform instead of the Fourier transform. This is interesting
as the images considered for image restoration (usually) do not contain complex values, i.e. the pixels in
the images only attain values in the real domain, R. Applying the Hartley transform to an image to be
manipulated in the frequency domain therefore guarantees that the values in the frequency domain are
all real and not imaginary. [LYR18] argued that this was advantageous for their implemented pipeline
because they avoided having to perform operations with imaginary numbers. Generally, performing an
arithmetic operation with an imaginary number, that is on the form α+iβ, requires two operations: one
on the real part, α, and one on the imaginary part, β. [Bra83] points out an interesting relation between
the Hartley transform and the Fourier transform. The Hartley transform of a real-valued function f can
be expressed as the difference between the real part and the imaginary part of the Fourier transform of
the same signal:
H(f) = ℜ[F(f)] −ℑ[F(f)]
(3.17)
In their work, [LYR18] computed the Hartley transform in this way.

3.2 Generating noisy images

3.2
Generating noisy images
This section the details of how a set of noisy images is generated from a set of clean images.
3.2.1
Additive white Gaussian noise
The corrupting noise considered in this work is additive white Gaussian noise which comes from a
Gaussian distribution N(0, σ). The value of σ determines the noise level in the noisy image.
3.2.2
Adding noise in the frequency domain
A method for adding Gaussian noise was presented in [Gre18]. To add Gaussian noise of percentage
level τ ∈[0, 1] to an image of size H × W, do:
1. Generate a matrix of size H × W with values picked from the Gaussian distribution N(0, 1) with
mean µ = 0 and variance σ2 = 1: R = N(0, 1)H×W ∈RH×W .
2. Normalize this matrix by its Frobenius norm: R = R / ∥R∥fro.
3. Compute the matrix product between the normalized noise matrix and the Frobenius norm of the
image, and multiply the result by τ: τ · R · ∥I∥fro ∈RH×W .
The definition of the Frobenius norm of a matrix is
∥I∥fro =

3.3
Quality evaluation methods
Methods for evaluation of image quality can be either subjective or objective. Subjective methods base
themselves on human judgement and do not reference explicit criteria. Objective evaluation methods
because they are based on comparisons using explicit (numerical) criteria.
Possible references are
ground truth or prior knowledge expressed with statistical parameters and tests. PSNR and SSIM are
objective evaluation methods. PSNR and SSIM are full-reference objective quality assessment metrics
for image restoration. PSNR is widely used. [HZ10] presents the PSNR and SSIM metrics and deduces
an analytical relation between the two. This section details how PSNR and SSIM are calculated. The
details on the derivation of the analytical relation between PSNR and SSIM can be found in Appendix A.
In the following, X ∈RH×W denotes a ground truth image (reference) and Y ∈RH×W denotes the
corresponding degraded image.

3.4 Model design

3.4
Model design
We shall now decribe the convolutional neural network model for denoising images in the frequency
domain. The two works [LYR18] and [Xue+20] provided inspiration for the model.
3.4.1
Model input
The input to the model is an image in the frequency-domain. This was chosen to have a square size
K × K, so H = W = K. The frequencies in the input image is shifted, so that the low-frequency
components lie in the center of the image and the high-frequency components lie toward the edges of
the image.
3.4.2
Convolutional layer operating in the frequency domain
Because the model operates on data in the frequency domain, it is necessary to adhere to the principles
of convolution when implementing a convolutional layer model. The convolution theorem [23b] [AP14]
applies to the Fourier transform and states that performing the convolution operation in the spatial
domain is equivalent to performing multiplication in the frequency domain:
F(f ∗g) = F(f) · F(g).
(3.35)
In conjunction with arguments that the Fourier transform and Hartley transform exhibit analogous
properties, [LYR18] used (3.35) to implement convolution in the frequency domain as a point-wise
matrix product between a weighting matrix and the image.

[LYR18] showed that the ReLU activation function can be redefined using the Heaviside function
and thus becoming a multiplication operation:
ReLU(f)(u, v) = max(0, f(u, v)) = f(u, v)HS(f(u, v)).

By (3.35), multiplication corresponds to convolution in the frequency domain. For this reason, [LH19]
followed the weighting layer by a convolutional layer. This was there way of implementing a convolu-
tional layer followed by activation in the frequency domain.
3.4.3
Model architecture
[Xue+20] built a model consisting of repeated architectural blocks of frequency-domain convolutional
layers akin to the model developed by [LYR18]. The model developed for this work consists of a similar
block architecture. In one block, the block input, Iin, is first weighted by a weighting matrix, W, in
the fashion of a point-wise product:
Iw = W ◦Iin.
(3.37)
Next, the weighted input is passed through N −1 convolutional layers each with several convolutional
filters and tanh() activation function. A final convolutional layer with only one filter produces the block
output feature image:
convj(Iw) = tanh(Cj ∗Iw + Bj),
for j = 1, 2, ..., N,
(3.38)
Iout = convN(· · · conv2(conv1(Iw))),
(3.39)
where Cj is the convolutional kernel and Bj is the bias of the j’th convolutional layer.
The final output of the model is the output of the final block.

Image dataset(s)

4.1.1
The Berkeley Segmentation Dataset (BSDS)
The Berkeley Segmentation Dataset (BSD) is an image dataset originally developed by the Computer
Vision Group at Berkeley University of California to further research in image segmentation.
But
it is also often used to test image restoration models. The data is a collection of natural images and
comes supplied with (human) ground truth annotations of the images as targets for image segmentation
algorithms. Since we are not performing image segmentation we discard the segmentations and just
use the unsegmented images. Different versions of the dataset exist. The first version, BSD100, was
presented by [Mar+01] and was intended exclusively as a test set. The version used in the experiments
conducted as part of this work is the BSD500 [Arb+11]. It is named so because it contains a total of
500 images. These are divided into a training set of 200 images, a validation set of 100 images, and a
test set of 200 images. The images are 8-bit RGB images in .jpg-file format.
Figure 4.1: Sample RGB-image, ”woman”, from the BSD500 test set with dimensions 481×321 pixels..
4.1.2
Image preprocessing
The images in the BSD500 dataset measure either 321 × 481 pixels (horizontal) or 321 × 481 pixels
(vertical). That is a total of >154.000 pixels per image. To run the model on images of this size would
be inconvenient, as the number of parameters would be very large. Thus training the model would
require hardware with more memory than was available in the making of this work. Therefore, the
choice was made to crop the images to 128 × 128 pixels. The crop is performed by keeping the center
patch of 128 × 128 pixels in the image.
Firstly, pixel value range is changed from [0,255] (8-bit) to the unit interval [0,1] simply by dividing
the pixel values by 255, the maximum possible pixel value in 8-bit images. Next, the image is converted
from RGB (3 channels) to grayscale (1 channel) so that the image consists of a single channel. The con-
version to grayscale is done by converting the image from the RGB colorspace to the YCbCr colorspace
and keeping only the Y-channel which stores the luminance information of the image. The values are
converted using the RGB-to-YCbCr conversion formula used in OpenCV [23d]:
Y = 0.229 · R + 0.587 · G + 0.114 · B.

CHAPTER 5
Discussion
[Tia+19] presented denoising benchmark results for several CNN denoisers on the BSD68 test dataset
which is a subset of the BSD500 dataset which was used to train and evaluate the models presented in
the previous chapter. The denoisers were evaluated at noise scales σ = 15, 25, 50 just as the models in
this work. For the three noise scales 15, 25, and 50, the models in [Tia+19] that achieved the highest
average PSNR on the BSD68 testset achieved the PSNR values 31.86, 29.68, and 26.76, respectively.
The average PSNR value obtained on the BSD500 testset by each of the denoiser models in the previous
chapter are more than 1 PSNR away from these benchmark results. However, the (Hartley) frequency-
domain denoiser models did, for each noise of the three scales, obtain a maximum PSNR value on the
BSD500 testset that is within 1 PSNR of the benchmarks. The models in this work were tested on
a larger set of images than the models in [Tia+19]. This may contribute to the lower average PSNR
values for the models developed in this work.
Comparing the average PSNR and SSIM results of the (Hartley) frequency-domain denoisers and
the spatial-domain denoisers, the frequency-domain models mostly performed slightly better than the
spatial-domain models. However, on the whole the performance is most similar, and it seems like the
models, during training, all converge to the same state. Therefore it seems to suggest that it does
not make much difference whether the denoising takes place in the frequency-domain or in the spatial-
domain. From the showcasing of the visual denoising results it is unfortunately clear that the models
do not achieve a perfect, or acceptable, level of denoising. In fact, generally, the denoised image seems
more blurred than the noisy image, suggesting that some of the high-frequency information in the image
is lost over the course of the denoising. This highlights that it is tricky to reduce (high-frequent) noise
without loosing information in the image that is also high-frequent. It is necessary to distinguish noise
from information in some way. [Xue+20] used a custom loss function which assigned different weight to
high-frequency and low-frequency parts of the frequency-domain data during training. Implementing a
similar loss function could perhaps improve the loss of hihg-frequency information.
The quality of the denoising results indicate that the models were constricted in some way from achiev-
ing better denoising. Admittedly, the training and validation datasets used are quite small and this
may explain why the models do not learn a better mapping from noisy to noise free images. Using
a larger sets of training and validation datasets therefore should result in better performance for the
models. Also, there may be ways to improve the model architecture by using more sophisticated deep
learning techniques.
The results of the last experiment in the previous chapter showed that denoising in the frequency-
domain, when the data has been transformed using the DCT instead of the Hartley transform, can
result in better PSNR. Therefore, it could be interesting to further examine if the DCT is a preferable
transform to use in frequency-domain image restoration.
==PAGE:42==
CHAPTER 6
Conclusion
This work aimed at developing convolutional neural network models for the image restoration task
of denoising. The denoising was supposed to take place in the frequency domain. This required the
establishment of frequency transforms for the purpose of transforming images from the spatial domain
to the frequency domain. Specifically, the Hartley transform and the DCT transform were chosen as
suitable candidates due to their property of being real-to-real transforms.
A most commonly used
frequency transform is the Fourier transform, and this could actually be utilized in the computation
of the Hartley transform.
The noise considered for restoration was additive Gaussian noise which
was added in to images the frequency domain when doing frequency domain denoising. The developed
denoiser model had to take into account how the convolution operation behaves in the frequency domain.
The denoiser model was used for denoising experiments in both the frequency and spatial domains. The
models were trained and tested on the BSD500 image dataset. The denoising results were measured
using both the PSNR and SSIM image quality metrics which are widely used in the literature. In the
experiments, the model was not able to get within 1 PSNR of other model benchmark results. These
results could be improved by training the models with more data or improve the model architecture.

17
Input:

19
appr: numpy-array (H x W), reference image that `appr` appproximates.

Output:
21

22

47 def ssim(appr, ref, consts=[1e-5,1e-5,1e-5], weights=[1.,1.,1.]):
48
'''
49
Compute the structure similarity index (SSIM) of a degraded (i.e. noisy) aproximation of an
image.
50
Input:
51
appr:
numpy-array (H x W), image approximation to compute SSIM for.
52
ref:
numpy-array (H x W), reference image that `appr` appproximates.
53
consts:
(optional) numpy-array (1 x 3), positive constants for numerical stability (
default: [1e-5,1e-5,1e-5]).
54
weights: (optional) numpy-array (1 x 3), weighting parameters (alpha, beta, gamma) (
default: [1.,1.,1.]).
55
Output:
56
ssimVal: float, the SSIM-value of the image approximation.
57
'''
58
assert (len(appr.shape) == 2) & (len(ref.shape) == 2)
59
assert (appr.shape[0] == ref.shape[0]) & (appr.shape[1] == ref.shape[1])
60
assert len(consts) == 3
61
assert len(weights) == 3
62
mu_a, sigma_a = appr.mean(), appr.std()
63
mu_r, sigma_r = ref.mean(), ref.std()
64
cov_ra = (appr.shape[0]*appr.shape[1])**(-1) * ((ref - mu_r)*(appr - mu_a)).sum()
65
l = (2 * mu_r * mu_a + consts[0]) / (mu_a**2 + mu_r**2 + consts[0])
66
c = (2 * sigma_r * sigma_a + consts[1]) / (sigma_r**2 + sigma_a**2 + consts[1])
67
s = (cov_ra + consts[2]) / (sigma_r * sigma_a + consts[2])
68
ssimVal = l**weights[0] * c**weights[1] * s**weights[2]
69
return ssimVal
