1
Introduction
Path finding is an important part of maritime navigation. In order to reduce the fuel
consumption of the ships, we want to find the shortest path. At the same time, the
journey should be safe, so the paths are better to avoid bad weather.
In this paper, we model the problem of the shortest path on the sea with bad weather,
apply Dijkstra’s algorithm and A* algorithm in a time-dependent graph, and propose a
hierarchical grid structure to speed up the A* search.
The remaining paper is organized as follows. Section 2 introduces the problem def-
inition. Section 3 lists related work. Section 4 for preliminary algorithms and Section
5 models the graph and obstacles. Section 6 describes the initial algorithm based on
Dijkstra’s. Section 7 introduces an improvement using A*. Section 8 introduces pre-
processing of the obstacles. Section 9 introduces a hierarchical grid graph for further
speeding up the query. Section 10 is about other speed-up ideas using the existing al-
gorithms. Section 11 introduces the graph instances and experiment results. Lastly,
Section 12 and 13 for discussion and conclusion.
2
Problem Definition
Time-dependent graph Let G = (V, E, W) be a directed graph where V = {vi} is a
set of vertices; E ⊆V × V is a set of edges; for each e ∈E, there is a function we ∈W
that defines the cost of e with respect to time. wuv(t) denotes how much time it takes
to travel from u to v when departing u at timestep t.
FIFO graph A time-dependent graph G = (V, E, W) is a FIFO graph if every edge
has FIFO property. An edge (vi, vj) has FIFO property iff for timestep t1 ≤t2, we have
t1 + wij(t1) ≤t2 + wij(t2). This implies that if departing from vi earlier, one arrives at
vj earlier. The time-dependent shortest path is polynomial solvable if the graph satisfies
FIFO property and otherwise in [1] it proves that it is a NP-hard problem.
In this paper, we focus on finding the minimum total travel time from source node s
to target node t given a fixed starting time t0 on a time-dependent FIFO graph. Assume
that time is discrete and all cost functions are known in advance over all time T.
3
Related work
There are several relevant research fields as to time-dependent shortest path on the sea.
3.1
Shortest path on road networks
3.1.1
Static graph
Shortest path on road networks have been intensively studied.
For static shortest
path, the base algorithms are Dijkstra’s[2], Bellman Ford[3] [4], Floyd-Warshall[5],
1
==PAGE:6==
A* algorithm[6] and bi-directional search[7]. Dijkstra is for graph with non-nagative
weights, while Bellman Ford is for detecting negative cycles and can do shortest path
on graph with negative weights. Floyd-Warshall is for all-pair-shortest-path (APSP)
that runs in O(n3). A* algorithm adds heuristic which estimates how much distance
remaining to guide the direction of search. Bi-directional search is based on A*, and it
searches from s and t at the same time until meeting in the middle.
A* based speed up ALT[8] algorithm selects landmark points and precompute
distances between all nodes and the landmarks.
Then it uses triangle inequality to
estimate distance. Precomputed Cluster Distances [9] divides the graph into several
clusters, precomputes shortest path between clusters and uses them as heuristic.
Search space reduction These speed-up ideas precompute information about
shortest path to prune edges while searching.
Geometric Containers [10] prunes an
edge e if it’s not in the container (can be a bounding box, circle, ecplise or other shapes)
of target node. A container contains all nodes that can be reached on a shortest path
from a node. Arc flag [11] has the similar idea as Geometric Containers, but it first
partitions the graph into K cells and each edge stores K bit flag of whether it is on
the shortest path to each of the cells. Both Geometric containers and Arc flag needs to
precompute all-pair-shortest-path (APSP) leading to long preprocesssing time. Reach
[12] precomputes the furthest reach distance for a node on the shortest path and prunes
a node if the reach is less than the current distance to the start or target node.
Hierarchical speed up Highway hierarchy [13] is one of the hierarchical solutions.
Later, contraction hierarchies (CH)[14] is a renowned algorithm that adds shortcuts to
bypass unimportant nodes on road network. In [15] it shows that CH can be applied in
grid graph as well. With a proper contraction order it speeds up the query 50 times in
average on different maps.
3.1.2
Time-dependent graph
Time-dependent shortest path algorithm is first studied in [16], in which a discrete time
algorithm based on Bellman Ford algorithm is proposed. In [17], it discusses the difficulty
of applying CH, SHARC and ALT algorithm to time-dependent graph, mainly because
the preprocessing steps become more complicated.
[18] introduces how to augment
ALT, arc flag and contraction hierarchies to time-dependent graph. Core-ALT[19] first
contracts the graph into a smaller one called Core, computes and stores shortest path
distances from all core vertices to each landmarks. The query is done by first running
time-dependent bidirectional search, then do Dijkstra search on the combined graph as
well as time-dependent ALT on the core graph. In time-dependent scenarios, an arc flag
is set to true if it is on the shortest path to at least one of the nodes in region i for
at least one timestep t ∈T. In [20], they propose a Dijkstra, multi-level arc-flag and
contraction based algorithm called SHARC and in [21] it is further enhanced to support
time-dependent query. For time-dependent CH [22], they assume the priority of the

nodes doesn’t change much by average so the overhead for supporting time-dependent
graph is to re-evaluate the edges for the time period when weight changes. In [23], a TD-
D-tree is designed for time-dependent shortest path query over a hierarchical partitioned
graph. [24] uses A* and boundary-node as heuristic to solve time-dependent shortest
path query with the road network that has speed patterns.
Apart from fixed start time time-dependent shortest path, there’s a variant ’best
start time query’, that tries to find the best start time in a given interval of time that
gives shortest path/minimum travel time. In some papers, it is called profile query.
For example, [25], [26] and [23] support this kind of query. [27] gives a comprehensive
review for both static networks as well as time-dependent graph as there are some more
variants. [28], [29] allow waiting to achieve for example minimum driving time. [30], [31],
[32] are about path with minimum fuel consumption, in which dynamic programming,
random algorithms and approximate algorithms are applied.
3.2
Shortest path in the air and on the sea
[33] shows contraction hierarchies doesn’t speed up much in the air, so they propose a
new heuristic, super-optimal wind. [34] introduce a static shortest path algorithm with
visibility graph on the sea.
3.3
Path finding in A.I. field
A* algorithm originates from the field of artificial intelligence. Lifelong A* [35] gives
an updated shortest path when graph updates, while keeping some explored path un-
touched.
Jump point search[36] is designed for uniform-cost grid graph in which it
selects the node to be expanded until meets an corner instead of every node during the
search. [37] and [38] deal with the scenario of avoiding a moving obstacle. Any angle
shortest path [39] on the grid is also an interesting direction as it makes it possible for
shortest path on the sea to follow the great circle.

Preliminaries
4.1
Parameters
First we introduce the notations used in this paper. G is the input graph. V is the set
of vertices and E is the set of edges. n is the number of vertices and m is the number of
edges. s and t are the source node and target node in a shortest path query. k is number
of obstacles. T is the set of total timesteps during the travel. Timestep is a integer t ∈T
without unit. In practise it could be 5 minutes, 1 hour, etc. d(u, v) denotes the original
weight of edge(u, v) without being affect by any obstacles. wuv(t) is the weight of edge
(u, v) at timestep t. auv(t) is the arrival time function which indicates the arriving time
if departing from u at timestep t.

4.2
Priority queue
A priority queue is a data structure that supports the following functions.
• insert(item, value) - insert an item with value as its priority
• pop() - return the item with minimum priority and remove it from the queue
• update(item, value) - update the value of the item
4.3
Dijkstra’s shortest path algorithm
Dijkstra’s algorithm is a classic shortest path algorithm for a directed graph with non-
negative edge weights. It is a Single-Source Shortest Path (SSSP) algorithm that finds
shortest path from one node to all other nodes.
Let G = (V, E) be a directed graph, s be the source node and t be the target node.
Dijkstra’s algorithm initializes s with s.d = 0, all other nodes v ∈V \{s} with v.d = ∞,
and insert s to a priority queue where the node with shortest distance to s has top
priority. Then it grows a shortest path tree T from s by iteratively popping a node u
with smallest distance and relaxing all outgoing edges of u. Let (u, v) be an outgoing
edge. By relaxing, it means to compare the distance between v.d and u.d + w(u, v). If
u.d + w(u, v) < v.d, it implies that a shorter path from s to v is found by taking the
edge(u, v). In this case, node v is either inserted to the queue or its priority in the queue
is updated to u.d + w(u, v).

4.4
A* algorithm
A* algorithm comes from the field of artificial intelligence and it’s a path finding al-
gorithm designed for artificial agents for example a robot.
It is an informed search
algorithm which gives priority to the cost from source to current node plus an estima-
tion of the cost between current node to the goal, so that the search is guided towards
the goal. To upgrade Dijkstra’s algorithm to A*, change Line 9 in Algorithm 1 to ’pop
a node from Q with minimum f, where f = u.distance + h’. h is called the heuristic
function that estimates the remaining cost. The estimation should not exceed the actual
cost for A* algorithm to guarantee optimally. If the heuristic never overestimates the
actual cost for reaching the goal, it’s called admissible.
5
Modelling
5.1
Modelling the graph
Regarding the surface of the sea as a plane, it is allowed to sail in any direction until
it meets an island or land. This would introduce numerous edges from one node to
any location on the sea. To make it simple, the graph instance is modeled as is an
8-connected grid graph with nodes placed every 1 or 2 degrees latitude/longitude or
at a uniform distance. Each node is embedded with geographic information, that is
its coordinates on the sphere in longitude and latitude. An 8-connected graph allows
moving in 8 directions and is sometimes called ’King’s graph’ because it represents all
allowed actions of the king on a chessboard.
5.2
Modelling the bad weather/obstacle
Wind, wave, current or storm are several types of weather that could interfere with the
speed of the ship. Non-linear mathematical formulas with several parameters have been
used to model the effect of different weather conditions as shown in [40].
We model the bad weather as a moving obstacles and show how to do a modification
to the arrival time function for maintaining the FIFO property of the graph.
For simplicity, the weather is modelled as a moving obstacle that appears during a
certain time period. An obstacle is a circle that has radius, start location, end location,
start time, end time and coefficient. The coefficient indicates how much the obstacle
affects the weight of the affected edge. The time-dependent weight of the affect edge can
be computed by (coefficient of the obstacle * d(u, v)), making it a linear change with
respect to the original weight. Coefficient less than 1 is regarded as a ’faster’ obstacle
for example when following an ocean current. Coefficient larger than 1 is regarded a
’slower’ obstacle which corresponds to a slightly bad weather and ∞means the obstacle
is inaccessible and there’s extreme weather. Noted that in this paper we mainly focus
on the ’slower’ and inaccessible obstacles are they best represent bad weather.

affected by the obstacle does not satisfy FIFO property. For example, an inaccessible
obstacle affects edge(u, v) from timestep t1 to t2.
Let t3 be a timestep after t2.
If
departing from u at t1, the arrival time is t1 + d(u, u) ∗∞which is ∞while departing

solve the issue, a mandatory waiting time has to be imposed on the edges affects by an

Dijkstra based time-dependent shortest path
We first introduce a time-dependent Dijkstra’s algorithm in FIFO graph. The algorithm
explores and updates the vertices following Dijkstra’s algorithm, while the weight of
each edge is decided on the fly by the current timestep, location of obstacles, and their
coefficients.
6.1
Algorithm description
As shown in Algorithm 3, the input instance is a FIFO graph G(V, E, W) where the
time-dependent weight of each edge satisfies FIFO property with k moving obstacles,
each has start location, end location, start time, end time, radius and coefficient. Noted
that if the graph is non-FIFO, it is required to apply Algorithm 2 first to modify it
to FIFO. The query is to find a shortest (minimun total travel time) given a source s,
target t and start timestep t0.
The algorithm starts by initializing n states for each of the vertices.
A state is
identified by the vertex and it has three more parameters, timestep, g and parent. g is
the total weight from source node to current node, in this case the travel time, equivalent
to the distance in static Dijkstra’s algorithm as in Algorithm 1. Timestep is representing
the current time, intended for getting the weight of an edge from its time-dependent
weight function. Parent is also the same as in static Dijkstra’s as it indicates what is
the previous node on the shortest path.
At first all vertices are initialized with g = ∞and the source node has g = 0. The in
the exploring phase, it gives priority to the node with minimum travel time and relax all
outgoing edges. The neighbouring node is the other endpoint of the outgoing edges and
it is in the form of a neighbouring state that contains information of current timestep
so that the timestep can be used to get time-dependent weight. Time-dependent weight
is calculated by checking the locations of all obstacles and whether the edge is affected
by the obstacle. If the distance between an edge and the center of the obstacle is less
than its radius, the obstacle is affecting the edge. If several obstacles are affecting the
edge at the same time, take the maximum coefficient among them. After getting the
time-dependent weight, relax the two states similar to static Dijkstra. If found a shorter
path to the neighbouring node through the current node, the algorithm updates the
neighbouring state. Do this repeatedly until reach the target.

Correctness
The correctness of A* algorithm depends on two conditions. First the heuristic should
be to be admissible, which is h(u) <= d ∗(u, t). The second is monotonic, that is, if
h(u) < h(v), then d∗(u, t) < d∗(v, t).
The correctness can be proved by contradiction. Suppose at some point, A* picked
an edge off the correct path. Let edge(x, y) be the first edge that leaves the correct path,

edge(x, z) be the correct edge, and the algorithm arrives at x at timestep tx. We know
that time dependent weight(x, y, tx) > time dependent weight(x, z, tx).
For y to be chosen by the algorithm, y has the minimum f value among x’s neigh-
bours. So we have g(y)+h(y) < g(z)+h(z), where g(y) = d∗(x)+time dependent weight(x, y, tx)
and g(z) = d∗(x)+time dependent weight(x, z, tx). Because z is on the correct shortest
path, we know d∗(z, t) < d ∗(y, t). By monotonic, we have h(z) < h(y). Combining the
two findings, it leads to time dependent weight(x, y, tx) < time dependent weight(x, z, tx)
which is a contradiction with the assumption.
7.2.4
Heuristic function
The performance of A* depends on a good choice of the heuristic function. The heuristic
should be as tight as possible to reduce the number of explored nodes.
Great Circle Distance One possible solution is to use the Euclidean distance as
heuristic. But since the graph is actually on a sphere, a more suitable choice would
be the Great Circle Distance for the shortest distance on a sphere. Given two points
on the surface of a sphere, create a circle whose center coincides with the center of the
sphere, then the circle is called the Great Circle. The curve length between the two
nodes with respect to the great circle is called Great Circle Distance, or geodesics. Since
the Great Circle Distance is already the shortest distance between two nodes, it cannot
overestimate the actual distance, making it admissible.
Octile distance Octile distance for uniform cost 8-connected grid. The octile dis-
tance regards weight of a cardinal edge 1 and diagonal edge 1.414 and it is calculated
by 1.414 × min(∆y, ∆x) + 1 × |(∆y −∆x)|, where ∆x and ∆y is the difference of x and
y coordinates of the two nodes. Because taking diagonal moves is always better than a
combination of horizontal and vertical moves, the heuristic guarantees to take as many
diagonal moves as possible.

In this section we propose a new A* based algorithm with hierarchical preprocessing
over the grid graph.
9.1
Algorithm description
Do a preprocessing of the grid graph by adding L layers of combined edges. Precompute
time-dependent weight/arrival functions for all edges in all levels. When querying, do
A*. If the f value of two nodes equals, take the one with smaller h(n) to break symmetry.
The preprocessing phase contains two main tasks, creating a level by adding edges
and edge composition. Given a 8-connected grid graph G = (V, E, W), add L levels of
edge-combined graph on top of the graph with non-overlapping combined edges at every

edges in the same direction in Level i. For example an edge at Level 1 is composed of
two edges from Level 0, which is the original graph. For each of the combined edges in
different levels, do a bottom-to-top composition of the time-dependent weight/arrival
time function starting from Level 0. Assume the time-dependent edge weight for Level
0 is already known.

consecutive steps. If such two edges exist, do edge composition and add the combined
edge to the graph. Scan the whole graph by moving the current node horizontally or
vertically 2L degrees longitude/latitude.

The correctness includes two parts.

graph. The edges are combined from bottom-to-top, and the L0 edges satisfies FIFO.

it is proved in Section 7.2.3.

Other speed up ideas
In this section, we dive deep into other speed up techniques for Dijkstra’s and A* based
shortest path algorithm and investigate how they can be applied to TDSP on the sea.
10.1
Landmark
ATL algorithm is a combination of A*, landmark and triangle inequality[8]. The main
idea is to use landmarks to estimate the distance between two vertices. The algorithm
first selects a small constant number of landmarks on the graph, compute and store
shortest path distances from all vertices to each of the landmarks. During the query
along with A*, the heuristic function h, which gives an estimation of the remaining
distance from the current vertex to the destination is calculated based on triangle in-
equality between the two vertices and a landmark. Given any pair of node (u, v), the
distance d(u, v) is then calculated by maxl∈L|d(l, v) −d(l, u)|.
It is natural to think of dynamic landmarks, e.g. landmark location/distance changes
over time.
But the main difficulty is that the triangle inequality does not hold in
time-dependent scenario. Given two nodes, u and v, let T(u,v,t) denote the minimum
time for travelling from u to v while departing u at timestep t, then it holds that
T(u, L, t) <= T(u, v, t) + T(v, L, t + T(u, v, t)). According to FIFO property, we also
know that T(v, L, t + T(u, v, t)) > T(v, L, t) because t + T(u, v, t) > t. But we cannot
infer the triangle inequality of T(u, L, t) < T(u, v, t) + T(v, L, t).
The A* based time-dependent algorithm Algorithm 4 can be modified to landmark
based A* algorithm by replacing line 11 to ”pop a state from Q with minimum f(n) =
g(n) + h(n)” where h(n) is calculated by maxl∈L|d(l, n) −d(l, t)| (t is target node).
10.1.1
Preprocessing
The preprocessing works as below. It takes O(m log n) for running a static Dijkstra’s
algorithm from one landmark to all nodes. So total preprocessing time is O(l · m log n)
for l landmarks.

random, farthest, planar and etc.
• For each landmark, run Dijkstra’s algorithm and store shortest path from the
landmark to all nodes.
10.1.2

The space for the algorithm itself takes O(n + m + k). Extra space is O(l · m) where
l is number of landmarks for storing distances between each of the landmarks and all

10.1.3
Admissible
Let u, v be an arbitrary pair of nodes and l ∈L a landmark. Let d∗(u, v) be the distance
of the shortest path between u and v. According to triangle inequality, the distance
between them can be written as d∗(u, v)+d(u, l) > d(v, l) and d∗(u, v)+d(v, l) > d(u, l).
From them we get d∗(u, v) > |d(u, l) −d(v, l)|. Do this for all landmarks and it holds
that d∗(u, v) > maxl∈L|d(l, v) −d(l, u)|. This shows the estimation is smaller than the
actual shortest distance, thus proves admissible.
10.1.4
Why useful
The landmarks can still be utilized because the main advantage of the ALT algorithm
is that with the pre-computed distances between nodes and a landmark, it can detect
static obstacles, in this case the continents, and guide to pass by.
10.2
Arc Flag
Arc flag, from its name, is about adding a flag to each arc (edge). A naive idea would
be to first pre-compute all pairs shortest path(APSP). Then for each edge, mark what
node can be reached through this edge following the shortest path. When querying, it is
easy to prune nodes not on the shortest path to the destination node using these flags.
The extra space for storing arc flags is O(n2) because there are n2 pairs. In [11], they
propose an algorithm based on graph partition. The graph is partitioned into k regions.
For each edge e ∈E, store a k bit vector, where the ith bit indicates whether e is on the
shortest path to at least one of the nodes in region i. In this case, the extra space can
be reduced to O(k · m).

12
Discussion
There are many speed-up algorithms for shortest path on the road networks, some
of which show orders of magnitude of improvement in running time. Time-dependent
shortest path on the sea is different from road networks in a way that the time-dependent
weight comes from unexpected bad weather instead of a rather fixed pattern of traffic
congestion that takes place at a certain time certain area. Also the sea surface is open so
the routes are much more flexible if we allow the edges to go to any angle. Given these
differences, it’s worth experimenting the algorithms on the road networks and evaluate
their performance on the sea with bad weather.
A* proves to be a good choice. But the limitation of A* is that it doesn’t support
’faster’ obstacles well. If there are ’faster’ obstacles, the heuristic has to be multiplied
26
==PAGE:31==
by the minimum coefficient among all obstacles, making the bound not tight.
Also
the choice of great circle distance as heuristic can be improved since the graph with
limited direction of edges doesn’t allow the path to follow the great circle. It would be
interesting to see how the octile distance performs if the graph is a uniform-cost grid.
Figure 8 shows the search space of A* with great circle distance and octile distance
on graph instance 2. It is also possible to try combinations of the speed-up ideas, for

example to combine hierarchical grid with landmark or arc flag.
One difficulty is that the implementation is based on python as there are more
geographical libraries, for example NetworkX[44], that help to read a geometrically
embedded map and convert to a graph. But the heap queue in python is not flexible
enough to customize the priority and comparison between objects.
13
Conclusion
In this paper, time-dependent shortest path on the sea have been studied and algorithms
based on Dijkstra’s and A* search and a speed-up algorithm based on hierarchical grid
graph preprocessing have been proposed. Our algorithm can take any time-dependent
weight and output the edge arrival time function that satisfies FIFO property. The
experiment results show A* works well with both of the graph instances. With extra
preprocessing time O(n+k·m) and space O(k·m), it is possible for the hierarchical grid
graph preprocessing to improve time-dependent A* by a factor of O(1/ log (len(path))).
If given more time, it’s worth implementing the the hierarchical grid and test the
performance. It’s also possible to add more combined edges, for example, instead of
adding combined edge for nodes every 2L degrees, do for all nodes.
