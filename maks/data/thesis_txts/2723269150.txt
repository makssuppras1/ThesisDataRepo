Abstract
Electric micro mobility is increasing its popularity in the last few years. One of the biggest
problems that companies in this ﬁeld have to face is the Inventory Problem, the daily number
of bicycles required to optimally run the service.
This optimization problem may save
operational costs for the companies and yield them more users. The goal of this Master
Thesis is to use Machine Learning techniques to estimate an optimal Inventory Level. In
order to do that, an open data source regarding Boston bikes has been used. After a ﬁrst
inspection of the literature in the ﬁeld, it has been found out that the main issue with
the Inventory Problem is that data sets become biased towards low demand values. This
makes the dataset imbalanced and complicates the usage of Machine Learning techniques as
Random Forest or Gradient Boosting. Veriﬁed this issue also in this case, a downsampling
algorithm called Balance Cascade has been used to overcome this problem. At the end,
considerations on the parameters of this algorithm have been done to address the Inventory
Problem and reach a better result. Furthermore, a comparison using an existing algorithm
shows if this Balance Cascade is a valide technique to solve this problem.

Introduction
Academic rephrasing: Over the past few years, bike sharing services (BSS) have experienced
remarkable growth due to their eco-friendly and cost-eﬀective nature. These services are
available from both companies and municipalities, and allow users to pick up or return bikes
at docking stations (in the case of station-based services).
It is evident that there are numerous aspects of BSS that can be ﬁne-tuned for optimal perfor-
mance. This study focuses on three key areas: Demand Forecasting, Inventory Problem, and
Rebalancing. These areas are interconnected, as the demand for bikes inﬂuences the number
required for the following day, which in turn aﬀects how operators redistribute bikes across
diﬀerent stations. The Inventory Problem component, which determines the optimal number
of bicycles required to operate the service, is the focal point of this research. The solution to
this issue will lead to an improved user experience and reduced costs for the service provider.
The objective of this thesis is to use Machine Learning to estimate the inventory level in
station-based bike sharing for static (overnight) rebalancing. However, the main challenge in
this task is the human behavior that creates an imbalanced system. Therefore, the Inventory
Problem will be treated as a classiﬁcation problem. However, traditional Machine Learning
techniques assume a balanced dataset, which is not the case for bike-sharing datasets, as
reported in [1]. After preliminary analysis, it was conﬁrmed that the dataset is imbalanced,
and popular techniques like Random Forest or Gradient Boosting do not provide satisfactory
results. Hence, this study conducted a literature review to explore new algorithms to address
this issue, and the selected algorithm is Balance Cascade, which uses AdaBoost classiﬁcation
on sub-problems. With the help of clustering, this method is iterated on all the diﬀerent
imbalanced sub-problems, following the iterative approach suggested in [1]. The last step
of the thesis has been the comparison between this algorithm and the one showed in [1],
applying the same dataset used here.
The data used for this study was obtained from Boston bikes service [2].
This thesis is structured as follows: Chapter 2 provides a comprehensive review of key
literature on the Inventory Problem, Imbalance Classiﬁcation, and undersampling and over-
sampling techniques. Chapter 3 presents an exploratory data analysis and the data cleaning
and preparation procedures. Verifying data imbalance was essential before implementing
the methodology. Chapter 4 explains the methodology in detail, with a focus on Balance
Cascade. Chapter 5 validates the study’s results and explores some modiﬁcations. Chapters
7 and 8 present the discussion and conclusion, respectively. The Appendix contains the
primary code implementation.

2
Literature Review
This section provides a summary of the academic sources reviewed as a basis for conduct-
ing the analysis. The literature can be categorized into three main sections: Inventory
Problem, Imbalanced Dataset, and Undersampling & Oversampling Techniques.
2.1
Inventory Problem
As previously mentioned, the Bike-sharing system faces several challenges, such as Demand
Forecasting, Inventory problem, and Rebalancing problem. According to Cantelmo et al.[3],
the Inventory and Rebalancing problems are closely connected, making it essential to address
them together. Therefore, this research focuses speciﬁcally on the Inventory problem.
Bike Sharing System operators need to tackle non-homogeneous asymmetric demand pro-
cesses, which result in an ineﬃcient level of bikes. To ensure the smooth functioning of the
system, the bikes need to be rebalanced, preventing stations from running out of bikes or
available docks for incoming bikes. As explained in Datner et al.[4], the non-homogeneous
asymmetric demand processes vary from station to station, making it challenging for Bike-
sharing companies to estimate the demand accurately.
Rebalancing can be done either through Static or Dynamic methods. In Static rebalancing,
bikes are moved overnight based on inventory levels, whereas Dynamic rebalancing involves
real-time tracking of demand and movement of bikes. However, the Dynamic approach can
be more expensive and time-consuming, making the Static approach more cost-eﬀective and
practical. Therefore, this research will focus on the Static approach.Solving the Inventory
Problem is a crucial aspect of the Static approach. According to Ceccarelli et al.[1], the
Inventory Problem involves estimating the number of bikes and available docks required to
meet the demand at each station. The accurate estimation of inventory levels is essential for
the successful implementation of Static rebalancing.
In summary, the Inventory and Rebalancing problems are interconnected and need to be
addressed together to ensure the smooth functioning of Bike-sharing systems. The Static
approach is more practical and cost-eﬀective, and solving the Inventory Problem is a crucial
aspect of the Static approach.
2.2
Imbalanced Dataset
The issue of imbalanced datasets is a main problem in various real-world applications, includ-
ing the analysis of BSS data, as highlighted by Ceccarelli et al [1]. In order to address this
challenge, a portion of the literature review was focused on understanding the implications
of imbalanced data. Ganganwar’s deﬁnition of an imbalanced dataset, as outlined in [5],
explains that it occurs when a class contains signiﬁcantly more samples than the remaining
classes. This condition is typical when at least one class is under-represented, referred to as
the minority class, and other classes dominate the majority of the dataset.
3
==PAGE:11==
2.3
Undersampling & Oversampling
The imbalanced nature of a dataset is a signiﬁcant challenge for standard learning algorithms
that typically rely on balanced data distributions.
As stated by He et al.
in [6], these
algorithms are unable to adequately understand the diﬀerences between the various classes,
resulting in misclassiﬁcation errors during prediction and classiﬁcation processes. Therefore,
it is crucial to employ speciﬁc techniques that can eﬀectively address imbalanced data in the
context of BSS analysis, such as oversampling, undersampling, and synthetic data generation,
among others [6].
2.3
Undersampling & Oversampling
In the last part of the literature review the aim was to understand how to overcome this data
imbalance, reaching decent results when it comes to classify. Therefore, the two methods
found are called undersampling and oversampling techniques: these allow to create balanced
subproblems. The diﬀerences highlighted in [5] and [7] can be summarised as follows:
• Undersampling: decreasing the size of the majority class (usually these observations
are thrown away) to balance the distribution between majority and minority class. The
main drawback of this method is the risk of discarding potential useful data (loss of
information). To avoid this loss of information, many diﬀerent methods are proposed,
like random majority under-sampling(RUS) [8].
• Oversampling: increasing the size of the minority class to balance the distribution
between majority and minority class. The main drawback of this method is the risk of
overﬁtting. The model may become too specialized in recognizing the minority class,
which may result in poor generalization and performance on new, unseen data. As a
result, it is essential to be cautious and to balance oversampling with other techniques
such as cross-validation, regularization, and hyperparameter tuning to reduce the risk
of overﬁtting. One of the most used approach is the SMOTE (Synthetic Minority Over-
sampling Technique), proposed in [9], in which synthetic samples for the minority class
using the k nearest neighbors (randomly chosen) of the same class. A new version
called Borderline-SMOTE is proposed in [10], which only replicates the cases in the
minority class close to the borderline and it is proved to provide better results.
In other articles, like [11], researchers tried to realized hybrid method to exploit the positive
sides of both approach.
The focus of this thesis will be on exploring the method proposed by Liu et al. in their re-
search paper titled "Exploratory Undersampling for Class-Imbalance Learning"[12]. Speciﬁ-
cally, we will examine a technique called Balance Cascade that was introduced in the paper.
This algorithm works by undersampling the dataset and then iterating on the remaining ob-
servations, without discarding the ones that were removed. This allows the algorithm to
retain information from the entire dataset, even though only a portion of it is used in each
iteration. The Balance Cascade approach has been praised for its ability to address class-
imbalance learning challenges.

3.2
Preprocessing Steps
The initial step in the study involved the handling of a dataset containing over 5 million
observations, which presented challenges of slow iteration and excessive information. To
address this, it was decided to only retain trips taken between 6-10am, as they better reﬂect
the aim of the study, which is to determine the inventory level in the morning. After initial
inspection, it was observed that the peak rental activity occurred during the night (2-3am).
This indicated that the dataset was set according to the Boston time zone, and as such, all
datatime features were adjusted by shifting them 6 hours forward.
3.2.1
Data Imbalance Preprocessing & Analysis
First, it is important to describe how to discuss whether the data is imbalanced or not. For
this purpose, the method used in [1] has been replicated. As said in the article, "In this
paper we assume that the solution of the inventory problem consists of a set of possible
conﬁgurations, moving between a lower and an upper bound".
The process is better comprehended by referring to the diagram in Figure 1, which delineates
the procedure utilized.

Figure 1: Explanation of the Cumulative Net Curve calculation
Having the Cumulative Net Curve as the diﬀerence between Arrivals and Departures, it is
possible to calculate the lower and upper bounds both for bikes and docks. Ceccarelli et al.
in [1] describe the dock and bike lower bound as "the maximum value of the cumulative net
curve represents the minimum number of available docks required in the station to satisfy
the arrivals demand (Dock’s lower bound); instead, the minimum value of the cumulative
net curve represents the minimum number of bikes required in the station to satisfy the
departures demand (Bike’s lower bound)".
Summarising what are these conﬁgurations:
• Bike’s Lower Bound: the minimum number of bikes required in the station to satisfy
the departing users’ request.
• Dock’s Lower Bound: the minimum number of available docks required in the

perform eﬀectively with the former, but are likely to yield satisfactory outcomes with the

3.2.2
Data Preparation for the Classiﬁcation task
After verifying the data’s imbalance, the objective of this section is to generate a dataset
for classiﬁcation purposes. To achieve this, the initial step involves aggregating the daily
records by station and pivoting the data to ﬁlter only the arrivals and departures occurring
between 6-10am. At this point, there will be a ﬁrst dataset with the arrivals and departures
for every day, for each station.
After this ﬁrst step, for every day the average trip duration is calculated.
In addition, it is feasible to calculate the mean number of arrivals/departures at the station
during the preceding two months. For this reason, the ﬁrst two months have been discarded.
Also, the departures/arrivals at the station in the previous day is added.
At this point, the target variable LB_bike and UB_bike is added and from this it is possible
to calculate the average LB in the previous two months, a feature which will be extremely
helpful when it will come to classify.
As other information, capacity of the station and number of nearby stations (with a search
radius of 500mt) are added to the dataset.
The last two information added were the month and the weekday of the trip, which have

Methodology
This study aims to evaluate how machine learning behaves in case of imbalanced data and
provides a new method to solve this issue. To repeat, the work considers a BSS station-based
and the problem it addresses is the Inventory Problem, which is the estimation of needed
bikes in a station, considering the free docks for the bikes dropped oﬀ. ML techniques will,
therefore, give a solution which is represented by these two values: Lower Bound and Upper
Bound. As noted in [1], despite the continuous nature of the values, these can be transformed
into a classiﬁcation problem rather than a regression problem. To implement this approach,
the authors of the cited study divided the various observations into clusters, explaining that
the goal was not to predict a speciﬁc number of bikes, but rather to determine whether a
station requires i.e. 0 bikes or 30 bikes. This study adopts the same approach.
All the ML methods used for the classiﬁcation will be explained in this section.
Additionally, the dataset generated in the preceding section was divided into training and
testing (70%/30% respectively) subsets without shuﬄing to preserve the chronological or-
der of the data. After that, the features that were available were standardized. In fact,
standardizing the data before classiﬁcation is important because it helps to ensure that all
the features are on the same scale and have the same distribution. This is crucial for many
machine learning algorithms that assume that all the features are equally important.

4.1.1
Random Forest
Random Forest is a machine learning technique that belongs to the family of decision tree
algorithms. It is a powerful and versatile method that can be applied to both regression and
classiﬁcation problems.
The concept behind Random Forest involves generating multiple decision trees, referred to as
a "forest", and allowing each tree to make a prediction independently. The ﬁnal prediction
is then derived by combining the predictions of the individual trees through a majority vote.
This combination of multiple trees gives the algorithm its name, "Random Forest".
Random Forest implements a random selection process for both the data points and features
used to build each tree. This randomization results in a diverse set of trees, reducing the
risk of overﬁtting, which is a common problem in decision tree algorithms.
In addition to randomization, Random Forest also incorporates "feature bagging", which
randomly selects a subset of features to use in splitting at each node of each tree. This fur-
ther randomization helps to reduce overﬁtting and enhance the performance of the algorithm.
The Random Forest classiﬁer in this work is applied using the Gini criterion which is a
measure of impurity. At each node in the decision tree, the Gini criterion is calculated based
on the distribution of class labels in the data samples that reach that node. The goal is to
split the data in such a way that the Gini criterion is minimized, meaning the split results
in a more pure subset of the data with respect to the class labels.
Mathematically, the Gini criterion is deﬁned as follows:

In the Random Forest algorithm, the Gini criterion is used to determine the best split at
each node of the decision tree by comparing the Gini impurity before and after the split.
The split that results in the largest decrease in the Gini impurity is selected as the best split.

4.2.1
Balance Cascade
The main focus of this study has been on the application of the Balance Cascade algo-
rithm, as outlined in the introduction. The algorithm, originally developed by Liu et al.
in [12], has been modiﬁed and explored in this work. This subsection will be divided into
four parts, starting with the original algorithm and progressing with the modiﬁcations made.
The Balance Cascade approach involves a two-step procedure to address the class imbalance
issue in the training data. Firstly, it performs an exploratory under-sampling of the pre-
dominant class to generate a well-balanced subset of the data. This is achieved by randomly
selecting a limited number of samples from the majority class.
After, a classiﬁer is trained on the balanced subset and used to predict the class labels of the
remaining majority class samples. In case the classiﬁer misclassiﬁes a majority class sample,
it is then added to the balanced subset. This cycle repeats until all majority class samples
have been correctly classiﬁed.
By continuously adding samples to the balanced subset until the classiﬁer can accurately
predict the class labels of all majority class samples, the Balance Cascade algorithm ensures
a balanced training set that can be used to train a more precise classiﬁer.

AdaBoost algorithm [15], can be summarised as:
• Initialize the weights of the training samples: Initially, all training samples are given
an equal weight.
• Train a weak learner: The ﬁrst weak learner is trained on the weighted samples, with
the samples that were misclassiﬁed in the previous iteration given a higher weight.
• Update the weights of the samples: After training the weak learner, the weights of
the samples are updated based on the error of the weak learner. Samples that were
correctly classiﬁed are given a lower weight, while samples that were misclassiﬁed are
given a higher weight.
• Combine weak learners: The weak learners are combined by taking a weighted average,
with the weights determined by the error of each weak learner.
• Repeat steps 2 to 4: The algorithm continues to repeat steps 2 to 4 until the maximum
number of weak learners is reached, or the minimum error is achieved.
• Final prediction: The ﬁnal prediction is given by the following formula:

• True Positive (TP): the number of positive samples that were correctly classiﬁed as
positive.
• False Positive (FP): the number of negative samples that were incorrectly classiﬁed
as positive.
• True Negative (TN): the number of negative samples that were correctly classiﬁed
as negative.
• False Negative (FN): the number of positive samples that were incorrectly classiﬁed
as negative.

Discussion
This section will discuss the ﬁndings of the study and their implications.
Firstly, the key results will be summarized. The Bounds approach showed that the Upper
Bound is easily calculated with standard machine learning methods such as Random Forest,
whereas the Lower limit can be challenging due to its imbalance. These results are consistent
with previous research, including that of Ceccarelli et al [1]. The study also found that an
iterative approach and modiﬁcations to the Balance Cascade algorithm [12] can overcome
this issue.
After that, the study’s limitations will be explored. One limitation is that the algorithm
has only been tested on station-based systems and only used for resolving static inventory
problems. Despite these constraints, the research contributes to the literature on solving
inventory problems through machine learning techniques.
Lastly, the future research directions will be discussed.
6.1
Findings
The primary focus of this study is based on the methodology employed in [1]. Consequently,
the project was partitioned into three distinct parts:
1. Veriﬁcation of the Imbalance Problem: the ﬁrst step aimed to determine whether the
imbalance issue is peculiar to the dataset used in [1] or it is a general challenge in bike
trip prediction. The approach adopted in both this work and [1] involves categorizing
the inventory problem into Upper Bound and Lower Bound bikes.
2. Demonstration of the Ineﬃcacy of Machine Learning Methods: the second question
aims to determine if a standard ML technique could eﬀectively tackle the classiﬁcation
problem.
3. Development of a Novel Approach: if the two mentioned stages were validated, a novel
approach had to be identiﬁed and utilized to address the Inventory Problem.
After conducting analysis and pre-processing steps, it was determined that the dataset was
imbalanced, with over 70% of data belonging to the Lower Bound class. This ﬁnding con-
ﬁrmed the ﬁrst point and suggested that standard machine learning techniques could predict
the Upper Bound with accuracy but not the Lower Bound. To overcome this limitation, we
explored the Balance Cascade algorithm from [12] and adapted it to address the problem.
An iterative approach was employed to tackle the classiﬁcation task, and the Balance Cas-
cade algorithm was utilized in multiple iterations to address the issue of imbalance. The
primary outcome of the study, pertaining to the prediction of the Lower Bound, was that

6.2
Limitations
The study’s major limitation is the absence of a feature that distinguishes between diﬀerent
classes or at least classes with a low or high value of Lower Bound, which prevented the
achievement of good results. The importance of the feature and the visualization did not
yield conclusive results for the objective at hand. There is clearly a need for improvement
in this area, particularly with regards to visualization. By examining the city map and in-
corporating newly observed feature values, the prediction could be enhanced. A thorough
analysis of this could lead to better results through the use of an iterative approach and a
random forest model. Avoiding the long process of writing an algorithm from scratch.
Another limitation could be the LB calculation method employed in this study, which is the
same as that used in [1] for comparison purposes. This method works in theory but it has
not been proved to work in a real-case application.
Furthermore, it has to be remembered that this study focused on a station-based bike-sharing
system with static rebalancing.
6.3

To enhance the classiﬁcation, visualization tools can be employed to identify fresh features
as previously stated.
Incorporating sociological data can also be a compelling addition,
whereby monitoring user types and acquiring additional trip-related details can aid in re-

into the various classes.
The Inventory Problem is being tackled using a Machine Learning approach in this case.
It would be worth exploring the possibility of addressing the same problem using Neural
Networks, which have been extensively researched and may yield even better outcomes.
Alternative methods that go beyond setting limits on the number of bikes can be explored.
For instance, clustering the city’s diﬀerent zones instead of relying on station positions could
be a promising avenue to pursue. In practice, Bike Sharing companies are concerned with

7
Conclusions
The objective of this thesis was to investigate the optimal inventory level estimation problem
using a large dataset of over 5 million bike trips from the Blue Bikes program in Boston,
comprising more than 350 stations. The ﬁrst step in the thesis was to develop a code for
calculating the Lower Bound (LB) and Upper Bound (UB) classiﬁcation. The UB classiﬁ-
cation achieved good results with standard machine learning (ML) techniques, while the LB
classiﬁcation required the use of a specialized algorithm. In particular, the Balance Cascade
algorithm was selected to address the imbalanced class distribution issue encountered in the
LB classiﬁcation, where the ﬁrst class alone represented more than 70% of the data.
To apply the classiﬁcation, a clustering approach similar to that used in a previous study
was employed, where each cluster represented a group of observations with similar LB.
A new dataset was created, which included features related to trip departures, arrivals, and
duration. Random Forest and XGBoost were initially used for both UB and LB prediction.
While the UB prediction was successful with both methods, there were issues in correctly
predicting the diﬀerent classes for LB, due to the assumption of a balanced dataset in stan-
dard ML methods. This resulted in overﬁtting of class 0 and no results for the other classes.
Therefore, the investigation focused solely on LB prediction.
After extensive research on undersampling and oversampling techniques, the Balance Cas-
cade method was chosen and manually implemented to better understand its iteration-by-
iteration functioning. The method splits the majority class to create a balanced problem
with the minority class and optimizes a value θ to reach a speciﬁc false positive rate f at
every iteration. The AdaBoost classiﬁer was used to solve the balanced classiﬁcation created
by the Balance Cascade. The method was found to be eﬀective, with an average f1-score of
21% and an average recall of 37%. However, some modiﬁcations were proposed to improve
the results, such as applying a penalty ω to the strict f and optimizing the value of ω for
each iteration. The modiﬁcations resulted in a signiﬁcant improvement in the average recall
and f1-score, from 30% to 46% and 32% to 48%, respectively.
Furthermore, the distribution of errors was analyzed to conﬁrm the improvements. The fea-
ture importance was visualized, revealing that the average LB was important for predicting
classes with low LB, while trip duration was the main feature for predicting classes with
high LB. The feature selection for every iteration could be another step towards improving
the results. This study demonstrates the potential of machine learning in estimating the
optimal inventory level, and future research could explore alternative approaches to address
the challenges encountered in this study.

The csv ﬁle created and needed to run the diﬀerent notebooks are:
DataBike.csv: contains all the trips downloaded from BlueBikes and preprocessed in order
to be ready to calculate Lower and Upper Bound
stations.csv: contains all the bike stations from BlueBikes and some little modiﬁcation to
make the dataset usable.
lost_of_days.csv: contains a list of the days for the trips.
data_work_experiment.csv: contains the dataset prepared for the classiﬁcation.
Data_Boston.csv: contains the same dataset for the classiﬁcation, but with some useless
column dropped or categorical data treated.
