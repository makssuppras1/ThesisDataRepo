Abstract
The aim of this thesis is to create an Acoustic Virtual Reality system with dynamic
source and receiver, meaning that the source/receiver can move freely in the virtual
scene. The acoustic simulations and auralization process operate entirely in real-
time and the system is able to adjust to any change in the virtual environment or
in the acoustical parameters of the space.
The two major objectives of this thesis are the combination of the image source
method and the ray tracing/secondary source method in a hybrid model as well as
the parallel programming of each method and the optimization in terms of perfor-
mance. Both methods are designed to operate on CPU architecture and the de-
terministic image source method is also implemented on GPU. The operation time
and limitations of each algorithm are measured for various acoustical conﬁgurations
and the performance of the image source method for both hardware architectures
was obtained. The simulation techniques are also validated in terms of computa-
tional accuracy and reverberation time, by comparing to Odeon software and to
Phased Acoustical Radiosity and Image Source Method (PARISM) model. Lastly,
the algorithms are embedded in Unity and Virtual Reality scene is build.
Regarding the hybrid model, an energy imbalance between the two methods is
investigated and some techniques are suggested. Moreover, it is revealed that the
GPU implementation of the IS method is computationally more eﬃcient, and it is
suggested to be compared to the stochastic image source method on GPU that has
already been developed.

Chapter 1
Introduction
1.1 Motivation
The Virtual Reality (VR) ﬁeld has gathered a lot of interest during the past decades
and the application range becomes wider over the years. Nowadays, VR techniques
are used for virtual training, medical rehabilitation, building design and for enter-
tainment purposes, such as in video games, movie theaters, concerts and museum
installations. The audio implementation in VR systems has gain the proper atten-
tion making the role of sound as essential as vision’s, contributing to a more realistic
and complete overall experience.
Many methods have been designed for interactive acoustic and virtual walk-
through applications. The majority uses hybrid room acoustic simulation techniques
to model the sound propagation in buildings and spaces. DIVA [28] implements the
image source method for the early reﬂections and uses the ray tracing method and
reverberation ﬁlters to create a pre-calculated artiﬁcial reverberation tail. EVER-
Tims [23] uses the beam tracing method for the early reﬂections and an algorithm
based on Sabine’s formula for the late reverberation. The recently developed Res-
onance Audio [24] application implements the ray-tracing method and Funkhouser
[11] implemented the beam tracing method. AD-Frustum [7] is based on the frustum
tracing technique and RAVEN [30] combines the image source and the ray tracing
method to model the early and late reﬂections respectively. These are just a few
examples of the research activity regarding acoustic virtual reality.
As the performance is of great interest in real-time applications, many methods
have been also implemented on Graphics Processing Units (GPU). For example, the
deterministic image source method has been implemented by Fu [10], but only for
rectangular enclosures and Taylor and Diaz-Guerra have implemented the stochastic
image source method on GPU [32] [8].

1.2. Research objectives
The Virtual Acoustics (VA) methods can be divided into two major categories.
On the ﬁrst one, the acoustic simulations are executed in advance for a grid of
spatial points. The position of the receiver deﬁnes the choice of a set of impulse
responses for recreating the sound ﬁeld.
The main advantage of these methods
is the implementation of more precise acoustic simulation techniques.
However,
these techniques are computationally ineﬃcient and thus pre-computation of the
IRs is necessary to achieve a smooth sound experience. On the second category,
the acoustic simulations run entirely in real-time and no information is deﬁned in
advance. These methods can be easily adapted to any acoustical environment but
they lack of precision.
1.2 Research objectives
The aim of this project is the development of an interactive virtual acoustic system
and the integration with Unity software to create an overall virtual reality experi-
ence. The system is designed to simulate the sound ﬁeld entirely in real-time and the
choice of the acoustic simulation methods aims to achieve the best possible accuracy
within time restricts of a real-time application. The project was mainly inspired by
Raven software [30] and was based on the previous work of Finnur Pind [22] and
Thomas Vestergaard [33] at the Acoustics department at DTU.
One of the main objectives of this thesis is the design of the deterministic image
source method for GPU, that it has not previously developed to the knowledge
of the author. Furthermore, the image source method will be combined with the
ray tracing/secondary source method while in most real-time geometrical acoustics
application the classical ray tracing method is used. Both choices aim to achieve a
more eﬃcient simulation model suitable for real-time VA. As part of the above, the
following research questions and objectives were also identiﬁed.
• Investigation of combining the image source method with the ray tracing-
secondary source method in a hybrid model.
• Design of the algorithms for parallel programming and determine eﬃcient op-
timizations in terms of operation time for CPU and GPU architectures.
• Investigation of the relation between diﬀerent acoustical conditions and the
operation time of each algorithm.
• Investigation of the limitations of the algorithm in diﬀerent acoustical situa-
tions.
• Exploration of Unity as a platform suitable for Virtual Reality research.

Chapter 2
Theory
2.1 Virtual acoustics
Virtual Acoustics is a general term for the simulating of the human acoustic expe-
rience in buildings and open spaces. The creation of a VA system comprises many
ﬁelds of Acoustics with the ambition of delivering an aural message to the listener/s
as it would happen in the real world. In this chapter, a virtual acoustics system and
its aspects are described brieﬂy.
2.1.1 Virtual acoustics system
A VA system is a system that recreates the human hearing impression under certain
acoustical conditions.
The process of creating a virtual acoustic system can be
divided into three main stages: a) deﬁnition, b) modeling and c) reproduction.
The deﬁnition of a virtual acoustic environment includes all the a priori knowledge
about the sound source, the space and medium of propagation and the listener. Both
physical and perceptual characteristics of each part of the system should be taken
into account. The modeling stage incorporates the source, space and listener models
that suit the purposes of the case. The reproduction refers to the production of the
three-dimensional (3D) sound image for a pair of loudspeakers with an adaptive
cross-talk cancellation, a set of loudspeakers or headphones. Figure 2.1 illustrates
the stages of implementing virtual acoustic environments.

is then processed and reproduced though headphones or a set of loudspeakers. This

2.1.3 Sound source model
The simplest model for a source is an omnidirectional point source.
This prac-
tice holds suﬃciently for far ﬁeld analysis of the sound ﬁeld and, additionally, it
is computationally eﬃcient. However, this does not hold for most real cases where
the sound source does not generate waves in the same way towards all directions.
For example, in a loudspeaker, low frequencies propagate approximately towards all
directions while high frequencies are directive only to a segment of the frontal hemi-
sphere. Directional patterns, so-called directivities, that are frequency-dependent
have been developed and implemented in some applications. Figure 2.4 shows an
example of the directivity pattern of a saxophone for two frequencies. In this report,
the source will be modeled as omnidirectional for simplicity reasons.
Figure 2.4: Directivity of an alto saxophone. Adapted from [29].

In most cases in VA, sound propagates inside an enclosure or building. The sound
waves bounce oﬀthe walls and interact with the obstacles inside the room before

is called direct sound and the rest of the waves are called reﬂections. They can

To model an acoustical environment, knowledge about the geometry of the
room, the shape and the irregularities of the reﬂecting surfaces, the objects that may
exist in the room as well as the characteristics of the medium of propagation, such
as temperature and density, are essential. Many techniques have been developed
towards modelling the sound ﬁeld and they can be divided into two major categories,
namely the geometrical acoustic (GA) and wave-based methods. Figure 2.6 shows

In the ﬁrst category, the waves are assumed to propagate rectilinearly and their
motion is dictated by the geometrical laws of optics. These techniques are in general
suitable in cases where the wavelength is small compared to the dimensions of the
room or the objects and large compared to the roughness of the surfaces. Therefore,
they are suﬃcient for simulating large spaces or high frequencies. They are also
characterized by computational eﬃciency and for that reason they are preferable for
real-time applications. Two major GA methods are the ray tracing (RT) and image
source (IS) method and they are described in the next chapters.
The wave-based techniques aim to solve numerically the diﬀerential equations
that govern the sound behaviour in the enclosure. To achieve that, the surfaces and
the medium are segmented into small ”areas” that form what is commonly called
’mesh’. The diﬀerential equations are then approximated within these areas as linear
equations. These methods consider any type of acoustic phenomena and therefore
they are very accurate. However, they are computationally demanding especially
at high frequencies and for large enclosures. Real-time applications that use these
methods, have to pre-compute and store a large amount of data in order to reduced
the real-time calculations as possible. Although they provide a more accurate result
than the GA methods, they are restricted to the geometry they are designed for.
2.1.4.1
Image source method
Image source method, introduced in [1] and described thoroughly in [3], is a widely
used method for modelling accurately the specular reﬂections in the room. As it was
mentioned above, it is a fair approximation for large spaces or for high frequencies as
well as for surfaces with low absorption. This method is simple and the construction
of the impulse response (IR) is also a computationally light process.
However,
for higher order of reﬂections and complex environments, it is unsuitable for real-
time applications as its operation time increases dramatically. Additionally, it does
not account the scattering eﬀect, which is a dominant phenomenon especially at
late reﬂections and, therefore, it is not usually chosen as a standalone method for
simulating sound ﬁelds [4]. Generally, IS method is suﬃcient for modeling the early
reﬂections but it does not provide valid results for the reverberation tail.
2.1.4.2
Ray tracing method
Ray tracing method is a stochastic process where a large amount of rays are emit-
ted randomly towards all directions inside an enclosure and the energy that hits a
point, centered at receiver’s position, is computed and stored for further process.
Opposed to IS method, it considers the scattering eﬀect but it lacks a proper tem-

2.1.4.3
Hybrid models
To exploit the beneﬁts each method has to oﬀer, many applications combine two
or more diﬀerent techniques.
Many criteria can be considered for splitting the
simulation to two or more parts. A common practice is to model separately the early
reﬂections with the late reﬂections. Subsequently, a hybrid model may combine the
IS and RT method for the early and late part of IR respectively or a wave-based
method with a statistical GA method for the respective IR parts. Another approach
is to divide the simulation to low and high frequencies and implement appropriate
methods for each section. The choice depends always on the speciﬁcations of each
application.

taken into account. Each ear can be modeled as a LTI system which is characterized
by its frequency response, called Head Related Transfer Function (HRTF). HRTFs
are functions of the frequency, the relative azimuth and relative elevation angle
between the source and receiver. Figure 2.7 illustrates the 3D polar system centered
at listener’s position.

2.1.6 Time constrains in real-time
The time a VR system needs to react to any action of the user is of great signiﬁcance.
There should be kept under certain limitations in order to ensure a smooth and
realistic walkthrough in the VR scene. There is remarkable research that investigates
the parameters that can deﬁne these time constrains. The acoustical perception
plays a signiﬁcant role and psycho-acoustical experiments have showed that a lag of
approximately 75 ms between the user’s action and system’s response is perceptually
unnoticeable [5]. However, this should not be a rule of thump in every situation as
the characteristics of spaces do not change signiﬁcantly in every event and the human
perception vary according to the scene. For example, in a virtual reality scene, the
early reﬂections play a crucial part to the localization process and the rotation of the
head is most of the times a rapid movement. On the other hand, the reverberation

2.2 Parallel programming
In parallel programming, large problems can be divided into smaller ones and ex-
ecuted at the same time. The main beneﬁts that come with parallelism are the
enhancement of the computer application performance and the reduction of the
power demands of the system. There are various types of parallelism but only the
instruction-level parallelism is described brieﬂy in this report. It refers to the par-
allel execution of instructions sets from the processor without aﬀecting the result of
the algorithm. As long as there is no data dependencies between the instructions,
parallelism is feasible.
All modern processors have multiple processing units that are called cores.
Each core can execute a diﬀerent task from the others or the same instructions
stream but on diﬀerent data. Moreover, each core contains multiple execution units
that diﬀerent instructions can be issued at the same time. Therefore, a multi-core
processor can execute multiple instructions from multiple instruction streams at the
same time. The distribution of the instruction streams is handled by OpenMP and
the handling of instruction sets is called vectorization. Both are presented brieﬂy
below.
2.2.1 OpenMP
OpenMP is a parallel programming model for shared and distributed shared mem-
ory multiprocessors. It is widely used and it has become a standard for parallelizing
algorithms. More information about OpenMP can be found in the oﬃcial website
[21].
To be able to exploit the possibilities of OpenMP, the algorithms were de-
signed in a way where each subproblem could read from the same data, execute
the same set of instruction stream and write to diﬀerent memory locations. Prag-
mas were applied to the code in order to indicate the parts of the script that are
designed for this puprose, and in particular the #pragma omp parallel and the
#pragma omp parallel for.

2.2.2 Vectorization
Vectorization is the process of modifying an algorithm from a scalar implementation,
where only one operation is executed at a time to a vector process, where a piece
of code handles vectors of data an the same time. This type of parallelism occurs
inside a CPU core exploiting the parallel execution units of the single core. This
can improve signiﬁcantly the operation time of the algorithm. While compilers are
able to vectorize automatically, higher control can be achieved through the use of
command line ﬂags, pragmas in the script and intrinsic instructions. in this project,
the #pragma omp simd directive is applied to any loop which is designed in a way
that multiple iterations can be executed at the same by using Single instruction,
multiple data (SIMD) instructions.
To make the usage of SIMD instructions possible, the following strategies were
applied. Whenever possible, loops were designed to be countable and termination
conditions that were data-dependent were avoided (e.g. ”break”, ”return” state-
ments). SIMD (Single instruction, multiple data) instructions were performed in
order to maintain the same control ﬂow for each iteration. Control ﬂow statements
(e.g. ”switch”, ”goto” statements) where also avoided and ”if” statements were im-
plemented only as masked assignments. Moreover, function calls replaced by inlined
functions and only functions compatible with vectorization where used. Regarding
the data handling, arrays were preferred and in each loop, the loop index was asso-
ciated directly with the array index in order to reserve data independencies between
iterations. The size of each array was also chosen towards that direction, so itera-
tions can have shared memory to read from but diﬀerent memory locations to write
to (i.e.
no write-after-write, write-after-read and read-after-write dependencies).
Moreover, data contiguously aligned in memory were designed and particularly in
IS method, large structure data were organized as structure of arrays (SoA) instead
of array of structures (AoS) because in cases where loops visit only some of the ele-
ments of the structure and SoA is chosen, some memory allocations of the fetched
cache lines remain unexploited. Below, the strategies implemented are presented
aggregated.
• Countable single entry and single exit ”for” loops
• SIMD instructions
• No functions calls, use of inlined functions
• Data independencies between loop iterations
• Arrays of data and structures organized as SoA

2.4 Ray tracing method
The ray tracing algorithm is described thoroughly by Kulowski [14]. It is a method to
simulate the specular and diﬀuse reﬂections inside an enclosure. Its basic principle
consists of emitting thousands of rays from a source point in a 3D space.
The
emission direction is distributed randomly towards all directions and the propagation
paths are tracked. When a ray reaches the receiver, its energy, time of arrival and
direction is stored for further processing. A common practice is to gather the energy
and time of arrival of each ray in an energy histogram that contains information
about the sound ﬁeld and the acoustical properties of the room, such as reverberation
time.
Ray tracing method is a stochastic operation and a large number of rays is
needed in order to cover all possible propagation paths in the room and obtain a
suﬃcient energy histogram. It is one of the most common practices in room acoustics
simulation due to its simplicity. In comparison to the image source method it is less
accurate due to the low resolution of the histogram in the time domain. However, it
incorporates the scattering process and is computationally more eﬃcient. For that
reason it is preferable in cases where the geometry of the room is quite complicated
or the high resolution of the impulse response is out of interest, for example in the
construction of the reverberation tail. In the following sections the algorithm is
described thoroughly.

order of 104 in order to gather a suﬃcient number of hits, making the algorithm com-
putationally demanding and time ineﬃcient. Zeng proposed the optimal receiver’s
size, expressed in equation 2.20 but this can increase the eﬃciency of the method
to a certain point.

receiver is now modeled as a point and the contribution of each SS is computed by

Chapter 3
Implementation
In this chapter the algorithms designed for real-time implementations are described.
In the ﬁrst and second part, the image source and ray tracing methods are presented
while in the last part the way these methods are combined is also explained.
3.1 Image source method
In this section two diﬀerent developments of the IS method are described. The ﬁrst
one, here called method A, is designed to reduce the computational cost as much as
possible while the second algorithm aims to be beneﬁt of parallel programming mak-
ing the threads operate independently from each other and use vector instructions
to optimally exploit data-parallelism.
3.1.1 Method A
In this method, the computation of the ISs and the IR is conducted separately in
two stages. The corresponding block diagram is illustrated in ﬁgure 3.1. In the ﬁrst
stage, the IS tree is constructed and the validity check is obtained while the second
contains the visibility check and the computation of the IR response. These two
functions were separated because it is not always necessary to recompute everything
in case that something in the scene changes. For example, in case the sound source
position and the geometry of the room do not change, the position of the ISs do
not change either. Therefore, in case that only the listener changes his position in
the room or rotates his head, only the visibility check and the IR construction must
be conducted again. However, if the source moves or the geometry changes, then
the tree must be reconstructed. Consequently, diﬀerent events in the virtual reality
scene demand diﬀerent update functions.

3.2 Ray tracing method
The ray tracing method algorithm operates in a single stage for eight octave fre-
quency bands, with center frequencies at 63, 125, 250, 500, 1000, 2000, 4000 and
8000 Hz. A predeﬁned number of rays are emitted by the source and their initial
energy is set to 1. The propagation of each ray stops when its energy is reduced
below −60 dB or the propagation time exceeds the length of the impulse response
which is also deﬁned statically in the script. Whenever a secondary source is com-
puted, its energy contribution and time of arrival are obtained and the histogram
is updated accordingly. The histogram is then averaged in 3ms chunks and every
Poisson distributed delta is scaled by the amplitude of the chuck that is associated to
the time shift of the delta function. The Poisson sequences for the speciﬁc geometry
are pre-computed and stored and they do not need to re-calculated if the geometry
does not change drastically.
The positions of the secondary sources are not stored like the ISs due to the
correspond huge data size. In the simple case where the number of rays is 1000 and
the absorption from the walls is low, the number of SSs can exceed the one million.
It is assumed that storing the SSs and separating the operation in two stages like
the IS method would increase the computational cost.

while the reﬂection density increases by the time of arrival. The SS-R distance is
always smaller than the larger cross dimension of the room and results the energy
imbalance between the two methods, especially for higher order of reﬂections, where
the IS-R squared distance increases signiﬁcantly.
A possible solution would be to truncate the IS method from the SS method at
the time point where the the energies match. Figure 3.7 presents an example of the
reﬂectogram for a single frequency band obtain as described above.

the IS method, small enough so the IS method will operate below the matching point
of the two reﬂectograms and implement a scattering model to compensate for the
energy imbalance at the transition point, including at the same time the scattering
eﬀect in the IS method. This would be similar to the ESR model of Odeon, as

4.2.2 Convolution routine
To ﬁlter long signals overlap-add and overlap-save methods have already proposed
as computational eﬃcient methods.
In the convolution routine the overlap-save
method is implemented. The main principle of this method is that a long signal is
segmented into smaller blocks and each block is then convolved with the impulse
response. The convolution is operated in the frequency domain by multiplying the
discrete Fourier transform (DFT) of each block with the frequency response of the
system. The result is then transformed back to the time domain.
In detail, the input mono audio signal is divided into equal sized blocks of length
L. The size of the DFTs and iDFT is N = L + M −1 where M is the length of the
impulse response. The input signal is appended with M −1 zeros and the impulse
response of each channel is padded to length N by adding L −1 zeros at the end
in order to prevent aliasing eﬀects. In the ﬁrst iteration, the ﬁrst N samples of the
audio signal are feeding to the convolution routine. The DFT of this N −length
block is computed and is multiplied with the DFT of the right and left channel.
The signals are then converted back to time domain and the ﬁrst M −1 samples
are discarded. The resulting L −length two blocks are then played through sound
card to the headphones. Figure 4.4 illustrates the overlap-save method.
To make the routine more time eﬃcient, the DFTs and iDFT were executed
eﬃciently by using Fast Fourier Transform (FFT) algorithms, included in FFTW

Chapter 6
Discussion
As described in chapter 3, the ﬁrst implementation of the IS method was designed
to be optimal in favor of number of operations and memory usage. This implies
the control of the program ﬂow through conditional statements in order to avoid
unnecessary operations. This choice is justiﬁed by the signiﬁcant smaller number of
ISs that contribute to the sound ﬁeld, compared to the total number of produced
nodes, as indicated in ﬁgure 2.14. A drawback of this method is that the computation
of each level of the tree is dependent from the previous ones and also each node
depends on its parent node. This aﬀects the way the ISs are stored in memory and
how these memory allocations are accessed in the algorithm. As a result, this causes
memory strides that prevent the exploitation of vectorization process. Moreover,
any conditional statement reduces the performance of the instruction execution by
the lanes of the CPU vector-units.
In conclusion, this method is unsuitable for
vectorization and an essential part of parallelism is lost up to a factor of 16.
The main challenge in the second method was to parallelize the operations
needed for a single IS. The conditions used in validity check and especially in the
visibility check which is the most computationally demanding stage of the algorithm,
reduced the operation time of this method. Although the number of conditions were
kept to the minimum, the achieved speedup was not satisfactory. In addition, the
recursive nature of the visibility algorithm requires the storage of the sequence of
nodes that produce each IS and subsequently this causes unaligned memory access,
reducing further the performance. Despite the eﬀorts by the author, this method
was not successfully parallelized for operating in a CPU and further optimizations
have to be done to make this method suitable for running on a CPU. Regarding the
performance of the same method on the GPU, a total speed-up of 12.5 was obtained
comparing to the HPC system performance, making this algorithm suitable for GPU
implementation.
By comparing the two methods, IS method A on HPC was found to be faster

Discussion
in the ﬁrst conﬁguration. However, the surfaces in the room were designed in a
way that the room produced many invalid sources, making this method faster. In
addition to that, in the second conﬁguration, where the number of surfaces is again
96, the operation time of this method increases. Therefore, the operation time of
the method A is considered to be strongly dependent on the geometry of the space.
The operation time of the ﬁrst method is dependent on geometry and increases with
the number of valid ISs, as described in section 5.3.
The GPU implementation operates faster in the second room conﬁguration,
where the number of valid ISs is signiﬁcantly higher than in the ﬁrst room. To
investigate thoroughly the performance of both methods, real and complex geometry
conﬁgurations should be taken into account. Nevertheless it is expected that in more
complex geometries, where the surfaces are not always parallel to other ones, the
operation time of the method B on GPU will be even smaller compared to the
method A on the HPC system. In conclusion, it would be reasonable to consider
the method B implemented on GPU as the most computational eﬃcient one. This
means that the exploitation of hardware in this case played a more important role
than the optimization of the algorithm in terms of the number of operations. A
further research objective would be the comparison of the deterministic IS method
to the stochastic IS method on GPU, developed by Taylor [32].
The secondary source showed great advantages. Within the time limit of 75 ms,
180 and 4000 surfaces can be handled on a commodity CPU and on the HPC system
respectively, if the number of rays equals to 1000. Additionally, 7.5 and 28 thousands
of rays can be emitted in the simple case of a rectangular room. The next step
would be to compute the binaural IR. The signiﬁcantly high performance of this
method would possibly provide suﬃcient time for computing the binaural IR before
exceeding any time constrain. Comparing to the classical ray tracing method, which
needs more than ﬁfty times more rays than the RT-SS method, it is expected to be
approximately faster by a factor of 50. As a result, through a better update rate in
the late part of the IR, a more realistic acoustic experience is possible to be achieved,
especially in cases where the geometry changes rapidly. Moreover, as the operation
time of the RT method is comparable to the IS method, the later method can be
prioritised in terms of computational power, aiming at even more accurate results
and optimizations.
Regarding the acoustical aspects of this project, the visibility check plays an
essential role even in a rectangular enclosure and must be implemented in both
deterministic and stochastic image source methods. As showed in section 2.3.2, the
number of invisible image sources increases dramatically by the order of reﬂection
in a simple box room and in complex environments this number is expected to be
even higher.

Discussion
A challenge arise in combining the IS method with the RT-SS method due to
energy imbalance between the ISs and SSs. It was suggested to truncate the reﬂec-
togram of both methods at the time point where the the energies match. However,
as this point is assumed to be dependent on the acoustical parameters of the room,
it would be challenging a predeﬁned value of transition order to be set. It could
be though computed in real-time but further computations would possibly reduce
the performance that is crucial for real-time applications. A suggested method to
overcome this would be to consider the energy of the IS in a diﬀerent way, by con-
sidering a partial distance from the IS to the receiver. However, the level of the
early reﬂections are of great signiﬁcance in localization process. A third suggestion
would be to include a scattering model in the IS method that would compensate any
potential energy imbalance, in case the maximum order is set to a reasonable low
value. However, the validity of these methods is unknown to the author and further
investigation should be conducted through psycho-acoustical experiments. Overall,
a comparison of the suggested VR system to RAVEN software would possibly be a
direct way to compare the eﬀect of classical ray tracing method to the secondary
method in terms of acoustical perception.
Lastly, Unity software was found to be suitable for real-time VA systems through
its compatibility with C ++ and it is convenient for research purposes as it provides
full control of the audio chain.

Chapter 7
Conclusion
The main objectives of this thesis are three. Firstly, the optimization of the IS and
RT methods in favour of operation time and the implementation of the ﬁrst one on
GPU. Regarding the deterministic IS method, the immense unnecessary calculations
computed by this method led to the design of two algorithms. The ﬁrst one is able
to reduce the operations to the minimum while it cannot parallelized optimally in
a modern hardware. In the second one, the number of computation reaches the
maximum possible limit but at the same time the method exploits both stages of
parallelism, threads and vector-units, being suitable for GPU implementation. It
was found that the performance of the second algorithm was higher, emphasizing
the dependence of the algorithmic design on the hardware architecture.
The secondary source method was found to decrease signiﬁcantly the operation
time of the RT method. As a result, in a VR application, more computational power
can be assigned to the IS method, or a low update rate can be set to the late part
of the IR improving possibly the overall VR experience.
The second goal of this project was the combination of the image source and
ray tracing method in a hybrid model.
It was found that the two methods are
not directly compatible. The diﬀerent way the energy is computed causes energy
imbalance at the transition point of the IR. Three solutions are suggested by the
author to overcome the mismatch between these methods, but further investigations
should be done to determine the most eﬃcient one in terms of acoustical perception.
The third objective is to implement these methods in Unity aiming to create
an interactive VR scene. The Unity software provides full control of the audio chain
and its compatibility with many programming languages and APIs makes it suitable
for research development and applications.

7.1. Further research
7.1 Further research
This project can be a starting point for further research and investigation for im-
proving the developed VA system and comparing with other VR applications in
order to achieve an overall knowledge of which VA techniques are perceptually ac-
ceptable and computationally eﬃcient. The potential research objectives are listed
below and can be divided into two categories, namely the ones that focus on the
acoustics and computational aspect respectively.
• Auralization for the late part of the impulse response that corresponds to the
RT method.
• Validation of the hybrid model by conducting acoustical perceptual measure-
ments.
• Integration of diﬀraction models for further improvement of the acoustic sim-
ulations.
• Investigation of the proposed techniques that combine the two methods.
• Comparison of this model with RAVEN software that combines the IS and the
classical RT method.
• Comparison of this real-time algorithm with a VA system based on pre-computed
acoustic simulations.
• Investigation if further optimizations can be achieved in terms of reducing the
operation time of the algorithm.
• Comparison of the deterministic IS method on GPU with the stochastic IS
method implemented in [32].

User Guide
This Appendix provides some useful guidelines for creating .dll ﬁles in Visual Studio
and how to implemented them in Unity.
Visual Studio - .dll ﬁles
Below, a simple example of a Dynamic Library ﬁle is presented. Two functions are
included in the ﬁle, namely the initialization() and the deg2rad(). The purpose of
the plugin is to convert degrees to radians. To do that, the initialization function
has to be called at the beginning. This function sets the proper value for π before
any other calculation is done. After this step, the function deg2rad() can be used.
To write a script that creates the DLL, open the Visual Studio and go to
File ¿ New ¿ Project ¿ Visual C++ ¿ Empty project. In the next step, as showed
in ﬁgure D.1, create the .cpp ﬁle that should contain the C + + script.
In the
last step, go to Project properties ¿ Conﬁguration properties ¿ General and set
the Conﬁguration type to setDynamic Library (.dll). Other settings that should
be modiﬁed properly are the hardware architecture, x86 or x64 and the compiler
options regarding algorithm optimizations.
