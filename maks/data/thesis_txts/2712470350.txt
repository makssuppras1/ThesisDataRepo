Abstract
Modern wind turbine blades are complex structures with variations in both material distribution,
geometry and coupling effects. Due to the beam-like structure of wind turbine blades, dynamic
analysis can be done using the finite element method with beam elements. A detailed dynamic
analysis requires a beam model that is suﬀiciently accurate and eﬀicient. The main objective of
this thesis is to investigate if the accuracy of a finite element beam model of a wind turbine blade
can be increased by using beam elements that take the varying properties along the element into
account. A more accurate beam element can potentially allow for an increase in the eﬀiciency
of the model as fewer elements are required.
The project is done in cooperation with the Department of Wind and Energy Systems at DTU,
who provided a standalone beam code that allows for analysis of a single wind turbine blade
and is based on the same beam elements as HAWC2. This beam element is based on average
properties of the beam. Two similar codes has been developed with other beam elements that
take the varying properties of the beam into account.
The first code is developed with an
equilibrium-based beam element, derived from complementary energy. This beam element allows
for an exact representation of the stiffness terms in the equation of motion of a beam with varying
properties. The formulation of the beam element is extended to include an expression for the
deformations along the element and it is shown that this expression can be used to assemble
the mass matrix. The second code is developed with a modification of the displacement-based
beam element implemented in the original code. The implementation of this beam element is
extended to take the varying properties along the beam elements into account, thereby allowing
for a better representation of beams with varying properties. An updated assembly of the beam
elements with varying element properties has increased the accuracy of the beam models when
they are used to model beams with varying pre-twist.
Through verification and a test case of a wind turbine blade, the two beam elements that allow
for varying cross-sectional properties have proven to be reliable. The two beam elements can
potentially reduce the error of a beam model of a blade by approximately an order of magnitude
compared to the original beam element based on average element properties. Therefore, the
project shows that the two beam elements that take the varying properties into account have a
large potential for implementation in HAWC2. The two elements have a similar accuracy when
effects from curvature of the blade and inertial forces are included. The displacement-based
beam element that allow for varying properties has a large resemblance to the beam element
currently implemented in HAWC2. At the same time, the description of the inertial terms and
deformation along the element is more simple compared to the equilibrium-based beam element.
Therefore, based on this project, the displacement-based beam element that allows for varying
properties seems to be the best candidate for a future implementation in HAWC2.

Resume
Moderne vindmøllevinger er komplekse konstruktioner med variationer i både materialer, ge-
ometri og koblingseffekter. Finit element-metoden med bjælkeelementer er velegnet til dynamisk
analyse af vindmøllevinger grundet vingernes bjælkelignende struktur. En detaljeret dynamisk
analyse kræver en bjælkemodel der er tilstrækkelig præcis og effektiv. Hovedformålet med dette
kandidatspeciale er at undersøge om præcisionen af en bjælkemodel af en vindmølle kan øges
ved at anvende bjælkeelementer der tager højde for variationer i bjælkens egenskaber langs el-
ementet. Et mere præcist bjælkeelement kan potentielt muliggøre en mere effektiv model, da
vindmøllevingen kan modelleres ved brug af færre elementer.
Projektet er udført i samarbejde med instituttet for Vind og Energisystemer ved DTU, som
har leveret en selvstændig kode med en bjælkemodel, der tillader analyse af en vindmøllevinge
og er baseret på det samme bjælkeelement som HAWC2. Implementeringen af dette bjælkeele-
ment er baseret på bjælker med konstante egenskaber. To nye koder er opbygget med samme
struktur som den oprindelige kode men med brug af bjælkeelementer, der tager højde for varia-
tioner i bjælkens egenskaber. Den ene kode er opbygget med et ligevægts-baseret bjælkeelement
baseret på komplementær energi. Dette element giver en eksakt repræsentation af stivheden
i bevægelsesligningen for bjælker med varierende tværsnit. Formuleringen af bjælkeelementet
er blevet udvidet til at inkludere et udtryk for deformationen langs elementet, og det er vist,
at dette udtryk kan bruges til at bestemme massematricen. Den anden kode er baseret på en
modificeret udgave af det flytnings-baserede bjælkeelement i den originale kode. Formuleringen
af elementet er udvidet til at tage højde for variationer i egenskaberne langs elementet, hvilket
giver en mere præcis beskrivelse af bjælker med varierende egenskaber. De to modeller med
bjælkeelementer, der tillader varierende egenskaber langs elementerne, er blevet opdateret til at
give en mere præcis beskrivelse af bjælker med varierende twist.
Testeksempler og et case study af en vindmøllevinge har vist, at de to bjælkeelementer, der
tillader for varierende egenskaber, er robuste. De to bjælkeelementer kan potentielt reducere
fejlen i en model af en vindmøllevinge med omkring en størrelsesorden sammenlignet med det
originale bjælkeelement med konstante egenskaber langs elementet. På denne baggrund viser
projektet, at der er et stort potentiale i implementering af de to bjælkeelementer, der tillader for
varierende egenskaber langs elementerne, i HAWC2. De to bjælkeelementer har en tilsvarende
nøjagighed, når krumning af vingen og inerti medtages. Det flytnings-baserede element, der
tillader for varierende egenskaber langs elementet, har store ligheder med det eksisterende bjæl-
keelement i HAWC2. Samtidig er beskrivelsen af inertien i elementet og deformationen langs
elementet simplere end i det ligevægtsbaserede bjælkeelement. Derfor tyder dette projekt på,
at det flytnings-baserede element, der tillader for varierende egenskaber langs elementet, er den
bedste kandidat med henblik på implementering i HAWC2.

Introduction
The wind turbine industry has expanded rapidly in terms of turbine size and total installed
capacity since the first industrial turbines in the late 1970s and early 1980s. The first generation
of wind turbines had a power of 22-30 kW and a rotor diameter of approximately 10 m [21]. To-
day, wind turbines are getting close to 15 MW with rotor diameters exceeding 200 m. Increasing
rotor sizes lead to significantly larger masses, aerodynamic loads and gravitational loads. As
a consequence, wind turbine blades have become structures with complex irregular geometry
and a non-trivial distribution of composite materials, leading to elastic coupling effects. Larger
rotors lead to reduced maintenance costs and increased capacity of wind turbines. Therefore it
is of high interest to increase the size and performance of the blades. One key aspect in the
development of wind turbine blades is to get as precise and eﬀicient a numerical model of the
blades as possible without being too computationally expensive. Thereby, potentially allowing
for full fidelity dynamic analysis solved in real-time and more accurate time history analysis in
aeroelastic analysis tools.
Wind turbine blades are very long and slender structures as seen in Figure 1.1 and can be
considered as cantilever beams. Therefore, wind turbine blades are well modelled with the finite
element method using beam elements. Furthermore, beam elements reduce the computational
cost significantly compared to a full 3D analysis of a wind turbine blade. The applied beam model
has to be able to accurately describe the energy of the 3D blade with the varying properties and
coupling effects. Classical beam elements based on interpolation of the kinematic field using
Hermitian shape functions do not take all coupling effects into account or allow for varying
properties along the element. Therefore, several alternative beam elements have been presented
for the modelling of wind turbine blades. Most of the relevant elements are based on a generalized

Timoshenko beam theory for a beam with anisotropic material properties and cross-sectional
properties determined from numerical cross-section analysis tools.
The anisotropic material
properties can be described by 21 independent parameters along the blade and allow coupling
to be introduced in the models.
An example of an applied beam element is seen in the aeroelastic code HAWC2 developed by
DTU Wind Energy to calculate the response of a full wind turbine in the time domain [22]. The
turbine is modelled using a multibody formulation, where the turbine is divided in a number of
local body coordinate systems. The effects of large rotations and deformation of the blades are
taken into account by the coupling between the bodies. This allows linear elastic beam elements
to be used within the local frame of reference in each of the bodies, with the assumption of small
deformations [6]. The beam element used in HAWC2 is derived by Kim et al. in 2013 in [13]
for an initially straight beam with anisotropic linear elastic material properties. The derivation
of the beam element is based on an approximation of the displacement field using polynomials
of arbitrary order. The implemented formulation of the beam element in HAWC2 is based on
constant element properties, corresponding to homogeneous beam elements.
A finite element model of a wind turbine blade requires a suﬀiciently large number of elements
to reduce the discretization error caused by the varying geometry and properties along the
blade.
However, the computational cost of time history analysis increases significantly with
the number of elements. As a consequence, it is of interest to reduce the required number of
elements to reduce the computational cost in time history analysis. A reduction of the number
of elements requires a larger accuracy of the used beam elements. One way of increasing the
accuracy of the beam elements and thereby reducing the required number of elements is to take
the nonhomogeneous properties of wind turbine blades into account in the formulation of the
beam elements.
Project Scope
This project looks into increasing the accuracy of the beam model used in HAWC2 by using
beam elements that allow for varying properties along the element, potentially reducing the com-
putational cost of time history analysis. The project considers the so-called equilibrium-based
beam element, described by Krenk & Couturier in 2017 in [17] for modelling of nonhomogeneous
anisotropic beams. The stiffness matrix and the nodal forces associated with distributed loads
are derived in the equilibrium-based beam element using complementary energy. The deriva-
tion is based on a description of the static field of the beam in terms of the section forces and
cross-sectional flexibility matrix along the beam element. As the section forces are independent
of the properties along the beam element, this allows for an accurate description of the element
stiffness matrix and nodal forces associated with distributed loads for a beam element with vary-
ing properties. The description is exact for a straight beam if the integration along the beam is
done with suﬀicient accuracy.
As the equilibrium-based beam element is based on interpolation of the static field, the defor-
mation along the beam element can not be described in terms of shape functions. As a conse-
quence, mixed element formulations have been used when the equilibrium-based beam element
has been used for dynamic analysis in the literature. In these cases the stiffness matrix from the
equilibrium-based beam element has been combined with a mass matrix derived from other sets
of shape functions. As a part of this project an expression for the deformation along the beam
element is derived and it is shown that parts of this expression can be used as shape functions
to assemble the mass matrix, thereby obtaining a beam element fully based on complementary
energy.
Furthermore, the project investigates if the displacement-based beam element implemented in
HAWC2 can be used in combination with varying properties along the beam element.

displacement-based beam element is based on an approximation of the displacement field with
polynomials of arbitrary order. It is therefore expected that if the deformation of a nonhomo-
geneous beam can be approximated by polynomials with suﬀicient accuracy, then the accuracy
of the beam element will be increased by including the effect of varying properties along the
beam element. In the following, the displacement-based beam element derived using average
properties of the element is referred to as the homogeneous displacement-based beam element.
The displacement-based beam element derived using the varying properties along the element
is referred to as the nonhomogeneous displacement-based beam element.
The equilibrium-based beam element and the two displacement-based beam elements use the
same assumptions, in terms of small deformations, linear elastic anisotropic material properties
and Timoshenko beam theory. This allows for a direct comparison of the accuracy of the three
beam elements and the influence of the elements on the accuracy of a beam model of a blade.
The main objective of the project is to investigate if the accuracy of the currently used beam
model of a blade in HAWC2 can be increased by using beam elements that allow for varying
properties along the elements. Therefore, the analysis is limited to beams in a fixed coordinate
system, without rotational effects. This corresponds to the formulation within a local frame of
reference in HAWC2 and allows for a direct comparison of the accuracy of the elements. The
beam elements are compared for static analysis of the beams in terms of required number of
elements to accurately describe the deformation of a beam subject to distributed loads. Inertial
forces leads to errors, when beam models are used for dynamic analysis. To obtain a better
comparison of the beam elements when used for dynamic analysis, the natural frequencies and
mode shapes are determined by solving the eigenvalue problem.
The analysis in this project is based on an implementation of the homogeneous displacement-
based beam element in a Python code. The code is referred to as TimoBeam and is developed by
the Department of Wind and Energy Systems at DTU. The code models a single wind turbine
blade in a fixed coordinate system and is based on the same input files, description of the
properties and center-line of the beam, as well as the same formulation of the beam element as
HAWC2. The code allows for modelling of curved and pre-twisted blades by discretizing the
beam into a number of straight beam elements. As described in Chapter 5, some small changes
and extensions are implemented in the code.
Two new versions of the TimoBeam code are developed for a direct comparison of the performance
of the beam elements. The nonhomogeneous displacement based beam element is implemented in
the TimoBeamV2 code and the equilibrium-based beam element is implemented in the ComplBeam
code. The ComplBeam model includes a formulation of the mass matrix based on the derived
shape functions from complementary energy and a formulation of the mass matrix based on the
shape functions used in the TimoBeamV2 model.
The thesis includes a self-standing description of the TimoBeam code and the new versions of
the beam code. Thus, all necessary equations and theory are presented in detail to allow a full
introduction to the three codes and relating beam elements.

Equilibrium-Based Beam Element
This chapter presents the equilibrium-based beam element, derived using complementary energy
for an anisotropic linear elastic beam with varying properties along the element. The main idea
of the element is to obtain the elastic stiffness matrix from an integration of the static field in
terms of the section forces in the beam and the cross-sectional flexibility matrix. The section
forces are independent from variations of the shape and properties of the cross-section along the
beam. This allows for an exact description of the potential energy in a beam with variable cross-
sectional properties and thereby an accurate description of the stiffness matrix and distributed
loads on the element.
It is often of interest to describe the kinematic field of a beam element in terms of the deflections
and rotations of the beam between the two nodes. The kinematic field is also needed for the
description of the kinetic energy of the beam element by the mass matrix in the equation of
motion. As the equilibrium-based beam element is based on interpolation of the static field, the
deformation of the beam is only known at the nodes. In the application of the beam element
for dynamic analysis found in the literature, a mixed formulation has been used, where the
kinematic field is described by other sets of shape functions. An alternative description of the
kinematic field is presented as a part of this chapter based on the complementary energy of the
beam. This formulation is computationally heavy when used for dynamic analysis, but can in
theory be used as shape functions of the kinematic field to determine the mass matrix.
The project adopts the formulation of the equilibrium-based beam element originally formulated
by Krenk in 1994 [15] for a beam in 2d. This derivation presents the stiffness matrix relative
to a line connecting the two nodes, allowing for e.g. curved beam elements. Loads acting on
the element are included in terms of the resulting section forces, and a simpler formulation
of the stiffness matrix is presented for straight beam elements. The assembly of the stiffness
matrix for a straight beam element in 3D only subject to nodal loading was presented by Krenk
and Høgsberg in 2006 [18] and applied by Svendsen in 2011 [26] to construct the elastic stiffness
matrix for a wind turbine blade in a rotating frame of reference. The beam element was extended
to include distributed loads for a beam in 3D by Krenk and Couturier in 2017 in [17] and used
to model an anisotropic wind turbine blade.
Other derivations and implementations of the stiffness matrix in an equilibrium-based beam
element, similar to the one presented by Krenk and colleagues, can be found in the literature.
A derivation of the stiffness matrix from complementary energy similar to the one presented by
Krenk in 1994 [15] was described by Ghiringhelli in 1997 [10]. Another, more general formulation
of the stiffness matrix is described by Cook et at. in 2001 [5], where interpolation of the static
field across an element is mixed with interpolation of the kinematic field across the boundaries
of the element.
This formulation is known as a hybrid element and was first presented by
Pian et al. in 1964 [24] for plate elements. The hybrid element formulation is similar to the
equilibrium-based beam element when the method is used on beam elements. The derivation
of the stiffness matrix in the equilibrium-based beam element is also similar to determining the
stiffness matrix by applying Castigliano’s second theorem. This can be seen by comparison with
the stiffness matrix derived by Friedman and Kosmatka for a Bernoulli beam in 1992 [7] and for
a Timoshenko beam in 1993 [8] using Castigliano’s second theorem.
An example of a mixed element formulation, where the stiffness matrix from the equilibrium-
based beam element is combined with other shape functions to assemble the mass matrix is seen
in the element described by Ghiringhelli in 1997 [10]. In this formulation, the axial and torsional

2.3
Stiffness Effects
The elastic stiffness contributes to the equation of motion in terms of the stiffness matrix de-
rived from the potential strain energy. In this section, the stiffness matrix is derived based on
static equilibrium states and complementary virtual work based on the derivation by Krenk and
Couturier in 2017 [17]. The derivation in [17] is based on complementary energy and virtual
work, whereas the stiffness matrix is derived in this project using complementary virtual work.
The two methods are very similar and lead to the same formulation. However, the use of com-
plementary virtual work leads to a slightly more simple derivation. The key concepts of virtual
complementary work and complementary energy are briefly summarised for a general elastic
body in Appendix A based on the theory presented by Krenk in 1989 [14].
2.3.1
Equilibrium Modes
Deformation of the beam element can be described in terms of 12 deformation modes, where the
first six deformation modes correspond to rigid displacements of the beam and the remaining
six modes correspond to deformation of the beam.
As rigid displacements of the beam do
not contribute to the elastic energy, the stiffness matrix is derived based on the six modes
corresponding to deformation of the element, referred to as the six equilibrium modes.
The six equilibrium modes are chosen such that they each correspond to a simple set of end
loads in equilibrium, as illustrated in Figure 2.4. The first three modes correspond to constant
normal force or constant transverse shear in the beam, whereas the last three modes correspond
to constant moment in the beam. The constant shear force in the first two modes leads to a
linearly varying moment.

Kinetic Effects
The analysis is limited to beams in a fixed coordinate system, whereby the only contribution
from kinetics effects to the equation of motion of the beam is the contribution from the kinetic
energy related to deformation of the element. As described in the introduction to this chapter,
the kinematic field is not known for the equilibrium-based beam element. In this section, a
formulation of the kinematic field of the beam element is presented based on complementary
energy. It is then shown that parts of this expression can be used as shape functions to derive
the mass matrix.
It is noted that other shape functions can be used in the assembly of the mass matrix, and that
a mixed formulation is used in the analysis in this thesis. The shape functions derived for the
nonhomogeneous displacement-based beam element in Chapter 3 are used to assemble the mass
matrix in the mixed formulation.

Displacement-Based Beam Element
This chapter presents the homogeneous and the nonhomogeneous displacement-based beam
elements used in this project. The homogeneous displacement-based beam element corresponds
to the beam element used in HAWC2 and the original TimoBeam code.
The beam element
is developed by Kim et al. in 2013 [13] for modelling of wind turbine blades in HAWC2. The
derivation of the stiffness and mass matrix is based on an approximation of the displacement field
using polynomials of arbitrary order of an initially straight beam with linear elastic anisotropic
materials. The main idea in the derivation is to define a part of the coeﬀicients of the polynomials
such that the boundary conditions are satisfied. The remaining coeﬀicients are then determined
by minimization of the strain energy of the element.
The implementation of the displacement-based beam element in the TimoCode is based on av-
erage properties of the elements. This is done by evaluating the integrals along the beam using
average values of the cross-sectional stiffness matrix D(z) = D and the cross-sectional mass
matrix Mcs(z) = Mcs. The implementation of the displacement-based beam element in combi-
nation with average element properties is referred to as the homogeneous displacement-based
beam element in this project.
The integration along the displacement-based beam element can be done using the varying values
of the cross-sectional stiffness matrix D(z) and cross-sectional mass matrix Mcs(z). The more
accurate integration along the beam potentially allows for a more accurate beam element. The
displacement-based beam element is implemented in combination with the varying properties
used for the integration in a new code in this project. The formulation is referred to as the
nonhomogeneous displacement-based beam element to clarify that average element properties
are not used for the integration.
This chapter presents a derivation of the stiffness matrix and shape functions of the homogeneous
and nonhomogeneous displacement-based beam elements, based on the derivation by Kim et al.
in 2013 [13]. The derivation is shown to illustrate the influence when the varying values of the
cross-sectional matrices are used for integration along the beam elements.
The description of the beam kinematics and constitutive relations correspond to the description
shown for the equilibrium-based beam element shown in Section 2.1 and 2.2. Thus, only the
derivation of the stiffness matrix, shape functions, and distributed loads are presented in this
chapter. When the shape functions have been derived, the element mass matrix can be deter-
mined using the same method as for the equilibrium-based beam element by replacing the shape
functions in Section 2.5.2.

Blade Finite Element Formulation
The purpose of this chapter is to present the assembly of a general finite element model of a
wind turbine blade in a global coordinate system, using the beam elements presented in Chapter
2 and 3. This includes the necessary equations for the analysis of the model.
Wind turbine blades can be curved and pre-twisted in the undeformed state. This is taken into
account in the formulation of the finite element model by discretization of the curved beam into
a number of straight beam elements. The elements are described in a local coordinate system,
whereby a transformation from the local element coordinate system to the global coordinate
system is required.
In the original TimoBeam code with the homogeneous displacement-based beam element, the
transformation from the element coordinate system to the main coordinate system is done for
the element stiffness and mass matrices based on the average twist of the element. An alter-
native transformation is introduced for the non-homogeneous beams. In this formulation, the
transformation related to pre-twist of the beam is done separately for the cross-sectional matri-
ces while the remaining transformation to the main coordinate system is done for the element
stiffness and mass matrices. The separate transformation of the cross-sectional matrices allows
for varying pre-twist along the elements and thereby leads to a more accurate representation of
beam elements with varying twist.
First, the main coordinate system is introduced and the change of basis is described in general
terms. Then, the two methods for assembly of the elements are presented. Finally the formula-
tion of the equation of motion and the needed equations for static and dynamic analysis of the
model are presented along with a presentation of the used integration methods along the beam
elements.

Program Structure and Model
Implementation
Three finite element beam models have been implemented in Python for a detailed analysis of the
three-beam elements described in Chapter 2 and 3. This includes modifications of the original
version of the TimoBeam code developed by the Department of Wind and Energy Systems at
DTU. The codes are developed with a similar structure to allow for comparisons of the influence
of the used beam elements on the accuracy of the finite element models. This chapter introduces
the three beam models and presents an overall description of the structure of the code and the
input files.
An important aspect of modelling wind turbine blades with complex geometry and varying
properties is the description of the geometry and properties in the model. Therefore, the de-
scription of the code includes a detailed description of the modelling of the beam geometry
and the interpolation of the cross-sectional properties along the beam. This description also in-
cludes a description of how the location and orientation of the beam elements is defined. These
descriptions are intended to serve as a general reference of the modelling in the TimoBeam code.

5.1
Model Characteristics
The following beam models have been implemented in Python based on the original version of
the TimoBeam code developed by the Department of Wind and Energy Systems at DTU.
• TimoBeam: Based on the original TimoBeam code with the homogeneous displacement-based
beam element described in Chapter 3. As a part of this project, the code has been extended
to include distributed loads and static analysis. The calculation of the distance from the
elastic center to the shear and mass center has been updated to include a change of basis
from the c2-coordinate system (introduced later in this section) to the element coordinate
system. It is noted that this correction does not correspond to the implementation in
HAWC2, but the change is implemented as it is more consistent with the definition of the
cross-section centers in the HAWC2 input files. It has been verified for the case study in
Chapter 7, that the model gives the same stiffness and mass matrix as HAWC2 when the
correction is not implemented. Additional smaller changes has been made to the structure
and format of the code.
• TimoBeamV2: New version of the TimoBeam code with implementation of the nonhomo-
geneous displacement-based beam element described in Chapter 3. The nonhomogeneous
displacement-based beam element allows for varying properties in the integration along the
elements. The change of basis from the element coordinate system to the main coordinate
system has been updated to allow for varying twist as described in Section 4.3. The code
is extended to allow use of Gauss quadrature or the trapezoidal rule for integration along
the beam elements based on the user input.
• ComplBeam: New code based on the equilibrium-based beam element in Chapter 2. The
code is similar to the TimoBeamV2 code, except for the calculation of the element matrices,
distributed loads and displacement field along each beam element. The code allows for the
mass matrix to being determined from the displacement field using the complementary
energy formulation or by using the mass matrix in the TimoBeamV2 model. The consistent
nodal loads from distributed loads and the displacement field along each beam element
are based on the section forces in the beam element, as described in 2.
Each finite element model is done as a class in Python and mainly contained in a single file.
The code have been developed in Python 3.8.8 and evaluated using Spyder IDE (5.1.5). The
program files are attached as an appendix to the report including an example of the input files.
The files can also be found at: https://gitlab.com/JensRichardt/master-thesis-beam-models.git.

The orientation of the element coordinate system is defined differently for the TimoBeam model
compared to the TimoBeamV2 and ComplBeam models. The TimoBeam model uses the average

angle between the ˆx-axis and the Xz-plane.
This corresponds to the average twist of the
element. The average values of the element are then calculated from 100 evenly spaced points
along the element. The location of the 100 points is defined for each element in the function
calculateNonDimSegments. The average values of the cross-sectional matrices are calculated
in the function calculateElementProperties. The assembly and orientation of the elements
in the TimoBeam model are described in Section 4.3.1.
In the TimoBeamV2 and ComplBeam model, the angle θelem = 0. The actual cross-section matrices
are used along each beam element. As described in in Section 4.3.2, the assembly allows for
varying values of the angle θ in (5.1) along the beam elements. This leads to a better represen-
tation of beams with pre-twist. The cross-section stiffness matrix D(z) and mass matrix Mcs
are calculated with respect to the element coordinate system without twist along each beam
element in the function calculateCrossSectionMatrices at the integration points along the
elements. The location of the integration points and the corresponding weights are defined in
the function calculateIntPoints. The function allows for both Gaussian-Quadrature and the
use of the trapezoidal rule.
The properties of the beam, including the cross-sectional stiffness matrix and cross-sectional mass
matrix, are defined with respect to the elastic center along the beam. As the beam elements are
linear, deviations between the location of the actual elastic center and the reference line of the
beam element can occur. A similar deviation can occur when the orientation of the plane where
the cross-sectional properties are defined and the orientation of the plane made by the x and
y-axis in the element coordinate system do not coincide. These deviations are neglected in all
three beam models. As the beam elements do not include the effects of curved beam elements,
the discretization of the beam has to be suﬀiciently fine to model curved beams, whereby the
error from the deviation of the reference axis of the beam element from the elastic center is
expected to be small.
5.2.4
Model Assembly
The element stiffness matrix K and mass matrix M are determined for all three models in the
element coordinate system in the function calculateElementMatrix. In TimoBeam, Gauss inte-
gration is always used. The TimoBeamV2 and ComplBeam models allow for both Gauss integration
and integration with the trapezoidal rule. The location and weights of the integration points is
determined in the function calculateIntPoints.
The TimoBeam model is based on the homogeneous displacement-based beam element with con-
stant properties along the elements. The analytical deflection of a cantilever beam subject to a
point load at the tip can be described analytically with third-order polynomials. Therefore, the
maximum power Np.max of the interpolating polynomials can be chosen as 3 for the TimoBeam
model. The integration of the strain-displacement matrix B(z) and the cross-section stiffness
matrix D in (3.13) can therefore be done accurately using four Gauss points. The maximum
power of the polynomials is verified in Section 6.1.3, later in the report.
The function calculateGeneralMatrix assembles the global stiffness matrix K and mass matrix
M. The assembly of the elements is described in Section 4.3. The nodal loads are determined
from the functions shown under ”Static Loads” in Figure 5.1. The boundary conditions are
applied in the function applyBoundary by eliminating the constrained d.o.f. from the stiffness
matrix, mass matrix and load vector.

Verification of Implementations
The purpose of the present chapter is to verify the implementation of the equilibrium-based
beam element and the two displacement-based beam elements. Another purpose is to compare
the accuracy of the three beam elements and the influence on the accuracy of the beam models.
This is done by considering several simple test cases that allow a separate analysis of different
factors affecting the accuracy of the beam models. The verification is done separately for static
and dynamic analysis in the following sections.
The displacement-based beam element used in the TimoBeam model has been verified for an-
isotropic materials as a part of the derivation by Kim et al. in 2013 in [13]. Similarly, the
equilibrium-based element has been tested by Krenk & Svendsen in 2017 in [17]. Thus, the
verification in this chapter is focused on the comparison of the accuracy of the three beam
models when a few elements are used, as well as new additions to the models.
For the static analysis, the implementation of distributed element loads for the three beam
models is verified first. Then, a tapered beam is considered to investigate the difference in the
accuracy of the elements for a straight beam with a varying cross-section. Then, the influence of
the maximum order of the interpolating polynomials on the accuracy of the two displacement-
based beam elements is tested. Next, the influence of linear interpolation of material properties is
tested to investigate if this creates an increased error for the models with nonhomogeneous beam
elements. Next, the effect of the updated assembly of the beam elements allowing for a better
description of pre-twist is investigated. This is done by considering a pre-twisted beam. Finally,
the expression for the deformation along the equilibrium-based beam elements is investigated
for a beam with distributed loads.
For the dynamic analysis, the natural frequencies are determined for a simply supported beam
with a constant cross-section. Next, a tapered cantilever beam is considered to test the influence
of the beam elements on the accuracy of the beam models when the properties of the beam are
varying.

could be that a numerical error is introduced when the strain energy is minimized.

model when 20 elements are used and three when a single element is used. This corresponds with
the expected maximum value of Np.max = 3 for the TimoBeam model as it is based on average
element properties, in which case the analytical deflection is described by polynomials with a
maximum power of 3 in case of nodal loads. There is no general optimal value for Np.max in the
TimoBeamV2 model as the optimal value of Np.max is seen to depend on both the geometry of
the beam and the number of elements. However the three cases show a general tendency, that
the optimal value of Np.max decreases as the number of elements increases.
Overall this example shows that for the TimoBeam model, the optimal value of Np.max corresponds
to the standard value of 3 in the model. There is no clear optimal value of Np.max for the
TimoBeamV2. In the following analysis, Np.max is chosen as 5 for the TimoBeamV2 model, as there
is a risk that large errors are introduced if a too large value is chosen. However, in some cases
the error is still significantly smaller when Np.max = 5, compared to the case where Np.max = 3.

Case study: 15MW Wind Turbine Blade
A case study is performed of the blade of the IEA Wind 15-MW Offshore Reference Wind Turbine
developed by the National Renewable Energy Laboratory (NREL), the Technical University of
Denmark (DTU) and the University of Maine in 2020. The reference turbine describes a 15-MW
offshore wind turbine with a rotor diameter of 240 m. A detailed description of the reference
turbine is found in the report [9].
The blade of the 15-MW reference turbine is particularly useful for a case study to test the
implementation of the beam elements, as the blade has a complex geometry with varying prop-
erties, coupling effects, pre-twist and curvature. Furthermore, the model is publicly available,
including HAWC2 input files for modelling the blade using beam elements. This allows the input
files to be used with a minimal number of corrections for the beam models in this project.
In the following, the geometry and properties of the blade are introduced. Then a static anal-
ysis is done for the curved blade by calculating the deformation at the tip and investigating
the required number of elements for an accurate model. Next, the input files are modified to
represent a similar straight blade to test the influence on the performance of the beam models
when errors from the curvature of the blade are neglected. Finally, a dynamic analysis is done
to test the influence of the accuracy of the models when inertial forces are taken into account.

7.3.2
Convergence of Natural Frequencies
A convergence study is done for the blade to investigate the accuracy of the three models with
a limited number of elements. The convergence study is done with Nelem + 1 equally spaced
nodes along the curved line describing the half-chord location of the cross-section by the Akima
interpolation in Figure 7.1. The interpolation of the aerodynamic twist is defined directly in
the code based on the curved distance along the blade. This is done to obtain a more accurate
interpolation of the twist when few elements are used.
Two versions of the ComplBeam model are included in the analysis. ComplBeam(A) corresponds
to the case where the stiffness matrix of the equilibrium-based beam element is combined with
the mass matrix of the nonhomogeneous displacement-based beam element. This corresponds to
the used mass matrix in TimoBeamV2 and thereby allow for a direct comparison of the stiffness
terms in the models. The formulation of the mass matrix presented in Chapter 2 based on
complementary energy is included in the model ComplBeam(B). The TimoBeam and TimoBeamV2
models are also included in the analysis.
A correction of the calculation of the average mass m per unit length of the homogeneous
displacement-based beam element is included in HAWC2 and TimoBeam. This correction adjusts
the mass by the expression shown in (5.7), leading to the new mass meigen. The correction is not
considered a part of this project. However, a version of the TimoBeam model with the correction
of the mass is included in the following convergence plots to show that the increased accuracy
from use of the nonhomogeneous beam elements is larger than the influence of the correction of
the mass.
Figure 7.10 shows the absolute relative error for the natural frequency of the first six modes
with respect to the TimoBeam model with 200 equally spaced elements. The number of modes is
chosen to include the first torsional vibration mode. The integration of the beam models that
allow for varying element properties is done using the trapezoidal rule with 1001 integration
points when up to 10 elements are used and 201 integration points when more than 10 elements
are used.
It is seen in Figure 7.10, that the two ComplBeam models and the TimoBeamV2 model have a
significantly larger accuracy for the first five vibration modes compared to the TimoBeam model.
This corresponds to the modes dominated by transverse vibrations. The difference in accuracy
between the ComplBeam(A), ComplBeam(B) and TimoBeamV2 models compared to the TimoBeam
model is seen to decrease as the mode number increases. For mode 6, corresponding to the first
torsional mode, all the models give a more similar relative error. This shows a general tendency
for the higher modes not included in the figure, that the modes with a significant contribution
from twist θz seem to give similar errors for all the models.
The ComplBeam(A), ComplBeam(B) and the TimoBeamV2 model have a similar accuracy after
approximately 10 elements. The large variation when less than 10 elements are used is expected
to be from an insuﬀicient discretization of the blade to capture the mode shapes. Furthermore,
as the blade is curved and the beam elements are linear, then the geometry of the assembled
model changes when the number of elements is changed. This effect is particularly large when
few elements are used.
As the ComplBeam(A) and ComplBeam(B) models have similar errors, it seems that the imple-
mented mass matrix in Chapter 2 can be used to describe the kinetic energy of the wind turbine
blade. The ComplBeam(A) model seems to give slightly smaller errors when less than 10 elements
are used compared to the TimoBeamV2 model and the ComplBeam(B) model. However, the effect
is limited.

Discussion, Conclusion and Further Studies
The examples shown for verification in Chapter 6 as well as the case study in Chapter 7 have to
a large extent shown the capabilities of the equilibrium-based beam element and the nonhomo-
geneous displacement-based beam element compared to the homogeneous displacement-based
beam element. The applicability of the beam elements in an aeroelastic code such as HAWC2 is
discussed in this chapter and general conclusions are presented for the project. Finally, potential
topics for further study of the accuracy and eﬀiciency of the beam elements are presented.
8.1
Applicability in Aeroelastic Code
Three separate programs have been implemented in python, including the small modifications
of the original TimoBeam code.
The similar code structure in the ComplBeam code and the
TimoBeamV2 code, allow for a direct comparison between the beam elements. The usefulness of
the beam elements implemented in the ComplBeam code and the TimoBeamV2 code is discussed
with respect to potential implementation in HAWC2 or similar codes involving dynamic analysis
in the time domain.
8.1.1
Representation of Loads
The representation of varying loads on beams with varying properties has been tested by static
analysis of a number of beams.
The equilibrium-based beam element implemented in the
ComplBeam code allows for an, in theory, exact representation of varying cross-sectional prop-
erties and loads for static analysis of initially straight beams using a single element. This is
illustrated in the test of the tapered beam in Section 6.1.2 and for the artificially straight wind
turbine blade in Section 7.2.2. In this case, the accuracy of the equilibrium-based beam element
is significantly better than for the two displacement-based beam elements. The accuracy of
the nonhomogeneous displacement-based beam element is significantly better compared to the
homogeneous displacement-based beam element.
The test of the pre-twisted beam in Section 6.1.5 shows that the updated formulation of the
change of basis in the assembly of the elements in Section 4.3.2 allows the ComplBeam code to
represent pre-twisted beams using a single element with the equilibrium-based element. This is
a significant reduction in the required number of elements compared to the TimoBeam code and
a small reduction compared to the TimoBeamV2 code.
For the curved blade in the 15 MW reference turbine, it is shown in Section 7.2.1 that the error
from the curvature becomes dominating for the equilibrium-based beam element and that more
than a single element is needed for an accurate representation of the deflection of the blade. In
this case, the accuracy of the equilibrium-based beam element corresponds to the accuracy of
the nonhomogeneous displacement-based beam element, whereas the error is significantly larger
for the homogeneous displacement-based beam element in the TimoBeam code.
8.1.2
Displacement Field
A current limitation of the equilibrium-based element is that it does not include any interpolation
of the deformation along the elements. This problem has been reduced by the new expression for
the displacement of the equilibrium-based beam element in (2.50). The two displacement-based
beam elements allow a direct interpolation of the displacement by shape functions. Thereby,
the displacement along the beam element is still a limitation for the computational eﬀiciency of
the equilibrium-based beam element, as the integrals in (2.50) must be evaluated every time the
load is updated.

8.1.3
Influence of Inertial Forces
The influence of the inertial forces has been tested by calculating the natural frequencies of a
tapered beam in Section 6.2.2 as well as the natural frequencies of the curved wind turbine
blade in Section 7.3.2.
The expression for the displacement of the equilibrium-based beam
element without distributed loads in (2.53) can be used to assemble the mass matrix, thereby
avoiding a mixed element formulation with other sets of shape functions. As seen in Sections
6.2.2 and 7.3.2, this mass matrix gives a similar accuracy for the natural frequencies as the
mixed formulation of the equilibrium-based beam element where the shape functions from the
nonhomogeneous displacement-based beam element are used to assemble the mass matrix. The
two implementations of the equilibrium-based element show the same tendency for dynamic
analysis as for the static analysis of the curved blade. The accuracy is significantly increased
compared to the homogeneous displacement-based beam element, whereas the accuracy is the
same as for the nonhomogeneous displacement-based beam element.
8.1.4
Integration and Interpolation of Properties
The two nonhomogeneous beam elements require a more accurate integration along the beam
elements compared to the homogeneous beam element. This can be illustrated by considering
a beam with a sudden local change in geometry or loads. As shown for the ComplBeam Code
in Section 6.1.1, the integration can miss local changes in the load or properties of a beam
if the integration points are not located at the local change. Therefore, integration with the
trapezoidal rule has been shown to be better than Gauss quadrature for the nonhomogeneous
beam elements.
Integration with the trapezoidal rule leads to larger computational costs in the assembly of the
models compared to Gauss quadrature with few elements. However, as the large computational
cost in time history analysis is in the time integration and not in the assembly of the model, the
larger computational time for assembly is not seen as a problem.
As shown in the example in section 6.1.4, the linear interpolation of the properties of the beam
in the TimoBeam, TimoBeamV2 and ComplBeam code can lead to significant errors when the points
where the properties are defined are not closely spaced. However, the test of the tapered beam
indicates that the error is not larger for the two nonhomogeneous beam elements compared to
the homogeneous beam element.
8.1.5
Overall Comparison
Overall, the equilibrium-based beam element and the nonhomogeneous displacement-based beam
element have shown to be well suited for modelling of beams with varying properties with a
significant increase in the accuracy compared to the homogeneous displacement-based beam
element. The project shows a large potential for implementation of beam elements that allow
for varying cross-section properties in HAWC2.
As the intended use of the beam elements
is for dynamic analysis, the nonhomogeneous displacement-based beam element seems like a
better choice for HAWC2 compared to the equilibrium-based beam element. This is due to
the large similarity to the currently implemented beam element, the direct interpolation of the
displacement and the simpler formulation of the mass matrix. However, in particular, for static
analysis of straight beams, the equilibrium based beam element has shown to be very useful as
it can provide an exact solution.

8.2
Conclusion
The main objective of this thesis was to investigate if the accuracy of a model of a wind turbine
blade in TimoBeam, and HAWC2, can be increased by using beam elements that include the effects
of varying properties along wind turbine blades. Implementation of the equilibrium-based beam
element and the nonhomogeneous displacement-based beam element in Python code similar to
the TimoBeam code, allows for a direct comparison of the influence of the beam elements. The
descriptions of the modelling of the geometry and properties of a beam in TimoBeam and of the
used beam elements, allow for a common source of reference for the code.
By updating the assembly of the elements in the TimoBeam code, the equilibrium-based beam
element and the nonhomogeneous displacement-based beam element have been extended to
include a more accurate description of varying twist along the elements.
The development
of an expression for the displacement of the equilibrium-based beam element allows for an
implementation of the beam element fully based on complementary energy and a representation
of the deformation along the beam element without using shape functions that do not correspond
with the stiffness matrix.
Verification examples and a case study of a wind turbine blade have shown that the two beam
elements based on varying element properties can describe beams with varying geometry and
coupling effects with large accuracy. The equilibrium-based beam element and the nonhomo-
geneous displacement-based beam element have proven to be reliable and potentially increase
the accuracy by an order of magnitude compared to the implemented beam element with con-
stant element properties in the TimoBeam code. Both beam elements with varying properties
have proven to have a large potential for implementation in HAWC2, as the increased accuracy
allows for a reduction in the number of elements and thereby an increased eﬀiciency for analysis
in the time domain.
While the equilibrium-based beam element is exact for static analysis of straight beams, the
nonhomogeneous displacement-based beam element has proven to have a similar accuracy when
used to determine the natural frequencies of a beam with varying properties and static analysis
of the curved wind turbine blade. Based on the similar accuracy when used for dynamic anal-
ysis, a simpler interpolation of the displacement field, and the large similarity to the existing
implementation of the beam element in HAWC2, the nonhomogeneous displacement-based beam
element is estimated to be the best choice for implementation in HAWC2.

8.3
Further Work
The following topics could serve as a potential addition to this project.
8.3.1
Comparison to 3D-Models
Comparison to analytical solutions has only been possible for simple test cases in this project.
For a more accurate validation of the modelling of more complex beams, such as the twisted
beam considered in Section 6.1.5, comparison with 3D finite element models or experiments is
required.
8.3.2
Geometric Stiffness and Rotational Effects
Rotational effects and centrifugal stiffening are expected to influence the accuracy of a beam
model of a rotating wind turbine blade. This could reduce the increase in the accuracy of a
beam model obtained by using nonhomogeneous beam elements compared to using homoge-
neous beam elements. The present project is limited to the analysis of beams subject to small
deformations and rotations in a stationary frame of reference. This means that the effect of
centrifugal stiffening (represented by the geometric stiffness matrix) and rotational effects can
not be investigated. A relevant extension of the project could therefore be to model a blade in a
rotating frame of reference or in a multibody formulation, and include the rotational effects and
centrifugal stiffening. This would allow for tests of the influence on the accuracy of the beam
model when either homogeneous beam elements or nonhomogeneous beam elements are used.
8.3.3
Time History Analysis
The current beam programs are limited to static analysis and determining the natural frequencies
and mode shapes by solving the classical eigenvalue problem. As the intended use of the beam
elements is for analysis in the time domain, a relevant extension could be to include time history
analysis with stochastic loading. This would allow for studies of the reduction in the calculation
time by using more accurate beam elements allowing for a reduced number of elements.
The integrals in the expression derived in Section 2.5.1 for the displacement field of the equilibrium-
based beam element depend on the distributed loads on the element. Another relevant test with
time history analysis would therefore be to investigate the influence on the total computational
cost when the displacement field is calculated at every time step. Thereby showing if the ex-
pression is of practical use in combination with dynamic analysis in the time domain.
