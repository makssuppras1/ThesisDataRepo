Abstract
This Thesis investigates how the Clustered Traveling Salesman Path Problem with Sub-
clusters can be solved heuristically using an approach that reduces the problem complex-
ity by solving each subcluster as a separate subproblem. Methods for computing lower
bounds of different computational complexity are presented using Minimum Spanning
Trees and the Open Traveling Salesman Problem, of which the latter creates a tighter
bound at the cost of higher computation time. The upper bounds are evaluated using
different approaches for candidate selection for the intersubcluster edges, with the most
promising approach being based on a reclustering of subcluster vertices that solve a sim-
plified version of the complete problem. Following an initial route generation based on
the presented candidates, an improvement heuristic is presented called the Subcluster-
2-opt in which the intersubcluster edges are updated and the connected subclusters are
reoptimized. The improvement heuristics show continuous improvement, but are com-
putationally expensive. A case study is presented for a postal route on Bornholm with
28,010 vertices which are divided into 5 clusters and 100 subclusters. The reclustering
approach generates an initial solution with a gap of 4.35% compared to the lower bound
in 2 hours, and has the benefit of parallelization which allows the generated subprob-
lems to be solved simultaneously. The generated solutions are compared to a state of
the art Traveling Salesman Problem heuristic Lin-Kernighan-Helsgaun, which generates
a solution with a gap of 2.78 % in 2 hours.

CHAPTER 1
Problem Introduction
In this Thesis the problem of large scale instances of the Clustered Traveling Salesman
Path Problem (CTSPP) with Subclusters is investigated. The problem is a variation on
the traditional Traveling Salesman Problem (TSP), with the extension that all vertices
belong to both a cluster and a subcluster. The requirement then is that all vertices
within a cluster and subcluster are visited consecutively. Additionally, the solution can
be a Hamiltonian path instead of a Hamiltonian cycle. The TSP is notoriously diﬀicult
to solve for large scale instances and the same goes for the CTSPP. In this Thesis
solution strategies are presented that deconstruct the problem into smaller subproblems
that can be solved to optimallity.
The problem occurs in real life situations seen in both the delivery and pick-up indus-
tries. The grouping of the vertices into clusters and subclusters ensures that the route
finalizes a neighborhood before moving on to the next. The subclusters can represent
these neighborhoods while the larger clusters can represent the set of customers that
receive their packages on a given day of the week. The different subclusters for a given
cluster can then be divided between the postmen.
This system allows for the same
postmen to serve the same area every week, which in turn increases postmen eﬀiciency
[ZHD07]. An additional benefit of this is, that the complexity of keychain management
decreases, as many apartment blocks require keys to enter the buildings.
The clusters and subclusters can be generated using a large selection of different ap-
proaches. Districting approaches can be a great way to offer highly specialized clusters
and subclusters for the task at hand. An overview of different districting approaches is
presented in Section 2.5.1. This Thesis will focus on solving the resulting routing prob-
lem for a fixed set of clusters, and therefore the clusters are generated using traditional
clustering techniques.
A mathematical formulation of the routing problem is presented in Chapter 2 which
is able to solve the problem to optimallity for small instances. Chapter 2 additionally
contains an overview of the available literature on the different variations of the TSP.
In Chapter 3 different approaches of varying computational complexity are presented
to generate a lower bound for the CTSPP. The approaches are based on the Minimum
Spanning Tree (MST) and the Open Traveling Salesman Problem (OTSP). Chapter 4
presents solution strategies to the problem. The solution strategies are based on a de-
construction of the complete problem into a set of smaller subproblems. The generation

work that can be done to further improve the performance of the proposed solution
strategies. Finally, the conclusion is presented in Chapter 7.

CHAPTER 2
Mathematical Models
Mathematical Optimization and Operations Research is the study of how systems can
be formulated mathematically and be arranged in an optimal manner to help decision-
making. One of the key elements in the toolkit for Operations Researchers is Linear
Programming (LP), which formulates a given problem as an objective function and a
set of constraints. This objective function will then either be maximized or minimized
subject to the constraints. One of the assumptions for LP is that the problem can be
modeled using linear and continuous functions. For problems that follow these require-
ments the simplex method can be applied. The simplex method was developed in 1947
by George Dantzig and researchers have since attempted to solve combinatorial prob-
lems of countless shapes. These attempts have resulted in both vast improvements and
cost reductions. Moreover, as computers have become more and more powerful, so have
the sizes of the problems being solved with these approaches. A great benefit of LP
is that the optimal solution will always be at an extreme point, a corner point, which
drastically reduces the amount of possible solutions that can be optimal.
Not all problems in Operations Research can be modeled as a LP, which results in
some problems being substantially harder to solve than others, despite being of similar
sizes. For problems that require integer values, alternative sets of algorithms other than
the simplex method should be applied, as the optimal solution is no longer required to be
at an extreme point. These problems commonly occur during production planning where
the number of units produced have to be integer and a binary variable can represent
whether or not a machine is in use. The following terminology is used when the set
of problems contain decision variables that are not discrete; Integer Programming (IP),
Mixed Integer Programming (MIP) or Mixed Integer Linear Programming (MILP).
2.1
Traveling Salesman Problem

studied problems. The classical TSP is defined as finding the optimal Hamiltonian cycle

city in a certain order will be discrete. Mathematically, this problem can be described

additional requirements. In the Clustered Traveling Salesman Problem (CTSP) variation
the vertices are split into a set of mV clusters. The vertices for each cluster are required

clusters. A requirement for this constraint is, that the vertices V are separated into mV
disjoint clusters V1, V 2, .., VmV such that V1 ∪V2 ∪.. ∪VmV = V .

2.5
Clustering approaches
The previous mathematical models require a predefined set of clusters as input, which
is not given in the dataset analyzed in this project. This in turn generates an additional
challenge in terms of partitioning the data into these clusters.
Clustering algorithms aims to group data according to a set of similarity measures
depending on which algorithm is applied. There are many approaches on solving a clus-
tering problem, each generating different solutions depending on which algorithm and
parameters are chosen to generate the clusters. Within Machine Learning clustering
often comes up as a classification problem that aims to identify data points that appear
similar on the measured metrics. An example of this is image identification, that can
identify a picture of a cat from a picture of a balloon.
The clustering problem concerning this task is a bit different, as the classification of
a node to be in a certain cluster is neither correct nor incorrect, as opposed to what it
would be in the case of identifying a cat as a balloon. Instead, the clusters represent
sections of a route, that can be done in a certain time-frame, such that a given neighbor-
hood will be finalized in one segment of route, as opposed to the TSP solution that might
come back to revisit a neighborhood at a later point. The latter only being the case
providing that it reduces the total travel distance, which in turn proposes challenges for
delivery and pickup companies that would like to finalize neighborhoods before moving
to the next.

2.5.1
Districting
Districting is a well studied problem which is divided into four main areas in the liter-
ature: service districting, political districting, sales territory districting and distribution
districting. Service districting is focused on researching the location of public services
such as fire stations and hospitals. Political districting studies the division of regions into
voting districts. Sales territory districting is focused on the allocation of market areas
to a sales force such that each member of the sales force has an equal market potential
for sales. Distribution districting is the study of delivery areas that are traditionally
modeled as a VRP. The problem faced in this project has many similarities with distri-
bution districting, the main difference being having to minimize one long route instead
of one route for every district.
[KR19] presents a thorough overview of the different areas of districting found in the
literature, including solution approaches and evaluation criteria. The paper describes
three typical criteria seen in districting problems: balance, contiguity and compactness.
Balance is a criteria keeping track of the sizes of the different districts. This can be in
the form a number of basic units, which, in the case of this project, will be households.
Contiguity ensures that it is possible to travel to basic units within the same district
without leaving the district. Compactness concerns a more visual aspect, ensuring that
the districts are undistorted and round-shaped.
Additionally, compactness can help
to ensure that the different districts contain equivalent travel distances. These travel
distances can be evaluated in several ways, and can include activity measures, such as
time required to make a drop-off. [BJ09] describes different approaches to large-scale
clustering based on sequential aggregation and grid search procedures.
[KR19] argues for the need of a common and generic framework for districting prob-

2.5.2
K-Means clustering
One of the most commonly applied clustering approaches is known as K-means cluster-
ing. The term K-means was first introduced in 1967 by J. MacQueen [Mac67], with the
Lloyd-Forgy algorithm being the classical implementation that is most widely used today
[For65] [Llo82]. One of the main benefits of clustering approach is that the algorithm
produces a fixed number of clusters that can be predefined as an input variable. The
algorithm is an iterative process with random start points, and there is no guarantee
that two different runs will generate the same clusters.
The algorithm works by initializing K random centroids and then allocating all ob-
servations to the closest centroid. The locations of the centroids are then updated to be
at the average of the observations allocated to that cluster, and next all the observations
are allocated to the closest centroid once again. This process repeats untill all observa-
tions are added to the same cluster in every iteration, or a certain number of iterations
is reached. For a more thorough walkthrough of the K-means clustering approach see
[Sre22].
K-means clustering has been chosen as the clustering algorithm for this project due
to its simplicity and its ability to control the amount of clusters generated. Having a
tool that controls the number of clusters is key for this project, however the algorithms
proposed in this project has no assumptions on the previous cluster generation.

Lower Bounds
Within optimization and operations research, Upper Bound (UB) and Lower Bound (LB)
are used to find the range in which the optimal solution value z∗to a problem can be
found. By increasing the lower bound and decreasing the upper bound, the feasible
region of a problem decreases. This reduced feasible region can then be explored using
a linear relaxation of the problem, which can help tighten the bound for the optimal
solution further. Algorithms such as branch-and-cut similarly make use of this decreased
feasible region to prune solution nodes that, in a minimization problem, have a higher
upper bound than an existing lower bound. For a maximization problem, the roles of
the lower and upper bound are switched.
UB ≥z∗≥LB
(3.1)
When generating heuristic solutions to problems, bounds can serve a different pur-
pose. The task researched in this project is a minimization problem to a very complex
problem for large instances, hence it will not always be possible to generate an optimal
solution. A lower bound can therefore serve the purpose of a measurement tool in terms
of how good a solution is compared to a bound that ensures that no feasible solution
better than the current one exists.
There can be advantages to having several approaches to finding these lower and
upper bounds as they can serve different purposes in terms of time and quality.
In this chapter, lower bounds will be generated through finding optimal solutions
to subsets of the complete problem. The problem will be split into two segments: In-
tersubcluster and Intrasubcluster edges. This way, a lower bound can be generated for
these subproblems, and the sum of the two will generate a lower bound for the complete
problem.

4.1
Lin-Kernighan-Helsgaun Heuristic
Lin-Kernighan-Helsgaun is an advanced heuristic specialized for solving different vari-
ations of the TSP and is commonly referred to as state of the art for TSP heuristics
[Hel09]. The LKH algorithm is a variant of the Lin-Kernighan (LK) algorithm, which
was first proposed in 1973 by Brian Kernighan and Shen Lin [LK73]. The LK algo-
rithm is based on a combination of local search and constructive heuristics to produce
high-quality solutions. The algorithm starts with a random initial solution and then
iteratively improves it by performing a series of k-opt moves to swap edges while pri-
oritizing smaller values of k. The LKH is based on the same idea of k-opt moves, but
these k-opt moves can now take any value of k greater than 2 and less than the number
of vertices. This adaptation along with clever problem partitioning and tour merging
results in the LKH algorithm being extremely effective.
One of the main advantages of the LKH algorithm is its ability to quickly find near-
optimal solutions for large-scale TSPs. Despite being a heuristic, the LKH has a high
frequency of finding optimal solutions. It is able to solve TSP instances with up to tens
of thousands of cities given enough computation time, and it has been used to solve
some of the largest TSP instances known. The LKH currently holds the record for the
best known solution on a number of extremely large TSP’s, including the World TSP,
which is a 1,904,711-city instance of locations throughout the world [Hel].
Due to its ability to quickly find high-quality solutions for large-scale optimization
problems, LKH is one of the most popular TSP solvers in use today.

proposed solution structures perform, but also to solve certain subproblems of the TSP

4.2.2
Generating Candidate lists
The first step in generating the graph is identifying a set of candidates for intercluster
connections for each of the subclusters. There are different approaches to identifying
a candidate that each have their own advantages and disadvantages. One of the main
problems concerning the task of candidate identification is, that as the number of can-
didates increases, so does the number of subproblems that have to be solved. Since a
TSPP has to be solved for every pair of intrasubcluster edges, an approach can be to
keep the number of candidates to the minimum.
A naive approach to candidate identification can be to look at the distance of each of
the vertices to the center of the following subcluster given by the top level route. This
naive approach minimizes the number of sub-problems generated, and can be a quick
way to construct an initial solution to the problem. In some instances, a vertex may
be the closest vertex to both the previous and the following subcluster centers. A node
cannot be both an entry and exit node, as this would create a cycle instead of a graph.
One way to avoid this is, to keep track of identified candidates and for the following
subcluster to take the second closest vertex.
An alternative, but similar approach can be to look at the closets k vertices to the
following subcluster. This approach is a good way to also limit the graph size, but has
a tendency to identify candidates in the same location, leading to solutions that do not
differ by a lot. This might be ideal for some subcluster structures in which a single area
contains the potential intersubcluster edges, however as a consequence some subcluster
structures might not be identify all good candidates with this approach.
Figure 4.2 and 4.3 visualize the candidate identification for a set of fixed clusters and
a fixed cluster route. The identified candidate nodes are visualized as stars, subcluster
centers as diamonds, while the cluster route is shown by the edges. The k closest ver-
tices to the following cluster are then identified as candidates. For the case of this test
instance, the approach successfully identifies the majority of the intuitive candidates
based on a visual inspection of the clusters. For larger instances, this approach may
lead to a lack of a diverse identification of vertices, and only recognize vertices within a
small neighborhood.

4.2.4
Concorde TSP Solver for TSPP
The optimal solution given by the Dijkstra’s Algorithm is only as good as the input that
the algorithm is given. Since the algorithm is fed with heuristic solutions to the sub-
problems, the actual costs will be different if the subproblems are solved to optimallity.
The state of the art TSP solver Concorde will be applied to solve the s −t TSPP to
optimallity using the fixed entry and exit points given by Dijkstra’s algorithm. A re-
quirement for the Concorde solver is that the problem instance is symmetrical which is
not the case for the OTSP with fixed start vertex, and hence the LKH will be used to
solve these problems.
Concorde is commonly referred to as the best exact solver for the TSP [HS14], and
has solved instances of up to 85,900 vertices [App+09]. Although Concorde has suc-
cessfully solved very large instances of the TSP, the resulting computation time is quite
extreme. Solving the 85,900 instance required a total of 136 CPU years, and hence for
most applications applying the Concorde to very large scale instances is infeasible.
Concorde is based on a branch-and-cut algorithm strategy that successfully solves
instances with up to 1,000 vertices in a reasonable computation time, but larger instances
quickly become very computationally heavy. In [HS14] the median computation time
for instances up to 3,000 vertices is estimated to scale according to Equation (4.2) with
constants a ≈0.21 and b ≈1.24. T represent CPU solution time in seconds. The study
similarly found that the solution times for separate TSP instances of the same size can
vary greatly.

CHAPTER 6
Discussion and
Future Work
The previous chapters have showcased different approaches for approximating the opti-
mal solution to large scale instances of the CTSPP with subclusters. Much of the work
presents ways of breaking down the problem into smaller subproblems, which are then
reassembled to construct a solution to the complete problem. This solution strategy is
a viable approach that offers solutions of a good quality, however this implementation
is unable to match the eﬀiciency of state of the art heuristics.
The idea of generating a candidate set of vertices that function as the vertices used
in intersubcluster connections is a strategy which can have a large potential, however
work needs to be put in to better limit these candidate sets. The candidate set proposed
in Candneighbor identifies many good candidates, however the number of candidates iden-
tified results in a very large number of subproblems, which is not desired nor feasible to
solve to optimallity within a reasonable time frame. One approach that was considered
but never applied that may lead to a large computational improvement of this strategy
is to start with an initial solution equal to an optimal solution to a separate instance of
a subcluster TSPP. Optimal solution paths for different entry and exit vertices often
contain many of the same edges as large portions of the problem remain unchanged.
Many heuristics such as LKH accept initial solutions, and thus may be able to generate
optimal solutions faster than Concorde. This initial solution could either be one of the
s−t TSPPs generated from the candidate lists, however it could also originate from the
OTSP solution to the subcluster.
Alternatively the approach can be enhanced by enhancing the candidate set. Candneighbor
offers too many candidates, Candclosest offers too few. Selecting good candidates, espe-
cially for large subclusters, can however be a challenge, and metrics need to be defined
that accurately identify a small set of great candidate vertices.
The reclustering approach offers a great way of getting an initial idea what a solution
route can look. However as the current applications of the reclustering process is based
on k-means which assumes a fixed number of clusters. This fixed number of clusters
may not be able to represent the internal layout of a subcluster. By applying alternative
clustering algorithms that do no take a fixed number of clusters, but instead fitting the
number of clusters required to describe the data may prove to be an eﬀicient tool to

improve this reclustering approach.
Another idea for improvement strategies is to group several of the smaller subclusters
into mid sized CTSPPs. This way the optimal solution will automatically detect the
optimal candidates and subcluster order. This approach leads to challenges in terms of
how these subclusters should be grouped, and a candidate list of entry and exit points
would have to be generated for this grouping of clusters. For most problem instances
this will not be trivial and require a clever idea.
Building on top of the idea of the Candneighbor candidate set, an approach could be
to build up a library of all TSPP solutions for the different combinations of candidates.
This task will be very computationally heavy, especially for the larger subclusters, how-
ever once the library has been created, recombining these in an optimal manner comes
down to arranging the subcluster order optimally and choosing the right entry and exit
points, which will be a lot faster as no TSPPs need to be solved, but solutions just
accessed from the solution library.
There are many ways to further improve on the generated improvement heuristics for
this problem. One idea is to search for local k-opt operations around subcluster borders
as a way to identify better candidates for entry and exit in the different subclusters.
The biggest potential for improvement is expected to be around these intersubcluster
connections, as the intrasubcluster connections are already solved to optimallity. These
classical k-opt operations can go through many more iterations in a given time-frame
compared to the proposed Subcluster-2-opt, and may lead to significant improvements.
It is however important to localize these operations to focus on the intersubcluster con-
nections. These k-opt operations can have varying depths similar to what is applied in
the LKH heuristic in order to increase the search space.
The Subcluster-2-opt can similarly be improved in different aspects. One of these
aspects is to better identify candidates within the heuristic, as this is currently based
on the naive candidate set of the closest vertex to the following subcluster. It is how-
ever important to limit the number of candidates to have any meaningful number of
iterations. There may be potential in having the Subcluster-2-opt of varying depths
as well, as some improvements may not show itself before several iterations of swaps
have occurred. The bottleneck again comes down to expensive running time for every
iteration of the Subcluster-2-opt, hence this is something that needs to be prioritized.
An alternative improvement heuristic could be based around the idea of prioritizing
poor performing subclusters compared to the OTSP LB and optimize these for an ex-
panded set of candidates, such as the Cneighbor candidate set. Combining this with the
Subcluster-2-opt would allow for continuous improvement on both the order in which
the subclusters are visited, but also on the entry and exit vertices used.

CHAPTER 7
Conclusion
This Thesis has presented solution strategies for the Clustered Traveling Salesman Path
Problem with Subclusters both in terms of lower bound generation and feasible solu-
tions. For the lower bounds, two approaches were proposed of different computational
complexity based on Minimum Spanning Trees and the Open Traveling Salesman Prob-
lem. Both approaches generate solutions for each of the separate subclusters, generating
a lower bound on a subcluster basis that can be expanded to a lower bound for the
complete problem by including the intersubcluster edges. The intersubcluster edges for
the lower bound are found solving an Open Traveling Salesman Problem for a distance
matrix based on the Single Linkage Distance between the different subclusters.
The solution strategy for solving the CTSPP with Subclusters is similarly based on
solving the different subclusters to optimallity based on a generated candidate list for
entry and exit points. Different approaches for candidate list generation were evalu-
ated, finding a strategy based on a reclustering of the subcluster vertices to be the most
eﬀicient. For the case study with 28,010 vertices and 100 subclusters, 1000 auxiliary clus-
ters were generated. An auxiliary route could then be solved through these generated
clusters, and the vertex subsets of the vertices used in the intersubcluster edges in the
auxiliary route could be used to generate a candidate list. This candidate list generation
has the advantage that it keeps track of the intrasubcluster route when considering which
candidates to identify. Following this, the subclusters are solved to optimallity using the
identified candidates for entry and exit vertices. Using the reclustering approach, a solu-
tion with optimal subproblems was generated in 2 hours of construction time for a fixed
set of entry and exit vertices. The gap for the complete problem compared to the Lower
Bound generated was 4.35%. Other candidate sets were generated based on the subclus-
ters of neighboring vertices and the distance to the following subcluster center. These
approaches generated gaps of 5.55% and 5.98% respectively. The generated solutions are
compared to state of the art heuristic for the TSP, the Lin-Kernighan-Helsgaun heuristic,
which was able to produce a solution with a gap of 2.78% in 2 hours of construction time.
An improvement heuristic is presented called Subcluster-2-opt, in which a 2-opt op-
eration is done on the order in which the subclusters are visited, and the intrasubcluster
routes are reoptimized for the new candidates. The improvement heuristic generates
better solutions, but is very computationally heavy as it requires 4 s −t TSPPs to be
solved in every iteration. Hence a prioritization needs to be made, on which 2-opts are
considered.
