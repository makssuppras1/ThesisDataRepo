Abstract
The advances in Machine Learning, and speciﬁcally Deep Learning, has led to
an explosion of work and research in the ﬁeld to solve various problems. In this
thesis, these new methods are applied to the problem of crop classiﬁcation.
Satellite imagery is used to classify which crops are growing on a given ﬁeld.
The implemented model is a Deep Convolutional Neural Network, based on the
U-net architecture.
The results of this work are very promising. Though the model struggles to
classify some speciﬁc crops, it is able to classify many diﬀerent crops with high
accuracy.

Introduction
1.1
Crop classiﬁcation using Deep Learning
The purpose of this project is to explore the opportunities of applying the recent
advances in image segmentation using Deep Learning, to perform crop classiﬁ-
cation based on satellite imagery.
Each year danish farmers apply for subsidies called Green Direct Payments
through the Danish Agricultural Agency. These subsidies are part of the Com-
mon Agricultural Policy subsidies from the European Union1 which, among
other things, seek to ensure sustainable agriculture in Europe, both economi-
cally and environmentally. Farmers are eligible for these subsidies if they meet
the following requirements.2
• Diversify crops (multiple crops on the farm)
• Maintain permanent grassland
• Dedicate 5 percent of the land as ’ecologically beneﬁcial elements’ (eco-
logical focus areas)
To ensure that the farmers are complying with the requirements, The Danish
Agricultural Agency performs a physical inspection of a small percentage of the
ﬁelds. To qualify this manual work, they have a yearly tender for satellite based
farm control (20183, 4 20195). The aim of the tender is use satellite imagery to
check that farmers meet the requirements. This enables the agency to identify
farms that are likely not to meet the requirements, and thus make the physical
inspection of the ﬁelds more eﬃcient.
The past two years, the task has been carried out by diﬀerent companies. Their
approach have been to use a Random Forest model to classify pixel values on
ﬁelds with quite good results. In recent years however, Deep Learning meth-
ods have outperformed classical statistical methods and other machine learning
methods in many image segmentation problems. Hence, it would be interesting
to explore the performance of Deep Learning methods on this speciﬁc problem.
The project is carried out in collaboration with the GIS and IT department at
COWI and supported by the Danish Agricultural Agency which has provided
data and reports on previous work on the subject.

2.1
Machine Learning
Machine learning is a term covering algorithms where computers progressively
learn how to solve a given problem. In general, there are two types of learning:
Unsupervised learning and supervised learning.
Unsupervised learning can be used to learn the underlying structure of a dataset
without any prior knowledge. Algorithms such as k-means clustering and prin-
cipal component analysis are widely used to highlight patterns in data but are
falling short when looking at complex problems. Neural Network architectures
such as Autoencoders can also be used to perform unsupervised learning, but
the biggest progress in learning has been in supervised learning in recent years.
Supervised learning is carried out using ground truth labels, i.e. prior knowledge
of what should be the output given an input. Thus, supervised learning is the
process of ﬁnding an optimal function that maps an input to an output. This
kind of learning is often used to solve regression and classiﬁcation problems,
which is what this project aims to do.
Supervised learning algorithms include support vector machines, linear and lo-
gistic regression, k-nearest neighbours, decision trees / random forests and ar-
tiﬁcial neural networks. The latter two are described further in the following
sections.
2.1.1
Random Forests
Random Forests, proposed by Tin Kam Ho in 1995 [Ho95], were, and still are,
considered one of the state of the art algorithms for classiﬁcation and regression
problems. The algorithm is an ensemble method where a number of decision
trees are constructed in order to counter the decision tree’s tendency for over-
ﬁtting (see subsection 3.3.2).
The Random Forest algorithm works by splitting the dataset into subsamples
as shown in Figure 2.1a. A decision tree is then constructed for each subsam-
ple. The trees together makes the Random Forest. When using the model for
prediction, the input data is run through each tree in the forest. The model’s
prediction is the result of the majority vote among the trees as shown in Fig-
ure 2.1b.

2.1.2
Artiﬁcial Neural Networks
The idea of Artiﬁcial Neural Networks (ANNs) was conceived in the sixties and
seventies and the ﬁeld has seen steady advances since then. However, in recent
years, the increase in available computational power in both consumer devices
and research labs has led to enormous progress in the ﬁeld. The results of this
work are some well performing network architectures, such as Convolutional
Neural Networks which will be covered in later sections. The name Artiﬁcial
Neural Networks stems from the notion of mimicking the neurons in a brain to
make a model that can learn the properties of data.
2.1.2.1
Fully connected networks
Common for all ANNs is that they consist of a graph of nodes with weights.
These weights are the parameters of the model, and they are updated through
training of the network.
In Figure 2.2 an example of a fully connected network is shown. Here we have
3-dimensional input feature x which we want to classify in two classes, y. The
nodes h in the hidden layers are the neurons of the network which mimic the
neurons of a biological brain. The biases b are nodes which are necessary to
train the network in most cases. The blue and orange lines between the nodes
in the graph represent the weights and biases respectively.

2.1.2.2
Optimisation and Gradient Descent
The basic principle of training an ANN is to treat the task as an optimisation
problem. We want to minimise the error between the output of the network and
the expected result. This error is called the loss function. To minimise the loss
function, diﬀerent optimisation algorithms can be used, but most of them rely
on the same principle: Gradient descent [GBC16, p. 80].

2.1.2.3
Inference and training - Forward and back propagation
Inference is the process of using the model to perform a prediction given a data
point, i.e. supply the model with x and get the output y. When doing this,
the initial information from x ﬂows through the network. This is called forward
propagation.
When training the network, the gradient of the loss function is estimated using
back propagation. The error from the loss function is propagated backwards
through the layers in order to estimate the gradient with respect to each pa-
rameter. [GBC16, p. 200]
Training an ANN is a non-convex optimisation problem similar to the function
seen in Figure 2.3b, just in a higher dimensional space than the 2D-case seen in
the ﬁgure.
For a human it looks just as easy to ﬁnd the minimum in Figure 2.3b as it
is in Figure 2.3a, but it is not trivial for a many-dimensional problem. With
many local minima it might seem unfeasible to train an ANN with a gradient
descent based algorithm. However, while gradient descent only guarantees a
global optimum for convex problems, it also does work for other problems if
precautions are taken to avoid getting stuck in a local minimum.

2.1.2.4
Activation functions
The output of the layers in the network is put through an activation function.
This function should be non-linear, as it has the purpose of introducing non-
linearity in the model. Without non-linear activation functions, the model would
be linear and have trouble mapping complex structures in the dataset.
Historically, activation functions such as the sigmoid and hyperbolic tangent
functions have been popular. Today, the rectiﬁed linear function (ReLU) has
taken the role as the most used function. The three functions are shown in
Figure 2.4.

The sigmoid and hyperbolic tangent functions suﬀer from a problem known as
vanishing gradients caused by the fact that their gradients are in the range
(0, 1). In the backpropagation algorithm, these gradients are multiplied, po-
tentially resulting smaller and smaller numbers the further the algorithm goes
back through the network. This causes the network to get stuck in the training
process and especially a problem for deep neural networks with many layers.
The ReLU function does not have the vanishing gradient problem, and it has
been proven to be highly successful in deep neural networks as described in
subsection 3.2.1.

2.2
TensorFlow and computational graphs
TensorFlow, which is the machine learning library that is used in this project,
uses a graph to represent the network and its layers and operations. Data (ten-
sors) ﬂow through this graph from input to output, hence the name TensorFlow.
Figure 2.5 shows an example of the graph representing the network shown in
Figure 2.2. The graph is relatively simple: Data ﬂows from the input x, through
the fully connected layers h1 and h2 to the output y.
The hidden layer h1 is expanded to show how the layer is deﬁned. We can see
that the input from x ﬂows to a MatMul operation with the weights of h1. After
the MatMul, the biases are added through the BiasAdd operation, and ﬁnally
the results are put through the ReLU activation function.
Using this graph, TensorFlow is able to parallelise computation of independent
operations, compute gradients and optimise operations to perform eﬃcient in-
ference and training of the network.1 TensorFlow is able to utilise the power of
the Graphics Processing Units (GPUs) to train neural networks in a highly eﬃ-
cient manner. The implementation of the network using Tensorﬂow is described
in detail in chapter 5.

3.1
Crop classiﬁcation
The European Copernicus programme and the Sentinel satellites has enabled
many new projects within the ﬁeld of data based agriculture. Both for precision
agriculture where fertilisation and watering can be optimised1 and for crop
detection.
The task of crop classiﬁcation using satellite imagery can be approached using
various methods. In [OAG12] a statistical Maximum Likelihood Classiﬁcation
method is compared to a Random Forest based method. In [BC18] a pixel-
based Random Forest method is compared to an object based Time-weighted
Dynamic Time Warping method. Also The Danish Agricultural Agency has
seen good results using pixel based a Random Forest algorithm to classify ﬁelds
using satellite imagery and radar data. Deep learning based crop classiﬁcation
based on Convolutional Neural Networks such as [JZX+18] has also been shown
to work well, which is promising for this project, even though the used methods
are diﬀerent.
3.2
Image Segmentation using Deep Learning
Image segmentation is the process of classifying parts of an image into diﬀerent
classes. The problem is occurring in many diﬀerent applications as shown in
Figure 3.1.
Self driving cars rely on image segmentation algorithms (among other things) to
navigate on a road. An example of such a segmentation is shown in Figure 3.1a.
Here the features in the image is segmented in classes such as road, cars, trees,
buildings and signs.
In medical imaging, segmentation algorithms are used for many diﬀerent pur-
poses. In Figure 3.1b, the cell segmentation problem from [RFB15] is shown.
3D image segmentation can be used to segment out the heart in MRI imagery
and measure the volume of blood ﬂowing through it. [MNA16]
Segmentation of satellite imagery is also used for many diﬀerent purposes from
mapping cities2 to performing crop classiﬁcation as shown in Figure 3.1c. An
interesting use case comes from the satellite imaging company Planet. They
use their imagery in a new application called Planet Analytics to measure land
use and land cover change and perform object detection to count ships and air
planes across the world. This allows their customers to easily query information
for any given area of the globe.

3.2.1
Convolutional Neural Networks
Convolutional Neural Network (CNNs) is a type of deep neural network that
contain convolutional layers and is typically used on image datasets. A convo-
lutional layer performs a convolution operation on the previous layer.
In a fully connected layer, the network learns a weight for each input value (pixel
for images) of the previous layer. For a convolutional layer, the network learns
the kernel of the convolution. This gives a much smaller number of parameters
in the model which enables much deeper networks to be trained.
The convolution operation is typically performed for each pixel in the input
image. The kernel, a small matrix, deﬁnes a neighbourhood around the input
pixel and weights for the neighbouring pixels. The pixel values and weight values
are multiplied and summed together. This gives the value for the convoluted
output image. Diﬀerent convolution kernels gives diﬀerent output images. Some
examples include edge detection and blurring as seen in Figure 3.2.

3.3
Training Deep Neural Networks
Deep Neural Networks are much harder to train than the more traditional and
relatively smaller networks such as fully connected networks. They require a
lot of computation time and tuning of the diﬀerent parameters that control
the training process and the layers of the network. Additionally, deep neural
networks often suﬀer from the problem of overﬁtting, which can be countered
with data augmentation and regularisation techniques.
The parameters of a model are called hyperparameters, and the process of tuning
them is called hyperparameter optimisation or hyperparameter tuning.
The
hyperparameters of a deep neural network model can include the following:
• Choice of loss function
• Learning rate
• Network size and depth
• Convolution kernel sizes
• Layer dropout

3.3.2
Overﬁtting
Overﬁtting is a common problem for deep neural networks. It happens when the
model is able to learn the training data too well, i.e. it memorises the training
data. When the model is then used to make predictions on data that it has not
seen before (i.e. the evaluation dataset) it will perform poorly. This typically
happens when there is too little training data or too many parameters in the
model.
The eﬀect of overﬁtting can often be seen when plotting the training loss versus
the evaluation loss. An example of how the two functions typically look is shown
in Figure 3.5. The loss functions follow each other nicely for a while. At some
point, the model starts to overﬁt the training data which causes the training
loss to continue downwards but the evaluation loss to start increasing.
When training a deep neural network, the model should be saved and tested
at the optimum of the evaluation loss function.
To counter the problem of
overﬁtting, methods such as data augmentation and regularisation can be used.

3.3.3
Data augmentation
Data augmentation [GBC16, p. 237] can be used when the model is overﬁtting
due to lack of training data. In the case of using imagery as input data, the fol-
lowing augmentations can be used to artiﬁcially increase the size of the dataset.
This will decrease the networks tendency to memorise the full training dataset.
• Rotation
• Translation
• Flipping
• Deformations (stretching / squeezing)
• Gaussian noise
In the case of using imagery as input data to the model, operations such as
rotation, translation, ﬂipping and deformations such as stretching or squeezing
can be applied to artiﬁcially increase the size of the dataset. Gaussian noise can
be added to the images to "confuse" the network which may prevent it from
memorising the full dataset.
3.3.4
Regularisation and Normalisation
Overﬁtting is not only a problem in deep neural networks, it also happens when
using many other machine learning algorithms. Decision trees are known for
overﬁtting which is why Random Forests are used instead. This use of models is
called ensemble training and is certainly also possible to do for neural networks,
but adds additional computation time and complexity. However, for deep neural
networks it is possible to perform regularisation of the network which mimics
the nature of ensemble models. One of these techniques is called dropout.
Dropout [SHK+14] [GBC16, p. 255] approximates training a number of networks
with diﬀerent architectures but is very inexpensive in terms of computational
power. It works by "dropping out", i.e. ignoring, random outputs in a layer.
It may sound counter intuitive to ignore parts of a trained layer, but dropout
introduces noise in the training process which helps prevent overﬁtting.
Another method that can improve training time is batch normalisation [IS15].
Input data are often normalised, when being prepared for the training process,
by substracting the mean and dividing by the standard deviation of the data.
Batch normalisation extends this concept to all the layers in the network. The
normalisation of the network layers allows for higher learning rates to be used
==PAGE:32==
20
Related Work
which decreases the training time drastically. Additionally, the normalisation
introduces some regularisation to the network, which may eliminate the need
for dropout layers in some cases.
3.3.5
Hyperparameter optimisation
Hyperparameter optimisation [GBC16, p. 422-431], sometimes called hyperpa-
rameter tuning, is the problem of ﬁnding the set of model parameters that gives
the best results. The parameters of a model include learning rate, regularization
and normalization described in the previous sections as well as model parame-
ters such as the number of hidden units in a fully connected layer or the kernel
size of a convolutional layer.
Hence, for a given model there can be many parameters that needs to be tuned
to make the model perform well. With knowledge about the problem, the model
and the data, some paramters may be tuned manually, but usually some sort of
automatic tuning is used to ﬁnd the optimal set of parameters.
The simplest of such automatic approaches is using a grid search. Deﬁne a set
of discrete values for each parameter and search through all combinations of all
parameters. This approach is feasible when relatively few parameters needs to
be optimised.
Random search can be used as an eﬃcient alternative to grid search. In random
search, the parameters are deﬁned using a probability distribution. For every
run of the model, a new set of parameters are generated from their distributions.
This has been shown to perform much better than grid search in some cases.
Finally, it may be possible to pose the tuning of hyperparameters as an ac-
tual optimisation problem where optimisation algorithms such as gradient de-
scent can be used to ﬁnd an optimum. This requires that the gradient of the
hyperparameter-optimisation problem can be calculated.
Alternatively, loss
function can be modelled using Bayesian regression and this can be used to
estimate a gradient for the hyperparameter-optimisation problem.

5.3.5
Tiling the dataset
The imagery and label image is tiled into smaller parts in order to produce a
dataset that can be used to train the machine learning model. As the diﬀerent
bands of Sentinel-2 imagery has three diﬀerent spatial resolutions (10m, 20m
and 60m), the dataset will contain tiles in three diﬀerent sizes. Examples of all
12 bands of a tile is shown in Figure 5.5.
The biggest tile size should be divisible by 2 and 6 to match the diﬀerent spatial
resolutions of the Sentinel-2 imagery.
Additionally it should ensure an even
layer size as input to the 2 × 2 max-pool layers in the network described in
subsection 5.5.1.
A tile size of 732 × 732 pixels for the imagery at 10 meter spatial resolution was
chosen. This gives a tile size of 366×366 for 20 meter imagery and 122×122 for
60 meter imagery. With the chosen network structure in subsection 5.5.1 this
gives even input sizes to the max-pool layers of 728 × 728, 360 × 360, 176 × 176
and 84 × 84 when using unpadded convolutions.
With the tile size of 732 × 732 pixels and a satellite image resolution of 10980 ×
10980, a total of 225 tiles are produced.

5.4.3
Saving the best model
The best possible model should be saved when training the network.
The
tf.estimator API contains a helper function for this purpose called BestExporter
that works with the train_and_evaluate function, which wraps the func-
tions described in subsection 5.4.2 into one.
A lot of time was spent trying
to implement the training loop using these functions. However, the TensorFlow
documentation is sometimes inadequate, and in this case it was not possible
to ﬁgure out how to use it properly. The BestExporter function requires a
serving_input_receiver_fn function as input argument, and it is not clear
how to implement this when loading data using a custom parser function for
the .tfrecords dataset as is done in this project.
Instead of using the helper functions, a manual method was implemented. When
evaluating the network during training, the loss and mean IoU metric is returned
and used to determine if the current model is better than what has been seen
before. If this is the case, the model is saved as the best model. Addition-
ally, this is also used as a stopping criterion. When the training has run for a
given number of iterations without seeing improvements to the best model, the
training is automatically stopped.

5.5.1
Standard U-net
The U-net architecture consists of a contracting path and a symmetric expanding
path. It is fully convolutional, i.e. there are no fully connected layers. This
enables the network to be used for arbitrary input image sizes, only limited by
the GPU memory.
Each layer in the contracting path consists of two convolutional layers and a
max-pool operation.
These layers will thus gradually decrease in width and
height but increase in depth.
The bottom of the U consists of two convolutional layers.
The expanding path consist of an up-convolution (transposed convolutions) and
two convolutional layers. Here the layers gradually increase in width and height
and decrease in depth.
When drawing the network graph, it is shaped like a U as shown in Figure 3.4a.
The layers in the contracting and expanding path are connected through con-
catenation horizontally across the U. A visualization of the graph generated
with TensorBoard is shown in Figure 5.8.

5.6.1.3
Weighted Cross Entropy
In order to account for the imbalance in class size, a weight map is multiplied on
to the cross entropy matrix before calculating the mean. This is implemented
as a variable weight which can be adjusted during experiments.
The max weight can be adjusted as a parameter: 1 is no weighting, 2 is weighting
up to a factor 2.
An example of an un-weighted cross-entropy image and a weighted cross entropy
image is shown in Figure 5.9.
The weight map consist of two parts: A map of the borders of binary mask and
a map of the label image where each ﬁeld is represented by the weight of its

class. These two maps are constructed in the parser function together with the
superclass label image.
The pixels around the borders of a ﬁeld is often a mix of reﬂected light from
the ﬁeld and its neighbouring ﬁeld or roads, forrests, etc. The purpose of the
border map is to decrease the loss function around the borders of the ﬁelds and
focus on minimising the loss within the ﬁeld.
The border map is constructed by using binary dilation on the binary mask.
This is shown in left side of Figure 5.10a. When implemented initially, this was
performed on the GPU and not in the parser function. However, TensorFlow
does not contain a binary dilation function that runs on the GPU. This caused
the computing time to increase drastically, as the CPU and GPU would have to
exchange data during training. Thus, a custom binary dilation function based on
the TensorFlow 2D convolution function was implemented to solve this problem.
The weight map is constructed using the label image and a list of the weights
for each class. The maximum weight is saturated to the value of the weight
parameter used when running the training. An example of the weight map is
shown in the middle of Figure 5.10a.
The border map and weight map is combined such that the weight map values
are used within the ﬁelds. At the borders of the ﬁelds, the border map is used
to decrease the weight to 0.5%. This combined map is shown in the right side
of Figure 5.10a.

The depth parameter is a way to change the number of parameters in the model.
Using a depth of 64 means that the convolutional layers in the ﬁrst layer of the
contracting path will have 64 channels. In the second layer, 128 and so forth. A
depth of 16 means that these convolutional layers will have 16 and 32 channels
respectively.
In this experiment, a basic network conﬁguration with a learning rate of 0.001
is used to compare how networks with diﬀerent depths perform. The mean IoU
of these runs are shown in Figure 6.10.
It is clear that the network with a depth of 16 does not contain enough parame-
ters to train properly. Varying the depth between 32 and 80 does, however, not
change much.
A depth of 64 performs well and is chosen for the following experiments in
order to provide a suﬃcient amount of parameters for the model when other
optimisations are performed.

Discussion
7.1
Discussion
Deep Learning models for image segmentation to perform crop classiﬁcation
using satellite imagery has been proved to work in this project. The implemented
U-net network architecture works quite well with the satellite imagery, even on
a relatively large number of crop categories compared to other similar works.
Many of the crop categories are classiﬁed with well above 90% accuracy.
Some categories are harder to train than others. While the network performs
quite poorly on crops such as Cabbage, Beets and Peas, it performs quite well
on crops such as Clover and Spinach, even though they are only represented by
a fraction of a percent of the data used to train the network. This shows that
the network is able to learn to distinguish crops even though it is trained with
a relatively small amount of data.
Larger categories, such as the grain crops Oats, Barley, Wheat and Rye, also
show varying performance. The winter sown grain crops are easily classiﬁed
correctly while the spring sown crops are harder to distinguish.
This shows
that there is a potential in this method for crop classiﬁcation, but there is also
potential for improvements.
During the project, a number of choices were made to limit the scope. One
of the most interesting subjects to explore would be to look at the errors in
the dataset found during the physical inspections of the ﬁelds and to compare
these to the output of the network. Is the network able to identify the errors?
Unfortunately, this was not investigated during the project.
Other choices made throughout the project include:
• Only using Sentinel-2 imagery from one area.
• Not using Sentinel-1 radar data.
• Using hand made cloud masks.
• Not using data on organic and conventional ﬁelds.
• Not taking full advantage of the ﬁeld polygons.
Improvements in accuracy could come from handling these limitations. Thus
the following sections contain suggestions for further research and improvement
of Deep Neural Networks for crop classiﬁcation.
The use of more satellite imagery would increase the size of the dataset and the
input features for each pixel drastically, which could increase the accuracy and
performance.
==PAGE:91==
7.1 Discussion
79
The ﬁeld polygons could be a valuable piece of information, which has not been
properly exploited using this network. If the location and bounds of each unique
ﬁeld in each image could be used as input data, the network could perhaps learn
how to distinguish neighbouring ﬁelds more eﬃciently.
There could be improvements to be gained in the implementation of the network
as well. The chosen loss function, softmax cross entropy, has been shown not
to be the best for image segmentation problems with high class imbalance. It
would be interesting to implement other loss functions such as Generalised Dice
Loss or IoU loss to see what gains in performance they could contribute.
The implemented U-net network architecture is not the only successful archi-
tecture for image segmentation using deep learning, and it could be interesting
to use other architectures on the same dataset and compare the performance.
The implementation was based on TensorFlow. TensorFlow proved to be quite
hard to work with at times, due to the lack of proper documentation and ex-
amples for many of the APIs used to implement and train the network. The
network is created using relatively common operations and should not be dif-
ﬁcult to implement using another machine learning library. Using higher level
tools and libraries is recommended to anyone who is working with image seg-
mentation and similar problems as the one in this project.

==PAGE:94==

Conclusion
8.1
Conclusion
Crop classiﬁcation on satellite imagery using Deep Learning methods based on
Convolutional Neural Networks is very promising.
In this project, a large number of diﬀerent crop types were classiﬁed. In other
crop classiﬁcation problems the number of crops are often signiﬁcantly smaller.
This project shows that it is possible to perform a detailed crop classiﬁcation
using using Deep Learning, where it is not only possible to diﬀerentiate grains
and maize, but also individual species of grains.
The results in this project are limited by the data used. Using only Sentinel-2
imagery, the model was able to classify many of the crop categories satisfactory,
but including additional imagery and radar data could increase the prediction
accuracy and improve the classiﬁcation.
Future work on the topic should extend the amount of ﬁelds used for training
and include imagery and data from multiple satellites.
