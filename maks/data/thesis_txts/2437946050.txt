Abstract
The aim of this Masters thesis is to find a solution that allows the introduction of vir-
tual reality in the architectural acoustics design process. As classic auralizations are
valid for a single position, the next level would be to achieve a method that freely pro-
vides movement around the space. Virtual reality allows that and many other options.
The approach followed to build such an Acoustic-Virtual-Reality (AVR) system
consists of pre-calculating all the acoustics simulations in a dedicated software (Odeon)
which provides reliable results. A grid of points of Ambisonic B-format impulse re-
sponses is exported to keep spatial information. On the other hand, the visuals are
managed by Unity and Oculus-Rift. Control information is sent by UDP to Pure
Data which manage all the audio parts of the project. The main patch contains two
modules: one designed to select the audio file which corresponds to the user position,
and another which computes the binaural synthesis so that a binaural signal is pre-
sented to the final user.
Three main spaces are modeled during the project: an invented rehearsal hall, a
well known real space from DTU, and a real project of Henning Larsen.
Finally, the system is validated through a pilot test which consists of two parts.
First, a spatialization test focused on the study of how accurate the spatial location
is with different scenarios and a second test which studies the subjective perception
of different grid size.

Preface
This thesis was made in collaboration with Henning Larsen Architects and the Acous-
tic Technology group at the Department of Electrical Engineering at the Technical
University of Denmark. The project contains a workload equal to 30 ECTS points
and it is submitted in compliance with all the requirements to achieve the degree of
Master of Science in Acoustical Engineering.
This project aims to contribute to the actual building design process as far as
acoustics is concerned. Nowadays, virtual reality tools for a professional purpose are
growing increasingly popular. The idea of integrating acoustics can help in the design
process to detect different problems and evaluate different solutions. Architectural
acoustics has been one of my main interests. The mix of architectural acoustics, au-
ralization, and virtual reality is a new and exciting topic that makes me glad to have
the chance to work on this project for my Master’s thesis.
The report is divided into four main chapters. It includes an Introduction, where
the motivation and the academic value of the project are clarified. The Theory section
explains the theoretical base of some relevant topics needed to carry out the project.
It also contains the mathematical and acoustic properties as well as the different
assumptions that are considered. The Methods chapter describes the methodology
followed in order to carry out a different process such as acoustic measurements and
validation.
It is one of the most important chapters of the thesis as it contains
a detailed description of the process in which the Acoustic Virtual Reality (AVR)
system is designed. The Result and discussion chapter presents different implemented
cases. Different rooms are modeled and analyzed. Finally, a conclusion summarizes
the main points of the project.
Kongens Lyngby, June 22, 2018
Hermes Sampedro Llopis

CHAPTER 1
Introduction
1.1
Motivation
Acoustic auralization has been around for a long time; it makes listening to a sim-
ulated acoustic space possible. It also plays an important role in the design of new
acoustic spaces as it helps to predict and adjust the acoustics, where a designed
space can be shown to a client by playing different auralized tracks. However, each
auralization computation is made for only a single position and in most cases, the
computation time is high. Thus, the next level would be to achieve an auralization
for all the points, being able to move with freedom around the space.
Nowadays, the interest in virtual reality (VR) is growing very fast. Many compa-
nies are developing new ideas which involve the use of VR for different purposes (e.g.
entertainment, medicine, tourism, education, etc). VR allows the user to experience
an alternative reality through 3D visuals and audio. With high quality equipment,
which include VR glasses, sensors and controls, the perception can feel incredibly real.
Currently, there are various products in the market for VR; however, companies are
looking for new solutions which can provide an even better immersion. Generally,
the visual part is more developed than the acoustic one. There are diverse solutions
available, mostly for video games, which provide developers the possibility of adding
binaural sound to the games.
Previous research work has been done regarding acoustic virtual reality. E. Balles-
teros [1] implemented a dynamic audio system that supports the immersion provided
by VR environments.
A Roman theatre was modeled and simulated; then, using
Ambisonics, the head tracking in a single fixed position is achieved. Another project
conducted by the Université Paris-Saday investigates the virtual reality auralization
in Notre-Dame cathedral [2] also by using ambisonics.
Moreover, [3] investigates
acoustic virtual reality and methodologies to compare the overall appreciation of fu-
ture renovation of urban public spaces. In this case, an Ambisonics recording was
made and reproduced in the virtual environment afterwards. All these projects have
been an inspiration for the development of this thesis. In all cases, various special-
ized softwares were used to take care of the different parts. The visuals and VR are
typically managed with either Blender or Unity together with Oculus Rift. Acoustic
simulations are carried out with acoustic software. Finally, Max/Msp is used to man-
age the audio part. Notice that in all these cases, the developed systems are designed
for fixed positions or established paths where the user is not allowed to move freely.

1.2
Research Objectives
The overall objective of this masters thesis is to find a solution that implements
auralization and VR together. The new contribution includes complete freedom of
movement and head tracking around the modeled space.
The advantages of the VR, which includes, visuals and free movement with 6 de-
grees of freedom (DoF) are summed to the advantages of the auralization. Thus, a
complete immersion into the VR model can be experienced. That would definitely
help through the architectural design process to have a realistic idea about how the
model is going to be and what problems can take place. It would be also useful in
making decisions between different acoustic solutions. Moreover, the architects are
involved in the visual part when designing a project. Applying VR to the process can
help to simulate how the acoustic materials affect to the overall visuals scape. With
these possibilities, the user is able to make an A/B comparison between different
acoustic treatments to perceive the acoustical and visual differences between them.
This Acoustic Virtual Reality (AVR) system will present all the previous advan-
tages and it would be designed to be used in the room acoustics field. As part of this
main objective, the following research objectives are identified:
• Investigation of an Acoustic Virtual Reality implementation method using reli-
able acoustic simulations such as those provided by Odeon, which are appropri-
ate for architectural acoustics applications.
• Development of an Acoustic Virtual Reality system based on pre-calculated
acoustics, that allows complete freedom of movement and head tracking (6
DoF).
• Investigation of different Ambisonics order decoders.
• Listening test.
Subjective evaluation of realism and sound localization with
different order of Ambisonics and interpolation schemes.

2.1
LTI systems and fundamentals of signal processing
This section will explain some fundamental knowledge about Linear Time-Invariant
(LTI) systems and signal processing, which is needed for the processing part of the
project.
2.1.1
LTI Systems
A LTI system is a system that satisfies the properties of linearity and invariance over
time.
The linearity properties imply superposition which includes proportionality
and additivity. It is assumed that x is the input signal and y is the output signal.
Figure 2.1 shows the proportionality principle. When the input signal is multiplied
by a factor α, the output will also be multiplied by the same factor.
Figure 2.1: Proportionality principle.
Figure 2.2 presents the additivity principle. Assuming two separated signals x1
and x2 which produces an output signal y1 and y2 respectively, the sum of both inputs
will produce the sum of the outputs.

2.1.2
Auralization
In general, a room is an LTI system where the input signal is created by the source
(or sum of sources in the room) and the output signal is obtained by performing
the convolution of the input with the impulse response. Auralization has been used
in room acoustic simulations by convolving the impulse response of the room with a
pre-recorded sound (normally in free field). Then, the result is convolved with a Head
Related Transfer Function (HRTF) so that the reproduction by headphones recreates
the acoustic space where the listener would be placed in the simulated point.

2.2.1
The impulse response of a room
The impulse response is the basic source information of the sound field in a room.
This signal contains all the acoustic information of the room for a certain position
of source and receiver. Figure 2.6 shows how a spherical wave is traveling from a
source to a receiver in a concert hall. The direct sound is the first sound that arrives
to the receiver. Then, other wave components, called early reflections, arrive to the
receiver after being reflected. These contains less energy as part of it is absorbed by
the surface. The sound continues reflecting and arriving to the receiver until all the
energy has been absorbed. These reflections are called the reverberant tail, and the
density of these late reflections increases with time. This is heard as reverberation

2.3
Spatial hearing
The position of the ears on both sides of the head makes humans capable to per-
ceive sound around them. Once the sound is received by the hearing system, the
information is passed to the brain to be processed and interpreted. This contributes
to the ability to localize sound in the 3D space. This ability is commonly referred
to as spatial hearing. As three dimensions are involved, a polar coordinate system
is considered more appropriate to describe the 3D space where the sound is located.
Thus, it is possible to describe the space with distance, azimuth and elevation as
shown in Figure 2.10. The horizontal plane is the plane parallel to the ground. The
height is at the ears’ position so that the origin is located at the center of the head.
The median plane is perpendicular (vertical) to the horizontal plane and the origin
is also located at the center of the head. The angle described between the median
plane and the vector from the center of the head to the source is called azimuth. This
angle is usually seen clockwise. On the other hand, the angle between the horizontal
plane and the vector from the center of the head to the source is called elevation. In
this case, 0 degrees indicates that the sound is placed in front of the head. When the
angle is increased, the sound is located above the head.
Figure 2.10: The coordinate system for sound localization [7].
For the sound localization, the human brain makes use of different cues [7]:
1. Interaural time difference (ITD)
2. Interaural level difference (ILD)
3. Spectral cues (monaural, relevant to the anthropometry of the listener)
4. Head movement cues
5. Intensity, loudness cues
==PAGE:28==
10
2 Theory
6. Familiarity to sound source
7. Direct to reverberant ratio (DRR)
8. Visual and other non-auditory cues
Cues 1 to 4 contribute to directional localization, whereas cues 5 to 7 contributes
to the distance perception.
The most important cues for the horizontal plane localization are the ITD and
ILD. The ITD is the time arrival difference between both ears which varies depending
on the source position. On the other hand, the ILD is the level difference between
both ears produced by the shadowing effect of the head. Figure 2.11 illustrates both
cues. In this case, as the source is located more on the right side of the listener, the
sounds arrive earlier to the right ear. Thus, the left ear receives a delayed version with
less amplitude as the head introduces some attenuation. ITD is related to the ability
of detecting interaural phase differences at low frequencies. ILD is more dominant
at high frequencies. Up to frequencies where the wavelength is comparable with the
head dimensions, the head does not produce a shadowing effect as the wave is able
to surround it. The cutoff frequency is determined by the distance between the two
ears which is commonly considered around 1500Hz which corresponds to the distance
between ears around 22cm [7]. Moreover, the distance between the human ears is half
the wavelength of a sound having a frequency of 700Hz. Below this frequency, it can
be considered low frequency. There is no obstacle produced by the head to the sound
waves, so the amplitudes that reach every ear are identical. The only information
that is available at low frequencies for localization is the phase difference between
ears. Hence, the ITD is more dominant at low frequencies.
Figure 2.11: Interaural Time Difference and Interaural Level Difference [8].

As commented before, ITD and ILD are the most important cues for the horizontal
plane, but they are not enough to determine the source location on the median plane.
The so called cone of confusion (Figure 2.12a) refers to the space where the ITD and
ILD has the same value, and without any other cue it would be impossible to localize
the sound. Therefore, in order to be able to localize on the median plane correctly,
spectral cues are needed.
The head, torso and pinna, filter the sound producing
spectral differences. A more exciting fact is that the brain does not interpret these
spectral differences as coloration, but rather as source localization. This filtering is
produced due to the reflections on the body, especially on the pinna. The head and
torso have more influence at low frequency, whereas the pinna has more influence
above 3kHz. As this filter is produced by the body and the pinna shape, it has a huge
variation between different persons. The movement of the head inside the sound field
helps to determine the location of the source. Figure 2.12b presents two cases. On
the left, the ITD is zero at both ears. In this case, the listener experiences ambiguity
as it is not possible to determine if the source is placed in front or behind (as both
cases result in the same ITD). The right diagram shows that the listener is able to
obtain information of the source location when moving the head. Moreover, the left
ear is closer to the source (placed in the front and not in the back) so that the ITD
value of the left ear is smaller than the one perceived with the right ear. The ITD
and ILD cues are good for locating sources on the horizontal plan; however, they are
insufficient for judging whether a sound is located in the front or back, and above or
below the listener.

2.3.1
HRTF
The Head Related Transfer Function (HRTF) is introduced to describe all the cues
that are taken into account. It describes not only the ITD and ILD but also the
spectral cues and how the sound field changes due to the presence of the listener.
Since the human has two ears, the HRTF are typically given in pairs. The impulse
response from the source to the ear drum is called the Head Related Impulse Response
==PAGE:30==
12
2 Theory
(HRIR). Once the HRTF for both ears are known, accurate binaural signals from a
monaural source can be synthesized.
The HRTF includes the three cues: ITDs,
ILDs and spectral filtering from the head, torso and pinna. Combining these cues,
it is sufficient to resolve all angles on the azimuth (horizontal) and median plane
(elevation). By including these cues, the transfer functions between each point of the
space and the ears of the listener are obtained. Notice that the HRTF is unique for
each point of the space, each listener and each ear.
From Figure 2.13 it is clear to see that when expressing the HRTF, the spectral
cues are shown. Moreover, for the HRIR, the ITD and ILD are clearly represented
(by the delay and the amplitude) as it is expressed in the time domain.
Figure 2.13: HRTF and HRIR [7]
The HRTF or the HRIR can be measured with a Head and Torso Simulator
(HATS) or directly on humans by placing microphones inside both ears. The first
method is normally preferred, as the HATS is made taking into account a standard
averaged human head, torso and pinna shape. Moreover, as the HRTF changes from
subject to subject, it is not common to measure on human subjects. Figure 2.14
presents a KEMAR head and torso of GRAS Sound & Vibration.
Nowadays, there are several HRTF databases. In order to share and work in an
organized way, the SOFA file format [10] is introduced. This format was designed
for storing spatially oriented acoustic data such as HRTF and binaural or directional
impulse responses. SOFA format has been standardized by the Audio Engineering
Society as AES69-2015.

2.3.2
Distance perception
The perception of sound distance is an significant factor in sound localization. It is
especially important in sound rendering where the distance plays a critical role in the
soundscape recreation. There are different cues used to judge the distance.
The direct-to-reverberant energy ratio is one of the main cues. As explained in
previous sections, the direct sound is followed by many reflections in a room. The
direct-to-reverberant ratio energy ratio varies with distance. Positions close to the
source present more direct sound, whereas further positions present less direct sound.
As the reflections are produced on the surfaces of the room, this cue depends on the
geometry, size and materials of the room. Moreover, for very dry environments, this
cue is not effective.
The intensity decreases when increasing the distance.
For an omnidirectional
source in free field the ratio is P ∼1/r. This cue is effective for distances below 15
meters.
Another cue is the spectral change. For long distances (more than 15 meters)
the air absorption begins to be noticeable (mainly at high frequencies), meaning that
sounds produced far away contain less high frequency energy. For short distances
(less than 0,5 meters) the HRTFs become distance-dependent and appear with some
monoaural and interaural changes. In addition, sound sources very close to one ear
produce very large ILD.

2.3.3
Influence of visuals cues on sound localization
As described in [11], the visual cues help to resolve localization ambiguities and
improve compactness perception. That means that the visuals have a direct impact
on the sound localization and can improve it as well. Gerzon [12] also highlights
that the localization ability can be affected by the expectations and attention of the
listener. The difference of having visuals or not when localizing a sound around the
listener can have a big impact.
2.4
Ambisonics
Ambisonics is a full-sphere surround sound reproduction technique that describes a
given sound field by using spherical harmonics. It was developed in the UK in the
1970s, predominantly by Michael Gerzon from the University of Oxford. The trans-
mission channels do not carry any speaker signal as other formats do. Instead, the
information is represented with the so called B-format, which contains components of
the given sound field and is then decoded to a speaker array. This makes it possible
to deal with multichannel audio from place to place without worrying about the final
reproduction system. B-format supports a full 3D audio and it is possible to process
the information easily, such as that in sound field rotation.
The B-format can present different kinds of orders. At zero order, there is only
a mono channel behaving as an omnidirectional microphone (W). The first order
consists of three additional channels behaving like a figure-of-eight microphone; there
are in total 4 channels: (W,X,Y,Z). The higher the order, the more channels and
spatial information. That means that high-order Ambisonics (HOA) can considerably
improve the spatial quality of the reproduced sound field [7].
2.4.1

The mathematical representation of Ambisonics is based on the directional properties
that the spherical harmonics present. It is also assumed that the sound field can be
decomposed in plane waves and summed, considering that the sources are far away

spherical coordinates is used. The coordinate system is defined in Figure 2.15. ϕ
represents the azimuth angle in positive orientation or counter-clockwise, and θ is the

2.4.4
Higher order Ambisonics (HOA)
Increasing the order improves the directional localization and allows to reconstruct a
more accurate sound field. In fact, the size of the sweet spot in which the sound field
is accurately reproduced is increased. The price paid for this is that the number of
channels is growing with the order, which also means that the minimum number of
loudspeakers needed to reproduce the sound field is also increased. The total number

2.4.5
Psychoacoustics and decoding
As was explained before, human beings use different mechanisms to achieve sound
localization. Different methods of sound localization are used in order to decide on a
”majority verdict”. This can be contradictory if used in isolation. This contradiction
in the localization also depends on the listeners’ experience, the expectations they
have and how they pay attention.
Gerzon’s [20] explains why both low frequency and high frequency decoding meth-
ods are needed in order to acquire the best possible localization.
As explained in previous sections, the localization of low frequencies is governed
by the phase difference or the so called ITD while at the high frequencies, the ILD
is more important. Gerzon proposed new theories and metrics to understand and
analyze this in a different way. It is thought that the total wave that arrives to the
ears, is the sound pressure that would be at the center of the head when no listener
is placed in the sound field; in other words, the one picked up by an omnidirectional
microphone placed in the same position. Thus, the remaining directional information
corresponds to the velocity of the sound field along the ear-axis (which corresponds
to the sound picked up by a figure of eight microphone). Then, the brain uses the
information picked up by an omnidirectional and sideways-facing velocity microphone
to localize sound. Note that it is assumed that no amplitude differences between ears
are considered (valid for low frequencies, below 700Hz) and that components of the
velocity microphone that are 90º out of phase with the omnidirectional, are not used
in deducing the directions of the given sound. Gerzon proposed the velocity vector
(rV) to characterize this theory.
For mid and high frequencies the phase relationship between ears is no longer
relevant. In this case the directional behaviour of the energy field around the listener
is what matters. Due to the nature of the energy, it is only possible to recreate the
exact energy field of a source through a loudspeaker array if the sound is located
at the exact position of one of the speakers.
Thus, in high frequencies, it is not
possible to satisfy all the ear localization mechanisms in a practical way. However,
it is possible to make a similar approach as in the low frequency case, and analyze
the directional energy field into omnidirectial and vector components. The subjective
sound direction can be estimated with these energy components (rE). The direction
can be calculated as the sum of the vectors pointing to each speaker, in which the
vector’s length is the energy of the sound from that speaker. The ratio of the length
of this vector compared to the total reproduced energy should be unity. However, in
practice, it is not possible to achieve this value and the larger it is, the better defined
the sound image. Taking all this information in consideration, it is clear that in order
to achieve a good spatial sound localization, both low and high frequency mechanisms
should be included in the decoding process.

2.4.6
Decoding
As explained before, the Ambisonics signals carry directional information but they
are not feeding any loudspeaker system, making them completely independent from
a certain loudspeaker layout. However, it is possible to manipulate these signals in
order to feed a specific loudspeaker array. The decoder is responsible for this. By
applying a linear combination of the different signals, it is possible to adapt the
sound field in different layouts. The only limitation is that the minimum number of
loudspeakers L should be equal to the number of Ambisonics channels M: L ≥M.
Gerzon has indicated that more stable sound images can be obtained if the number
of loudspeakers in the array is greater than the number of Ambisonics channels [12].
In theory, increasing the number of loudspeakers beyond the minimum requirement
can reduce the possible angle between a sound image and the nearest speaker. Thus,
the sound localization can be enhanced [21]. Still, it is found that high frequencies
can be damaged if the Ambisonics order is low and the number of speakers is high.
The basic aspects of Ambisonics decoding equations for 3D layouts are going to
be presented. It is assumed that L is the number of loudspeakrs, {bvj} is a set of n
directions defined by an azimuth and an elevation, Sp is a plane wave signal coming
from {bvj}, Sij are the signals of each loudpeaker i located at direction bui and the
Ambisonic channels expressed by Furse-Malham normalization.
As commented in the previous section, there are a series of metrics developed
by Gerzon in order to predict the localization. These metrics are simpler than us-
ing HRTF. There are two models, which are the simplest of these metrics, that are
characterized by the velocity localization vector rV and energy localization vector
rE which encapsulate the ITD and ILD. These vectors are defined by a direction,
which indicates the direction of the localization perception, and a magnitude, which
indicates the quality of the localization. The magnitude is the unity for the case of
a single source in natural hearing and the direction corresponds with the direction of
the source. The localization is assumed to be optimum when the direction of both
vectors coincides and the magnitudes are both unity. However, in practice it is not
possible to obtain rE = 1. Only it is only possible, when the sound is coming from
one loudspeaker [20, 22]. In general, the best solution is to maximize the energy
vector which leads into a compromising situation.

CHAPTER 3
Method
This chapter presents the methods and procedures used in the project. It shows how
the Acoustic Virtual Reality system is developed by describing the different meth-
ods and approaches. Moreover, it describes how the measurements of the acoustic
parameters were carried out and how the final system is validated.
3.1
Acoustic Virtual Reality development
This section contains the main core of the project, where the approach, design and
configuration of the method used to obtain AVR is explained. This method allows
the integration of auralization with VR so that acoustics and visuals are experienced
together. To fulfill such needs, a head-position-tracked auralization system was cre-
ated using virtual game features along with Geometrical Acoustics predictions and
audio signal processing. This process allows to recreate a virtual 3D sound field by
using pre-calculated B-format impulse responses, and decoding it to a certain virtual
loudspeaker array that, in the end, is virtually synthesized for a binarual listening
experience through the use of HRTFs.
AVR builds on the foundation of auralization, but the main difference is that in
auralization, the receiver is stationary and no head movement is allowed. AVR allows
the user to move with freedom through the modeled space with a complete free head
movement (6 DoF). These features add considerable additional value when applying
AVR to building acoustics as it allows to have a better idea about how visuals and
acoustics interact during the design process. Moreover, additional options are possible
such as instantaneous A/B comparison between different acoustic materials, which
allows the listener to experience the differences in a realistic way.
The approach followed to implement the AVR is explained in depth in this chapter.
However, the basic idea will be explained here as an overview for the reader. First, a
virtual model is built. Acoustic simulations are carried out in a grid of points (space
discretization). As the user walks through the virtual environment, his position will
be tracked and used to calculate the distance to each of the points that form the grid.
The convolved audio file that corresponds to the IR calculated for the closest point
will be played through a virtual loudspeaker array around the user that is always fol-
lowing him, so that the center of the array is fixed at the center of the user’s head. Of
course, interpolation is taken into account to provide a better result in between points.

second order Ambisonics B-format IR.
3.1.2
Previous considerations
Before explaining the designed system, it is important to mention the previous con-
siderations that were taken into account before getting the final idea of the project.
In the earlier steps, the Steam Audio plug-in was considered. This plug-in can be
added directly to Unity and implements Geometrical Acoustic methods that allows
the integration of the virtual model a fully-featured spatial audio. It improves the VR
immersion as it is physics-based sound propagation method, which interacts with the
virtual environment. It works in real time taking into account occlusion and reflection.
A simple model was implemented were the modeled space was an L-shaped room.
Different informal tests were conducted. It was concluded that the accuracy of the
tool works for video games, but it is not precise enough for architectural acoustics.
Two approaches were found: Real-time calculations approach, compute the
acoustics in the 3D model in real time. The Steam Audio plug-in is an example of this
method. The advantage is that no pre-calculations are needed. However, either the
computational load is heavy or the result is not very accurate. Thus, Pre-calculated
approach was preferred for this project. This approach consists of computing all the
previous IRs in a dedicated software such as Odeon. It takes long to make all the
computations, but once it is done, they can be saved and no more computations are
needed. Moreover, the result is precise and valid for acoustic purposes.

3.1.3.1
Acoustic simulation (Odeon) and Pre-processing
The first step when the AVR system is going to be used is to create a 3D model. For
this project, Sketchup is used due to its simplicity. The model of the room is then
exported to Odeon and to a 3D object that will be imported later in Unity.
Once the model is imported in Odeon and materials are assigned, the sources
and receivers can be placed. The receivers should be located so that a grid of points
is generated (Figure 4.15b). All the points should be separated the same distance
between each other. The resolution of the grid is investigated in the listening test
in the following chapters. Once the calculations are done, Odeon provides the IRs.
It is important to obtain the Ambisonics IRs in order to maintain some directional
information. Version 14 of Odeon allows first and second order Ambisonics, and both
are generated in FuMa format.
Odeon exports the IRs with a normalized amplitude, so that all the points have
the same maximum value. As the IRs are going to be placed in a grid of points, the
level difference between points needs to be taken into account in order to provide the
correct level on the corresponding distance to the source. The latest version of Odeon
(v15) provides a new feature which allows one to export a text file with the attenua-
tion factors for each point. This file is imported into Matlab in order to compute the
corresponding amplitude of each IRs just by dividing the IR by the attenuation factor.
Once the IRs have been corrected, they can be convolved with any anechoic sound.
The convolution is done for the 4 channels (for first order) or 9 channels (for second
order).
It is possible to use more than one source simultaneously.
In this case, it is
important to compute each combination of source and grid points separately. After
convolving all the combinations (all the points with all the sources), the final audio
file for each point is obtained by summing the different audio files of the different
sources. That can be easily understood with the following example. Assuming that
there are two sources S1 and S2 and a grid of four points (p1, p2, p3, p4), the IRs
corresponding to S1 and each point is IR11, IR12, IR13, IR14. In the same way, the
IRs of S2 and each point is IR21, IR22, IR23, IR24. Then, for two different anechoic
sounds x1 and x2 (coming from S1 and S2 respectively) , the convolved audio files y1
for S1, for each point are obtained as:
y1 = {IR11 ∗x, IR12 ∗x, IR13 ∗x, IR14 ∗x}

position (which is controlled with the Oculus Rift Controllers) is always tracked. This
information is used to perform an amplitude panning between the two nearest points.

3.2
Measurements
Objective acoustic parameters, more specifically T30 and/or T20, were measured by
capturing the impulse response of a room in several different points determined by
different source-receiver positions in compliance with ISO 3382-2 [28]. These mea-
surements were done in one room in order to recreate the space in AVR as it will be
explained in section 4.2.
3.2.1
Equipment
The measurement of the reverberation time of the room is conducted following the
standard [28]. A laptop with the acoustic software DIRAC from Brüel & Kjær was
used as a signal generator and processor. The sound source was a Dodecahedron
speaker connected to a LAB GRUPPEN 300 amplifier. The microphone was a Brüel
& Kjær 4192-L-001 (serial No. 3018512 with a sensitivity of 13.6mV/Pa) connected
to a Brüel & Kjær Nexus Conditioning Amplifier.
3.2.2
Procedure
In accordance to the standard, the sound source needs to produce enough level in
order to achieve a decay curve to start at least 35dB above the background noise for
calculations of T20 and 45dB for T30. Due to technical problems, it was difficult to
produce enough level in order to reach 45dB above the background noise, therefore
only T20 was considered for these measurements. In order to improve the signal to
noise ratio (SNR), an exponential sweet was chosen with a length of 5s. Each mea-
surement was averaged 5 times in order to obtain a better SNR.
An ”engineering” approach was follow, so that 6 source-receiver combinations
were used.
Specifically, two source positions and tree microphones positions were
used. The microphone positions should be separated at least 2m from each other
and preferably 1m away from any reflecting surface. It is also important to avoid
symmetrical positions [28].

objective of the present thesis is to make a system which puts together auralization
and virtual reality, it makes sense to investigate if when both together, they produce
any improvement in the perception of the acoustics.

3.3.2
Test 2: Spatialization
In this second test, the spatialization with different implementations is studied. This
experiment allows one to compare the perceived spatialization when using binaural,

allowed to move, so that the listening point was always fixed at the same position,

Finally, the last case allows the subject to move the head when visuals and sound are
presented.

5 degrees is used for the test. However, for the elevation, a resolution of 15 degrees
is used. In order to generate the different source position for each case, a function in
Matlab was implemented which provides random values of azimuth between -120 to
120 degrees with 5 degrees of resolution and random elevation values between 0 and
45 degrees with a resolution of 15 degrees.

CHAPTER 4
Results and discussion
This chapter presents the results obtained for the different parts of the project. The
first section of the chapter is focused on the analysis of the designed Ambisonics
decoder. The rest of the chapter presents different modeled spaces, starting from a
very simple room and ending with a real project that Henning Larsen is working with.
4.1
Ambisonics Decoder Analysis
In this section the different Ambisonics decoders used throughout the project are an-
alyzed and discussed. Though the most important decoder for this project is for the
second order Ambisonics system, built with an icosahedron loudspeaker array, other
decoders are also analyzed as they have also been used during the project. All the
decoders analyzed in this chapter are developed and presented in Appendix A.
The Ambisonic Toolbox [29] is used in order to generate plots of the different
decoders. There are different graphs for each decoder. First, the direction of rV and
rE to each loudspeaker is given. This is done assuming that for each value, the source
is only positioned in the corresponding speaker position. A plot with the differences
between both vectors is also presented.
This representation helps to know if the
direction of the vectors is correct and if there is some deviation between them. Here,
wrong values would mean wrong source location. For example, if the source is coming
from a single loudspeaker, and one of the vectors is not pointing to the corresponding
direction, the source localization would be wrong, as it would not come from the
speaker.
Then, two plots are presented with the magnitude of the velocity and energy vec-
tors. The magnitude is presented in an XY graph where the x-axis represents azimuth
and the y-axis the elevation. It is possible to see how the magnitude changes with
the direction. Notice that in some cases, especially for array configurations where the
speakers are not equally separated in all the axes, the magnitude can change with
the direction (as in the case of rectangular arrays). The pressure and energy gains
are also presented in these plots.

4.1.1
Decoder for an Icoshaedron loudspeaker array (first and

In this section, a second order Ambisonics for an icosahedron loudspeaker array is
studied. A first order version for the same configuration is also used and analyzed.
First of all, the first order Ambisonics’ basic decoder is analyzed. Figure 4.1 shows
the direction of the velocity and energy vectors and the difference between both. In
comparison to Table A.1, where the direction of the loudspeakers is listed, both vec-
tors are pointing to the correct directions. In addition, the difference between both
is zero. Looking at the magnitude of the velocity vector, plotted in Figure 4.2, it is
possible to confirm that for all the directions, rV = 1. This result was expected since
a basic decoder is used, which maximizes the unity to the velocity vector. However,
the magnitude of the energy vector presented in Figure 4.3 is rE = 0.5 for all the
directions. This value is almost in the limit, as Gerzon observes that ”values below
0.5 gives rather poor image stability” [12]. Looking at the max-rE decoder, some
differences can be noticed. First of all, the direction of the vectors is not changed,
thus no graphs are presented. On the other hand, the magnitude of both vectors is
changed. For the velocity vector, the value is decreased to rV = 0.5774 (Figure 4.4).
For the energy vector, the value is increased to rE = 0.5774 (Figure 4.5). This value
corresponds with the one given in Table 2.2 for the maximizing rE. Some observations
can be made when taking all this information together. The results are showing that
the basic decoder provides a larger magnitude value for the velocity vector (unity).
As it was explained in detail in section 2.4.5, this is important for the localization
at low frequencies. It also provides a lower value for the energy localization. On the
other hand, the max-rE decoder shows a higher value for the energy vector, which
is important for the localization at high frequencies but presents a value below the
unity for the velocity vector. This agrees with the theory, as the basic decoder can be
used for the low frequency part of the signal while the max-rE decoder can be used
for the high frequency part of the signal.

AVIL’s front room, DTU
Before implementing the designed tool in a real project, it is preferable to consider
a well known space in order to make a comparison between real and virtual spaces.
This approach can contribute to the detection of possible errors and problems. In
this section, a room was chosen and modeled in AVR. The process is similar to the
previous one for the rehearsal hall. However, other aspects have to be considered as
in this case, a real room is going to be modeled. Hence, the process is explained in
more detail.
Different DTU rooms were considered. After different examinations, the space
in front of the AVIL (Audio Visual Immersion Lab) in building 354 was preferred.
The space provides a low reverberation time, which is good in order to make compar-
isons between reality and simulations. Too much reverberation time in a room could
make the comparison difficult due to the fact that the sound field is diffused, while
rooms that have been acoustically treated could be harder to simulate. For this first
application a simple model was favored and the selected space being a good option.
Moreover, the T shape of the space is an advantage in order to test the accuracy of
the model around the corners.

simulated room in order to import it to the AVR system. It is important to make an
accurate model in Odeon as it will be compared to the real room in the AVR system
afterwards. In this subsection, the Odeon model will be created by importing the
geometry of the room and assigning materials to the surfaces. Then, the simulations
will be matched with the measurements by changing the absorption coefficient and
the scattering of the different surfaces of the room. Once the averaged measurement
values match with the averaged simulated ones, the model will be ready to generate
the impulse responses of a grid of points in order to use it for the AVR system.
The Sketchup model was imported into Odeon with the SU2Odeon plugin pro-
vided by Odeon. A visual inspection of the real room allowed to know what kind
of materials should be imported in Odeon for each surface. Then, the sources and
receivers used in the measurements (Figure 4.22) are placed in Odeon at the same
positions. After running the first simulation, the averaged reverberation time is pre-
sented and compared with the measured one in Figure 4.24a. As seen, there is not
a good match between the two, probably due to some differences in the material
properties. A fine tuning of the different materials was made in order to get a better
match between the both. The final material properties are presented in Table 4.3.
The adjustment was done manually by slightly changing the value of the absorption
coefficients. While there are no rules for making these changes, it is important to use
common sense and make the changes according to the real world. That means that,
knowing what kind of materials are present in the room and how they behave make
it possible to add more or less absorption in certain frequency bands. For example, if
the absorption needs to be increased in all the bands, it would not be wise to increase
the absorption coefficient in the high frequencies on a glass wall, as in the real world
it is a very reflective material at this region of the spectrum. However, it can be
possible to increase it in low frequencies due to the fact that boards behave as low
frequency resonators. This kind of approach has been followed with all the materials
in order to achieve the desired reverberation time.
Figure 4.24b shows that the adjusted simulated results are practically identical to
the measurements. Moreover, the difference was computed and expressed as just no-
ticeable difference (JND). The ISO 3382-1 [30] lists the JND of reverberation metrics
at 5% based on work by Seraphim (1958). That means that if the difference between
measurements and simulations is less than 5% (1 JND), it will not be noticeable.
Figure 4.25 shows that for all the bands the JND is less than 0.5, which is a very
good match.
Once the Odeon model is matched with the measurements, it is ready for adding
the grid of points that discretizes the space. For this model, a grid resolution of
1.2 meters is chosen, providing 65 points which is shown in Figure 4.26. The grid
resolution of this space will be studied in the listening test in this chapter.

Henning Larsen project: Uppsala’s City Hall
The final step of this project is to apply the designed AVR system to a real project.
For this task, Henning Larsen proposed the City Hall in Uppsala (Sweden), which is
a real project taking place in Henning Larsen. Uppsala City Hall is expected to open
in its new shape in 2020. The existing City Hall in Uppsala from 1957 is going to be
extended. The new building adds a modern quality but is designed with respect for
the original version. The project consists of a big atrium with a building inside. The
atrium is divided into two areas. One half is public, where pedestrians are allowed to
walk through. The other half is private, and it is only accessible through the reception.
As the atrium walls are mainly made of glass, the reverberation time seems to be
high. An Odeon model was made in order to estimate the acoustics. After computa-
tions, the resulted reverberation time was around 8s. As this value is very high, some
acoustic treatment were designed by Henning Larsen which is mainly focused on the
windows and the inside building facade. A new simulation was carried out in Odeon.
In this case the resulted reverberation time was reduced considerably. Figure 4.30
presents the reverberation time for both cases.
This part of the project is not focused on the acoustic treatment of the atrium, but
rather in the implementation of the AVR system using the results. Thus, Henning
Larsen has provided the CAD files and the Odeon model. The first step is to create
the grid of points by placing the receivers. For this model, four sources are going to
be used. Figure 4.27 shows the model with a total of 67 receivers spaced 5m apart
from each other. This resolution was chosen due to the large dimensions of the model.
Finer resolution implies more points and more computational time. Also, Pure Data
can be saturated with an excessive number of audio files.
As more than one source is used, it is necessary to compute so many grid of points
as sources. For this case, the simulations was run four times. Then, the pre-processing
part was also run four times. Finally, the four sets of audio files, corresponding to
the four different grids are summed point to point. The result is a single grid with
67 points which contains the information of all the sources.
5 meters can be considered a coarse grid size. Big separations between points can
spread the source, producing a bigger apparent source, which can be a problem in
certain cases. However, for this model, the idea was to simulate how the acoustic
behaves with the noise of people inside the building. Thus, the audio sources consist
of people speaking, laughing and making noises. In this case, the fact that the source
is spread is not a problem. In order to match visuals with acoustics, silhouettes of
people were placed around the source positions, covering the apparent source size.
Note that using silhouettes of people is preferable to using more realistic human
models, thus avoiding the uncanny valley effect which produces a huge decrease in
the realism when humans are modeled in 3D with high precision but no animations
are applied.

For the scenarios in which visuals are allowed, something curious happens. In this
case, the subjects were allowed to see different sources but only one was correct. For
the big room cases, the results are close to the source position values. However, in the
other two rooms the results present big deviations, especially in the medium room.
This can be explained due to the fact that if the subject is confused and perceives the
source in a wrong direction, then the visuals have a negative influence on the source
location. That can be explained with a simple example, assuming that the source is
located at 15 degrees azimuth and 0 degrees elevation. If the subject perceives the
source at 5 degrees azimuth and two sources are placed, one at 15 degrees and other
at -5 degrees azimuth respectively, then the subject can be influenced by the visuals
and choose the wrong one due to the confusion.
Finally, the last scenario corresponds to the case when both, visuals and head
tracking are allowed. These results present the closest values to the position of the
sources for all three rooms.
Putting together all this information, it can be concluded that having visuals and
head tracking together with auralization, considerably improves the sound localiza-
tion. Moreover, in most cases, the results seem to be confusing when only binaural
sound is presented. This is probably due to the fact that human beings use head
movement (even if it is small) to localize the sound. It is important to mention that
in general, the standard deviation of the elevation (Figure B.1) presents higher values
than the azimuth. This can be explained by the fact that the human beings are less
sensitive to the median plane when locating sounds. Moreover, the resolution chosen
for this test in the elevation was 15 degrees, while for the azimuth it was 5 degrees.
Also the elevation was taken between 0 and 45 degrees. Thus, the elevation differ-
ence presented in Figures 4.31, 4.32 and 4.33 is lower than the one for the azimuth.
Moreover, in general, second order presents better results than first order even if in
some cases it is not so clear.
However, in order to obtain more reliable results and make stronger conclusions,
a proper test should be carried out following the same method as the presented pilot.

by the interpolation made in Pure Data. When the user walks around the virtual
space, the new volume value used to reproduce the sound in the different points is
interpolated with the previous one. That means that a fade in and fade out is made
in order to make a smooth transition and avoid audible steps. Because in this case
the points are very close to each other, the fade in/out time has to be shorter than in
the other grid sizes. A long fade time can produce an audible step transition, which
produces an unreal feeling. Thus, this time should be adjusted if a real test is going
to be carried out.

4.6
Discussion
In this section, a brief discussion is going to take place by having an overall view of
the previous results.
First of all, looking at the theory part and the test results, it can be concluded
that second order Ambisonics is a better choice for this project. The higher the order,
the better the sound reproduction is. Notice that equation (2.13) is an approxima-
tion of the sound field up to a certain order. The results given by the spatialization
test agree with this by showing, in general, more precise results for second order.
Moreover, subjectively speaking, it is possible to notice differences between first and
second order Ambisonics. For the the first one, the sound sources results are more
spreaded and blurred while for the second one, the sources seems to be better defined.
On the other hand, different loudspeaker arrays could be also studied. However, one
of the reasons why it has not been done is because of the high number of loudspeakers
that other options provide. The number of loudspeakrs has to be higher than 9 and
the geometry has to be defined by a regular polyhedron. That means that for the
second order the given options are an icoshaedron, which provides 12 vertices (and
loudspeakrs) or a dodecahedron, which provides 20 vertices (and loudspeakrs). Thus,
the dodecahedron was not tested due to the high number of loudspeaker signals that
has to be processed in real time.
It is also possible to make some conclusions looking at the different rooms modeled
during the project. The first modeled space was the rehearsal hall. In this model
it is possible to notice the differences when the acoustic materials are present and
when they are not. Subjectively speaking, the localization works well, as it is possible
to always precisely localize the source.
It is also important to highlight that the
delay time when changing from one case to another is negligible and change can be
considered as instantaneous. This was one of the main priorities when designing the
AVR. A big delay can ruin the comparison and change the perception of the user.
The modeled space in DTU was used in order to make comparisons with a real
space possible. Of course, the simulation is limited to the precision of the simulation
software. From here, it can be concluded by subjective exploration that the AVR
model works similar to the real space even if there are some differences. One of the
main differences were found in the corner of the L shape space. There, no diffraction
is produced so that as the user moves around the corner, the feeling is not as real
as it should be. This is due to the fact that the simulations were carried out with
Geometrical Acoustics methods which does not take into account diffraction. However,
the designed AVR system allows one to used all kind of impulse responses so that
different methods can be combined. Also, real measured impulse responses can be
used.
Finally, the Uppsala’s City Hall was modeled in colaboration with Henning Larsen.
The main purpose of this model was to show how the AVR system can be applied
to a real project to experience how the acoustic materials affect the acoustics and

visuals. As it was explained before, four sources were placed so that different noises
of people can be simulated. Subjectively speaking, there is a huge difference between
both cases. The case without treatment presents higher sound pressure levels around
the space, with a very long reveberation background which can be very inconvenient.
The case with acoustic treatment presents a huge improvement. The overall sound
pressure level is reduced and the background reverberation disappears.
The visu-

presents a very good solution, where the acoustics is highly improved and the visuals
do not cause discomfort.
It is also important to discuss the size of the grid.
Even if the listening test
presents different results from the expected ones, the reason has been explained be-
fore. Thus, it is expected to produce more realistic simulations with a finer grid. The
grid of points is a discretization of the space. That means that a finer grid presents
more information and therefore is more precise. However, the finer the grid, the more
audio files have to be processed in real time which can overload the computer. On
the other hand, a coarse grid avoids overloading the computer as it uses less audio
files. However, big separation between points can produce a lack of acoustic infor-
mation that could be crucial to detect certain problems such as echos, flutter echos,
etc. Moreover, a coarse grid tends to spread the sound sources. Subjectively speaking
it feels as the sound source is bigger than what it was during the simulation. This
apparent source size can be a problem in certains cases. Thus, when choosing the
grid size it is a matter of compromise.
Summarizing, in general it can be concluded that the AVR system can generate
models that fulfill the function for which it was designed: provide a virtual real-
ity acoustic auralization suitable for architectural acoustics where visuals and acous-
tics can be experienced simultaneously in a completely free movement environment.
Therefore, this fulfills all the research objectives exposed in the beginning of the thesis.

CHAPTER 5
Conclusion
The main goal of this thesis was to investigate and design a system to combine the
acoustic auralization with the VR. In this study, implementing the AVR so that the
user can experience both, VR visuals and auralization through accurate acoustic sim-
ulations was found to be a solution. Different tools have been used. The acoustic
simulations were carried out with Odeon. The space is discretized in a grid of N
points providing N impulse response files. Second order Ambisonics has been used
in order to keep the spatial information. Then, the impulse responses were convolved
in Matlab with the desired anechoic sound. The visuals are managed by Unity and
Oculus Rift. The position of the user in the Unity 3D model is always tracked, which
allows one to compute the distance between each point of the grid. This informa-
tion is sent by UDP to Pure Data. A patch takes all the received information and
reproduces the corresponding audio file to the user position. Then, the sound field
is rotated according to the head position of the user. The B-format is decoded to
a virtual loudspeaker array around the listener. Finally, each loudspeaker signal is
convolved with the corresponding HRTF to provide a binaural signal.
Different Ambisonic decoders are analyzed. The conclusion agrees with the the-
ory. A better localization can be achieved when using two different decoders, a basic
decoder for the low frequencies and a max-rE decoder for the high frequencies. Thus,
the signal is divided, decoded and reconstructed. This allows one to provide the dif-
ferent mechanisms that human beings use for localizing sounds in the low and high
frequencies (ILD and ITD). It is important to highlight that the high frequency de-
coder is not able to maximize the energy vector to unity. So that the larger the better.
However, it was shown that using second order Ambisonics with an icosahedron loud-
speaker array, the energy vector presents values close to unity.
Two pilot listening tests were carried out. The first one consists of testing how
well the sound localization works for different scenarios. It was concluded that the
best results were obtained when using visuals and head tracking. Thus, it can be
concluded that the AVR improves when the listener is immersed in the acoustic sim-
ulated field. The purpose of the second test was to test the realism of the virtual
model for different grid sizes. It was concluded that a medium grid provides better
results. However, it was expected that the higher the grid, the better. The reason for
these differences were as followed: for small grid sizes, the Pure Data patch needs to
be adjusted. It is important to keep this in mind if a real test is going to be made.

5 Conclusion
Further work can involve carrying out a proper test to extract more reliable conclu-
sions. That implies testing a higher number of subjects and correcting the problems
found for the fine grid. Another idea to help improve the project would be to improve
the visuals, as it was found that the visuals help the subjects become more immersed
in the model. Unity allows one to create very impressive visuals by using lightmaps.
That was beyond the scope of this project, but can definitely add a positive value to
the final result. It is known that the 15th version of Odeon will provide third order
Ambisonics. Thus, a new decoder can be designed by using third order and choosing
another loudspeaker array that fulfills the requirements. Based on the results ob-
tained in this thesis, using higher orders will improve the final result and the sound
localization.

APPENDIX C
User Guide
This Appendix presents a user guide to provide the necessary information to start
building and using an AVR model from scratch. Not all the processes are going to be
explained (as for example how to work in Odeon) but the more relevant steps that
concern the designed system.
1. Sketchup 3D model
First, a 3D model needs to be created. This task can be done easily with Skecthup
as it is shown in Figure C.1. Once the model is built, it can be exported to a 3D
object: File →Export →3D Model.
Odeon provides a plugin for Sketchup in order to export the model into Odeon.
The plugin can be downloaded for free (SU2Odeon).
Figure C.1: Sketchup 3D model.
==PAGE:112==
94
C User Guide
2. Odeon: Acoustic simulations
Once the model is imported to Odeon, the materials can be applied to each surface.
Then, the sources and receivers should be placed so that a grid of points discretizes the
space as it is shown in Figure C.2. For the actual room with dimensions 30×20×10m,
a grid of 1, 2m is chosen.
Odeon normalizes the exported impulses responses.
In order to keep the real
amplitude (to consider the distance attenuation) it is important to export the atten-
uation factors. Version 15 allows that. On the Job List menu , click on job. On the
top menu: Job List →Exctract attenuation factors for IR’s and save in file (Figure
C.3). This generates a text file with the corresponding attenuation factor per receiver.
Finally, the computations can be carried out.
Figure C.2: Odeon, grid of receivers.

the compensated impulse responses which are saved in the folder called IRAT.

In order to configure the audio part, a new folder has to be created which con-

folders: files_A and files_B. The content of the folder is shown in Figure C.8. This

Figure C.8: Pure Data files folder.
5. Unity
Together with this report, a Unity template is given. This includes the basic con-
figurations as for example the Oculus Rift packages and the scripts to communicate
with Pure Data. Also the grid of points is already created. The Sketchup model is
imported into Unity by: Assets →Import New Asset. It is recommended to import
a fbx file, as it is possible to export double sided faces, and materials.
Next step is to add materials to the surfaces. Assets →Create →Material, allows
one to create new materials. It is also possible to import images with textures to
create textured materials. Then it can be assigned to each surface by dragging it to
the corresponding surface.
By adding lights, it is possible to create the desired atmosphere: GameObject →
Light, then, choose the desired light.
Once the visuals are created, it is important to configure the technical part. First,
the grid of points have to be created (if not using the given template). Each point is
represented by a sphere which can be created by: GameObject →3D Object →Sphere.
