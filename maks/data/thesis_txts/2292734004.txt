Abstract
Information retrieval is concerned with ﬁnding relevant information from a large collec-
tion of documents given a search query. We present a novel method for information retrieval
that computes a probability distribution over document labels, which can be used for rank-
ing documents, given a text search query. This method process raw character-level text
queries, avoiding any need for preprocessing, and map character-vectors to context-aware
word-embeddings which are then mapped to a vector representation of the whole text. Our
method is based on deep learning; a machine learning framework using mutiple layers of
processing units with non-linear transformation functions. We trained our model on a cor-
pus of 14,261 articles concerning medical diseases and conditions from the web search engine
FindZebra.com. We show that our model is able to learn to ignore stop words and notice
key words in a query, and automatically captures the semantics of medical terms. On a set
of 248 test queries we were able to demonstrate performance on pair with current state-of-
the-art, but more robust to spelling errors. Moreover, by combining the current FindZebra
seach engine with our method were able to show a signiﬁcant performance boost.

Introduction
Traditional Information Retrieval and Neural Language Models
Information retrieval (IR) is the act of retrieving a set of relevant documents from a collection of
documents given a query. Web search engines is the most common application for IR systems.
In order to display only the most relevant documents to the user, the retrieved documents
must be ranked by relevance.
Hence, we have to represent both documents and queries in
a way that allow us to measure the relevance of a document to a query.
A traditional way
of representing documents is the bag-of-words model which is a sparse vector representation
with occurrence counts of the terms in the document where each dimension corresponds to a
separate term. Classical methods for measuring relevance between query and document based
on the bag-of-words model include Standard Boolean models (logical matching between query and
documents) and Vector Space Models (distance measures between query and document). Bag-
of-words based models are computationally very eﬃcient and can scale to millions of documents.
However, they suﬀer from the fact that the information about the ordering of terms is lost in the
representation. This limitation can be combated by using probabilistic markov models (N-gram
models) over sequences of length N. However, for these models, the possible word combinations
(N-grams) grows with V N where V is the size of the vocabulary and thus, suﬀer from the curse of
dimensionality. Another weakness of both bag-of-words and N-gram models is that they ignore
the semantics of the words (same meaning but diﬀerent words).
In 2003 Bengio et al. (2003) introduced a so-called neural language model that overcomes
the curse of dimensionality problem that N-gram models suﬀer under.
This model learns a
distributed representation for words (a real valued feature vector for each word in a vocabulary;
word embeddings) by ﬁtting a probabilistic function to a sequence of words given the previous
words (context), similarly to the N-gram model. The diﬀerence, however, is that the neural
language probabilistic function is modelled by a neural network1 with shared parameters across
words in the context. The model learns the parameters of the probabilistic function and the word
features vectors simultaneously.
Since then, deep neural network-based models have become
increasingly popular in natural language processing and information retrieval (LeCun et al.,
2015; Le and Mikolov, 2014; Kim, 2014).
The FindZebra Search Engine and Deep Supervised Learning of Text Documents
Most information retrieval systems are based on so-called unsupervised methods. That is, docu-
ments have no labels, and hence, document relevance to a query must be based on some similarity
measures. However, for IR systems with labeled documents, it is possible to apply supervised
methods that predicts a document label, given a search query. FindZebra ( www.findzebra.com)
is a web search engine for rare diseases (Dragusin et al., 2013). FindZebra only retrieves docu-
ments from high quality sources for medical information. Each document is labeled with a disease
label. In this thesis, we present supervised models that computes a conditional probability dis-
tribution over these document labels given an input query. These models are based on deep
learning; deep architectures composed of multiple processing layers. Our models process raw
character-level text avoiding any pre-processing steps. We show that with a speciﬁc architecture
that maps from a distributed representation of characters to word vectors and then from words

Basic Supervised Deep Learning
Deep learning is a set of machine learning algorithms that models complex architectures composed
of multiple layers of nonlinear-processing units inspired by how the brain is believed to process
information (Bengio et al., 2015; Murphy, 2012; Deng and Yu, 2013). Dependent on the modelling
task, these algorithms can broadly be divided into 3 categories (Deng and Yu, 2013);
• Deep supervised or discriminative models, where we map input data {xi}N
i=1 to some targets
{yi}N
i=1. That is, our model tries to directly ﬁt a posterior distribution p(y|x).
• Deep unsupervised or generative models, where we try to model the input data. That is,
the model tries to ﬁt densities over the input data. If the targets are available then, using
Bayes’ rule, these model can also be used to output a posterior distribution.
• Deep hybrid models. These are deep discriminative models which are somehow assisted by
some unsupervised learning methods (including reinforcement learning).
The proposed models in this thesis are deep discriminative models.
Therefore, we will only
consider these types of models in the following section. We will give a brief background in deep
supervised learning and the most commonly used architectures. The terms; model, architecture
and network will be used interchangeably.

2.5
Convolutional Neural Network
Convolutional neural networks (CNNs / ConvNets) are deep learning architectures with one
or more convolutional layers typically followed by a pooling/sub-sampling layer (LeCun et al.,
2015). The two ﬁnal layers of a CNN is typically a MLP. CNNs can be applied on input data
with a grid-like structure such as images or time-series. A sentence or a document of text has for
example a 2D grid-like structure if each word or character is represented as a numerical vector.
CNNs have shown state-of-the-art performance in tasks such as image, video, speech and audio
processing and recognition (LeCun et al., 2015; Schmidhuber, 2014).

2.7
Long Short-Term Memory
Long short-term memory (LSTM) is a RNN model containing so-called memory units which over-
come the vanishing gradient problem. On various synthetic tasks, such precise timing, counting
and memorizing long strings, LSTMs has been shown to be able to learn long-range dependencies
where other RNN models could not (Schmidhuber, 2014). Moveover, LSTMs and variants thereof
has been applied successfully with state-of-the-art performance on tasks such as machine trans-
lation, robot control, protein structure prediction, speech recognition, handwriting recognition
and time-series prediction (Schmidhuber, 2014; Greﬀet al., 2015).

2.11
Eﬃcient Implementation
Training deep learning models is very computationally expensive, and much of the success of these
models have been credited to recent advances in computational power and graphics processing
unit (GPU) computing (Schmidhuber, 2014). Element-wise activation functions, hadamard prod-
ucts, matrix-multiplications and convolutions are embarrassingly parallel operations, especially,
when performed with large minibatches of input data with large input spaces. Hence, GPUs
with thousands of processing cores can speed up training time of large deep learning models with
a factor of ∼20 compared with multicore CPUs. Therefore, GPU computing is essential to deep
learning.

3.1
Word-level Text Modelling
Most previous work on text classiﬁcation has been done an a word-level. The following section
will give a brief overview of recent contributions to deep learning for word-level text classiﬁcation
and information retrieval.
Recursive Neural Tensor Networks
In order to capture the meaning of longer phrases Socher et al. (2011) introduced so-called
Recursive Neural Networks. These models are trained on phrases which are parsed into sub-
phrases in a tree-like structure by human annotators. At test-time the model is then able to
construct a parse tree and capture the compositionality of words in phrases in vector space which
can then be used for tasks such as sentiment analysis. A speciﬁc recursive neural network called
Recursive Neural Tensor Network (RNTN) achieved record-breaking performance on sentiment
prediction of movie reviews ( Socher et al.) in 2013. More recently, LSTMs on tree-structures
were introduced (Tai et al., 2015; Zhu et al., 2015) demonstrated even higher performance on
sentiment classiﬁcation.
Word2Vec and Paragraph Vectors
A widely used unsupervised method for mapping words to a vector space is the Word2Vec model
(Mikolov et al., 2013), which is very similar to the ﬁrst neural-language model introduced by
Bengio et al. (2003). This model comes in 2 variants; continuous bag-of-words (CBOW) model
and continuous skip-gram model. The CBOW tries to predict a word given it’s context. That
is, given a sequence of one-hot encoded words {x(t)}T

Le and Mikolov (2014) showed that the Word2Vec model can be extended to sentences and

both the word embeddings and a unique learnable vector for each paragraph as inputs. The
paragraph vectors can then be used as input features for task such as sentiment analysis and
information retrieval. Using paragraph vectors and a simple logistic layer Le and Mikolov (2014)
was able to outperform much more complicated models such as RNTN.

Character-level Language modelling
Character-level text modeling using RNNs were ﬁrst introduced in 2011 by Sutskever et al. (2011),
with the goal of predicting the next character in a stream of text (similar to other language models
such as the N-gram and Neural language model, but on characters instead of words). Formally,

and long-range dependencies on a character-level. Recently, Kim et al. (2015) where able to
demonstrate state-of-the-art language modeling performance using character-level inputs and
word-level predictions by combining a CNN with a LSTM.
Character-level Text Classiﬁcation
To our knowledge, character-level document classiﬁcation using deep learning have only been
demonstrated by Zhang et al. (2015). They recently showed that CNNs can eﬀectively be used
to extract information from raw text at a character-level in order to classify documents. These
character-CNNs achieved competitive performance compared with word-level models.
Character-level Neural Machine Translation
A very recent paper (Ling, 2015), yet to be published, apply character-level models to the task
of machine translation. Similarly to the approach used in this thesis, they map a character-
sequences into a word-sequence using a BLSTM model and whitespace boundaries to build
word-representations. Their model achieved performance on pair with word-based models.

6.1
Performance Evaluation
Validation and Test Queries
In order to estimate the performance of our classiﬁer we used queries from (Dragusin et al.,
2013) and (Svenstrup et al., 2015). Dragusin et al. (2013) created a set of 56 search queries
and answers based on diﬃcult clinical cases, primarily extracted from other journal articles.
Svenstrup et al. (2015) obtained 546 question and answers for rare diseases from a set of 3000
medical questions from the American College of Physicians (ACP). We merged these two sets of
queries and excluded those where the answer did not have a corresponding CUI present in our
corpus. We then split the queries into a validation and test queries, randomly assigning each
query to one of the sets. This left a total of 248 validation queries and 248 test queries. The
validation queries were used to select model hyperparameters and the test queries were used to
asses performance.

Benchmark
The current version of the FindZebra search engine was previously shown to outperform both

symptoms (Dragusin et al., 2013). We will use the current version of FindZebra as a performance

benchmark for our proposed models. Currently, FindZebra utilizes the open-source enterprise
search platform Apache Solr12 which is built on the information retrieval software library Apache
Lucene13. Solr provides a state-of-the-art high-performance full-featured text search engine. It
uses a combination of the Vector Space Model14 and the Boolean model15 to ﬁnd documents
relevant to a query. The boolean model is used as a preprocessing step to exclude documents
that does not contain any of the words in the query. Subsequently, the remaining documents
are scored on the basis of how many times query terms appear in the documents in order to
compute (dis)similarity/distance measures between query and documents (bag-of-words vector
space model).
6.2
Training Procedure
Optimization scheme
We train models using the stochastic gradient descent scheme ADAM (Kingma and Ba, 2015)
with default settings. Our mini-batch size was set between 192 and 320 snippets depending on
model size.
Dropout regularization
In our initial trials we quickly found that dropout regularization signiﬁcantly increased valida-
tion performance. Therefore, we applied dropout with a dropout-ratio of 50% at the following
locations of our models:
• Chars2Doc CNN:
We apply dropout at the output of the ﬁrst fully-connected layer (15).
• Chars2Doc BLSTM:
We apply dropout to the character embeddings (14).
• Chars2Words2Doc BLSTM
We apply dropout to the character embeddings (14), the word embeddings (19) and at the
output of the fully-connected layer in (23)
Number of training iterations and Early-stopping
In order to set the number of training iterations (parameter updates) and avoid overﬁtting we
apply so-called early-stopping; we monitor the performance on the validation set and while the
performance continues to increase we continue iterating. A detailed description of our early-
stopping procedure is provided in the appendix A.3

single times-step for a word in our character-level BLSTM layer, whereas in a word-level model,
as used in Solr, a misspelling might result in the word being totally useless.

Character Embeddings
We chose to train our models on both upper and lower case letters because in the context of
medicine the semantics of some terms might depend on the letter case (e.g. genes and other
abbreviations). Therefore it is interesting to see whether the upper and lower case of the same
letter is in close proximity to each other in the character embedded space. We applied kernel PCA

8
Conclusions and Future Work
In this thesis we presented a novel method for IR that learns to compute a probability distribution
over document labels, given a search query. This method process raw character-level text se-
quences, avoiding any need for preprocessing, and map these to context-aware word-embeddings
which are then mapped to a distributed representation of the whole text. We demonstrated
state-of-the-art performance and robustness to spelling errors.
Moreover, by combining cur-
rent state-of-the-art bag-of-words method with our method we showed a signiﬁcant performance
boost.
Currently, our method only works with labeled documents. Many of the labels for the set
of documents used in this thesis where unique for a given document. Hence, our method would
probably still work even if each document was treated as a unique label. However, our classiﬁ-
cation method is not likely to scale to millions of documents, due to long training times.
Future work could focus on optimizing our method. For example, exploring more model sizes
and combinations or training on only lower case letters. Future work could also be dedicated to
using our approach for unsupervised training. Speciﬁcally, we could try to map each of our word
embeddings in a document to the surrounding words, similar to skip-gram Word2Vec (Mikolov
et al., 2013), but instead of a softmax cross entropy error we would use a distance measure. Sub-
sequently or possibly simultaneously, we could train a BLSTM document autoencoder (Li et al.,
2015; Sutskever et al., 2014) using these word embeddings. Then using some similarity measure,
an IR system would rank documents based on the similarity between query and documents in
the hidden space.
