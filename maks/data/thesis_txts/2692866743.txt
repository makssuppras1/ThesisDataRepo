Abstract
Travel demand estimation is crucial to ensure reliable transportation network analysis.
Since the traﬃc demand is not directly observed, for each traveler, we have to estimate traﬃc
demand from other data and information available (e.g. traﬃc surveys, traﬃc counts and
smartphone data). This makes the OD matrix estimation a challenge.
This work presents a method to solve high-dimensional calibration problems applied to
large-scale simulation-based traﬃc networks. The method is a simulation-based optimization
(SO). The method is validated in a simple toy network. After that, it is applied to a large-
scale traﬃc network of Porto and compared against a benchmark calibration algorithm. It
is shown that the proposed approach is able to outperform the benchmark by one order of
magnitude.
The proposed approach includes an analytical traﬃc model deﬁned as a system of non-
linear equations that is diﬀerentiable and provides the SO algorithm with structural infor-
mation. This information enables to solve the high-dimensional calibration problem in few
iterations, therefore being computationally eﬃcient.

Introduction
High-resolution urban traﬃc and mobility data is becoming increasingly available world-
wide. This has sparked an interest in the development of traﬃc models to inform the design
and operations of urban mobility networks. In order to enable the use of these high-resolution
traﬃc models, there is a need for building robust calibration and validation tools that ensure
that the models ﬁt with the observed data.
Calibration problems face some diﬃculties (e.g. high-dimensional, stochastic, simulation-
based and non-convex) and there is a lack of algorithms that can eﬃciently address these
challenges.
The problem of model calibration has been widely studied in the literature.
Nevertheless, current practices rely on expertise with heavily supervised approaches. The
solutions consume a lot of time and computation and are not easily transferable.
This thesis focuses on the calibration of the Traﬃc Demand.
More speciﬁcally, the
calibration of the OD ﬂows. Other works focus on the calibration of other parameters. From
the demand side, behavioral parameters such as the ones used in mode and route choice
models. On the other hand, parameters used in the supply side can also be calibrated (i.e.
the parameters used for car-following or lane-changing models)
The calibration performance is validated by comparing the simulation performance and
the observed values from real the traﬃc network. The most accessible data is usually vehicle
loop counts and public transport counts. After the simulation is calibrated it can be validated
with other real data observations, e.g. point-to-point travel time and stop-to-stop ride time.
The main objective of this thesis is to develop a method that guarantees global optimum
in a reduced number of iterations, therefore being computationally eﬃcient.

2
Calibration
Technology improvements in computing and power memory size have increased the sim-
ulation of complex stochastic systems in most engineering ﬁelds. In transport engineering,
the agent-based and microsimulation of traﬃc ﬂows are growing in importance. The applica-
tion of such techniques has shifted from low dimensions and complexity, such as isolated toll
plazas and intersections to complex networks. The wide use of such techniques calls for care-
ful inspection of their ability to accurately represent the systems under analysis. Calibrating
or identifying the model parameters is, therefore, a crucial issue. This problem is generally
solved as an optimization problem where the objective function to minimize represents the
distance between the real observed traﬃc data and the simulation outputs.
The simulation model is not analytical, hence the objective function f is in the form of a
black-box model - given a set of parameters β and its feasible domain, the simulation outputs
f(β). This problem is generally referred to as a black-box or simulation-based optimization
and it is a constrained nonlinear, nonconvex optimization problem.
In traﬃc microsimulation, every single evaluation of the objective function is very costly,
depending on the size of the network and the number of vehicles simulated. The cost of each
evaluation of the objective function is high, as it requires to run the simulation. Further-
more, the optimization problem has computational complexity due to the high-dimension of
the parameter vector. This has lead the calibration studies towards the use of heuristic op-
timization techniques. The most common approach is simultaneous perturbation stochastic
simulation (SPSA) algorithm [Spall, 1992]. This method computes an approximation of the
gradient function after perturbing all the parameter values simultaneously. It uses informa-
tion in the descending direction to obtain a new parameter set. SPSA has a relatively low
computational eﬀort as it only requires two function evaluations to compute the gradient.
The genetic algorithm (GA) has also been used [Kim et al., 2001]. These general-purpose
algorithms are guaranteed to achieve asymptotic convergence properties for a wide range of
problems. However, this generality comes with a lack of computational eﬃciency because
the algorithms are not designed to identify good solutions fast.
An alternative approach to the problem resolution is to develop models that can predict the
simulation output as a function of the parameters. This model is an approximation of the
simulation which is easier to optimize. At each iteration, the simulation is run to get one
observation data point (input-output) and the current and previous observations are used to
train the model that is a simpliﬁcation of the simulation function. Once the model is ﬁtted
with the observations, it is optimized to obtain a new parameter set. Adding this extra
Page 2
==PAGE:6==
step to the optimization problem increases the complexity and computation time at each
iteration. Nevertheless, this approach reduces the number of iterations needed and therefore
the number of total simulation evaluations.
Table 1 shows a summary of diﬀerent Simulation Calibration methods used in literature
in recent years. The chosen methods are all oﬄine calibration, not real-time as it is out of
the scope of this thesis. The W-DSPSA [Oh et al., 2019] is a variant of SPSA that incorpo-
rates spatio-temporal correlations between parameters and measurements in approximating
the gradient, thereby minimizing the impact of uncorrelated measurements. The enhanced
algorithms outperform the original SPSA showing improved robustness and convergence.
Another approach is the Gaussian Process [Sha et al., 2020, Schultz and Sokolov, 2018], the
idea is to use all the available information from the current and previous evaluations of f,
and not simply rely on local gradient and Hessian approximations. This makes the algorithm
able to ﬁnd the minimum of diﬃcult non-convex functions with relatively few evaluations,
at a cost of performing more calculations to determine the next point at each iteration. A
similar approach is to use a metamodel [Patwary et al., 2020, Osorio, 2019, Osorio, 2013].
The metamodel replaces the simulation-based objective function with an analytical objective
function. The key component is the use of problem-speciﬁc, analytical and diﬀerentiable,
approximation of f. The goal is to provide structural information of the entire feasible region
with few simulation runs.

2.1
Calibration parameters
Eﬃcient traﬃc simulators have been designed to capture the interactions between trans-
portation demand and supply. Demand models estimate and predict origin-destination (O-
D) trip patterns and simulate the behavior of individual drivers (including pretrip departure
time, mode and route choice, and response to information). Supply models capture traf-
ﬁc phenomena through detailed representations of the capacities of the network; the traﬃc
dynamics resulting from speed and acceleration, lane changing, and merging and weaving
behavior; and the impact of incidents.
Each of these model components requires a set
of parameters and inputs whose values must be deﬁned or initialized before model execu-
tion. Various algorithms then tie together the demand-and-supply components to assign the

2.1
Calibration parameters
dynamic demand to the network and determine the temporal propagation of ﬂows. The re-
sulting traﬃc conditions (including speeds, densities, travel times, and delays) may be used
for a variety of planning and real-time management applications.
The main challenge then becomes the calibration of the supply and demand parameters
to reﬂect the real phenomena. Diﬀerent requirements are expected for dynamic traﬃc as-
signment models (DTA) and for microscopic traﬃc simulation. For example, DTA models
usually utilize mesoscopic demand and supply simulator components, that employ a mix of
microscopic and macroscopic models to capture the decision of the travelers and the move-
ment of vehicles throughout the network. They consider the (often thousands or tens of
thousands) OD ﬂows in the network as inputs that need to be calibrated. Similarly, on the
supply side, segment output capacities are among the parameters that need to be calibrated,
and these are easily in the order of thousands. Microscopic traﬃc simulator models also
require OD ﬂows as inputs, but on the supply side, they require a much smaller number of
parameters to be calibrated (used in the individual models, such as car-following, merging,
lane–changing).

pair. The objective function f has two summation terms. The ﬁrst term represents the
distance between the traﬃc conditions estimated by the simulator and the traﬃc conditions
described by ﬁeld measurements. More speciﬁcally, a set L1 of links are equipped with sen-
sors. These links have an average ﬂow yi computed from ﬁeld data and an expected ﬂow

is normalized by |Z| and by the scalar weight factor δ. The second term is introduced to
yield solutions that are physically plausible (e.g. OD ﬂows that are consistent with travel
patterns in the city). In the literature, the prior matrix is often referred as the initial or
seed matrix. This matrix is usually obtained from past traﬃc studies, such as studies with
lower resolution traﬃc data (e.g., static assignment models). The parameter δ is used to
weight the importance of the initial OD matrix. The problem has upper and lower bound
constraints (2).
Problem (1)-(2) has an objective function that is non-convex and non-diﬀerentiable simulation-
based. This SO problem has some challenges. First, it is high-dimensional. the dimensions
of d is often in the thousands. Second, the problem has many local minima. The simulation-
based term E[Fi(d, u1; u2)] is computationally costly to estimate. The goal of this work is to
design an algorithm that tackles the above challenges and is able to identify good solutions
with good performance with few simulation runs.

3.2
Metamodel formulation
First, the SO framework used in this paper is summarized. Then, the metamodel for-
mulation is presented. The metamodel SO approach presented is a variation of Carolina
Osorio [Osorio, 2019].
The main steps of the SO algorithm are displayed in Figure 1. The main idea of the meta-
model approach is to approximate the Simulation-Based objective function with an ana-
lytical function, known as the metamodel, and use this formulation to solve an analytical
and diﬀerentiable optimization problem. The use of the metamodel allows performing the
optimization problem more eﬃciently, as in each iteration the model ﬁnds a solution that
improves the previous trial point.
Let mobj
k
denote the metamodel at iteration k of the Algorithm. The metamodel is a para-
metric function with parameter vector βk. The following steps are done at each iteration of
the algorithm. At each iteration of the algorithm, the metamodel is ﬁtted with the previous
observations of demand-counts pairs obtained from the simulation. This problem is solved
by minimizing the square distance between the simulated expected ﬂow (i.e. E [Fi(d, u1; u2)])
and the ﬂow from the metamodel approximation (i.e. mi,k). (This is step 1 in Figure 1.)
Once the metamodel is ﬁtted, the metamodel minimization problem is solved (3)-(11). This
formulation ﬁnds the demand that minimizes the distance between the ﬂow - observed vs.
the analytical approximation (step 2a). At some iterations the next iterate is sampled ran-
domly from the solution from a uniform distribution, this is done to expand the sampling
of the metamodel in case the algorithm is currently in a local minimum (step 2b). The
previous step deﬁnes the next trial point, the next step is to run a simulation to obtain the
simulated traﬃc ﬂow (step 3). The performance of the new trial point is done by evaluating
the point using the simulated objective function. At step 4, it is decided if the new trial
point is accepted or not. Given the new set of observations, the metamodel is again ﬁtted
and the algorithm loop is closed.
In this proposed framework, a metamodel is ﬁtted and a metamodel optimization problem
is solved, at every iteration of the algorithm. This diﬀers from other frameworks used in
the literature which they ﬁrst collect several number of simulation observations using design
of experiments and then ﬁt a metamodel and solve an analytical metamodel optimization
problem. Instead, in the iterative approach adopted here the role of the metamodel is not to
replace the simulation. The role of the metamodel is to provide the SO problem analytical
structural information. This allows the SO problem not to treat the simulation as a black-
box, which eases the search in high-dimensional feasible regions.

mi,k(d; βi,k) (i.e. metamodel approximation of the expected ﬂow for link i). Furthermore, the
metamodel includes several constraints (4)-(11) deﬁned as a system of nonlinear equations.
These serve to deﬁne the analytical components of the metamodel. These constraints are
now described in more detail.
Note that mi,k(d; βi,k) has two components. The ﬁrst summation term, λi(d), is deﬁned by
constrains (6)-(11) and provides the metamodel with structural information. The second
summation term φi(d; βi,k) is deﬁned by constraint (5) and is a linear polynomial function
that provides a general-purpose component.

route demand, for all routes that include link i. The expected route demand is calculated
by the product of the route choice probability and the demand of the OD of that route,
dO(r). The route choice probability, as deﬁned in Eq. (7), is calculated using a logit model

The OD-pairs to modify are determined by drawing randomly from a binomial distribution
of size z0 (i.e. number of OD-pairs) with a probability of success of 0.3. Note that the vector
drawn from the binomial distribution has the same size as the previous iterated value, so
the indexes of the vector that draw a 1 determine which OD-pairs are modiﬁed.

Improve the performance of the algorithm
This section explores the bottlenecks in the algorithm as well as discusses the methods
used to reduce the computational time for each iteration. There are two main bottlenecks in

at every iteration to ﬁt the metamodel to the simulated sampled points.
The following implementations have been done to reduce the computational time of the
algorithm:
(A) Provide the jacobian of the objective function and the constraints.
(B) Reduce the number of parameters of the metamodel (β).
(C) Provide a good initial solution to the minimization problem.
(D) Use a python compiler to accelerate functions.

Observability - traﬃc counts allocation
The previous results for the toy network are calibrated having traﬃc counts at each
link. In reality, this information is not available, and only a few detectors are allocated

5.1
Observability - traﬃc counts allocation
in a subset of network links.
The allocation of traﬃc network ﬂow detectors is usually
performed by maximizing the coverage of the traﬃc (e.g. link ﬂows, route ﬂows, OD-pairs).
There are diﬀerent ways to specify how this coverage should be accomplished, the most
popular in the literature is OD-pair coverage (ODPC) [Yang et al., 1990]. Other methods
have been studied, such as Maximum Flow Coverage (MFC) [Yang et al., 1990] and OD-
Demand Coverage (ODDC) [Yang and Zhou, 1998]. These methods work by maximizing
the coverage of the estimated OD-matrix given the subset of link counts, and a budget
constraint (i.e. maximum number of detectors).
Below an introduction to the methods used.
• MFC: At each iteration, a new detector is allocated at the link that has the highest
ﬂow.
• ODPC: At each iteration of the ODPC problem, a new detector is allocated at the link
that covers the most OD-pairs.
• ODDC: Similarly to ODPC, a new detector is allocated at the link that covers the
most OD-pairs. But in this case the demand of each OD pair is considered as a weight
(i.e. the OD-pairs with higher demand are preferable)
To chose which method to use to allocate the detectors, the three methods are tested in
the Porto network using a budget constraint of 100 detectors. The results are shown in table
7. The MFC method has the highest ﬂow coverage (65.42 %) whereas the ODDC method
has the highest OD-pair coverage. The ODPC method has a lower performance in both
OD-pair and ﬂow coverage.

5.4
Computational performance analysis
In this section, the performance of the proposed method is presented and compared
against the benchmark. The results for the metamodel are shown in Table 11. Each it-
eration of the proposed method requires running three main problems as presented in the
Methodology (i.e. ﬁnd next trial point, simulation run, ﬁtting of the metamodel given pre-
vious trial points). It can be observed that each simulation run is performed in about 2
minutes (5% of the total time), whereas obtaining the next simulation sample point (i.e.
Fitting + Optimization) requires almost 41 minutes, on average, of computation (95% of the
total time). This means that each iteration is slowed down because the proposed method
required solving complex optimization problems. Other traditional SO techniques such as
SPSA require less computational time for ﬁnding the next iterate and are able to perform
more iterations within the same amount of time. On the other hand, the proposed method
is able to ﬁnd high quality iterates at each iteration and therefore required fewer iterations
overall. Furthermore, the proposed method can be improved if implemented using a better
performing programming language instead of Python and a more powerful mathematical
optimization solver instead of SciPy.

6
Conclusions
This thesis proposes a variation of the metamodel approach introduced in [Osorio, 2019].
This method can be used to solve high-dimensional calibration problems applied to large-
scale simulation-based traﬃc networks.
The method uses an analytical network model that allows ﬁnding solutions in a high-
dimensional problem using few simulation observations. Moreover, having the structural
information allows the algorithm to become robust to the initial solution.
The performance of the method is evaluated with both a simple toy network and a
large-scale traﬃc network of Porto for several scenarios (i.e.
uncongested, unbiased and
congested scenario). For the large network, the performance is compared against a bench-
mark calibration algorithm. It is shown that the proposed method is able to outperform
the benchmark by one order of magnitude even when allowing the benchmark method to
sample twice as many simulation observations. However, the proposed approach requires
more computational time, being about 57 times slower than the benchmark method.
It is remarkable how the metamodel is able to ﬁt the observed data given few simulation
observations. This is achieved thanks to demand-dependent route choice model and the fact
that the metamodel is ﬁtted to the simulation output at each iteration. This allows the
metamodel to replicate the simulated counts and therefore search for good solutions in a
high-dimensional feasible region.

7
Future work
Other conﬁgurations of the parameters could be studied; note that the current approach
includes a metamodel at each link that models its traﬃc counts. This metamodel is a func-
tion of the analytical model (βi,0λi(d) + βi,1) and at each iteration, this function is ﬁtted to
the simulated data by adjusting each β. Another approach could be to adjust the analytical
model instead, by modifying parameters of the traﬃc model that are now ﬁxed (Eq. (7-11)).
The presented work is able to ﬁt, exceptionally, the simulated traﬃc counts to the
observed. However, the OD problem is underdetermined, which means that ﬁtting to the
observed counts does not guarantee ﬁtting to the real traﬃc demand (variable of interest).
The problem of estimating travel demand based on sensor data often results in an ill-posed
inverse problem, where solution uniqueness cannot be ensured. Combining the metamodel
approach with the use of, recently available, more disaggregate travel demand data (e.g.
license plate, cell phone) has great potential to address this problem [Mínguez et al., 2010].
