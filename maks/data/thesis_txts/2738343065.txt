Abstract
The global electricity production remains heavily reliant on fossil fuels, with gas contribut-
ing significantly to overall output. Gas turbines, commonly used for electricity generation, emit
pollutants such as NOx and greenhouse gases like CO2 and CH4, posing severe health and environ-
mental risks. Therefore, it is imperative to develop robust and accurate methods for quantifying
these emissions. Predictive Emission Monitoring Systems (PEMS) offer a solution by predicting
unmeasured emissions through multivariate sensor data. However, sensor errors and system mode
variations present challenges in achieving accurate predictions.
This thesis investigates methodologies for real-time anomaly detection in multivariate time
series data, specifically within gas turbine operations. Existing approaches, including statistical
methods, machine learning, and deep learning techniques, exhibit limitations in handling non-
linear data, system mode transitions, and computational complexity. Consequently, this research
proposes a novel framework comprising two models: a Quadratic Discriminant Analysis (QDA)
model for predicting turbine modes and a k-Nearest Neighbors (K-NN) based anomaly detection
framework.
The QDA model effectively classifies turbine modes using spatial and temporal features, en-
hancing the robustness of emission predictions. The K-NN algorithm provides real-time anomaly
detection and imputes anomalous sensor values with quantified uncertainty, leveraging mean dis-
tance methods and physical checks for validation. This integrated framework demonstrates high
accuracy and adaptability across various operational modes, significantly improving PEMS preci-
sion and addressing periods of incalculable emissions.
Future work should extend the application of these models to other fields with weaker input
correlations to validate performance and enhance generalizability. This thesis contributes to ad-
vancing real-time anomaly detection techniques, ensuring more reliable and efficient monitoring of
emissions in gas turbine systems.

Chapter 1
Introduction
The electricity produced in the world still relies mainly on fossil fuels. For example, in 2022,
gas represents 22 % of overall electricity production [1]. The electricity produced using gas uses
gas turbine or gas engine, which releases many harmful pollutants [2]. In fact, emissions from gas
turbines are harmful to human health and the environment. Pollutants, such as NOx, have the
potential to cause respiratory diseases and can cause acid deposition, which harms plant life and
aquatic environments [3]. Additionally, CO2 and CH4 emissions are the main greenhouse gases
that contribute to climate change [4]. Therefore, there is an increasing need to have robust and
accurate methods to quantify these emissions.
Many applications in many fields depend on physical and mathematical models to predict
values which can support decision or provide insight on physical or not phenomena. For instance,
emissions of pollutants from gas turbines can be predicted by models, either first principal models
using physical equations or data driven models using machine learning tools are used to predict
the emissions. In all cases, the models use input data to predict outcomes.
Predictive Emission Monitoring Systems (PEMS), are software sensors that predict the emis-
sions from gas turbines that is not directly measured [5], and rely on multivariate input data from
sensors. Sensors provide information such as temperature, pressure, or fuel flow. Sensor errors
can compromise the accuracy of models that rely on their data, potentially leading to incorrect or
highly uncertain results. Therefore, it is important to detect the possible anomalies in the inputs
in real-time to adapt the uncertainty of the outcomes from models accordingly and correct the
input if possible.
Furthermore, gas turbine system can operate in several modes. For instance, the outlet pres-
sure before the combustion chamber or the fuel flow will not have the same range of values if the
turbine is operating or if it is running in idle mode (there is no electricity generation, but there is
combustion in the combustion chamber). Thus, the sensor values can have several ranges of plau-
sible values according to the system mode. Therefore, the anomaly detection system should be
able to handle anomalies according to the system mode, and to handle sudden changes of modes,
which leads to sudden change in sensor values and should not be flagged as anomaly.
Over the years, researchers have explored various methodologies to address the challenges
associated with real-time anomaly detection in multivariate time series. These approaches aim to
capture complex dependencies between sensors, as well as temporal patterns, and identify failing
sensors without extensive manual setup. Several notable methodologies have been proposed in the
literature to tackle anomaly detection in industrial settings.
One prevalent approach involves the application of deep learning methods for the detection of
1
==PAGE:7==
anomalies. Deep learning techniques, as reviewed in the work of Pang et al. [6], offer the ability
to capture intricate relationships and dependencies within multivariate time series data. Methods
such as the Convolutional Neural Network (CNN) [7][8], the Bayesian Network (BN) [9], or the
Multivariate Convolutional Neural Network (MCN) [10][11] enable to extract meaningful features
and complex relationship between sensors, thus allowing to capture spatial patterns. Furthermore,
these methods are coupled with other methods such as long-short-term memory (LSTM) [7] or
Hierarchical Temporal Memory (HTM) [9] to capture the temporal pattern of the dataset. These
frameworks can also deal with complex high-dimensional data sets by lowering the dimension using
AutoEncoder for instance [12], which will capture the essential characteristics of the input data in
a more compact form, facilitating learning and analysis. These models can offer real-time as well as
post-processing anomaly detection. However, these methods often need important computational
resources, they can be complex to implement, and they do not provide in most cases filtering and
prediction features. These models focus on detecting the anomalies and thus are not suited to
denoise the sensors value.
Another main approach is to use time series methods and particularly univariate time series.
Several models have been used, such as the Hotl-Winters [13], the autoregressive integrated moving
average (ARIMA) [14], the local autocorrelated polynomial (TVLAP) model [15]. These models
often use historical data using various parameters to consider trends and seasonality, for instance.
The aim is to compare the predicted value from the model with the measurement one to de-
tect anomalies. However, these methods have difficulties handling non-linear connections, small
anomalies, and past anomalies can heavily influence future forecast. Furthermore, they are not
suitable for handling changes in system mode, as they are not multivariate techniques, and they
cannot capture spatial patterns between sensors. In the case of multi-mode systems, such as a gas
turbine, using these methods would require developing a specific set of parameters for each mode
and adding mode prediction framework.
Furthermore, interesting work using Kalman filter has been investigated. As evidenced in the
research by Cong et al. [16], Kalman Filters are well suited to capture dependencies between
sensors and provide real-time estimates of system states, making them valuable for identifying
anomalies and failing sensors. This article, introduces the Field Kalman Filter, which considers
all sensors as a whole and monitors multimode, capturing complex interactions and dependencies
between sensors and process variables. However, this method does not enable to detect which
sensor has an anomaly. Additionally, the calibration process and training requirements of Kalman
Filters may necessitate careful consideration for each gas turbine application and therefore are not
easily implementable across gas turbines.
This thesis aims to investigate the available frameworks for anomaly detection and assess
a framework that effectively addresses the requirements for multivariate time series real-time
anomaly detection in the context of gas turbine operations.
The framework developed needs
to not require heavy calculations, it must detect sensor input anomaly and provide an imputed
value with an uncertainty. Thus, what is a suitable real-time anomaly detection method, which
can also provide imputed with an uncertainty, for multivariate time series sensors value in the case
of gas turbine system running in several modes? Thus, this thesis seeks to:
A. Exploring existing literature to identify approaches suitable for real-time anomaly detection
and imputation of anomalous values within multi-mode systems;
B. Develop a model to predict the gas turbine mode. This mode information can enhance the
emission prediction model and may also be utilized by the anomaly detection framework;
2
==PAGE:8==
C. Develop a robust and practical framework for real-time anomaly detection and imputation
of anomalous values in multivariate time series sensor data within multi-mode systems.
The first chapter 2, Background knowledge will focus on describing the Predictive Emission
Monitoring System (PEMS) and explaining the two main challenges of this system which are the
various operation modes of the system and the anomalies coming from the sensors. Therefore, the
chapter 3 Method & Materials will suggest frameworks to tackle these challenges. Objectives A,
B and C will be respectively explored in section 3.1, 3.2 and 3.3.

Background knowledge
2.1
Predictive Emission Monitoring Systems (PEMS)
Predictive Emission Monitoring Systems (PEMS) is used to define systems that allow predicting
the emissions from power plants that, for instance, use gas or oil to generate power and/or heat,
these systems will be called emission sources. PEMS, are increasingly used by companies to
report their yearly emissions in compliance with the regulations standards. It is more reliable over
time than Cotinuous Emission Monitoring Systems (CEMS) which measure the emission directly
in the flue gas but they very often undergo issues, because they are very sensitive. PEMS does
not rely on measurement from the flue gas but it uses sensor inputs from the system itself. For
instance, to predict the emissions of a gas turbine it will use ambient variables such as the ambient
temperature and pressure, as well as variables in the system itself such as fuel flow, generated
power, inlet temperature and pressure before the combustion chamber.
These inputs will be
combined with thermodynamic laws as well as energy and mass balance to find the state of the
system. Once the state of the system is known, the emissions can be computed using two ways. For
the stoichiometric emissions, i.e. the reaction is assumed completed, the emissions are computed
using the fuel composition. For the rate formation emissions i.e. the reaction is not completed,
the emissions will depend on the combustion process. The first group includes the formation of
SO4 and CO2, while the second includes NOx, CO and CH4.

There are many types of turbine, each type can be distinguished by the components in the
system, the way it works, or the dimension of the turbine. In this thesis, only gas turbines of the

various gas turbines of the same type. The dataset have a minutely frequency, they provide the
inputs used by PEMS as well as the outputs. For this thesis, the focus has been on the inputs.
They can be divided into two categories: ambient inputs and combusion inputs. These inputs
are described in table 2.1 and figure 2.1, [17]. More detailed description of the inputs on their

2.2
Considerations on operational modes
Power plants, as many systems can be operated in several modes, i.e. according to the mode
the inputs value can be in specific range values. An important input to distinguish between the
5
==PAGE:11==
various modes is the load. The load is often the amount of electrical or mechanical power that
the system delivers over the maximum of this amount that the system can provide. The load is
defined differently according to the system; it can use, for instance, the generated power or the
speed rotation of the turbine. Four modes can be defined as follows; the definitions are taken from
report [18]:
• Not running: Fuel is not provided in the turbine. This mode does not emit emissions.
• Start-up: The turbine was in not running mode, and gradually the fuel flow provided to the
machine is no longer null. In this mode, fuel flow, valve position, T45avg, dPinlet and P3
increase rapidly. Usually, no power is generated or very few. Although this mode can last
for a few minutes, this phase can represent a very high proportion of the methane emissions.
As the turbine is not in steady state, the usual steady state equations cannot be applied,
and the chemical reactions in the combustion chamber are strongly impacted.
• Idle: This mode is often after the start-up one or before not running, the turbine is not
completely in steady state, and the generated power is null.
• Operational: This is the most common mode when the turbine is not in a running mode.
The turbine is considered in steady state and the load is usually above 50 %. In this mode,
the combustion of methane with oxygen, for example, is almost total.

The information of the turbine mode at a given time is important to adjust the system param-
eters or the equations used in the model. Indeed, according to the turbine mode, some equations
may not be valid and the model needs to be adjusted, as well as some parameters. Thus, it is
important that the model is calibrated using calibration data for all modes to provide more ac-
curate outputs. Moreover, some emissions should not be reported for specific turbine modes. For
example, NOx should not be reported when the turbine is in a start-up mode. Finally, knowing the
mode of a set of inputs can provide more insight that can be used for anomaly detection. Indeed,
the inputs according to the turbine mode can have different range of values. For instance, a mea-
surement can be considered wrong if the turbine is in a certain mode but right if the model is in
another mode. Thus, the mode of the turbine plays an important role and shall not be neglected.
Currently, PEMS does not distinguish between the modes and use same equations and parameters
regardless of the mode. Therefore, the section 3.2 will focus on developing a framework to predict
the turbine mode.

Chapter 3
Methods & Materials
3.1
Overview of existing anomaly detection methods
Anomaly detection in multivariate time series data, particularly from sensor networks, is cru-
cial for various applications such as industrial monitoring, healthcare, and cybersecurity. This
literature review explores various methods employed for real-time anomaly detection, focusing on
their advantages and disadvantages.
3.1.1
Statistical Methods
Traditional statistical methods have been widely used for anomaly detection in time series
data. Techniques such as Principal Component Analysis (PCA) and Multivariate Autoregressive
Integrated Moving Average (MARIMA) models [14] have been employed due to their simplicity
and effectiveness in detecting deviations from expected patterns. PCA, for instance, reduces the
dimensionality of the data, capturing the most significant variance, which can help in identifying
outliers [19]. MARIMA models, on the other hand, are well-suited for modeling time series data
by capturing trends and seasonality, making them effective for detecting anomalies in predictable
patterns [20].
The main advantage of these methods is their simplicity and ease of implementation. They are
often straightforward to understand and interpret, providing clear reasons for detected anomalies.
Additionally, they typically have low computational costs, making them suitable for applications
with limited resources.
However, these methods assume linearity in the data, which make it
complicated to use for real-world datasets, and can be sensitive to noise and outliers, leading to
false positives. Moreover, these methods are not really suitable in the case of this thesis, since the
sensor values can be in different modes which would require to create several MARIMA models
and it should also be able to handle quick changes from one mode to another.
3.1.2
Machine Learning Methods
Machine learning techniques, including both supervised and unsupervised learning, have gained
prominence for anomaly detection. Algorithms such as Support Vector Machines (SVM), k-Nearest
Neighbors (k-NN), and Isolation Forests have been extensively applied. SVMs, for instance, can
separate normal and anomalous data by finding an optimal hyperplane, even in high-dimensional
spaces [21]. k-NN methods detect anomalies by measuring the distance to the nearest neighbors,
identifying points that deviate significantly from the norm [22]. However, since it uses the nearest
10
==PAGE:16==
neighbors it is highly sensitive to anomalous points in the dataset used to find the neighbour. This
requires to carefully build the dataset and then can make difficult to scale the method to hundred
of emission sources.
An interesting approach combining k-NN and Vector Autoregressive (VAR) space has been
proposed to improve the robustness and accuracy of anomaly detection in time series data [23].
This method leverages the strengths of both k-NN, which excels in detecting local anomalies, and
VAR, which models temporal dependencies effectively.
Machine learning models offer significant flexibility, capable of handling non-linear relationships
and complex patterns in the data. They also allow for the incorporation of domain-specific features,
enhancing detection accuracy. Moreover, certain machine learning models can be scaled to handle
large datasets efficiently. However, supervised methods require labeled training data, which make
it difficult to scale the model to hundred of emission sources. Additionally, training and deploying
these models can be computationally expensive. The complexity of these models can also make
them challenging to interpret, obscuring the reasoning behind anomaly detection.
3.1.3
Deep Learning Methods
Deep learning approaches, particularly Recurrent Neural Networks (RNNs) and Long Short-
Term Memory (LSTM) networks, have shown significant promise in handling the temporal de-
pendencies in time series data. RNNs and LSTMs are designed to remember past inputs, making
them suitable for sequences where past information is crucial for understanding the current state
[24]. These models have been effectively used to detect anomalies by learning the normal patterns
of time series data and identifying deviations from these learned patterns [25].
CNNs, traditionally used for image data, have been adapted for time series data due to their
ability to capture spatial hierarchies through convolutional operations. When applied to time series
data, CNNs can automatically learn to extract features such as trends and periodicity, which are
crucial for anomaly detection [8][7].
MCNs extend the concept of CNNs by handling multiple channels of data, which is particularly
relevant for multivariate time series data from sensors.
MCNs can process each sensor’s data
channel independently while learning the correlations between them, leading to more accurate and
robust anomaly detection [11][10].
The primary advantage of deep learning models is their ability to capture temporal depen-
dencies and non-linear relationships in the data. They often achieve higher accuracy compared
to traditional and basic machine learning methods. Furthermore, deep learning models can auto-
matically learn relevant features from raw data, reducing the need for manual feature engineering.
However, they require large amounts of data for training, which is a limitation to scale these mod-
els to hundred of emission sources. Training deep learning models is resource-intensive, requiring
significant computational power and memory. Moreover, these models are often seen as black
boxes, lacking interpretability and making it difficult to understand the decision process.
3.1.4
Hybrid Methods
Hybrid methods combine statistical, machine learning, and deep learning techniques to leverage
the strengths of each approach. For instance, combining PCA with neural networks or integrating
clustering with anomaly detection algorithms has been explored to enhance detection capabilities
[26]. These methods aim to balance the trade-offs between computational efficiency and detection
performance.
11
==PAGE:17==
Hybrid approaches can achieve higher accuracy and robustness by combining multiple meth-
ods. They offer versatility, allowing customization to specific applications and data characteristics.
However, designing and implementing hybrid methods can be complex and time-consuming. En-
suring seamless integration and interaction between different methods can also be challenging.
Additionally, hybrid methods may incur higher computational costs due to the combined use of
multiple algorithms.
Consideration on the existing methods
Real-time anomaly detection in multivariate time series data from sensors is a complex and
evolving field. Each method discussed has its advantages and disadvantages, and the choice of
method often depends on the specific requirements of the application, such as the need for inter-
pretability, computational resources, and the nature of the data. In the case of this thesis, the
aim is to develop an anomaly detection framework for real time anomaly detection of multivariate
time series within multi-mode systems. The framework should also be able to impute the potential
anomalous points with a corresponding uncertainty. Unfortunately, the models described above
do not fulfill all these criteria. These methods only provide the information that a new point is
anomalous but it does not give the knowledge of which inputs that make the point are anomalous
and neither provide an imputed value with an uncertainty. Moreover the deep learning and hybrid
models are more suitable for more complex dataset with a great number of dimensions. Therefore,
they are too complex for the case studied in this thesis. Thus, this thesis in section 3.3 proposes
a new anomaly detection framework based on the K-NN algorithm that take into all the criteria.

3.2
Prediction mode framework
3.2.1
Overview of existing classification methods
Classification of time series data is a critical task in many scientific and engineering disciplines,
including finance, medicine, and environmental science. Time series classification involves predict-
ing the category of a given sequence of data points that are indexed in time order. This literature
review provides an overview of existing models for time series classification, highlighting various
approaches and methodologies used in this domain, along with their respective advantages and
disadvantages.
Time series data are characterized by temporal dependencies and patterns that traditional
classification models often struggle to capture. Consequently, specialized techniques and algorithms
have been developed to address these challenges. One common approach is the use of distance-
based methods, such as Dynamic Time Warping (DTW). DTW measures the similarity between
two time series by finding an optimal alignment between them, allowing for sequences that may
vary in speed. Berndt and Clifford [27] demonstrated the effectiveness of DTW in time series
classification, showing that it can handle variations in temporal dynamics better than simple
Euclidean distance. However, DTW can be computationally expensive, especially for long time
series, and may require extensive computation to align sequences, making it less suitable for real-
time applications.
Another prominent approach is the use of feature-based methods, where time series data are
transformed into a set of features before classification.
These features can include statistical
measures and frequency domain characteristics. For instance, Fulcher and Jones [28] presented
a method for constructing feature-based representations of time series, which enabled the use of
conventional classifiers such as decision trees and support vector machines (SVMs). This approach
often involves selecting a subset of relevant features that best capture the underlying patterns in
the data. While feature-based methods can be very effective, they may suffer from the ”curse of
dimensionality” if too many features are used, leading to overfitting and increased computational
complexity.
Machine learning algorithms, particularly neural networks, have shown great promise in time
series classification.
Recurrent Neural Networks (RNNs), including Long Short-Term Memory
(LSTM) networks, are designed to capture temporal dependencies by maintaining a memory of
previous inputs.
Hochreiter and Schmidhuber [24] introduced LSTM networks to address the
vanishing gradient problem in RNNs, enabling the modeling of long-term dependencies in time
series data. Despite their powerful capabilities, RNNs and LSTMs require large amounts of data
and significant computational resources to train effectively, and they can be prone to overfitting if
not properly regularized.
More recently, Convolutional Neural Networks (CNNs) have also been applied to time series
classification. Wang, Yan, and Oates [29] demonstrated that CNNs can automatically extract
discriminative features from raw time series data, achieving state-of-the-art performance on several
benchmark datasets. However, CNNs may struggle with capturing long-range dependencies in time
series data, which can limit their effectiveness in certain applications.
Finally, hybrid models that combine different approaches have also been explored. For instance,
Karim et al. [30] proposed a hybrid model that integrates CNNs and LSTMs to capture both local
and long-term dependencies in time series data. This combination leverages the strengths of both
architectures, resulting in improved classification performance.
However, hybrid models often
13
==PAGE:19==
require careful tuning and significant computational resources to achieve optimal performance,
which can be a barrier to their practical implementation.
In summary, the classification of time series data is a complex and evolving field, with vari-
ous approaches trying to temporal dependencies and patterns in time series data. However, the
methods described above also come with their own set of limitations, including computational
complexity, risk of overfitting, and challenges in capturing long-term dependencies. In this thesis,
an approaching using feature-based method by creating feature using statistical measures com-
bined with several Quadratic Discriminant Analysis (QDA) to distinguish between turbine modes
has been developed requiring few points in the training datasets and little computations.

3.3
Anomaly detection framework
3.3.1
Proposed method
This section on anomaly detection aims to define an anomaly detection framework that can
flag anomalies in real time and be able to replace the flagged values with imputed one with an
uncertainty. The objective is that the PEMS can still provide emission results even if the original
values from the sensors do not allow it. This framework need to be carefully implemented, because
we would rather have a conservative approach by providing higher emission results when the
calculation are not possible than provide wrong emission results using imputed inputs that could
underestimate the real emissions.
3.3.1.1
K-Nearest Neighbours Algorithm
The anomaly detection framework in this thesis is based on K-Nearest Neighbours Algorithm
(K-NN). The K-NN is a versatile, easy-to-implement supervised machine learning algorithm used
both for classification and regression, but primarily known for its role in classification tasks [33].
For classification K-NN operates on a simple principle: it classifies a new data point based on
the majority label of the ’k’ closest points in the feature space. The feature space is typically
represented by a multidimensional space where each dimension corresponds to a distinct feature
of the data points. By calculating the distance between different points in this space, the K-NN
algorithm can determine which points are nearest to a new, unlabeled point. This distance can
be measured in various ways, with the Euclidean distance being the most common. The K-NN
is particularly effective when the dataset has well-defined clusters of normal behavior, making
deviations from these clusters easier to detect. In our case, the feature space are divided into two
groups, as for the prediction mode model, the combustion and ambient inputs, there are described
figure 2.1. For the same reasons than for the prediction mode model we do not want to use the
ambient conditions to predict the combustion inputs and the other way round. It would not make
physically sens to use the ambient temperature to impute the fuel flow. The distance are computed
using the Euclidean distance.
In this thesis, the K-NN algorithm is both used to detect anomalies of a new point using
anomaly score and to impute missing or anomaly inputs. The anomaly score can be calculated
in various ways, several definitions of the anomaly score are proposed in section 3.3.1.2. For each
emission source, it can be gas engine or gas turbine for instance, a training dataset is generated
using historical values. This training dataset is used to find the k closest neighbours of a new
point. In this section a point refers to the set of input values (combustion or ambient) for a spe-
cific timestamp and any point can be associated to an anomaly score.
K-NN to flag anomaly: For a new point that needs to be analysed under the anomaly
detection framework, its k closest neighbours are found, using the euclidean distance, the distance
between this new point and each of its k closest neighbours are calculated. Then, an anomaly
score is calculated, this score is then compared with an anomaly threshold to know if more inves-
tigation must be conducted on this point to flag potential anomalous inputs.
K-NN to impute missing or anomaly inputs: For a specific point if some inputs are
missing or some inputs are flagged as anomaly, the K-NN algorithm can be used to impute these
inputs by employing the Euclidean distance as a weight. This method involves identifying the ’k’

features. Each of these neighbours contributes to the imputation, with their influence weighted

3.3.1.2
Consideration on anomaly score
The definition of the anomaly score plays a significant role in the anomaly detection frame-
work, since it is used as the metric to detect anomalies. Therefore, the anomaly score could be
calculated in several ways according to the patterns inside the dataset. A first simple approach
is to use the Euclidean distance between the point that undergo the anomaly detection frame-
work and its neighbours from the training dataset. This distance would be then compared with
an anomaly threshold to detect if this point may contains anomalous inputs. Since the anomaly
score is a distance in this case the threshold must be a constant representing limit distance. Yet,
the datasets from emission sources used in this thesis have patterns that challenge this definition.
Indeed, the majority of their points are contain in operational mode, characterized by stable values
with relatively high fuel flow, while very few points are in start-up mode, characterized by high
variability and lower values, see section on prediction mode 3.2.
Thus, if the new point is in
27
==PAGE:33==
operational mode its distances with its neighbours are likely to be low, while, if the point is in
start-up mode the distances will be greater, since very few points are in this mode with specific
input values. Therefore, these patterns challenge the definition of the anomaly score, since the
threshold is a constant and cannot be both adapted to points in operation mode and points in
start-up mode, since according to the mode the same distance could mean an anomaly point for
operational mode while it could be a normal point for start-up mode. In other words the density
of points, in the dataset used to find the neighbours of a new point, in the area corresponding to
the operational mode is greater than the density of points in the area corresponding of the start-up
mode in the feature space.
Distance based anomaly score: Two ways using distances are considered in this anomaly
detection framework. The first method is using the distance between the new point and its k-th
farthest neighbour in its feature space, combustion or ambient inputs, named farthest distance
method. The second definition is the normalization of the mean distance between the new point
and its neighbours by dividing it by the mean distance of its neighbours and their own neighbours,
named mean distance. This two methods are more precisely described below:
A. Farthest Distance Method: This method uses the distance between the new point and
its k-th farthest neighbour in its feature space. The anomaly score, A(x), is defined as:
A(x) = dist(x, xk)
where xk is the k-th farthest neighbour of x.

point and its neighbours by the mean distance among the neighbours themselves.

3.3.1.3
Anomaly detection framework overview
The anomaly detection framework developed in this thesis is a real time anomaly detection,
thus this framework is apply to the input values before emission calculations. Let an emission
source where a training data has been generated using historical values from this emission source
and let a new point from this emission source at specific timestamp, i.e. a set of input values.
This new point will undergo a pre processing step, then the anomaly score of this point will
be calculated. If this anomaly score is higher than a threshold, called anomaly threshold, then
further analysis will be conducted otherwise the model that computes the emission or the
prediction mode model can use this point without further analysis. Further analysis includes,
flagging potential anomalous inputs, a physical check and finally a post processing step.
Figure 3.8 shows the general overview of the anomaly detection framework, each part is described
in details in the following paragraphs.

==PAGE:36==
Figure 3.8: Overview of the anomaly detection framework.
3.3.1.4
Training dataset
For each emission source and feature space (combustion and ambient inputs) a training dataset
is built using the historical data of this emission source, the values are normalized using Standard-
Scaler form the python package sklearn [32]. This function applies a Z-score normalization on each
input by removing its mean and dividing by its standard deviation; the mean and standard devia-
tion of each input are stored to be able to use it again to normalized points for which the anomaly
detection framework will be performed. This step is a really crucial step because the efficiency of
the K-NN algorithm is completely based on the training dataset. Indeed, if measurements, which
can be defined as anomalies, are in the training dataset, the anomaly detection framework may not
flag some anomalies, since this new abnormal point has close neighbours in the training dataset.
Furthermore, the training dataset needs to cover the whole range of possibilities of how the emis-
sion source can be operated. Effectively, if the new point has a fluel flow and all others features
higher than the maximum of these features in training dataset, the anomaly detection framework
will consider this point has unknown and therefore it will be detected as anomaly, whereas this
point could be a normal point. Thus, the training dataset should at least cover one year of data
since some emission sources can have seasonal pattern, and the training dataset should contains
points where the fuel flow is between 0 and the maximum that the emission source can operate.
Furthermore, the training data need to undergo some processing to ensure that training points
do not contain anomaly points, which would jeopardize the anomaly detection framework. The
minimum processing is to remove all the points that contains input values outside their physical
31
==PAGE:37==
range and only keep points where the system is actually running, so where the fuel flow is pos-
itive. In a later step a model is used to compensate cases where the inputs still fulfill physical
relationships but are unknown for the training dataset, to make this anomaly detection framework
more robust. A training dataset is both used to feed the K-NN algorithm, the neighbours of a new
points will be amongst the training dataset, and to calibrate the models for the physical check.
The physical check is explained later in the core model; it consists in defining for each input a
model where an input is a function of the other inputs and the load. Thus, the training dataset is
also used to calibrate all these models. Figure 3.9 summarizes the flow to build the training dataset.
Figure 3.9: Framework to build the training dataset.
3.3.1.5
Pre processing
The first step of the anomaly detection framework is to perform simple processings. The first
processing is to flag missing values.
The system that provides the input values every minute
provides the value of each input and a quality. Thus, the missing values can be flagged using the
quality or the value itself. Indeed, sometimes the system provides good quality, while the value is
missing or set to a default value. The second processing is to flag the inputs outside their physical
range. If an input is outside its physical range, then the value is replaced by the closest boundary
and the quality of the value is set to outside physical range. Finally, the inputs are normalized
using the same Z-score normalization as the one used to build the training dataset.
3.3.1.6
Anomaly detection, core model
Anomaly score
Once the new point has undergone pre processing, the core of the anomaly detection model can
be applied to it. The first step is to compute the anomaly score of this new point. In this step the
32
==PAGE:38==
missing values are replaced using the imputed one to be able to compute the anomaly score. The
values are imputed following equation 3.8. This first calculation is twofold, firstly is to quantify the
anomaly of this point, if the anomaly score is higher than the anomaly threshold, then this point
might has some potential anomalous inputs. Secondly, this calculation will be used as a baseline
for further analysis, with all the inputs available for this point we know what is the anomaly score.
In a later stage we will be able to compare this baseline with other anomaly scores computed by
masking some inputs. The anomaly score can be computed as described before, either by using
the furthest distance or the mean distance methods. These different definitions of the anomaly
score will be tested in the result part. Table 3.4 summarizes the three various ways to compute
the anomaly score, proposed in this thesis.
Once the anomaly score has been computed for the new point the results is compared with the
anomaly score threshold. If the anomaly score is below this threshold the point does not undergo
more investigation and the inputs from this point are then used by the PEMS model to compute
the emissions. The potential missing values of this point are replaced by the imputed one along
with the uncertainty. Otherwise, if the anomaly score is greater than the threshold further analysis
are conducted.

Flag potential anomalous inputs
Now that the point has been identified as potentially anomalous, it is crucial to determine
which inputs are responsible for the high anomaly score. The challenge arises because it is not
feasible to directly compare a specific measured input with its imputed counterpart. This difficulty
stems from the fact that other inputs, which influence the imputed value, may themselves contain
anomalies, thus skewing the imputed result. Consequently, it becomes challenging to ascertain
whether the discrepancy between the measured and imputed values is due to anomalies in the
measurement, other inputs, or a combination of both.
To address this, the proposed method involves iterating over each measured input, masking
it, and recalculating the anomaly score using the imputed value in place of the actual measure-
ment. This recalculated anomaly score is then compared to a baseline anomaly score, which was
previously established using all available measurements for the new data point. If the recalculated
score is significantly lower, such as showing an absolute relative change greater than a threshold,
33
==PAGE:39==
it suggests that the masked measurement was contributing to the original high anomaly score, and
subsequently flagged as anomalous. The anomaly score for this new point, where the value of the
flagged inputs has been replaced by their imputed one, is then recalculated, and this score serves
as the new baseline. This baseline is assessed against an anomaly threshold, and the process is re-
peated as long as the new baseline exceeds the anomaly threshold, and anomalies are flagged based
on the changes in the anomaly score resulting from sequentially masking each input. Throughout
this process the input values of the new point can change several times, after each loop if some
inputs are flagged as anomalous the new point change, since the imputed value is used instead of
the measurement one for each anomalous point. Thus, after each loop, the new point and the new
anomaly score calculated from it are stored. Later, the physical check step will require to access
specific input values, more precisely, the input values for which the anomaly score fulfills some
conditions.
In this anomaly detection framework, the anomaly score serves two distinct purposes, each
associated with a specific threshold. The first purpose, accompanied by a designated anomaly
threshold, is to determine when to initiate an analysis to know which inputs are anomalous. It
is crucial to stop examining input anomalies at the right moment to avoid making the framework
overly sensitive and mistakenly flagging too many inputs as anomalies. The second purpose of
the anomaly score, with a different threshold, is to compare the baseline anomaly score to a new
score obtained after masking a specific input. Consequently, two different methodologies can be
employed to calculate the anomaly score for each respective purpose.
To serve the second purpose of the anomaly score, i.e. analyzing changes in anomaly scores
from a baseline to a score following the masking of specific inputs, a distance-based anomaly scor-
ing method is preferred. This approach facilitates a precise quantification of any changes that
occur when an input is masked. In contrast, using a mutual neighbour method complicates the
assessment of such changes, as it only allows for integer variations in the count of mutual neigh-
bours—for example, a change from 3 to 3.5 is not possible. Therefore, if the mutual neighbour
approach is used to compute the anomaly score, it will only be used as the condition to stop
investigating the inputs anomaly. Thus, anyway a distance based approach is used to investigate
the inputs anomaly. Figure 3.10 shows the framework to flag potential anomalous inputs. As men-
tioned above the anomaly score mentioned in the drawing to stop running the loop on the normal
inputs can use the three ways defined in this thesis, however the anomaly score used to compute
the relative change can only use the distance based anomaly score. Nevertheless, as mentioned
previously, only the farthest distance and mean distance methods will be used.

At the conclusion of this step, certain inputs which belong to the new point may have been
identified as anomalous, and for each flagged value, an imputed value is used instead. The goal is
to supply the PEMS with a refined version of the input values to accurately compute emissions.
As discussed in characterization of the anomalies, in the background knowledge section, see 2.3,
using anomalous inputs in the PEMS could impede its ability to calculate emissions accurately.
It is thus crucial to replace these anomalous inputs with imputed values that more accurately
reflect reality. However, as previously mentioned, it might be more prudent to avoid calculating
emissions for timestamps where the inputs are inadequate for the PEMS to perform accurately. In
such cases, a conservative approach should be adopted by assigning high emission values to these
timestamps rather than using potentially incorrect input values supplied by the anomaly detection
framework. Consequently, the next steps will focus on minimizing false alarms, i.e., verifying that
the inputs flagged as anomalous are genuinely so.
Physical check
The aim of this step is to compensate the challenge regarding the obligation of the dataset to
contain all the possible input values. Indeed, as explained before if the new point has input values
unknown for the dataset but that are not anomalous, the anomaly detection framework will flagged
almost all the inputs as anomaly. Since, it is impossible to absolutely cover all the possibilities, this
35
==PAGE:41==
step will verify if the inputs, even if they have been flagged as anomaly, fulfill some relationships.
The inputs that indeed verify the relationships will be removed from the anomalous inputs and
their measurement value will be used, instead of the imputed one.
In order to not have the opposite behaviour, where most of the inputs that are flagged as
anomalies are ultimately not assessed as anomaly, this physical check is not performed all the time
and it is only performed on specific inputs. Firstly, the inputs that have a really huge anomaly
score should not be checked and should by flagged as anomaly anyway. Secondly, this physical
check must be performed only if enough inputs are flagged as anomaly. Indeed, if only one input is
flagged as anomalous it is not likely that the point is in an unknown area of the feature space. The
proportion of inputs flagged as anomaly to trigger the physical check must be carefully defined.
Normally, the situation where some inputs of a new point are normal despite being in an unknown
area of the feature space, should leads to a great proportion of anomalies.
To check if inputs must not be flagged as anomaly because there are in an unknown area of
the feature space that correspond to normal values, each input are defined as a function of the
other inputs using the training dataset. Furthermore, since the behavior of an input according
the others can be different according to the load, i.e. according to the fuel flow or the generated
power, the functions are different according to load ranges. For instance, if an input has a linear
relation with the others inputs and that two load ranges are defined, then the linear coefficients
will not be the same for the two load ranges. The functions can be linear, polynomial with degree
greater than 1, or other, the functions are depending on the relationships between the inputs. In
the case of the combustion inputs used in this thesis, linear models are used since the inputs have
strong correlations between each other.
In practice, a specific anomaly score threshold, called physical threshold is defined to distinguish
between the strong anomalies that should not be checked with relationships and weaker anomalies
that might need to be checked. Additionally, as mentioned a great proportion of inputs must be
flagged as anomalous to trigger the physical check. If the proportion of inputs flagged as anomaly
is greater than a defined threshold then the following steps are performed. Firstly, retrieving the
inputs values on which the physical check is going to be performed. In the core model, a loop is
performed to flagged the anomalous inputs, at each step in the loop a new anomaly score is com-
puted along with the inputs values, corresponding to measurement values for the normal inputs
and imputed values for the anomalous inputs. To retrieve the inputs values on which the physical
check will be applied, the inputs values corresponding to the greatest anomaly score below the
physical threshold are selected. Thus, for the strong anomalies the imputed value is used whereas
for the weak anomalies the measurement value is used. It is important to select the imputed values
for the strong anomalies to not have bias when using the predicted value using the relationships
between inputs. Finally, each anomalous input value is compared to the predicted value using its
function for a given load range that gives the input value according to the other inputs. If the
relative change between the input value and the predicted value is lower than a defined threshold
then the input is flagged as normal. Figure 3.11 displays the framework of the physical check step.

than a specific threshold, that can be a constant or depending on the standard deviation of the
input, then the input is not flagged as anomalous anymore and the measurement value is used

Validation
The previous section explained the anomaly detection model. For each emission source and

and Critical Success Index.

A lower FAR indicates better performance, with zero being the ideal value.

4.1.5
Discussion
According to the validation of the prediction turbine mode model, more research is needed to
formalize how this model can be applied to new turbines. It has been shown that with a robust
datasets, built from many turbines, turbines of the same type can use this model, even though
no data point from these turbines has been used to build the labeled dataset. Nevertheless, it
required to add more points to the labeled dataset. Thus, it would be interesting to investigate
how it would be possible to create a pool of data points for each turbine type, which would be
enough to be able to use this pool for each new turbine of the same type, without need of adding
more points. It would required to examine which and how many points from how many turbines
it would be needed to have one labeled dataset per turbine type. According to how the features
are distributed amongst the turbine of the same type, what are the most relevant points to add
from several emission sources to create a dataset that can then be used for all the new turbines of
the type.
Despite the need for more investigation, this model has already showed good results in the
case of combustion inputs to predict turbine operation mode. This turbine mode model required
few points around 100 points per mode, which reduce the work of creating labeled dataset. Thus,
it also allows systems with very low frequency for some populations to use it as classification
model. Additionally it showed optimistic results toward not requiring one model with specific
hyperparameters and labeled dataset per emission source, which is promising since this model
should be able to be scaled up to hundred of emission sources. Finally, the approach using features
and QDA enable to understand how the model works, it is not a ”black box” model.
However, since the lack of literature on QDA models in a context of multivariate time series, it
would be necessary to apply this framework to other fields to assess if the model can be generalized
or if it can only be used for specific cases, such as the one in this thesis. Moreover, the uncertainty
on the possibility to create robust common dataset for several turbines is questioning the possibility
to scale this method to a huge amount of emission sources. Thus, this prediction mode model might
be limited to specific cases and may show some limitations. As mentioned previously, more research
is needed to conclude on these matters.

AGT-42 and AGT-43 the dense areas are the FuelFlowPrimaryRange that have more than 300
points and for the emission source ES-132 the dense areas must have a more than 1000 points in
their FuelFlowPrimaryRange. Since the anomaly detection framework is based on the distances
of a new points with its neighbours from the training dataset, the model is expected to perform
better in the dense areas of the feature space than in the parse areas. More precisely, the parse
areas are expected to have a greater false alarm ratio than in the dense areas.

4.2.4
Discussion
The anomaly detection framework showed promising results in the particular case of combus-
tion inputs for three emission sources. The combustion inputs have a high correlation between
each others ≥0.90 and from the emission sources a large number of historical points are available.
Despite the low number of points for low values of FuelFlowPrimary corresponding mainly for
points in start-up mode, the anomaly detection framework was able to distinguish between the
none anomalous input values and the anomalous one. The high correlation between the inputs
allow to have parse areas in the feature space, since the inputs have strong linear relationship
between each other, so not all the possibilities of the inputs values need to be in the training
dataset. Therefore it would be interesting to investigate how the model perform according to the
correlation of the inputs and quantify the density of points needed in the feature space according
to this correlation for the anomaly detection framework to be efficient. For example, due to a
lack of time this model has not been applied on the ambient inputs, which need to be done since
there are also used by the PEMS to compute the emissions. However, the ambient inputs do not
have strong correlation between each other, so it would be interesting to assess how the anomaly
detection framework is performing on these inputs.
This anomaly detection framework relies on two main elements: the training dataset, used to
find the neighbours of a new point, and the way to compute anomaly score; how to quantify the
anomaly of an input value. The challenges regarding the training dataset are that it requires cover-
ing as much as possible realistic cases and to not contain anomalous points that would potentially
lead to an underestimation of the anomalous points. The way of computing the anomaly score is
also primordial because it is the anomaly score that enables to mark an input value as anomalous
or not. As discussed in the thesis this anomaly score can be computed in several ways which
require more or less computations. The mean distance method seems to be the most complete,
since it takes into account the density of the training dataset in the area in the feature space of a
new point. Also, by using the mean distance between all the neighbours it reduces the impacts of
potential anomalous points in the training dataset.
Finally, it has been seen that the anomaly detection framework has many hyperparameters that
need to be optimized for each emission source. Thus, to use this framework on an emission source,
it requires building a robust training dataset with no anomalous points and that covers as much
as possible all the possibilities of the inputs values and the hyperparameters need to be optimized.
In this thesis, the hyperparameters used have not been found using an optimizer because of the
lack of time to implement it. Therefore, to use this framework and implement it for many emission
sources, it would require implementation of an optimizer and efficient processes to build a robust
training dataset for each emission source. It would also require creating a validation dataset where
the anomalies are defined to be used by the optimizer to find the best hyperparameters. This
60
==PAGE:66==
validation dataset would be divided into at least two parts, one subset would be used to optimize
the hyperparameters and one to validate the framework with the hyperparameters found.

4.3
Future outlook
4.3.1
Prediction mode model
To reduce the work necessary to implement the prediction mode model for new emission sources
such as gas turbine or gas engine, a unique labeled dataset could be explored. Indeed, it could be
investigated if it is possible to create only one labeled dataset that could be used for all turbines
to predict the turbine mode. To achieve this it would required to find a way of normalizing the
features used by the labeled dataset according to some physical values that depend on the turbine
dimensions and type of instance. Then, for a new turbine, its features would be normalized and it
would use the normalized labeled dataset to predict the turbine modes. Moreover, as mentioned
previously, some future work could be to implement this prediction mode model on other fields and
to also compared it using known dataset to assess its efficiency with already known classification
models. This work will enable to assess if the prediction mode model proposed in this thesis can
be generalized to other fields and then be a relevant solution to classification problems in context
of time series.
The prediction mode model could also be used as another layer of the anomaly detection
framework. Indeed, the prediction mode model using QDA compute the likelihood of a set of
inputs to belong to one of the system modes.
Some investigation could be done to define a
likelihood threshold, which would indicate if the point is good enough. If the highest likelihood
of a point to belong to one of the system modes is below a threshold the point could still contain
anomalous inputs, that have not been marked by the anomaly detection framework.
4.3.2
Anomaly detection model
The anomaly detection framework has only been tested on combustion inputs of power plants.
These inputs are highly correlated which make the use of K-NN algorithm efficient. It also enabled
to define linear models that predict an input value from the others inputs according the load.
These models are used in the physical check step. Thus, it would be interesting to implement this
model in another context. If the system studied has many inputs they could be divided in sub set
of inputs according to the relation between them. In the case of the emission sources the inputs
have been divided into combustion and ambient inputs for physical reasons. By implementing this
framework in another field it would assess the generality of the model, if it is a robust model that
can be used in several contexts. It could be assessed what are the requirements for the model to
perform well.
Furthermore, this framework must be tested in a real time scenario. In this thesis, the frame-
work has been developed to be used in real-time, however, it has only been tested on data as a
postprocessing tools. Thus, some works need to be done to explore how this model can be actually
implemented in real-time. Moreover, it should be also explored how to use this framework. Indeed,
how can the results from this model be trusted, should only the imputed value from the strong
anomalous inputs be used and for the weak anomalous inputs maybe just provide feedback on
potential anomalous inputs, avoiding computing emissions on fake inputs. How define the limit
between the strong and the weak anomalous points?
Finally, it could be interesting to find ways to easily implement this framework to new emission
sources and fields. One idea could be to be able to create common dataset which would avoid to
create a dataset per emission source, requiring a good amount of work, especially if the anomaly
detection framework needs to be implemented in many systems.

Chapter 5
Conclusion
This thesis aimed to define a real-time anomaly detection model for multivariate time series
sensors value within multi-mode systems. The model needed to be able to mark the anomalous
inputs and replace the marked one using imputed value with a quantify uncertainty.
A. The existing anomaly detection methods shown some flaws and are not complete
enough to fulfill all the requirements defined. Statistical methods like PCA and MARIMA are
simple and computationally efficient but fail to handle non-linear data and quick transitions be-
tween modes. Machine learning methods such as SVM, k-NN, and Isolation Forests offer flexibility
and handle non-linear relationships but require extensive labeled data and are computationally
intensive. Deep learning methods, including RNNs, LSTMs, and CNNs, excel at capturing com-
plex patterns but demand significant data and computational resources, and lack interpretability.
Hybrid methods combine various techniques to enhance accuracy but are complex to design and
implement, and have high computational costs.
These methods do not enable to detect in real-time anomalous inputs and to provide a re-
placement value for the anomalous ones with an uncertainty, in a context of multi-mode systems.
Therefore, this thesis developed two models, one to predict the mode of the system and the second
to detect the anomalous inputs and replace missing inputs or anomalous inputs with an imputed
value along with an uncertainty.
B. Prediction of turbine modes model enables to make PEMS model used to predict the
emissions more precise. Indeed, according to the system mode various assumptions can be applied
on the system which enable to model the system, for instance the assumption of steady-state
cannot be applied in start-up mode but can be applied in operational mode.
• This thesis shown that Quadratic Discriminant Analysis (QDA) models combined with spe-
cific features both spatial and temporal, were able to predict in real-time with high accuracy
the mode of the turbine for each timestamp, using current point and the previous 2 to 13
data points.
• The method developed in this thesis requires around 100 data points per mode to build an
efficient labeled dataset. It has also been shown that the same dataset could be used to
predict the turbine mode of several turbines of the same type.
• Therefore, this model can be easily applied to a large amount of emission sources requiring
few implementation work and with high performance.
63
==PAGE:69==
• However, this model needs to be compared with known classification models to assess its
performance compare to existing models.
C. Anomaly detection framework improve the robustness of PEMS model which predict
the emissions for emission sources such as gas engine or gas turbine. Indeed, this framework can
lower the impact of sensor errors on the inputs used by the PEMS by detecting and replacing
anomalous inputs and missing ones with imputed value.
• The anomaly detection framework developed in this thesis is based on K-NN algorithm. The
use of K-NN algorithm is twofold.
• Firstly it enables to quantify the anomaly using an anomaly score calculated using the mean
distance method. This method computes the anomaly score by normalizing the mean distance
between the new point and its neighbours by the mean distance among the neighbours
themselves. This method enables when computing the anomaly score of a new point to take
into account the density of points in the area of the feature space where is the new point, in
the training datasets used to find the neighbours. This method allows to efficiently gives an
anomaly score representative of the different areas of the feature space.
• Secondly, the K-NN algorithm is used to impute a missing input or an anomalous input using
the nearest neighbours value weighted by their distance with the imputed point.
• Moreover, using the central limit theorem on the distances between the imputed point and its
nearest neighbours, an uncertainty is provided with the imputed value. A specific approach
is developed to be able to find the anomalous inputs that are responsible for the anomaly of
the point.
• Finally, to make the model more robust to unknown areas of the feature space, a physical
check is proposed to verify if the inputs marked as anomalous are genuinely so. This check is
based on models that predict each input value using the other input values, and the model
parameters of these models depend on the load of the emission source.
• This framework shown promising results, it is able to detect anomalies in each of the system
mode even the modes that have a parse density in the feature space of the training dataset
used to find the neighbours. Using this framework has been shown to improve the precision
of the PEMS model by providing information that allows for the computation of emissions
over periods that might otherwise be incalculable, which can account for more than 10 % of
the year, in some cases.
Both the prediction mode and anomaly detection models demonstrated good results. Specif-
ically, the anomaly detection framework developed in this thesis satisfies all the requirements
mentioned in the objectives of this thesis. It enables to detect anomalous inputs in real-time,
providing imputed value for the anomalous inputs or the missing ones. However, they perform
well in the particular case of combustion inputs, with strong correlations between them and in the
context of emission sources, such as gas turbines and gas engines. Therefore, these models should
be implemented in other fields where inputs may have weak correlations and compared to existing
models to assess their performance compared to known methods.

B.2
Anomaly detection
B.2.1
Metrics results
This section described the calculations of the metrics bias score (BS), false alarm ration (FAR)
and critical success index (CSI) for several scenario and hyperparameters values. The hyperparam-
eters values assessed are described in the table B.1, the values of some hyperparameters depend
on the anomaly score method used. To compute the metrics, all the hyperparameters are fixed to
their default value also mentioned in the table B.1 except one that will vary from its minimum
value to its maximum using a specific step. The metrics are also computed using various anomaly
score methods, explained in section 3.3.1.2:
• Farthest neighbour: Distance between a new point and its k-th farthest neighbour;
• Mean distance: Normalization of the mean distance between a new point and its k neigh-
bours by dividing this mean by the mean distance of the mean distance of its neighbour
between them and their own neighbours.
