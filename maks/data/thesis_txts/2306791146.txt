Abstract
No environment is static, and this is neither the case for the cityscape areas.
Due to a contentious growth of available data, it has recently become more
relevant to include a larger variety of data types when doing change detection.
A data type that is very relevant to detect changes in the cityscape is the
digital surface model. Previously change detection has primarily been carried
out using spectral images. The relevant question to ask is: How can these
data types be combined to identify relevant changes in diﬀerent applications,
and speciﬁcally for the cityscape application: how can spectral orthophotos
be combined with digital surface models to identify relevant objects such as
buildings. Diﬀerent approaches have been tested, including extending the
IRMAD method developed by A. A. Nielsen [7] to include the digital surface
model. In order to ease the process of inspecting the changes a post ﬁltering
method has been tested and applied to remove irrelevant changes. The result
showed promising detection certainty, since the data contains the information
to discriminate the relevant objects namely changed buildings. The approach
of handling multiple data types have been clariﬁed and it has become clear
that there are challenges when diﬀerent data types are to be used together
in a change detection algorithm.

Chapter 1
Introduction
1.1
Motivation
No matter where you look in the environment changes are happening. Bridges
are being built, crops are growing, seawater levels are changing. In a large
number of cases it is of interest for companies, governmental departments
and research studies to monitor these changes. Due to the availability of
high quality, widely covered and temporally frequent update of data, this
monitoring can be conducted.
In many cases related to urban planning, management and control, it
is interesting to know where new constructions have been made or where
buildings have been torn down.
This information can be extracted from
satellite images or aerial images and used in these administrative contexts
etc.
When the national GIS-maps must be updated orthophotos are manually
inspected and and the extend of a building or a road is mapped into the
system. This process requires a lot of employees and is also rather trivial.
Using a change detection algorithm the GIS operator only has to inspect areas
where change has occurred, which can potentially save a lot of time. Already
today methods are being developed to classify buildings in orthophotos. To
minimize the data that has to be processed by the classiﬁcation algorithm, a
change detection algorithm could preprocess the image and select the relevant
area.
In contexts such as deforestation it is also relevant to point out areas
where forest has been removed in order carry out control and prevent illegal
deforestation. In this case change detection algorithms have been used to
point out areas of interest.
The applications of change detection are many and therefore the improve-
6
==PAGE:11==
ment of these change detection algorithms will improve the decision making
world wide in many departments.
In the past years an increasing number of satellites, aircrafts and drones
has been mapping areas not only using spectral images but also using laser-
scanners (referred to as LIDAR). These LIDAR systems provide high res-
olution Digital Surface Models (referred to as DSM). Recently the Geo-
datastyrrelsen (referred to as GST) has updated the freely available DSM
in a 40 cm resolution. These DSM’s have been shown to be useful in change
detection cases such as building change detection shown by Hiroshi Murakami
[6].
Now these 2 types of data provide diﬀerent information about the scene,
both available to identify changes in typical applications. A relevant inves-
tigation would be to combine this information in an intelligent manner to
improve the change detection certainty.
The mapping department at the consultant company NIRAS has shown
interest in this perspective of change detection.
A collaboration in this
project has been decided. The interest from NIRAS is to satisfy their cus-
tomers’ need of detecting changes in the cityscape, and therefore the case in
this project has been chosen accordingly.
1.2
Objectives
Both data types, spectral ortophotos and DSM’s, explains some characteristic
of the surface of the earth.
Each some speciﬁc characteristic, which are
not strictly correlated. Some objects might change shape, and others might
change in color.
What could be interesting here is to see if an object is
removed from or added to the scene.
This will often cause a change in
both parameters namely the reduced height and the revelation of the surface
below, which often has a diﬀerent spectral footprint. Therefore it is relevant
to develop a method that can combine the spectral and height information
in a change detection algorithm, in order to increase the certainty of change
detection methods. The overall objective is:
How can ortophotos and DSM be combined in a change detec-
tion algorithm?
In collaboration with NIRAS a dataset and a case have been chosen.
An area in Copenhagen is being mapped by drones this spring 2016, and
spectral orthophotos are constructed. From GST orthophotos and DSM are
available from a ﬂight campaign in 2014. The data from GST are available
from the following link: http://download.kortforsyningen.dk/. The only
missing part is the DSM from 2016. This dataset will be constructed from
7
==PAGE:12==
the orthophotos using stereo vision methods. These methods will not be
covered in details in this report.
This project will deal with the case where the 2 datasets must be com-
bined. By now methods are known to handle each of the two data types
separately. These methods will be the foundation of the investigation and
will therefore be applied to respectively the spectral part, and the DSM.
For the spectral part methods from multivariate statistics, including canon-
ical correlation analysis (referred to as CCA) developed by Hotelling [4], are
used for change detection. For example the IRMAD algorithm developed by
A. A. Nielsen [7].
The standard method for change detection on DSM is not very sophisti-
cated due to the fact that there is only 1 parameter in the dataset namely
the surface height. The method is to subtract the heights and compare with
the uncertainty of the height measurements. An example of this method
applied is the Murakami’s Change detection of buildings using an airborne
laser scanner [6].
1.3
Risk analysis
The main risk in this project is that the data from NIRAS will not be avail-
able. But this is not a severe risk for the project since the GST can provide
both DSM and orthophotos for the project. The main focus which is the
methodology does not require a speciﬁc dataset.

This chapter will cover some of the most relevant existing methods regarding
the applied change detection method in this report.
The covered meth-
ods in this chapter is Image diﬀerencing applied to DSM, IRMAD applied
to spectral images, Markov Random Field modelling [5] and Mathematical

2.1
Digital Surface Models
Digital surface models can either be sampled using a a laser scanner or a
camera by post processing the images with stereophotogrammetry methods.
DSMs are in general more expensive to produce than spectral orthophotos
especially if they are mapped using the laserscanner due to the instruments
required. Normally the resolution is not as high as for the spectral images.
Especially not compared to RGB images taken with modern cameras.
Figure 2.1 show 2 stretched gray scale images corresponding to the DSM
heights at 2 diﬀerent timestamps. The standard method for detecting changes
in DSM images is to subtract them from each other, and compare the diﬀer-
ence with a threshold.
The white areas in the change image indicates change and black areas are
no-change. Several larger structures appear as changed which is expected,
but also a lot of smaller areas indicate change which might be true, but maybe
not as relevant for a typical case such as building change detection due to
the typical size of buildings. These small changes could be cars, trees or
similar objects that have changed position between the two ﬂight campaigns.

2.2
Change detection in Spectral Images
2.2.1
Multivariate Alteration Detection
The method Multivariate Alteration Detection (referred to as MAD) is a
statistical change detection method that uses elements from multivariate
statistics such as CCA. The method is developed by A. A. Nielsen [7]. The
method is applied to bi-temporal multivariate datasets fx spectral bands. As
the case is for change detection of spectral images the method is applied to
2 sets of images with the same spectral bands. Shown below p corresponds
to the number of spectral bands.
X = [X1 · · · Xp]
(2.1)
Y = [Y1 · · · Yp]
(2.2)
The method assumes that the mean of each variable is 0, and since im-
age pixels shows light intensity, the mean is not 0. Therefore the mean is
subtracted for each spectrum such that the mean is now zero.
In order to compensate for linear diﬀerences in the two images not caused
by real diﬀerences, the CCA is applied to the 2 image. These diﬀerences
could be caused by that the images are not taken by the same camera which
might result in scaling diﬀerences. Other linear eﬀect will also be removed
by applying the CCA. The CCA takes each variable (here spectral bands)
in each dataset and transforms it linearly such that the correlations between
them are maximized. The linear transformation is carried out by multiplying
each pixel in the two images with its corresponding vector estimated by the
CCA. The two vectors applied which leads us to the canonical variates (CV )
are shown below.
CVX(p) = aTX = [a1X1 + · · · + apXp]
(2.3)
CVY (p) = bTY = [b1Y1 + · · · + bpYp]
(2.4)
These two new datasets (images) are called the canonical variates, each
containing p variables, where n is the number of spectral bands for the initial
images.

no-change probability of each pixel this can be used as input. The formula
for no-change probability is shown in eq. 2.12

2.2.3
Dispersion matrix determination
The dispersion matrix determination is an essential part of the MAD algo-
rithm. The deﬁnition of the dispersion matrix is:
Σij = cov(Xi, Xj) = E[(Xi −µi)(Xj −µj)]
(2.13)
, where Xi is a vector of the i’th variable of all pixels. The case is similar
with Xj. The number of variables correspond to the total number of spectral
bands from both images. So 2 images with 3 spectral bands will result in a
dispersion matrix with size of 6 by 6. µi and µj are the means of the i’th
and j’th variable.

The case is often, if a statistical method is applied to a large data set,
that the entire data set can not be contained in memory, and therefore the
data must be loaded in chunks fx. line-by-line. Since the mean must be
estimated, the trade oﬀis so that either the data set must be loaded twice
(in order to determine the mean) or the provisional means must be used.
The method of loading data twice was implemented naively by ﬁrst load-
ing each row by turn, accumulating the pixel values for each variable, in each
image, dividing by the total number of pixels in the end. Loading each row
again by turn allows to accumulate the product of the standard deviation
of each image for each variable, again dividing by the number of pixel at
the end. This approach leads to small computational errors, which is accu-
mulated. The reason is that when small numbers are added to a very large
number, the small number might not contribute due to the bit resolution
being larger.
Provisional means and dispersion
The provisional means method was implemented in order to reduce compu-
tation time by avoiding loading data twice but also due to the previously
mentioned computational errors.
The method is described in BMDP Statistical software, p. 662 [3].
The comparison of the two methods shows very little diﬀerence in the test
data. Only small deviations in the residuals in the order of 10−3 shows that
the precision is almost the same. Regarding computation time, the result is
also approximately the same.
Therefore the provisional means method was applied in the IRMAD im-
plementation, since this method is the most accurate.
2.3
Thresholding by means of χ2-fractile
When selecting the threshold value, the theoretical solution is to take all
the no-change pixels and ﬁnd the value that selects a high fractile of that
group of data, for example 0.95 or 0.995. This means that all values that
are above that limit are classiﬁed as change. In a set of images where many
things have changed a bit it might not be the best solution to just select all
changes. In some cases it might be more interesting just to take the most
signiﬁcant changes. This is highly dependent on what you are looking for
and how the data looks like.
Since each variable is normalized to unit variance before calculating the
sum of squares of MAD-variates, the χ2-image, follows the χ2-distribution,

Markov random ﬁeld modelling can be divided into 2 parts. The mod-

The modelling part is a formulation of an optimization criteria namely the

The optimization is the algorithm that ﬁnds the state of the image that

Monte Carlo methods. It depends on the approach.

2.6
Mathematical morphology
The basics of Mathematical morphology consists of 2 basic operations called
erosion and dilation. From these two operations 2 others are formed, called
opening and closing. By ﬁrst performing the erosion followed by the dilation
is called the opening method and in reverse order we have the closing.
These operations was ﬁrst implemented for binary images, and later for
gray scale as well. Relevant literature would be such as Serra’s Image Anal-
ysis and Mathematical Morphology [8]. The erosion method is an operation
where a kernel, called the structure element is mapped over the image testing

for occurrence of a logical 0, if found the corresponding pixel of the location
of the kernel becomes 0. if none is found within the structure element the
pixel will remain the same. The dilation is a very similar algorithm except
it triggers on 1 and the pixel becomes 1, instead of 0. Basically these two
methods are each others opposite.
When performing the opening, clusters smaller than the structure element
will be removed completely by the erosion. Clusters larger than the structure
element will remain but with a slight change in shape and dependent on the
shape of the structure element. Objects tends to have shapes similar to the
structure element.
The structure element can be chosen to have diﬀerent shape and size
dependent on the desired result. The typical structure element is chosen to
be either a disk, diamond or a square. The decision of choosing one or the
other depends highly on the desired shape of the resulting objects.
This method is used to remove single pixel changes, which are often not
of interest but more likely noise.

Chapter 5
Discussion
For some change detection applications it is relevant to observe a very speciﬁc
type of change, and therefore the sampled data type is then selected to map
that speciﬁc change. But for some applications it might not be a case where
there is a speciﬁc type of change that you are interested in. Therefore the
approach developed in this project can be relevant in other cases where a
broad range of changes are of interest, and therefore multiple data types
must be incorporated.
The algorithm that has been developed have though not been tested as
a general change detection method, since a large variety of data was not
available, but could potentially be relevant in any change detection case
where multiple data types must be integrated, to capture a broad range of
types of changes.
The challenge by combining data types of diﬀerent physical origin, is that
there is not a natural scaling between the two data types. When using the
binary and/or method the operator must select a threshold for each of the
diﬀerent data types, which might result in a biased threshold in relation to
the desired changes. The advantage of using the combined IRMAD method is
that the CCA selects the data types that shows the most signiﬁcant changes
by means of maximizing correlations, where large deviations from highly
correlated variables are considered as signiﬁcant changes. This means that
the variables that are the most signiﬁcant for determining a speciﬁc type of
change can be determined by comparing the MAD variables (see fx. ﬁgure
4.13) with the correlations of MADs with the input data (see fx. ﬁgure 4.17).
In this example it can be determined that the 1st MAD variate can alone
be used to classify changed buildings.
By inspecting the correlations be-
tween MAD and input data, it is shown that this 1st MAD variable is only
dependent on the DSM. One might think that this is obvious of the cityscape
example but for other examples it might not be as obvious. For other appli-

cations it might be so that the relevant change is not located in the 1st MAD
variable.

Chapter 6
Conclusion
The purpose of this project was to investigate methods for including the
2 data types e.g. spectral orthophotos and DSMs into a common change
detection algorithm. It has been successfully shown that the 2 data types
can be combined to a common change detection algorithm that can detect
changes in either 1 or both of the data sets. Diﬀerent methods have been
developed to investigate and compare performance. Since no ground truth is
available the methods have been evaluated by more qualitative means. The
combined IRMAD method has some advantages, that makes it suitable for
change detection, especially where the desired type of change is not known,
and where the data types are correlated (which was not the case in this
application)
The investigation does only cover this 1 application which successfully
detected changes in a cityscape with focus on buildings. For this speciﬁc
application it has been shown that it is possible to identify what the most
signiﬁcant variable is, which for this application is the DSM. For further
applications where only buildings are of interest it might not be needed to
include the spectral information.
Furthermore the second goal was to implement the method so that the
change detection can be applied in data production in companies such as NI-
RAS. The method has been fully implemented with the goal of making the
solution scalable and streamlined for production. How successful the imple-
mentation of making the algorithm fully scalable for large data productions
would require further software tests to verify the algorithms but that is not
in the scope of this project.

Chapter 7
Perspective
In order to further develop the method of combining multiple types of data,
such as spectral orhtophotos and DSMs, a number of data sets would be
required in order to evaluate how well the method performs. As having only
1 application with just 1 data set the method can only be adapted to this
speciﬁc application. A large variety of spatial raster data could be included
in the method to improve a speciﬁc type of change that is relevant.
Keeping in mind that the purpose partly was to construct a software
program, that can perform the change detection, further improvements re-
garding the functionality and interface is required in order to be able to use
it eﬃciently for operators without any programming expertise. Speciﬁcally
it could be useful for the operator to be able to select the type of change that
is of interest in this speciﬁc case, and let a classiﬁcation algorithm select the
object belonging to that speciﬁc class.
Classiﬁcation
For cases where the the desired type of change is not known, a unsupervised
classiﬁcation of the changes could be useful to separate the type of changes in
order for the operator to get a deeper understanding of the types of changes
in the scene.
For a case where a speciﬁc type of change is desired a supervised classi-
ﬁcation. It might not be the case that the most signiﬁcant changes are the
ones of interest. A supervised classiﬁcation of changes would make it possible
to reduce the number of irrelevant changed objects when a speciﬁc change
is desired. A good example is the application here where the the changes of
the trees are undesired, but sometimes overlapping with the buildings. The
trees can easily be separated as shown in the DSM change detection section
4.2.1 in chapter 4.
57
==PAGE:62==
Having a classiﬁcation algorithm coupled to the change detection method
would make the algorithm useful in many general change detection applica-
tions.
Software development
The desire have been to make the algorithms able to run on any size of
images, which have not yet been tested on large images, but also some of the
functionality have not been implemented line-by-line due to time restrictions,
and would probably require some optimization.
The IRMAD have been
implemented as line-by-line calculations, but some of the extra functionality
such as plotting diﬀerent types of data, or generating other results, have been
implemented by loading the whole image into memory at once.
Unlinear change detection methods
This project has only been working with the linear elements of statisics such
as CCA, but an nonlinear method has been developed called Canonical In-
formation analysis (CIA) that has also been applied for change detection.
This metods has some advantages over CCA, but is also more comprehen-
sive. For some applications the nonlinear approach migt be able to solve
some correlation isssues that the linear might not.
