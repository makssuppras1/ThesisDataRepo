Abstract
This thesis considers the problem Sequence Building with Narrow Time Windows,
which can be seen as the Traveling Salesman Problem with Time Windows (TSPTW).
The problem consists of determining a route that starts at a depot, then visits all
orders exactly once within their time window before returning to the depot with an
objective of minimizing the weighted sum of the total distance and duration.
The proposed solution method builds the sequence around the orders with narrow
time windows and then improves the solution using an Adaptive Large Neighborhood
Search heuristic.
The method was tested on 68 instances with orders ranging from 23 to 331, showing
very good results. The proportion of orders with narrow time windows varies between
0-18%. The approach finds a feasible sequence, respecting all time windows for all
instances. When comparing results, the heuristic improves the distance for all Fast
mode instances. In 20 of 23 instances, the heuristic improves the distance in Normal
and Thorough Mode.

CHAPTER 1
Introduction
Transportation is essential to modern society. Transportation enables the movement
of goods between locations, including waste collection and recycling. However, trans-
portation accounts for about 64% of global oil consumption, 27% of all energy use,
and 23% of the world’s energy-related carbon dioxide emissions (Mead, 2021). As a
result, increasing numbers of companies are emphasizing sustainable business prac-
tices. One option is route optimization. Route optimization is the process of finding
the best possible route for better eﬀiciency and cost-effectiveness. By increasing ef-
ficiency, companies can reduce fuel use, cut carbon emissions and work in a more
environmentally conscious way.
In this thesis, the problem Sequence Building with Narrow Time Windows, which
can be seen as the Traveling Salesman Problem with Time Windows (TSPTW), is
addressed. The TSPTW is a variant of the well-known Traveling Salesman Problem
(TSP). The standard TSP is defined as follows. Given a set of cities and the distance
between each city. The problem is to find the shortest possible route so that each city
is visited exactly once before returning to the starting point. The TSPTW problem
has been addressed using many techniques, including exact methods and heuristic
approaches. However, because the problem is NP-hard, since it is a variant of the
TSP, exact methods can be limited when the problem size increases.
This thesis addresses an asymmetric TSPTW. The objective is to find a minimum-cost
path that starts at a depot and services each order exactly once before returning to
the depot. Each order must be visited within a given time window. Some orders have
a very narrow time window, increasing the problem’s diﬀiculty. The cost function
is a weighted sum that considers the total distance and duration of the route. The
problem and real-life instances were provided by AMCS, a software company that
provides solutions for the waste, recycling, and resource industries.
Because of the instance size and the added complication with the narrow time win-
dows, a metaheuristic method is proposed to solve the problem. The metaheuristic
proposed is an Adaptive Large Neighborhood Search (ALNS) heuristic, which has
an adaptive nature and provides robust performance in different instances. First, an
initial solution is constructed using a construction heuristic. This method builds the
sequence around the orders with narrow time windows. Then, the ALNS is used to

CHAPTER 2
Literature Review
This chapter reviews the relevant literature on the problem addressed in this thesis.
The problem is a variant of the Traveling Salesman Problem with Time Windows
(TSPTW). First, the TSPTW and other routing problems are described. Then, rele-
vant solution methods from the published literature are reviewed.
2.1
Routing Problems
The Traveling Salesman Problem (TSP) is a well-known problem in theoretical com-
puter science and operations research. The objective of a standard TSP is to minimize
the cost of traveling to a fixed set of customers from a single depot before returning
to the depot. Each customer must be visited once. The standard version of TSP is
a complex problem to solve and belongs to the NP-Hard class. Another problem is
the Vehicle Routing Problem (VRP) which was introduced by (Dantzig & Ramser,
1959). and is a generalization of the TSP. As with TSP, the standard VRP consists
of a single depot and a set of customers, but it also includes a fleet of vehicles at the
depot. In other words, the TSP generates a single route that connects all customer
locations, whereas the VRP can generate several routes that connect all customer
locations. Therefore, TSP can be considered a VRP with only one vehicle with no
capacity limitations.

The Traveling Salesman Problem with Time Windows (TSPTW) is a time-constrained
variant of the TSP. It entails determining the cheapest route for a vehicle that begins
and ends at the same depot and must visit a set of n customers (nodes) exactly once.
Each pair of customers has a travel time associated with it. The service at node
i should begin inside the node’s time window [ai, bi]. Early arrivals are permitted
because the vehicle may arrive before the lower bound of the time window. However,
in this situation, the vehicle must wait until the node is ready to begin service before
proceeding. (M. W. P. Savelsbergh, 1984) demonstrated that even finding a feasible
solution to TSPTW is NP-complete.
2.2
Solution Methods
The following section discusses some of the most relevant methods in the literature.
For example, TSP and TSPTW can be solved optimally using exact methods if the
instance is not too large. However, with increasing numbers of customer locations,
the computational complexity of an exact algorithm for solving these problems is ex-
ponential. Therefore, when the instance is too large, heuristic methods are preferred.
However, those methods can not guarantee optimality.
2.2.1
Exact Methods
Dynamic programming is an algorithmic strategy that supports many TSPTW solu-
tion approaches.
(Dumas, Desrosiers, Gelinas, & Solomon, 1995) present a dynamic programming-
based algorithm for the TSPTW to minimize the total traveling cost. It uses elimina-
tion tests that take advantage of time window constraints to significantly reduce the
state space and the number of state transitions. Presented are computational results
on problems with up to 200 nodes and fairly wide time windows.
(Mingozzi, Bianco, & Ricciardelli, 1997) and (Baldacci, Mingozzi, & Roberti, 2012)
present a dynamic programming-based algorithm using state-space relaxation that
was introduced by (Christofides, Mingozzi, & Toth, 1981). (Mingozzi et al., 1997)
presented computational results for randomly generated test problems with up to
120 nodes. In (Baldacci et al., 2012), the solution was computed using a dynamic
programming algorithm that uses bounding functions based on numerous tour relax-
ations and the resulting dual solution. The authors present computational results
of nearly all TSPTW instances, with up to 233 nodes, found in the literature and
compare those results with four state-of-the-art exact methods. The presented re-
sults demonstrate that, except for one instance, the proposed method outperforms
all known exact methods at the time.
==PAGE:17==

(Tilk & Irnich, 2017) presents a new dynamic programming algorithm for minimiz-
ing the tour duration of the TSPTW. They name this variant of the TSPTW the
Minimum Tour Duration Problem (MTDP). It generalizes the method presented by
(Baldacci et al., 2012) and offers a new dynamic programming formulation in which
a new resource extension function is defined to permit relaxation and forward and
backward dynamic programming. Results are presented where the approach can solve
over 90% of the instances with up to 125 nodes.
Integer Programming is another algorithmic strategy that has been used to solve the
TSPTW.
(Ascheuer, Fischetti, & Grotschel, 2001) presented a branch-and-cut algorithm for
the Asymmetric TSPTW. It presents three models using a cutting plane algorithm
as an integer linear program and solves instances with up to 233 nodes.
Another branch-and-cut algorithm is presented by (Dash, Günlük, Lodi, & Tramon-
tani, 2012).
The authors presented a formulation based on partitioning the time
windows into buckets by solving linear programming problems. The LP relaxation
of this formulation generates strong lower bounds, which are then used in a branch-
and-cut algorithm. It presents computational results on asymmetric instances from
the literature and randomly generated symmetric instances.
(Boland, Hewitt, Vu, & Savelsbergh, 2017) proposes an integer programming ap-
proach for the TSPTW. The approach is based on formulating the problem on time-
expanded networks and then solving the formulation with a procedure called Dynamic
Discretization discovery (DDD). This procedure was proposed in (Boland, Hewitt,
Marshall, & Savelsbergh, 2017) to solve the Service Network Design problem. They
present computational results that show that the DDD procedure, using partially de-
signed time-expanded networks, produces high-quality solutions for the TSPTW and
that the algorithm can easily be modified for other variants of the TSPTW.
(Langevin, Desrochers, Desrosiers, Gélinas, & Soumis, 1993) present a two-commodity
flow formulation to solve the TSPTW. The problem was solved with an arc elimina-
tion procedure by adding violated sub-tour elimination constraints and a branch-and-
bound scheme. Presented are numerical experiments on problems with up to 60 nodes.
They note that the algorithm is suitable for the standard TSPTW and the makespan
problem, where the objective is to minimize the completion time of the route.

2.2.2
Heuristics
Due to the complexity of the problem, exact methods frequently cannot solve large
instances in a reasonable amount of time. Due to these limitations, much research
focuses on solving the TSPTW using heuristic methods.
(Gendreau, Hertz, Laporte, & Stan, 1998) presented an insertion heuristic to mini-
mize the travel times of TSPTW, based on the GENIUS heuristic for the symmetric
TSP introduced by (Gendreau, Hertz, & Laporte, 1992). The algorithm creates a
set with all vertices in non-decreasing order of their time window width and then
gradually builds a route by attempting to insert those vertices between two of its
nearest neighbors. If no feasible insertion is possible, a backtracking method is used.
This process continues until all vertices are placed into a feasible position on the
route. The algorithm avoids cycling by keeping track of how many times a vertex is
removed from the route during the backtracking process. When that value exceeds
θ, the process terminates with an infeasible solution. A post-optimization procedure
is applied if the algorithm produces a feasible initial solution. The procedure entails
removing and reinserting each vertex in turn. Acceptance of a solution is dependent
upon its improvement over the current one. The algorithm is tested on 375 instances.
The algorithm presented in this thesis utilizes a similar method for building an initial
solution as presented by (Gendreau et al., 1998).
(Ohlmann & Thomas, 2007) introduced a compressed-annealing heuristic to mini-
mize the cost of the tour, a variant of simulated annealing that incorporates variable
penalty methods. The cost of the tour is the travel time between customers, includ-
ing the service time for each customer. The algorithm considers infeasible solutions
by introducing a penalty function into the objective function that relaxes the con-
straint on time windows. Using this method, they obtained the best-known results
for numerous instances.
(da Silva & Urrutia, 2010) presents a General Variable Neighborhood Search (GVNS)
heuristic. The algorithm is divided into two stages. The first stage is the constructive
stage, during which the algorithm utilizes a Variable Neighborhood Search (VNS)
heuristic to construct a feasible solution. The second stage is the optimization stage,
where the algorithm improves the feasible solution constructed in the first stage with a
GVNS heuristic. They present computational results for four distinct sets of instances.
The results demonstrate that the approach is eﬀicient, saving time and improving on
previously published results in some instances.
All the methods mentioned above aim to minimize the travel time between all cus-
tomers, but they do not account for waiting time. In some cases, this can be costly due
to the time spent waiting required to complete the route. Therefore, another objec-
tive to consider is minimizing the solution’s route duration, defined as the difference

between departure and arrival times at the depot.
The first approach that considered route duration minimization was (M. W. P. Savels-
bergh, 1992) on the Vehicle Routing Problem with Time Windows (VRPTW). The
approach is a generalization and extension of the local search algorithm presented
in (M. Savelsbergh, 1990) and allows time window constraints in edge-exchange im-
provement methods. In addition, the approach considers variable departure times
from the depot because the departure time of vehicles is not fixed but must still fall
inside the given time window.
(Bretin, Desaulniers, & Rousseau, 2021) presents a three-step clustering heuristic to
solve the TSPTW problem in postal service. The problem varies from the classical
TSPTW in two ways. First, route duration is essential. Second, many customers
have no time window, which reduces the eﬀiciency of the standard time windows pre-
processing techniques. They propose an approach with a weighted objective function
to address the importance of route duration, allowing for optimizing both route dura-
tion and travel time between customers. To address the low time window density and
the large real-world instances, they introduce a cluster/solve sequence. The heuristic
starts with clustering some service points to reduce the size of the instance. Then it
solves the clustered problem. Finally, it disaggregates the clustered solution. They
present a sensitivity analysis of the objective function’s weights. They present compu-
tational results on small and medium-sized instances from the literature for various
objective function weights, as well as computational results from their research.
The objective function used in (Bretin et al., 2021) is similar to the one used in this
thesis.
The objective is to minimize a weighted sum of the travel times between
all customers and the route duration. In contrast, in this thesis, the objective is to
minimize a weighted sum of the distance between all orders/customers and the route
duration.
An Adaptive Large Neighborhood Search Heuristic (ALNS) has been used on many
routing problems and has performed well on large-scale instances. ALNS was intro-
duced by (Ropke & Pisinger, 2006) for the Pickup and Delivery Problem with Time
Windows (PDPTW). PDPTW is a generalization of the VRPTW where each request
needs to be picked up and delivered, meaning that corresponding deliveries need to
be placed on the same route. In (Ropke & Pisinger, 2006) the objective is to minimize
the weighted sum of thee components: The distance, the time spent by each vehicle,
and the number of requests not included in a route. The heuristic is tested on more
than 350 benchmark instances with up to 500 requests. The presented results improve
the best-known solutions for over half of the instances.
In (Pisinger & Ropke, 2007) the same authors present a general heuristic for solv-
ing five variants of the vehicle routing problem using the ALNS framework.
The
computational results indicate that for large-scale VRPTW instances, the heuristic

thermore, they note that as the size of the problem increases, the method becomes
more attractive compared to other heuristic methods.
As mentioned above, the ALNS heuristic has been used widely in the literature. For
example, (Sarasola & Doerner, 2020) solves the VRP with synchronization constraints
where idle times at delivery locations should be reduced. In addition, (Yu, Zhang, Yu,
Sun, & Huang, 2020) presents an ALNS for Green VRPTW where the objective is to
minimize the total carbon emissions in the transportation, and (Konstantakopoulos,
Gayialis, Kechagias, Papadopoulos, & Tatsiopoulos, 2020) presents a multi-objective
Large Neighborhood Search for the VRPTW where the objective is to minimize both
the number of used vehicles and the distance traveled. They all result in good quality
results for given problems.

CHAPTER 3
Problem Description
AMCS is a software company that guides and supports other businesses on their
journey towards sustainability. Their software solutions are used for multiple pur-
poses, including waste-specific route optimization which enables businesses to deliver
significant cost reduction in fuel, wages and more.
One of the problems is Sequence Building with Narrow Time Windows. It is a route
optimization problem where the main goal is finding a minimum-cost sequence that
starts at a depot, visits each order exactly once, and returns to the depot. There are
specific orders that need to be visited on each route. AMCS receives the information
on the orders from their clients, where the orders are located and when they can
be visited, information on the depot, where the route should start and end, when
the route can start, and when it needs to be completed. The cost of the solution is
calculated as a weighted sum of the route’s total distance and duration. This problem
is very similar to the Traveling Salesman Problem (TSP) since only a single route
is optimized at a time. However, the problem includes time windows, which means
each order must be visited within a given time window. This variant is called the
Traveling Salesman Problem with Time Windows (TSPTW). Some time windows are
narrow, where an order has a time window of 30-120 minutes.

CHAPTER 5
Methodology
This chapter presents the methods used to solve the problem: Sequence Building With
Narrow Time Windows. As mentioned above, the problem is NP-Hard, meaning for
large instances, exact methods will not solve it within a reasonable time. Hence, a
metaheuristic approach is used to obtain a near-optimal and feasible solution. First,
a feasible initial solution is constructed using a construction heuristic presented in
Section 5.1. Then, the solution is gradually improved with the Adaptive Large Neigh-
borhood Search (ALNS) approach presented in Section 5.2.
5.1
Construction Heuristic
An initial solution can be, for example, a random solution or a solution by construc-
tion. A Construction heuristic begins with an empty solution and then repeatedly
extends the solution until it is complete. When creating an initial solution for a prob-
lem with time windows, it could be problematic for the last orders if all orders are
inserted randomly. Therefore, the orders are ranked according to their time window
width and then inserted into the route in a non-decreasing order (Gendreau et al.,
1998).
Algorithm 1 shows the heuristic for generating the initial solution. The heuristic is
based on the Nearest Insertion for TSP. In the Nearest Insertion, an order that adds
the least to the objective is inserted into the partial route. In the presented heuristic,
however, only one order is considered at a time, and that order is placed into the
partial route where it adds the least to the objective. This approach ensures that
the orders with narrow time windows are not left until the end. Furthermore, each
order is only placed in a feasible position in the sequence to avoid diﬀiculties moving
between a feasible and an unfeasible route.
The solution z is a set of orders in a route and when the service at each order begins.
R keeps track of orders that are not in the route. The objective value, or the cost of
the solution c(z), is the weighted sum of the distance traveled on the route and the
duration of the route, as mentioned in Chapter 4.

route.
Even though service times are assigned to the depots based on their time window, the
route duration is calculated based on the orders. The route’s start time is calculated
by deducting the travel time from the depot to the first order from the service time
of the first order on the route. The route’s end time is determined by adding the
driving time from the last order to the depot to the service time of the last order.

5.2
Metaheuristic
This section presents the metaheuristic approach used to optimize the problem, start-
ing from the solution obtained with the construction heuristic. The metaheuristic
approach that is implemented is an Adaptive Large Neighborhood Search heuristic.
5.2.1
Adaptive Large Neighborhood Search
The Adaptive Large Neighborhood Search (ALNS) was introduced in (Ropke & Pisinger,
2006) and is an extension of the Large Neighborhood Search (LNS) heuristic. LNS
was presented by (Shaw, 1998) and the neighborhood is defined by a destroy method
that destructs a part of the current solution and a repair method that rebuilds the
destroyed solution. The LNS heuristic belongs to the class of heuristics known as
Very Large Scale Neighborhood search (VLSN) algorithms. All these algorithms are
based on the observation that by searching a large neighborhood, the algorithm can
find local optima of high quality; therefore, a VLSN algorithm may provide better
solutions. However, this can be time-consuming, so various filtering techniques are
used to limit the search.
Algorithm 3: Large Neigborhood Search
Input: A Feasible Solution z
Output: zb
zb = z
while Stop Criteria NOT met do
zt ←r(d(z));
if accept(zt, z) then
z = zt;
end
if c(zt) < c(zb) then
zb = zt;
end
end
The LNS algorithm starts with an initial feasible solution. The main elements are
the destroy method, the repair method, and the acceptance criteria. Algorithm 3
shows the methodology of LNS. The algorithm has three variables: zb is the best
solution discovered during the search, z is the current solution, and zt is a temporary
solution.
The temporary solution is either accepted and promoted to z, or it is
disregarded. There are many different ways to implement the accept function, the
simplest being only to accept improving solutions. As mentioned above, the destroy
method (d(z)) removes a part from the solution, whereas the repair method (r(z))
repairs the solution by reinserting the removed part.

5.2.2
Acceptance Criteria
As mentioned before, the simplest choice of acceptance criteria would be to accept only
solutions that improve the current solution. However, this could cause the heuristic
to get trapped in a local minimum. Therefore, another option is to use the acceptance
criteria from simulated annealing, where the temporary solution is always accepted if
it improves the current solution. If it does not improve, it is accepted with probability
given in Equation 5.1.
e−(c(zt)−c(z))/T
(5.1)
This acceptance criteria can diversify the search and allow the algorithm to explore
new neighborhoods with new solutions. The presented algorithm uses the simulated
annealing acceptance criteria.
The initial temperature T0 is set to a high value,
allowing many solutions that do not improve to be accepted. Due to the diversity of
the problem instances, T0 is determined by examining the initial solution as proposed
by (Ropke & Pisinger, 2006), rather than simply assigning it a value (see Equation
5.2). This initial temperature allows the acceptance of w% worse solution than the
initial solution, with a probability of 50%.
T0 = −w · c(z)
ln(0.5)
(5.2)
In each iteration, the temperature is decreased with a cooling rate of 0 < c < 1,
thereby decreasing the probability that the algorithm will accept non-improving so-
lutions as the search progresses.

will be tuned when testing the ALNS.
The stopping criteria for the algorithm is time.

The most straightforward removal heuristic is a random removal. It removes k num-

Worst Removal Constant
The Worst Removal method updates L before each order is removed. Another option

When an instance contains orders with narrow time windows, it may be diﬀicult to

placed in improving positions. The algorithm for this method is the same as Worst
Removal Constant because the time windows do not change when orders are removed.
L is replaced with the orders in decreasing order of their time window width.
5.2.4
Repair Methods
The repair methods are responsible for reinserting the removed orders into the route.
When the repair methods have identified an order number and position, they are
inserted into the route as described in Section 5.1.3. Two repair methods were im-
plemented, a Greedy heuristic and a Regret heuristic. These methods are prevalent
in the literature and have been utilized by authors such as (Ropke & Pisinger, 2006)
and (Lusby, Schwierz, Range, & Larsen, 2016).
In some instances, a removed order cannot be reinserted because there is no feasible
position along the route. In these instances, the methods return an infeasible partial
solution with an infinite cost. On the other hand, when all orders can be inserted
into the route, the methods return the newly constructed solution.
Greedy heuristic
The Greedy Insertion is a simple heuristic where orders are greedily inserted into the
route based on the cheapest insertion.

When orders have narrow time windows, there are frequently few positions where they
can be placed along the route. In order to reduce the number of infeasible solutions,
the regret heuristic also considers the cost incurred if a particular order is not placed
in its best position during the current iteration.
The regret insertion heuristic closely resembles the greedy insertion heuristic described
above. Similar to the greedy insertion heuristic, it determines the optimal position
for each order in the list R and the cost of placing the order in that position. In
addition, however, it also determines the regret cost of the order. The regret cost is
the difference between the cost of placing an order in its best position and the cost
of placing it in its next-best position. When the algorithm has assessed all orders in
the list R, it reinserts into the route the order with the highest regret cost at position
pi and removes that order from R. This process continues until the algorithm has
placed all orders into the route.

5.2.6
Restarting
With each iteration, the algorithm decreases the temperature, so the probability of
accepting a less optimal solution decreases. As a result, the algorithm may be able
to locate a high-quality solution at some point, then move away from it and never
return to its neighborhood.
Similar to the restarting method in (Perumal, Larsen, Lusby, Riis, & Sørensen, 2019),
the restarting replaces the current solution with the best solution, allowing for a more
thorough examination of the neighborhood surrounding that solution. In addition,
the temperature is recalculated based on the cost of the best solution.
At the beginning of each iteration, a restarting check is conducted. The algorithm
performs a restart if ψ iterations have passed since the algorithm improved the best
solution and ψ iterations have passed since the last restart.

6 Testing
The main objective of parameter tuning is to ensure that the developed heuristic
performs as well as possible for the given problem. Ideally, a tuning strategy where
all combinations of parameters are tested is preferred, but that would require con-
siderable computation time. Therefore, the following tuning strategy is used: All
parameters are set to a specific value. This value was found while developing the al-
gorithm and from the literature. Then, each parameter is tuned individually, allowing
its value to change while the values of the other parameters remain unchanged. When
a parameter has been tuned, its value is set to the determined value, and the following
parameter is tuned. The parameters that the process has not considered keep their
initial value until they are tuned. The improvement from the initial solution cost is
used to determine the performance for a given parameter value.

CHAPTER 7
Results
This chapter presents computational results obtained with the developed ALNS heuris-
tic and an analysis of the ALNS heuristic. First, in Section 7.1 the computational
results obtained by construction heuristic, an LNS heuristic, and an ALNS heuristic
are presented and compared with the obtained results from the AMCS system. Then,
Section 7.2 presents experiments and discussion on the performance of the ALNS.
7.1
Computational Results
The heuristic method presented is cost centered. This cost is a weighted sum of the
distance and duration of the solution route. In this section, computational results are
presented for each instance.
Compared are the results of the initial solution obtained by the construction heuristic,
results from an LNS heuristic using a random destroy method and a greedy repair
method, results from the ALNS heuristic, and the results obtained from AMCS from
their system, for some instances.
Several results from the AMCS system were obtained and compared to the results of
the heuristic method presented in this thesis. Important distinctions appear between
the current AMCS system and the proposed method. The current system at AMCS
provides three available modes: Fast (F), Normal (N), and Thorough (T). Since the
system does not terminate after a predetermined amount of time, the run times of
each instance vary greatly. The time portion of the results is rounded to minutes, so
the exact number of seconds is not presented, and the coeﬀicients of each run can
vary. The main difference, however, is that the current system allows for soft time
windows, so an order’s service time can begin 15 minutes early, and there is no limit
on how late it can be visited. These soft time windows are accomplished using a
penalty function.

Conclusion
The main objective of this thesis was to propose a solution method for the problem
of Sequence Building with Narrow Time Windows. The problem is the same as the
Traveling Salesman Problem with Time Windows (TSPTW), a variant of the well-
known Traveling Salesman Problem, which is an algorithmic problem tasked with
finding the shortest route between a set of points and locations that must be visited.
Most of the literature focuses on minimizing the travel time between each point.
However, for the problem presented, the objective is to minimize the cost, which is a
weighted sum of the total distance and the duration.
The literature revealed that even finding a feasible solution for the TSPTW is NP-
Complete. First, a mathematical model was formulated, implemented, and experi-
ments were conducted. The results of the experiments confirmed that because of how
complex the problem is, it could not be solved within reasonable computational time
using linear programming techniques. Because of the narrow time windows and the
requirement for a fast response from clients, a metaheuristic approach was selected.
An Adaptive Large Neighborhood Search (ALNS) was designed and implemented to
optimize the problem. The heuristic requires an initial solution which was developed
using a construction heuristic. The construction heuristic builds the sequence around
the orders with narrow time windows while minimizing the cost simultaneously. The
construction heuristic only allows an order to be placed in a feasible position in the
sequence. Then, the ALNS is used to improve that solution. The ALNS also only
accepts feasible solutions.
In the parameter tuning of the heuristic, only one parameter was tuned at a time, and
each parameter was tested three times. Because of the randomness in the heuristic,
a more effective strategy would be to test all parameter combinations and each value
more frequently. Nonetheless, the parameter values gave good solutions. Furthermore,
the coeﬀicients of the cost function were also evaluated and determined.
The heuristic approach was analyzed on real-life instances provided by AMCS. The
main advantage of the presented heuristic is that it returns a feasible sequence for
all instances received from AMCS. Two solutions were presented with different coef-
ficients in the cost function. One heuristic has coeﬀicients similar to the ones used
==PAGE:76==
64
Conclusion
in the current system at AMCS. The results are very similar to the AMCS solutions.
Some clients, however, prefer to reduce their driving, so the other heuristic focuses
on that. The solutions have a shorter distance when compared to the three methods
in the current system for all instances except three. For those three instances, the
heuristic only improves the distance of the solution in the Fast method.
Further work could be exploring more ways of placing orders in the sequence to
minimize the duration or consider a multi-objective cost function to optimize both
the distance and the duration at the same time.
This thesis provides a solution method that addresses some of the main challenges
with the current system of a feasible sequence and a good split of orders. The results
on real-life instances have highlighted that the method is practically viable and returns
good results.
