Abstract
The problem of closure, is one of the great problems in turbulence modelling. This
problem arises from Reynolds averaging of the equations of fluid motion - termed
the Reynolds Averaged Navier-Stokes Equations, or RANS. This results in more un-
knowns than equations - as there is the addition of the Reynolds stress tensor. These
equations are then referred to as ’unclosed’. In order to solve the equations, models
have been created in order to estimate the Reynolds stress tensor - however, these
are far from optimal. Thus, the aim of this work is to use a machine learning (ML)
method in order to predict the Reynolds stress tensor. Both a neural network (TBNN)
and a random forest (TBRF) model are implemented - based on the work in [KD20]
and [LKT16a]. The models are compared to a common turbulence model (k −ω)
which obtained a baseline root mean squared error (RMSE) of 0.1792. The accuracy
of the ML models in relation to the number of input features used is also investigated
for the neural network (5, 17 and 28). The results of the ML models indicate that all
result in a lower error than the baseline modelling method (TBNN 5 features: 0.1033,
TBNN 28 features: 0.0695 , TBRF 17 features: 0.0736) - the greatest accuracy was
obtained with the neural network using 17 input features (RMSE = 0.0661).
Furthermore, in order to use this method to model unsteady flow (time-dependent),
the Reynolds Stress Tensor would need to be predicted at every time-step and fed
back into the solver. Thus, another aim was to create an integrated framework in
which this would be realizable. While this was successful, the iterative predictions
were not. This is likely due to issues relating to the data used in this work - which
was a limiting factor. Instead, propagation of a one-time prediction into the solver
was investigated - which could be used for steady flow cases. It was found that the re-
sults largely agreed with the previous work ([KD20]), and could possibly be improved
further - through optimization of the framework parameters.
Overall, the results of this work show the superiority of a neural network based al-
gorithm for prediction of the Reynolds stress tensor (and thus, turbulence modelling),
compared to traditional techniques. With the changes to the framework outlined in
this thesis, it is believed that this could be modified to produce a novel and accurate
method for modelling unsteady turbulent flows.

CHAPTER 1
Introduction
1.1
Background
1.1.1
Big Data and Fluid Mechanics
Historically, while other areas of research may have been limited by the amount of data
available, fluid mechanics has long dealt with large amounts of data from experiments
and simulations [BNK19]. The question then becomes how to process and analyse
this data. Previously this has largely been done through the use of expertise as well
as algorithms which have been based in experimental observations.
Progress within computational fluid dynamics (CFD) has been driven by advance-
ments in computational hardware capabilities (Moore’s Law), and these advancements
have themselves been pushed by large and complex data sets - such as those gath-
ered in fluid flow experiments. Similarly, although many machine learning techniques
have been around for multiple decades, the recent surge in the use of these techniques
within research and the industry is largely due to innovations in hardware. This has
now come full circle, and fluid dynamics researchers are now fully utilizing the power
of machine learning to analyze the complex and high-dimensional data associated
with the topic.
1.1.2
On the Problem of Closure
The Navier-Stokes equations, which are described in greater detail in subsequent chap-
ters, are a set of partial differential equations which describe fluid motion. Turbulent
flows, which can be observed in most real-life applications, as opposed to laminar
flows, are more irregular and thus diﬀicult to describe. For turbulent flow, each of
the terms of the Navier-Stokes equations, can be decomposed into a mean and fluctu-
ating part via averaging, resulting in the Reynolds Averaged Navier-Stokes Equations
(RANS).
This is where the problem of closure becomes apparent. Through this process we
obtain four equations; however, in addition to the four unknowns given in these equa-
tions, one ends up with with the Reynolds stresses (u′
iu′
j). The Reynolds stress tensor
is symmetric, thus it adds 6 additional unknowns into the mix. A set of equations
which have more unknowns than equations are said to be ’unclosed’ [Pop+00]. Hence,
in order to solve the equations, the Reynolds stress tensor must be determined - and

can be approximated through modelling. The closure problem and a few of these
models are described in detail in chapter 2, while here recent research on closure
models and the use of machine learning in this area will be discussed.
1.1.3
Machine Learning and Closure Models
The advancements made in machine learning in tasks such as natural language pro-
cessing and image analysis can be exploited by fluid mechanics researchers and, along
with expert knowledge, may elucidate new insights into complex problems [BHT20].
As this work is based on the intersection between machine learning and closure models,
it would be prudent to discuss previous work on the subject.
A great deal of research has been performed in recent years combining machine
learning and closure models, illustrated in [BNK19] and [DIX19]. Although RANS
models are popular, it is clear that there are disadvantages to the method, and the
output is often inaccurate [LT15]. Thus, one area of research which has been partic-
ularly active is quantifying or characterizing the uncertainty between a RANS model
in relation to high-fidelity simulations. High-fidelity simulations are extremely com-
putationally expensive, and not often available for the problem at hand. Thus, when
using a sub-optimal solution (RANS), one would want to identify discrepancies, and
what factors contribute to these discrepancies. [LT15] used several machine learning
algorithms, including Random Forests and Support Vector Machines (SVM), in order
to determine regions of high uncertainty within RANS results. Similarly, [WWX17]
used a Random Forest algorithm trained on high-fidelity simulation data in order to
predict the discrepancies in RANS data sets. This would allow for the prediction of
discrepancies in the absence of high-fidelity simulation data - which is often the case.
Another perspective, and the one focused on in this work, could be to use machine
learning algorithms to model the Reynolds stress tensor u′
iu′
j. In [LKT16b], a neural
network architecture was created and implemented in order to predict u′
iu′
j. Using
features extracted from RANS data, the anisotropic Reynolds stress tensor could
be predicted based on learning from high-fidelity simulations (DNS). Building upon
this, [KD20] implemented a Random Forest variant of the neural network, as well
as the original neural network presented by [LKT16b], and found they had similar
performance.
Recent studies combining the problem of closure and machine learning, including
but not limited to those presented above, show that machine learning likely will have
a big role in furthering the knowledge within the area of turbulence closure, as well
as fluid dynamics as a whole.

The aim of this work was to re-create and build upon both the random forest algorithm
and neural network presented in [KD20] and [LKT16b], respectively, for prediction

Using the flow cases obtained from the authors of [KD20], once the model architec-

integrated into the RANS solver, such that predictions can be made recursively.

CHAPTER 3
Machine learning
In this chapter, a brief overview of machine learning algorithms are given. To keep
consistent with the previous chapter the background for the machine learning algo-
rithms will be given from a probabilistic perspective, i.e the unknowns variables will
be treated as random variables with a corresponding probability distribution.
”A computer program is said to learn from experience E with respect to some class
of tasks T,and performance measure P, if its performance at tasks in T, as measured
by P, improves with experience E.” - Thomas Mitchell [Mit97]
The goal of machine learning is then to learn from the data. This could be in
the form of finding patterns or making predictions.
There are different forms of
machine learning depending on the task we want the computer program to learn,
the performance measure used, and the nature of the signal. In this work a we will
consider the form of supervised learning, but first a short description of the other
classes of machine learning is given.
Reinforcement learning is a class of ML that has grown in recent years, due to
the fact that it is a class that is applicable for a wide range of problems [Mur21]. The
main idea behind reinforcement learning is for a system to make a series of decisions
in an environment and achieve a desired state, guided by an occasional reward or
punishment. The system which makes these decisions is termed the ’agent’. Rather
than being given the correct answer, by the use of a training set, the agent must
learn from trial and error. An example of this could be playing the game checkers.
The agent can learn to move pieces to maximize the total reward - e.g. it could be
rewarded based on how few of the opponents pieces are left on the board.
Unsupervised learning, as the name suggests, is a type of machine learning in
which there is no correct answer, or ’supervision’. This is a particularly interesting
class of machine learning, as instead of labelling a whole data set and training the
algorithm on the ’correct answer’, unsupervised learning let’s the data speak for
itself. This is typically used when there is no specific goal in the machine learning
task, rather the goal is to discover inherent patterns and describe the data in some
way. An example of this is clustering, in which, based on a set of features, the data
can be separated into groups of similar observations. This can determine the number
of classes the data set is comprised of.

Forward pass is the term used to describe passing the features of a sample (input

is passed into the first hidden layer, at which z = W x + b is computed. The acti-

how well the model is fitting to the sample, by comparing the predicted output to
the actual output. The loss function used depends on the application. We can collate
the loss for each observation into a cost function (J), describing the error over the
entire training set. The cost function is used to update the parameters of the network
(the weight (W) and bias (b) vectors) through a method termed ’back-propagation’,

3.1.2
Back-Propagation
During the first forward pass, the weights are randomly initialised (and the bias is set
to 0). The model ’learns’ through the updating of the weights and biases in relation to
the cost function. By applying gradient descent, the model computes the derivative
of the weights and biases with respect to the cost function, J. The error can then be
transferred back to the output layers (most recent layer) corresponding weight and
bias - however, we want to transfer this error back through all layers and all nodes,
to update all the weight and bias parameters. This can be ’backpropagated’ through
partial differentiation and the chain rule - i.e. the error is carried from the output
back to the first hidden layer. The specific calculus involved in this process is not
given here as it is outside the scope of this project. With gradient descent, iterating
through the training data a number of times should result in a convergence of the
model - i.e. the error will not decrease as the iteration number increases.

fact that each step in gradient descent is likely not entirely in the direction of the
minimum, and rather gradient descent proceeds in an oscillatory or ’zig-zag’ fashion,
towards the minimum. This has resulted in a number of optimization algorithms
which typically perform better and result in a more time-eﬀicient training process.
One of the most popular of these algorithms was implemented in this work, and is
described in greater detail in section 4.3.2.
3.1.3
Regularization
The neural network illustrated in figure 3.2 is very simple - typically a higher number
of hidden layers would be included, with a varying number of nodes in each layer. If a
significant number of hidden layers are used, ’deep learning’ or ’deep neural network’
is the term applied to the method. Using a large number of layers allows for more
complex functions to describe the mapping to the output - a large number of layers
could technically lead to over-fitting, but this can be avoided through regularization
techniques. Over-fitting is the term used to describe a situation in which there is
a large discrepancy in the performance of the model on training data versus unseen
(test) data. Essentially, the model is fitting too well to the training data. In this
case, the model is said to have high variance. The opposite, a model which is not
complex enough, is said to have high bias. Hence, when building a model, one must
take into consideration the variance/bias trade-off.
Thus, as a higher number of
layers will likely lead to high variance, regularization techniques can be implemented
during training to reduce the complexity of the fit. These, in lay-man terms, make it
harder for the model to fit perfectly to the training data. In this work, two popular
regularization techniques were implemented, and are described in section 4.3.2.

3.2.3
Regularization
By utilizing a greedy, ’top-down’ approach to fitting the model, this model is very
prone to overfitting. Essentially, without regularization techniques, the tree could
continue splitting nodes until it reaches 0 error on the training data ([Mur21]). How-
ever, as mentioned in relation to neural networks, the aim is to produce a model
which generalises well to unseen data. There are two approaches to regularizing a
decision tree, although the main idea is the same as stated previously - to reduce the
complexity of the fit.
One method relies on stopping the fitting process based on some measure - for ex-
ample the tree depth. Another is to allow the model to fit to the data completely, and
then ’prune’ the decision tree. With ’pruning’, certain splits are merged backwards
to the original node.
3.2.4

Decision trees have a number of attractive characteristics. They are easy to train and

trees differ from one another. Thus, to ensure variation between the decision trees,

on a random subset of the original data set (see Figure 3.5). The subsets are chosen
by sampling the original data set with replacement - using replacement means that
observations can appear more than once. To further ensure that the decision trees
are not correlated, only a subset of features are considered at each node. By using
multiple uncorrelated decision trees, the ensemble method typically performs better
than a single classifier ([HSM19]).

5.3
Propagating the TBNN Predictions
Based on the results presented in the two previous sections of this chapter, when
moving forward with the framework, only the TBNN with 17 features was considered.
While the aim of this work was to develop a corrective model for unsteady turbulent
flow, when integrating the neural network into the OpenFoam solver, the solver gave
highly inaccurate predictions. Reasons as to this are discussed in Section 6.3. There-
fore, it was decided to investigate the propagation of the neural network prediction
for the steady flow case, using the framework given in Section 4.1.
In order to create a corrective turbulence model, the bij tensor obtained from the
neural network was propagated into the flow field. This entails that at convergence
of the k −ω model, the anisotropic tensor is replaced with the predicted bij. The
k −ω model is likely to be ’thrown-off’ by the addition of a new parameter, as the
convergence is based on an different bij tensor. Thus, the model continues to iterate
until a new convergence is obtained - i.e. when the threshold for residuals is met.
This required 739 iterations in the OpenFoam solver.
To visualise the result of propagating the anisotropic tensor, the in-plane velocity
magnitude is plotted in Figure 5.3(a). Here propagating the TBNN bij is compared to
propagating the bij tensor from the actual DNS data, as well as the in-plane velocity
magnitude of the DNS data itself. Note that the k −ω model is not shown in this
plot as it cannot predict secondary flows. To show the effect of the blending factor,
γ, three additional values were tested. This is illustrated in Figure 5.3(b).
Furthermore, the resulting stream-wise velocity and in-plane vector (square-duct
case) for the DNS data, the propagated DNS bij, and the propagated TBNN bij (γ
= 0.6) are plotted in Figure 5.4.

6.1
Comments on Data Provided
In this work, five flow cases were considered.
These were provided to us by the
authors of [KD20]. A common approach to training and testing a machine learning
algorithm is to split the data into a training set, development set and test set, at a
ratio of 80%:10%:10%. This ensures adequate sample size when evaluating the model;
however, this requires a suﬀiciently large data set. Unfortunately, due to the fact that
only five flow cases were available to us, the data set had to be split such that it was
tested on one of the flow cases. This limits the amount of data that the model can
be trained on. Typically, the performance of a machine learning algorithm increases
with the amount of data that is available for training. This is because it gives a better
estimation of the distribution of the population. Furthermore, by testing on only one
flow case - the results are only applicable to that case, meaning that we do not know
if the model is able to generalize to other flow cases.
In terms of the quality of the data, one of the provided flow cases was corrupted
by noise. Thus, making the data unusable and further limiting our data set.
6.2
Predictions using the TBNN and TBRF
For the prediction of the anisotropic tensor using the Tensor Basis Neural Network
(TBNN), it was found that a model using 17 of the 28 available features resulted in
the lowest root mean squared error (RMSE), at 0.0661. This is a significant reduction
from the baseline k −ω model, which obtained an RMSE of 0.1792. The 5 feature
TBNN obtained a relatively meager reduction in error, with an RMSE of 0.1033.
While the use of 5 or 17 features can be rationalized by the findings in [KD20], it
would be prudent to compare the results with a model using all 28 features. In the
results in can be seen that this performed similarly to the 17 feature TBNN, with
a slight increase in the RMSE. This may validate the use of fewer features in the
aforementioned work; however, it may also be an indication that the neural network
==PAGE:64==
50

needed a higher number of training iterations, in order to converge, when using a
larger input vector.
We may compare these results to the output of the Tensor Basis Random Forest
(TBRF). Using the same 17 features, the TBRF was also able to significantly out-
perform the baseline model. The raw output had an RMSE of 0.815; however, due
to the piece-wise characteristic of the random forest as a regressor, it is reasonable
to use a Gaussian smoothing filter on the predictions, and base our evaluation upon
this output. As expected, after smoothing, the RMSE was reduced further, to 0.0736.
Thus, this shows that the random forest method is comparable to the neural network.
However, neural networks typically outperform other machine learning models when
there is a large amount of data available for training. This is exemplified by the rise
of the use of deep neural networks for a wide range of applications in the last decade
or so. Thus, as the results of this work show that the TBNN is superior, we may also
hypothesize that this would be the case when utilizing a larger data set.
Typically, when evaluating a machine learning model, one would analyze the ob-
servations upon which the model obtained a low accuracy. This allows for a more
in-depth evaluation of the model in terms of generalization, such as assessing if the
model is unable to generalize to certain types of observations and possible reasons for
this. However, due to the lack of data in this project it was not possible to do this
analysis. One possibility would be to create a number of different models in which
each model would be trained and tested on a different subset of the data. However,
this would be time consuming, and a sub-par solution compared to simply acquiring
more data.
6.3
Propagation
The original aim of this work was to extend the work done in [KD20] and [LKT16a] by
integrating the neural network into the OpenFoam solver - predicting the anisotropic
stress tensor at each iteration and propagating this prediction back into OpenFoam.
The main concept being that this could be used in unsteady flow cases. However,
although the integration of the neural network into Openfoam was successful, it was
found that the predictions were highly inaccurate. Essentially the program failed.
The most likely reason for this is that the neural network is not trained to make a
prediction based on data which is highly similar to DNS data - the neural network is
trained on mapping RANS data to DNS data. Thus, when, after a few iterations, the
resulting input to the neural network is more similar to DNS data than the original
k −ω models output, it is unable to map this to a suitable prediction. In other words,
this is an issue related to a difference in the distribution of the training data and the
test data during propagation.
As I was unable to produce any relevant results for the iterative propagation, the
work was focused instead on re-creating the propagation experiment from [KD20] for
steady flows. However, in the cited work, this was done using the predictions from
the TBRF, rather than the TBNN. From the resulting in-plane velocity magnitude

To fully evaluate the work done in this thesis, we may compare the results obtained
here to those in the papers upon which the thesis was based. Firstly, we may compare
the resulting accuracy obtained in the prediction of the anisotropic stress tensor using
the TBNN and TBRF. In [LKT16a], an RMSE of 0.14 was obtained for the TBNN
model.
While a significantly lower RMSE is obtained in this work, it cannot be
certain that these two evaluations are directly comparable. This is due to the fact
that the RMSE in the cited paper is based solely on the 6 distinctive bij components,
where as here all 9 are considered. Is can be assumed that basing the RMSE on
only 6 components would result in a somewhat different result - although one cannot
comment on the extent of the difference.
In Table 6.1, the RMSE obtained for each of the models explored in this work are

seen that the 17 feature TBNN in this work, slightly outperforms that of the cited
paper. However, the performance of the 5 feature TBNN and the 17 feature TBRF
is reduced in this work. However, the aim of this work was to reproduce the TBRF
rather than improve upon it - and thus, little was done in terms of optimization of the
random forest here. Furthermore, the TBRF was not regularized and likely over-fit
to the training data. Optimization efforts were focused rather on the neural network
model.

6.5
Future Work
Future work should be focused on modifying the framework such that it is suitable
for unsteady flows. As discussed above, in order to iteratively predict and propagate
the anisotropic stress tensor, we can expect that the neural network would have to
be trained on a mix of RANS and high-fidelity/DNS data - such that the distribution
of the training data and test data are similar. This coincides with the need for more
data in order to create a robust and generalized model.
Furthermore, it would be prudent to investigate improvements that can be made
on the TBNN. Although some parameters of the neural network were optimized in
this work, it would be interesting to further optimize parameters such as the learning
rate and the number of hidden layers. A deeper neural network may result in more
favourable results; however, as stated previously this too relies on having a large
enough data set.
Moreover, although the full 28 feature TBNN was presented in
this work, it was not robustly evaluated. This model would likely require a longer
training period, and as the neural network architecture was optimized for 17 input
features, the aforementioned note on optimization would be even more applicable to
this new model. Other parameters in the propagation framework must also likely
be optimized in order to obtain a higher performance. This was exemplified in the
results - i.e. optimizing the blending factor would likely have a large effect on the
accuracy, as the results showed quite a significant difference between using a blending
factor of 0.8 versus 0.6.
It is expected that with these changes, the new modified framework would be
able to accurately predict the anisotropic stress tensor for unsteady flows. However,
we may also explore other methods for achieving the overall aim of this work. An

used graph networks in order to simulate the complex physics of a moving fluid, by

stated in this work, as there is a possibility that by viewing each mesh cell as a graph,
the anisotropic stress tensor could be predicted based on the interaction with nearby
cells. Furthermore, graph networks are Galilean invariant by design.

CHAPTER 7
Conclusion
The main aim of this work was to create a novel integrated neural network and
OpenFoam framework in which the anisotropic stress tensor (bij) could be predicted
for unsteady flows. In order to investigate this, a specific plan was set - here the
measure to which this plan was achieved is presented.
Firstly, the papers upon which the thesis aim was based needed to be studied and
understood in detail. While researching other studies which combine the problem of
machine learning the problem of closure, it was found that this is a relatively new
area of research. There were not a significant number of papers published on the
topic, and the two presented in this work were those which are most relevant for the
aim of the thesis. Understanding the work done in [LKT16a] and [KD20] was crucial
to implementing the methods and building a basis for the aim of this work.
In terms of the attempt to re-create the frameworks in the aforementioned papers,
we were able to successfully implement both the Tensor Basis Neural Network (TBNN)
and Tensor Basis Random Forest (TBRF). In total, four models were created, and all
of these significantly outperformed the baseline k −ω model. Due to time constraints,
only one of these models (TBNN 17 features) was optimized, and this model obtained
a slightly lower RMSE compared to [KD20]. However, as discussed previously, further
optimization could be implemented.
The main bulk of the practical work done in this thesis was based on integrating
the neural network architecture into the OpenFoam solver. While the practical aspect
of this integration was successful, this work failed to obtain relevant results. However,
although the main aim of this thesis was unattainable at this time, the steps which
should be taken in order to realize this are outlined in Section 6.3. Furthermore, in this
work the propagation of the predicted bij tensor using a neural network architecture
has been shown to be successful for steady flows - while the previously cited paper
used the TBRF model.
In conclusion, this work was mainly limited by the amount of data available.
With a larger data set and the other improvements mentioned in this work, it is likely
possible to solve the problem of closure for unsteady flows using this framework. Thus,
this shows the advantages and potential of using a data-driven, machine learning
framework, for not only the problem outlined here, but also in consideration in a
wider-scope - for fluid dynamics in general.
