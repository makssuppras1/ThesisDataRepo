ABSTRACT
Recent advancements of data-driven algorithms, such as deep learning, opened a new
wave of research for applications in power system security assessment. The development
of such methods require large amount of data, but due to the highly reliable operation of
power systems, historical data is not suﬃcient. Therefore, the data has to be generated,
however, it was shown that creating such data sets is a challenging problem and becomes
computationally intractable, even in case of low dimensional problems [1]. This M.Sc.
thesis investigates the problem of data set generation for data-driven security assessment
of power systems.
A literature overview of relevant theoretical background and state-of-the-art data set
generation methods is given in Chapter 2.
This work proposes an eﬃcient data set generation methodology based on the works of
[1] and [2].
The proposed methodology consists of the following steps.
First bound-
tightening algorithms are applied to the given case. Then, using the convex relaxation of
the AC-OPF problem, infeasibility certiﬁcates (based on separating hyperplanes) are used
to signiﬁcantly reduce the event space. Following that, highly parallelizable directed walk
algorithms are used to characterize the dynamic stability boundary of the power system.
Finally, using the well-characterized description of the stability boundary, a sampling
method is applied to draw a large number of points from the entire stability region and
create a balanced data set. The methodology is detailed in Chapter 3.
The proposed methodology is implemented as a generic data set creation framework, Power
System Data set Creation (PSDC). Using PSDC, several IEEE benchmark test systems
are thoroughly analyzed and the resulting data sets are made publicly available at Zenodo.
The software implementation details and case studies are presented in Chapter 4.
The analysis of several case studies showed that the implemented methodology is capable
of generating large scale and high quality data sets.
Furthermore, the implemented
methodology provides substantial computational speedup compared to previous works
such as [1]. Additionally, the methodology is applied to problems with up to |x| = 31
control variables, whereas in [1] only cases up to 17 control variables were investigated.
The analysis of the resulting data sets showed that PSDC creates high-quality data sets.
The data sets were subjected to thorough analysis including evaluation using several
classiﬁers. Despite the simplicity of the models, the classiﬁers were able to reach high
accuracy in case of each case study.
Hence, the resulting data sets are suitable for
development of data-driven algorithms for power system security assessment.
This work required the development of various software tools and the creation of data
sets that are meant to be used by future researchers. All software tools and data sets
that were developed throughout this work are published as open-source projects and are
publicly available.

INTRODUCTION
Maintaining stable and secure power system operation is one of the most essential problems
in electrical power systems. The size of interconnected large power systems grew in the
past decades, while at the same time power system stability and operational security
became more important as our society highly depends on the continuous stable and secure
operation of the power grid. Furthermore, the high-scale integration of renewable energy
sources (RES) presents further challenges to maintaining the highly reliable operation of
the electrical power system [5].
As power system instability can have a variety of causes, a multitude of simulations
and assessments are required to ensure safe operation. As the share of RES is rapidly
increasing in the power system, the uncertainty of power generation is also growing. Such
uncertainties strongly increase the number of possible operating scenarios for security
assessment. It is computationally impossible to assess every possible operating point of the
power system, thus novel, computationally eﬃcient methods are needed [6]. Furthermore,
ensuring economic operation and optimal power dispatch also becomes a diﬃcult problem
when uncertainty of generation is included in the optimization models [7].
In recent years machine learning became a common tool in many ﬁelds of science.
Data-driven algorithms (such as decision trees, neural networks) became well-known
mathematical models that exhibit unprecedented performances [8].
However, despite
the fact that in today’s modern power grid there is an abundance of data with the rise
of advanced measurement devices such as Phasor Measurement Units (PMUs), applied
machine learning methods remain scarce in power systems.
Several research projects attempted to mitigate the computational challenge of security
assessments under uncertainty e.g.: prediction of transient instability [9]. In most of these
novel methods it is common that they all build on machine learning approaches. Due
to recent advancements both in computational power and eﬃciency of machine learning
algorithms advancements are fueling an interest in applications regarding electrical power
systems.
A fundamental obstacle for applying machine learning in power systems is the lack of
high quality, well-balanced data set, where both stable and unstable operational points
are equally represented. Due to the highly stable operation of the power system, historical
data per se is not adequate for training. It is therefore necessary to create a data set using
simulations.
A straight forward data set generation method, is to check all possible operating points of
a given case with brute-force computation. However, due to the high dimensionality of the
problem, the computational complexity of such method explodes when security criteria
such as N-1 criterion are also considered. For the IEEE 14 bus system with 5 generators

the number of possible operating points is |ψ| ≈2.5·106 for a single load proﬁle [1]. Hence,
such brute-force data set generation methods are computationally intractable.
Therefore, computationally eﬃcient data set creation methods are necessary to enable the
development of data driven methods for power systems. There are various approaches to
generate well-balanced data sets, however due to the large imbalance between the size of
feasible and infeasible operating regions, not all methods proved to be equally successful.
Previous work
The authors in [10] present a data set creation method that relies on importance sampling
of the feasible operating region. Yet, due to the large imbalance between feasible and
infeasible points, this method can be iterative and time consuming. Also the small size
of the data sets produced by this method (|ψ| ≈103−4) can be challenging to use for
advanced data driven approaches such as neural networks.
The authors in [11] propose a method to classify safe and unsafe operating point using on
deep autoencoders. The results of classiﬁers look promising, however the authors highlight
multiple times the fact that the imbalance of the training data set is the major factor that
reduces the performance of their methodology.
The data set creation methodology of
[11] uses Monte Carlo simulations to enrich the data set, however they do not manage to
suﬃciently balance the data set, compared to the original(2.417 %), their enriched data
set has only 2.95 % feasible-to-infeasible set point ratio. In this case the size of the data
set |ψ| ≈105 is suitable for training data driven algorithms, however the substantial over
representation of infeasible points introduces an inherent bias to data driven methods.
The authors of [1] and [2] introduce a methodology that outperforms other existing data
set generation methods. These methods have a superior computational eﬃciency as they
can discard large regions of the high-dimensional state space using infeasibility certiﬁcates
based on the quadratic convex relaxation of the AC-OPF problem.
Furthermore, the
resulting data sets are more suitable for data driven applications as they maintain a close
to 50 % −50 % ratio of feasible-infeasible points and the size of the generated data sets is
also more suitable for applications with advanced machine learning models.
The authors in [12], using the data set generation method of [1], show a successful
application of data driven power system security assessment as their neural network (NN)
based small-signal stability assessment reduced the computational time more than 200
times compared to traditional methods while a close to 99 % accuracy was maintained.
Contributions of this work
The goal of this work is to generalize the methodology of [1] and [2] to arbitrary power
systems and enable the development of data-driven power system security methods.
To allow the creation of data sets the generalized methodology has to be available to
all researchers, thus an open source tool Power System Data set Creation (PSDC) is
developed.
Besides, a number of reference data sets are published along with PSDC,
generated from well-known power system test cases.

BACKGROUND
The goal of this section is to introduce the reader to the relevant deﬁnitions and necessary
theoretical background of the thesis. Furthermore a literature overview of state of the art

2.1
Theoretical overview
This section aims to provide the reader with suﬃcient theoretical background that is
necessary to introduce the data set creation methodology in Chapter 3. It is assumed
that the reader has a basic understanding of electrical power systems. This section will
start from revisiting the basic principles of load ﬂow calculation in Subsection 2.1.1.
Following that, in Subsection 2.1.2, power system stability is brieﬂy reviewed with the
aim of providing suﬃcient background for small-signal stability analysis. Finally, the AC-
OPF and Quadratic Constraint (QC) relaxation of the AC-OPF problem is introduced in
Subsection 2.1.4. The concepts introduced in this section construct the basis of eﬃcient
data set creation methods introduced in Section 2.2 and the methodology detailed in
Chapter 3.
2.1.1
Power Flow problem
Power ﬂow (or load-ﬂow) analysis of an electrical transmission network is used as one
of the basic analytical tools that enables the calculation of bus voltages and the ﬂow of
complex power.
The goal of power ﬂow calculations is that given a deﬁned load proﬁle (power system
generator dispatch and load dispatch) the voltages and voltage angles shall be found at
each node of the power system.
That means, after carrying out the calculations, the
amount of power ﬂowing through branches is known around the whole system [14]. Power
ﬂow calculations use a balanced three phase representation and are applicable to the
steady state conditions of the power system.
The power ﬂow equations are well described in [14, 15, 16]. A partial formulation of the
power ﬂow problem is given in the following, however it is important to note that the power
ﬂow problem has many diﬀerent modiﬁcations and solution methods, e.g., basic Newton-
Raphson power ﬂow, Guass-Seidel method, dishonest-Newton, very dishonest Newton,
fast-decoupled load ﬂow. In this work, if otherwise stated, the Newton-Raphson method
is used, as described in [13, 16], to obtain a power ﬂow solution.

2. Background
Steady state model of the power system
A power system can be modeled as a directed graph with N nodes (vertices) and L edges.
The set of N nodes represent the buses. The set of L edges represent the connecting
branches of the power system. G ⊂N represents the generators (PV buses), and D ⊂N
represents the load buses (PQ buses) of the power system. Generators and loads can be
thought of as sources and sinks of the graph [16].
Any i ∈N bus of the system is characterized by four variables xi = (pi, qi, |vi|, θi) which
are respectively active power, reactive power, voltage magnitude and voltage angle. The
|N| bus power system’s buses are divided into the following 3 categories:
• Slack bus - There is only one slack bus in the system. The slack bus is the reference
bus where v and θ are known.
In this work the slack bus is denoted by i = 1,
consequently x1 = (p1, q1, |v1|, θ1).
v1,θ1 are also called as reference values and
commonly set to v1 = 1.0 [pu], θ1 = 0◦.
• PQ bus - PQ buses are also know as load buses. p and q are known at these buses.
• PV bus - voltage controlled buses, where p and v are known.
The power system is considered to be in balanced three phase, steady state operation.

2.1.2
Dynamic model of Power system
In this section a short overview of dynamic power system simulations is given. As this
work focuses on small-signal stability of the power system it is necessary to establish a
basic understanding of the mathematical and physical models behind the simulations.
In order to study the dynamic behavior of the power system additional modeling
parameters are required compared to steady state studies such a load ﬂow analysis.
Furthermore, the purpose of modeling also deﬁnes certain properties of the simulation,
e.g., the time step and time span of the simulation depends on the analyzed phenomenon.
When it comes to building a model of the power system, all dynamic components of
the power system has to be modeled adequately.
Meaning that the various dynamic
components such as automatic voltage regulators (AVRs), power system stabilizers (PSS),
generator units, have to be modeled with respect to the simulation’s purpose. Generating
units (synchronous machines) are components, that have various forms, e.g.
classical
model, 4th order model or 6th order model, and the appropriate models have to be chosen
to adequately represent the behavior of the components.
In dynamic power system simulations a diﬀerential and algebraic equation system (DAE)
has to be solved. In this section the most important modeling aspects are summarized in
order to establish an understanding of building such DAE systems that represent power
systems.
General considerations
Dynamic simulations are not time indiﬀerent (in contrary to steady state simulations)
and the initial condition of the system at t0 has to be in normal steady-state, balanced
operation. Initialization of a dynamic simulation means that the initial values of variables
are computed. The initial conditions of the power system are computed by solving load-
ﬂow equations of the system and then calculating the algebraic and state variables of
the DAE. Several methods have been proposed to solve such equation systems, [15] gives
a good overview of the diﬀerent numeric methods, the author names two categories -
Simultaneous-implicit (SI) method and Partitioned-explicit (PE) method. While building
the DAE and solving it using the SI method is more straightforward, the PE method
handles stiﬀequations better, thus the PE method is used in most commercial applications
[15].

Building on the basics of steady state and dynamic power system simulations given in

stability. First, a general overview of power system stability studies is given. Following
that N-1 and small-signal stability is discussed in detail, as the present work focuses on
these two subproblems.
Power system stability is the ability of the power system to remain within acceptable
operating limits after subjected to a disturbance [3, 16].

N-1 security criterion
N-1 stability of the power system dictates that in case of a given operational point, if
a component spontaneously falls out of operation - e.g.
due to a fault - the system
has to remain in a stable state. This deterministic approach means that not only the
initial intact operating point has to be analyzed in terms of various stability criteria (e.g.,
voltage stability, transient stability) but also all the N-1 scenarios are subject to the same
analysis. Hence the name, that the analysis examines an N-component system after loss
of any (major) component [3]. However, due to the fact that N-1 criterion handles every
component failure similarly, without considering severity of loss or probability of loss
makes it inadequate in many real-life cases and economically more beneﬁcial probabilistic
approaches are preferred e.g. the probabilistic approach detailed in [19, 20].
This work considers N-1 criterion as ensuring that the static load ﬂow converges without
violating any security criteria, e.g., (2.4).
The dynamic eﬀects of losing a component
under steady state operation is not considered under the N-1 criterion.
In some cases an N-k analysis is necessary. That refers to the analysis of the power system
when multiple contingencies are applied at the same time. Contingency denotes to an
unexpected outage of a single (or multiple) power system component(s), e.g., generating
unit, transmission line, transformer [21].

Small-signal stability
Eigenvalue analysis of the power system has two main purposes - small-signal stability
and dynamic voltage stability (the topic of voltage collapse) analysis [3, 13, 16].
The eigenvalue analysis of the power system is the eigenvalue analysis of state matrix
A given by (2.8). A power system is small-signal stable (small-disturbance rotor angle
stable) if the power system is capable to maintain synchronism under small disturbances.
A disturbance is considered small if the system equations (2.7) can be linearized with
suﬃcient accuracy around the operating point [3, 16].
The main principle of such analysis is that the dynamic model of the power system,
described by the set of DAE system is linearized around a given operation point (initial
condition). Following the linearization of the system a small ∆x disturbance is applied to
the system and its stability is determined based on Lyapunov’s stability criteria.

used: In case of small-signal stability (SSS) the eigenvalues of A (roots of the characteristic
equation of the system’s ﬁrst approximation (2.9)) must be on the left half plane of the
complex plane and the smallest damping ratio of the system must be above a critical ζmin
value [16]. The damping ratio is given by (2.11). An illustration of eigenvalue analysis

This section gives a brief overview of the AC Optimal Power Flow (AC-OPF) problem.
The AC-OPF problem was ﬁrst introduced in 1962 [23]. The AC-OPF is a non-linear,

(2.13a). Due to the non-convex nature of the problem (that arises from the power ﬂow

reliably and eﬃciently for arbitrary power systems causes a challenge until this day.
As mentioned in Subsection 2.1.1 the power system has to comply with physical constraints
given by (2.4).

solution, that minimizes the objective function while respecting the network constraints.

The quadratic convex (QC) relaxation

cannot be ensured. By using convex relaxations of the AC-OPF problem eﬃcient solvers
can be used to reach a solution and the solution is guaranteed to be a lower bound of
the original problem’s objective value.
There are several diﬀerent convex relaxations,
however the QC relaxation represents a good trade-oﬀbetween computational complexity
and tightness of the relaxation. Therefore, this work builds on the QC relaxation, hence
this relaxation method is explained in detail in the following.

This section describes the state-of-the-art data set creation methods. The main focus

As it was seen in Chapter 1 there are several approaches that are aiming to solve the

convex relaxation of the AC-OPF problem.

infeasibility certiﬁcates obtained from the relaxation of the AC-OPF problem to discard

2.2.1
Eﬃcient data set generation for power systems

4. Perform N-k analysis of the sampled points.

3.1
Bound tightening methods
The ﬁrst step of the data set creation methodology is to run two bound tightening
algorithms on the power system case. In Subsection 3.1.1, the ﬁrst, a computationally
cheap bound tightening method is introduced, and in Subsection 3.1.2, the second bound
tightening method is detailed.
Bound tightening is a general approach to strengthen the relaxation of nonlinear
optimization problems.
In this work, tighter optimization variable bounds also mean
reducing the high dimensional state space, hence improving the eﬃciency of using
separating hyperplanes as infeasibility certiﬁcates.
Bound tightening is used as a ﬁrst step of the methodology to enhance the tightness of the
quadratic convex relaxation in the following step (see Section 3.2). By enforcing tighter
bounds of the optimization variables of Model 2 (2.15b)–(2.15f), the distance between
upper and lower bounds can be signiﬁcantly reduced, which leads to a smaller optimality
gap and an overall better solution of the QC relaxation [27]. Furthermore, having tighter
bounds around the decision variables (2.20) reduces the initial infeasible space.
The ﬁrst bound tightening method is computationally eﬃcient in terms of memory usage
and in terms of wall-clock time. Furthermore, the algorithm can be parallelized [27]. The
second, optimization based algorithm is computationally more expensive both in terms
of memory usage and in terms of wall-clock time as it relies on solving an optimization
problem.
The procedure of applying a ﬁrst computationally cheap and secondly an OPF based,
computationally more expensive bound tightening method is proposed by [2].

3.4.1
Problem deﬁnition
The term directed walk is not related to the conventional directed walks known from graph
theory. This work borrows the terminology from [1] and uses the directed walk term as
a collective name of the algorithms described in the following. A directed walk algorithm
has the following parts: initialization, gradient descent part, and the HIC sampling part.
In the gradient descent and HIC sampling parts, the following sub-processes may occur:
step size calculation, step direction calculation, and HIC sampling.
A directed walk is an algorithm that iteratively converges from an initial point towards a
set point with speciﬁc desired qualiﬁcations, such as a certain damping ratio. This point is
said to be part of the HIC region. After the HIC region is reached, the algorithm collects
a large number of samples by moving along the HIC region [1].
The ﬁrst half of a directed walk algorithm is quite similar to the well-known gradient
descent method, where the goal of the algorithm is to ﬁnd the point that (locally)
minimizes a particular function. However, directed walks also contain a HIC sampling

part where a large number of samples are drawn around the set point found at the gradient
descent part.
In the following, directed walk algorithms are used to ﬁnd power system set points that
have a damping ratio close to a predeﬁned ζmin value, and to draw a large number of
samples close to the ζmin boundary. The gradient descent part of the algorithm minimizes
the diﬀerence between the current step’s damping ratio and the target ζmin.

Detailed overview
The brute-force directed walk algorithm (Algorithm 1) starts from OP1 initial set point.
From this point, the distance from the HIC region dk is calculated. In Algorithm 1 K is
the maximum iteration and µ is the HIC limit, the minimum distance from ζ.
After initialization, Algorithm 1 starts the gradient descent loop (Algorithm 1 line 3-10) if
the damping ratio of the initial set point is out of the HIC region, else it starts at the HIC
loop (Algorithm 1 line 11-20). In both loops, the following actions take place: calculate the
next step size (αk+1) based on the current distance (dk) from the HIC region; Calculate
the next step direction (∇k+1) by selecting the direction of the steepest decrease of the
distance from the HIC region. After that, the algorithm steps to the new set point deﬁned
by the current set point (OPk), the newly calculated step size (αk+1) and step direction
(∇k+1). From OPk+1 new set point, the distance and damping ratio is recalculated. If this
new point’s damping ratio is inside the HIC region, the algorithm proceeds to the HIC
loop; else it repeats this process until the maximum number of iterations (k) is reached,
or the set point’s damping ratio is inside the HIC region.
The HIC loop starts with adding the current set point to the output points Ωj. Following
that, a large number of samples are computed around the set point and added to the
output (more details on the HIC sampling are given in Subsection 3.4.4). After that, a
similar process takes place as before, the new set point and the current distance from the
target damping ratio are calculated. If the newly computed set point is already part of
Ωj, the algorithm halts and returns the current contents of Ωj.

HIC sampling
The HIC sampling, a common step of the brute-force and greedy algorithms, is discussed
in this subsection.
The HIC sampling step is responsible for drawing a large number of samples around the
stability margin. At the end of the directed walk algorithm, these points construct Ωj
subset of set points that describe the Ωsecurity boundary as given by (3.10). As it was
seen in [1] and [2] a detailed characterization of the HIC region is essential, in order to
achieve suﬃcient performance of data-driven methods.
In the following, two diﬀerent
approaches of HIC sampling are described.
Axial sampling
The axial HIC sampling method is described in [1]. In the HIC sampling process, the
input is a set point that is suﬃciently close to the ζ critical damping ratio. The axial
sampling strategy takes samples that change only along one dimension at a time. The λ
discretization factor deﬁnes the change applied to each possible direction. The number of
steps taken into each possible direction is deﬁned by an R radius hyperparameter, which
is set to 1 by default.

Random sampling
The random HIC sampling method is one of the contributions of this work. Similarly to
the axial HIC sampling method, the input point is a set point that is suﬃciently close

generated around the input set point. After that, the samples are pruned by enforcing
the minimal Euclidean-distance between these points. The minimum distance is deﬁned
by α smallest discrete step size.

4.1.1
About dynamic power system simulation tools
In recent years the de facto data ﬁle format of static and optimal power ﬂow related
research is the MATPOWER [36] case format.
One of the main reasons is that the
MATPOWER data format is relatively easy to read, write and at the same time, it
is still ﬂexible and can be easily extended.
Another reason why MATPOWER’s data
format became popular is the fact that MATPOWER itself became one of the most
popular research tools as it is free and openly accessible. Another excellent example of
the MATPOWER case format’s acceptance is the benchmark library of optimal power
ﬂow cases Power Grid Lib - Optimal Power Flow [26].
The built-in MATPOWER
test cases and OPF benchmark library have paved the way towards more transparent
and reproducible research results. Having said that, currently developed power system
simulation tools build on or compatible with the MATPOWER data format, e.g.,
PowerModels.jl [31], PyPSA [37], PSAT [13].
In the case of dynamic power system studies, there is no widely accepted or commonly
used data format. There are multitudes of dynamic power system simulation tools. Some
of the well known commercial simulation tools are Siemens PSS/E [38], Neplan [39],
EUROSTAG [40], DIgSILENT PowerFactory [41], DOME [42], PowerWorld Simulator/DS
[43], GE PSLF [44]. There are also several open-source software tools that are capable of
dynamic analysis, e.g., PSAT [13], MatDyn [45], PowerDynamics.jl [46], PowerSystems.jl
[47], LITS.jl [48]. The purpose of the former lengthy lists of commercial and open-source
dynamic simulation tools is to show the chaotic and tumultuous situation of dynamic
power system analysis software. Almost all of the formerly listed tools implement their

reference in case of dynamic power system studies. Besides that these documents detail
various power system test cases, they also analyze the test cases with multiple tools used
by academia and industry. By comparing the results of the various dynamic simulation
and analysis tools they attempt to give reference values when it comes to dynamic power
system simulations. The benchmark test systems are publicly available 2 3 . Nevertheless,
due to the vast number of dynamic simulation tools, the benchmark power system cases
are provided in several case formats.
Another notable eﬀort to establish a commonly acceptable and high-quality dynamic
power system test cases is the Electric Grid Test Case Repository [51]. This collection of
static, OPF, and dynamic power system models clearly shows an increasing push towards
well-documented benchmark power system test cases. As seen before, this repository also
provides the test systems in 3–4 various case formats.
The vast number of dynamic data formats hinders the research community and creates
an obstacle in the industry. As the share of RES and the need for a more economic grid
operation continually increase, the need for data exchange between Transmission System
Operators (TSO) also increases. Consequently, the Common Information Model (CIM)
was established by the European Network of Transmission System Operators (ENTSO-

International Electrotechnical Commission (IEC) with close cooperation with both TSOs

4.2
Simulation environment
In this subsection the simulation environment, the implementation of the methodology
discussed in Chapter 3 is given. The main purpose of this subsection is to explain software
design choices that lead to maintainable, ﬂexible, modular and easy-to-use software
framework. Also, a brief overview of the software components is introduced.
To ensure maintainability and modularity the core simulation tool, that implements
general power system analysis tools, is implemented as a standalone MATLAB toolbox.
This tool is an object oriented re-implementation of PSAT. PSAT was not suitable to
use due to the excessive usage of global variables that leads to high-jacking the work-
space [13]. The object oriented re-implementation of PSAT enables eﬃcient and parallel
simulation runs, while preserving the mathematical models of the original PSAT thus
ensuring accurate simulation results. The developed tool, Object Oriented Power System
Analysis Toolbox (OOPSAT) is available as an open-source simulation tool4.
The implementation of the data set creation algorithm is PSDC. PSDC maintains a high
code quality to ensure re-usability and modularity of functions. These principles are often
lost, especially in academia, due to the high-paced research environment. Moreover, PSDC
includes detailed documentation of each function. Leading to an overall easy-to-use and
maintainable framework. This comes with the price of excess development time of features
of the framework. However, this projects follows the principle that the code has to be
well-documented and tested on the expense of additional features. It has been shown that
these general principles largely enhance the usability of open-source code [54].
PSDC is an open-source tool, therefore it builds on only open-source software. The initial
bound tightening algorithm detailed in Subsection 3.1.1 is used as is provided by the
author in MATLAB. The second bound tightening algorithm, discussed in Subsection 3.1.2
is implemented by PowerModels.jl [31] in Julia [30].
Model 6 and Model 7 are implemented in Julia. For better reliability and maintainability
these model are implemented by modifying the built in models of PowerModels.jl.
The polytope construction and sampling are provided by Volesti [55] R [56] library.
Therefore, it is a requirement of PSDC that R and Volesti is installed, however Volesti is
called from the Julia package of PSDC.
The directed walk algorithms (Section 3.4), ﬁtting a multi variate normal distribution
(Section 3.5) and the ﬁnal evaluation of the set points are implemented in MATLAB.

modular implementation of PSDC enables the extension of N-1 stability by considering

In the followings the extension of the methodology to static N-1 analysis is presented.
OOPSAT makes it possible to consider N-1 contingency analysis, that considers large
disturbance voltage stability (see Figure 2.1), however this is out of the scope of this
work.
To assess all N-1 line outages of the system, directed walk samples are evaluated using
an extended version of DirectedWalks.checkSetpoint. N-1 criterion means that |L| + 1
number of evaluations are run. The +1 stands for the intact power system case. In this
case only line outages are considered, generator or other component outages are neglected.
Code 4.7 shows the implementation of extending the methodology to consider N-1
criterion. The current implementation considers each set point evaluation as an atomic
operation. Therefore, the N-1 subcases are evaluated sequentially. This is a disadvantage
of the current implementation. However, this implementation has the following advantage.
If the ﬁrst unstable contingency is found, the execution of N-1 stability analysis can be
interrupted before checking for all other cases.
In case of larger power systems, e.g.,
100 < |L| parallelization of the algorithm might be necessary. However, as detailed in
Subsection 2.1.3, the brute-force approach of checking each outage can be unrealistic and
unnecessary.
The implementation shown by Code 4.7 makes it possible to pass a list of contingencies,
therefore only the speciﬁc lines that are part of the list are evaluated for N-1 outages.
This makes it possible to deﬁne a list of most critical contingencies based on preliminary
assessment of the power system case as done in [20].
Further extensions
Another possible direction of extending the methodology is to include transient stability.
The works of [9] discusses the prediction of transient stability using data-driven algorithms.
By building on OOPSAT the methodology can be extended easily to dynamic stability
criteria assessment.
To implement such a functionality, OOPSAT has to be extended ﬁrst.
That requires
rewriting the old PSAT functional interface into an object oriented method of OOPSAT.
After that, the new OOPSAT functionality has to be tested on several power system test
cases. Finally, this criterion can be added similarly to the N-1 checks in the methodology.
The overall implementation can take up to 3–5 weeks including writing the relevant
documentation and excluding generating data sets. Due to the time constraint of this work
dynamic voltage stability criteria are not implemented. To assess dynamic stability criteria
— e.g., voltage stability — time domain simulations have to be carried out. Therefore, a
possible challenge of including further dynamic assessments is the increase in performance
requirements.

CONCLUSION AND FUTURE PERSPECTIVES
5.1
Conclusion
In this thesis, the problem of data set generation for power systems is addressed. The
contributions of this work include several theoretical improvements of the methodologies
presented in [1] and [2]. Furthermore, this work contributes open-source software tools
and data sets to the research community, to enable the development of data-driven power
system security assessment algorithms.
In Chapter 3 an eﬃcient data set generation methodology is proposed that enables not
only the evaluation of AC-OPF security criteria, but also further dynamic assessment of
power system security such as small-signal stability. One of the contributions of this thesis
is the generalization of the methods proposed in [1] and [2]. A further contribution of this
work is the development of a novel directed walk algorithm, that enables the investigation
of high-dimensional problems.
As none of the currently available open-source dynamic simulation tools provide
functionality to assess small-signal stability of a power system, OOPSAT was developed.
OOPSAT is one of the contributions of this work.
OOPSAT is currently the only
open-source dynamic power system analysis tool that can be run in a (highly) parallel
environment and provides small-signal stability assessment.
To analyze several test cases, IEEE benchmark systems were used. In this M.Sc. thesis
four test cases are thoroughly analyzed. The test ﬁles for these case studies are well-known
power system test cases in the literature.
The software implementation of the methodology, PSDC, is a comprehensive data set
generation software framework. PSDC provides an easy to use interface, hence user written
code can be used to conﬁgure the data set generation methodology. PSDC was designed to
be easy to use and provides a high-level interface that enables the generation of data sets
with a single MATLAB script ﬁle. Additionally, PSDC provides a rich and comprehensive
documentation with examples and unit testing functionalities.
The main focus of the thesis is the application of the methodology to several case studies,
presented in Section 4.3 – Section 4.6. All resulting data sets were created using PSDC
and subjected to exhaustive analysis. The case studies were subject to statistical analysis,
software performance analysis and ﬁnally the resulting data sets were also analyzed by
means of several simple classiﬁers such as decision trees or neural networks.
During the case studies it was found that the proposed methodology is capable of creating
well-balanced, high quality data sets. Furthermore, the implementation of the data set
creation methodology can make full use of parallel computing environments. Therefore,
Page 63 of 70

the implemented methodology can create larger data sets within a fraction of time
compared to the results of previous works.
The modularity of PSDC was presented by extending the methodology to N-1 analysis.
This extension also shows the easy to use interface of PSDC, however the deeper analysis of
N-1 case studies remains to be future work. Furthermore, PSDC was built on OOPSAT so
it can be extended to include further dynamic security assessments such voltage stability.
Finally, to contribute to the research community all software tools are published on GitHub
and the data sets are available at Zenodo:
• PSDC public repository: https://github.com/timonviola/PSDC.
• OOPSAT public repository: https://github.com/timonviola/OOPSAT.
• Published data sets: https://zenodo.org/communities/psdc/.
5.2
Future work
PSDC enables the development of data-driven algorithms for power system security
assessment. There are vast possibilities to explore more advanced models than the models
used in the case studies of this work.
A possible direction is to apply autoencoders
following the lines of [11]. Another possible application is to reformulate a pruned neural
network into a mixed-integer linear programming problem as done in [57].
PSDC, just like any open-source project, welcomes code improvements and extensions.
PSDC currently involves Julia, R, and MATLAB (and shell scripts). This technology
stack can be reduced to a single programming language.
Therefore, the installation
and maintainability of PSDC would improve.
Choosing MATLAB as an overall
implementation language has the disadvantage that MATLAB is a proprietary language.
Hence, not everyone has access to MATLAB. Julia has the disadvantage that most of the
analysis tools, directed walks have to be reimplemented. However, in case of a pure Julia
implementation, the performance of PSDC could be further improved.
This work also proposes a novel directed walk algorithm, the greedy directed walk. Future
works might be able to propose more eﬃcient directed walks and investigate further
improvements the algorithm, such as considering AC-OPF criteria.
Future work might also include the improvement of small-signal stability assessment based
on state-of-the-art methods. The works of [58] proposes a polynomial approximation based
method, that promises signiﬁcant performance beneﬁts compared to the conventional
perturbation based eigenvalue analysis.
On the other hand, another line of research
proposes a reformulation of the AC-OPF to characterize small-signal stable space [59].
PSDC can be further extended.
Currently the following criteria can be combined to
create/analyze data sets: AC-OPF criteria, small-signal stability criterion and static N-1
analysis. By building on OOPSAT, further extensions regarding dynamic stability criteria
can be conveniently accommodated.
Additionally, the data set creation method presented in this work can be extended to
consider load uncertainty similarly to the methodology presented in [2]. With the modular
