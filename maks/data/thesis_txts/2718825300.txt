Introduction
Given a finite number of strings of various lengths as input, Shortest Superstring problem
aims to find the shortest string which includes all the input strings as a substring. For
example, given the set of strings {abcd, cde, def}, one of the superstring can be found by just
concatenating all the strings i.e. the string “abcdcdedef”. However, there are some obvious
overlaps in the strings which can be removed to find the shortest superstring “abcdef”.
The superstring problem has many applications in data compression and DNA sequencing.
Since the problem is NP-hard, we only have polynomial time approximate algorithms. In this
thesis, we discuss about a few approximate algorithms, namely Greedy and Mgreedy. Greedy
algorithm is 4-approximate (but later on, proved to be 3.5) and Mgreedy is 4-approximate
as well. However, it is conjectured that Greedy algorithm is 2-approximate and is yet to be
formally proved.
We will first give some basic definitions and concepts that will be used in the analysis of
the algorithms. We will first explain MGreedy and then Greedy. The reason for this is
that MGreedy has some very interesting observations(given as Lemmas) that will help us
in analysing Greedy algorithm better later on. And finally, we have done some empirical
analysis by running the algorithms on different kinds of strings to see the length of the
superstrings returned by them. All the implementation has been done using Java as the
programming language.
2
Preliminaries
2.1
Basic Definitions
Firstly we define the problem more formally. Given a set of strings S = {s1, s2, s3, ...
sn} over a finite set of alphabet, shortest superstring problem finds the shortest string which
contains all si as substrings. We assume all the strings in the set are substring-free, i.e.
no string is a substring of another. If the set is not substring-free, we can just remove the
strings which are substring of another.
For two strings s and t, overlap of s w.r.t. t is denoted by ov(s,t) and is defined to be
the longest string y such that s = xy and t = yz for some non-empty strings x and z.
Furthermore, we define prefix of s w.r.t. t as pr(s,t) to be the string x as defined before. So
we have s = pr(s,t)ov(s,t) and we can say that two strings s and t can be merged together
to get a superstring pr(s,t)t. Let <s1, s2, s3, ... sn> represent the string pr(s1, s2)pr(s2,
s3). . . . . . .. pr(sn−1, sn)sn . Clearly this is the shortest string containing the strings s1, s2,
s3, ... sn in that order. So the optimal shortest superstring would be of the form <si1, si2,
si3, ... sin> for some order of the strings. We denote the length of shortest superstring by
OPTS for the set of strings S. And lastly we define ||S|| to be the sum of lengths of all strings
in the set S.

2.2
Overlap Graph
Given the set S of strings, we can define an overlap graph Gov to be a directed graph with
vertex set V=S and directed edges from string si to sj representing the overlap of si w.r.t
sj with a weight of |ov(si , sj)|. Clearly, Gov is a complete directed graph since the overlap
between any two strings will always have a weight value ≥0. As an example, let’s say S is
{“abcd”, “defabc”, “cdf” }. The overlap graph for the set S looks like below in Figure 1 :

Prefix Graph
Similarly, we can define a prefix graph Gpr to be a directed graph with vertex set V=S

of |pr(si , sj)|. Clearly, Gpr is a complete directed graph since the edges between any two

2.4
Cycle cover of a graph
A cycle cover of graph G is defined to be a set of cycles which are subgraphs of G and
each and every vertex of G is present in exactly once cycle. And a minimum cycle cover is
one of the many possible cycle covers of a graph G but the total sum of weights of all cycles
is minimized.

overlap much larger than the sum of total lengths of all strings. In this thesis, we do not
discuss about the use of any approximate Max-TSP solution, but we do find another set
of strings (called representatives) in the MGreedy algorithm. These representative strings
have low overlaps among each other and can be used as input to an approximate Max-TSP
algorithm.
3
Brute force Algorithm
The brute force algorithm involves generating all possible permutations of the given
strings. For each permutation, we would merge strings together from the left to the right
to get the corresponding superstring. And finally return the superstring with the shortest
length.
It is easy to see that the time complexity of generating all the possible permutations is O(n!)
where n is the number of strings in the given set S. Since it is an algorithm of factorial com-
plexity, the algorithm is just not feasible for problems with a large n and that is why we look
at approximation algorithms which can solve the problem approximately but in polynomial
time. In our implementation of the brute force, we use recursion to generate all the possible
permutations and is run only for a small number of strings(n < 10) to compare the length
of superstrings generated by the Greedy and MGreedy on the same set of strings.
In our java implementation, the class NaiveShortestSuperString.java is used to find the short-
est superstring using brute force. This class contains two methods, namely findSuperString()
and getOverlapString(). The method findSuperString() generates all possible permutations

takes two distinct strings si and sj as arguments and returns the merge <si, sj>.

Modified Greedy Algorithm
4.1
Description
MGreedy was described first in this paper [1] . MGreedy algorithm does the following
repeatedly: Select a pair of strings with the maximum overlap (breaking ties arbitarily) and
replaces the pair with their merge. When selecting a pair of strings, MGreedy also checks
the overlap of a string with itself (which is reffered to as a self-overlap from now on). In the
case of self-overlap, MGreedy removes the self-overlapping string from the set of strings and
stops processing that string in the future iterations. After processing all the strings, we will
be left with self-overlapping strings which were removed by MGreedy during its execution.
And finally MGreedy returns the concatenation of all these self-overlapping strings.

4.2
Correctness
Each iteration reduces the number of strings in set S by 1, either by being moving a
string from S to T or adding a merge of two string to S and removing the chosen two strings
from S. The last iteration of the loop will just move the last string from S to T causing the
size of S to become 0 and hence loop will always finish. And since all strings in S and T,
at any iteration of the loop, have been formed by either merging some strings in previous
iteration or they are the original strings that we began with. So all the strings in T after

the while loop would be superstrings of some subset of the original set of strings. Hence the
concatenated string would be a superstring for the original set of strings.
4.3
Approximation
In order to prove that MGreedy is a 4-approximate solution to Shortest superstring prob-
lem (SSP from now on), we will first consider another algorithm denoted by ’Concat Cycles’
(also mentioned in [1]) and prove that it is 4-approximate. Then we will prove that MGreedy
mimics it and hence MGreedy is also 4-approximate.
Let us first explain the algorithm Concat-cycles below:
Concat-Cycles Algorithm:
1. Given the set of strings S, create the graph Gpr and find a minimum cycle cover of it.
Let C represent the set of cycles C= {c1, c2, ... cp }
2. We do the following for each cycle c to obtain its representative string r(c) : Let the
cycle c be represented as si1, si2, si3 ... sik, si1 . Break the cycle arbitrarily at some
edge to get a path of nodes(recall each node is a representation of one of the original
strings). For simplicity, let’s say the path is si1, si2, si3 ... sik (the edge between sik
and si1 is removed). We can define r(c) as <si1, si2, si3 ... sik>
3. Concatenate all the representative strings r(c1), r(c2),. . . .r(cp) and return the resulting
concatenated string.
We first introduce a few notations and describe them before we prove the 4-approximation
of Concat-Cycles:
• w(C) :- Total sum of weights of all cycles in the minimum cycle cover of a graph.
• For some cycle c = si1, si2, si3 ... sik, si1 in the Gpr , we define s(c) as
s(c) = pr(si1, si2)pr(si3, si3)...pr(sik−1, sik)pr(sik, si1)
i.e. each r(c) corresponding to cycle c can be defined as r(c) = s(c)ov(sik, si1)
• Given a string s, s∞is defined to be a semi-infinite string and has the form ssss...
• For a superstring s = <s1, s2, s3 ... sk>, we call first(s) as s1 and last(s) as sk i.e. the
first and last string in the order of strings used to generate the superstring s.

Lemma 4.8. Modified Greedy algorithm actually generates a cycle cover.
Proof. We need to consider two cases: 1) When two strings are merged together (the else
part of the pseudo code) and 2) When it is a self overlap (the if part of the pseudo code)
and the string is removed from the set of strings. In case 1), Lemma 4.7 implies that the
merge of two strings, say x and y,(which may also be superstrings) corresponds directly to
selecting the edge between two nodes in the overlap graph. One node would correspond
to the first(x) and the other node would correspond to the last(y) which are both original
strings that we started out with. So case 1) would just be creating a directed path in the
overlap graph until case 2) happens. Case 2) corresponds to actually closing the path into
a cycle since the edge that is selected in case 2) is between the last(x) and first(x) where x
is the superstring corresponding to the path. It is easy to see that each node in the graph
generated by MGreedy would be head of exactly one edge and tail of exactly one other edge,
hence MGreedy generates a cycle cover.

Theorem 4.10. Modified Greedy algorithm generates a cycle cover of maximum weight.And
in turn, the cycle cover corresponds to a minimum cycle cover.
Proof. From Lemma 4.8 we have proved that MGreedy(Modified Greedy) generates cycles.
Now we will prove that weight of these cycles are maximized.
Let’s call M the cycle cover generated by MGreedy algorithm and N be a maximum weight
cycle cover that has maximum number of edges in common with M. (Note here that there
can be multiple cycle covers which are maximum). We shall show that M is the same as N.
Suppose that this is not the case.
Let e be the edge of maximum overlap which is in symmetric difference of M and N.
Suppose that this edge is in N \ M (in N but not in M). Since MGreedy did not include
e, it must have included some other adjacent edge f that is of higher overlap than e since
MGreedy is a cycle cover. Edge f cannot be in N since N is a cycle cover, therefore f is
in M \ N which contradicts the assumption that edge e is a maximum overlap edge in the
symmetric difference of M and N.
Now let’s suppose the edge e = k →j is in M \ N. The two N edges i →j and k →l
that share head and tail with e are not in M(since M is a cycle cover) and must have overlap
weights less than e. Since ov(k,j) ≥max{ov(i,j), ov(k,l)} then ov(k,j) + ov(i,l) ≥ov(k,l) +
ov(i,j) by Lemma 4.9. Thus we could replace the two N edges with e = k →j and i →l to
find another cycle cover N ′ that will have more edges in common with M and total weight
not less than N, which contradicts our choice of N.
Hence no such edge e exists which implies N=M.

5
Greedy Algorithm
5.1
Description
Greedy algorithm was first introduced in this paper [1]. Similar to MGreedy, it chooses
a pair of strings with the maximum overlap and replaces them with their merged string and
it continues doing the same until we are left with one string. The difference between Greedy
and MGreedy is that Greedy always chooses two distinct string while MGreedy can choose
the same string to check for self-overlaps.
Consider S = {c(ab)k, (ba)k, (ab)kc}. It is easy to see that Greedy will first join c(ab)k and
(ab)kc to create the string c(ab)kc, and then it will concatenate c(ab)kc with (ba)k to get the
string c(ab)kc(ba)k of length 4k+2. However, the optimal string is c(ab)k+1c and its length
is 2k+4. Interestingly enough, this case is considered to be the conjectured worst case for
Greedy.
Below is the pseudo code for Greedy Algorithm

2: while S.size() > 1 do

4:

7: return last string in S

5.2
Correctness
Each iteration reduces the number of strings in set S by 1 by merging two distinct strings
from the set S and replacing the chosen two strings from S with the merged string. And
since each merged string that replaces a pair of strings is a superstring of the said strings so
the last remaining string would also be a superstring of the original set of strings.
5.3
Approximation
After our analysis of MGreedy algorithm, it is easy to see that Greedy algorithm forms
a Hamilitonian path (not a cycle) at the end of its execution. Hence, we need different
approach to get the approximation factor for Greedy but we will nevertheless prove that
Greedy is 4-approximation solution as well.
Note that in the recent years, Greedy has been shown to have better approximation factor
than 4 but the conjecture that Greedy is 2-approximate is yet to be shown. In this thesis,
we only prove that Greedy is 4-approximate.
Theorem 5.1. Greedy is a 4-approximate solution for the shortest superstring problem.
Before we give its proof, we need to define some new notations, observations and lemmas
that will be used in the proof. Similar to MGreedy, we can think of greedy as going down a
list of edges sorted by their overlaps and deciding for each edge whether to include it in the
Hamiltonian path or not. We say that an edge is better(worse) if it appears before(after)
another edge in this list. Additionally, we say that an edge e dominates another edge f if e
is better than f and e shares its head(or tail) with the head(or tail respectively) of f.
Let’s say that Greedy produces the superstring <s1, s2, s3 ... sn> where the strings have
been numbered according to their order in the superstring. Now, we can observe that Greedy
does not include an edge e if and only if
1. e is dominated by an already chosen edge f, or
2. e is not dominated by any edge but it would form a cycle (as in MGreedy)
We call the edges in the latter case as "bad back edges", e = si −→sj where i < j. Every
bad back edge corresponds to a string <si, si+1, ... sj> which at some point in the execution
of Greedy has more self-overlap than overlap between any two other strings. The bad back
edge e is said to span the closed interval Ie=[i,j]. When Greedy considers the edge e when
going down the sorted list of edges, it would have already chosen all the edges on the greedy
path from si to sj, but not yet the worse edges si−1 −→si and sj −→sj+1 since if this was not
the case then <si, si+1, ... sj> would not be a self-overlapping string.
Now we will prove below that any two intervals corresponding to two bad back edges are
either disjoint or one contains the other.
Lemma 5.2. Let e and f be two bad back edges. The closed intervals Ie and If are either
disjoint or one contains the other. If If ⊂Ie, then e is worse than f i.e. ov(e) < ov(f).

it is 2-approximate. In each experiment, we do the following: Generate a random DNA
sequence of some length and create 4 more copies of it. Then for each copy, we cut the
DNA sequence into non-overlapping contiguous fragments of lengths between 20 and 30.
Then we combine all these fragments from each copy and give it as an input to the greedy
algorithm. It is highly likely that the final input set of fragments will be overlapping and
hence, it is not completely random strings as we did in the previous section of experiments.
The shortest superstring length would be close to the length of the random DNA sequence
we generated since all the fragments would be substrings of it. The experiment was run for
random DNA sequences of length 200, 300, 400 upto 1000. The results are shown below in
table 5. The DNA fragments generated from the DNA sequence has been created by the
method testNotTotallyRandomSuperString() of the class StringGenerator.java.
