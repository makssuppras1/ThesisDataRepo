Abstract
Fractional order derivatives applied to loudspeaker models have shown great improvements in the
prediction of the loudspeaker response. Especially, in the modelling of the diaphragm outer suspension
and the voice coil inductance. However, these kind of models require an excessive computational load,
since all the past history is used for the calculation of each new sample. Therefore, the implementation
in real time applications, such as feed-forward compensation systems, is limited.
In order to reduce the computational load, a multi-stage processing is proposed in [3]. This method
basically removes the less relevant part of the history, thus the number of computations is signiﬁcantly
reduced. Nevertheless, this approximation implies an reduction in the accuracy of the fractional order
model.
The main goal of this project is the improvement of the multi-stage processing model in terms
of eﬃciency and accuracy. In this way, speciﬁc elements of the algorithm are analyzed along the
diﬀerent chapters of the report. The analysis is mainly focused on the anti-alasing ﬁlter design, the
interpolation and the downsampling applied in the multi-stage processing. Based on these analysis,
diﬀerent solutions and modiﬁcations of the multi-stage processing are proposed. Finally, measurements
are carried out in order to evaluate the accuracy of the multi-stage processing model as a function of
the computational load.
In this project, a linear loudspeaker model is used. However, since the ﬁnal goal of the model is the
implementation in a feed-forward compensation system, in future modiﬁcations, the non-linearities of
the loudspeaker should also be included in the model.

Chapter 1
Introduction
A loudspeaker is an electro-acoustic transducer whose main task is to transform an electrical signal
into a sound wave. However, due to their physical properties, loudspeakers are non-linear devices,
especially at high signal levels. This means that the sound wave generated has frequency components
not contained in the input signal. In order to solve this problem, many methods have been proposed
along the years. The relevant one for this project is the feed-forward compensation, which predicts
the loudspeaker response based on a model and compensate the non-linearities by modifying the input
signal.
In order to model the loudspeaker response, a common solution is the state-space model representa-
tion, where the physical properties of the loudspeaker components are included in order to obtain
the loudspeaker response. One of the advantages of this kind of model is the possibility of adding
the non-linearities to the system. The state-space loudspeaker model has shown precise results in
many cases, although it has two important limitations in the prediction of the loudspeaker response.
On one hand, the creep eﬀect caused by the viscoelastic surround of the diaphragm produces an
increasing displacement response below the resonance frequency, which is not adequately predicted
by the model. On the other hand, the voice coil inductance losses produced by the eddy currents
are not often considered in this model, thus the loudspeaker impedance predicted at high frequency
is not totally accurate. In order to solve these limitations, recent studies showed that by adding
fractional order components to the model, a precise prediction of these two eﬀects can be achieved [1] [2].
Nevertheless, the solution of fractional order derivatives in the time domain depends on all the past
history. What is a great disadvantage when they are included in the stage-space model, since the
prediction of a new state depends on all the previous states. As there is no real system with enough
computational power and memory for processing such a number of samples, the history of the system
has to be truncated at some point. Moreover, taking into account that the model should be able to run
in real time, in order to be implemented in a feed-forward compensation system, the computational
load has to be drastically reduced.
This project is the continuation of a previous project [3], where multi-stage processing was implemented
in order to reduce the computational load, while keeping a reasonable model accuracy. Basically,
the multi-stage processing uses an speciﬁc wavelet transform to split the history signal in time and
frequency, so the less relevant part of the history is removed from the calculation, reducing in this way
the number of computations.

The multi-stage processing has shown a great reduction of the computational load, maintaining a
reasonable accuracy with respect to the full history solution or the measurements. However, the real
time performance has not been achieved yet. Furthermore, some aspects in the algorithm were still
not totally clear, especially the eﬀect of some of the steps in the multi-stage processing, such as the
anti-aliasing ﬁlter or the interpolation.
After this introduction chapter, three chapters explaining the fractional order derivatives, the fractional
order loudspeaker model and the multi-stage processing are included. Along these three chapters
the main theoretical concepts of the project are exposed. In chapter 5, the eﬀect of the anti-aliasing
ﬁlter used in the multi-stage processing is analyzed and discussed, a modiﬁcation of this ﬁlter is also
proposed. Then, in chapter 6, polynomial interpolation is implemented in the multi-stage processing
and the results are compared with the linear one, already implemented. Chapter 7 is focused on the
downsampling of the multi-stage processing, where higher factor downsampling is investigated. The
measurements results are presented in chapter 8, at the same time as they are compared with the
model simulations. Finally, in chapter 9 the main conclusions of the project are presented, as well as
some ideas for a future work.
1.1
State of the art
Fractional order derivatives have been studied in a pure theoretical way during the last three centuries.
However, along the last decades, several applications have been found, especially in the modeling of
physical properties of diﬀerent materials, but also in control systems techniques [4] [5].
Regarding the loudspeaker modelling, fractional order derivatives have been used in two diﬀerent
ways: to model the losses in the voice coil inductance [2], and to model the creep eﬀect caused by the
viscoelastic properties of the surrounding material [1].
Since the clearest disadvantage of the fractional order derivative is the high computational load,
many authors have deal with this problem in bibliography. From really simple solutions like the
"Short-Memory principle" [5], to complicated algorithms and transformations [6] [7]. Nevertheless, it
was not found any reference to a fractional order loudspeaker model able to work in real time. The
solution used in this project is the multi-stage processing algorithm implemented in [3].
1.2
Goals
The main goal of this project is to analyze and modify the multi-stage model, in order to improve its
eﬃciency and accuracy. This is necessary in order to use the model in a real time application, such
as a feedback control system, where both eﬃciency and accuracy are essential. The speciﬁc goals to
achieve along the project are listed below.

Chapter 2
Fractional Order Derivatives
First references to fractional order derivatives were in 1965, in conversations between Leibniz and
L’Hôpital, where they discussed the idea of derivatives with non-integer order. Since then, the study of
the fractional derivatives has been developed in a theoretical way, without many practical applications.
However, in the last decades fractional calculus has been applied in diﬀerent ﬁelds such as automatic
control techniques, or the modelling of dynamic systems [4]. The advantage of the fractional derivatives
lies in the fact that the derivative order does not have to be exactly one (or any other integer order),
but it can be adjusted to any value in between zero and one, allowing a more precise model. The
accuracy improvement is especially noticed in the modelling of systems with memory, which are those
systems where the present state depends on the previous states, as for the loudspeaker.
Recent studies showed the advantage of using the fractional order derivatives in the modelling of
loudspeakers. Speciﬁcally the outer suspension, normally made of a viscoelastic material, and the
voice coil inductance, are well modeled using these techniques. In this chapter, the main theoretical
concepts regarding the fractional order model used in this project are exposed. A more detailed
explanation of this model can be found in [3].

CHAPTER 2. FRACTIONAL ORDER DERIVATIVES
The fractional order formulation presented in equation 2.2 considering n ∈R+ is known as the
Grünwald–Letnikov’s deﬁnition of the fractional order derivative. There are other formulations of the
fractional order derivative such as Caputo or Riemann-Lioville. However, the Grünwald–Letnikov’s
deﬁnition is used in this case because of its discrete formulation [3].
As can be seen in equation 2.2 the summatory is limited to n, since for all the values above n
(k = n + 1, n + 2, n + 3, ...) the binomial coeﬃcient would be zero. However, this is not true if
n ∈R+, since the diﬀerence between a fractional and an integer number cannot be zero [3]. This is
the reason why the fractional order derivative depends on all the previous states, thus it requires a
large computational load and memory.
In order to simplify the calculation, the easiest solution is the "Short-Memory principle", which
basically truncates the history by removing the oldest samples from a certain time length [4]. The
consequences of this calculation reduction in the model accuracy are presented in section 4.2.

representation of the loudspeaker linear model is used. The fractional order state-space solution is
presented in equations 2.4 and 2.5 [3].

Loudspeaker Model
A loudspeaker can be described as an electro-acoustic transducer which transforms an electrical signal
into mechanical movement, and this mechanical movement produces a sound wave. In ﬁgure 3.1 1, a
sketch with the main parts of a standard moving-coil loudspeaker is presented. Although there are
many kinds of loudspeaker, this one is just an example of the most common conﬁguration.
The electrical part of the loudspeaker consists in a moving voice coil inside a magnetic ﬁeld. When
the electrical current signal ﬂows through the voice coil, the magnetic ﬁeld generates the force that
moves the voice coil and therefore the diaphragm, producing the acoustic vibration. As it can be seen
in the sketch, the diaphragm is attached to the voice coil, but also to the loudspeaker frame through
the surround and the spider, also known as outer and inner suspension. These two elements force the
diaphragm to come back to the resting position when there is no signal.

Chapter 4
Multi-Stage Processing
In order to make the fractional order calculation useful for a real time application it is necessary to
reduce the number of computations in the model. As it was explained in chapter 2, the fractional
order solution depends on all the previous states, which means that the number of computations will
increase with the signal length, and therefore the calculation time. When dealing with a real time
application, the signal could be considered inﬁnitely long, since its length is not speciﬁed. This is a
problem, not only because of the memory limitations, but also because the amount of computations
would be extremely large. For this reason, the history length used in the fractional order calculation
must be truncated at some point.
Several methods have been developed in order to reduce the computational load in the fractional order
calculation. The most basic is the "Short-Memory Principle" [5], which keeps only a certain length of
the memory, removing the oldest part. The accuracy of this method depends on the memory length
used for the calculation, although the longer the memory, the more higher computational time.
Another kind of solutions keep the history in a logarithmic spaced way [6], so the most recent part of
the history is highly sampled while a lower sampling rate is used for the older memory. In this way,
the memory length is kept fairly large with a lower number of computations. The solution used in
this project is based on this method. Also, another method proposed by Ikeda [7] implements an
inhomogeneous sampling algorithm which performs a speciﬁc time transformation depending on the
fractional order used.
The method implemented by Madsen [3] and used in this project is based on a multi-stage processing
of the history used for the calculation. Basically, this method performs an analysis of the weighting
function in the time and the frequency domain. From this analysis, the most relevant part of the
history is selected for the calculation in such a way that the number of computations is considerably
reduced while the error is acceptable. This chapter gives an overview of the multi-stage processing
method used in the fractional order model. Also, in the last sections, the most relevant concepts for
next chapters are explained in detail.

4.2
Wavelet Transform and Multi-Stage Processing
Considering both time and spectrotemporal analysis of the weighting function, it is concluded that
the length of the history considered for the calculation does not need to be so long at high frequencies,
while for lower frequencies, a longer part of the history is required. Also, the most recent history is
more relevant than the older part of the history. In this way, the goal of the multi-stage processing is
to keep all the frequency content of the recent history, while removing the high frequencies as the
history is older. This conclusion leads to the use of the wavelet transform [10], which basically split
the signal spectrum allowing the multi-stage processing to select relevant content of the history. It has
to be mentioned that this kind of processing is commonly known as "multi-rate", although in this
case the "multi-stage" term is used, since it was found more appropriated.
In ﬁgure 4.31, it can be seen how the wavelet transform split the signal in time and frequency. The
approach of this transformation is the fact that a long period of the signal is required to analyze its
low frequency content, since its wavelength is larger. On the other hand, for high frequencies a shorter
period of the signal can be used. In this case, the grey part of the signal shows the part of the history
that is going to be kept for the fractional order calculation. The rest of the signal will be removed,
reducing signiﬁcantly the memory used and number of computations.
Figure 4.3: Wavelet transform time and frequency distribution. Grey area shows the part of the signal
obtained by the multi-stage processing
The wavelet transform is implemented as a set of lowpass and highpass ﬁlters, where the signal
is downsampled after the lowpass ﬁlter. The diagram used in this wavelet transform is presented
in ﬁgure 4.4, where the lowpass and highpass ﬁlters are represented by H0(z) and H1(z) respect-
ively. The downsampling is represented by the sign ↓2, and it basically removes the highest half
part of the spectrum and keeps the lowest. After downsampling, each sample corresponds to two
samples before downsampling, which means that it represents double the time. In this way, each
time the signal is downsampled, not only the spectrum is reduced to the half, but the same amount
of samples are representing double the time. This process is repeated several times in cascade until
the signal is reduced to a narrow frequency band at low frequencies, which represents a long time period.

the third stage only one forth and so on.
The stages are implemented in buﬀers of memory which are ﬁlled with the incoming samples from the
history. The new sample is placed in the ﬁrst position and it is moved to the next one each time a
new sample arrives. Once the buﬀer is full, the last sample is simply discarded when there is a new
one. The only stage that receive the samples directly from the output of the model is the ﬁrst one,
while the others receive the new samples from the previous stage. For example, each 2 samples, the
second stage includes in its ﬁrst position the last sample from the ﬁrst stage. In the same way, each 4
samples, the third stage takes its ﬁrst sample from the end of the second stage, and so on. In this way,
the history is saved in several stages until the memory is ﬁlled.
In the same way as the history is successively downsampled along the diﬀerent stages, the weighting
function is also downsampled in order to multiply each value with its corresponding part of the signal.
However, the downsampled weighting function is not calculated in real time as the history, but it is
calculated only once and its values are kept in memory, from where they are used in the fractional
order calculation. This approximation will also contribute to the error of the multi-stage processing.
Regarding the downsampled version of the weighting function, the weight of the new samples has to
be reconsidered. As the new values are representing several values of the original weighting function,
each of them is multiplied by the number of samples that it is representing. Basically, the weighting
function is multiplied by 2stageN−1. The ﬁrst stage is multiplied by 1, since no downsampling has
been done yet and each sample represents one sample. The second stage is multiplied by 2, since each
sample represents two samples, the third stage is multiplied by 4 and so on.

4.3
Aliasing and reconstruction
One of the problems when dealing with this kind of signal processing is due to the fact that the signal
is downsampled and reconstructed. In this way, two issues must be considered: the aliasing due to
downsampling the signal, and the signal replications due to the reconstruction. Moreover, since these
two processes are repeated many times along the multi-stage processing, their analysis and solution
becomes more complex.
17
==PAGE:26==
CHAPTER 4. MULTI-STAGE PROCESSING
The aliasing is produced when the sampling frequency is not high enough to sample the signal properly.
From Nyquist sampling theorem it is known that the sampling frequency has to be at least half the
maximum frequency of the signal, f2/2 ≥fmax [9]. Otherwise, the frequency content above fs/2
would be mirrored in the downsampled version of the signal.
In order to prevent this eﬀect, the signal is normally ﬁltered by a lowpass ﬁlter called anti-aliasing
ﬁlter, which attenuates the frequencies above Nyquist limit. Since a perfect ﬁlter is impossible to
implement in a real situation, the frequency content over fs/2 will not be totally removed. For this
reason, aliasing cannot be completely avoided, just reduced more or less depending on the ﬁlter used.
Regarding the reconstruction process, its task is to upsample the downsampled signal in order to
obtain a new signal as close as possible to the original one. Of course, the frequency content ﬁltered by
the anti-aliasing ﬁlter will not be recovered. The simplest way of reconstruction is just a replication of
each sample in the signal, obtaining a stepped signal. In the frequency domain, this reconstruction is
translated as periodic repetition of the spectrum at high frequencies, which obviously is not the best
solution. The most common way of signal reconstruction is to apply some kind of average between the
signal samples, what is known as interpolation. In the frequency domain, the interpolation can be
seen as a low pass ﬁlter which attenuates the spectrum replication.
A simple example is presented in ﬁgure 4.6 in order to visualize the consequences of these problems
in the model. In this example, the displacement response is obtained for a pure tone at 2 kHz. The
diﬀerence between the four plots is the number of stages used in the multi-stage processing. The
sampling frequency is ﬁxed at 20 kHz, so the signal frequency range only goes until 10 kHz in order to
satisfy the Nyquist theorem. In the ﬁrst case, only the ﬁrst stage is used, which is the stage where all
the samples are kept, in other words, the signal is not downsampled any time. This is the reason why
only the pure tone can be seen in the spectrum.

5 to 2.5 kHz in order to downsample the third stage and obtain the fourth one. Since the sampling
frequency is 2.5 kHz, only the signal until 1.25 kHz (fs/2) can be sampled. For this reason, the pure
tone cannot be represented in the fourth stage, and it creates an aliasing product at 500 Hz, which is
exactly the mirrored version of the 2 kHz tone from half the sampling frequency, 1.25 kHz. Although
the pure tone is not present anymore in the the fourth stage, it is still present in the previous stages,
so it appears in the multi-stage solution. Regarding the reconstruction process, more reconstruc-
tion products can be seen in this case simply because the reconstruction has been performed more times.
Finally, in the last plot, many aliasing and reconstruction products appear in the multi-stage solution,
in such a way that is not totally clear which peaks come from the aliasing and which come from the
reconstruction. As a general rule, the reconstruction products appear mostly at high frequency, while
the aliasing products at low frequency, although at a certain point they are mixed. The example
presented is fairly simple, since the signal is a pure tone and not many stages are used. However, the
signal is usually more complex and more stages are normally used. Thereby, a great amount of aliasing
and reconstruction products are present in the model solution, where it is practically impossible to
distinguish between them.
4.3.1
Anti-aliasing Filter
The anti-aliasing ﬁlter is a crucial element in the multi-stage processing in order to reduce the aliasing
eﬀect produced when downsampling the signal. In this case, a Finite Impulse Response (FIR) ﬁlter
is used in the multi-stage model. The cutoﬀfrequency of the ﬁlter has to be at fs/2, since above
this frequency is the part which is going to be removed by the downsampling. The ﬁlter is created
with Matlab and its frequency range is normalized with the sampling frequency. For this reason it is
possible to apply the same ﬁlter in all the stages independently on the frequency range of the signal,
which is halved in each stage.
The way the ﬁlter is implemented in the multi-stage processing is by multiplying its coeﬃcients by
the last samples of the buﬀer. Taking into account that the buﬀer length does not correspond to the
number of samples per stage, but its the number of samples per stage plus half the ﬁlter length. In
this way, the ﬁlter is perfectly centred at the end of the stage, which corresponds to the exact sample
required for the next stage. For example, in a case where 10 samples per stage and a third order ﬁlter
with 4 coeﬃcients are used. The buﬀer length is 10 + 4/2 = 12 samples long, and the ﬁlter is centered
between samples 10 and 11. The sample calculated corresponds to the value at the 10.5 sample and is
added at the beginning of the next stage.

The order of the ﬁlter is also a relevant parameter in the multi-stage processing. Odd order ﬁlters
have an even number of samples, so it would be centred between two samples. On the other hand,
even order ﬁlters have an odd number of samples, which means that it would be centred in a speciﬁc
sample. Since a second order downsampling is applied in this case, half of the samples are removed.
Considering an impulse signal where the impulse is placed in one of the samples that are going to be
removed, the even order ﬁlter cannot get the impulse. However, the odd order ﬁlter ensures that the
impulse will be placed in one of two samples where it is centered. Although, this is just a worst case
example, the odd order ﬁlter was chosen, since in this case it is considered that it is less time variant
to the input signal.
In chapter 5 a more detailed explanation of the ﬁlter design is given, as well as a discussion about the
eﬀect of this ﬁlter in the multi-stage processing and in the model accuracy.
4.3.2
Interpolation
When reconstructing the saved history for the fractional order calculation, some of the samples are
missing due to the downsampling. For example, when downsampling, the average between samples 1
and 2, 3 and 4 or 5 and 6 is calculated, so the values at the samples 1.5, 3.5 or 5.5 are obtained, but
not the ones at 2.5 or 4.5, and at some point these points are required in the history reconstruction.
In the example, the diﬀerence between the required and the saved value is just 1 sample since the
second stage is considered. However, as the stage number increase, the downsample increase as well,
so the diﬀerence between the value of a required sample and the calculated one might be huge.
In order to solve these discontinuities in the saved history, a linear interpolation was included right
before the multiplication by the weighting function. The linear interpolation is the simplest way of
interpolation and is the one chosen in this model. It basically consists on a calculation of the value
that the function would have in a speciﬁc point, considering the function as a straight line between
the two closest samples to that point. In equation 4.5 it can be seen how the signal is interpolated.
Basically (x, y) is the sample required, while (x0, y0) and (x1, y1) are the two closest samples.

The computational load is obtained just by calculating the amount of operations required to ob-

used, it is possible to obtain the computational power a DSP needs in order to run the model in real time.
The main computational load in the full solution comes from the summatory, where the weighting
function is multiplied by each state of the history saved. In this case, the multiplication and the sum
are considered as just one operation, so for calculating a set of k samples, the number of operations
required is equal to the amount samples to the power of two, NOp = k2. Since for each new sample
all the previous are multiplied by the weighting function, the number of operations has a polynomial
growth of factor 2. This is the reason why the full solution cannot be implemented in a real time
application.

the summatory is only considering the number of samples per stage η by the number of stages NS.

samples and stages, when the downsampling is performed. In order to simplify the calculation, in [3]

Chapter 5
Anti-Aliasing Filter
As it was explained in the previous chapter, the anti-aliasing ﬁlter is used in each stage before the
downsampling in order to reduce as much as possible the aliasing eﬀect, and in this way improve the
model accuracy. From Madsen’s results it was concluded that the anti-aliasing ﬁlter has a signiﬁcant
impact in the model error, although some aspects were not totally clear [3].
An analysis of the ﬁlter behaviour has been carried out in order to understand its eﬀect in the model.
Also, a modiﬁcation of the ﬁlter is proposed in order to reduce the ﬁlter order without decrease
the model accuracy. In this chapter, the main aspects related with the ﬁlter design and the ﬁlter
modiﬁcation are explained, as well as the conclusions from the eﬀect of the ﬁlter analysis.
5.1
Filter Design
The anti-aliasing ﬁlter used in the multi-stage processing is designed with the Matlab function
fdesign(). This function gives the ﬁlter coeﬃcients based on diﬀerent parameters, in this case the
parameters used are:
• Apass: Largest amplitude in the pass band ripple.
• Astop: Minimum attenuation in the stop band.
• Fpass (ωpass): Frequency until where the pass band is considered.
• Fstop (ωstop): Frequency from where the stop band is considered.
In ﬁgure 5.1, the template Matlab uses for the ﬁlter design is presented. Basically, the speciﬁed
parameters are the limitations of the maximum values the ﬁlter response can take. Diﬀerent ﬁlter
parameters can be chosen for the ﬁlter design, for example the ﬁlter order (N) or the cutoﬀfrequency
(fc). In this case, the ﬁlter order is automatically selected based on the limitations of the other
parameters. The cutoﬀfrequency could also be found as the frequency in the transition band where
the ﬁlter response is 6 dB lower than the lowest pass band value.
Since the ﬁlter pass band is supposed to be as ﬂat as possible, the ′equiripple′ option is selected in
Matlab when designing the ﬁlter. This option makes the ripple maximum amplitude constant along all
23
==PAGE:32==
CHAPTER 5. ANTI-ALIASING FILTER
the ﬁlter pass band. Other options selected in the ﬁlter design are: ′odd′, in order to have an odd order
ﬁlter, or ′MinOrder′, which makes the ﬁlter as short as possible taking into account the limitations.
Figure 5.1: Filter design template in Matlab showing the main constrictions.
When designing the ﬁlter, the computational eﬀort is an aspect to consider, since each time one stage
is ﬁltered, a multiplication by each of the ﬁlter coeﬃcients is performed. In this way, a lower order
ﬁlter will reduce the number of operations. On the other hand, a higher order ﬁlter it is expected to
produce a lower error in the ﬁnal solution. Depending on the requirements of a speciﬁc application in
terms of eﬃciency and accuracy, it might worth to use a high order ﬁlter or not.
Another limitation in the ﬁlter design could be the buﬀer length. As it was explained in the previous
chapter, the samples of each stage are saved in a buﬀer. Since the ﬁltering is performed as a multiplic-
ation of the ﬁlter coeﬃcients by the last samples of the buﬀer, and the ﬁlter has to be centered at the
end of the stage, the length of the buﬀer is determined by the number of samples per stage plus half
the ﬁlter length. For this reason, if there is a limitation in the buﬀer length, the ﬁlter length is also
limited.
5.2
Filter Modiﬁcation
Based on Madsen conclusions regarding the anti-aliasing ﬁlter, it was found that the error caused by
this ﬁlter in the multi-stage processing was not always proportional to the ripple amplitude, as was
be expected [3]. It was also observed that the amplitude of the ﬁlter at the DC component has a
signiﬁcant consequence in the error. In some cases, a positive gain at the DC component of the ﬁlter
produce lower error than a negative gain, even if the negative gain is lower in absolute terms. This
eﬀect was not expected since the pass band of the ﬁlter is supposed to be as ﬂat as possible (lower
ripple), in such a way that the desire part of the spectrum is kept with as low modiﬁcation as possible.
Observing the ﬁlter pass band response, especially the ripple amplitude, some answers can be found.
It has to be considered that the signal is not only ﬁltered once, but it is ﬁltered one time in each stage.
Also, since the ﬁlters are in cascade, the ﬁlter response in each stage corresponds to the product of the
ﬁlter at that speciﬁc stage by all the previous stages ﬁlter. As it was explained, all ﬁlters used in the
multi-stage processing have the same shape, although their frequency range is halved from one stage
to the next one due to the downsampling. In this way, a maximum in the ripple amplitude might
24
==PAGE:33==
CHAPTER 5. ANTI-ALIASING FILTER
correspond with a minimum in the next stage and the other way around. For this reason, it can be
assumed that the ripple maximums and minimums are somehow compensated from one stage to the
next one.
The previous explanation is assumed to be true for all the frequency range except for the DC compon-
ent. Since the frequency range is halved in each stage, the ﬁlter cutoﬀfrequency is halved, although
its value at the DC component is the same in all stages. In this way, if the ripple amplitude at the
DC component is positive or negative, since it has the same value in all ﬁlters, the amplitude in
the total ﬁlter response is accumulated from one stage to the next ones, ending up with a huge deviation.
This eﬀect can be clearly seen in ﬁgure 5.3 (left), where the total ﬁlter response of the ﬁrst six stages
are presented. In this case, the ripple amplitude at 0 Hz is -0.5 dB, and it is accumulated in each
of the stages. On the other hand, the ripple in the rest of the pass band keeps a value close to the
Apass = 1dB limitation.
The solution proposed in this project to solve the ﬁlter problem at the DC component is to modify
the ﬁlter in such a way that its gain at this point is 0 dB. In this way, the gain of the ﬁlter at the DC
component is neither positive or negative, instead there is no gain. The ﬁlter modiﬁcation is done by
ﬁxing the ﬁrst value of the ﬁlter frequency response to one. In ﬁgure 5.2 it can be seen an example of
this modiﬁcation. It has to be considered that 1024 points were used in the FFT of the ﬁgure in order
to visualize clearly the ﬁlter shape. This is the reason why the ﬁlter responses are not only diﬀerent
at the DC component. However, in the FFT used for the ﬁlter modiﬁcation, the number of points was
equal to the number of coeﬃcients of the ﬁlter, 36 in this case, represented in the ﬁgure as the N-ﬀt
response. Looking at the values of this response, it can be seen how they are exactly the same in both
ﬁlter responses, except at the DC component.

Chapter 6
Higher Order Interpolation
As it was explained in chapter 4, the signal reconstruction is one of the crucial steps in the multi-stage
processing, although it is also one of the causes of error in the solution. A linear interpolation is
included in order to reduce the error produced by the reconstruction. In this chapter, the linear
interpolation is substituted by a higher order interpolation in order to analyze its eﬀect in the model
accuracy. Also, it is evaluated if the improvement in accuracy worths the increase in computational load.
6.1
Polynomial interpolation
The higher order interpolation chosen in this case is called polynomial interpolation. In section 4.3.2
it was explained how the linear interpolation calculates the new sample value based on the the two
closest samples, and their distance with respect to the new sample. The polynomial interpolation
considers more than two samples in the new sample calculation. Basically, this kind of interpolation
consists in the calculation of a polynomial function containing the known samples, so the interpolated
samples can be found just by looking at the polynomial function in the speciﬁc time position [12].
A simple example is presented in ﬁgure 6.1 in order to show the diﬀerent interpolation kinds. The red
curve shows the reconstructed signal without interpolation, just by replicating the value of the closest
sample, also known as "Nearest Neighbor Interpolation". Since the reconstructed curve is stepped, a
high level in the reconstruction products at high frequency is expected. The yellow curve corresponds
to the linear interpolation, where basically the samples are joined by a straight line. Finally, the
purple curve shows the polynomial interpolation. As it can be observed, the polynomial interpolation
has a smoother shape than the previous ones. In the frequency domain, the smoothness of the curve
is translated into a greater reduction in the reconstruction products.

Chapter 7
Higher Downsampling Factor
In order to reduce the computational load of the fractional order calculation, the most eﬀective method
found in this project is to increase the downsampling factor in the multi-stage processing. In this
way, the same history length can be saved in memory with a lower number of samples, and therefore,
the number of computations is reduced. However, a reduction in the model accuracy, due to the
computational load improve, is expected. In this chapter, two multi-stage processing modiﬁcations
are proposed in order to reduce the computational load by increasing the downsampling factor. Also,
the consequences of these modiﬁcations in the model response are analyzed by comparison with the
original multi-stage processing and with the full solution. Moreover, a problem in the multi-stage
processing related with the continuity between stages is found and discussed.
7.1
Downampling factor 4
The downsampling factor implemented in the multi-stage processing until now is 2. In this section,
the downsampling factor is increased until 4. This modiﬁcation involves many small variations in
the multi-stage processing although the algorithm is basically the same. The main diﬀerence in the
multi-stage processing is the number of samples removed from the history. While in the original multi-
stage processing, 1 in every 2 samples is saved from one stage to the next one, in this case, only 1 in
every 4 samples is saved. In the following paragraphs, the consequences of this modiﬁcation are exposed.
First of all, the number of stages required in order to ﬁll the memory length considered in the fractional
order calculation has to be recalculated. From equation 4.2, the new calculation is obtained as it can
be seen in equation 7.1. It has to be noticed that the number of stages used in the new multi-stage
processing is half the number used in the original multi-stage processing plus one.

Another fact to consider when increasing the downsampling factor is the anti-aliasing ﬁlter used.
Downsampling by a factor of 4 implies that only the ﬁrst quarter of the spectrum is kept, so the

in the ﬁlter length, the buﬀer length has to be longer as well, since the buﬀer size is the stage length
plus half the ﬁlter length.
Finally, in the history signal reconstruction for the fractional order calculation, the interpolation also
has to consider the new spacing between samples in order to estimate the value of the samples removed
by downsampling.

7.2
Downsampling factor 2 and 4
In the previous section it was shown how the computational load can be reduced by increasing the
downsampling factor, although the model accuracy is reduced. Depending on the speciﬁc application,
this error might not be acceptable, and if the system properties allows it, the number of operations
might be increased in order to reduce the error. Trying to achieve a compromise in terms of accuracy
and eﬃciency between downsampling by a factor of 2 and by a factor of 4, a new modiﬁcation of
the multi-stage processing is proposed. The goal of this modiﬁcation is to adjust the multi-stage
processing to the application requirements.
The idea of this algorithm is to ﬁx the number of stages where factor 2 downsampling is applied, and
calculate the number of stages with factor 4 downsampling required to achieve the memory length
needed in the calculation. In this way, the larger the number of stages with factor 2 downsampling
used, the more accurate solution is achieved with a higher computational load. On the other hand, a
small number of stages with factor 2 downsampling, means a lower computational load and an increase
in error.
The modiﬁcations performed in the multi-stage processing are basically the same as in the previous
modiﬁcation: number of samples to remove, number of stages calculation, sample index vector and
weighting function, anti-aliasing ﬁlter design and interpolation. However, the implementation is a bit
more complex since in some cases the downsampling factor is 2, while in some others it is 4.
As it was explained in the previous section, the ﬁlter used when downsampling by a factor 4 is twice
longer than the one used when the factor is 2. Thereby, the ﬁlter applied in the multi-stage processing
is diﬀerent depending on the downsampling factor. Due to the diﬀerence in the ﬁlter length, the buﬀer
size has to be reconsidered, since it is determined by the number of samples per stage plus half the
ﬁlter length. This problem is avoided if the long ﬁlter is considered to calculate the buﬀer length.
However, in those stages where the short ﬁlter is applied, the last samples of the buﬀer are not used.
In this project it was not investigated a solution for this problem, thus the longer ﬁlter is used in the
buﬀer size calculation.
However, another modiﬁcation is performed in order to select the number of samples per stage for
each downsampling factor. Basically, the matrix containing all the stages is divided in three matrices.
In the previous multi-stage processing implementation, the content of all the stages is saved in a
three-dimensional matrix, where the ﬁrst dimension is the buﬀer length, the second one corresponds
41
==PAGE:50==
CHAPTER 7. HIGHER DOWNSAMPLING FACTOR
to the four variables of the state-space model, and the third one to the number of stages. In this case,
the stages are divided into three kinds: the stages where no downsampling is applied, which is only
the ﬁrst one, the stages with factor 2 downsampling, and the ones with factor 4 downsampling. Each
kind of stage is saved in a diﬀerent matrix, so the size of these matrices can be adjusted to the number
of samples per stage used in each case. As a consequence of this separation of the stages in three parts,
all the operations performed in the multi-stage processing are also divided in three parts. This might
increase the complexity of the algorithm, although should not aﬀect the total number of operations.
This modiﬁcation improves signiﬁcantly the ﬂexibility of the multi-stage processing, and therefore, the
computational load and accuracy can be easily adjusted depending on the number of samples per stage
selected in each case. For example, considering the ﬁrst stage, where the most recent samples are kept
without downsampling, thus it contains the whole frequency spectrum of the signal. If the number of
samples in the ﬁrst stage is increased, the recent history is saved with more accuracy, reducing the
model error at high frequencies. On the other hand, if the model accuracy at high frequency is not a
requirement, the number of samples in the ﬁrst stage could be reduced, so the computational load is
reduced as well.
Although the variable size stages are implemented and the correct performance is veriﬁed, the results
were not evaluated since in this case the analysis is focused on the downsampling factor. Therefore, in
all the examples presented in this project, the number of samples per stage is the same in the three
kinds of stage.
The Matlab code for this multi-stage processing implementation is presented in the appendix. This
code is a modiﬁcation of the function developed in [3]. In the program it can be compared the
downsampling by each of the factors 2 and 4.
Finally, the obtained results in terms of accuracy and eﬃciency are in between the results when the
downsampling factor is 2 and when it is 4, as it was expected. In the next section, a more detailed
analysis of the error obtained is presented.

7.4
Discontinuity between stages
One of the problems observed in the multi-stage processing is the discontinuity between the diﬀerent
stages in the reconstructed signal. These discontinuities produce jumps as the ones presented in ﬁgure
7.3. These jumps are especially noticeable in the impulse response, while for other kinds of signal they
do not appear so clearly.
Looking at the jumps position in time and the spacing between them, it is concluded that the jumps
are produced by the stages. Each time the impulse is removed from one stage and inserted in the next
one, a jump is produced. Moreover, it is found that the ﬁlter impulse response also have an eﬀect in
the broadness and amplitude of the jumps. In such a way that a higher order ﬁlter produces broad
jumps with low amplitude, while a low order ﬁlter produces shorter jumps, but larger in amplitude.
In order to analyze and understand how the multi-stage processing produce the jumps, the content
of the stages is observed one by one. In this case, instead of using the loudspeaker model, only the
fractional derivative is used, so the multi-stage processing eﬀect can be analyze without the eﬀect
of the loudspeaker dynamics. Thus, basically the multi-stage processing is calculating the fractional
derivative of an impulse signal. Furthermore, the simulation is performed for 2 diﬀerent anti-aliasing
ﬁlters, one of them is a 65 order ﬁlter, with a ripple amplitude (Apass) of 10−5 dB. The other ﬁlter
used is a ﬁrst order ﬁlter, with only two coeﬃcients [0.5 0.5], which is considered the simplest low pass
ﬁlter.
The results from the stage by stage analysis are presented in ﬁgure 7.4. In this ﬁgure it is presented
the content of each new sample inserted in each of the stages along all the ﬁrst 5000 processed samples

In order to verify the model accuracy at low frequencies, where the creep eﬀect becomes important,
long simulations must be considered. In this case, the lowest frequency considered is 0.01Hz, which cor-
responds to at least a 100 seconds signal. Since the time it would take to run the full solution model is

the fractional order model is ﬁtted. This model is included in the multi-stage processing and the
frequency response is simulated for the diﬀerent mutli-stage processing implementations. Finally,
the simulations are compared with the measurements in order to obtain the error produced by the

8.2.2
Fitting algorithm
The ﬁtting algorithm applied is based on the algorithm used in [3], which uses the Matlab function
fmincon() in order to minimize the error between the measurements and the model frequency re-
sponses. The model parameters are initialized with a starting guess, and their maximum values are
deﬁned as constraint limits. Then, the loudspeaker frequency responses are calculated based on the
parameters, and the relative errors with respect to the measurements are obtained. The sum of the
three relative errors gives the cost function, which is the value the algorithm is trying to minimize.
This process is repeated successively, modifying the parameter values within the constraints, until the
lowest cost function is achieved.
Since the model ﬁtted is basically an approximation to the frequency response curve measured,
the noise in the measurement might reduce the accuracy of the approximation. For this reason,
the noise at low and high frequency should be avoided in the ﬁtting algorithm, thus the frequency
range considered is from 1 Hz to 1 kHz for the displacement and from 30 Hz to 3 kHz for the impedance.

this model. However, it has to be considered that the integer order model used in this case is a fairly

8.4
Discussion
Along this chapter it could be seen that the error between the measurements and the simulations
from the ﬁtted model, was much larger than expected. Therefore, it was not possible to evaluate the
diﬀerences in accuracy of the three multi-stage processing implementations. The error with respect to
the full solution is much lower than the error obtained with respect to the model frequency response,
or to the measurement. In this way, it is is safe to conclude that the error is not coming from the
multi-stage processing.
On the other hand, since the error with respect to the measurements is the same in the three mutli-stage
processing implementations, it could be said that the higher downsampling factor does not aﬀect the
model accuracy in this case. Based on the fact that the multi-stage processing error with respect
to the full solution is around -70 dB, while this error with respect to the measurement is around
-30 dB. The computational load could be drastically reduced, before a larger error with respect to
the measurements is obtained. In this way, an even higher downsampling factor could be used, and
the number of samples per stage could also be reduced. Of course, this is only true if the error with
respect to the full solution is much lower than the error with respect to the measurements. However, a
low error with respect to the measurements is diﬃcult to achieve due to the noise at the displacement,
as it has been seen in the measurement results.
In order to reduce the error between measurements and simulations, two factors have to be considered:
the noise at the displacement, and the deviation at the resonance. Regarding the noise produced at
the displacement, especially at low frequency, a reduction in this noise would allow a broader frequency
range in the ﬁtting algorithm, and also a comparison at lower frequencies would be possible. In order
to measure the displacement frequency response properly, the measurements should be carried out in
a low noise environment, such as the anechoic chamber. Another possible solution is to increase the
signal length or apply several averages in the solution.
Regarding the frequency response deviation at the resonance frequency due to the ﬁnite element
approximation error, it should also be reduced in order to evaluate the multi-stage processing accuracy
properly. Although a higher sampling frequency reduces this error, the computational load also has to
be considered. A deeper analysis should be carried out in order to determine the minimum sampling

frequency required to achieve an acceptable accuracy at the resonance. Moreover, the application
of numerical methods might also be considered in order to reduce the sampling frequency without
obtaining such a deviation at the resonance. A numerical method for the Grünwald approximation is
proposed in [13].
Finally, although from ﬁgures 8.7, 8.8 and 8.9, the model seems to ﬁt quite well the measured frequency
responses, a better ﬁt might be found. It has to be considered that the model obtained from the
ﬁtting algorithm is quite dependant on diﬀerent factors of the ﬁtting algorithm, such as the initial
values, the constraints or the cost functions. Also, a modiﬁcation of the fractional order model could
be considered. Some of these modiﬁcations are proposed in [1], such as removing the mechanical
compliance C0 in order to avoid the displacement limitation at low frequencies. Also, adding a resistor
in parallel with the fractional order inductance, might produce a better prediction of the impedance
above the resonance frequency.

Chapter 9
Conclusions
Along the diﬀerent chapters of this report, an analysis of the multi-stage processing model is carried out
in order to achieve the goals proposed in the introduction chapter. The analysis is especially focused
on the anti-aliasing ﬁlter, the interpolation, and the downsampling of the multi-stage processing.
Based on the results found in the analysis, diﬀerent solutions and suggestions are proposed in order to
improve the eﬃciency and accuracy of the model.
In chapter 5, the anti-aliasing ﬁlter’s eﬀect on the multi-stage processing is evaluated. It was found
that the ﬁlter gain at the DC component of the ﬁlter has a relevant eﬀect in the error introduced by
the ﬁlter. A ﬁlter modiﬁcation is proposed in order to avoid the ﬁlter gain at the DC component.
From the results obtained with the modiﬁed ﬁlter, it was concluded that the model accuracy does
not depend on the ripple amplitude. The ﬁlter attenuation is also analyzed in order to observe its
eﬀect in the aliasing products reduction. Since the aliasing products are relatively low compared
with the signal level due to the multiplication by the weighting function, it is concluded that the
attenuation does not have a signiﬁcant error in the model accuracy. Therefore, due to the fact that the
error produced by the multi-stage processing is not aﬀected by the ripple amplitude nor the attenu-
ation of the anti-aliasing ﬁlter, a lower order ﬁlter can be used in order to reduce the computational load.
The eﬀect of the reconstruction on the multi-stage processing frequency response, and the improve-
ments when including a linear interpolation were analyzed in chapter 6. The linear interpolation
is substituted by a polynomial interpolation, since it is expected to improve the accuracy of the
model. However, it was found that there is not such improvement. Since the frequency components
produced by reconstruction are drastically decreased by the linear interpolation, they do not have
a signiﬁcant eﬀect in the error produced by the multi-stage processing. In this way, although the
polynomial interpolation produce a greater reduction the reconstruction products, it does not improve
the model accuracy. Moreover, the polynomial interpolation is also evaluated for higher downsampling
factors in the multi-stage processing, ﬁnding similar results in terms of accuracy. On the other hand,
the polynomial interpolation entails a great increase in the computational load. In this case, the
number of operations is doubled since the kind of polynomial interpolation is the cubic. Therefore, it
is concluded that in this case, a higher order interpolation is not convenient in the multi-stage processing.
In order to reduce the multi-stage processing computational load, the eﬀect of increasing the down-
sampling factor is evaluated in chapter 7. First of all, a modiﬁcation of the multi-stage processing is
proposed where the downsampling factor is doubled from 2 to 4. Although the number of operations
63
==PAGE:72==
CHAPTER 9. CONCLUSIONS
with the modiﬁed model is halved, the error with respect to the full solution is signiﬁcantly larger.
Thereby, another modiﬁcation is proposed in order to achieve a compromise in terms of accuracy
and eﬃciency, but also to increase the ﬂexibility of the multi-stage processing. In this case, factor 2
downsampling is applied in the ﬁrst stages, while factor 4 downsampling in the last ones. The results
from this model are in between the factor 2 and the factor 4 downsampling results. A higher or lower
accuracy and computational load is obtained depending on the number of stages selected from each kind.
Looking at the impulse response from the multi-stage processing, a deviation was found at the
beginning of each stage, especially in the higher factor downsampling cases. Based on the analysis of
these jumps, it is known that they are produced by the ﬁltering and downsampling process. Moreover,
it is veriﬁed that using a low order anti-alasing ﬁlter, the deviation is drastically reduced.
Finally, displacement measurements with a loudspeaker are carried out in order to verify the diﬀerent
multi-stage processing modiﬁcations. Therefore, a model was ﬁtted and the simulations from this
model were compared with the measurements. From this comparison, the same error was obtained for
the three downsampling factors in the multi-stage processing. In this case, it is concluded that the com-
putational load could be minimize without aﬀect the model accuracy. Nevertheless, the error between
the measurements and the fractional order model was fairly large, due to noise in the displacement
measurements and a deviation at the resonance frequency due to the ﬁnite element approximation error.
9.1
Future work
Based on the conclusions found in this project, some ideas are proposed in order to continue with the
improvement of the multi-stage processing algorithm. The main suggestions are listed below.
• A veriﬁcation of the model accuracy based on measurements with lower noise should be carried
out. Especially at low frequency, the error from the diﬀerent downsampling factors should be
determined in order to evaluate the improvement of these modiﬁcations.
• The error produced by the ﬁnite element approximation should be analyzed, in order to determine
the minimum sampling frequency required as a function of the fractional order model accuracy.
• The advantages in terms of accuracy and eﬃciency of the variable number of samples per stage
should be evaluated in order to obtain the optimal number of samples in each stage.
• The deviation produced by the jumps at the beginning of the stages should be avoided in order
to reduce the multi-stage processing error.
• The non-linear components of the loudspeaker should be included in the fractional order model
since the ﬁnal goal is the prediction of those non-linearities in order to compensate them.
• Once the non-linear model is implemented in the multi-stage processing, it should be included
in a feed forward compensation system.
