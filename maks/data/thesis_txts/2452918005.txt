Abstract
Digital microﬂuidics (DMF) is technique for manipulating droplets on a surface, using electro-
static control of the surface hydrophobicity. This enables advanced and accurate control of the
droplet, and is used as an alternative to traditional microﬂuidic systems, which typically use
pressure diﬀerentials to move ﬂuids around in a passive device with ﬁxed micro-channels. DMF
oﬀers the ability to create an endless number of microﬂuidic systems, by simply making changes
in a software environment.
Polymerase chain reaction (PCR) is a technique used to copy speciﬁc segments of DNA, and is a
very commonly used technique in many biomedical research ﬁelds, as a medical diagnostics tool,
and in forensic investigations. It relies on thermal cycling of a sample, in often big and expen-
sive equipment. Lab-on-chip (LOC), which is a concept that essentially attempts to down-size a
complete laboratory to the size of a single chip, can be combined with PCR to get PCR-on-chip,
with the goal of reducing the laboratory equipment requirements, by running the PCR process
on a single small chip. Merging DMF with PCR-on-chip has the potential to unite one of the
most used methods for DNA ampliﬁcation, with the reduced equipment requirements of LOC,
and the potential microﬂuidic system possibilities of DMF.
In this thesis project I investigated the feasibility of making an inexpensive PCR module for a
modular DMF platform, using standard printed circuit board (PCB) technology. In addition
to this I examined the potential of creating a simulation environment, which could be used to
predict the outcome of future hardware alterations.
After designing and building a functional PCR module, the feasibility of running the PCR-on-
chip was demonstrated successfully, by running PCR tests on the system. The PCR module
uses hardware and software architecture inherited from an existing DMF platform, which means
that a future integration into that platform should be relatively straightforward to realize.

1
Introduction
Polymerase chain reaction (PCR) is a method for copying deoxyribonucleic acid (DNA) se-
quences, and is a very used technique[1] for analysis in many biomedical ﬁelds[2], as part of
medical diagnostics[3], and in forensic investigations[4].
Lab-on-a-chip (LOC) is the idea of
miniaturizing an entire lab down to the size of a single small device[5], which typically performs
only a single task, but very eﬃciently. LOC devices are traditionally passive devices made using
either mechanical or lithographic micromachining. Digital microﬂuidics (DMF) is a type of LOC
system where droplets can be controlled individually, which changes the LOC from being a pas-
sive device, with only a single purpose, to an active device, with many possible user-controlled
conﬁgurations[6].
A DMF platform would be capable of oﬀering the advantages of a LOC system, but with the
added possibility of many diﬀerent biomedical applications instead of just one, as is usually
the case with LOC devices. One possible application for a DMF platform is a PCR module[7].
Combining PCR, with LOC and DMF, could potentially merge one of the most used DNA
ampliﬁcation techniques, with the reduced size and hardware requirement of LOC, and the
system ﬂexibility of DMF.
1.1
Project motivation
This project is part of a larger project named ”Magic Box”. The Magic Box project is a modular
DMF platform[8], capable of working with many diﬀerent types of biomedical processes, by
giving the user a simple graphical interface to allow speciﬁc and complicated droplet control.
Any desired task can then be deﬁned by the user, enabling a wide variety of point-of-care (POC)
biomedical functionality in a small device, or complicated experimental research setups, which
would normally require multiple expensive instruments and laboratory space.
One possible use for the DMF platform is to perform PCR, but such a PCR module must ﬁrst
be designed and tested, which is the motivation for this project.
1.2
Thesis goals
The main goal of this master thesis project is to build a PCR module for a DMF platform. Its
hardware must ﬁt into the already existing DMF platform, and have the ability to use droplet
manipulation. It should preferably be based on using printed circuit board (PCB) technology as
heating and sensing elements, as this is both a cheap solution and are easy to make, using the
very well established electronics industry. Furthermore it is desirable to have a design simulation
environment, which would enable testing new design ideas, before making the physical hardware
changes. Testing new hardware designs can take weeks, depending on the speed of delivery for
new PCBs, and having the ability to test the hardware changes could potentially reduce time
wasted on designing, ordering, and testing ineﬀective hardware changes.
The thesis goals are:
• Design and build a functional PCR module for the DMF platform.
• Make reliable simulation environment usable to test future design ideas.
• Perform a successful PCR using the PCR module.
1.3
Polymerase chain reaction on chip
PCR is a technique used to make copies of a DNA segment for use in research, forensic and
medical applications, where studying DNA can be crucial for certain tasks. The DNA segment
2 of 102
==PAGE:9==
copying is done in a machine speciﬁcally designed for this process. PCR is normally done in small
plastic vials, loaded into an industrial PCR machine, some of which can run hundreds of samples
simultaneously. These machines typically work by heating and cooling a large thermal mass,
using thermoelectric heating, which makes the process slow. PCR on chip, which is PCR running
on an LOC platform, has the potential to drastically reduce the thermal response time, by
essentially reducing the thermal mass of the system, to the thermal mass of the sample volume[9].
A traditional PCR process runs for ∼90 minutes, of which the actual biological processing time
is ∼60 minutes, and the rest is time loss due to thermal response time of the system. PCR on
chip has the potential to reduce the total processing time to 12 minutes[10]. Another beneﬁt of
reducing the size of the PCR system, is a lower sample volume requirement. This reduces the
material consumption cost, and allows for smaller sample volumes to be analyzed[11].
1.4
Digital microﬂuidics
Digital microﬂuidics is a technique used to manipulate droplets by surface tension. The system
uses electrowetting on dielectric (EWOD) to change the hydrophobicity of a surface coating,
which is used to control the movement of the droplet on the surface, illustrated in ﬁgure 1.1.
EWOD works by altering the wetting properties of a surface, by applying an electric ﬁeld to
that surface[12]. The DMF platform essentially changes the LOC device from a passive to an
active device, where the user can control the ”layout” of the chip, by simply making changes
in a software interface.
Having a reduced size could also make such a system portable[13].
DMF is in principle able to perform any microﬂuidic task, which could be achieved by using
traditional passive microﬂuidic devices, but also has the ability to alter the functionality of its
current setup, even while running, enabling a much greater range of possible test setups without
the need for replacing components, which in traditional microﬂuidic equipment would require
separate physical devices.
The Magic Box DMF platform diﬀers from competing systems, by being designed from the
beginning to be able to control generic DMF equipment, instead of being designed for a speciﬁc
task, or with a speciﬁc hardware layout.
The modularity of the Magic Box DMF platform
makes it behave more like a translator between the user, operating a basic computer, and the
hardware, which could potentially be any desired microﬂuidic biomedical application, such as
analysis instrumentation, or chemical and biological reaction instruments.

image is illustrated by the droplet (cyan) being moved from the upper left corner, to the lower
right.

1.5
Project background
The PCR module is part of a larger project called Magic Box, which is a modular DMF platform
intended to be capable of creating many diﬀerent microﬂuidic setups on a single hardware setup,
by allowing a user to create virtual microﬂuidic setups through software. The platform uses
EWOD to move sample droplets, mimicking a traditional micromachined passive microﬂuidic
device with closed channels, usually run by pressure diﬀerentials created by gravity, a pump,
centripetal force, or osmosis.
The beneﬁt of having a software-deﬁned microﬂuidic setup is that the ”geometry” of any setup
can be changed instantaneously at no extra cost, and without the need to wait for new hardware
to arrive, and be assembled and tested.
This project is directly based on work done by Ph.d Georgi Plamenov Tanev, who built a modular
re-conﬁgurable digital microﬂuidics platform [8]. It is also an extension of a previous project, by
the author of this thesis, on resistive heating and temperature sensing in PCB.

Fundamentals of PCR theory
This section introduces the basic concepts of the PCR method, which are later used as guide-
lines in the thermodynamic analysis and the prototype design process. The standard method
used to perform PCR is from the book PCR Technology - Principles and Applications for DNA
Ampliﬁcation by Henry A. Erlich[14] and Molecular Biology and Genomics, chapter 4 The Poly-
merase Chain Reaction, by Cornel M¨ulhardt and E.W. Beese[15], which describes the basic steps
required for performing PCR. The basic principles have been modiﬁed, where needed, to match
the speciﬁc PCR materials used for this project, but the overall process is the same as describe
in the books.
The main goal of this thesis project is to build a module, which can perform PCR, for an
existing DMF platform. PCR is a method for amplifying speciﬁc DNA sequences, using a DNA
polymerase enzyme and two nucleotide primers. The reaction involves repeated cycles of DNA
denaturation, primer annealing, and primer expansion by the polymerase enzyme. Depending
on the number of cycles, and the eﬃciency of the reaction, the target segment of DNA can be
copied millions, or even billions of times, during the PCR. The main goal of PCR is to increase
the amount of the target DNA, as the amount of target DNA determines how well, and by which
methods, it can be analyzed.
2.1
Materials required for PCR
The actual copying mechanism is provided by a DNA polymerase enzyme. In our experiments
we used an enzyme called Taq polymerase, which is very temperature stable and has high
enzymatic activity at ∼72 °C[14] [15]. The temperature stability of the enzyme is required, as
high temperature is needed for the denaturation step.
The Taq polymerase enzyme can only create DNA copies, if it has been given a starting location.
These are provided by PCR primers, which are short strings of single stranded DNA, known as
oligonucleotides. They are made of the same four nucleobases as DNA; cytosine C, guanine G,
adenine A, and thymine T. The primers simultaneously deﬁne the region of DNA to be copied
and allow the Taq polymerase enzyme to perform the copying process[14] [15]. The sample mix
contains a forward primer and a reverse primer; one for each of the two separated DNA strands.
The sample mix must also contain the raw materials for constructing the copied DNA segments,
which comes in the form of deoxynucleoside triphosphates (dNTPs), and the sample is mixed in
a buﬀer solution, to keep the polymerase stable and active[14] [15].
A standard PCR sample volume is 50 µL[14] [15] [16]. The sample volume is intended to be as
small as 1 µL in a ﬁnal DMF platform PCR module.
2.2
PCR temperature steps
The reaction requires cyclic heating of a solution of DNA to three speciﬁc temperatures and
durations, and repeating this for a desired number of cycles. Each completed cycle will double
the amount of target DNA (assuming 100% eﬃciency), until it has been copied millions, or
billions of times during the PCR.
The speciﬁc temperature and duration for each step is determined by the type of polymerase
enzyme, the melting temperature of the primers, and the length of the DNA targeted for copying.
There are three main temperature steps in the thermal cycling process, with an additional initial
step and ﬁnalizing step, as seen in the ﬂowchart in ﬁgure 2.1.

The initial denaturation step is required for some thermostable polymerase enzymes to activate,

The duration of the denature step during the cycling process, is considerably shorter at only 10

2.2.2

During the denaturation step, the temperature is raised to 90-98°C for up to 30 seconds[14] [15],
depending on the type of polymerase enzyme used for processing. This breaks the hydrogen
bonds between the two strands of the DNA, creating two separate single-stranded nucleotide
molecules, which will act as templates for the new DNA copies, illustrated in ﬁgure 2.2.

The temperature for this step must be high enough to denature the DNA, but not so high that it
negatively aﬀects the Taq polymerase. Setting the temperature too high, will reduce the half-life
of the enzyme from 2 hours at 92.5°C to 9 minutes at 97.5°C[18].
The duration for this step must be long enough to separate the DNA completely, but not so long
that it negatively aﬀects the Taq polymerase. Setting the denaturation duration too long, may
denature the polymerase enzyme as well, reducing the activity during the extension step[14] [15].
We used a denature temperature of 94°C, with a duration of 10 seconds, in our PCR tests, as
this was the denature processing parameters speciﬁed by the manufacturer of the polymerase
mix used for the PCR testing[17].
In order to keep the activity of the polymerase enzyme high, while being able to run the process
for multiple hours, we must simultaneously be able to ensure that the sample temperature is

speciﬁc target locations, illustrated in ﬁgure 2.4. The primers serve as starting locations for the
Taq polymerase during the extension step, which is the next step in the cycle.

The ﬁnal extension step ensures that any remaining partially copied single strands are completed

2.3
Repeated cycling
The 3 main steps of the reaction: denaturation, annealing, and extension, are repeated 20-40
times[14] [15]. Each completed cycle doubles the amount of target DNA (given 100% eﬃciency),
which results in 220−240 copies of the original DNA segment, due to the exponential accumulation
of target DNA, which is visualized in ﬁgure 2.7.
The exponential accumulation response is only valid as long as no process-limiting factors are
present.
In reality the accumulation response follows a logistic response (an S-curve) if the
process is run for long enough or at incorrect temperatures, due to limiting factors like polymerase
activity decrease and solution depletion.

2.4
PCR requirements
In order to have a successful PCR we need to ensure that the sample temperature is within ±2°C
of the setpoint temperatures for each step. If the denature temperature is too low, the DNA
will not denature, and if the temperature is too high the polymerase enzyme deactivates, as it
also denatures. If the anneal temperature is too low, the primers may attach to incorrect sites,
making erroneous copies, and if the temperature is too high, the primers will not attach at all.
If the extend temperature is too low, or too high, the polymerase activity decreases, making the
process ineﬃcient.

thermally in time, with a stationary sample on top of it.

4
Thermodynamics of droplet heating
This section gives a qualitative description of the basic thermodynamics of droplet heating, which
are later used as guidelines in the prototype design process.
One of the main requirements
for performing high quality PCR, is to have good temperature control of the sample during
processing.
4.1
Ideal droplet heating
The ideal droplet heating would be a step-like response, going instantly from temperature A
to temperature B. This is of course not possible in the real world, where objects have thermal
inertia based on thermal conductivity and heat capacity.
An example of a heating and cooling proﬁle is illustrated in ﬁgure 4.1, showing an idealized
perfect heat source, and a more realistic temperature proﬁle of a typical heating system. The
realistic heat source has thermal inertia, which makes it have a gradual temperature increase
(1), and when the setpoint temperature is reached, any thermal distance between the heater and
the temperature sensor may make the system overshoot (2) and possibly also oscillate around
the setpoint (3).
This is because there is a thermal distance between the heater and the sensor, making the
sensor temperature lag behind the heater temperature. By the time the sensor registers that the
setpoint temperature has been reached, the heaters are already at a higher temperature. This
eﬀect is also present during the subsequent cool-down, which means that the sensor registers
the setpoint temperature at a point in time, where the heaters have already dropped below the
setpoint, which makes the system oscillate around the setpoint temperature.
When the heating cycle is stopped, the system will slowly drop to ambient temperature (4). As
I am not using thermoelectric cooling, the only way I can reduce the cool-down time, is with
forced air cooling. This can easily be realized, by adding a cooling fan to the setup. The forced
air cooling will help reduce the cool-down times, but can never cool below ambient temperature.

Thermal camera measurements
Thermal imaging was used to obtain 2D temperature measurements of the board and heat
zones. The camera captures infrared radiation to image the temperature of a surface, which
produces very nice looking images, but is notoriously diﬃcult to use properly, especially on
metallic surfaces. The thermal camera used was a Satir E8, seen in ﬁgure 5.5.
Figure 5.5: The thermal camera used to image the DMF module board.
If used correctly, the absolute accuracy of the camera, stated by the manufacturer, is ±2°C, while
the relative accuracy is ±0.1°C. These accuracies assume completely perfect measurement setup.
The relative accuracy is how good the camera is to distinguish relative temperature diﬀerences
in a single image, and the absolute accuracy is how good the camera is at measuring the correct
absolute temperature.
The main problem with using a thermal camera is the diﬀerence in emissivity of diﬀerent materi-
als. This property is a materials ability to emit thermal radiation, and can have a value between
0 (a perfect thermal reﬂector) to 1 (a perfect thermal emitter). Gold, which is the material the
electrodes on the DMF module PCB are coated with, has an emissivity of 0.03 - 0.5 depending
on the level of polish. The white solder mask on the PCB on the other hand has an emissivity
of ∼0.95.
The connection between emissivity and the measured temperature comes from Stefan-Boltzmanns
law, seen in equation (5.3), which derives the radiation intensity Q, using the material emissivity
ε, the Stefan–Boltzmann constant σ, and the absolute temperature T:
Q = εσT 4
(5.3)
The consequence of the diﬀerences in emissivity is, that it is impossible to image the entire
board correctly; setting the emissivity setting to match the solder mask, causes the electrodes
to appear almost unheated, as seen in ﬁgure 5.6, while setting the emissivity setting to match
the gold coating, causes the solder mask to appear as if it is hundreds of degrees warm, as seen
in ﬁgure 5.7. The heated strip, visible in the center of the board, is one of the PCB heaters.

The thermal camera has a 0.1°C relative sensitivity, but the absolute accuracy is ±2°C, and
this assumes that the emissivity settings are perfectly adjusted, and that the entire imaged

5.6
PID control scheme
The temperature on the heat zones, both on the board and in the lid, are automatically adjusted
by a PID controller, which is a type of feedback control method.
It is used to apply an amount of power to the heaters which, when tuned properly, produces a
system with fast thermal response time, while keeping thermal overshoot to a minimum. It is
also capable of reducing the inﬂuence of thermal transients, making the system more stable.
It is a control algorithm, which works by continuously calculating an error value, in this case
the distance between the actual temperature and the setpoint temperature, and then applying
an appropriate output power, based on its algorithm and tuning parameters.
The PID algorithm consists of three diﬀerent terms: a proportional term, an integral term, and
a derivative term, and a general expression for this is shown in equation (5.4). The output
power, u(t), is the sum of the error value times a proportional constant, plus the integral of the
error value times an integral constant, plus the derivative of the error value times a derivative
constant.

The PID control is an advanced control scheme, and requires careful tuning of the three constants
KP , KI, KD. Incorrect settings may cause the system to never reach setpoint, or cause the system
to oscillate. An example of the thermal response of a heating system is shown in ﬁgure 5.18,
which shows the eﬀect of having two poorly tuned PID controls, and one well tuned. These

The proportional term in equation (5.4) adds a value proportional to the diﬀerence between
the current temperature and the setpoint temperature. This term gets smaller as the measured
temperature approaches the setpoint temperature. If this setting is too low, the temperature
may never reach the setpoint, but if it is too high, the system may overshoot and oscillate around

the setpoint. Having a high proportional term also has the eﬀect of making the system more
sensitive to thermal perturbations, which decreases the stability.
The integral term in equation (5.4) adds a value, which is the sum of the accumulated error since
the algorithm started - the integral of the thermal curve, relative to the setpoint temperature.
This term reduces thermal response times and increases the systems ability to reach the setpoint
temperature. If this setting is too low, the system may never reach the setpoint, but if it is too
high, the system may overshoot and oscillate around the setpoint. Having a high integral term
may increase the time it takes to settle at the setpoint.
The derivative term in equation (5.4) adds a value, which is based on how fast the error value
changes. A powerful, but brief, thermal perturbation could create a high response from the
system, which is not always desired. Depending on the system requirements, this term can be
used to either attenuate or amplify the thermal response to ambient perturbations. This term
helps in reducing overshoot and the time it takes for the system to settle at the setpoint, but
may cause instability if set too high.

Thermodynamic simulations in COMSOL
This section describes how COMSOL was used to make thermodynamic simulations of the design,
which could potentially reduce resource costs and development time for future design changes,
by oﬀering the ability to test design changes in simulation, before making hardware changes.
COMSOL is a simulation environment, which uses the ﬁnite element method to solve physics
problems. Deﬁning an equation which can model how a large, or complex, structure behaves,
can be very diﬃcult. The ﬁnite element method divides large geometries into many smaller
parts, called ﬁnite elements, which are much easier to calculate. The results from calculating
the smaller ﬁnite elements are then combined, and the combined calculations is used to model
the larger, or more complex, geometry. A simulation using the ﬁnite element method can easily
become very time consuming, or require very powerful computing power, so it is important to
keep the simulated system as simple as possible.
In this thesis project, COMSOL was used to simulate thermal transport in solids and ﬂuids, as
well as Joule heating of resistive elements.
The section begins with a simulation of a much simpler design, which is used to compare how well
diﬀerent simulation methods match real world measurements. The rest of the section describes
simulations of the PCR heater module PCB, speciﬁcally about the heat zone temperature uni-
formity, and the required minimum distance needed between thermally separated heat zones. In
the last part of the section, I simulated the module PCB with samples and an oil-ﬁlled reaction
chamber.
The main reason for simulating a diﬀerent design ﬁrst, is that the size and geometries of the
DMF module PCB are too resource demanding to perform in 3D, while a smaller and simpler
design is not. The simple design is used to compare the resource-demanding, but more realistic,
3D physics simulations, with less demanding 2D simulations on the DMF module PCB.
6.1
Simulation physics and study types
In order to simulate thermal transport in the PCR module, I used ”Heat transfer in solids and
ﬂuids”, and to simulate the resistive heaters, I used ”Electromagnetic heating - Joule heating”.
The simulations can be run in either stationary mode, which is a steady state simulation that
assumes the system is in equilibrium, or in time-dependent mode, which simulates the system
evolution over time. The stationary simulations are by far the least resource demanding of the
two.
The ”Heat transfer in solids and ﬂuids” physics must be used in every simulation. The ”Joule
heating” physics is only required for those simulations in which I simulated the resistive heaters,
by driving a current through them. For simulations without ”Joule heating” physics, the heaters
are simulated by applying a ﬁxed temperature to the heater geometry.
Running the two physics simultaneously requires more resources, as they will have to run as
coupled multi-physics simulations, however running with ﬁxed temperature geometries generates
an unrealistic heat proﬁle.

mode. The 2D ﬁxed temperature time-dependent simulations, were a bit too optimistic, and if

would either need to be further optimized, or alternatively they would need to be run as 3D Joule
heating simulations instead, which would give a better, although a bit conservative, estimations
of the heating times of the heat zones.

6.4
Simulations on the DMF module PCB
Running the more accurate 3D Joule heating simulations on the entire module PCB turned out
to be too resource requiring to be performed on a generic oﬃce computer.
The geometry of the PCB is simultaneously large and contains many small features. This means
that the number of ﬁnite elements needed for the simulations increases the memory requirement
and processing time beyond the capability of standard generic oﬃce computer performance.
3D simulations were only possible after reducing and simplifying the PCB geometries to the point
were they could equally well have been replaced by regular 2D simulations. Instead of trying to
manipulate the 3D geometry to the point where it no longer were a good representation of the
actual board geometry, the 2D simulations were used instead.
The steady state 2D ﬁxed temperature simulations oﬀer a thermal estimate which can be con-
sidered good enough, although a bit on the conservative side. I do not recommend using the
time-dependent 2D ﬁxed temperature simulations without further simulation optimization.

required.

PCR heater design and characteristics
This section describes the PCR module design. The design uses considerations from basic PCR
theory, requirements from the DMF platform, thermodynamics of multilayered structures, and
results from COMSOL simulations, to construct a functional PCR module for the DMF platform.
7.1
Initial design considerations
From the basic PCR theory I deﬁned that the temperature must be accurate within ±2°C. From
the thermodynamics section I concluded that a sandwich construction, with dual heating, is
necessary for the system to work properly. From the DMF platform itself I am required to
use the DMF module PCB, that the lid should be transparent, and that droplet control is not
available for this thesis project. From the simulations I know that the spacing between individual
heat zones must be at least ﬁve columns of electrodes. This gives us a ﬁnal list of initial design
considerations:
• The temperature of the entire sample volume must not deviate more than ±2°C from the
setpoint temperature.
• The module must be heated from bottom and top.
• The bottom part of the heater module, needs to be able to allow for droplet movement
control by the DMF platform. This means that I have to use the DMF module PCB as
the bottom heater.
• The lid needs to be transparent, in order to be able to observe the droplets. This means
that I cannot build heaters directly into the lid, but must use another method to heat the
lid.
• Since the droplet control is not used in this project, I can reduce the size of the chamber
o-ring, so it only contains the sample itself. The oil immersion can be left out of the initial
test phase, as it is only required for a reaction chamber with moving droplets.
• the lack of droplet control means that I must run the board in time-domain mode, instead

9.1
First iteration: sample temperature too low
The ﬁrst hardware iteration did not heat the sample enough. At this point the heater was only
using the bottom heater, but due to a very fast thermal response of the module PCB heater,
and the small sample volume of 50 µL, we assumed that any thermal gradient would be small
enough for the system to still function satisfactory. This assumption made us begin PCR testing
before testing the actual sample temperature during processing.
We also found the sample loading for this design iteration to be a bit cumbersome.
The suggested solution was to increase the temperature on the module PCB heater, until the
top of the sample was at setpoint temperature, and to make a lid which could help facilitate the
sample loading.
9.2
Second iteration: air leaks in lid
In order to easier load the system for testing, connectors were mounted in the lid, seen in ﬁgure
9.2. These connectors caused air to leak into the system during testing, which caused the sample
to violently disperse in the reaction chamber. This was probably made worse by the increase in
the heater temperature, which caused the sample to shock boil, as the heater temperature was
now running at ∼110°C.
69 of 102
==PAGE:76==
Figure 9.2: The second hardware iteration used increased heater temperature, and lid con-
nectors for easy reaction chamber loading. Both of these conditions would turn out to be
problematic.
The lid material at this point was poly(methyl methacrylate) (PMMA), which is a fairly brittle
polymer. The machining required to make the holes for temperature sensors and connectors,
caused microscopic fractures in the lid, which would increase the air leaking. The fracturing
were accelerated by the process, probably due to thermal expansion and contraction, as well as
ﬂuid interaction.
The suggested solution was to decrease the complexity of the lid, by removing the connectors,
and to switch lid material from PMMA to PC.
9.3
Third iteration: sample temperature too high
After making a new simpler lid, and using PC instead of PMMA, we resumed the PCR testing,
only to ﬁnd out that the increased heater temperature made the Taq polymerase deactivate
completely.
The suggested solution was to add the second heater, which could heat the sample from the top,
eliminating the need for the heater to work above setpoint temperature.
9.4
Fourth iteration: material incompatibility
After ﬁnishing the fourth test, which thermally, mechanically, and electrically completed without
any problems, but still ended up unsuccessful, we noticed that the sample had completely lost
its color. As a result we decided to test the materials used for building the PCR module.
The materials in direct contact with the chamber were the polyimide ﬁlm, the immersion oil,
the NBR o-ring, the PC lid, the lid temperature sensor, and the epoxy used to mount the sensor
in the lid. We had also added the top heater, which was a high intensity halogen lamp.
70 of 102
==PAGE:77==
At this point we did not know which of these materials could have an inﬂuence on the PCR,
so we set up a test to see if any of the materials would interact with the sample, using small
pieces of each material and a standard PCR machine. Each material was tested individually, but
also in a combination of all materials, and they were tested with and without the high intensity
lamp, to best mimic the PCR module setup.
The results from the tests showed very clearly, that we had a materials problem; the epoxy and
NBR completely inhibited the reaction, and the PT1000 partially inhibited the reaction, which
can be seen in the gel electrophoresis analysis in ﬁgure 9.4, where the lack of bands indicate an
unsuccessful reaction for these two materials, and the low intensity of the PT1000 indicates a
low activity reaction.
The epoxy also had the eﬀect of interfering with the dye in the sample mix, making it go from
green to transparent, as seen in ﬁgure 9.3, where test#3 and test#6, the ones containing epoxy,
have gone completely transparent.
The gel electrophoresis analysis shows a small diﬀerence in the intensity of the lines, with and
without the added light source, but in favor of the light. This diﬀerence is probably just an
artefact of the analysis, but could potentially mean that the light has a beneﬁcial inﬂuence on
the reaction.

Four failed design iterations preceded the ﬁnal version. The failed designs led to discovering that
the system is unable to heat the sample eﬃciently with only a single heater, that overheating the
sample is a critical error, and that air leaks in microﬂuidic systems, especially heated systems,
can be detrimental to the process.
Furthermore we discovered that it is vital to consider material choices, both regarding mechanical
properties, but also when it comes to chemical interactions.

10
Discussion
In this section I will discuss some issues encountered during the project, as well as design upgrade
ideas and future outlook.
10.1
PCR in Time-domain vs.space-domain mode
We managed to perform a successful PCR test using the time-domain mode, but running the
system in this mode drastically increased the processing time, because the system had to heat
and cool a relatively large thermal mass, during each PCR cycle. The total bio-processing time,
which is the sum of all the time spent in each step is 56 minutes, but the total processing time
for the PCR test was 154 minutes, giving a total time lost due to thermal cycling of 98 minutes.
A commercial PCR machine has a thermal cycling time loss of ∼10-30 minutes, depending on
machine and settings, and experimental PCR-on-chip has demonstrated provessing times down
to 12 minutes[10].
Enabling the DMF droplet manipulation and running the PCR module in space-domain mode
instead, has the potential to reduce the thermal time loss considerably, and possibly also the
bio-processing time.
10.2
Heater characteristics
The heater uniformity was only acceptable in the center of the heat zones. In a ﬁnal product,
when using the space-domain mode, this would not be a practical problem, as the droplet heating
can be restricted to only using the area of the heat zones with acceptable uniformity.
Adding internal copper planes, below each heat zone, would increase the uniformity. Future
designs are recommended to add this feature.
10.3
Heat zone layout
The large heat zones could potentially be used for heating multiple droplets simultaneously, if the
uniformity is improved, but the small heat zones would probably be suﬃcient for single-droplet
processing. The large heat zones use more PCB area than the small ones, which makes the need
for either type of heat zone size, a question about having either many small heat zones, or fewer
large heat zones.
10.4
Simulation tool as part of future design changes
The simulation tool had a high degree of agreement with measurements of the physical device.
This indicates that future design ideas could be modelled, before making the actual hardware
change. Hardware changes often come with a waiting time, both in terms of designing, fabricating
and validating but also in terms of assembling new hardware into a functional system, before
any test can be performed.
It is of course not possible to completely eliminate hardware changes and testing, but the simu-
lation tool could eliminate time loss due to design changes which, after testing, proves ineﬃcient.
10.5
Heated lid
The requirement for a heated lid is most likely unavoidable. The lid used during this project
was made of PC, which is a relatively stable polymer, but not suitable for use with the DMF
73 of 102
==PAGE:80==
platform. The droplet manipulation requires having a potential between the surface electrodes
on the module PCB, and the bottom of the lid, which means that the bottom of the lid must
be electrically conductive. Having the top heaters and sensors built into the lid, which would
greatly simplify the construction, also requires electrically conductive properties.
Instead it is suggested to use indium-tin-oxide (ITO) coated glass.
ITO is both electrically
conductive, and optically transparent. This makes it possible to have a transparent lid, which
can oﬀer the electrical properties needed for the lid.
10.6
Next step
Test results of the PCR module testing shows that the concept is feasible, and enabling the
space-domain mode is a natural next step, which could potentially make the PCR modules
processing time comparable with commercially available machines. This requires coatings which
are still in the testing phase.
The product from the PCR test was only tested quantitatively, and another natural step would
be to test the contents of the product, to see if it matches the expected result.
10.7
Future outlook
Combining PCR with lab-on-chip and digitalmicroﬂuidics, to create DMF PCR-on-chip, has
great future potential. The ability to perform one of the most widely used techniques for DNA
ampliﬁcation with a system capable of a near-endless setup conﬁgurations, is exactly the type
of technological evolution which leads to discoveries never thought of before, because it allows
for an almost cost-free exploration of oour world; suddenly it becomes easy and cheap to thread
new paths, and try strange things, which would traditionally not be explored due to high costs
of money or time.
The DMF system oﬀers great ﬂexibility in its modular setup, as well as portability, which also
has the potential to make it possible to have entire laboratories in a very small, even mobile,
space. Having the ability to perform PCR on the spot, could open the door to completely new
ways of medical diagnostics, perhaps one day even allowing on-site genotyping, followed directly
by gene-speceﬁc drug synthesizes,and administration.

Conclusion
All of the stated goals were achieved. The ﬁrst thesis goal was to design and build a functional
PCR module for a digital microﬂuidic platform, using hardware from an existing DMF platform,
which would enable droplet manipulation by eletrowetting on dielectric. The second thesis goal
was to make a reliable simulation tool, which could be used for testing future design ideas before
making and testing new hardware. The ﬁnal thesis goal was to test the system, by performing
a successful PCR.
During the design and test phase of the project, it was discovered that a dual heating system
was necessary for successful operation, and that the heat zone temperature uniformity could be
drastically improved by adding a copper plane to the heat zone area. Furthermore it is suggested
to use an ITO glass as lid, as this would drastically reduce the system complexity.
It was also discovered that the materials choice for the PCR module, can be a critical issue, as
incorrect materials can inﬂuence, or completely inhibit, the reaction.
The system could not be tested using the droplet manipulation, which has the potential to
drastically reduce the processing times[10], but it was still possible to perform a complete PCR
with the system, using stationary droplets and thermal cycling of the system. A natural next
step for the Magic Box project, is to complete the surface coatings necessary for space-domain
operation.
The PCR module was thermally tested with three heat zones running simultaneously, and the
conclusion of that test showed that the thermal separation, between the individual heat zones,
is good enough to run multiple heat zones together. Proper testing of this setup should be done,
with moving droplet, as soon as that becomes possible.
The simulation tool turned out to have a high degree of agreement with measurements, and it
could potentially be a powerful tool for testing new design ideas, before making actual hard-
ware changes. The tool could be optimized further by acquiring better hardware, enabling the
possibility of running 3D simulations instead of 2D simulations, and further ﬁne-tuning of the
parameters could give the time-dependent simulations better accuracy.

Thesis project plan
The following is a copy of the thesis project plan, made in the beginning of the project, and then
updated with the auto-evaluation section at the project ending.
Thesis objective
Polymerase chain reaction (PCR) is a technique used to make copies of a DNA segment for use
in research, forensic and medical applications, where studying DNA can be crucial for certain
tasks. The DNA segment copying is done in a speciﬁc piece of equipment designed for that
purpose.
Conventional PCR equipment consists of stationary instruments, with sizes ranging from ordi-
nary tabletop printers to big oﬃce copying machines. They use a plastic tube to hold the PCR
mixture, and a large heating/cooling chamber to handle the tube and the PCR processing.
The main goal of this master project is to build a PCR module for a digital microﬂuidic (DMF)
platform, with smaller footprint than conventional sized equipment, as well as droplet based
sample control using cheap printed circuit board (PCB) technology as heating and sensing ele-
ments.
Learning objectives
• Understanding, building and programming embedded control systems.
• Understanding the fundamentals of PCR theory.
• Understanding thermal transfer mechanisms in multilayer structures.
