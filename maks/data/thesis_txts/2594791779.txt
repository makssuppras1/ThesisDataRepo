This paper proposes a proved and optimised implementation for an algorithm (written by Valk and Jantzen) to

Introduction and Motivations
Petri nets provide an elegant and useful mathematical formalism for modelling concurrent systems and their
behaviors. In a Petri net, the occurence of an event or an action of the system is represented by the ﬁring of
a transition and the movement of tokens from one place to another. There are many points of interest when
studying a Petri net, such as computing the reachability set, which is the set of all reachable marking from
the possible transitions to be ﬁred. There is also the coverability set, which is the downward closure of the
reachability set. More precisely, a particular property of Petri nets and Vector Additions Systems (which have
been proved to be equivalent by Hack in [12]) is the so called property of monotonicity. It is the basis of many
decisions procedures in Petri nets theory. For instance, in a Petri net, if a transitions can be ﬁred for a marking
m, it is also ﬁrable for any marking m’ greater or equal (componentwise) to m. This can therefore lead to
unboundedness in the Petri net : A marking is unbounded if for any integer n there is a place p and a ﬁring
sequence t, such that ﬁring t in m brings more than n tokens to p. Consequently, unboundedness is a monotone
property of markings, and we can deﬁne a set of unbounded markings, K∞. The point of this paper is to compute
a ﬁnite and minimal basis of such a set K∞, but also of the set of unbounded markings on a precise place, or
on simultaneously unbounded places. This can be done using the Valk and Jantzen algorithm, proposed in the
paper [16] in 1985.
In this paper, we therefore proposed an implementation for this particular algorithm, which aims at ﬁnding the
minimal ﬁnite basis of set of vectors of integers.
In section 2, we presend the notions used and deﬁnitions needed in the paper. Although, the Valk and Jantzen
algorithm has been designed for application to every upward closed set, we will restrict ourself to an imple-
mentation on Petri nets and Vector Additions Systems (which are upward closed set). In section 3, we make
a review of the termination, correctness and completude proofs of the algorithm before giving an optimised
version of it in section 4. After insuring the correctness of the optimisation we added to the original algorithm,
we propose a framework for an actual use in section 5 and solve all the problems related to the implementation
in section 6: the handling of the inﬁnite, the problem of veriﬁcation of the predicate Pk(m) and especially when
applied to ω-markings. To do so, we used the tool MinCover, a tool presented and implemented in [6] and [7] by
Finkel, Haddad and Khmelnitsky. The MinCover tool is an algorithm to implement and compute the minimal
coverability set for a precise Petri net. In our study, it is useful since we can use it to verify the properties of
some particular sets of Petri nets, like for instance the sets of unbounded markings or the set of markings with
a speciﬁc unbounded place, that we are going to test through this paper. We also give results about complexity
of both the Valk and Jantzen algorithm and the optimised one that we proposed.

Upward and Downward closed Sets
Let us deﬁne the upper closure (in Np) denoted by ↑K of a set K ⊆Np by :
↑K = {x ∈Np
|
∃y ∈K ∧y ≤x}
and we say that a set K ⊆Np is upward closed iﬀK =↑K.
Deﬁnition 2. A basis of an upward closed set K is a set B ⊆K such that K =↑B.
A basis B of an upward closed K is ﬁnite if B is ﬁnite and it is minimal if B is minimal (for inclusion) among
all other basis.
In another way, if K is an upward closed set, then its elements verify the following implication : ∀m ∈K and
∀m’ ∈Np, m’ ≥m =⇒m’ ∈K.
We will use similar deﬁnitions of downward closure, downward closed sets except that we will consider downward
closed subsets in Np
ω. Moreover, the region of m ∈Np
ω, denoted reg(m), is deﬁned as the downward closure of
m excluding the vectors containing ω: formally, reg(m) =↓m ∩Np.
Theorem 1. Each upward closed set K ⊆Np has an unique minimal basis.
Proof. For the seek of the proof, let’s suppose that there exists two minimal basis of K ⊆Np, an upward closed
set. We denote them b1(K) and b2(K). In such a case, we either have b1(K) ≥b2(K) or b1(K) ≤b2(K) which
then leads to choosing the minimal one for our b(K). However, there is also a possibility for b1(K) and b2(K)
to be incomparable. If so, using the minV function on vectors will create a new vector b(K) which will still be
a basis of K (because it is the concatenation of two basis of K), and which will be minimal for K.
Notation 1. This unique minimal basis of a set K ⊆Np is written b(K).
Remark 4. : To be precise and clear, we let the reader know that in [16], b(K) is called res(K).
2.2
Petri Nets
A Petri Net, N, is a 5-tuple deﬁned by :
N = (P, T, B, F, m0)
 where :
P = p1, p2, ..., pp is a ﬁnite set of places,
T = t1, t2, ..., tt is a ﬁnite set of transitions,
B : P ∗T →N is the backwards incidence function,
F : P ∗T →N is the forwards incidence function,
m0 ∈Np is the initial marking.
A Petri net is said to be marked if it is associated to an initial marking m0 not empty. When the Petri Net
is marked, it can also be refered as N = (R, m0) with R being a 4-tuple gathering all the other parts of the
5-tuple.
To evaluate the global state of a Petri net at every transition, one can look at the number of tokens in each place
of the net. Whenever a transition is ﬁred, the number of tokens in each place will change, since the ﬁring of a

2.3
Vector Addition Systems
As described in [18], Vector Addition Systems (VAS) were introduced by Karp and Miller in [14]. They were
later be shown as equivalent to Petri nets (PNs) by Hack in [12].
Deﬁnition 6. An n-dimensionnal VAS is a pair G = (x,W) where :
 x ∈Np is called the start point (or start vector)
 W is a ﬁnite set of vectors (called addition vectors) in Zp
For VAS, reachability set and coverability set can be deﬁned in the same way as for PN.
Moreover, as deﬁned in [13], one can deﬁned n-dimensional Vector Addition Systems with States (VASS) as
being a VAS (x, W) along with a ﬁnite set T of transitions of the form p →(q, v) where q and p are states and
v is in W. The meaning of such a transition is that it can be applied at point y in the state p and yields the
point y + v in state q (provided that y + v ≥0). The previous VASS is speciﬁed by G = (x, W, T, p0) with p0
being the initial state (also called the starting state).
Remark 10. A VAS can also be seen as a single state VASS.
Since the result and the algorithm of Valk and Jantzen in [16] are eﬃcient for VAS, and since VAS are equivalent to
PNs, we will apply the result of this article to PNs in order to compute the ﬁnite minimal basis of some interesting
and particular sets of a Petri Net (Unbounded markings, Unbounded places, Simultaneously Unbounded etc)
that will be described later on.

The Algorithm of Valk and Jantzen
In their paper [16], published in 1985, Valk and Jantzen proposed an algorithm in order to compute the ﬁnite
minimal basis b(K) of an upward closed set K ⊆Np.
However, some conditions need to be fulﬁll in order for the algorithm to be decidable. Indeed, the set K of
which the ﬁnite minimal basis will be computed should ﬁrst be an upward closed set and should veriﬁes the RES
property (as called by Valk and Jantzen in [16]). The property RES is deﬁned by the existence of an element in
the intersection of the set K and the region of an element m of Np. Further on this report, their algorithm will
be explained, proved and improved. (To make this paper easier to read, the algorithm of [16] will be referred as
the VJ algorithm).
Now we deﬁned PK(m) as the predicate (reg(m) ∩K ̸= ∅) which will be tested in the algorithm in order to
insure that the RES property holds for the current iteration.

3.1
Algorithm’s explanation
The VJ algorithm computes the ﬁnite minimal basis b(K) of an upward closed set K ⊆Np. It can be decomposed
into three major parts :
 Find a new value in K from a greater set Mi (at ﬁrst iteration M0 = Np, then Mi is a set that still contains
elements of K)
 Find the minimum value of the vector m for which it is still contained in K, and then add it to Ri.
 Truncate the set Mi (that still contains K at each iteration) in order to have a strictly new element found
by the algorithm at every new iteration.
More precisely, the operations done by the algorithm are :

the product of the cardinal of both set to be minimized together. As the dimension of the space is p, M ′ is of
cardinal p, and thus the complexity is bounded by the maximal repetition of the algorithm, which is p times (as
b(k) is also of cardinal p). So the complexity is bounded by pp.

We present some ideas, solutions and thoughts in this section.
