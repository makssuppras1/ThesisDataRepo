Abstract
In this thesis, we use dynamic epistemic logic to formalise single-agent learning
in both fully and partially observable domains. These domains are unknown
to the agent, as the agent has never interacted with them before. The agent
can learn about the underlying dependencies between objects of a domain by
executing actions and observing the eﬀects.
We present a learning function
and two learning algorithms solving the learning problem under three diﬀerent
restrictions. The work is a continuation of the work by Bolander et al. [7],
Bolander and Gierasimczuk [6], inspired by Baltag and van Benthem [2].

Chapter 1
Introduction
Learning is something all cognitive agents instinctively do from the moment
they are ‘born’. A part of learning is causal learning where the agent strives to
learn cause-eﬀect relations of their environment. Bonawitz and Muentener [11]
argue that children as young as the age of six months are able to identify some
categories of cause-eﬀect relations. It therefore seems natural to formalise the
learning of cause-eﬀect relations in order to implement this ability in artiﬁcial
agents and allow them to gain deeper understanding of their environment. As
causation is a term with numerous interpretations, Pearl and Mackenxie [22]
divide causality into three levels, on the ‘Ladder of Causality’; 1) prediction, 2)
intervention, and 3) counterfactuals. The higher up on the ladder the type of
causation is placed, the more causal questions can be answered by it. We have
chosen to focus on the second level of Pearl’s Ladder, that is intervention.
In this thesis, we will consider learning as a goal, where the agent should be
able to learn as much as possible in a given situation. The type of learning we
study is called domain learning, where an agent will attempt to learn about an
unknown environment. We will focus on single-agent learning scenarios, where
we will use formal learning theory and Dynamic Epistemic Logic (DEL) to
formalise learning. Formal learning theory and some logics of the DEL family,
are developed from the wish to understand certain aspects of natural language.
Formal learning theory with the purpose of understanding natural language
acquisition [8] and DEL with the purpose of understanding communication in

In previous work by Bolander and Gierasimczuk [6], and
Bolander et al. [7] functions and algorithms that learn the structure of action
models are deﬁned. These functions concern single actions and learn the eﬀects
of sequences of actions, but they do not provide insights into the knowledge
of underlying causes of eﬀects of actions. If the learner instead examined the
causal relations a domain, it could potentially gain more knowledge of domain
structure eﬃciently, and from this build a more complete representation of the
domain.
1.1
Goal of the thesis
The goal of this thesis is to develop a causal learning method enabling the
agent to learn the underlying causalities of an unknown domain.
This also
allows the agent to infer the dynamic structures of the domain. The aim is to
create learning functions with the property of ﬁnite identiﬁability, using update
learning (see chapter 2) where an epistemic model is updated step-wise.
The structure of this thesis is as follows. First, in chapter 2 we introduce the
basic notions and the formalised learning of action models. Next, in chapter 3,
we will examine interpretations of causality as dependence and deﬁne the de-
pendence models; the inference thereof will be the learning goal of our learning
agents. From here, we will deﬁne two dependence learning methods in fully
observable domains. In chapter 4, we will extend the analysis to cover partially
observable domains and we will discuss the consequences of the partial observ-
ability restriction. Furthermore, we will investigate algorithms for solving the
problem of generating the observation traces needed as input for learning meth-
ods in the partially observable domain. After this, in chapter 5, we present the
dependence learning algorithm for partially observable domains. In the ﬁnal
chapter 6, the ﬁndings of this thesis are summarised, and some open problems
for further work is discussed.

Chapter 2
Learning action models
In this chapter, the basic notation and the type of action learning we study are
deﬁned. Bolander and Gierasimczuk [5] and [6] study domain learning where
an agent attempts to learn a representation of the dynamics of a domain.
2.1
Dynamic Epistemic Logic (DEL)
Let us start by deﬁning the epistemic language Lepis over a set of propositional
symbols:
Definition 2.1 (DEL) The language of DEL Lepis is deﬁned as:
φ ::= p | ¬φ | φ ∨φ | Kφ | [A]φ
Here, [A] denotes the dynamic modality of an event model A (see 2.5). We
interpret [A]φ as ‘executing A will lead to an epistemic model where φ holds’.
To allow reasoning about the uncertainty of agents, the possible world model or
epistemic model can be used.

Definition 2.2 (Epistemic model) An epistemic model is a tuple M =
(W, R, V ), where W is a ﬁnite set of worlds, R ⊆W ×W is the equivalence rela-
tion, and V : P →P(W) is a valuation function mapping worlds to valuations.
A pointed epistemic model (M, w), consists of an epistemic model M and a
distinguished world w ∈W.
Finally, let us deﬁne the truth of an epistemic formula φ in a given epistemic
model M and world w ∈W:
Definition 2.3 (Truth of epistemic formula)
M, w |= p iﬀp ∈V (w)

2.2
Actions
As we are considering action learning, a formal deﬁnition of actions is needed.
Actions will here be interpreted as mappings that transform states.
Definition 2.4 (Actions) An action is a subset α ⊆2P × 2P . An action
α is deterministic if for every s ∈2P , there is one s′ ∈2P , with (s, s′) ∈α. An
action is universally applicable if for every s ∈2P , there is at least one s′ ∈2P
with (s, s′) ∈α.
When modelling changes caused by actions DEL includes the concept of action
models or event models.
Definition 2.5 (Action model) An action model over propositions P,
is a tuple A = (E, Q, pre, post), where E is a ﬁnite set of events, Q ⊆E × E is
the indistinguishability relation, pre : E →Lepis assigns a precondition to each
event while post : E →Lprop assigns postconditions to events. The set of all
action models over P is called Actions(P).

For an event e, pre(e) denotes which propositions must be satisﬁed before e
can take place, while post(e) denotes which propositions are true or false after
e has occurred. We will distinguish between diﬀerent types of preconditions of
actions.
Definition 2.6 (Precondition types) We deﬁne preconditions pre(e)
of an event e as being one of the following types:
• Basic, if all preconditions are conjunctions of literals.
• Maximal, if all preconditions are maximally consistent conjunctions of
literals.
Example 2.1 (Train track control and preconditions) Consider
a scenario where an agent is learning to control the merge of two train tracks,
using two levers. The agent can use the ‘move’ action to move between two
rooms, the room containing the two levers, l, and the room containing a screen
showing the status of the tracks, ¬l. In the lever room, the two levers are colored
red, and blue respectively. When the red lever is pulled it is denoted r while b
denotes that the blue lever is pulled. Figure 2.1 shows the rules of the domain.
In order to complete the merge m, both levers must be pulled, which the agent
can do by executing the two pull actions, ‘pullr’,‘pullb’ in any order. If only the
red lever is pulled r, then the traﬃc on both tracks is stopped, ¬t.
The agent is able to observe any eﬀect of actions, making the domain fully
observable.

The ﬁrst concept is streams.

Definition 2.10 (Stream) A stream, E ∈(2P × 2P )ω is an inﬁnite se-
quence of propositional state pairs (s, t). These (s, t)-pairs are called observa-
tions. For n ∈N and E being a stream we use the following notation:
1. E[n] is the initial segment of E of length n.
2. En is the n-th element of E.
3. set (E) := {(s, t) | (s, t) ∈E}, denotes the set of all observations in E.
A stream E is sound with respect to an action α, if set(E) ⊆α. A stream E is
complete with respect to an action α, if α ⊆set(E). If a stream is both sound
and complete with respect to α, the abbreviation ‘E is for α’ is used. These
notions are used for initial segments of streams as well.
Definition 2.11 (Learning function) A learning function is a func-
tion that as input takes a ﬁnite sequence of observations and gives an action
model as output: L : (2P × 2P )∗→ActionModels∪{↑}, where ↑denotes ‘un-
decided’.
Bolander and Gierasimczuk [6] introduce two types of identiﬁability of actions:
ﬁnite identiﬁability and identiﬁability in the limit. In ﬁnite identiﬁability, the
learning function will observe some ﬁnite amount of action executions and will
always output a correct model for the action. This type of learning function is
at most once deﬁned, meaning that a learning function L will never correct its
answer after outputting an action model. It can, however, also give the answer
‘undecided’.
Identiﬁability in the limit, a weaker type of learning, requires that after the
learning function has observed some ﬁnite amount of action executions outputs
a correct model for this action, and then stick to this answer forever on all the
following output. As this cannot be achieved in ﬁnite time, this type of learning
is called inconclusive. However, this learning type covers the non-deterministic
actions as well, where ﬁnite identiﬁability only handles deterministic actions.
The learning functions presented later in this section will be ﬁnitely identiﬁable,
which means that they have the following properties.
Definition 2.12 (Finite identifiable learning functions) For a
set of actions X, where α ∈X, a learning function L, and a stream E, we say
that:

1. L ﬁnitely identiﬁes α on E and there is an n ∈N such that act(L(E[n])) =
α, where act(a) = {(s, t) | t ∈s ⊗a}.
2. L ﬁnitely identiﬁes α, if L ﬁnitely identiﬁes α on every stream for α.
3. L ﬁnitely identiﬁes X if L ﬁnitely identiﬁes every α ∈X.
4. X is ﬁnitely identiﬁable if there exists a function L which ﬁnitely identiﬁes
X.
Definition 2.13 (Definite Finite Tell-Tale) A set Dα ⊆2P × 2P
is a deﬁnite ﬁnite tell-tale (DFTT) set for α ∈X where X is a set of actions, if
the following hold:
1. Dα ⊆α
2. Dα is ﬁnite
3. For any β ∈X, if Dα ⊆β then α = β.
A set of actions X is ﬁnitely identiﬁable if and only if there exists an eﬀective
procedure D: X →2(2P ×2P ), that given an action model a returns a DFTT
of α. Bolander and Gierasimczuk [6], proves that the set of deterministic and
universally applicable actions are indeed ﬁnitely identiﬁable.

Furthermore, the formalisation of dependencies between propositions of a for-

3.1
Causation and dependence
According to Wright [31], when two propositions "are causally connected means
that we can inﬂuence the one by manipulating the other." Wright calls this type
of causal connection for manipulative causation, as there is an essential connec-
tion between causation and action. Recall the ‘Ladder of Causality’ by Pearl
and Mackenxie [22], from chapter 1. It is clear that Wright’s deﬁnition of manip-
ulative causation corresponds to the second level of Pearls Ladder, intervention
causality. Barbero et al. [3], introduces an epistemic causal model combining
causation and epistemic reasoning, based on the causal models founded by Pearl
[21]. In these models, the agent distinguishes between two types of variables;
==PAGE:22==
14
Modelling Causality and Dependence Model Learning
exogenous variables, that are causally independent, and endogenous variables
that are causally dependent. The causal epistemic model allows for interven-
tions to take place, where some variables are set to certain values, similarly to
the learner executing an action in the domain of Bolander and Gierasimczuk
[6]. We will thus adapt the learner to learn a dependence model of an unknown
domain.
When designing a dependence model, one must be aware of two paradigms
in formalisation of dependence: Independence Friendly Logic (IF Logic) and
Dependence Logic (DL). IF Logic was founded by Hintikka and Sandu [16],
and is an extension of First-Order Logic (FOL), where the independencies of
formula are investigated on a quantiﬁer level. Väänänen [29], later developed
DL, where the dependencies are described on the atomic level of formulas, by
adding dependence atoms to FOL formulas.
As we are interested in studying the dependencies of formulas in the proposi-
tional language, we will choose the DL perspective. The semantics of DL is
called Team Semantics (TS). TS is a generalisation of Tarski’s semantics [26],
where the satisﬁability of variables is evaluated with respect to sets of variable
assignments instead of single assignments [12].
Baltag and van Benthem [2], use this semantics as an inspiration in their paper,
where they present a decidable logic of functional dependence, LFD. LFD is an
extension of propositional logic added dependence atoms of DL combined with
dependence quantiﬁers treated as modalities within the string of generalised
semantics for FOL. Various language extensions are discussed as well, including
dynamic logics of changing dependence models. These dependence models will
be, in turn, used as an inspiration for our dependence model.
3.2

Now we will deﬁne a model for dependencies between propositions of a domain.
We will adapt the model for FOL dependencies by Baltag and van Benthem [2]
to propositional dependencies.
Definition 3.1 (Dependence model for FOL) A dependence model,
DF OL = (V, O, S, P) consists of a set of variables V , a set of objects or values
that variables can be assigned to O, a set of states S, mapping variables to
objects and a set of predicates P of objects.
Our dependence model for propositional logic will be a tuple, as there are no

Definition 3.2 (Dependence model) A dependence model over a set

P, and a set of dependencies D between singleton elements of U and elements
of V . Note that V ∪U = P and V ∩U = ∅, as no proposition in P can be both
exogenous and endogenous.
From a dependence model, the epistemic model can be induced, as the knowledge
of which propositions in P are dependent implicitly provides the knowledge
of which propositions in P are independent.

3.3.1
Action models and dependence types
Let us ﬁrst consider the relation between dependence types from Baltag and
van Benthem [2] and the precondition types of action models from Bolander et
al. [7].
Let l and k be literals. Let us ﬁrst consider the relationship between global
dependencies and the precondition type of an action model. If l globally deter-
mines k, then whenever the value of l changes between the precondition and
postcondition of an action, so will the value of k.
An example of a global dependence between two propositions in an action model
with basic preconditions could be the red lever r and the traﬃc ﬂow t, from
example 2.2. Here the values of r globally determines t, which is shown in the
second column of the action model. Anytime the value of r changes between the
pre- and postcondition of the pullr action, so does t, i.e. when r is 0 (negated),
t will always be 1, and when r is 1, t is always 0. The same can be shown for
the action model with maximal preconditions in example 2.1.
Now let us consider the connection between local dependence and the precon-
dition type of an action model For action models with basic preconditions, see
example 2.2, if l locally determines k, then whenever l appears in the precondi-
tion of an action while taking the determining values, the value of k will only be
able to change if l is no longer the determining value in the action postcondition.
For action models with maximal preconditions, see example 2.1, if l locally
determines k, then whenever l takes the determining values, the value of k will
only be able to change if l is no longer the determining value in the action
postcondition.
An example of the local dependence between two propositions in an action model
with basic preconditions could be the blue lever b and the merge of the tracks

4.3
Traces in AI
We will now examine how traces are used in a more general setting. To describe
interaction with a system, often represented as a ﬁnite state machine, a trace or
execution trace can be used. An execution trace is simply a sequence of actions.
In AI, execution traces of a system are commonly used as means to verify and
investigate its behavior. This system could be a planning agent attempting to
ﬁnd a plan to reach a goal. After ﬁnding such a plan, the agent uses execution
traces to verify that this plan is indeed valid, and thus does not break any known
rules in the given environment as in Russel and Norvig [24]. Bertoli et al., [4]
illustrate another use of execution traces where only partial execution traces are
available, namely to detect and reconstruct missing knowledge of the system.
In Machine Learning, execution traces can be used to design and train classiﬁ-
cation models. The work of Tsimpourlas et al., [28] is an example of such use.
In their paper, the goal is to design a neural network, which is able to label exe-
cution traces in order to solve the test oracle problem in the area of automated
testing of systems. Furthermore, execution traces have been investigated in the
ﬁeld of Cognitive Science, in order to compare traces of automated reasoning to
traces of human subjects, a method initiated by Newell and Simon [19].
In Bolander et al. [7], execution traces of a domain transition system are crucial
part of the learning process. They deﬁne an execution trace as an alternating
sequence of states and actions, ending in a state, while the observation trace of
an execution trace tells what an agent can observe from this execution trace.
The agent is to learns an unknown domain from a given observation traces from
the initial state of the domain transition system.

4.4.1
From trace generation to exploration
Let D = (T, Ω, Obs) be an unknown domain. Given the initial state s0 ∈T,
and the set of universally applicable actions A, the task is now to generate all
observational traces of D from s0. The transformation of the trace generation
problem of a domain D to a graph exploration problem ﬁrst needs to transform
the transition system into a graph in the following way:
• Let the initial state s0 and all other states in the underlying transition
system of D, be nodes in a graph. Let the value of propositions of each
state be node labels.
• From s0, consider A to be a set of unexplored, labeled, directed edges
leaving s0.
• Executing an action ai ∈A, corresponds to traversing or exploring an
edge in an unknown graph. If the endpoint of the edge is not yet known,
this node along with its outgoing edges is discovered.
The goal is now to explore all edges of the graph. When exploring an unknown
graph, a partial graph of what nodes and edges have been explored so far is
maintained, as well as the unexplored edges for all discovered nodes. All paths
from s0 are saved. To generate the set of all observational traces of the unknown
domain, use the labels from the nodes and edges to identify actions.
We will now consider some algorithms for solving this exploration problem. In
the following algorithms, we assume that the graphs to be explored are strongly
connected, i.e., for any nodes u and v there is a path from u to v. In this section
we will give a short description of each of the three main algorithms we study.
To get a more detailed description of each of them see appendix B. We begin
by introducing some basic notions for graphs.

4.4.2
Graph properties
We deﬁne a node in a directed graph as a sink if the outdegree of this node is
greater than its indegree (meaning that more edges leave the node than enter it).
The strength of a sink is the diﬀerence between the indegree and the outdegree.
The sum of all strengths of the graph is called the deﬁciency of the graph. An
alternative deﬁnition of deﬁciency is the number of edges to be added to a graph
to make it Eulerian. This implies that an Eulerian graph has deﬁciency zero,
i.e., it has smaller outdegree than indegree,
While exploring, a partial graph is built of the explored edges and nodes so far.
This graph is known to the algorithm.
Definition 4.15 (Partial Graph) Let P = (V ′, E′, c) be a partial graph,
where V ′ ⊆V and E′ ⊆E ∩V 2 is the nodes and edges in the partial graph,
while c ∈V ′ is the current node in the exploration. Initially, P = ({s}, ∅, s),
where s is the starting node.
In the partial graph, nodes have a poutdegree and pindegree, being the number
of edges leaving and entering the node in the partial graph. If a node v have
poutdegree(v) <pindegree(v), it is called a partial sink, and if poutdegree(v)
>pindegree(v) it is called a partial source. Note that partial sinks are sinks in
the whole graph as well, but this does not necessarily hold for a partial source.
A state is a pair (P, u), where P is a partial graph, and u is a function mapping
V ′ to non-negative integers. The number of unexplored edges going out of v
is denoted u(v) for v ∈V ′. Initially, u(s) =outdegree(s), for the starting node
s. The algorithms terminate when the partial graph coincides with the whole
graph, i.e. when there are no unexplored edges left.
4.4.3
Known algorithms
Let us ﬁrst consider some known algorithms to solve this exploration problem.
Three of the most famous exploration algorithms are: Greedy, Depth-First, and
Breath-First. The overall exploration strategy of all three is to choose unex-
plored edges whenever possible. If the algorithm is stuck in a node where all
its edges have been explored, the algorithm considers the path it has currently
explored consisting of previously unexplored edges. The algorithm is recursively
run from any node on this path which have unexplored edges. The diﬀerence
between these algorithms lies in which node with unexplored edges they choose
ﬁrst when they get stuck:

• Greedy: Move to the nearest node from the current node with unexplored
outgoing edges
• Depth-First: Move to the most recently discovered node with unexplored
outgoing edges
• Breath-First: Move to the node with the smallest distance to the start
node with unexplored outgoing edges that can be reached in the partial
graph.
Albers and Henzinger [1], show that for all three algorithms, there exist graphs of
deﬁciency d that require 2Ω(d)|E| edge traversals. Since 2Ω(d)|E| edge traversals

of exploring unknown graphs.
4.4.4
The Stem Algorithm
Deng and Papadimitriou [9] show the following theorem hold.
Theorem 4.16 A graph with a deﬁciency d, can be explored such that no
edges are traversed more than 2d log d times.
Compared to the lower bound of 2Ω(d)|E| edge traversals of the Greedy, Breath-
First and Depth-First algorithms, obtaining an an upper bound for the algo-
rithm is a signiﬁcant beneﬁt of this algorithm.
The idea behind the algorithm is to apply the algorithm for traversing an Eu-
lerian graph. The algorithm for traversing an Eulerian graph was created by
Hierholzer [15], starts by choosing any starting node, v and follows the path of
edges from v until returning to v. As the property of the graph being Eulerain
means that it is impossible to get stuck at any other node that v, as all nodes
have even degree. This ensures that the algorithm can never enter any other
node than v without being able to leave it again. When returning to v, the cycle
discovered may contain all vertices and edges of the graph. However, as long
as there exists a node u belonging to the current path that has edges not being
part of the path, another path will be started following previously unused edges
until returning to u, and being joined with the previous paths. This procedure
is continued until the entire graph have been traversed.
When considering non-Eulerian graphs, the traversal can get stuck in a node,
and thus a strategy to decide whether to choose an unexplored or previously

Definition 4.21 (The Balance algorithm)
• Start by running a modiﬁed version of the Eulerian algorithm (presented
in the Stem Algorithm B.1), where missing edges are replaced with paths
between endpoints.
• To ﬁnd missing edges, maintain chains where the tail of chain i is si, and

• When the algorithm gets stuck at some sink s, the explored graph contains

• Now, visit the current node and explore unvisited edges. If there are no
unvisited edges from the current node, relocate to the startnode of the
chain.
• When all nodes on a chain is ﬁnished (having explored all their edges)
relocate to a new chain that is not yet ﬁnished and explore unvisited
edges.
• To decide which chain should be chosen next, tokens are placed on the
nodes to balance the work, such that the agent is less at risk of getting
stuck.
• Terminate when all chains are ﬁnished.

To determine the time and space complexity of the Learner(P, Q, Σ) let us ﬁrst
us time complexity of Paths(q, τ). Paths(q, τ) traverses τ once to ﬁnd any q-
paths in τ. Simultaneously, the algorithm traverses any cycle in τ to remove it,
which adds at most |τ| to the time complexity. q-Index(q, τ) adds an additional
traversal of τ, which does not add to the asymptotic time complexity. This
gives a total time complexity for Paths(q, τ) of O(|τ 2|). The space complexity
of τ is linear in the size of the output T, i.e. the size of all paths in τ. The
q-Index(q, τ) procedure adds O(1) space to the complexity, which again is in-
signiﬁcant. Paths(q, τ) therefore have a space complexity of O(|T|). This can be
further reduced by considering the number of distinct paths in a graph. Donald
[10] show that for any graph, with V nodes, there exists and upper bound of
|V | −1 distinct paths in the graph. As the state space of any domain transition
system can be at most 2|P |, there must exist at most 2|P | −1 distinct paths in
the transition system. This results in Paths(q, τ) having a space complexity of
O(|2P |).

Chapter 6
Conclusion and Future work
6.1
Conclusion
In this thesis, the aim was to develop causal learning functions or learning al-
gorithms to enable the learner to learn the underlying casualties of an unknown
domain, which additionally allows the learner to learn the dynamic structures of
this domain. To achieve this two questions needed to be answered: What should
be the underlying framework for our learning function? How should we interpret
causal learning? To answer the ﬁrst question, we have examined learning func-
tions for learning action model representations of a domain, which provided the
basis for our learning functions. To answer the second question, an investigation
of how to interpret and represent causal relations between objects in the domain
lead us to our choice of interpreting causal relations as dependence relations in
our domains. Combining these answers resulted in a framework for dependence
model learning, for learning global dependencies in a domain. Initially, a simple
learning function for learning singleton dependencies between two propositions
of a fully observable domain was created. To learn more complex dependencies,
where one proposition can depend on multiple other propositions of the fully
observable domain, a learning algorithm was developed. Both learning methods
exploit the fact that the learner always can see the full extent of the eﬀects of
an action. After this, we extended the domains to include partially observable
domains. In order to learn the dependencies of partially observable domains, the
==PAGE:62==
54

observation traces of the domain, and generation thereof were studied. We found
that the restricted observability resulted in the dependence learning functions
previously deﬁned being unable to detect dependencies that are not observable
simultaneously. To enable the learning these dependencies, a ﬁnal learning al-
gorithm was developed. This learning algorithm used the observation traces
to ﬁnd all sequences in the observation trace where a dependant proposition
changes value. By determining the common elements of all such sequences, the
dependence of a proposition is revealed. Summing up, we managed to construct
a causal learning method for our domains. There are, however, still many open
problems that deserve to be mentioned.
6.2
Future work
We will now discuss some open problems that could expand the work of this
thesis into diﬀerent areas.
DNF learning
Another logic-based method for learning global dependencies
in a fully observable domain could be to ﬁnd the minimal formula in the Disjunc-
tive Normal Form (DNF) of the states in the unknown domain. This method
is inspired by the dependence tables of Baltag and van Benthem [2]. Here the
learner will be given a stream containing all states at least once. Firstly, the
state space of the domain would be represented as a DNF, where each state
would be a conjunction of the values of propositions in this state. By ﬁnding
the minimal DNF describing the domain, the independent propositions would
be eliminated as there for any state would be another state being equivalent to
it except for the independent proposition. Finding the minimal DNF is a non-
trivial problem, however, the Quine-McCluskey algorithm [17], does provide a
solution to ﬁnding minimal Boolean functions.
Dynamic domains
To be able to consider domains closer to real-life scenar-
ios, the extension to learn dynamic domains is essential.
Van Benthem [30]
(implicitly) represents these dynamic domains as graphs. As the partially ob-
servable domains are represented as a graph, using a similar representation for
dynamic domains seems very natural. This would give the possibility of applying
algorithms developed for dynamic graphs to solve sub-problems in a supervised
learning process. Here the teacher could give hints to the learner for instance,
by removing edges to states that provide no relevant information at this point
of the process, as studied in Gierasimczuk et al. [13]. The learner must then be
==PAGE:63==
6.2 Future work
55
able to detect these changes in order to reason about the underlying structure
of the domain and its dependencies.
Cognitive Science and dependence learning
Understanding the relation-
ship between the dependence learners in this thesis and the cognitive perspective
on learning dependencies in an unknown domain could also be an interesting
aspect to investigate.
Causation and Local Dependence
We have presented two types of de-
pendence, local and global dependence. In this thesis, we have focused on the
dependence model learning for global dependencies as this type of dependen-
cies provides the most knowledge of the domain structure. However, a learning
function for local dependence is still useful in understanding the dependence of
the domain and could be used to learn which propositions of a domain might
be in the dependence, but not as a singleton. This learning algorithm might
resemble the learning algorithm from chapter 5.
Interpretations of causation
A diﬀerent perspective on learning cause-
eﬀect relationships between objects in a domain, could associated with the ﬁrst
lever of Pearls Ladder, prediction. Here, the predictive coding theory from cog-
nitive science (see Rao [23]), could provide inspiration to a model for prediction-
based causal learning. As machine learning and prediction often is related, an
implementation of this model could be done using Bayesian Inference as pre-
sented Tenenbaum et al. [27].
Considering the third level of Pearls Ladder, the counterfactual reasoning could
be another approach (see Starr [25]). To model causality using counterfactuals,
is far from a new invention, and there already exits a framework for analysing
counterfactual statements, Bayesian Networks. By using Bayes Networks on
Structural Equations we can obtain an undirected acyclic graph that can be
used to evaluate counterfactuals, as described by Menzies and Beebee [18].
Complexity of dependence learning
Finally, as discussed in chapter 5, the
complexity of the learner is far from optimal, which could be improved either
by a more compact state space representation or by relaxing the problem to
learning an approximation of the dependence.

edges are assigned ages. Initially, all edges have an age of ∞, and after ﬁnishing
an edge, its age will be 0. The algorithm will attempts to to minimize the age of
a path t. An edge is old if they are not in the current portion, but still included
in the partial graph. An edge is new after it is ﬁnished.
Lemma B.8 The algorithm will prefer connection paths consisting of ﬁnalized
new edges to paths containing old edges. When there exists a path between nodes
only consisting of ﬁnalized new edges, then the path between these two nodes
consists of ﬁnalized new edges.
The overall results of Deng and Papadimitriou [9], is the following theorem.
Theorem B.9 A graph with a deﬁciency d, can be explored such that no edges
are traversed more than 2d log d times.

B.2
The Complete Algorithm
The Complete algorithm solves the same exploration problem as the Stem algo-
rithm, the exploration of an unknown, directed graph, and the overall general
strategy of the exploration is the traversal of Eulerian graphs, just as the Stem
algorithm. Albers and Henzinger [1] do however add a divide and conquer strat-
egy that increases the eﬃciency of the algorithm signiﬁcantly.
The ﬁrst step to achieving this improvement is the Balance algorithm. This
algorithm explores an unknown, strongly connected graph with deﬁciency d. Let
us now deﬁne the basic notions for this algorithm. Edges are initially marked
unvisited or new and become visited when traversed. A node is ﬁnished when
all its outgoing edges are visited. The algorithm is stuck if it enters a ﬁnished
node on a unvisited edge. A sink is discovered when the algorithm is stuck at
this sink for the ﬁrst time. An underlying assumption of this algorithm is that
whenever the algorithm is stuck in a previously undiscovered sink, the partial
graph of explored edges is strongly connected.
For any graph with deﬁciency d, the Balance algorithm requires to know the d
missing edges (u, v) needed to make the graph an Eulerian graph. Furthermore,
a path from the start node u to the end node v, in the graph should be known.
The algorithm will then try to ﬁnd these missing edges by maintaining d edge-
disjoint chains, from u to the current estimate of v. This is done in two phases;
Phase 1; the algorithm for Eulerian graphs, presented by Deng and Papadim-
itriou [9] is executed on the graph.Phase 2; contains two subphases, where the
algorithm in each will visit the current node of the current chain C and work
on the chain to ﬁnish C. These chains will be in one of the three following
states: fresh, in progress, or ﬁnished. A chain is ﬁnished if all its nodes are
ﬁnished, a chain C is in progress if was abandoned in a previous subphase for
after ﬁnding the start of another chain, and a chain is fresh if it is not yet in
progress. At most d + 1 chains can be in progress and at most d chains can be
fresh simultaneously. When the algorithm works on a chain, it will use tokens
to mark the node being the current estimate of the end of this chain. A node
can be assigned multiple tokens from diﬀerent chains at once.
Let us now consider a subphase in a little more detail. From a current node v
==PAGE:72==
64
Detailed descriptions of Algorithms
of the current chain C, the algorithm check for unvisited edges. There is three
cases for the algorithm to consider:
1. v does not have any unvisited outgoing edges, and it is not the end of C.
In this case, the algorithm should continue to the next node of C.
2. v does not have any unvisited outgoing edges, and it is the end of C. The
algorithm should decide on a new chain to work on and relocate to this.
3. If v has unvisited outgoing edges, repeatedly explore unvisited edges until
the algorithm is stuck at some node u. Let P be a path from v to u. When
discovering u, four scenarios arise. a) u = v, and the cycle is added to the
chain. b) u ̸= v, u have a token and is the startnode of fresh chain D.
Then P is added to D and the token from u is placed on v. The chain
containing the previous owner of the token becomes the new current chain.
c) u ̸= v, u have a token and is not the startnode of the fresh chain. The
algorithm proceeds as in b), but creates a new fresh chain. d) u ̸= v, u
does not have a token. The algorithm places a new token on v and creates
a fresh chain from u, which becomes the new current chain.
Repeating the above procedure is essentially the Balance algorithm.
The requirement in the Balance algorithm to know the d missing edges (u, v),
is however problematic, as these edges might not be trivial to identify before
running the algorithm. And Albers and Henzinger [1] therefore present another
version of the algorithm, the P-Balance algorithm. The P-Balance algorithm
is, just as the Balance algorithm provided with a start node and its chain,
also given a set of paths between various nodes in the graph.
Initially, this
set might be empty and the P-Balance algorithm will then add edges to ﬁnd
paths between the missing edges in the graph. P-Balance adds at most d −1
edges during execution. This brings us to the Complete algorithm, which is the
ﬁnal algorithm in Albers and Henzinger [1]. From and initially empty P, the
P-Balance is executed until the algorithm either terminate is stuck at a node u.
In the former case, Complete also terminates, and in the latter case, paths from
each visited node in the explored partial graph are added to P and P-Balance
is executed again from u given the updated set P.
Definition B.10 (The Complete algorithm) The Complete algorithm
executes P-Balance from an initially empty P, until P-Balance terminates, or
until the algorithm is stuck in a node u. In the former case the Complete algo-
rithm terminates. In the latter it adds a path to the set P, such that a subset of
traversed nodes in the explored subgraph is now adjacent to u. It then restarts
the P-Balance algorithm from u.

B.3
The Explore Algorithm
In this section, a detailed description of the Explore algorithm is given.
The problem that the Explore algorithm solves is the exploration of an undi-
rected, unknown graph, G = (V, E), where v is the set of nodes in G and E is
the set of edges in G. The algorithm will be given a startnode and will always
know the endpoints of an explored edge, but for any node, only the number of
unexplored edges is known. When exploring the algorithm faces the choice of
traversing a speciﬁed explored edge or a random unexplored edge.
The complexity of the algorithm will be stated in the number of edge traversals
above the optimal, here being a traversal of the corresponding Eulerian graph
|E|, the algorithm need. This measure is called the penalty of the algorithm.
The main idea behind the Explore algorithm is to maintain most of the relo-
cations when stuck in a node where all edges have been explored along with a
spanning tree of the graph.
The Explore algorithm uses the procedure Saturate(v) to explore edges from a
speciﬁc node of the graph. The Saturate(v) procedure will explore the graph
from v until v is saturated, meaning that all edges from v are explored. To mark
which nodes are saturated, the Explore algorithm will explore the graph using
colors, {white, red, blue}. Initially all nodes are white, and when a node is satu-
rated during the Saturate(v) procedure it is marked red, and will never change
color. These red nodes are then used as a stopping point of the Saturate(v) pro-
cedure, to avoid exploring any further from this node. A node is marked blue if
it occurs on any path from v during Saturate(v). If the Saturate(v) procedure
is stuck on a blue path, it will mark the endpoint red and try to explore any
unexplored edges further back on this path.
The Explore algorithm will try to use the Saturate(v) procedure on all nodes of
G, skipping those that might already be saturated during the exploration from
other nodes.
Panaite and Pelc [20] proves that the Explore algorithm has a penalty of O(|V |),
given any undirected connected graph.
