Summary
In this thesis we have investigated if a neural net system based on reinforcement learning
can be used to create a control system for a robotic manipulator operating in a contin-
uous dynamic environment containing both static and dynamic obstacles, and if such
a system would be configuration independent, i.e heterogeneous controllability with the
same system structure trained automatically on varying manipulator designs.
After analysing various methodologies, we based our system on the Deep Deterministic
Policy Gradient, which was designed for use in continuous environments, where both
the state and action spaces can be high dimensional.
We created an environment by adapting the Cartpole problem from OpenAI to reflect
our needs by adding functionality for electro magnet control and collision detection.
A position control system was created for a 3 DOF robotic manipulator, which achieved
high accuracy in position control tests, but path analysis revealed that the trajectories
was less than ideal compared to straight line movements.
We tested our position control system with a more complex 5 DOF robotic manipulator,
and while the performance did not match the 3 DOF case, we showed that a reinforce-
ment learning based system does have the potential for heterogeneous controllability.
Finally we have investigated if the position control system could be expanded to include
static obstacle avoidance, however we were not able to accomplish this while retaining
the previously achieved position control performance.
Based on our results we have concluded that a control system built on the methods
applied in this thesis does not appear suitable for real world implementation.

Reinforcement Learning
terminology
Action space - The scope of possible actions the agent can perform, like the state space
it can be either discrete or continuous.
Agent - The first component of the reinforcement learning, the neural net(s) that are
responsible for deciding what action to perform.
Environment - The representation of the environment, responsible for generating the
states that are the information the agent receives.
Experience replay - The creation of pseudo i.i.d data by storing N transitions in a
replay buffer, and then drawing random samples from this when performing weight op-
timization.
Exploration - The process of actively mapping the environments state and action
spaces, thus ensuring that the global minima is found.
Model free algorithm - Information regarding the system dynamics are not required
to solve the task, ”Learn by doing”.
Off policy - The policy does not directly influence exploration in the next step, thereby
allowing independent exploration strategies, as well as the ability to have separate poli-
cies for performing the actions and updating the weights.
On policy - The policy directly influences which states and actions will be explored /
encountered in the next step.
Policy - The definition of how an agent will react to a given state, either as a specific
action for a deterministic policy, or the probability distribution of taking action a in
state s for a stochastic policy.
Q value - Also known as Q function or action value function, a measure of the overall
expected reward associated with performing action a at state s.
Reward - A numerical value generated by the environment and send to the agent, as
a consequence of performing the action a in state s, a reflection of the quality of the
action vis-a-vis solving the objective.
State space - The delineation of states that can occur, can either be discrete - E.g
the possible outcomes of rolling a dice or continuous - E.g the span of configurations a
robotic manipulator can assume.
Total discounted reward - The cumulative reward associated with performing action
at in state st and then continuing to perform actions until the episode terminates (t = T),
with discount factor γ to increase the weight of rewards received near t.

CHAPTER 1
Introduction
The purpose of this project is to investigate the ability of a neural network based solution
to control a robotic manipulator in a dynamic setting.
This chapter will provide a high level introduction to the general problems that must be
solved and the motivation for doing so.
Finally a brief overview of the thesis structure will be provided as a readers guide.
1.1
Motivation
While classic control provides well established tools for the control of robotic manipu-
lators in the form of kinematic control theory, these methods are generally only well
suited for deterministic environments, or at least settings in which non deterministic
components can be ignored.
I.e. in a human / robot collaboration, the primary responsibility for avoiding collisions
resides with the human.
1.2
Objectives
The theme of the thesis was to investigate if a reinforcement learning based system could
be a substitute for classic control.
The objectives of the thesis was structured into three sequential phases:
Phase 1 - Position control:
Implement a position control system based on reinforcement learning, and investigate
its performance and behaviour compared to ”ideal” control.
Phase 2 - Static obstacle avoidance:
Investigate how the position control system can be expanded to avoid static obstacles
on the playfield.
Phase 3 - Dynamic obstacle avoidance:
Determine if the developed neural network system can adapt to dynamic obstacles in
the environment, based on the results and conclusions from phase 1 and 2.

CHAPTER 2

The following chapter will be the analysis of the project, starting with an introduction

2.1
State of the art
2.1.1
Reinforcement learning methodologies
The following discussion of RL methodologies assumes that the reader is familiar with
basic concepts of reinforcement learning, a brief terminology section is included in the
pre-matter of this document, and in the following discussions cursive notation indicates
that the concept is included in this section.
For an in depth representation of the theories behind the methods and algorithms, the
reader is encouraged to read chapter 5.1 of this thesis, titled ”Reinforcement learning in
a continuous action space”.
2.1.1.1
Q Learning
Q learning, Watkins et. al - 1992 [WD92] is the original foundation of a large segment
of the reinforcement learning methods.
At its core is the action value function, also known as the Q value or Q function, which
describes the total discounted reward associated with performing action a in state s,
visualized on figure 2.1.
In its base form Q-learning centres around generating a lookup table (LUT) that contains
the Q-value - Qπ(s, a), of all possible state/action pairs, and thus it is only suitable for
reinforcement learning problems where both the state and action space are small and
discrete.

2.1.1.2
Deep Q Network - DQN
Originally presented by Mnih et. al - 2013 [Mni+13], the Deep Q Network was capable of
solving seven different Atari 2600 games, and even surpassing human level performance
in three of those games.
They achieved this by adding experience replay to Q learning and utilizing the off policy
nature of Q learning.
This was a demonstration of the algorithms ability to solve a task with a large state
space, but it was still limited to small discrete action spaces, as each possible action was
represented by a unique output neuron.
2.1.1.3
Stochastic policy gradient - SPG
The stochastic policy gradient (SPG) was presented by Sutton et. al. - 2000 [Sut+00],
as a method of solving reinforcement learning problems in continuous action spaces by
approximating a stochastic policy directly instead of approximating the action value
function (Q learning).
The primary complication of the stochastic policy gradient was its reliance on integrating
over both the state and action spaces, which was both computationally expensive, and
also required a large number of samples before converging in high dimensional action
spaces.
2.1.1.4
Deterministic policy gradient - DPG
Presented by Silver et.
al - 2014 [Sil+14], the deterministic policy gradient was an
evolution of the SPG, which eliminated the need to integrate over the action space,
thus severely outperforming its stochastic counter part in tasks containing large action
spaces.
With the elimination of the stochastic component, DPG did loose the policy gradients
inherent ability to explore its state and action spaces, requiring alternative exploration
methods.
==PAGE:21==
2.2 Physical implementation
5
2.1.2
Deep determinisitic policy gradient - DDPG
The Deep Deterministic Policy Gradient, (DDPG), Lillicrap et. al - 2016 [Lil+16], was
the result of combining the methods of the Deep Q network with the deterministic policy
gradient.
The paper demonstrated the ability of the DDPG algorithm to solve a number of re-
inforcement learning problems with continuous state and action spaces, including a
moderately successful ability to control a 7 DOF robotic manipulator in a vacuum envi-
ronment.
Based on these results we selected the deep deterministic policy gradient as our workhorse
for the project, and will therefore be investigating its application towards solving our
objectives.
2.2
Physical implementation
While the core premise of the project scope could be achieved in a purely software based
solution, the feasibility of constructing a simple robotic manipulator will be investigated,
as this could provide insight into how well the developed system could transition from
the simplified software based environment, to the non linear complexities of the real
world.
The physical design was limited to a single 5 DOF manipulator, while the primary focus
of the software implementation was centred around a simpler manipulator with 3 degrees
of freedom.
This both reduced the complexity of the problem thereby accelerating development
cycles, as well as provided a basis for testing the performance of the solution across
various types of robotic manipulators, accomplishing the heterogeneous component of
the thesis.
The tool attached to the end effector of the robotic manipulators will be a small electro
magnet.

2.3
Heterogenous control systems
A significant complication of classic control based systems is their dependency on config-
uration specific design solutions, but with the ”learn by doing” strategy of reinforcement
learning this complication can potentially be removed, as the developed system should
be configuration independent and thus capable of learning to control various configura-
tions of robotic manipulators.
This hypothesis will be tested by investigating the performance of the developed sys-
tem on both a 3 DOF and 5 DOF design, with as few modifications as possible to the
reinforcement learning parametrization.

2.5
Project approach
The problems represented by the three project phases introduced in the previous chapter,
and the heterogeneous complication, will serve as the foundation for determining if a
reinforcement learning system based on the deep deterministic policy gradient algorithm
can replace classic control systems.
The evaluation will be based on its ability to achieve comparable performance to classic
control theory with respect to position control, while allowing the robotic manipulator
to function in an environment that contains both static and dynamic obstacles.
The tasks necessary to achieve this provided the structure for this report, with each
chapter representing a specific task or objective.
• The Environment: Delineate the characteristics of the playfield based on the hard-
ware constraints.
• The Simulator: Design and implement a software representation of the environ-
ment with sub-functions including collision detection, electromagnet control and
visualizations.
• DDPG: Represent the RL architecture used in the thesis, and present the theory
of its support foundations.
• Position control: Design, implement and test a position control system based on
a 3 DOF robotic manipulator, while characterizing the underlying motivations for
the parameter choices and investigate its behaviour.
• Heterogeneous manipulator control: Investigate if the position control solution
designed for a 3 DOF manipulator can be applied to the 5 DOF manipulator and
compare the performance between the two.
• Perspective: Expand the position control systems to include static obstacle avoid-
ance, and based on these results determine the viability of a dynamic obstacle
avoidance system and thus the potential for collaboration.

5.1
Reinforcement learning in a continuous

Reinforcement learning is a branch of deep learning, the task of which can be summa-
rized as:
”Identifying the optimal action to perform in all possible configurations of the environ-
ment”.
The reinforcement learning structure used throughout this thesis, is the single agent
setup which consists of two parts: an environment and an agent.
The agent must learn how to solve a specific task by interacting with the environment
and observing the consequences of its actions via a reward.
Figure 5.1: The general reinforcement learning loop between agent and environment.

5 Deep Deterministic Policy Gradient
Figure 5.1 shows how this interaction is structured:
1. At time t the agent receives the current state St, and reward Rt.
2. Based on St the appropriate action, At is selected.
3. The environment receives and performs the action, resulting in the new state St+1.
4. At time t + 1 the agent receives the new state St+1 and the reward Rt+1 which
describes the effect of the action At.
Thus the Agent / Environment symbiosis provides the basis for causation based learning,
as it captures both the cause At and the effect St+1 while providing an assessment Rt+1.

Q learning, Watkins et. al - 1992 [WD92], is an off policy reinforcement learning al-

The Q value, also known as Q function or action-value function is a measure of the

Actor - Critic
The actor critic approach combines the policy-based method with Q learning, by split-
ting the reinforcement learning problem into two segments:
The Critic (Q) estimates the action value function, analogues to Q learning.
The Actor (µ) updates the policy distribution along the trajectory that the Critic indi-
cates.
With this separation it became possible to combine methods from Q learning with the
policy gradient methods, which is the principle behind the Deep Deterministic Policy
Gradient algorithm.

5.3 Exploration / Exploitation

5.3
Exploration / Exploitation
A central aspect of reinforcement learning is the ”Exploration/Exploitation” dilemma:
We are about to select an action, but do we want to explore our action space, or choose
the current best known action to solve the task?
In order to learn how to control the robotic manipulator across a range of situations
that captures the entire state / action space interception, the system must explore these
spaces while learning.
At the same time we also wish to spend a non insignificant portion of the total training
time in the vicinity of the target.
As the DDPG algorithm is an off policy approach we are free to choose an exploration
process independent of the algorithm.
Our initial choice was to use process noise, where we add a small random noise component
to our action before each step.
By using 0 mean process noise, the overall direction of the movement is predominantly
controlled by the actions of the neural network, as the noise effectively cancels out across
a large number of steps.
Thus the robotic manipulator will initially move across a large section of its action space
because the actions of the neural network will not have a general direction, until the
training starts to converge.
At which point the actions of the agent will start driving the position of the end effector
towards the target, thus increasing the time spend per episode in this area, while the
continued presence of the process noise ensures that the area is investigated.
Later this noise was augmented with a distance (d) and velocity (v) dependencies -
equation 5.9, such that less noise was introduced when the end effector approached the
target at a low velocity, allowing the system to experience a reward associated with the
accomplishment of a success criteria.
noise = noise · 2v + noise · 2d
(5.9)
5.4
Learning while doing
Another advantage of the actor - critic approach is that it is well suited for a ”learning
while doing” setup, as the tasks of choosing what action to perform and training the
network are naturally separated.
Thus at any time the best case actor can be deployed independently to solve a control
problem, while the system continues to collect data in the form of state transitions, and
via the soft target updates pushes improvements to the deployed actor.

6.3
Hyper parameters
The set of variables that define network structure and controls training are called hyper
parameters, all of which are initialized prior to training the neural network.
6.3.1
Learning rate
Perhaps the most important hyper parameter is the learning rate, which controls the
rate of change in the weights during training according to:
Weight update = LR · estimated weight error
Thus the learning rate controls the rate at which a network adapts to solve a given
problem, by influencing how large the impact of each update cycle is.
I.e a large learning rate will allow a network to learn faster, at the risk of arriving at a
sub optimal solution (Local minimal).
Reversely if the learning rate is too small the network will never converge, as each ob-
servation is not given enough consideration to generate a trajectory in the state solution
space.
This behaviour can be observed on figures 6.5 and 6.6.
Figure 6.4: Various learning rates applied to the same network structure.

6.3.2
Dropout
To reduce overfitting deep learning systems often rely on dropout, which introduces a
probability that a given neuron in a network is temporarily disabled, along with its
incoming and outgoing connections, thereby varying the network architecture during
training.
The result of this variation is that the significance of a specific neuron fluctuates, as their
influence on the outcome changes, which essentially means that the training process
becomes noisy by introducing additional variance, thus reducing potential overfitting
issues.
However in reinforcement learning the input signals are the states of the system, which
generally already exhibits a large degree of variation, which is further increased by the
previously mentioned moving target problem.
Therefore we did not rely on dropout in our networks, as weight decay can also reduce
overfitting without adding noise to training process.
6.3.3
Weight decay (L2)
L2 weight decay is the sum of squared weights and this is another method which aims
to reduce over fitting by decaying the weights by a small factor after each update step,
thus preventing the individual weights from exploding in value, which is desirable as
large weights are generally a sign of over fitting - The network has become fixated on
solving a specific situation.
A suitable weight decay was experimentally determined to be:
Weight decay = 0.0001

Perspective
This chapter presents the results of implementing static obstacle avoidance to the po-

presented results, the perspective with respect to dynamic obstacle avoidance and thus

8.1
Static obstacle avoidance
Due to time constraints collision detection was reduced such that it only detected if the
end effector was inside a set boundary surrounding a 10 cm tall wall placed along the X
axis at X = 0.15, as seen on figure 8.1 instead of the matrix based implementation.
Figure 8.1: Penalty based on simple collision detection.
==PAGE:68==
52
8 Perspective
8.1.1
Experience replay complication
While the experience replay component of the DDPG algorithm provides independent
and identically distributed training data, a consequence of its random sampling nature
is that all transitions are considered equal.
With each training round consisting of 64 samples extracted from the replay buffer which
contained 50.000 transitions, the probability of observing collisions frequently enough to
be able to impact overall system behaviour was unacceptably low.
To compensate for this we introduced a unique transition variable into the system, such
that when a collision occurred the transition related to this event was stored both in the
replay buffer, and as the unique variable.
During training the number of samples extracted from the replay buffer was then reduced
to 63, and the 64th transition was instead provided by the unique variable.
This ensured that the special circumstance was always taken into consideration when
updating the network weights.
8.1.2
Wall collision results
Figure 8.2 shows how the manipulator moved during training of a 3rd gen system prior
to the inclusion of collision detection, and it can clearly be seen that the movements are
focused around the 9 goal zones.
Figure 8.3 is the 4th generation of the system on figure 8.2, but with collision detection
enabled, which showcases the results of our attempts to apply collision detection:
The penalty associated with wall collisions caused the system to predominately black-
list the three goal positions located closest to the manipulator along the X axis, thus
avoiding collisions.
Tests of the trained system revealed that the highest success rate of three positions was
position 2, at just 10 [%], but the path to each of those ”successes” also contained a
collision with the wall.
Increasing the collision penalty simply increased the probability of the three goal fields
being blacklisted, and decreasing it caused the system to retain its ability to reach the
goal positions but with a guaranteed collision, when the target was one of the goal fields
that required the manipulator to cross the wall.
Similar results was observed when training a clean system with collision detection added
before training was commenced.
Thus we were forced to conclude that static obstacle avoidance was not feasible with the
tested methods and the constraints on network size introduced by the available comput-
ing power.
This was also a confirmation of our suspicions regarding the fixed height movements
encountered in the path analysis of chapter 6, as seen on figures 6.10 and 6.12.

Evaluation
8.2.1
Dynamic obstacle avoidance
Based on the unsatisfactory results of the static obstacle avoidance tests, we did not
proceed with the implementation of dynamic obstacle avoidance.
Our findings have therefore lead us to conclude that the methods applied in this thesis
towards solving the reinforcement learning problem, does not appear well suited for
static nor dynamic obstacle avoidances.
Despite these findings we cannot definitively state that a system containing several
thousand neurons per layer could potential achieve better results, but the computing
power required to test this hypothesis was not available in this project.
8.2.2
Physical manipulator
Due to the complications with the software implementation and the resulting time con-
straints, the physical manipulator was not manufactured, but the design work was not
in vain as it provided the premise for delineating the playfield, such that it reflected a
realistic real world setting.
==PAGE:71==
CHAPTER 9
Conclusion
We have confirmed that a control system based on reinforcement learning is feasible
for a robotic manipulator in a continuous environment, and that the behaviour can be
influenced via reward shaping.
Furthermore our results proved that heterogeneous controllability can be achieved, thus
potentially eliminating the need for manipulator specific control design.
Despite these results we did not achieve a position control system with comparable
performance to classic control systems, nor was the DDPG based system capable of
static obstacle avoidance, thus voiding the dynamic collaboration aspect.
The final conclusion is therefore that while reinforcement learning shows potential in the
field of robotic control, a system built on the methods investigated in this thesis does
not appear suitable for real world implementation, based on our findings.
We must however note that our results may have been affected by the network size
constraints imposed by the available computing power.
9.1
Final remarks
Reinforcement learning and classic control theory are more interconnected than they
may appear at first glance.
In fact the core concept of reinforcement learnings agent - environment interaction can
be considered directly related to the field of adaptive control.
Furthermore the back propagation used to update the weights of the neural network
could also be interpreted as an abstract type of feedback control.
