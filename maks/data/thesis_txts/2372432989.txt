Wave Force Identiﬁcation Using OMA
Kim Petersen
Abstract
Operational modal analysis (OMA) is an eﬃcient technology for identiﬁcation of modal
parameters. In traditional experimental modal analysis (EMA) both input and output is
measured, whereas in OMA only the output is measured. This makes OMA useful for
large structures where the input cannot be measured. Still the force input can be of interest
for instance in analysis of remaining fatigue life or stress estimation from a ﬁnite element
model (FEM). This thesis is focused on estimation of wave forces using the structural
response. The problem statement is:
Can the wave forces acting on a model structure be identiﬁed based on re-
sponse measurements using operational modal analysis?
It is attempted to solve the problem by using the frequency transform of the convolution
integral. In the frequency domain the FRF is inverted in order to estimate the frequency
domain input force. Then the frequency input force is inverse transformed to obtain the
time domain response. This is implemented using two diﬀerent approaches. One method
(method 1) just as described above. In the second method (method 2) the input force is
assumed to be the product of a force distribution vector and a scalar function of time. Both
methods can be considered to be deterministic methods, where it is assumed that the FRF
can describe the relation between input and output completely.
A simulation model is used, to assess how well the force estimation methods respond
to noise in the structural response signal. In the simulation it is found that both of the
proposed methods can estimate the load accurately, assuming that the noise in the response
is not excessive. The force estimation methods are then used to estimate loads from wave
tank tests performed in Newcastle. First the modal parameters are estimated using both
an FEM and OMA identiﬁcation methods. The estimated modal parameters are then used
to identify loads from harmonic waves, irregular waves and breaking waves. It is found,
that the proposed methods can estimate the total load variation in time with correlation
between estimate and measurements in the order of 0.8 to 1.0. In all tests it was found
that the magnitude of the total force could not be estimated correctly. The relative error
between estimated and measured force was in the order of 0.11 to 0.36. For the moment the
relative error varied from 0.02 to 0.061. It is concluded, that the incorrect force magnitude
is likely due to incorrect scaling of the estimated mode shapes. It is also concluded that,
neither of the applied methods are able to identify the actual distribution of forces. A
more probable distribution of forces can be obtained by using the method 2. Although
assuming a constant force distribution has the limitation of only being able to produce
force estimates, that are ﬁxed ratios of the assumed distribution.

Chapter 1
Introduction
Problem statement
Can the wave forces acting on a model structure be identiﬁed based on re-
sponse measurements using operational modal analysis?
OMA is a very eﬃcient technology for identiﬁcation of modal parameters of a wide range
of structures, where traditional modal testing methods either are very diﬃcult to use or
does not work at all [5]. In OMA it is only the response, that is measured, and the input
force is unknown.
In monitoring of oﬀshore structures it is of interest, to know what the environmental
loads is. The wave forces can be used for assessing the fatigue load or updating a FEM,
which ultimately will inﬂuence business decisions. Applying the actual forces on an up-
dated FEM can also be used to estimate stress and strain. However very little is known
about the actual wave forces on oﬀshore structures. Design codes for oﬀshore structures
base the estimation of wave forces on linear wave theory and the Morison equation [9,
13]. The Morison equation can predict the quasi-static component of the wave force quite
reliably. It is not so good for predicting the slamming or impact part of breaking waves.
Experimental research measuring wave forces have been conducted, in attempts to ﬁnd
a relation between wave kinematics and wave forces on structures. This has mostly been
done using dynamometer or pressure transducers to record the force or pressure from wave
impact [7, 18]. The problem is that on full scale oﬀshore structures, it is not possible to
mount force transducers. Instead of measuring the actual force the structural response can
be measured using either accelerometers, strain gauges or other kinds of response sensors.
Then the response can be used to estimate the actual load.
There have been developed methods that can estimate the loads on a structure based
on output only. The method used in this thesis was used in [2] to estimate the load from a
hammer impact experiment. Some researchers are attempting to solve the force estimation
problem by expanding the measured response to an updated FEM and then estimating the
load [17]. In recent years a lot of attention has been given to the approach of using a state
space formulation and Kalman ﬁltering to identify loads from output only [10], [12].
This thesis is focused only on deterministic estimation methods, that make use of the
response measurements and the FRF of a structure. Two methods for identifying wave
forces are used in this thesis. Both methods are directly or indirectly using the inverse
FRF matrix, to ﬁnd the wave forces based on a measured response. The Fourier transform

forces can then be compared to the actual force that was measured.
1.1
Thesis Outline
In chapter 2 the most important theoretical framework is presented. This includes the con-
struction of the FRF matrix, modal scaling and inversion of ill posed matrices. Addition-
ally a couple of appendices that elaborate on some of the theory used has been included.
Appendix A is a brief overview of dynamics, appendix B is a short description of OMA.
In appendix C some of the signal processing methods used extensively in the analysis is
presented, and in appendix D the topic of transforms (primarily Fourier transform) is de-
scribed. In chapter 3 the methods of force identiﬁcation are presented in greater detail. In
chapter 4 the methods presented in the preceding chapter is used to perform simulations
in order to assess the robustness of the force identiﬁcation when the response is noisy. In
chapter 5 the results of the analysis on the Newcastle data is presented. First the modal
parameters are estimated based on a FEM and OMA using the data collected in the New-
castle tests. The force identiﬁcation have been performed on three wave types: harmonic
waves, irregular waves and breaking waves. Chapter 6 provides a more elaborate inter-
pretation and discussion of the analysis, as well as observations made during the analysis,
which might not be immediately apparent by looking at the results. Chapter 7 provides
a conclusion for the thesis. Appendix E contains a source code listing for some of the
functions developed and used throughout the thesis. Appendix F is a source code listing
for ANSYS macros used to set up the FE models and perform the modal analysis used in
the chapters 4 and 5.

• Least squares regression.
• Using singular value decomposition.

Chapter 3
Force identiﬁcation methods
In this chapter the methods to perform the force identiﬁcation are presented and discussed.
Two methods are used.
• Estimation by deconvolution / inversion of FRF (method 1).
• Estimation by using an assumed force distribution (method 2).
Both methods can be considered deterministic force identiﬁcation methods that rely on the
FRF to describe the relation between input and output.
3.1
Method 1: Deconvolution by FRF Inversion
This method will onward be referred to as method 1.
A mathematically intuitive solution to force identiﬁcation is to perform the deconvo-
lution of the convolution integral in frequency domain.
𝐘(𝜔) = 𝐇(𝜔)𝐗(𝜔) →𝐗(𝜔) = 𝐇−1(𝜔)𝐘(𝜔)

methods might not work. Based on the approach presented in section 2.3, the FRF can be
inverted by using the pseudo inverse.
𝐗(𝜔) = 𝐇+(𝜔)𝐘(𝜔)
(3.2)
Pseudo inversion will typically circumvent the problem of singular or close to singular
FRF matrices.
Using the SVD pseudo inverse has the beneﬁt of having a mechanism to cope with
close to zero singular values by only inverting singular values above a deﬁned threshold.

One limitation of this method is that in most cases it will only be able to predict the
sum of forces, but not the distribution of forces. Consider the case of an oﬀshore platform
on a monopile. The static system can be idealised as a cantilever beam with a large mass
on top. The wave loads will act where the wave slams on the monopile. However if the
response is recorded on the topside and on the pile the greatest displacements would be
found on the topside. In this case direct deconvolution by inversion of the FRF would yield
the greater forces on the topside because that is where the greatest response is measured.
Physically the force is distributed very narrowly in the area where the wave impinge on
the pile.
Another central problem is that when using the SVD pseudo inverse the tolerance of
singular values to invert has a very large impact on the estimated load. A good rule of
thumb to avoid getting too much noise in the force estimate is to invert one singular value
per mode in a given frequency band. In the case that there is no closely spaced modes
it is only the largest singular value that is inverted and everything else is forced to zero.
If there are two closely spaced modes then the guideline would be to include the same
number of singular values as the number of modes in a limited frequency band. The
intuitive interpretation of this guideline is that the FRF is used only to identify forces in
frequency bands where there is modal information (peak from natural frequency). It is
typically also around the natural frequencies that the structural response will be focused.
Following this guideline will cause problems when trying to estimate loads from a peak
excitation frequency outside the frequency band of modes. In that case it must come down
to a matter of good engineering judgement.

Based on the simulation results it would be easy to jump to the conclusion that method

distribution is obtained. So it must be expected that this level of performance only can be
obtained in a simulation. Still both the platform and Newcastle simulation proves that
both methods are quite robust even with noise added, as long the appropriate parameters
for force estimation are used.

5.2
Harmonic waves
Intuitively the simplest case to perform wave force identiﬁcation on should be the harmonic
wave. However harmonic waves force the response at a frequency that might not be close to
the frequency bands of modes. This can make it diﬃcult to identify forces if the harmonic
frequency fall outside the frequency range where the FRF has meaningful information.
Common wave periods for wind generated waves are in the order of 3 - 15 s translating
to a wave frequency of 0.3 - 0.06 Hz. In the Newcastle tests the harmonic tests have
frequencies in the range 0.5 - 0.7 Hz.
The acceleration data is decimated to by a factor of 28 bringing the Nyquist frequency
30
==PAGE:45==
down to 4.98 Hz 2. The data is lowpass ﬁltered 0.2 Hz above the excitation frequency.
The data is then integrated with cutoﬀfrequencies 0.1 Hz above and below the harmonic
frequency.
For the purpose of estimating the forces it was found that only inverting the ﬁrst sin-
gular would result in the most accurate estimation. Lowering the tolerance causes the
individual forces to be out of phase and some in anti phase. The cause of this behaviour
might be due to the relative scaling of the diﬀerent mode shapes making the modal con-
tribution from higher order modes larger than what is actually the case. It seems fair to
assume that in the case of harmonic waves the load primarily will be focused at still water
level, and then assuming a decay towards zero below the water surface. In the context
of potential wave theory the logic motivating this assumption is that the velocity proﬁle
has this shape, and for linear waves the drag force is proportional to the velocity. Two
diﬀerent force distributions are proposed a linear decay and a logarithmic decay which are
sketched in ﬁgure 5.8 and 5.9. This implicitly assumes that there is no force contribution
on accelerometer 4 and 5 since they both are placed above the water level.

and irregular waves. Therefore this force distribution is not used further in the analysis.
Instead the simple decaying force distribution used for the other wave types will also be
used here.

Chapter 6
Discussion
In this chapter the methods used and the analysis of the Newcastle data is discussed more
generally.
Incorrect scale of estimated forces.
In the analysis of the Newcastle data it was gen-
erally found that the correlation between estimates and measurements was relatively high
(0.80-1.00). However in all cases there was an error in the magnitude of the estimated
forces compared to the measured ones. The relative error varied from 0.11 to 0.46 for
force and 0.02 to 0.61 for moment. Having a high correlation and simultaneously having
a high error suggests that the scaling of the model used to estimate forces are wrong. This
interpretation is also supported by inspecting the time series plot in ﬁgures 5.10, 5.12 and
5.16 where the estimated loads are close to the measurements in terms of temporal dis-
tribution but lack the magnitude of the measurements. There might be more explanations
why this is the case.
• The mode shapes are scaled incorrectly. As shown in the identiﬁcation of the
modal parameters it was observed that the modal mass matrix was not an exact
identity matrix. This alone does not prove that the scaling is incorrect, since the
modal mass is based on the mass matrix from the FEM. It has to be remembered
that the mass matrix from the FEM might not represent the actual structure that was
used in the tests. As argued elsewhere the FEM is not modelling damping or the
presence of water in the tank. Another approach to scaling the model would be to
use the mass change method. In the case of model tests this approach is certainly
possible whereas for real structures it is a more involved procedure. In the case of
real oﬀshore structures the testing engineer is also likely to have a FEM available.
If the purpose of the testing is to update the FEM it should be expected that the
existing FEM is inaccurate. So using the existing FEM to scale the experimental
mode shapes should be done cautiously.
• The identiﬁed mode shapes are not the operational mode shapes. In the iden-
tiﬁcation the quality of the mode shapes were evaluated based on the MAC value.
This is not an inherently poor way of evaluating the quality of the mode shapes. In
this case the operational mode shapes were evaluated against the mode shapes from
the FEM. This way of evaluating the mode shapes suggests that the modes from
the FEM should correspond to the ones found experimentally using OMA. As dis-
cussed before the model used in the experiments might not be very well represented
41
==PAGE:56==
by the FEM. So it could be the case that the experimental mode shapes should not
necessarily correspond to the analytical ones.
As stated in the analysis of the Newcastle data the force is primarily focused at the fre-
quency of excitation, and around the ﬁrst mode. So even if the higher order mode shapes
are incorrectly identiﬁed the error related to incorrect scaling higher order modes will for
practical purposes be negligible, when trying to identify the input force.
Energy distribution of force measurements and estimate.
In the analysis of the New-
castle tests it was generally found that the energy distribution of the measured force was
dense around the excitation frequency and the modes of the model. The identiﬁed forces
also had a very dense energy distribution at the frequency of excitation, but the force
spectrum was generally more ﬂat at the frequency of the modes. Physically it seems more
appropriate that the input force is distributed as was seen for the identiﬁed force. If the
problem is considered in more general terms then the input force spectrum should be some-
what proportional to the wave spectrum. Typically the wave spectrum will have a single
peak frequency1, which the estimated force spectrum more closely resembles compared to
the measured force spectrum. So it can be argued that the estimation methods presented
are better at capturing the spectral distribution of forces compared to direct measurement.
The reason that this artefact is present in the measured force spectrum is that the force
measurements are based on measurements of strains. The force estimate is also based on
response measurements, but the estimation itself is the product of the inverse FRF and the
response. The inverse FRF will have a negative peak at the modes which makes sense
because it will take less force to cause a response at the natural frequency.
Eﬀect of using diﬀerent force distributions.
During the analysis a couple of diﬀerent
force distributions have been proposed. It was found that as long a somewhat sensible
distribution was used the identiﬁed total force would turn out quite well. Even for the case
of the breaking wave where it was stated that the proposed distribution sketched in 5.15
performed worse compared to the distribution shown in ﬁgure 5.9, the actual diﬀerence
in estimated reaction force was marginal. This is probably not a universal truth. It is very
likely that the assumed force distribution will be of greater importance in cases that are
more complex than a controlled one dimensional wave tank test. Furthermore in the case
that the identiﬁed loads is going to be used for estimating stresses from a FEM the actual
force distribution will be of greater importance. This is so because the location of forces
will aﬀect local strains and stresses. Using a force distribution that is constant to estimate
load might also be a too crude assumption in many cases. In the harmonic wave tests it is
probable that an assumed constant distribution could be used to accurately estimate loads.
In more common wave conditions such as irregular waves using an assumed constant load
distribution vector might result in a ﬂawed estimation of the force distribution.

Chapter 7
Conclusion
The problem formulation stated in the introduction was.
Can the wave forces acting on a model structure be identiﬁed based on re-
sponse measurements using OMA?
To answer this question two methods to identify the loads based on response data was
used.
• Direct deconvolution of forces (method 1).
• Using an assumed force distribution (method 2).
Both methods rely directly or indirectly on inversion of the FRF matrix. By perform-
ing simulations on a model platform it was concluded that the loads could be estimated
accurately as long that the response signal was relatively noise free. Simulations on the
Newcastle model showed that the forces could be estimated quite accurately even at SNR
as low as 15 dB if suitable parameters was used. In the analysis of wave tank tests from
Newcastle it was concluded that the measured and estimated forces correlated well, but
there was an error in the magnitude of the estimate relative to the measurements. Both of
the methods applied were able to estimate the temporal distribution of loads with correla-
tion coeﬃcients between the measured and estimated forces of 0.80 - 1.00. Method 2 was
in most cases marginally closer to the actual magnitude of the reaction force. For the force
estimate the relative error ranged from 0.11 to 0.36. For the moment estimate the relative
error ranged from 0.02 to 0.61. It would seem that the reason for the error in magnitude of
the estimates were related to an incorrect scaling of the mode shapes used to build the FRF.
Based on these observations it is concluded that it is possible to estimate the total wave
force assuming that the response measurements are relatively noise free and a well scaled
FRF matrix is available. In terms of spatial distribution of loads method 2 obtains more
probable load distributions, but is at the same time limited by being restrained to force
distributions that are ﬁxed ratios of the assumed distribution. Using method 1 to estimate
the force the spatial distribution of forces will in most cases be wrong, since the identiﬁed
forces are proportional to the measured response in a given location. In the analysis this
resulted in force contributions in DOFS where the wave was not present.

Appendix B
Short overview of OMA
OMA is the use of operational measurements of the response to perform modal analysis.
This is in contrast to experimental modal analysis (EMA) where both input and output is
measured. For OMA to work the following axioms are assumed to be true.
Axiom 1 The structure is being excited by an excitation that has white noise characteris-
tics.
Axiom 2 The structure response is Gaussian distributed
Axiom 3 The structure should be exposed to stationary input process, in the statistical
sense.
Axiom 4 The correlation function of the structure response can be interpreted as a free
decay.
B.1
White noise
For OMA to work it is required that the output has white noise characteristics. White noise
is characterised by having a ﬂat spectrum across all frequencies. It is not necessary for the
excitation to be exactly white noise. What is important is that the excitation has broad
enough spectrum to suﬃciently excite the modes of interest.
B.2
Central limit theorem
The second requirement for OMA to work is that the response is Gaussian distributed.
This is ensured by the central limit theorem (CLT) which can be formulated as follows:
Theorem 1 When independent random variables are added, their sum will tend toward a
Gaussian distribution, even if the original variables themselves does not follow a Gaussian
distribution.
This is important because the system response will tend to a Gaussian distribution even if
the underlying excitation does not follow a Gaussian distribution. The Gaussian distribu-
tion has the important feature that it can be described by only two parameters.
• The mean, usually denoted 𝜇.
53
==PAGE:68==
• The variance or standard deviation, usually denoted 𝜎2 and 𝜎respectively.
However when a structure’s response is measured the signal is zero mean, therefore the
signals probability distribution is entirely described by the variance. This is a very useful
fact since the correlation function can describe all the information of a Gaussian signal
assuming zero mean.
B.3
Stationariness
The excitation used for OMA is assumed to be stationary. This is to be understood in the
statistical sense. This has to do with the use of correlation functions. If correlation func-
tions are to be used the response signal should be zero mean and constant variance/standard
deviation. In practise it means that the source of excitation should not change during test-
ing. For instance the response distribution from a bridge carrying highway traﬃc will be
diﬀerent depending on the type of traﬃc which can be composed, quite diﬀerently.
• Many small vehicles
• Few large trucks
• Train traﬃc
• Etc.
In order to ensure that the response distribution has stationary properties the measurements
should be segmented such that there is a homogeneous variance in the response.

Correlation Function

Appendix C
Signal Processing
Most of the data collected in the Newcastle tests are digital signals. So before the data is
analysed the signals should go through some processing. This chapter is primarily based
on [6] chapter 8.
C.1
Filtering
Filtering a signal is the process of removing some unwanted component of the signal.
Most often that is removing unwanted frequencies. Usually the ﬁlter types can be divided
in to three main types.
1. Lowpass ﬁlters that excludes frequencies above a certain cutoﬀfrequency. Used for
removing high frequency energy in relation to decimation.
2. Highpass ﬁlters that excludes frequencies beyond a certain cutoﬀfrequency. Used
to remove DC oﬀset in signal.
3. Bandpass ﬁlters that excludes frequencies outside a frequency band. Used to divide
a signal with many modes in to smaller frequency bands with fewer modes.
Since what is being dealt with is digital signals only digital ﬁltering is considered. Tra-
ditionally digital ﬁltering is designed as inﬁnite impulse response ﬁlters (IIR) and ﬁnite
impulse response ﬁlters (FIR). These types of ﬁlters are very useful for realtime signal ﬁl-
tering. In the case of OMA and analysis of structural response signals, the need for realtime
ﬁltering is not necessary. Instead one can rely on FFT ﬁltering, in which the time series is
transformed to frequency domain, multiplied by a ﬁlter response function (window) and
then inverse transformed to time domain. This also circumvents some of the drawbacks of
digital ﬁlters such as phase distortion and ripple close to the cutoﬀfrequency. One thing
to remember when using FFT for signal ﬁltering is that the signal needs to be periodic,
since only circular signal convolution can be performed in this way as opposed to linear
convolution. The periodicity can be ensured by applying a tapering window on the time
signal such that the beginning and end of the ﬁnite signal tapers toward zero. Additionally
the signal can be segmented and then each segment can be tapered by a window and the
full signal can then be synthesized from the segments.

𝑓𝜈= 𝑓𝑠

Most often a signal will be oversampled which means higher than the required Nyquist
frequency. However if the frequency of interest is less than 10 % or greater than 90 %
of the Nyquist frequency the frequency of interest will be diﬃcult to analyse. So if the
signal for instance is sampled at 160 Hz the Nyquist frequency will be 80 Hz, in that case
only the frequency band between 8 and 72 Hz can be analysed properly. If however the
frequency of interest is 1 Hz it is necessary to at least bring down the Nyquist frequency to
10 Hz. That is to reduce the number of samples by a factor of 8. Simply downsampling by
keeping every eighth sample and calling it a day will cause aliasing, since high frequency
components will leak in to the signal. So in order to downsample a signal by a factor of 𝑛
the following two step procedure is performed
• Filter high frequency signal components above the reduced Nyquist frequency (also
known as anti aliasing).
• Downsample by keeping every 𝑛sample.
This will ensure that high frequency components outside the Nyquist band will not be
present. This procedure is also known as decimation.
In the case of the Newcastle tests the sampling interval was 𝑑𝑡= 0.000391s so the sam-
pling frequency is 𝑓𝑠= 2557.54Hz, which makes the Nyquist frequency 𝑓𝜈= 1278.77Hz.
The frequency band that can be analysed without decimation is 128 to 1150 Hz. The ﬁrst
mode of vibration has a natural frequency in the order of 2 Hz, so the signal will in most
cases need to be decimated by a factor of roughly 26 = 64.
C.3
Detrending
Usually the zero reading of electronic sensors is unreliable due to noise in the very low
frequency range. This is because of very low frequent oscillations due to for instance
temperature variations. The result is a DC oﬀset, which is handled by detrending the
signal. Detrending can be performed by high pass ﬁltering the signal with a suitable cutoﬀ
frequency. Another option is to segment the data, remove the mean of each segment and
then reassembling the signal. To avoid signal discontinuities overlapping segments are
tapered with a window that is zero at the extremities and unity in the middle.

For structural response measurements it is not feasible to record both acceleration, veloc-

twice to obtain displacement. One option is to use a numerical integration scheme such as
cumulative trapezoid integration.
y = cumtrapz(x, t)
However these methods can produce very large errors and are susceptible to drift.
Another option is to use the approach of FFT ﬁltering since the diﬀerentiation property of
the Fourier transform gives.

D.3
Fast Fourier Transform
The discrete Fourier transform has the major drawback that the naive computation will be
an algorithm of complexity 𝑁2. Which means that doubling the amount of samples will
quadruple the number of operations. The fast Fourier transform (FFT) is a divide and con-
quer algorithm that partitions the DFT in to smaller DFTs and adds them up in the end. In
theory the FFT algorithm only has complexity 𝑁log2 𝑁, which is a very large improve-
ment. Consider a signal sampled at 𝑑𝑡= 0.0005s and a sampling duration of 𝑇= 60s,
which will be 𝑁= 𝑇/𝑑𝑡= 120000 samples. Using the naive approach to compute the
DFT will require 1.44⋅1010 operations whereas the FFT will require 2.02⋅106 operations.
That is an improvement of 4 orders of magnitude. Another beneﬁt is that the reduced

shapes correlate [3]. It is calculated according to the following equation:

|𝐚𝐻𝐛|2
(𝐚𝐻𝐚)(𝐛𝐻𝐛)

F.1
Newcastle ﬂexible model
The ﬂexible Newcastle FE model is built of mainly beam elements with a hollow circular
section. In addition a single mass element is added to the top as was the case for the actual
model used in the wave tank tests. At the bottom the model is ﬁxed against translation and
rotation. The force transducer is represented by a one dimensional rotation spring. This
is because the team that performed the test in Newcastle found that the natural frequency
of the model was not as high as expected. This is likely due to the fact that the boundary
conditions did not perfectly ﬁx the model against rotation at the support. The spring ele-
ment stiﬀness is calibrated such that the ﬁrst mode has natural frequency close to that of
the actual model. To simulate the eﬀect of added mass from the surrounding water a small
amount of density is added to the submerged beam elements. Fluid structure interaction
and damping is not modelled which could be a signiﬁcant source of error.
