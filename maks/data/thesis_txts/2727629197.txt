Abstract
Outdoor navigation for autonomous mobile robots is a rapidly developing field of re-
search. It holds practical significance and values in many areas. This project devises a
navigation system that enables the robot to navigate on an outdoor running path using
3D vision techniques. The system lets the robot correctly perceive and understand its
surrounding world and finds safe and driveable routes driving along detected paths.
Through the RGB-D camera, the robot collects RGB and depth images to generate
point clouds that present its surrounding environments. After that, a 3D plane segmen-
tation method RANSAC and a deep learning model SPVCNN are applied to identify
obstacles, paths, and other ground terrains. Then the robot creates grid maps based on
the identification results to represent the semantic environment and relative positions.
The Hybrid A∗planner then applies to the grid maps, helping the robot find suitable
driving routes.
Through testing, the system displays reliable performance and fulfils project objec-
tives, and some improvements are come up with for further research in future.

CHAPTER 1
Introduction
The autonomous mobile robot is a robotic system capable of autonomous movement
and task execution. It gets widely applied in various fields, including industrial automa-
tion, healthcare, agriculture, logistics, and services. Navigation is a core technology to
achieve these tasks for enabling autonomous movement in autonomous mobile robots.
Autonomous mobile robot navigation refers to the ability of autonomous mobile robots
to autonomously perform path planning, obstacle avoidance, and accurate localization
without human supervision or intervention.
In its early development stage, autonomous mobile robot navigation relied on preset
paths. Robots would move along predefined paths, typically guided by physical guide
wires or floor markings. This method was suitable for simple industrial production lines
but could have been more effective in complex environments.
Then with advancements in sensor technology, autonomous mobile robots began
using sensors to perceive the surrounding environment for navigation, for example, Light
Detection and Ranging (LIDAR) for obstacle detection, distance measurement, and
cameras for visual perception. The robot can analyze sensor data to calculate its position
and plan suitable paths.
Autonomous mobile robots started utilizing mapping and localization technologies
to achieve more accurate navigation. Maps provide structural and feature information
about the environment, while localization technologies determine the robot’s position
on the map. Typical localization technologies include Global Positioning System (GPS),
Inertial Navigation Systems (INS), and visual localization. Combining maps and local-
ization allows the robot to plan paths more accurately and avoid obstacles.
Finally, with the advancement of artificial intelligence and machine learning, au-
tonomous mobile robot navigation has gained autonomous decision-making and learn-
ing capabilities. Based on changing environments and task requirements, the robot can
learn and optimize its navigation and path planning in real-time. It enables autonomous
mobile robots to adapt to complex environments and autonomously solve navigation
problems.
Based on the above contents, this project aims to devise a reliable system for mobile
robots autonomously navigating on a chosen outdoor path which needs to find solutions
to let the robot perceive surrounding environments, map the perceived environments,
and plan a route to the destination with an obstacle avoidance mechanism.

1.1
Objectives
In order to achieve the project’s aim, the navigation system should fulfil specific objec-
tives so that the robot can self-decisions for environment sensation, mapping and path
planning. These objectives are listed below:
• To allow the robot to perceive surrounding environments.
• To detect obstacles and recognise paths from different terrains based on perceived
environment information.
• To map the path and obstacles with accurate locations relative to the robot.
• To find a suitable map route so the robot can follow the path.
• To enable the robot to avoid obstacles while driving.

CHAPTER 2
Analysis
This Chapter carries on the analysis to decide how to achieve the objectives described in
Chapter 1. It includes the descriptions of the environment where the robot gets operated,
the robot platform on which the navigation system gets tested, the methodologies to
solve problems which may get encountered when designing the navigation system, mod-
ules and software for implementing those solutions, as well as the state of art researches
of autonomous mobile robot navigation.
2.1
Environment
One of the prerequisites of robot navigation is the running path. This project decides to
operate on an outdoor running path with complex terrain environments. In this way, the
navigation system can gain more remarkable universality. A running path surrounding
the DTU Lyngby campus gets selected on this basis. The environment and location
of the running path are as Figure 2.1 shows. This gravel-like trail encircles the whole
campus, with several pavements and motorways crossing it. Varies with seasons, grass
may grow on both sides of the path. It should consider many obstacles along the path,
such as trees, stones, signs, rubbish bins and pedestrians. Besides, falling leaves and
shadows may cover the path.
2.2
Hardware
This project uses the Terrain Hopper mobile robot owned by DTU Electrical and Pho-
tonics Engineering Department as a platform for test. The picture of the DTU Terrain
Hopper robot is shown in Figure 2.2. The Terrain Hopper is an autonomous robot that
navigates uneven and rough terrain, including sand, gravel, and rocky surfaces. It has a
suspension system allows it to adjust to uneven terrain and maintain stability. It also
has the robot control software - Mobotware running on Ubuntu 18.04 operating system.
Terrain Hopper is a large vehicle of about 1.8m × 0.8m, giving it enough space to equip
the camera and sensors. Although its weight reaches 170kg, the driven ability of Ter-
rain Hopper is enough for undertaking many tasks on the chosen path. Its maximum
steering angle is around 33◦, and its maximum speed is 1m/s. However, it still has some
shortcomings due to its large size and heavy weight. The Terrain Hopper can diﬀiculty
run on narrow tracks, sharp turns and steep slopes.

2.3
Related Work
The study field of navigation for autonomous mobile robots has developed rapidly and
has gained significant attention in recent years. Many studies related to autonomous
mobile robots have contributed to advancements in robot navigation and inspired later
research. Therefore, it is essential to review recent research in this field to get guides on
the navigation system design of this project.
The camera is a commonly used sensor to help the robot perceive and understands
the surrounding environments. Especially the use of RGB-D camera makes it possible
for robots to construct 3D scenes containing position and shape information of objects

2 Analysis
Path planning enables the robot to make informed decisions on how to navigate and
reach its goals effectively with consideration of environmental factors, constraints, and
desired objectives. [17] presents the Hybrid A∗algorithm designed for autonomous vehi-
cles operating in unfamiliar environments with real-time obstacle detection by employing
the A∗search algorithm[18] that tailored to the kinematic state space of the vehicle and
Reed-Shepp model[19]. [20] applies the Hybrid A∗algorithm in largely uncharted and
potentially intricate environments that enable the robot to navigate structured and un-
structured regions. [21] proposes a flexible multi-criteria adjustment to the Hybrid A∗
algorithm. It enhances obstacle avoidance by incorporating a smooth evaluation of col-
lision danger using danger fields and improves node expansion to cover the robot’s state
space more evenly.
2.4
Problem Analysis
This project’s critical factor is letting the robot correctly distinguish paths and obstacles
from the environment presented in Section 2.1. In order to handle this task, first, to
represent the perceived environment, there is a choice between a 2D image and a 3D
point cloud. However, 2D images have limited spatial information and require addi-
tional processing and inference to obtain depth and distance information. Besides, 2D
images are more susceptible to variations in viewpoint and lighting, often requiring ad-
ditional preprocessing and data augmentation techniques to improve robustness. These
drawbacks make 2D images less advantageous in outdoor environments.
Compared to 2D images, 3D point clouds directly represent objects’ geometric shape
and position in the environment while preserving the distance and relative positional
relationships between objects. On the other hand, point clouds can model objects from
point cloud data captured from different angles and lighting conditions, exhibiting a
certain degree of robustness to viewpoint and lighting variations. Therefore, this project
chooses a point cloud to represent the world that the robot sees and to do further
processing.
In 3D point clouds, objects usually have different surface normal vectors. For exam-
ple, the ground is vertical to the objects on it, such as trees, pedestrians, and dustbins.
The running path and the ground it belongs to can be regarded as in the same plane,
but the obstacles are not. According to this characteristic, the robot can filter obsta-
cles employing 3D plane segmentation in point clouds like Random Sample Consensus
(RANSAC).
After implementing 3D plane segmentation, the robot also gets a plane which is a
ground where the running path is. Apart from the path, the ground may contain other
terrain types like pavement, motorway, grassland, and bare surface. Different terrain

CHAPTER 4
3D Plane Segmentation
4.1
Introduction
In a point cloud, a plane refers to a collection of points with consistent normal vectors
and approximate planar shape. 3D plane segmentation divides the points in a 3D point
cloud data into sets corresponding to different planes. The process helps understand
the geometric structure of the scene or object represented by the point cloud and ex-
tract information specific to individual planes from the point cloud. In robotics and
autonomous driving, segmenting the point cloud of the environment into planes can
help identify plane structures like roads, obstacles, and buildings.
The commonly used methods for 3D plane segmentation include model-fitting-based
methods and region-based methods. Model-fitting-based methods attempt to fit mathe-
matical models, such as plane models, to the point cloud data and determine the plane
parameters by optimizing the fitting results. Region-based methods segment the point
cloud into contiguous regions with similar features like normal vectors, colour, or curva-
ture and label these regions as different planes.
In this project, as discussed in Section 2.4, the robot must find the ground where the
running path belongs to detect the path. This project adopts RANSAC to do 3D plane
segmentation to find such a ground. It is an algorithm based on model-fitting-based
methods. After the RANSAC process, the point cloud is expected to be segmented into
two parts: the ground and the other objects.
4.2
RANSAC
In this project, the world that the robot sensing can be classified into two categories.
One is the earth’s surface. It is a ground with multiple terrains such as running paths,
pavements, motorways, grassland, and bare surface. Another category is other objects,
including trees, stones, dustbins, signs, poles, and pedestrians.
The ground can be
regarded as a plane in point cloud data, which means the robot can implement RANSAC
on the point cloud to separate ground and other objects. Due to other objects not being
in the same plane as the ground and the robot always driving on the ground’s surface,
it implies that the robot can only drive through other objects with collision. In other
words, the other objects can be considered obstacles in total. Implementing RANSAC
means the robot can directly find the ground and filter obstacles from point cloud data.

4 3D Plane Segmentation
RANSAC is an iterative parameter estimation algorithm used to estimate the optimal
model parameters from a data set containing noise and outliers. It is widely used in 3D
plane segmentation and often serves as the fundamental algorithm for implementing it.
The RANSAC algorithm can segment plane structures from point cloud data through
iterations and model fitting.
In 3D plane segmentation, RANSAC is utilized to estimate the parameters of the
plane model, such as the plane’s normal vector and origin position. It randomly samples
a small set of points, fits a plane model, calculates the distances between the model and
other points, and classifies the points as inliers if their distances fall below a predefined
threshold or as outliers if they do not conform to the model. Through multiple iterations,
RANSAC seeks the best-fitting plane model and filters out the outliers that do not adhere
to the model.

CHAPTER 5
3D Semantic Segmentation
5.1
Introduction
3D semantic segmentation refers to the pixel-level semantic labelling of a point cloud
in a 3D scene, where each point is assigned a specific semantic category. This seman-
tic labelling allows the robot to identify and segment different objects, regions, and
parts in the scene, enabling a fine-grained understanding of the scene. To achieve 3D
semantic segmentation, deep learning techniques and specific network architectures are
commonly used. Deep learning models learn semantic features and spatial relationships
from large-scale annotated training data to realize semantic understanding and segmen-
tation prediction of point clouds.
In this project, through 3D semantic segmentation, the robot should recognize var-
ious elements such as paths, pavements, and motorways within the ground. SPVCNN
architecture is utilized to carry on the task of achieving such results. This chapter de-
tails the processes of accomplishing 3D semantic segmentation, including building the
SPVCNN model, training and optimizing.
5.2
SPVCNN Architecture
5.2.1
Architecture
SPVCNN is a deep learning network model that processes point cloud and voxel data.
It combines the advantages of point cloud data and voxel data, enabling effective infor-
mation propagation and feature extraction between the sparsity of point clouds and the
density of voxels. Let it can handle irregular point cloud data and be computationally
eﬀicient.
The architecture of the SPVCNN is illustrated in Figure 5.1. As the figure shows,
it comprises two branches: Sparse Voxel-Based Branch and Point-Based Branch. Each
branch consists of many blocks. The Sparse Voxel-Based Branch has a U-Net-like struc-
ture. It can capture information across various receptive field sizes by utilizing sparse
convolution. At the same time, the Point-Based Branch maintains a high-resolution
representation and retains intricate details of expansive outdoor environments.
The
information on two branches converts to each other seamlessly with minimal computa-

Loss Function
The loss function is a function used to measure the discrepancy between the predicted
output of the model and the actual label. The loss function is a crucial component in
model optimization by serving two primary purposes: model evaluation and model op-
timization. It assesses the model’s performance by evaluating the dissimilarity between
the predicted and actual values. A lower loss value indicates better consistency between
the model’s predictions and the actual values. Besides, the value of the loss function
is used as the objective function for the optimization algorithm. The optimization al-
gorithm utilizes the gradient information of the loss function to update the model’s
parameters, gradually reducing the loss function’s value and improving the model’s pre-
dictive capability.

5.4.2
Optimizer
The optimizer is an algorithm that automatically adjusts the model’s parameters during
training to make the model’s predictions as close to the actual labels as possible. Deep
learning models typically consist of many parameters, such as weights and biases in
neural networks. The function of the optimizer is to optimize the model by continually
adjusting these parameters during training to minimize the loss function. The optimizer
calculates the gradients of the parameters and determines the direction and magnitude
of parameter updates, gradually optimizing the model’s performance.
This project uses Adaptive Moment Estimation with Weight Decay (ADMW)[23]
algorithm as the optimizer in the training of SPVCNN. It is an improved version of the
Adaptive Moment Estimation (ADAM)[24] optimizer, combining the concepts of mo-
mentum and weight decay to improve the model’s convergence speed and generalization
ability. The core idea behind the ADAMW algorithm is to balance the direction and
magnitude of parameter updates by combining momentum and weight decay. It main-
tains the continuity and stability of parameter updates through the momentum term
and controls the complexity of the model through weight decay.

learning rate to meet the requirements of different stages of training. Typically, a larger
learning rate is beneficial in the early stages of training as it helps to explore the loss
function space and accelerate model convergence quickly. As training progresses, grad-
ually reducing the learning rate allows the model to make finer parameter adjustments,

This experiment uses the OneCycleLR method to schedule the learning rate for
each batch. OneCycleLR is based on the concept of ”one cycle policy”[25], where the
learning rate gradually increases to a higher value and then decreases back to a lower
value throughout the training process.
Let η0 be the initial learning rate at the beginning of training, ηmax be the highest
desired value of the learning rate during the training process, ηmin be the lowest desired
value of the learning rate during training, Tcurr is the value of current training steps, Td
is the value of training step after which the learning rate begins to decrease from the
maximum value to the minimum value, and Tm is the number of total training steps.

CHAPTER 6
Mapping
6.1
Introduction
Mapping is the technique of creating a map of the environment using the robot’s per-
ception system and motion information. The map is an abstract representation of the
spatial structure and features of the robot’s surroundings. Mapping plays a crucial role
in outdoor navigation by providing the robot with an understanding of the spatial struc-
ture and distribution of obstacles in the surrounding environment. This information is
essential for tasks such as path planning and obstacle avoidance.
In this project, the robot utilizes obstacle information from Chapter 4 and inference
results from Chapter 5 to determine its position and construct a representation of the
environment by creating grid maps.
6.2
Grid Mapping
Grid Mapping is a commonly used technique in the outdoor navigation of autonomous
mobile robots for modelling and representing the environment. It divides the environ-
ment into a regular grid of 2D cells, where each cell represents a discrete area and stores
information about its attributes and states. Grid maps are constructed by continuously
updating the information based on real-time vision data from the robot, enabling adap-
tation to dynamic environments. It also provides a straightforward and intuitive way
to represent the environment, making it easy to understand and use. Meanwhile, by
dividing the environment into discrete grid cells, grid maps facilitate the processing and
analysis of environmental information and can be applied to navigation tasks.
In this project, the grid maps are created to represent the robot’s surrounding envi-
ronment. The procedures to build such maps are the following:
• Step 1: Reconstruct the point cloud to represent the perceived world the robot
views from the camera. The point cloud comprises obstacles, paths, movements,
motorways, and unlabeled ground obtained from the 3D plane segmentation and
3D semantic segmentation modules. Each point in the point cloud has labelled
with different colour features in terms of its category.

CHAPTER 7
Path Planning
7.1
Introduction
Path planning is the technique used to find a path that allows the robot to reach the
target point while avoiding obstacles and adhering to constraints such as safety, eﬀiciency,
and smoothness. The path planning algorithm considers factors such as the environment
map, the robot’s motion capabilities and limitations, obstacle positions and shapes, and
task requirements to generate a path that satisfies the conditions.
In this project, the robot utilizes the Hybrid A∗algorithm to plan and select routes to
navigate a perceived environment autonomously. The robot can get path and obstacle
positions from grid maps generated by the mapping module. As discussed in Section 2.4,
the robot should satisfy three tasks: driving along the path, driving through the pave-
ment if the pavement separates the path, and stopping navigation when encountering
the motorway. These factors and the enormous size of the robot introduced in Section
2.2 are all considered when planning the routes.
7.2
Hybrid A∗Planner
The Hybrid A* Planner was first introduced in the 2007 DARPA Urban Challenge, and
then its insights and algorithm were made public in 2008. As a global path planning
algorithm, it plans paths from the start to a goal in a known environment. By combining
the advantages of discrete search and continuous search, it can eﬀiciently explore the
search space and find a path that satisfies given constraints and objectives.
The Hybrid A* Planner comprises robot constraints, node expansion, and analytic
expansion. In this project, the grid map, a discretized representation of the continuous
state space, is where the Hybrid A∗Planner operates.
In order to plan the routes,
the widely-used conventional A∗search algorithm is employed to explore the kinematic
state space of the robot. Meanwhile, the analytic expansion is applied to adapt the A∗
algorithm to generate the robot’s continuous state within the search’s discrete nodes.
This hybrid discrete-continuous path planning strategy enables the Hybrid A∗Planner
to adapt to robot non-holonomic constraints and effectively search for paths in complex
environments.

7.2.3
Analytic Expansion
A discretized set of steering angles is used in the node expansion process explained
earlier. As a result, the search will not directly reach the exact goal state with continuous
coordinates. To address this precision issue and further enhance the search speed, the
Hybrid A∗Planner incorporate analytic expansions into the search based on the Reed-
Shepp model[19].
The Reeds-Shepp model is a commonly used curve model that describes the shortest
route of a vehicle with upper bound curvature, finite turning radius and limited steering
operations. Applying the Reeds-Shepp model, the routes with a maximum of two cusps
can form a set of possible routes. Each route consists of a series of line segments and
circular arcs, representing the motion route from an initial state to a target state, and
the number of segments is at most five. The minimum length route in the set of possible
routes is the solution.
Apart from expanding nodes in the manner introduced above, some states are aug-
mented with an additional successor state by computing an optimal Reed-Shepp route
==PAGE:73==
7.3 Module Test
63
from the current state to the goal, assuming an environment free of obstacles. The gen-
erated Reed-Shepp route is then checked for collisions through Eq 7.1 and Eq 7.2. Only
if the route is collision-free the search terminates. However, applying the Reed-Shepp
expansion to every state is computationally expensive. This project implements the
Reed-Shepp expansion every N th iteration to reduce the computational burden.

CHAPTER 8
Test
8.1
Introduction
The test is a critical step in ensuring system quality and performance. It helps this
project uncover issues, validate system correctness and functionality, and provide valu-
able data for system optimization and improvement.
In this project, however, the navigation system does not include a control module to
command the robot directly. For this reason, it conducts tests by manually controlling
the Terrain Hopper robot on the running path surrounding the DTU Lyngby campus
to validate the whole navigation system’s performance. During the test, the system’s
inputs are RGB images and depth images acquired from an Intel D455 camera, process-
ing through five functional modules; the outputs should be planned routes for Terrain
Hopper safely drive along the paths.
8.2
Scenarios
This project carries on four tests in different scenarios.
The first scenario is on an
ordinal running path. The path covers both straight and bend trails, with trees and
grasses growing on two sides of the path. This scenario tests the navigation system’s
performance on an ordinary path. The second scenario locates where the path ends
with a motorway to test whether the navigation system can correctly command the
robot to stop before it drives on the motorway. The third scenario is where a running
path crosses with a pavement. It tests whether the navigation system can command the
robot to drive across the pavement to the opposite path. In the fourth scenario, it tests
the navigation system’s reactions when a pedestrian walks on the path. The locations
of four scenarios are shown in Figure 8.1.
During all tests, to reduce the effects of computation latency on the robot, the
system processes only 1 frame of images per second, and the robot is driven manually
at a relatively slow speed.

CHAPTER 9
Future Work
This project only covers some aspects of outdoor autonomous mobile robot navigation,
and there are some methods this project does not study but might improve the navigation
performance. These possible improvements are worth being further researched in future
and are listed below:
• Improve the plane segmentation: When there are potholes on the ground, or
the ground is not even enough, the plane segmented by RANSAC will not contain
the whole ground. The uneven ground parts and potholes are wrongly recognised
as obstacles and become unpassable even though the robot can safely drive across
them. It can obstruct the robot from finding routes in path planning. This problem
may get settled by identifying and incorporating potholes and uneven ground parts
or turning to another 2D plane segmentation method.
• Let model adapt to more weather conditions: The SPVCNN model is trained
with data acquired in only sunny and cloudy weather. Herefore, the model might
display poor inference ability in other weather conditions such as snowy and rainy.
The solution to this is collecting more data in snowy and rainy weather and then
training the model with it.
• Merge generated maps to one large map: The grid maps generated in this
project are separate single maps. The shortcoming of such a map is that it discards
past environment information and cannot plan navigation based on a comprehen-
sive understanding of the environment.
The possible methods to merge maps
include Visual Odometry and Simultaneous Localization and Mapping (SLAM).
• Navigating to a fixed goal: This project chooses different goals on separate
maps. However, in practice, the robot is usually needed to navigate to a fixed
destination. This problem may get solved by incorporating the GPS and utilising
GPS coordinates.
• Add loop closure detection and fork selection: In this project, the robot
follows the running path, neglecting the situations of driving into a loop and two
or more paths crossing together. Therefore, introducing loop closure detection
and fork selection techniques can significantly improve navigation performance.
A commonly used loop closure detection method is Bag-of-Word.
As for fork
selection, it might get solved by incorporating forks into a representative global
map and then applying the Hybrid A∗planner to choose a suitable route.
==PAGE:86==
76
9 Future Work
• Optimise computation speed: The navigation system has relatively high la-
tency between the input of RGB-D images and the output of navigation paths.
The latency will affect the navigation performance of the robot.
The possible
methods to solve it include changing the model to reduce inference time and im-
proving the path planning algorithm to reduce path search time.

CHAPTER 10
Conclusion
This project devised a navigation system to enable the DTU Terrain Hopper robot to
drive autonomously along the running path around the DTU Lyngby campus.
The
system was divided into five modules with different functions.
Firstly, the robot perceived the surrounding environment through the data acquisi-
tion module. The module used an Intel D455 camera to capture RGB and depth images.
Then the data acquisition module generated the point clouds to represent the world that
the robot saw based on these images. Next, the 3D plane segmentation module utilises
the RANSAC method to segment grounds and obstacles for point clouds. Meanwhile,
the 3D semantic segmentation module built and trained the SPVCNN model to detect
paths with other terrains on the grounds. The segmentation results were merged into
semantic point clouds. Based on semantic point clouds, the mapping module created
the grid maps using the grid mapping technique. Through the path planning module,
after implementing the Hybrid A∗planner, the safe and driveable routes were found on
grid maps for robot navigation.
With a series of tests on each module and the system, the objectives of this project
were all fulfilled as follows:
• Objective 1: To allow the robot to perceive surrounding environments.
This objective was achieved through the data acquisition module. With an Intel
D455 camera mounted on the robot, the robot could acquire RGB and depth
images to generate point clouds representing perceived surrounding environments.
• Objective 2:
To detect obstacles and recognise paths from different
terrains based on perceived environment information. This objective was
achieved through 3D plane segmentation and 3D semantic segmentation modules.
The obstacles and grounds were recognised and separated through RANSAC. Then
using the SPVCNN model to make inferences for grounds, the paths and other
terrains like pavements and motorways were detected.
• Objective 3: To map the path and obstacles with accurate locations
relative to the robot. This objective was achieved through the mapping module.
Based on segmentation results, a grid mapping method was developed to create
grid maps with semantic information and precise location for each grid.
• Objective 4: To find a suitable map route so the robot can follow the
path. This objective was achieved through the path planning module. The Hybrid

A∗planner was applied here to find safe and driveable routes on grid maps. With
the help of the Hybrid A∗planner, the robot could plan such a route not only
following the path but also allowing the robot to drive across the pavement and
stop when encountering the motorway.
• Objective 5: To enable the robot to avoid obstacles while driving. This
objective was also achieved through the path planning module. The routes planned
by the Hybrid A∗planner enabled the robot to avoid obstacles on the two sides of
the paths, such as trees and poles, while bypassing obstacles on the running path
like pedestrians.
