Summary
In this thesis we design a data structure supporting random access queries on
a collection of highly repetitive strings represented as a tree of m edits, namely
insertions, deletions and replacements. Let mI and mD be the number of in-
sertions and deletions in the tree, respectively. Our data structure uses linear
space in the number of strings belonging to the collection and answers random
access queries in O(log log m+log(mI +mD)) time. In order to accomplish this
query time we present a solution for answering a variant of the range selection
query. Given a sequence of integers and natural numbers i and j, this variant of
range selection reports the ith lowest integer among the values occurring exactly
once during the ﬁrst j positions of the sequence. We also present specialized
solutions for when the tree of edits are restricted to a path and only some of
the edit types.

Chapter 1
Introduction
Highly repetitive data can be viewed as a text string over an alphabet of symbols.
We can chop this highly repetitive text into very similar strings. We will in this
thesis describe such a collection of similar strings by a tree of edits whose root
represents one of the strings. Each node in the tree also represents one of the
strings in the collection such that the string of this node is the same as the string
of its parent except that one of the following edits has been made: insertion,
deletion or replacement.
A collection of DNA from diﬀerent people is an example of highly repetitive data
for which the chopping is obvious. Moreover, we reckon that the relationship
between DNA of people in a group ﬁts well with the tree structure relation
imposed by the tree of edits. A random access query given DNA of a group of
people would then be: What is the 1001173th base pair in Stacy’s DNA?
Version control of e.g. a text document is another example of highly repetitive
data. Each time a letter is typed or deleted in the text document, a new version
of the text is created for which we support random access. However, in this
thesis we restrict ourselves to a version of the problem where the entire tree of
edits is known before creating the data structure and hence we do not support
later updates.
We do not deal with the inconvenience that the diﬀerent nodes in the tree can
==PAGE:12==

represent the same string, i.e if the same symbol is inserted and deleted at the
same position multiple times in a row. In the case of just storing DNA this
would be a problem of designing the tree such that as few nodes as possible
share string.
In the process of designing a solution for random access given a path of edits, we
solved restricted versions of the problem. We restricted the tree to be a path and
we restricted the type of edits allowed. Not only did these simpliﬁed problems
function as a natural todo list during the project, but for path of insertions and
path of deletions they lead to specialized solutions, which achieves faster query
times than if our more general solution for path of edits was used.
We do not investigate construction time of the presented data structures, but
we do analyze the time of the transformations we present.
After this introduction we formally deﬁne the problem of the thesis in Chapter
2, namely to support random access on a collection of strings represented as a
tree of edits.
Chapter 3 is the preliminaries and deals with notation, queries relevant for our
solutions and data structures used to answer those queries. In the beginning
of the preliminaries we motivate its content by mentioning which restricted
problems they help to solve.
In chapter 4 the solutions for the diﬀerent restricted versions of the problem
together with the full problem are presented. As a general approach, we solve
the path restricted version of the problem and transform the tree to a path using
a depth ﬁrst traversal of the tree.
Finally, in Chapter 5 we summarize the presented solutions and make a sugges-
tion for further work, which will improve the query time of both random access
on a path of edits and on a tree of edits. Here you will also ﬁnd a table of time
and space for the presented solutions of the diﬀerent restricted problems.

Chapter 2
Problem Description
In this section we present the problem of the thesis, namely to support random
access for a collection of highly repetitive strings represented as a tree of edits.
2.1
Random Access using Tree of Edits
An instance of the problem is a string, S, and a tree of edits T where each
node is given an id from the set {0, ..., |T| −1} according to a pre-order walk
of T. Each node v of the tree (except the root) holds string edit information
consisting of three parts: edit type, index information i and symbol information
c. We consider three diﬀerent edit types to perform on a string, S:
Replacement: A replacement rep(i, c) performed on S overwrites the symbol
at position i in S by the symbol c. Thus creating the string S[1, i−1].c.S[i+1, n].
Insertion: An insertion ins(i, c) performed on string S shifts all symbols in
S[i, n] one position to the right causing the length of the string to increment.
Then we assign the symbol c to the now empty position i in S. Thus creating
the string S[1, i −1].c.S[i, n].
==PAGE:14==
4
Problem Description
Deletion: A deletion del(i) performed on a string S shifts all symbols in S[i +
1, n] one position to the left causing the length of the string to decrease by one
and the value S[i] has now been overwritten by S[i + 1]. Thus creating the
string S[1, i −1].S[i + 1, n]. Note that this edit type has no symbol information
associated.
Consider now again the tree of edits T. An example is seen on Figure 2.1. Each
node v in T represents a version, Sv, of the string S. The root of T represents
the original string S. The string Sv is obtained by walking the path from root
node to node v while performing the edit of each visited node on the string
belonging to the parent of that visited node.
We assume that the index of any edit from T is valid. Meaning for rep(i, c)
or del(i) performed on string S we assume that i ∈{1, ..., |S|}, and for ins(i, c)
performed on string S we assume that i ∈{1, ..., |S| + 1}.
Together the string S and the tree of edits T represent a collection of strings.
We want to support fast random access queries; RandomAccess(v, i) returning
Sv[i] for v ∈T, on this collection of strings while using little space.

Chapter 3
Preliminaries
In this chapter we introduce notation and underlying algorithms and data struc-
tures used in the report.
We now motivate the diﬀerent sections of preliminaries. We use label predecessor
queries to answer random access queries when replacements are available. We
use predecessor queries when both insertions and replacements are available. We
use range select queries when only insertions are available. We use our adapted
range select query when both insertions and deletions are available. The adapted
range selection reports the ith lowest value among the values occurring exactly
once in a preﬁx of a sequence. The solution, which we present in section 4.4,
for this query is based on a wavelet tree representation of the sequence.
3.1
Notation and Concepts
• With log(x) we refer to log2(x).
• We assume all arrays to be 1-indexed.
• Let A and B be strings. Then A.B is the string: A concatenated with B.

• Let the array S of size n be a permutation of the integers {1, ..., n}. The
inverse of S is named S−1 and is deﬁned as S−1[S[i]] = i for i ∈{1..n}.
• Let B be a bitvector and j a natural number, then the query rank1(B, j)
returns the number of ones in B[1, j]. Likewise, rank0(B, j) returns the
number of zeroes in B[1, j].
• Given a tree T with labeled nodes and a label i, the label ancestor of a
node v ∈T has two properties: it is an ancestor of v and it has label i.
Note that we assume that v is an ancestor of itself.
• A subsequence of a sequence S is any sequence which can be created by
deleting some of the elements in S. The number of deletions can be zero.
3.2
Word RAM Model
The space and time analysis of the queries and data structures are carried out
using the word RAM (Random Access Memory) model.
In the word RAM
model we have access to an inﬁnite memory of computer words, which can be
accessed by indexing as shown in Figure 3.1. Each computer word consists of w
bits. The model assumes it takes constant time to read/write a word from/to
memory to/from local memory of the CPU. The memory directly available by
the CPU consists of a constant number of words.
In the model the CPU can perform boolean operations and shifts on a word
in constant time. Thus allowing constant time basic arithmetic operations on
the numbers which a word can represent.
The model assumes that for a problem of some size n the size of the computer
word is w ≥log n bits. To get a feeling of this assumption and the "capacity"
of a computer word, consider that the human DNA consists of approximately 3
billion base pairs [HGP]. Since log(3 · 109) ≈31.5, we can represent the number
of base pairs in the human DNA within one 32 or 64 bit computer word.

3.3
Label Predecessor
The static predecessor problem on a set S ⊆U = {1, ..., u} is to support prede-
cessor queries:
Predecessor(x): Return the largest element in S that is less than or equal to
x.
Let n be the number of elements in S.
The static predecessor problem can
be solved using Y-fast tries in O(n) space answering predecessor queries in
O(log log u) time as explained in [Bad].
The static label predecessor problem is an extension of the static predecessor
problem. Now each integer in S ⊆U = {1, ..., u} is given a label from {1, ..., σ}.
On this set of labeled integers we want to support the label predecessor query:
LabelPredecessor(x, ℓ): Return the largest element among the ℓ-labeled inte-
gers from S that is less than or equal to x.
The static label predecessor problem can be solved by partitioning S into the

Sℓwe construct a static predecessor data structure. The partition is illustrated

The query LabelPredecessor(x, ℓ) is then executed in two steps; ﬁrst the data

3.5
Range Select
The range select problem on a sequence S of n integers from {1, ..., σ} is for the
integers i, j, k, where 1 ≤i ≤j and k ∈[1, j −i+1], to answer the range select
query:
==PAGE:21==
3.5 Range Select
11
RangeSelect(S, i, j, k): Return the kth smallest symbol in S[i, j].
For the purpose of this thesis it is only necessary to consider range queries
on the form S[1, j] and therefore we redeﬁne the range select query:
RangeSelect(S, j, k): Return the kth smallest symbol in S[1, j].
One solution to the range select problem is representing S as a wavelet tree.
Then the query for RangeSelect(S, j, k) is as follows. Let v be the root of the
wavelet tree and let Bv be the bitvector of v.
If l = rank0(Bv, j) < k, then there are fewer than k symbols in Sv[1, j] belonging
to the left child and hence all these symbols are smaller than the kth smallest
symbol in Sv[1, j]. Therefore the value we seek must be the (k −l)th small-
est value in Svr. We move to the right child vr of v by updating the position
j ←rank1(Bv, j), the value k ←k −l and ﬁnally the node v ←vr.
Otherwise there are k or more zeroes in Bv[1, j], and the kth smallest symbol
of Sv[1, j] must therefore belong to the left child vl of v, since all symbols in Svl
are smaller than every symbol in Svr. We move to the left child by updating
the position j ←rank0(Bv, j) and then the node v ←vl.
This is done until we arrive at the leaf node of symbol c. Now we know that the
kth smallest symbol of S[1, j] is symbol c and that c occurs j times in S[i, j].
An example of a range selection query is shown in Figure 3.3.
The time of this range select query is O(height of wavelet tree) = O(log σ),
since we traverse from root to leaf using a constant number of constant time
rank queries on bitvectors at each level of the tree.
The space of the wavelet tree representation of S is O(n) as explained in section
3.4.

4.1
Path of Replacements
Now we examine a path of replacements. We restrict the tree T to be a path P
and we only allow the edit type replacement. We want to support fast random
access on the collection of strings represented by the original string S and the
path of replacements P.
Given version number v and string position i, the
random access query should return the symbol Sv[i]. Let s = |S| be the length
of the original string and thereby also the length of all other strings on the path.
Let m = |P| −1 be the number of replacements in the path.
Solution
Create the set {1, 2, ...m} of integers with labels. For each replacement v ∈
{1, 2, ...m}, if replacement v was made at position i of the string, give integer v
label i. Construct a label predecessor data structure for this sequence of labeled
integers. Furthermore, store the original string S together with an array R of
size m such that R[v] holds the symbol of replacement v. The grey box in Figure
4.1 shows the components of the data structure for an instance of the path of
replacements.
The query RandomAccess(v, i) ﬁrst calls u ←LabelPredecessor(v, i) for the cre-
ated set of labeled integers. If a label predecessor exists, return R[u]. If a label
predecessor does not exist, then return S[i].
Correctness
When doing a random access query, RandomAccess(v, i), the symbol at Sv[i]
must either be the original symbol S[i] or it must be the latest replacement
symbol at position i before or at version v.
If a label predecessor u ←LabelPredecessor(v, i) exists, then that label prede-
cessor must: 1) be a replacement at index i in the string and 2) it must be the
latest replacement before or at version v. Hence it holds that Sv[i] is the symbol
of the uth replacement, R[u].
If no label predecessor for the query LabelPredecessor(v, i) exists, it must be
because no replacements have been made earlier on the path. Therefore the
symbol at Sv[i] must be the original symbol S[i]. Hence the query is correct.

Tree of Replacements
Now we present the tree of replacements T. Let S be the original string belong-
ing to the root of T. As in last section, since only replacements are allowed, all
strings have the same length s. Let m = |T|−1 be the number of replacements.
We assume that each node v ∈T is given a number from {0, .., m} according to
a pre-order walk of T.

Solution
We transform the tree of replacements T to a path of replacements P and reuse
the solution for path of replacements.
The transformation is done using a depth ﬁrst walk of T as follows. Initialize
the path as empty. When a node v ∈T is visited for the ﬁrst time, node v′ is
added to the end of the current path and when the node v is visited for the last
time, node v′′ is added to the end of the current path. The ﬁrst node added to
P is the ﬁrst-visit node r′ of the root r of T. Let r′ = 0. Then each time a node
is added to the path, let its id be the id of the previously last node plus one.
Let the index information of v′ and v′′ be the index information of v.
The root still represents the original string S and thus it does not have index
information or symbol information. Except for the root, let the symbol infor-
mation of ﬁrst-visit node v′ ∈P be the symbol information of v ∈T. Let the
symbol information of last-visit node v′′ ∈P be the symbol which was replaced
by replacement v ∈T. The latter is diﬃcult to ﬁnd. Therefore we do not assign
the symbols to the last-visit nodes yet.
Even though the created path is missing the symbol information for its last-
visit nodes, create for P and S the data structure presented in section 4.1 to
support random access on a path of replacements. Just use some placeholder
information for symbol information of the last-visit symbols temporarily.
Let F be an array of |T| integers such that F[v + 1] holds the version number
v′ ∈P of the ﬁrst-visit node of v ∈T. Note that the +1 is because we do
not use 0-indexing and the lowest valued ﬁrst-visit node in T is the root with
version number 0. The array F is a mapping from node v ∈T to its ﬁrst-visit
node v′ ∈P.
The random access query for Sv[i] on a tree of replacements with v ∈{0, ..., m}
and i ∈{1, ..., s} is then:
RandomAccess(v, i) = RandomAccessP (F[v], i)
The query RandomAccessP is a random access query on the path of replacements
deﬁned by the original string S and P which we created.
Now we ﬁnd the missing last-visit symbols using random access on the almost
ﬁnished path of replacement in the order which the last-visit nodes occur in P.
By ﬁnding the symbols in this order we make sure not to query one of the missing
symbols. Let i be the index information of replacement v ∈T and let w be the
parent of v ∈T, then the symbol information of v′′ is RandomAccessP (F[w], i).

• The data structure presented in section 4.1 for supporting random access

• The mapping F which maps nodes v ∈T to their ﬁrst-visit node v′ ∈P
Correctness
As for the path of replacements, when doing a random access query, the value at
Sv[i] must either be the original value S[i] or it must be the latest replacement
at index i, which happened on the path from the root of T to v. In the context of
the tree of replacements, the latest replacement at index i is the replacement at
the lowest ancestor (including v itself) of v ∈T which has a replacement at index
i. Hence to do a random access query we must ﬁnd the lowest node among the
ancestors of v which have a replacement at index i: the lowest i-labeled ancestor
of v.
Now we prove that the random access query yields the lowest i-labeled ancestor
of v ∈T.
Since we always use the ﬁrst-visit array F when making random access queries,
we only query on ﬁrst-visit nodes, but the symbol returned can belong to a
ﬁrst-visit or a last-visit node.
Now we examine the query RandomAccessP (v′, i) with the assumption that the
predecessor returned is a ﬁrst-visit node, u′. Then the last-visit node u′′ must
be later in the path than v′, since u′ and u′′ both have label i. Hence the node
u ∈T must be an ancestor to v ∈T. This is illustrated in Figure 4.2. We now
argue why u is the lowest ancestor of v. No replacement at index i exists on the
path between u ∈T and v ∈T or in the subtree of u or in the subtree of any
node on the path between u ∈T and v ∈T. This holds since if a replacement
at index i existed any of these places, then the label predecessor query would
have returned either a ﬁrst-visit node of some node between u ∈T and v ∈T
or it would have returned a last-visit node of one of the mentioned subtrees.
We conclude that u is the lowest i-labeled ancestor of v. Since u′ has the same
symbol information as u the correct symbol is returned.

Now we present the path of insertions. Thereby restricting the tree T to be
a path P and only allowing the edit type insertion. Let m = |P| −1 be the
number of insertions in the path and let s = |S| be the length of the original

The path of deletions problem is very similar to the path of insertions problem
and can be transformed to a path of insertions problem, such that the solution
can be reused. This transformation is explained in the end of this section.

Correctness
We now argue why the query RandomAccess(v, i) is correct for v ̸= 0. We can
view Ψ as a set of coordinates as shown in Figure 4.6. Remember that Ψ[v + s]

We now provide some intuition of why the calculation of l is changed.
In
each iteration of the ordinary range selection query we ﬁnd that there are
l = rank0(Bw, v) values smaller than the rest of the values in the valid range.
However for this application there are some of these values we do not want
to count, namely the values corresponding to deletions and the original inser-
tions of those deleted elements. Therefore we subtract 2 · rank1(Dw, v). Now in
each iteration of the adapted range selection query for which we move to the
right child in the wavelet tree, we acknowledge that we have encountered the
l = rank0(Bw, v) −2 · rank1(Dw, v) leftmost elements in Sv.
Correctness of Adapted Range Selection Query
Let Ψ and D be constructed as described and represent Ψ as a wavelet tree. We
now want to prove that the adapted range selection query AdaptedRangeSelect(Ψ, D, v, i)
returns the ith smallest value among the values only occurring once in Ψ[1, v].
Let Ψw be the subsequence of Ψ for node w of its wavelet tree as described in
section 3.4. Let Bw be the wavelet tree bitvector of node w and let Dw be the
corresponding bitvector from D.
We ﬁrst derive the meaning of the new l calculation. The number of integers
in Ψw, which belong to the low half of the interval [aw, bw], is rank0(Bw, v).
Remember that bit Dw[j] indicates whether the corresponding value Ψw[j] has
the following two properties: it belongs to the low half of the interval [aw, bw]
and it is a second occurrence in Ψw[1, j]. Therefore rank1(Dw, v) is the number
of low valued second occurrences in Ψw[1, v].
The ﬁrst occurrences of these
values must also be present in Ψw[1, v]. Because of the construction of Ψ we
know that an integer can occur at most twice. Therefore l is the number of
small integers in Ψw[1, v] minus the number of small integers occurring twice in
that part of Ψ. This corresponds to the number of small integers in Ψw[1, v]
occurring exactly once.
The query AdaptedRangeSelect(Ψ, D, v, i) goes as follows. Let w be the root of
the wavelet tree.
If l = rank0(Bw, v) −2 · rank1(Dw, v) < i, then there are fewer than i 0s in
Bw[1, v] whose corresponding values each occurs exactly once in Ψw[1, v]. These
l values are all smaller than the ith smallest among the values occurring once
in Ψw[1, v]. Therefore the value we seek must be the (i −l)th smallest among
the values occurring exactly once in Swr. We move to the right child wr of w
by updating the position v ←rank1(Bw, v), the value i ←i −l and ﬁnally the
node w ←wr.
Otherwise there are i or more 0s in Bw[1, v] whose corresponding values occur

Complexity
The wavelet tree representation of Ψ uses O(mI + mD) words of space.
As
we argued in the complexity for path of insertions and deletions, the tree of
bitvectors D falls within this big-O notation.
The predecessor structure of C uses linear space in the number of elements in
C when using Y-fast tries. Hence C uses O(mI + mD) words of space.
The label predecessor data structure presented in section 3.3 on Z uses linear
space in its number of elements mD plus the number of labels mI. Hence Z
uses O(mR + mI) words of space.
The string ˆSm uses O(mI) space.
Therefore the presented data structure for supporting random access given a
path of m edits uses O(m) space.
The predecessor query on C takes O(log log m) time when using Y-fast tries,
since C can consist of integers from {1, ..., m}. Likewise, the label predecessor
query described in section 3.3 on Z takes O(log log m) time, since Z can consist
of integers from {1, ..., m}.
The adapted range query on Ψ using the presented wavelet tree solution takes
O(log mI) time.
The query time of this solution for random access given a path of edits is there-
fore O(log log m + log mI).
Remember that m = mI + mD + mR. The number of deletions on the path is
upper bounded by the number of insertions: mD ≤mI. However the number
of replacements is not upper bounded by the number of insertions.

4.6
Tree of Edits
We now look at the full problem: tree of edits.
Let T be the tree with mI
insertions, mD deletions and mR replacements. Let m = mI + mD + mR be the
total number of edits. We assume that each node in T is given a unique number
from {0, ..., m} according to a pre-order walk of T.
Yet again, let the original string S be the empty string, which can be done
without loss of generality, since any original string can be seen as a series of
insertions. Hence the root of the tree holds the empty string.
Solution
We transform the tree of edits T to a path of edits P and reuse the solution for
path of edits.
Using a depth ﬁrst walk of T we create a path P of edits. Initially let the path
be empty. When node v is visited for the ﬁrst time, add the ﬁrst-visit node v′
to the end of the path. The last time node v is visited, add the last-visit node
v′′ to the path.
The ﬁrst node added to the end of the path is r′, namely the ﬁrst-visit node of
the root r. Let r′ = 0 and each time a new node is added to P, let its value be
one plus the value of the previously last node in P. Remember that the root is
a special case, since it does not have edit type, index information and symbol
information. It represents the empty string.
Create the mapping F, which maps nodes in T to their ﬁrst-visit node in P.
More speciﬁcally let F[v] = v′ for v ∈{1, ..., m}. Note that it is not necessary
to support random access for the empty string S0.

Let all ﬁrst-visit nodes v′ in the created path P have the same edit type, index
information and symbol information as v from T.
For last-visit nodes v′′ the assignment of edit type, index information and symbol
information is as follows:
If v ∈T holds a replacement, then let v′′ ∈P be a replacement. The index in-
formation of v′′ is the same as for v. The symbol information of v′′ is the symbol
at Sw[iv], where w ∈T is the parent of v ∈T and iv is the index information of
v. We access Sw[iv] by making the query3 RandomAccessP (F[w], iv).
Similarly as to what we did in the solution for path of replacements, we can
build this data structure with placeholder symbol information for all replace-
ments. Then we are able to perform RandomAccessP (F[w], iv) to ﬁnd the correct
symbol for the replacements as long as we query and assign the symbols in the
order which the replacements occur in P.
If v ∈T holds an insertion, then let the last-visit node v′′ ∈P be a deletion and
let the index information of v′′ be the index information of v. Deletions do not
have symbol information.
If v ∈T holds a deletion, then let the last-visit node v′′ ∈P be an insertion. Let
the index information of v′′ be the index information of v and let the symbol
information of v′′ be the symbol deleted by edit v. This symbol information
is found and assigned in the exact same manner as we found the reversing
replacement symbols, namely by making a random access query for the ivth
symbol in the string represented by the ﬁrst-visit node of the parent of v ∈T.
Data structure
The data structure to support random access on a string collection represented
by a tree of edits consists of:
• The data structure presented in section 4.5 for supporting random access
given the path of edits P, which was created by the transformation of T
• The mapping F which maps nodes v ∈T to their ﬁrst-visit node v′ ∈P
Then the random access query is:
RandomAccess(v, i) = RandomAccessP (F[v], i)

Correctness
The correctness of the random access query relies on the following insight. The
edit of any last-visit node v′′ reverses the edit of v′ such that the string repre-
sented by the last-visit node is the same as the string represented by the parent
node of node v ∈T.
Now we prove by induction that the string Sv′′ represented by last-visit node v′′
is the same as the string Sv′−1 belonging to the node just before the ﬁrst-visit
node v′ on the path. The node on P just before a last-visit node will either
be another last-visit node or it will be its own ﬁrst-visit node. First we do the
inductive step and assume that Sv′′−1 = Sv′ holds. The set up of the inductive
step is shown in Figure 4.16 to the right. Furthermore, the three boxes to the
left in Figure 4.16 show that no matter which edit type v′ is, it holds that
Sv′′ = Sv′−1. The base case is that v′′ −1 = v′ as illustrated on the Figure, and
we immediately see that the assumption Sv′′−1 = Sv′ holds.
The node before v′ on the path can be one of two things. It can either be the
ﬁrst-visit node of the parent of v ∈T as illustrated in Figure 4.17 to the left.
Using what we just proved in the above paragraph, the string Sv′′ must be the
same as string Sv′−1 = Sparent(v). Else the node v′ −1 must be the last-visit
node of the sibling just to the left of v ∈T and therefore v′′ shares string with
the last-visit node of its sibling. In general we conclude that strings of last-visit
nodes are the same if their corresponding nodes are siblings in T as illustrated
in Figure 4.17. The string of the last-visit node of the leftmost sibling must be
the same as the string of their shared parent in T. This is illustrated in Figure
4.17 to the right. Hence we conclude that the string belonging to any last-visit
node v′′ is the same as the string belonging to the parent of v ∈T.
Now we have proven that last-visit nodes reverse the string such that the edit of
any ﬁrst-visit node v′ is performed on a string precisely the same as the string
belonging to the parent of v ∈T. Therefore the string Sv′ = Sv.
Hence when we use the mapping F to map the value v to the value v′, the
random access query must return the correct symbol, since Sv′[i] = Sv[i] is true
for any position i in the string.

Complexity
The path created has twice as many nodes as T, since two nodes are added to
P for every node in T. The space used to support random access on a string
collection represented by the path P is therefore O(2m) = O(m). The mapping
F uses space O(m). Hence the total space used by this data structure is O(m).
The query time of random access is the same as for random access given a path
of edits, since the only additional operation is a constant time look-up in F.
There are mI +mD insertions on the path and there are 2·m edits on the path.
Therefore we answer the random access query in O(log log m + log(mI + mD)).
It is worth noting that for the created path it always holds that the number of
insertions and the number of deletions on the path are the same, since insertions
and deletions are used to reverse each other.
Transforming the tree to a path takes O(m · (log log m + log(mI + mD))) time.
We traverse the tree adding 2 nodes to the path for each of the m nodes we
encounter.
It takes constant time to add a ﬁrst-visit node.
However, when
adding a last-visit node for an insertion or replacement, the symbol information
must be found by a random access query, which takes O(log log m + log(mI +
mD)) time. Note that the time it takes to construct the data structure to answer
random access queries for path of edits is not included in the transformation
time.

Chapter 5
Conclusion
We have presented a linear space data structure which supports O(log log m +
log mI) time random access queries on a collection of highly repetitive strings
represented as a path of edits.
We have also presented a transformation, which transforms any tree of edits to
a path of edits, such that we can reuse the solution for random access given a
path of edits. This lead to a linear space data structure which supports random
access queries given a tree of edits in O(log log m + log(mI + mD)) time.
In the process of creating the solution for random access given a tree of edits
we solved specialized versions of the tree of edits by constricting the tree to a
path and allowing only some edit types. The space and time complexities of the
solutions for the diﬀerent versions of the problem are seen in Table 5.1.
We reduced random access given a path of insertions to a range selection query
on a sequence holding the ﬁnal position of the inserted elements in the order
they were inserted. We suggested two diﬀerent linear space data structures to
solve the range selection problem: First we suggested to use a wavelet tree to
represent the sequence of ﬁnal positions, thus leading to the random access query
taking O(log m) time. However we also suggested using the solution presented
in [BJ09], allowing random access given a path of insertions to be solved in
linear space and O(
