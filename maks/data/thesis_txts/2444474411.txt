1.1
Machine Learning: a brief introduction
1.1.1
Categorisation of ML problems
In order to organise the different problems faced in the Machine Learning field, three
main categories have been identified, with each task pertaining to one or more of
them. In the following we will outline said categorise, which will allow us the address
the types of challenges that will be tackled in the Thesis.
1.1.1.1
Supervised learning
We call supervised learning that set of problems that try to find a mapping form a set
of inputs xi to a set of outputs yi. In these cases, we have at our disposal a dateset,
identified by a collection of annotated annotated observations of the type {xi, yi}.
The usual methodology adopted in ML tasks implies the splitting of the full dataset
into subsets, identified as the training, validation and test sets. The training set, as
the name suggests, is used to run the algorithm of choice and perform learning. The
validation set is used to perform tuning of hyperparameters, whilst the test set, left
alone up until the end, is used to get a better estimate of the ectual generalization
error of the used model. It is possible to reformulate the Supervised learning problem
in probabilistic terms. In this case, we are interested in finding a posterior probability
p(yi|xi, θ) that represents the distribution of the targets conditioned on the observed
data. The distribution’s parameters are represented by the vector θ.
It is possible to further categorise supervised learning tasks by identifying them as
either classification or regression tasks. In the first case, our target is a discrete vari-
able yi ∈{1, . . . , K} that represents the membership of an observation xi to one of
a set of K possible classes. Examples of said tasks include image and audio classifi-
cation. In the latter case, the target is a continuous variable yi ∈R that represents
a corresponding quantity, for instance the amount of a chemical in a solution, or the
expected return in an investment.

1.1.1.2
Unsupervised learning
In the unsupervised learning setting, the set of possible tasks is similar to the one
pertaining to the supervised learning problems, but in this case no labels are available.
This noticeably complicates things, since a direct comparison between predictions
and targets is not possible. One of the most general tasks that can be tackled in
this setting is density estimation, in which the algorithm tries to model the data
distribution p(xi|θ).
1.1.1.3
Reinforcement Learning
Reinforcement Learning was defined as a third type of Machine Problem because it
didn’t quite fit in either the supervised or the unsupervised setting, with enough pe-
culiarities to be classified as its own thing [SB98]. In the RL setting the model, called
agent, has to interact with a set environment by performing a set of actions. After
performing an action, the agent changes its current state. Rewards are given to the
agent accordingly to how good or poor its performance was w.r.t. the specific objec-
tive presented by the environment. Rewards can be assigned densely, i.e. after each
step taken by the algorithm, or conversely sparsely, so e.g. only after the algorithms
reaches a terminal state.

1.2.2
The Convolutional Neural Network
The Convoloutional Neural Network (CNN) is a specialised type of Neural Network
for handling data with an inherit grid-like structure [GBC16]. Examples of such data
are time-series and images. Such models have been already around the early eighties
[Fuk80], but it’s only with results like [LeC+89] in 1989 that the model has been
popularized and affirmed as superior methods for image classification problems.

1. Sparse connectivity: each element in the output feature map is only influ-
enced from a part of the input which size is proportional to the kernel size. Since
the kernel is chosen to be smaller than the input, this gives rises to the concept
of local receptive field. A layer’s receptive field is understood as the range of
the input’s elements that contribute to the calculation of a certain feature map
element. Thus, a convolutional layer’s receptive field is determined by the size
of the chosen kernel. By stacking convolutional layers, we will then increasingly
widen the receptive field by connecting to a wider range of inputs through the
feature maps of preceeding layers;
2. Parameter sharing: convolving the kernel with the input makes it so that
the weights in k act on the entire image by ”sliding” across it. This is a form of
sharing parameters, that not only drastically reduces the amount of parameters
in a network but also makes it so that the model can exploit spatial correlation
between neighboring elements in the input;
3. Translation equivariance: by the properties of the convolution operator, we
already get that the network is equivariant to translation in its inputs, by that
meaning that a translation in the input before applying the convolution operator
corresponds to first convolving and then translating the resulting feature map.
A way to further reduce the dimensionality of the feature maps and to achieve trans-
lational invariance is represented by the application of pooling operations. Said opera-
tions act on a feature map, summarising its content by ”pooling” together neighboring
information. The most common way of performing such summarisation is through
max-pooling. Considering the two-dimensional case, the max-pooling acts similarly to
a kernel, sliding a rectangular window over a feature map and returning the maximum
element contained in it.
The usual CNN architecture comprises a concatenation of a certain number of con-
volutional and pooling layers, usually followed by a classifier/regressor that uses the
features extracted by the CNN as input. Said model is usually a FNN itself.

1.2.3.3
Optimization Algorithms
After producing an estimation of the error’s gradient w.r.t. the network’s parameters,
different algorithms can be applied to perform gradient-based optimisation. The sim-
plest algorithm of such algorithms is the Stochastic Gradient Descent (SGD), which,
given a set of weights wt at a given iteration t produces an updated set of parameters
wt+1 by moving them in the direction defined by the error’s gradient ∇wE(X, w)
with a step size defined by the learning rate η following the equation
wt+1 ←wt −η∇wE(X, w)
(1.25)
The name derives from the fact that the error’s gradient is usually dependant on the
particular set of observed data x, which makes the estimate stochastically dependant
on the training set (or batch of examples).
A common modification to the standard SGD optimiser is realised by introducing mo-
mentum. The general idea is to reduce possible sharp turns in the gradient’s direction
by adding to the current gradient’s estimate a portion of the gradient computed at
the previous iteration. The amount of gradient that is ”remembered” through itera-
tion is regulated by the forgetting factor γ. The update to the model’s parameters is
then changed to

1.2.4
Network’s regularisation techniques
There are many pitfalls in which the learning could fall short, and one in particular
that should be avoided is overfitting. This is the situation in which our model learns
a representation of the data that is too close to the one of the given training exam-
ples. This leads to a poor generalisation of the true, underlying distribution of the
data. It is up to the model’s designer to come up with methods that can prevent
overfitting. Given the usually very large number of parameters in a Neural Network,
such techniques are often paramount in order to achieve good predictions. We will
briefly present some of such regularization procedures, namely weight decay, batch
normalisation1 and dropout.
1.2.4.1
Weight decay
The most intuitive way to force the network’s parameters to be kept within reasonable
limits (i.e. not explode) is to introduce a penalisation term dependant on the p-norm
of the network’s weights in the overall error computation.
E(w, α) = E(w) + α||w||p
(1.27)
One very well understood example is the L2 norm regularisation, which penalises
the error with the 2-norm of the weights, ||w||2 = 1
2wT w. This can also be seen as
introducing a normal Gaussian prior on the weights, which translates to equation
(1.27) if one considers the prior in the calculation of the log-likelihood of the data.

1.2.4.3
Dropout
Dropout is a modern regularisation scheme [Ha12], [Sa14] specifically designed for
ANNs. It works by forcing, during training, the activation of neurons in non-output
layers to zero with probability p. This effectively creates a scheme that emulates
the training of an ensemble of networks. This methods introduces regularisation by
reducing interdependency effects of the learned wieghts for neighboring neurons.

2.1
Derivation of ES gradient through Taylor expansion
Usually Taylor expansions are used to get an approximation of a function in a neigh-
borhood of a chosen point through it’s high-order derivatives. It can also be used in
the converse way, i.e. the information about the function and it’s high order deriva-
tives can be used to derive information about the function’s gradient. We will use
this to give an intuitive derivation of the ES gradient presented in [90].
2.1.1

2.2
Variational Optimization as a generalization of the
ES gradient
We presented the derivation of the ES gradient in the case of generally ”well-behaved”
functions, that is by assuming f differentiable on its domain and x being a contin-
uous vector. While in these cases gradient-based approaches are preferred over non-
gradient based approaches, there are cases in which such techniques are not applicable
(for instance, for non differentiable f or discrete x). [SB12] We present Variational
Optimization as a general function optimization framework that poses less restrictive
assumptions on the target function by introducing a ”surrogate” objective which is
assured to be smooth.
2.2.1

2.2.2.1
Using an isotropic search distribution
In the general case, using a Gaussian search distribution with full covariance matrix
will be the choice that allows the most flexibility. This is not necessarily desirable,
since a highly flexible distribution is more prone to overfitting [J63], especially in
the case of very complex loss landscapes. In addition, using a full covariance matrix
will result in a number of parameters that exponentially grows with the number of
the distribution’s dimensions. This exponential growth quickly makes the problem
intractable, because of both computational costs (inversion of very big matrices) and
quality of estimation (a good estimation of the gradient might be achievable only
through a huge amount of function evaluation).
It’s not unusual for optimization problems to be placed in high-dimensional space:
for instance, in the case of Neural Networks optimization, the number of dimensions
corresponds to the amount of optimizable parameters (i.e. the network’s weights),

2.5
Natural gradient
Up until this point we approached the general minimization problem by applying tra-
ditional gradient-descent techniques that follow the regular direction of the gradient.
In the following, we will start by highlighting the shortcomings of the regular gradient
direction, presenting an alternative that follows the optimal search direction offered
by the optimisation of the variational bound but subject to constraints in dissimilarity
in the old and updated distributions, called natural gradient.
2.5.1
Drawbacks of using the regular gradient
One critical problem with the regular search gradients is the impossibility of precisely
locating an optimum, x∗, irrespective of the problem’s dimensionality. In fact, in
order to locate an optimum with at least a certain degree of precision, any objective
function needs to have a variance that tends towards zero, σ →0, in order to allow its
probability mass to be centred around the interested point. However, as we have seen
in the Gaussian case (take for instance (2.41)), we have that the gradients for both
the µ and σ parameters are proportional to 1
σ, so that for progressively decreasing
variance the estimate will explode. This of course is not admissible, since it would
lead to extremely unstable learning: every time the search leads in the proximity of an
optimal point, the search distribution’s gradient will start to decrease, which will in
turn yield increasingly large gradients estimate that will likely overshoot the target.
The converse is also not admissible: for very large values of the variance, σ ≫1,
the search distribution will become too wide, yielding updates with a significantly
reduced magnitude.
2.5.2
The Kullback-Leibler divergence
To understand why the normal gradient direction is so ill-behaved, we have to remem-
ber that the way regular updates are derived. Coming from the notion of steepest
gradient descent, it is computed as the direction with the maximum change in the
function value corresponding to the minimal change o the search distribution’s pa-
rameters, where the changes are all computed in terms of Euclidean distance. This
clearly points out how the choice of the Euclidean distance as a measure of difference
between distributions is unsuitable. We have thus to come up with another notion of
distance to use to compute our updates.

Reinforcement Learning
All the environments used on the Reinforcement Learning setting were made through
the OpenAI Gym toolkit [J10]. This library allows to have an unified framework in
which to train a RL algorithm, so that no changes are needed to the code itself when
switching from an environment to the other. After some initial testing, we decided
to focus our attention on two environments in particular:
• CartPole: even though this is more of a classical control problem, its relative
simplicity allows for easy training on a Reinforcement Learning setting. The
environment consists of a cart, on which a pole is attached and initialised verti-
cally, at a small angle w.r.t. the perpendicular to the cart plane. The cart has
to balance the pole by applying, at each timestep, a force of ±1. A reward of
+1 is given for each timestep that the cart manages to keep the pole within a
certain threshold angle. The event terminates either if the pole falls off or after
200 timesteps.
• SeaQuest: this Atari game presents itself as a way more complicated RL problem
compared to the CartPole, due to its larger action space and more complex
rewards.
In this game the agent controls a submarine that, with a limited
supply of oxygen at its disposal, has to rescue divers by picking them up from
their underwater location while fending off enemies by shooting torpedoes at
them. The agent can replenish its oxygen by resurfacing and immediately dies
if he runs out of it. A positive reward is given for each diver rescued and for
each enemy slain. The game ends if in a single rollout the agents dies four times
or after 10000 timesteps have elapsed.

could display at once, was displaying certain objects only on even frames and other
objects on odd ones. To solve this problem, the same preprocessing method used in
[TODO J68] was applied. Each Atari 2600 frame is a 210 × 160 × 3 RGB image. At
each timestep, a frame is encoded as the maximum pixel value between the current
frame and the previous one. This encoding is then converted to greyscale and resized

We can interpret this effect by remembering how the baseline is understood as an
operation that centers the returns around (in first approximation) the average return.
Remembering that the VO gradient estimate (2.26) using a baseline depends on the
averaging of terms (f(xi)−f0)∇θ log p(xi|θ), we can infer that the terms dominating
the expected value will be the ones in which the term (f(xi) −f0) is far away from
zero, i.e. the baseline has the effect of suppressing the contribution to the gradient
of observation with returns close to the average. This is particularly helpful in cases
where the search distribution is close to collapsing: in this situation, where the value
of σ tend towards close-to-zero values (in one or more directions, depending on the
used search distribution), we will likely have very extremes values of the log-likelihood
because of its dependency from 1
σ. While usually this leads to spikes in the estimated
gradient’s norm, using a baseline reassures us that relevant contributions to the es-
timates are given only by observations which actually carry some information about
the loss surface, i.e. perturbations that manage to deviate from the average return.
We discussed how the baseline can help in improving the algorithm’s stability. Whilst
this is a welcomed property, we will now show and discuss how, unfortunately, the
baseline can also lead to a loss in the algorithm’s performances.
Following the same line of reasoning that we used to justify the increased stability
given by the baseline, it’s not difficult to argue how in cases where this stability is
not needed the baseline is introducing a bias in the gradient estimation without signif-
icantly reducing its variance. In particular, it will not use all the information given
by the full population, using mostly extremely-performing observations to compute
the updates. Thus, we can expect a degradation in the algorithm performances (loss
and convergence speed) in those cases where learning is expected to behave prop-
erly. This can already be seen in Figure (3.7), where in the first half of the iteration
the algorithm with the baseline constantly underperforms when compared to the one
without baseline.
All the effects shown above can be also seen using the VO algorithm on the CartPole
problem but with different settings for the variance sharing, even though they are not
as noticeable as in the single case. The interested reader can find the relative plots
in Appendix [TODO: populate appendix].

The answer naturally lies in the return given by the environment. We have to remem-
ber that the effect of variance reduction achieved by introducing the baseline in the
gradient’s estimation is only the one of centering the rewards. There is absolutely no
impact on the variance of the returns themselves. If we consider the different nature
of the MNIST return (given by the NLL loss between the algorithm’s predictions and
the training targets, which, looking at our experiments, has a usual range between
[−3, 0]) and the CartPole rewards (which are integers between [0, 200]), it’s not diffi-
cult to imagine that without using fitness shaping the gradient estimates in the RL
settings will be extremely unstable. The effects of using the natural gradient on the
algorithm’s stability can be easily seen as a reduction of the learning rate, which could
prevent the algorithm from collapsing or from having an exploding gradient.

3.5
Discussion
Sharing the variance leads to instability and not noticeable increase of performances.
It seems that most of the times non adapting sigma is the most viable option. It
should be noted that the more freedom is given to the sigma parameter the more
it behaves like the unadapted version.
Some regularization techinques should be
introduced to avoid search distribution collapse.
The effects of the baseline, while not outstanding per se, become very interesting when
paired to the removal of the fitness shaping. The fact that learning is still achievable
without rt but by using a variance reduction method like the baseline highlights how
an a priori analysis of the fitness shaping should be made in order to come up with an
optimal transformations of the reward. generally speaking, the current frt proposed
seems to be empirically working, but without a solid theoretical derivation to back
it up. While the baseline is a more formal way to reduce the estimator’s variance,
