Abstract
This thesis presents a solution algorithm for the problem of identifying maximum
imbalances in a transmission system caused by the removal of k lines where
k ≤5, as presented by Energinet in [AM17]. The problem is formulated as a
type of maximum cut problem. The algorithm uses the relationship between
edge-connectivity and the number of edge-disjoint paths [Men27] to recognise
the existence of cuts in a graph, and then a local search to locate cuts.
Tests show that some variants of the solution algorithm perform better, and how
diﬀerent problem properties aﬀect the performance. The algorithm is applied
to the real-life instances provided by Energinet, and the results show that the
algorithm performs very well for small graph problems, and for large graphs
with smaller k. The problem is time sensitive, and the challenge lies in solving
large scale problems within a time limit of 15 minutes. The algorithm locates
critical imbalance cuts, and can be used to improve the current safety standard
in the European transmission system.

Chapter 1
Introduction
1.1
Transmission System
An electricity market has suppliers who produce electricity and consumers who
demand it. The commodity is distributed from supplier to consumer through a
transmission system that is operated by transmission system operators (TSO).
The TSOs are responsible for operating, maintaining and developing the network
[VD08].
Consumers are highly dependent on electricity, and the consequences of a system
breakdown can be serious, therefore safety is a primary goal for the operators.
Faults frequently occur in a power system. A transmission system therefore
operates with some power reserves so that it is able to handle some loss of
generation or consumption (load).
1.1.1
The European Power System
According to a study from 2014, [Ee14], the interconnected European (high
voltage) transmission system extends from Portugal to Poland and from Den-
mark to Turkey, and feeds a load between 220GW and 440GW. The system is
operated by many diﬀerent operators; together they are called the European
==PAGE:12==
2
Introduction
Network of Transmission System Operators (ENTSO-e).
In order to have a
safe system, the TSOs are required to operate according to the N −1 security
principle, where N denotes all pre-fault branches and −1 is for a faulty element
[Ee09]. Following this principle makes the system ’N −1 secured’; this means
that a fault in any one element (a line or shunt element) would not cause any
cascading disconnections in the system. Cascading disconnections happen when
the loss caused by a fault causes overloads in the remaining system.
Out-of-range contingencies can lead to the loss of more than just one element.
The loss of several lines can lead to a system separation where the system is
separated into two subsystems (or islands). This scenario is critical if the mis-
match between load and generation in the subsystems is signiﬁcant. Typically,
a small island will be separated from the main system, and only the island will
be aﬀected, since the main system is designed to be able to handle a loss of
3,000MW of import (generation) or 2,000MW of export (load). [AM17]. The
study from 2014, [Ee14], says that a contingency that leads to higher losses
of power is of low probability, however, it is considered conceivable. If such a
contingency occurs, the European power system is likely to experience a system
blackout, and the consequences of a system blackout are not calculable.
The ideal would be to evaluate the system for all such critical scenarios, however,
the largest graph model of the system considered in this thesis has 21,289 nodes
and 30,344 branches. The map in Figure 1.1 shows the transmission system net-
work, operated by members of ENTSO-e, with lines designed for 220kV voltage
and higher, and generation stations with net generation capacity of more than
100MW [Ent]. In addition to the large scale of the model, the power ﬂow on
the branches is changed according to the electricity markets, and every time at
least one branch fails there is a change in the topology of the graph. Evaluating
the system, therefore, is a time-sensitive task. The size of the network and time
sensitivity makes evaluating the system for all N −k contingencies where k ∈Z+
is the number of faulty branches impossible, due to the computational burden.
1.2
Problem Description
As previously mentioned, the primary focus of the transmission system oper-
ators is safety, and even if it is not possible to evaluate the system for all k
contingencies, the current N −1 safety principle might be improved upon.
The Danish TSO, Energinet, wants to go beyond N−1 safety and be able to han-
dle some out-of-range contingencies. They are therefore seeking ’a method which
is able to identify the largest imbalance (positive and negative) which occurs af-
ter removing k branches that separates the interconnected system in exactly two
separated sub systems. k can be 1...5.’ [AM17]. Moreover, the method has to

be simple and robust, and able to provide a result every 15 minutes.
Knowing the largest possible imbalance in the system in the case of k faulty
branches allows the TSO to estimate if there is a suﬃcient power reserve to
restore the imbalance within a required time limit, currently 15 minutes. In a
case where the power reserve is not suﬃcient, just being aware of the situation
allows the TSO to improve the transmission system if it is deemed necessary.
The objective of this thesis is to develop a method which ﬁnds the maximum
imbalance in a power system after splitting it into two separate systems by
removing k lines. A transmission system can be formulated as a graph; this
transformation is explained in Section 1.3.
An interconnected transmission system (as the European system) can be oper-
ated as separate islands. This means that the power balance is kept on every
island, and no electricity is imported or exported from the area. In a system
that is operated as separate islands, the diﬀerent islands are independent from
each other, therefore, each island has a separate graph representation.

1.3
From Power System to Graph
This section explains the structure and components of a transmission system
and how it relates to the graph model based on the note [Dyn18] from Energinet
(Nan Qin). In the graph model, a node corresponds to a busbar, and an edge
corresponds to the transmission lines and transformers that connect the bus-
bars. Additionally, the power ﬂowing through the branches is represented by a
directional ﬂow on the edges. Figure 1.2 shows an example of a small transmis-
sion system. The diﬀerent colors represent grids with diﬀerent voltage levels;
these grids are connected by transformers. Some components in the ﬁgure are
emphasised, and are described below.
Figure 1.2: A 14 busbar system example from [Dyn18].
Busbar:
is an electrical node represented by a node in the graph. Power can
be produced, consumed and distributed from a node, and the demand,
d ∈R, in a node can be determined based on the power ﬂow to and from
the node.
Transmission lines and Transformers:
are series elements (or branches)
which connect between two busbars (nodes), represented by edges in the

graph. Transformers connect between two grids with diﬀerent voltage lev-
els, given by diﬀerent colors in Figure 1.2, and transmission lines connect
two busbars with the same voltage level and the same color in the ﬁgure.
Generator, load consumption, and compensation components:
are
shunt elements which only connect to one electrical node (one busbar).
The current ﬂows from generators to the loads through these branches
maintaining the power balance in the network, so the total generation
is equal to the total consumption on an island. These elements are not
directly represented on the graph, but can be determined based on the
power ﬂow.
Power ﬂow:
is current which ﬂows from generators to the loads through these
branches, maintaining the power balance (it is assumed to be lossless in
this study).
Projecting the power system to a graph lets nodes represent the busbars and
edges represent the connections between the busbars. Several parallel branches
can connect two busbars, and this is the case between busbar 1 and busbar 2
in Figure 1.2. The graph representation does not have parallel edges; instead,
the edge has a capacity that shows the number of branches it represents. Shunt
elements are not directly modeled in the graph, however, positive or negative
ﬂow being absorbed into a node indicates the presence of consumption or gen-
eration. The power ﬂow between two nodes is given by the ﬂow on the edge
between two nodes. It can ﬂow in both directions of an edge, and the sign of
the ﬂow is used to indicate the direction. A positive ﬂow means that the power
follows the direction of the edge, and a negative ﬂow means that the power is
ﬂowing against the edge’s direction.
The transmission system from Figure 1.2 projects into the graph in Figure 1.3,
where the blue number on the edge is the number of branches the edge repre-
sents (i.e., capacity), and the red number is the ﬂow of power in MW, where
the ﬂows are given by the data ﬁle in Figure 1.5. Note the edge between node
1 and 2 represents two branches, as Figure 1.2 shows.

Literature Review
Section 1.3 explains how a transmission system can be represented by a graph
model. Let G(V, E) denote this graph, where V is the set of vertices and E is
the set of edges. This chapter will start looking into k-connectivity and weighted
maximum cut graph theory, as is suggested by Energinet in [AM17].
2.1
Connectivity
Connectivity can be used as a measure for the ﬂexibility or resilience of a graph,
and when talking about connectivity, it can be diﬀerentiated between edge-
connectivity and vertex-connectivity. This thesis considers the resilience of a
network in cases where branches (represented by edges) fail, therefore it is edge-
connectivity which is of relevance here. Vertex-connectivity could for example
be used in cases where the failure of busbars or shunt elements is considered.
2.1.1
Edge Connectivity and Edge-disjoint Paths
Menger’s Theorem, presented in 1927 by Karl Menger, relates the edge-connectivity
and the number of edge-disjoint paths in a graph. The edge-connectivity be-
==PAGE:22==
12
Literature Review
tween two nodes, i, j ∈V , is the maximum number of edge-disjoint paths be-
tween the two nodes. Then the size of a minimum edge cut that disconnects
node i from node j is the number of edge-disjoint paths between the nodes.
Moreover, the edge-connectivity of a graph is the minimum number of edges
whose removal cut the graph into two subsets, and it is equal to the smallest
maximum number of pairwise edge-independent paths from one node to any
other node in the graph [Men27]. Menger’s theorem can be used to show that
a graph is k-edge connected if k edge-disjoint paths exist between a node i ∈V
and every other node j ∈V on the graph.
The subject of edge-connectivity has been dealt with in many articles, and most
of them suggest algorithms for determining the connectivity of a graph. How-
ever, the graph model of a transmission system is expected to not be k edge
(branch) connected, therefore, their algorithms are not suitable for this prob-
lem. The challenge of the problem is instead to identify the edges which make a
cut after the graph is determined to be the most k-connected, and no literature
on that speciﬁc subject has been found.
2.2
Maximum Cut Problem
The maximum cut problem is a graph partitioning problem, where the objec-
tive is to partition the vertices of a graph into two subsets, in a way where
the number of edges with one end in each set is maximized - the cut size is
maximized. A more general version of the problem is the weighted maximum
cut problem, where the sum of the weight of the edges is maximized in stead
of the number of edges [Com15]. The problem this thesis attempts to solve is
formulated mathematically in Chapter 3, as a type of weighted maximum cut
problem with some extra constraints. The extra constraints are that the number
of branches allowed in a cut can at most be k, and the two subsets of vertices
need to be connected. The objective of this problem is the same as that of the
maximum cut problem, however, the solution space is more restricted because
of the additional constraints.
The weighted maximum cut problem is NP-complete, therefore, also NP-hard.
No algorithm exists (yet) that can solve the problem in polynomial time [Kar10].
A lot of literature can be found on the maximum cut problem, and many ap-
proximation and heuristic algorithms have been proposed to solve it. Since the
objective of the Maximum Cut Problem is the same as the objective of the
problem in this thesis, a optimal solution to the maximum cut problem is also
optimal if it is feasible. However, that will most likely not be the case because
of the additional constraints.

2.2.1
Maximum Cut with Limited Unbalance
The maximum cut problem with limited unbalance is a version of the maximum
cut problem where an extra constraint is considered. The extra constraint sets a
limit on the size diﬀerence between the two sets of vertices on either side of the
cut. This problem is strongly NP-hard [GM06]. In this case an extra constrain
does not make the problem easier to solve, and because the constraints are not
similar the proposed solution approaches are again not suitable.
2.3
Most Vital Links in a Network
An interesting subject related to connectivity is the problem of ﬁnding the most
vital links in a network. The article [CS82] by H. W. Corley and David Y.
Sha presents a result that is of interest. They consider a ﬁnite, directed, and
connected network with non-negative weights (distances), which is modelled by
a graph G(V, A). Given two nodes s, t ∈V , they provide the following results:
’If G(V, A) is n-separable with respect to s and t, then any set of n links in A
which separate s and t is a set of k most vital links.’ [CS82]
In this case, these sets of n links cause a cut, which is what this thesis is searching
for. They do not provide an algorithm for ﬁnding these vital links, just that the
vital links lie on some of the shortest paths between s and t.
This problem is also considered in [BnKS95], where it is proved that ﬁnding k
most vital arcs is NP-hard, even when all arcs have unit length.

The computational complexity of a problem reveals how diﬃcult it can be to
solve a problem. The maximum cut problem is NP-Complete [Kar10], which
means that a solution can be checked quickly, but there is no fast way to solve
the problem. Also, the generalized version of the k-constrained maximum cut

4.3.1
Connectivity and Branch-Disjoint Paths
A thing to note here is that since an edge can represent several branches, the
size of a minimum cut equals the maximum number of branch-disjoint paths.
The capacity, lij, of an edge (i, j) ∈E determines how many times the edge can
be used so the paths are branch-disjoint. The term branch-disjoint paths will
therefore be used from now, and means that an edge can be used as many times
as the branches it represents.
Assumption 4.1 assumes that a feasible cut (S, ¯S) disconnects a smaller set of
nodes from the main graph. This assumption can be used to argue that if the
connectivity of a node and potentially its neighbouring nodes is more than k
with regards to the main graph, the node cannot be disconnected from the main
graph. On the other hand, if the connectivity is at most k, a cut exists where
the node, and potentially some neighbourhood nodes, can be disconnected. This
connectivity of a node i ∈V together with its neighbours, with regards to the
main graph, can be determined by searching for k branch-disjoint paths from
node i to k nodes on the main graph.
Determining the existence of a cut is only the ﬁrst step of solving the problem,
thereafter potential maximum imbalance cuts need to be located. The edges
making up a minimum cut are among those used on the branch-disjoint paths
and have no free capacity left. Even though the number of edges covered on
the paths might be signiﬁcantly less than all edges in the graph, the number
of combinations might still be large, especially for the larger instances from
Energinet. The literature review did not reveal any research on actually locating
speciﬁc cuts in a way that can be used here. Another thing to note is that the
ﬂow on the branches is not considered when searching for branch-disjoint paths,
an the imbalance of a cut cannot be calculated before the edges which make
up a cut are located. This might be impractical, since the number of feasible
solutions is large, according to Observation 4.3. Also, a maximum imbalance
cut is not necessarily a minimum cut. However, the hope is that a node i ∈V
and k points on the main graph can be chosen in a wise way, which leads to
high imbalance cuts being located, even if the cut’s size is not minimum. Next,
ways to choose these nodes, and how a cut can be located after the existence of
a cut is determined, are considered.

4.3.3
Observations and Conclusions
The examples above gave an idea about how branch-disjoint paths can be used
to determine the existence of a cut.
Based on the examples, the following
observations and conclusions can be made.
• The start node, the end nodes and the paths from start node to end nodes
aﬀect which cuts are or can be found.
• Choosing end nodes far from the start node will make the paths cover
more branches, possibly allowing more cuts to be found.
• Prioritising start nodes with a node degree of at least k might make it
possible to ﬁnd branch-disjoint paths to all k end nodes.
• A high demand (positive and negative) in a node indicates that a high ﬂow
is transported to or from the node, and if this node can be disconnected
from the main graph, the imbalance of the cut is probably high. Therefore,
prioritising high demand and supply nodes as start nodes could be a way
to target high imbalance cuts.
The idea is that a set of start and end nodes is to be selected. Branch-disjoint
paths can be used to determine if a start node can be disconnected from end
nodes by k or less branches, and a cut therefore exists and remains to be located.
As long as there is still time, this procedure can be repeated.
The point of a start node is to examine if this node and maybe some of its sur-
rounding nodes can be disconnected from the main graph and cause a maximum
imbalance cut in the graph. The end nodes should be chosen as points on the
main graph; however, there is no way of knowing in advance which end nodes,
together with the chosen start node, will lead to a maximum or minimum im-
balance cut being found. It is suggested that if nodes are chosen far away from
the start node, more edges are covered. Also, the nodes will probably not be on
the same side of a cut as the start node if it is disconnected. However, increas-
ing the number of cuts that can be found with every combination of start and
end nodes also increases complexity of locating these cuts. Section 5.2 explains
more explicitly diﬀerent methods of selecting start and end nodes and how the
methods are implemented.

Implementation
The following chapter explains the main elements in the implementation of the
k-constrained max imbalance cut problem, and variations of the randomized
solution algorithm. First, the implementation of the k-constrained maximum
imbalance cut problem is described in Section 5.1, and thereafter the implemen-
tation of the proposed solution algorithm is described in Section 5.2.
5.1
Implementation of the Problem
Before the solution algorithm is implemented, the problem itself has to be de-
ﬁned and generated. A transmission system is made up of nodes, branches,
edges and islands, and the graph representation of a transmission system is de-
ﬁned based on two data ﬁles: a node ﬁle and a branch ﬁle. The node ﬁle deﬁnes
all the nodes in the transmission system, and the islands with which they are
associated. The branch ﬁle deﬁnes the branches by the nodes they connect to,
and a ﬂow that represents the amount of power ﬂowing through the branch in
MW. The edges are generated by identifying all the branches which connect the
same two nodes.
==PAGE:44==
34
Solution Algorithm and Implementation
A problem is deﬁned by an island, which is represented by a graph, G(V, E),
and a given value of k. A solution to the problem is in the form of a cut. A cut
that partitions the nodes, V , into two disjoint subsets, (S, ¯S), by the removal
of a set of edges, ¯E. This cut is feasible if each of the node sets S and ¯S are
separately connected and the cut is caused by at most k branches.
5.2
Solution Algorithm
Chapter 4 analyzed a solution approach for the k-constrained maximum im-
balance cut problem, and this section describes the most important parts of
the implementation and algorithm. The ﬂow diagram in Figure 5.1 gives an
overview of the algorithm. The solution algorithm takes a problem and a max-
imum run time, T max, as input. The ﬁrst step in the solution algorithm is to
evaluate all cuts that disconnect one node by a single edge, and since those edges
cannot be part of causing any other cut, they can be reduced from the graph.
The best positive and negative imbalance cuts are saved. For as long as time
remains, the algorithm evaluates diﬀerent combinations of start and end nodes.
A breadth ﬁrst search (BFS) method is used to search for branch-disjoint paths
and to determine if the graph is disconnected. If the graph is disconnected,
the last part is to locate cuts, and if a better maximum cut is found, the best
maximum and minimum cuts are updated. When no more time is left, the al-
gorithm terminates and returns the best maximum positive and negative cut.
The diﬀerent parts of the algorithm are explained in detail below.
5.2.1
Breadth First Search (BFS)
The breadth ﬁrst search method is a shortest path algorithm, which is used to
ﬁnd the shortest distance from a root node to all other nodes in a graph, where
all the edges have a length of 1. The BFS algorithm explores all of the neighbour
nodes at the present depth prior to moving on to the nodes at the next depth
level. The solution algorithm uses BFS many times in diﬀerent parts of the
algorithm in some of the end node methods, when searching for disjoint-paths,
when checking if a graph is disconnected, and when checking if a cut is feasible.
The BFS method is implemented by following the instructions in [KT05]. The
basic version of the BFS method is described in Algorithm 1, where r ∈V is
the root node in which the search begins.

5.2.5
Locate Cut Search
In the cases where the start node is disconnected from one or more end nodes
after the branch-disjoint paths, the existence of at least one cut is determined,
and the next step is to locate cuts. A disconnected graph consists of subgraphs
which are individually connected. Section 4.4 explained how each connected
subgraph can be represented by a node, and edges from the branch-disjoint
paths which cause the graph to become disconnected connect the subgraph
nodes. Therefore, a graph now consists of sets of nodes which are connected
by edges from the branch-disjoint paths between the start and end nodes. Let
N denote the set of node sets, and M the set of edges connecting the node
sets. The task is now to evaluate which combinations of two sets (S, ¯S) make
up feasible cuts, and hopefully a maximum imbalance cut is identiﬁed.
In the small example graph used in Section 4.4, the number of node sets is
small. The number of node sets for large graphs is also expected to be relatively
small, however number of possible combinations grows exponentially (2|N|), and
there is a risk of spending a lot of time on evaluating cuts with regards to
one combination of start and end nodes if all combinations were considered.
Fortunately, a local search should be capable of ﬁnding all feasible cut if enough
start and end node combinations are evaluated.
Local search
In a local search, the size of a neighbourhood can be determined by two param-
eters, p4 ∈[0, | ˜V |] and p5 ∈[0, | ˜V |], where p4 gives the maximum distance from
the start node. If p4 = 1, the neighbourhood includes all of the start node set’s
ﬁrst hand neighbours, and if p4 = 2, all the ﬁrst and second hand neighbours,
and so on. Also, to make sure the number of neighbours is not too large, param-
eter p5 gives the maximum number of neighbours allowed in a neighbourhood.
The algorithm for the local search method is given in Algorithm 3. The method
in line 2 uses a BFS to ﬁnd the neighbourhood node sets based on parameters
p4 and p5. All the combinations of ways to divide the local node sets into two
sets (S, ¯S) are found with the method in line 5. The for loop in line 6 evaluates
all combinations of local sets where the start node set is in the same side of a
cut. For each combination, it is necessary to consider if the cut is feasible. This

located with all the solution approaches are diﬃcult to ﬁnd, a closer look is
taken at these cuts. The properties of the optimal solutions are shown in Table

of start and end nodes to locate these cuts.
Figure 6.1 represents instance

the same way, Figure 6.2 represents instance 100n126b2.5, and the maximum

of the end nodes covers all the cut edges. The low average node degree of the
graph causes the graph to get easily disconnected before enough branch-disjoint
paths from a start node can cover these edges.

7.6
Discussion and Possible Improvements
The randomized solution algorithm has been tested, ﬁrst on a set of random
test instances in Chapter 6, and then on some real-life instances provided by
Energinet in this chapter. For the problems where the optimal solution is known,
the smaller networks and larger networks for k ≤2, the solution algorithm has
been able to solve all problems. For larger networks and k = {3, 4, 5}, it is more
diﬃcult to evaluate the performance of the algorithm since the optimal solutions
are unknown. For the solution algorithm and a given neighbourhood size, a(n)
(optimal) cut can be described by one of the following types of cuts.
Type 1 - Easy to locate:
The cut is easy to locate, since it can be found with
many diﬀerent combinations of start nodes.
Type 2 - Possible, but hard to locate:
The cut can be located with the cur-
rent neighbourhood size, but it requires a speciﬁc combination of start and
end nodes. Given enough run time, the cut will eventually be located.
Type 3 - Not possible to locate with current neighbourhood size:
The cut can-
not be located with the current neighbourhood size. For it to be possible
for the algorithm to locate the cut, a larger neighbourhood is needed.
Obviously, it would be ideal if all optimal cuts were type 1, but the results show
that this is not always the case. Other than a higher run time, type 2 cuts might
be found if the start and end nodes could be chosen more wisely or with a larger
neighbourhood. The specialized start and end node methods attempt to help
with this. The high demand start node method does, in the test instances prob-
lems, speed up the solution time a bit, but would need to consider the demand
in the reduced nodes for it to make a positive diﬀerence in the real instances.
If more information about the transmission network was available, such knowl-
edge could allow the selection of nodes to be prioritized better. This could for
example be information about the location of nodes so that Assumption 4.5 can
be used. Assumption 4.5 expects actual islands and areas on the outer edge of
the continent to be disconnected. Even though the topology of a transmission
network can change (in theory every hour), knowledge about the network and
previous maximum imbalance cuts could be used to target cuts or areas which
potentially lead to maximum imbalance cuts.
For type 3 cuts, the only solution is to increase the neighbourhood size. How-
ever, a larger neighbourhood is more time-consuming for each start and end
node combination, which can have a negative eﬀect on the performance of the
algorithm. There is, therefore, a trade-oﬀbetween being able to ﬁnd cuts that
require a speciﬁc combination of start and end nodes, and less local cuts that
==PAGE:86==
76
Final Results
disconnect a larger number of nodes. The two test instance problems which
were hard to solve, see Figure 6.1 and Figure 6.2, required a large enough neigh-
bourhood to be located. The results for the real-life instances do not indicate
any cuts of this type; the results just seem to vary. Also, it is diﬃcult to say
anything about the reason why the algorithm does not locate the best known
maximum imbalance cuts every time, without being able to look at the graph
of the problem.
A last potential type of cut which has to be mentioned is the following.
Type 4 - Impossible to locate:
If all the branches in a cut cannot be covered
by branch-disjoint paths between any combination of start and end nodes,
the algorithm cannot locate the cut.
No example of a cut of type 4 has been identiﬁed, but the possibility for such a
cut cannot be denied based on the research in this thesis. A maximum imbal-
ance cut of this type cannot be located with the solution algorithm, however,
with no example of such a cut, it is not considered a signiﬁcant issue for the
algorithm.
Overall, the algorithm has proved to be able to solve small scale problems opti-
mally with a short solution time. Compared to the solution time of the complete
search, which evaluates all combinations of up to k branches, the randomized
algorithm is much faster for k = {3, 4, 5}. The algorithm was also able so solve
the large scale problems with k ≤2 optimally, and the problem with case3isl1
and k = 2 was solved with a shorter solution time than the complete search
method with all the variants of the algorithm. However, the average solution
time for the other large problems with k ≤2 was larger than it for the complete
search method. For the large scale problems with k ≥3, the optimal solutions
are unknown, and with bad bounds, no guarantee can be given about the best
found solutions being optimal. Therefore, the solution algorithm cannot be used
to guarantee that the system is ’N −k secured’ for all k ≤5 with a time limit
of only 15 minutes. However, the algorithm can evaluate with high accuracy
whether a transmission network is ’N −2 secured’, which is better than the
principle currently used which makes the system ’N −1 secured’. Moreover, the
results for the problems with case2isl1 and k = {3, 4} and case3isl1 and k = 3
also look promising. Even though the algorithm cannot guarantee the system
is ’N −k secure’ for k = {3, 4, 5}, it can locate critical cuts, and being aware of
such a cut allows the safety of the transmission system to be improved upon.

Chapter 8
Conclusion
This thesis takes on a problem presented by Energinet. They seek to improve
the safety of a large scale transmission system, and request ’a method which
is able to identify the largest imbalance (positive and negative) which occurs
after removing k branches that separates the interconnected system in exactly
two separated sub systems. k can be 1...5.’
[AM17].
Moreover, the method
should be able to provide results within 15 minutes. A graph representation
of a transmission system is presented, and a mathematical formulation of the
problem as a type of maximum cut problem is given. The problem is denoted
as a k-constrained maximum imbalance cut problem, and requires the number
of branches causing a cut to be no more than k, and the two disjoint sets S and
¯S to be connected.
An analysis of the graph representation of Energinet’s transmission system re-
vealed, among other things, that around 25-30% of the nodes are connected by
only one edge. Such an edge can only cause a cut where a single node is discon-
nected. The graph, therefore, can be reduced by those nodes and edges. Such a
reduction in a large graph can make a signiﬁcant diﬀerence in the solution time
of the problem.
This thesis uses connectivity theory and Menger’s theorem, to determine if the
connectivity of a node and possibly neighbour nodes with regards to the main
graph is at most k. This is done by searching for branch-disjoint paths between
a start node and k end nodes. Moreover, if the branch-disjoint paths cause the
==PAGE:88==
78
Conclusion
graph to disconnect, a cut exists and is made up of branches on the paths.
A randomized solution algorithm to solve a k-constrained maximum imbalance
problem presented by Energinet is successfully developed and implemented.
Tests on randomly generated test instances show the algorithm performs best
with a random end node method and evaluating node sets separately in ad-
dition to a local search. Regarding the start node method, the test indicated
the high demand method performed better than the random method, however,
the same does not seem to be the case for the real-life problems. This is most
likely because nodes reduced from the graph can have a large demand, and this
demand is therefore not considered after the reduction. Tests also showed that
the diﬃculty of solving a problem depends, in general, on the value of k, the
size of the graph and the average node degree in the graph.
The results show that the algorithm provides optimal solutions for all the small
problems, and for large problem for k = {1, 2} with a shorter solution time
than an exact method also implemented for this thesis. For the large graphs
and k ≥3, the actual optimal solutions are unknown, therefore, it is diﬃcult
to evaluate exactly how close to optimal the solutions provided by the solution
algorithm are. For the smaller of the large graphs with 6,278 nodes (case 2
island 1), the algorithm is very consistent with the maximum cuts it provides
for the problems with k = 3 and k = 4 in the best of ﬁve runs. There is a vari-
ation in the maximum imbalances for k = 5. For the largest of the graphs with
21,289 nodes (case 3 island 1), the low number of start and end combinations
evaluated compared to the size of the graph shows the time constraint provides
a challenge for the solution algorithm. The algorithm still manages to locate
two critical positive cuts for k = 5, and being aware of such cuts allows the
TSO to improve upon the safety of the transmission system. The results show
that the algorithm cannot be used to guarantee that a large scale transmission
system is N −k branch secured for k = 5. However, the results show that for
k ≤2, the algorithm consistently provides optimal solutions, and the results for
k = 3 also seem convincing. Therefore, the proposed algorithm can be used to
improve the N −1 security principle ENTSO-e currently uses.
The solution algorithm can be optimized with some modiﬁcations, which could
decrease the solution time slightly, however, it is unreasonable to expect a large
improvement in the performance of the algorithm. The performance of the al-
gorithm could also be improved if a wiser way of selecting start and end nodes
could be a found. The main time complexity of the algorithm is in the locate
cut search, therefore, that is also where there is room for most improvement, if
further research with this algorithm and problem is continued. It would also be
interesting to test the solution algorithm on other large scale problems, to see
if the results are similar.

Appendix B
Implementation
This Chapter describes the implementation of the exact solution approach used
to ﬁnd the optimal solutions to smaller problems, and the method used to
generate random test instances.
B.1
Exact Solution Approach
An exact solution approach, which evaluates all combinations of up to k branches
is implemented. Given a graph/network and a value k all possible combinations
of up to k-branches are considered and the optimal solutions for both the maxi-
mization and minimization problem are returned. The list of branches is sorted
so that combinations with largest ﬂow edges are tested ﬁrst, as a way to speed
up the search. Also by giving the best known cuts as input, some combinations
of edges can be eliminated by only looking at the ﬂow on the edges. For every
combination of edges the algorithm checks if the branches make up a feasible
cut. This is done with a BFS search to ﬁrst check if the graph is disconnected, if
the graph is disconnected a second BFS is performed. The second BFS reveals
if the nodes that were not discovered in the ﬁrst BFS are connected or not. If
the nodes are connected the branches cut the graph into exactly two sets and
the cut is feasible. Otherwise the cut is not feasible.

B.2
Random Test Instance Generator
In order to have some instances for testing purposes and also be able to plot
the graph, a graph generator is implemented. The graph generator takes the
number of wanted nodes and an average node degree (or number of branches)
as input. First the nodes are given random coordinates (x, y) in (0, 1) × (0, 1),
next, to make sure the graph is connected, the nodes are connected by a branch
to the closest node that is not yet connected using a greedy algorithm. Last the
remaining branches are placed connecting a random node to one of the closest
( a
|V |) nodes. Where a was set to 0.2 for the test instances used in this thesis.
Each branch was assigned a ﬂow based on a normal distribution N = (µ, σ2).
The test instances are generated with a mean of 0 (µ = 0) and a variance of
four (σ2 = 4). The data is given as output in a node ﬁle and a branch ﬁle, as
described in Chapter 1.
