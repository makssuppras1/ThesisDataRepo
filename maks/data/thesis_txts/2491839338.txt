Abstract
Search engines have become a common way of accessing information and
our expectations to their search results are high.
When the search result is
unsatisfying you ﬁnd yourself substituting words and rephrasing the query. This
process can be done automatically to save time and frustrations. Automatic
query reformulation is the task of automatically modifying a query to improve
the search results.
This project focuses on automatic query reformulation for a document re-
trieval task on the SQuAD dataset with the Lucene search engine. The query
reformulation model being a sequence-to-sequence neural network producing
reformulations based on the original queries.
The query reformulation models were ﬁrst trained with supervised learning
to generate queries from paragraphs using the SQuAD dataset. A pretrained
BERT model produced subword embeddings from the paragraphs which again
were used as input for the models. The models were evaluated based on the
word error rate between the target query and the generated query. The queries
generated had a word error rate of 10.03 and were of poor quality.
The models were further trained with reinforcement learning on a document
retrieval task using search results as reward. The REINFORCE algorithm with
a moving average of the rewards as baseline was used for training the models.
The R@40 and reciprocal rank was used as reward. The models only managed
to marginally increase the reward and the reformulations received less reward
than the original queries, making use of these reformulations counterproductive.

Chapter 1
Introduction
The ever increasing amount of information on the internet is available at our
convenience through search engines.
Search engines have become a natural
extension of ourselves and the demands and expectations of their performance
are high. Facing a complex problem the ﬁrst thing that comes to your mind is
to use a search engine to gather relevant information, but a query in natural
language can be ambiguous, contain irrelevant information, or simply be
spelled wrong. These could be reasons why a search engine often have trouble
interpreting what the user is looking for and it often takes multiple iterations of
searches to retrieve satisfying results. You often ﬁnd yourself having to modify
your query with diﬀerent keywords or completely reformulating it in order to
ﬁnd a satisfying answer. Improving search engine results via automating the
process of query reformulation can save time and frustrations.
A search engine returns a list of ranked results (documents), which are
either relevant or not. Automatic query reformulation is the task of modifying
a search query to increase the number of relevant documents returned by the
search engine.
There are diﬀerent approaches to modifying the query to improve the
results.
One approach aims to add relevant words to the query, another
approach completely rephrases the query, others try to simplify the user query
by removing irrelevant information.
Within these approaches you still need to know what words are relevant
to add to the query or what rephrasing is better than the original. There are
two main strategies to this; local method and global method [1], which are
illustrated in ﬁgure 1.1.
• The local method assumes that the documents returned by the original
query (pseudo documents) contain relevant information related to the
query. With that assumption the pseudo documents are used to expand
or modify the original query. Local methods can either involve the user
in the reformulation process or not. The user can be a part of the re-
formulation process by for example choosing what terms to add to the
query given some suggestions. That is called relevance feedback. When
the reformulation happens without involving feedback from the user it is

Outline
This project will focus on automatic query reformulation with global methods.
The objective is to train query reformulation model that can improve the search
result compared to the original question by reformulating the query.
This query reformulation model is a sequence-to-sequence neural network. It is
ﬁrst trained to produce queries based on paragraphs using supervised learning.
Second, it is further trained with reinforcement learning maximizing the reward
returned by the search engine for a document retrieval task.

Chapter 2
Background Theory
This chapter describes the background theory needed to understand the query
reformulation model.
2.1
Classiﬁcation
In a classiﬁcation problem the objective is to assign a discrete class value k to
an input x [2]. The model produces a probability distribution of size k where
each index gives the probability for class ki.
Producing a sequence of words can be considered a classiﬁcation problem
where each word is a class. The number of classes is the size of the vocabulary
that the model has access to. The objective for the model is to produce the
correct sequence of classiﬁcations.
2.2
Supervised learning
Supervised learning attaches a correct value (target value) y to an input x.
When the model predicts a class it produces probabilities for each class. The
distribution is compared to the target value distribution. The model’s objective
is to match its output distribution to the target value distribution [2].
When training a model to classify a sequence of words using supervised
learning there is a target sentence which is a sequence of target distributions.
This way the model has, for each word it produces, a target distribution
comparable to its output distribution.

2.9
Wordpiece
The Wordpiece model proposed by [6] aims to represent text by subwords that
the model can use as its vocabulary. All words should be represented by one
or more subwords to ensure that all words in a corpora can be composed by
subwords. The following is an example of a sentence represented by subwords:
Before: The unavoidable price of reliability is simplicity.
After: the una ##vo ##ida ##ble price of reliability is simplicity .

The word unavoidable is not in the vocabulary but instead it can be composed
by multiple subwords. The "##" symbols means that the following subword is
part of the previous subword. This way the original word can be reconstructed
from the subwords.
The subwords are found by selecting a maximum amount of subwords
that the model can use, which is the vocabulary size D. The vocabulary is
then initialized with each letter being a subword.
The objective is then to
deconstruct words in a given corpus down to a set of subwords which have the
shortest length when constructing the corpus once again. This would make the
initial state with only single letters the worst solution because the total amount
of subwords needed is the total character length of the corpora. Instead the
subwords will be the longest occurring subwords that will minimize the total
amount of subwords needed to represent the corpora.
2.10
Recurrent neural network
Recurrent neural networks (RNN) is an extension to the feed forward network
where instead of only having forward connections to the next layer there are now
cyclic connections going to the same layer. This makes the network able to get
information about the previous words that were predicted. A simple recurrent
neural network is illustrated on ﬁgure 2.4. On the left the RNN cell is shown
with its cyclic weights. The RNN is unfolded to the right, meaning that the
RNN cell is shown for each time step where the previous state is sent to the
next time step.
Figure 2.4: Simple recurrent neural network [7]. Folded to the left and unfolded
per time step on the right
The RNN cell has issues with long sequences since the important information
from the start of the sequence can easily be overridden at the end of the sequence.
The long short term memory (LSTM) proposed by [8] is a recurrent model that
addresses this issue. The LSTM is able to decide what to remember and output
for each time step. This is possible through a number of gates which control
the ﬂow. Each gate takes information from the previous time step to determine
what to output.

2.12
Attention
Consider an encoder-decoder setup where both the encoder and decoder is an
LSTM cell. The encoder produces a hidden state h for each input in the se-
quence. The last hidden state is used to initialize the decoder and from that
the decoder is asked to generate a sequence of words. When working with long
sentence the last hidden state of the encoder might not contain enough infor-
mation for the decoder. Attention, proposed by [10] and described by [11], is

Transformer
In cases where the whole input sequence is known from the start, the recurrent
neural networks still have to make all the computations sequentially one time
step at a time. The transformer model proposed by [12] computes the entire
sequence at once and is still able to handle sequential dependencies. The Trans-
former is built on the attention mechanism, which [12] describes attention as
mapping an input and a key-value pair to an output. The output is a weighted
sum of the input depending on the keys. The transformer architecture proposed,
shown on ﬁgure 2.8, is an auto-encoder structure. The encoder and the decoder
are composed of multi-head attention and feedforward layers. The encoder and
the decoder shown are just a single module, one could have multiple layers of
encoders and decoders.

Mask
The decoder has a multi-head attention layer with masking. This means that for

can only use the features for previous words to predict the next word for each
position. This again means that whenever the decoder would predict a class for
a position it is predicting the next subword in the sequence.

Related works
This chapter describes the research that has been done within query reformu-

4.1
Early attempts
An early query expansion with pseudo relevance feedback is the relevance model
proposed by [21]. They deﬁne probability of adding a term from the pseudo
documents as co-occurrence between the query terms and terms in the pseudo
documents.
A continuation of this was proposed by [22]. Considering both the query
and pseudo documents as language models they minimize the average Kullback-
leibler divergence.
This is done by adding the most likely terms from the
pseudo documents to the query.
One of the earliest attempts with a global technique for query expansion
was done by [23].
They cluster words in the entire corpus and expand the
query with words that belong to the same cluster as words in query.
[24]
performed query expansion using synonyms from a thesaurus (WordNet [25]).
The result showed that imprecise queries beneﬁted more from query expansion
than already precise queries. It also showed that shorter queries beneﬁt more
than longer queries from query expansion.

Chapter 5
Aim of project
Based on the related research described in chapter 4 I consider two approaches
to query reformulation:
1. Training the model with reinforcement learning using the search engine
results as reward.
2. Combining supervised learning and reinforcement learning by ﬁrst training
a language model using supervised learning and then further train it with
reinforcement learning using the search engine results as reward.
An third approach could be an only supervised learning approach, but
the quality of results returned by the search engine is diﬃcult to relate to a
supervised loss.
A loss function that would relate the query to documents
could be a ranking loss, but it cannot be used in this case. For a seq-to-seq
model producing a sequence of words the output is related to the probability of
choosing those words. The ranking loss function considers similarity between
documents and queries and would not depend directly on the models output.
The loss needs to be a function related to the last activation’s of the model in
order to be diﬀerentiable with respect to the loss.
Training a seq-to-seq model using supervised learning to reformulate a
question requires a target reformulated question. If the objective is to increase
the quality of results returned by the search engine it would require prior knowl-
edge of which reformulations increase the result. It would be more desirable to
train the model based on search engine results directly. This requires a loss that
is not related to the model’s last activation’s. Reinforcement learning oﬀers
solutions to this via policy gradient and the REINFORCE algorithm.
This
is why the research in query reformulation leans towards reinforcement learning.
With that said [31] made a supervised baseline.
They precomputed the
impact on the search result for a limited set of candidate words.
A binary
classiﬁer was trained to predict what words to expand the query with depending
on the precomputed impact on the search results. Because the amount of can-
didate words used for expansion is limited, they could precompute the impact
on the search result. The supervised learning approach was outperformed by
[31] reinforcement learning approach.

The ﬁrst approach towards query reformulation poses some issues.
Here
only reinforcement learning is used to train the model based on the results
from the search engine.
Both [31] and [32] reported that they had issues
with the model converging at sub optimal solutions. With that in mind, my
assessment is that the approach of only using reinforcement learning to train a
model would require an advanced loss function, as the one in [35], to ensure
that the model produces both coherent and useful reformulations. Another ap-
proach is to simplify the problem as [31] did by using pseudo relevance feedback.
Since the focus in this project is global methods I am not going to choose the
pseudo relevance feedback approach of [31]. [35] perform query reformulation
using global methods but use an advanced reward function and training
algorithm (PPO). The reward function requires a BERT model to get a word
level reward and two LSTM encoders to get compute a answer correlation re-
ward. Considering the time frame of this project a simpler approach is desirable.
Instead I choose the second approach that combines supervised learning
and reinforcement learning by ﬁrst training a language model and then training
it with reinforcement learning to improve the search engine results. The use
of a language model would be a starting point to generate meaningful and
coherent reformulations.
[32] and [34] showed that this approach is able to
produce reformulations that improve the search engine results. In addition this
approach can be broken down into steps by ﬁrst creating a language model and
then moving on to reinforcement learning.
The language model used in [32] is a machine translation model trained
on a large corpora of multiple languages.
It was then ﬁne tune for English
with a small amount of English-to-English data.
The model would then be
trained with reinforcement learning. An alternative model could the question
generation model proposed by [36]. The model is trained to generate questions
from a paragraph input.
My hypothesis is that if the model can learn the
general structure and composition of questions and be able to generate mean-
ingful questions given a paragraph it can be further trained via reinforcement
learning to perform query reformulation.
The aim of this project is to perform query reformulation using global
methods with the objective of improving the result returned by a search engine.
Based on related research I have chosen to ﬁrst train a model to generate
questions and then further training it with reinforcement learning using the
search engine results.
The following describes the steps towards the query reformulation model I
have chosen.
1. Following [36] I will train a question generation model. The model will
be trained by supervised learning on the inverse SQuAD dataset meaning
from paragraph to question. Described in chapter 6.
2. Since a query to a search engine rarely is a paragraph from wikipedia I will
investigate whether the model can produce reformulations from shorter

I perform experiments with two approaches, either using the word piece

Discussion
The test and training loss of the experiment with subword id as input is shown
in ﬁgure 6.1.
The general trend among the models is that the training loss
keeps decreasing while the test loss increases.
This is a clear indication of
overﬁtting. The models would beneﬁt from more regularizing such as increased
dropout. The large LSTM with attention is the only model that appears not
to be overﬁtting and suitable for further training to see if the test loss would
decrease. The large LSTM with attention is also the model that achieves the
lowest test loss.
The results of the experiments with BERT embeddings as input for the
encoder is shown in ﬁgure 6.1. Again, the majority of the models appear to be
overﬁtting. More regularization should be applied. The large transformer and
large LSTM with attention could also be trained further to see whether the
test loss kept increasing. The major thing that stands out is the test loss of
the large Transformer. The train loss decreases very slowly compared to the
other transformers and the test loss has huge variations. This indicates that
the model needs a lower learning rate for a more stable convergence.
Comparing the test loss between using BERT embeddings and not (ﬁg-
ure 6.1 and ﬁgure 6.2), the major diﬀerence is that the test loss is lower using
the BERT embeddings for the medium transformer and the large LSTM with
attention. They achieve a minimum test loss at 5.5 with BERT embeddings
and only around 6 without. This indicates that the models beneﬁts from the
BERT embeddings.
The experiments where BERT embedding were used both in the encoder
and decoder is shown on ﬁgure 6.3. The models converge very slowly but the
test loss keeps increasing.
Based on the previous experiments the following
adjustments have been made:
• The size of the transformer in this experiment is increased from 512 to 768
compared to the large transformer in the previous experiments. Because
the large transformers from the previous experiments had trouble converg-
ing and the transformer for this experiment is even larger, I decreased the
learning rate from 0.0001 to 0.00001.
• To decrease overﬁtting and help the models generalize I increased the
dropout of the transformer model from 20% to 40%.
• To keep the training time of the LSTM with attention at a reasonable
level I decreased the encoder layers to 1.
The experiments with using BERT embeddings as input for the encoder and
the decoder did not perform as well as the smaller models. My assessment is
that with large models like this hyperparameter optimization for learning rate
and dropout becomes crucial for proper convergence.
The LSTM with attention did not converge properly. I suspect it is because
I decreased the number of encoder layers. I suggest optimizing the attention
implementation to decrease the training time and keep the multiple layers of

Chapter 7
Query Reformulation with
Reinforcement learning
This chapter describes the experiments with query reformulation using reinforce-
ment learning. The objective is reformulating the original query to improve the
search results. Query reformulation can be applied to both a document retrieval
task and to a question answering task. Since a question answering model re-
quires setup and training and since a document retrieval task only requires a
standard search engine, I consider query reformulation on a document retrieval
task.
7.1
Search engine
These are the search engines I consider:
• Lucene: A widely used search engine implemented in Java by Apache.
Lucene is used by [31],[35] and [35].
• Google: The Google search engine would a good representation of a
real world search engine. [31] use Google by limiting the search to the
wikipedia domain.
• Ranking algorithm: A ranking function like BM25. The search speed
would probably be faster since documents could be stored in cache during
training.
• Elasticsearch: Elasticsearch is a service built around the Lucene search
engine. It has a REST API with a python integration. In addition it
supports concurrent searches.
Using Google as the search engine for reinforcement learning would mean
that every search would have to go through Google servers.
That would
add signiﬁcant latency.
Another issue is maintaining the correct labels for
documents returned by Google since they are not kept locally. Based on this I
choose not to use Google.
45
==PAGE:50==

January 5, 2020
Technical University of Denmark
Lucene has the downside of not always return top k results, it returns
up to top k results.
If no relevant documents are found, no documents are
returned. The in memory ranking function always returns the top k results.
Since [32],[31] and [35] use Lucene, I choose to use Lucene for the rein-
forcement learning experiments.
7.1.1
Data
The SQuAD dataset is a question answering dataset but can also be used
for document retrieval.
Another choice of dataset is the TREC-car dataset
which is used by [32],[31]. For the purpose of showing that query reformulation
models can learn reformulating a query to improve the search result, SQuAD
is suﬃcient.
I deﬁne the document retrieval task as:
Given a query from SQuAD
that on a paragraph, the relevant documents are the paragraphs belonging
to the same wiki. For example the question Who else appeared with Beyonce
in Telephone?
belongs to a paragraph in the Beyonce wiki. All paragraphs
from the Beyonce wiki is considered relevant document for this question. This
approach is inspired by the TREC-car dataset where the queries are head-
lines from a wiki and the relevant documents are paragraphs from the same wiki.
The entire SQuAD data set has a total of 20.000 paragraphs which are
indexed in Elasticsearch (Lucene) without replacing entities.
7.2
Models
As candidate models for further training with reinforcement learning I consider
the two models having produced the most coherent questions: The medium
sized transformer and the large LSTM with attention.
The search engine will add a latency for each training iteration and I
expect the convergence with reinforcement learning to be slower because of
the variance in reward. Therefore I choose the medium sized transformer as it
faster to train than the large LSTM with attention. I refer to this model as
SQuAD transformer.
Since the SQuAD transformer was not able to produce questions of very
high quality I consider the following alternative.
I propose a model which
aimed towards producing questions close to the input question.
With the
assumption that a better reformulation is close to the original query by
substituting a few words, I train a medium sized Transformer model with the
question as input and the same question as target. The task is pretty simple
because the model just has to mimic the input from the representation created
by the encoder. The training is also much faster since the input is questions
and signiﬁcantly shorter than paragraphs. This simple model will have a better
starting point since its default predictions already are relevant. The issue with
such a simple model is lack of a general understanding of language since it only
mimics the input. I will refer to this model as Q2Q transformer.

Discussion
One Question
The results for the experiments with only one question showed that the models
did learn to reformulate the question maximizing the reward with suﬃcient
training.
Even the SQuAD transformer managed to increase the reciprocal
rank though it was pretrained with entity replacement. These experiments are
of course a result of overﬁtting but indicate that the models are able to learn
with the REINFORCE algorithm implemented.
The models trained with the reciprocal rank as reward in ﬁgure 7.1 and 7.3
managed to improve the rank of the paragrpah from 21 result to the top ranked
result. The experiments with dropout above 30% take longer to converge which
indicates that the models do not beneﬁt from increased dropout.
The models trained with R@40 as reward in ﬁgure 7.2 achieve a maximum
recall of 0.175 with dropout at 40%. The experiments with dropout at 40% and
50% keep increasing the reward and have not converged. This indicates that
the models trained with R@40 as reward beneﬁt from increased dropout.
The recall reward is related to multiple diﬀerent paragraphs compared to
the reciprocal rank reward that is only related to one paragraph. The models
trained with recall as reward have to explore more diverse paths to retrieve
more paragraphs. This could explain why the recall reward converges better
with increased dropout than without.
Both models needed at least 500 training iterations before they started to
improve. To decrease this learning rate could be increased.
The reformulations of the Q2Q transformer with reciprocal rank as reward
are close the original query.

The reward baseline for the REINFORCE algorithm is an essential tool
for reducing variance in reward and to have stable convergence.
[32] use
the reward of the original query as a baseline.
This makes sense since any
reformulation with less reward than the original query will produce a negative
reward. When the models produce queries of signiﬁcantly worse quality than
the original query, using this baseline will produce a negative reward for all
input queries. The models could not converge since they never received any
positive reward. Even if the models predicted a relevant word that returns a
relatively high reward, it would still result in a negative reward since it is less
than the original query. The models would eventually be guessing at random.
A strategy could be to have the moving average as a baseline until the model
becomes good enough to compete with the original queries and then change
the baseline to the original queries.
[35] train for multiple episodes on one batch of queries before moving to
the next batch. The models in this project are only trained on each query once
every epoch.
The experiments with one question indicated that the models
needed a few hundred training iterations before increasing the reward. Since
the models need a lot of training iterations just to improve one single question

Chapter 8
Conclusion and future work
The aim of the project was to perform automatic query reformulation improving
results returned by the search engine. The approach I chose was ﬁrst training
a language model with supervised learning to produce queries.
Second, the
model was trained further with reinforcement learning using the search engine
results as reward.
The language model is a sequence-to-sequence model trained to produce
queries from paragraphs. It was trained on the inverse SQuAD dataset with
a paragraph as input and a query as the target sequence. Diﬀerent sizes of
transformer auto-encoders and LSTM auto-encoders were tested (table 6.1).
The models were evaluated based on word error rate (table 6.3) and the quality
of selected questions (table 6.4 and ??).
The model that scorin the lowest
WER 10.09 was the large LSTM auto-encoder with attention that used BERT
embeddings as input to the decoder. The WER was signiﬁcantly higher than
9.66 by [36]. The questions generated were of poor quality compared to [36]
and most were not coherent or related to the input paragraph.
The models suﬀered from overﬁtting as a result of too little regularization.
In addition the larger models had issues with converging properly because the
learning rate was to high. Compared to [36] the models need more training, at
least four times as much.
The two candidate model for further training with reinforcement learning
were the medium sized transformer and the large LSTM with attention. As
the medium sized transformer model is faster to train and produces queries
of similar quality it was chosen to being trained further with reinforcement
learning.
This medium sized transformer model trained on the SQuAD dataset
was further trained using reinforcement learning to reformulate a question for a
document retrieval task. The SQuAD dataset was used for document retrieval
with the search engine Lucene. The REINFORCE algorithm was used to train
the model with the search engine results as reward.
The two rewards being considered were recall@40 and the reciprocal rank.
The moving average of the previous rewards was used as baseline for the
REINFORCE algorithm to reduce the variance in reward.

coherent questions from paragraphs nor was it able to increase the reward
when trained with reinforcement learning. Compared to the original query, the
reformulations received less reward making them counterproductive.
8.1
Future Work
In this section I propose a series of improvements towards a query reformulation
model able to produce reformulations with improved search result compared to
the original query.
First step concerns the query generation model (chapter 6) and would be
regularizing the models by increasing the dropout. When enough regularization
is applied the models have to train, according to my estimation in section 6.4, at
least four times as much compared to [36]. To prepare and facilitate the models
for further training with reinforcement learning would chose not to replace the
entities in the SQuAD paragraphs with common tags.
This would provide
a better query generation model for further training with reinforcement learning.
To improve the training with reinforcement learning I would implement
Entropy regularization (equation 7.2) like [31] and [32].
This would punish
peaks in the model’s output distribution and thereby increase the exploration.
In addition I would implement multiple training iterations on each batch of
queries like [35]. This would allow the model to learn each question and lead
to faster convergence.
Once the query reformulation model is able to successfully learn using
the REINFORCE algorithm with above suggested improvements, multiple
query reformulation models can be trained like [34]. The data would be divided
into random subsets and a query reformulation model would then be trained
on each of the subsets.
Having multiple query reformulation models each
reformulating the query based on its policy an aggregator would select the best
reformulation among them.

In addition to the proposed improvements there are plenty of hyper pa-

be tested along with the number of encoder/decoder layers and sizes.
Regarding the experiments with reinforcement learning I would investigate
the eﬀect of diﬀerent learning rates to speed up the convergence. The window
size of the moving average baseline can be adjusted along the the number of
episodes and regularization coeﬃcient.
Implementing the proposed improvements would be the next steps towards a
successful query reformulation model.
