Summary
This thesis’s main goal is to explore the possibility of using machine learning methods
to predict the onset of void coalescence in ductile fracture. To explore this possibility,
a large set of finite element calculations, on void coalescence, are borrowed from Cihan
Tekoglu. The problem involves trying to predict critical coalescence stress (stress for
the onset of coalescence) based on the initial configuration of a void in a unit cell.
Through data exploration, it is found that not all of the variables in the data set
are significant to predict the critical coalescence stress. The initial void configuration
variables that are significant are three, namely, the size and shape of a primary void
and size of a secondary void.
Two machine learning models are considered for the task. These machine learning
models are Support Vector Regression (SVR) and Gradient Tree Boosting (GTB).
These two methods are widely used and have been successfully implemented in the
field of fracture mechanics. Both machine learning models perform very well in pre-
dicting the testing set, but the SVR model performs significantly better. The testing
set is unseen data for these models and is used to verify how good they generalize
to unseen data. Analytical solutions are used to compare with the results from the
machine learning models, and the SVR method outperforms both analytical solutions.
Even though the analytical solutions have an unfair advantage, as they are fitted to
the whole data set, whereas the SVR model has only the training set to train on.
The effect of using fewer data to train the SVR model is studied. That can open
up the possibility to gather a small data set using finite element calculations and then
using the SVR model to calculate more data points. That can save much time since
FE calculations are computationally heavy. In this study, the SVR model performs
excellently and only needs around 1/4 of the whole data set to train on, to get the
same score as the analytical solutions.
The trade-off between good predictions and interpretability is addressed. One
of the advantages of the GTB model is its interpretability, and we suggest using
SHAP values to interpret the GTB model. The SHAP values help to visualize the
impact each variable has on the model’s prediction. The SHAP values can also look
at each individual prediction to interpret why the model predicted particular critical
coalescence stress.

CHAPTER 1
Introduction
Fractures in materials have been a problem and of great interest for a long time,
because knowing how and when material fractures can increase the lifetime of a struc-
ture and prevent failure. Fractures are usually split up in brittle fracture and ductile
fracture. Brittle fracture is when no plastic deformation occurs before fracture, and
thus the fracture absorbs little energy and usually happens very fast. Ductile frac-
ture, however, is when the material deforms plastically before a fracture occurs. As
the material deforms plastically, the microscopic voids grow and change shape and
their relative position. Initially, the void growth is stable, and the plastic deforma-
tion is roughly uniformly distributed in the material. Then at a certain point, the
deformation localizes in the ligament connecting neighboring voids while regions that
are off the localization plane experience elastic unloading. This point in the void
growth, where the transition from uniform to localized plasticity happens suddenly,
is referred to as the onset of void coalescence (Tekoglu et al., 2012[21]). This process
is visually illustrated in Figure 1.1. Three modes of void coalescence have been ob-
served. The modes are internal necking, shear coalescence, and necklace coalescence.
The internal necking mode, which will be in focus in this thesis, is the most common
mode observed, and it occurs when the localization plane is near perpendicular to
the main loading direction, see Figure 1.1e. It is called internal necking because the
intervoid ligament shrinks similarly to a necking process in a macroscopic specimen
under uniaxial tension (Tekoglu et al., 2012[21]).
In ductile materials the failure is usually due to nucleation, growth and coalescence
of microscopic voids that initiate at inclusions and second-phase particles (Anderson,
2016[2]). The reason why the onset of coalescence is so essential is that for many en-
gineering materials, under low-stress triaxiality, the macroscopic fracture strain can
safely be taken to be equal to the strain at the onset of coalescence. For engineer-
ing materials subjected to high-stress triaxiality, then the strain increase after the
onset of coalescence is not negligible. In either case, an accurate prediction for the
onset of coalescence is vital for an accurate prediction of the fracture strain (Tekoglu,
2015[20]). Therefore, we are interested in finding the stress which initiates the onset
of coalescence, often referred as the critical coalescence stress. It is called the criti-
cal coalescence stress because as soon as the ductile matrix reaches this stress state,
coalescence occurs and most likely fracture occurs.
Most of the work done on building accurate models for void coalescence is based
on the work done by Thomason (Thomason, 1990 [22]). Thomason performed plastic

limit load analysis on perfectly plastic materials and developed an empirical equation
for the critical stress required to initiate internal necking. Since then, much work
has been done to extend and improve Thomason’s work, e.g. Benzerga and Leblond
came up with a closed-form analytical solution for the plastic limit load at the onset
of coalescence (Benzerga and Leblond, 2014 [4]).
One of the problems with studying void coalescence is that it is challenging to
verify analytical models with experimental results, as it is hard to perform experi-
ments to study void coalescence. However, finite element (FE) calculations are now
widely used to verify analytical models. The finite element calculations are based on
works done by (Koplik and Needleman, 1988 [13]). The finite element calculations
are discussed in more detail in Chapter 2.1.
This thesis’s main goal is to use void coalescence data, gathered by finite element
calculations, to make machine learning models to predict the critical coalescence
stress. The void coalescence data consists of the initial configuration of a void with
corresponding coalescence stress. The aim is to have a machine learning model that
takes in the initial configuration of the void (void size, void aspect ratio, secondary
void size) and then gives a prediction for the critical coalescence stress for this con-
figuration.
The application of machine learning are many in the research of void
coalescence, if the machine learning predictions prove to be accurate enough. Finite
element calculations are computationally heavy, and obtaining an error free mesh
can be cumbersome in certain configurations. One application could be if one were to
use FE calculations to gather 400 data points on void coalescence, but instead only
calculated 200 or even 100 data points. Then trained a machine learning model on
these 200 or 100 data points and used the machine learning model to predict the rest.
This can save a lot of computing time, and even allow the researcher to gather even
more data points in a short time. Because when a machine learning model has been
trained, the time it takes to make new predictions is almost instantaneous. Another
application could be to borrow FE data sets on void coalescence, train a machine
learning model on these data sets, and use it to produce new data points.
In this thesis, we explore the potential of using machine learning models to predict
the critical coalescence stress given initial void configuration and how much data the
model needs to train on, to give good predictions. The interpretability of the machine
learning model will also be explored, that is the possibility of visualizing why the
model is giving a certain prediction. In some cases, the interpretability of the model
is so important that simple models are chosen over complex models because of the
interpretability of the simple models, even though the complex models might give
better predictions (Lundberg and Lee, 2017[15]). Even though the interpretability
is not very important in our case, it is always beneficial to be able to interpret the
results.
The thesis is organized as follows: Chapter 2 goes over the technical aspect of
the data set and describes how the data points were calculated. In Chapter 3, the
data set is visualized, providing information about the distribution and correlation.
Chapter 4, gives a review of how machine learning has been used in the field of fracture
mechanics. In Chapter 5, a short introduction to machine learning is given along with

CHAPTER 2
The data and Finite
Element calculations
The data set used in this thesis consists of results from finite element (FE) calculations
provided by Cihan Tekoglu, from the papers (Tekoglu, 2015 [20]) and (Tekoglu et al.,
2012 [21]). In this chapter, the setup of the finite element (FE) calculations used to
calculate the onset of void coalescence is described, for further detail see (Tekoglu,
2015 [20]) and (Tekoglu et al., 2012 [21]).
The technical aspects of the data are
discussed, as well as we calculated a few extra data points to use as further validation,
using the same FE setup as in (Tekoglu, 2015 [20]) and (Tekoglu et al., 2012 [21]).
2.1
Finite element setup
The first and main part of the data comes from (Tekoglu, 2015 [20]), which studies
how a secondary void population impacts the onset of coalescence. The impact of
the secondary void population is investigated by using 3D unit cell FE calculations.
The second part of the data comes from (Tekoglu et al., 2012[21]), which studies the
effect of combined tension and shear for the onset of void coalescence.
In the FE framework, the calculations are performed on a so-called unit cell. This
unit cell represents a small piece of a ductile material, and in the middle of the unit
cell is a void. In this case, the unit cell is cubical, and we are representing the ductile
material in such a way that to each side of the cubic unit cell is another identical
cubic unit cell with a void in its middle. The ductile material we are representing
in the FE framework is therefore made up of many of those cubic unit cells. In the
main part of the data set, the cubic unit cell has a void in the middle, and secondary
voids, which are distributed in the x1 −x3 plane, see Figure 2.1. From now on, the
void in the middle will be called primary void, and the other secondary voids referred
to as secondary void. The main loading axis is in the x2 direction. This is how the
general setup is, and in the following, the setup of the FE framework for the study
of secondary void population is described, for further detail see (Tekoglu, 2015 [20])
and (Tekoglu et al., 2012 [21]).
The lengths of the cubic unit cell are L1 = L2 = L3 = L. The macroscopic stress
state of the unit cell is a axisymmetric tension with a predominant axial stress, thus

As mentioned earlier, the main part of the data set was used to study the effects

Thus the main data set consists of a primary

These initial
configurations refer to the aspect ratio of the primary void, and sizes of both the
primary void and the secondary void. The initial configurations are then changed
by varying the size and shape of the primary void, and changing the size secondary
void.
The shape of the secondary void is kept constant.
By changing the initial
configurations and obtaining the critical coalescence stress for each configuration, a
total of 457 data points are gathered in this main part of the data set (Tekoglu, 2015
[20]).
The material of the unit cell is elastic-perfectly plastic. The Poisson’s ratio is
taken to be ν = 0.49, but for many metals the realistic ratio is around ν = 0.3. The
reason for taking such a high ratio is to satisfy the theoretical limit load analysis
of the condition of having an incompressible velocity field. Taking ν = 0.49 ensures
that the first elastoplastic iteration produces a near incompressible velocity field. The
Young’s modulus, E, over the yield stress ratio is E/σ0 = 104, with the yield stress
σ0 = 450 MPa (Tekoglu, 2015 [20]).

2.1 Finite element setup
7
The FE calculations are performed in the commercial software ABAQUS. The unit
cell is meshed using C3D8 (8-node linear brick) elements. In (Tekoglu et al., 2012[21]),
they argue that that the fully integrated quadrilateral and brick elements without
the hybrid formulation, like C3D20 (20-node quadratic brick), will not handle the
incompressible behavior. Which is what we have by taking ν = 0.49. These elements
tend to become over-constrained with increasing incompressibility. However, using
hybrid elements, like C3D20H (20-node quadratic brick, hybrid with linear pressure),
is beneficial if the material behavior is fully incompressible (or almost incompressible)
and hyperelastic. However, they are also computationally more expensive. For these
reasons, they use the C3D8 elements for the mesh. A separate mesh convergence study
is performed on each void configuration to ensure that the results are not dependent
on the mesh (Tekoglu et al., 2012[21]).
As previously stated the macroscopic stress state of the unit cell corresponds to
axisymmetric tension with a predominant axial stress, Σ22 > Σ11 = Σ33.
When
void coalescence sets in, the deformation is fully confined in the ligaments while the
regions off the ligaments connecting the voids unload elastically. Now in limit load
analysis where elasticity is neglected, the strain components for the unit cell become
D11 = D33 = 0, and D22 ̸= 0. As a result, (along with that the property of normality
is obeyed at the macroscale), the macroscopic yield function for void coalescence
depends only on the stress component Σ22. Therefore, the problem consists of finding
the critical stress value Σc
22 required to initiate void coalescence for a given initial void
configuration. This means that the equilibrium equations must be solved on the unit
cell’s initial geometry instead of the deformed one. That is done by switching off the
non-linear geometry option in ABAQUS (NLGEOM = No) (Tekoglu, 2015 [20]).

2.3
Additional data points
In order to provide further validation for the machine learning models, a few extra
data points are calculated. In order for the data points to be useful for validation,
we need to use the same setup of the FE calculations as described Chapter 2.1. The
calculations are performed in ABAQUS and they are based on a model provided by
Ingrid Holte, a PhD student at the Technical University of Denmark (DTU).
The void configurations that are calculated are provided in Table 2.3. For each
configuration, the process is the following: the 1/8 of the unit cell, along with the
geometry of the void in the unit cell is drawn in ABAQUS. The unit cell with a void
is assigned a material, i.e., elastic-perfectly plastic with a yield stress of σ0 = 450
MPa. Young’s modulus over yield stress ratio of E/σ0 = 104 and a Poisson’s ratio
of ν = 0.49. Then the unit cell is meshed using C3D8 elements (8-node linear brick
elements). On each side of the unit cell, the nodes are assigned to a section. Three of
these sides have boundary conditions imposed on them, while the other three sections
have force applied to them, which apply the axisymmetric tension with predominant
axial stress. In our case we use Σ22 > Σ11 = Σ33 = 0.9, which was used in (Tekoglu,
2015 [20]) and (Tekoglu et al., 2012[21]). To enforce the constant triaxiality, a user-
subroutine called MPC is used. ABAQUS then applies the force incrementally until
the solution has converged. The stresses are then calculated according to Equation
2.1 with a python script, and then we have our critical void coalescence stress, Σc
22 or
simply σc. A separate mesh convergence study is performed on each void configuration
to ensure that the results are not mesh dependant, see Table 2.3.
Before we can start to gather our own data, we need to verify that these results can
be used to validate the machine learning model. That is we need to confirm that the
results calculated here match the values calculated by Tekoglu. Four different initial
void configurations from Tekoglu’s data set are calculated to see if the critical void
coalescence stress we get matches Tekoglu’s values. The results of these calculations
are given in Table 2.2. In Table 2.2, the initial void configuration is given on the
left side, in the middle a convergence study is performed by increasing the number
of elements in the mesh for the same initial void configuration. The resulting critical
coalescence stress is given below each number of elements. To the right, we then have
the critical coalescence stress from the Tekoglu data set. Our critical coalescence
stress is converging to a value that is a few MPa’s lower than Tekoglu’s value, but it
is still within a reasonable range. Therefore, we can safely use our new data points for
further validation. It is worth noting in order to make the modeling of the geometry

CHAPTER 4
Literature Review
Machine learning has been applied in many fields with great success. This chapter
reviews what has been done to explore how machine learning methods have been used
in fracture mechanics. The author is not aware of any work that has been done on
predicting the onset of void coalescence in a ductile fracture, as is the main goal of
this thesis. However, machine learning methods that have been applied in fracture
mechanics could provide insight into what methods are well suited for this thesis.
In (Moore et al., 2018 [17]), machine learning (ML) methods are used on fracture
growth in brittle materials. Hybrid Optimization Software Suite (HOSS) is used to
run simulations to gather data on fracture propagation and interaction, leading to
coalescence and eventually failure. For more information on HOSS, see (Moore et al.,
2018 [17]) and references therein. Using this data, they train ML models to predict
whether or not two fractures coalesce and how much time that process takes. They
employ three popular ML methods, namely Random Forests, Decision Trees, and
Artificial Neural Networks. The ML methods were successful in catching key patterns
and trends in the brittle material exploration and provided an accurate predictive
model. Not only does it give good results, but it also vastly reduces computational
time. They argue that using ML methods can be a stepping stone to couple microscale
models, that are computationally efficient enough, with macroscale models.
Damage evolution in a representative volume element (RVE) is investigated in
(Denise Reimann et al., 2019 [19]) using the machine learning methods, Support Vec-
tor Regression (SVR) and Random Forest. Both ML models succeeded in predicting
the damage evolution correctly and with very high accuracy. Predicting the necessary
grain size in microstructure models to produce given flow curves was also investigated
using the same ML methods. Again both models predicted the grain size with good
accuracy. They also argue that when they test data out-of-range for the grain size
training data, the prediction deviates sharply from the reference value. Thus, they
conclude that the trained ML models are restricted to the range of values covered by
the training data.
Artificial Neural Network (ANN) is again successfully implemented in (Fethi Ab-
bassi et al., 2013[1]). They employ the Gurson-Tvergaard-Needleman (GTN) model
to evaluate ductile damage and fracture phenomena. Then ANN ML method is used
to determine the material parameters of the GTN damage model, with promising
results. In (J. L. Zapico et al., 2008[25]), the ML method ANN is used in a similar
manner as in (Fethi Abbassi et al., 2013[1]). They use ANN to determine parameters
==PAGE:30==
20
4 Literature Review
in a FEM model to match dynamic tests of fractures in a small steel frame. Using
ANN, they get an increased accuracy in the simulation output.
All of the methods described above were successfully implemented and with good
success. However, not all of them are the best suited for the problem at hand. Neural
network methods are very powerful, and they are especially effective in problems
with a high signal-to-noise ratio (Hastie et al., 2009[23]. The Neural network method
requires, in general, a large training set to get accurate predictions (Stewart, 2019[16]).
The data set used in this thesis is quite small and has very little noise. Therefore, the
ANN method is not suited for our task. On the other hand, the SVR method, which
was successfully implemented in (Denise Reimann et al., 2019 [19]), works well with
small data sets and is not as effective with large data sets. Thus, the SVR method has
good potential to work well on our data set. Decision tree-based methods are both
powerful and how they work is very intuitive, and they proved to be very successful in
both (Moore et al., 2018 [17]) and (Denise Reimann et al., 2019 [19]). Therefore, it is
prudent to use a decision tree-based model on our data set. We choose to use a model
called Gradient Tree Boosting (GTB), which generally outperforms other powerful
decision tree-based model such as Random Forests. The two chosen machine learning
models to model the critical coalescence stress are Support Vector Regression (SVR)
and Gradient Tree Boosting (GTB).

5.1
Machine Learning Introduction
Machine learning is usually split into three categories: supervised learning, unsuper-
vised learning, and reinforcement learning. Supervised learning is when the task is
to predict a quantity of interest, based on other quantities. Supervised learning is
then split into regression and classification. The difference between the two is that in
regression, the goal is to predict a continuous quantity, and classification is when the
goal is to predict a discrete quantity (Herlau et al., 2019[10]). An example of regres-
sion is the goal of this thesis, where the aim is to predict the critical coalescence stress,
which is a continuous variable, based on the initial configuration of the void in the
unit cell. An example of classification is if we changed the problem to predict whether
coalescence sets in or not, for a given stress value and initial configuration of the void
in the unit cell. Then the problem is a binary classification, that is, either coalescence
sets in or it does not. Unsupervised learning is the opposite of supervised learning.
With this type of learning, the ground truth is not accessible, and instead, the goal is
to learn what is similar in the data and to find underlying patterns by, e.g., grouping
them (which is called clustering)(Herlau et al., 2019[10]). Unsupervised learning and
reinforcement learning are out of the scope of this thesis, but more information can
be found in (Herlau et al., 2019[10]) and (Hastie et al., 2009[23]). In the following
sections, some of the terms in machine learning that are relevant to this thesis will
be introduced.

5.3.3
Gradient Tree boosting
Boosting is one of the most powerful methods in machine learning. The main idea
is to combine the output of many ”weak” learners or simple models to produce a
more powerful learner. Gradient tree boosting (GTB) is a very powerful and popular
type of boosting algorithm. Gradient tree boosting combines the output of decision
trees to make predictions. It is, in essence, a sum of many decision trees, where each
added tree is a small step in the right direction (Hastie et al., 2009[23]). Gradient tree
boosting is a very good machine learning model to use when we want good predictions
without sacrificing interpretability.
The trees partition the space of all joint predictor variable values into disjoint
regions Rj where j = 1, 2, ..., J. The j variable represents the number of regions
that the tree divides the predictor space into, which is known as the terminal node or
simply leafs. We assign a constant γj to each such region, and all values in that region
are predicted with this value. Therefore the predictive rule is x ∈Rj ⇒f(x) = γj.
Estimating γj is typically trivial, and most often, it is simply the mean of yi in the
region Rj (Hastie et al., 2009[23]).

for ν in the GTB package is ν = 0.1 (Hastie et al., 2009[23]). To find the optimal
value of the learning rate, a few values of between 0 < ν < 1 are chosen, and the
optimal learning rate is found by choosing the one with the lowest test error in the
cross-validation.

5.4
Interpretability and SHAP values
One advantage of a decision tree-based models is their interpretability.
In many
cases, it is imperative to be able to interpret the model, for example, if machine
learning models are used in business where a company might need to be able to justify
and explain why the model is predicting particular value. Being able to interpret
predictions from our model correctly can provide insight into how the model may
be improved, e.g., the user might have good domain knowledge of the data, which
could help to improve the model. In some applications, simple models that are easy
to interpret (e.g., linear models) are often chosen over complex models that give
more accurate results but are harder to interpret. Therefore, there can be a trade-off
between accuracy and interpretability of a model’s predictions (Lundberg and Lee,
2017[15]).
In (Lundberg and Lee, 2017[15]), they motivate that SHAP(SHapley Additive
exPlanation) values can be used to interpret tree ensemble models. The SHAP values
originate from game theory, and they essentially detect how the observation (or data
point) changes for all possible combinations of a subset of features. Then the change
between the features is divided in a fair way (Lundberg and Lee, 2017[15]).
As interpretability is not of great importance in our case, it is out of the scope
of this thesis to go into the SHAP values in more details. For more information on
SHAP values, see (Lundberg and Lee, 2017[15]) and references therein.

What the bootstrap method does is, it
randomly picks a value from the APE results, but the value remains in the APE
results. Therefore, some values can be chosen multiple times, and some values may
not be chosen at all. The bootstrap method randomly selects values until we have a
pool of data that is the same size as the APE results (same size as the test set). This
process is iterated thousand times, which produces a normal distribution around the
mean value, as shown in Figure 7.1. Figure 7.1 shows results for the GTB method,
where the peak is around 0.9, which is the MAPE value presented in Table 7.2. The
confidence interval is then the 95% confidence interval for the normal distribution,
indicated by the light blue box in Figure 7.1. Then we have our confidence interval
(CI), which indicates that we are 95% confident that the MAPE of a new test set lies
within. It is worth noting that the CI for the SVR model does not coincide with the

CHAPTER 8
Interpretability
One of the advantages of using a decision tree-based model is its interpretability.
This chapter provides an overview of what is possible. In this chapter, only the GTB
model is considered. As for the moment, using tools like SHAP values for the SVR
is computationally infeasible but might be possible later on.
8.1
Feature Importance
As we saw in Figure 5.7, single decision trees are very interpretable and easily visu-
alized. That is lost when we use the sum of many such trees, Equation 5.22, and we,
therefore, need to find another way to interpret it. That is done by finding the relative
importance of the input variables (initial void configuration). The importance of each
variable is calculated for each tree, by measuring the improvements over all internal
nodes for which it was chosen as the splitting variable. The variable importance is
then averaged across all of the trees in the model (Hastie et al., 2009 [23]). We then
have the importance of our variables relative to each other. The feature importance
for our GTB model is presented in Figure 8.1. The relative spacing of the primary
void, χp, is by far the most important feature. That is not surprising since we expect
the primary void’s size to have the most impact on the critical void coalescence stress.
8.2
SHAP values
Another way of interpreting our GTB model is by using SHAP (SHapley Additive
exPlanations) to produce visual interpretation (Lundberg and Lee, 2017[15]). Figure
8.2 shows the summary of the SHAP values for each variable. SHAP works with a
base value, and then the variables are assigned a SHAP value, which either pulls this
base value up or down. In our case, the base value is the average critical coalescence

CHAPTER 9
Discussion and Future
work
9.1
Discussion
The main goal of this thesis is to explore the possibility of using machine learning
methods to predict the onset of void coalescence in ductile fracture. The data set
used in this study is FE data on critical void coalescence, both with a secondary void
population and without a secondary void population. Each data point consists of
four variables, three which describe the initial void configuration, and one variable
being the corresponding critical coalescence stress. Two different machine learning
models were considered for the task of predicting the critical void coalescence, given
the initial void configuration as input, namely Support Vector Regression (SVR) and
Gradient Tree Boosting (GTB). SVR is a very powerful machine learning method,
and it generally works well ”out of the box.” It is an excellent method when we care
more about getting good predictions than interpreting the results. The GTB method
is also very powerful and is very well suited when we want good predictions, but we
also want to be able to interpret the results.
These two models were trained on the same training data and verified on the same
testing set, which is unseen data for the two models. The SVR model performed
considerably better than the GTB model on the testing set. The GTB model would
likely have needed much more data to perform on the same level as the SVR model.
To get a reference for how well our machine learning models are performing, they are
compared with two analytical solutions. However, these analytical solutions have an
unfair advantage. They have been fitted to the whole data set, and thus the test set
is not unseen data for them. Even so, the SVR model manages to outperform them,
and the confidence interval of the SVR model and the analytical solutions do not
coincide. Therefore, we can conclude that machine learning can be effectively applied
to void coalescence in ductile fractures.
The possibility of using less data to train our machine learning models while
still getting good predictions was explored. Due to the fact that the SVR model
outperformed the GTB model and is generally efficient with smaller data sets, only
the SVR model was considered for this part. The data set is randomly split into
training and testing sets. A random small subset of the training set is used to train
==PAGE:70==
60
9 Discussion and Future work
the SVR model, and then this trained model is verified on the testing set. Then
a larger subset of the data is used to train the model and then verified using the
same test set as before. This is repeated periodically until the subset becomes the
whole training set. The results are very promising; the test set error decreases very
fast as the number of training points is increased. At around 100 training points,
the SVR model begins to outperform the analytical models. The results are a good
proof of concept, that if the goal is to generate a large data set on void coalescence;
it is possible to decide what initial void configurations we want to study, then we
take a random part of these initial void configurations and perform finite element
calculations to find the critical coalescence stress. Then this FE data is used to train
the SVR model, and the SVR model predicts the rest of the critical coalescence stress
for the remaining initial void configurations. A table such as Table 7.3 could then be
used to choose how much of the data needs to be calculated using FE, depending on
how much accuracy is needed for the study.
An important result in Table 7.3, is the last row before the analytical results,
where we have a MAPE score that is significantly lower than for the full data set.
That is, for 364 training points a MAPE score of 0.085% is achieved, compared to
the score for the full data set, MAPE= 0.2509%. That happens because the variables
have less range and more data points for each value in the data set used for this
part of the study (tightly distributed and many data points). In more detail, the Wp
variable is only these four values, Wp = [0.25, 0.5, 1, 1.5], but there is a total of 457
data points. That means that there is much information for each of the Wp value.
The second data set has Wp = [0.2, 0.5, 0.625, 0.75, 0.875, 1, 1.5, 2, 2.5, 3], but it only
has 100 data points. Therefore, we have much less information about each Wp value
in the second data set. Now, the score became slightly worse when the full data set
was used (the two data set merged), which means we are sacrificing a small amount
of accuracy if we merge the two sets. But in turn we are able to predict new data
points for a greater range of the Wp variable.
The same study is performed on a less complex data set, by not including a
secondary void population. This study explores if we can use even less data to train
our machine learning model if the data is less complex. In this case amount of training
data required to start to outperform the analytical solutions is around 60 data points.
However, in this part, a much smaller data set was used, and in turn, a smaller testing
set, and thus the test results are not as reliable. The analytical solutions are also not
as good as for the more complex data study. For these reasons, it is hard to conclude
that less data is required to train the SVR model when the data set is less complex.
To validate the two machine learning models further, a new set of data points
was calculated, including data points with initial void configurations that were very
different from the periodically distributed data set used to train the models. The new
set of data points also included data points on the limit of the range of variables in
the training set. That is explored to study the robustness of our models. The SVR
model performed incredibly well on this new test set and did not have any considerable
problems with any of the new data points. The worst predicted data point was the
data point that was most out of the range of the training set. We, therefore, confirm
==PAGE:71==
9.2 Future Work
61
that the trained machine learning models are restricted to the range of values covered
by the training data, as was found in (Denise Reimann et al., 2019[19]) and discussed
in Chapter 4. The GTB model did not perform well on this data set. The GTB
method does not extrapolate well to data that is different from the training set, and
therefore we would need more data to train on the GTB model to be as robust as the
SVR model.
Interpretability, that is being able to motivate why our model gives us a particular
prediction, was not of vital importance in this study. However, interpretability could
be very important in similar problems. We might want to be able to examine each
prediction and understand why the model predicts a certain value, e.g., a value we did
not expect. The trade-off between interpretability and good predictions in machine
learning is, therefore, addressed. The SVR model is very powerful and is often the
method of choice when we care more about good predictions rather than being able to
interpret the results of the predictions, as well as when the data set is not very large,
because the SVR method does not scale well to larger data sets. On the other hand,
the GTB model offers strong predictions and interpretable results, but is, in general,
stronger on larger data sets. The use of SHAP values makes the interpretability of
the GTB model intuitive and can be very beneficial for certain scenarios. For now,
the SHAP value method is not computationally feasible for the SVR method, but
much progress has been made in the interpretability of machine learning in the last
few years; thus, it might be possible in the near future.
9.2
Future Work
In this thesis, we have confirmed that machine learning can be applied to void co-
alescence in ductile fracture. It would be exciting to extend the machine learning
models to a more complex void coalescence data sets. For example, having the voids
randomly distributed in the unit cell, or changing the macroscopic stress state, such
as including shear.
Gathering larger training data sets allows for a more interpretable model such
as the GTB model to study if it can perform on par with the SVR model, if given
enough data. If the predictions for the GTB model would be accurate enough, then it
might be beneficial to use the interpretability of the GTB model to catch interactions
of variables that are hard to understand. Thus, machine learning could be used to
directly study void coalescence and how the initial void configuration impacts the
critical coalescence stress.
