Abstract
In the distributed vertex-coloring problem, the goal is to distributively
assign colors to the vertices of a graph such that all vertices are colored
and no adjacent vertices are assigned the same color. We present algo-
rithms and techniques for this problem in the LOCAL model. First, we
present algorithms for the class of graphs that are paths or cycles. Then
we turn to general graphs and present deterministic solutions to the prob-
lem. Specifically we present a 2O(√log n) algorithm due to Srinivasan et al.
[21] that employs the technique of graph shattering to obtain a (∆+ 1)-
coloring of a graph with maximum degree ∆. We then turn to randomized
algorithms where we present a (∆+ 1)-coloring algorithm [15] and a very
fast O(log ∆+2O(√log log n)) (∆+1)-coloring algorithm due to Barenboim
et al. [3]. Finally, we consider lower bounds on graph coloring and show a
necessary exponential separation between deterministic and randomized
algorithms for the coloring problem. The latter essentially shows that the
complexity of any randomized coloring algorithm on instances of size n
is at least that of an optimal deterministic algorithm on instances of size
√log n.

Introduction
In theoretical computer science and mathematics, vertex coloring is one of the
most well studied topics. Besides its interesting theoretical perspectives it has
numerous applications, examples of which are pattern matching and schedul-
ing problems. Consider the following scheduling situation. There are n tasks
T = {t1...tn} that must be solved with the restriction that some jobs ti, tj
conflict, so they cannot be solved simultaneously. Construct a graph with the
vertex set T and edges between jobs ti and tj that conflict. It turns out that
the makespan of this schedule is exactly the number of colors needed to vertex
color this graph [4] [20].
In the distributed setting, particularly in the LOCAL model, vertex coloring is
one of the classical symmetry breaking problems that has received much atten-
tion. Specifically (∆+ 1)-coloring of graphs with maximum degree ∆is very
well studied. This is also the main topic of this thesis.
We begin by showing some bounds from probability theory that will be useful
in the analysis of the randomized algorithms that we will present.
We then turn to some simple algorithms for 3-coloring paths. The aim is to
introduce, in a straightforward setting, a few of the strategies used to design
simple coloring algorithms.
For the main part of the thesis, we focus on the tools and techniques em-
ployed by state of the art deterministic and randomized algorithms. The proper-
ties of the LOCAL model implies that we can can trivially compute any function
on the graph by electing a vertex to gather information on the topology of the
graph, compute the desired function and then distribute the result to the re-
maining vertices. So, spending time proportional to the diameter of the network
O(diam(G)) = O(n) is clearly an upper bound. It is, however, for most prob-
lems, not a very tight one. We can, as we will show using even a fairly simple
randomized algorithm, obtain a (∆+ 1)-coloring in O(log n) rounds.
In 1996, Srinivasan and Panconesi [21] presented a fast way to determinis-
tically compute a decomposition of a graph, which essentially splits the graph
into smaller subgraphs (clusters) with the property that the diameter of these
subgraphs is small and that the cluster-graph is colored using few colors. This
allows us to use the trivial algorithm from before to compute local solutions
that can then be combined into a global solution.
It turns out though, that we can speed up the process even further by using
randomization [3] and applying a technique known as graph shattering which
essentially shatters the graph into connected components of size poly(log n).
We can then use the deterministic approach on these small instances to quickly
obtain a coloring of the entire graph.
The final part of the thesis will show a lower bound on coloring [19] and an
exponential separation between deterministic and randomized algorithms for
coloring and a whole class of other problems [7]. As it turns out, our best hope
for improving the complexity of the best randomized algorithms is to improve
their deterministic parts.

Preliminaries and Notation
2.1
The Distributed Model of Computation
In the distributed model of computation we model a network of n processors
as an undirected graph G = (V, E), |V |= n, such that there is an edge be-
tween any two processors that have a direct communication link. There is no
globally shared memory and the nodes of the network communicate without
failure in synchronized rounds with their neighbors. The time complexity of
any algorithm run in the distributed model is measured in the number of com-
munication rounds used to solve the problem. Lastly, local computation is free
in the sense that nodes can compute arbitrary functions without effecting the
time complexity [3].
2.1.1
The Local Model
The LOCAL model, which is the computational model used in this thesis, is a
distributed model of computation originally formulated by Linial [19]. In this
model we make the following assumptions about the nodes of the network:
• All nodes know global parameters such as the maximum degree ∆and the
number of nodes n
• Nodes can send messages of unbounded size through the edges of the
network
• Nodes are assigned a unique identifier from {0, 1}c log n
Note that the implications of the LOCAL model are such that we can com-
pute any function on G in time O(diam(G)). We just assign one node to col-
lect information about the topology of the network, compute the function and
distribute the result to nodes in the network, all in time proportional to the
diameter of the network.
Another important distributed model is the CONGEST model in which we
model congestion on the edges by restricting the message size over any edge in
any round to be of size O(log n) [14].
We now provide a few definitions and some notation that will be used
throughout the thesis.
2.2
Definitions and Notation
We first formally define two important problems which, as we shall see later,
turns out to be closely related.
Definition 2.1. (Vertex coloring) Given a graph G = (V, E), a vertex coloring
is an assignment of colors (usually integers) to the vertices such that no two
adjacent vertices are assigned the same color. If there is an assignment using
at most k colors then we say that G is k-colorable.
7
==PAGE:8==
Definition 2.2. (Independent set) Given a graph G = (V, E), an independent
set I ⊆V is a subset of the vertices such that no two vertices of I are adjacent
in G. If I is not a subset of any other independent set, we call I a maximal
independent set (MIS).
Note that a maximal independent set differs from a maximum independent
set which is an independent set containing the maximum amount of vertices.
Computing a maximum independent set is NP-hard, whereas maximal indepen-
dent sets can be computed efficiently both sequentially and distributively (e.g
Luby’s algorithm [1]).
We always denote by G a generic undirected graph whose vertex set is V ,
|V |= n, and edge set E. Usually we will use H to denote the subgraph of G
induced by the vertices that are uncolored after a call to some coloring algorithm.

iteratively when, say, reducing the number of uncolored vertices in a graph. The

3.1

Since all nodes on a path has at most two neighbors, it is clear that we can color
any path using at most 3 colors. Consider the following strategy.
Each node that has not stopped repeatedly checks whether it has the largest
identifier and if so, picks a color from {1, 2, 3}. The basic intuition is that by
the uniqueness of coloring, in each round, at least one node will have the largest
identifier among its neighbors and can then safely pick a color and stop. Each
node runs the following algorithm:

Proof. We show by induction that coloring is valid in each round of the al-

If node

least one node will obtain its final color ∈{1, 2, 3} in each round. Hence the
running time is O(n).

3.2.2
Modified Solution for Undirected Paths
Clearly, Algorithm 2 requires that each node knows its predecessor and succes-
sor. However, we can make a slight modification to the algorithm by partitioning
the paths in a clever way to obtain partial solutions that can then be combined
into a global solution.
Use figure 2 for support of the following strategy. Each node sends its ID
to its neighbors. If node v is not a local minima or maxima, then v assigns its
predecessor as the neighbor with the smaller ID and assigns the other neighbor
as its successor. On the other hand, if v is a local minima or maxima, then
(unless it only has 1 neighbor) it cuts the path in two and sets both neighbors
as successors or predecessors, respectively. In this way each node participates
in at most two subpaths.
Figure 2: Illustration of partitioning the path into multiple directed subpaths.
The beginning and end of each subpath has a unique identifier that is either a
local minima or a local maxima.
Now, each node can run the solution for directed paths obtaining a 3-coloring
for each of the at most two subpaths that it participates in. Finally, each node
resolves any potential coloring conflicts with its neighbors. If node v participates
in 1 subpath there is nothing to resolve since the obtained 3-coloring is valid.
If v participates in 2 subpaths then it might be the case that it was assigned
different colors for the two paths that it participated in. In this case, v can
resolve the conflict by picking a color that is different from its neighbors. Since
v has at most 2 neighbors and there are 3 colors to choose from, resolving such
a conflict is always possible.
With this modification, it is also possible to 3-color undirected paths in
O(log∗n) rounds.

4.1
(∆+ 1)-coloring in 2O(√log n) rounds using Network De-

In this section, we present an algorithm due to Srinivasan et al. [21] that ob-
tains a (∆+ 1)-coloring in 2O(√log n) rounds using a technique called network
decomposition.
In the introduction we mentioned that in time proportional to the diameter
of the graph, one node can collect information about the topology of the net-
work, compute any function (as local computation is free) and then distribute
the result to the remaining nodes. This strategy is usually too slow since the
diameter of the graph can be O(n).
However, in this algorithm, this is exactly what we are going to do. But
we will do it by decomposing the network into smaller subgraphs where we
have upper bounds on the diameter. We will then show how to combine these
solutions into a global solution for the whole graph.
4.1.1
Network Decomposition
Network decomposition is an important primitive for many graph algorithms,
particularly distributed algorithms. Informally, a network decomposition par-
titions the graph into disjoint clusters of subgraphs that satisfy bounds on the
diameter of the clusters and the number of colors needed to vertex-color them.
More formally:
Definition 4.1. Given an n-vertex graph G = (V, E), a (d(n), c(n))-decomposition
of G is the construction of vertex disjoint clusters {Ci} such that the graph in-
duced by each cluster has diameter O(d(n)) and each cluster is colored using
O(c(n)) colors.
From the decomposition of G into the clusters C, we can construct the
cluster-graph GC:
Definition 4.2. Given a set of clusters C from a decomposition of a graph G,
the cluster-graph GC is a graph whose vertex set is C. There is an edge between
Ci and Cj if there exists a vertex u ∈Ci and a vertex v ∈Cj such that (u, v) is
an edge of G.

with respect to a graph G = (V, E) and a subset P ⊆V , the ruling forest is a

• Each tree is a subgraph of G, so the vertices of Ti also form a tree in G
• For all trees Ti, the root (or the leader) l(Ti) of the tree is in P
• Every vertex in P belongs to a unique tree
• Trees are vertex disjoint
• The leaders of the trees in the forest are not too close: For any i ̸= j:
dG(l(Ti), l(Tj)) ≥α
• The depth of any tree in the forest is no more than β

For the algorithm presented shortly, we need to be able to merge decomposi-

a partitioning of a graph. We need the following claim:
Claim 4.2. Let G be a graph whose vertices are partitioned into two sets A
and B. Given a (α, β)-decomposition of A on G[A] and a β-coloring of G[B]
with maximum degree β −1, we can compute a full (α, β)-decomposition of G
in O(β) time.
Proof. First of all, let us establish that a β-coloring of a graph trivially defines a
(1, β)-decomposition. The cluster graph in this case is just equal to the original
graph and is then, of course, immediately colored using β colors.
So, we want to show that we can merge the (α, β)-decomposition of G[A]
and the (1, β)-decomposition of G[B] into a (α, β)-decomposition of G.
Consider the following strategy. Construct the clusters of the full graph as
the union of the clusters of A and the vertices of B. Now, we let the colors
of the clusters coming from A be intact. The |B| clusters coming from B are
updated in parallel according to their color class. That is, for each color c in
[1..β], vertices from B with color c picks a new color in [1..β] that is not taken
by any neighbors.
It is safe to do so in parallel since, by the same argument as earlier, vertices
with the same color class constitute an independent set. It is possible to do so
as each vertex from B has degree at most β −1 so there is always at least 1
available color.
Clearly, this procedure can be done in O(β) time since we need O(1) com-
munication rounds for each of the β colors. Figure 7 provides the intuition.

5
Randomized Algorithms for Graph Coloring
5.1
Introduction
Randomization is a powerful way to obtain fast colorings. Usually, randomness
will be used to either decide whether a node is active in a given round, or is
used to randomly select a color from the color palette of the node. Randomized
coloring algorithms are run for a fixed number of rounds, but due to its non-
deterministic behavior, we cannot guarantee a valid solution. We say instead
that a coloring algorithm finds a coloring with high probability:
Definition 5.1. A randomized coloring algorithm produces a proper coloring in
k rounds with high probability if the probability that all nodes are colored after
k rounds is 1 −1/nC for some constant C > 0
Usually, C is controlled by the number of rounds we choose to run the
randomized algorithm.

Lower Bounds and Exponential Separation
6.1
Introduction
In this final section, we contextualize the complexities of some of the algorithms
we have seen. Using the famous lower bound of Linial [19], we show that the fast
path coloring algorithm we presented in Section 3.2 is optimal. Subsequently, we
show that there exists an exponential gap between deterministic and randomized
coloring algorithms [6]. As some of the fastest known randomized algorithms
(e.g Algorithm 10) employs the graph shattering technique to yield components
that are then colored using a deterministic algorithm, this result enforces some
limitations on our ability to improve the complexity of randomized coloring
algorithms.
6.2
Linial’s Lower Bound
Linial’s lower bound is one of the most fundamental bounds in distributed com-
puting. It states that we cannot 3-color an n-cycle in o(log∗n) rounds. This
implies that the algorithm we presented in Section 3.2 is in fact optimal. We
now provide a proof the Linial’s lower bound which is formulated as follows:
Theorem 6.1. Given a graph G which a directed n-cycle, we need Ω(log∗n)
communication rounds to 3-color G deterministically in the local model
The proof given is largely due to the techniques shown in [18] which is ar-
guably a simpler proof than the original one provided by Linial [19]. The general
idea is to show that any T-round 3-coloring algorithm implies the existence of a
(2T +1)-ary 3-coloring function (see Definition 6.1). We then, by the properties
of the coloring function, show that we must have T = Ω(log∗n).
Let us first establish that any coloring algorithm that runs for T commu-
nication rounds is bound to have its vertices pick a color based on vertices of
distance at most T. In this way, we can view a color algorithm as a function A

6.3
An Exponential Separation
In this section, we study a necessary exponential separation between random-
ized and deterministic complexities. This influences our view on necessity of
the graph shattering technique used in Algorithm 5 that is also a prerequisite
for the fast randomized (∆+ 1)-coloring algorithm presented in Section 5.3.
The work is based on the 2019 paper by Chang et al. [6]. In their paper, they
establish that this graph shattering technique is absolutely necessary for fast
randomized (∆+1)-colorings in the LOCAL model. In fact, they show that the
bottleneck of a randomized algorithm for any LCL (see Definition 6.2) problem
is its deterministic complexity.
Specifically, it is shown that the randomized complexity of any symmetry-
breaking problem on a graph instance with n vertices is at least that of a de-
terministic algorithm on an instance of size √log n. The randomized algorithm
presented in Section 5.3 had complexity O(log ∆+ 2O(√log log n)). What we will
show is that the term with exponential dependence cannot be improved without
improving the complexity of the deterministic approach in Section 4.1.
The main theorem of this section is as follows:
Theorem 6.4. [6] Let P be an LCL problem. Define DetP(n, ∆) to the the
complexity of the optimal deterministic algorithm for P and define RandP(n, ∆)
to be the randomized complexity with error probability 1/n. For sufficiently large
n,
DetP(n, ∆) ≤RandP(2n2, ∆)

Definition 6.2. [5] An LCL problem, or locally checkable labelling problem, is a
graph problem for which the feasibility of the solution can be verified by checking
each O(1)-radius neighbourhood. If all such neighborhoods are valid then the
global solution is also valid.

Each vertex just needs to verify that its assigned coloring is different from the
colors of its neighbors which requires O(1) communication rounds.

7
Conclusion
In this thesis, we have surveyed the vertex coloring problem in the distributed
setting, particularly in the LOCAL model. We have shown a very simple al-
gorithm for coloring a path in O(n) rounds and subsequently a much faster
algorithm that runs in O(log∗n) rounds.
We then showed a deterministic algorithm for general graphs due to Srini-
vasan et al. [21] using the technique of network decomposition.
Then we presented randomized algorithms for the coloring problem and
showed how the graph shattering technique can be used to quickly split the
problem into smaller subproblems, that can then be solved deterministically
using the network decomposition technique.
We showed the lower bound of Linial [19] which implies that our fast path-
coloring algorithm is in fact optimal. Finally, we proved an exponential sepa-
ration between randomized and deterministic algorithms which showed that we
can never obtain a better randomized complexity on instances of size n than the
deterministic complexity on instances of size √log n which is exactly the term
hinging the second randomized algorithm in Section 5.3.
7.1
Future Work
The problem of (∆+ 1)-coloring is very well studied in the distributed setting.
Recent breakthroughs, as shown on the timeline in the introduction, have im-
proved on the deterministic complexity of Srinivasan et al. [21] and showed
O(poly(log n)) algorithm both with and without network decomposition [22]
[13].
The improvements in the deterministic complexity naturally sparked im-
provements in the randomized complexity.
In their paper, Chang et al.
[8]
obtains a O(poly(log log n)) randomized algorithm using a technique they call
ultra fast graph shattering.
It would be interesting to study these new results, particularly the improve-
ments in deterministic complexity as it is, as we now understand, the key to
improving randomized complexities as well.
