Abstract
In this master’s thesis, a machine learning model based on neural networks is implemented
to perform one of the transmissibility-based methods within Transfer Path Analysis (TPA),
specifically Operational Transfer Path Analysis (OTPA), in an airborne system with two speakers
acting as noise sources and one receiver. This system will be represented using the model of a
baffled circular piston. Many datasets, containing the spectrum of sound pressure at relevant
points in the system, will be generated using this model. These datasets will be used to train
numerous neural networks that will model the system’s transmissibility in the frequency domain.
Cross-validation will be applied to ensure that overfitting does not occur and to choose the best
model complexity. Subsequently, through measurements taken in an anechoic chamber under
the same conditions as in the airborne system, the model will be tested again, and its results
will be compared with the OTPA method.
The results will be analyzed, and it will be discussed how the project could be extended to cover
more types of airborne systems. Finally, it will be presented how this project demonstrates that
it could be investigated how to apply the neural networks approach to OTPA in a commercial
product that separates noise sources in a car.

Introduction
In this section, the background, prior work and the hypothesis of this project in collaboration
with Hottinger Brüel & Kjær (HBK), will be explained.
1.1
Background
This Thesis unites to different fields, Transfer Path Analysis (TPA) and Machine Learning. In
one hand, TPA can be described as a family of methodologies that study the transmission of
vibrations and noise in a system so the noise contribution of different paths is evaluated. In the
other hand, artificial neural networks is one of the most powerful and used methodologies inside
machine learning, since they can be used applied in any technological field.
Currently, two of the most used family of methods within TPA are, Classical TPA and Transmissibility-
based TPA. Classical TPA methods are conditionated since is often not feasible to apply them,
as it can be a tedious process since it requires frequency response function (FRF) measurements
on several sub-assemblies, depending on the system [1].
Operational Transfer Path Analysis
(OTPA) is the most used method within transmissibility-based methods, as it is a much more
practical method, determining the path contribution in a system by the transmissibility between
indicators, obtained form operational measurements; the indicators, which work as inputs to
the transfer path analysis method, could be accelerometers or microphones (depending on the
system) located close to the noise sources [1].
With everything said, this project aims to find if the deep neural network (DNN) approach is
feasible in order to perform OTPA in airborne systems.
1.2
Prior work
HBK has no previous work about this, thus, the starting point for this project is a paper called
"Operational Transfer Path Analysis: Theory, Guidelines & Tire Noise Application" [2] published
by Delft University of Technology, in which the OTPA method is used and explained in order to
find the contribution of different noise sources in a vehicle’s tire noise propagation.
As reference for the machine learning part, two papers about the neural network model approach
to OTPA be used. The first one, called "Contribution analysis of road noise with transfer path
analysis based on neural network" [3] published by the Seoul National University, investigates the
training of a neural network using a large number of data sets generated using a box-car Finite
Element Method (FEM) model.
The second one, called "Operational transfer path analysis
based on deep neural network: Numerical validation" published by the Dongeui University [4],
investigates the neural network model approach to OTPA for a multi-structural system and
validates it using numerical validation.
One of the most important features of a deep learning model based on neural networks made

1.3
Problem statement
This thesis project mainly focuses on implementing OTPA using neural networks in an airborne
system. The main reason why this topic is investigated is because neural networks have demon-
strated their exceptional ability to discern patterns and input-output relationships from extensive
datasets that may not be appreciable otherwise [4]. Therefore, after training a neural network
and validating its performance, it could be used to predict the system response at the receiver
position in situations where measuring at this position is difficult.
Within the system of concern, choosing the number of sources and receivers for the systems is
crucial since a large number of noise sources or receiver could make the problem too complex.
Therefore, in order to keep all the aspects surrounding this project simple, it was chosen to use
a simple set up: two noise sources, two indicators for each source and one receiver, in a free field,
which means, not considering reflections, only the direct signal created by the sources.
As each frequency line requires a separate neural network [3], an excessively broad frequency
range could result in excessive computational time. Thus, the frequency range for the data was
initially limited to just 400 Hz based on the ranges used in [3] and [4]. Even though this fre-
quency range is small, only 9 frequency points within the 400 Hz range were selected, requiring
training only 9 neural networks.
The ideal case for training and testing the neural networks would be with real data measured in
an anechoic chamber, so in the first month of this project, some measurements were conducted
at DTU’s small anechoic chamber. But due to the availability of both the needed equipment
and the anechoic chamber, it was decided to generate all the data using an analytical model and
validate the trained neural networks with real measurements. In the results section, it will be
shown that after conducting cross-validation to determine the ideal number of hidden layers and
neurons per hidden layer for the already mentioned 9 frequencies, 256 networks were trained and
tested, covering the entire frequency range of the conducted measurements.
The analytical model used to generate all the datasets, is based on the model of a plane circular
piston emitting in a infinite baffle [5]. Initially, the plan was to generate different data points
by varying the distance and angles between the receiver and the sources, thereby creating a
deep learning model capable of predicting noise contributions from numerous different systems.
However, to simplify the project, it was decided to design the analytical model and generate all
the data for a fixed distance and angle.

2.1
Operational transfer path analysis
Operational Transfer Path Analysis (OTPA) is one of the test-based methodologies within Trans-
fer Path Analysis (TPA) techniques used to characterize and study the transmission of vibrations
in mechanical or acoustic systems. Its primary objective is to identify or estimate the contri-
bution of different paths to the overall system response. TPA can be divided in three types
depending on the type of measurements that they require and its approach; classical transfer
path analysis, component-based transfer path analysis and transmissibility-based transfer path
analysis [1].
To better understand how the different types of TPA methods are applied, the normal frame-
work of a TPA problem is going to be explained. This framework concerns a system (could be
a mechanical system or an acoustical system) which can be divided in its active part and its
passive part. The active part is characterize by the source, which is the component causing the
excitation that is transmitted by the interface, between the active and the passive part, to the
receiver in the passive part.
Classical TPA and component-based TPA accurately model vibration transmission by determin-
ing the forces and moments necessary to describe connectivity between the active and passive
parts. However, they require detailed data on how vibrations transfer from one part to another,
which can be a tedious process, especially if there are many different sources of vibration. This
is why transmissibility methods are preferred in such cases or systems. Transmissibility methods
do not require explicit determination of forces between parts of the system. Instead, path con-
tributions are obtained from "transmissibilities" between sensors, which can be computed from
operational measurements [1].
OTPA is a specific approach of the transmisibility-based methods, in which sensors are used
to measure while the system is operating [1]. This measurements are the key to find the tran-
simissibility matrix T which relates the sources with the receiver. In this project, the method
that has been followed in order to perform OTPA is based on the Least-squares algorithm [2].
This method considers the linear system model with a set of inputs and outputs related by the
transsmisibility matrix T. It is important to make it clear that the inputs of the system represent
the measurements performed by indicators very close to the sources (active part of the system)
and the system output represents the measurements performed in the receiver position (passive
part of the system).

2.2
Neural networks
Neural networks are one of the most powerful and widely used machine learning models, as they
have a renowned reputation for their excellent performance in various tasks such as classification,
pattern recognition, or variable prediction (which is the focus of this project). Their structure
and performance are based on the human brain’s ability to process information [6].
Neural networks are composed of several layers. Each layer is made up of a number of neurons,
which can be seen as information processing units connected to all other neurons of the previous
and next layers via weighted connections.
One can differentiate the neural network layers as the input layers and the output layers, as well
as the hidden layers. The input layer receives the input data and passes it to the subsequent
layers, which are consequently referred to as hidden layers. All hidden layers process and transmit
the information until it reaches the output layer, which consists of a number of neurons equal
to the number of elements in the output vector. This process is often called the forward pass,
hence these networks are termed feedforward neural networks. This is the type of network used
in this project, but there are many other types depending on how they process information and
how they are constructed, such as convolutional neural networks, recurrent neural networks, etc
[6].

2.2.1
Cross-validation
After the training phase, test or validation must be performed so the ability of the model to
generalize well to new data can be assessed. This testing should never, under any circumstances,
be based on prediction made with data that was used to train the model otherwise, overfitting
could occur and it wouldn’t be detected. Overfitting happens when a neural network learns the
training data too well, thus, it doesn’t generalize well to new data [6].
In order to assess the performance of the network, after training it, it has to be fed with new
data, normally called test data, so the test error Etest is introduced as the mean over all the

The test error comes with a problem, since it depends on the specific test data set that it’s been
using, therefore, the generalization error must be introduced.
The generalization error Egen
refers to the error obtained on average if the model could be tested on an infinite amount of
data, thus, it is an idealized quantity [6].
Since normally when working on a machine learning model one doesn’t have infinite data to train
and test the model, the solution to estimate the Egen is cross-validation. In cross-validation, the
dataset is divided into parts, with one portion used to train the model and the remaining portion
used to validate it.
There are three different versions of cross-validation, depending on the way to divide the data
[6].
• The hold-out method: in which the data set is just divided into train and test sets, so first
the model is trained using the train set and then the Etest is computed as Equation 16. In
this method the Etest can be seen as an approximation of the Egen
• K-fold cross-validation: The data set is divided in K different sets of test and training data,
so the model is trained and tested using all the splits. Therefore, the Egen is the average
of the Etest of each split
• Leave-one-out method: This is the most computationally heavy method as it could be seen
as K-fold method but using being K the total number of data points (N), therefor, the
model will be trained N times and each time it will be tested in only one data point, and
the Egen is the average of the Etest of each split
2.2.2
Two-level cross-validation
When not only model validation but also model selection has to be performed, the way to achieve
this goal is through two-level cross-validation.
Two-level cross-validation can be applied using any of the three cross-validation methods already
explained, which actually will determine the total number of splits performed in each loop. This
method basically performs cross-validation twice on a whole data set diving it into segments [6].
Figure 2 shows the scheme of two-level cross-validation using the hold out method, since it’s the
method used in this project.

3.2.1
Implementation
In order to implement this formula, a Matlab function was written, which receives as inputs:
the emitted signal U, the distance r and the angle θ of the calculation point; and the frequency
range of the calculation.
If Equation 17 is computed as a function of frequency and the excitation signal U is eliminated,
what is actually obtained is the transfer function between the piston and the calculation point.
Therefore, the written Matlab function, first computes the mentioned transfer function, performs
the inverse of the fast Fourier transform (IFFT) of it, and convolves it with the excitation signal
in order to obtain the sound pressure created by the piston at the receiver point in the time
domain. Finally, it computes the fast Fourier transform (FFT) of this result. The size of the
performed FFT was set to 512, therefore, there are 256 frequency lines in the positive side of the
spectrum. From those 256 lines, and as mentioned before, only 9 lines were selected in the 400
Hz frequency range.
As the OTPA method uses indicators to measure the sound pressure really close to the sources,
these indicators are modeled as points with a distance of r = 0.15m from the source and an
angle θ = 0.1◦.

to add to the generated signals white gaussian noise, getting as input the SNR desired. This
gaussian noise could be seen as the residual µ explained in the OTPA section.

3.3
Data generation
In this subsection, the process for data generation by means of the just introduced analytical
model is explained.
Since neural network models learn and generalize better with a large amount of data, it is
important to generate as much data as possible [4]. Taking into a account the lack of time when
working on data generation and the design of the neural network models, only four types of signals
where used; white noise, pink noise, a lineal tonal sweep and a exponential tonal sweep. Using
this type of signals, seven pairs of signals (one for each source) where used: pink and white noise,
white noise and lineal sweep, pink noise and lineal sweep, white noise and exponential sweep,
pink noise and exponential sweep, white noise and white noise; and lastly, pink noise and pink
noise.
A part for this seven signal pairs, these signals were generated in four intervals of amplitudes:
(0, 0.01) (0, 0.1), (0, 1) and (0, 10). 250 pairs of amplitudes where generated randomly using an
uniform distribution in the mentioned intervals.
Thus, the total number of data generated is obtained from 7 (pairs of signal)* 4 (amplitude
intervals)* 250 (amplitude pairs) = 7000 data point, which can be also called situations. Thus,
each situation actually consists of 4 indicators called I1 and I2 for source 1, I3 and I4 for source
2 (system input) and one receiver called Y (system output). This 4 indicator form the X matrix,
which represents the inputs of the system mentioned in subsection 2.1.
Nevertheless, this data set was augmented multiplying each situation by a random number
between 0 and 1 and adding it to the data set. Thus, the final length of the data set used to
train the neural networks contains 14000 data points or situations.
3.4
Neural networks models
The whole implementation of the neural networks on this project was made using python, more
concretely the PyTorch library [8].
As already introduced, one neural network must be constructed for each frequency line [3], being
its inputs the spectrum of the generated sound pressure of the four indicators united in the X
vector; and the output, the sound pressure at the receiver point Y as explained in subsection 2.1.
Since neural networks do not directly accept complex numbers as inputs, and the spectrum data
(both from measurements and the analytical model) is expressed using complex numbers, an
adaptation must be made to enable neural networks to support them. Each complex number is
split into its real and imaginary parts, resulting in each input to the network being divided by
two. Consequently having 8 inputs and 2 output.
Since the neural network is trained to solve the current OTPA problem in which the position (and

5
Discussion
In this section, all the results shown so far are investigated and discussed. Afterward, how to
improve the results and how this project could be used in the future is going to be explained as
well.
Starting with the case where the number of hidden layers was fixed to 5 and different numbers
of neurons per hidden layer where tested as shown in Figure 7, Figure 8 and Figure 9; it is
acknowledge that from 5 neurons per hidden layer and on, the ability of the net to predict the
system output eY is practically the same, showing that it is not need a very complex model in
terms of number of neurons per hidden layer for the net to perform well. This is confirmed by
Figure 10 where it is proved that there is no significant improvement when increasing the number
of hidden layers from 1 to 6 (when the number of neurons per hidden layer is fixed at 5). Even
the 6 hidden layers network at 208 Hz performs worse than the other numbers of hidden layers.
This results show that the linear regression problem of predicting the system output Y by means
of the inputs X (indicators) when working with just 2 sources and 1 receiver is relatively simple
for an approach with neural networks, yielding low complexity models (only 1 hidden layer and
5 hidden neurons).
In Figure 11, the comparison between the Eval and Etest obtained through hold-out cross-
validation suggest that no overfitting is happening since both errors show the same trend and the
differences between them are minor. This results show a good ability of the networks estimating
the system output Y with data that the they haven’t seen before. However, since the data used
for testing and training is generated from the same analytical model, the generalization ability
of the networks must be assessed using measured data.
When extending the number of designed neural networks to cover the full frequency of the
measurements, it is important to clarify that actually the two-level cross-validation to find the
perfect number of hidden layers and hidden neurons should have been performed again but since
the computational time would have been excessive, it was decided to go on with the ones already
obtained.
Figure 12 shows the Etest after feeding the already trained nets with a new data set. In this
particular case, since the relative mean squared error is a number between 0 and 1, the closer
the relative error is to 0, the more negative the dB value will be. Therefore, the neural networks
in the range from 400 Hz to 1 kHz are the ones that are performing better.
Figure 13 shows that in that particular situation, almost all the neural networks are giving
a fairly good estimate of the system output eY which is observed from Figure 13a, where the
estimated spectrum closely resembles the test one, and in Figure 13b since the relative error in
dB is reasonably negative.

When assessing the ability of networks to estimate the contribution of each source, the results
will be compared with those obtained through the OTPA method. In the specific case shown
in Figure 14 and Figure 15, due to the real level difference between S1 and S2, both the neural
networks and OTPA provide good predictions for the source with the highest level (S1 in this
case). However, for the other source (S2 in this case), both the neural network estimate and
OTPA are inaccurate, with OTPA yielding the worst estimate. This misestimation is believed
to be caused by the fact that the transmissibility matrix T is computed inside OTPA as the
solution of a least-squares estimation. Reference [2] investigates computing the transmissibility
matrix T by means of a Singular Value Decomposition (SVD) of the inputs matrix X, which
should yield a better estimation of the source contribution. A positive aspect of these results is
that, although its estimation is not accurate, the neural network significantly outperforms the
OTPA method when estimating the source with the lowest level (Figure 15b and Figure 15c),
caused by the fact that the networks is solving a linear regression problem to estimate the system
output Y (unlike OTPA). Therefore, all the weights inside the network could be considered as
the transmissibility function between the input and output of the system.
When looking at the results of feeding the real measurements to the 256 nets, it is important to
carefully analyze the Etest shown in Figure 16 as it represents the Etest across only four measured
scenarios. When comparing Figure 12 with Figure 16, even thought this results show a good
performance when estimating the system output Y after 400 Hz, it becomes apparent that more
measurements are necessary to accurately assess the performance of the neural networks.
In the specific case where both sources are emitting pink noise, as depicted in Figure 17, it
is observed that the networks provide a good estimation for the system output Y from 400
Hz onwards. However, in scenarios such as when both sources emit white noise (illustrated in
appendix A Figure 20), the networks yield estimates that are higher in level than the actual
measurements.
Unlike in the testing of the 256 networks with newly generated data, the networks do not provide
accurate estimations for the contribution of the sources, as shown in Figure 18 and Figure 19.
Several factors influence the performance of the network when predicting the system output
Y, with probably the most significant one being that the neural networks are trained using
data obtained from an analytical formula, while the measurement data provided to the network
is generated by a two-driver speaker. Therefore, discrepancies likely exist between the sound
pressure levels created by the model and the actual speakers. These discrepancies can also be
augmented due to mistakes when measuring the distance and the incident angle between the
sources and the receiver; or by the fact that the height of the position of the microphones was
not exactly the same. In summary, multiple measurement factors potentially impact the results.
Additionally, the fact that the measurements were made in an anechoic chamber that has a finite
size makes the free-field assumption invalid for frequencies which wavelength is comparable with

the size of chamber. Consequently, below around 80 Hz (which wavelength is around 4 m) the
relative error increases fast (Figure 12). Furthermore, the speaker response in this frequency
range is likely not uniform.
The performance of the networks is also limited due to the fact that the generated data was not
augmented in terms of cross-spectrum or phase, like done in [4], and only 14000 data points were
created to train the data, which contribute to make the training of the networks not as complete
at it could have been. Additionally, the use of only white noise, pink noise, and tonal sweeps as
excitation signals limits the learning capabilities of the networks.
Lastly, instead of obtaining and analyzing the spectrum of the measured and generated signals
by means of the FFT, the STFT could have been used resulting in more accurate spectra.
5.1
Room for improvement
There are numerous ways through which this project could be improved, beginning with a more
comprehensive plan to generate all the training data. More types of signals could be used, as in
[4] where only harmonic excitation is performed. Furthermore, instead of generating amplitude
values for the excitation signals from a uniform distribution within a certain range, as it was
done, it would have been more favourable to determine the levels of the sound pressure from a
normal distribution with specific mean and variance, like done in [4]. Augmenting the data by
incorporating phase and cross-spectrum information, along with an increased number of datasets,
would also enhance the results.
The measurement procedure could be executed in a more structured manner to obtain a greater
number of test measurements. Narrowing the frequency range of the measurements and enhanc-
ing the frequency resolution would be beneficial [4]. Additionally, the microphones used were
not suitable for these measurements and were not calibrated.
Furthermore, this project could be expanded to enable the neural network to predict system out-
put Y in scenarios with varying distances and incident angles between the source and receiver.
Equation 17 shows that the change on distance and angle is a non-linear process, in which case
the neural network build should not use the identity function as activation function (as it was
in this project). Instead, non-lineal activation functions such the hyperbolic tangent or RElu
should be used. Normalizing the sound pressure could also improve results. Additionally, it
could be investigated whether including geometry values (such as distance and incident angle)
as inputs to the network would enhance its performance. The ideal case for data generation
in order to obtain an useful data set to train neural networks under this conditions would be
in an anechoic chamber, using something like a rotating machine with an attached microphone
that measures as it moves, allowing for recording in different situations of distance and angle of
incidence. After getting around 1000 data points, and performing the cross-spectrum and phase
data augmentations, I strongly believe that the obtained results could show a good performance

5.2
Future work
Taking the results where the neural networks are tested with the generated data (Figure 12,
Figure 13, Figure 14 and Figure 15), it can be concluded that neural networks seem a good
approach to predict the system’s output Y and the source contributions after the networks are
trained.
Therefore, this project could be used to keep investigating the design of a product which, after
training all the neural networks, predict the system output Y only based on indicator information.
For example, I’m thinking that in a car, where small indicator microphones could be placed near
noise sources (engine, wheels, etc.), with another microphone as a receiver positioned at the
driver location. Then, with the entire system in operation, that is the car in motion, record
many different datasets (varying the speed, the surface the car moves on, etc) in order to train
a neural network capable of predicting the sound level at the driver position in any type of car.
Obviously, for this to work, a huge number of datasets obtained from many different types of
cars would be needed.

Conclusion
In this project, the OTPA method has been developed using neural networks for an airborne
system, and it was tried to validate it using real measurements giving results that are quite
improvable. The conclusions that have been reached are:
1. The measurement process to obtain data for validating the model is a tedious one, and if
not done carefully, it can result in unusable data. Developing a detailed plan regarding
the types of signals to use and the range of sound pressure levels to employ is crucial and
completely determines the measured results. Additionally, defining the type of microphones
and speakers to use is crucial when measuring, as well as the duration and frequency range
of the measurements
2. When developing an analytical model to be used for generating a dataset to train a neural
network, it is crucial to choose the model variables correctly (such as frequency range of
interest, piston radius, distance and angle relative to the calculation point, etc), as these
will determine the validity of the data generated with respect to the real system that the
model represents
3. The method used within OTPA was not sufficiently effective, as the estimation of noise
source contributions using OTPA when the level of one source is lower than that of the
other yielded poor results. The OTPA method could have been applied using SVD to
explore how the results could have been improved
4. This project serves as a proof of concept that it is possible to construct a neural network
model that acts as the transmissibility between the inputs and outputs of an airborne
system.
Making possible a product that (after being trained with real measurements)
could predict the system output just with the measurements of indicators placed close to
the noise sources.
