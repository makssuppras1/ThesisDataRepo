Abstract
 
 
Though the concept of autonomous vehicles is not new, there has been tremendous development and 
interest in the last couple of years with self-parking and highway-driving features already a reality. The 
problem however comes in cities where the number of interactable objects increases tremendously and 
decisions based on environment becomes critical, leading to an increased computational expenditure.  
 
Object detection and classification is one of highly researched area mainly due to its nature of being one of 
the foundational challenges for not only autonomous vehicles but also for indoor and outdoor mobile 
robots and industrial automation solutions. One of the main problem with current state of the art 
solutions is the enormous computational setup required to run them, which limits their scalability. 
 
This thesis aims to develop a supervised learning based real time object detection and tracking algorithm 
using a combination of convolutional neural networks (CNN) and recurrent neural networks (RNN). The 
input source for this algorithm would be a USB camera. To analyze the scalability of the algorithm, its 
deployed on a Nvidia Jetson TX2 that belongs to a new category of SoCвҖҷs combining both CPUвҖҷs and GPUвҖҷs 
at a very low power consumption (< 15 W) making them ideal for CNN based algorithms. In this thesis, 
object detection is considered as a regression problem, thus an end-to-end detector is developed.  A 
multilayered LSTM block is added to the proposed detector for providing the tracking features. The objects 
of interest, in this thesis, are limited to pedestrians, cars and trucks.  
 
The proposed algorithm is then tested for evaluating the performance and accuracy of the network under 
varying conditions such as lighting, occlusion, motion blur, fast and partially visible objects.  Apart from the 
environment constraints, hardware constraints would be enforced by the Jetson TX2 thereby allowing for a 
detailed analysis of scalability to similar systems requiring object detection. 
 
 
Keywords 
Autonomous Vehicle, Object Classification, Obstacle Detection, Obstacle Tracking, Computer Vision, 
Supervised Learning.

1. Introduction  
 
Deep learning based algorithms gained prominence from 2012, when AlexNet (Krizhevsky A. S., 2012) 
obtained a new state of the art result in the ImageNet challenge (ILSVRC2012) (Deng, 2009). The rapid boost 
in different deep learning algorithms along with new and improved computer architectures (Graphical 
Processing Unit and Tensor Processing Unit) allowed these computationally extensive algorithms to be 
deployed in different application areas like security and surveillance, medical diagnosis and autonomous 
vehicles to name a few. 
However the recent accidents involving self-driving vehicles from Google (now Waymo) (DMV, 2012), 
Tesla, Uber suggest that a lot still needs to be achieved for a fully autonomous vehicle. This chapter 
covers the motivation behind this project as well as the problem formulation of the actual project. The 
goals of the project along with a short thesis outline is presented. 
 
1.1 Motivation 
 
Object detection and classification is one of the main foundational blocks of an autonomous vehicle 
providing it with the ability to perform different maneuvers depending on its interactions with 
environment. The problem of object detection and classification applies to different domains like indoor 
navigation of mobile robots to unmanned aerial vehicles to autonomous vehicles, making object detection 
and classification one of highly researched area with different methods being developed aiming to 
optimize speed and accuracy. 
 
The problem of object detection can be solved with different sensors like a LIDAR (Light detection and 
Ranging), Radar (Radio detection and ranging), Vision sensors or a combination of these, namely sensor 
fusion. This thesis however focuses on using a vision sensor to detect and classify objects of interest, one 
of the main reason for choosing a vision sensor is, despite all the shortcomings, the amount of information 
that can be accessed via camera is simply not possible to obtain with any other sensor currently out there. 
There has been different approaches towards constructing an object detector, however the crux of all the 
state of the art remains the same, i.e. a deep convolutional neural network is used as a feature extractor 
and these features are then used to predict the locations, in form of a bounding box that encloses the 
object of interest. 
 
One of the primary drawbacks of the present state of the art object detectors and feature extractors is the 
computational resources needed to run them, they leverage heavy GPU instances to run the inference 
models and achieve real time (>15 fps) performance which restricts its usage in a lot of mobile devices. 
Another drawback is the power requirement of these systems. A typical autonomous vehicle has a power 
requirement of around 2500 watts (Gawron, 2018). The early applications of an autonomous vehicle is 
aimed for operation in cities (having ample charging spots) and not for long distance travels, thereby 
countering the problem of power consumption for the time being. 
 
1.2 Problem Formulation 
 
A typical city environment consists of multiple interactable objects like vehicles, pedestrians, cyclists, 
buses, trucks, traffic lights and traffic signs that may be subject to occlusion different lighting conditions, 
scaling effects and crops when captured by a camera.  
What makes object detection challenging is the fact that apart from the above-mentioned effects, the 
system should be real time to be worthy enough for use in an autonomous vehicle. Taking cues from the 
current problems of autonomous vehicles, this thesis is aimed at constructing an object detector which

2.2 Evaluation Metrics 
 
This section focuses on the different metrics used to by various publicly available datasets (section 2.3) to 
evaluate the performance of different neural network models for the task of classification and detection. 
 
2.2.1. Intersection over Union (IoU) 
 
Intersection over union is an evaluation metric used in the KITTI (Geiger, 2012) and Udacity datasets 
(Udacity, 2017) for measuring the performance of an object detector. For applying this metric ground truth 
and predicted values of the bounding box is required.  
Intersection over union is the ratio of area of overlap to area of union (Figure 6). The ground truth is 
typically hand-annotated labels in a particular dataset and are usually provided with the training and 
validation set.  
 
Figure 6: Example showing the calculation of Intersection over Union. The Predicted bounding box is 
generated by the Neural Network while the Ground Truth is typically a hand-annotated label (usually 
provided with a training and validation dataset) (Rosebrock, 2016) 
 
While defining an evaluation metric, a certain threshold is considered above which the prediction is 
considered as a successful prediction.  
 
2.2.2 Precision Recall Curves 
 
==PAGE:16==
 
Page | 10  
 
Precision is a measure of accurate predictions i.e. percentage of positive predictions which are correct. 
Recall on the other hand measures the performance in finding all the positives. 
 
 
A precision вҖ“ recall curve shows the relation between precision and recall for every possible cutoff. Model 
recall can be increased by reducing the threshold increasing the potential positives however, a higher 
precision is much harder to achieve. Precision to some extent can be considered as a measure of accuracy 
while recall can be considered as a measure of completeness.   
 
2.2.3. Average Precision (AP) 
 
Average precision is the weighted mean of precision obtained by the network at each threshold (Retrieval 
cutoff), weighted by increase in previous recall. In this thesis the evaluation method used by PASCAL VOC 
is used, i.e. average precision is obtained by weighted the sum of precision over 11 uniformly distributed 
recall values.  
 
Mean Average Precision (mAP) can be defined as the mean of average precisions computed across all the 
classes.

2.3.1. CIFAR 10  
 
CIFAR 10 dataset (Krizhevsky A. &., 2009) is a subset of 80 million tiny images dataset and is aimed towards 
training algorithms for object recognition. The dataset consists of 60000 images of size 32 x 32 pixels 
comprising 10 classes, each class having the same number of training and testing images. The training set 
comprises of 50000 images and the remaining 10000 images are for testing the network.  
The classes are вҖ“ airplane, automobile, bird, cat, deer, dog, frog, horse, ship, truck and each class is 
mutually exclusive. 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
 
2.3.2. ImageNet 
 
The ImageNet (Deng, 2009) dataset can be considered a ramped up version of the CIFAR 10. It is also 
aimed at image classification with a bigger image size (~640 pixels, ~640 pixels) and 1000 classes. Each 
class having 1300 training images and 50 validation images.  
ImageNet dataset evaluates an algorithm based on the average error it produces over all the test images 
i.e. the algorithms will give a top prediction along with a list of the top 5 object categories in descending 
order of prediction confidence. Thus leading to top 1 prediction error percent and top 5 prediction error 
percent.

3. Image Classification 
 
Convolutional neural networks (CNN) have gained prominence especially in the domain of computer vision 
ever since the breakthrough results achieved by Alexnet on the ImageNet challenge in 2012. Since then 
different networks have been developed that have reduced the classification error rates to the extent of 
surpassing human capabilities (error rate of 5%). These networks, GoogleNet (Szegedy, 2015), VGGNet 
(Simonyan, 2014), ResNet Invalid source specified. and others focused on improving the accuracy of the 
network at the cost of increased layer count and varying the network architecture of feed forward neural 
networks. 
 
However, the problem of scalability restricted the use of these high performance networks in embedded 
devices like smart phones, embedded CPUвҖҷs and GPUвҖҷs. This chapter focuses on the custom lightweight 
classifier developed with the intent of being deployed on Jetson TX2, along with a performance 
comparison with other state of the art architectures developed with the same intent. 
The purpose of constructing a new image classifier instead of using a pre-trained one is the fact that apart 
from being computationally expensive, most models do not incorporate strategies developed in different 
algorithms and restrict in making changes to the pretrained model architectures. 
 
3.1. State of the Art 
 
Recently, the focus of the computer vision community shifted towards development of CNN architectures 
with the end deployment device being a smartphone or any other embedded device. Some of the popular 
architectures that were developed to be deployed on embedded devices are Mobilenet-v1 (Howard, 2017) 
, ShuffleNet (Zhang, 2017) and Mobilenet-v2 (Sandler, 2018), which introduced different architectural 
changes like depthwise separable convolutions, layer shuffling and non-inverted linear bottlenecks, for 
maintaining the accuracy with reduced number of computations.  
Apart from the above architectural changes there are different strategies for obtaining small networks like 
shrinking, quantization (Han, 2015), distillation (Hinton, 2015), using low bit networks (Rastegari, 2016).  
 
Mobilenet-v1 proposed the idea of depthwise separable convolutions that aimed to reduce the number of 
computations and learning parameters by replacing a standard convolution with a depthwise and point 
wise convolution (kernel size equals 1).

3.2. Constructing Image Classifier for CIFAR 10 dataset 
 
When developing a model for image classification the goal is to map the input image to the probabilistic 
prediction. The model is trained to make predictions closer to the ground-truth.  
 
The training strategy used in this thesis is different from the traditional approach (training the network 
from scratch on the ImageNet dataset). An incremental approach of training is used, i.e. the model is first 
trained on CIFAR 10 dataset and then on ImageNet dataset. By doing so, it is observed that the training 
speed of the model is much faster 
 
One of the benefits of using CAFFE is that being written in C, it allows for easy modifications and layer 
constructions. The depthwise separable shuffle layer is easily incorporated into the convolution layer of

4. Object Detector 
 
The purpose of an object detector is to provide a class label along with a bounding box enclosing the 
object. In this section, the classifier developed is modified to act as a feature extractor (by removing the 
softmax and pooling layers) for localization of objects.  
Removing the Softmax and pooling layers allows for moving towards dynamic resolutions where the 
network is flexible enough to  
The modified network is retrained on the detection and localization datasets (Pascal VOC, KITTI, Udacity) 
so that the layers are updated to provide a bounding box coordinate for the object of interest. The objects 
of interest are car, pedestrians and trucks mainly because these were the objects having highest number of 
labels across all the datasets. 
 
4.1. State of the Art 
 
There are different algorithms currently available for object detection, each having its own set of tradeoffs 
with respect to accuracy and speed. The different algorithms can be roughly grouped, based on their 
principle of working as вҖ“ 
1. Sliding windows and Support Vector Machines 
2. Region proposal based Convolutional Neural Networks вҖ“ RCNN (Girshick R. D., 2014) , Fast RCNN 
(Girshick R. , 2015),    Faster RCNN (Ren, 2015). 
3. Regression based Convolutional Neural Networks вҖ“ YOLO (Redmon, 2016) , SSD (Liu, 2016) 
4. Segmentation based Convolutional Neural Networks вҖ“ Mask RCNN (He K. G., 2017) 
 
This thesis builds upon the method of regression based convolutional neural networks, similar to You only 
look once (YOLO) and Single Shot Multibox Detector (SSD).  
 
4.2. Working of the designed regression based object detector 
 
In a regression based object detection network, the detector is trained to predict the bounding 
boxes (рқ‘Ҙ, рқ‘Ұ, рқ‘Өрқ‘–рқ‘‘рқ‘Ўв„Һ, в„Һрқ‘’рқ‘–рқ‘”в„Һрқ‘Ў) along with the class of the object enclosed in the box [].  
where, (рқ‘Ҙ, рқ‘Ұ) is the top left corner point of the bounding box, рқ‘Өрқ‘–рқ‘‘рқ‘Ўв„Һ and в„Һрқ‘’рқ‘–рқ‘”в„Һрқ‘Ў correspond to the width 
and height of the bounding box.  
 
For detecting objects, the input image is divided into a grid with the cell size (20 pixels) that is smaller than 
the smallest object to be detected (25 pixels, KITTI benchmark). During the training process, the network is 
trained to label each cell with the class of the object and coordinates of the top left edge of the bounding 
box of that object relative to the center of the cell (рқ‘Ҙрқ‘җ, рқ‘Ұрқ‘җ, рқ‘Өрқ‘–рқ‘‘рқ‘Ўв„Һрқ‘җ, в„Һрқ‘’рқ‘–рқ‘”в„Һрқ‘Ўрқ‘җ). If there is no object of 
interest in the cell, it is assigned a donвҖҷt care label to maintain the fixed size of the cell labelling. If multiple 
objects are present in a cell, the object having a more pixel coverage is considered for label assignment. In 
majority of the cases there wonвҖҷt be two objects having the same pixel coverage, if however, they do occur 
then the object that has an edge of the bounding box near the cell is considered for pixel coverage. 
 
The drawback of only using this approach is that the detector would predict multiple overlapping bounding 
boxes, which cannot be used for object detection in its current form (Figure).

To overcome the problem of multiple overlapping bounding boxes, during the training phase of the object 
detector, group rectangles implementation of OpenCV is used to filter the generated bounding boxes 
based on the coverage threshold (0.7). The idea is to suppress the multiple predictions and use the 
bounding box having highest coverage threshold. Doing this allows the network in to select the bounding 
box that covers the detected object with highest overlap.

5. Object Tracking 
 
Tracking different objects is one of the most challenging problems for autonomous vehicles. Current 
object detectors locate objects of interest in an image from scratch i.e. treating every image as new 
thereby losing information about their trajectories, which is quite important for path planning of 
autonomous vehicle.  
In the previous section, the drawback of increased resolution on frame rate was observed. In this section 
a deep learning based tracker is developed, which allows to increase the speed of the proposed object 
detector by alternating between detection and tracking algorithm. Thus tracking objects provides dual 
benefits of increasing frame rate and maintaining the inter-frame relation of different objects. 
Tracking objects however poses a challenge in its own, since the objects may suffer from occlusion or 
there may be new objects in the environment and motion blur. Thus, the tracking algorithm should be 
invariant to these conditions. 
 
5.1. State of the Art 
 
Based on its working object tracking can be divided into two categories namely, tracking by detection and 
tracking without detection.  
 
Tracking by detection is a two-part tracker and works by first detecting all the objects using an object 
detector and then associate the detected objects in the next frame using location and appearance 
features. The drawback of such a method is that its performance is directly linked to the accuracy of 
object detector.  
 
Usually trackers are used to predict the path of the object, however in this thesis apart from tracking the 
objects the focus is on increasing the frame rate of detection. Thus a novel idea is proposed, which to the 
authors knowledge has not been implemented or proposed. 
 
5.2. Constructing the Tracking Module 
 
For tracking, Long Short Term Memory (LSTM) is used with its inputs being bounding box coordinates, 
object class and the output of the feature vector. The idea behind using the feature vector is that they act 
as historical visual information allowing for effective tracking by learning from the past information.  
 
For training the multiclass object tracker, a custom dataset is used, which consists of 2635 images of size 
1920 x 1200 pixels and has classes belonging to the categories cars, pedestrians and trucks. The LSTM 
based tracker is then trained on the custom dataset.

5.6. Summary 
 
In this chapter, the frame rate of the detector is increased using a switching combination between object 
detector and tracker, using which the detection frame rate increases to 18.4 fps at 1920 x 1200 pixels on 
the Nvidia Jetson TX2 from an earlier rate of 10.8 fps.   
 
The LSTM based tracker was trained on the custom dataset.  
 
The detector initialized the tracker at frame рқ‘Ў and is not used (turned off) for the frames, рқ‘Ў+ 1, рқ‘Ў+ 2, рқ‘Ў+
3, during which the tracker tracks the objects of interest using the bounding boxes predicted by itself 
along with the features from the Feature Extractor network. 
 
At рқ‘Ў+ 4 frame, the tracker is reinitialized using the bounding boxes predicted by the detector.

6. Conclusion 
 
 
 
Throughout this project, the intent was to develop a real time object detector and classifier that can run in 
real time > 15 fps on an Nvidia Jetson TX2 using a USB camera as the source of input. 
 
 
The problem of detecting and classifying objects of interest namely pedestrians, cars and trucks   is 
considered as a supervised learning  problem  and is intended to be solved using convolutional neural 
networks due to the operational  flexibility they provide . 
 
 
A custom feature extractor was created that combined depthwise separable convolutions and shuffling 
operation to create a new layer, named depthwise separable shuffle operation. This classifier achieved a 
top1 accuracy of 68.93% and a top5 accuracy of 89.73% using only 2.016 Million trainable parameters. 
A mini classifier was initially trained on the CIFAR10 dataset and was then expanded to the proposed 
configuration sharing   a major portion of the architecture between them. This allowed using the weights 
of the model used for training on the CIFAR 10 dataset to be used for training the proposed model on the 
ImageNet dataset. It was observed that using this method helped in reducing the training time as 
compared to training the model from scratch on the ImageNet dataset. The classifier worked well in 
predicting the class labels of the objects however, it wasnвҖҷt able to localize the object position in the 
image. 
 
The classifier was then modified to   perform the task of feature extraction and further layers were added 
to perform the task of detection using those features. The combined network would predict the 
coordinates of the bounding box of the object in the image along with the category of the object. The 
proposed detector was then tested with varying camera resolution settings in order to check the best 
configuration for autonomous vehicles. 
It was observed that the operating rate of the detector at the desired resolution was not real time. In order 
to overcome this a tracking module was incorporated which allowed to track the objects.  The idea behind 
this approach was to switch between the detector and tracker to allow for better optimization and 
increase in frame rate. 
It was observed that the optimal period between update of the tracker was 3 frames. Thus at every 4th 
frame the detector was used to re initialize the bounding box coordinates of the tracker. This approach 
resulted in an increased frame rate from 10.8 fps to 18.4 fps. 
 
Thus the objective of constructing a real time object detector to be deployed on an embedded platform 
was met.

7. Future Work 
  
In this section, proposals for the work in future based on current results is described вҖ“ 
1. The tracking module need to be retrained with a much larger dataset to increase the current frame 
deterioration.  
2. The effect of quantizing the neural network, which theoretically increases the frame rate, needs to be 
studied. 
3. Another method of constructing smaller neural networks using bigger specialized networks i.e. the 
process of distillation needs to be experimented upon. 
4. Using the USB camera has a computational overhead of 20ms, the Jetson provides dedicated hardware 
camera ports. The effects of switching from USB to MIPI camera protocol needs to be studied.
