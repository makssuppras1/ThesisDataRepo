Summary (English)
Increasing demand on mobile data leads operators towards new solutions that
can be cost eﬀective. Cloud RAN is a mobile network architecture where several
RRHs can connect to a BBU pool for statistical multiplexing gain. C-RAN has
introduced also challenges that need to be solved in order to utilize the beneﬁts
of this new architecture. On major challenge that this thesis project talks about
is Data Compression of LTE signals. Transmission of I/Q data between RRH
and BBU is done mostly through CPRI optic ﬁber channels. For utilizing the
capacity of a CPRI ﬁber channel a lossless compression scheme is proposed.
A two stage compression scheme, using Linear Prediction as preprocessor and
a Huﬀman entropy coder, is designed and realized in hardware, targeting on
FPGA’s.
This system has to be compatible with LTE’s network standards,
CPRI’s constant bit rate and ORI’s speciﬁcations.

Introduction
Continuous rising of mobile data transmission push network operators to in-
crease network capacity to face the increasing demand. According to the latest
forecasts by Cisco[1] mobile data traﬃc reached for 2015 3.7 exabytes per month,
increased by 73% comparing to 2014 and this traﬃc is expected to reach 30.6
EB per month by 2020. Further 4th Generation(4G) traﬃc exceeded 3rd Gen-
eration(3G) traﬃc for ﬁrst time in 2015.
These facts and forecasts lead mobile operators to consider new network archi-
tectures to counterbalance the increasing demand with Total Cost of Ownership
(TCO). TCO includes Capital expenditures (CAPEX) and operating expendi-
tures (OPEX). CAPEX refers to costs of hardware, site acquisition, installation,
site support etc. OPEX covers costs i.e for rentals, electricity and maintenance.
The most common architecture that introduced along with 3G networks is the
Radio Access Network (RAN)[2]. Is a Base Station Unit (BBU) with a Remote
Radio Head (RRH)(RAN ﬁgure to be included). Traditional RAN is structured
as follows: A RRH is separated from one BBU and its main functions to interface
with the BBU using ﬁber, to perform digital to analog (DAC), analog to digital
(ADC), power ampliﬁcation and ﬁltering.
In-phase Quadrature (I/Q) Data
transmission between RRH and BBU is done over communication protocols.
The most common speciﬁcation that is used for the transport of the front-haul
traﬃc is Common Public Radio interface (CPRI)[3], another speciﬁcation is
Open Base Station Architecture Initiative (OBSAI)[4]. This scheme forms the

1.2
LTE Overview
Mobile communications as evolve through time their standards approach design
limitations. GSM and GPRS are some examples of technologies that reached
their limits more than a decade ago.
LTE (Long Term Evolution) or the E-UTRAN (Evolved Universal Terrestrial
Access Network), introduced in 3GPP R8, is the access part of the Evolved
Packet System (EPS). The main requirements for the new access network are
high spectral eﬃciency, high peak data rates, short round trip time as well
as ﬂexibility in frequency and bandwidth.
LTE downlink signals are modu-
lated using Orthogonal Frequency Division Multiplexing (OFDMA) and for the
uplink are modulated using Single carrier Frequency Division Multiplexing (SC-
FDMA)[7][8].
The advantage of OFDM that LTE/LTE-A exploits is that instead spreading
the signal over the whole carrier bandwidth (5 MHz i.e for UMTS), the signal is
split and distributed in many narrow carriers of 180 KHz where these carriers
can be transmitted simultaneously[7]. LTE can adapt to any available band-
width between 1.25 MHz up to 20 MHz and narrow or expand the available
carriers. Devices that support LTE must be compatible with all bandwidths
and the one is used in practice depend on the amount of spectrum that is avail-
able for a network operator. LTE is developed for a number of frequency bands
– E-UTRA operating bands- currently ranging from 700 MHz up to 2.7GHz in
order to be available and enable any possible deployment around the word[8].

1.3
Statement of Problem
New technologies and networks architectures make the mobile front haul a ﬁeld
where investigation takes place for ﬁnding new techniques to optimize network’s
performance. Compression of LTE signals is a challenge that raised not only
from the user demand for higher bandwidth on their mobile devices but also
from the new architectures that recently proposed where use of more ﬁber links
is necessary. An eﬀective compression scheme can reduce the overall cost for
ﬁber links.
This gives the motivation for researching compression techniques and algorithms
that can be used on this application. Compression can be categorized in loss-
less and lossy. Field Programmable Gate Arrays (FPGAs) are commonly used
in communication systems where compression and decompression modules are
realized inside an FPGA.
In network architecture as C-RAN data rates in the front-haul scheme is ex-
pected to 12 to 55 times higher depending on the ﬁber I/Q sample size[2].
Eﬃcient compression schemes will be needed to manage this bandwidth rise.
This Master thesis project a lossless compression technique is studied. Typical
a preprocessor de-correlates the input samples from a time domain process to
produce residuals of the input signal as close as possible to White Gaussian
Noise (WGN). Estimated residuals then are the input to an entropy encoder
for encoding and reduce signal’s data rate. On the receivers’s side codewords
that have been produced and transferred through the channel and then by a de-
coder are reconstructed as residuals and then an inverse preprocessing procedure
returns the original values without any distortion or data loss.

2.1
Data compression
The emerge of mobile technologies, new products that are always online in the
new era of Internet of Things create more data than ever before. Compression
methods have been introduced for many years but now it seems they will be
most useful than ever. The rate of data generation has been exploited last years
and it expected to grow with even more.
Compression is divided into two methods, lossless and lossy. In a lossless coding
technique source data accuracy is preserved and any redundancy is removed.
The encoded sequence on the decoder side can be reconstructed exactly without
any distortion. This technique is useful for applications where data loss tolerance
is limited. Trade oﬀs of a lossless techniques is a lower compression ratio.
==PAGE:20==
12
Theoretical background
On the other hand in a lossy compression method some data from the original
source information are removed along with any redundancy. Original signal on
the decoder side in this case cannot be totally recovered. If some distortion can
be tolerated lossy compression schemes can achieve better compression ratio
than lossless techniques.
The basic measure of performance for a coding scheme is Compression Ratio
(CR) and it is deﬁned as the number of bits that represent the data before com-
pression to the number of bits required to represent data after compression[10].
A measure of the average bits needed to code the symbols from a source is
Entropy H [10]. Entropy express the fundamental limit of how much the data
generated from a source can be compressed. This value depends on the statistical
nature of the source. In lossless manner is possible to compress the source with
compression rate close to H.
For a set of symbols A, called also alphabet, a source S with alphabet A generates
a sequence X1, X2..., Xn and the symbols are generated independently.

Entropy Coder
One of the main types of entropy coding creates and assigns a unique preﬁx-free
code to each unique symbol that occurs in the input. These entropy encoders
then compress data by replacing each ﬁxed-length input symbol with the corre-
sponding variable-length preﬁx-free output codeword. The length of each code-
word is approximately proportional to the negative logarithm of the probability.
Therefore, the most common symbols use the shortest codes.
Packetizer
Most of entropy encoders take ﬁxed-length as input producing variable length
output. As long this compression system’s output meant to be mapped into
CPRI channel, where has a constant bit rate (synchronous channel), these vari-
able length codes that are expected as output from the entropy encoder require
to be buﬀered before they sent over the transmission channel[11]. This buﬀer
is of ﬁnite length and it might overﬂow or underﬂow with some probability.

Huﬀman Coding
Huﬀman code is an optimal preﬁx code for a given model, that is commonly
used for lossless data compression. The developed method presented by David A.
Huﬀman "A Method for the Construction of Minimum-Redundancy Codes"[17].
The output from Huﬀman’s algorithm can be viewed as a variable-length code
table for encoding a source symbol. The algorithm derives this table from the
==PAGE:25==
2.2 I/Q Data

estimated probability or frequency of occurrence (weight) for each possible value
of the source symbol. In optimum codes, more frequent symbols are represented
using fewer bits than less common symbols and in an optimum code the two
symbols that occur least frequently will the same code length[10] . Huﬀman’s
method can be eﬃciently implemented, ﬁnding a code in linear time to the
number of input weights if these weights are sorted. However, although optimal
among methods encoding symbols separately, Huﬀman coding is not always
optimal among all compression methods[18].
2.2
I/Q Data
I/Q data shows the changes in magnitude (or amplitude) and phase of a sine
wave. If amplitude and phase changes occur in an orderly, predetermined fash-
ion, you can use these amplitude and phase changes to encode information upon
a sine wave, a process known as modulation. In LTE system the modulations
that are used are QPSK 2 bits per symbol encoding, 16-QAM 4 bits per symbols
and 64-QAM where each symbol is modulated with 8 bits.

2.3.1
SC-FDMA for Up-Link
SC-FDMA is a Modiﬁed from OFDMA with similar throughput performance
and complexity. A major beneﬁt of it that makes it suitable for broadband
signals, especially for uplink transmission, is the low Peak to Average Power
Ratio (PAPR). (it is useful for power saving on user equipment). In SC-FDMA
instead of dividing the data stream and putting the resulting substreams directly
on the individual subcarriers a FFT function is executed for transforming the
incoming signal to frequency based and so the redundant subcarriers to be
removed. This distributes the information of each bit into all subcarriers and
thus reducing the power diﬀerences between the subcarriers. The number of
subcarriers that are assigned to each UE depends on the signal condition the
transmission power and the number of devices in the uplink. This procedure
has as result lower PAPR.

OFDMA for Down-Link
In downlink, traﬃc from BBU RRH, LTE uses Orthogonal Frequency Division
Multiple Access (OFDMA). OFDMA modulation splits the data into many slow
data streams that are transported over many narrow carriers in parallel. The
advantage of many slow parallel streams is that transmission steps can be long
enough to avoid issues of multipath transmission[7].
The orthogonality property derives from the fact that subcarriers are spaced in
such way that the side lobes of each subcarrier wave are exactly zero at the
center of the neighboring subcarrier.

C-RAN network architecture along with other net-

Adaptive LPC Model
Overall target for the compression system is to comply with CPRI speciﬁcations.
CPRI timing is aligned with LTE timing speciﬁcations as shown in table 2.2.
For an 20 MHz LTE carrier sampling frequency is Fs = 30.72 MHz. One I/Q
sample (2x16 bits) is generated every 1/Fs = 32.6 ns. In CPRI FC = 3.84MHz
thus TC = 260.416 ns, during this time are generated 8 I/Q samples or 256
bits. It has to be noticed that Fs = 8 ∗FC. So for an 20 MHz LTE carrier the
minimum CPRI link that can be used is 1228.8 Mbit/s and 8 I/Q samples can
be transfered every TC.
Respectively for diﬀerent LTE conﬁgurations like 2 antennas and 4x4 MIMO,
which is the most common, will require a line rate of 9830.4 Mbit/s(8*1228.8
Mbit/s). This is translated to 64 I/Q symbols transmitted every TC.

4.1.5
Bit Packetizer
As presented earlier in order the variable length output from the Huﬀman en-
coder to comply with CPRI’s constant bit rate a buﬀer need to be added at the
end of the encoder. For the current setup where a line rate of 1228.8 Mbit/s the
system targets 256 (128 I & 128 Q) bits construct a basic frame. The variable
length codewords make buﬀering quite challenging. Code lengths in the model
vary from 12 bits to 30 bits. A buﬀer like this can be modeled as M/M/1 queue.
The average code length shown to be 15 bits. The speciﬁcations of the queue
are
– arrival rate λ = 15bits/t. Assuming on code word per clock cycle
– service rate µ = 128bits/TC
– utilization ρ = λ
µ
– Probability of n bits in the buﬀer: (1 −ρ)ρn
– Mean number of bits in the buﬀer: ρ/(1 −ρ)
– Probability of buﬀer overﬂow: P(more than 128 bits in the buffer) = ρ128
How big the probabillity P of buﬀer overﬂow depends on the t/TC ratio. To
have less probability for buﬀer overﬂow it must be ρ < 1.

Chapter 5
Conclusions & Discussion
Conclusions
Rise of mobile traﬃc creates the need for new network architectures. C-RAN
mobile network architecture for the front haul seems to be a real good candidate
for future deployments.
LTE and CPRI are now the de facto standards in
mobile communications, LTE for the network and CPRI for transmitting data
between RRU and BBU. Concepts of all these technologies presented in this
thesis project.
For this Master thesis project a typical lossless compression scheme presented for
compressing time domain LTE I/Q data for the front haul. A compression and
decompression module is added in both RRU and BBU before ﬁber interface.
In this scheme as preprocessor for de-correlating and removal of redundancies in
the signal designed and implemented a Linear predictor, followed by a Huﬀman
entropy encoder. As for Huﬀman encoder a JPEG recommendation followed
and designed for encoding resulting residuals with a manageable way.
Two cases of Linear prediction investigated, one with ﬁxed parameters and an
adaptive model. In both cases data seem to be de- correlated and redundancies
to be removed although the system wasn’t able to lead into low entropy for the
==PAGE:64==
56
Conclusions & Discussion
residuals in order to cover the required speciﬁcation for compression ration in
front haul. Model for both case simulated in MATLAB.
The major reasons for not being able the system to meet the required speciﬁca-
tions can be summed up as:
– Alphabet. Size of the alphabet in this case is 216 leading into residuals
inside an alphabet of 217.
– Statistics. A large amount of statistics need to be run on samples to get
a safe result for the source behavior.
– Nature of the source. In compression an algorithm doesn’t ﬁt all. LTE
signals seems to be cumbersome to be compressed in an 100% lossless
manner with the proposed compression scheme.
Also in UL that this
project concentrated a lot of factors can aﬀect such a systems performance.
– Timing restrictions. Strict latency times are needed in order communica-
tion protocols to operate in these high bandwidths. Lossless compression
can be highly complex and lead into big delays. This shown also in this
report by the diﬀerent results a long LPC ﬁlter can produce compared to
short ﬁlters.
This adaptive model also implemented for FPGA’s. For hardware implemen-
tation the autocorrelation method for calculating linear prediction coeﬃcients
used. The most eﬃcient way to estimate prediction coeﬃcients is the Levin-
son - Durbin recursive algorithm. In hardware requires several multiplication
operations along with division which created a lot of problems during imple-
mentation due to quantization errors. The system as hardware design can cover
timing speciﬁcations.
Discussion
Compression for the front haul is quite challenging. Still is a sector under inves-
tigation and in literature not a lot of information and suﬃcient speciﬁcations
exists for this kind of systems. Linear predictive coding for lossless compression
seems to be insuﬃcient to cover the speciﬁcations in [26]. For the uplink an
overall CR of 1.14 could be achieved in the best case scenario.
Seems that with this scheme is diﬃcult to be achieved a compression ratio up
to 50% that is required.
I most of the literature the technique of removing

57
redundancies in the frequency domain, exploiting the fact that LTE signal is
oversampled using empty subcarriers, seems to be a de-facto procedure. This
technique lead to an overall data rate reduction of 25%. Although it causes some
loss of information even in the presented literature about lossless compression
on the front - haul is used[9][27]. On this technique although did not presented
here, simulations in MATLAB where obtained.
Linear predictive coding is used widely in speech compression.
It is a quite
common technique to introduce quantization to the input samples creating again
some loss but better compression ratio. Using both of this techniques and by
introducing some data loss this compression scheme can be suitable for the front
- haul.
