Abstract
The following project will give an introduction to current state-of-the-art dialogue sys-
tems technology, which is based on generative methods. A sequence to sequence model
based on long short-term memory networks is implemented and tested using the BLEU
and perplexity along with a qualitative assessment. The model is expanded by using
pre-trained word embeddings trained by GloVe as an initialisation and statically for the
embedding layer. This does not provide a clear improvement of the performance and
it seems the simple sequence to sequence model is not able to utilise the pre-trained
embeddings. Some basic flaws in the sequence to sequence model is identified including
the difference in the model when training and testing.
We find that current methods for training and evaluating dialogue systems does not re-
semble human dialogue very well and some ideas for improving this is proposed including
a new loss function based on aligning pre-trained embeddings.

CHAPTER 1
Introduction
Background
The idea that computers might one day obtain human level intelligence has been around
for almost as long as computers themselves. One of the first to propose that it might
even be a possibility was Alan Turing, whom with his famous Turing Test [56] pro-
posed a way to determine whether a computer had reached human level intelligence. A
computer, which during a dialogue with a human, was indistinguishable from a human
could according to Turing be defined as having reached human level intelligence, which
is popularly called strong AI.
Whether a computer being able to communicate on human level will have reached strong
AI is difficult to say before it has actually been achieved, but many believe that it is
actually necessary with strong AI for a computer to pass the Turing Test. Many previous
achievements within AI research were thought to be solvable only by strong AI, but is
today rather considered weak AI. Models solving these problems are generally only
applicable to domain-specific problems and not easily generalizable to other domains.
Many of the things which are possible within image and speech recognition today have
been though to require strong AI only decades ago.
A major part in obtaining human level intelligence is being able to communicate, which
is also the primary problem in the Turing Test. Dialogue systems have been an active
research field, since Weizenbaum and Joseph [59] released his chatbot ELIZA, which was
the first chatbot able to have something remotely similar to a human dialogue. It was
a rule-based system looking for keywords in the input and returning general and vague
answers and questions. Having a dialogue with ELIZA has been described as talking
with a psychiatrist. If you say a sentence containing ”mother” it will say ”Tell me about
your mother”.
The idea behind ELIZA has since been developed and have produced several very human-
like chatbots. ALICA [58] was released by Wallace in 1995 and the scheme used to build
ALICE has later turned into the AIML-standard. It is today the standard for building
rule-based chatbots and it is being used all over the world to produce chatbots for fun
and commercial purposes alike. Despite the ease and transparency of AIML, with its
obvious benefits for building predictable chatbots, these rule based chatbots are still
very limited to a specific domain. They are furthermore expensive to create due to the
needed human effort in writing down patterns and rules for the dialogue. This rule based
system is furthermore very fragile, which becomes apparent when the dialogue moves
==PAGE:9==

outside of the domain for which rules have been defined. Then the bot will generally
respond with predefined answers, which are very general or with answers, which are not
necessarily related to the input, since it has no knowledge of how to respond outside the
predefined domains.
Jabberwacky [12], which was released in 1998, was one of the first chatbots being able
to learn from conversation. It builds a dictionary of all sentences it has ever seen and
then finds the best answer given the input sentence. This method makes it possible for
Jabberwacky to learn through interaction. Still it is restricted to the sentences it has
seen and is not able to generate new sentences.
The rise of deep learning, which are deep neural networks trained using lots of data, have
provided new methods for developing chatbots, which are not based on rules mapping
an input to an output, but rather generative methods, which maps an input sentence to
a latent representation from which it is able to generate an output sentence. Different
methods have been used for these generative systems including Partially Observable
Markov Decision Models [64] and more recently neural networks [57, 32, 27]. These
systems are being trained end-to-end on large amounts of text and yield interesting
results and a good basis for creating more human-like chatbots, which generalize better
to several domains.
Even though there has been great progress within these generative methods rule based
dialogue systems are still the standard. Their predictability and ease of comprehen-
sion makes these chatbots great for small domain requests like pizza deliveries, weather
forecasting and news. That bots are able to learn from new data, can also backfire
as Microsoft had to admit after the internet turned on their twitter-bot Tay and used
its learning algorithms to make it tweet racist comments and make posts denying the
holocaust.
Obviously we’d like our chatbot to possess some sort ethical awareness, but many other
traits and attributes are important in order to create a system which is able to resemble
human interaction for example common knowledge and personality. Human language is
very ambiguous and understanding natural language demands knowledge of the world
not immediately available to a computer. We’d like the system to be consistent in it’s
answers. Sentences, which are syntactically different but semantically the same, should
receive the same answers, but it should also know that it can’t be from both England
and Berlin as Berlin is a city in Germany and not England. Persona models which could
contain physical traits or geographic information, but also personality and relations,
have formerly been investigated as a means to create this consistency [28].
The project
The following project will investigate how deep neural networks can be used for sequence
generation. Sequence generation can be used within many fields, but some of the most
promising use cases has been within language translation and dialogue systems. This

project will focus on aspect of generating sequences for use within a dialogue system
framework.
Dialogue system research draws upon experience within both natural language process-
ing and dialogue generation, but also increasingly upon advanced machine learning al-
gorithms to train deep neural networks for creating general open domain generative
models. The above section introduced the research and related work which have led to
the current state within dialogue system technology.
The main research questions are whether it is actually possible to create a viable dia-
logue system using these deep learning methods and how one such model differs from
former retrieval based models. Furthermore we will investigate how dedicated embed-
ding methods can be combined with these dialogue systems.
This project will be based on the work done by Sutskever, Vinyals, and Le [53] who
used a recurrent neural networks model for sequence to sequence learning in a machine
translation task.
Several others [57, 49, 32] have later expanded this model to con-
versational tasks. The task for these systems is to predict a target sequence given a
source sequence. The model used in this project will be based on the original model by
Sutskever, Vinyals, and Le [53], but some of the other methods used within dialogue
systems will be introduced. Singh et al. [50] introduced reinforcement learning on top of
these dialogue system as a mean of optimizing long-time rewards instead of only trying
to optimize for separate sentence pairs. Word embeddings have had big influence on
natural language processing research since the publication by Mikolov et al. [36]. It
will be investigated whether using these pre-trained word embeddings in a sequence to
sequence model instead of training them as an end-to-end model increases performance.
Chapter 3 will introduce the models and theory required for the construction of these
generative models and training word embeddings separately.
The analysis is based on data from the OpenSubtitles dataset [30]. The full dataset
contains sentences in 2.6 billion sentences in 60 languages, but since the goal is to build
a dialogue systems only the English subset will be used. This subset includes subtitles
from approximately 106 thousand movies containing a total of approximately 337 million
sentences, which will be used to train and test the model. Several assumptions are made
about the dataset, which intuitively are not always true meaning that the dataset is very
noisy. The full model including training and testing is implemented using python and
Tensorflow. The dataset and the implementation of the model along with third party
modules used for this project is introduced in Chapter 4.
The results of the analysis and discussion of these along with the research question,
which have been investigated will be introduced in Chapter 5. Main points of interest
have been to investigate how the system learns over time and how the complexity of
the network influences the performance, but also the influence of using pre-trained word
embeddings is investigated and if the pre-trained word embeddings and the learned ones
in the end-to-end network are similar.

3.1
Natural Language Processing (NLP)
Natural language processing is a broad field within computer science concerned with
making computers understand human natural languages. Subtasks for achieving this
include predictive modelling, which concerns predicting the next word or sentence based
on the context, constructing vector representations for characters, words or sentences
representing semantic relations and generative models. NLP has recently benefited from
the increased overlap between NLP and machine learning as machine learning algorithms
have shown to work well within this domain also. Especially neural network architectures
have been applied to many natural language tasks with great success.
This section will introduce some general concepts from NLP, which are relevant for
the project. Section 3.1.1 will introduce general language modelling in a NLP context.
Section 3.1.3 will introduce vector representations of words. It has been a very active
research field for several years and are extremely useful especially in neural network
models. Section 3.1.4 will generalise this to sentences, which is still a very active research
field.

Language models are a general concept within NLP which concerns predicting or mod-
elling some part of the language. It could be to predict the next word in a sentence or
to translate a sentence from one language to another. The following section is based on

3.1.2
Tokenization
Tokenization is usually part of the preprocessing or parsing for the text which is to
be analysed or used for in modelling task. It is the task of splitting a document into
separate elements or tokens based on a number of predefined rules, which will ease the
upcoming modelling tasks or the analysis but removing unnecessary noise providing a
clearer signal.
A sentence can be split into separate tokens in a lot of different ways. The sentence:
”He’s a very nice, handsome and kind man” could be split using only the spaces resulting
in the tokens: ”He’s”, ”a”, ”very”, ”nice,”, ”handsome”, ”and”, ”kind” and ”man”. But
as can be seen this would result in ”nice,” and ”nice” being considered different tokens
and similarly for ”He’s” and ”He” and ”is”. For our case this is not optimal, since the
most clear signal and information is obtained by letting it recognise each of words.
Sometimes one might want separate words to be considered a single tokens though. ”Bus
stop” is a single object but information might still be obtained by the separate tokens.
Constructing the right tokenizer is therefore a balance to find the best signal to noise
ratio. One should remove as much noise as possible, but not removing too much of the
signal.
Tokenizers can be combined with part-of-speech taggers for increased precision. It marks
what part of a sentence a given word is, which for example would make it possible for
a model to distinguish between the different meanings of leaves. There are the leaves
in the forest and there can be someone who leaves the room. Without a part-of-speech
tagger the word leaves will be considered the same word, which could lower performance.
3.1.3
Word embeddings
What is today called word embeddings have a long history in the field of linguistics and
are often also called distributional semantic models or distributed representations. The
first attempts at creating vector representations capturing semantic relations between
words used hand-crafted features [1]. An example of this is semantic differentials [51],
were each word was rated on several attributes, which would produce a vector in some
way captured the semantic relations between words.

Most machine learning methods today required the input to be a fixed dimensional input
and embedding methods are therefore very relevant today also as a preprocessing step
or as a part of the model. A basic way to represent a word is as a one-hot vector, which
is a vector the length of the vocabulary with zeros everywhere except for the given word
which has a one. Summing these one-hot vectors up for a sentence, will produce the
bag-of-words sentence embedding explained in Section 3.1.4. The problem with these
representations are that they are very sparse and don’t carry much information. Each
one-hot vector has the same distance to the others no matter the difference in actual
semantic meaning. For example would the words big, huge and chair all have the same
distance to each other even though the words big and huge are much more similar in
semantic relations.
Similarly to the methods used in 1970s and 80s, word embeddings today are used for
constructing a dense low dimensional representation of a word carrying as much infor-
mation as possible. Generally two methods exist for for constructing these, which are
matrix factorisation or sliding window methods. Some of the methods using matrix
factorisation are LSA/LSI [10] introduced in 1990 and HAL [33] introduced in 1996, but
many extensions have been developed including PLSI [19] and LDA [19]. Often these
vectors will capture semantic relations between words, which are useful as a first step in
many models for tasks such as topic modelling or sentiment analysis.
In 2003 Bengio et al. [6] introduced word embeddings in combination with a neural
network in a NLP task of predicting the next word in a sentence given the previous m
words: P(tn|tn−1, ..., tn−m). This is called a sliding window method, since you’re consid-
ering m + 1 words at a time and then slide across the sentences training the method.
They used a neural network architecture for estimating the probabilities, but construct-
ing these embeddings were still very computational demanding. In 2012 Mikolov and
Zweig [35] came up with Word2Vec, which provides a computationally effective method
for computing word embeddings based on the sliding window method, but using a much
simpler architecture than what was previously used in a neural network architecture.
Being the first computationally feasible method presented it got quite a lot of attention
in the NLP community and the machine learning community in general. Mikolov and
Zweig [35] also showed that the embeddings learning possess semantic relations, which
can be used for example in the relation: woman + king −man = queen, but it also
clusters words of similar meaning close together, which can be used for finding synonyms.
Furthermore the words embeddings have proven to be global, which means they can be
used for translating from one language to another using linear mappings [34]. Some of
the relations found can be seen in Figure 3.1. The pictured relations are from GloVe,
but would look similar for Word2Vec.
Since Word2Vec was published other methods producing word embeddings have been
introduced the most notable one probably being GloVe [44], which is based on matrix
factorisation. Even though the methods are different in their approach to constructing
the embeddings it has been found that the resulting representations in general are quite
similar and capture the same semantic relations [44, 26].

When using deep neural networks for general image tasks the features found in the first

3.1.3.1
Word2Vec
Word2Vec is a method presented by Mikolov and Zweig [35] in 2012 and it provides
an efficient way to construct word embeddings from a large corpus of text.
It is a
predictive model using a sliding window method, which optimizes it’s prediction us-
ing back-propagation. Mikolov and Zweig [35] actually presented two methods as part

of Word2Vec: continuous bag-of-words (CBOW) and skip-gram. They have a slightly
different setup, but are both based on a linear neural network architecture using back-
propagation.
In CBOW one tries to predict the target word given the 2k words in the context:
p(tn|tn−k, ..., tn−1, tn+1, ..., tn+k). In skip-gram one tries to predict the 2k words in the
context given the target word, p(tn−k, ..., tn−1, tn+1, ..., tn+k|tn). Both models have two
linear layers. The first project from either the context (CBOW) or the target word
(skip-gram) to a latent representation of size, q, which is the wanted size of the word
embeddings. For CBOW the average of the projections is taken to be the latent rep-
resentation. The latent representation is then mapped to a softmax layer predicting
either the target word (CBOW) or the context (skip-gram). For both models the weight
matrix from the input layer to the latent representation will be the one containing the
word embeddings. An illustration of the two models can be seen in Figure 3.2.
3.1.3.2
GloVe
GloVe [44] is another method recently published which is commonly used for constructing
word embeddings. Similarly to Word2Vec it provides an efficient method for estimating
word embeddings, but where Word2Vec uses a predictive method and back-propagation,
GloVe uses a matrix factorisation method. For very large corpus’ these methods are
more efficient than predictive methods, since they better utilise the repetition in the
corpus and co-occurrence of the words, which in turn allows training on larger corpus’ of
text than for predictive methods. For word embedding methods more data is generally
very beneficial, since it provides a clearer picture of the relations between the words.
The method is based on co-occurrences of words. They note though that a more infor-
mative way to represent the information than using just the co-occurrence frequencies

steam and solid. Similarly for k = gas the ratio would be small and for k = water it
would be approximately 1.

3.1.4
Document embeddings
Similarly to word embeddings it can be beneficial to have a fixed length representation
of a sentence or document. A document in this context might be a single sentence or
a collection of sentences. A common way to represent documents have been the bag-of-
words representation, which is a vector the length of the vocabulary having the count
of how many times each word appears in the document. The word order is obviously
lost in this representation, since only the count is registered, but some extensions have
been developed, which are able to keep the information of word order for example bag
of n-grams. The two sentences: ”She is hungry” and ”Is she hungry” obviously have
different meaning and intention, but without the order of the words is not clear whether
the sentence is a question or a statement. Furthermore the bag-of-words representation
is generally very sparse and high-dimensional with no explicit semantic information.
A simple way to produce lower dimensional representations, have been use pre-trained
word embeddings by averaging the word embeddings for the words in the given document.
The hypothesis is that this should keep a lot of the semantic information given by the
word embeddings, although that similarly to bag-of-words the word order will be lost.
An improvement to this method has shown to be weighting the word embeddings by
a TF-IDF score [45], which gives an indication of how important a word is to a given
document.
More advanced methods for producing document representations include Doc2Vec [25]
and Skip-Thought [24]. Doc2Vec is an extension to Word2Vec which trains an embedding
for the document along with leaning the embeddings for each word.
Skip-thoughts
basic idea is very similar to the idea behind skip-gram [35], since it tries to predict
the surrounding sentences given a sentence, where skip-gram does the same, but on a

3.2
Artificial Neural Networks (ANN or NN)
Articifial neural networks (ANN or just NN) has been an active research field since
Rosenblatt [48] came up with the Perceptron in 1958. Its architecture was inspired by
the neural networks found in the brain and it was able to learn patterns by observing
data. Since the initial description of the Perceptron many developments have followed,
but the blooming of neural networks, which today are called deep learning can largely be
contributed to a few discoveries. The description of applying back-propagation to train
multi-layered networks, which was first presented by Werbos [61] in his 1974, sparked
another rise in the interest of neural networks, but the networks proved too computation-
ally demanding to train. The general increase in computing power and the low price of
GPU’s during the 2010’s sparked another rise in interest, sin the increased computation
power now allowed the training of deep networks.
Today neural networks are increasingly being used in more and more fields where one
earlier did not think of applying ANNs. Neural networks are now state of the art within
many of these fields including image and voice recognition surpassing former methods
used due to the large amount of available data and computation power, which is readily
utilised by these neural network architectures.
Section 3.2.1 will describe recurrent neural networks, which is a method often used for
sequences. Section 3.2.1.3 will introduce the long short-term memory network, which is
a special kind of this neural network architecture being used in the sequence to sequence
model. It is especially good at capturing both the short and long term dynamics in
sequences and have been shown to be easier to train than conventional recurrent neural
networks.

3.2.1.1
Vanishing (and exploding) gradients:
The following section is based on the section about vanishing gradients in [40].
Deep neural networks were during the 1980’s found to be very difficult to train. This
was both a problem for feed-forward neural network, but also recurrent neural networks,
which were unrolled through time to become a deep feed-forward neural network. It
was showed during the 1990’s that the difficulties in training these networks were due to
vanishing gradients [16, 3, 5, 18], which have a large influence when the networks have
a depth beyond a single layer.
The two most commonly used activation functions at this time were the tanh and sigmoid
functions. Both have a gradient strictly lower than one. The gradients of the units in
a neural network are found by backpropagation, which is based on the chain rule. For
a network with n layers, the first layer will be multiplying n numbers less than one
to calculate gradient. This means that the gradients will decay exponentially with the
depth of the network, such that the further in the front the layer is the slower it will be
trained due to lower gradients.
The opposite problem to the vanishing gradient problem are exploding gradients. These
might occur if the weights in the initialised model are too big allowing for them too
grow exponentially, meaning that the first layers will be trained much faster than the
last layers. The combination of vanishing and exploding gradients make training all the
layers at the same speed a very delicate task, since either the last layers will be trained
much faster than the first layers or the opposite.
The exploding gradient problem have been shown to be solvable by clipping the gradients,
which makes sure that they do not grow too fast [43]. Vanishing gradients are a much
more difficult problem to solve, since it is difficult to know actually when they are present.
Several solution have been proposed to solve the vanishing gradient problem, but the one
we’ll focus on here is the LSTM-cell, which will be introduced in Section 3.2.1.3. It still
has the exploding gradient problem, but this can be solved by clipping the gradients.

3.2.1.2
Backpropagation through time
Backpropagation through time (BPTT) is a method for training recurrent neural network
using the backpropagation algorithm, which is used to train feed-forward neural networks.
The method was developed independently by [37, 47, 62].
For applying BFTT one unrolls the recurrent neural network and applies what closely
resembles a normal backpropagation algorithm. The main difference is that the weights
are shared in the unrolled network, which results in the updates for each time step being
summed up and applied to the weights.
3.2.1.3
Long short-term memory networks (LSTM)
As mentioned in Section 3.2.1 the following description is based on the blog post by
Christopher Olah ([41]). Many different architectures for LSTM’s exist and [15] is a
very good reference for additions and the history of these. Here the focus will be on the
original LSTM architecture proposed by Hochreiter and Schmidhuber [17] in 1997, with
the addition of forget gates proposed by Gers, Schmidhuber, and Cummins [13] in 1999.
The long short-term memory network (LSTM) was developed by Hochreiter and Schmid-
huber [17] partly to solve some of the problems traditional RNNs have. Its structure
allows it to capture both long and short term dependencies much better than a normal
RNN and it doesn’t have the problems with vanishing gradients due to a concept called
the constant error carousel basically being the hidden state cj,n, which makes sure the
gradients stay stable. The gradients will not vanish since the gradient of the hidden
state is actually the activation from the forget gate, which with the right initialisation
can be made sure not to be zero. This makes it much faster to train with BPTT using
gradient descent or similar optimisation techniques.

A single LSTM-cell, j, in a layer with m cells has two hidden states. A long term hidden
state usually called cn,j ∈R and a short term hidden state called hn,j ∈R. The short
term hidden state, hn,j, is also the output of the cell, which is fed to itself and all the
other cells in the layer. A sketch of a single LSTM-cell can be seen in Figure 3.5. It
is fed the output of all the cells in the network at the former time step, hn−1, which
is concatenated with the input at the current time step, xn, similarly to a traditional
recurrent neural network. The difference is in what happens inside the cell. The cell is
also fed the hidden state cj,n, which is the long term hidden state from itself at the former
time step, which is updated by the forget and update gate and thereafter outputted by
the forget gate.
As mentioned each cell has 3 gates: a forget gate, an input gate and an output gate
which all use a sigmoid activation function. The forget gate determines how much of the
long-term hidden state from the time step before, cn−1,j, should be forgotten. The input
gate determines how much of the input to the cell should be added to the hidden state.
The input consists of actual new input to the layer, xn ∈Rk, which is concatenated with
the output from the cells in the layer at the time step before, hn−1 ∈Rm. If the given
layer is the first in the network xn will be the input to the model. In our case this will be
the individual word embeddings. If the layer is one of the hidden layers beyond the first,
xn will be the output from layer before the given one. The output gate determines how
much of the updated hidden state, cn,j, should be output as hn,j. The three gates protect
both the hidden state of the given cell, but also protects the other cells by controlling
what should be output from the cell, since the errors only will propagate back when the
input gate is open. If the input gate is not opened the errors will stay inside the long
term memory loop.

outputs from the cells in the layer at the former time step, hn−1 ∈Rm. Note that hn−1
is a vector, since it is the output from all the k cells in the layer including cell j.

Retrieval-based models are based on finding the best candidate within a set of candidate
solutions. The candidate solutions are can be former seen pairs on which the model has
been trained or candidates, which have been obtained through interaction with humans.
Being retrieval based implies that the model can only come up with answers within the
set of candidates and thus can’t generate new or unseen candidates.
Generative models on the other hand generate a responses given an input sequence.
Sutskever, Vinyals, and Le [53] introduced sequence to sequence models based on LSTM
networks for machine translation, which has then been expanded to dialogue systems.
An illustration of the model can be seen in Figure 3.6. The model consists of two separate
LSTM networks - one is the encoder and one the decoder. The source sequence is first
fed to the encoder one embedding at a time to create a latent representation of the input
sequence. The latent representation is captured in the long and short term hidden states
of the cells, which is copied to the decoder network as an initial state, when the entire
source sequence has been fed to the encoder.
The decoder is then given a go-token, which indicates that the source sequence has
been encoded and it can start decoding. At each time step the embedding from the
token, with the maximum probability in the softmax from the time step before is fed
as input to the network, xn, along with the output at the time step before hn−1. An
end-of-sentence-token marks the end of the sequence.
During training of the network the true embedding from the target sequence is fed to the
decoder at each time step instead of the token with maximum probability in the time
step before. This is to ease the training of the network. Some have tried to first use
this method and then switch to feeding the previous most likely as a sort of dictionary

3.3.2
Candidate sampling algorithm
The conventional softmax described in 3.3.1 is very expensive to calculate for a problem
which has very large number of classes. For our problem this corresponding to a large
vocabulary. This is due to the denominator in (3.30) for which it has to calculate the
product between the output vector, hn, and weight vector, ok, for every class, k.
There exists many ways to optimize calculation of the softmax, but here we’ll use the
method proposed by Jean et al. [20] based on [4]. It subsamples from the full vocabulary
V, such that it doesn’t have to calculate the sum in the normalization of 3.30 over all
the classes in V, but only over a subset, V′. The following derivation is based on [20]
and [63].
For each batch, B, we create a set of candidate classes, V′, which is the union of sampled
candidates and the actual classes of the targets in the batch: V′ = Vsampled ∪{t}t∈B. Per
the definition of the union a class is only included once in the set V′ even though it might
be in both Vsampled and {t}t∈B. Each class, k, is included in Vsampled with a probability
Q(k). Q(k) can be modelled in many ways, but most often it is taken to be a uniform
==PAGE:30==
3.3 Sequence to sequence models
23
or log-uniform distribution. Theoretically it could be conditioned on the other words in
the sequence for increased accuracy, but since it is just an approximation here it will be
approximated using the log-uniform distribution in (3.6).
In practice all of the classes in the training batch are included first. Then a specified
number of classes are sampled from the vocabulary using the distribution Q(k) to con-
stitute Vsampled.
The union of these two sets then constitute the final set of classes
considered in the sampled softmax.
If Vsampled includes a true class, which by definition is already included in {t}t∈B, this
is considered an accidental hit, but since V′ is the union of the two sets a class will
only be represented once, meaning that V′ will have fewer samples if there are more
accidental hits. The number of accidental hits depend on the chosen distribution, Q(k),
but generally there will be more accidental hits using a log-uniform distribution rather
than a uniform distribution, since this is the distribution which closest resembles the
true distribution of the words. This means that Vsampled and {t}t∈B comes from the same
distribution rather than two different ones, which will make the overlap between the two
larger.

Instead of optimizing the log likelihood as proposed by Sutskever, Vinyals, and Le [53] we
could take a different approach toward the problem by maximizing the pointwise mutual
information as proposed by Li et al. [27]. The idea is that we’d like the information
shared between the source and target sentence to be as high as possible. The pointwise
mutual information of two sentences S and T is defined as:
PMI(S, T) = log
P(S, T)
P(S)P(T)
(3.35)

expected value of the pointwise mutual information across all possible S and T. If two
sentences are completely independent we have that: p(S, T) = p(S)p(T), which would
mean that: PMI(S, T) = 0. If P(S, T) < P(S)P(T) ⇒PMI(S, T) < 0 and if P(S, T) >
P(S)P(T) ⇒PMI(S, T) > 0.

Baseline
As a baseline comparison, which we can compare the trained models with, the retrieval-

has so much data that in many cases it might seem human.
Still Cleverbot is only able to answer with answers, which are part of the interactions

4.1
Dataset
The dataset used for the training and testing of the dialogue system is a corpus consist-
ing of movie subtitles compiled by Lison and Tiedemann [30] coming from the website
www.opensubtitles.org. It has been updated several times, but in the 2016 version used
for this project it contains monolingual and bilingual corpus’ for 65 languages and has a
total of 17 billion tokens across all languages. The English subset from the 2016 version
used here has a total of approximately 2.5 billion tokens and 337.8 million lines.
The main assumption in using this dataset for building a dialogue system is that the
lines in subtitles contain dialogue, i.e. that a given line in the dataset is the answer
or response to the line before it. In many cases this assumption is clearly wrong. For
example when the source and target lines comes from two different movies. But they
could also be from different scenes or by the same person. In some cases the subtitles
may not be verbal communication from the movie, but rather specifying some action,
sound or background music in the movie. For example: ”♪Soft music ♪”. At other
times some words in the sentence might not be part of the actual dialogue, but rather to
provide a style for the dialogue. For example: ”[Yells] Come over here!”. This obviously
means that the dataset has a lot of noise, but we assume that the model is able to
generalise due to the large amount of data.

4.3.1
Preparing the data
The preparation of the data has two main steps. Building the vocabulary and writing
a file with the tokenized corpus. A basic component in both of these two steps is the
tokenizer. It splits the sentence into tokens based on some rules and this tokenized
sentence is then used for building the vocabulary or written as the tokenized corpus.
The vocabulary thus actually contains tokens and not actual words as one might be
led to believe. Example 4.1 shows how some sentences from the actual dataset are first
being cleaned, then tokenized and then indexed based on the build vocabulary, which is
then written to the final file.
Tokenizer:
The original Tensorflow implementation of the sequence to sequence model
came with a very basic tokenizer. It had the following rules:
• Split the sentence into segments at each space. Furthermore split each segment at
the following characters: .,!?’̈‘:;)(
• Replace all digits with 0, such that they all are treated the same way.
The following rules were added to this tokenizer for better processing of the corpus:
• Also split at the following characters: [ ]
• Make all characters lowercase (Makes sure that words using different case aren’t
treated differently)
• Strip each sentence for all spaces and dashes in the beginning and end of the
sentence (Dashes proved to be present in many of the sentences in the beginning
of the sentences, but it was decided that they are more an artefact of the
The above rules provides a consistent style of the sentences fed to the network. Words are
not treated different based on case and styling or grammatical characters are separated
from the words.
==PAGE:45==
4.3 Implementation
38
Build vocabulary:
The original Tensorflow implementation provided a function for
building a vocabulary by looping over all the sentences in the corpus, tokenizing each
sentence and adding each token to a dictionary.
When the entire corpus has been
processed the word-list is sorted by the count, such that the most common words are
in the beginning. Then at the beginning 4 artificial tokens are added, which are part of
the sentence abstraction used in the sequence to sequence model. They are:
• PAD: Used to make sentences a certain length. Since most recurrent neural net-
work models need their input to be a certain length we pad sentences of shorter
length, such that they are all the same length.
• GO: Depending on the setup this token is used in the beginning of the target
sequence to indicate that the source sequence has ended and the decoder can now
begin constructing the target sentence.
This token is substituted for the EOS
token in some models.
• EOS: Indicates the end of the generated target sequence. In some cases it is also
used as the end of the source sequence instead of the GO token.
• UNK: The unknown-token is used for all the tokens not included in the vocabulary.
Having added these four tokens to the sorted vocabulary the 80.000 first tokens are
written to a vocabulary file.
The tokens not included will be given the UNK-token.
Having found this vocabulary it is checked if there exists a GloVe embedding in the
Spacy vocabulary for the given token. If there isn’t an embedding the token is removed
from the final vocabulary. The final vocabulary is therefore only has a subset of the
80.000 most frequent words. The final vocabulary used for the rest of the pipeline has
64.602 tokens.
Tokenize corpus:
Having build the vocabulary another pass through the corpus is
done to write the final tokenized corpus. It loops over all the sentences in the corpus
and splits the sentence into tokens using the tokenizer described above. For each token
in the sentences it finds the index from the vocabulary build before. If the token is not
in the vocabulary it is given the unknown-token. Then the indexes are written to a new
file, which is the one used for the training of the model. If a line is empty it is removed
from the corpus.

For some this can be a difficult abstraction and it might be more difficult to debug, but

Tensorflow has two different implementations of RNN networks. The one which was
originally used in the sequence to sequence implementation used static length input,
which means that the RNN network would run for the entire padded length for all
sentences in the batch. This is not computationally efficient and Tensorflow had recently
released a dynamic length RNN implementation, which only runs for the specified length
of the sequence and then copies the state to the rest of the padded states, which is much
more efficient.
But this also means that almost the entire model implementation was changed from the
original implementation except for the implemented loss function.
Model input:
Tensorflow generally works with tensors and the input to the model
is also tensors. The model needs all the source sentences in the batch to be the same
length and all the target sequences in the batch to be the same length. The maximum
sequence length is the same for the source and target sequences: Tmax. The sentences
==PAGE:48==
4.3 Implementation
41
are padded using the PAD-token. The model takes 5 inputs:
• Encoder inputs: The pre-processed tokenized source sequence padded to length
Tmax. It becomes a matrix of shape Tmax × |B|.
• Encoder lengths: The length of each source sentence in the batch before being
padded to length Tmax. It is a vector of length |B|.
• Decoder inputs: The pre-processed tokenized target sentences padded to length
Tmax. It has shape Tmax + 1 × |B|.
• Decoder lengths: The length of each target sentence in the batch. It is a vector
of length |B|.
• Decoder weights: A binary weight tensor. It has a one for all entries of to the
length of the actual target sequence. It has shape Tmax × |B|.
Word embedding matrices:
The embedding matrix, E, for the tokens are variables,
which can either be trained, static or a combination. They have an embedding for each
token, n, in the vocabulary. The embedding matrix has shape |V| × q with q being the
size of the embeddings. The encoder and decoder can either share the embedding matrix
or have their own, but generally the embedding matrix is shared if the same language
is used for the encoder and decoder. In sequence to sequence translation tasks from one
language to another one will tend to have separate embeddings matrices for the encoder
and decoder. The original implementation being for a machine translation task used
separate encoder and decoder matrices, but for this implementation they are shared.
If the embeddings are learned during training they are updated during each step of
the training using backpropagation. But one can also use pre-trained word embeddings
learned using for example GloVe or Word2Vec. These pre-trained embedding can either
be used as an initialisation for the training of the embeddings or statically as is, with
no further training.
Since the four control tokens: PAD, GO, EOS and UNK aren’t included in the vo-
cabulary of pre-trained vectors they will either still need to be trained or set to some
predefined vector. The embedding matrix of the four tokens can then be concatenated
with the matrix of the pre-trained embeddings, when they have been initialised and it
has been specified whether they can be trained or not. One needs to make sure that the
embeddings being trained are initialised to have the same magnitude as the pre-trained
embeddings and a magnitude corresponding to the initialisation of the weights, such
that the input to the activation functions are right.
The embeddings are applied to every element in the encoder inputs and decoder
inputs.
This means that the matrices of size: Tmax × |B| and Tmax + 1 × |B| are
expanded by one dimension such that the input to the encoder and decoder have size:
Tmax × |B| × q and Tmax + 1 × |B| × q respectively.

Encoder and decoder:
The encoder and decoder of the sequence to sequence model
are recurrent neural networks and for this project they both are LSTM-networks as spec-
ified in Section 3.3. In some models the encoder and decoder networks can share the
weights, but using separate weights for the networks is preferred in most articles. It pro-
vides the most weights, making it possible to better approximate the mapping between
the source and target sequence, but more importantly the encoder and decoder solve
different tasks. The encoder takes a sentence and encodes it to a latent representation,
while the decoder uses this latent representation of the source sequence to generate the
target sequence.
As mentioned in Section 4.3 the encoder and decoder have been heavily modified to utilise
the more computationally efficient dynamic recurrent neural network implementation.
Possibilities for investigating the states of the network during training have been added,
such that state of each cell for each example in the batch and for each time step can be
inspected.
The encoder and decoder networks are set to the same size, such that they have the
same number of layers and units in each layer. This is because the latent representation
is actually the internal state of each cell in the network and not a single output of the
network. The final state of the encoder network is thus copied to the decoder network
as the initial state instead of a zero initialisation as is used for the encoder network.
The networks takes an input of shape |B| × q at each time step and updates the states
of each cell for each sentence pair in the batch. Inputting all the examples in the batch
at a time makes it possible to utilise fast implementations of matrix-matrix products
instead of calculating the matrix vector products separately.
Loss function and gradients:
The loss function as specified in Section 3.3.1 and
3.3.2 is calculated using the outputs of the decoder. The output at each time step is
given to a sampled softmax classifier calculating the likelihood. This is averaged over
the sequence length and number of sequences in the batch as given in Section 3.3.1.
Tensorflow have special operations for calculating the gradients of the trainable variables
in the model. The gradient-operation is called directly on the output of the loss function,
since what we’d like is the partial derivative of the loss function for each of the variables.
Then one can add an optimizer, the most common one being gradient descent, but
Tensorflow also has interfaces for using optimizers such as for the Adagrad algorithm
[11], the Adam algorithm [23] or the RMSProp algorithm [55], which will update the
variables based on the calculated gradients. For this project the gradient descent method
has been used.
Checkpoints:
Tensorflow has a special interface for saving checkpoints during training
of the model variables. This means that when the entire computational graph is defined
we can load a checkpoint from a former training session of a model with the same
computational graph, which will set the values of the variables to the ones given in the
==PAGE:50==
4.3 Implementation

checkpoint. Checkpoints is furthermore a nice way to investigate the development of a
variable during training.
4.3.4
Training
Having prepared the data one can now begin the training of the actual model. The
training can be split into two parts, which are repeated for a specified number of steps
or until the loss is below a certain threshold. The first step is to obtain a batch of data
and prepare it such that it corresponds to the input of the model. Then the input is
fed to the encoder and decoder. The output of the decoder is used to calculate the loss
from which the gradients are found and applied to the trainable variables. During the
training checkpoints are saved, which can be loaded when training should continue or
for inspection of the variables and how they develop during training.
Get batch of data:
The original sequence to sequence implementation loaded all of
the data in the tokenized dataset into memory and then randomly sampled from this.
But due to large amount of data in the current dataset another approach was needed,
which didn’t need to hold all of the data in memory.
In order to sample sentences from a file without having a bias towards sentences of
certain length the following algorithm was used inspired by [46]:
while size(batch) < batch_size
jump to random place in document
if next character is a newline character
source = next whole sentence
if position is at end of document
# We have hit the last sentence in the corpus
source = first sentence in corpus
if 0 < length(source) <= max_length
# The source sequence is not in the test set and
# it is below the maximum length
target = next whole sentence
Add EOS-token to target sequence
if 0 < length(target) <= max_length
# The target sequence is not in the test set and
# it is below the maximum length
add source and target to batch
return batch
The algorithm keeps jumping to a random place in the file until the next character is a
newline character. This ensures that sentences are sampled with the same probability
regardless of length.
When it has found a newline character it takes the next two
==PAGE:51==
4.3 Implementation
44
sentences as the source and target sequence and checks that they are not in the test set
and that they are below the maximum length. The first sentence will be sampled only
if the algorithm jumps to the last position in the file.
The above algorithm can be quite computationally demanding if the sentences are long,
since it will have to resample a lot of times before hitting a newline-character. In the
beginning 25 times the needed batch size random numbers are sampled from a uniform
distribution over the filesize and sorted. This is done because it is faster to run through
a file from the beginning until the end, which means that if we sample and sort the
places we’d like to jump in the beginning we can run from the beginning to the end of
the file. We sample 25 times as many as needed, since this is approximately the average
length of the lines in characters, when it is tokenized, which means we will hit an new-
line character on average every 25th time. There is on average approximately 21 tokens
in a sentence and since low index tokens (which will have an index consisting of few
characters) will be much more frequent sampling 25 times as may seems appropriate.
When the lines have been sampled the lengths are recorded in two vectors. The target
sequence gets either a GO or an EOS token at the beginning and an EOS token at
the end. Then the sentences are padded to Tmax and Tmax + 1 for source and target
respectively and the weight matrix for the target sentence is constructed.
Calculate loss and update weight:
Having retrieved a batch of data the operation
applying the gradient-updates is called directly. Using the defined graph Tensorflow fig-
ures out which input is needed for calculating the wanted operation by going backwards
through the graph. If the input has been provided correctly it calculates the loss and
applies the gradients to the variables.
4.3.5
Decoding
When one wants the model to generate a target sentence given some input source sen-
tence a slightly different model is used compared to when training the model. During
training the true target word is given to the RNN network at time n instead of the
output from the network at time n−1. When decoding the predicted word at time n−1
is fed back to the network at time n as input. This means that one has to calculate
the full softmax before one can find the predicted word, compared to during training
where the output is just calculated at each timestep and then the sampled softmax is
calculated at the end. The predicted target word is fed to the decoder and a new target
word for n + 1 will then be found. This is repeated until Tmax iterations have been done
or an EOS-token is predicted.

The methods designed specifically for training word embeddings do this much faster

the right magnitude. LSTM-cells use both tanh and sigmoid functions as activation
functions in the cell, which makes this something one should consider.
Outside the
range [−4, 4] the gradient of tanh is almost zero, which means that the network will not
learn anything. For sigmoid this happens outside the range [−9, 9].

CHAPTER 6
Conclusion
This project has made an investigation into the current technology for dialogue systems
and machine learning techniques for sequence generation.
A sequence to sequence model was designed and implemented in Tensorflow and it was
trained on the subtitle dataset from www.opensubtitles.com by having it predict sentences
in the dataset based on the sentence before, which should simulate a dialogue between
two entities. The trained models were tested on 3 separate test sets one which were in
the context of the training data and two which weren’t. The methods used for testing the
dataset were perplexity and BLEU along with a qualitative assessment of the responses
generated. The basic sequence to sequence model performed very poorly on the test sets
compared to the training error observed. This could be due to the different methods used
for the target generation when training and testing and future work could investigate
whether switching during training to the same method as during testing would improve
performance.
It was investigated whether initialising the embedding layer of the model using pre-
trained word embeddings from GloVe improved performance.
From this it was not
clear that using pre-trained embeddings provide an improvement in performance. This
could be due to the simplicity of the model, which is not able to utilise the information
provided in the pre-trained. It was showed that the embeddings obtained when training
the embedding layer from a random initialisation were not a linear transformation of
the GloVe embeddings. This could be because the embeddings found for the sequence
to sequence model are different from the GloVe or due to the embeddings not having
converged. Inspection of the trained sequence to sequence embeddings show that they
have much less structure than the ones from GloVe.
The analysis of the methods showed that current methods for dialogue generation lack
several aspect of human dialogue. Typical loss functions like negative log likelihood are
not very good at modelling the diverse aspects of dialogue in which a source sentence
has several possible very different answers. Utilising word and sentence embeddings for
aligning the generated target sequence with the true one could provide a method for
better modelling the many aspect in a dialogue.

CHAPTER 7
Future Work
7.1
Data
The current dataset used from [30] only consist of single turn dialogues and is generally
of a very noise nature. It would be very interesting to have a dataset, which consist
of dialogue across several turns. This would also allow the model to be trained across
several iterations, which could possibly provide more diverse answer as the amount of
information gets bigger.
7.2
Dictionary learning
Using dictionary learning for the two different ways of handling the input to decoder
in the sequence to sequence model could possibly improve performance on the test set.
Feeding the true target token to the decoder during training instead of the one which
was output at the last time step makes it easier for the decoder to solve the problem and
it might be beneficial to use this type of training in the beginning. But since the model
is tested by having to generate the entire sequence switching to the other method, when
it has converged on the first might further improve the performance.
7.3
Different architectures
Many different architecture exist for an RNN cell and it would be very interesting to see
which have an influence on the performance of a dialogue system. Just for LSTM-cells
many different versions exist and it is not clear which parts of the cells have an influence
on the performance. Forget gates might for example not be necessary for short term
dialogue consisting of a source and a target sentence, since the states are reset after each
turn.
GRU-cells have become very popular recently as an alternative to the LSTM-cell. It has
a simpler architecture and has shown to perform equally well on several tasks.
==PAGE:69==
7.4 Training objectives
62
7.4
Training objectives
This project used negative log likelihood as the loss function for optimizing the model,
but this loss function is in general not very good at mimicking the diversity of human
dialogue. Others have tried using maximisation of mutual information as a loss function,
but including word embeddings or using reinforcement learning might provide more
diverse answers to the questions posed.
7.5
Preparation of data
The data pre-processing has many parameters and methods which can be tuned for
increased performance. The tokenizer is the major component of processing the corpus
and it would be interesting to see if using part-of-speech taggers as a part of the tokenizer
improved performance. A hypothesis is that it would be able to remove some of the
disambiguation from similar spelled words, which have a different meaning. Another
possibility would be to pass the part-of-speech as a separate input to the encoder and
decoder along with the word.
At the moment all the tokens, which are not in the vocabulary are set to the unknown-
token. This will often be names or places, which occur were little in a corpus like this.
By adding global tokens for names and places one could obtain improved performance
in these domains. The words which are rarely used might be the ones, which give rise
to the flavour and diversity of human language. Part-of-speech taggers could readily be
used for finding these.
7.6
Query expansion
Query expansion is a field of computer science used to expand a given query with extra
context or relevant information. In natural language processing this could involve finding
synonyms for a given word and using these in search queries along with the given one
or fixing spelling errors to obtain a query, which is more likely to yield a nice result.
In dialogue systems one could imagine using query expansion to provide context or
real world knowledge about concepts, which is not immediately available to the model
through what it has learned during training. Wikipedia is a nice source for providing
extra context to a query, which could give a model information about people, places or
concepts, which are relevant for the dialogue. A different case could be for a technical
service chatbot at a helpdesk, which could use technical articles or instruction manuals
through query expansion to help users on specific problems.
Although it might be possible to learn this information through extensive training of the
model using query expansion could be an easier way to include specific information in
the model.
