Abstract
Security teams nowadays face a difficult challenge: how much information
about a system should be revealed? Even though transparency helps oper-
ations, it can also give attackers information to refine their strategies. This
thesis tackles this dilemma through the Optimal Obfuscation Tree Problem
(OOTP): identifying what parts of an attack tree should be hidden from pos-
sible attackers to enhance protection without disrupting service.
We introduce LatticeXML, a tool that connects conceptual security frame-
works and hands-on threat assessment. Through the combination of attack
tree analysis and Partially Observable Markov Decision Processes (POMDPs),
we convert an exponentially complicated task into a manageable challenge.
Across six progressive iterations, we achieve an 88.8× efficiency gain, expand-
ing functional usage from 60 to above 200 nodes.
Principal contributions include: (1) the first formal treatment for obfusca-
tion aimed at XML attack trees, (2) validated refinement techniques for AND
gates reaching 100% precision, and (3) the introduction of trace reduction al-
gorithms that can reach up to 62% reduction. The system is also designed to
integrate with ADTool, automatically selecting which node is best for hiding
within existing workflows.
Practical testing on both random and actual threat diagrams verifies that
LatticeXML effectively converts the art of security concealment into a data-
driven methodology, allowing security specialists decide what to hide based on
concrete evidence and not just guess work.

Introduction
Modern systems nowadays face a hard challenge, as the quality of cyber attacks
increases, improving their defense mechanisms is a must. Organizations worldwide
need to figure out what information about their systems needs to be revealed or
concealed in order to minimize attack success and keep, at the same time, operational
functionalities.
This thesis addresses the Optimal Obfuscated Tree Problem (OOTP), creating dif-
ferent tools that aim to help in balancing transparency and security. The difficulty
involves not merely obscuring information, but also implementing strategic conceal-
ment.
1.1
Motivation and Problem
1.1.1
Context and Relevance
Modern organizations rely on diverse and complex interconnected systems that
present numerous attack paths. Security experts are known to use attack trees [1, 2]
in order to model these vulnerabilities, where each node represents a potential at-
tack step and edges interconnecting them represent logical dependencies (AND/OR
relationships). These kinds of models help identifying critical paths that attackers
might be able to exploit.
However, complete transparency about system architecture aids both attackers and
defenders. First, defenders need visibility when it comes to monitoring and man-
agement, on the other hand, attackers can exploit this information to enhance their
strategies.
Where the challenge lies is in selective obfuscation; it is necessary to specifically hide
certain attack steps to not only maximize security but also preserve the essential
functionalities of our system.
For example, consider a banking system in which an attack tree might model paths
that compromise customer data. Some of the attack steps might include physical
access, while others could exploit software vulnerabilities, and most of them require
a specific sequence of actions. This raises the question: given limited resources,
which attack steps should be hidden from potential attackers to protect the system?

1.1.2
The Optimal Obfuscation Problem
This thesis addresses a problem that combines two theoretical frameworks:
• Attack Trees: Hierarchical models used to represent possible attack strate-
gies, where nodes are connected using logical AND/OR gates.
• Partially Observable Markov Decision Processes (POMDPs): A math-
ematical model for decision-making under uncertainty, where the true state of
the system is not fully known.

Key Insight
Given a budget of B = 1 (hide one node), which node should we hide to
maximize security? The attacker’s optimal strategy changes based on what
they can observe:
• No hiding: Attacker chooses path B →E (success probability = 0.6)
• Hide D: Attacker is uncertain about A’s viability, as the expected cost
increases
• Hide E: Attacker still prefers visible path through A
Formal Definition
The goal of the Optimal Obfuscated Tree Problem is the fol-
lowing:
Given an attack tree T with a set of attack steps A, a budget B that limits how
many nodes can be hidden, and a security threshold τ, find a subset of nodes X ⊆A
such that:
1. |X| ≤B
(budget constraint)
2. min E[rew(T)] ≥τ
(security guarantee)
This problem is computationally difficult for several reasons:

• The number of possible obfuscation strategies grows exponentially: O(2n) com-

• Hidden (obfuscated) nodes introduce uncertainty, which creates complex belief

• Practical scenarios involve trees with hundreds of nodes, making efficient com-
putation even more challenging.

1.2
Objectives
The main objective is to develop a practical tool capable of solving the Optimal
Obfuscated Tree Problem by combining Lattice-POMDP theory with attack tree
analysis. This extensive goal is achieved through the following concrete research
questions:
RQ1: How can the Lattice-POMDP framework be adapted to work with
standard XML attack trees while maintaining theoretical guarantees?
We want to adapt the Lattice-POMDP framework to be able to work with stan-
dard XML attack trees while keeping theoretical guarantees.
RQ2: Is it possible to scale POMDP analysis from toy examples (10-20
nodes) to realistic attack trees (100+ nodes)?
Through algorithmic optimizations including trace reduction, parallel processing,
and intelligent bounds computation, we achieved practical scalability for trees with
hundreds of nodes.
RQ3: Does there exist an optimal strategy for AND gates that can be
validated empirically and exploited algorithmically?

1.3
Contributions
We proved and validated with 100% accuracy that hiding the lowest-probability
child nodes in AND gates produces optimal security outcomes.
RQ4: Which optimization techniques most effectively reduce computa-
tional complexity while preserving solution quality?
Multi-stage trace reduction proved most effective, eliminating up to 62% of re-
dundant computations while maintaining identical security guarantees.
RQ5: How can this problem be practically integrated?
We created a seamless integration with ADTool in order to allow users to visualize
their attack trees after execution.
1.3
Contributions
This thesis aims to make several contributions across theory, implementation and
practice:
1.3.1
Theoretical Contributions (Addressing RQ1)
1. Formalization of OOTP for XML Attack Trees: First formal approach
of the obfuscation problem for standard attack tree formats, establishing a
framework that connects attack trees with Lattice-POMDPs while maintaining
compatibility with existing security tools.
2. Mathematical Security Guarantees: We develop bound calculations and
belief state representations that ensure correctness and also enable practical
computation.
1.3.2
Algorithmic Contributions (Addressing RQ2, RQ3 and RQ4)
• Scalability Breakthrough: We were capable of extending applicability from
60 to 200+ nodes through multi-stage reduction algorithms, memory-efficient

2.1
Attack Trees
An attack tree is a hierarchical, graphical representation used to model potential
attacks against a system [3]. It consists of:
• Root node:
represents the ultimate goal of an attacker.
• Branches:
represent different paths that an attacker can take to reach their
goal.
• Leaf nodes:
represent the most basic attack actions.
There exist two types of connections between nodes:
OR nodes offer choices between two or more nodes. To reach the goal in a parent
node, it just needs to succeed in one of the child nodes. As shown in Figure 5, to
reach the root node "Bank Account", it is sufficient to execute one of its children,
"ATM" or "Online".
Figure 5: OR node
AND nodes require completeness. To reach the goal in a parent node, all child
nodes must be achieved. Contrary to the OR node, as it can be seen in Figure 6, in
order to reach the node "ATM", it is mandatory to execute both its children "PIN"
and "Card" successfully.

2.1.1
Formal Definition
An attack tree T = (V, E, r, λ, p) consists of:
• V : Set of nodes representing attack goals or steps
• E ⊆V × V : Edges showing parent-child relationships
• r ∈V : Root node (the main attack goal)
• λ : V →{conjunctive, disjunctive, leaf}: Refinement type
• p : V →[0, 1]: Success probability for leaf nodes
The root node represents the attacker’s ultimate objective. Leaf nodes are atomic
actions that cannot be decomposed further.

2.2.1
The Observability Problem
Traditionally, when analyzing attack trees we assume attackers have all the infor-
mation needed. All probabilities, vulnerabilities, defenses, however, attackers have
to try, guess and learn how the system works as they go.
Here is where Partially Observable Markov Decision Processes have an impact. As
POMDPs model decision-making in a situation in which you cannot see everything,
it is perfect for the problem described in this thesis [4].
2.2.2
Lattice-POMDPs
Attack trees can be mapped to a POMDP structure. In a standard POMDP, the
state space can be anything, but in a Lattice-POMDP for attack trees:
- States = 2Act (all possible subsets of completed attack steps)
- Initial state = ∅(no attacks completed)
- Goal state = all necessary attacks completed
- Actions = attempting any uncompleted attack step
The "lattice" comes from the subset relationship. For example, if an attacker has al-
ready completed attacks {A, B}, they will be able to reach {A, B, C} by successfully
executing attack C. All these kind of transitions form a lattice structure.

In this section, we introduce the Optimal Observability Problem, which is to control
what an attacker can observe. We define:
• A budget B, a positive integer (N+) representing the maximum number of

2.4
The Optimal Obfuscation Problem (OOTP)
The Optimal Obfuscation Problem reformulates the previously defined problem:
Given an attack tree T, find a subset X of attack nodes to hide such that:
1. |X| ≤B (States the number of nodes that can be hidden)
2. The resulting security level meets our threshold τ
This formulation of the problem allows us to directly answer the question: "Which
attack steps should we obfuscate to reach our goal?"

Hiding nodes in an attack tree directly translates to observation constraints in the
Lattice-POMDPs. If we hide a node n in an attack tree, then all the states that differ
only in whether n has been completed or not instantly become indistinguishable to
an attacker.

ADTool
ADTool [5] is a tool for modelling and analyzing attack-defense trees, which can be
useful for structuring attack strategies in OOTP. While ADTool does not natively
support POMDPs or obfuscation, its output can be integrated into our framework
by transforming the attack tree into a Lattice-POMDP model.
3.1.1
How can it be integrated
1. Model the Attack Tree in ADTool: Define attack steps as nodes and
refine dependencies using AND/OR refinements.
2. Convert to Lattice-POMDP: Map the extracted attack tree into a Lattice-
POMDP model:
• Nodes in ADTool correspond to states in POMDP.
• Edges represent transitions between states.
• Attack step success probabilities define transition probabilities.
3. Apply Observation Constraints: Define an observation function such that
at most B attack steps remain observable.
4. Solve for Optimal Observability: Use an external solver, like Alyzia’s, to
optimize the observation function while ensuring that the expected minimum
reward remains above the threshold τ.
5. Evaluate Different Obfuscation Strategies: Experiment with different
values of B to analyze how observation limitations affect attack success prob-
abilities.

Conclusions and Future Work
In this thesis, I tackled the Optimal Obfuscation Tree Problem (OOTP) thanks to
the development of the LatticeXML system, a tool capable of connecting theoretical
security frameworks with actual security analysis. With an iterative development
this exponentially complex problem has become a tractable challenge.
9.1
Contributions Summary
9.1.1
Theoretical Contributions
The thesis has made several significant theoretical advances:
1. Formalization of OOTP for XML Attack Trees: We provided the first
formal treatment of the obfuscation problem for standard attack tree for-
mats, establishing a mathematical framework that connects attack trees with
Lattice-POMDPs while maintaining compatibility with existing security tools.
2. AND Gate Optimization Theory: We developed and validated a hypoth-
esis that optimal obfuscation strategies for AND gates follow predictable pat-
terns. With an 100% validation rate across diverse test cases, this finding
enables targeted optimization that significantly reduces computational require-
ments.
3. Dominance-Based Reduction Framework: We introduced novel algo-
rithms for identifying and eliminating redundant attack traces while preserving
optimal solutions, achieving up to 62% reduction in computational complexity
without compromising solution quality.
9.1.2
Implementation Achievements
The LatticeXML system represents a substantial engineering achievement:
1. Scalability Breakthrough:
