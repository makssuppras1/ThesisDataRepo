Abstract
This master’s thesis is written by Sival Ekner Bobek in collaboration with DTU,
Technical University of Denmark, as part of the Mechanical Engineering program.
The project is constructed with the guidance of Associate Professor Casper Sc-
housboe Andreasen.
The purpose of this project is to simulate and make an experimental hydraulic
test-rig with the goal of obtaining an identiﬁcation of a position servo. For this,
an inverted pendulum on a cart is used to create the foundation for the computer
simulation and the physical test-rig. The inverted pendulum is chosen because
of its generic unstable nature. If a computer simulation can simulate a highly
nonlinear and unstable system with matching results from the test-rig, it is possible
to simulate relevant test cases.
This thesis utilizes a state-space analysis in order to investigate the stability
of the inverted pendulum assembly. This is done by establishing the eigenvalues,
which allows for the pinpointing of the uncontrolled state of the system’s stability.
The experimental test-rig consists of a hydraulic pump system and a propor-
tional valve that is used to control a hydraulic actuator. The hydraulic system is
balancing an inverted pendulum. Further, an Arduino UNU microcontroller with
two PID controllers is attached to the test-rig, which is used to control the ﬂow
through the hydraulic valve. The ﬁrst PID controller is for the angle of the pen-
dulum and the second is to control the position of the cart. The PID controllers
do so by each using an input. One input is from a potentiometer that monitors
the angle of the pendulum and one is from an ultrasonic rangeﬁnder that mea-
sures the position of the cart. In addition to building the experimental hydraulic
test-rig, a simulation has been created. It consists of a multi-body mathematical
model of the inverted pendulum, as well as a simulation of the hydraulic system,
all connected.
Newton’s 2nd law is used to calculate which parameters are relevant when an
inverted pendulum is to be kept upright. It is deducted that the distance from the
rotating joint to the center of mass is the primary factor.
Diﬀerent types of proportional valves are tested to ﬁnd the best type for this
==PAGE:3==
ii
particular purpose. The result was a 4/3 way critical lapped proportional valve.
The response time of the system is crucial to the stability of the system. The
method by which the Arduino transmits data to the valve is reviewed to ﬁnd an
optimized response time. The best solution is a 12bit DAC (Analog to Digital
Converter) as it showed the best response time and, thereby, the most stable for
controlling the balance of the pendulum.
The results from the experimental setup and simulation show similar trends.
Theory and practice match well.

Chapter 1
Introduction
As automatization becomes an increasing important cost-saving factor and tool for
consistent reliability in most factories, a demand for cheap but ﬂexible automation
solutions that can be implemented on an existing system will continue to rise.
Imagine having an old hydraulic system that, with only very few modiﬁcations,
can become automated without the need for expensive control systems. But, as
this thesis proves, only an Arduino microcontroller and a few sensors are needed
to automate a system.
Likewise, being able to use hydraulic systems for robotics, instead of linear
electric motors will enable the manufactures to save space and cost, by having one
pump system for multiple actuators, instead of having an electric motor of each
actuator.
In this project, a hydraulic system is maintaining an inverted pendulum up-
right, by having an Arduino UNO microcontroller run a closed-loop PID (Propor-
tional–Integral–Derivative) controller on an ordinary hydraulic proportional valve.
This is done in order to prove that automation can be done relatively easy and
cheap, using microcontroller. (See ﬁgure 1.1)
==PAGE:8==

Figure 1.1: The test-rig
Inverted pendulums are used everywhere, from kids’ toys (hover-boards) to
skyscraper buildings, where sensors detect movement caused by wind, pump water
or oil into diﬀerent reservoirs of the skyscraper, to control the balance of it. The
inverted pendulum is a generic unstable problem that is important to control
in many real-world scenarios and makes a ﬁne baseline for this project. If one
can hold the balance of an inverted pendulum, one can control various relevant
components in modern mechanical solutions. While humans have built-in control
systems, engineers need to design control systems for mechanical mechanisms. As
proven in this thesis, a PID control system can be applied to make it possible for
a nonlinear system to control its balance.

1.1
Motivation
In times where automation is a big part of the industrial world, being able as
an engineer to design and build an automated system, as well as getting it to
work, is a relevant issue to solve in today’s world. Every mechanical engineer can
construct a simple hydraulic system, but writing the code, creating the simulation,
and getting it all to work as one unit is what makes this project interesting. The
inverted pendulum is a classical control engineering problem due to its unstable
behavior. Therefore this issue was chosen to be solved by combining hydraulics

with control engineering, which makes an interesting combination of pure analogue
mechanical force transmitting and the digital control system.
Most robotic solutions use electric motors, as the primary actuator. However,
that is expensive, space-consuming, and relatively weak compared to hydraulics.
Hydraulic systems can often be used instead, as long as their are properly con-
trolled.
Utilizing the hydraulic method enables a system to lift heavier things or move
faster, using the same space. This in turn, enables factories to save space and
money.
The downside to this is the controllability of the system.
An electric
motor is much easier to control compared to a hydraulic system and does not
contaminate product with hydraulic oil.
1.2
Project Scope
The main objective is to design, simulation, and conduct experimental identiﬁca-
tion of a servo mechanism, by constructing a hydraulic actuator test-rig, as well
as a simulation of the test-rig to see if the results match. The test-rig is to be
constructed using oﬀ-the-shelf products, to keep the price at a feasible level for
real-world scenarios.
An Arduino UNO cost €20 [16] where the most basic control system from

The Pendulum
It is necessary to design the test-rig and simulation, with a suitable ratio between
the parts. For instance, the movement of the cart and the length of the pendulum.
By knowing the eﬀect of each parameter used to describe the system mathemati-
cally, it is possible to deduce a design that is likely to be successful.
In the following section, the mathematical models necessary to describe the
pendulum and cart system is established and discussed.
First, an FBD (Free Body Diagram) of the pendulum is made (See ﬁgure
2.1), and afterward, Newtons 2nd law is used to calculate the acceleration of the
pendulum.
The mass located in the FBD is the COM (Center Of Mass) of the entire
pendulum. The coordinate system is ﬁxed.

The amount of torque the pendulum delivers to the joint (O) can be described
in two ways. As the inertia (I = mp ·L2 ) times the angular acceleration M = I · ¨θ
or as M = mp · g · d where d is the length the mass have moved away from the
upright position in the x-direction d = L · sin(θ).
Setting those two equations equal to each other leads to the following equation:
mgL · d = md2 · ¨θ which reduced will look like this: ¨θ = g
L · d.
The angular acceleration is, therefore, dependent on the length of the pendulum
L.
A pendulum where the center of mass is further away from the rotating joint
will accelerate slower, and therefore easier to maintain upright, whereas a short
pendulum will accelerate much faster and therefore require a faster response time.
However, a long pendulum will also require the actuator to move a long distance
in order to get the rotating joint under the COM and thereby get the pendulum
upright if the pendulum starts at the same angle.
The pendulum in this project has one DOF (Degree Of Freedom), which is the
rotation around the z-axis. By adding the cart to the pendulum, a second DOF
needs to be taken into account, which is the movement on the x-axis.

2.2
Hydraulics
Hydraulic systems are based on Pascal’s principle of transmission of ﬂuid pressure.
The principle states that if a conﬁned incompressible ﬂuid is exposed to a change in
pressure, the pressure will throughout the ﬂuid have the same change in pressure.
And since the pressure is equal to force times area, a hydraulic system can be used
as a gearing.

2.3
PID Controller
A PID - proportional, integral, and derivative controller, is a tool to adjust a
speciﬁc input. It works by adding a feedback loop to a system. This way, a sensor
can tell the PID controller if the system is in the correct state, and if that is not
the case, then the PID controller can calculate the needed input, for the system
to return to the desired state.
The process of a PID controller is corresponding to a person on a bicycle. If
he is tilting to the right, he will turn to the right side in order to get the center of
==PAGE:17==
2.3 PID Controller
11
mass under him again. There are three parameters, the angle of the handlebars,
the duration of the turn, and the velocity he is carrying.
Just like the example with the cyclist, the PID controller also has three things
happening at the same time. The proportional gain (Kp); Times the error (see
ﬁgure 2.4) with the Kp value. This is similar to turning the handlebars when the
cyclist is falling to the side. If the bike is tilting 5°from upright, the rider will
turn the handlebars 2.5°if Kp is set to 0.5, and both the integral part (Ki) and the
derivative part (Kd) are set to zero.
Figure 2.4: PID Controller [4]
The integral part (Ki) looks at the steady-state error. This is what the cyclist
does to avoid riding in circles. If a cyclist has a steady-state error, it will result in
the cyclist trying to get the bike upright, but never fully succeeding, and therefore
have to keep turning the handlebars in order to not fall over entirely.
The derivative part (Kd) is monitoring the speed of the bike. If the bike is
going too slow, the handlebars have to be turned a lot to compensate, but if the
cycle is going fast enough, only a small adjustment is needed to keep the balance
of the bike. The Kd is looking at the rate of change of the bikes upright angle.
By looking at ﬁgure 2.5, one can see how the diﬀerent parameters aﬀect a
system.
==PAGE:18==
2.3 PID Controller

Figure 2.5: PID Tune Table [12]
A well-tuned PID controller will manage to get a system to a certain point
relatively fast and stay there while still being stable.
In this thesis, two PID controllers are used to control the single input to a
valve.
The ﬁrst PID controller (the angle PID) exists to ensure the correct angle of
the pendulum, this one is very aggressive, and overshoot will occur. The second
PID controller (the distance PID) exists to control the position of the cart. This
is tuned to be much slower since it just needs to get the pendulum back to the
middle of the actuator stroke. This enables the system to react to a disturbance
in any direction.
The two PID controllers are positioned so that the output from the distance
PID controller changes the setpoint of the angle PID controller. See ﬁgure 2.6.
This means that the angle PID will try to keep the pendulum at the upright
position, plus the output from the distance PID, which will turn the angle setpoint
in the direction of the actuator stroke center.
The angle PID will, therefore, at all times, aim the pendulum in the direction
of the actuator center. The output of the angle PID is then sent as a signal to the
valve. This way, each PID can be tuned individually, which makes it much easier
to tune the diﬀerent parameters.

If the
pendulum is at one side, aiming towards the center, then the distance PID will force
the actuator away from the center, while the angle PID will force the pendulum
towards the center. This will result in zero movement of the cart.
PID controllers are good at adjusting one thing at a time. However, if more
than one controller is controlling the same input, as is the case in this thesis, the
PID controllers tend to interfere with one another, unless they are arranged in a
clever way.

ball bearings. At the end of the axle, a rectangular wheel is attached, this wheel
is connected to a smaller wheel, using a belt, creating a gearing of i = 70mm
9mm = 7.8.

degrees. The gearing allows the pendulum to rotate 46 degrees before the angle
sensor has rotated 360 degrees. See ﬁgure 3.1. This results in a more accurate
reading of the angle of the pendulum. Two bolts limit the rotation of the pendulum
so that the pendulum can only rotate 23 degrees, 11.5 degrees to each side of the

The control valve used in this thesis is a directly operated directional control
proportional valve. The reason why a servo valve is not used instead, which is

This thesis is meant to show that it is possible to control a servo system using

be accomplished using a combination of a servo valve and a fast responding control

The diﬀerence between a servo valve and a proportional valve is the mechanism
that controls the spool inside the valve.
When choosing a valve, there are multiple factors to consider. The main one
is the type. For this thesis, a proportional hydraulic valve has been chosen. The
second-most important things are the size of the valve (L/min), the number of
inlets/outlets, the number of diﬀerent positions the valve has, the type of lapping,
and the signal type needed to control the valve. For this project a 12L/min at

The width of the spool compared to the width of the port determines the type
of lapping. There are three diﬀerent types. The ﬁrst one is under-lap, where the
width of the spool is smaller than the width of the valve port, resulting in leakage
when the valve is in the neutral position, forcing the actuator to extrude, due to
diﬀerence in the area on either side of the piston.
The second is zero-lap also called critical center. This type of lapping is deﬁned
by the width of the spool being exactly the same width as the port in the valve.
In theory, this valve will not have any leakage, but in reality, a small leakage will
occur.
The third and ﬁnal lapping is called over-lap. It is deﬁned by the spool’s width
being larger than the valve’s port. This results in zero ﬂow, even if the spool is
moved slightly, as seen in ﬁgure 3.3 and 3.4.

When choosing the control signal, it is important to consider, that there are
two diﬀerent types of control systems.
One is controlled by current, generally
between 4..20mA with a constant power supply of 24Vdc, this kind of valve comes
with either two input plugs, one for each solenoid, or it can come with integrated
electronics that control one or two solenoids. The other type of control is a voltage
signal between -10..10V.
DTU had one of each type in a suitable size, which could be used for this
thesis. Both types were tested in this thesis, but the one with the +-10V signal is
over-lapped, and therefore too slow for this particular project.
There are a few things to consider when choosing the type of control signal.
If a valve without electronics is chosen, one would have to control the solenoids
individually. This would force the Arduino to run two output signals, and therefore
have two 0..5V to 4..20mA converters as well. An Arduino can only output an
on/oﬀsignal of 5Vdc.
For the Arduino to produce an analog signal there are two options: An PWM
signal from 0 to 5V can be used in combination with a low pass ﬁlter or a DAC
(Digital to Analog Converter) can be used. See 3.5.1 for more info about analog
signals.
To be able to control a valve that is controlled with 2 x 4-20mA, two analog
signals are needed and two electric circuits are needed that can convert 0..5Vdc
into 4..20mA, 24Vdc.
The other type of proportional controlled hydraulic valve is controlled by a
voltage signal +-10Vdc and a power supply of 24Vdc. A major issue with this
type of control signal is that the resistance of a cobber wire is dependent on the
temperature of the wire.
So if a system is calibrated when the wire is hot the voltage drop across the

cable will be higher than when the cable is cold, resulting in a system that is not
calibrated correctly. This system is much more likely to have disturbance in the
signal. Therefor industrial valves usually use the current controlled valves.
Since the precision of the valve is very important in this type project, this can
become a problem. But because this system is not going to require a high current
or run for a long time, it might not be an issue for this project. Furthermore, the
fact that a voltage drop when 0V is sent will not change anything (0Vdc is when
the valve is dead center), will ensure the Arduino still knows where zero movement
will occur, and thereby know when a direction shift will happen.
Another thing to consider is that fact that the signal is bipolar (uses both

be sent to a 0..5Vdc to -10..10Vdc bipolar opamp, before it is transferred to the

for this project.

oil and the velocity of the piston / rod.
When investigating the friction caused by the seal and the relative motion of
the piston, there are three primary phenomenas to take into account:
• Friction lag is where the friction will change some time after the velocity has
changed due to the time it takes to build up a new height of oil ﬁlm.

diﬀerence has build-up for the piston to move again. This is apparent as a

3.4.1
Sampling Time
The speed of the system is critical to ensure a stable outcome. As seen in ﬁgure
3.9, there is a big diﬀerence in how the system reacts when the frequency of the
PID controllers is changed.
(a) Angle error, simulated at 1000Hz
(b) Angle error, simulated at 50Hz
Figure 3.9: Diﬀerence in stability due to sample time
In 3.9, all the parameters are the same - the only thing that changes between
plot (a) and (b) is the sampling rate of the PID controllers.
As seen in ﬁgure 3.9, the system overshoots much more, when the frequency
of the system is low.
A fast system is, therefore, essential to ensure stability.
However, an Arduino does not have a constant frequency for its calculations. It
very much depends on how many and the type of calculations it has to perform
every loop. Therefore it is diﬃcult to predict how fast the Arduino is. However,
as a baseline, an Arduino UNO can have a sample rate of 9615 Hz [23]), if it does
not have to calculate anything. If the Arduino has to produce a PWM signal, the
frequency will be around 976Hz [9]. A DAC can handle a frequency of around
400Khz, depending on the DAC and the way it is coded [15].
Having a fast
sampling time means that the time it takes for the Arduino to determine how
much the valve should open will be much faster. Using a DAC instead of a PWM
signal will therefore result in a more stable outcome.
If a PWM signal was used instead of a DAC, one might have to add an RC
network (resistor and capacitor circuit), to be used as a lowpass ﬁlter, to ensure a
steady signal for the valve [5].

The Pendulum and sensing subsystem is receiving one signal from the control

The Motion_actuator signal is used to control a modeled pin slot joint (px),
which represents both the x-axial movement of the cart as well as the rotational
movement of the pendulum around the z-axis. The coordinate system is created
in the top left corner and used to place everything in a global coordinate system,
as well as adding gravity to the system. Simscape multibody has been used to
create a mathematical model of the pendulum and cart. Just like in chapter 2.1,
there is no damping or friction associated with the movement or rotation.
The pendulum and cart are visualized in an animation showing how the pen-
dulum assembly will rotate and move, according to the movement of the hydraulic
actuator. This makes it easy to understand what is happening when the simulation
is running.
The cart is made as a solid sphere, with the same weight as the real cart. It
can move on a 500mm long rail, with the default position being in the middle.
The pendulum is attached to the cart, and modeled as a rod with zero mass
and a sphere on top, with the mass of the pendulum being 252g. This makes it
simple to calculate and adjust the center of mass.

From the slot joint, three signals are sent out, the position of the cart, the
angle of the pendulum and the velocity of the cart:

The setpoint / ref value (the position the PID controllers aim for) is then
subtracted from the position signal before it is sent into the position PID controller.
The output from the position PID controller is then added as the setpoint value
of the angle PID. As described in section 2.3.
The angle PID output is then converted to a command signal for the valve.
Afterward, a sample rate limiter of 50Hz is added, in order to ensure the sample
rate of the simulation is matched with the sample rate of the Arduino.
By placing a sample rate limiter here, instead of limiting the solver, will enable
the solver (ode15s) to solve the rest of the system at the required speed, making
the simulation more realistic.
After the sample rate limiter, a signal rate limiter is placed; this is used to
simulate the speed of the spool inside the valve. The valve used for in this thesis
does not have any information about the movement speed of the spool, so the data
used is from a similar sized valve, made by Rexroth.
Figure 4.5: Rexroth valve size 6 (@ ∆P = 10bar Q=16l/min)
The rising and falling slew rate is set to 0.4, making the spool take 25ms to go
from 0 to 100 % stroke. After this, the command signal is sent to the valve in the
hydraulic system.

Hydraulic System
The hydraulic system consists of a variable-displacement pressure-compensated
pump that is rotated with a constant rotation speed, a 4/3 way directional valve,

Figure 4.6: Simulink system (Hydraulic)
The variable-displacement pressure-compensated pump is connected to a con-
stant rotational source of 188PRM. This is done to keep the pump producing
20l/min at 75bars. The pressure relief valve (75bars), is added in case of overﬂow
from the pump to the valve. The oil used is 50wt at 60◦C.
The valve is a 4/3 way valve, which has been modiﬁed to match the one from
the test-rig. The data used for this can be seen in ﬁgure. 4.7.
This data is made by using equation (3.1) with data given by the manufacture,
and can be see in ﬁgure 3.5.
The valve makes an interpolation of the data given. Two vectors and a matrix
is used as the ﬂow / pressure data given to the valve. One vector is the amount of
pressure, the other is the command signal, and the matrix is the amount of ﬂow
at the given vectors. The valve conﬁguration then makes an interpolation of the
values given.
If the valve is told to open 80% with the pressure diﬀerence of 70bars, but
the data that have been given to the valve is 75% and 100% at 75bars, it will
interpolate between these two ﬂow values, and between the two closest pressures
given. Making the valve linear.

Chapter 6
Conclusion
Hydraulics can be diﬃcult to both simulate and control, due to its nonlinear
behavior.
An inverted pendulum can be linearized, but it is not easy to keep
balanced. This thesis addresses whether it is possible to control a hydraulic system
precisely enough to balance an inverted pendulum and to create a simulation
that can match the results. Hereby making an identiﬁcation of a position servo
mechanism.
Throughout this thesis, a hydraulic test-rig setup has been constructed phys-
ically, alongside a computer simulation setup, acting as an approximate virtual
copy. This was done in order to make an identiﬁcation of a position servo.
An Arduino UNO was successfully used to control the test-rig, by running
two PID controllers.
This was done to prove it is possible to use a relatively
cheap alternative to the much more expensive solutions, such products oﬀered by
commercial vendors. Using two PID controllers to control one valve, showed to
be a diﬃcult task, nevertheless a clever PID arrangement made it possible for the
two PID controllers to work along side eacohter, creating a stable solution for the
system.
By measuring the hydraulic pump characteristics used in the test-rig, and using
the corresponding data-sheets of all the parts used in this project, it was possible to
create a computer simulation that matches the test-rig. The computer simulation
was able to mimic the characteristics of the physical test-rig while also providing
very similar results.
The simulation does diﬀer slightly from the test-rig mostly due to unaccounted
variables such as poor sensor precision performance and vibrations issues in the
physical setup.
Nonetheless, the two setups do show similar characteristics in
terms of magnitude and speed, both the ﬂow through the valve and the movement
of the actuator, when balancing the pendulum.
This thesis proves that a hydraulic system can be integrated into automated
solutions, providing a more cost eﬃcient and less spacious solution for some me-

chanical solutions. At the moment, most robotic arms used in production lines are
equipped with electric motors because they are linear and therefore easy to control.
However, Hydraulics tent to have a better mass to power ratio, can might therefore
become handy in some applications, where strength is an important aspect.
6.1
Future Works
The hydraulic system created in this project is currently using two PID controllers
to keep the pendulum in balance at the desired position. This is not the best way
to arrange a closed loop system with multiple inputs, but only one output. A
diﬀerent type of controller would, therefore, be a needed upgrade, to ensure more
stability in the system. An LQR (Linear–Quadratic Regulator) controller might
be a better choice. This requires additional research.
Regarding the test-rig, a new valve with a larger ﬂow would most likely be
a good thing to use, since the valve used at the moment is the bottleneck in
the hydraulic system regarding acceleration. This would enable the actuator to
accelerate quicker and thereby be able to balance the pendulum better. New and
more precise sensors are also needed to get the test-rig to perform better.
It would be interesting to redesign the pendulum assembly, so the initial po-
sition of the pendulum is downward, requiring an upswings algorithm, to get the
pendulum at an angle where the controllers can balance the pendulum.
Adding a second pendulum to be balanced on top of the ﬁrst pendulum would
also be interesting. Having two pendulums in series, producing multiple inputs for
the PID controller(s) to monitor.
Regarding the simulation, it would be interesting to see how much the diﬀerent
parameters aﬀect the outcome. For instance, add friction to the system to see if
the simulation becomes more accurate or stable due to the added damping eﬀect,
or by linearizing the valve even more, in order to see how simple the simulation
can become, while still producing useful results.
One can also discard the pendulum and attach a load cell on the test-rig and
within the simulation, to see if it is possible to establish a precise force transmission
by controlling the proportional valve.
Note: An interesting thing is how diﬃcult it is to ﬁnd hydraulic control systems
online, for automated solutions.

Interface Program
ProPxD interface program 
The ProPxD software allows quick and easy setting of 
the digital valve electronics. Individual parameters as well 
as complete settings can be viewed, changed and saved 
via the comfortable user interface. Parameter sets saved 
in the non-volatile memory can be loaded to other valves 
of the same type or printed out for documentation pur-
poses.
The PC software can be downloaded free of charge at 
www.parker.com/isde – see page “Support" or directly at 
www.parker.com/propxd. 
Features
•	 Comfortable editing of valve parameters
•	 Saving and loading of customized parameter sets
•	 Executable with all Windows® operating systems from 
Windows® XP upwards
•	 Simple communication between PC and valve electron-
ics via serial interface RS232C
The valve electronics cannot be connected to a PC with 
a standard USB cable – this can result in damages of PC 
and/or valve electronics.
The parametrizing cable may be ordered under item 
no. 40982923.
