Summary (English)
Most of the large datasets for visual question answering and reading compre-
hension are created using annotated data. Annotating large amounts of data
using crowdworkers is expensive. The goal of this thesis is to address this is-
sue by creating a model that can achieve near state of the art results for the
GQA visual question answering dataset using only a small fraction of annotated
supervised data. We propose a solution using the Meta Module Network in com-
bination with the modiﬁed MAPO model of another thesis [Joh20]. We recreate
the Meta Module Network model and achieve 58.85% performance which is less
than 2% lower than the original paper, and argue that the performance of the
modiﬁed MAPO model can be stabilized using new hyper parameters, as it has
shown high variance in performance at diﬀerent runs of the model. We show
that accuracy of the program generator for the Meta Module Network using just
0.6% of the annotated training data is within 1% of the performance using the
full training data. This lays the foundations for combining it with the modiﬁed
MAPO model which should increase the performance of the execution engine
when using less annotated training data. The code for the modiﬁed MAPO
model has also been refactored to make it easier to apply it to new models.

Chapter 1
Introduction
In the process of deep learning models evolving and becoming better, the need
for more new and more complex dataset arises. In the creation of these new
datasets a lot of annotated supervised data is often added to assist models in
the training process. When working in the ﬁeld of question answering anno-
tating this data is usually slow and expensive. Many of the datasets for multi-
hop reading comprehension (questions requiring multiple steps of reasoning)
and visual question answering (VQA) has been constructed using paid crowd-
workers for annotating the data, constructing questions, create reasoning steps,
etc [How06] [KCS08]. Some of the newer datasets such as DROP[DWD+19],
SQuAD[RZLL16] and BREAK[WGG+20] has budgets in the 10 000’s of dollars
just for crowdsourcing. Even though the exact budgets for VQA datasets such
GQA[HM19b] and VQA1.0[AAL+15] are not publicly known, they also utilize
crowdworkers to annotate data. To reduce the needed amount of annotated
data we wish to create a model that performs similar to current state of the art
models, while requiring signiﬁcantly less annotated training data.
The foundations for this has already been made by [Joh20] by achieving state
of the art results on the CLEVR[JHvdM+17a] dataset while only using a small
fraction of the training data. The CLEVR dataset uses synthetic image data,
which lacks the complexity of real life scenarios[JHvdM+17a]. The aim of this
project is to see if similar results can be achieved on a larger and more com-
plex dataset consisting of real life images. Speciﬁcally we want to investigate
==PAGE:16==

how much annotated data is needed to achieve near state of the art results on
the GQA dataset, and investigate if applying the modiﬁed MAPO algorithm
used in [Joh20] on a model for the GQA dataset can increase the model perfor-
mance when using less annotated data. The GQA dataset is one of the most
extensive visual reasoning datasets in terms of size, complexity and semantic
compositionality[HM19b].
As the MAPO model was not originally intended for VQA tasks, [Joh20] mod-
iﬁed the MAPO model to work speciﬁcally for the CLEVR dataset using the
Neural Module Networks model[JHVDM+17b]. We refactored the code base to
act more as a framework, making it is easier for the data science community to
apply new models to it, and thus in the long run limit the need for annotated
data.
To achieve near state of the art results for the GQA dataset, we have:
• found a model for GQA that would ﬁt the MAPO structure
• recreated the results of that model
• shown that it works using less annotated data
• refactored [Joh20] code base and recreated his results
• tuned the hyper parameters of [Joh20] to give more stable results
After researching the current state of the art models for the GQA dataset,
we found that the Meta Module Network (MMN)[CGL+19] would ﬁt well for
the objectives listed above. MMN is created to preserve compositionality and
interpretability through a modularized design, all while performing versatile
functionalities without depending too much on the input data.
This aligns
perfectly with our ambition of reducing the amount of annotated supervised
data with the modiﬁed MAPO model, as MMN does not depend on the speciﬁc
inputs rather than the functional semantics of the question[CGL+19]. Training a
program generator to create functional programs from questions, enables MMN
to answer the question for a given image with the generated functional programs.
Following the works of [Joh20], applying the modiﬁed MAPO algorithm should
then improve the model performance when using less annotated supervised data
during training.

Chapter 2
Related Works
This chapter will dive into some of the works related to our project. First we will
describe some of the relevant visual question answering datasets, then explain
the Neural Networks models leading up to the works of [Joh20], followed by
a brief run through of the Meta Module Network and MAPO explaining why
they are relevant for our project. After this we will go through some of the
other GQA state of the art solutions, explaining them and why they don’t ﬁt
our project.
The ﬁnal two subsections will be a perspectivation to reading
comprehension, describing some of the datasets and how crowdworkers were
used to create them, along with two models that could ﬁt well if MAPO was
applied to reading comprehension.
2.1
VQA Datasets
2.1.1
CLEVR
CLEVR is a visual question answering dataset with images of 3D shapes (cubes,
spheres and cylinders) of various colors. The images are generated based on
randomly sampled scene graphs and each image contains 3-10 objects.
Ob-
jects can be of diﬀerent sizes and be of either metal (shiny) or rubber (matte)

Each question is based on a program, which is a sequence of functions (i.e.
count, relate, exists, etc.) that will answer the question. Figure 2.1 could have
questions such as: Does the metal sphere have the same color as the metal cylin-
der? (yes) and Are there more small cylinders than metal things? (no). The

also the models trained on this dataset. This means that models trained on this
dataset will not have a deep understand of real life images, as they have little
conceptual knowledge about most real life objects.
2.1.2
VQA and VQA 2.0
VQA is a dataset with natural language questions about images. It contains
204 721 real life images from the MS COCO dataset[LMB+14], and 50 000 ab-
stract scenes generated for this dataset, see ﬁgure 2.2 for examples. The abstract
scenes contain scenes with clipart “paperdolls”, which were added to attract in-
terest in high level reasoning because real life images often requires very complex
visual recognizers[AAL+15].
The questions and answers were created by crowdworkers[KCS08].
As sev-
eral diﬀerent answers can be correct for a questions, each question has mul-
tiple answers. The dataset includes a total of 614 163 questions and 7 984 119
answers[AAL+15].
Abstract Scenes
The abstract scenes are made up using an abstract library.
This library contains 20 diﬀerent human models with diﬀerent gender, race, etc.

VQA 2.0
A problem with the ﬁrst version of the VQA dataset was, that
some questions types tend to have similar answers when they are asked in the
dataset. If a question begins with “Do you see a . . . ”, the answer is yes 87%
of the time. This means that the models could answer some of the questions
with high accuracy, because they were able to learn the bias in the question. To
ﬁx this they introduced counter examples to each question, that has a diﬀerent
answer. So if a question Q on an image I has the answer yes, VQA 2.0 also
contains an entry where the same question Q on a diﬀerent image I′ has the
answer no. I′ was picked so it is similar to I[GKSS+17].
2.1.3
GQA
GQA is a dataset with questions over real world images. [HM19b] drew inspira-
tion from the CLEVR dataset when designing GQA. Each image is annotated
with a scene graph, which contains information about the objects in the image
and the relation between the objects. The scene graph can be considered to
be a formal representation of the image, see ﬁgure 2.3. The scene graphs are
taken from the Visual Genome dataset[KZG+17], which primarily has been con-
structed using crowdworkers from Amazon Mechanical Turk[KCS08]. The ques-

patterns outline the high level structure of the questions[HM19b]. An example of

2.2
Neural Module Networks
[ARDK16] introduced Neural Module Networks (NMN) as a method for visual
question answering. The model uses a semantic parser to analyze the question.
The result of this is used to generate a sequence of modules that can answer the
question. Each module is a neural network that has a speciﬁc function such as
find[red] (ﬁnd the color red in an image) or describe[color] (returns the
color in a speciﬁed part of the image).
[HAR+17] and [JHVDM+17b] both expand on the idea of NMN, by using a
trainable program generator and reinforcement learning for end-to-end training
of the model. They both use a program generator based on LSTM[SVL14].
[HAR+17] pre-trains the program generator using behavioral cloning from an
expert policy, and then reinforcement learning for joint training of the model.
[JHVDM+17b] pre-trains both the program generator and the execution engine
with supervised learning. The model then jointly trains both using reinforce-
ment learning without supervised programs, where the program generator sug-
gests programs for the execution engine. They also suggest a way to reduce the
amount of supervised programs need. First the program generator is trained
on a small amount of supervised samples, then the program generator gener-
ates programs for pre-training the execution engine, and ﬁnally they are jointly
trained using reinforcement learning.

2.3
Meta Module Network
The Meta Module Network (MMN) aims to combine the interpretability of
NMN and performance of traditional monolithic networks such as [WZL+19]
and [YYC+19].
Like the MAC network [HM18] MMN aims to have strong
compositionality, but instead of general reasoning it uses meta modules.
It
only has a single meta module which is executed recurrently. This means that
MMN has a monolithic structure, however as the meta module is instantiated
as explicit modules using predicted programs, MMN keeps the interpretability
of NMN’s [CGL+19].
Instead of the task speciﬁc modules from NMN, the abstract meta module in
[CGL+19] is instantiated based on a function recipe. A program generator gen-
erates the function recipes for the meta module. This is done in two steps using
a coarse-to-ﬁne approach[DL18]. First a coarse program is generated where the
type of each function is decided. In the second part the parameters for each
functions is then decided. This model has shown promising results for handling
unknown questions and adapting for more challenging environments[CGL+19].
This model ﬁts well with the structure of the modiﬁed MAPO model in [Joh20],
and it also seem to be able to provide good results using less annotated data, due
to its adaptabilities. For a more detailed description please refer to section 3.8.
2.4
MAPO
Memory Augmented Policy Optimization (MAPO) is an algorithm for policy
optimization. It is similar to the policy gradient algorithm, but it introduces a
memory buﬀer that stores promising paths, to prevent forgetting high reward
paths [LNB+18]. When a long sequence of correct actions is required to ﬁnd a
high reward, traditional policy gradient descent algorithms tend to have a hard
time remembering the high reward trajectories.
MAPO[LNB+18] introduces
systematic exploration that eﬃciently ﬁnds high reward trajectories and stores
them in a memory buﬀer. By using unbiased gradient estimates, it also gives a
lower variance in training. Furthermore, using weakly supervised data to obtain
close to state-of-the-art results could be advantageous, as annotation of data in
general is expensive (see section 2.6.1 for expenses on crowd workers).
[JHVDM+17b] uses reinforcement learning for joint training. [Joh20] takes that
approach, but replaces the REINFORCE[Wil92] algorithm with a variation of
MAPO. This allows the model to use a smaller amounts of supervised data,
and still get the same performance, which is a perfect match for our project.

2.5
GQA Solutions
This section focuses on some of the current state-of-the-art solutions other than
the Meta Module Network for the GQA dataset. It should be mentioned, that
only oﬃcial models with papers associated at the start of this project were
considered. Those are the models we are going to compare our results with.
2.5.1
State of the Art
Several diﬀerent approaches to visual question answering has yielded good re-
sults on the GQA dataset, as well as on other similar datasets like VQA 2.0 or
CLEVR. For Neural Module Networks see section 2.2. The following paragraphs
are some of the state-of-the-art approaches to the GQA dataset.
NSM
The Neural State Machine (NSM) is a model created by the original
developers of the GQA dataset. The idea is to create a state machine in which
reasoning is performed by traversing edges between nodes [HM19a]. Each node
in the graph corresponds to an object which have a set of properties, while
edges in the graph describe the relation between nodes. A node could be a bowl
object with the properties Color:
brown and Material:
wood. This node
could then have an edge to a node representing an apple object, with the relation
of the edge being inside, meaning the apple is inside the brown wooden bowl.
LXMERT
Learning Cross-Modality Encoder Representations from Trans-
formers (LXMERT) is a large scale transformer based model for discovering
cross-modalities between object relations and questions [TB19].
The overall
model is made up of three encoders; one for object relations, another for encod-
ing the question, and lastly one for encoding the cross-modalities between the
two other encoders. The model is thus trained to understand relations between
images and language. This model has proven to be good at solving VQA tasks.
However, it does not ﬁt our problem as it relies on pre-training using supervised
data, and it does not have a ﬁtting search space in which MAPO can perform
systematic exploration.
GRN
The Graph Reasoning Network (GRN) two graphs and joint embed-
dings from bilinear attention networks to create joint embeddings of images and
question. Having a combination of the two graphs creates more context in the
==PAGE:24==
10
Related Works
embeddings in terms of both semantics and factual information [GXT19]. Be-
cause the model relies on the creation of joint embeddings between objects and
words, it does not have an obvious application for MAPO.
MAC
The Memory, Attention and Composition (MAC) network is designed
by the GQA authors to create explicit reasoning and also act as a baseline model
to compare results with[HM18]. The idea is to have a recurrent neural network
of MAC cells, which perform attention based reasoning steps. By using self-
attention between the cells it is able to perform very complex reasoning without
necessarily using the full training data. The MAC model does not quite have the
performance of the other mentioned models, and furthermore it is end-to-end
diﬀerentiable, which encourages training of the full model as one unit.
2.6

Even though this project is working with visual question answering, the concept

will increase the probability of taking a successful step. Over time the neurons
in the brain will then have a strong connection between stepping forward and
walking, and a weak connection leaning backwards and walking[ACK+12]. The
neural networks work in the same way. They amplify connections signals be-

3.3
Seq2seq
A sequence to sequence network, or in short seq2seq, is a neural network that
takes one sequence as an input and outputs another sequence [SVL14]. Seq2seq
is often used in natural language processing (NLP) tasks such as translation,
summarizing, and question answering where the sequence length varies in input
and output. It can also be applied to a wide variety of tasks with diﬀerent
input and output formats such as text recognition and sound to text (speech
recognition)[CSW+18][MLGZ19]. For this project it will be used to transform
the preprocessed question data into programs.
Traditionally seq2seq models has consisted of recurrent neural networks (RNN)
such as LSTM’s or GRU’s [SVL14]. The way RNN work is that it takes an input
sequence X of dimension m and passes it through a hidden state H of dimension
n and outputs a new sequence Y of dimension o. Each step xt is passed into a
hidden state ht along with the previous hidden state ht−1. ht then outputs yt
based on xt and ht−1. Diﬀerent variations of this is depicted in ﬁgure 3.5.
3.3.1
Encoder-Decoder Structure
The traditional way seq2seq works is inspired by the encoder-decoder structure
in cryptography. The input sequence is encoded by the encoder into a higher
dimensional hidden state which is fed through to the decoder which decodes it

into the new sequence. The hidden state makes the output independent of the
input length and type, which means any input encoded to that hidden state can
be decoded into that type of output.
3.3.2
Padding
One of the challenges in computing sequences is that they often vary in length.
To solve this issue padding is introduced. The basic idea is to ﬁxed the length of
the sequence and add a unique padding token, this could ’0’ or another distinct
token, on the empty part of the sequence, i.e. if the ﬁxed length is 32 and the
sequence only consists of 26 tokens, 6 padding tokens would be added to match
the length. Then when training the seq2seq model, we specify that it should
not include the padding tokens in the training to avoid the bias that this could
add.
3.3.3
Attention
Attention in seq2seq is a way of emphasising in each step of the input sequence
what other parts of the sequence are important. The attention weights are fed
as input to the decode along with the hidden state. This means the decoder
can pay extra attention to important parts of the encoded sequence in each step
of decoding. This usually gives better performance as the decoder will have
more context from the original sequence.
The attention mechanism is often
described as a function mapping a query Q and a key-value pair into an output.
This is done by computing the weighted sum over the values, where the weight

Neural Module Networks
Neural Module Networks is a model for VQA. It takes as input a natural-
language question w and an image x and tries to predict the answer y[ARDK16].
A neural module network model consists of a collection of modules and a pro-
gram generator π (referred to as network layout predictor by [ARDK16]). The
program generator predicts programs from questions. Given an image x and a
question w. A program z = π(q) is generated. The execution engine φ executes
this program to predict an answer a = φ(x, z) to the question. [JHVDM+17b]

functions that altogether can answer the question. Each function in the pro-
gram then corresponds to a speciﬁc execution of a module. Each function can
be considered a reasoning step which together with the other functions answers
the question. In [ARDK16] the question “What color is his tie?" is parsed into
the functions, find[tie] and describe[color] and a dependency from the
image and find function to the describe function. A visual description of the

MAPO
The exploration buﬀer Be is then used in the MAPO algorithm, see algorithm 2
in appendix 7. The way MAPO is executed is by having a set of actors collect-
ing training data asynchronously, and then a learner that trains the parame-
ters based on the training data from all the actors. This architecture is called
an actor-learner architecture which utilizes distributed sampling[ESM+18]. As
it would add too much complexity to update the policy for all the actors at
each step, it is only synchronized periodically, and thus the actors will most of
the time use a "stale" policy πold
θ
that is behind the policy of the learner, see
[LNB+18]. When running MAPO each actor maintains a buﬀer Bi for the high
reward trajectories. The actor then sample from the buﬀer and on-policy sam-
ples according to the current πold
θ . Weights are then computed with the policy
as max(πθ(B), α) for samples in the buﬀer and 1 −max(πθ(B), α) for samples
outside. The samples from all the actors are pushed asynchronously to a queue
from which the learner reads samples and compute gradients as described in
equation 3.32 and updates the parameters.
3.7
MAPO and Neural Module Networks
The MAPO model presented by [LNB+18] uses some assumptions that are not
applicable for the Neural Module Networks.
It assumes that the set of all
possible trajectories A are countable, this can be countable inﬁnite, and that
evaluating the return of a trajectory always results in the same return. Since
the execution engine in the neural module networks is not static, that means
that and that evaluating the return of a trajectory will not always results in the
same return. This means that we cant eliminate programs just because it gives
an incorrect answer, as the execution engine could later give a correct answer
based on the same question. The size of the problem also means that systematic
exploring of all the possible trajectories is also not possible. Since the execution
engine is very complex model, it is also not feasible to test all generated program
structures.
To resolve these problems [Joh20] suggests the following approach. The actors
will suggest K program structures, kn of these are novel program structures and
K −kn are not novel, but assigned a high probability by the program generator.
This way the algorithm both explore new programs, but also revisit already
explored programs, if they have high probability.
The systematic exploration suggested by [LNB+18] generates programs in a

the path is already explored, then the actor simple goes one step back and
continue from there. Since it is not feasible to test all programs, the systematic

Consider that at some point t1 during the exploration the algorithm select m1

3.8
Meta Module Network
As datasets such as [HM19b] are becoming larger and more complex than pre-
vious datasets [AAL+15][JHvdM+17a], the compositionality of NMN’s start to
require more complex modules as a consequence of more complex semantics.
Traditional monolithic networks does still perform well on these datasets, how-
ever they do not have the same interpretability as NMN’s and does not generalize
as well do to the lack of compositionality.
The Meta Module Network (MMN) seeks to build on the compositional strengths
and interpretability of Neural Module Network (NMN) but furthermore combine
that with the performance and eﬃciency of more traditional monolithic networks
[CGL+19]. The goal is to create a model that can perform complex reasoning
on an abstract set of sub-tasks, without being dependent on the speciﬁc input.
In its essence, the Meta Module is a monolithic neural network which takes a
function recipe and instantiates an instance of a module responsible of solving a
sub-task, see 3.8. As each module is created, an execution graph containing the
relations of the modules is built concurrently to maintain compositionality and
allow visual reasoning, see ﬁgure 3.9 for example. This implies the model could
solve for instance a ﬁltering sub-task for a ball by training on a set of items
that does not include the ball object, i.e. the models ability to solve ﬁltering
sub-tasks does not depend on the object it is ﬁltering.
Figure 3.8: Upper left: Monolithic network. Upper right: Neural Module Net-
work. Bottom: Meta Module Network creates abstract modules
which take function recipes as inputs and combine those to solve
a set of sub-tasks. [source: [CGL+19]]

The meta module is originated in the concept of meta learning [FAL17], the

3.8.1
Program generation
It should be noted that this model was primarily created for the GQA [HM19b]

operations, arguments and dependencies. This means there are two straight-

and the other is to use a seq2seq transformer based approach, see section 3.3
and 3.8.1.2 for more details. We combine these two approaches by using a coarse-
to-ﬁne method like [DL18] to parse the semantics into the programs which are
then used for supervising when training the transformer based program gener-

3.8.1.1
Coarse to Fine
The idea is to ﬁrst generate the functions and in a second generation step to
apply the arguments for the function. By generating the functions ﬁrst it is
easier to apply the arguments as they have a ﬁxed structure of how they work,
i.e. a Relate function takes three inputs, the subject object in question (lamp),
the relation (above), and the relative object (bed), which describes a function
for ﬁnding the lamp above the bed.
Using the dependencies provided by the
dataset, it is also possible to generate an execution graph in which the function
dependencies are stored, see ﬁgure 3.9 for a visual representation.

3.8.2
Bottom-up Feature Extraction
Bottom-up feature extraction uses Faster R-CNN[RHGS15] to extract image
features.
Faster R-CNN is designed to identify objects belonging to certain
classes and predict bounding boxes for them. It detects objects in two stages.
First it predicts object proposals. Then at each spatial location, a network is
used to predict a class-agnostic objectness score. The object proposals with the
highest scores continue to stage two. In stage two region of interest (RoI) pooling
is used to extract a feature map, which is inputted into a CNN. The output is
a distribution over class labels, and class-speciﬁc bounding box reﬁnements for
each proposal[AHB+18].

Visual Encoder
The purpose of the visual encoder is to encode visual features extracted from
the input image and output the features into the correct modules. I.e. if a meta
module tries to select all the lamps in the picture, the visual encoder should
map all the extracted lamp features to that meta module. This is done using a
transformer based attention model and self-attention. Simply put self-attention
is an attention model where key = value, where the goal is to learn the de-
pendencies between tokens in a sequence, this could be words in a sentence or
features in an image. This is computationally very eﬃcient and also gives good
performance [VSP+17]. The self-attention is applied using multi-head attention
where key = value on both the extracted features and the question. These
encodings are then combined in a cross-attention network using multi-head at-
tention to output the visual features guided by the question. The structure of
the visual encoder is shown in ﬁgure 3.12.

In Jacobs setup the MAPO workers always have an up to date version of the

loading in the model before each new question.
The dataloaders for the MAPO workers maintain a list of which questions are
answered correctly and which ones are not. To transfer changes to this list, we
use a database. After each batch there are going to be some questions that can
now be answered and some that can no longer be answered. These are found by
the part of the code that runs on the GPU. The list of changes is then serialized
using JSON and pushed to a database . The dataloader for the MAPO workers
then continuously check if new updates have been pushed to the database, if
there are new updates, it retrieves them, updates the list of questions that can
be solved and remove the updates from the database.

4.2.2
Preprocessing
When preprocessing the data to create the inputs to the model, there was no
scene graph nor bounding box data in the original code base. To add this we
use the generated programs to identify the correct objects and corresponding
bounding boxes in the scene graph provided by the dataset. These bounding
boxes are then used for the teacher modules described in 3.8.5. If the gener-
ated programs do not return anything or have a wrong format then we just
return a bounding box with −1 on all coordinates. This functionality has been
implemented in the preprocess.py script. Program parsing is done using the
API_provider.py script. This script has also undergone some changes in the
function implementations to work properly with the data.

4.2.3
Feature Extraction
The original paper uses features extracted with bottom-up attention [AHB+18].
However due to technical limitations with using Caﬀe1 on the DTU Compute
servers, we were forced to use an alternative method of extracting the features.
We settled with the py-bottom-up-attention implemented in https://github.
com/airsplay/py-bottom-up-attention, which simply uses the same model
and weights is the original implementation made with Caﬀe. It is an easy few
line installation instead of the very cumbersome Caﬀe installation, and it is
implemented using PyTorch like our code is. It uses the Detectron22 object
detection framework. It is expected that there will a slight diﬀerence in the
extracted features however the author argues that this diﬀerence is insigniﬁcant.
These features are then given as input to the visual encoder. Some examples of
the feature extraction using Detectron2 are shown in appendix 7.

Question Encoder
The question encoder consists of an embedding layer for
the Glove word embeddings with 300 dimensions, a positional embedding layer
for the question positions from [VSP+17], a linear layer for the projection of the
question, and a sequence of self attention layers for encoding the question.
Visual Encoder
The visual encoder consists of a projection layer for the
visual features, a linear layer for projecting the coordinates of those features
and a sequence of self-guided attention layers to encode the visual features. It
also has an embedding layer for the positions of the bounding boxes.
Module
The module consists of the two attention layers, also described in
section 3.8.4, the self attention layer with the function recipes and execution
graph, and the cross attention layer for incorporating the visual features.

The setup made up of two components; the ﬁrst is the dataset which deﬁnes what

Dataset
All experiments for the meta module network has been performed
on the GQA dataset.
The GQA dataset in general requires more reasoning
than the VQA and CLEVR dataset, and as the aim of the model is to learn
complex reasoning using few samples, the GQA dataset ﬁts perfectly.
The

Chapter 6
Future Works
The setup that [Joh20] uses has three steps.
The ﬁrst step is to train the
program generator with a small amount of supervised data.
We have done
this with the program generator from MMN and seen that we can still achieve
very good performance with the program generator. This is a precondition for
the next step, which is to train the execution engine. This should be done by
using programs generated by the pre-trained program generator. Currently the
execution engine is trained on the programs that are generated using the coarse-
to-ﬁne approach, i.e. the supervised data. In the future this should be changed
to use the programs generated by the pre-trained program generator.
The third step is to integrate the MMN model into the MAPO setup. In order
to do this we have to edit the MAPO algorithm so everything works with the
GQA dataset. The systematic exploration algorithm also needs to be edited to
ﬁt the MMN program generator.
After putting MMN and MAPO together, we should run experiments to see
how much we can lower the amount of supervised data and still get good per-
formance. [Joh20] still gets good performance when using just 1k supervised
samples to train the program generator, even though the accuracy of the pre-
trained program generator is less than 40%. This does not necessarily imply
that 1k supervised samples are enough for our model, but since we get 59%
accuracy for our program generator with 1k samples, this is at least grounds
==PAGE:76==
62
Future Works
for optimism. It is also worth noting that the results of [Joh20] show that if
the program generator is too poor after pre-training, then it is hard to get good
accuracy for the execution engine.
One way of increasing the performance of the program generator would be to
ensemble it and reduce to variance of the model predictions. As we have experi-
enced high variance during training of the execution engine, it would also make
sense to ensemble this.
[Joh20] balances the dataset depending on the answer type by oversampling
observations with answers that does not occur often in the dataset. Doing this
may allow us to increase the performance of the execution engine after combining
MMN with MAPO.

Chapter 7
Conclusion
In this project we set out to create a visual question answering model, that re-
duces the need of annotated supervised data. This lead to the reconstruction of
the Meta Module Network, a state of the art VQA model capable of performing
complex reasoning over questions, even without prior experience with the ob-
jects and relations that could be associated with the question. With only part of
the original code base provided for this model, we were able to achieve 97.43%
performance of the original article. We got a validation accuracy of 58.85% on
the GQA dataset which outperforms the original benchmark models of MAC,
Neural Module Networks and MCAN. We have shown that the programs gen-
erated by the program generator of the model can achieve more than 92.2%
validation accuracy using as little as 0.06% of the annotated training data on
the GQA dataset, and more than 97.4% when using 0.6% of the annotated train-
ing data. As the GQA dataset has real life images and complex questions, which
require multiple reasoning steps, this shows promising prospects for applying the
modiﬁed MAPO of [Joh20], as the program generator acts as a bottleneck for
good performance on the execution engine. As the modiﬁed MAPO model was
created for an entirely diﬀerent setup, the code base for it has been refactored to
have a modularized design, which makes it easier to apply new model structures
to work with the modiﬁed MAPO model. Furthermore we discovered that the
modiﬁed MAPO model can achieve better performance and reduce variance in
pre-training by ﬁne-tuning the hyper parameters.
