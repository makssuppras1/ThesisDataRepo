Abstract
The aim of this thesis was to benchmark and measure the performance of three recon­
struction methods for the use of Spectral CT threat detection. The reconstruction meth­
ods in question were two Algebraic Reconstruction Technique (ART) and a Convolutional
Neural Network (NN). Here one ART and the NN were based on scans made with nine
projection, while the other ART were based on 74 projections. These three reconstruction
methods produced several sets of data, based on the scans calculated Linear Attenuation
Coefficient (LAC). The data sets also had paired material labels, which were used to train
and test a Random Forest Classifier (RFC). Using several differently trained RFCs the
three reconstruction methods were scored on accuracy, precision, recall and a F1 score.
Having scored the three reconstruction methods on six differently themed data set, which
was done in order to identify how well they work on specific materials, as well as a large
combined data set; it was found and concluded that out of the three ART9 performed the
best, being the most reliably accurate solution. The NN9 ranked as second best, closely
followed by ART74.

Introduction
Spectral Computed Tomography (or Spectral CT) is a technology which have seen use
in the medical field as a computerized x­ray imaging procedure, due to the allowance of
viewing a patient’s body in two dimensional slices from any angle. In combination with
imagine recognition algorithms, this could be used to identify various abnormalities etc. In
another field, this combination could be used to automate and possibly optimize various
processes.
One such opportunity is the x­ray scans of luggage in airports. The current wide spread
solution used world wide is taking a x­ray scan of a person’s luggage, and then having
a security officer look through the scan and possibly taking a closer look if needed. This
could be further optimized with the addition of automatic threat detection.
Granted, most airports already have some measure of threat detect, which flags various
items which could be classified as a threat. To fully implement an automatic threat detec­
tion solution would require a larger machine than what is already required in airports, but
would allow for faster luggage security screenings while needing fewer officers (though
these would still be needed if a threat is identified, which would need a manual check).
The way the automatic threat detection algorithm would work, is after the Spectral CT
scans of the luggage have been taken, the algorithm would work to identify any shape
and density which would require a human to further investigate. The identification of
threats using the density will be the main focus of this thesis.
Using a CT scanner to provide a three dimensional scan of luggage comes with many
challenges for a final workable solution. The ability to differentiate several items, the
variations of containers of liquids, and having a database with a reference value for almost
every possible item which could appear, in order to have the most precise identification
possible. For the thesis, this will be simplified to having several different types of items,
ranging from liquids to fruits, being scanned in next to each other outside of an bag. This
is done as to not complicate the development of the algorithm, where as the potential
scan of a realistic luggage would be for a later stage of development.
1.1
Problem Statement
The aim of this thesis is to examine and measure the performance of three different recon­
struction methods: two Algebraic Reconstruction Technique (ART) solutions and a neural
network (NN) solution. These reconstruction methods’ performance are to be bench­
marked, in order to determine which solution should be used going forward for a Spectral
CT threat detection pipeline.
The reconstruction methods have been provided before hand, where they already have
reconstructed several Spectral CT scans, divided into different sample data sets measur­
ing the Linear Attenuation Coefficient (LAC) for several objects and materials. The reason
for two ART methods, is measure the performance increase when using more projections
when scanning this data. The methods and their projections are: ART9, ART74 and NN9
(9 and 74 being the amount of projections). By providing a classifier to this data, the
performance, like accuracy, can be measured and compared.
In order to fulfill these goals, the needed theory and knowledge for the reconstruction
methods and Spectral CT scan data will be explained.

Theory

2.1
The Pipeline
[3]
This report will work with working prototypes of a Spectral Computed Tomography scan­
ner, which aim to be used in airport security by handling material analysis and threat
detection. These prototypes all follow the same pipeline approach to perform these anal­
yses, from scanning an object, to segmentation and reconstruction of the images, and
lastly the classification and threat detection. In order to properly discuss this pipeline,
each step of this process will be explained in order: starting with scanning, segmentation,
reconstruction, classification.
2.1.1
Spectral Computed Tomography
Computerized Tomography started as a revolutionary technology within radiology, allow­
ing for a digital representation of x­ray scans. These x­ray scans are performed by using
a x­ray source to send x­rays through an object, and measuring the x­ray on the other
side with an detector. The detector can then measure the attenuation of x­rays, which is
based on the density of the object being scanned. Using the attenuation is therefore key
to material analysis and detection, which is what the kind of CT scan which this thesis will
be revolving around.
In these kinds of material analysis CT scanners, it is important to achieve the highest
accuracy of scans in order to better pin point the attributes of the object in questions
(be it shape, density etc.). To this end, adding several additional detectors and energy
sources to acquire more precise scans resulted in the idea of Spectral CT. Spectral CT
covers many types of scans, dual­source or single source, with this thesis focusing on the
single­source with photon­counting detectors.
Using a single­source for Spectral CT scanner works much like a normal CT scanner
works, with a single x­ray source firing x­rays at an object and measuring the attenuation
on the other side. Where it differs though, is when measuring the attenuation, having
photon­counting detectors allows for the distribution of photons into discrete channels
according to their energies. This results in the attenuation coefficient being measured as
multi­channel energy profiles, allowing for a more in depth and specific scan of an object’s
chemical structure. This specificity is very useful when it comes to using this technology
as threat detection, as it does not leave a lot of room for error.
Energy Channels
The Spectral CT scanner this thesis will be dealing with uses 32 energy channels. The
energy channels are proton detectors set to a specific energy interval. The specific in­
tervals in use for each of the 32 energy channels is unknown, since not knowing these
values does affect this project. Having a higher number of energy channels leads to more
precise scans of an objects chemical structure, but it also produces a lot more data per
scan. Handling all of this data will require more computational power, needing more com­
puting power per additional energy channel. For Spectral CT scanner being used in an

Projections
Another facet of CT scanning is the use of projections. These refer to the amount of
angles a CT scanner uses for making the scans. Having a CT scanner make a scan for
all 360 degrees of an object is useful, but also requires a lot more computing power for
the image reconstruction (more on this in 2.1.2). For this thesis, the use of two different
amounts of projections will be examined.
As the Spectral CT scanner being used for threat detection, needs to be both precise and
fast, the amount of projection used makes a difference. There are two larger categories
of projection amounts: Dense view, which feature many projections, and sparse view,
which feature a small amount of projections. This thesis will be dealing with both of these
types, in order to benchmark and analyse the differences in performance. The dense view
solution will be using 74 projections, and for the sparse view solution 9 projections.
2.1.2
Image Reconstruction
Once the scans have been made, in order to combine these into a 3D model of a scanned
object, the image have to be reconstructed. There are several methods used to recon­
struct Spectral CT scans, the main two focused in this report is the Algebraic Recon­
struction Technique, total variation (ART.TV, will be referred to as ART), and the use of a
Convolutional Neural Network (CNN, will also be referred to as NN for simplicity). Both of
these methods will be examined closely in the following sections.
Algebraic Reconstruction Technique
[5]
There are several ART methods for image reconstruction, all of which are iterative in na­
ture. By projecting density across the reconstructed space already created, it can modify
itself until fit with all measured projections. This is easier for a uniform object, and gets
harder to get correct with more complex shapes.
The way it projects measured density is through using rays. These rays refer to a square
region between two parallel lines. Using this ray, it can better modify itself to a shape
by focusing on smaller regions. It then iterative goes through each measured density,
changing itself to match every pixel with its measured value. This is how both the ART
methods work.
Convolutional Neural Networks
Convolutional Neural Network (CNN) is a type of neural network, which is used to process
data, especially images. This is due to it working with grid based data representations,
which images are perfect for because of its pixels structure. The NN9 being used in this
project have three different kinds of layers it uses to process data.
Convolution layers. This function takes the dot product of a matrix of learning parameters
and a matrix of a square cluster of the grid. By doing this, the CNN can change the grid
based on the learning parameters.
Pooling layers. These further combine various values using a specified operation. For
the NN9 the operation is a max pool, which combines and only keeps the max value in a
cluster in the grid. Pooling layers shrink the image.
Up sampling layers. These can be utilized to do the opposite of pooling, by expanding
the grid by spreading the values of the grid out.

Using this same principle, by taking the Spectral CT scans through two layers of convo­
lution, then a max pooling layer and save the value for later. Repeat twice, before two
addition convolution layers. Per up sampling layer, use three convolution layers as to
better improve image quality. Before the second convolution layer, use the saved data
from earlier since this will keep the image from being too dilated. By the third up sample
cycle a clean reconstructed picture have been achieved. This is how the NN9 works.
2.1.3
Imagine Recognition and Machine Learning
For the classifier, the Random Forest Classifier is used. This creates a decision tree
based on its training, which it uses to to identify values it is presented with.
For the scoring the classifier several metrics will be used all measured as a value between
0 and 1, where 1 mean 100 % correct. Accuracy, the amount of correctly label voxels.
Precision, is the ability of the classifier to not label a positive label as a negative. Recall,
is the ability to find all positive labels.

3.1
Python
The pipeline used to handle the Spectral CT scans’ development is programmed in Python.
Python is widely known as one of the best programming languages for data science and
machine learning, which is why it was an obvious choice for a project like this. It should
be mentioned, though, that this choice was made before this thesis started.
Python is used for these kinds of projects due to its programming framework and wide
selection of useful libraries. The programming framework of Python provides a simple and
understandable way to write code, since itself can identify and associate data types. This
results in Python needing the least amount of code written compared to other languages.
Using the robust framework of Python, several different Integrated Development Environ­
ments (IDE) are available to use. For this thesis, the provided code was imported as a
project in PyCharm and a Python 3.8 interpreter was setup. The interpreter was created
to import all the necessary libraries through the Anaconda distribution. The Anaconda
distribution contains many of the packages used in the code, the most important of these
will be discussed in 3.1.2.
3.1.1
From Linux to Windows
The pipeline was not only developed using Python, but on Linux. This thesis would be
executed using Windows though, which provide the first challenge of the project. Luckily,
Python is widely available on most platforms, meaning the code itself would not be a
problem, as well as the export­import process for the project folders. The only problem
was in the choice of packages.
This being a machine learning project, meant that Python would have to gain access to
the hardware of the computers in use. The packages which makes this possible are highly
Operating System (OS) specific, since granting access to the hardware as well as utilizing
it needs to use commands unique to the OS in question. This meant that for the switch to
Windows to proceed, some packages needed to have a specific Windows version. This
was made a even bigger deal, since some of these packages only had an experimental
Windows branch for specific versions of Python. At the start of the project, Python 3.9 was
used, then downgraded to Python 3.7, before finally landing on a selection of packages
compatible with Python 3.8. The packages which needed Windows specific branches
was gpustat, while TensorFlow, Keras and CUDA needed the Windows version installed
as well.
The system the benchmarking was performed is comprised of the following:
NVIDIA GeForce RTX 3060 GPU, which have a Core Clock of 1320 MHz, with it being pos­
sible to overclock it to 1837 MHz. It features 3584 CUDA cores and have 12GB GDDR6
15GHz memory clock. Intel Core i9­9990K CPU at 3.60GHz, and 16 GB Memory.

The segmented data are what these reconstruction are based upon, specifically the Linear
Attenuation Coefficient (LAC) for each voxel (3D pixel) of the scan. Accompanying these
LAC values are assigned labels to each of these voxels, telling what material each voxel
represents. The LAC values are all different, and based on the reconstruction solutions
calculations per voxel based on the number of projections, meaning that ART74 will have
little over 8 times as much information per voxel compared to the others.
A Random Forest Classifier will be trained and tested for each reconstruction method’s
LAC data, using the labels information as the ground truth. This process is seen in Clas­
sification.py (used for training and testing the classifier using only a specific data set) and
ClassificationAll.py (used for training and testing the classifier on the combined data of all
data sets). Scoring these classifiers will be the basis for the benchmarking comparison
and analysis. All code can be found at A.1.
3.2.1
Classification.py
Starting classification.py does not need any input to start running, using an IDE’s build in
Run function or launching it through a command­prompt equivalent program is enough
(though stating which Python interpreter is in use is needed). Changing this to require
input is a possibility in order to make it more functionally effective.
The first section of code states which data set is in use, based on the location of the
data folder in question. The following section is the corresponding set of labels to data
set in use, and should be paired with the correct data set. This is an ineffective way of
differentiating data sets and corresponding labels, but it works.
The next 100 lines of code handle the importing and setup of the LAC files and labels.
LAC values and labels are all imported for the three reconstruction methods at once, which
again is a bit ineffective since this can be a slow process for less powerful machines. From
this point forward, variables will be named such that 1 refers to ART9 related values, 2
refers to ART74 and 3 refers to NN9.
Next, three sklearn classification models are setup. This is done by pairing LAC values
and labels for each of the reconstruction methods, followed by training three different
Random Forest Classifiers using the paired data. For each of these classification mod­
els, scoring is performed. Scores are calculated in regards to each model’s accuracy,
precision, recall and ROC values. These will all be used to compare each model and
benchmarking these models.
Finally, two confusion matrices are created (a normalized version and a non­normalized
version) for each reconstruction method, and these are plotted, shown and exported. This
was used to get the information seen in 4.1.
3.2.2
ClassificationAll.py
This functions much like the Classification.py program. The difference is mainly in the
importation and handling of all the data sets. These changes mainly pertain to the con­
catenation of several sets of values in order to get the combined data set.
Every data set and label information is imported, and handled for each reconstruction
method. The LAC values and labels are paired and used to train Random Forest Clas­
sifiers, and these are evaluated. Since this handles the combined data sets the Jaccard
Coefficient is calculated in order to determine the overlap of labels seen in several data
sets. This was used to get the information seen in 4.1.7.

3.3.7
Additional Notes
There are a few thing which should be noted in addition to the individual values. Firstly,
as seen in the full LAC plots at A.2, it appears as if there is a lot of noise in the all of the
data sets except for A.2.10. This is to be expected, but it will result in classifier not being
able to get a perfect 100 % accuracy (the accuracy was not expected to be that high, but
it should be noted nonetheless).
Additionally, there seem to be some spelling errors and inconsistency in the labelling in
some of the data sets. There are two different version of ”olive_oil” and ”Olive_Oil”, the
capitalisation will make a difference when combining the data sets, since it will see it
as two different labels. ”Toothpaste” also exists as ”Toothaste”. Unfortunately, changing
these labels to fit each other have not been possible, due to the code rejecting chances.

Results and Discussion
Having gone through how the benchmarking is performed as well as having discussed
the difference of the data sets, the benchmarking will be performed. The results of the
benchmarking will below be presented followed by a discussion of these.
4.1
Results
Much like the presentation of the data sets in the methodology section, each data set will
be presented on their own, in order to better highlight the differences in performance and
accuracy. The different solutions were trained on the data set individually, using 80 % of
the data set for training, while testing on the remaining 20 %. This size was chosen as it
sufficiently trains the solutions, which also leaves a large enough test set to get precise
metrics for the benchmarking. Only the normalized confusion matrices will be shown in
the following section, the non­normalized version can all be found in the appendix at A.3.

Looking at the accuracy of the three reconstruction methods, its clear to ART9 outper­

showcased with the different reconstructions, 3.1, where there is an image quality in­
crease from ART9 to ART74. Its hard to say what causes this, since this is going to be
a trend for all data sets, its quite unlikely to be caused by mislabelling and switching the
ART9 and ART74 data for every data set. What also disproves this theory, is that the
resulting reconstructed images does have the expected increase in quality.
Based on the accuracy of received, the performance of ART9 can be deemed as Good,
with ART74 and NN9 being deemed Acceptable.
Comparing the precision, recall and F1 scores, ART9 performs very well when it comes
not labeling positive occurrences as negative at 0.88. Again, ART74 and NN9 are just
acceptable, with NN9 being on the border of being deemed bad.
Since these values are nearly the same for each of the methods, the accuracy is not
being skewered by a problem, but simply the classifier having a hard time differentiating
between the different labels. For the ART solutions it seems to be acetone, nitric acid and
whiskey which is hard to identify. Looking at the mean values, its hard to identify why this
is the case, since the mean values is supposedly far from each other. But as seen in the
full LAC data plots at A.2, most are very noisy, which seem to be the only explanation for
the inaccuracies with fluids.
