Summary
When using co-packaged optics it is important to have a stable optical source. This
source is usually a laser, but unfortunately a laser’s performance is very temperature
dependant. Therefore it is important to have a stable temperature-control system for
the laser, and an accurate understanding of the thermal behavior of the laser.
In this report the thermal behavior of a laser has been determined, and a control-
system is designed and implemented for a prototype of an ”optical power supply” to be
used in co-packaged optical systems, made by Nvidia’s interconnect design department
in Roskilde. To this end a model which can accurately predict the behavior of the setup
has also been derived and simulated with SPICE software.
Although the derived model has some limitations, it is able to predict the behavior
of the setup. The control system keeps the temperature constant, in spite of technical
challenges with the equipment.

CHAPTER 1
Introduction
Though they are primarily known for their work on graphic cards and artificial intel-
ligence, the multinational technology company Nvidia have recently expanded into de-
veloping optical systems. This includes working with co-packaged optics, a subject of
increasing interesting in optical communications.
The basic principle is to create a
system which relies as little as possible on high speed electronics. This is done by incor-
porating the transceiver modules in the system, and doing so has many advantages; it
saves power, it can expand the throughput, and it reduces the need for equalisation and
re-timing. In order for a system such as this to be reliable, it needs to be possible to
replace components, which becomes diﬀicult if everything is incorporated in the same
system, i.e. co-packaged. For this reason, Nvidia is working on developing an ”optical
power supply”, an easily replaceable device with 16 high power lasers. Each laser has
dedicated cooling, which increases their lifetime.
In this dissertation, a high power laser will be modelled, and the circuits, which
are used to control and monitor the laser, will be designed and described. A model
for the thermal behaviour of the system will be derived, based on thermal/electrical
equivalences, allowing a SPICE model to simulate the thermal behaviour. A PID loop
will be tuned to control the temperature, and the optical behaviour will be determined
experimentally.
The derived model and the process of deriving it will bring further clarity and addi-
tional insight into the thermal behaviour of this and any future setups used in Nvidia’s
optical communication project(s).
1.1
Problem Statement
The purpose of the project is to investigate and implement systems for stabilising, mon-
itoring and controlling high-power laser modules for use in co-packaged optics solutions.
The project will involve:
• Establishing a model representing the thermal, electrical and optical characteristics
of the laser.
• Identifying the relevant operational window regarding temperature, electrical and
optical properties.
==PAGE:9==

• Developing a control algorithm enabling stable operation of the laser within the
identified operating window.
• Implementing the system in microcontroller technology for controlling a prototype
laser.

CHAPTER 2
The Setup
To investigate the behaviour of the laser a setup is constructed. In this chapter, the
setup will be thoroughly described in three parts: electrical, optical and thermal.
2.1
Electrical Setup
The electrical setup consist of the following sub-circuits: a thermoelectric cooler (TEC);
a thermistor; a TEC Driver with an external PID loop; a laser; a laser driver; a micro-
controller; an optical power meter; and a computer for data collection. This can all be
seen in Figure 2.1.
Figure 2.1: A simplified model of the electrical setup showing how the different parts of
the setup are connected to each other. The boxes represents circuits or microprocessors
too advanced to be represented with capacitors and resistors. Additionally, this overview
does not show how the various parts are connected to the various power supplies that
are used in the setup.
The microcontroller is connected to the laser driver with I2C, which it uses to set
the laser’s diode current. The microcontroller is also connected to the TEC driver, from

which it reads data from, such as the temperature from the thermistor or the TEC

Furthermore, the microcontroller reads the optical power output by measuring an
amplified voltage from an optical power meter. After having read the optical power,
the microcontroller sends all of this data through a serial connection to a PC, where
the data is processed. The TEC driver is mounted on an evaluation board, which has
various peripheral circuits setup in advance. Through these circuits, it is connected to
the thermistor, the TEC, and to an external PID loop, which regulates the current given
to the TEC, based on the temperature measured by the thermistor.
2.2
Optical Setup
The optical setup consists of a laser, an attenuator, and an optical power meter, all of
which is connected through a short fiber. The setup can be seen in Figure 2.2. The laser
used in this project is of lower power (∼30 mW) than the one which will be used in the
final project (200 mW).
Figure 2.2: The laser, the fiber, the attenuator and the power meter.
In this setup, the output of the laser goes through a short fiber, and then through a
20dB attenuator before it reaches the optical power meter. The purpose of the attenuator
is to ensure that the level is low enough for the power meter. Furthermore, it should
be noted that the fiber goes from a LC connector to a PC connector, which leads to
more loss than the 20dB. The exact loss is never determined, as the loss in the fiber
conversion will change just by touching the converter [11] making it almost impossible
to accurately measure the loss.
2.3
Thermal Setup
The thermal setup has four main components: a laser, a heat sink, a heat spreader and
a TEC. The four components are stacked on top of each other as seen in Figure 2.3. In
addition to the four main components, a thermistor is also shown in the figure. The
thermistor and the TEC both act as couplings between the thermal and electrical side,

with the thermistor converting temperature to voltage, and the TEC converting voltage
to heat.
Figure 2.3: The four main components of the thermal setup and the thermistor.
The power lost in the laser will generate heat, which is distributed evenly across
the TEC through the heat spreader.
The TEC is controlled by the electrical setup,
based on temperature measured by the thermistor. It is used to transfer heat from the
heat spreader to the heat sink in order to cool the laser. The heat sink speeds up the
rate at which the heat is dissipated into the air, improving the eﬀiciency of the process.
Additionally, thermal paste and pads are added between each component in order to
improve thermal conductivity.

3.3
Laser Driver
The laser driver was controlled from a microcontroller over I2C. It has four output current
channels, with a maximum output current of 150 mA each. To increase the max output
current, two channels were connected in parallel to the diode. To protect the DACs, the
two channels are always set to the same current. These channels have an ADC built in
to measure the voltage of each output, which is used to measure the diode voltage, and
afterwards, the measurements are sent back to a microcontroller.

3.5
Microcontroller
The microcontroller mostly serves three purposes: measuring data, controlling the laser
driver and passing data back to a PC.
The microcontroller is programmed to have a main loop and an interrupt driven by
a callback timer. The main loop transmits data via serial communication to the PC
and handles any calculations for current updates if relevant. When a new current is
==PAGE:17==
3.6 TEC
10
calculated, the main loop raises a flag. The interrupt handles I2C communication with
the laser driver, and it updates the diode voltage measurements. If a flag is raised, the
interrupt requests the updated current from the laser driver. It was found that the
I2C communication and the serial communication should be carefully timed, as they
would otherwise occasionally overlap, causing the I2C to fail and needing a reboot to be
reestablished. In the end, it was chosen to have the main loop be somewhat slow, going
off once roughly every ∼0.12s, while the interrupt timer was set up to go off once every
41.17 ms, and only do one I2C operation for each interrupt. The main loop’s update
rate is effectively the sampling frequency of the system, making the sampling frequency
fs = 8.333 Hz. This means that it will not be possible to sample anything above half of
this frequency, according to Nyquist’s sampling theorem. This is not a problem however,
as most of the thermal behaviour is much slower than this, as will be shown in later
sections. The methods for reading data can be seen in Table 3.3.

4.1
Sources of Errors
The setup suffered from an error which seems to be related to the setup’s ground connec-
tion. The error was never completely solved. In this section the behaviour of the error
will be examined. There are two distinct variations of this error, which will be referred
to as the Mode Error and the Switching Error.
4.1.1
The Mode Error
The Mode Error occurs when the TEC driver changes mode between cooling and heating.
This manifests as large offsets in the voltage, which, in the worst cases, can lead to a sort
of oscillation, where the temperature never stabilises. Generally, it seems that there is
some voltage threshold, where – if the thermistor voltage goes below it – it changes into
heating mode and causes the offset. Examples of this behaviour can be seen in Figure
4.1, where a best case and a worst case of the Mode Error can be seen.
The best way to reduce the Mode Error is to alter the PID control; in general, a
lower gain reduces the Mode Error. The source of the Mode Error was never determined.
The primary theory is that the error is related to the electrical ground of the system. In
order to investigate the problem, all of the probes of an oscilloscope were connected to
the circuit, and it was observed that this almost completely removed the Mode Error.
Afterwards, an attempt was made to recreate this effect by connecting all of the ground
pins of the electronic system to no avail. This implies that the effect of adding more
probes was to connect more of the circuit to the electrical ground of the electrical grid
through the oscilloscope.
It was never determined what the exact cause of this problem was and ultimately,
it was decided that as long as the Mode Error was minimised by improving the PID
control and by applying the oscilloscope probes, its effects were acceptable.

4.1.2
The Switching Error
The Switching Error occurs when the input current changes quickly. Like the Mode
Error, it leads to a leap in the voltages, which causes the TEC to receive a current that
is either too high or too low; however, this offset is often temporary, and the temperature
is brought back in a few seconds. An example of this can be seen in Figure 4.2.
As both the laser driver and the TEC driver are powered through two different
channels of the same power supply, the most likely cause of this error is the power
supply. This seems odd, as 300 mA is not a huge change in output current, but it is
clear that the error is somehow triggered by the change in current. As the temperature
settles down again fairly quickly, it was decided to accept this error, as it has a fairly
low impact on the temperature.

done as described in the guide [4].
5.1
Tuning P
First, the circuit it is reduced to look as is shown in Figure 5.1. In this configuration,
the circuit essentially acts as an amplifier with a gain of RP/RI, and as per the guide,
RP is set to be 100 kΩ, while RI is changed. According to the guide, the gain should be
increased until oscillations occur, at which point the gain should be halved to find the
value of the resistors. This approach was not possible, as the output started showing
Mode Errors before the oscillations could be reached. Instead, it was chosen to find a the
gain which caused Mode Errors to occur and then find a resistor more than twice that
value to ensure that the response is fast, while still keeping it far above the threshold for
Mode Errors. The process can be seen in Figure 5.2. Here, a resistance of RI = 47 kΩ
is chosen, leading to a gain of 2.13.

CHAPTER 6
Optical Characterisation
In this chapter, it will be described how to model the laser’s optical behaviour, based
on measurements.
6.1
Optical Power
It is beyond the scope of this project to accurately model the optical power output as
a function of diode temperature and input current. Instead, a more simple method will
be used: the output power of the laser diode will be measured at different temperatures
and with different input currents. From this, a lookup table can be created:
Poptical = LUTo(ID, TD)
(6.1)
Here, ID is the diode current and TD is the diode temperature. In order to model the
laser, measurements of the optical output power will have to be made.

ments for each laser: the diode current, the diode voltage, and the optical output power.

and decreased. To investigate the laser’s behaviour even further, the sawtooth wave is
also inverted, so the current is decreased gradually instead of increased. Examples of
the input current waves can be seen in Figure 6.5.
In theory, this should produce similar current power curves as seen in Figure 6.1, 6.2
and 6.3. The results of these tests can be seen in Figures 6.6 and 6.7. It should be noted
that the power measured is overall lower than the power measured by Lumentum; this is
the result of there being some unknown attenuation somewhere in the fiber connections.
This loss is assumed to constant, therefore its impact can be ignored, as the shape of the
curve will not be effected.
Looking at the plots, it is quite clear that the rate had little
impact on the optical output power when increasing the current, and that the output
power in general stays almost constant, decreasing slightly at the highest temperature.
However, looking at the data when ramping down reveals different behaviour: firstly,
there seems to be a small upwards bend at the highest current, and secondly, it is
clear that the power is lowest when the temperature is low in all cases. This behaviour
matches the data from Lumentum, as can be seen in Figure 6.8, where the output is
higher for higher temperatures.

7.1
Thermal/Electrical Equivalence
There are three ways that heat can be transferred: by conduction, convection and ra-
diation. Conduction is the transfer of heat through solid objects in contact with each
other. Convection is when the heat transfer is the result of fluids or gasses flowing and
moving the heat of a system to other parts. Radiation is the transfer of energy through
electromagnetic radiation.
In this case, most of the heat transferred from the laser diode will be transferred
through conduction. As described in [6] and [10], thermal conduction is equivalent to
electrical conduction in a few key manners. The specific correspondences can be seen in
Table 7.1. These correspondences make it possible to solve heat problems with a lumped
element approach.

predicts the behaviour of the circuit, in that it reaches the correct temperature in the
correct amount of time.

CHAPTER 9
Discussion
9.1
Measurement Errors
The Mode and Switching Errors were significant problems encountered in this project,
and they were never solved. A few different approaches were taken to mitigate them,
as the Mode Errors could be reduced by tuning the PID network to operate above the
threshold for this type of error, and the Switching Errors could often be disregarded.
Still, before any further work is done, it should be investigated if these errors could
be avoided; e.g. if another power supply could be acquired, it is very likely that the
Switching Errors could be resolved. A solution for the Mode Errors is less clear. The fact
that the problem lessened when probing the circuit with an oscilloscope indicated that
a better ground connection resolved the error. However all other attempts at improving
the ground connection did not show similar improvements. Any further work with the
TEC driver should begin with a thorough investigation of this behaviour.
9.2
Power Losses
Every single component was in some way connected to the surrounding air. This meant
that Newton’s Law of Cooling applied and that parasitic resistances connected many
of the circuit nodes down to the ambient temperature. It was originally assumed that
the impact from this phenomena could be ignored, as the resistances were assumed to
be quite large – however, this was not the case. The best solution to this issue would
have been to add some form of isolating material around the setup, so that the path of
least resistance was through the heat sink. Another way to resolve this, without making
new measurements, would be to extend the model to have more resistances in order to
account for the parasitic losses. The second option was used to determine the size of the
convection resistance.
9.3
PID Measurements
From Figures 8.1 and 8.3, it is evident that the PID network will keep the diode temper-
ature stable. When the diode current is rapidly changed, the temperature is kept within
roughly 0.5 ◦C, and when an input current step is applied, the temperature has a brief
overshoot before the temperature stabilises after 20s. It is possible that a even faster
==PAGE:64==
9.4 Comparisons to the Simulation Measurements
57
temperature control could be implemented if it was not for the Mode and Switching
Errors that limited the values of the PID network. However, as it stands, the PID is
concluded to work as intended, keeping the temperature stable under different operating
conditions.
9.4
Comparisons to the Simulation
Measurements
The comparisons of the steps, as seen in Figure 8.4, may at first seem to indicate that the
simulation is far off; it has roughly the same shape, as it is a step with an overshoot an no
ripples, but the rise time is faster, the settling time is slower, and the overshoot is smaller.
This is due to the fact that the step was measured with the 13EE004 laser and not the
13EE013 laser as modelled in the simulation. As the step was recorded at much earlier
point in time, and the laser was later changed, the parasitic resistances must have been
altered between the two measurements. Unfortunately, no step response was measured
with the 13EE013 laser, so the same comparison could not be made. Instead another
comparison can be seen in Figure 8.5. Here, the measured temperature is compared to
the simulated temperature when ramping the input current down. The measurement was
taken with the 13EE013 laser, resulting in much better correlation between the behaviour
of the measurement and the simulation. The main difference is that the simulation does
not produce the Switching Errors that the measured temperatures contain. In addition
to the lack of errors, the simulation correctly predicts the behaviour of the setup.
Besides showing the accuracy of the simulation, the results discussed above also
highlight an important limitation. In order for the model to be accurate, it is necessary
to update the laser capacitance, and possibly all three convection resistances.
It is
likely, however, that if measurements with more stable convection resistances could be
constructed – for instance by putting the setup inside a box – the model would be much
easier to fit to other lasers.
9.5
Predictions made with the Simulation
Model
Figure 8.6 shows the behaviour of the simulation under operating conditions outside
the scope of what was measured; specifically how the setup reacts to higher thermal
powers. This is relevant as the final setup will have four lasers per TEC, and as these
lasers are higher power, it is important to investigate how the control loop responds.
Unfortunately, there is a maximum limit to the thermal power that the simulation can
handle. Once it reaches 1.4W, it begins to display some very unstable behaviour. For
this reason, the power was increased in increments of 100mW until 1.4W was reached,
==PAGE:65==
9.5 Predictions made with the Simulation Model
58
and the step response when changing VTEMPSET was investigated. The figure shows
how the overshoot decreases as the power increases and that the rise time becomes slower
as the power increases. This is essentially the result of there being significantly more
heat to transfer, so it is done slower, which results in a lower overshoot and a longer rise
time.
Based on the steps, it is evident that the setup will be able to regulate up to 1.4W
of thermal power, and it is expected that the setup is able to handle even more heat,
even though the simulation is not.
==PAGE:66==
CHAPTER 10
Conclusion
The process of deriving an equivalent model for the electrical, optical and thermal compo-
nents have been described. The electrical model was constructed based on the circuits
provided in the datasheets for the various components.
The optical model was con-
structed based on measurements of the different lasers used in the project. Finally, the
thermal model was mostly derived by constructing a thermal/electrical equivalent model
and estimating the values of the components, either by calculating them or by finding
them experimentally.
The model has been compared to the measurements and has been found to correctly
describe the behaviour of the laser. However, it was also found that in order to achieve
this accuracy, the model needs to be updated to match the specific laser exactly. It
has been considered how the model could be improved, i.e. by decreasing the impact of
the parasitic resistances induced by convection cooling of the various components of the
setup.
A PID network used for controlling the temperature of the setup has also been derived.
Using the control circuit, the setup has been shown to generate a stable temperature
under different operating conditions. The PID was limited by the Mode Error, which
was an important factor when determining the sizes of the components in the control
circuit. Reducing this error would allow for more freedom in the PID tuning process,
and possibly allow for faster temperature control.

Calculations for PID Transfer

This appendix details the derivation of the decomposed transfer function described in
Section 3.7. The exact calculations performed are somewhat tedious, but for complete-

As mentioned in Section 3.7, the transfer function is:
Z2
Z1
= Z =
(CIRPs + 1)(1 + CD(RD + RI)s)
s(CFCIRPs + CF + CI)RI(CDRDs + 1)
(A.1)
Which can be expanded to become:
Z =
CDCIRP(RP + RI)s2 + (CD(RD + RI) + RPCI)s + 1
