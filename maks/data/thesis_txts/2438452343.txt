Data Augmentation for Volumetric Images
Asger Ougaard, s134556
Abstract
In this thesis we look at a new approach to a learned data augmenta-
tion scheme for volumetric images, discussing the beneﬁts and disad-
vantages of this approach compared to a standard data augmentation
scheme. In discussing the required properties of the more advanced
transformations used for this data augmentation scheme, we present
a way to use Continuous Piecewise-Afﬁne Based (CPAB) transforma-
tions. We investigate how CPAB transformations work, and discuss
how different parameter settings affect the performance of the data
augmentation. The data augmentation scheme is used to augment a
dataset containing ten CT scans of the upper bodies of ten subjects.
Results of the data augmentation is presented, and the quality of the
augmented data is discussed. Furthermore a convolutional neural
network is trained on 2D slices of the augmented as well as the un-
augmented data, learning to segment the spine in these. The perfor-
mance of the networks trained on the two data sets are compared,
and shows that the data augmentation scheme has indeed improved
performance. Lastly, ideas to improve performance, as well as reﬂec-
tions on choices made during the process of the thesis are presented,
indicating that this data augmentation scheme warrants further in-
vestigation.

1
Introduction
The performance of machine learning models rely on data, and especially the amount of
available data. In many cases this is not an issue, companies like Netﬂix and Facebook
have enormous amounts of data on their users, and are therefore able to target their ser-
vices to the individual user. Also, in many image classiﬁcation problem data is readily
available, no more than a quick search on the internet away. In other cases, however,
data is much more scarce, which makes difﬁcult to obtain good results with machine
learning. Often this is the case for medical data, where each data point corresponds to
some data about a patient. In that case the size of the dataset is limited by the number of
patients. In other cases, the dataset is used for a segmentation problem, which requires
an annotated image for each image in the dataset. Creating these annotated images take
a lot of time and, in the case of medical data, need to be done by a doctor or some other
trained professional, who does not have a lot of spare time to make them. This holds
especially true for volumetric images, where the number of voxels in an image very
quickly become massive.
To solve the issue of data availability, data augmentation is used to generate new data
points from the existing elements of the dataset. A typical image data augmentation
scheme involves rotating, shearing, ﬂipping and shifting the original images, and in that
way obtain new slightly different images. These transformations seem rather arbitrary
when considering that a data augmentation scheme seeks to model the variations in the
true data, and that this method is used on a plethora of different datasets. The reason
for using this scheme is it’s simplicity, and that it is simple for the user to implement
the transformations in such a way that the transformations are class invariant. Consider
MNIST, a dataset consisting of 60.000 hand written digits from 0 to 9. Rotating images
of the number 6 more than 90 degrees will make the image closer to a 9 than a 6.
However, if the rotations are kept within a range of 25 degrees from the original image,
the new images will fall within a realistically slanted version of a 6. The simplicity
of this data augmentation scheme comes at a cost. Depending on the dataset, it is not
necessarily possible to capture the variation in the dataset by simply rotating images.
Considering MNIST again, the number 4 is typically written in two distinct ways; with
an open or a closed top. We would expect a dataset to reﬂect this, by containing images
ranging from a completely open top, somewhat open top, and all the way to a closed
top. Another issue with this data augmentation scheme is that it is up to the individual
user to specify which rotations, ﬂips etc. is allowed, and hope that these are the best
choices. This also goes against the general idea behind machine learning, where as
little human interaction as possible is desired. In this project a new approach to data
augmentation is presented, which seeks to combat these issues.
Any given dataset consist of elements sampled from some space. The elements of the
dataset span a subspace of this larger space. The proposed data augmentation scheme
aims to ﬁll out this subspace, by learning a transformation between every pair of data
points, which aligns one data point to the other. The new data points are then generated,
by subsequently sampling from the learned transformations. In this way the model
is able to learn the variations in the dataset, as the model attempts to transform one
element into another. Using the example of handwritten 4’s, if the model encounters a
4 with an open top and one with a closed top, the transformation will attempt to push
the top of the open 4 together. The generated images will then be 4’s with varying

The beneﬁt of this approach is that the data augmentation scheme only introduces vari-
ation that is already present in the data set, thus making sure that the created data points
are indeed realistic. Also, the parameters of the transformation are not set by the user,
but are learned in the process of alignment. The disadvantages of the proposed aug-
mentation scheme are the relative complexity of implementing the data augmentation
scheme, compared to the typical data augmentation scheme, and if a data set contain
very little variation in itself, the proposed data augmentation will not be able to generate
new images with a lot of variation.
Furthermore the performance of the data augmentation scheme will be tested by train-
ing a convolutional neural network on an augmented dataset, and compare it to the
performance without data augmentation.

Data
The data used in this project consist of 10 CT-scans of upper body of young adults (aged
16-35), with corresponding annotated images [5]. This is a dataset, where the simple
data augmentation scheme will be unable to capture the variations of the data points.
The spines vary between somewhat straight spines, and spines with a more pronounced
s-shape. The images are volumetric images, each consisting of between 500 and 600
layers, where each layers is a 512×512 image. To simplify alignment and segmentation
in the images, each image is resized to a 512×512×560 image, and then translated so
that all spines have the same centre. Cross-sections of the spines are shown in Figure

Data augmentation
The proposed data augmentation scheme is based on more advanced transformations,
than those use in a typical data augmentation scheme. The transformations should be
able to locally change the shape of an image, rather than imposing the same change
on the entire image. For a pair of images in the image space xi, xj ∈Ω, we are
looking for a transformations T i,j : Ω→Ω, such that T i,j(xi) ≈xj. Where Ωis the
[0, 1] × [0, 1] × [0, 1] box in R3.
These transformations used to align image pairs should meet certain requirements.
Differentiability
The transformation should be differentiable, such that the alignment between pairs of
images can be optimized using gradient-based optimizations. Furthermore the differen-
tiability ensures a smooth transformation without sharp edges in the transformed image.
Inverse should exist
An invertible transformation allows the possibility of taking at step back, if the trans-
formation is has taken an unfortunate step. Also, an invertible transformation ensures
that the transformation avoids folds in the resulting image, preserving the topology.
Differentiable inverse
To take a step back using a gradient-based optimization, the derivative of the inverse
transformation should exist.
These requirements together are the deﬁnition of a diffeomorphism.
2.1
CPAB
To ﬁnd transformations that meet all three requirements, continuous piecewise-afﬁne
(CPA) based transformations are used. The transformations are based on CPA velocity
ﬁelds. The piecewise refers to a tessellation P of the image space Ω, that is the image
space is split into several cells C1, ..., CN, each with an associated afﬁne transforma-
tion. Additionally equality constraints are imposed on the transformations along the
boundaries between cells, to ensure continuity between neighbouring cells.

A continuous piecewise-afﬁne based (CPAB) transformation is a transformation based
on a CPA velocity ﬁeld. The trajectories of the points through the CPA velocity ﬁeld,
gives the CPAB transformation. The trajectories are found by integrating the points
over the CPA velocity ﬁeld.
Given that the CPA transformations are initially deﬁned by their derivative, i.e. the
velocity ﬁeld, the transformations are clearly differentiable. Also, the transformations
are invertible, since the inverse transformation can be found by integrating over the
negative velocity ﬁeld, and the derivative of the inverse transformation exist, namely
the negative velocity ﬁeld. The integration over velocity ﬁelds will be dealt with in
section 2.3.
Let VΩ,P be a space of such CPA velocity ﬁelds and let d = dim VΩ,P, VΩ,P is ﬁnite
dimensional and linear [2]. Then there exist a bijective function Z : Rd →VΩ,P. This
means that for every element θ ∈Rd, we can ﬁnd a corresponding element vθ ∈VΩ,P.
Being able to go from R3 into VΩ,P would be very convenient, since it would mean that
any random element of R3 could be transformed into a valid CPA velocity ﬁeld. This
makes the optimization process much easier, as optimizing over elements of R3, instead
of VΩ,P, makes sure that every step lands at a valid point.
To ﬁnd the transformation from Rd to VΩ,P, we look at the conditions on the parameters
of the velocity ﬁelds.
First we have the continuity conditions between neighbouring cells. Furthermore con-
ditions to keep the boundary of the image ﬁxed are added, such that points on the image
boundary can move along the boundary, but never away from it.

2.2
Tessellation
To transform an image, we need to calculate where each voxel ends up in the trans-
formed image. For any given voxel in the image, the path taken by that voxel during
transformation must be tracked to know in which cell it is currently located, and there-
fore which afﬁne transformation is affecting it’s path. This part of the algorithm will be
referred to as the findcellidx function.
Given the number of voxels in a volumetric image, calculating the cell index of every
single voxel several times during transformation takes up a large part of the compu-
tation. Ideally the chosen tessellation should make these computations as simple as
possible, such that the findcellidx function takes minimal computational effort.
However, the tessellation also determines the ﬂexibility of the CPAB-transformation.
The number of cells in the tessellation determines the degrees of freedom of the trans-
formation. A ﬁne tessellation will result in the more free, but computationally ex-
pensive, transformation, whereas a coarse tessellation will require less computational
effort, but may result in a too restricted transformation.
Initially a tessellation with each cell as a box was considered. This was mostly due to
the simplicity of the corresponding ﬁndcellidx function. However, the resulting trans-
formations were all too restricted by the continuity constraints, regardless of the number
of cells. This meant that the alignments were only able to capture translational variance,

2.4
Alignment
To align two images xi and xj we are interested in ﬁnding the transformation T θi,j,
such that T θi,j(xi) ≈xj. transforming an entire image is computationally expensive,
since the images are of size 512 × 512 × 560, approximately 140 million voxels. In
the alignment between images, the image has to be transformed several times to ﬁnd
the optimal θ. Furthermore, since the spine only takes up a smaller part of the entire
image, a lot of the computations will be irrelevant in regards to the alignment.
To save computational effort we let each spine be represented by a curve through it’s
centre, and align these instead of the entire image. Representing each curve as 100
points along the spine, this approach is much less computationally expensive with only
one millionth the points of an entire image. The downside to this approach is that
the alignment cannot catch rotations along the spine, but only the general shape. We
believe that this will not affect the performance overly much since the alignment will
still be able to capture the curvature of the spine, and from the images, it does not seem
there is a lot of twisting of the vertebrae in the images.

The gradient of the loss function w.r.t. θ is then estimated using ﬁnite difference ap-

Having aligned all pairs of of images, the corresponding θi,j are all stored. To per-
form the data augmentation, we need to be able to sample new θ vectors randomly,
and apply the corresponding transformation to some image. We choose a symmetric
prior which will penalize large values, namely a zero-mean Gaussian(0, Σ). Since the
optimal transformation from image xi to xj, T θi,j = T −θj,i, the optimized θ should
have zero-mean. However, given that the alignments are not necessarily perfect, the
mean is likely a little of from the expected. To enforce a zero-mean in the computed
θ’s, we replace θi,j by −θj,i, for i > j. Having imposed a zero-mean on the θ’s we
compute the covariance Σθ. Sampling from the Gaussian(0, Σθ) distribution, we can
obtain new transformations centered around the identity transformation. During data
augmentation, each image in the dataset is then transformed with several sampled θ’s.
In this project we sample a total of around 120 transformed images.

3
Results
The dataset was split into a training and a test set, using spines 1 through 7 for training
and 8 through 10 for testing. 120 new images were generated for the training set by
randomly sampling iθ’s and applying the corresponding transformation to the images.
Examples of the transformations for each of the seven training images can be seen in
Figure 4. The comparisons in Figure 4 shows the way the data augmentation scheme
alters the shape of the spines. In 4e the curvature in the upper spine was accentuated by
the transformation, whereas in 4f the spines was straightened. In other cases the spines
was bent to the side (4b, 4g). Since the alignment was done between curves instead of
the entire image, the transformation does not take the volume of the spine into account.
In Figure 4a some of the vertebrae have been squeezed, making the disproportionally
small, and in Figure 4e some vertebrae have been stretched. From the example images,
this does not seem to be much of a problem, since most vertebrae seem to maintain a
reasonable size. However, other methods of alignment could be considered to prevent
this from happening. In ﬁgure 5 example images are shown, where the transformation
has distorted the vertebrae signiﬁcantly.
A convolutional neural network based on the U-net [4] architecture is trained both on
the original dataset, and on the augmented dataset. The architecture of the original U-
net can be see in Figure 6. Each image is split into 512×512 slices along the spine.
This is done for simplicity, and the fact that a full image takes up a lot of memory by
itself, which can create memory problems for the network.
The loss for each of the two datasets can be seen in Figure 7. The loss of the network
trained on the augmented dataset converges to a value lower than the los of the network
trained on the un-augmented dataset.
The output of the network is an image containing values between 0 and 1, each entry
is considered the probability that this position is spine. The predictions of the network
are then found by setting all entries greater than 0.5 to 1 and 0 elsewhere. The test error
of the network predictions are found by computing the mean square error between the
probabilities and the true segmentation. This, along with the prediction accuracy, gives
an idea of how certain the predictions of the networks arre.
Comparing the two networks on the test set, the prediction accuracies are almost iden-
tical, with the data augmentation slightly ahead, at 99.4% and 99.5%, while there is
a larger gap between the mean square errors at 385.5 and 325.3, also with the data

4
Conclusion
We have successfully implemented a learned data augmentation scheme for volumetric
images, and shown that a network trained on the augmented dataset performed better
than a network trained on the un-augmented data. However, the difference in perfor-
mance was not as great as expected. In the alignment of data points, we chose to
represent spines as points along their centres. This resulted in several instances where
the transformed vertebrae appeared distorted, which can have affected the network dur-
ing training. A way of improving this part of the augmentation scheme could be to
represent spines as several curves along the outer edge of the spine, this would help
preserve the volume of the vertebrae, without requiring too much added computational
effort. Another interesting approach would be to accept the much increased runtime,
and simply align entire images to each other to understand what effect this has on the
end result, compared to the simpliﬁed alignments.
Furthermore the convolutional neural network was implemented for 2D images, which
meant that the data has to be split up into slices. It would be interesting to see how
the performance had compared on a 3D network, where the whole shape of the spine
would have been available for the network.

We would expect the data augmentation scheme to perform well on other datasets than

umetric images tend to be very large, making the alignment process computationally
quite expensive. Because of this, in some cases it might be more reasonable to use
the typical data augmentation schemes to generate data a lot faster. Having said that,
when some kind of simpliﬁcation of the alignment process can be done, like represent-
ing spines by a curve, the runtime is far from unreasonable for most cases, making it a
viable option.
Overall we saw some improvements in the performance of the network when training
on the augmented dataset. Also, the generated images seemed to behave as expected, to
some extent, stretching and bending spines, giving new images ranging from a some-
what straight spine, to a pronounced s-shaped bend. This considered we would expect
data augmentation scheme to perform even better with some slight moderations, war-
ranting further investigation into data augmentation based on CPAB transformations.
