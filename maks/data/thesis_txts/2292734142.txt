Abstract
In modern society the most notable change in robotics have been the introduction of autonomous
navigating robots. Several companies have created robots which are self driving or ﬂying but they all rely
on GPS navigation. This thesis introduces different methods in order to detect and segment objects with
the purpose of creating a situational awareness for a robot. The awareness gives the robot the possibility to
navigate by the use of recognizing objects and relate them to already found object.
The two methods used for mainly detection was a HSV Histogram Threshold and a 4D Histogram
Thresholding both based on the HSV color model. From the results it was shown that detection with the use
of only a color model was inﬂuenced on the lighting of the objects. Although the two methods was found to
be successful in detection and could be combined for better results.
Besides the two methods a landmark segmentation method was presented in order to ﬁnd a speciﬁc object
common in the area of operation. The results showed that the detection and segmentation was successful and
added an extra dimension to the situational awareness.

CHAPTER 1
Analysis
For the modern society robots have become something that normal families can buy in the form of for
example a robot vacuum. The production industry have seen an inﬂux of new ways to use robots enabling
faster and more efﬁcient production. Although these robots are autonomous but only work within a small
conﬁnement, such as the living room or a production hall. The most notable change is the few robots
operating in the public space having to work in a changing environment with many inﬂuences from people.
Google have developed a self driving car [1, Google Car] which are used for autonomous passenger
transportation in cities and on highways. This means that the robot, being the car itself, have to adjust and
navigate amongst driving cars, which itself can be quite the task. The obstacle avoidance for the car is done

A method to resolve this problem is to mimic the human navigation method, which is done by visu-
ally recognizing objects and landmarks to give a bearing or sense of relative positioning to other objects.
Knowing the position of these markers in for example a forest, where GPS signal is very difﬁcult to obtain,

1.1
Objectives
The objective of this thesis is to propose methods in order to segment the objects needed to create a map.
Using this map gives the robot situational awareness due to both positional information but also the overall
shape of known objects to compare to.
In order to visually ﬁnd an object, the images taken are processed and evaluated to achieve this. The
goal is to separate different objects into regions and then be characterized such that they can be added to a
map later.
Using images as the source of detection requires that the pixels are represented such that further
processing yields the best result. The deﬁnition of the pixels are described by a speciﬁc color model and the
choice is the ﬁrst and most essential task towards detection.
From the color model representing the pixels of the image, the objects can be detected by the use of
different methods. The model have an inﬂuence on these methods since it is deﬁning the information
available for the pixels. Closely relating the methods to the structure of the model gives a detection method
that work on different images without applying constraints to the shape of objects which can be segmented.
The objects that have been found by the segmentation have to be post processed such that the positional
information can be stored for later used. This data is found by relating points and lines of the objects, and
constructing polygons, to the coordinate system of the robot.
Besides the objects of the image the next goal is to detect the landmarks and give the necessary positional
information to store alongside the objects in order to create a map.

CHAPTER 2
Color Model
2.1
Introduction
When working with images the representation of them are the fundamentals on which further manipula-
tion is build upon. The representation is deﬁned by the use of a color model which specify the color of a
single pixel within the image. These models use different combinations of color components in order to
describe the color. The choice of components leads to the deﬁnition of how the color can be expressed and
are important for how detection of objects are done.
The structure of a color model is a coordinate system where each of the axis corresponds to each of
the color components. This means that a point given by coordinates within the model can be translated
directly into a speciﬁc color for each of the pixels within the image. Each of the coordinate axes are made
into layers of the image such that the combination of all the layers gives the ﬁnal result i.e. the original image.
Each of these layers are all quantized into a resolution for which the component can be characterized
by. This normally consists of an 8-bit resolution for three layers giving a 24-bit color image, which is used
such that it us easier to implement into hardware.
With respect to the goal of ﬁnding objects in an image, the color model is used to detect similarities
between the color components used. This means that a region which deﬁnes an object can be assumed
to have very similar values for each of the layers in the model. Although this is only true when the color
within the captured image of the object is the same. With the change of lighting or lack of it, objects can be
obscured in the image and appear very different compared to if a human was to observe the object.
In the following two sub sections two color models will be examined for the use of object detection.
The ﬁrst one being the most common model, RGB, and the second one the HSV model.
2.1.1
RGB
The most common color model used is RGB, which stands for "Red", "Green", and "Blue". This color
space are deﬁned by a cube in the Cartesian coordinate system, which is normalized to be within 0 to 1. The
three axis deﬁnes the amount of red, green, and blue needed in order to represent the color.
For the color to be bright the overall value of the layers or coordinates must be closer to 1 than 0, since
it describes the amount of red, green, or blue and not the brightness. This then means that darker colors

between these two points a diagonal line is formed through the cube, which is called the natural axis. This
line deﬁnes all the grey colors from black to white. The RGB cube and the natural axis can be seen in the

As described before when representing an image with the RGB model each pixel are given by three layers,
corresponding to the axes which is the red, green, and blue color. This gives each the x-y positions in the
image a value in the all of the three layers, and when superimposing these together gives the original colors
of the image.
When looking at the structure of the RGB model, it requires all colors to be composed of the three colors.
As described before this then means that the characterization of the speciﬁc color is not well represented.
When an image suddenly becomes bright the overall values within the model are increased, driving the mean
value of the pixels towards 1.
The model itself is very good for generating a color but lack the overall characterization. This evident when
trying to detect a certain color within an image of the same object in both darker and brighter lighting. It
could be compensated for by looking at the overall mean value of each of the layers in the RGB image and
scaling them accordingly [7, p. 289-294].
Instead of post processing the image and correcting for the change in brightness, a different color model
can be used.

2.1.2
HSV
This model is contrary to RGB formed as a cylinder and consists of the following three layers: Hue,
Saturation, and Value. The difference between this model and RGB is that instead of using colors to
described the separate layers, they are deﬁned by ﬁrstly what spectrum of color it is, how much of the color
is present, lastly the brightness of the color.
These three descriptions are corresponding the Hue, Saturation, and intensity. The Hue of the color is
given by an angle in a color circle where all colors are represented. The Saturation and Value are both given
between 0 and 1, describing the percentage of pollution of white and intensity of the color respectively.
Together this creates a cylinder where the angle is the Hue, Saturation is the width, and Value is the
height. Combining these three together gives the color to be described.
Differently from the RGB model, the color black is deﬁned as the slice of the cylinder where the Value is
0 and the Hue and Saturation ranges from 0 to 360 degrees and 0 to 1. White is deﬁned as the top of the
cylinder with zero Saturation and the Hue ranging from 0 to 1. The natural axis of the cylinder is the line
starting from the origin in the black plane to white point and similarly describes all grey colors.
The cylinder can be seen in the ﬁgure below:
Figure 2.2: HSV Color Model, the Hue is given as an angle around the cylinder, the Saturation as the radius
and the Value as the height of the cylinder.
As described this model decouples the speciﬁc color from the characteristics of the color, which makes it
possible to detect regions of same Saturation and Value. This possibility will be described in the next section
comparing the RGB and HSV color models.

CHAPTER 3
Segmentation
3.1
Introduction
For a robot to discern objects and recognize the object itself the ﬁrst step is to segment an image into
regions, where these can then be processed and classiﬁed as objects. Segmentation of objects consists of two
tasks, the ﬁrst one is to detect the object of interest from different assumptions or known parameters and
secondly to deﬁne the region of which the object is segmented to.
There are several different methods to detect objects in an image and it is very dependent on the spe-
ciﬁc object. The characteristics of the object, such as the shape, the color, and size, can be used to detect if
the object is present in an image. The objective of the segmentation for the robot is to ﬁnd objects that are
present in a forest environment, which means that two objects of different size or shape still can be of the
same classiﬁcation.
An example of this could be trees in different heights, widths and even shape. The geometry change
with the perspective and movement of the robot, which imposes a constraint on the method for detection.
It cannot rely on templates or assumptions of size and shape in order to detect similar objects since the
environment is not controlled or inﬂuenced.
Another common used method for detection is the color components, which gives information about
how the color is deﬁned. With the use of the HSV color model it is possible to neglect the geometry of the
object and purely detect the speciﬁc color representation. Although this means that all trees and bushes, as
used in the previous example, all are detected as one and further more are prone to changes in lighting.
The segmentation is only to deﬁne where objects are in an image and not speciﬁcally what the object is.
This method is the simplest since it only concerns the color in the image and by using the information about
the color of object, a simple classiﬁcation can be approximated in order to give some knowledge of the object.
For segmentation with the use of the color model the information of the layers must be organized into a
structure which can be used for interpretation. This is necessary to distinguish different objects from each
other and help create thresholds that can detect the objects. By thresholding the image, regions of which
objects reside in are found but not the speciﬁc object. That requires the knowledge of how an object generally
is represented in the color model for it to be thresholded from the image.
This chapter focuses on the use of the color model HSV for detection, where two different methods
will be shown. The ﬁrst method uses histograms of the layers to determine thresholds for objects. The

3.2.2
Valley Detection
When examining the histogram for the three channels of the HSV color model it can be seen that there
is several valleys that deﬁne distinct regions of which pixels are grouped into. By simply detecting these
valleys it is possible to ﬁnd suitable threshold limits in order to segment out speciﬁc objects within the image.
The assumption is that if there is a very prominent valley between two curves in the histogram, this most
likely indicate a separation of two objects.
The method for valley detection builds on a weighted window sliding across the histogram, ﬁnding values
which have a signiﬁcantly smaller amount of pixels compared to the surrounding values. The window
consists of 5 values where the middle of the window, called ’x’ for purpose of explanation, is the value of
interest.
This value is checked if it is a valley by using the 4 other values which is situated around the index of
x. The ﬁrst and last values of the windows are the values 10 indices before and after x respectively. The
second and fourth values are the value 5 indices before and after x respectively. This window can be seen in
Figure 3.4 with the 4 values surrounding x.
Figure 3.4: Search window for valley detection showing the 5 values used, where the data is an example.
The ’x’ is the value of interest which is checked if it is a valley by using the other values around it.
To ensure that the full search window does not slide out of bounds in the histogram, x is limited to start at the
11th index and stop 10 indices before the end. For each time the window slide, the index for the minimum
value within the section from (x −10) to (x + 10) is found. If this minimum is not at the index of x it cannot
be a valley and the window slides one position to the right. On the other hand if x is the smallest value a
weight for the window is calculated by the use of the following equation:
w = H(x −10) + H(x + 10) + H(x −5)

3.2.3
Morphology
The two operations used in morphology are opening and closing, which consists of a consecutive dilation
and erosion operation [7, p. 523-527]. Morphology in general uses a structure element as a template for
the region which is affected and is described by a binary matrix. The matrix have a deﬁned centre which is
placed on pixels which is ’1’ or true in the binary image.
The ﬁrst operation is dilation which expands the current pixel structure by adding pixels where the structure
element covers a pixel which is ’0’ or false. The second operation is erosion which oppositely removes any
surrounding pixels from the centre which is covered by the structure element. The placement of the element
for dilation is on the every pixel which is ’1’ but for erosion it is only on those where the the structure
element ﬁt within the set of pixels.

As described before the combinations of the two explained operations create Opening and Closing.
Opening consists of an erosion and then a dilation using the same structure element. By removing the
outer layers of pixel structures via erosion, any protrusions and narrow paths between regions are removed.
The following dilation restores the size of the structures and completes the operation.
Closing uses the reverse order of the two operations by ﬁrstly dilating to close any holes and ﬁlling gaps
in the structures. Secondly as for opening the size of the original region is restored by eroding the additional
pixels added by dilation.

3.3
4D Histogram Thresholding
In the previous method the principle of ﬁnding thresholds for objects were based on the histograms for
each of the channels, where the valley between two large sections could be detected. By combining the
histograms into a coordinate system for the HSV a 3D space with each of the color components as the axis
can be created. Each pixel in the original image can be represented in this space and clustered in order to
detect different objects. Detecting these clusters gives the possibility to ﬁnd groupings of pixels which are
assumed to have the same color and therefore also the same object.
The clusters can then be used to segment the image into detected objects, and by imposing classiﬁcation
assumptions the clusters can be determined as speciﬁc objects. The clustering is done by a K-Means
algorithm with a seed determined by a weighted 4D window peak detection.
3.3.1
4D Histogram
In order to create the 4D histogram such that it is fast to search through, the 3D HSV space is limited
to 25 intervals per axes. This subdivision reduces the amount of points that needs to be processed when
performing the seeding for the K-Means algorithm.
To create this histogram the three layer values for every pixel are ﬁtted individually into the axis intervals to
create coordinates in the 3D space. Pixels which are close in all the values will be put into the same interval
creating a grouping of points. The amount of pixels or weight of each of the interval is put into the 3D
structure creating a fourth dimension in the histogram.
An illustration of the 4D histogram can be seen in Figure 3.11 where the weight of each of the intervals
are displayed as a increased or decreased circle. The size of the diameter describes the amount of pixels at
the interval coordinate, which can be correlated into a grouping of pixels of the same type. The assumption
is that these large weighted coordinates corresponds to individual objects and can be found by clustering
them together.
3.3.2
Seeding
In order to seed the K-Means algorithm the largest weighted coordinates are found and used for the
initial starting point. The method is using the same principle as the Valley Detection in 3.2.2 but instead
looks for prominent peaks in the 4D histogram, which are the presence of large weights. The coordinate
system which the peaks have to be found in are 3 dimensional, therefore the search window is enlarged to a
3D structure.
The shape of the window is formed as a cross where the centre is the value of interest when search-
ing for peaks. Just as the valley detection the values in the window are weighted to create measure for
the potential peak. The window can be seen in Figure 3.12 which consists of the centre and the 6 closest
coordinates in the histogram. This gives a width of 1 and uses the neighbours to create a spatial relationship
to the middle coordinate. As for the valley detection this window slides through all pixels except the outer
points of the coordinate system, since placing the search window in any of these points will cause it go out
of bounds.
Each time the window slides to the next coordinate, the centre and the 8 neighbours surrounding it (this
includes the diagonal neighbours) are checked if ’x’ is largest value in the window. If this is not the case
the window slides to the next coordinate but if it is the measure for the peak detection is calculated. This is

3.3.3
K-Means Clustering
The clustering algorithm used is k-means which partitions a dataset ’X’ into ’k’ centroids by the use of
the euclidean distance to the initial centres. Normally these starting centroids would be chosen randomly
from X with a probability in relation to the ﬁrst chosen centre. This method of centroid initialization is
called k-means++[8] which effectively chooses new centroids in X.
For the histogram clustering a predeﬁned seeding for the k-means is used to reduce the running time
of the algorithm. The main reason is to ensure that the centroids are closely placed to the largest weights in
the histogram but also to remove the initialization method and therefore decrease the run time. The method
of seeding from previous subsection is used for clustering where an example of the seeds can be seen in
Figure 3.13.
Figure 3.14: The principle of K-means clustering visualized on an example data set. Step 1: Initialize
k cluster centres for the data set, which are shown as rectangles. Here the choice of k is 3. Step 2: The
distance to all k centres for every observation is calculated and the closest centre is associated with the data
point, which can be seen as the coloring. Step 3: The cluster centre is placed at the centre of mass for the
clustered data point. Step 4: The new centres are used to perform the same clustering is in Step 2.
With the use of the seed each initial centroids are already chosen and the general k-means algorithm is used.
This consists of 3 steps which are repeated until the method converges.
The ﬁrst step is to calculate the euclidean distance from every data point in X to the k cluster centres.
For every point the closest cluster is assigned to them such that a cluster set c can be formed.

3.3.4
Classiﬁcation
The classiﬁcation of objects is the last part of the 4D Histogram clustering and introduces some
assumptions in order to interpret the clusters. The three different objects which are distinguished from the
clusters are the sky, the road, and the trees.
The reason why the grass is not included in this classiﬁcation is that the general information of the
position does not contribute to the operation of the robot. It cannot be used for navigation since it is very
homogeneous and cannot be differentiated from other observations of grass. Further more it is an area that
in it self is navigable by the robot.
Sky classiﬁcation
The ﬁrst classiﬁcation that is the detection of the sky. The clusters found from the k-means algorithm are
positioned in the 3D HSV space and can be used for designation. The centroid which have the largest Value
coordinate in the cluster list represents the pixels that have the largest brightness.
The assumption is that these pixels are indicating the sky since it is the source of the largest illumination.
The operational environment of the robot is a dirt road in an small forest and here the largest source of
lighting are either reﬂections of the sun or the sky itself.
Figure 3.16: Inspection of 3 pixels which represent the sky where the corresponding HSV values are giving
for each of them. The pixels are seen to have small Saturation due to the lack of any color and oppositely a
large Value due to the brightness.
In Figure 3.16 a magniﬁcation of the sky can be seen. Looking at the HSV values for the 3 pixels placed
closely together, it can be seen that these indeed have a large Value. Since the cluster representing these
pixels are placed in the centre of mass of all the included pixels, the largest valued cluster are assumed to be
the detection of the sky.
Another observation is that the saturation generally are small for these pixels which is due to the deﬁnition
of the color component. A small Saturation value corresponds to a pixel which have almost no color left and
is polluted with white. The last indication to support the cluster being the sky, is to calculate the centre of

Summary
This chapter have shown the use of two different segmentation methods both using the HSV color model
as the starting point. From the results the ﬁrst method, the HSV Histogram Thresholding, was seen to detect
the road in the image without greater disturbances from lighting. The use of the saturation histogram for
thresholding detected other regions such as the sky or trees, but with the knowledge of the environment these
segments was removed or reduced.

and can therefore be segmented. This makes the solution to the detection problem dependent on that the
road always have this property and that it cannot discern different roads from each other. This could be seen
in the crossing roads test where the different saturation of the brick road did not get detected or the overall
saturation threshold was introducing non-road segments.
To alleviate these dependencies on speciﬁc properties the 4D Histogram Thresholding were introduced,
where grouping of pixels were used to detect different objects within the image. Rather than relying on
assumptions made towards the speciﬁc color of an object, each pixel was clustered into regions such that
the individual sections could be segmented. Looking a the clusters generated three main objects could be
segmented, which were the road, the sky, and the trees.
The results for the method showed that the different objects could be detected and classiﬁed but with the
introduction of spotlighting from the sun made the task of separation difﬁcult.
Comparing the two methods the simple HSV thresholding from the single histogram had the best re-
sults in ﬁnding the road. When looking overall at the detection of speciﬁc objects this method would require
many assumptions in order to detect the correct threshold in the histogram. By using the clustering of the 4D
Histogram method these assumptions was not needed.
Combining the two methods together will give an effective road detection with the possibilities of
segmenting other objects such as the sky and trees. The single histogram method will then be used as a
quality check for the cluster classiﬁed as the road.

CHAPTER 4
Post Processing
4.1
Introduction
The obtained regions representing the different segmented objects can be processed such that potential
incorrectly detected structures are discarded. In order to perform this process some assumptions are made
for the characteristics of the sections.
The general assumption is based on the amount of pixels detected for each structure, where objects
known to be of a certain size compared to the image can be thresholded such that any smaller would be
removed. To ﬁnd the different structures of the binary image the Connected Components algorithm [9,
Mathworks] can be applied, which uses neighbouring pixels to ﬁnd and separate individual pixel groups.
From the components found the position of them is required to be found such that it can be saved into a map.
Here a polygon representation gives a structure which can be translated into the coordinate system of the
robot such that it can be related to the position of the robot. This can give a relative measure of distance to
each of the components and ﬁnalize the described map structure.
In order to create the polygon, the edge of the structure can be traced and used as a rough outline. The
detection of the edge does neglect holes within the structure.
The whole edge of an object can potentially comprise of a large amount of pixels and be difﬁcult to
process and later saved. A reduction algorithm can be used on the edge to alleviate the amount points and
create a reduced polygon.
This leads to three main tasks presented in this chapter, separation of individual structures in the binary
images, detection of the edge for each of these, and creation of a polygon from the edge with a reduction
algorithm.
4.2
Connected Components
In binary images pixels that are connected can be described as having a neighbourhood with another
pixel. For 2D images two connectivities can be used, the ﬁrst only concerns the closest neighbours to the
pixel of interest. This is called 4-connectivity and by including the diagonal pixels in relation to the centre
pixel additional 4 neighbours are added which gives 8-connectivity. The two neighbourhoods can be seen in
Figure 4.1

For the 4-connectivity the pixels in the grey area are detected as neighbours where the one in the white is
not. This pixel would have be detected with the 8-connectivity since it includes the diagonal pixels which
can be seen.
Deﬁning the pixels which are connected in the binary image can be done with the use of Connect-Components,
which uses the described connectivity. The algorithm which is used mainly is called Flood ﬁll[10] which
will be described only shortly.
For a set of pixels the algorithm starts in one unmarked pixel, from this point it uses the connectivity
ﬁnd the nearby pixels which is done recursively until all connecting pixels are labelled. The analogy for the
algorithm is to take a ’bucket’ and spread water unto pixels until there is no more places for it to go.
Then another unlabelled pixel is found and used ﬂood ﬁll upon until there is no more in the set of pixels.
For each of the different labelling a component is created, splitting these regions into separate structures.
The use of the connected components are to remove those which have smaller size than a threshold
set. For the detection of the road the threshold used for both of the segmentation method was chosen to be
150 pixels. The result of discarding the smaller structures can be seen in Figure 4.2.

4.3
Edge detection
In order to create the polygon of the detected components, the edge is found in order to approximate
the shape. To accomplish this the overall method is to trace the edge and explore all protrusions from the
structure, ending in at the starting point to create the full edge.
The tracing is done by evaluating the 4-connectivity for the pixel which the algorithm currently is on,
moving clockwise around the edge according to rules.
The starting point for the tracing is found by taking the ﬁrst pixel for the structure, found by the con-
nected component, and then move furthest to the left assuming that this ends on the edge. From this point it
starts by ﬁnding the value of the surrounding pixels such that the movement can be done.
A portion of this algorithm can be seen in Algorithm 1 where the position of the last pixel is used for the
movement. The value of the variable lastPos are chosen according to Figure 4.3, where if the last position
was above, to the right, below, or to the left the value becomes 1, 2, 3, or 4 respectively.

4.4
Polygon Reduction
The ﬁnal edges found in the previous section could be used to describe the polygons for the structures,
although the amount of pixels used would make the processing fairly slow. By reducing the points for which
the polygon is represented as, the polygon can be handled faster with less points.
The method used for the reduction of points is called the Douglas-Peucker algorithm[11] which uses the
idea of splitting the points into segments and approximating the curve. The pseudocode for the algorithm
can be seen in Algorithm 2 where the splitting is done by recursively call the method itself twice with the
two segments.
Algorithm 2 Polygon Reduction (Douglas-Peucker)
procedure POLYGONREDUCTION(PointList, ϵ)
Initialize the maximum distance dmax = 0
Initialize splitting index = 0
Initialize end of point list end = length(PointList)
for i = 2 to (end −1) do
d = FarthestDistance(PointList[i], Line(PointList[1], PointList[end]))
if d > dmax then
index = i
dmax = d
end if
end for
if dmax > ϵ then
Result1 = POLYGONREDUCTION(PointList[1...index], ϵ)
Result2 = POLYGONREDUCTION(PointList[index...end], ϵ)
ResultList = Result1[1...length(Result1)], Result2
else
ResultList = PointList[1], PointList[end]
end if
Return ResultList
end procedure
The function called FarthestDistance calculates the distance from the line described by the ﬁrst point and
last point in PointList, to the rest of the points in the PointList. The index of this point is saved and used to
split the points into two lists, which is used as the input for two different calls of the same function.
The algorithm stops dividing the set of points until the maximum distance is below or equal to ϵ, which is
a threshold for the maximum distance. This is chosen fairly low, at 2.7, in order to keep the shape of potential
protrusions. Choosing this value larger results in the algorithm using fewer points for approximating the
segments and gives a polygon with less edges. The results of reducing the points for the edge polygons
in 4.4 can be seen in Figure 4.5.
It can be seen that the structure is approximated with a polygon, but reduces the amount of detail of
the edge. The assumption is that for the general detected road or object the shape are not required to be
detailed. This is due to the fact that these polygons are not further processed but are only to be saved into a
general map.

4.5
Summary
The objective of this chapter was to illustrate the post processing done to create polygons. The separation
of individual components gave the starting point for detecting the boundary. The connected components was
used for discarding smaller structures which was classiﬁed as noise or miss detection, but also resulted in
individual objects to be handled.
The edge of these objects was then found by an algorithm tracing the circumference. If the structures
included any holes these would be neglected, which was due to the objective only being detecting the position
of the objects.
From the edges detected the polygons could be created, but the structures generally included to many
points. The polygons was then reduced by the Douglas-Peucker algorithm which with the use of dividing the
points into smaller segments approximated the curve resulting in the ﬁnal polygons.
The quality factor ϵ was chosen to preserve the general shape and some protrusions of the objects.
Although the value could have been increased to lower the amount of points used for the polygons, but was
kept small to keep smaller thinner structures somewhat represented.

CHAPTER 5
Landmark Detection
5.1
Introduction
When humans navigate any area it is done by recognizing certain features and places such that a visual
observation gives a relative position to the destination at hand. These features can be described as landmarks
which are navigated from in order to travel to the desired target. Combining all these markers a map can be
formed such that it can be maneuvered from.
Detecting these speciﬁc objects require some prior knowledge which for humans are simply memories
that can be remembered and processed for later use.
Comparing this to a robot the exact same methodology can be applied by saving positions of detected
objects into memory which forms a map of the environment. By using predeﬁned landmarks as the objects
the process of detection can be made easier for the robot.
In the environment for the robot milestones are placed to deﬁne the path and mark every kilometre. All
of the markers have a distinctively red sign on them and are painted in a black color. This information gives
the robot a speciﬁc object to ﬁnd when driving on the road.
This chapter presents the different methods used in order to detect the markers and highlight them for
further processing. The ﬁrst step is the detection of the sign on the landmark followed by segmentation of
the marker. The third and ﬁnal step is to ﬁnd the two parallel lines which are formed by the marker it self.
5.2
Detection
Detecting the distinct red sign on the landmark is done by the use of the same method as the HSV
Histogram Thresholding. Using the hue histogram instead, the speciﬁc region in the color circle for red is
extracted such that the red sign can be found.
The color wheel is cut such that red is represented both as 0 and 1 in the histogram. This means that
there are one section on either side of the histogram where the red color can be found. These two areas are
stitched together such that the two ends of the histogram is connected, giving a small region of only the red
color.

lines in the segmented image. This process uses a kernel for convolution with the image such that the correct
features are highlighted. The choice of the structure of the kernel deﬁnes the operation which is applied. The
vertical lines can be found by calculating the gradient along the horizontal axis, where an appropriate kernel
is chosen for this operation.

right pixel is used: [1 0 −1]. Applying this kernel ﬁnds the change in pixels from left to right, highlighting
only the left side of the pole. The inverse kernel [−1 0 1] can then be used to ﬁnd the opposite side and
complete the vertical line detection. The convolution of the two kernels and the combined result can be seen

5.6
Summary
The overall goal for the landmark detection was to separate the speciﬁc marker from the image in order
to process it into data that could be saved into a map. The method was to look for a prominent peak in the
hue region deﬁning the red color and using it as the threshold to ﬁnd the sign. From the section below the
detected sign a threshold could be found in order to detect the pole. From the thresholded image the edges of
the sides could be found and the corresponding lines was correctly found.
The general method was also seen to discard wrongly detected areas which was not the sign, but when
the saturation of the overall image was lowered a incorrect detection could occur in brown or reddish areas.
If the sections below then also was dark enough the thresholding for the pole would be made and wrongly
detect an incorrect landmark. The threshold found by this area below the detected sign was seen to segment
the pole giving a structure which could be used to ﬁnd the sides with. Even when the sign was incorrectly
discerned the resulting thresholded image was seen to detect the structure, producing an edge map and
corresponding lines, although the result was useless.
The further processing of the lines needed was to ﬁnd the correct star and end point for the pole, such that
they could be saved into a map.

CHAPTER 6
Conclusion
The aim of the thesis was to present solutions to object detection and segmentation, and propose a
method for situational awareness in the area of operation.
The overall result of the different methods was shown to correctly identify speciﬁc objects and landmarks,
in different situations and for several images. Classifying the objects gave the robot the possibility of
recognition and a platform to further build upon the situational awareness.
6.1
Results
The detection solutions based on the choice of the HSV color model proved to both detect the road in
images where sunlight did not interfere. The coverage of the original road compared to the solutions showed
that the two methods was both above 85% and resembled the correct shape of the road. From these detected
sections post processing created useful polygons with a small amount of points such that storing them for
later use was feasible.
The detection for increased sunlight showed that the coverage dropped in some situations for the 4D
histogram method but the HSV thresholding remained very close to the ground truth. The results shown for
the second image for comparison between the methods had a notable difference in coverage of more than
36%.
The image with the largest amount of spotlights, over exposing the image made the detection of the road
very difﬁcult for the 4D thresholding but the HSV thresholding detected the road with above 90% coverage.
From these results it can be concluded that for purely segmenting the uniform road the ﬁrst method have the
highest detection rate in any form of lighting.
When the two methods was used to detect crossing roads unto the main road, the result was the com-
pletely the opposite. Here the HSV thresholding had difﬁculties ﬁnding the different colored road, since it
was made from a brick and did not have the same Saturation. The 4D histogram method on the other hand
detected the three different scenarios for the crossing road correctly without any issues.
The HSV histogram threshold was only explored for road detection because the creation regions for
detecting other objects required several assumptions about the general color of the objects. The 4D histogram
thresholding segmented the image into a number of clusters equal to the amount of prominent peaks in the
histogram. With this method the clustering of the images was based on the position of the pixels in the
3D HSV space and did not require these assumptions. The clusters that was found was then classiﬁed into
speciﬁc objects which then was based on color assumptions, but was not restricting the sections which could
be detected.

CHAPTER 6. CONCLUSION
The two other objects which was searched for in clusters was the sky and the trees, where the sky was
very simple to test. The overall detection of this object was found to be successful and very few false
negatives was found. The pixels with the largest Value were often clustered into the same group, since they
was positioned in the same area in the HSV space. This classiﬁed them as the sky but since the detection
was based on color meant that reﬂections of light and other light sources would be wrongly detected.
Detections of the clusters which was trees was based on assumptions for the color just as the sky
classiﬁcation. The tests showed that clusters often included other objects which was segmented into the same
sections and interfered with the cluster centre. The trees was often detected as low probability and a few was
high probability. This was due to the threshold which was found by averaging the manually detected tree
clusters and creating a threshold for the two probabilities.
The method for detecting the landmarks on the path was seen to ﬁnd the signs when the red was present in
the image. Although some trees in the correct lighting was detected as the sign but the area below eliminated
most of the incorrect detections. The tests also showed that it was needed that the sign to be detected, was
larger than the morphology structure element of a 5x5 square.
As for the 4D histogram thresholding, the over exposed images had areas which was dark enough to be
detected as the pole of the landmark. This is due to the assumption that the area just below the sign that was
found had a speciﬁc color and not the particular shape of a pole.
6.2
Future works
In this section the presented solutions enhancements and possible improvements are described.
• Restricting the detection of cluster peaks. The current restriction for ﬁnding clusters close to
already found peaks in the 4D histogram could be improved with a more strict removal of smaller
peaks.
• Thresholding for tree clusters. The thresholds which was found for the tree classiﬁcation could be
improved by using a larger amount of clusters for ﬁnding the average cluster coordinates.
• Land mark detection. The method for ﬁnding the pole of the landmark could be improved by
inspecting the structure of the detected pole. The shape could be used to discard wrongly detected
landmarks.
• Post processing of the polygons. In order to relate the polygons to the position of the robot, rotational
matrices would have to be applied. The images that are taken can be referenced to the camera
coordinates by a transformation matrix, projecting them onto a plane in front of the camera. From
these points geometry and rotation can then be applied in order to relate them to ground which the
robot is situated on. With these coordinates the world representation can be found for the polygons.
• Creation a map of detected objects. From the projected polygons and landmarks a map structure
could be created and saved such that the robot could use it for later referencing of detected objects.
