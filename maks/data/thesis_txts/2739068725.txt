Advanced Control for Magnetic Levitation System
Abstract
This master project is an extension on previous work done on a ECP M730 maglev
system.
The mathematical model of the system has been improved to be able to
handle SISO, SIMO, MISO, and MIMO cases.
The simulink model of the system
has also been improved thanks to the improvements of the mathematical model. Three
different nonlinear controllers have been designed and tested on the system in 4 different
scenarios, both in a simulink and on the physical ECP test rig. The controllers were
compared to each other and to conventionel PID-controllers tuned to the system on five
different performance metrics, which are settling time, maximum absolute error, time
weighted error, integral of square of input, and number of parameters. The metrics show
that of the nonlinear controllers tested the feedback linearization controller generally
has the fastest settling time while the super twisting sliding mode controller generally
has a lower maximum absolute error and lower time weighted error. The PID-controller
was not able to be implemented on two of the scenarios, but of the remaining two all
the nonlinear controllers generally performed better than the tuned PID-controllers.

Advanced Control for Magnetic Levitation System
1
Introduction
To control various systems in the industry, the PID-controller is the standard practice
to use. The lineararity of the PID makes it intuitive to understand and put into practice,
and many tools has been made to assist in the tuning of a PID-controller to a linear system,
such as bode plots, pole-zero diagrams, and root locus. However, most systems in reality
are not linear. While an approximation around a linearization point can ease the calculation
and allow the use of the aforemented tools, the PID may not be able to adequitly control the
system when the output is too far from the reference and the linearization is no longer valid.
Nonlinear controllers allows control of the plant across the entire range, and so is able to
overcome a lot of the obstacles that conventionel PID controllers face, especially on highly
nonlinear systems. While the versatility of nonlinear controllers is a great advantage, they
are much more complex to design, and without the aforemented tools, they are much more
difficult to evaluate in a theoretical sense. Instead they need to be put into practice so their
performance can be compared, which this project aims to do.
The tests will be performed on an ECP M730 Magnetic Levitation System,[1] which is
a test rig that contains one or two magnets sliding on a vertical low-friction glass rod be-
tween two controllable electromagnets. Because of uncertainties in the mathematical model,
especially friction, and nonlineararity on the input exact control of the magnetic disks can
be difficult to attain.
In previous work at DTU, the constants for the sensors and actuators have been de-
termined, and a mathematical model of the rig has been developed and implemented in
simulink as a model.[2][3] However, this current model can only handle SISO configurations
of the model, and needs improvement before experiments of higher order can be conducted
on it. One of the goals of this project is to improve the current model to facilitate further
and more involved work on the test rig.
Another objective of this project is to design various nonlinear controllers for position-
ing of the magnetic disks. These nonlinear controllers will be the sliding mode controller,
feedback linearization controller, and super twisting sliding mode controller. They will be
tested in four different scenarios both in simulink using the improved model and on the
test rig. Each scenario will present a more difficult task for the controller. First scenario is
to simply follow a reference in an open loop stable configuration, second is to stabilize the
magnet in an open loop unstable configuration, third is to follow a reference in open loop
unstable configuration, fourth is to follow a reference in an open loop unstable configuration
while compensating for a disturbance. The performance of each of the controllers in each
of these scenarios will be compared to each other and to a conventional PID solution on
settling time maximum absolute error, time weighted error, integral of square of input, and
number of parameters.

Literature analysis
This project builds upon two previous projects conducted at DTU; one is a special
course conducted by Sigurd Engvik Rasch in 2020[2] where the author creates a basic model
in simulink for future use, the other is a master project done by Kun Du in 2021[3] where the
author deals with fault detection and developing a linear controller for the system. Another
important reference is the manual[1] which contains essential information for working with
the system.
2.1
The manual of the system[1]
Chapter 1 of the manual provides a short overview of the system and the manual itself.
Chapter 2 offers a description of the accompanying software, along with instructions on
how to physically handle the system. This includes topics such as cleaning, safety during
operation, storage of the magnets when not in use, and how to touch the magnets when
manually moving them. Chapter 3 is a short guide on how to start up the system along with
a self-guided demonstration. Chapter 4 contains instructions for the physical system, and
chapter 4.5 describes how the laser measuring sensors work, and that they are designed to
work up to 5 cm away. Many details about the laser sensors are proprietary. Chapter 5 gives
the full order nonlinear model of the system, and also simplified and linearized equations of
motion. The appendix contains links and references to various studies. In the instructors
part of the manual, 6.1i, there are coefficients for fitting curves to the sensor values output
by the sensors, which was used to perform the sensor fitting in previous work.
2.2
Sigurd’s report[2]
The focus of Sigurd’s report is to identify the system and create a suitable simulink
model of it which future projects can depend on. Most of the content useful to this project
is to be found in chapter 3, which also takes up the majority of his report. Chapter 3.1
makes a calibration for the sensors by experimentally measuring the output from the sensors
in counts and noting the corresponding position of the magnet and then fitting a curve to
the output by finding the best fit for the quotients in the sensor equation. This is done
both for the upper and lower sensor. It also found that the output from the sensors were
different depending on whether the built-in program from ECP or the simulink extension
were used, this was caused by a temperature compensation feature that is not included in
the simulink extension. After sensor calibration has been completed, the report moves on
to model the actuators in chapter 3.2.1, in which the chapter also states that the coil tem-
perature significantly affects the open loop response. He does not note how this may affect
closed loop performance. For the lower magnet the modelling is performed by inputting an
open loop voltage in counts and noting the calibrated results from the sensors. For the upper
magnet, the method is similar, but instead the stable equilibrium being noted the position in
which the magnet transitions from accelerating upwards to downwards is noted. The range
of validity is defined as 0 cm to 4 cm away from both coils. Chapter 3.2.2 makes a nonlinear
plant compensation, which acts as a simple controller that counteracts the nonlinearities

3
Description of the physical system
On Figure 1 is provided a descriptionary figure from the manual,[1] and on Figure 2 is
a picture of the specific maglev system used in this project.
The physical system consists of two coils with embedded sensors. The coils can provide
a magnetic force to one or two magnets that slide along a vertical glass rod between the coils.
Each coil has an associated LED that lights green when the coil is provided with positive
counts and red when the coil is provided with negative counts. The intensity of the light is
proportional to absolute value of the provided counts, and acts as a visual aid for the user.
Clamp screws allows the glass rod to be removed from the system to replace one or both
of the magnets. When only one magnet is attached to the rod (with its north pole facing
upwards) positive counts to the lower coil provides a repulsive force while positive counts to
the upper coil provides and attractive force, both forces result in lifting the magnet. When
two magnets are installed the second magnet will have its north pole facing downwards, and
thus the forces on the second magnet from the actuators will be reversed. The laser sensors
provide non-contacting position feedback and allows reading the positions of the magnetic
disks. The system is connected to a control box that handles I/O like the DAC’s for the
coils and the ADC’s for the sensors. The system has a built-in ruler to manually read the
position of the magnet, which can be adjusted using the ruler clamp screws. A storage pin
with a screw is also provided to hold any unused magnets.

a simple version of the system with only one magnet installed and building it up from there.

Control methods
One of the objectives of this project is to test different control methods and compare
them against each other, these control methods are all presented here.
6.1
Sliding mode control
Sliding mode control is, at its core, the idea that two bad, e.g. unstable. controllers can
be combined to make a good controller. A good example of this is shown on Figure 8. This
is done by defining a sliding manifold within the system that the controller tries to reach
and follow. A classic example of such sliding manifold is s := x + ˙x = 0 for a stabilizing
system. The controller starts in what is called reaching mode, where it tries to get the
system states onto the previously defined sliding manifold. After that, the controller stays
in sliding mode, where it rapidly switches back and forth between the two sub-controllers
in order to stay on the sliding surface, as illustrated on Figure 9. It stays in sliding mode
until given a new reference to follow or the system is perturbed enough to push it sufficiently
away from the sliding manifold. In practice sliding mode controllers generally need to be
modified to introduce a gradual shift between the two sub-controllers for them to work
properly.[14] Doing so, however, means a loss of precision. This loss of precision may be
worth it in exchange for less strain on the actuators and thus increasing their lifetime, which
is also generally preferred in the industry. One such modification will be used in this project.
Instead of sign(s), the function sat( s
Φ) will be used for this controller. The value Φ sets the
saturation range and can be adjusted depending on the needed precision of the controller.
The relation sign(s) = sat( s
Φ) will hold true outside the saturation range. A continuous
shifting function like this cannot guarantee asymptotic stability within finite time, but can
guarantee boundedness.[15]
6.2
Feedback linearization
Feedback linerization for nonlinear systems is very similar to full state feedback for linear
systems, in that certain eigenvalues is imposed on the system in order for it to achieve a the
desired behavior. For nonlinear systems there is an additional hurdle that the nonlinearities
also needs to be cancelled by the controller in order to induce the desired linear dynamics.
Feedback linearization ideally requires that the nonlinear system is perfectly known, other-
wise the linearization will not perfectly cancel out the non-linearaties but will also introduce
more. In the best and most realistic case the result will be boundedness but not asymptotic
stability, while in the worst case the result will be instability. There are two subtypes of
feecback linerization: Input-output linearization and full-state linearization. Input-output
linearization only concerns itself with controlling a particular output of the system, and not
the system in its entirety. One way to create and input-output linearization controller is to
differentiate the output until the input term appears, counteract the found dynamics, and
then impose the desired behavior. While easy in theory, this can result in a very complex
controller for higher order systems, which will not be easy to derive by hand. In addition,
input-output linearization may not capture all the states within the system, leading the

6.3
Super twisting sliding mode controller
The super twisting sliding mode controller is a development on the regular SMC where
the control signal is integrated based on the sign of the error.
This integration allows
increased robustness against constant disturbances, and may even be able to account for
model inaccuracies when designing the controller. The architecture also somewhat eliminates
chattering as the controller by default is doing a square root on the absolute value of the
error and multiplaying with the sign of the error rather than just taking the sign like the
regular SMC. In addition to that, the integral action will cause a smooth change in control
action, rather than an abrupt change that the regular SMC does.

Scenarios
The scenarios that the nonlinear controllers are tested on in this project are presented
here. When implemented on the test rig, for the first scenario the magnet will start resting
on top of the bottom coil, for the rest of the scenarios the magnet will be held up using the
blunt end of a pencil, since the upper coil cannot produce a strong enough magnetic field on
its own to lift the magnet from its lowest position.
8.1
Simple oscillation
This scenario uses a single magnet setup and the lower coil. The goal of the scenario
to make the magnet follow a simple sinusoidal reference with a frequency of 1rad/s. As the
system is open loop stable in this configuration, it should not be hard to get the magnet to
follow the reference. This scenario is intended as the initial test to test of the design method
of the controller holds up on the real system.
8.2
Upper hold
This scenario uses a single magnet setup and the upper coil. Switching from an open
loop stable configuration to an open loop unstable configuration presents a challenge to the
controller, since fast switching is needed to move the magnet from one position to another.
In this scenario, the goal is not to follow a reference, but just to move the magnet to a certain

height and stabilize it once there. The simpler reference also makes it easier to troubleshoot
the controller should any problems arise.
8.3
Upper oscillation
This scenario uses a single magnet setup and the upper coil. As with the simple oscil-
lation scenario, the goal is to make the magnet follow a simple sinusoidal reference with the
same frequency as that scenario. This is considerably harder than the simple oscillation sce-
nario since this configuration is open loop unstable, so the controller needs to make constant
adjustments to make sure that the magnet neither falls down nor shoots up while following
the reference.
8.4

This scenario uses a single magnet setup and both coils. This scenario is very similar
to the upper oscillation scenario where the magnet follows the sinusoidal reference. The key
difference in this scenario is that the lower coil is acting as a disturbance, meaning the driving
coil has to do additional work to compensate for this disturbance and keep the magnet close
to the reference. This scenario is intended as a test of how well the controllers handle an
environment where disturbances on the controlled object are present.

9.4
Oscillation compensation
As this scenario is very similar to the upper oscillation scenario, it will use the same
controller design as that. However, instead of a constant count of 0 being applied to the other
coil, it is instead decided that it generates a disturbance that the driving coil has to compen-
sate for. The disturbance generator consists of a uniform random number generator input
into a lowpass filter added to a white noise generator. This generator is shown on Figure 29.
The reason for this design is that it combines a larger disturbance that changes less often
along with smaller rapidly changing disturbance. The lowpass filter is there to make sure the
coil doesn’t change from a very large value to a very small value too quickly. The uniform
random number block is configured to range from -14000 to 14000 and with a sample time of
1 second, the white noise block has a noise power of 10000 and a sampling time of 0.1 second.

As mentioned in earlier chapters, a lot of the design choices for this controller was arrived
at by experimentation on the real system. The considerations from that will be explained
now.
Initially, the controller would have ω0 = π and all its gains adjusted hereafter. Coulomb
friction was included in the controller design and there was no lowpass filter before the fric-
tion estimation. During testing, this setup resulted in either the coil not supplying enough
force for the magnet to rise up to meet the reference, or the coil forcing the magnet to
oscillate up and down, slamming the magnet down on the floor of the mechanism multiple
times a second, way more often that it should by the given ω0 for the controller. During
testing the control box was turned off prematurely much of the time in order to minimize
damage to the equipment, and thus very few graphs with the results exists, as it would be
irresponsible to conduct multiple tests that almost certainly would damage the system just
to learn why the controller behaves the way it does. There was one test where the duration
was limited to 5 seconds and the control effort artificially limited to not be below 0 for the
purpose of collecting data of the problem and showing it to the supervisor of this project.

it to slam down with more force than if it was in free fall. This constraint does not show up

The first breakthrough was placing a lowpass filter before the friction estimation. this
attenuated the noise that was exacerbated from the derivative enough for the controller to
be able to work with it. While this meant the controller was no longer going haywire, it also
showed that the current controller did not have enough effort to lift the magnet from the
floor of the machine, and so ω0 was gradually increased until a satisfactory tracking result
was found. During this time, the coulomb friction was also turned off and back on again to
investigate its effect on the tracking. At ω0 = 4π an interesting difference of the effect of
the coulomb friction was found, this effect is shown in Figure 40. Looking at the reference
tracking plots, one can see that the controller with coulomb friction compensation is not as
smooth as without coulomb friction compensation, indicating that the current model of the
coulomb friction overestimates the coulomb friction in the real system, and thus needs to
slow down when trying to reach the reference. On the error plots neither controller manages
to consistently stay within 1 cm of the reference, and while the controller with coulomb fric-

Evaluation of controllers
11.1
Conventional linear controllers
To establish the usefulness of the nonlinear controllers created in this project, they need
to be compared to conventional solutions, i.e. PID-controllers. Two different PID-controllers
was tuned to the simple oscillation scenario and upper hold scenario respectively. Attempts
to tune the PID-controller for the upper oscillation scenario was not successful. Already
at this point one advantage of nonlinear controllers tested in this project is clear: Their
versatility and adaptability.
The PID-controller needed different a tuning depending on
whether is was the lower or the upper coil it was applied to, which was not the case for
the nonlinear controllers, where one could simply swap u1 with u2 and y1 with −y2 and the
controller would still work adequately. This shows the nonlinear controllers ability to still
perform the same, or at least very similar, task from a different position, at least in this
case. The nonlinear controllers also managed to follow a reference in a nonlinear plant that
is unstable in open loop, which the conventionel PID-controller did not manage to do. The
PID for the simple oscillation scenario was tuned using the Ziegler–Nichols method[17] while
the PID for the upper hold scenario was hand tuned, since it was not possible to find a
proportionel gain where the magnet would enter stable and consistent oscillations.

the task is stabilizing at a specific position rather than following a trajectory.
In terms of settling time, the feedback linearization controller is the fastest controller
in 6 out of the 8 instances, with the SMC being faster on the real system in the upper hold
and oscillation compensation scenario. This shows the speed at which a well-tuned feedback
linearization controller can reach the reference, even if the system is not perfectly known.
Interestingly, the STSMC is faster than the SMC in all scenarioes in simulation, but on the
real system the SMC is the faster controller.
The STSMC has the lowest value both for maximum absolute error and for time weighted
error in 6 out the 8 instances, only beaten by the feedback linearization controller on the real
system in the simple oscillation and upper oscillation scenario. The STSMC is very good at
minimizing tracking error over the entire run, due to the integration term included in the
control loop.
The integral of square of input is comparable between all of the controllers in each sce-
nario. While one controller may have have a lower-end value in one scenario and a higher in

Future Work
The sensor noise that was amplified by the derivative proved to be a point of great
trouble during this project, especially in regards to designing the nonlinear controllers. This
was solved by using a large λ for the SMC and STSMC and a lowpass filter for the feedback
linearization controller and the resulting controllers managed to follow the reference without
too much trouble. However, using numerical differentiation is not always possible or practi-
cal, so the controllers should also be tested using an observer, such as the one from Sigurd’s
report.
The current sensor and actuator models are only calibrated with a limited physical
range and limited counts, meaning there is a high chance neither will be valid when outside
of this range. While this decision may be valid for the sensors, given that they are made to
measure at a close distance, the coils will still affect a magnet that is outside of sensor range.
The edges of validity, both in terms of distance and temperature, for both the actuators and
the sensors should be investigated to assess how to best improve the models. As seen in the
oscillation compensation scenario, the lower coil can still affect the magnet while it is far
outside the range at which the coil can levitate the magnet. Getting measurements for the
force applied out at those ranges can help design controllers for MISO configurations and
help troubleshoot MIMO configurations. If incorporating data from ranges further away
leads to the model being invalid for levitation distances, consider implementing different
models for the coil depending on the height of the magnet, and implement gain scheduling
to change between the models as needed.
While the simulink model of the system has been improved greatly in this project, there
is still room for further improvement. First of, sensor noise could be added to the output of
the model, in order to anticipate the effects the noise will have on the controller and make
adjustments before applying the controller to the real system. This is not as simple as just
adding a Gaussian noise block to the model, as the sensor resolution decreases the further
the magnet is from the sensor. The consequence of this is that sensor noise has a greater
impact on the distance calculation the further away the magnet is from the sensor. This
can probably be extrapolated in simulink by converting the distance to its would-be counts,
the applying the noise to the counts, before returning the distance as it would have been
calculated from the sensor data. All this is of course assuming that the sensor noise on the
counts is not affected by external factors.
Another improvement to the simulink model is to further investigate the friction in the
system. As it stands, both the viscous friction and coulomb friction models are insufficient
in completely explaining the friction in the system. Adding static friction or the stribeck
effect or other friction modeling effects to the friction model may improve the accuracy.
As there could be made improvements to the simulink model, so can there be made im-
provements to the model containing the driver. First of saturation blocks should be added
Page 52 of 58
==PAGE:59==
Advanced Control for Magnetic Levitation System
at the input to the driver in order to avoid integer overflow and cause the actuators to un-
expectedly switch from a maximimum positive value to a maximum negative value or vice
versa. Another improvement that is a bit more involved is to is to expose the temperature
output to the rest of the simulink environment. This accomplishes two things: First is that
it allows direct detection of the temperature inside the coil and to stop the experiment be-
fore the coil becomes too hot so that the internal safety measures trigger, since they are
much more involved to deal with. Of course this require extensive testing of the system in
order to verify at what temperature counts the internal safety mechanism triggers. This
can allow faster resets of the system and greater control greater control when experimenting
with controllers that require high input counts and thus generates more heat. Secondly, it
can allow the creation of compensation algorithms to compensate for the heat affecting the
output counts of the sensors and the magnetic fields of the coils. This is already part of
the built-in ECP software, but not part of the simulink environment.[2] This may require
testing to know the exact effect, but if it is possible to extract this information from the
ECP software it will be significantly easier to create those algorithms.
As there were no scenarios in which both magnets where in use, it is unknown weither
the mathematical model or the simulink model is still valid for SIMO or MIMO configura-
tions. The coefficients for the magnet to magnet force should be verified before commencing
such operations. Theoretically, the simulink model should be able to accurately model two
magnets installed on the system, so given that the interaction is verified one should be able
to test SIMO and MIMO scenarios on it.
Since the different controllers in this project have different advantages and disadvan-
tages, one possible step for improvement is to implement gain scheduling between multiple
controller types. For example, have the feedback linearization controller active during startup
and the transient for a faster settling time and then switch to the STSMC to reduce steady
state error.
There is limited variation in tracking references in this project, only sinusoidal and
constant references have been utilized. This means that there are no instances where the ve-
locity or acceleration of the reference is either undefined or infinite. A possible consequence
of this is that certain edge cases has not been accounted for and the controllers may exhibit
undesired behavior with certain references. The controllers should be tested with different
references, e.g. trapezoid, to validate the controllers’ abilities to track different kinds of
references.
The results from the test rig does not perfectly match the results from the simulations,
and while this is to be expected, it can make it difficult to discern whether the simulation or
the experiment is more representative. This is because while the simulation always returns
the same result, there might be mistakes in the model, but the test rig is susceptible to
outside influence and random chance. Running multiple experiments with the same scenario
and controller can give more data to work with and better gauge the performance of the
controllers. Even so, statistical analysis can be done on the current results to find significant

interactions between different variables to help decide next steps.

Conclusion
There is many different project objectives completed in this project.
An improved
mathematical model has been created based upon previous work on the ECP 730 Magnetic
Levitation System, this new model is able to handle both lower and upper SISO configu-
rations, and also MISO, SIMO, and MIMO configurations of the system. This model has
also been implemented in the simulink environment for use in simulation. A different model
containing the driver for the maglev system has also been created to streamline bringing
simulink models using the simulated model onto the real system.
This improved model
along with the models created in simulink will be beneficial to future projects based on this
magnetic levitation system.
Three different nonlinear controllers, a sliding mode controller, a feedback linearization
controller, and a super twisting sliding mode controller, have been designed for positioning
of the magnetic disks. Inititial values where calculated in implemented in simulink with
success, but applying the same controller to the test rig rarely lead to instant success. Pa-
rameters for each controller had to be adjusted for the controller to successfully work on
the rig. These adjustments mostly consisted of increasing λ for the SMC and STSMC and
adding a lowpass filter to the feedback linearization controller, showing how to compensate
for a noisy derivative when designing one of these controllers for a plant. Of course the
simulink model is not perfect either, and this shows that nonlinear controllers can still work
well despite an imperfect model.
Four different operation scenarios have been defined for the controllers to be tested on
the system, which are simple oscillation, upper hold, upper oscillation, and oscillation com-
pensation. All of the nonlinear controllers have been tested both in simulink and on the test
rig, totalling 24 different runs of nonlinear controllers included in this report. Two different
PID controllers was also tuned to the test rig, one for the simple oscillation scenario and one
for the upper oscillation scenario. It was not successful to find tuning values for the upper
oscillation and oscillation compensation scenarios. With the PID controllers tested both in
simulation and on the test rig, a total of 28 runs of of various controllers is included in the
report.
The fact that attempts to find suitable tuning values for all scenarios shows that while
there are situations where tuning a PID controller is not possible or feasable, designing a
working nonlinear controller is still a viable option. In addition, the same nonlinear con-
troller could be moved to another part of the system and still function adequately while just
needing some signs to be flipped, the PID controller needed to be tuned again ground up in
the new position.
Five different performance metrics has been defined to evaluate the performance of the
Page 54 of 58
==PAGE:61==
Advanced Control for Magnetic Levitation System
controllers in the various scenarios, these performance metrics are settling time, maximum
absolute error, time weighted error, integral of square of input, and number of parameters.
The nonlinear controllers was compared to the conventional PID controllers and against
each other according to those defined performance metrics. It was found that there was no
significant difference in integral of square of input between the controllers. The feedback
linearization controller generally had the fastest settling time, and the super twisting sliding
mode controller generally had the lowest maximum absolute error and lowest time weighted
error across the scenarios. The PID controllers were generally worse in all the defined metrics
than any of the nonlinear controllers designed in this project, except for the integral of square
of input as mentioned.
