Abstract
As ADAS technologies become more and more popular in the automobile industry,
drivers are slowly adapting to vehicles that get more and more automated. A fully
autonomous vehicle needs to be able to visualize and understand its surroundings in
order to make the right decisions and drive safely. The detection and understanding
of the surface where the vehicle can drive consists in a basic information for the ve-
hicle and is crucial for decisions making. Therefore the segmentation of the road is a
problem that is looked into with high interest for autonomous vehicles. Such vehicles
are often thought to be only urban car for individual users, but other applications
can be concerned by self-driven vehicle. In the military, drones where first invented
for information gathering and exploration. Autonomous military vehicles could do
the same on the ground. The ground type is therefore often different as the military
vehicle drives on unstructured dirt road called ruts. Subject to diverse visual modifi-
cations like lighting, shadows and ground type, the driving surface becomes variable.
Classical image processing methods are limited in these situations and a lot of differ-
ent algorithm would be needed for all the different cases. Deep learning algorithm
can overcome this diversity and generalize it’s understanding of a road or a rut. In
this thesis, two strategies are explored and a Convolutional Neural Network for each
of them is trained. The dataset is prepared differently for each approach. The final
model achieves a 84.4% Dice score and a 78.1% IoU score on the test set from the
dataset used for this thesis.

CHAPTER 1
Introduction
Altran is a 37 years old french company that is currently leading the Engineering R&D
consulting field. Being historical partners of significant french car manufacturers like
Renault and Groupe PSA, Altran is nowadays focusing on the development of a
learning autopilot for vehicles. As part of the development of this autonomous car,
the vehicle’s External Perception (EP) is crucial. The EP aims to generate a local
map of the environment (road, vehicle, pedestrian and sign detection) by taking as
an input raw data from different sensors such as cameras, radars, lidars, etc.
This thesis is part of Altran’s AV2A project and focuses on the conception of path
detection model for vehicles driving on ruts; a non urban area where the road is not
clearly defined with asphalt, but mostly by a vehicle’s wheel tracks. An example is
given in Figure 1.1.
Figure 1.1: Example of a rut
This type of road is regularly used by military vehicles and an autopilot fit for
this unconventional environment could be useful in an exploration context. A military
vehicle could autonomously drive around an unknown or a dangerous zone and gather
data without putting a human life at risk. Although, the detection of rut could also
be of value for general public self driven cars. It is reasonable to think that, in the
future, many users of self driven cars will not necessarily live in the city, and may
have to use a dirt road to get to their home.
With a camera as only sensor, the goal of this thesis is to train model using
deep learning techniques and architectures in order for the self driving vehicle to
understand where it is allowed to go.

2.1.1
The Principle
In computer vision, semantic segmentation partitions an image into coherent parts.
Pixels from the same object should be put in the same class. This exercise is highly
useful in the context of autonomous vehicles as it gives an understanding of the
vehicle’s surroundings. Large-scales projects such as the Cityscapes dataset [Cor+16]
aim to improve semantic segmentation performances for autonomous cars by offering
an open source manually and precisely annotated dataset. An example of the data
from Cityscapes is given in Figure 2.1.

trians”, ”buildings”, ”road signs”...
In the context of this thesis, the surroundings of the vehicle is mainly the rut on
which it drives on and the surrounding nature (trees, grass, rocks, etc...). There-
fore, two classes only are needed: ”rut” and ”background” which includes everything

2.1.2
The Method
Semantic segmentation is an image based concept which is why CNN is the type of
network that is used. In this section, the theory behind a basic CNN will be explained,
then the architectures used for segmentation will be presented.
Introduction to CNNs
In computer vision, object detection or segmentation is done based on the identifica-
tion of distinctive characteristics. For example, if one wants to build an algorithm
capable of detecting whether or not an image contains a dog, the algorithm will need
to detect features like fur color, head, body and tail shape, ears, etc...
However,
defining these features in a deterministic manner is highly restrictive. What if the
fur color is a crucial feature and the algorithm is fed an image of a dog with a fur
color it’s never seen before? What if the head shape is important and the dog in the
picture is hiding it’s head? Classical computer vision methods for problems similar
to the dog recognition are either too simple or too specific. A good way to generalize
the recognition would be to learn what features are important to focus on. This is
where neural networks revolutionizes computer vision.
Training a neural network on images require a convolutionnal neural network (CNN).
In the context of road segmentation, a CNN outperforms classical image processing
algorithms when it comes to detecting the road in different light and weather con-
ditions. Classical methods can be competent but are not able to handle real time
changes that a road could be affect by. Deep learning convolutional networks repre-
sent a robust real-time technique for the generalization of road detection. They are
composed of several layers of image processing and deep learning techniques that are
going to be described in the following sections. The different layers and elements of
a basic CNN are presented in Figure 2.2.
Figure 2.2: Basic CNN architecture [Adv19]

Convolution Layer
When working with images, a basic statement would be that a pixel is more strongly
correlated to it’s neighbors than by pixels that are far away. Indeed, there is a higher
probability for pixels to be part of the same object if they are neighbors.
From
this status, it becomes understandable that CNNs are based on the analysis of the
influence of nearby pixels. This work is done with the use of filters. Filters are kernels
of weights with a size determined by the user (usually 3x3 or 5x5). Filters move all
over the image with a step called a stride (a stride of 2 means that the filters moves
2 pixels at a time) and performs a convolution to generate a value. This convolution
is represented in Figure 2.3. The generated values form a new image called ”feature
map”.
Figure 2.3: The 2D convolution operation [Cor18]
Different weights in the filter will create different feature maps. The network will
learn to adjust the weights of these image characteristics in order to make a prediction
that is closest to the reality. Although this is only the first step in the convolution
network. One convolution is usually not enough for the model to correctly learn. More
convolutions are needed and before that, the features maps need to be adjusted.
Pooling Layer
The pooling layer is a type of down-scaling of the feature map in order to reduce the
computational power needed for the convolutions. Also it helps extracting dominant
features that are not subject to rotation and position changes. For example, a road
will always be at the bottom of the image. Similar to the convolutional operation, a
filter travels through the image and generates another feature map. There are two
types of pooling: Max Pooling and Average Pooling.
Max Pooling generates the
new feature map by taking the maximum value of the filter whereas average pooling
outputs the average value of the filter. Depending on the stride and the size of the

output feature map will be a fourth of the size of the input as shown in Figure 2.4.

Activation Layer
The activation layer simply consists of a transfer function that determines the acti-
vation of the output of a filter. The most famous activation function, the Rectified
Liner Units (ReLU), filters out the negative values in the feature map that could be
generated by the convolutions. The ReLU function is usually used after max pooling
layers and is shown in Figure 2.5. The sigmoid is another activation function that is
usually used as a classifier at the end of the architecture. It brings output values to
a probability between 0 and 1 as seen in Figure 2.6.

Global Functioning
When zooming out of all these technical and mathematical aspects, the segmentation
by deep learning process can be popularized: The neural network is fed with an image.
The images goes through all the different convolutional layers also composed of max
pooling operations and ReLU activations. When at the bottom of this encoder, the
last activation function classifies each pixel. The resulting mask is compared with
the true ground which is the annotated mask that represents the actual segmenta-
tion, the one that is aimed for. The comparison yields an error and this error is
then propagated backwards in the network in order to update the weights at each
layer accordingly. The weights are updated with the help of gradient descend that
guides the weights towards an optimal value that will minimize the cost function as
illustrated in Figure 2.7. Once the weights are updated, a new image goes through
the network and this iteration operates until all of the training data has been seen;
at this point the network has been trained for one epoch. Training on several epochs
is essential as the model can’t converge after only seeing the data once.
Figure 2.7: Loss function optimization with a single weight coefficient [Ras19]

2.2.2
The U-net used in the POC [RFB15]
The U-net architecture, presented in Figure 2.11, was created for semantic segmen-
tation of cells. A new type of architecture was introduced using de-convolutions to
reconstruct the image. De-convolutions, also called up-convolutions or transposed
convolutions increase the output size and thus increases the resolution of the output
while keeping the information resulting from the previous convolutional layer. This is
done by zero-padding around each pixel and then performing a standard convolution
as shown in Figure 2.10.
Figure 2.10: Illustration of the up-convolution principle [DV16]
==PAGE:24==
10
2 Problem Analysis
The architecture of the U-net is composed by two main parts: a classical convo-
lutionnal network followed by an up-convolution part. In addition, high resolution
features from the contracting part are cropped and concatenated to the corresponding
feature maps of the expanding part in order to add context and improve localization.
Cropping is needed since the convolutions are done without padding, making the fea-
ture map lose border pixel at every convolutions. In the expansive path, the number
of feature channels is divided by two at every step. In the end, a 1 by 1 convolution
layer is applied to map features to either one of the classes: cell or background. In
the paper, a pixel-wise softmax is used in this last layer to activate pixels to either
1 (cell) or 0 (background) depending on their value. An example of their result is
shown in Figure 2.12.
Figure 2.11: The U-net architecture [RFB15]

2.3
The Dataset
The dataset was though for military vehicles and therefore was taken in a military
base. It is composed by 40GB of Full HD and 15 fps videos (1920x1080p) of ruts
and dirt roads explored by a military vehicle. The camera is placed on vehicles’ hood
and films the front of the car. On the examples showed in Figure 2.13, it can be
noted that the rut can be of slightly different colors, shapes, and shadows sometimes
change the appearance of the drivable zone. However, the dataset is limited since it
was recorded on 2 different days with very similar weather conditions: sunny with
few clouds. It is reasonable to believe that no model trained on this dataset will be
able to generalize dirt road detection. Therefore, the model trained in this thesis will
be evaluated on data coming from this dataset.
In order to train the model on images, the dataset was made on frames from the
videos taken every 60 frames, that is every 4 seconds.

4.4
Batch Size Analysis
The batch size is the number of images that the model needs to work through before
updating it’s weights. This means that this parameter determines the behavior of the
gradient descent:
• Batch Gradient Descent when the batch size is the size of the training set
• Stochastic Gradient Descent when the batch size is 1 image
• Mini-Batch Gradient Descent when the batch size is between 1 and the size
of the training set
The batch gradient descent calculates the error after each image of the training set
but only updates the model weights after all of the training set has been seen. In other
words, the model is only updated after each epoch which requires less computational
resources. It also results in a stable convergence since the update is always made
on the same data: all of it. However, the implementation of the training algorithms
usually requires all of the images from a batch to be available and loaded in memory.
Also, since the model is not updated frequently, the training can become very slow
depending on the size of the dataset. The stability of the training could turn into a
drawback and get the model stuck on a local minimum.
The stochastic gradient descent is the other extreme. The model is updated after each
image. Since the updates are frequent, the training can converge faster depending on
the dataset and it’s noisy update process can get the model to avoid local minima.
On the downside, it is harder for the model to converge and stay close to the true
minimum and the frequent updates require more computational resources.
A trade-off between the robustness of stochastic gradient descent and the efficiency of
the batch gradient descent emerges and introduces the mini-batch. The mini-batch
is a portion of the training data such that:
Batch Size x Number of Batches = Training Set Size
Updating the model after each mini-batch allows the model to avoid local minima
while benefiting from the computation and memory efficiency.
In the blog post from [K19], a study on the size of mini-batches is made on a
model trained for image classification. It is shown that there is an optimal batch size
between 1 and the size of the dataset that yields a most accurate model. Therefore,
the mini-batch gradient descent will be used to get the best model possible.

on a small and specific dataset. The MODEL NAME model performs well on the test
set that is close to the training and validation data. The dataset used for training and
testing the model is too restricted to hope for a generalization of the segmentation.
The model is only good for this particular military base. In order to generalize the
model, data from other places and during different times of the day, year, etc... would
provide the deep learning algorithm with more cases and situations from which it will
learn.
Also, merging a 2D image dataset with data from different sensors would add dimen-
sions to the dataset. For example the use of stereo cameras or lidars would add 3D
information that could be valuable for the road cut case and the vehicles understand-
ing of it’s surroundings.
Batch Size Optimization
In section 4.4, the importance of choosing a suitable batch size was raised. With the
augmented dataset, it was unthinkable to use batch gradient descent since the dataset
was too large for the GPU. Also, a stochastic gradient descent would have been too
long to train. Naturally, as explained in section 4.4, the mini-batch option was used.
The batch size was chosen to be 25 as it was handled by the GPU and the training
set could easily be divided by 25 (107 batches of 25 images). Time constraints did
not allow an optimization of this parameter that can drastically change the training
process and the model’s accuracy as experimented in [K19] where it is shown that
Changing the batch size influences the model accuracy on the test set. It is also
stated that batch sizes should be a power of 2 because of the implementation of the
matrix operations libraries. Therefore training models with batch sizes of 16, 32, and
64 could output a more accurate segmentation model.

pre-trained network. This neural network should be trained on a relevant dataset
and serves as a starting point for the training.
