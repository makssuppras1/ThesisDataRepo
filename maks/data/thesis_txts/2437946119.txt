Summary
Every year over 1.7 million people is diagnosed with breast cancer.
When the cancerous growth is
to be determined a tissue sample from a tumour is extracted and sent through a process to turn it into
histopathology images. In the process it is stained to highlight Ki-67 antigen, which can give an indication
of the cancerous growth. This indication is known as the proliferation index which is the ratio between
positive tumour nuclei and total number of nuclei in a tissue sample. It is therefore essential to be able
to detect and classify each nuclei properly.
The current implementation to detect and classify nuclei by Visiopharm, App. 10004 - Ki-67, Breast
Cancer, relies on labelling each pixel in the image by a dictionary. Afterwards a watershed-based nuclei
segmentation is applied to separate nuclei. Even though the proliferation index from this method is
not bad, it is known not to detect all nuclei and also detect too many false positives. Visiopharm have
therefore been interested to see if it was possible to optimise detection of Ki-67 by using deep learning.
In this project the goal have been to take a deep learning method designed for image detection and
train on annotated histopathology(Ki-67 tagged) images. The method used is Faster R-CNN which is
designed as a uniﬁed network with an object detection part, Region Proposal Network(RPN), and an
object classiﬁcation part, Fast R-CNN. Both relies on a feature map from a pre-trained network. In this
project a VGG16 network has been the desired network to generate the feature map. The implementation
for the method that was used in this project originates from CNTK as an example script. The code has
been slightly modiﬁed for this use.
By doing pre-processing to the images it has been possible to maintain areas which have been annotated
and extract location of ground truths. Each ground truth nuclei have been given a boundary box of equal
size, 26 x 26 pixels, in images with size 256 x 256 at magniﬁcation x20.
After doing several experiments and changing parameters, the model was tested and give a mean Average
Precision(mAP) at 0.6010 and a recall of 0.8924. After examining the results it was decided to change
the Non-Maximum Supression(NMS) threshold for Intersection over Union(IoU), setting a threshold on
the class score, changing hyperparameters of the loss functions and changing the metric IoU threshold.
In the ﬁnal outcome the model performed with a mAP at 0.7568 and 0.7547 and still keep the metric.
By changing the metric the mAP and recall increased by 0.08.
When compared with Visiopharms
implementation the performance was very similar but by analysing visually it looked like Faster R-CNN
predicted better and also found some nuclei which should have been annotated as ground truth.
Overall Faster R-CNN performs very well on tumour nuclei datasets but in this project have not been able
to perform its best. This is due to the data quality where lacking annotated nuclei, missing information
about the size of each nuclei and true nuclei center location have been the most essentials. This would
need further study in the future.

Summary (Danish)
Hvert år får mere end 1.7 million personer diagnostiseret brystkræft. Når man ønsker at bestemme hvor
hurtigt kræften spreder sig, tages en vævsprøve af tumouren hvorefter den køres igennem in proces som
giver et histopathologisk billede. I processen indfarves vævsprøven til at synligøre Ki-67 antigenet som kan
indikere kræftens sprædningshastighed. Indikatoren er kendt som proliferationsindekset, som er forholdet
mellem positive kræftceller og samlet antal kræftceller. Det er derfor essentielt at kunne detektere og
klassiﬁcere alle kræftsceller korrekt.

watershed baseret segmentering til at adskille alle cellerne. Selvom proliferation indekset fra metoden
ikke er dårlig, detekter metoden ikke alle cellerne og ﬁnder for mange falske celler. Visiopharm er derfor
interresert i at se om det er muligt at optimere detektionen af Ki-67 ved hjælp af deep learning.
Målet med projektet har været at tage en deep learning metode designet til billede detektion og træne
på annoteret histopathologi(Ki-67 tagged) billeder. Den anvendte metode er Faster R-CNN som er et
sammensat netværk med en detektions del, Region Proposal Network(RPN), og en klassiﬁkations del,
Fast R-CNN. Begge dele bruger et feature map fra et pre-trained netværk. I det her projekt er et VGG16
netværk benyttet som det ønskede til at få et feature map. Implementationen af metoden som bruges i
projektet stammer fra CNTK som et kode eksemple. Koden er modiﬁceret en smule til dette formål.
Ved at bruge pre-processing på billederne har det været muligt at bevare de annoterede områder, samt
udtrække lokationerne af ground truths. Hver ground truth har fået en fast boundary box, på 26 x 26
pixels, i billeder af størrelse 256 x 256 pixels ved forstørrelse x20.
Efter udførelse af ﬂere eksperimenter samt ændring af parametre, var modellen testet og gav en mean
Average Precision(mAP) på 0.6010 og en recall på 0.8924. Efter at have undersøgt disse resultater blev det
besluttet at ændre på Non-Maximum Supression(NMS) grænseværdien for Intersection over Union(IoU),
sætte en grænseværdi for class scoren, ændre hyperparametrene i loss functionerne og ændre på metric
IoU grænseværdi.
I den endelige model præsterede den mAP på 0.7568 og 0.7547 recall ved normal metric. Ved at ændre
metric blev precision og recall forøget med 0.08. Ved sammenligning med Visiopharms implementering
var udfaldet at de præsterede nogenlunde ens, dog viste de visuelle sammenligninger at Faster R-CNN
fandt ﬂere korrekte celler som ikke var blevet annoteret i første omgang.
Generelt præsterer Faster R-CNN rigtig godt på tumour kræftcelle datasæt, men i dette projekt har den
ikke præsteret optimalt. Dette skyldes den manglende kvalitet i data hvor celler ikke var annoteret, mang-
lende information om størrelsen på hver enkelt celle samt center lokationen ikke var korrekt. Modellen
vil derfor kræve ydeligere forskning i fremtiden.

Chapter 1
Introduction
1.1
Motivation
Breast cancer is the single most common diagnosed cancer for women in the world where 1.7 million
is diagnosed each year[1]. In 2015, 64,718 people(alive) in Denmark was or had been diagnosed with
breast cancer[2]. Even though the number of deaths from breast cancer have decreased over the last
10-15 years, around 1140 people(primarily women) still dies every year in Denmark[2]. The faster breast
cancer is detected the higher is the chance to survive[3]. This is both in terms of when cancerous growth
is discovered but also how fast a diagnose can be given.
When a patient is under suspicion of having breast cancer they are sent to a mammography screening
(most common), ultrasound or a MRI scanning[4]. Based on one of these examinations a doctor can
determine if there could be cancerous growth(tumour) or not. If there is a tumour the next step is to
determine if it is benign or malignant cancer. This is done by taking a biopsy of the tumor where the
sample goes through several steps before it can be analysed[5]. In the staining phase hematoxylin is
applied as background contrast and the antigen Ki-67, which is highly present in cells with cancerous
growth, is tagged with DAB to highlight cells with cancerous growth from regular cells[6]. Based on
this analysis it can be determine the type of cancer, if it is benign or malignant cancer, the quantity
of positive and negative cancer cells which can give a proliferation index of the cancerous growth. The
analysis is carried out by pathologist either manually, by looking and counting nuclei, or assisted by
digital histopathology[7].
Letting a pathologist manually count nuclei is expensive and time consuming but most of the times very
accurate[7]. On the other hand there is more automated methods which quickly can detect and classify
nuclei but is yet not as accurate as pathologist[8]. These automated methods also requires a pathologist to
determine if the results are valid or not[7]. Current methods also requires independent methods to detect
the nuclei and classify them. Based on this it is suggested to use the state-of-the-art within image object
detection and classiﬁcation, Faster R-CNN[9], to detect and classify nuclei faster and more eﬃcient.

1.2 State-of-the-art
2
1.2
State-of-the-art
Today, the methods applied by Visiopharm in detection and classiﬁcation of nuclei is based on labelling
pixels by their intensity[10].
Hereafter a watershed-based nuclei segmentation is applied to separate
clusters of nuclei. This method is simple but can also easily lead to undetected nuclei which eventually
lead to a incorrect proliferation Index[6].
At Visiopharm their current implementation gives a good
proliferation Index but is lacking in detecting all the nuclei. Visiopharm is therefore seeking to ﬁnd a
better approach to detect and classify nuclei.
In the ﬁeld of histopathology detection and classiﬁcation of nuclei the main goal is often segmentation[11].
This is either as main output or as a way to make the detection of the nuclei. In general the state-of-
the-art within nuclei detection, for example Automatic CAD(Computer-Aided-Diagnosis) system[8] is a
combination of simple methods where a pre-processing is performed to extract some information, like
boundaries, where the nuclei afterwards is segmented with various ﬁltering methods, like Diﬀerence of
Gaussians(DoG). Lastly a classiﬁcation algorithm can be applied to the detected/segmented areas.
Other state-of-the-art nuclei detection relies on Gaussian Mixture Model, threshold or other ﬁltering
methods for detecting nuclei[12]. These methods seem to perform well but based on the fact that this
relies a lot on diﬀerent independent methods this could take some time and also be expensive to improve
since there could be a lot of parameters to tune[13].
When it comes to current state-of-the-art object detection and classiﬁcations with neural networks the
most popular methods included is Single Shot MultiBox Detector(SSD)[14], You Only Look Once(YOLO)[15]
and Faster R-CNN[9]. What is generally seen for most of these state-of-the-art object detection methods
is that most of them was uniﬁed networks with two tasks; object detection and classiﬁcation. Before these
uniﬁed networks was present the state-of-the-art object classiﬁcation networks relied on other methods to
do the object detection which could then be fed into the classiﬁcation networks[16]. When these networks
relied on other methods they became signiﬁcantly slower and less accurate[17]. Therefore state-of-the-art
object detection quickly became deep learning networks as well as the possibility to train the networks
to speciﬁc problems like face detection, objects on the road (autonomous cars) or nuclei detection[18].
These state-of-the-art object detection networks (or similar networks) have been participating in various
competitions and been some of the winning entries[19].
When comparing in speed and accuracy most of these networks is quite similar where performance and
choosing the "best" network is depending on the task it should perform (generally/speciﬁc detection,
segmentation etc.), data (availability, details, noise, quality etc.) and more or less what you as a user
prefer[20][21][17][22].
==PAGE:12==
1.3 Aim and outline
3
When it comes to combining nuclei detection and deep learning there have been an uprise the last couple
years, mostly with focus in segmenting nuclei. This was also the focus for Data Science Bowl 2018[23].
Combining Nuclei and deep learning have been tried by Eliezer Farrant Braz and Roberto de Alencar
Lotufo in 2017[24]. They trained a CNN to classify each pixels in an image to either be nuclei, cytoplasm
or background. Based on this classiﬁcation the number of nuclei can be determined.
1.3
Aim and outline
The aim for this thesis is to take state-of-the-art object detection and classiﬁcation algorithm, Faster
R-CNN[9], and optimise it for nuclei detection and classiﬁcation. This deep learning network will take
histopathology images as inputs, run it through the network and output the location of each nuclei and
their class(positive or negative). The network will train on images which is annotated by pathologist.
The ﬁnal outcome of this project aims to be a network which detect all nuclei and their class better and
quicker than current state-of-the-art methods.
The outline of this thesis is as follows: Chapter 2, Deep learning, will provide a brief description of the
history of deep learning with focus on object detection and classiﬁcation in images, introduction to deep
learning and its components, a review of the main method in this thesis, Faster R-CNN, with all its stages
along with its training application and metric. Chapter 3, Data, gives an introduction of transforming
a biopsy to a digital whole slide image, information about the whole slide images in general and deeper
analyse of the raw data along with decision making and assumptions of the data. Chapter 4, Baseline,
describes which decisions that have been made in pre-processing the data, the model and its parameter
but also about the hardware used for training. Chapter 5, Results, gives the results from the baseline
model, provide decisions on which path to follow for optimising the network and its results as well. A
comparison with the current solution from Visiopharm will also be made in this chapter. Chapter 6,
Discussion, will take some of the issues and advantages, of the model, data, other aspects and discussion.
Lastely a perspective of future work within this project and the ﬁeld itself will be provided. Chapter 7,
Conclusion, will draw the conclusion of this thesis.

Chapter 2
Deep learning
Since the early beginning of machine learning the concept of artiﬁcial neural network have been fascinated
the science community. The reason for their fascination can be found in the concept itself. Artiﬁcial neural
network is inspired by the biologic of how the brain functions. Neurons sending and passing signals to
each other and thereby create a certain output based on the input signal. This philosophy also create
the foundation for artiﬁcial neural networks but where neurons is put in layers[25].
Through the years these artiﬁcial neural networks have had their breakthroughs and stagnation af-
terwards. Before 2012 the most resent breakthrough was with the MNIST dataset(handwritten digits
recognition). With this data set it was shown that neural networks could actually perform just as well
as other state-of-the-art algorithms if not even better, this was also a simple data set of images. The
creators came with their own suggestion of a neural network, LeNet. One of its huge drawback was
computational time. Back in 1998 only the CPU was applied to do computations[26][27][28].
In 2012 the major breakthrough for neural networks was made and deep learning was invented. The
game changer consisted of multiple things; In 2010 a new image database was made, ImageNet. This
contained over a million images which all had an object from one of a 1,000 classes. This was then made
to a competition, ImageNet Large Scale Visual Recognition Challenge(ILSVRC), the challenge was to
recognise as many images as possible. At that time GPU’s was not that big of a deal and everything was
mostly run on a couple of CPU’s[29].
In 2012 parallelizing of code to run on GPU’s was made possible.
This made a huge impact in the
computation time. Instead of only having a couple of cores in the CPU to do all the calculations of the
training phase, it was now possible to use 1,000s of cores to do the calculations. This lead a group of
researchers at University of Toronto to try their luck on the ILSVRC10 with a invented neural network.
Their network ended up beating the winning team with an improvement of 10% in the error-rate. Also in
the ILSVRC12 their submission ended up winning also with more than 10% better error-rate. This newly
invented network, was one of the biggest(deepest) networks ever seen and is today known as AlexNet[30].
This contributed to the new term Deep learning which was a new way of talking about deeper neural
networks[27][28][29].
As the years past and the ILSVRC was better solved, new main challenges was added.
ILSVRC15
consisted of two challenges, object detection and object localisation.
This added a new layer which

To understand what a deep learning method does it is necessary to understand the concepts behind deep

2.1.1
Feedforward
The most simple and straight forward neural network is a feedforward neural network consisting of only
one Fully Connected (FC) layer(further deﬁnition see subsection 2.1.3). An example of such network can
be seen in ﬁgure 2.1.
Figure 2.1: A simple feedforward network with one hidden layer
In this network all input nodes, x, is connected to all neurons, n, in the hidden layer each with a weight,
w. The output from the neurons is then connected with all the output nodes once again with a weight
each. As seen each neuron/output node is connected with a bias term, b, which is the same for one layer.

2.1.3
Layers and terminology
When it comes to deep learning a lot of terminology is used and diﬀerent types of layers is used in
various networks. To clarify some of the terminology and layers a brief description have been given in
this subsection.
Fully connected layer
As mentioned in subsection 2.1.1, the most intuitive layer to think of is a Fully Connected layer. All
input is connected with all neurons in the ﬁrst layer which is connected to all neurons in the next layer
etc. The structure is illustrated in ﬁgure 2.1. More over how FC layers works is described in subsection
2.1.1.
In CNNs FC layers is often placed at the end of the network to help performing classiﬁcations. When
applying FC layers the number of parameters can easily increase drastically, why FC layers is kept to a
minimum[38].
Convolutional layer
A network only consisting of FC layers would be very heavy and need high amounts of computer power
to train such a network for object detection.
When operating with images it is not expected that pixels in the bottom left corner aﬀects features in
the top right corner. In images, features are expected to be placed locally and have local sensitivity[39].
Due to this FC layers often have weights which are irrelevant to use or could send wrong information.
Another drawback of only using FC layers is the amount of parameters to train[38]. For an input image
==PAGE:21==
2.1 Deep learning in general
12
of size 256 x 256 x 3 it will require more than 196,000 parameters per neuron in the ﬁrst layer and will
require a couple of neurons and hidden layers to perform anything.
A way to only apply information to a local area in an image/feature map is to use convolutional layers.
A convolutional layer can be seen as a window(receptive ﬁeld) sliding over the image/feature map. A dot
product between each window and a ﬁlter(kernel or weights) is then calculated and one value is extracted.
This is then sent through an activation function and the ﬁnal output is generated. When this is done for
all the windows a new image/feature map is made by these activated dot products[38]
An example can be seen in ﬁgure 2.4.
Figure 2.4: An illustration of a 3x3 convolutional layer with no padding and stride 1. The dot product
of a window of 3x3 from the input map and the ﬁlter is calculated and send through a ReLU
activation. The red area ends up having a dot product of 25 while the green have −2 and
its activation is 0.
Often one ﬁlter will only provide one type of output which only highlight one feature. It is therefore
desired to have more ﬁlters for each image/feature map. If the input image/feature map have multiple
layers then each output layer have a set of ﬁlters which correspond to the number of input layers. It is
then the sum of all the dot products for each ﬁlter which is the output sent through the activation[38].
In deep learning language when a convolutional layer is said to have x kernels, this describes the number
of output layers.
By adding a number of convolutional layers together it is possible to extract high hierarchy features
throughout an image which can be used in detection and classiﬁcation of objects[38].
Pooling layer
One of the essential layers in most CNN is pooling. When a feature map is pooled it is downsampled
while the most essential features is still preserved[38]. The most common way is to have a pooling layer
with receptive ﬁeld 2x2 and stride(step for receptive ﬁeld) 2 which leads to 75% of a map being discarded.
This will normally downsample each feature map with spatial size NxN to a feature map which is N
2 x N
2 .

Transfer learning
Before training a deep learning network the weights have to be initialised. This can easily be done by
giving all weights and biases the same value, making random values or by creating them by a function[42].

An alternative to always train a network from scratch is to use a pre-trained network, known as Transfer
learning[43]. When taking this pre-trained network it already knows how to identify diﬀerent kind of
features through the layers. This information can therefore be reused and ﬁnetuned to the desired dataset
is necessary. From this pre-trained network it is also possible to freeze parameters to minimise the number
of parameters to train.
These pre-trained networks are often trained with large datasets often from competitions like ILSVRC.

2.2
Faster R-CNN
Faster R-CNN is a uniﬁed network consisting of two modules[9], a Region Proposal Network(RPN) and
a Fast Region based CNN(Fast R-CNN).
The core idea for Faster R-CNN is based in Region based CNN(R-CNN)[44]. The concept for R-CNN
is to take an image and take an external algorithm, like Selective Search[45], to generate proposals for
where objects is based in the image. Each of these proposals were extracted as a region and send through
a CNN which could classify them. This method performed very well compared to, at that time, good
methods. One major problem for this method was time. Here each proposal/region had to run through a
CNN which easily could take a couple of milliseconds. The number of proposals suggested in the original
paper[44] was 2000 proposals. This could take around 50 seconds to run through one image[46].
One of the authors of the original paper, Ross Girshick, decided to further develop on R-CNN and handled
the time problem and thereby made a Fast R-CNN[16]. He suggested to take the entire image, run it
through a CNN, take each proposal and make Region of Interest(RoI) pooling after a feature map. From
that each RoI pool was sent through the remaining network and classiﬁed. This method made sure that
the CNN should only run one time for each image and thereby lowering the runtime drastically from 50
seconds per image to around 2 seconds per image[46].
Even though Fast R-CNN made a signiﬁcant increase in the speed it still had one major drawback. It
relied on proposals from an external method. This was then replaced with a new network, RPN, in the
paper Faster R-CNN[9]. This uniﬁed network, which consists of the RPN and Fast R-CNN, took the lead
and became the state-of-the-art network in object detection. Once again the runtime went down. This
time at around 0.2 second per image[46]. This uniﬁed network can be seen in ﬁgure 2.9.
Figure 2.9: Illustration of Faster R-CNN[9]

Region Proposal Network - RPN
The purpose for RPN is to identify where objects are in the image by classifying them as objects or
not objects and make a box regression to ﬁt the proposals better. This is done with anchors(see further
down).
When a feature map is generated it will consists of a speciﬁc amount of layers where each layer is a
representation of one feature in the image. This feature could be faces, letters or nuclei[51]. The location
of these highlighted features in the feature map correspond to the location in the original image. A feature
in top left corner of an image would also be represented in the top left corner of the feature map[50]. A
highlight in the feature map would likely reﬂect an object in the image[51].
A RPN is therefore put into action here to identify and localise objects in the image[9]. The structure
can be found in ﬁgure 2.12.
Figure 2.12: Region proposal network
To simplify the process of the RPN an example of one anchor center is represented in ﬁgure 2.12 and in
the explanation here. It is to note that the process for the rest of the anchor centers is the same.
When the RPN receives the feature map a 3 x 3 convolutional layer runs through the feature map with
padding and stride 1[9]. Each vector in this new map is then parallel sent through two 1 x 1 convolutional
layers. One layer for classiﬁcation using a softmax with two options: object vs. not object. The other
layer is for box regression to calculate the oﬀset of each boundary box, where the oﬀset is then added to
original coordinates in the image.
The classiﬁcation outputs 2k outputs for each anchor center, while the box regression outputs 4k also for
each anchor center. Here k is the number of anchors in each anchor center. In total the RPN will output
16 x 16 x 6 x k outputs or just 16 x 16 x k proposals each containing score for object and not object,

To do all of this a new concept was invented in the Faster R-CNN, anchors[9].
Anchors
As mentioned the nodes in the feature map represents objects and a circa location. To deﬁne the location
even better it is desired to have a reference to the image it self instead of only to the feature map. Anchors
was a solution for this question[9].
An anchor is a ﬁxed sized boundary box which correspond to the original image, through anchor centers
spread out in the image. Each anchor center correspond to a node i the feature map[9][11][52].
These anchors are expected to capture all objects by having diﬀerent scale and aspect ratio.
These
diﬀerent scales and aspect ratio is suggested to be 3 each resulting in 9 possible combinations[9]. The
idea is that RPN in that way can capture any object in the image no matter the size and shape. An
example can be found in ﬁgure 2.13.
Figure 2.13: Example of 9 anchors in one anchor center in the image.
A basic thing for these anchors is that they do not change shape or size from anchor center to anchor
center[9].
Handling these anchors in the RPN happens in the two parallel 1 x 1 convolutional layers. The classiﬁ-
cation layer apply the 1 x 1 convolutional with a kernel of 18(2 x 9 anchors), where a softmax is applied
afterwards resulting in 9 pairs which sums up to 1[52].
For the box regression layer a 1 x 1 convolutional with a kernel of 36(4 x 9 anchors) is applied and gives
out a 4 set oﬀset. This oﬀset is then added to the image coordinates which the anchors represents.
The kernels in the two 1 x 1 convolutional layers correspond to only one anchor from the anchor center.
The exact location for where the anchor is placed in the original image is then added afterwards by where
the anchor center is located.

2.2.3
Post processing
Post processing after RPN is made to remove all non relevant proposals through a couple of processes
and only keep the best proposals which leads to minimal runtime.
When all proposals from the RPN is generated, 2304(16 x 16 x 9), a cleaning of all the proposals is
necessary to keep the best proposals. In the paper it is suggested to be top 300 proposals without losing
accuracy[9]. If all proposals from the RPN had to go through the Fast R-CNN it could eventually explode
the runtime(image size 512 x 512, 9216 proposals). It is also not expected that all proposals contain any
objects.
Therefore all proposals are send through a post-processing part to remove all proposals which is not in top
300. The number of proposals in the end is maximum 300 but could be much less. The post-processing
can be found in ﬁgure 2.14. It can be seen that it starts to sort all proposals by score, hereafter only
top 6,000 is kept. Then Non-Maximum Suppression (NMS) is applied before the top 300 is kept for Fast
R-CNN.
Figure 2.14: Post processing after RPN
When the proposals have been made each contain six numbers: object score, non object score, x-center,
y-center, width and height. Each of these six outputs is also used in the loss function.
Threshold - Remove worst proposals
All these proposals are sorted by their object score with the highest ﬁrst and decreasing from there. This
sorted list is needed for all processes.
First top 6,000 scoring proposals are kept, the rest is removed from the list. It is expected that not all
proposals contains objects and it is therefore easier to remove them to save time in the non-maximum
suppression.

together with other boundary boxes. In the ﬁnal step we are only interested to look at one proposal for
each object. The remaining proposals for each object is therefore needed to be removed.

calculate the overlap, Intersection over Union(IoU) is used. IoU is a measure of how much a two objects
intersect with each other divided with the union of their areas.

The authors state that NMS will reduce the number from 6,000 proposals to around 2,000 objects[9].

reduced the number of proposals to a suﬃcient size which is below 2,000. In the training the number of
proposals which is kept for training is remains 2,000.
The authors state that it was tested with various size of proposals to keep and they concluded that
even with a maximum of 300 proposals it still gave accurate and reliable results[9]. Based on this a
ﬁnal threshold of the remaining proposals was made keeping top 300 proposals.
The output of this
post-processing part is a maximum of 300 proposals and their x-center, y-center, width and height.
2.2.4
Fast R-CNN
When all proposals with objects within them is found Fast R-CNN can then take each proposal and
classify the object in the proposal to a class. At the same time it ﬁts the boundary box better to the
proposed object.
From R-CNN this task would be just to extract each proposal from the image and send it through a
classiﬁcation network like VGG16[44]. Since a feature map is already extracted from the image, it could
be expected that all the relevant information in each proposals already is present in the feature map[16].
Fast R-CNN handles each proposal by making a RoI-pooling. Hereafter the extracted feature map is
ﬂatten out and sent through two fully connected layers. At last each proposal is classiﬁed with a FC
softmax layer while there is made a box regression to the boundary box to ﬁt it better, this is done
parallel with the classiﬁcation. This is done for all proposals and gives out the same number of proposals
with a probability for each class and the new boundary box size[16].
An overview of Fast R-CNN can be found in ﬁgure 2.16.

RoI-pooling
The goal for RoI-pooling is to extract a feature map which correspond to a speciﬁc proposal[16]. From
SPP-Net[53] it was suggested to take the last feature map from a feature extractor and apply a "pyramid"
model to extract diﬀerent feature regions within this feature map. Each "pyramid" layer is the feature
map split into regions where max pooling is applied to each region.
The "pyramid" output is then
combined into 1 vector which can be sent through the remaining classiﬁcations layers[53]. An example
of this "pyramid" can be seen in ﬁgure 2.17.

A feature by performing this "pyramid" method is that the output is always consistent which is required

For Fast R-CNN it was suggested to use the idea from SPP-Net of taking a feature map of random size and
extract that RoI which corresponds to a speciﬁc proposal and output a ﬁxed size feature map. Instead
of using a "pyramid" of diﬀerent ﬁlters, it was suggested to use only one ﬁlter and extract a feature
map which correspond to the size of after the last max pooling layer in the desired feature extractor
network[16]. For VGG16 the desired output is 7 x 7 x 512, so in each feature map a patch of 7 x 7 have
to be extracted[48].
The desired output is made by ﬁnding the RoI in the feature map for a proposal and sectioning the RoI
in 7 x 7 ﬁelds where max pooling is applied[54]. This will then give out a 7 x 7 x 512 feature map which
is desired to feed to FC layers in the VGG16[48].
When the feature map is extracted it is ﬂatten to a 25088 element vector which is then connected to 4096
in the ﬁrst FC layer. This results in above 100,000,000 parameters to train which is where the majority
of the parameters in VGG16 goes to[48].
Classiﬁcation and box regression
After a RoI have been sent through a couple of FC layers it is then sent parallel through 2 paths. One
which handles classiﬁcation of the region and one which handles box regression of the region[16].
The classiﬁcation layer takes the output from the last FC layer and sent it through a smaller FC layer
==PAGE:35==
2.2 Faster R-CNN
26
and hereafter a softmax function is applied to determine which class it belongs to in class k + 1. An
additional class is added since it is needed to have a proposal that is classiﬁed as background[16].
The other part takes the output from the last FC layer as well and also apply a smaller FC layer. After
the FC layer it is through a box regression layer which then calculate the oﬀset of the boundary box for
the proposal. This is done with respect to the original image and only in a close region to the RoI[16][46].
The Fast R-CNN outputs the same amount of proposals as given in the beginning. Each proposal have
a set of boundary box coordinates(xcenter, ycenter, h, w), a label(highest probability class) and a score
for the label class[16].

Fixation
When the tumour tissue have been extracted from the patient it is quickly being ﬁxated. This have to be

Embedding
When the tissue is ﬁxated it can be embedded. The main reason to embed the sample is to make it
stronger when it is being sectioned in thin slices[56]. Before embedding the tissue sample have to be
dehydrated. The remaining space can then be ﬁlled with a paraﬃn wax.
When the tissue have been embedded it is possible to go on with the sectioning. Another possibility is to
extract a small sample from the embedded tissue and put into another block of paraﬃn wax along with
other extracted samples. This will then create a TMA. This can contain samples of the most interesting
part of a tissue sample from multiple patients and makes the entire process of examine tissue samples
much more fast. The data used in this thesis comes from a TMA[56].
Sectioning
When the paraﬃn is solidiﬁed it is time for the sectioning. Here, very thin slices is cut from the embedded
tissue sample. This size of these slices is a ﬁxed size normally between 3-5µm[56]. Each slice is then
mounted on a glass slide.
Staining
After the sectioning the thin slices is more or less transparent since nuclei is transparent. To make the
nuclei visible the thin slices are stained. This is done for all the slices.
The staining have the purpose to highlight and visualise features. These features highlighted can be
generally structure in the sample but it can also be highlighting speciﬁc cells[56]. In this project the
main focus is to detect and classify positive and negative nuclei. It is therefore essential to have a clear
indicator of a nuclei and also a way to distinguish between positive and negative nuclei. A way to do
so is to stain the Ki-67 protein which is only present when cells divide. The quicker the cell divides the
higher the quantity of Ki-67, which is the case for cancerous growth[6].
The most common staining method is to take a solution of a speciﬁc antibody and put the slices into

antibody to the ﬁrst antibody. This new antibody carries a marker which will then add a colour. This

3.5
Assumptions
To make sure that the data is usable and that the model perform properly assumptions related to the
data have to be clariﬁed. Most assumptions are related to the label and nuclei data since these can easily
eﬀect the performance of the deep learning model.
Labels are in the center of a nucleus.
This may seem quite trivial when looking at ﬁgure 3.6c, that each label is a center of a nuclei. This
assumption is important since these labels set the base for the boundary boxes for each nucleus. If this
assumption does not hold the ground truth boundary boxes will be placed incorrectly which leads to
incorrect training.
All nuclei are annotated and in the same way.
This assumption is related to the work of the pathologist. It is assumed that pathologist 2(the preferred
pathologist) have been consistent in her way to annotate nucleus. This is both for annotating in the
center of each nucleus, correct labelling of the annotations and annotate all visible nuclei. The last one
also includes that if any nucleus is not annotated because it is too obscure all similar nuclei should not
be annotated either.
All nuclei can be completely separated
In ImageNet images one of the challenges is to identify objects which is overlapping each other[31]. It
could be a person in front of a car, a man on a horse or a cat on a table.
For the nuclei images it is known that each slice could be up to 5µm and nuclei could therefore overlap[56].
To be able to detect each nuclei it is therefore assumed that all nuclei can be completely separated or at
least to an extent where it is assumed that they do not overlap with each other. If this is not possible a
segmentation of the nuclei would not be possible either.
Images have to be independent of each other.
Generally for data applied to a deep learning model it is expected that each input data is independent of
the rest of the data. This also implies that each data input is unique and only exist once. Even though
it is already known that the data overlap with each other, the data is still accepted since it is only a
part-wise overlap.
Each class have nuclei in the same size.
The reason to have this assumption is to make the entire process as fully automated as possible and
thereby making equal size boundary boxes.
Since no information related to the size of the nuclei is
provided a generally size for each class is required. When looking at ﬁgure 3.4 it seems to hold for most
of the nuclei. If each nucleus had to be ﬁtted with a costume boundary box the time consumption would
==PAGE:48==
3.5 Assumptions
39
explode and it would be too ineﬀectively. Therefore, it is required to make the assumption that all nuclei
within a class have the same size.
The model is background independent.
As it can be seen in ﬁgure 3.6 the data have certain areas which is not annotated. These areas could
possibly contain nuclei which therefore have not been annotated. To ensure that these nuclei does not
aﬀect the model they have to be removed in the pre-processing phase. Since there is already a mask(ﬁgure
4.1), the easiest way would be to ﬁll the non annotated areas with one color. This color should therefore be
considered as a background color. The assumption is that the model is independent on which background
color that is applied to these non annotated areas.

Model
The model along with data is now ready and a baseline of the parameters can be found.
The ﬁrst
parameter to secure is one of the most essential, the feature extractor network.
As a general guideline for the baseline settings of various parameters, it is the same as for the original
provided code for Faster R-CNN by CNTK[58]. Some of these parameters have been modiﬁed slightly to
suit the nuclei data better.
Feature extractor network
From the paper they used two diﬀerent pre-trained feature extractor networks(CNN), ZFNet and VGG16[9].
ZFNet was used to win the ILSVRC 2013 competition and is generally a deeper version of AlexNet where
parameters have been ﬁne tuned[26]. In the paper they found that VGG16 was superior to ZFNet but
also that ResNet101 performed even better.
CNTK provides a couple of pre-trained networks for example VGG16, AlexNet, ResNet50[59].
One
problem occurs when it comes to ResNet, dimensions of the layers does not ﬁt to the standard of Faster
R-CNN. Therefore average pooling of RoIs is necessary. Unfortunately this have not been implemented
yet[58].
Since ResNet is not possible to use directly and AlexNet have been shown multiple times not to be as
eﬃcient as VGG. It have been decided that VGG16 is the feature extractor network to use.
Layers in VGG16
As a standard from CNTK the layers used in Faster R-CNN for VGG16 is the following[58].
• Start training at Pool2
• Take out feature map at relu5_3
• Use layer Pool5 to drop7 for classifying
These settings make the number of parameters to train 136,449,349 in 32 parameter tensors.
As a reference to where those layers are placed in the VGG16 network, the network can be seen in ﬁgure
4.3.

indicated that it was better to keep drop7 than trying to improve with drop6.
Samples per epoch
The chosen feature extractor, VGG16, takes around 5 fps(frames per second) according to the paper[9].
There is approximately 350,000 images in total. If all images are included in each epoch, one epoch would
last around 20 hours without validation images. This would be necessary if the model should learn more
than 50 object-classes but in this case the number of object-classes is 2(3 including background). Further
it can be seen that each image easily can contain above 100 nuclei while Pascal or COCO normally
contains less than 10 objects in each image. This will eﬀect the amount of training images which is
necessary to use since fewer images is needed to obtain the same number of objects.
After having tried diﬀerent settings varying from 500 images per epoch to 50,000 images per epoch it
became clear that the size of samples per epoch should not be too high. At 50,000 over ﬁtting occurred
after the ﬁrst epoch which took around 3 hours to run. Based on the images this also makes sense. Each
image is quite similar to each other and contain at least 100 nuclei in each. This sums up to above

5,000,000 nuclei for each epoch. For a sample size per epoch at 500 the outcome could vary a bit between
each run.
It was decided to set the size at 2,000 training images per epoch. This will give above 100,000 nuclei
in each epoch which is a reasonable representation of the data when it is 2,000 random images from the
350,000 images. This gave a fair ﬁt(without too much overﬁtting), stability and a reasonable run time.
At this setting it still converges quick with above 100,000 nuclei per epoch.

Above 2,000 did not improve signiﬁcantly but only made the runtime explode while under 1,000 images

Early stopping
Since the amount of images used in each epoch is not that high compared to the amount in total it can be
diﬃcult to predict when to stop training. An easy way to determine when to stop training is to analyse
when the model does not optimise anymore, also know as early stopping[60]. There is not a clear way to
set a maximum number of epochs or when to use early stopping except it should not under- or over-ﬁt.
This is of course also dependent on the data that the network is training on.
Normally early stopping is based on accuracy/prediction errors. For competitions like COCO and Pascal
this also makes sense since there is multiple object-classes, few objects per image and each object can
look very diﬀerently from one another. In this case there is 2 object-classes, plenty of objects per image
and each nuclei does not look completely diﬀerent from each other.

Chapter 6
Discussion
6.1
Model
6.1.1
Pre-implementation
The base idea of this project have been to take an implementation of Faster R-CNN and try to make
it work with nuclei data. Since the implementation have already been done, as little modiﬁcation to
the main code as possible should be made. If major changes was to be made the idea of using a pre-
implemented network would fail since the time spent on debugging could easily explode. This will make
it more appealing to just implement the network from scratch(with inspiration from the example code)
since every detail can be considered and determined.
It has therefore also been a process to clearly
understand what is going on within the network since the paper on Faster R-CNN describes the general
solution and not all the implementation details. Some of the changes made were to include validation to
the training and make early stopping possible but also introducing weights on the centers in the loss and
making augmentation with colour transformation possible.
Some time also went into trying to make a ResNet work with Faster R-CNN. The main problem is
that ResNet architecture requires RoI average pooling but CNTK only supports RoI max pooling. This
is a well known problem in the CNTK community but have not been added yet. Such a feature will
require digging into the CNTK platform and adding it at the root. Even though it will improve the
model it will also exceed the time limit for this project. By not being able to use any other networks
for feature extracting than VGG networks and AlexNet this put up a major obstacle from CNTK in
the improvements which can be made[26]. If any other networks were to be used, it will at the moment
require switching to another platform like Tensorﬂow.
6.1.2
Baseline
What is very interesting when looking at ﬁgure 5.3 and 5.4 is that just from the baseline predictions was
actually pretty good. It would seem from these images that all nuclei is detected even though they are
detected multiple times. What is even more impressive is that the model classify the predictions correctly.
By insuring that each prediction is classiﬁed correctly the purpose of the model, given a proliferation
==PAGE:78==
6.1 Model
69
index is met.
Generally, the model predicts the right nuclei by having a recall at ∼0.9 as mean, which could be higher
by changing the metric IoU. The amount of TP is great especially when it is seen from various ﬁgures in
this report that the GT is not always correctly placed in the center of the nucleus.
Already at the baseline it was interesting to observe that the predictions most of the times was not in
the center of the nuclei either was the GT. This resulted in predictions which was in the opposite from
GT in the nucleus. When the distance is big enough this results in a correct prediction which is not set
to TP since it does not overlap enough with the GT.
It did also look from the baseline images that most of the nuclei which was detected by Faster R-CNN
but did not have a GT, even though it should have been annotated as tumour nuclei.
6.1.3
Training and testing the model
To train the model it was necessary to use a Nvidia GPU with a good performance score. The best way
to require this was to use the GPU cluster at DTU. Since the cluster runs entirely through the terminal
and there were some restrictions on which software and packages that was allowed on the cluster, which
made it a challenge to debug the code if any errors acquired. For any future work it is recommended to
make test runs on a computer instead of using the cluster.
6.1.4
Fine tuning parameters
When data is of high quality ﬁnetuning parameters is where performance will increase when working
with deep learning. For this project it will not make sense to do much ﬁnetuning of parameters since the
data quality was bad. If data had been perfect with no missing annotations, annotations in the center
of all nuclei and a size for all the nuclei data would have been perfect to make improvements on the
parameters. However they are not perfect and it was not prioritised to improve the data in this project.
When working with ImageNet data it can be assumed that all the data is annotated correctly and with
correct boundary boxes[31]. If this was not the case the competition would have died long ago. The
performances from ImageNet is therefore also based on better data where just small adjustments could
make a signiﬁcant diﬀerences.
The small adjustment and its impact in this project was seen in the post processing which did not include
any deep learning parameters. This deﬁnitely indicates that the model performs well but is stopped by
the data.

6.2 Optimising the loss function
70
6.2
Optimising the loss function
From the start it was observed that the annotations were not always in the center of the nucleus. This
can be observed in ﬁgure 3.4. It was therefore of interest from the beginning to try and put a weight on
the center locations to make the loss function punish anchors which were too far away from a GT. This
was tried in a lot of various ways but always ended up performing worse than no weights on the center.
The reason for worse performance is that the loss function will see that the anchor is then placed further
away from a GT. If the loss function sees that the anchor is further away from the GT it will try and
minimising this distance. This will end up pushing the prediction away from the actual GT rather than
closer. If the center weights was increased above 1.0 it will end up sorting anchors away because it crossed
the boundary of the image. When an image has no anchors which hits a GT Faster R-CNN ends up
crashing. The training of Faster R-CNN will continue but does not train and gives out errors instead.
Unfortunately this happened every time the code was to run with center weights above 1.0.
When varying σ it was interesting that it changed signiﬁcantly for the RPN loss but did not have any
signiﬁcant impact on the Fast R-CNN except making the mAP slightly worse for σ ̸= 1.0. For the RPN
it makes great sense since the images contains many objects all around the image and therefore needs
some more smoothing to perform optimally.
6.3
Optimising post processing
After evaluation of the baseline training and results were produced it quickly became clear that there
was a problem with multiple predictions within the same nucleus. By having "small" boundary boxes
changes to the x and y have major impact on the IoU with a GT which therefore resulted in predictions
not removed but should have been. By implementing these changes mAP quickly became better but the
recall declined because TP was removed.
There could be several reasons for the decline in the recall.
One could be that a prediction with a better score removes a TP but is not overlapping enough with GT
however hits within the correct nuclei(Metric error). This will occur when a nucleus is annotated in one
end and the TP is predicted in the other end of the nucleus.
Second, a prediction with a better score removes a TP but is also classiﬁed as TP but with another
GT(boundary box size error). This will happen when two nuclei have annotations close to each other.
Third, a TP prediction is removed but the kept prediction does not hit any nuclei. This can happen
when the method predict an artefact around the nuclei and thereby removes a TP.
By looking at ﬁgure 5.6 it does seem that most of the coincidences where a TP is removed is either
because of the ﬁrst or second reason. Especially when the nuclei is clustered together they are smaller
==PAGE:80==
6.4 Metric IoU
71
but also a lot closer to their neighbouring nuclei which ﬁt into the second reason. When the nuclei are
bigger it is clear that reason one causes TP to be removed.
Since the third reason is not activated in any images it thereby implies that it around nuclei makes very
good predictions.
By setting a threshold on the score the idea would be that the model in a way does the same as a
pathologist does when they annotate, since the score is determining on how likely a prediction is to be a
nuclei. When looking at ﬁgure 5.7 it can be seen for the images with highest threshold that the predictions
that is left also hits what truly looks like nuclei. This is also what is intended by changing the score
threshold. Unfortunately it is very tough on the predictions which indicates that the model does not
completely know the diﬀerences between background and nuclei. If the model could distinguish better,
the score for each prediction would be a lot higher then it is, and it would be easier to ﬁlter away bad
predictions just by the score. When threshold on the low scores it does seem to hit the right predictions
which should be removed. Somewhere in the middle, or higher middle, would be a good place to set such
a threshold.
6.4
Metric IoU
The Metric was ﬁrst deﬁned as IoU= 0.5 which is the standard for ImageNet competition[31]. In the
competition the data is assumed to be well deﬁned but for this project it is not. This makes it diﬃcult
to have a good metric since the metric itself expect one of its inputs to be a real GT.
When trying to analyse what happens when adjusting the metric it became clear that most of the TP
predictions overlaps very well with a GT(ﬁgure 5.9). It is only in the very low end that it could be further
improved by changing the metric IoU. Around IoU = 0.4 it does not improve anymore and by looking
at ﬁgure 5.15, 5.14 and 5.13. At the low level metric IoU it looked like nuclei was detected but had not
been annotated by the pathologist. This is once again not the model failing but the data. If the data
had been better it would not be relevant to change the metric IoU but in this case it actually seem to
improve the results correctly.
If the data was perfect this metric would be perfectly ﬁne to use. When the quality in the data is lacking
as for the data quality in this project it would be preferable to use a diﬀerent metric. Which metric to
use instead would need further study in the future.

6.5
Comparison with App. 10004
The App. 10004 has a high advantage comparing to Faster R-CNN, it has been trained(dictionaries have
been created) and is independent on other inputs than the image. Another advantage is that it outputs
segmentation of each nuclei and not only the detected location. This makes it possible to extract the
exact area/boundary box of each nucleus.
The method could also have been used as a pre-processing to the data to generate better GT boundary
boxes but was not tried out in this project.
When comparing App. 10004 it was interesting to see that the true size of the boundary boxes have
much inﬂuences on the mAP. This clearly indicates that GT boundary boxes should not be equally ﬁxed
size but should be varying more. The reason to include the ﬁxed size boxes was also to make it more
comparable with GT and the predictions.
From the original implementation of App. 10004 it looks like it performed very well but when comparing
in this project it did not seem to perform that well[62]. The TP nuclei it misses in ﬁgure 4.7 looks to be
very easy to detect when visualising the image. These nuclei is detected by Faster R-CNN in ﬁgure 5.18.
Overall from ﬁgure 5.18 it seems that both App. 10004 and Faster R-CNN agrees on most nuclei where
most of them is TP. The ones they agree on which is not annotated with a GT looks not as FP but as
nuclei which have not been annotated correctly by the pathologist.
Even though it visually looks better with Faster R-CNN recall points towards that App. 10004 is better
at ﬁnding TP but also have more FP by looking at mAP. Whether this is cause by bad data as GT or
because it just performs better is not to determine. Faster R-CNN could properly be improved to surpass
App. 10004 by optimising parameters. But when comparing visually Faster R-CNN it generally seems
to predict better than App. 10004.
6.6
Data
From the start there were set some assumptions which were crucial to make the network perform and
also perform well. What can be concluded is that the most of these assumptions did not ﬁt to the data
at all. Already in the beginning this was observed and discussed with the supervisors at Visiopharm who
agreed that the data was far from perfect. After the baseline results it was decided that the time was
best spent looking into the post processing instead of trying ﬁxing the data instead of trying to ﬁx GT
data with regular image analysis.
==PAGE:82==
6.7 Future work
73
With the boundary boxes ﬁxed at size 26 x 26 the original idea was to vary this size for the two classes.
After the baseline it seemed superﬂuous since most of the nuclei were actually detected.
Instead of
wasting time trying to maybe improve the mAP a bit, it was decided not to do more for this point. This
could have made the NMS remove predictions more correctly but could also lead to more errors in the
predictions.
Overall it seems that the data is actually the biggest obstacle to receiving better results. If the data was
better the model could train better and thereby perform better when testing. After doing this project
it is believed that Faster R-CNN could detect and classify nuclei almost to perfection if the data was
perfect in its annotations.
6.7
Future work
For future work a couple of interesting aspects could be looked further into. These suggestions mostly
points toward larger changes or new ideas. In general it is diﬃcult to point at parameter updates since
the model seem to predict very well even though the data is not that good.
• Better annotated data - After the discussion this point may be trivial. The data should either
be segmented or at least contain a boundary box around every single nucleus. When this dataset
is acquired it would also be possible to ﬁnetune all parameters.
• Other datasets - It would be ideal to use one model to detect all sorts of diﬀerently stained nuclei.
One well trained network to detect all sorts of nuclei would be time saving and ideal. This requires
more datasets which is also better annotated then the current dataset.
• Change the feature extractor to another network, ex. ResNet - By having another network
it has previously been shown that better or similar results would be obtained. VGG16 is a very
heavy network with above 130 million parameters and take up 500mb(primarily for the classiﬁcation
part, convolutional to FC layer). By using another network fewer parameters is to be trained and
saved and thereby making the model run faster. Alternative only changing the classiﬁcation layer
could also be beneﬁcially.
• Segmenting nuclei using deep learning - From this project it have been shown that it is
possible to detect nuclei but within the community of histopathology segmentation is the primary
goal. Since this model can detect and classify nuclei an extra step could be added to obtain the
segmentation either by feeding the output to another algorithm or by adding an extra deep learning
layer. Since everything already runs with deep learning, it is suggested to keep it in that way and
for example use Mask R-CNN[63].

Chapter 7
Conclusion
For this thesis the goal was to detect and classify nuclei eﬃcient with deep learning. It has been shown
that by applying a deep learning model, like Faster R-CNN, it is possible. Just by using a pre-implemented
model and its initial values for detecting and classifying objects, good results can be obtained. By updat-
ing few of the parameters in the post processing the model can improve signiﬁcantly. This even performs
just as well as the implementation Visiopharm already have(which also segments) but outperforms their
implementation in runtime.
As it has also been shown in this project a known phrase is still relevant, a chain is only as strong as its
weakest link. The weakest link in this chain/project is the data, its inconsistency and all the unknown
factors within it. By having data where no object size is deﬁned, a model like Faster R-CNN would end
up predicting objects wrongly since object size ends up being estimated.
It is deﬁnitely possible to apply Faster R-CNN to obtain good and reliable results when detecting and
classifying nuclei. However, it requires good data to perform well and should be studied further in the
future.
