Introduction
Classical computers are extremely powerful tools that accurately and quickly perform calcula-
tions that are not feasible for humans.
However, computers are limited by needing to solve
problems with finite hardware within a reasonable time limit. Thus computers can only prac-
tically solve problems of limited size. For example, much modern cryptogtraphy, such as the RSA
(Rivest–Shamir–Adleman) [1], key protocol relies on encoding information with math problems
that, while technically solvable, are not feasible to complete within the constraints of modern
computers. Quantum computers are a novel technology, which while still in their infancy, can
theoretically solve problems that are infeasible to run on classical computers with a comparatively
modest amount of memory.
One of the most important bounds for computer algorithms is the time complexity, which expresses
the relation between the size of the input and the time the problem will take to run.
Time
complexity is usually expressed in big O notation, O(f(n)), where f(n) is the relation between
time and input size n as n →∞. Note that expressing a number N requires O(log(N)) bits.
Many interesting problems unfortunately, scale extremely quickly classically. For example, the
best known classical prime factorization algorithm, the general number field sieve, runs in super
polynomial, but sub-exponential time. This means it runs faster than exponential complexity but
slower than polynomial complexity, specifically O(e1.9(log(n))1/3(log(log(n))(2/3))). On the other hand,
a quantum computer running Shor’s Algorithm could solve the same problem in polynomial time,
specifically O(log(n)2(log(log(n)))(log(log(log(n))))), an almost exponential speed up.
While quantum computers exist, even the most powerful of them have extreme limitations [2], that
make them unsuited for practical use as of 2023. However, many quantum computing schemes, such
as measurement-based quantum computing (MBQC) show promise. This project aims to create a
code base designed to convert circuits to an MBQC graph format and simulate them to analyze
the effect of error on their results. The code for this project can be found at [3].
Classical Computers
While computers are extremely complicated machines, they can be described by relatively simple
mathematical models.
The primary model used to describe classical computers is the Turing
machine, a theoretical machine that consists of two components, a machine head and an infinite
tape. The machine head has a finite number of states it can be in. The tape contains an infinite
number of cells, but each cell contains a finite amount of information. A Turing machine, despite
its simplicity, is capable of calculating any real world calculation on natural numbers[5][6] in finite
time.
Indeed, a function is only effectively calculable if and only if it can be calculated by a
Turing Machine, according the The Church-Turing Thesis [7]. Any machine capable of emulating
a Turing Machine is called ’Turing complete’, and is therefore capable of any calculation that can
be performed by a physical device, so long as it has enough time and memory.
Classical computers are digital meaning they encode information in binary bits, ie 0 and 1. Since a
single bit holds very little information, bits are organized into groups called registers to hold larger
values. The architecture of classical computers is more complex than the simple Turing machine
model, and have specialized parts for handling specific calculations, such as arithmetic logic units
(ALU). ALU’s are digital circuits that are used to quickly evaluate logical and mathematical
operations. These units are composed of devices called logic gates. Logic gates are simple devices
that take in binary information, perform a logical operation, and return an output. For example,
the AND gate is a logic gate that takes in two bits, and performs the AND operation seen in
table 1, and returns the result. It is important for ALU’s to be universal, meaning that they can
implement any possible Boolean function, if enough of them are put together. This requires a set
of gates able to perform the AND, OR, and NOT logic operations. Any gate set that can perform
these three operations is a universal gate set, since any boolean function can be implemented using
only the gates in the set. While powerful, a universal gate set is not Turing complete by itself,
and ALU’s are therefore not full computers. For example, there is no set of gates that could run
a while (true) loop, which is trivial to implement on a Turing machine.

Time Complexity
Unlike the theoretical Turing Machine, real computers must perform calculations in finite time
with finite memory. This limits the size of inputs of algorithms run on real machines, as described
by the time complexity of the algorithm. As stated before, time complexity is described with big
O notation, O(f(n)), where f(n) is the relation between time and input size n as n →∞. As can
be seen in figure 1, time complexity has a massive effect on computation time. Problems that scale
in polynomial time, or p time are p problems, and can be effectively solved.
There exists a more challenging class of problems known as nondeterministic polynomial problems,
or np problems. These problems can be solved in polynomial time by a nondeterministic Turing
machine, a modification of the Turing machine that has multiple possible outcomes for each step,
and either performs all of them simultaneously or is imagined to pick the one that leads to the so-
lution the fastest. While a deterministic Turing machine can solve any problem a nondeterministic
Turing machine can, many np problems run in exponential or worse time on deterministic Turing
machines.
For example, let’s say we wanted to prime factorize an integer N by checking every prime number

guess a prime factor. In either case it performs O(1) calculation steps. Of course in practice,
an implementation of the nondeterministic Turing machine would not be able to rely on luck,
and would involve separate computer CPU’s trying different solutions simultaneously. Since the
maximum size of N is N = 2n −1, where n is the number of bits used to store N, the size of
numbers a computer can store rapidly outpaces the ability to factor them, an asymmetry that is
vital to ensuring security in many cryptogrphic keys, such as RSA [1].
A deterministic Turing machine may verify the solution of any np problem in polynomial time, in
our example, checking if a prime number is a prime factor of N is only one calculation for any
machine. The problem of turning NP problems into P problems known as the P = NP problem.
While technically unsolved, most computer scientists believe that P ̸= N [8], and therefore hard
NP problems can not be solved in polynomial time classically. Therefore, to efficiently solve these
problems, a new kind of computer not based on the classical Turing machine must be used.
Quantum Computers
The Turing machine, and therefore classical computers, are a classical devices, which store and
manipulate classical information. A Quantum Computer is a computer that stores and manipulates
quantum information. Quantum computers can be described using a quantum Turing machine [9], a
theoretical machine that describes the behavior of a quantum computer in the same way the classical
Turing machine describes the behavior of a classical computer. The quantum Turing machine is
capable of perfectly simulating a classical Turing machine [9], and therefore can run any classical
algorithm efficiently. A classical Turing can likewise simulate any quantum algorithm, albeit at the
cost of increased time complexity, and the inability to perfectly store arbitrary irrational numbers
[9].
The advantage of the quantum Turing machine is that it is able to solve some problems
much quicker than a classical computer, such as the aforementioned Shor’s algorithm, with which
a quantum computer can prime factorize almost exponentially faster than a classical computer.
Shor’s algorithm, like many quantum algorithms, works by creating a superposition of all possible
results, and then amplifying the amplitude of the correct answer.
While quantum computers are theoretically powerful they are practically challenging to create. As
of November 2022, IBM’s most powerful quantum computer has 433 quantum bits of information
[2], no long term quantum information storage, and requires a room full of cooling equipment. For
reference, MacBook released in the same year has at minimum 8 GB, or 6.9 ∗1010 classical bits, or
cbits, of RAM, and over an order of magnitude more in storage [10]. Because quantum computers
have such severe memory limitations, the goal of quantum computer projects is to create a device
more analogous to the ALU’s mentioned in the classical computing section than a full machine.

In this kind of machine, a classical computer is connected to a quantum circuit, and uses the
quantum circuit to aid in solving problems that would not be feasible to solve classically. This
kind of machine is not a Quantum Turing Machine, so we will not express quantum algorithms as
functions to be run on a Quantum Turing Machine. We instead express quantum algorithms as
quantum circuits, using a universal quantum gate set.
Quantum Gates
Just as a classical circuit is created from classical gates, quantum circuits are created from quantum
gates. Quantum information is stored in quantum bits, or qubits, which are two level quantum
systems. We will express qubits as a two dimensional Hilbert space with basis vectors |0⟩and
|1⟩. Quantum gates are physical devices that perform a unitary operation on one or more qubits.
Because they are unitary, quantum gates must be reversable. The number of inputs and outputs of
a quantum gate are therefore always the same, ensuring that no information is lost. As can be seen
in table 1, this is not true for classical gates such as the AND and NOT gates. Additionally, qubits
cannot be copied due to the no cloning theorem [11], while cbits are trivially copied. The result
of these properties is that, before measurement, a quantum circuit must have the same number of
qubits at every step of computation, placing significant restrictions on the architecture of quantum
circuits.
Like classical gates, quantum gates have universal gate sets, finite gate sets that can be combined
to perform any unitary operation on a qubit register. A universal quantum gate set can still be
made with only a handful of gates, for example the following set of three rotations of arbitrary

Theory: Measurement Based Quantum Computers
Classical computers encode classical bits, or cbits in electric current or voltage levels.
These
electric currents are propagated through physical gate devices to apply gate logical operations to
the information they encode. Thus, in order to perform an operation on two cbits, the physical
encodings of those two cbits must interact when the gate operation is performed. This of course
requires all of the cbits to be physically encoded in such a way that their physical representation
can be moved through the computer and through gates without error, or at the very least with
small enough error that the computer can correct it.
6
==PAGE:7==
Superconducting quantum computers operate similarly, encoding quantum information in the two
lowest eigenstates of an artificial atom made in a superconducting circuit [12]. As with a classical
computer, when a gate operation is performed on 2 qubits, the 2 qubits must physically interact to
exchange information. Like the physical encoding of a classical computer, the encoding of qubits in
a superconducting quantum computer must retain information long enough for the entire quantum
function to be computed. A superconducting quantum computer is therefore strictly limited in
time and space, it needs to complete its computation before its qubits become unusable, and it
must be designed such that every qubit can physically reach, or otherwise have its information
transferred to, all the qubits it must interact with to perform a calculation.
Measurement based quantum computers, also known as one way quantum computing, is a quan-
tum computing scheme that can circumvent some of these limitations. It relies on propagating
information through quantum teleportation.
Quantum Teleportation
Unlike classical information, which must travel through space, quantum information can also be
teleported via quantum entanglement. Quantum teleportation is a protocol by which 1 qubit of
quantum information can be from one place to another via quantum entanglement, requiring only
a single cbit of classical information to be physically transmitted.

Description of an MBQC
A measurement based quantum computer first prepares a highly entangled state of many qubits
called a graph state, like the one seen in figure 3. The graph state is composed of input, bulk,
and output vertices. Input vertexes are initialized holding the input to the circuit, bulk vertexes
are analogous to gates and wires in a conventional computer, and output nodes are the nodes that
contain completed calculation once the program has run. To perform a calculation, qubits are
measured, as seen in figure 4. By measuring input and bulk nodes, information travels through
the cluster via quantum teleportation. Therefore, our qubits only have to physically interact with
each-other when the state is prepared, all further operations are single qubit measurements and
the transmission of classical information. As with other teleportation protocols, corrections must
be applied to the output bits.
The cluster operates as a quantum circuit that performs a function on its input qubits. Since all
vertexes in the graph are qubits, the input qubits are only a subset of the all the qubits in the
cluster. Therefore the number of qubits in the cluster is higher than the qubits that make up the
information it performs a function on. We must therefore make a distinction between physical
qubits and logical qubits. Physical qubits are all the two level quantum systems that physically
exist in our machine, for a graph state, each vertex is a physical system. Logical qubits are abstract

Theory: Graph States
Graph states are a type of quantum state that can be expressed as a mathematical graph, where
each vertex is a qubit, and each edge represents entanglement.
Graphs
A mathematical graph, G = (V, E) is defined as a set of vertices, V , and a set of edges E. Each
edge in the graph is a connection between two different vertices. In the graph states used for
quantum computing, vertices represent qubits, and edges represent entanglement between two
different qubits.

Advantages of MBQC
While a cluster may be of arbitrary size, each physical qubit in the cluster only undergoes a small
number, c, of entanglement operations and a single measurement. The number c depends only on
the architecture of the cluster, for example, in the square grid, each physical qubit is entangled to
its four neighbors, requiring c = 4 entangling operations, regardless of the size of the cluster.
This provides an advantage over superconducting quantum computers. For a circuit of k gates,
a superconducting quantum computer must perform at least k unitary operations on its physical
qubits, requiring that the machine is capable of storing its qubits of a time that is O(k), and
capable of performing a number of operations on that qubit that also is also O(k). Given that
storing quantum information is extremely difficult, this is a significant challenge when it comes to
running longer algorithms.
For a measurement based quantum computer, it is not required that a physical qubit survive for
the entire computation. All that matters is that a qubit survives long enough to go through a
number of entanglement operations that does not scale with n, and then be measured. Proposed
implementations of MBQC’s take advantage of this [14] [15], and use a scheme where only a small
fraction of the cluster exists at any given time, as shown in figure 5.

Theory: Quantum Gates in MBQC
Quantum circuits consist of an input register and a series of gates that perform unitary operations
on that input register. In order to perform quantum calculations, a measurement based quantum
computer must have the same functionality.
Our input register is simply a series of vertexes
initialized with our input values that aren’t entangled to eachother. To implement a gate, we add
a subgraph, a subset of a graph, that performs that gate unitary operation to our graph. Like the
graph as a whole, each gate subgraph has input nodes, bulk nodes, and output nodes. The input
nodes connect to either our input register or the output of another gate, the bulk nodes connect
only to other nodes in the gate subgraph, and the output nodes are either connected to a new gate,
or are the output of the circuit as a whole.

Clifford Gates
Clifford gates are a subset of quantum gates that normalize the Pauli group.
A group, G, is defined as a set of elements, {A, B...} and an operation ∗, which satisfy the following
requirements [18]:
• Closure: if elements A, B are in group G, then C = A ∗B must also be in group G
• Associativity: If elements A, B, C are in group G, then (A ∗B) ∗C = A ∗(B ∗C)
• Identity: There is an identity element, I, in group G such that I ∗A = A ∗I = A for {A ∈G
• Inverse: For each element A in group G, there is a corresponding element A−1 such that
A ∗A−1 = I
The identity matrix and Pauli matrices with the phase factors ±1 and ±i and product operation
together create a group called the Pauli group.
A normalizer, g is a set of elements, of S in a group, G, is defined as:
g ∗S ∗g−1 = S | g ∈S
(32)
Note, that for an element s in S, g ∗s ∗g−1 does not necessarily evaluate to s, but it will return an
element in the set S. This is important to us, because if a unitary, U is a normalizer of the Pauli
group, then the following is true for any combination of Pauli operators P1:
P1U = (UP2U †)U
P1U = UP2
(33)
Where P2 is also a combination of Pauli operators. Therefore, when chaining Clifford gates together,
while the Pauli corrections we apply to our output state will vary as we add more gates, we never
have to adjust any properties of the Clifford gates themselves based on the rest of the circuit.

Simulating MBQC
Converting to MBQC
We now have a clear method of converting any quantum circuit, c, into a graph state.
19
==PAGE:20==
1. We rewrite the circuit c as c′, a circuit that performs the same function but using only gates
in the set S = {Rx(α), Rz(γ), H, CNOT}.
2. We create a graph, G, consisting of one node for each qubit in the circuit c, this graph does
not have edges yet.
3. We also create a correction list for each logical qubit, which contains the corrections that
need to be applied to the logical qubit after measurement. These correction lists are initially
empty.
4. For each gate in the rewritten circuit c′:
(a) We add the corresponding gate subgraph to our graph state. The gate’s input nodes are
already nodes in G, but for each bulk and output node in the gate subgraph, we add a
corresponding node to our graph G. We add every edge in the gate subgraph to G.
(b) We record the measurement basis for each bulk node and input node, including any
dependencies on previous measurements.
(c) We update the correction lists by commutating previous corrections through the added
gate and add new corrections from the added gate.
After these steps, the graph is ready to be run or simulated. In the code attached to this thesis, we
implemented these steps in python. The rewriting of the circuit into the basis gate set S was done
using Qiskit [21], an open source software development kit (SDK) for quantum computing. The
other steps are handled by a python class we created, which converts the transpiled qiskit circuit
into a graph, and stores the necessary auxiliary data. To simulate the graph, we used Paddle
Quantum [22], another quantum platform. We also implemented simulated error into our code,
and simulated the Deutsch-Jozsa Algorithm, Grover’s Algorithm, and Shor’s error correcting code.
Simulating Error
Like any physical system, a quantum computer computer will have error. Simulating a full cluster
state of GKP states is computationally infeasible. After all, if it were possible to efficiently simulate
quantum computers on classical computers, there would be no advantage to quantum computers.
Because of this we do not fully simulate a cluster of entangled GKP states, we instead use Paddle
Quantum’s ?? more abstract simulations.
As seen in figure ??, it is possible for the peaks of GKP state to be measured in the wrong band,
especially if the peaks are not squeezed tightly. This can result in us measuring a 1 for a GKP in
state 0 and vice versa, which would affect the corrections we apply to the final state and to the
angles in our rotation gates. While the full effects of finite squeezing on GKP states is outside the
scope of this project, incorrect measurement results are trivial to implement, we simply can change
the power of the terms in our correction by 1 for each error we expect will occur. We refer to this
as measuremnt error.
In order for the rotation gates described in previous sections to work, it must be possible to measure
at any angle in the XY plane. Optical states such as GKP states are measured with homodyne
measurements [23], which measure the input signal by combining it with a local oscillator, as
seen in figure 11. Homodyne measurements return information on the quadratures, and allow us
to measure at arbitrary in phase space. In practice of course, there will be some error, so we
implemented an option in the code for random rotations in phase space within a given parameter.
We refer to this as rotation error.

Deutsch-Jozsa Algorithm
The Deutsch-Jozsa Algorithm was one of the first quantum algorithms developed that exhibits an
exponential speed up over classical algorithms. While it has no current practical use, it is still
useful to demonstrate quantum advantage.
We consider a black box oracle that takes in an n bit string input and outputs a single bit and
performs the following function:
F : {0, 1}n →{0, 1}
(46)

balanced, in which it returns a result of 0 exactly half the time. The problem is to evaluate if the
oracle is constant or not.

balanced. It is worth noting that evaluating the oracle a constant number of times, k, times will
either tell us it’s balanced or allow us to conclude it is constant with a probability of error ϵ ≤
1
2k .

Rotation Error
When we apply a rotation error, we find that the chance of measuring correctly decreases slowly
as the error increases. As seen in figure 16, it takes a rotational error of around .5 radians to lower
the probability of a correct result to less than .95 for a constant 2 qubit function. This isn’t that
unexpected, given that the graph consists of short, unconnected lines, error is very limited in its
ability to propagate.
Our graph for an n qubit Deutsch-Jozsa algorithm consists of n unconnected subgraphs that must
all return 0 without error. Let us call the error rate for a subgraph ϵ, and the success rate of
(1 −ϵ). For an n qubit constant Deutsch-Jozsa circuit, all outputs must be measured correctly. So
we expect the circuit to have a failure rate of 1 −(1 −ϵ)n. This can be seen in figure 17.

Grover’s Algorithm
While the Deutsch Jozsa algorithm is of very limited practical use, later quantum algorithms have
much more obvious applications.
One example is the Grover Search Algorithm [24], which provides a quadratic speed up in an
unstructured search problem. An unstructured search problems considers the following situation:
there is a randomized list of N items. One of these items, with index ω, is ’correct’. We want to
find the index of the correct element. Classically, we check the elements of the list one at a time
until we find ω, which is of complexity O(N). It should be noted that in the case where there
are multiple correct indices, Grover’s algorithm will still work, returning a single correct index at
random, so long as the correct indices are a minority of all indices.
In practice, an unstructured search of a list is rare, as classical data structures can drastically
increase search speed. Hash tables, for example, allow for searching in constant, O(1) time [25].
However, Grover’s algorithm can be used for other applications, such as brute force attacks on
cryptographic keys, so long as those problems can be expressed as an unstructured search. In these
cases, the polynomial speed up could be a significant advantage.

Conclusion
Grover’s algorithm is a much more complicated and more useful algorithm than the Deutsch-
Jozsa.
As such it is harder to implement and more sensitive to error, especially measurement
error. This brief analysis of Grover’s algorithm shows how for a more complicated cluster state,
error propagates and can quickly render the results useless. It is therefore important to implement
methods to deal with error.
A practical Grover oracle would need to solve a logical problem in order to be useful, such as an
oracle that attempts a prime factorization or modified traveling salesmen. These oracles would
have to be optimised to contain as few gates as possible, since as we see from our tests, the graph
state is extremely sensitive to error.
Error Correcting Codes
While creating more accurate measurements and more tightly squeezed states would reduce the
effect of error, we can also employ error correcting codes, techniques that allow us to detect and
remove error. The idea of error correcting codes is ancient, such as Masorah, a method for ensuring
accurate reproductions of the Torah used in antiquity [26]. Most classical error correcting codes deal
with bit flip errors and loss of data through a channel. They usually work by copying and repeating
information. This is not possible in quantum, due to the no cloning theorem. Additionally, quantum
information can from sign flip error, where the sign of the amplitude of a state is flipped. Sign flip
error has no classical counterpart, and poses an additional challenge for quantum error correction.
Bit Flip Error
The simplest classical error correction code is a repetition code, where one cbit is copied at least
twice, for a total of at least three identical cbits. The cbits are then decoded by majority decision.
While this code is relatively inefficient, it requires at minimum three physical cbits for one logical
cbit, it is a useful starting place when making quantum error correcting codes.
While we cannot clone a quantum state, we can entangle it with a known state. We use the circuit
seen in figure 38. Let’s say we want to encode some state |ψ⟩= a |0⟩+ b |1⟩. When a bit flip error
occurs, our state will become |ψe⟩= a |1⟩+ b |0⟩. We have a bit flip chance of ϵ, so our chance of
correctly transmitting |ψ⟩is 1 −ϵ without any error correction. The first set of CNOT’s encodes
our logical |0L⟩and |1L⟩states as follows:
|0L⟩= |000⟩
|1L⟩= |111⟩
|ψ⟩→|ψ′⟩= a |000⟩+ b |111⟩
(64)
We now apply our bit flip error and decode. Our circuit will correctly decode the initial state if
no more than one qubit is flipped. The chance that all 3 qubits are un-flipped is (1 −ϵ)3, and the
chance one is flipped is 3ϵ(1 −ϵ)2. As seen in figure 37, so long as ϵ < .5, our error correcting code
will increase the chance of getting the correct state back.

Shor Code
To cover all possible errors, we combine the bit flip and sign flip into the Shor Code [27]. The
Shor code circuit is the same as the sign flip correction circuit, only each physical qubit has been
replaced with a bit flip error correction circuit. This code is appealing, after all, the effect of a
measurement error is a bit flip and/or phase flip on the final state. Having a code that handles
both should then allow us to deal with measurement error, which tends to have a worse effect on
the final results than bit flip error.

operations the graph performs in pre-processing before adding the qubits to the graph state.

Other Methods
The above mentioned error correcting codes are implemented on the level of logical qubits in our
cluster state, as we are translating a conventional quantum circuit into MBQC. An more effective
method, such as the one here, [28], would work on the level of physical qubits. However, since
these methods modify the underlying graph state, they are outside the scope of this project.
Conclusion
In this work, we discuss the motivations for and limitations of quantum computing. We analyzed
Measurement Based Quantum Computing, and created a software library to assist in simulating
it’s capabilities and limitations. By analyzing the Deutsch-Jozsa and Grover quantum algorithms,
we quantified the effect of some types of error on an MBQC. Using this code, we demonstrated
that MBQC is very sensitive to error, and would require error correction and very efficient circuits
to be implemented practically.
