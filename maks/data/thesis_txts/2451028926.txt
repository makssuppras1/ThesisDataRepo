ABSTRACT 
Life is becoming extremely depending on electricity, consumers are demanding for a 
stable and continuous delivery of energy. This increases the importance for security and 
stability assessment of the system operation. While the former identifies contingencies 
the later assess the current state of the system. With the need to have a stable delivery of 
power the stability studies are being more and more common. This thesis implements a 
large power system model for testing stability assessment method. There different kind 
of stability problems, being the most commons the voltage instability and transient 
stability. This study will focus on the aperiodic small signal stability, what it is and how 
to assess it.  
In this thesis two different data will be used. The first is a large system model provided 
by Texas A&M University is studied and will be used to try to evaluate if it is possible to 
implement a stability assessment for such a large data. The second data is a smaller one 
representing ta part of the Swedish network, and it is used to for the implementation of 
the real-time performance testing of assessment method. Since it was discovered that 
dynamic simulations of power system are not so easily accomplished, when dealing with 
a large system. With so many models and equipment if one parameter is not properly 
defined the simulations will not converge. Issues are not as straight forward to identify, 
and the actions needed to fix problems may are not trivial. 
This thesis aims to create a realistic application to visualize and assess stability problems 
within the power system.

INTRODUCTION 
1.1 
Background 
In future power system real-time stability and security assessment will be increasingly 
important with the introduction of renewable energy sources. In recent years this has 
resulted in the development of a range of methods for assessing the state of a power 
system. To ensure that these methods can run in real-time, performance testing on 
different power systems is needed. The scaling of the method with the system size needs 
to be investigated such that the system can run on not only small test systems but also 
large system. Therefore, there is a need to acquire models of large systems and simulate 
instability scenarios for them. 
As part of the research on real-time stability assessment a state-of-the-art SW-platform 
for wide-area applications has been developed to enable testing and demonstrations. The 
SW-platform provides structured access to model parameters and real-time phasor 
measurement unit (PMU) snapshots. 
The development of new models for large power system together with the SW-platform 
will help in the development and optimization of stability methods to ensure their 
capability to run in real-time. 
1.2 
Formulation of the problem 
Security and stability assessment are important to system operation, the former can 
identify the contingencies while the later assess the current state of the system. Prior to 
the deregulation and the increase of renewable energy sources (RES) in the system 
operators were aware of which of the critical contingencies could be a treat to the system 
stability. The consumption patterns, such as daily, hourly and weekly variations remained 
the same for several years. Therefore, security and stability assessment could be done 
through offline process due to the predictable characteristics of the system. 
However, the deregulation of power system and higher economics objectives force the 
operation of the power system closer to its physical limits In addition, with the world-
wide incentive that the supply for the power consumption to be produced mainly by green 
energy, for example, the Danish Government intent to cover 50% of the energy demand 
by renewables sources by 2030, there are more RES in the power system [1]. These 
sources contribute to the fluctuation of the operation point of the system, due to the 
unpredictable and the fluctuate nature of this sources. Hence, offline assessment is no

longer enough, and a real-time approach is necessary. Real-time testing of assessment 
methods demand a lot of computational effort, however, the phasor measurement 
technology [2] creates a door for the development of real-time applications, such as 
monitoring and control. [3]. 
In [4], [5] different assessment methods were tested in real-time using a small test system. 
Nevertheless, the scaling of the method with the size of the system also needs to be 
investigated such that the system can run not only for small test systems but also large 
ones. Therefore, this project will than focus on testing the Aperiodic Small Signal Rotor 
Angle Stability (ASSRAS) method in real-time for a large system, possibly with more 
than 5000 busses. This means it will be necessary to find a large dynamic model, apply 
different dynamic instable simulations that might be interesting to perform the assessment 
method and develop a performance measure to be able to visualize and verify the results 
of the method. [5] describes how to perform the ASSRAS assessment. 
This project will not focus on how to do a stability assessment, but some important 
statements will be described and revisited. The goal of this project is to verify if 
assessment methods can be used in real-time for large power systems. To accomplish this 
different test and implementations must be done. The first is to acquire a model of a large 
power system and simulate different instable scenarios for testing the assessment method. 
Next the computation of the ASSRAS method will be done for those instable scenarios. 
If one of these cases have a stability problem, preventive measurements should be 
implemented to avoid the instability. Here after, a way to visualize the performance and 
the preventive measures, if they were necessary, should be implemented. Lastly, perform 
a real time test of the assessment method and the performance measure

2.2 
Load flow calculations 
Knowing the characteristics of the system the next step was to perform a load flow 
calculation. The load flow was calculated using the Full Newton Raphson method and the 
software PSSE34. Figure 2 presents the results of the load flow. 
 
Figure 2. Load flow solutions 
Although the load flow converted, many generators were working on the limit of the 
reactive power production, either on the minimum limit of reactive power, which means 
it is absorbing power, or in the maximum limit, machine is injecting power in the system. 
In order to avoid the generators being in its reactive power production limit (minimum or 
maximum), fixed shunts were added to the buses where those generators are connected 
starting from the high voltage, 500 kV. For example, when performing a load flow, all 11 
generators connected at bus 6349 where generating the maximum of its reactive power, 
giving a total of 421,97 Mvar being injected in the system from the generators. Therefore, 
it was added 3 fixed shunts of 100 Mvar each, acting as a capacitor to compensate and 
relieve the stress of the machine. This measure was only partially successful since in some 
cases, the new fixed shunt did not had an impact on the production of the generator. 
In some other events, adding a shunt at bus “A” had a negative effect at a generator at bus 
“B”. For example, assuming the small system in Figure 3 are generating within its limits. 
When adding a fixed shunt in the “Bus A”, G1 was no longer on the limit even though G2 
remained violated. Next step would be compensating the reactive power of G2. However, 
when a shunt was added to “Bus B” it violated the limit of the G1 again. the generator in 
the “from bus” violated again. This happens when the generators needed different type of 
compensation. One generator was injecting power in the system and the other was 
absorbing.

DYNAMIC SIMULATIONS OF LARGE SYSTEMS 
This section will cover the challenges faced when trying to perform dynamic simulations 
of a large system. 
3.1 
Challenges 
Throughout the studies of this system several different challenges where encounter when 
performing a dynamic simulation. In this section those challenges will be explore and the 
tentative of solution for each of them will be cover as well. 
3.1.1 
Missing models 
The first challenge faced when performing a dynamic simulation on this dataset was that 
not all generators and none of the switched shunts had their dynamic models already 
specified. On a first check all wind machines and a few of conventional ones did not have 
any of the necessary models, such as the generator model, stabilizer model. excitation 
mode and the turbine-governor model. This prohibits the dynamic simulation to run. 
The first solution for this was to put all equipment that did not had a defined model, both 
generator and switched shunt as out of service. This was not successful, PSSE continue 
to complain about the missing models. So, the next tentative of solution was to create new 
models for that equipment. 
For the wind machines it was used the models for a type 3 machine, since it was the type 
used in this synthetic data, it was specified in [7]. The parameters used were the 
parameters set on the Program Application Guide [8] where there is some generic 
standard for the wind machine models. This fixed the issue of the missing models of the 
wind turbines. 
For the conventional machines it was a bit harder to create the models. Instead of using 
the standard values like for the wind machines. The new models were created based on 
the already existing ones. Meaning that it was used the same model as the machine that 
had the closest characteristic to the machine without model.  
To do that, a code was created, in python. First the conventional machines were divided 
in two groups, with models and without models. Then the code read all data from the 
group with models and compare with the data of one without model machine. The model 
used was the same one used for the generator that had the closest size (Sbase) and the

4.1 
Aperiodic small signal stability 
When it comes to stability of the power system different approaches can be make, such 
as voltage stability or transient stability. In this study it will be explore the aperiodic small 
signal stability. The aperiodic small signal instability is cause due to the lack of 
synchronizing torque. It is presented on work from a Ph.D. dissertation [9] and a patent 
for the method has been granted in [10].  
Each generator has a capacity to generate enough synchronizing torque so that the system 
can maintain the operation at a steady state. The deficiency of the synchronizing torque 
causes the increase of the rotor angle and loss of synchronism. This section will explore 
how to calculate the boundaries of the method. 
The input of the method is a measurement data that provides the monitoring of the system 
grid in real-time. The phasor measurement unit (PMU), [2], was used as input. In this 
study, the PMU provides the following values among others: 
• Bus voltages; 
• Bus frequencies; 
• Branch currents; 
• Generator currents; 
• Generator OEL status. 
With the status of the system (PMU) a deterministic representation of the system 
conditions can be established, and the following assumptions can be made: 
• All generators injections enter the network in a node of constant voltage

already operating in a stable situation, a simple disturbance can make the operation point 
reach the unstable region. Usually, blecautes happens when the operation point reaches 
the unstable region, causing equipment to disconnect trying to reach the stable region 
again.

This chapter will present the results of the implementation of the assessment method in a 
wide area application using the SW-platform previously mentioned. 
The output file shows the response of the grid after a disturbance occurs. The dynamic 
simulation starts with the system operation in the steady state. After three seconds a fault 
occurs on generator 3 at bus 101 disconnecting it from the system, causing the loss of 
1200 MW. 
For a dipper analysis of what happens on the system after the outage, it will be used the  
bus 68 voltage and frequency shown in Figure 26 and Figure 27, respectively. The first 
shows a voltage drops at the same moment of the outage, which is expected, since now 
there is an unbalance between the production and consumption, the demand is higher than 
the generation. Thus, the lack of power causes the decrease of the voltage. After the 
outage, the system begins to take actions to try to find the new equilibrium point, for 
example the remaining generators increase their production to compensate the loss of 
power. This phase is represented by the oscillation of the voltage after the perturbance, 
trying to go back to within its limits. For all busses in this network the voltage limit is the 
same: 𝑉𝑚𝑎𝑥= 1,1 𝑝𝑢 and 𝑉𝑚𝑖𝑛= 0,9 𝑝𝑢.  
At first moment the outage does not causes instability to the system, even though the 
voltage drops to a value lower than the minimum limit. However, approximately one 
minute (60 seconds) after the disturbance occurs the network becomes instable.

CONCLUSION 
This chapter contains a resume of the work and the results obtains in through this study 
and what would be the important to continue work when moving forward. 
7.1 
Results 
The implementation of a large power system model was not successful due to all the 
challenges faced with the dynamic simulations. In addition, with the difficulty of the 
dynamic simulation work there was not enough time to implement calculation of 
countermeasures into the application. The application only assesses the stability problem 
in a level of understanding and investigating if the instability is considered the aperiodic 
small signal stability. But it does not calculate any actions that could be taken in other to 
avoid the stability. 
However, the study proves that the application works for assessing the method. When 
analyzing the results from the stability analysis with together with the simulations of the 
computation of the assessment method. It indicates the occurrence of the aperiodic small 
signal instability. The simulations of PSS/E accuse of the network stop converging after 
65 seconds. And at the same time the assessment method shows that around the same 
time two of the generator crosses the stability boundary. The application developed also 
show the loss of synchronism of one of the generators happens in the same instant. 
Indicating that the aperiodic small signal stability occurs. 
7.2 
Perspectives 
This study validates the importance of assessment method in real time. If the method was 
applied to a grid in real time operation, the application would be able to detect the problem 
and warn the operators in real time. Normally, when a blackout happens due to a rapidly 
decrease of the voltage the incident is labelled as voltage instability. However, this study 
shows that other types of instability, that does not happen so often, can cause major 
problems in the system. 
The result of this studies should increase the awareness to develop platforms to assess the 
aperiodic small signal stability. The contingency plan for power system disturbance 
usually covers all the N-1 disturbance that can happen, and also contingencies that might 
affect more than one key equipment to that system, such as an important power plant or 
a transmission line that is the main tool of transferring power for on area to the another. 
==PAGE:62==
 
Conclusion 
60 
With a bigger focus on this type of stability, the contingency plan may give different 
countermeasures to those disturbances. Or also discovered new outage that at first look 
seams it will not have a big impact of the system but after a minute or two it can have 
catastrophic consequences. 
This paper also incentives the development of different platforms and software in order 
to improve the performance measures. Results can always be more accurate and faster. 
Or it could also influence the work to develop an application that not only assess the 
instability, trying to understand it, but it also suggests countermeasures to the operator. 
7.3 
Future work 
Perform a dynamic simulation to a large data requires much knowledge in dynamic 
models. To improve the work, it would be important to able to implement the assessment 
method to a true large data, as it was the initial proposal. To do that a complete 
understanding of how dynamic models and dynamic simulations works is necessary, to 
be able to cope and fix all the issues that can happen when during this step of the work. 
In addition, as future work it would also be helpful if the application could not only assess 
the stability but provide measures to prevent the instability to happen. It would be 
interesting if the application could indicate that if nothing is done the system will become 
unstable, and even suggest what should be the preventive measure. Or instead of 
suggesting the application could automatically applied the countermeasure in real-time. 
The same study that was made for the aperiodic small signal stability could be made for 
different stability assessments, such as voltage stability and transient stability. The 
application that assess the method would work different, since it would be have a different 
focus than the stability margins, but the structure of the platform form the performance 
measure would remain the same.
