Introduction
1.1
Scope and Goal
With the growing crowd density in public transportation hubs, security inspection has beôÙ
come critical in ensuring public safety from safety threatening such as terrorism. To deterôÙ
mine whether there are any prohibited items in passenger luggage, security inspections
typically use XôÙray scanners.
Operators typically donãt have a lot of time to make an assessment of whether to clear a
bag through security or not. To allow operators to work better and to create a more efficient
screening process, checkpoint screening systems must provide high resolution images,
a higher probability of automatic threat detection, and low false alarm rate. The rapid
development of deep learning, particularly, the Convolutional Neural Networks (CNN), in
recent years has brought in a revolution in image processing, including object detection
and recognition.
Conventional XôÙray scanners produce 2D images by directing these penetrating rays at
something and measuring how much energy comes out the other side. That does not
provide a layered view of a bagãs contents and they might not be smart enough to detect
sophisticated threats. When the image is limited, automatic threat detection capability is
reduced. Computed Tomography (CT) is the latest technology to be introduced to the
world of baggage screening that helps overcome this problem. It was the initial approach
of the thesis, to use CT data in order to run experiments, but later it was decided to work
with Spectral XôÙrays that we obtained from the same source that it would provide the same
data for the reconstruction of CT images.
With the ability to make more informed onôÙscreenôÙinspection and resolution, the number
of rechecks and physical inspection will be significantly cut back, resulting in a more efôÙ
fective flow of passengers through security and a lower cost of security operation. The
new technology also reduces the burden on operators, making their work more efficient
and will ultimately result in better security outcome, improved passenger experience and
operational efficiency and reduced operational costs. All these can be achieved utilizing
Artificial Intelligence, and more specifically, Deep Learning[19] methods.
The aim of the thesis, is to examine if an already tested model that automatically detects
a number of prohibited items in XôÙray images, can also detect them by providing it images
taken from a CT equipment by using Spectral XôÙrays. It is attempted to approach a 3D
imaging by providing to the model many XôÙray images from various angles of the same
luggage. A new dataset is created in order to test the model and its capacity to provide
reliable results. In this project, the classôÙbalanced hierarchical refinement (CHR)[11] is
used, which performs well in situations where there are baggage with a lot of overlapping
items. Itãs a common challenge that any security check system faces and itãs difficult to
address.

But, there was a restriction to the capabilities regarding research, since there were no

2.1
Deep Learning
2.1.1
What is a Deep Learning
Deep Learning is a subset of Machine Learning, which is a subset of Artificial intelligence
(AI). Artificial intelligence is a broad phrase for approaches that allow computers to mimic
human behavior. Machine Learning is a collection of algorithms that have been trained on
data to enable all of this. Deep Learning is a subset of Machine Learning that is inspired
by the structure of the human brain. Deep Learning systems seek to reach similar conôÙ
clusions as humans by continuously evaluating data with a predefined logical framework.
Deep learning does this through the use of a multiôÙlayered structure of algorithms known
as Neural Networks.
2.1.2
Artificial Neural Networks
Neural networks can be trained to do the same tasks on data that our brains do when
identifying patterns and classifying different sorts of information. Individual layers of neuôÙ
ral networks can also be thought of as a type of filter that works from gross to subtle,
increasing the possibility of detecting and producing a correct result. The human brain
operates in a similar manner. When we learn new knowledge, our brain attempts to comôÙ
pare it to previously known objects. Deep neural networks employ the same concept.
We can use neural networks to perform a variety of tasks in different industries such as
medicine, biology, military industries, finance, customer research and recognition science.
We can use neural networks to group or sort unlabeled data based on similarities between
samples in this data. Deep learning requires vast amounts of data and a huge amount
of processing power. But, since industries across the spectrum are generating tons of
ãBig Data,ã i.e. digital data now being produced at an unprecedented rate and massive
volumes, there is a clear opportunity for deep learningôÙpowered applications.
2.1.3
Layout of Neural Network Architecture
A simple neural network, as shown in Figure 2.1, is made up of layers and connection
weights. Communication between network members is also important for network behavôÙ
ior. In neural networks, there are three types of neuronal layers:
1. Input layer: This layer receives raw data that is fed into the network.
2. Hidden layers: Their performance is determined by the inputs as well as the weight
of the connection between them and the hidden layers. The weights between input
and hidden units determine when a hidden unit should be enabled.
3. Output layer: The output of this unit is determined by the hidden unitãs activity and
the weight of the connection between the hidden unit and the output.

2.1.4
Learning Process in a Neural Network
In order to generate some output, the input data should be fed in a neural network in a
forward manner. This process is call Forward Propagation. Each hidden layer accepts
input data, processes it according to the activation function, and then passes it on to the
next layer. At each neuron in a hidden or output layer, the processing happens in two
steps:
1. PreôÙactivation: It is a weighted sum of inputs, which is the linear transformation of
weights with respect to the available inputs. The neuron decides whether or not to
pass this information on based on the aggregated sum and activation function.
2. Activation: The activation function receives the calculated weighted sum of inputs. A
mathematical function that adds nonlinearity to a network is known as an activation
function. There are four types of activation functions: Sigmoid, Hyperbolic Tangent
(Tanh), Rectified Linear Unit (ReLU), and Softmax.
Another key aspect of neural networks is the training process and how a model is able
to optimize its weights in order to achieve higher accuracy. To do this, the network comôÙ
pares initial outputs with a provided correct answer, or target. A technique called a cost
function is used to modify initial outputs based on the degree to which they differed from
the target values. Finally, cost function results are then pushed back across all neurons
and connections to adjust the biases and weights. This pushôÙback method is called Back
Propagation. More specifically, the back propagation algorithm works by computing the
gradient of the loss function with respect to each weight by the chain rule, computing the
gradient one layer at a time, iterating backward from the last layer.

2.2
Object Detection
The task of identifying images and categorizing them into one of several predefined disôÙ
tinct classes is known as Image Recognition (or image classification). Computer Vision
(CV) is the field of study that aims to provide machines with this ability. Image RecogôÙ
nition, as one of the computer vision tasks, serves as the foundation for solving various
CV problems, such as Object Detection and Image Classification. Object Detection is the
first step wherein machines detect a certain object in an image. A step further, multiple
object detection involves locating several objects in an image by drawing bounding boxes
around them. Image Classification assigns a class label or a category to the detected
object, such as airplane, car, or bike.
Object Detection is one of the tasks in which Deep Neural Networks (DNNs)[15] excel.
The main factor that contributes to the depth of a DNN is the number of hidden layers.
Deep neural networks can contain hundreds of hidden layers, like ResNet[5], which we
will talk more later in the paper, where conventional neural networks may include no more
than three of five like AlexNet[7]. The more hidden layers a neural network has, the more
patterns it can recognise and thus the model can differentiate classes of items. Such a
class of networks are known as Convolutional Neural Networks (CNN)[1] as can be seen
in figure 2.2.
The main feature of a CNN model is the convolution layers. In every layer, specialized
filters(or kernels) are used at the input data and based on the results of those operations,
that layer has a distinguished knowledge about the input data. This process is called
feature extraction. For example, given an image, the first convolution layer may detect
edges, the second one shapes and the third one colours.

2.2.2
Core concepts of Convolutional Layers
Convolution
A convolution in the context of a convolutional neural network is a linear process that
includes multiplying a set of weights by the input, as illustrated with the figure 2.5. The
multiplication is performed between an array of input data and a twoôÙdimensional array of
weights, called a filter or a kernel. The filter is smaller than the input data, and the type of
multiplication used is a dot product between a filterôÙsized patch of the input and the filter.
When the filter is applied to the input array numerous times, the result is a twoôÙdimensional
array of output values that reflect a filtering of the input. As a result, the twoôÙdimensional
output array from this procedure is referred to as a Feature Map. Depending of the number
of kernels applied on an input, we get the corresponding number of feauture Maps.
Figure 2.5: A representation of a convolution operation.
Pooling
Pooling is performed in neural networks to reduce variance and computation complexity
by reducing the number of parameters to learn. An example can be seen at figure 2.6.
There are three main pooling methods:
1. Max pooling: The maximum pixel value of the batch is selected
2. Average pooling: The average value of all the pixels in the batch is selected.
3. Min pooling: The minimum pixel value of the batch is selected.
The term ãbatchã refers to a group of pixels with a size equal to the filter size, which
is determined by the image size or by the needs of the model. The pooling operation is
chosen based on the data available. Because the average pooling method smooths down
the image, the sharp features may be missed when using this pooling approach. Max
pooling chooses the imageãs brightest pixels. It is useful when the imageãs background
is dark and we are just interested in the imageãs lighter pixels. Min pooling is the exact
opposite of the Max operation.

Attenuation
A key feature in XôÙray imaging is attenuation. Attenuation is the reduction of the number
of particles or photons entering a body of matter by absorption(energy is transferred from
the photons to atoms of the target) and scattering. The intensity of an XôÙray beam going
through a material is determined by the thickness and the type of the material. Several
factors affect attenuation. Attenuation coefficient is a measure of how easily a volume of
material can be penetrated by a beam of light, in our case an XôÙray beam. A high attenôÙ
uation coefficient indicates that the beam is weakened as it passes through the medium,
whereas a low attenuation coefficient indicates that the medium is relatively transparent
to the beam.
Several factor have an affect on attenuation. Others are related to the XôÙray beam and
others to the properties of the matter that the beam goes through. Some of those those
factors are the energy of the XôÙray beam, the number of photons that manage to pass

There are two main interactions between XôÙrays and matter. The Photoelectric Effect and

2.3.1
Photoelectric Effect
In Photoelectric Effect, a photon interacts with an atom causing an electron to be ionized
and thus ejected from its orbit around the atom. The energy of the ejected electron is
equal with the energy of the photon minus the binding energy between the electron and
the nucleus. Since a photon is absorbed, the phenomenon is also call Photoelectric AbôÙ
sorption. The photons that are absorbed by the medium they traverse through, create
white areas on the image and those that are transmitted through are dark.
2.3.2
Compton scattering
A Compton scattering happens when a photon collides with an electron from an atomãs
outer shell, causing it to move from its orbital shell. The photon is deflected at an angle
and hence travels in a new path with less energy. The likelihood of this effect occurring
is inversely related to the XôÙray energy and independent of the atomic number. Compton
scattering has a great effect on the xôÙray image as it causes reduction on the contrast.
Those interactions between XôÙray and matter, are responsible for the information we acôÙ
quire about the composition of an object and its geometry.

Transfer Learning
For this thesis, the ResNetôÙ101 [5] architecture was used as the backbone upon which
ClassôÙBalanced Hierarchical Refinement was added on top, by executing Transfer LearnôÙ
ing. Transfer learning is the improvement of learning in a new task through the transfer of
knowledge from a related task that has already been learned. An simple illustration can
be seen at figure 3.1. This technique is popular in deep learning due to the enormous
resources required to train deep learning models or the large and challenging datasets
used to train deep learning models. Transfer learning only works in deep learning if the

with a subset of of the ImageNet[3] dataset, called ImageNet Large Scale Visual RecogôÙ

The weights in reôÙused layers can be utilized as a starting point for the training process
and changed in response to the new situation. In this context, transfer learning is regarded
as a type of weight initialization strategy. This may be advantageous when the first related
problem contains a lot more labeled data than the problem of interest, and the similarity
in the structure of the problems may be relevant in both cases. Since the layers of the
first model are initialized with preôÙdefined weights, we get the benefit of decreasing the
training time for new neural network model as we only have to train the top layers.

Residual Neural Network
The first part of the model that was used for the experiments is the Residual Neural NetôÙ
work, also known as ResNet. ResNet has many variations as you can see in the table
at figure figure 3.2. Those different variations have to do with the number of layers that
constitute the final model. In this paper, the ResNetôÙ101 was used, meaning that there
are 101 layers in the model. ResNet is a CNN made up of series of residual blocks with
skip connections. The layout of the model can be seen at figure 3.5.
The wellôÙknown vanishing gradient problem is one of the challenges that ResNet handles.
This is due to the fact that when the network is excessively deep, the gradients from which
the loss function is calculated simply drop to zero after numerous chain rule applications.
As a result, the weights in the initial layers never update their values, and hence no learnôÙ
ing occurs. Gradients in ResNet can flow directly through the skip connections from later
layers to initial filters.
Figure 3.2: Variations of the ResNet model. Each variation contains different number of
hidden layers. We can also observe that the first Convolution Layer is common for all
variations.[5]
Another issue that is addressed is the degradation problem. As we already mentioned,
the more layers we add to our models, the bigger the accuracy of the expected outcome.
It has been observed, that as we increase the number of hidden layers, the accuracy gets
saturated. This is expected, as we expect sufficient deep neural networks to model all
the details of the data. But if we move past the saturation point, the accuracy decreases.
This was observed during the training process. It was theorized that it is overfitting of
the model. But the results shown that the training error was increased instead of getting
decreased.
The problem can be solved by utilizing skip connections (or shortcuts) in order to propaôÙ
gate the information that is gain from the ãshallowã layers to the deepest ones. Let g(x)
be the function that the layers have learned. Consider the layers h(x) = g(x) + x, which
have skip connections. The skip connection is denoted by the +x term. The +x term in
h(x) = g(x) + x will bring the original value, while layer g(x) must learn only the changes
in the value, or the residue or ãx . Whatever is taught in g(x) is simply the residue,
which can be positive or negative, to change the value of x to the needed value. To make
h(x) an identity function, the residue g(x) must simply become a zero, which is relatively

Data Acquisition and Properties
4.1
Data for training and validation
For the training and validation of the model, the Security Inspection XôÙray (SIXray)[11]
Dataset was used. As a whole, the dataset consists of 1,059,231 XôÙRay images in JPG file
format as seen in Figure 4.1. The total size of the dataset is 73 GB. It contains 6 prohibited
items : guns, knives, wrenches, pliers, scissors and hammers.Figure 4.2 contains detailed
data regarding the distribution of the positive items. Out of 1,059,231 images, only 8,929
are positive samples.The hammer class includes only 60 samples and thus it is not used
in either training or validation. The rest of the dataset is negative. All the images are in
RGB and depending on the items, each one has different colouring.
Alongside the pictures, there are CSV files, one for training the model and the other is used
for the validation.The structure of those files is identical.Both contain the name of the picôÙ
tures in the first column, and the next 5 columns are filled with the 5 classes(prohibited
items). If a picture is positive in any of the 5 classes it will be labeled with 1 in the corôÙ
responding class and with ôÙ1 if it is negative, and thatãs how we declare the presence or
absence of prohibited items.
The Dataset is be divided into 3 subsets, SIXray10, SIXray100 and SIXray1000, indicating
different levels of class imbalance. The number indicates the ratio of negative images with
respect to positive ones. In the SIXray10 and SIXray100 subsets, all positive images are
used and there are 10x and 100x negative images respectively. For our experiments, only
the SIXray10 is used because it provided the necessary accuracy and it didnãt require a
lot of computational power during the training/validation process. Each subset is then
subdivided into a training set and a testing set, with the former having 80% of the images
and the latter comprising 20%.
The SIXray dataset has a number of properties that can decrease the accuracy of visual
recognition. Firstly, prohibited items can come in a wide variety of shapes, sizes and subôÙ
types, as well as many possible angles to be viewed from(mostly from the top), thus makôÙ
ing it quite misleading to tell what exactly the piece of luggage contains. Secondly,since
these images were gathered from XôÙray scans of personal luggage,the content(of the lugôÙ
gage) will inevitably be randomly arranged.Since XôÙRays can penetrate through baggage,
that allows for even the most impeded items to be seen,leading to the most prominent
property of this dataset,called overlapping. Lastly, since most of the images do not conôÙ
tain any of the five prohibited classes, statistically, guessing a negative label achieves
sufficiently high accuracy and it is possible for the training stage to bias towards the negôÙ
ative classes.

4.2
Data acquisition using a CT scanner
As it has been already mention, the equipment that was used to acquire data for the
creation of the test dataset, was meant for Computed Tomography (CT) scans. In CT
scans, an XôÙray source is circling around an object shooting beams of XôÙrays that go
through it. CT scanners use XôÙray detectors, which are located on the opposite side of
the XôÙray source. Depending on the objects, the XôÙrays that pass though the objects are
picked by the detectors.
After each revolution, all those images that had been taken from different angles(also
called projections) are stored in a sinogram. More specifically, a sinogram is a 2D array
of data containing the projections. An example of acquiring a sinogram from an object
can be seen in figure 4.3. The acquisition of these data is called forward projection. For
the purpose of this thesis, this is the point were we stop the process and we do not move
into back projection which is the reconstruction of the 3D image.
After the collection of all sinograms that correspond to the slices of an object that has
been scanned, we go through every sinogram, and we take the horizontal line(projection)
that corresponds to a specific angle one by one. Then we add each horizontal line and
we end up with a new XôÙray image. This way we acquire the XôÙray images for an object
from different perspectives.

Discussion
6.1
Results Interpretation
The results of the this study showcase that apart from one trial, none other of the remaining
approaches that were attempted were close to the ideal results. With the sole exception
of one experiment, the model was not able to provide accurate detection with the test data
that it was provided with.
One possible explanation might be the fact that because the second part of the model was
trained on XôÙray images in RGB, it was not calibrated for the grayscale XôÙray images that it
was provided with for testing. Another possible scenario might be that the model was not
introduced, during the training process, to the prohibited classes from all possible angles.
The SIXray dataset contains images that had been taken from the top of the baggage.
Lastly, the size and the synthesis of the newly created dataset itself might be factors for
the resulted outcome. Both factors have to do the the size(number of images) and the
differentiation of the prohibited items.
If we examine the first experiment closely, we will see that the majority of the detected
images of the prohibited class (pliers) that were correctly labeled were mostly noise. An
example can be seen in figure 5.3. The model interpreted the noise mostly as pliers,
even though there is not enough information. But the same phenomenon did not happen
for the knife, which was entirely missôÙlabeled. As for the card knife, even with the an
acceptable amount of information, the model was not able to label it correctly. It is safe to
assume that there was not a set of images in the training data that would resemble this
kind of knives. It is worth noticing the fact that is the only time when a prohibited class
was missôÙlabeled as a gun. As for the second experiment, there is enough contrast to
provide some information regarding the interior of the luggage, but the noise is still a key
factor and does not allow an accurate detection.
As we move to the medium energy channels, we can see that the model does not label
any of of the prohibited classes correctly. Regarding the individual items, there is missôÙ
labeling in all projections from all the samples regardless the reduced noise. The cluttered
luggage though, provide a better result, especially for the knifes and the pliers.
Finally, the high energy channels don not improve the quality of the detection. The samôÙ
ples with the individual items do not classify correctly the majority of the pliersã projections
and the knives are not labeled correctly at all. we must take into consideration the fact that
the images are becoming more transparent, meaning the XôÙray beam is passing through
the objects. The cluttered items provide again better results but still far from the acceptôÙ
able threshold.
It is safe to assume that the model works better in the medium energy channels, as the
contrastôÙtoôÙnoise ration provides detailed enough images. Also, is worth noticing that the
model performs well when there are cluttered items rather than being individually in the
luggage. In order to improve the quality of the image in this energy channel range, PrinciôÙ
ple Component Analysis (PCA) would help greatly. By performing PCA to each projection
of the scanned luggage, would would be in a position to deconstruct the image into comôÙ
ponents and reconstruct it with only those components that provide the most information.
This way we can eliminate the noise and get better results. The only downside of this

to happen for each projection.
At this point it must be acknowledged the fact that the dataset that was used for the testing
did not contain any spectral information since the samples were transformed to grayscale
images.
6.2
Further Work
In addition to the work that has been done for this thesis, there is a number of things that
can be done in order to get more robust and accurate results and of course expand the
range of capabilities of the model.
Firstly, the SIXray dataset can be turned into greyscale, or at least a part of it, and use
it as the training and validation basis. Another possibility is to expand the new dataset
with more images and include all the prohibited classes and use it also in the training and
validation process. Apart from the experiments that were conducted, more test regarding
the individual samples could be done and collect the findings for the 8 of them.
Furthermore, Spectral XôÙray images should be utilized. Spectral imaging could provide
colour on items depending on the attenuation provided by the selected energy channel.
Since different materials attenuate differently on XôÙray exposure, we would be in a position
to differentiate between prohibited classes and their counterfeit. For example a plastic
toy gun is posing no threat but right now the model can not make the distinction between
plastic and metal.
Additionally, Object Detection for 2D images could be implemented based on CHR. It
is documented that it can perform well on this task. Instead of just declaring if there is
a prohibited item in the image, it would be useful to have a box surrounding the object
of interest. Since CHR is based on transfer learning technique, any other algorithm reôÙ
lated to Object Detection can be used as the backbone, for example the latest version of
YOLOv4[17].
Lastly, since for this thesis, computerized tomography equipment was used to collect the
data necessary for the experiments, CT reconstruction should be utilized and create 3D
images. This would make possible to create robust datasets with no data imbalance
and design models that would be able to receive 3D images and detect prohibited items
in the 3ôÙdimensional space. Instead of using 2D bounding boxes we could use a 3D
box. This way we could have the exact position of an item inside a luggage. Such a
model has already been designed by researchers in Google[6] but they have applied it on
smartphones.

In this thesis we examined the possibility of using an existing Deep Learning model called

It was shown that the model could not provide meaningful results for the majority of the
experiments since a lot of missôÙclassification occurred between the prohibited classes.
