Chapter 1
Abstract
Markov Decision Processes are an essential tool to model many AI-related problems,
and it is often useful to ﬁnd a strategy that maximizes a property (goal) in the model. A
strategy is a choice to be taken for each state that maximizes the probability of reaching
the goal. The prevalent algorithm to calculate this probability and its related strategy is
Value Iteration, a reinforcement learning algorithm. Since Value Iteration may not be
feasible or usable at all on extremely large MDPs, in this paper a Genetic Algorithm
is analysed. This version aims to challenge the Value Iteration algorithm in terms of
computing speed, while providing an approximation of the true result. The results
show that this algorithm is indeed faster on MDPs without a speciﬁc structure, but
ends up falling behind Value Iteration in tree-structured MDPs. The Genetic Algorithm
used for this paper, the results and the MDPs can be found at https://github.com/
ZecchinLeonardo/DTU_thesis.

Chapter 2
Introduction
Markov Decision Processes are a core component of artiﬁcial intelligence, as they are
used to model many aspects of it, such as a problem’s search space. As it is extremely
versatile, it is equally useful to solve a plethora of real problems. Therefore, ﬁnding a
strategy that maximizes a certain property is often needed, yet ﬁnding the best one is a
np-complete problem as it requires a complete exploration of the search space, in fact,
its size increases non-polynomially with regard to the state and the non-deterministic
choice number. This is deﬁned as the state explosion problem (Clarke et al. (2012)).
This is why AI algorithms are used to solve this problem. Currently a reinforcement
learning algorithm, Value Iteration, is on average the best algorithm to face the property
maximization problem (Forejt et al. (2011)). This algorithm produces a strategy to take
for each state, which is the choice to take in order to maximize the given property.
Once a strategy is combined with the MDP, a Deterministic MDP is deﬁned as all the
non-deterministic choices are discarded.
The intention of this study is to try and challenge this algorithm with a Genetic
Algorithm that gives an approximation of the best strategy, but possibly faster. This is
because GA could supposedly handle a MDP that would be too large for VI, because
it would work with its Markov Chains instead of the whole MDP. The algorithm has a
population of possible strategies, checks their ﬁtness based on how high the probability
of the property is, combines and mutates these strategies, and ﬁnally produces a new
population of strategies for the next round. Once a probability of 1 is reached or the
generations are over, the algorithm returns the best strategies found throughout the
whole evolution.
In order to encode Markov Chains (that from now on will be referred to as "strate-
gies") that could be used for crossover, each strategy is composed of a sequence of ﬂoat-
ing point numbers between 0 and 1 (genes), which form a chromosome (see Section
4.1, Unity Distribution). This system is also easy to mutate (see Section 4.2, Uniform
Shrink). The algorithm has been tested on different sized MDPs with many different
settings, the results are reported in Chapter 5: Results, and they are globally analyzed
in Chapter 6: Conclusion.

Chapter 3
Background
3.1
Markov Decision Processes
Markov Decision Processes, or MDPs, are models used to infer results from stochas-
tic processes. They use states which are connected by non-deterministic choices which
can be taken arbitrarily by using a strategy, in other words, given a state, a strategy can
be formulated such that a choice is always taken such that the non-determinism is re-
moved. Finally, a choice connects a state to a set of transitions, which has a destination
state and a probability to be picked (randomly by the environment). Certain states can
have labels to represent goal states and failure states.
3.2
Value Iteration
Value iteration is a reinforcement learning algorithm that tends to work better than other
algorithms of its type for solving MDPs maximization problems Forejt et al. (2011).
It starts with a random value function and, for each state, it updates the value function
with the best choice value while looking ahead. Instead of evaluating and then improv-
ing, the value iteration algorithm updates the state value function in a single step. This
is possible by calculating all possible rewards by looking ahead. Once the value func-
tion is updated within a given ε, the algorithm has converged and returns the optimal
policy.
3.3
Genetic Algorithms
Genetic algorithms are a branch of artiﬁcial intelligence algorithms that are based on
the concept of evolution. A randomly-generated population set is created and tested on
a problem, the set’s results are ranked based on the ﬁtness of the given output, and then
used to select the best elements in the set to crossover and mutate to populate the next
generation’s set. Crossover are Mutation are essential in order to balance exploration
and exploitation which is a core concept of search algorithms ( ˇCrepinšek et al. (2013)).
This behaviour resembles the natural evolution of life that adapts to an environment. In
this context, the population members are strategies (or chromosomes) that are ranked
based on the probability of reaching a goal state; the probability is treated as a ﬁtness
function and thus maximization is encouraged. If a chromosome is a strategy, then a

3.3.1
Selection
Selection is the ﬁrst phase of the GA’s evolution, in which the members of the former
generation’s population are selected to create an offspring for the next generation. The
selection methods in general favour the population member with the highest ﬁtness.
Steady State
Steady state selection is a type of selection that chooses chromosomes that have a ﬁtness
higher than some criteria, which in this case is above average in a given generation.
These chromosomes then go through crossover and mutation for the next generation.
3.3.2
Crossover
Crossover is the second phase of the GA’s evolution, in which two parents are chosen
from the selection phase and mixed in order to produce a new offspring for the next
generation.
Cycle crossover
Crossovers have a start and end index. Cycle crossover copies all genes before the start
index from the ﬁrst parent, then copies the genes with index between the start and end
index from the second parent and ﬁnally the last ones from the end index to the actual
end of the chromosome from the ﬁrst parent. Every time a gene is copied it is done
at the index equal to the length of the array that is currently being ﬁlled (not from the
start).

3.3.3
Order crossover
The behaviour is similar to the previous one but each parent has an index that starts from
the beginning and when a gene is copied from one of them only that index increases.
3.3.4
Mutation
Mutation is the third and ﬁnal phase of the GA’s evolution, in which the crossed-over’s
offspring is randomly mutated to ensure a sufﬁcient exploration of the search space.
One gene mutation
With this setting, only one gene can be mutated. This is highly ineffective with large
chromosomes because the search space is much bigger and there’s not enough explo-
ration. This has been implemented but not used.
All genes mutation
With this setting, all genes are mutated randomly. This is still quite ineffective because
the ﬁnal result strays too far from the high-ﬁtness parents and ends up being a random
result. This has been implemented but not used.
Adaptive Mutation
With this setting, all genes can be mutated with a certain probability. Each generation
the average ﬁtness is computed and if the ﬁtness of each individual is higher than the
average, each gene has a lower chance of being mutated. If the individual has a lower
than average ﬁtness, each gene has a higher chance of being mutated. This system en-
courages mutations with ineffective strategies and discourages mutations with effective
ones.

Chapter 4
Encoding
In order to build a GA on an MDP, the latter needs to be encoded in a precise and clear
way such that it works with the former. The task however, is not trivial. Traditionally,
in GA, the problem is adapted to a bitvector which can be manipulated using selection,
crossover and mutation. In a general MDP case, however, this is not possible. Suppose
that there are two states in an MDP, one with 2 choices such that choosing one of the
choices could be encoded with a single bit, and one with 9 choices, which would need
4 bits. The issues that arise with a classical approach are multiple: ﬁrstly, since we
encode the choice taken, these have variable length and we need to account for this.
But the ﬁrst issue leads to the second: crossover is impossible. Suppose we want to
perform crossover on the aforementioned example, the ordering does not change the
choice we are referring to, since they are not of compatible length. Suppose we had a
simple strategy of 1|0010 ("|" marks the end of a chromosome and the beginning of the
next one), changing the order to 0100|1 is meaningless as we are still referring to the
same choice as before (there is no other valid interpretation). This leads to yet another
issue: mutation. Suppose we wanted to apply bit inversion to the previous example:
this would leave us with 0|1101, but there are not 14 choices in the second state, there
are only 9. The modulo operation can be used to ﬁx this (again, this implies that we
need the number of choices that can be taken for every state), but overall these issues
force us to use a different, non-trivial approach to the problem, which we will refer to
as "Unity Distribution".
4.1

To circumvent the issues we faced in the previous point, the solution proposed is to
distribute the choices in continuous intervals between 0 and 1 of equal length, which is
1/|As| where As is the set of outgoing choices from a given state s. Formally:

4.2
Uniform Shrink
Now we will see that this encoding works as-is with crossover and mutation operators
(selection is independent of the encoding). Mutations are typically bit ﬂips, but the
genes are real numbers, therefore we cannot just ﬂip them, it would not explore the en-
tire search space. An approach found in the literature to mutate genes with continuous
distributions is a "Shrink" operation, which entails a mutation using a random num-
ber obtained from a Gaussian distribution that models the chromosome values. This
approach is still not ideal for this problem because a Gaussian distribution does not
model the Unity Distribution approach correctly. Therefore, we will use an operator
that we will refer to as "Uniform Shrink". Instead of drawing from a Gaussian distri-
bution we will use a uniformly random number between 0 and 1, we will add it to the
original value and we will use the modulo operator such that the ﬁnal value will fall in
the [0,1] interval. This ensures that the entire search space is in the scope of the mu-
tation operator. Finally, changing order between chromosomes also works because the
value is still the same, but it now falls under a new interval which represents another
choice from another state.

Chapter 6
Conclusion
6.1
Discussion and conclusion
Taking into account the results mentioned in chapter 5, we can summarily judge how
the algorithm performs. In randomly connected MDPs, the result is on average better
than VI, speciﬁcally, up to 2.5 times faster (hp 5.2.2) and with an approximation ratio
that is proportional to the number of non-deterministic choices leading to the goal (hp
5.2.7, hp 5.2.8). This is because many ways to reach the goal ensure a higher probability
of ﬁnding a strategy that has probability 1 of reaching it (hp 5.2.1). In tree-structured
MDPs however, the VI algorithm wins by a large margin. The population size (hp
5.2.3) and generation size (hp 5.2.4) can be tweaked to have a better approximation
result for the GA but it is never comparable in terms of speed to VI (hp 5.2.5). Some
parameters like the probability of mutating each state’s strategy when the ﬁtness is
above the population’s average can also be tweaked to try and increase the accuracy of
the GA (hp 5.2.6). 10% works well with the smaller and medium MDP tested but falls
off with the larger one, while 20% always yields satisfying results. Higher mutation
rates tend to always underperform, this is supposedly because of bigger changes that
stray too much from the good strategies. Finally, the worst case for the GA is when
there is a single strategy that leads to the goal in a tree-structure. Any strategy that does
not include the goal will have ﬁtness 0, so evolution is impossible, it is simply a blind
guess among a huge search space.
Unfortunately, this version of the GA is not able to handle MDPs that would be
too large for VI because the current encoding makes handling the huge strategies too
cumbersome and thus slows the evolution down too much.
6.2
Future Work
In this paper, GA and VI were tested on problems that both could handle, but VI (and
this version of the GA) are not scalable to very large MDPs. The results on tree-
structured MDPs for the GA need improvement; in the past a different encoding was
tested to support a more efﬁcient application of this algorithm (Godefroid and Khurshid
(2002)), but it was on a restricted version with just 1 transition per non-deterministic
choice. This restriction is justiﬁed because the solution proposed had a strategy that
is composed of choices to be taken sequentially and not one strategy for each state
like in this paper. Unfortunately if we consider an unrestricted version of the problem,
this solution cannot work as the same strategy would yield different results based on
