Abstract
This thesis is a further study of the spatial integration method ﬁrst introduced
by Dean and Hutchinson and then further studied with an inclusion of self-
similarity and a history dependant behaviour found in the Ph.D thesis made
by Kristian J. Juul. This thesis expands the 2D self-similar framework to 3D
with 20-node elements. It is shown that it is possible to get the same results
in 3D and thus showing that the model can be expanded from a plane strain
condtion to be a full state constitutive relation,

1
Introduction
This master thesis is a continuation of the project worked out in Kristian Jør-
gensen Juul’s Ph.D thesis[1]. Which is a further development of the work carried
out by Dean and Hutchinson in their paper "Quasi-static Crack growth in small
scale yielding"[2], where they found a way to describe the development of stress
for a perfectly plastic problem using a steady state condition which followed the
tip growth of a crack. Juul further develops on this idea by working out a way
to develop this system in a self-similar environment, the system can be described
by only one independent variable. Self-similar problems makes it possible to use a
spatial integration method to solve plasticity, due to the conditions that are in place
within such a framework. A self-similar environments physical ﬁelds will remain
unchanging in time, for a coordinate system that follows the path of a crack or
an indenter. Making it possible to make use of an integration technique that only
considers spatial derivative and not time derivative.
By using the methods that Dean and Hutchinson created to transform variables that
are usually expressed on a traditional incremental form in time derivatives, to a form
where only spatial derivatives are considered, it is then possible to ﬁnd the usual
history dependence from these variables by integrating along lines that describe
the material ﬂow.
However Dean and Hutchinsons methods in the steady-state
environment only takes one length scale into account and is not directly applicable
in industrial methods.
By making this in the self-similar environment developed by Juul with multiple
length scales dependent of only one variable scaling factor. This spatial integration
method becomes a powerful tool for analysing processing methods such as drawing
rolling and welding, by making it possible solve a fast numerical models of these
methods. It is possible to ﬁnd the pre-settings of the machines, by numerical means
using these methods and thus potentially saving months of preparation time in ﬁne
tuning these machines. With Juuls thesis he proved that there was a signiﬁcant
time-saving potential in solving these kinds of systems, he also found that there is
no loss in accuracy associated to the models.
The current limitations of the model is that they are based of the assumption of
small strains and it is thus still a problem for major manufacturing processes where
the strain can easily exceed 10%. However the development of these methods is
nowhere ﬁnished and still has a huge potential for manufacturing if these methods
are developed further.

1.1
Motivation
Iterative models are normally very computationally heavy when referring to the ﬁnite
element method. However there is a huge potential in reducing this computational
time when working with problems of a self-similar or steady state formulation, as the
need for continuously updating the stiﬀness matrix will not be necessary, by using
the elastic solution as a starting point for an iterative procedure only accounting for
the gradient in the displacement ﬁeld, it is possible to ﬁnd the plastic ﬁeld by only
adjusting for the gradients in the stress and strain across the solution. So far this
method has only been developed for 2D ﬁrst by Dean and Hutchinson in 1980 and
then by Juul in 2018. But they have shown that the method works and that There
is a huge time saving for running simulations within this framework.
With more development this method should be able to take over most plastic iter-
ative models for self-similar or steady state behaviour such as rolling, welding and
crack problems, as methods used today can take several days to solve to ﬁnd ﬁne
tuning parameters for machinery.
1.2
Problem Statement
This project will focus on expanding the self-similar framework by Juul to work in
a 3D framework, thus expanding the potential to indenters that aren’t limited to
an assumption of plain strain or plain stress model. The goal is to get a working
model, and to show the possibility of further development of the framework.

2
Theory
The methods that are included in this project, will focus on those that exhibit self-
similarity. Where the ﬁelds of deformation scale with the solution of the problem. In
these problems it is possible to change the iterative solution from a time derivatives
to a spatial derivative. This has several beneﬁts for models such as a ﬁnite element
model. Here becomes possible to keep the stiﬀness matrix constant in each iteration
and only the ﬁeld quantities will change till convergence is achieved.

Spatial integration method
Solving the self-similar system with iterative spatial derivatives is much like the
method for solving a steady state problem. Juul[1, p 15] describes how to alter the
integration method, to ﬁt the unique characteristics of the self-similar problem. The
integration lines of the self-similar problem can be placed anywhere in the model,
however they have to follow the ﬂow of material, meaning that the mesh would best
ﬁt a self-similar problem if the mesh is swept about the point of impact, if this is
not possible, the streamlines will have to be extrapolated to form a line following
the material ﬂow, but this can be quite extensive. The altered spatial integration
model follows 5 steps.
1. Determine the current displacement ﬁeld, for the ﬁrst iteration this is just the
elastic solution of the problem.
2. Determine the gradients of this displacement ﬁeld.
3. Determine the ﬁeld quantities outside the spatial integration domain with only
elastic relations.
4. Determine values of the ﬁeld quantities inside the spatial integration domain.

3
Implementation
To introduce these methods it is ﬁrst important to build a ﬁnite element model.
The ﬁnite element model is to be displacement based to make the shapes of the
indenters.
3.1
3-D element implementation
Most of the theory in the ﬁnite element method remains the same when going from
2 to 3 dimensions, the largest change is the size of the systems, an 8 node plane
element will have a local stiﬀness matrix of the size 16x16. For a 20 node solid
element this will turn into a 60x60 matrix as it includes an extra dimension per
node. likewise the shape functions are dependent of an extra variable and goes from
a 16x2 to a 60x3 system, when used in calculations.
Figure 3.1: Illustration of 20-Node element, with node numbering for connectivity
The ﬁrst 8 nodes of the 20-node solid element are corner nodes and thus have 3
connections per node, whilst the mid-line nodes are only connected to 2 nodes. An
isoparametric element like this is deﬁned by 20 shape-functions, which is used to
interpolate the solution between discrete values obtained by the mesh. Isoparametric
elements makes it possible to discretise a mesh with several diﬀerent shapes as the
shape-functions account for variations in shape and form, even making it possible
to collapse certain nodes thus creating diﬀerent shapes such as pyramids or other
triangular shapes.

3.4
Indenter
There are many diﬀerent indenter types that would ﬁt a self-similar condition. The
type implemented in 2D is known as a wedge indenter, this indenter is self-similar
if there is no deformation in the z-direction.
Other types of indenters could be pyramid or conic shaped where there is only
one characteristic length that can describe the indentation. Meaning the rate of
indentation and the width of the indentation can remain constant in a moving
reference.
3.4.1
Wedge-indenter
The next natural step of developing the 3D model is to make the indenter. There
are many ways to incorporate an indenter for a self-similar problem. The indenter
can be conic in shape or be shaped like a pyramid. Another possibility is to make
the indenter as it is introduced in 2D but on a extruded plane where the ends are
symmetric on the z axis. The x-y plane remains unchanged from the 2D case and
thus the results of the 3D model should align almost perfectly with what can be
found with an 8-node 2D elastic model.

3.5
Streamline deﬁnition
To make the streamlines through the elements stated in section 2.2.1 it is important
that the Gauss-points align throughout the elements so that it is possible to create
a line from outside the plastic domain with an indenter, should make all elements
propagate towards the indenter tip. An example of how this could be done for a
wedge indenter is seen in ﬁg. 3.6
Figure 3.6: Mesh for a wedge-indenter where no change of the mesh occurs in the
z-direction, extruded along the z axis.
The mesh that this project continues working on has a thickness of 1 element mean-
ing that there are going to be twice as many streamlines as there are in a 2-D model
of the same mesh quality, as each element will include 8 Gauss points instead of
4. Assuming the mesh is made in a sweep, where there is no need to extrapolate
the Gauss points. Each streamline will pierce 2 Gauss points for every element they
pass, starting from the edge of the elastic domain moving towards the plastic domain
to the tip of the indenter.
Most of the mesh is made with Ansys mechanical apdl, however it was not possible
to sweep the entire mesh, as Ansys could not ﬁgure out a way to collapse the last
elements, and it was thus necessary to make a function to make the last elements
in the mesh.

The mesh only has to have one element in the z direction. The nodes of the elements
facing the tip are found by taking the radial distance to the point (0,0,z), the nodes
are then sorted according to their z value. Mid line nodes are then created from the
corner nodes of every element, all the elements have collapsed nodes at (0,0,z) and
the new elements created are thus in a long triangular bar shape.
With the mesh in place the streamlines are found by looking through every element
and ﬁnding the coordinates of each Gauss point. These Gauss points then have
to be sorted by angle and for the wedge indenter they also have to be sorted with
respect to their z value. After sorting these Gauss points by angle and length of z,
they are sorted for the radial distance towards the point where the indenter touches
the domain. When this is done it is now possible to make the spatial integration,
by starting with a Gauss point from outside the plastic domain.

Discussion
5.1
Pyramid and conic indenters
Although not implemented, a pyramid or cylindrical indenter type should be self-
similar, however, streamlines and domain has to be slightly altered to account for
the shape of this indenter. All streamlines have to move towards the tip of this
indenter, meaning the mesh should be a half sphere, boundary conditions on this
problem should be rather easy to implement as a full constraint along the edges of
this sphere, however Ansys had major issues in making a mesh that did not have
distorted lines somewhere in the mesh.
(a) Swept mesh made in ansys
(b) Section of the mesh with issues
Figure 5.1: Mesh made with ansys for diﬀerent self-similar indenters
Initially the mesh looked ﬁne especially the top plane ﬁg. 5.1a, however when in-
specting the inner parts of the mesh ﬁg. 5.1b, it is clear that the sweep is not
perfect, here lines are slightly twisted when moving towards the centre, meaning
that to make streamlines most of the points of the streamline would have to be
extrapolated, instead of just being located on the Gauss points.
Just like the mesh for the wedge indenter it could not sweep the mesh without cutting
out the very tip of the domain, meaning it is necessary to mesh a tip just like for
the wedge-indenter. However due to the other issues encountered in Ansys it might
be easier to program the entire mesh instead of just creating the tip separately.
The streamlines of an indenter of this type, should unlike the wedge and 2D problem
be sorted by spherical coordinates. After being sorted by the 2 angles, they should
then be sorted by the radial distance to the tip of the indenter.

5.2
Future work
There are many implementations that could be done to this framework. One being
making more indenter types such as pyramid, conic and triangular. Secondly making
the framework valid for large deformations would increase the possibilities of this
framework to be used in industry, without the need for updating the stiﬀness matrix
between iterations, it would be able to compute the plasticity much faster than
the models that are currently used in the industry, for ﬁne tuning and setting up
production machinery that exhibit the self-similar or steady state behaviour.
Another thing to consider is testing the robustness of the model, so far there has
been no issues for getting convergence if there has been both a plastic and elastic
domain, However, the eﬀects on the convergence with changing mesh size has not
been shown for the 3D-model.
6
Conclusion
With this project it is shown that the Theory developed by Juul has no issues in
being extended to work in a 3D framework, as there is almost no loss in accuracy
and convergence is still very fast considering the huge change in size of the systems.
The method of streamline-integration has a huge potential, as 3D models can go
beyond plane stress and strain assumptions. It is possible to describe more complex
geometries and indentations, thus further expanding the possibilities of using this
method for making much quicker simulations, then the current methods in use. One
of the biggest issues with a 3D FEM model is that the stiﬀness matrix will become
huge with higher resolutions, but this method will only need to build the system
once, then all changes between iterations occur only in corrections of the loads and
displacements. The convergence of the streamline integration is also incredibly fast,
as the use of the elastic solution serves as a good starting point for the actual plastic
ﬁeld.
There is still much work to be done in the development of these methods. The 3D
model has much potential, and by expanding the theory from small strain theory
to ﬁnite strain theory it should also be possible to make the models work for larger
deformations, however, this method does show a huge potential.
