Abstract
In this report methods for estimation and control of a 2-tank system are explored, and the
solution can possibly be extended to include more tanks, input, and outputs.
Methods for state estimation and rain disturbance modelling are developed, and design of a
full state feedback controller of these state estimates are developed.
Lastly parameter and structure estimation methods are developed, on the basis of simple
least squares estimation, and model selection of the basis of F-test.

Introduction
In most modern sewer systems the load of waste water (sewage and rainwater) going into
them can be quite large, which can result in the overÔ¨āow of waste water onto the above-
ground surface. Having tanks along the sewer system going to the waste water treatment
plant (WWTP) can help mediate this overÔ¨āow with optimal level control of said tanks.
Optimal level control of tanks in an entire sewer system is however not an arbitrary problem
to solve, given enough tanks, Ô¨āow delays due to pipes, and restrictions on tank outputs and
control, the solution quickly becomes very complicated.
Modelling and subsequent order reduction of large sewer systems can be very cumbersome
and each model will always contain some model uncertainty, by using system identiÔ¨Ācation
and estimation this process could potentially be automated. If the estimated system stays
somewhat constant the control design would then be simpliÔ¨Āed.
The practise of stochastic adaptive control revolves around system identiÔ¨Ācation, estima-
tion and control of systems with noise and/or uncertainties. This project will attempt to
apply these principles to the benchmark sewer system Astlingen [1] by modelling, estimating
and controlling a multi-tank system representing the Astlingen model.
See appendix 7.1

can be seen below and in Ô¨Āgure 1.1.1 respectively.

1.5.3
Controllability and Observability
Having less inputs than outputs could suggest that the system is not completely control-
lable, and having less outputs than states could suggest that the system is not completely
observable.
Checking the controllability and observability of the system yields that the system has 10
controllable states out of 18, and 18 observable states out of 18.
The reason for the system being completely observable, is because of the integrating struc-
ture of the system. Each integrator state terminates at a state that is visible through the
14
==PAGE:19==
output matrix. The system is actually in an Observer Cononical form.
The reason for the system not being completely controllable is not immediately obvious,
but is due to the Ô¨āow delays not running into Ô¨āow from controlled sources. At tank 5 (see
Ô¨Āgure 1.1.1) there is a Ô¨āow delay with no time delay, meaning that no extra state is created.
Counting up the uncontrollable Ô¨āows:
‚ÄĘ The outgoing Ô¨āow of tank 5 is uncontrolled, created 1 uncontrolled delay state.
‚ÄĘ The Ô¨āow delays of 10 minutes to tank 4 Ô¨āows together, and creates 2 Ô¨āow delay states
of 1 and 2.
‚ÄĘ Tank 6 have 3 uncontrolled delay states created from an incoming Ô¨āow.
‚ÄĘ Tank 1 have 2 uncontrolled delay states created at 20 and 15 minutes, one originating
from tank 5, and one from a disturbance Ô¨āow.
Counting up the created uncontrolled delay states we get to 8, accounting for the all un-
controlled states when calculating controllability. The reason why the state of the outgoing
Ô¨āow of tank 1 is not considered uncontrollable, is because the outgoing Ô¨āow of tank 1 is
dependant on the volume, and the Ô¨āows entering and Ô¨Ālling up the tank is controllable.
Additionally ignoring the disturbance matrix G, and computing the minimal realization
thereof, yields a state space description with 9 states, 4 control inputs, and 6 outputs. Check-

meaning that the minimal realization has 10 controllable states out of 10, and 10 observable
states out of 10, conÔ¨Ārming that the uncontrollable states come from the disturbances and
not from the rest of the tank model.

In this chapter we explore structure estimation by model selection, with least squares estima-

5.1
Design of model selection algorithm
Going into model selection there is a need for training, test, and validation data, then either a
maximum model order or a stopping criterion, and Ô¨Ānally a loss function of score to evaluate
how well a certain model performed.
Then the algorithm goes over each output as follows:
1. Generate training, test, and validation data
2. (if the stopping condition is false) Select a model S
(a) Generate regressor ő¶train from training data using model S
(b) Estimate parameters ňÜőł using global LS
(c) Generate regressor ő¶test from test data using model S
(d) Calculate prediction error őĶ = Ytest ‚ąíő¶testňÜőł
(e) Calculate model performance P using őĶ
3. Select model S with best performance P
4. Generate ő¶ with training and test data using model S
5. Estimate parameters ňÜőł using global LS
6. Generate ő¶val from validation data using S
7. Calculate prediction error őĶ = Yval ‚ąíő¶valňÜőł and validate the selected model.

Chapter 6
Conclusion
Designing a state feedback controller for a linear fully estimated tank system can be very
simple and eÔ¨Écient, however once the water Ô¨āows of the tank system behaves nonlinearly,
then control becomes diÔ¨Écult because of state estimation error. With more complex state
observer design, it might be possible to estimate states better and thereby improving control.
The nonlinear tank system has complicated estimated structures and parameters, but it is
still possible to predict the tank volumes with simple LS estimation over time, although a
bit more imprecise than for a linear tank system.
To summarize
‚ÄĘ Full state feedback control is easy given good state estimates, and full state feedback
control still works well in nonlinear tank systems given good state estimation.
‚ÄĘ Parameter, structure, and state estimation of a linear system is simple, however in
nonlinear tank systems structure and parameter estimation is complicated.
‚ÄĘ An estimated nonlinear tank system can still predict some tank volumes well, even
with simple LS estimation.
6.1
Further development
In order to simplify the modelled tank system, it is possible to include the diÔ¨Äerent sub-
catchment areas in the rain disturbance gain. This way there is only need for a single rain
disturbance signal, that being the rain forecast.
If the subcatchment areas are assumed to be included in the model, it would also be possible
to estimate these, given knowledge about the sampling time.
Another disturbance to be included in modelling and estimation, could be the waste water
Ô¨āow from houses and industry.
When it comes to the nonlinear Ô¨āow a few thing could be done. This could be simply making
a noise model for the nonlinearity, using diÔ¨Äerent artiÔ¨Ācial regressor data during estimation,
or maybe abandoning the simple LS estimation method altogether.
