Abstract
The segment rank and selection problem consists of preprocessing a set of n horizontal line
segments in the plane, such that given a point in the plane, we can compute the number
of segments intersecting the vertical line going up to the point and find the jth intersecting
segment on the vertical line. We present a succinct solution to the segment rank and selection
problem in rank space [1, 2n] × [1, n] using 2n log2 n + O(n) bits of space and can answer
segment rank and select queries in O(log n) time. This is slightly slower compared to the
optimal query time O(log n/ log log n) when using space O(n logO(1) n) space [3]. We also
provide a succinct solution for horizontal line segments in the plane [1, U] × [1, U] using

Introduction
The segment rank problem is to preprocess a set of n horizontal line segments in the plane into a
representation such that given a point (x, y) we can find the number of segments intersecting the
vertical line from (x, 1) to (x, y). The segment selection problem is to return the jth intersecting
segment ordered by increasing y-coordinate among the segments intersecting the vertical line for
a given x-coordinate. The segment selection query is the primary query used to answer random
access queries on persistent strings and was first considered in conjunction with persistent
strings [3]. The segment selection problem can be solved in O(n) space and O(log n/ log log n)
query time, as shown by Bille and Gørtz [3]. Furthermore, they showed that this is optimal for
any data structure using n logO(1) n space.
The random access problem is to preprocess a data set to support fast retrieval of any part of the
data set. A persistent string is a collection of strings derived from a rooted tree called the version
tree. The version tree is a rooted tree where each edge is labeled with an edit operations and
each node represents the string achieved from applying the edit operations on the path from the
root to the node on an empty string. Given a node v in the version tree and index j, a random
access query returns the character at position j in the string represented by v. It has been
shown that random access in persistent strings can be reduced to segment selection [3]. Access,
rank, and select queries stand at the core of many compact and compressed data structures,
and are used in various scenarios for instance, document listing in sequence collections [10] and
XPath search queries on XML [2]. As such we will consider rank and selection on persistent
strings as well. For a node v in the version tree, index j, and character α, we say that the
persistent string rank query returns the number of occurrences of the character α in the first j
characters of the string represented by v. For the same input, the persistent string select query
returns the index of the jth α character in the string represented by v.
In recent years there has been an increasing interest in succinct data structures. A succinct
data structure is a data structure that uses space close to the optimal space needed to represent
the data according to the information-theoretical concept of worst-case entropy, while still
supporting efficient queries on the represented data [7].
In this paper, we show the worst-case entropy for a set of n horizontal line segments in rank
space and introduce a new succinct representation of horizontal line segments in rank space.
Our representation uses 2n lg n + O(n) bits of space and supports segment rank and select in
O(lg n) time, where lg n = log2 n. Using our representation for horizontal line segments we
show two representations of a persistent string of n edit operations from the alphabet [1, σ].
The first representation uses 4n lg n + n lg σ + O(n) bits of space and can answer random access
in O(lg n) time. The second representation uses 8n lg n + 4n lg σ + O(n + σ) bits of space and
has O(lg n + lg σ) query time for random access, rank, and select. Throughout this paper we
will use lg n = log2 n. The next section will formally define the concepts of segment rank and
select and then random access, rank, and select on persistent strings.
1.1
Segment Rank and Selection
Let L be a set of n horizontal line segments in the plane.
We say that a horizontal line
segment (x1, x2, y) crosses the vertical line through x-coordinate i iff x1 ≤i < x2. We will
refer to (x1, y) and (x2, y) as the left and right endpoint of a segment respectively. The segment
selection problem is to preprocess L to support the operation:
• segment-select(i, j): return the jth segment according to increasing y-coordinate among
the segments crossing the vertical line through x-coordinate i.

It has previously [3] been shown that the segment selection problem can be solved in O(n) space
and O(log n/ log log n) query time. Additionally, we will work on a problem that we call the
segment rank problem that is the problem of preprocessing L to support the operation:
• segment-rank(i, y): return the number of line segments crossing the vertical line through
x-coordinate i with a y-coordinate in [1, y].
This is a natural extension to the segment selection problem since segment-select(i, segment-rank(i, y)) =
y for a segment (x1, x2, y) ∈L where x1 ≤i < x2, that is segment-rank is the dual of segment-
select.
Although it was left unmentioned in [3] it follows trivially by their results that the
segment rank problem can be solved in O(n) space and O(log n/ log log n) using their data
structure.
We will investigate the segment rank and selection problems in the rank space
[1, 2n] × [1, n] such that there is exactly one endpoint on each x-coordinate and no segments
with the same y-coordinate are overlapping. We present a succinct representation of the segment
rank and selection problem in rank space that achieves the following bounds:

Access, Rank and Select On Persistent Strings
Given an alphabet [1, σ] and a version tree T consisting of n nodes where each edge is labeled
with one of the edit operation:
• replace(k, α) : replace the kth character with α ∈[1, σ].
• insert(k, α) : insert character α ∈[1, σ] after position k.
• delete(k) : delete the kth character.
Let v be a node in T, then the string S(v) is the string achieved from applying the edit operations
on the path from the root to v on an empty string ε. Our goal is to preprocess T to support
the queries:
• access(v, i): returns the character S(v)[i], for any 1 ≤i ≤|S(v)|.
• rank(v, α, i): returns the number occurrences of α in S(v)[1, i], for any 0 ≤i ≤|S(v)|.
• select(v, α, i): returns the position in S(v) of the ith occurrence of α, for any i ≥0.

1.3
Previous Work
There already exists a solution to the segment rank and selection problem using O(n) space
and O(log n/ log log n) query time [3]. Still, to the best of our knowledge, no prior work has
been done on answering segment rank and select succinctly. There exists a related query called
the stabbing-semigroup query. For a set of n intervals where each has an associated weight,
for a given point, the stabbing-semigroup query asks for computing the sum of weights of the
intervals intersecting that point.
The stabbing-semigroup problem has been solved in O(n)
space and O(log n) query time [1].
They also showed how to make the structure dynamic
allowing adding and removing intervals in O(log n) time and how it can be adapted to work
in external memory. Another related query working on intervals is the stabbing max, which
is the problem of finding the max weight interval intersecting a point [12]. To the best of our
knowledge, neither of these problems has been considered in a succinct setting. Many geometric
queries on two-dimensional points have been considered in a succinct setting [11]. Their results
are however not directly applicable to horizontal line segments, since a horizontal line segment
(x1, x2, y) can be considered to be present as the points (x, y) for x1 ≤x < x2.

Preliminaries
In this section, we will provide the reader with the preliminary knowledge required for our
proposed data structure. The concept of succinct and compact data structures begins with

an understanding of worst-case entropy.
Let the set U be the set of all instances of some
problem P. The worst-case entropy for a problem P is defined as Hwc(U) = ⌈lg |U|⌉and is the
minimum number of bits required such that each instance of P can have a unique identifier if
all identifiers have equal length [9]. For a data structure to be called compact in regards to the
problem P it has to use O(Hwc(U)) bits of space, furthermore for it to be called succinct it has
to use Hwc(U) + o(Hwc(U)) bits of space [7, 9]. In the next sections, we will show two different
succinct data structures.
2.1
Bitvectors
A bitvector is a type of bit array B[1, n] such that we can perform random access, rank and
select queries on the bits in B. We define rank and select as follows on a bitvector B[1, n]:
• rank(B, b, i) : return the number of occurrences of bit b ∈[0, 1] in B[1, i], for any 0 ≤i ≤
|B|. In particular, rank(B, b, 0) = 0.
• select(B, b, i) : return the position in B of the ith occurrence of bit b, for any i ≥0. In
particular, select(B, b, 0) = 0 and for any j > rank(B, b, |B|) that select(B, b, j) = |B| + 1.
Bitvectors are a well-studied data structure that is fundamental to most compact data struc-
tures. We will be using bitvectors with the following properties:
Lemma 1. A bitvector of the bit array B[1, n] uses n+o(n) bits of space and can answer access,
rank, and select queries in constant time [8].
In many use cases, B will contain mostly 0s and only a few 1s. This has led to the study of
bit arrays B[1, n] where only m positions have a 1 bit, where we are only interested in answering
rank and select queries on the 1 bits. This is usually referred to as a compressed rank and select
dictionary. A particular compressed rank and select dictionary invented by Okanohara and
Sadakane have the following properties:
Lemma 2. A bitvector of the bit array B[1, n] such that m positions have a 1 bit, can be repre-
sented using m lg n
m + O(m) bits of space and can answer rank queries in O(lg n
m + lg4 m/ lg n)

The next data structure we look at generalizes rank and select queries for a sequence of
characters and makes heavy use of bitvectors to support these queries efficiently.
2.2
Wavelet Tree
A wavelet tree [6] is a structure on a sequence S[1, n] of n symbols from the alphabet [1, σ] such
that we can perform random access, rank and select on S, where rank and select are defined as:
• rank(S, α, i) : return the number of occurrences of character α ∈[1, σ] in S[1, i], for any
0 ≤i ≤|S|. In particular, rank(S, α, 0) = 0.
• select(S, α, i) : return the position in S of the ith occurrence of character α ∈[1, σ],
for any i ≥0.
In particular, select(S, α, 0) = 0 and for any j > rank(S, α, |S|) that
select(S, α, j) = |S| + 1.
The wavelet tree is a type of binary tree that partitions the alphabet [1, σ]. Each node v in the
wavelet tree represents but does not store an alphabet Σ(v) = [a, b] ⊆[1, σ] and a subsequence
S(v) such that S(v) is the subsequence of the symbols in S from the alphabet [a, b] in the order
they appear in S. The alphabet associated with the root is [1, σ] and for any non-leaf node

Σ(v1) = [a + ⌊(b −a)/2⌋+ 1, b]. Furthermore, for any leaf node v, that Σ(v) = [c, c] where

Segment Wavelet Tree
In this section, we will describe our proposed solution to the segment rank and selection problem
in rank space. We will then show that we can generalize this solution to any space. We will begin
by describing the content of our data structure and then show how we can answer segment-select
and segment-rank queries using this structure.
4.1
Data Structure
For a set L of n horizontal line segments in rank space, let L and R be the set of the left
endpoints and right endpoints respectively of the segments in L. let YL[1, n] be the sequence of
y-coordinates of the left endpoints (x, y) ∈L ordered by increasing x-coordinate. Let YR[1, n] be
the corresponding sequence for the right endpoints. We represent the set L with the following
structures:
• A wavelet tree for each of the sequences YL and YR over the alphabet [1, n].
• A bitvector BR[1, 2n] where bit i is set iff there is a right endpoint (i, y) ∈R that has
x-coordinate i.
Each wavelet tree uses n⌈lg n⌉+ o(n) bits of space by lemma 5 and the bitvector BR uses
2n + o(n) bits of space by lemma 1, and thus the total space in bits is 2n lg n + O(n). Since the
content of the sequences YL and YR is the same, then the two wavelet trees will have a similar
topology. Let v be a node in the wavelet tree for YL, then we will refer to the corresponding
node in YR as R(v). We will refer to the bitvector and subalphabet of a node v in one of the
wavelet trees as B(v) and Σ(v) respectively.

Lemma 7. Let v be a node in the wavelet tree of YL with computed local variables lv and rv,
then the number of segments with y-coordinate in Σ(v) crossing at time i is lv −rv.
Proof. When v is the root of the wavelet tree of YL we have rv = rank(BR, 1, i). Since BR
contains a 1 in every index k such that there exists a right endpoint with x-coordinate k, then
rv is the number of right endpoints in the interval [1, i]. Since there is exactly one segment
endpoint in each x-coordinate the number of left endpoints in the interval [1, i] is i −rv = lv.
Thus lv and rv are the number of left and right endpoints in [1, i] respectively, and the number
of segments crossing at time i is the number of left endpoints in the interval [1, i] that does
not have a matching right endpoint in the interval [1, i]. Since a right endpoint cannot come
before a left endpoint, each right endpoint in the interval [1, i] must match a left endpoint in
the interval [1, i], and thus lv −rv is the number of segments crossing at time i.
For a non-root node v, and parent u let the edge (u, v) have label b. Let lv = rank(B(u), b, lu).
By lemma 4 lv is the number of segments in B(v) such that their index in B(u) is less or equal
to lu. Since the left endpoints are ordered by increasing x-coordinate in B(v) and B(u), then
the number of left endpoints with x-coordinate in [1, i] and y-coordinate in Σ(v) is lv. The same
holds for rv with respect to the right endpoints; thus, lv −rv is the number of segments in v
crossing at time i.
The correctness of the navigation follows from lemma 7, since we can keep track of the

Persistent Strings
In this section, we will show how we can reduce random access in persistent strings on a version
tree T of n nodes to segment selection on 2n −2 segments in the plane [1, 4n −4] × [1, n −1].
This plane differs slightly from what was shown in [3], where they reduced it to the plane
[1, 8n −4] × [1, n −1]. Later on, we will show a structure that also supports rank and selection
on the strings S(v) of a node v in the version tree T. Initially, we want to show the information-
theoretical optimal number of bits required to represent a version tree T of n nodes, so we have
a benchmark for the space efficiency of our solutions.
Lemma 11. let U be the set of version tree instances of n nodes with characters from [1, σ].
Then the worst-case entropy of Hwc(U) = n(lg n + lg σ) + O(n).
Proof. Let T ′ be a version tree such that T ′ is a path consisting solely of insert edges. For
an edge (u, v) the number of places a character can be inserted is |S(u)|. Thus the number of
ways we can configure the insert position of all the edges is 1 · 2 · . . . · (n −1) = (n −1)!. The
number of ways we can choose n −1 characters from the alphabet [1, σ] with replacement is
σn−1. We thus have the number of bits required to distinguish the instances of version trees
that are a path consisting solely of insert edges is ⌈lg((n −1)! · σn−1)⌉= n(lg n + lg σ) −o(n).
A crude upper bound on the number of topologies of a tree of n nodes can be derived from
the Euler tour of a tree. The number of bits to store an Euler tour of a tree of n nodes is
2n −2. The number of ways to configure 2n −2 bits is 22n−2. The additional number of bits to
distinguish the topologies is thus ⌈lg 22n−2⌉= O(n). Assuming that any edge can be any edit
operation we would have that 3n−1 is the number of ways the edges can be labeled with an edit
operation, requiring ⌈lg 3n−1⌉= O(n) bits to distinguish between these instances. Note that

both the delete and the replace edit operation is done on a position in the string represented
on the parent node, as the insert edit operation. Thus the number of ways we can configure
the positions of the other edit operations is already accounted for by the number of ways we
can configure the insert positions. We have clearly overcounted both the number of topologies
and the number of configurations of the edit operations, yet shown that even for the subset of
instances where the version tree is a path it is required to use n lg n+n lg σ −o(n) bits of space.
The additional instances where the version tree is no longer a path and can choose any edit
operation only lead to factor O(2n · 3n) more instances, and thus the worst-case entropy of the

As has already been shown [3] we can model the replace edit operation as a delete followed by

in T ′ in O(lg3 n + lg σ) time.

Project Plan
This is the original project plan for the thesis. As we were studying persistent strings we got
the idea to use wavelet trees to represent segments. When we discovered that this would allow
us to represent horizontal line segments succinctly, we changed our focus to segment selection
and rank instead of pattern matching in persistent strings.
A.1
Aim of the project
The project aims to develop a solution to the pattern matching problem in persistent strings
based on state-of-the-art algorithms for the compressed indexing problem.

applying existing techniques or identifying special properties of persistent strings that allow for
a faster or more space-efficient solution. A solution has already been identified, and thus the
