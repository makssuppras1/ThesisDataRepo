Summary
Recent advancements in non-linear filtering techniques provide for an opportunity to
improve and rid past consistency issues in large scale simultaneous localization and
mapping based on the extended Kalman filter as the chosen optimization method. This
is important for the multitude of industries that rely on such algorithms when the
Global Positioning System or other means of extracting absolute measurements of one’s
whereabouts are not available. Especially in robotics and augmented reality is exact and
reliable localization information crucial for operation as well as acquiring the structure
of the environment.
In this study, the invariant extended Kalman filter is rigorously investigated for its
ability to correctly model uncertainty of the estimated states by avoiding the unfortunate
mathematical properties of the current state of the art implementations of filter-based
simultaneous localization and mapping; that is, the need to linearize around the true tra-
jectory but as this information is obviously unknown, one must evaluate the linearization
around the estimate instead which can lead to inconsistencies.
This enables robust and consistent fusing of local maps via the means of the infor-
mation filter formulation, which will save tremendous computational effort and reduce
memory requirements. In order to use the non-linear uncertainty presentation for local
mapping a novel extension of the framework of local mapping using the extended infor-
mation filter is presented and tested in simulation and on a real data set using a camera
and inertial measurements.
Keywords Simultaneous localization and mapping, invariant extended Kalman filter,
extended information filter, local mapping, map fusion.

CHAPTER 1
Introduction
1.1
Motivation and background
An essential element of an autonomous vehicle, such as a mobile robot, is the ability
to answer the question: ”Where am I?” However, that question is ambiguous as it is
two-fold. A commonly used way to answer that question is by using the widely known
Global Positioning System (GPS) for localization and navigation; however, under many
circumstances, the system is unavailable. This frequently happens in daily life when
one finds themselves inside a building or in an urban environment where the signal
is distorted by skyscrapers. Additional caveats of the GPS are that one only receives
updates at a frequency of 1 Hertz. In addition, the United States government currently
claims only a 4 meter RMS (7.8 meters 95% confidence interval) horizontal accuracy
for civilian (SPS) GPS and even lower for vertical accuracy. This can be very limiting
in many applications. Therefore, the map app on smart-phones already has systems
in place to predict movement in-between GPS updates. The convenience of the GPS
system is that it provides absolute measurements of position, i.e. with respect to a global
reference coordinate system. This is precisely why, the question of one’s pose is two-fold
as in order to know one’s location, one must first contextualize the environment to a
map or some chosen coordinate system before answering.
Examples are provided in the following to further illustrate this point. After the
Fukushima Daiichi nuclear disaster, a mobile robot developed jointly with Toshiba and
the International Research Institute for Nuclear Decommissioning was deployed to survey
the damaged buildings and examine fuel debris in the Fukushima unit[1]. Under those
circumstances, there was no GPS signal for localization, and neither was construction
blueprints of much help when buildings were either severely damaged or collapsed. Even
though the robot was not autonomously operated but remotely controlled, having the
location information relative to its initial starting point as well as information of its
surroundings was still of utmost help to the human operator.
When exploring the
unexplored, a significant factor is remembering how to get back to its initial starting
point and also saving the gathered information. This was especially important for the
cave divers of the Tham Luang cave rescue in Thailand where they relied on manually
placed lines and markers to map their path into the unknown cave. That being said,
having as a backup a digital mapping and localization system on hand which by using
sensors could track one’s localization relative to one’s starting point in a simultaneously
created map of the explored environment could be life-saving.

1 Introduction
1.1.1
Simultaneous Localization And Mapping
This is exactly what the robotics community have solved by implementing Simultane-
ous Localization And Mapping (SLAM). It is a technique that aggregates a multitude
of approaches for self-localization and automated map generation using only on-board
sensors that measures relative information. It works when the mobile robot or the plat-
form, on which the sensors are attached, localizes itself on the map it has constructed
and continuously improve and optimize. The localization and map construction of the
robot using SLAM are two sides of the same coin; that is if one is faulty, the other will
also be.
The main difference between that of using a GPS system and that of SLAM is that
one is using absolute information and the other only relative.
Figure 1.1: The unobservable global pose.
This is illustrated in Fig. 1.1, where a robot, represented by the purple triangle,
makes relative measurements of two nearby distinctive markers in the environment rep-
resented by two yellow circles. The aim of SLAM is to construct an estimated map
constituting of gathered information such as the two measurements of the markers and
position the mobile robot in it. However, using only such measurements makes it im-
possible to observe the position of the markers and the robot itself in a global reference
system, as is what happens when one is using GPS. This is because the two configura-
tions are indistinguishable from each other; that is, when departing from either, it is not
possible for the robot to tell that there is an actual difference between its position and
orientation as well as the positions of the markers in the two cases.
1.1.2
Front-end
The sensory system of SLAM can be completely arbitrary; in fact, there exist many
different methods of gathering surrounding information. Although classically GPS mea-
surements are not considered in the rich literature of SLAM systems, they can, in fact,
be included to reference the constructed map and the localization within it globally.
Thus, the goal is to fuse all available information from all measurements to get the most

accurate estimate. Early research of SLAM, the first being the 1986 paper by Smith and
Cheeseman in [2], involved mainly smaller ground-based vehicles with laser range finders
to measure distances to objects in the environment. Thus, by a known scan motion, one
can estimate a 3-dimensional (3D) point cloud with the robot at its origin. Other known
SLAM systems employ sonar, radar or LIDAR to obtain relative measurements in order
to construct a map of the background.
Figure 1.2: LIDAR (Light Detection and Ranging) measurements of a self-driving car.
However, these type of systems tend to be more expensive, and there is indeed a
correlation between the accuracy of one’s measurements and the amount of money one
is willing to spend on the system. Therefore, in recent times, the most researched SLAM
systems are those based on computer vision. This stems from the fact that computer
vision has been rapidly evolving as cheap camera systems that are small in size, low
weight, and low energy consumption have come to be available. In addition, another
appealing area of research that is tightly knit to SLAM is augmented reality. Hence,
knowing the camera orientation and position relative to a plane observed in the frame of
the camera is what enables the augmentation. Therefore, real-time and computationally
low demanding SLAM systems are of surging interest.
These systems go under the name visual SLAM which covers two different configu-
rations of camera setups: Mono- or stereo-vision. The latter being two synced cameras
facing the same scene arranged such that there is a fixed known transformation between
them.
Then mutually shared landmarks in a pair of images can be obtained as 3D
information relative to the stereo setup via triangulation and epipolar geometry. How-
ever, as consumer products usually only carry a single camera, monocular visual SLAM
systems are almost exclusively published in academia as compared to the stereo vision
sibling. This comes down to that recovering structure and camera pose to scale is far
less trivial. Mono-camera setups relies strongly on the estimated camera poses in order
to acquire 3D information from image frames as opposed to e.g. range sensors. The
contrast lies in that for the monocular system to at all obtain measurements, the pose
and localization needs to be known and accurately estimated for consistent map gener-
ation. In Fitzgibbon and Zisserman[3] a complete sequence of non-stereo images with

known correspondences can be used to estimate camera movement and landmarks in a
3D reference frame. However, this is known as offline reconstruction or structure from
motion whereas SLAM is constrained to online estimation; that is, in real-time. The
process of extracting information through the sensory system is known as the front-end
of the SLAM system.
The front-end of visual SLAM can generally be divided into two parts each with two
categories: direct versus indirect and sparse versus dense. These classifications do actu-
ally extend to any sensor modality, although they are most often used in the context of
monocular, stereo, visual-inertial, and RGB-D SLAM. Underlying most formulations are
typically noisy measurements fed into a probabilistic model that computes an estimate
for the unknown, hidden model parameters that characterize the 3D world and camera
motion. This probabilistic model is what is known as the back-end of the SLAM system
and will be introduced in the next section. However, part of the problem is to generate
an intermediate representation that is suited for the said model. Indirect methods will ei-
ther extract and match a sparse set of features, alternatively called keypoints, found in a
pair of images (be it consecutive or a stereo images) or using other established correspon-
dence techniques such as dense, regularized optical flow. This can also include matching
geometries like lines or curve segments instead of points. Klein et al. in [4] showed in a
state of the art implementation PTAM (Parallel Tracking and Mapping), that real-time
SLAM can be realized with sparse bipartite graph of keypoints and keyframes, which is
optimized in a separate thread using non-linear optimization. The maps generated from
sparse methods consists of a points cloud, which is a coarse representation of the envi-
ronment and mainly used to track the camera pose for localization such as illustrated
on Fig. 1.3c.
(a) DTAM (dense).
(b) LSD (semi-dense).
(c) PTAM (sparse).
Figure 1.3: Sparse and Dense Methods.
Direct methods need not find feature correspondences and can immediately use all
image values as input to the back-end optimizer. The difference lies in that that proba-
bilistic model will optimize a photometric error, since an image sensor directly provides
photometric measurements. This is fundamentally different to the indirect methods that
extract geometric information out of the image. Hence, the back-end must optimize geo-
metric error or in more specific terms: Reprojection error minimization. Dense Tracking
and Mapping (DTAM)[5] was one of the first published direct SLAM methods that op-

erate on a single monocular camera. However, direct methods suffer from their inability
to recognize already visited places. The advantage of map generation, other than the
intuitive uses for a human operator, is that it presents a solution to the drift problem of
dead-reckoning: By identifying previously found landmarks, also known as loop closures,
it limits the error committed from estimation in the back-end. Hence, direct methods
usually classifies as visual odometry, whereas indirect methods use descriptors of its
selected point features in the image frame for convenient data association which is an
important and necessary part of SLAM.
From the perspective of the amount of information used in an acquired frame from
an image sensor, the front-end of the SLAM system can be classified as either dense or
sparse. Dense methods attempt to use and reconstruct almost all, if not all pixels in
the image, whereas sparse methods merely use and reconstruct a carefully selected set
of points. Naturally, this choice is reflected in the map generation. An illustration of
this can be found in Fig. 1.3a and 1.3b. However, intermediate approaches - semi-dense
- refrain from reconstructing the whole surface of all objects in the scene, but rather a
subset of it[6].
1.1.3
Back-end
Generally, the types of inferences that the back-end of indirect SLAM performs on
abstracted data produced by the front-end are either filtering methods or bundle adjust-
ment (BA) methods. In order to illustrate the differences, the probabilistic Bayesian
models of SLAM are visualized by means of probabilistic graphical models in Fig 1.4.
(a) Global BA.
(b) Keyframe BA.
(c) Filter.
Figure 1.4: Optimization methods.
The variables Tt each represent an instant of time of the camera pose and other
states of interest such as velocity to be estimated, and xk represents the static position
of landmarks. These are connected by edges representing the feature measurements in
the frame of the respective image. As time goes by, this Bayesian network of the SLAM
problem will continue to grow without bounds as new measurements of unexplored fea-
tures are added. The most successful inference techniques are all least-squares methods
which minimize the reprojection error by assuming Gaussian distributions for states and
observations. This stems from the special status of the Gaussian distribution of proba-
==PAGE:18==
6
1 Introduction
bility theory given the central limit theorem, which makes it the most natural way to
represent dispersion in a lot of practical inference problems.
The most exact solution would be the application of global bundle adjustment by
finding the full maximum likelihood solution to the graph illustrated in Fig. 1.4a each
time a new measurement is added. The computational efforts would preclude an online
implementation.
Instead, the state of the art approach is by using a sparse bundle
adjustment by removing all but a selected of past poses as illustrated in Fig. 1.4b; usually,
these keyframes are in a sliding window of the most recently obtained poses. They tend
to be chosen in an intelligent manner, such that the least blurred frames are selected.
However, all other poses, including the edges linked to them, are simply discarded and
do not contribute to the estimation. The upside given this loss of information is that
the graph will attain a sparsely inter-connected network that lowers the computational
burden. This is in stark contrast to filtering, as illustrated in Fig. 1.4c where information
is not discarded but instead marginalized out such that only the most recent state is
available. Landmarks, which may be measured again in a loop-closure, are retained.
The advantage of filtering is a very compact graph that will not expand during repeated
movement in already explored areas. On the flip side, the result is a fully inter-connected
graph, since each marginalization of a past pose causes change in links between all the
variables. Thus, the joint distribution of all of these cross-correlated variables must be
kept and stored. However, the propagation of a joint distribution of an unbounding
growing number of variables is intractable to perform in real-time. Strasdat et al. in
[7] concluded that filtering has a better accuracy per unit of computation trade-off for
systems on computationally constraint budgets. On the other hand, keyframe based
bundle adjustment attain a better trade-off for larger scales of mapping using more
points. This is because increasing the number of landmarks increases the accuracy, while
increasing the number of intermediate keyframes only has a minor effect. Considering
the cost of bundle adjustment which has a linear complexity in number of landmarks
to the cost of filtering which is cubic in complexity, it becomes evident that keyframe
bundle adjustment is the more efficient technique.
However, complexity only illustrates the increase in computational burden but does
not indicate the required starting point of computational effort. This is important to
consider as equating lower complexity to real-time performance can be futile. One must
specify the targeted computer architecture and its available resources. The most well
known and complete SLAM framework in the state of the art is ORB-SLAM2[8]. It is a
classic sparse indirect monocular SLAM system based on bundle adjustment of a sliding
window of keyframes. However, this system only runs real-time with adequate framerates
(18-20 Hz) on multi-core CPUs with decent clock speeds (3 GHz boost and above) and
with modest RAM available. On the other hand, the Multi-State Constraint Kalman
Filter[9], the back-end of which is a filtering method, has a very low computational
requirement – running in real-time on a low power ARM processor such as the one in a
modern day smart phone. They do not, however, provide the same functionality so the
comparison is not a fair one and should not be read as such but emphasizes the low bar
in terms of computational budget that filtering requires.

1.2 Related work

1.2
Related work
The purpose of this thesis is to alleviate the burden of a fully inter-connected graph
due to marginalization and elevate the benefits of filtering to large scale mapping in
sparse indirect monocular SLAM. There has been multiple studies on this matter and
the general approach is to generate smaller maps - submaps - and consecutively fuse
them together instead of using the extended Kalman filter to build one single large map,
which as a whole would have to be updated whenever a new landmark is found. This was
originally proposed in [10] by Leonard et al. which involved dividing the environment
into multiple overlapping submap regions, each with its own stochastic map. However,
this entailed approximation schemes in order to convey information from one submap
to the other. This was later improved upon in [11] by Bosse et al. which resulted in the
Atlas framework where an interconnected set of local coordinate frames are maintained.
Local maps denotes submaps that are not globally referenced but instead in a local
reference frame. This idea has lead to a plethora of publications in local mapping for
SLAM. The first being the works on independent local map joining as proposed in [12]
and [13] by Tardos et al.
and Williams respectively, which involved a map joining
technique that allows the system to build a sequence of independent, limited in size,
stochastic, local maps and join them in a globally consistent way. The process of joining
two consecutively built local maps is by transforming the latter map using an estimate of
relative pose from the former map to have both submaps in a common reference frame,
and then apply the coincidence constraint to the common elements. Williams uses an
extended Kalman filter (EKF) to fuse consecutively build local maps in a sequential
manner by treating each map as an integrated observation.
However, as the global
map grows in size and becomes larger and larger after fusing in more maps, the dense
covariance matrix of the extended Kalman filter for the mapping algorithm will be less
efficient than what is proposed in the paper [14] which demonstrates sparseness using
an information filter.
This is achieved by keeping the robot starting poses and end
poses of the local maps in the global state vector in addition to all landmarks but
removing intermediate vehicle poses.
Combined Kalman-Information Filter[15] with
log(n) complexity further demonstrated that by using a Divide and Conquer strategy,
the efficiency could be improved even more.
However, the state of the art methods all suffer from the choice of filtering technique
when building local maps. In any non-linear estimation problem, the extended Kalman
filter is the go-to formulation for implementing the back-end of SLAM, primarily because
of its ability to close loops thanks to the maintenance of correlations between landmarks.
Then, the era of graph optimization came to be, which has lasted throughout the last
decade. The primary issue leading to the shift was the fact that EKF-SLAM returns
an overly optimistic covariance matrix, resulting in inaccurate estimates, i.e., being
inconsistent. This inconsistency stems from the fact that only relative measurements
are available when running SLAM. Thus, the pose (origin and orientation) of the earth
fixed global frame can never be correctly estimated. However, EKF-SLAM tends to
try to estimate it as its output covariance matrix exhibits an information gain in these

In other words, the EKF does not handle the above lack of observability in a prefer-

in practice as the true trajectory is obviously unknown. Therefore, a filter in which the
error propagation does not depend on the initial linearization points would circumvent
the inconsistency.
This is especially relevant in local mapping as the covariance estimation is of utmost
importance when fusing maps. The said paper [14] experiences that the linearizations
of the extended Kalman filter resulted in an inconsistent global map at more substantial
noise levels. It is stated that ”This is likely due to the fact that in any submap joining
algorithm, inconsistency in even one of the submaps, will lead to an inconsistent global
map.”
The recent paper [16] has provided an alternative variant of the extended Kalman
filter filter that rides it of its inconsistency issues, namely, the invariant extended Kalman
filter(IEKF). It does this by preventing covariance reduction in the directions of the
unobservable state space by modifying the extended Kalman filter to account for the
geometrical structure of the system. The caveat of the invariant extended Kalman filter
is that it only covers systems able to be defined on Lie Groups. Fortunately, as will be
shown later, SLAM can achieve that.
One might consider extending the exactly sparse delayed state filter to Lie groups as
in [17]. However, no one has yet provided a way to efficiently access probabilities, i.e., the
covariance matrix, for tasks such as proposing link hypotheses and detecting loop closure
as stated in [18]. Therefore, for the best robustness and, at the same time adhering to
computational requirement, map-joining with consistency may prove fruitful.

2.1

We define a matrix Lie group G as the subset of d × d square invertible matrices Md(R)
that satisfy the following properties:
Id ∈G,
∀X ∈G, X−1 ∈G,
∀X1, X2 ∈G, X1X2 ∈G
where Id is the identity of G and X is any point in G.
At every point X ∈G there can be associated a tangent space Tχ(G). The tangent
space TId(G) at the identity Id is called the Lie algebra. We denote the Lie algebra as
g ⊂Md(R) whose dimension is the same as G. Why is this important? A fundamental
property dictates that the tangent space of different points in the same group is isomor-
phic (a mapping that can be reversed exists), and as the tangent space at the identity
is less difficult to describe, we start from it.
“The miracle of Lie theory is that a curved object, a Lie group G, can be
almost completely captured by a flat one, the Tangent space TId(G) of G at
the identity.”
John Stillwell

different sampling times, this dependency should be properly modeled to guarantee the
same dynamical behavior of the filter. In addition, in sensor data, one does not always
have equidistant samplings, thus demanding that the process noise covariance to be
rescaled at each time instant which is the case in Gaussian process regression.

3.2
Summary
This chapter presents the novel findings of this study.
A local mapping scheme for
right-invariant retractions. The reasoning for needing such a scheme is that the uncer-
tainty representation of the invariant extended Kalman filter does not fit into the usual
==PAGE:50==
38
3 Local Mapping
framework of linear systems theory. In other words, it will not make sense to use the
current state of the art formulation of the information filter to fuse local maps together.
It simply will not work. The first part of the chapter introduces a way to store local
maps in a Lie group element. This is essential in order to adhere to the structure of the
uncertainty representation of the invariant extended Kalman filter. The remaining of
the chapter then involves constructing an observation function and an innovation that
consists of actions of the space of the Lie group. Of course, the complicated Jacobians
are derived as well as they are not trivial. These right-invariant type functions along
with the Jacobians are what is needed for the presented information filter scheme.

CHAPTER 4
Vision
The following chapter will introduce the front-end of the proposed method. Although
this is appropriate and suitable for testing a real-world scenario and proving the back-
end’s applicability, it is not state of the art and not directed for industrial application.
Bear in mind there exist a dedicated field to the SLAM front-end involving camera
sensors, and it is still ongoing.
There is no one solution to the visual front-end of
SLAM as it is a complicated task indeed. Previously in the introductory chapter, it is
mentioned that visual systems typically employ point-feature tracking using descriptors
for convenient data association. However, as convenient as it might be, the question
still arises when exploring the already explored: Where to look in order to find a feature
that will ideally share the same descriptor as previously discovered features?
There have been proposed many solutions to this problem. George Klein proposed
in his PTAM[4] that selected down-sampled non-blurry images to be saved in memory.
These images can then be compared with each other to find matches.
Indeed, if a
match is found a loop closure has occurred which will initiate a comprehensive search
for identical descriptors in both images processed in a separate thread to not disturb
the real-time operation of SLAM. When point-feature correspondences are found, the
back-end can optimize and reduce the uncertainty of the generated trajectory and map.
A later alternative can be found in ORB-SLAM [8] where a compact description of
the selected images are saved in a bag of words representation using a multitude of
descriptors in the images. However, these are very brief descriptions and additional
geometric constraints such as RANSAC (Random sample consensus), and also temporal
constraints are needed to make the performance reliable.
The introduced visual system will not be using feature-point descriptors, nor will it
include global loop closure. Naturally, this is a downside and should be implemented
for industrial application. To compromise on this downside, optical flows are used to
track point-features. This allows for robust tracking of points in the field of view of the
camera. Of course, as soon as the tracked point leaves the field of view of the camera,
it is forever lost, and new optical flows must be computed.
4.1
The pinhole camera model
The wide availability of inexpensive digital cameras using CMOS sensors makes it a pop-
ular choice for SLAM front-ends. A convenient way to model the camera is the pinhole
==PAGE:52==
40
4 Vision
camera model due to its mathematical simplicity. Of course, it is only an approximation
of real cameras. However, the model can be extended with a distortion model to account
for imperfections in manufacturing. The camera calibration and remapping, along with
the relation between actual real-world coordinates and camera coordinates, are briefly
introduced in the following sections.
The pinhole camera model considers the central projection of points in 3D space onto
a 2D plane. The centre of projection is the origin of a Euclidean coordinate system in
which there is a plane z = f, which is usually named the image plane or the focal plane.
The pinhole model involves mapping a point in space X = (X, Y, Z)T to the point on
the image plane where a line joining the point X to the centre of projection coincides
with the image plane.
Figure 4.1: Pinhole camera model[3]. C is the camera centre and p is the principal
point.
Using similar triangles as seen in Fig. 4.1, it is obvious that the point (X, Y, Z)T is
mapped to the point (fX/Z, fY/Z, f)T on the image plane. It is clear that this describes
the central projection mapping from 3D Euclidean world coordinates to 2D Euclidean
image coordinates: R3 →R2. The centre of projection is by convention called the camera
centre or sometimes the optical centre. The perpendicular line from the camera centre
to the image plane is the principal axis of the camera and the point of intersection is
the so called principal point.

4.2
Distortion model
The camera calibration matrix does not account for lens distortion as the pinhole camera
model obviously does not have a lens. Therefore, in order to accurately represent a real
camera, one must extend this pinhole camera model with a distortion model. Usually,
certain types of camera lenses introduce radial distortion such that objects along the
optical axis of the lens occupy disproportionately large areas of the image so that objects
in the periphery will take up a smaller area. This is called barrel distortion and can
be seen in Fig. 4.3. Of course, the inverse of this effect exists as well and is called
pincushion distortion.
(a) No distortion.
(b) Barrel distortion.
(c) Pincushion distortion.
Figure 4.3: Types of radial distortions.
Another distortion occurs when the camera lenses are not perfectly parallel to the
imaging plane. This is called tangential distortion. These type of distortions will occur
in any camera with a lens and will be unique to that camera as any camera will not be
perfectly identically manufactured.
This results in non-linear relations for distance in the camera image which does not
align with how distance relations are in the real world frame. One needs to account for
these camera characteristics when triangulating image points to real world coordinates.
The widely adapted Brown-Conrady-model proposed by Brown in 1964 [27] will be
used as the distortion model. This model uses 5 distortion coefficients to describe the
imperfections found in a camera image. Using these coefficients the camera images can
be corrected by a remapping to a rectilinear representation.
For the radial distortion the 3 coefficients are usually denoted as (k1, k2, k3) and can
be corrected using the following relations:
˜x = x(1 + k1r2 + k2r4 + k3r6)
(4.12)
˜y = x(1 + k1r2 + k2r4 + k3r6)
(4.13)

(4.14)

image coordinates. Normalized image coordinates are calculated from pixel coordinates
by translating to the optical centre and dividing by the focal length making the focal
length of the normalized image coordinate f = 1.
Most often k3 = 0 unless there is severe distortion. This is the case for the real
data set on which the proposed SLAM algorithm will be evaluated on. These distortion
coefficients are intrinsic to the camera and remain the same regardless of the captured
image resolution. The intrinsic distortion parameters along with the intrinsic and extrin-
sic parameters of the pinhole model can be estimated using established computer vision
tools such as OpenCV or MATLAB’s Computer Vision Toolbox. These tools base their
calibration algorithms on [28] and [29]. They require using objects with known geometry
such as chessboards as calibration rig for easily finding 3D object points and their 2D
corresponding projections in each view. The algorithms use PnP solvers and Levenberg-
Marquardt optimizations to achieve accurate results with minimal reprojection error.
4.3
Triangulation
The triangulation of a point in 3D space is illustrated given its projections in two images.
This can be thought of having two cameras with known intrinsic parameters along with
a known transformation between them. Obviously, in the monocular case the intrinsic
calibration matrix K is the same as the two views are from the same camera but at
different time stamps. The SLAM algorithm handles the estimation of rotation matrices
and translation vectors of the robot in the world coordinate frame and the extrinsic
parameters of the camera with respect to the body frame are constant and known. Thus,
the two views will have a known transformation between them. This information enables
the computation of two lines defined from the camera centres of the two views that passes
through the exact location of the points in image coordinates to then intersect at the
triangulated 3D point in space.

the real data set, 9 views of a 3D point is used when triangulating. Naturally, using the
total number of views a point has been viewed in a current local map will provide for
more accurate results but on the other hand increase the computational burden. The
choice of 9 views comes merely from trial and error and is not founded on extensive
research. Triangulating from more than 2 views increases the robustness as the depth
of the 3D point will only be estimated well if the camera translates enough to produce
significant parallax.
The downside of this linear method is that it assumes that the solution point X
is not at infinity, for otherwise one could not assume that X = (X, Y, Z, 1)T. This is
mainly a disadvantage when one is seeking to carry out a projective reconstruction, when
reconstructed points may lie on the plane at infinity. However, due to its efficiency and
computational speed it is preferred over other non-linear approaches.
4.4
Tracking
The filter operation of the proposed SLAM algorithm is dependent on updates, which
will be performed at recurrences of landmarks. This entails augmenting an initial set of
point features representing landmarks into the state of the invariant extended Kalman
filter to subsequently include observations of these to reduce uncertainty. When tracked
points leaves the field of view of the camera, new point features will be initialised and
augmented into the state of the filter to again be updated via tracking. The Kanade-
Lucas-Tomasi (KLT) feature-tracking algorithm constitute the combined work of [30],
[31], [32], and [33], which will be used for tracking and point selection.
In general, when tracking point-features in image frames, one must choose how to
select relevant and robust features as well as how to track them from frame to frame.
The KLT algorithm solve the tracking problem using the Lucas-Kanade method for
registering and aligning two image patches of two consecutive images. The matter of
choosing good features is done via the Tomasi-Kanade method of minimum eigenvalue
corner detection.
As the camera moves and generate new frames of the environment in the field of view,
the pattern of image intensities of each frame change in a very complex way. However,
consider a specific pixel intensity I(x, y, t) of a frame, where (x, y) are spatial variables
and t is a temporal variable. It will usually be strongly related to a pixel in sequential
frames taken at near time instants. This is obviously only for the case when the frames
refer to the same scene taken from only slightly different viewpoints. Such an image can
be described as:

I(x, y, t) = I(x + dx, y + dy, t + dt)
(4.22)

every pixel in the previous image by an amount δ = (dx, dy). However, even in static en-

happens when the photometric appearance of a region on a surface in the environment
changes due to its reflectivity being a function of the angle of which it is viewed. In
addition, at occluding boundaries, image points will come into sight to then disappear
between frames. Despite the obvious violations of the invariance property, it turns out
that it is satisfied in many cases at surface markings abound in different environments;
be they natural or man-made.

Several measures are taken in order to reduce erroneous pixel tracking. A forward-
backward error is limited. It is calculated from tracking each point back and forth in
successive windows from which the bidirectional error can then be found. This value is
the distance in pixels from the original location of the points to the final location after
the backward tracking. The corresponding points are considered invalid when the error
is greater than a set threshold of 3 pixels.

In addition, if an estimated point from the invariant extended Kalman filter repro-
jected onto the image plane are is far from the tracked point estimated by KLT, the
point is rejected. Also, if landmarks are too close to each other (within 10 pixels), the
most uncertain landmarks are removed. Lastly, when the tracker is reset due to too few
points being tracked, new points are uniformly chosen in the image.
4.5
Summary
This chapter presented the front-end of the SLAM system that will later be used on
a real data set. The back-end will be fed tracked feature-points from a KLT tracking
algorithm, which constructs optical flow trajectories. Thus, the back-end will be able
to locally update on augmented landmarks in the state. However, as they are optical
flows and not feature descriptors, the tracking will be forever lost when landmarks leave
the field of view of the camera. The reasoning for choosing a KLT tracker is that the
actual tracking is very robust and that very few inaccurate data associations occur. The
front-end has multiple constraints that ensure the very best points are always tracked
and that they cannot be confused with other tracked points. To summarize the settings
of the KLT algorithm: It is run with a 31 pixel square window on a 3 level Gaussian scale

pyramid of 752 × 480 resolution images. A maximum iteration count for convergence is
set at 30. A forward-backward error is limited at 3 pixels. Point-features are ensured to
be uniform and not within 10 pixels of one another.

5.1
Consistency simulation
In this section, simulation results are presented to illustrate the consistency of the in-
variant extended Kalman filter. A comparison has been made to the usual state of the
art extended Kalman filter, which is based on quaternions or axis-angle representation
of its orientation in 3D. This is key to understand the important reason behind the
consistency issues in local map fusion.
The quaternions or axis-angle representation are in fact choices that characterize the
natural system as it form a nonlinear uncertainty representation and retraction. This
has been an obvious progression since the early works of SLAM based on the problematic
Euler angles. However, as opposed to the invariant extended Kalman filter, it is not a
holistic approach where the retraction is natural to its state space. Instead, other states
are kept with their separate linear retraction which will through the inter-connection
corrupt the correction gain. In addition, in quaternion space there are possible con-
vergence issues in many situations as the matrices to compute the correction gain are
only constant in the equilibrium points for level motion because of trajectory dependent
terms whereas the main benefit of the invariant representation is that these matrices are
constant on a much bigger set of trajectories.

seen nearby landmarks and augment those who is not already in the state and thus

advantages of the invariant extended Kalman filter.

CHAPTER 6
Conclusion
In this chapter, a summary of the work done in this study will be presented as well as
some possible future work directions that could be followed in order to improve upon
the current results.
6.1
Summary
The proposed solution tackles a major challenge in large scale SLAM arising from the
fact that the robot and observations from the on-board vision sensor is coupled which
will inevitable occur as noisy measurement are taken from an uncertain robot position.
This coupling is especially apparent between mapped landmarks in filtering. The error in
estimation of landmarks will be linked together as they all were observed from uncertain
robot locations. In the case where the robot location was certain, then the errors of the
landmarks would be independent from one another but this will never be reality. When
using the invariant extended Kalman filter for localization and mapping, an observation
of subsequent landmark improves the knowledge and uncertainty of the robot’s pose
and hence consequently also betters the position of other landmarks. As this coupling is
represented with Gaussian error distributions in the parameterization of the Lie algebra,
the non-zero off diagonals in the maintained covariance matrix cannot be neglected -
even if landmarks are not visible in the same frame, the cross-variance is significant.
The computational requirement becomes intractable as more unexplored parts of the
environment are mapped since the associated error will be related to all other parts of
the map through the uncertain position and orientation of the robot. Subsequently, loop
closures will result in all other parts of the map, in addition to re-visited landmarks, to
be updated. However, using the information representation and the proposed formula-
tion of the extended information filter in a local mapping scheme, one can drastically
reduce memory consumption without taking steps to limit the statistical coupling by
approximation. The computational complexity of building local maps are constant in
time and map fusing can be done linear in time as oppose to usual quadratic in time of
large scale SLAM using filtering methods.
The consistency issues of using a linear error exacerbated the obstacle of local map-
ping in all previous publications that had the extended Kalman filter as the back-end.
The proposed algorithm circumvent any use of linear error representation. Even in the
local map fusion itself. A novel local mapping scheme was presented and evaluated in a
==PAGE:82==
70
6 Conclusion
real world data set, namely, the EuRoC data set. The performance was measured to be
on a par with current state of the art visual-inertial SLAM algorithms.
Lastly, this study intends to be a reference work for the actual implementation
of invariant extended Kalman filtering in visual-intertial SLAM by providing verbose
derivations and necessary definitions. An effort has been put into providing a proper
discretization scheme along with Jacbobians that can be used directly in implementa-
tions. Thus, the reader can directly extract information and implementation of SLAM
elements when demanding the presented non-linear error definition.
6.2
Future work
Along this dissertation several measures have been proposed that could yield an improve-
ment of the results presented. The scope and time allowance of the study did not allow
for such turns in focus and deroutes.
• Use on-manifold preintegration of the inertial measurements for improvement as
showed in [37] instead of assuming constant change in-between samplings.
• Use descriptors for tracked points for possible global loop closure. This is an es-
stential element missing for the proposed algorithm to be a proper SLAM, however,
as this is well established in literature an effort into its implementation was not
made in this study.
• Look into the effect of marginalization of the velocity and bias terms when pro-
ducing a local map to be fused to the global map. There are possible conditional
dependencies that are not properly broken when using an IMU.
• Look into the benefits of using invariant extended Kalman filtering for local maps
in the proposed algorithm. As opposed to using the provided linear transformation
to then use a linear framework.
