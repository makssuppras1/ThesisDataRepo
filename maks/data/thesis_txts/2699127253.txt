Abstract
In this thesis we explore the decidability of the model checking problem of c-
semiring CTL (C-CTL). We already know that the model checking problem is
decidable for restricted instances of c-semirings so we consider the more general
case. We ﬁrst develop the necessary supporting deﬁnitions and results related
to the grouping of paths of a transition system. We then propose alternative
semantics for C-CTL that allow us to compute the solutions to some operators
in ﬁnite time. We have not yet determined the decidability of C-CTL in general
but we are closer to an answer than before.

1.1
Motivation
Model checking [BK08] is a formal automated method for veriﬁcation of systems.
Model checking requires us to model the system, and the properties to be checked
in a mathematically precise and unambiguous manner. Commonly the system
is described as a labeled transition system, and the properties are speciﬁed
in a property speciﬁcation language, for example with a temporal logic such
as computation tree logic (CTL). It is then possible to check if the described
properties hold of a system in question by returning either true or false. This
allows us to reason about qualitative properties of a system, which is not enough
when we are interested in answers that are not boolean i.e. quantitative.
As model checking is an automated method, it is a reasonable constraint that
the algorithms that are used to verify properties of a system are decidable. That
is, we can guarantee that the algorithm returns an answer for all inputs. It is
also possible to show that a restricted fragment of the problem, that is either
the domain or the logic is restricted in some way, is decidable. However that
does not answer the question, if the general problem at hand is decidable nor
where the boundaries of decidability lie.
In this thesis, the decidability of C-Semiring CTL (hereafter C-CTL), a quan-

2.1
C-Semirings
The notion of a constraint semiring (c-semiring) was introduced by Bistarelli,
Montanari, and Rossi [BMR97].
A c-semiring is a semiring with additional
properties of the two operations.
Formally a c-semiring is a tuple ⟨A, +, ×, 0, 1⟩such that:
• A is a set
• 0 and 1 are elements of A
• + : 2A →A is deﬁned over (possibly inﬁnite) sets of elements of A as
follows: P {a} = a, P ∅= 0, P A = 1 and P (S Ai) = P {P Ai}, for
Ai ⊆A, i ≥0
• × : A × A →A is a binary associative, commutative operation that
distributes over +, has 1 as its unit element and 0 its absorbing element.
Note the fact that + is deﬁned over sets and not pairs or tuples as is conven-
tional in semirings. This automatically makes such an operation associative,

two elements we use + as a binary inﬁx operator such that P {a, b} ≡a + b,
while in all other cases we use P in preﬁx notation.
We use the following abbreviations and conventions proposed in Lluch-Lafuente

We will now list some instances of c-semirings. First we will list some instances

• ⟨{true, false}, ∨, ∧, false, true⟩(boolean)

And here we list some instances where the multiplicative operator is not idem-
potent:
• ⟨R+, min, +, +∞, 0⟩(optimization)
• ⟨[0, 1], max, ·, 0, 1⟩(probabilistic)

2.2
C-CTL
In [LM05] C-CTL is presented and an interpretation is given over transition
systems. A transition system is the usual one extended with a c-semiring, that
is, the tuple ⟨S, T, C, P, I⟩where:
• S is a set of states
• T ⊆S × S is a set of transitions
• C is a c-semiring
• P is a set of propositions
• I is a function mapping propositions of P into valuations, which are map-
pings S →A where A is the domain of the c-semiring under consideration
In the transition systems we assume that the set of states S is ﬁnite, and we
also require that T is total, i.e. for every state s there is at least one outgoing
transition (s, s′) ∈T. Runs of a system are maximal paths in the underlying
state transition graph, that is paths that are inﬁnite.
A path is a sequence
s0, s1, s2, . . . such that for all i ≥0 we have si ∈S and (si, si+1) ∈T. We
denote by |p| the length of a ﬁnite path or path fragment p, by sp
i the (i + 1)th
state of p and by γ(s) the set of runs starting in s.

Strongly Connected Components
Let us now deﬁne strongly connected components (SCCs) as maximal sets of
states such that every state is reachable from any other state in the set. We
assume that T is total, so with this deﬁnition transient states are not in SCCs,
that is states that are not contained in any cycle. Let us also deﬁne the con-
densation of a graph as the contraction of each SCC into a single vertex. The
resulting condensed graph is a directed acyclic graph.
With these two deﬁnitions, we can now observe some properties of the transition
systems that will make computations of certain operators tractable.

4.1
Fixpoints
In the following sections we will assume that we can always calculate the ﬁx-
points presented in ﬁnite time for every domain.
The ﬁxpoints that we are
calculating are always iterative multiplicative operations, i.e. for some a ∈A
we want to calculate a×a×a×· · · and ﬁnd the ﬁxpoint in ﬁnite time. We know
that this calculation is monotone by the intensive property of the multiplicative
operation hence by Knaster-Tarski Theorem [Tar55] we know that the ﬁxpoints
are well deﬁned.
Now let us consider the special case where the only idempotent relations for the
multiplicative operation are 0 and 1:
Lemma 4.1. Let C = ⟨A, +, ×, 0, 1⟩be a c-semiring, a ∈A and a × a ̸= a
unless a is 1 or 0. Then a × a × a × · · · = limi→∞ai = 1 if a = 1 and 0
otherwise.
Recall the two instances of c-semirings where × is not idempotent from chap-
ter 2:
• Copt: ⟨R+, min, +, +∞, 0⟩(optimization)
• Cprob: ⟨[0, 1], max, ·, 0, 1⟩(probabilistic)
First notice that here the multiplicative operation for both of these instances
are as speciﬁed in lemma 4.1. It is then apparent that for any value in the
domain we can trivially calculate the ﬁxpoints in ﬁnite time, this follows from
the intensive property of × which states that a × a ⩽S a.
As was mentioned before, c-semirings are composable. In the case of cartesian
products, the multiplicative operation is vectorized and operates independently
on each component of the combined c-semiring tuple. It is clear that lemma 4.1
still applies for the components of the composed c-semiring as before, but the
new c-semiring does not fulﬁll lemma 4.1 unless all of the c-semirings of the
cartesian product fulﬁll it. We assume for the other cases that similar behavior
is observed.

5.1
Fixpoints of inﬁnite domains
In section 4.1 we assumed that we could calculate all the ﬁxpoints that were
later presented in ﬁnite time for every domain. We discuss that the ﬁxpoints are
well deﬁned by Knaster-Tarski Theorem [Tar55] and then show how they can be
calculated for c-semirings that are as speciﬁed in lemma 4.1. Let us now consider
the c-semirings that do not have an idempotent multiplicative operation, but
have some relations other than 0 or 1 that are idempotent. Consider the ﬁnite
domain A = {0, a, b, 1}:
• The additive operation is totally ordered such that 0 ⩽S a ⩽S b ⩽S 1
which implicitly deﬁnes it.
• The multiplicative operation is commutative and deﬁned such that a×a =
a, a×b = a, b×b = a with 0 as the absorbing element and 1 as the unit. We
can show that it is associative and distributes over the additive operation.
So the deﬁned structure is a c-semiring and lemma 4.1 does not apply to this
speciﬁc case. Since the domain is ﬁnite even though lemma 4.1 does not apply,
it is apparent that the calculation can still be done in ﬁnite time.
Now let us consider a similar c-semiring as the one we just considered, with the
diﬀerence that it is deﬁned over an inﬁnite domain. Even though the ﬁxpoints
are well deﬁned, we cannot say for certain when the calculation tends toward 0
without knowing all the possible ﬁxpoints. This calls into question if we have
solved the general decidability problem for all c-semirings, but we can say with
certainty that we have solved the problem for a subset of them. On the other
hand we are unaware of any practical c-semirings at this time that that do not
fulﬁll lemma 4.1 given that their multiplicative operator is not idempotent.
5.2
Related Work
The model checker PRISM supports the speciﬁcation and analysis of properties
based on costs/rewards [PRI21]. The implementation is based on the ideas of
Kwiatkowska, Norman, and Parker in [KNP07] where they enhance DTMCs
(discrete-time Markov chains) with reward structures and extend PCTL (Prob-
abilistic CTL) to allow speciﬁcations over these structures.
The limitations
of this method, is that compared to C-CTL the rewards are limited to being
real-valued numerical values and so does not oﬀer the expressiveness of C-CTL.
==PAGE:43==
5.3 Future Work
33
Another limitation is that we can only accumulate values with summation over
paths which compared to C-CTL diﬀerent types of quantiﬁers are possible. The
obvious advantage of this method is that it has been implemented and can
already be used in a real model checker for real problems.
To specify quantitative security-requirements Martinelli, Matteucci, and San-
tini introduce c-semiring H-M Logic (c-HM Logic) which is a semiring based
extension of the classical Hennesy-Milner logic in [MMS15]. They use the logic
in the frame of Partial Model Checking which focuses on the veriﬁcation of a
part of a system only, this is diﬀerent from Classical Model Checking which
performs the veriﬁcation on the whole system. With this method they avoid the
combinatorial explosion of the state space which is also a problem in this thesis.
A speciﬁcation language for software architectures with quantitative features is
proposed by Paraponiari and Rahonis in [PR20]. The logic is called weighted
PCL (wPCL) which is an extension to propositional conﬁguration logic (PCL)
and is deﬁned over a commutative semiring. PCL can describe in a strict math-
ematical way qualitiative features of architectures, wPCL can describe quan-
titative characteristic such as the costs of interactions, the time needed, the
probability of the implementation of a concrete interaction.
5.3
Future Work
Working on the remaining question marks in table 5.1 is obvious future work.
Finding more eﬃcient ways to solve the cases presented in this thesis is some-
thing that could also be tackled. The solutions presented in this thesis are brute-
force approaches which do not scale well with larger transition systems. The
most obvious avenue to pursue is to propose algorithms speciﬁc to c-semirings
that are deﬁned as in lemma 4.1.
Finally it is important to consider c-semirings similar to the one proposed in
section 5.1. As was mentioned we are unaware of any such practical c-semiring,
but if there are any, we are uncertain if we can solve such an instance in ﬁnite
time.
