Abstract
This paper introduces the AnnoFormer, an interactive deep learning model for auto-regressive
annotation of object instances. Manual object segmentations can be a slow and expen-
sive process when thousands of segmentations are needed to create a dataset. To reduce
the price, interactive segmentation is used, which is computer assisted annotation meth-
ods, that create segmentations guided by a few inputs from an annotator. AnnoFormer is
a pure transformer model for point-wise contour prediction, with a transformer backbone
for feature extraction. At the center of the model is the transformer encoder/decoder
structure, which uses image features to find boundaries of objects and predict new pos-
sible boundary points, auto-regressively creating a predicted contour. The model uses
a ground truth bounding box as well as 4 clicks from an annotator as inputs. The An-
noFormer model does not yet reach results on par with current state-of-the-art methods.
The best model trained on real images reaches a mean intersection over union of 0.325,
which is far lower than results like the SegFormer from Xie et al.[1], which scores a 0.84
mIoU on a statistically similar dataset. However, on real images the AnnoFormer can
reach single intersection of union values as high as 0.91, showing great potential as a
interactive segmentation tool.

Introduction
Deep learning is on the rise in the computer vision field, and have been for some time.
The benefit of deep learning is the models is learning in a similar way to how humans
learn, by seeing examples and finding connections. Seeing enough examples lets a deep
learning model make a predictive guess for similar cases. By exposing a model to enough
examples it can learn to recognize patterns in images. The data for these visual models
is images with motifs for the purpose of the model. Models for self driving vehicles could
benefit from data with many images of cars, roads, street signs and other traffic relevant
objects. A model for cancer research would benefit more from many images of different
types of cell structures. Deep learning models learn by examples and because of that,
the amount of data needed to actually have them learn anything meaningful is very high.
While it would be best to train on unlimited data or at least towards several thousand
or millions samples, that case is highly impractical. More data takes more training time.
It also raises the prise of data, since all data needs to be annotated before use. These
annotations is made by humans, which depending on the quality of the work can be a slow
process. Manual annotations are very expensive and time consuming. The large COCO
[2] dataset required 22 worker hours per 1000 segmentations. At 2,500,000 different
segmentations that is 55,000 hours total. The detailed CityScapes [3] dataset took 1.5
hours per image, at a total of 25.000 images. Depending on the use case, an annotation
can come in different shapes and sizes, but for object detection or classification, datasets
needs to have a ground truth about object shape, location and class label annotated.
The reason for these segmentations is to have a detailed ground truth to train, validate
and test models on. An instance segmentation is a boundary of pixels along an object
instance. This segmentation can be a sparse type that only records corner point coordi-
nates. This would ignore points along straight lines, since a straight line can be described
by two points. Or it can be a full segmentation that includes coordinates of every pixel
along the object contour. Even a 4 point bounding box encircling the object instance is
useful and often used as input for models to let them work on single objects instead of a
full image [4, 5, 6, 7, 8]. An object segmentation can be used to create an object mask.
This mask is used to put focus on the object instance, highlighting it from everything else
in the image and letting a computer ”see” the object instance. Like the bounding box,
this is to help the model focus on the important areas in the data. The difference is that
a bounding box will still include many pixels which are not part of the object instance. A
high quality mask is made to more precisely split the pixels of the object instance from
other pixels. But the mask takes more time and is therefore more expensive.
To help with the high price and time of manual annotations, one can use interactive anno-
tations instead [8, 7, 5, 9]. Here the annotator will be assisted by a computer to create the
annotations, which speeds up the whole process. This lets the annotator posses a more
supervisory role, which requires less focus and precision, making the job easier, leading
to possibly more annotators [10]. The annotator will make a series of clicks [11], scribbles
[12, 13, 14] or bounding boxes [4, 5, 6, 7, 8, 15, 16] which all helps inform a model what
is important and what is not. Another interactive segmentation could be in the form of
reinforcement learning[8], by correcting a segmentation, a point or a polygon , instructing
the model where it went wrong. The polygon method [7, 8, 5, 17] is interesting since it
skips the pixel-wise classifications and starts with polygons instead, since a polygon is
also the end goal of such models.

In this paper, an interactive annotation method will be explored to create a supervised
deep learning model for auto-regressive object instance segmentation. The idea is to
have an annotator give a few examples of boundary points, by selecting them from an
object instance boundary on an image. The points should then be attributed with image
features and the model then use these features to find a connection within these boundary
points features. This will allow the model to recognise similar features throughout the
image, which should let it predict a contour around the object instance one point at a
time. The AnnoFormer model, which is the product of this master thesis, is designed to
predict points and creating a contour, which will then be evaluated using its intersection
over union with the ground truth segmentations of object instances.
The deep learning model will use a Visual Transformer [18] to extract image features from
an input image. These features will be used along with pixels chosen by an annotator to
train the AnnoFormer model to classify the next probable pixel in the boundary sequence.
For AnnoFormer to train successfully, there needs to be a learnable connection between
the image features. The hypothesis of this thesis is therefore that this connection is that
this connection exist in some unknown feature dimension, and that it is learnable by a
deep learning model, such as AnnoFormer.
The AnnoFormer model is able to auto-regressevely create segmentations, with as high
as 0.85 mean Intersection of Union on toy data made with masks from the CityScapes[3]
fine annotation dataset, and as high as 0.325 mIoU on the real images from the same
dataset. Individual contours score even higher, at 0.982 maximum IoU for the toy data,
and 0.910 maximum IoU on real images. The mIoU results from real images are far below
the current state of the art models. The mIoU’s are not in the same size range as e.g
SegFormer [1] at 0.84mIoU or Curve-GCN [5] at 0.737 mIoU. The AnnoFormer is tested
on statistically similar data, but still a different dataset, making it incomparable with official
results. The AnnoFormer results are instead compared to an untrained model and also
to the mIoU of the ground truth bounding boxes, to give a perspective of how well the
predicted contours align with real object instance segmentations.
In chapter 2 and 3 this paper list projects and articles with similar methods as that of
AnnoFormer, and explains tools used in the creation of the model. Chapter 4 will give an
overview of the model, explaining the different steps it uses to create a prediction. Chapter
5 will go through the different model iterations and the many tests and experiments, as
well as well as results in the form of Inter section over union statistics and examples of
predicted contours on the different data, and compare them with a baseline models IoU
statistics. Chapter 6 will validate and explain the meaning of the results, as well as explain
what did and did not work in the model iterations. Finally chapter 7 will comment on the
project as whole, and chapter 8 will mention ideas for improvement of AnnoFormer, and
what is next should the work continue.

2
Background
The background for the AnnoFormer is the Transformer deep learning architecture, which
will be described in sections 2.3 and 2.4. A short describtion of Convolutional and Recur-
rent neural networks can be found in sec 2.1 and 2.2, to highlight some methods that the
AnnoFormer model will be replacing with a pure transformer implementation.
2.1
Convolutional Neural Networks
Convolutional Neural Networks [19], abbreviated as CNN is included here to highlight an
alternatively commonly used method for Visual learning models. While the AnnoFormer
does not use CNN, it could have used them both for feature extraction in section 4.3 and
the encoder/decoder section of the model, section 4.5.
Within deep learning Convolutional Neural Networks are used to extract important fea-
tures from images. These features are anything that describes the image. Edges, corners
or colors are three major features. CNN’s are often used to reduce the dimensionality of
data, since the convolution can easily be made to output a smaller image than the input.
CNN is also used as the baseline for classification models, where the extracted features
are used for input to a Fully Connected layer (FC), which are trained to label object in-
stances, from the extracted features.
Larger models of CNN’s are used and trained to extract important features. Examples are
the VGGNet [20] and ResNet[21], which have great success and are used as the baseline
for many larger deep learning models.
2.2
Recurrent Neural networks
Recurrent Neural Network (RNN) [22] is a type of neural network which is used for se-
quential data. This data could be a time series or a sentence of words, making it popular
in the Natural language processing field. The neural network is described as recurrent
since the RNN is performing the same action on every sample in a sequence. After each
sample, a memory is kept, called the hidden state, so that previous samples in a sequence
will affect the next. The output for each time step t is therefore dependent on the current
input and the previous hidden state memory.
A known limitation for RNN is the sequence lengths of the inputs. When using backprop-
agation in an RNN, called backpropagation through time (BPTT), their gradients will, with
time, either vanish or explode, depending on the gradient sizes. This limitation was par-
tially fixed with the introduction of Long-Short Term Memory (LSTM) modules[23]. These
have more gates implemented in each layer, that can be used to ignore the layers caus-
ing gradient problems. This enables the network to learn longer sequences. However
LSTM’s are still limited by being a very horizontal model, since each unit is dependent on
the previous one, in a long chain of units.
Both RNN and LSTM are mentioned here, to highlight some of the previous methods used
for sequential data. These methods can largely be replaced by transformers as seen in
Devlin et al.[24].
2.3
Transformer
The transformer model was introduced by Vaswani et al. in Attention is all you need, 2017
[25]. Instead of using RNN (sec:2.2) to handle sequential data, the Transformer model is
AnnoFormer
3
==PAGE:14==
based fully on the attention mechanism. Attention is the method of focusing on specific
parts of an input, which are deemed more important than others. These important parts
depend on the context. In this paper the important features are object instances, more
specifically their boundaries. An example of attention of object instance in a visual model
can be seen in figure 2.2. A great feature of the Transformer is that it can run data in
parallel, even when used with sequential data. A feature not found in RNN. Instead of
working with sequence data one step at a time, it will use the full sequence as input and
calculate attention across all samples. This creates attention scores across the sequence,
which the model uses to learn connections in the data.
The Transformer architecture as described by Vaswani et al. can be seen in figure 2.1.
It is made up of N layers of transformer encoder/decoder layers, each with self-attention
and fully connected sub-layers. After the last decoder layer, the transformer follows the
architecture of a normal classification model, by mapping the model output to class labels
using a dense layer, and calculate probabilities using a softmax layer, that scale the num-
bers so they sum up to 1 and lie within 0-1, essentially translating them to probabilities.
Encoder
Each encoder layer is made up of 2 sub-layers; the self-attention layer, or as in figure 2.1
shows, a multi-head attention layer, followed by a Feed Forward Neural Network (FFNN)
layer. Following each sub-layer there is a residual connection and a normalization layer.
For N layers, the output of the first encoder or decoder layer is the input for the next layer.
The first layer needs inputs prepared with an embedding and a positional encoding. The
embedding can be any type of encryption of data into numbers. In Attention is all you
need, 2017 [25] the subject is NLP, so the data is sentences. In [25] they used learned
embeddings, which gives each token (which is a number representing a word) a large
vector of additional numbers describing it. This results in similar words having a similar
embedding, and therefore a similar meaning for a computer. A positional encoding is
added to that embedding to keep track of the order of the input sequence. Normally
with RNNs and sequential data, information is passed a single step at a time, so the
sequence of data is not changed and can be read in the correct order. Like a sentence
which is expressed one word at a time. That is not the case for the transformer, which as
mentioned, uses the full sequence at the same time. Therefore the data needs a position
encoded to it, to keep track of the correct order of the data, if the position is important at
the output level. When the input is prepared, it is split into a Key, Query and Value vector.
These are used as parameters for the attention mechanism. The input is also passed
through a residual connection, past the attention sub-layer.
Decoder
The decoder layer has the same function and principle architecture as the the encoder.
First it has the same self-attention sub-layer as the encoder, that splits the decoder input
into a Key, Query and Value vector, with a residual connection and a normalization layer
too. Then a new attention sub-layer is introduced, an encoder-decoder attention layer,
which is similar to self-attention, except the output from the encoder is used as the Key
and Value vectors, while the decoder supplies the Query vector from its own previous
self-attention sub-layer.

To further improve the attention mechanism, the multi-headed attention is introduced.
From the same input, several scaled dot-product attentions are set up, and each weight
matrix associated with a Key, Query and Value are initialized randomly. This gives several
representations of the same input data, which helps find a good representation. They are
trained in parallel, resulting in h different outputs, which are then concatenated. This
creates a large attention score matrix, which is multiplied with a learnable weight matrix
to reduce the dimensions back again. It now contains a vector of values for each sample
in the input sequence.
2.3.2
Softmax
After the encoder, the output is often a high dimensional vector or matrix, depending on
the model embedding size. This is passed through a linear dense layer, that maps the
output to a size equal to the used vocabulary size. The vocabulary is here represented
by a one-hot encoded vector for each vocabulary token. Now a value for each token in
the vocabulary is present. A softmax layer normalizes this long output vector, so that all

values are between 0-1 and they sum up to 1, making them into probability values for each
token in the vocabulary. The highest probability, will be chosen as the model prediction
at the current sequence step.
2.4
Visual Transformer
The Visual Transformer is the name of the model made by Dosovitskiy et al. in An Image
Is Worth 16x16 Words: Transformers For Image Recognition At Scale[18]. They use the
original transformer model from [25] as close as possible, but the input data is replaced
with images instead if sentences.
Figure 2.3: Visual Transformer Architecture from An image is worth 16x16 words [18]
In order to use images as input, they are first embedded using patch embedding. This
turns the image into a sequence of patches, of a size p, which are then flattened to a
1d sequence using a linear projection layer. A positional embedding are added so the
sequence order is kept through the model. This could be a simple embedding of adding
the patch positions in order (0,1,2...d), but it could also be a more complex embedding like
a trigonometric function, that uses a sin or cos frequency to describe the inputs position.
As an extra input embedding, a learnable classification token is also added, to help the
model recognize different classes. The encoder and decoder are used as they are from
[25]. The outputs are classified with a multi-layer perceptron, which gives a probability for
all classes, to show which class the input is most likely to belong to.

Related Work
Interactive methods for semantic segmentation as well as Transformers in computer vi-
sion have both been used in many projects before. Transformers as standalone tools
or for parts of a larger model. Interactive annotation is a popular alternative to manual
annotations.
3.1
Interactive segmentation methods
Interactive methods is when a computer is used to assist a human annotator to create
annotations. Extreme clicking[9] is an example of an interactive bounding box segmen-
tation. Here a bounding box is created by a user by selecting the 4 extreme points of
an object instance. Bounding boxes are often used as the minimum pre-processing for
computer vision tasks [4, 5, 6, 7, 8] and they are created interactive annotations. Ob-
ject detection tasks such as [4] can help find object instances and create these bounding
boxes as well. Another method for providing additional meaning by interacting with the
model, is a scribble, introduced by Boykov et al.,2001 [12]. This is a sketch or notation
to the image made by an annotator that helps outline the foreground or background for
the model to better see the object instance. In the same way, boundary points can be
pointed out to a model, to let the model have extra features to work with at the contour of
an object instance, as done in [11], where an annotator can place points by clicking.
These methods, with the exception of the boundary box methods, operates on a pixel
level of notation, where indicidial pixels are assigned a label such as background, object
or boundary. Another type of interaction is with polygon models [7, 8, 5, 17], that as the
name suggests, utilize polygons instead of pixel labeling. Here the annotator can as an
interaction place the polygon closer to the true boundary, by dragging its vertices around.
These models [7, 8, 5] skip over the pixel level methods, and straight to the polygons,
since in many cases a polygon or mask is the end results of a segmentation model. The
polygon models [7] and [8] are quite similar in method to the AnnoFormer. They each
predict a single vertex at a time, creating a polygon around an object, like the AnnoFormer
predicts a single point at a time and creates a contour.
3.2
Click methods
One of the subsections of interactive segmentation is the click methods. Here an anno-
tator will provide indications or corrections to the model by clicking on the input image.
This could be used as a prior for setting an input to a model, which is what this paper will
use. A few points along the object instance boundary used as examples of correct feature
representations for a boundary pixel, as done by Maninis et al.[11] and Papadopoulos et
al.[9].
In Extreme Clicking[9] they expand on the bounding box in 2 ways. Normally a bound-
ing box can be created by 2 points, namely a set of the upper-left/lower-right, or upper-
right/lower-left corners of the box. This will give the bounding box a corner with start
coordinates as well as a width and a height. Or created by the 4 corners of the box, which
gives the same information. This can be difficult to place by a user, since none of the clicks
involved are actually placed on the object instance, but around it. The vertices made by
the bounding box is what the touches the object instance. Extreme clicking instead pro-
poses to create the bounding box by its vertices. The annotator picks the 4 extreme points
from the object instance. Top, bottom, left and right. This makes the annotation easier for
AnnoFormer
9
==PAGE:20==
the annotator. The second difference is the added importance of the 4 clicks. Now they
notate places on the object contour, instead of being placed on the image background far
from the object. A correctly adjusted deep learning model will be able to use the extreme
bounding box as well as the 4 points to find a better contour to work with. An example,
as shown in [9] is the use of an edge detection, where any edges between 2 of the 4
boundary points will have a high probability of being part of the object instance contour.
Other click based methods could be to notate an object with a single, or a few clicks like
in [26, 27]. This is similar to the scribble method, in that it gives the model an indication
of what is the background and foreground.
3.3
Transformer methods
Transformers and attention have seen great results within the natural language process-
ing field. The Bert model by Devlin et al., 2019 [24] is a great example of a transformer
improving on the state of the art. Within short time the transformer was introduced to com-
puter vision as well, with the Visual Transformer by Dosovitskiy et al., 2020[18] being the
first pure Transformer model in computer vision, demonstrating a possible replacement
for CNN. The SETR model by Zheng et al. 2021 [28] is an example of a pure trans-
former model used for sequence to sequence translation within computer vision, wherein
they even provide a comparison FCN model as a baseline. They follow the embedding
method of the ViT [18], using patch embedding and then flattening the input to a 1d se-
quence. A focus of Zheng et al. [28] is the size of the receptive field of the transformer,
which is larger than in the FCN comparison model.
DETR by Carion et al. [4], is an object detection transformer, that operates on image
features, similar to the proposed AnnoFormer. However DETR uses a CNN backbone
to extract the features, instead of a Visual Transformer. PolyTransform Liang et al. [17],
as previously mentioned is a polygon instance segmentation method, which also uses
a transformer. They use an initial mask as a backbone, then translate it to polygons
which are deformed by their transformer model to a closer approximation of the true mask
shape. The transformer is also a fine supplementary tool, as seen in SegFormer [1], Xie
et al.,2021. Segformer uses a transfomer encoder, but a MLP layer as a decoder, making
it a hybrid model. Here the transformer functions similar to a CNN for feature extraction, to
get both high resolution coarse features and low resolution fine features, which the MLP
then fuses an upsamples to a semantic segmentation.

4
Method
In this chapter is an overview for the model architecture and the methods used to process
the inputs.
4.1
Model Overview
AnnoFormer uses pre-processing to prepare the input images. A pre-trained Visual Trans-
former to extract image features, along with cropping and resizing of object instances
using ground truth bounding boxes. The resulting feature image is sparsely used, by
only using a minimum of 2 starting points provided by an annotator, as well as m candi-
date points, sampled from the feature image. The unique candidate points are calculated
during run time, for each sequence step. The transformer encoder/decoder then finds a
probability for each of the m candidate points, and uses the most probable as the pre-
dicted next point. The model then discards the first starting point and adds the prediction
to the end of the start sequence, making the model predict a full contour auto-regressively
one point at a time.

output choices of the model to m possible points. Sec:4.5 - The input points features are
used to find the most probable output among the candidate points.
4.2
Inputs
The inputs for the model are an image with an object instance and a series of start points
from an annotator. The object instance is first specified by a bounding box, to reduce the
number of objects the model can see at the same time. The points is the annotators way
of choosing what object to annotate, and with them the model will begin predicting similar
points and create the complete object contour.
4.2.1
Image cropping
When an image is loaded it is cropped to a ground truth bounding box provided and
resized, as illustrated in figure 4.2. A single image contains several object instances, but
the bounding box informs which object instance is the current one to focus on. To avoid

4.3
Pre-trained Visual Transformer
To extract image features from the input, a pre-trained Visual Transformer model from
the timm library (PyTorch Image Models), called vit_base_patch16_224, is used. It is a
Visual Transformer as described in section 2.4, with a patch embedding size of p = 16
and an input image size of 224. While similar feature extracting models exist, for example
the well trained ResNet-50[21], a visual transformer was chosen to keep the backbone
within the Transformer architecture and avoid CNN. This visual transformer is an image
classification model, so it can extract image features and use these to classify objects.
By removing the last linear layers from the model, the output will be the extracted image
features instead of a classification.
The output is of size 196 x 768, which is reshaped to 14 x 14 x 768, a spatial size equal
to the input image size divided by the patch size.
The output is then upsampled to
224 x 224 x 768, which is the same as the resized image size, as seen in figure 4.3. Up-
sampling is done with bicubic interpolation, giving a larger resolution to the feature image.
The feature image now contains a vector of 768 feature dimensions, which describes each
pixel in the resized input image.

4.3.1
Additional features
Normally for a transformer model, the inputs would have an embedding created by some
type of encoding, including a positional encoding. The extracted features are used as
embedding, and then the pixel coordinates are added as additional features, to give the
model a sense of the spatial dimensions of the contour shapes. Each point of interest
in the model lies at a very specific distance from each other, depending on the run time
resolution as described in section 4.4.1. The xy coordinates also acts as positional en-
coding for the model. While being a very basic form of encoding, it makes it possible to
get a spatial position from the model results, by reading the xy features of the predictions
as coordinates. This makes it possible to plot the predicted pixels as well as use them for
inputs in the auto-regressive model. The additional features are added by concatenating
an xy array [224 x 224 x 2] of coordinates, with the feature image [224 x 224 x 768] result-
ing in the final [224 x 224 x 770] feature image that the model will use. The xy features are
normalized to the 0-1 range, which lies closer to the other features in size, so they will not
have an exploding effect on the model weights later, or become the dominant features
the model will train on.
4.4
Candidate points
The candidate points are the possible vocabulary for the transformer model at each itera-
tion. In a NLP task, the vocabulary would be a list of words to chose from, and the model
would give a probability for each. Since this is a visual model, it would normally give a
probability to each pixel in the image. With an image of size 224x224x770, that would be
50176 different pixels, which is 1.64 times larger than the Bert [24] vocab at 30522.
However, most of the image pixels are unlikely to be part of the actual contour, and a
smaller selection can be defined for every input sequence instead. The vocabulary is set
to a very specific selection of points, referred to as the candidate points, which for the
start sequence is pre-calculated, but otherwise calculated during model runtime.

4.4.1
Resolution Sampling
Resolution in this paper is the term used to describe the amount of sampled points of a
contour. The resolution number corresponds to the total pixel distance between points in
a contour. Making a resolution of 1, approximately 10x larger than a resolution of 10. A
resolution of 10 would mean that there needs to be a distance of 10 pixels between to
pixels, measuring 1 unit with a standard 4 pixel connectivity, and
√
2 units for diagonally
placed pixels. The distance is counted cumulative along the full contour, and only when a
contour point is at least a distance from the previous point, measured along the contour,
will it be added to the resolution sampled boundary. A faster method can also be achieved
by letting all pixel distance be equal to 1. This can be used with simple segmentation
shapes that require less details, as this methods is slightly less precise.
The resolution defines how many points are in the contour the model is training on, but it
can also be used as a tool for generating additional training contours. Different resolutions
would mean different points from the full contour being used. To do this, a random resolu-
tion is implemented in later models, used each time a contour is calculated. That is, a new
random resolution for each sample in each batch in each epoch during training. Creating
several training contours from a single segmentation. A random resolution is based on
the original segmentation size. The extreme resolution sizes is set to 8 minimum and 40
maximum. Below 8 and the candidate points as described in section 4.4 would risk being
duplicates because of the rounding to integers. Above 40 would for most segmentations
create a sampled contour of too few points. The list of the random resolutions is found
by taking the modulo ’%’ operation of the segmentation size, for each value in the range
8-40. This way, all the values that the segmentation size can be divided by and equalling
to an integer is found, which are the optimal resolutions for that segmentation since they
wont have any points left over in either end of the segmentation when skipping points.
Should the segmentation size be equal to a prime number the highest resolution of 8 is
used.
4.5
Transformer Model
The transformer model consists of a transformer encoder, with N layers, a transformer
decoder with N layers and a dense layer. The model depth is defined by the number of
encoder/decoder layers, and can be increased to heighten the complexity of the model
The encoder/decoder is set to have an embedding of 770, to work with the pre-trained
ViT outputs with additional features. They are using H heads in the Multi-Head Attention,
and have a hidden dimension size of 200. The hidden dimensions can be increased to
heighten the width of the model.
As an example, let the pre-processed contour be a sequence of points ABCDEFGHIJK.
This is split into smaller sequences for training, according to a variable, in this case 4,
resulting in a number of sequences ABCD, BCDE, ...KABC, equal to the size of the
contour. The lower size limit for these input sequences is 2, since the candidate points
requires 2 points to calculate the rotating vector used to find the other candidate points.
Each 4 size sequence is in turn passed to the encoder. For the sequence ABCD, the
decoder would then be supplied with the candidate points for E, denoted E0, E1, ...Em.
The candidate point Ei, closest to the ground truth next point E is then the candidate point
with the lowest mean squared error to E and the index i of Ei is saved, to be used with
Cross Entropy Loss which is the models loss function. The model will return a selection
of probabilities to the vocabulary of size m, same size as the number of candidate points
Em. The highest probability, found with a softmax layer, is used as the model ”prediction”.
The model moves on, the input sequence is now BCDEi and the candidate points are

All contour tests are made at 3 resolutions. Resolution 8, where there are 8 pixels be-

All contour tests are made with an input of 4 start points, from the ground truth segmen-

the only input points the testing will use. All other points of the predicted contours will be
made by auto-regressive predicting 1 point at a time, and adding this to the existing point
sequence.
The chance of a failed contour rises with each additional predicted next point, so to in-
crease the chance of success, double inference is used when testing. The input points
from the annotator are used in both directions around the object, making the final contour
from a clockwise and counter-clockwise contour each covering half of the object instance.
This halves the length of each predicted contour, lowering the chance of bad predictions
significantly. The testing will continue to auto-regressive predict new points, until the
clockwise and counterclockwise contours meet. They will then be joined to create the
final contour. In the case of a failed test, the contours will be limited to 80 predicted points
total, and the contour is given a test result of 0.0 IoU.
To further help with the success rate of the tests, any new predicted points must be located
a distance equal to 1/4 the test resolution, away from the currently predicted contour
points. This is done by setting the probability to 0, of the model outputs located within
this distance of the current contour. Should all model output probabilities be set to 0, the
original highest probability will then be used, instead of just stopping the contour. This fail
can be recognized by the contour forming an asterisk (*) shape on the images, as seen
in figure 5.13b.
For all tests on toy data, the predicted contours will be displayed in green, and the ground
truth are represented by the shape of the object instance in white. For all tests on real
data, the predicted contour will be displayed in green, and a ground truth contour will be
set as reference in blue. The contour examples will all be made with a test resolution of
8, as this is the currently highest resolution the model can use.
Failed tests
A large amount of the predicted contours are scoring a low IoU. An un-
known amount of these failed tests emerge from the way the test are currently performed,
and not necessarily from an insufficiently trained model. To test the Intersection over
Union of the contours, they are made into polygon objects from a python library (shapely).
This library have build in IoU methods, but these requires valid polygons, with a good ge-
ometry. Many polygons created from the predicted contours do not have a good geometry,
and will therefore be unable to calculate an IoU value, instead resulting in a 0.0. This low-
ers the mIoU of the tests significantly, so additional statistics is used to better show the
real IoU distribution. The minIoU, which is the smallest measured Non-Zero IoU. That
is, the worst predicted contour, that has not failed. The maxIoU is the highest IoU value
recorded for the model. This is the overall best result from the model. The medIoU is
the median value for the IoU test. 50% of the IoU results are above this value. A low
medIoU indicates a lot of failed contours, and will together with the mIoU give a better
representation of the IoU test distributions. For all the IoU tests, higher values are always

Experiments on image features
The model builds on the assumption that contour pixels are in some dimension different
than other pixel, and that this difference is enough to train a model on. For an input image
wxhxc, it could be an RGB image of size 640x440, a very normal image format, every
pixel is defined by 5 dimensions. 2 spatial dimensions and 3 color. Together they define
how the pixel will look, and where it is. Group 1000 pixel together and you have an image
and this image often have a motif. A motif in an image, it could be a car or a dog or a
million other objects, is defined by the pixels it is made of. This means a neighborhood
of pixels in an image have more information to offer when grouped together. They have
a context. A structure between them that only makes sense when they are grouped the
whey they are.

5.3.2
Principal Component Analysis
Principal Component Analysis, PCA, uses Singular Value Decomposition (SVD) to find
the dimensions that are responsible for the highest variance of the data. In other words,
it finds ways to ”rotate” the dimensions around and finds which directions the data can be
viewed from that reduces the dimensions the most, without loosing too much information.
A number of principal components equal to the number of data dimensions can be found
in data. In this case 768 dimensions will result in 768 principal components, and they
will describe 100% of the data. They can be sorted by percentage of variance explained,
so they are ranked from the largest principal component to the lowest. Depending on
the image, the 768 components would describe different percentages of the explained
variation, but they are mostly the same, differing by ±5%.

5.4
Initial models
Transformer Encoder ViT
One of the initial Transformer models used only a trans-
former encoder with 6 layers, and a dense layer after this, mapping the encoder output
to the vocabulary size, and then to probabilities with a softmax layer. The model was
designed to predict a single pixel from a vocabulary, which would be the next sequence
point. The vocabulary was set to be the ground truth contour, same as the contour it was
trained on. This meant it had a great success rate in predicting pixels from the contour,
since this was the only pixels available to it. But it failed in predicting the points in the
correct sequence order, which was its main goal. It did not utilize the 768 feature di-
mensions of the pixels for anything directly, in that it had no way of seeing the features
when calculating the probabilities of the next points. Instead it chose the blindly from the
available vocab. Another problem of the model, was it’s inability to train on more than a
single contour, since the vocabulary size was decided from the amount of contour points
available. A changing vocabulary size would not be possible, so changing contour sizes
could not change either. This could be reworked by padding the contours. The results
can be seen in figure 5.5b. What figure 5.5b can’t show here, is the amount of predicted
pixel which are repetitions. Figure 5.5a and 5.5b have the same amount of contour pixel,
but in 5.5b they are mostly layered on top of each other.

Angle Transformer
The next Transformer model worth mentioning is the angle predic-
tion model. The vocabulary changed to a constant size of 36, representing 36 different
angles to rotate, to find the next point at a constant distance. A prediction of class 1 would
mean the direction of the current contour line should be rotated by 10◦, class 2 would be
20◦, etc. Running in to some of the same flaws as the Transformer Encoder ViT model,
the Angle Transformer did not get to see the 770 feature dimensions of the 36 possible
classes it would classify as the next points.
To test this model, the toy dataset from section 5.1.1 was created. The idea being that
simple shapes like a circle would result in the model predicting the same angle with every
input sequence. Which it did. Then for a simple square, it should predict an angle of 0◦
along the vertices, and then a 90◦at the corners. Which it did not. It would just predict a
straight line instead of a square, with a constant angle of 0◦.

Edge feature experiments
Initially 3 models with a limited amount of features were created, to test if less features
would make it easier for the AnnoFormer to create a contour. To scale down the complexity
of the image features and determine if the AnnoFormer is capable of learning from features
as intended, a new feature image is made with only 3 features, as examples seen in
figures 5.1b, 5.1e and 5.1h. The additional features as described in section 4.3.1, and
then a new feature from a canny edge detection [31]. The edge feature will be a high
value around the edge and a low value everywhere else in the feature image. The high
value is a magnitude larger than the xy features, which is scaled to 0-1. This should make
the edge feature the dominant feature the model will follow.
The 3 model versions are tested at 8, 15 and 25 resolution. For complex shapes the res-
olution at 8 are expected to perform best, for larger shapes with less detail the resolution

6
Discussion
6.1
Image features
The initial experiments on image features in sections (5.3.1, 5.3.2, 5.3.3) show that the
boundary features in the images is somewhat learnable, which is good news for the
project. The t-sne fails to separate boundary points from non-boundary points. While
this does not mean a difference can’t exist, it does mean it is not an obvious difference
that are easy to quantify.
To test if a model can find a connection in the features, the support vector machine
(sec:5.3.3) is used before moving on to larger experiments. As seen in figure 5.4 this
model can label pixels well enough to make the object instances recognisable when the
input image is present. This is with the feature dimensions downscaled to 32, and only
94% of the data variance explained, as described in section 5.3.2. The original dimen-
sionality of 770 should include more information but it will also require a more complex
model to find a connection. But this SVM is a great indicator that the data can support the
AnnoFormer idea.
6.2
Initial models
Transformer Encoder ViT
was trained so that it could predict a sequence of contour
points in the correct order, but it did not succeed very well. It mostly predicted the same
few points several times, over and over again, instead of a new point at each sequence
step. Since the points represents the classes in the model classification, the model mostly
predicted the same class. With the inability to scale the training to more images without
adding padding to the inputs, it was also difficult move to larger tests of this model version.
The vocabulary size of the model, which is equal to the contour size, could not change
after the model is initialized, and so the model could only be trained on the single image.
The vocab issue was improved by rethinking the prediction process. If each next point
lies at a constant distance from the current point, then the next point could be expressed
by an angle. It was upgraded to the Angle Transformer model, with a constant number of
classes.
Angle Transformer
had some of the same problems as the Transformer Encoder ViT. It
had a tendency to predict a single class only. For the angle transformer, the class index is
used as an angle, meaning a continous prediction of class 8 would change the direction of
the next point by 80◦. The only possible shapes created by this would be regular polygons,
with all angles at the same values, as well as a straight line. The Angle Transformer
with its 36 classes, representing angles turned out to be the test of the candidate point
4.4 component, which ended up becoming an important part of the AnnoFormer models
architecture, making the Angle Transformer a crucial experiemt in the project.

The 3 edge feature models (EF-CP33-Res10, EF-CP69-
Res10 and EF-CP69-Res15 ) showed great results as seen in table 5.2, 5.3 and 5.4, and
managed to predict complete contour as seen by the examples in figure 5.6. The edge
feature models could have been trained further, seeing as both the training and validation
loss in figure 5.7 did not converge. The accuracy and the average validation angle error

could have continued, for testing more combinations of hyperparameters. So far the can-

Starting with DFToy-Init, the first test with deep features on the

consistent values in the 3 resolution tests.
6.4
Deep features on real data
The tests on real images shows a lot of difficulty with finding a pattern in features. Al-

more data was available for training, which should have helped with overfitting, but this
would have increased the training time. Training a model with 4000 samples already
took more than 80 hours, so more time would be needed for larger tests. Many of the
Deep feature models on real data was trained in parallel due to the long training time,
so they did not influence each other as much as the deep feature models trained on toy

baseline tests are made to compare the deep feature models trained on real images.

Conclusion
In this thesis, an auto-regressive deep learning model, AnnoFormer, for interactive ob-
ject annotation was designed. It was first trained and tested on a toy dataset created
specifically for this project, using fine detail object masks. Then it was trained and tested
on the well known CityScapes [3] fine annotations dataset. AnnoFormer is designed as
the first pure transformer model for interactive segmentation. Studies with similar work
from the deep learning community was needed to learn how current transformer models
work, and how current interactive segmentation methods work. There have been many
improvements within this field in recent years, reaching very high intersection over union
values, like the SegFormer[1] model at 0.84 mIoU on a similar CityScapes dataset. How-
ever since AnnoFormer is tested on a different partition of the dataset than official papers
like SegFormer, they can not be compared directly. The closest in method to this project,
is the polygon methods [7, 8], which in a similar way predicts the boundary around an
object instance. At the time [7, 8] showed new state-of-the art results for interactive seg-
mentation with a 0.7138 mIoU, but have now been surpassed many times by newer papers
such as SegFormer [1].
Creating the AnnoFormer model required a series of experiments to first confirm that
there in the feature dimensions were a difference between an object instance boundary,
and every other pixel in an image, and that this difference could be learned when training
the model. It was not possible to visualize this connection in the data with dimension
reduction methods such as PCA or T-sne tests, but a support vector machine model was
able to label the correct pixels in an image as the boundary of an object instance, with
very little training. Then a series of experiments showed that a pure transformer deep
learning model was also able to learn the boundary pixels features and predict successful
contours from them. First on the easier toy dataset, then again on a dataset with real
images.
On toy data, the AnnoFormer model scored very high intersection over union values. As
high as 0.835 mIoU, and on real images it scored a max of 0.325 mIoU. The model was
able to score individual intersection over union values at 0.8 and above, and the highest
score on real images is 0.91 IoU. Comparing these results with the bounding box baseline
results, the bounding box mIoU at 0.567 is still on average a closer approximation to the
object instance shapes than the AnnoFormer contours at 0.325 mIoU. Lastly the model
components effect on the IoU values were measured in the extra ablation studies. This
showed that the current parameter values are mostly optimal, with the exception of training
data amount, and amount of candidate points the model used. The tests was however
insufficient in showing if the values should be higher or lower for future experiments. In
conclusion, the AnnoFormer model does not yet reach state-of-the-art levels of precision,
but shows potential.

Future Work
Interactive segmentation is not only about creating accurate segmentation. It is also about
creating them faster than manually segmenting object instances. The processing time of
the AnnoFormer have not yet been measured on images with an annotator. A future
goal would be to test the actual prediction time, and push it down to 5 seconds or less,
if possible. If the time is sufficiently low, it could also be interesting to see if the model
can improve on its own predictions, by letting it use its previous contours as additional
feature dimensions. The current implementation leaves plenty of room for improvements
like these.
So far all tests are started using ground truth segmentation points as inputs. It would be
important to test how robust the model is to noise by using less precise points. Different
challenges are also presented in occluded object instances, bad lighting, shadows or low
image resolutions. The current implementation needs more testing and troubleshooting,
in order to better quantify the real potential of AnnoFormer. The bounding box baseline
test shows that some of the ground truth segmentations are loosing details or completely
failing during loading and processing. If this happens frequently during testing, it might
also happen during training, resulting in training the model with bad data.
The AnnoFormer shows potential as an interactive segmentation tool, but it needs to be
tested in a more regulated way, like other papers in the deep learning field, before it can
be recognized as a useful tool. It should be prepared for testing on datasets like the
official CityScapes [3] test set, ADE20K [32], Pascal [33] or ImageNet [34]. The best
model version of AnnoFormer was trained on twice the training data as the second best
model version, and it still only used about 10% of the available CityScapes training split.
With many of the models suffering from overfitting, it would be exciting to see how far the
current model version can be pushed, if trained on 40.000 segmentations instead of only
4.000. It could even benefit from a pre-training on the detailed CityScapes segmentations
and then test if it can generalize to another dataset like COCO [2] or ImageNet [34].
