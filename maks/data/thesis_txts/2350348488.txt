Abstract
Presenting aÔ¨Éne Grassmann codes based on the work of P. Beelen et. al. in the
articles [1], [2], and [3]. We determine the length, dimension, minimum distance, and
number of minimum weight codewords of any aÔ¨Éne Grassmann code. We explicitly
determine the dual of the aÔ¨Éne Grassmann code and compute its minimum distance.
The minimum weight codewords of the dual of any aÔ¨Éne Grassmann code shown
to be 4 when considering the Ô¨Ānite Ô¨Āeld of size two and 3 when the size is greater
then two. Moreover we classify and count the minimum weight codewords of the
dual aÔ¨Éne Grassmann code.
Furthermore we give an implementation of aÔ¨Éne Grassmann codes in SageMath.
The code is implemented as a class containing many useful methods. A key feature
is the ability to generate all minimum weight codewords of the dual constructively.
Two decoding algorithms are presented and implemented, both of them exploiting
the minimum weight parity checks. One has a local iterative approach and the other
a global error detecting approach. The idea of the global decoder emerge as a part of
this study. Simulations on numerous aÔ¨Éne Grassmann codes have been conducted
over Ô¨Āelds of size 2,3 and 5. Many interesting observations have been made. It
was believed that the local decoding algorithm would be able to guarantee correct
decoding upto half the minimum distance. But we have managed to Ô¨Ānd counter
examples on this conjecture.

Introduction
This thesis presents the research done by P. Beelen et. al. on aÔ¨Éne Grassmann
codes in articles [1], [2], and [3]. By collecting the work from 2010 to 2015 into one
cohesive paper. The majority of the work will be presented in its full extend with
complete proofs.
In the attempt of making the thesis as self contained as possible we introduce
the most essential concepts of abstract algebra on which the aÔ¨Éne Grassmann codes
are based upon. Furthermore we introduce the most central aspect of coding theory.
Our goal is to establish the theoretical foundation needed to fully comprehend
the aÔ¨Éne Grassmann codes. Enabling us to make a practical implementation and
hopefully obtain a deeper understanding of these codes.
The aÔ¨Éne Grassmann code is introduced in chapter 2. In section 2.1 the con-
struction of the code is presented along with several examples. Thereafter we deter-
mine the minimum distance of any aÔ¨Éne Grassmann code in section 2.2. In section
2.3 we present the automorphisms we will apply on the aÔ¨Éne space on which the
aÔ¨Éne Grassmann code is evaluated upon. In eÔ¨Äect we prove that our group of trans-
formations is a subgroup of the automorphism group of the aÔ¨Éne Grassmann code.
In section 2.4 the dual of the aÔ¨Éne Grassmann code is explicitly given in terms of
its basis. And the minimum distance is found to be 4 over the binary Ô¨Āeld and 3
over any non-binary Ô¨Āeld. Lastly we classify these minimum weight codewords of
the dual code in regard to our group of transformations. Furthermore the exact
number of minimum weight codewords in the dual of any aÔ¨Éne Grassmann code is
given.
A signiÔ¨Ācant part of this study is the practical implementation of an aÔ¨Éne Grass-
mann code in the programming language SageMath. In chapter 3 we document this
work and go through some examples of interaction with the implemented code. Fur-
thermore we present two decoding algorithms. Both of them exploiting the large
number of minimum weight parity checks. The Ô¨Ārst algorithm is based on a local
point of view trying, to correct each coordinate of a received word by exploiting the
minimum weight parity checks containing that coordinate. The second algorithm
is based on a global point of view.
In this we compute a vote for each coordi-
nate exploiting all the minimum weight parity checks not containing that speciÔ¨Āc
coordinate. Having these votes we decide which coordinates are most likely to be
erroneous and correct these by erasure decoding.
In chapter 4 we present the results of the simulations on the diÔ¨Äerent decoding
algorithms. And give examples on some of the minimal error patterns that result in
an unsuccessful error correction. In relation to the recent work on aÔ¨Éne Grassmann
codes there was a believe that the local decoder could correct errors upto half the
minimum distance. This is falsiÔ¨Āed by found counter examples. As a contribution
to the research on aÔ¨Éne Grassmann codes there have been conducted experiments
on the decoding of aÔ¨Éne Grassmann codes over a Ô¨Ānite Ô¨Āelds of size greater than 2.

3
Practical implementation and decoding
A signiÔ¨Ācant part of the work presented in this thesis, is the practical implementation
of an aÔ¨Éne Grassmann code in SageMath.
In this section we will present and
document this work and go through some of the consideration concerning the chosen
data structures.
We will present two diÔ¨Äerent decoding algorithms of the aÔ¨Éne Grassmann code.
Both of them exploiting the fact that it has so many parity checks of weight 4
and 3 for q = 2 and q > 2 respectively. The Ô¨Ārst algorithm, local decoding, comes
from a local point of view. Here we consider one coordinate at a time and deter-
mines which value this coordinate should have based on the minimum weight parity
checks containing that coordinate. The second algorithm, global decoding, is quite
the contrary. This time we go through all coordinates and calculate how many of
the syndromes gives a zero. But in complete opposition to the local approach we
disregard all minimum weight parity checks relevant to that coordinate. In this way
we end up with a global view of which coordinate is most likely to be erroneous.
To correct these errors we simply erase the d‚ąí1 most likely coordinates. If all the
erroneous coordinates are erased we are sure to recover the sent word by solving the
erasure problem explained in example 8. If not we have no solution. In the following
we will go into further details on these two approaches. Both algorithms have been
implemented and several results will be presented. The entire source code can be
referred to in the appendix 6.3.

3.2

In section 2.5 we determined the exact number of minimum weight parity checks
of an aÔ¨Éne Grassmann code. Furthermore we showed how these minimum weight
parity checks lies in diÔ¨Äerent orbits when the group G(‚Ąď,m) of transformations acts

This fact oÔ¨Äers a direct approach of generating these minimum weight parity
checks simply by using a representative of each orbit and letting all transformations

been implemented for the case q > 2. This was the Ô¨Ārst priority, since there has not
been any experimental experiences for q > 2 so far. For q = 2 the brute force method
has been used merely to save time on programming. The constructive method is
really fast and allows us to generate, e.g. over 2 million parity checks for CA(3,6) over
F3 in a matter of minutes. The method generate_minimum_weight_parity_checks
called on an instance of an aÔ¨Éne Grassmann code constructs all minimum weight
parity checks for the code. The source code of this method can be found in the
appendix 6.3 line 195-286.

3.3
Local decoding
The Ô¨Ārst decoding algorithm is based on a local approach of correcting the errors.
It is local in the sense that we single out one coordinate at a time of the received
word and try to correct that single coordinate once and for all based on the relevant
minimum weight parity checks. Formally speaking let r be the received word. Con-
sider some coordinate rs1 of r. Pick out all the minimum weight parity checks with a
non-zero entry in coordinate s1. Let h be one of these parity checks with support set
{s1,s2,s3,s4} in the binary case and {s1,s2,s3} in the case where q > 2. By solving
the equation rs1 +rs2 +rs3 +rs4 = 0 or hs1rs1 +hs2rs2 +hs3rs3 = 0 respectively for
rs1 we get an indication of what rs1 should be. If none of the other coordinates,
from the support set of h, is erroneous then surely rs1 will be the correct value. So
by solving the equation for all minimum weight parity checks and keeping track of
what value of rs1 each parity check prefers, we get a most likely value of coordinate
rs1 by choosing the one most parity checks satisÔ¨Āes. This in turn implies that the
received value of rs1 is irrelevant for the decision. The procedure is carried out
for each and every coordinate of the received word, and we call this a sweep. The
algorithm is presented in algorithm 1.

3.4
Global decoding
The second decoding algorithm is based on a global approach of correcting the errors.
The idea here is that we go through all coordinates of the received word, but this
time we only regard all the minimum weight parity checks that does not concern
a given coordinate.
The incentive is that when we count how many syndromes
gives a zero, the erroneous coordinates will have the highest counts, simply because
the erroneous coordinate is disregarded when calculating its syndromes. When all
coordinates have been calculated we have a count, vote, for each coordinate.
Now considering all the votes for each coordinate position one could hope that
the erroneous coordinates would be among the highest d ‚ąí1 votes. Then by sim-
ply erasing the d‚ąí1 highest votes from the received word and solving the erasure
problem one would get the sent word. Now if just one erroneous coordinate is not
erased there is not any solution to the erasure decoding problem.

5
Conclusion
We have presented the aÔ¨Éne Grassmann code. This has been done on the basis
of the most essential preliminary theory from abstract algebra and coding theory.
The main results of this study has been the following: The determination of the
minimum distance of the code. We have shown that the group of transformations
is isomorphic to a subgroup of the automorphism group of the code. An explicit
determination of the dual aÔ¨Éne Grassmann code has been given and the minimum
distance of this has been computed to be 4 over the Ô¨Āeld of size 2 and 3 otherwise.
Furthermore the minimum weight codewords of the dual code have been classiÔ¨Āed
into three and one orbit, respectively in regard to the group of transformations.
A signiÔ¨Ācant part of this study has been the implementation of the aÔ¨Éne Grass-
mann code in the language SageMath. The code has been implemented as a class
with a variety of useful methods. Worth mentioning is the method which generates
all minimum weight codewords of the dual code. Doing this in a constructive an
very eÔ¨Äective way. Allowing us to generate hundreds of thousands minimum weight
codeword in a matter of minutes. While implementing a data structure specially
suited to save space and do fast calculations.
Two main decoding algorithms was presented. One with a iterative local ap-
proach and the other with a global error detecting approach. Both of which have
been implemented along with some interesting variants.
The idea of the global
decoder emerge as a part of this study. We have successfully conducted decoding
simulations to measure the performance of the diÔ¨Äerent decoding algorithms. Many
interesting observations have been made. It was believed that the local decoding
algorithm would be able to guarantee correct decoding upto half the minimum dis-
tance. But we have managed to Ô¨Ānd counter examples on when the local decoding
algorithm fails to correct errors under half the minimum distance. The simulations
performed on the aÔ¨Éne Grassmann code over a Ô¨Āeld of size greater than 2 have
never been done before.
For the further work on aÔ¨Éne Grassmann codes it would be interesting to Ô¨Ānd
out what the maximal number of errors is, for which we are guaranteed a correct
decoding for each of the two main decoding algorithms.
Hopefully this study will contribute to the further research on aÔ¨Éne Grassmann
codes.
