Summary
Large-scale Vision-Language models have shown great capability on open-
vocabulary image classification. However, vision-language models fail when
applied directly in an open-vocabulary object detection setting. To overcome
the domain shift between vanilla classification and classification for object de-
tection, instead of the commonly used approach of knowledge distillation, I
propose Vision-Language Object Detection Adapter (VLODA). By freezing
the vision-language model and training components which adapt its predictions
to the object detection task we can fully take advantage of the generalization
capabilities of the open-vocabulary classifier. My model archives 20.6 segmen-
tation APr on the LVIS dataset, which is an improvement of 0.6 segmentation
APr compared to the previous state-of-the-art.

Chapter 1
Introduction
The more and higher quality data is available the better a deep learning model
will perform. Most use cases do not have much training data available and
data collection is expensive. Open-vocabulary models are trained in a way that
allows them to dynamically change the categories at inference time. This can be
achieved by training on image-text pairs scrapped from the internet, see Figure
2.5 for sample data. The image-text pairs are ideal to train classifiers, as the
text describes what is on the image, and the vast amount of data available on
the internet allows for a big vocabulary size. Object detection is the task of
both localizing and classifying objects in the image by predicting instances with
bounding box coordinates and class probabilities. The image-text pairs are not
suitable for training an open vocabulary object detector, it requires different
data, where objects with location are paired up with text description, which is
not as widely available. It was demonstrated by VILD [2] that localizers trained
on a set of base categories can generalize well to novel categories. Thus a strong
open-vocabulary object detector can be constructed by training the localizer on
base categories and adapting an open-vocabulary classifier to object detection.
The main line of research [2], [3], [4], [5], [6] focuses on distilling the CLIP [7]
open vocabulary classifier into an open vocabulary object detector. To be able
to take full advantage of the generalization capability of the open-vocabulary
classifier on novel categories, I decided to adapt the CLIP classifier by freezing
it and training additional components instead of using distillation.

the bounding box. In other words, if an object is small, and does not cover the
whole bounding box, then the object detector should predict a low probability
for the given class. An object detector only using the frozen CLIP classifier for
classification, and not adjusting the class probabilities with localization quality
achieves 4.8 novel and 2.7 overall segmentation average precision (AP) [8] on
the LVIS [9] dataset, compared to the 22.6 novel and 20.6 overall AP of my
best model. In open-vocabulary classification training data, the text description
usually describes a dominant feature in the image, which does not need to fill the
whole image tightly. Applying a vanilla classifier to object detection classification
leads to many false positives, as some large bounding box, with a small object in
it will be classified as that object with a high confidence. This can be overcome
by predicting how tightly the object fits in the image and if any other objects fill
the image more. A high classification score for an object should only be given if
the following three conditions are true: 1) the open-vocabulary classifier predicts
that category, 2) the object fits tightly into the image (counter-example in Figure
1.1 top), 3) there are no other objects which fill the image even more (counter-
example in Figure 1.1 bottom). I propose to ensemble the final classification
score from 1) the frozen CLIP classifier, 2) an intersection over union (IoU)
head to detect if there are no objects fitting tightly in the bounding box, 3) a
distillation head to detect of there are other objects filling the bounding box
even more.
Localization can be solved by training a class-agnostic localizer on the base classes.
Although the CLIP features contain the necessary information to localize objects,
they are not as suitable as features of a fine-tuned backbone. This can be
mitigated by using a more complex neck to transform the frozen features.
Main contributions of this work are:
• Proposing an ensemble classifier using localization quality scoring modules,
more specifically intersection over union (IoU) scoring and classification
distillation modules, with which the open-vocabulary classification scores
can be adapted to open-vocabulary object detection.
• Demonstrating that cascading features work better on frozen classification
pre-trained backbone features.
I achieve state-of-the-art performance by focusing on the shortcomings of the
open-vocabulary classifier in the object detection setting.

Before talking about my model, is it important to familiarize ourselves with
previous works. For an open-vocabulary object detector, we need to discuss object
detectors, open-vocabulary classifiers, and open-vocabulary object detectors.
My VLODA model builds on the CLIP [7], Sparse-RCNN [10], F-VLM [11],

2.1
Object detectors
Object detection is the task of localizing and classifying objects in an image. The
training dataset consists of images with a list of instance annotations for each
image. Each instance is annotated with a class label ci bounding box coordinates
bi ∈R1×4 and might be annotated with a segmentation mask si ∈Rw×h, where
w and h are the width and height of the bounding box in pixels. At inference time
the object detector needs to identify each object, then classify it by predicting
the class ci, and localize it by predicting the bounding box bi, or segmentation
mask si. Samples from two datasets, the COCO [14] and LVIS [9] dataset can
be seen in Figure 2.1. The LVIS dataset has 1203 categories with a long-tailed
distribution compared to the COCO dataset with only 80 categories. You can
see the category distribution in Figure 2.2. LVIS is a federated dataset, this
means the categories are not exclusive. Each image is annotated with some

Faster-RCNN: Faster RCNN [15] is a commonly used two-stage detector model.
The input image is first transformed by a feature extractor backbone. The two
stages are a region proposal network (RPN) and the detector heads. The region
proposal network predicts a class-agnostic objectness score for pre-defined anchor
boxes. Based on the objectness scores the bounding boxes are sorted, and
non-maximum-suppression (NMS) is applied to reduce the number of duplicate
predictions. The RPN selected proposal boxes, are fed to the second stage.
Each proposal box corresponds to a region of interest (RoI), by applying RoI
aligning on the backbone features (which can be thought of as cropping out the
bounding box in the feature space), we can get a proposal-specific feature matrix.
These RoI-aligned features are fed into the two detection heads: bounding box
regression and classification. The bounding box regressor predicts a correction
of the proposal bounding box coordinates. The classification head predicts the
category distribution for each instance. For the final prediction, the proposal

Cascade-RCNN: The second stage in the Faster RCNN corrects the proposals.
Cascade-RCNN [17] repeats the second stage n times and predicts a bounding
box correction and class distribution at each stage. This allows high-quality
predictions even if the proposals are of bad quality.
Sparse-RCNN: Sparse RCNN [10], see Figure 2.3 takes advantage of the
power of Cascade-RCNN, by instead of using an RPN network, it learns a fixed

propagated between the layers. The instance feature vectors and the RoI-aligned
backbone features are mixed by two dynamic convolutions at each stage. For
coordination between the proposals, a self-attention in the instance dimension is

DETR [18] Sparse-RCNN replaces the matching with the Hungarian assigner,
which computes the optimal bipartite matching between predictions and ground
truth objects. This is only possible as there is information flow between the
proposals, and the model can learn to only detect each object once. Using the

DiffusionDet: Riding on the popularity of diffusion models, DiffusionDet [12]
formulates the detection problem as a reverse diffusion process. From random
bounding boxes, it learns the reverse process, denoising them into bounding
boxes from the data distribution. Instead of conditioning the reverse diffusion
process on some text, like in text-to-image diffusion models, it conditions the
reverse process on the image features. DiffusionDet achieves this by replacing
the learned proposals from Sparse-RCNN [10] with random proposals.
Mask Scoring RCNN: Object detector classifiers do not take into account
the quality of the localization. By reducing confidence for object predictions
with poor localization quality, the high-confidence instance predictions will have
better localization. The Mask Soring RCNN [13] predicts the intersection over
union (IoU) between the predicted and ground truth instance mask, from the
RoI pooled features and the predicted segmentation mask. The IoU is defined
as the area of overlap over the area of union, in other words, how much of the
prediction overlaps with the ground truth relative to the overall prediction and
ground truth size. The Mask Scoring RCNN optimizes the squared L2 distance
between the predicted score and the segmentation IoU:

VILD: Vision and Language Knowledge Distillation (VILD) [2] was one of the
first papers building on the CLIP open-vocabulary classifier [7] to create an
open-vocabulary object detector. The VILD model has three different versions:
VILD-text, VILD-image, and VILD-ens. VILD-text distills the text encoder and
VILD-image distills the image encoder into the detector. VILD-text encodes
the CB base categories in the training dataset and replaces the last layer of the
classification head in a Mask-RCNN [16] detector with these embeddings. By
normalizing the features before the last layer, and replacing the class weights
with class text embeddings, the last layer becomes a cosine similarity between

CLIP text encoder vocabulary. VILD-image is similar to the VILD-text detector,

with the student features on the RoI-aligned regions, the CLIP text encoder
can be used at test time for classification. The VILD-ens model trains both
VILD-text and VILD-image heads and ensembles them by geometric mean at
test time.
RegionCLIP: It was demonstrated by [2], that object detectors trained on
base categories CB have similar localization performance as detectors trained
on both base CB and novel CN categories. As the localization works well for
open-vocabulary object detection, RegionCLIP [4] disentangles the localization
from classification by using a separate backbone and detector for the localizer
and only training a network for classification. RegionCLIP inserts an extra
step between the contrastive image-text pretraining of CLIP [7] and the object
detection knowledge distillation, by introducing a region-text pretraining step on
a region-text dataset, where frozen text features are distilled into the RoI-aligned
visual features by a contrastive classification and an L1 loss. The region-text
pretrained model can already be used for zero-shot object detection, and by
fine-tuning it on the training set, it outperforms VILD on both LVIS [9] and
COCO [14].
OV DETR: Detection Transformer (DETR) [18] is an end-to-end trained one-
stage object detector with comparable performance to two-stage object detectors.
It uses a transformer encoder-decoder architecture, where the encoder is used
to extract features from the image, and the decoder predicts the bounding box
coordinates and the class probabilities. The decoder queries are learned and
each token corresponds to one predicted instance. Open-vocabulary Detection
Transformer (OV DETR) [6] adapts the DETR architecture by summing the
learned queries with CLIP text embeddings, thus making each query class
conditioned. Similarly to other open-vocabulary object detectors, it is trained
on the base classes CB, but by replacing the category embeddings to the novel
classes CN it can generalize to the novel objects at test time.

Chapter 3
Methodology
In an open-vocabulary setting, the training data consist of annotations from CB
base classes, but during inference, the model needs to be able to classify CN
novel classes. Novel categories CN can contain categories that are not in CB,
thus not seen at training time.
My method, the Vision-Language Open-Vocabulary Adapter (VLODA) adapts
the CLIP classifier to open-vocabulary object detection. To be able to capitalize
on the generalization capabilities of the CLIP model, I freeze the image encoder
and adapt it for object detection, by adding an RoI-align to the classifier and
training additional localization quality measuring components on top of the
frozen features. The model architecture can be seen in Figure 3.1.
First, in Section 3.1 I am describing how the frozen CLIP classifier can be
adapted to predict the category probabilities for several bounding boxes without
inferencing the backbone multiple times. Then in Section 3.2 I continue by
describing the trained components of the detector and finally, in Section 3.3 I
write about how the trained components can be mixed with the frozen classifier
to get the final prediction.

3.1
Frozen CLIP classifier on bounding boxes
Before it can be discussed how the classification probabilities can be corrected
by taking into consideration the localization quality, we need to discuss how the

The ResNet50 CLIP [7] image encoder was pre-trained on 224X224 images and it

3.2
Trained modules
I adapt the CLIP classifier to object detection by introducing localization and
localization quality modules. Localization is done by class-agnostic bounding
box, and class-agnostic segmentation mask heads. The localization quality is
measured by a bounding box IoU score and a CLIP distilled open-vocabulary
classifier. The VLODA head can be seen in Figure 3.2. To be able to use the
frozen classifier during test time, the ResNet backbone is frozen. Using a frozen
backbone reduces GPU memory requirements and decrases training time.
Proposals: The inputs of the detector head are the frozen backbone features
F(I) and bounding box proposals bp. I test three different proposal types: RPN
[15], learned proposals [10], and random proposals [12]. The baseline Mask-
RCNN [16] model uses its default RPN for proposals. For the cascading stages, I
experiment with two additional settings, learned and random which were inspired
by Sparse-RCNN [10] and DiffusionDet [12] respectively.

The random boxes are parameterized by relative center x, center y, width, and

deviation of 0.25, bp ∼N(0.5, 0.25). The initial proposal features are also not
learned, but they are initialized by the RoI pooled FPN features. An advantage
of using random proposals instead of learned ones is that at test time the number
of proposals can be adjusted.
Assigner: Predictions need to be matched up with ground truth instances in
order to be able to compute the loss. I experiment with two matching strategies,
1 to 1 matching with a Hungarian assigner [18] and a 1 to K matching using the
simOTA assigned [12]. Both assigners compute the matching cost, which is the
loss between the predictions and ground truth labels. The Hungarian assigner
solves an optimal transport assignment problem between the predictions and the
ground truth boxes, using the above-mentioned cost matrix and the Hungarian
algorithm. The 1 to K simOTA assigner uses an optimal transport assignment
algorithm and selects the up to K predictions dynamically for each ground truth.

classification and classification for object detection. VLODA outperforms other
models by not distilling the backbone, but capitalizing on the frozen classifiers’
generalization capability and correcting its prediction for object detection.

Chapter 5
Discussion
Deep learning model performance depends greatly on available computing power,
data quality, and data quantity. It is cheaper, and thus more practical to fine-tune
pre-trained models. I believe that open-source multimodal deep representation
learning will play a major part in the future of deep learning. These models
can be fine-tuned into task-specific models. These pre-trained models need to
have a dense representation space that captures the semantic information in
the input data. Using self-supervised or weakly supervised learning, models are
not limited by the available data. The explosion in natural language processing
would not have been possible without the use of unlabeled datasets, and pre-
training. Similar techniques can improve model performance on other modalities
as well. Models using a pre-trained backbone with a good embedding space can
achieve high performance with low training cost and limited data, as they do
not need to learn the feature transformation. Language was created for efficient
communication between people, by using models which can understand human
language, allows us a more direct interaction between the user and the software.
As there is limited data for training open-vocabulary object detectors, it is a
perfect example of pre-trained open-vocabulary classifier adaptation. The CLIP
open-vocabulary classifier can be trained on the vast amount of image-text pairs
on the internet, and its generalization capability can be exploited by adapting it
for object detection. Open-vocabulary object detectors can be easily integrated
into proof of concept products, and if higher performance is needed, further

the user has a more direct interaction with the model, as it can understand
human language and dynamically adapt to the needs of the user.
In this work, I have demonstrated how a multimodal representation learning
model, the CLIP open-vocabulary classifier can be fine-tuned and applied to
another task, object detection with minimal data and compute resources. Vision-
language models are strong open-vocabulary classifiers, but they do not work
out of the box in the classification for object detection setting. The vanilla
classification does not need to take into account the localization quality, and
it can give high confidence to any category which is partially present in the
picture. By freezing the CLIP classifier and training localization quality models,
the frozen classification score can be adjusted. Freezing the backbone has the
added benefits of reduced GPU memory usage and faster training time. The
frozen CLIP features need to be transformed to perform localization, as the
CLIP model was pretrained on the classification task. I have demonstrated
that state-of-the-art performance can be achieved by correcting the classification
scores with the localization quality and using a stronger localizer.
Future work should focus on both better pre-training, and how the pre-trained
models can be adapted for the given task. As proven by the performance of
the RegionCLIP [4] models, the CLIP embedding space has limited localization
information. By using a more appropriate representation learning task, the
pre-trained backbones would be easier to apply to the downstream tasks. It is
impossible to take into consideration all use cases when selecting the pretext
tasks, thus use cases should focus on the shortcomings of the pre-trained model
for the given task and correct for it. The pre-trained encoder should have better
representation for localization, and the CLIP-based open-vocabulary object
detectors should focus on the task-specific shortcomings CLIP model, namely
how to adapt the vanilla open-vocabulary classifier to classification for object
detection.
