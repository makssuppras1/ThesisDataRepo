Abstract
The navigation of the mobile robot has been a famous topic for decades. It is an The
navigation of the mobile robot has been a famous topic for decades. It is an engineering
that combines artificial intelligence, control theory, and mechatronics. The task is usually
divided into perception, path planning, and control. These traditional methods require
deep understanding and solid skills to implement as each module is essential to the
success of navigation.
Reinforcement learning (RL), initially inspired by dynamic programming, is designed for
optimizing decision-making problems. With the popularity of deep learning (DL) using
neural networks (NN) methods, reinforcement learning evolved into deep reinforcement
learning (DRL). DRL can be applied to continuous control tasks, which is a perfect fit
for the navigation problem.
In this thesis, a complete pipeline from building custom environments to training and
testing the DRL agent is presented. The mobile robot learns to follow a predefined
path while avoiding obstacles. The result also shows some robustness to the changes
happening in the environment such as a moving obstacle or slippery floor. If provided
with a well-designed environment, DRL shows great potential in solving the navigation
problem without the need for internal dynamics of the mobile robot and the complex
path planning algorithm. The end-to-end way of learning and action is the beauty of
DRL.

CHAPTER 1
Introduction
This project implemented the training of a neural network using Deep Reinforcement
Learning (DRL) in a simulation environment that can control a differential drive mobile
robots able to follow a reference path while avoiding obstacles.
1.1
Background
Autonomous Transport Robot (ATR) is designed to transport goods in a factory with-
out requiring human operation. It operates by following different predefined paths to
complete different tasks.
For instance, leave the charging station and then move to
the storage house. In most of the cases, not all the areas in the factory are accessible.
There are boundaries, for example, walls or fences, to prevent the robot from entering
restricted areas such as manufacturing zones. Furthermore, an ATR is required to com-
plete deliveries within strict deadlines. Failure or delay in delivery will heavily affect
subsequent processes in the production line. Thus, the ATR must be capable of handling
different situations, including unexpected obstacles or potential slips on the wheels, to
ensure uninterrupted operation.
To accomplish these tasks, ATRs are often equipped with rangefinder sensors to detect
obstacles. A commonly used strategy is to simply stop moving to avoid collision. How-
ever, this strategy is not always suﬀicient. When carrying a heavy load, a hard break
will cause the load to fall off. And if the obstacle keeps blocking the path, the robot
will be stuck in the deadend. Thus, intelligent navigation and obstacle avoidance are
developed to address these issues.
Traditional navigation stack consists of localization, path planning, and control. For
indoor navigation, localization can be done by motion capture systems such as the Opti-
TRACK system [4] or SLAM(Simultaneous Localization And Mapping). Path planners
such as A*, Hybrid A* [3], D*, and RRT* are often used to find an optimal path that
can reach the goal without hitting obstacles. If pursuing more accurate planning, a tra-
jectory planner can be used to generate a feasible trajectory that ensures the feasibility
of the path. Lattice planner [27] which includes the kinematics constraints of the robot
is a good combination of path planner and trajectory planner. Once a path or trajectory
is obtained, a controller such as a PID controller, LQR controller, or MPC controller
can be used to control the robot to follow the path.
The traditional navigation stack is widely used in the industry and has been proven to

be effective in simple scenarios. However, the navigation pipeline relies on the accurate
result from each module, which is hard to achieve in all cases. A path planner like A* has
drawbacks of high time complexity and poor adaptability to dynamic and unknown envi-
ronments [10]. The controller needs a precise model of the robot otherwise performance
will be degraded. Moreover, optimal controllers such as MPC draw great computational
power to perform online optimization. Thus, it’s challenging for traditional methods
to tackle complicated scenarios, like the environment with many obstacles or unknown
cases, like slippery tires.
The control problem from another perspective is a decision-making problem - the con-
troller decides what control input should be commanded to the plant based on the cur-
rent state. LQR controller uses the dynamic programming idea as it is used in solving
MDP (Markov Decision Process) problems. With the development of Artificial Neural
Networks (ANNs) and great improvement in computing power, Deep Learning (DL) has
become a powerful tool for tasks such as image classification [16], speech recognition [11].
Thereafter, when AlphaGo developed by Google DeepMind which is trained by Deep Re-
inforcement Learning(DRL) [34], defeated the human champion in Go, the capability of
DRL in decision-making problems started to be popular.
When Atlas Robot developed by Boston Dynamics which uses MPC to do full body con-
trol [6] shocks humans about its capability of doing fancy movements such as backflips,
other amazing groups are using RL to make Bipedal Robot [18] and Mini Cheetah [20]
learn to do locomotion control in a simulation environment. Even greater, a team at
the University of California, Berkeley uses RL to train the robot to walk from scratch,
in the real world, in just 60 minutes [40].
These excellent works inspired the project to use DRL to make the robot navigate making
intelligent control decisions.
1.2
Objective and methods
The purpose of this project is to train a neural network with model-free Reinforcement
Learning techniques to make a differential drive robot navigate in a corridor with ran-
dom obstacles. The obstacles could be static or dynamic. The robot should reach the
destination before the time limitation. Worst cases such as a slippery tire can also be
considered.
The map for the training environment will be designed to have one curved path, multiple
obstacles, and the walls of the corridor.
The differential drive mobile robot equipped with a rangefinder sensor will be imple-
mented for simulating the robot locomotion and sensor data.
Proximal Policy Optimization (PPO) [31] is selected as the DRL algorithm for training
the neural network.

CHAPTER 2
Background
In this chapter, the theoretical details of mobile robot navigation, control, Reinforcement
Learning algorithms, and neural networks will be introduced.
2.1
Mobile Robot Navigation
The navigation of autonomous mobile robots has been a hot topic for decades since it is
widely used in fields such as military, industry, and agriculture. The navigation problem
is generally divided into three parts, localization, path-planning, and path-following.
The aim is that by giving a start and end point on a map, the robot can find its way to
the destination without collisions.
From the perspective of the robot, it needs to know where it is and where it should
go.
Wheel odometry, SLAM, or motion capture system provides robot localization
information, which solves the where-it-is problem. Path planning then sketches up a
feasible route that can avoid obstacles. The controller then computes control inputs
based on the deviation from this reference path. Ideally, the result control signal will
drive the robot to follow the path and reach its destination.
2.1.1
Path Planning
In practice, path planning task is divided into global path-planning and local path-
planning.
Global path planning is to find a path from the start point to the destination, taking
the static or dynamic obstacles, and walls into consideration. Two types of algorithms
are developed for accomplishing such tasks, search-based and sample-based.
Search-based algorithms such as Best-First, Dijkstra, A*, and their variants are robust
and widely applicable to path-planning problems in 2D or 3D. The advantages of these
methods are that they are robust, general, and easy to implement. More importantly,
they are predictable, which means it’s easy to verify whether the algorithm works prop-
erly. However, BFS and Dijkstra don’t guarantee the optimal result. The computational
complexity of Dijkstra and A* could reach up to O(|V |2). This complexity requires ei-
ther reducing the resolution of the map or sacrificing the precision and quality of the
==PAGE:18==

path. Moreover, these algorithms are designed mainly for static environments and they
struggle with high-dimensional spaces. These approaches are shown in Figure 2.1(1).1
Sample-based algorithms are mainly RRT (Rapid Random Tree) and its variants. RRT
can handle high-dimensional spaces and non-static environments. The variant, RRT*
improves the path quality and thus provides an optimal solution. However, RRT* per-
forms worse in narrow passages since there might not be enough points generated in such
areas. It also requires significant computational resources in complex tasks. The final
product of RRT or its variants is often a tree-liked path rather than a smoothed curve
which is what the robot controller requires. Thus post-processing such as smoothing or
interpolation is needed in practice. These approaches are shown in Figure 2.1(2).
However, the path generated by both types of algorithms is not strictly feasible in
reality, since they do not consider the kinematics and dynamics of the robot. In cases
where the kinematics DOF (Degrees of Freedom) of the robot is constrained, such as
an Ackermann steering car, where the vehicle cannot move sideways, the path needs
further process. Since the global planner might only run once, unexpected changes in
the environment like the sudden appearance of a pedestrian will lead to collision. Thus,
local path planning is needed for real-time decision-making.
Local path planning often uses sensor data to perceive the surroundings so that it can
react to moving obstacles or new obstacles.
Normally there is a limited horizon to
reduce the computations. Common local path planning algorithms include Vector Field
Histogram (VFH), Dynamic Window Approach (DWA), Artificial Potential Fields, and
variants of the RRT and PRM for dynamic environments. Take DWA as an example,
it has strengths in adapting to the changes in the environment in real time and can
directly compute possible linear and angular velocities. However, DWA can suffer from
poor stability and high-energy consumption due to ineﬀicient path planning [17] and it
has diﬀiculties dealing with fast-moving obstacles [28].
A special type of planner called Lattice Planner [26] is designed for global path plan-
ning. The kinematics constraints are encoded when calculating feasible local paths. The
planner dynamically generates optimal routes from the state grid based on the current
situation. It adapts to the dynamic environment in real time. The lattice planner can
be seen as a combination of path planning and trajectory generation. Due to the high
dimensionality of the discretized state space, the lattice planner is computationally ex-
pensive. Although it can handle dynamic environments, it can struggle with moving
obstacles with non-constant velocity since the responsive behavior of the control ob-
ject is not modeled [35]. A simplified lattice planner is implemented in this project for
performance comparison, see Section 3.3.2.
In a typical robotic system, both global and local path planning work together. The
global planner provides a general route for the robot to follow, and the local planner
adjusts this route in real time to avoid unforeseen obstacles and local changes.

2.1.2
Trajectory Generation
Path planning is the process of finding the shortest or safest path from a starting point
to a destination in the environment. The path therefore mainly consists of a series of
waypoints in space, without considering the time or the dynamics of the robot. Once
the path is determined, a trajectory generation method could be applied to give better
guidance on how the robot should move toward the goal. Polynomial interpolation and
optimization techniques are often used to generate those trajectories [38].
2.1.3
Modelling
With the reference path given by the path planner, the mobile robot needs a controller
that can drive it to follow the path. The most common controllers are PID, LQR, MPC,
and their variants.
Modelling
The design and implementation of these controllers need a precise model of the ATR
which is a differential drive two-wheel mobile robot. The ATR is shown in Figure 2.2.
Only the kinematics of the robot are considered in this project. The dynamics in the
motor and wheels are neglected, for example, gear transmission, and ground friction.

2.1.5
Lattice Planner
As introduced in Section 2.1.1, a lattice planner is a special type of planner that com-
bines path planning and trajectory generation. It is designed for non-holonomic robots,
which is compatible with the differential robot used in this project, the robot cannot
move sideways. Lattice planner reduces the search space to a uniform discretization of
vertices corresponding to position and direction. Each vertex in the discretization space
is connected to other points via kinematically feasible motion primitives. These feasible
motion primitives are also known as control actions.[1].
The feasible trajectories can be graded by a cost function. For instance, remove the
trajectories that collide with obstacles. The trajectory with the lowest cost is selected as
the optimal trajectory. The robot then follows this trajectory until the next time step.
In a simpler case, the control action that is used to generate this trajectory is therefore
the control input for the robot.
The visualization of trajectories is shown in Figure 2.4. Each curve is a sequence of
edges in the state lattice, representing a motion plan. By defining different control sets,
different shapes of the curves are generated.

Short Summary
The navigation, path following and collision avoidance problems can be solved via path
planners and a controller based on the system model.
Since the controller is designed by mathematical models, for instance, solving an LQ
problem, an accurate system model is essential. However, in practice, the model can be
heavily inaccurate or even unknown, which makes it impossible to design the controller.
Even with everything provided, these methods are usually computationally expensive
because the path planning needs to compute the entire environment and the controller
needs to solve the LQ problem in real-time.
The highly dynamic environment with randomly moving obstacles can make it even
harder to handle all the possible situations since the global and local path planning re-
sults will possibly give unfeasible paths, and the reference state is beyond the controller’s
capability.
Another issue is, due the the feedback architecture, and the path-finding strategy, the
planner and controller are more likely to fall into local minima which leads to sub-optimal
results.
Even though this topic has been studied for decades, it appears that when applied to a
nonholonomic and real-time constrained system, all the existing methods do not provide
a complete solution. [41].
This thesis tries to use the reinforcement learning method to provide an end-to-end
solution for robot navigation tasks.
2.2
Reinforcement Learning
The control problem can be considered as a decision-making process. The controller, in
control theory, or the agent, in reinforcement learning, gets the state feedback (obser-
vations) and then computes control input (makes decisions), to bring about a desired
outcome. If RL can solve Go or video games, can it be applied to the guidance and
control of a mobile robot?

2.2.1
Markov Decision Process
The ‘learning‘ term in reinforcement learning is learning what to do given a specific
situation (State)- how to map situations to actions - to maximize a numerical reward
signal [36]. That is, to make a series of good decisions to get a good final result. This
process of making sequential decisions is called a Markov Decision Process (MDP).
Figure 2.6: Reinforcement learning diagram
The agent is facing a stochastic or deterministic environment in which the dynamics are
known or not and is using a policy to choose actions at given times/states. The reward
is given immediately after the action is taken. The goal of the agent is to maximize the
cumulative reward. Thus, several concepts are introduced to describe the MDP.
• Agent: The object that interacts with the environment. Also called controller, in
control theory.
• Environment: Everything outside the agent. It responds to the agent’s actions
and then transfers to the next situation (state). The environment also gives rise to
==PAGE:27==
2.2 Reinforcement Learning
15
rewards that the agent tries to maximize. The term environment here represents
the plant or the controlled object in control theory.
• State S:
The observations of the situation of the environment. It can be one
frame of image or elements that can describe the situation of the robot and its
environment.
• Action A: In control theory it is called ‘input‘ or ‘command‘. It is the decision
that the agent has made to go to the next step.
• Reward R: After each interaction, i.e., after the action is taken in the environ-
ment, a reward will be given to compensate or penalize the agent based on the
new state it is in. The accumulated reward is what the agent needs to maximize.
In such an iteration, the agent first observes the state St ∈S, then takes action At ∈A
based on the state St, then receives a reward Rt+1 and changes to the new states St+1
at the next time step.

2.2.3
Temporal Difference Methods
To solve a DP problem without a model, a method called Monte Carlo (MC) is intro-
duced. Sampling the whole state space in each episode then get a return for each state.
Averaging the return for each state, this is in general the expected return that represents
the value function for one state. A useful incremental method is introduced in the simple
every-visit MC method. Then, the update rule is defined as:
V (St) ←V (St) + α [Gt −V (St)]

where α is the learning rate, usually a constant between 0 and 1.
However, the MC methods need to wait until the termination of an episode and, there-
after, get the return Gt to update the value functions. This property makes it impossible
to be applied to continuous tasks.
The core idea of temporal difference (TD) is to use bootstrapping to estimate the return.
From Equation 2.5, the return can be decomposed as the sum of the immediate reward
and the discounted return of the next state. The return of the next state can also be
represented by the estimate value function. Thus, the update rule of TD(0) or one-step
TD methods is:
V (St) ←V (St) + α [Rt+1 + γV (St+1) −V (St)]
(2.8)
Need to know that, the MC discounted return Gt is an unbiased estimate of vπ(St) with
some variance. The TD target Rt+1 + γV (St+1) is a biased estimate of vπ(St) with low
variance. However, the bias is usually not a problem, and the TD methods are often
more eﬀicient than MC methods.
The second part of Equation 2.8, δt = Rt+1 + γV (St+1) −V (St), is called TD error or
TD residual. This residual or error describes the difference between the estimated value
of St and the better estimate Rt+1 + γV (St+1) [36]. This error changes over time, thus
the method is called temporal difference.
The above update rule can be seen as a prediction method for the control problem.
The goal of the agent is to maximize the return, thus the action-value function is more
important. Applying TD methods to state-action value functions leads to the Q-learning

2.2.9
Exploration and exploitation
In reinforcement learning, exploration and exploitation are two important concepts. The
agent needs to explore so that it can perhaps take better action in the future. It also
needs to exploit the learned policy to obtain rewards and then make evaluations. A
common practice is using ϵ−greedy or ϵ−soft to allow non-optimal action to be taken by
a possibility. When training with PPO, the exploration and exploitation are controlled
by the standard deviation of the normal distribution and the entropy term embedded in
the object function Equation 2.21.
The entropy term S[πθ](st) in the context of information theory and probability is a mea-
sure of the unpredictability and randomness of a system. Using the normal distribution
as the policy, the corresponding entropy is given as:

the normal distribution. The larger the standard deviation, the larger the entropy. The
object function wants to maximize the entropy by gradient ascent. The derivative of the
entropy with respect to the standard deviation is given as:

This represents that when the standard deviation is small, a small change in σ will lead
to a larger change in S. As σ becomes larger, the change in S becomes less sensitive
to the change in σ. Thus, adding this entropy term encourages the agent to maintain
a higher level of randomness to explore more. In practice, adding entropy term often
leads to robust learning, as the agent is less likely to get stuck in a local optimum. This
benefits a lot in path following and obstacle avoidance tasks because sometimes the
feasible path is not often nearby.
In conclusion, maximizing the object function will encourage exploration by pursuing
a higher entropy that brings a higher standard deviation in action distribution, and
exploitation by maximizing the advantage estimate for the current policy.

2.3.2
Transfer Learning
Transfer learning is a method that allows the network to reuse the knowledge from the
previous task to speed up the learning process of the new task. The weights from the
training in the first time are kept and used as a starting point in the second training [19].
To apply transfer learning in this project, the robot can first learn to follow the path
without any obstacles. Then the knowledge is transferred to the second training where
the robot learns to avoid obstacles while following the path. The two tasks are related
==PAGE:44==
32
2 Background
but not the same. This helps reduce the time of random exploration action needed for
reinforcement learning because the robot doesn’t need to learn to follow the path again.
One special case in transfer learning is domain adaptation. It is a method that allows the
network to learn from the source domain and apply the knowledge to the target domain
[5]. Domain represents the data distribution of the environment. In this project, the
training environment uses a method to emulate the data generated from a laser scanner.
The data from this simple sensor model is the source domain. The data from a sensor
in the real world is the target domain. The model trained by the emulated sensor can
be used with the real sensor. Furthermore, the model can be trained further with the
real sensor data to improve the performance.
2.3.3
Incremental Learning
Similar to transfer learning, incremental learning allows the network to learn new knowl-
edge based on the old knowledge as well. However, it focuses on gradually increasing
the complexity of the tasks while keeping the learned knowledge.
A common practice is to start learning with a simple task and then migrate the model
from the simple task to more complex tasks by transfer learning. Thereafter, increase
the complexity of the tasks gradually to adapt the model to the complex environment
[12]. Due to the similarity of two tasks, and the similarity of avoiding one or more
dynamic obstacles, this method can be used in this project. The network can learn
a model to follow the path first and learn to avoid one obstacle, then more obstacles
once the previous learning is stable. Compared with the learning method where the two
tasks are learned at the same time, the incremental learning method allows the agent to
develop a solid foundation before tackling more diﬀicult challenges.
The challenge for incremental learning is the catastrophic forgetting problem. The net-
work tends to forget the knowledge learned from the previous task when learning the
new task. However, in path following and obstacle avoidance tasks, the following com-
plex tasks still contain the simple tasks. Then, this disadvantage is not a problem in
this project.
The experiments using transfer learning and incremental learning will be shown in Chap-
ter 4.

no matter what action it has taken or will take.
After this step of interaction, the

The fundamental concept of creating a random path involves initially defining several
waypoints and subsequently interpolating them to form a smooth path.
The 2D coordinate of the first waypoint can be easily created by taking the cosine and
sine of a random angle θstart, multiplying with the desired length Lp and dividing by 2 to
make it at a reasonable position in the map. An average segment length is calculated by
dividing the desired total length Lp by the number of desired waypoints -1. It represents
how long the next waypoint should be away from the previous one. Thus the total length
of the path created later will be close to the desired length.
To prevent the path from crossing itself, the direction of the new waypoint is generated
randomly until it is in the forward direction of the previous one. It is done by calculating
the dot product of the direction vectors.

Path interpolation
The procedure is shown in Algorithm 2. The source code can be found in the function
‘generate_path’ in ‘env_path.py’
Initially, PCHIP interpolator is instantiated with a normalized parametric domain rang-
ing form [0, 1]. It is a common practice for generality, numerical stability, and precision.
The result is a mapping function pp(ω). It can map a normalized parameter t in the
range [0, 1] to the waypoints’ coordinates. Subsequently, the parametric domain is evenly
divided by 1000 points. This step is for discretely differentiating pp(ω) to compute the
total length of it. By calculating the derivative of the path at these points, and their Eu-
clidean norms, the small arc length for each segment is obtained. The cumulative sum of
these lengths yields a representation of the total arc length of the whole path. By giving
an interval d which represents the desired distance between two path points, the number
of points needed for representing the reference path can be calculated. The domain is
evenly separated by this number, and the accumulated arclength at each needed point
is therefore computed. Using this set of arclength again with pp(ω), the coordinate of
each point in the real-world reference path can be finally obtained. The last step is to
turn these 2D points into a KDTree for a fast nearest neighbor search when querying
the closest point on the path to the robot.
The yaw angle of each point on the reference path should be calculated as well. This
will be used for calculating the tracking error when formalizing the reward function.

So far, we have obtained a reference path and the obstacles around it. The next step is
to create a boundary around the path to represent the corridor. The idea is to shift the
reference path to the orthogonal direction of each segment. This can prevent the corridor
from crossing the path. However, due to the curvature of the path, the shifted walls will
sometimes look twisted and not parallel to each other. Thus Cubic Spline Interpolation
is used to smooth the walls and resample the points by the desired interval distance.
Since the first waypoint is considered the starting position of the ATR, the walls need to
be extended with an offset to create space before and after the path. Finally, two other
walls are created to close the corridor. They are lines interpolated between the terminal
points of each wall.
The last step is to convert these four walls into KDTree, Sharpely Polygon, and Linear-
Ring objects. The KDTree will be used to find the closest points from the robot. The
Polygon helps with quickly checking if a point is inside the corridor. The LinearRing
helps with calculating the intersection point between the walls and the laser sensor. The
details will be explained in Section 3.1.4.

Dynamic obstacles
Another interesting feature is to make obstacles move. To keep the obstacles on the

random vector from a normal distribution where the mean value is a positive number
representing the moving speed. A random obstacle selector is designed to make some
obstacles move in a positive direction but some move backwards. This in general will
constrain the obstacles around the path while allowing some changes. The new state of
the obstacles will be updated in each step of the environment.

3.2.2
Reward function design
The goal of an RL agent is to maximize the cumulative reward. Thus, the reward design
is essential for the training process. One straightforward strategy is to give a reward at
the end of each episode. The agent receives a reward if the policy drives the robot to
the goal successfully, or a penalty if the robot collides. However, an extremely sparse
reward function, such as this one, is not suitable for navigation tasks. With the initial
random walk behavior, it’s nearly impossible for the agent to learn anything. Thus,
a continuous reward function that can reflect precisely the situation of the robot is
needed. Considering the dual task of this project, which is to follow a reference path
while avoiding obstacles and walls, the reward function should be designed to reflect
both the path following and obstacle avoidance performance. The two functions are
named rpf and roa respectively. The reward function outlined in this section is adapted
from the approach detailed in this paper [22] but with key modifications. Specifically,
a quantization technique has been applied to make the function sparse, diverging from
the original method proposed in the paper. Some trick rewards are added for different
requirements in this project.
A trade-off parameter λ is suggested to give weights to the two objectives, rpf and roa.
The final reward function should be a combination of the two performance measurements:
r = λrpf + (1 −λ)roa
(3.8)

3.2.3
PPO implementation
The implementation of the Proximal Policy Optimization (PPO) algorithm in this
project draws significant inspiration from the explanations and detailed insights pro-
vided in the blog by Huang (2022), referenced as [13].
The training process with PPO algorithm can be summarized as follows:
Data Collection Loop: The agent runs a specific number of steps to collect data.
It stores the observations, actions, rewards, and other information in a buffer. If the
episode is terminated or truncated, the termination term needs to be noted in the buffer
as well since only the non-terminated step will be used for
Calculating Advantages and Returns: The algorithm computes advantages using
GAE Section 2.2.6
Data Preparation:
Observations, actions, advantages, returns, and other relevant
data are reshaped or flattened to prepare for batch processing.
Policy and Value Function Updates: The actor and critic are updated over several
epochs. In each epoch, the data are shuffled and split into batches. For each mini-batch,
the algorithm computes new log probabilities, entropy, and value estimates by actor and
critic network. This procedure is so-called replay. The ratio of the new and old policy
is computed and clipped. Then the loss function Equation 2.21 is computed and then
backpropagated to update the network parameters.
Performance matrices: The code tracks the value loss, policy loss, entropy, approx-
imate KL divergence, clip frequency, variance, and other performance matrices. These
matrices are used to monitor the training process and to make sure the algorithm is
working as expected.

4.3
Problems and discussion
There are several problems found in the implementation of the environment and the
training process.
4.3.1
Environment implementation
Although a custom environment provides great flexibility in adapting to different tasks,
accuracy and eﬀiciency are not suﬀicient. The simulated rangefinder behaves poorly
when the robot is close to multiple obstacles. The emulated rangefinder is too slow to
be used in the training. A third-party library could be used to improve the performance.
The noise is an essential part of the real world, but it is missing in the implemented
environment. This creates a gap between the training and the real world and limits the
usage of the learned model.
4.3.2
Reward function
The reward function for obstacle avoidance is not good enough. When the robot points
directly to the obstacle, it gets the highest penalty correctly. However, when the robot
starts turning left or right, it does receive a smaller penalty, but this change is not
smooth. The penalty will increase again when the robot is turning at a certain angle,
then decrease if the robot keeps turning. This leads to the reward function not being
monotonic, which will confuse the agent and lead to a bad performance.
Due to the speed penalty, the robot is required to move as fast as possible even when
it is close to the obstacle. This leads to the behavior of collision in most of the cases.
However, if this penalty is removed, the robot will move very slowly or even stay still as
the reward in this case is maximum.
==PAGE:105==
4.3 Problems and discussion
93
The design of the reward function in this model is quite complex and problematic. It
incorporates numerous terms, making it not only challenging to fine-tune but also lacking
in intuitiveness. While employing quantized rewards can mitigate the issue of the robot
being overly responsive to minor actions, finding the right balance in resolution is critical.
An improperly calibrated resolution can adversely affect performance. There’s also the
risk of the agent getting trapped in local minima, particularly in situations where all
available actions yield identical rewards, leading to a lack of encourage for the agent to
explore potentially better strategies.
4.3.3
Exploration
One phenomenon observed during the testing is that the robot always starts turning to
avoid the obstacle until it is very close to the obstacle. This results in either the robot
sticking besides the obstacle or crashing into the obstacle. From Section 2.2.9, we know
that the entropy term is used to encourage the exploration. However, even if setting a
very small coeﬀicient for entropy, it will lead to uncontrolled behavior of the agent, while
the entropy does keep increasing during the training. One possible solution is to use the
same method as the learning rate, using annealing to decrease the entropy coeﬀicient as
the training goes on. It will encourage the exploration at the beginning and then focus
on the exploitation at later stages.
4.3.4
Training
Due to the time limit, only one algorithm (PPO) was implemented and used for training
in this project. Although the clipping method in PPO helps stabilize the training, the
policy and value estimate still oscillate considerably. Usually, it can be improved by
tuning the hyperparameters such as the learning rate, the number of epochs, and the
number of mini-batches.
The network design could be improved as well. One possible method is using recurrent
neural networks and Long Short-Term Memory (LSTM). RNNs and LSTMs are designed
to handle not just the latest observation, but the recent history of observations [2]. This
matches the demand in the navigation task that the robot should make more aggressive
actions when it’s getting close to the obstacles. Using LSTM neural network for path
planning has been studied [24], which can be generalized to control problems.
The maximum time steps in one training was 2 million. Sometimes it shows great success
in the training, that the logs show the robot reaches the goal every time at the end of
the training. However, this model doesn’t perform well in the testing. Normally the
reason is that the model is overfitting the training data. However, both the environment
for training and testing are generated in the same way, thus if it overfits in the training,
it should still perform well in the testing. The reason for this behavior requires further
analysis.
==PAGE:106==
94
4 Experimental Setup, Results, and Discussion
Another unknown issue is that the training script will stop running randomly. It doesn’t
throw any error messages, but the training process just stops.
This happens in the
sampling stage, where one batch of data is generated from the interaction between the
actor network and the environment. We need to further analyze this problem to find a
reason.
4.3.5
Generality, incremental learning, and transfer
learning
The model trained in the training environment doesn’t perform well in the simulation
environment. One of the reasons could be that the simulation map is more narrow than
the training map. If the robot walks on the reference path in the training map, normally
it won’t detect any walls. However, in the simulation map, the robot will detect the
walls all the time, which might lead to the confusion of the agent.
Both incremental learning and transfer learning have been tried in this project. As the
practice is simply loading the model from previous training, there are not many parame-
ters to tune. When transferring from a path-following task to an obstacle-avoidance task,
the method doesn’t work well compared with training in an environment with obstacles
from scratch.
4.3.6
Robustness
The robot doesn’t perform well in an environment with dynamic obstacles due to a
lack of information on the prediction of the obstacle motion. This needs to be further
investigated.
As mentioned in Section 4.1.4, the model shows the capability to handle the noise on
the wheels. However, if the dynamic on the wheels is too large, for example, one of
the wheels always rotates backward, the model will fail. If the robot always moves in
this way, then this feature can be learned by using the same dynamic in the training
environment. However, if the robot only moves in this way for a short period and the
malfunction happens randomly, then the training will fail unless the observation space
covers the feature of the wheel dynamics.
In general, the robustness of an end-to-end controller is hard to guarantee and evaluate.
The unexplainable neural network structure could lead to good performance in some
cases, but it could also fail unreasonably. By training with more data, and experiencing
as many as possible situations, the agent will be more robust.

4.4
Future work
Future work should resolve the problems mentioned in Section 4.3. A better implemen-
tation of the rangefinder should be used in training. The accuracy and eﬀiciency of the
sensor simulation should be balanced carefully. This needs testing and verifying in the
real world. The noise model of the sensor and robot dynamics should be added to the
environment as well. The training effect of using static maps and dynamic maps should
be compared. This can perhaps explain the different performances shown in the training
and the testing.
The reward function for obstacle avoidance needs to be improved to remove the non-
monotonicity. The heuristic rewards should be reformulated to become more intuitive
and clear. Furthermore, additional rewards or new states to observe the robot when
completing the task within the time limit can be added.
Using different neural network structures such as recurrent neural networks and LSTM
schemes may benefit knowledge transfer.
The low explained variance and abnormal
behavior of the policy estimate loss should be further investigated.
Since the object is making the ATR navigate in the factory environment, specific oper-
ations can be added to guarantee safety, for example, the robot should stop when it’s
getting too close to an obstacle.

CHAPTER 5
Conclusion
This thesis presents a complete pipeline for addressing mobile robot navigation control
tasks, including the creation of custom environments, training, and testing of the Deep
Reinforcement Learning (DRL) agent. The environment contains a differential-driven
mobile robot, a randomly generated curved path with obstacles, and a simulated laser
scanner. The mobile robot learns to follow a predefined path while avoiding obstacles in
a corridor using a deep reinforcement learning algorithm. Unlike most existing methods,
this project uses a model-free approach to solve the path planning and optimal control
problem. By letting the robot explore the environment and continuously get rewards,
the agent learns the optimal policy to deal with each situation. The reward functions
are separately designed for path-following, obstacle avoidance, and time eﬀiciency. A
proper quantization method was used to make the reward sparse to speed up the training
process.
The trained model shows great tracking performance in path-following tasks, keeping
minimal cross-track error with the reference path. Compared to traditional A* and lat-
tice planners, the DRL agent shows better obstacle avoidance capabilities with lesser
path deviation. The neural network controller, as an end-to-end method, shows advan-
tages in ease of application and relatively cheap computational cost. The result also
shows some robustness to the dynamics in the environment such as a moving obstacle
or slippery surfaces.
While the DRL method has shown great potential in solving the navigation problem,
there are still some challenges.
The training process is resource-intensive and time-
consuming.
Improving policy and value function estimation requires expanding the
neural network’s depth and units. This will lead to a more diﬀicult training process.
Additionally, the stability of the training is not guaranteed. Quite an effort is needed
to tune the hyperparameters to achieve a good result. The cause of failure in training
or testing is hard to identify, requiring various adjustments and compromises to achieve
intermediate targets.
Despite the drawbacks, the end-to-end way of learning and action selection mirrors
human behavior. With the development of learning algorithms addressing stability issues
and a universal approach to represent reward functions, DRL will be a promising method
for navigation tasks.
