I. INTRODUCTION
Bisimulation is a fundamental tool in computer science
for abstraction and minimization, relating models by
useful logical and dynamical properties [1]. Originally
developed to reason about concurrent processes in a
non-quantitative setting [2], it has been extended to
quantitative models based on labeled transition systems,
such as, for instance, the celebrated notion of probabilistic
bisimulation [3], closely related to ordinary lumpability for
Markov chains [4].
This report is mainly based and does the extension
on the report Minimization of Dynamical Systems over
Monoids [5]. The report‚Äôs main theory starts from the
generalized forward bisimulation (GFB). To explain GFB,
we first introduce forward bisimulation and forward dif-
ferential equivalence (FDE). Opposed to backward bisim-
ulations, forward bisimulations focus on the outgoing
transitions of the states and identifies equivalence classes
among the chemical species that preserve the total concen-
tration [6], [7]. And similarly, FDE identifies a partition
that induces a quotient ordinary differential equations
(ODE) that tracks sums of variables in each equivalence
class [8]. Here we use a small example to explain:
Consider a ODE system with variables x1, x2 and x3, k
represents for the discrete time:

system the Equation 1 can be changed into

In this section, we introduce the basic definition of the
dynamical system and explain the notion of notations used

Definition 1 (Dynamical System). A discrete-time dynam-
ical system (DS) is a system with several variables xi
inside. For each xi in the system, it has its own unique
update function fxi : MX ‚ÜíM. Here elements of MX
means states. We use X = {x1, . . . , xn} to represent
the set of variables xi as well as F = {fx1, . . . , fxn} to
represent the set of update functions. In this way we can
use (X, F) to describe the basic information of a discrete-
time dynamical system.
Based on the definition form the report Minimization
of Dynamical Systems over Monoids [5], here we use R
to denote an equivalence relation over X, and XR the
induced partition. We often do not distinguish among
an equivalence relation and its induced partition. If not
explicitly mentioned, we assume that ‚äē: M √ó M ‚ÜíM is
such that (M, ‚äē) is a commutative monoid with neutral
element 0‚äēin this report. Moreover, GI denotes the set
of all (total) functions from I to G and f[a/b] is the term
arising by replacing each occurrence of a by b in f.

V. APPLICATION
In this part we discuss about the application of this
theory in the boolean network case study. Most of these
applications are not the exclusive application - which
means other reducing algorithm may have the same effect.
But using this extension may find more results and
sometimes make the number of coarest GFB smaller than
the original reducing algorithm.
A. Simplify a DS system
Boolean networks suffer from the state space explosion
problem: there are exponentially many states with respect
to the number of Boolean networks variables, so decrease
the number of variables and simplify a DS system shows
great importance.
Using Example 1 in this paper as a running example.
To show the relation of each element in this Boolean
network, we will use a typical graphical named influence
graphs, as shown in Figure 3. In this figure, each note
represent a variable in this Boolean network and the
arrow means the influence between two different variables.
In particular, green arrows denote a positive influence
(promotion), while red dashed ones denote a negative one
(inhibition). For example, in the update function of x1,
the term x5 shows x5 promote x1 so there exists a green
arrow from x5 to x1, while x9 inhibits x5 due to term ¬¨x9
in the update function of x5 so the arrow from x9 to x5
shows red.
When we applying the GFB from example 3 and using
reduced model for XR from Example 4, we can see that all
red nodes in Figure 3 get collapsed into one node C1, as
well as yellow nodes get collapsed into C2. It means that
this BN are reduced into only five nodes now. For the
reason that we do not have to consider the transportation
between the nodes in C1 as well as those in C2, we finally
simplify the model.
Figure 3: Pictorial representation of the Boolean network
from Example 1.
B. Change the number of input/output variables
Variables in a Boolean network are always divided into
three groups: inputs, outputs and internal nodes [13].
Input variables (x9 in example 1), with a constant up-
date function, do not have any incoming edges. Outputs
variables (x0 in example 1) do not have outgoing edges,
and never appear in the update function of other variables.

==PAGE:8==
Internal nodes have both incoming and outgoing edges. In
particular, input variables are often set by the modeler to
perform different what if experiments, where as output
ones are used to observe the response dynamics of the
model [5].
We notice that after applying the reduced model,
sometimes we changed the number of input or output
variables, and these changes can bring different benefits:
1) increase the number of input variables: Increasing
the number of input variables means the modeler can have
more choice to change the input variables so that the
model can have more chance to perform different. Using
Figure 4 as an example, suppose that after applying the
reduced model all yellow nodes get collapsed into one node.
Then the modeler are able to make y1 ‚äēy2 as a new input
variables and find the relations between y1 ‚äēy2 and r1.
2) decrease the number of input variables: By decreas-
ing the number of input variables, modelers can decease
the number of useless test cases. Considering the figure 4
again, suppose after applying the reduced model all red
nodes get collapsed into one node. Then before reduction
the modeler need to go through all cases of x1 and x2,
but now the modeler only need to consider x1 ‚äēx2 and
check the response dynamics.
Figure 4: examples of change the number of input vari-
ables.
3) increase the number of output variables: More out-
put variables means more response dynamics of the model
can be observed. Using Figure 5 as an example, again
suppose that after applying the reduced model all yellow
nodes get collapsed into one node, here we get a new
output variable r1 ‚äēr2. And it can be a new response
which the modeler may be interested in.
4) decrease the number of output variables: Decreasing
the number of output variables means the modeler can
avoid to focus on the details and can observe the response
much easier. Suppose that after applying the reduced
model all red nodes get collapsed into one node in figure 5,
now the model offering a coarser representation of the
system focusing on one pathway (from x1 to r3 ‚äēr4),
and the modeler doesn‚Äôt have to focus on the r3 and r4
separately.

VI. RELATED WORK
All our research is mainly based on the bisimulation.
Bisimulations have been introduced in computer science
by Milner [14] and Park [15], it is always used to reduce
the number of states of a system by find other systems
which are model equivalence with less states. For bisim-
ulation, as mentioned in the senction I, opposed to the
forward bisimulation which we do the extension in this
report there also exists backward bisimulation. And L-
bisimulation [16], [17] can be seen as a generalization of
backward differential equivalence (BDE) [18]. Here [16],
[17], [19] shows that it is thus complementary to FDE.
However, neither BDE nor L-bisimulation allow for model
reduction through nonlinear transformations.
For Boolean network, here [20] shows how to re-
duce Boolean networks with backward Boolean equiva-
lence(BBE), a backward-type bisimulation in line with
those for Markov chains [4] and BDE. But just like the
8
==PAGE:9==
notions for Markov chains and ODEs [18]. When applied
to BNs, BBE and GFB (when applied to BNs) are not
comparable.
As we mentioned before, this report started from Min-
imization of Dynamical Systems over Monoids [5]. And
in [5] the algorithm is only able to find GFB equivalent
variables separately, which we proofed in section I that the
GFB partition found in this way may not be the coarsest
one. Meanwhile, our extension allowed to show all results
of reducing a BN with N variables into a bisimilar one
with M variables and limit some specific variables in one
set. However, their framework can be applied not only
on Boolean networks, but also difference equations with
monoid (R, ¬∑) and multi-valued networks, which is still
waited to be proofed for our extension.
VII. CONCLUSION
In this report, we discussed about generalized for-
ward bisimulation for dynamical systems over commuta-
tive monoids and extended the existing reduction tech-
niques from Minimization of Dynamical Systems over
Monoids [5], making reducing with disjunction and con-
junction at the same time possible. However, two reduction
algorithms mentioned in this report are both ineÔ¨Äicient
and still needed to be improved. We implemented the
dividing algorithm in the report and tested the time it
will cost in different situation using cases from GINsim
repository. While in this experiment, we notice that the
reduction algorithm still has lots of ways to improve:
1) In the merging algorithm (see Remark 3), we notice
that ‚äēcan be different in one set. For example, giving
a DS with X = {x1, . . . , xn} and F = {f1, . . . , fn}, then
x1‚ą®x2‚ąßx3 also can be one possible set of induced partition.
However, in the dividing algorithm, in one set it can
only have one kind of ‚äē, which means the number of
equivalence classes still has possibility to be decreased.
2) Our algorithm now only support Boolean networks,
but it‚Äôs also possible to do an extension and try to
implement in on the Multi-valued network. In this kind
of network the state variables can be defined over two
values. Here [21] extensively study the conditions for
the conversion of multivalued networks to Boolean ones
using Van Ham code [22]. And [23] define a method
inferring a Boolean network bisimilar to the multivalued
one whatever the coding procedure. Which means when
input a multi-valued network, we can transform it into a
Boolean network and then applying our algorithm.
