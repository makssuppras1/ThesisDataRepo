Abstract
Spatio-temporal prediction of passenger-demand is of interest for many shared mobility
services. By having a model that can predict passenger-demand, the service provider can
have an eﬀective strategy for allocating resources where it is needed. As shared mobility
services are rather new, many cities only have a very limited oﬀering of these services
and in many cities there is none. This also means that there is no previous usage data
for many cities where the goal is to deploy a new service. This makes it hard to create
a model that can predict passenger-demand with satisfying results as regular machine
learning model generally requires a lot of data for training. In this report, we explore how
to create a model that is optimized for predicting in a setting with no or very limited data
is available. Speciﬁcally, the proposed model is based on graph neural networks combined
with a meta learning strategy. By collecting data from multiple cities and for multiple
types of shared mobility services, the model will be trained on a variety of datasets and
learn how to generalize across cities and services. The model is compared to a transfer
learning model and evaluated on multiple datasets. Here we show that a meta learning
model achieves better results than a transfer learning model. Finally, the model is used in
a practical setting where it is explored how well it will perform over diﬀerent time horizons
on a speciﬁc dataset. Here we ﬁnd that the meta learning model is eﬃcient at adapting to
a new city and that it achieves satisfying results even when used in a zero-shot setting.

Introduction
The goal of this project is to create a model to predict passenger-demand for both diﬀerent
cities and diﬀerent passenger-demand types. Passenger-demand is in this project deﬁned
as pickups of any mobility service that can be used by people within a city. This could
be taxis, shared bikes, for-hire-vehicles etc. As the demand for such services increase,
the need for eﬀective passenger-demand forecasts also grows. Additionally, since these
services will deploy to new cities it is desired to have a model that can generalize to new
cities across multiple services. This problem is exactly what is studied within this report,
where the goal is to create a model that can generalize to new cities and services and be
used as an ”initial deployment” prediction-model for anyone that wish to deploy a new
passenger-demand service.
1.1
Forecasting passenger-demand
The problem of predicting passenger-demand is not new and a lot of researchers are already
exploring this ﬁeld[1][2]. With the recent development within deep learning, state-of-the-art
within the ﬁeld of predicting passenger-demand is continuously being pushed. The newest
modeling techniques are using graph neural networks to model interactions within cities.
Here cities are split up into regions that represent nodes within a graph. Each node is
then connected to other nodes using some metric or historical information. The most
active research within this ﬁeld revolves around two main ways to push the state-of-the-art.
Either a new graph model is developed that is eﬃcient at learning complex demand
patterns or a new way to model the city with a graph is created that gives a better signal
for the models to ﬁnally obtain a better result. This focus is quite narrow and in this

project the main focus will not be on either of those.
1.2
Learning on new cities
As previously mentioned, the goal is to create a model that can generalize to new cities
and multiple services. One could think of it as a general model for all passenger-demand
problems. Such a model has great beneﬁts as it removes the need for collecting data
before a model can be deployed. Instead, this general model can be used to bootstrap the
automation of the service by predicting passenger-demand from the ﬁrst day of rolling out
the service.
The method that will be used to solve this problem is meta learning. Meta learning is
an area within machine learning that has gained a lot of traction lately, especially after
the MAML algorithm was introduced[3]. The goal of meta learning is to optimize a meta
objective. The meta objective is most often related to fast adaptation to new tasks. In
this project, a task is deﬁned as a single service within a city, i.e. a dataset of taxi pickups
in New York or bike pickups in Washington DC. So in order to be able to optimize a
model for this meta objective, multiple datasets will be collected that span multiple cities
across diﬀerent types of passenger-demand.

2.1
Graph Neural Networks
The term Geometric Deep Learning was ﬁrst formalized by Fererico Monti et.al in 2016 [4].
In Bronsteins recent book [5], he goes into detail about what Geometric Deep Learning
is and the applications of it. In short, the Geometric Deep Learning framework provides
methods to model data that is generated from non-euclidean domains. More speciﬁcally,
by using graphs with nodes and edges to represent connections and complex relationships,
a more expressive model can be created that often gives a better representation of the
real world. This has also led to a big increase in the research done within this ﬁeld, as
many researchers ﬁnd that using methods from Geometric Deep Learning enables them to
outperform existing methods[6]. Since these models computes directly on graphs, a more
commonly used term for Geometric Deep Learning is Graph Neural Networks (GNN).
The idea of GNNs however is not a new idea developed within recent years. One of the
ﬁrst papers to introduce GNNs was published in 2009[7]. Here the authors developed a
Recurrent Graph Neural Network (RecGNN), which operated on each node by recurrently
updating the hidden state of the node using information from neighbouring nodes. This
had some limitations, both in terms of computation but also because of the requirement
that the recurrence was limited to be a contraction mapping in order to converge. Since
then, a lot of research has been done within GNNs and the main methods used now are
Convolutional Graph Neural Networks (ConvGNN). ConvGNNs can be further separated
into Spectral and Spatial based methods.

2.2
Meta Learning
With recent advances in deep learning, it has been possible to achieve impressive results
on tasks such as image recognition, machine translation and game-playing[22][23][24].
However, a major hurdle still exists for all these tasks as none of them generalize very well
and are super-optimized to a speciﬁc task. With standard deep learning techniques the
models are trained from scratch to predict a target that minimizes some loss function.
This means that the models do not use any prior experience to learn a new task. The only
priors given are those that are set by the one modeling the problem. This makes deep
learning ineﬃcient as the datasets being used for training has to be big in order for the
algorithm to learn the task with satisfying results.
Meta learning provides an alternative way of training machine learning models that more
closely resembles how humans learn[25]. In meta learning, some meta-knowledge ω is
learnt from a distribution of tasks P(T ). A task within this distribution is deﬁned as
T = {D, L}, where D is a dataset and L is a loss function. The meta-knowledge can
represent anything i.e. parameters of a model[3], embedding of previously seen data[26], a
learnt optimization schedule[27] etc.
The training procedure of a meta learning algorithm can be split into two levels: An
inner-level and an outer-level. In the inner-level, a new task Ti is presented to the
meta-learner. Here two subsets (support set and query set) are sampled from the dataset
Di. The support set, is used to optimize the meta-learner w.r.t the loss function Li with an
optimization procedure fω, this is called the inner-loss. In the outer-level the optimized
model is used to predict on the query set and with this we can compute the outer-loss or
query-loss. Intuitively, the outer-level, accumulates meta-knowledge ω from the inner-
level and since the inner-level is an optimization w.r.t to a distribution of tasks, the
outer-level optimizes the meta-knowledge ω to eﬃciently learn new tasks. Formally, we
can put the training procedure as the following optimization of the meta-knowledge ω.

A popular setting is K-shot learning where only K examples are shown at the inner-level.
The inner-level then needs to eﬃciently adapt the meta-learner, for the outer-level

”Learning-to-learn”. In the case of K = 0, a Zero-shot learning approach is used.

There exist other methods such as transfer learning[28], where pre-trained models can
be used to eﬃciently train a model on a new task. This is similar to meta learning,
although it is not the same. Meta learning is deﬁned by explicitly having a meta-objective
(the outer-level), that optimizes for learning on new tasks. The meta-objective can
roughly be split into the three following categories Metric-based, Model-based and
Optimization-based. These will be explained by examples in the next sections.

2.2.2
Model-based meta learning
Model-based or black-box meta learning models, aim to learn an internal state of some
model. This is diﬀerent from the metric-based approaches, where the model is ﬁxed at
test-time. The internal state is updated by sequentially taking in data points from the
support set. Here the goal of the model-based approach is to quickly adapt to the support
set by updating its internal state appropriately. The meta-knowledge ω of a model-based
approach is given by how it updates its internal state, which is also why it is termed
black-box meta learning.
The ﬁrst methods using model-based meta learning used recurrent layers to sequentially
take in data points from the support set to update the internal state[30]. However, using
recurrent layers poses a problem as the order of the data points matter and the fact
that it needs to keep previous data points in memory. A solution to this is given by the
Conditional Neural Process(CNP)[31].
The CNP does not take datapoints from the support set sequentially and it does not rely
on some external memory. The general overview of the model can be seen in Figure 2.2.2.
Here the datapoints {(x1, y1), (x2, y2), (x3, y3)} are part of the support set. For each of
them an embedding ri is computed by the same neural network h. These embeddings are
put into an aggregation module a that computes the internal representation r of the CNP.
Finally, predictions on the query set are performed by a neural network g that takes the
internal representation r as an input.

3.1
Representing passenger-demand with graphs
3.1.1
Dataset structure
For this project several datasets were collected. All the datasets have one thing in common;
they represent passenger-demand in diﬀerent locations of any shared mobility service over
a period of time. This is the only assumption about every dataset collected within this
project. A table of all datasets can be found in Table A.0.1.
Every record within a dataset represents a passenger interacting/transacting with the
transportation method. An example could be a dataset of a bike sharing service, where
every record is a passenger picking up a bike. The records contain timestamps and
longitude/latitude positions. In some cases, the records also has pre-speciﬁed regions or
stations that represents where the interaction started. If this is not the case, a grid-structure
will be used to create regions. This is described in Section 3.1.2.
When regions are deﬁned, the records are split into 1-hour intervals where the number
of records for each region is counted. This number is the passenger-demand and will
be used both as a feature in our models and the target to predict. In addition to this,
information about the weather and a time-encoding of the timestamp is used to provide
external features.
3.1.2
Creating the graph
To explain how passenger demand at each timestep t is represented an example will be
used. For the example, the Citibike2014 dataset is used which contains bike pickups in
Page 15 of 66
==PAGE:25==

New York. This dataset has 313 stations where bikes can be picked up. See Figure 3.1.1
where each dot represents the location of a single station/region.
Figure 3.1.1: Red dots represents regions where bikes can be picked up.
Naturally, each station can be represented as a node with a graph. However, this method
gives rise to a problem; how to represent edges/connections between the nodes. In [2],
the authors create a multi-graph that is given by a weighted sum of three graphs. The
multi-graph is used as the adjacency matrix and represents weighted connections. The
three graphs are Distance Graph, Interaction Graph and Correlation Graph. Both
the Interaction Graph and Correlation Graph rely on historical usage patterns. This
makes these two graphs non-feasible in a meta learning fashion, as the premise of meta
learning is that the model will learn quickly on data it has not yet seen. If there is no or
very limited historical data, the Interaction Graph and Correlation Graph will have
a lot of noise and can not be trusted for representing a connection between two nodes.
The Distance Graph is feasible to use in a meta learning fashion. However, the resulting
adjacency matrix is weighted. This is a problem for some GNN models such as GAT [13].
Also, distance is most likely not a good measure for similarity between nodes. Instead,
the pickups were partitioned into grids, where the grids span the whole area of all regions.
Now each pickup can be assigned to a speciﬁc square within the grid, which means that
nodes are represented by grids. The adjacency matrix in the grid system, will be given by
the neighbouring squares. In Figure 3.1.2 a 10×10 grid is used. The left plot shows each
square within the grid. The number is how many pickups happened within that speciﬁc
square over an hour, where each dot represents a station where pickups are possible. The
right plot illustrates the connectivity of four squares (nodes), where they are connected to

One major advantage of the EdgeConv model described in Section 2.1.4 is that it does
not require edges. Instead it learns the edges itself dynamically. This makes the EdgeConv
model easy to use and removes the need for using historical data to create the edges.
3.1.3
Data processing pipeline
Before training any models, a data processing pipeline was used to standardize the format
of every dataset within the project. The pipeline consists of a series of sequential steps
that ensures consistency and that can be applied to every dataset collected. The steps are
as follows:
1. Partition dataset into regions/grids and create adjacency matrix accordingly.
2. Count number of pickups in each region over intervals of time. Each interval is a
single timestep.
3. Encode time information into a binary vector representing time, weekday, month etc.
4. Fetch historical weather information at each timestep using a weather API[34].
5. Combine pickups, longitude/latitude, time encoding and weather information for
each hourly interval for each region/grid. This will be the features.
Page 17 of 66
==PAGE:27==
Models and baselines
6. Create target values for timestep t by using the pickups at timestep t + 1 for each
region/grid.
7. Normalize all information for each region/grid to µ = 0 and σ = 1 (except time
encoding).
The pipeline ensures that every dataset is in the same scale. This will make it easier to
compare performance across datasets but it will also be crucial when the datasets will
be used for meta learning. If the datasets are not on the same scale, it would mean that
the loss would not be either. This would cause the meta learning approach to put higher
emphasis on datasets with a greater number of pickups as the variance/noise of those are
most likely higher.
3.2
Models and baselines
3.2.1
Selecting a model
In order to select a suitable model for meta learning, some testing was done with diﬀerent
models and architectures. Their limitations and beneﬁts were evaluated together with the
performance they achieved on two benchmark datasets (Citibike2014 & Citibike2021). An
important thing to note about the meta learning setup, is that the model that will be
used has to be ﬂexible. This means that it should be able to adapt to diﬀerent adjacency
matrices. Both in terms of the edges, but also in terms of the number of nodes. This
limitation makes pure spectral GNNs unusable as the graph-laplacian will change when-
ever the adjacency matrix changes. Additionally, many state-of-the-art models within
passenger-demand prediction requires that the number of nodes are ﬁxed and known
beforehand [35]. This is also infeasible for the meta learning model.
With this in mind, a total of 3 models were developed. All the models follow a similar
structure as in Figure 3.2.1, where the Graph layers capture spatial eﬀects that are then
fed into a LSTM layer to capture the temporal eﬀects. In all the models, only the previous
12 timesteps are used for predicting the demand at the next timestep, i.e. q = 12. The
timesteps are seperated into hourly intervals. Finally, the external features such as weather
and time is concatenated with the last hidden state. This is then used as the input for a
Feed-Forward Network that predicts the demand in each region of the graph. This model
structure is ﬂexible, as the graph layers are invariant to changes in the adjacency matrix.

Algorithm 2, the training procedure of the transfer learning model is given. Here it is
noted that the query set is not used and that the model parameters are updated each time
the loss is computed on the support data.

3.3.3
Augmenting datasets
For training the meta learning model a number of tasks has to be deﬁned. For every
dataset that is collected, there can be created multiple tasks. Creating multiple tasks
from a single dataset, can be seen as augmenting the dataset to provide multiple ”new”
tasks. For this project, all datasets that are not augmented use a time resolution of 1-hour
and a grid size of 10x10. In cases where regions are deﬁned these are also used in the
non-augmented pool of datasets. From here, multiple other tasks can be created from the
same dataset. All diﬀerent combinations of the following augmentations were done:
• Partition with a grid structure with diﬀerent grid sizes (5x5, 10x10)
• Partition by regions deﬁned in dataset (if pre-deﬁned regions exists in dataset)
• aggregate pickups over diﬀerent time intervals (1 hour, 4 hours & 8 hours).
This means that although not a lot of datasets have been collected, they can be processed
in multiple ways to potentially make the meta learning model generalize even better or
adapt faster in settings where only a few datasets are collected.
The notation for augmented dataset will include a grid size and/or a time resolution. An
example of the Uber dataset could be Uber-GRID-5X5-4HOUR. Here a 5x5 grid is used
and pickups are aggregated over a 4-hour time interval.
3.3.4
Filtering datasets
When datasets are collected for training a meta learning model, it would be beneﬁcial to
have a method for ﬁltering out datasets that do no provide value for the meta learning
model. To solve this, a heuristic was developed.
The heuristic computes a matrix of losses. The matrix is created as follows: A model is
trained on each dataset. These models are regular models (no meta learning). Now for
each model, the loss on the test data from all datasets will be computed. This means that
if there is 10 datasets, in total 100 losses will be computed. In the matrix, the values in
the rows represent losses for each model on one of the datasets. The columns are then
normalized, by the loss where the model matches the dataset it was trained on. In Figure
3.3.2 the structure of a loss matrix is shown with 3 datasets.
Page 27 of 66
==PAGE:37==

Figure 3.3.2: Example of loss matrix. Values in a row represent the loss of
diﬀerent model evaluated on a speciﬁc dataset. The columns are normalized,
where the model matches the dataset it was trained on.
When the matrix has been computed, it can be roughly viewed as a distance matrix. I.e.
when model X is evaluated on dataset Y, a higher loss represents a higher distance between
the two. Vice versa, if the inverse is computed on all the losses, it can be viewed as a
similarity matrix. This gives rise to using clustering techniques to ﬁlter out datasets that
do not follow the same pattern as other datasets.

4.1
Evaluating meta learning model
4.1.1
Experimental setups
The following experiments in this section all follow the same underlying structure for
evaluating a meta learning approach to the passenger-demand problem. In addition to this,
the Capital Bikeshare dataset is not used in any of these experiments as it will be held-out
for a case study, however all the other datasets from Appendix A is used. The goal of
the experiments are to evaluate the meta learning model and compare it against other
relevant models/baselines. A transfer learning model is compared to the meta learning
model. The setup for all the experiments can be summarized as follows:
A number of datasets are collected, these are called meta-datasets. The meta-datasets
are split into two; a meta-training set and a meta-evaluation set. The meta-training set
is used for training the meta and transfer learning models using the methods described
in Sections 3.3.1 & 3.3.2. Each dataset in the meta-training and meta-evaluation set
can be further split into a training and test split. For an example, if we pick out any
meta-dataset, a model is only trained on the training set and only evaluated on the
test set of this speciﬁc dataset. Here it is emphasized, that meta-training and meta-
evaluation sets refer to whole datasets, while the training and test sets refers to parts of a
single dataset. These rules for evaluation applies to all models in the following experiments.
An important thing to note is that for all datasets, the same train/test split is used. This
Page 29 of 66
==PAGE:39==
Evaluating meta learning model
means that when the models are compared to each other, it is guaranteed that they have
been exposed to the exact same data. For evaluating the models, the test data is used and
none of the models have been exposed to this data before. This creates a fair comparison
between all models.
The list of models is given below. The results reported in this chapter is based on these
models, where the loss has been computed on the test set of a speciﬁc meta-evaluation set.
• An Edgeconv model trained on the dataset used for evaluation.
• A ﬁne-tuned meta learning model. Fine-tuned on the dataset used for evaluation.
• A ﬁne-tuned transfer learning model. Fine-tuned on the dataset used for evaluation.
• A meta learning model. Trained on the meta-training sets.
• A transfer learning model. Trained on the meta-training sets.
• An Edgeconv model with randomly initialized parameters.
• A baseline that uses the passenger demand in previous time step to predict the next.
The ﬁne-tuned models refers to a model where the initial parameters of the model are
not randomly generated. Instead, the parameters of either the meta or transfer learning
model is used to initialize the corresponding ﬁne-tuned model. With these parameters,
the ﬁne-tuned models are then trained on the training set of a single meta-evaluation set,
in a similar fashion as the Edgeconv model. This means that in total 3 of the models have
been exposed to training data from the dataset that is used for evaluation. These models
are referred to as the trained models. The other models, i.e. the meta and transfer learning
model and the randomly initialized Edgeconv model is referred to as the adaptable models.
When evaluating the models, the adaptable models are ﬁrst exposed to the training set of
a single meta-evaluation set. Here k examples from the training set are randomly sampled
and used to adapt the adaptable models. This follows the principle of a meta learning
setting where the model is supposed to adapt quickly to a new setting. The reason for
including both a transfer learning model and a randomly initialized Edgeconv model in
the adaptation, is that this makes the comparison between these fair. I.e. they have been
exposed to the same part of the meta-evaluation set.
This evaluation scheme is shown in Figure 4.1.1. When k examples are sampled, it is
done several times to estimate a mean and standard deviation. This can only be done for
the models that are adapted.
In all the experiments, either 0, 1 or 5 shots are used for the adaptable models. A single
shot is in this case, deﬁned as the number of pickups in each region over the last 12

observations. What the model has to predict is then the number of pickups in the next
time period. If hourly pickups are used, a 1-shot setting is where a model is given the
pickups over a period of the last 12 consecutive hours and should predict the number of
pickups in the next hour (13th hour). The 0 −shot (Zero-shot) problem now refers to an
prediction from an adaptable model, that has not yet been adapted to the problem. Lastly,
a 5-shot setting, is where 5 periods of 12 hours have been sampled and can be used for
adaptation.

to see, whether the augmentations achieve a better result in the ablation study. The
experiment is performed as follows:

4.3
Case study - Deploying a shared bike service to
a new city
In this section, a more practical use of the meta learning model will be explored. The case
that is considered is a bike sharing company that has planned to deploy their service in a
new city. This is done using a already existing dataset for the city Washington DC. The
dataset used is the Capital Bikeshare dataset.
The goal is to deploy a model that can be used to predict passenger-demand of their bikes
in diﬀerent regions of the city. As they have no prior knowledge about the city, they split
up the city into regions which are deﬁned by a grid. In the best case, the model that they
will deploy can make accurate predictions shortly after the service has been deployed.
4.3.1
Filtering out datasets
In the process of experimenting with diﬀerent meta learning settings in Section 4.1, it
was found that some datasets provided more value for the meta learning model than other
datasets. Because of that, it was decided to look into creating a method for discovering
which datasets provide less value and then ﬁlter those out before using them for the case
study. For this the method described in Section 3.3.4 was used to compute a distance
matrix.
The distance matrix was computed using all the datasets that were considered for training
a meta learning model. This includes all datasets, except the Capital Bikeshare dataset.
Additionally, a simulated dataset was included to validate the method.
The simulated dataset was created, by modeling 200 nodes, where the values to predict
for each node was independent from all the other nodes. I.e. there was no graph structure
to learn between the nodes. More speciﬁcally, the values in each node followed this simple
linear function f(t) = a · t + b, where a and b varied between each node and t was the time
from the ﬁrst observation. It is expected that a model trained on this dataset performs
poorly on all the passenger-demand datasets and all models trained on passenger-demand
datasets perform poorly on the simulated data.

4.3.2
Deploying a online learning model
Now a collection of datasets have been gathered, and they have been ﬁltered to ensure that
knowledge can be transferred between them. The next step is to train a meta learning
model and see how it performs in diﬀerent scenarios on the Capital Bikeshare dataset. The
Capital Bikeshare dataset has been processed in the same way as all the other datasets in
this project. No augmentations were used for this part and therefore it uses a 1-hour time
resolution and separates the city into a 10X10 grid.
In total four models will be used for evaluating the meta learning model. The models will
be evaluated in two diﬀerent settings. In the ﬁrst setting, the models will be evaluated
without online learning. The second setting, a online learning approach will be used.
Here online learning refers to the models being adapted/updated with a gradient descent
algorithm every time a new observation is observed. The adaptation is always performed
after the model has predicted on the observed data. For the online learning setting a
batch-training procedure was also used where the online learning models would be trained

on batches using all observed data. The idea was to see if this achieved better results than

not included in the report.
The following models are used in the case study:
• Edgeconv: This model is a simple Edgeconv model. It has not been pre-trained in
anyway. However, every time it observes 25 datapoints, it is retrained on all the data
it has observed up until that point. This model is the same with/without online
learning.
• Partly trained Edgeconv: This is a simple Edgeconv model that has been trained

• Partly trained meta model: This is a meta learning model that has been pre-

the online learning setting it is also being adapted on each observation after the
black line.
• Meta model: This is a meta learning model that has been pre-trained on the
collected datasets in a meta learning setting. It has not been ﬁne-tuned. In the

Discussion
5.1
Reprensenting passenger-demand with graphs
In Section 3.1, a multi-graph is mentioned where edges between nodes are created/weighted
by some metric, i.e correlation or interaction etc. However, using such a metric would
cause issues in a real-world setting where it is not possible to compute a correlation before
data is actually captured. In the literature these multi-graphs were quite popular. Another
often used alternative to these are simply ignoring pre-deﬁned regions and manually create
a grid that captures all pickups within a dataset. With this, one can exploit the structure
of the grid by using neighbouring nodes etc. Intuitively, one would expect that adjacent
nodes within a grid has some similarities and that the usage patterns for all neighbouring
nodes can be used to infer the usage pattern of the node itself. By using a grid structure,
it was also possible to collect more datasets, as the datasets were not required to contain
pre-deﬁned regions. Although the regions may be a better representation for nodes within
a city, the grid provided a decent alternative when the regions were not pre-deﬁned
5.2
Selecting Edgeconv model
When selecting the most appropriate graph neural network model to implement in a meta
learning setting two main issues were addressed. First, the model had to be ﬂexible in the
sense that it should not be limited to a speciﬁc structure of the graph that was presented
to it. Secondly, it needed to have at least a decent performance when comparing it to
other graph-models.
Page 50 of 66
==PAGE:60==
Data augmentation
In Section 3.2.1, 3 models were tested. A simple baseline Graph Neural Network model,
a Graph Attention model and ﬁnally the Edgeconv model. It was found that the Edgeconv
model actually showed the best performance on two diﬀerent datasets. One where a region
partioning was used and another one where a grid partioning was used. In addition to
showing good results, the Edgeconv model is also a very ﬂexible model. Since it does
not require pre-speciﬁed edges it removes the choice of how the edges should be created,
as it is done by the model itself. What this means is that it removes the inductive bias
of the one who is modelling the problem. Especially in a case where there is no right
or wrong way to create the edges this seems to be the most objective solution. For
the passenger-demand problem, there is no clear way of creating the edges and in the
literature it is done in multiple diﬀerent ways. For that reason, these dynamic graph
layers could be interesting to apply to some of the existing research. It should be noted
however, that some other popular metrics such as demographic information was not
used for creating edges within a graph. As the other two graph models use information
from edges, it could be that their performance would have been better if more eﬀort
was put into optimizing the graph structure. But as mentioned before, this would be in-
feasible in a meta learning setting and was therefore not considered important for this work.
In Section 3.2.2, it was explored how the Edgeconv model learned relationships within a
graph. In this case, it was applied to one of the datasets used in this report. A interesting
ﬁnding was that the Edgeconv model seems to learn relationships between some nodes
that are distant from each other. These relationships would not be found if the edges were
created using a distance metric. For that reason, the Edgeconv model could potentially
also be used to explain node relationships in other problems where it is relevant. This
is also why this model is powerful for image segmentation. The transformations in the
Edgeconv model are non-linear as they are done by deep neural networks. For that reason
it may be hard to understand how the nodes are actually embedded. However, if it treated
as a black-box one could use it as a proxy for node similarity.
5.3
Data augmentation
In Section 4.2, an ablation experiment was done to evaluate the performance of aug-
menting datasets. The reasoning for using augmentations was to see whether it can boost
the meta learning model when few datasets were included for training the model. When
datasets are augmented, each augmentation can be viewed as a separate dataset and thereby
increasing the number of diﬀerent tasks that the meta learning model can use for training.
By increasing the number of datasets, the hypothesis is that the adaptation/generalization
Page 51 of 66
==PAGE:61==
Meta learning vs. transfer learning
performance of the meta learning model would improve.
From Figure 4.2.1, it can be seen that the augmentations has a poor performance when
the models are evaluated without adapting to any examples, i.e. k = 0. When k is
increased, the augmentations achieves a similar performance to the non-augmented setting.
Additionally, it is observed that the mean loss increases when more datasets are removed.
This makes sense, as more datasets during training means that the meta learning model is
trained to adapt to multiple diﬀerent tasks. This should make it better at adapting to
new unseen tasks.
It should be noted that for this experiment, that whenever the models are evaluated it
is done on non-augmented datasets. This does give an advantage to the model trained
on non-augmented datasets. For an example, all the non-augmented datasets are using a
1-hour time resolution. When the datasets are augmented, a 4 and 8-hour time resolution
is also included. Using diﬀerent time resolutions, most likely changes the dynamics of the
graph model. Therefore, the augmented models are most likely trained on some tasks, that
are less related to the dataset that is used for evaluation. This could potentially decrease
the performance of this model.
These augmentations should be researched further and this ablation study was done to get
an indication for whether it can be useful to augment datasets, to increase the number
of tasks when training a meta learning model. However nothing can be concluded with
certainty from this result.
5.4
Meta learning vs. transfer learning
In Section 4.1, the beneﬁts of a meta learning model was compared to a transfer learning
model. Two baselines were also added to make sure that the two models actually learned
something useful. The two baselines were an untrained Edgeconv model and a model
that used the observation at the previous timestep to predict the next. Across all the
experiments, the meta learning and transfer learning models outperformed these two
baselines.
Four diﬀerent experiments were performed, where the datasets used for training the models
varied across the experiments. In the ﬁrst experiment, all data was included for training
except the dataset that was used for evaluating the models. Here it was found that the
two models had very similar performance. On some datasets the meta learning had the
lowest loss, while on some the transfer learning was the lowest. Also, the error bars which
represents 1 standard deviation of the two models very often overlapped.
The last three experiments showed some more interesting results. Here there was a
general trend where the meta learning model actually outperformed the transfer learning
Page 52 of 66
==PAGE:62==
Meta learning vs. transfer learning
by a signiﬁcant margin. What had changed in these experiments, were which datasets
was included in training. In the second experiment (Section 4.1.3), the datasets were
augmented with a diﬀerent time-resolution. The models were only exposed to datasets
with a 4 and 8-hour time resolution during training and they were then evaluated on a
1-hour time resolution. From this experiment, it can be concluded that the meta learning
model is better at generalizing to diﬀerent time resolutions. The transfer learning model
is in this case not even able to outperform the two baseline models. This shows that it
has really not been able to learn from the other time resolutions and thereby transfer
knowledge from this. The meta learning model does outperform the baselines and it can
be concluded that it has learnt something from the other time resolutions. Given that the
meta learning model is optimized for fast adaptation, it makes sense that this model is
better in this case. Since both a 4 and 8-hour time resolution was included in the training,
the meta learning model was optimized to adapt between the two time resolutions. This
most likely gives it an clear advantage in this case. However for the zero-shot case the
meta learning model also outperformed the transfer learning model, so in general it simply
ended up as a superior model.
The last two experiments were similar. The only diﬀerence is that in one of them the
datasets were augmented (Section 4.1.4), while in the other one they were not (Section
4.1.5). In both experiments, all datasets were used for training except the ones that were
related to bike pickups. With this setup, it would be apparent whether the meta learning
model is better at adapting to new passenger-demand types. Here the meta learning
model again outperformed the transfer learning model in both experiments. This shows
two things. First, that bike pickups have a diﬀerent usage pattern than other services,
i.e. taxis or cars as the transfer learning model could not transfer knowledge from other
transportation types to the bike pickups. And second, that the meta learning model is
able to adapt to this new setting better than a transfer learning model. Here it was also
ensured that the meta learning model did not outperform the transfer learning model
because of augmenting the data, as one of these experiments included only non-augmented
data.
With these ﬁndings, it was concluded that the meta learning model is better at gener-
alizing to new settings. This was also expected, as this model is optimized speciﬁcally
to adapt to a new setting quickly. However, the meta learning also performs well when
k = 0, which shows that the parameters it has learned are generally good at solving
the passenger-demand problem. In fact, the beneﬁts of increasing k was quite small and
therefore it can be concluded that it is possible to create a zero-shot prediction model for
the passenger-demand problem in the way a zero-shot model is presented in this report.
Page 53 of 66
==PAGE:63==
Filtering datasets
The transfer learning model is optimized to perform well on the data it was trained on.
Instead, of treating several datasets as diﬀerent tasks, the transfer learning model pools
all datasets together and ﬁnds a good model for this combined dataset. When changes
occur in the dataset, i.e. a new passenger-demand type, the transfer learning model will
have problems with performing well on this new setting. This is most likely the cause for
why a transfer learning model is not as good as the meta learning model. This is even-
though there were exposed to the exact same data, both throughout training and evaluation.
For all the experiments, diﬀerent number of k-shots were used to see how the adaptable
models beneﬁted from this. Across all the experiments, it was discovered that increasing k
did not beneﬁt the models signiﬁcantly. In the case of k = 1, the error bars were quite
large, which means that it caused a high variation in the performance. The variation in
performance is most likely based on which example from the training data was included.
This could indicate that the meta learning model is able to signiﬁcantly change the
parameters using a single observation.
Finally, a ﬁne-tuned meta learning and transfer learning model was also included. Here
the model to compare against was the Trained Edgeconv model. The ﬁne-tuned models
were ﬁne-tuned using the same training setup as the Trained Edgeconv. This means that
they were exposed to the same data. However they were ”pre-trained” in either a meta
learning or transfer learning fashion on some other data beforehand. It is generally noted
that the performance for all the three models are similar. However, in some cases it seems
that both the meta learning and transfer learning model is able to outperform the Trained
Edgeconv. An example of this, can be seen in all the ﬁgures of experiment 1 (Section
4.1.2). The reason for why they can outperform the Trained Edgeconv, is that their initial
parameters before ﬁne-tuning can potentially lead to a better local minima compared to
the Trained Edgeconv. It could be that the parameters of the models are already close
to a good minima, since they have already been pre-trained. Additionally, if the training
time was restricted for training the models, the meta learning model would most likely
ﬁnd a good set of parameters faster than the other two models due to its eﬃciency in
adapting itself.
5.5
Filtering datasets
In Section 4.1, it was noticed that the Green Mobility dataset generally had a diﬀerent
loss pattern than the other models during evaluation. An example of this is that the
trained model was barely any better than the baseline model in many cases when a trained
model was to predict on this dataset. The reason for this, was because this dataset was
Page 54 of 66
==PAGE:64==
Meta learning in an online learning setting
quite small and therefore contained few pickups in each region for every hour. In many of
the regions, the number of pickups was 0 and in this case the baseline is expected to be
good as it will predict 0 in many cases.
Because of this, a heuristic was created to investigate whether the Green Mobility dataset
was actually providing value to the meta learning setup. If not, it could be that it would
make the results worse compared to excluding it when training a meta learning model. In
Section 4.3.1, the setup for ﬁltering out datasets is explained. Here a simulated dataset
is included where the nodes are completely independent. As the Edgeconv model learns to
extract information from similar nodes it was expected that models trained on this dataset
would perform poorly on the passenger-demand datasets and vice-versa. In Figure 4.3.1
a dendogram of the similarity matrix of losses is shown. Here it is clear that the simulated
data has a high dissimilarity with all the other datasets as models trained on non-simulated
data show a diﬀerent pattern when evaluated on the simulated data. With this ﬁnding
it was concluded that this could be a potential heuristic for ﬁltering out datasets. Since
the Green Mobility dataset also had a high dissimilarity with the other datasets and was
actually closer to the simulated data, it was concluded that the Green Mobility dataset
should not be included in the case study.
This method is only a heuristic and for that reason it can not be concluded to be a valid
strategy for ﬁltering out datasets. Also, losses may not be the best proxy for similarity.
Instead one could exploit the fact that the Edgeconv model is using a node embedding.
This node embedding could have been compared across datasets to see if a model trained
on the simulated data embeds nodes in a diﬀerent way compared to models trained on the
other datasets.
5.6
Meta learning in an online learning setting
In Section 4.3 a case study was performed. This was done to showcase how a meta
learning model can be useful for a shared mobility service that wants to deploy their service
in a new city. Since the setting is a time-series problem where data can be continuously
captured, the models can be deployed in a online learning setting. Here it is expected that
the models become better over time as more data has been captured. However, using a
meta learning model the goal is to have a predictive model that can perform well from the
beginning. It is expected that the beneﬁt of using a meta learning model is greatest in
a low-data setting, i.e. when not a lot of data has been observed. This means, that the
beneﬁts of the meta learning model will be diluted as more data is being used for training
the models.
In total, three diﬀerent settings were studied. The diﬀerence between them is how much
Page 55 of 66
==PAGE:65==
Meta learning in an online learning setting
data is observed before the models start training in an online learning fashion and how
much is used for pre-training. Also, the total amount of data varies between the settings,
to see the longer term beneﬁts of the meta learning model.
In the ﬁrst setting, a total of 400 hours of data is used and the models are deployed
in a online learning fashion from the ﬁrst observation. Here the Partly trained meta
model is actually the same as the Meta model, since the Partly trained meta model has
not been trained on a subset of the data. Without online learning it was noticed that
the partly trained meta model did not outperform a model that was retrained every 25
hour. This is a result of the amount of data being used, as the retrained model eventually
performs quite well on the problem as this model gradually observes more of the data,
while the other models does not observe any of the data. When the models are trained
in a online learning setting this changes and now the meta learning model shows the
best result. This is encouraging, as this shows that the meta learning model can be used
to give a predictive model a ”head-start” in terms of predicting passenger-demand in a
new city. Especially since the meta learning model is signiﬁcantly better at predicting
passenger-demand initially before a lot of data is observed.
In the other two settings the online learning starts after 50 and 75 hours. This allows one
to capture some data before deploying the models. As an example, this could be used
if the ones deploying the service are not conﬁdent that the model would perform well
without some initial data that the models can learn from. In both of these settings, the
Partly trained meta model outperforms all the other models. In these settings, less data is
also used which means that the beneﬁt of the meta learning model has not been diluted
yet.
The ﬁgures in Section 4.3 also gives some insights into how the meta learning models
are able to achieve a better performance than the regular models. A good example of this
is Figures 4.3.7 & 4.3.8. Here it can be seen that mean prediction of all nodes peak
during the day at around 14.00, while it is the lowest at around 02.00 in the night. In the
setting with online learning (Figure 4.3.8) before the online learning starts, the meta
model underestimates the number of pickups at the middle of the day and overestimates
it at the middle of the night. However at all other times during the day, it is quite good
when comparing it to the Partly trained meta model. When the online learning starts
(indicated by black line), the meta model immediately gets better at predicting the highs
and lows. This shows that the meta model, before training has learnt a good general
model that can predict well on the data that is not in the extremes. This could indicate
that diﬀerences in passenger-demand across cities and transportation methods lies in the
extremes, which is what the meta learning model quickly adapts to when observing data.

Future work
5.7
Future work
The work done within this project opens up multiple paths of research. In some cases,
when meta learning is applied it may not be simple to collect relevant datasets that can
improve the meta learning model. In many cases data is not open-sourced as it is a
valuable asset and sometimes only few relevant datasets has been captured. To overcome
this, augmentations were introduced. For multiple applications, it is possible to augment
datasets in some way. Similar to what was done in this work, when working with time-series
data diﬀerent time resolutions could be used. The results in Section 4.2 did not show
anything that could be used for concluding the beneﬁt of augmenting datasets. However,
a more comprehensive experiment with a fairer evaluation method could indicate if the
augmentations is really useful or not. This could also change based on the application
where it is used.
The model used in all the meta learning experiments was the Edgeconv model. This
model has not been applied to other passenger-demand problems in the literature yet. It
could be interesting to see how it actually performs against state-of-the-art models, as
this model is both more ﬂexible but it also does not include any inductive bias in the
modelling step. This could potentially make it superior to other models/methods where
the graph is created based on some decisions from the user.
For the meta learning model, only the MAML algorithm was considered. It could be
interesting to see how other optimization-based techniques would perform. Also, a ﬁrst-
order approximation was used due to GPU memory limitations. It is very likely that
not restricting the algorithm to use ﬁrst-order approximation could improve the results
further.

CHAPTER 6. CONCLUSION
6
Chapter
Conclusion
In this project, a graph based meta learning model was developed to predict passenger-
demand in cities where data is not available yet. The model was compared against a
transfer learning model trained with the same data. It was found that the meta learning
model outperformed the transfer learning model in many cases and that it is eﬀective
using diﬀerent number of K-shot learning. Even for a Zero-shot learning problem the
model proved to be eﬀective in predicting passenger-demand. The meta learning model
was also applied in an online learning setting using a bike pickup dataset. Here the beneﬁt
of using the meta learning model as an initial deployment model when data was not
collected yet was evaluated. The meta learning model proved to be eﬀective at quickly
adapting to the new setting outperform a model with a random initialization. From this it
can be concluded that using a meta learning model is an eﬀective strategy for creating a
model that can learn from very little data and achieve satisfying results. Additionally, an
augmentation strategy was used to increase the number of tasks that the meta learning
model could adapt to. This was not found to increase the performance of the model, but
it did not have a negative impact either. Therefore, it can not be concluded whether this
is a viable strategy for boosting the performance of a meta learning model.
