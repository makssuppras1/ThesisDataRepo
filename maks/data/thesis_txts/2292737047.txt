Chapter

Abstract
Reliable localisation when given a map of the environment or estimating the map knowing the robots
position have been solved using diﬀerent approaches. But it is a problem when both the map and the
position of the robot is unknown. In this case the robot start in an unknown location and environment
and proceed to build a navigation map of the environment while simultaneously use this map to update is
position. The mapping and localisation is done with the use of a single camera pointed toward the ceiling.
A map of the environment is created using the distinct features found on a ceiling. These features
includes lamps, corner, the material of the ceiling and others. Then a world coordinate system is made
using the position of the robot as the origin point, and the X-, Y- and Z-coordinates of the detected
features are found. The position of the features are found using stereoscopic images and the relation
between them.
Diﬀerent methods are used to calculate the distance and position to ﬁnd the most optimal solution. The
robots position is calculated using the translation of the features between images.

Chapter

Introduction
Reliable localisation is an essential component of any autonomous robot system. The problem of local-
isation in a known area have already been addressed and solved using many diﬀerent approaches. But
it is a problem when both the position of the robot and the surrounding area is unknown.
In this case the robot needs to build and navigate a map of the environment simultaneously while up-
dating its position. This method is usually known as Simultaneous Localization and Mapping (SLAM)
and in this thesis we use a vision based approach of the concept.
A map of the environment can be created using a single camera pointing towards the ceiling, detecting
local features and landmarks, and the calculate the position of the robot. Only using one camera makes
it a cheap way to create a localisation system, but it does add some diﬃculties to the calculation process.
The main problem of this thesis is to create a localisation algorithm using the principles of V-SLAM
with a single camera pointed towards the ceiling. This is done using a robot from The Department of
Electrical Engineering, where it is assumed that the odometry position and angle of the robot is calcu-
lated correctly. After adding a camera to the center of the robot and pointing it towards the ceiling a
prototype to perform test of the algorithm is ready.
Pointing the camera towards the ceiling eliminates some of the problems other vision based camera sys-
tems have. Problems with scaling are almost non existent, because the distance to the ceiling is almost
constant. Other factors like people or objects moving in front of the camera are also highly unlikely.
But there are problems when looking at the ceiling. The diﬀerent types of features on a ceiling are quite
unique, easy to ﬁnd and spread across the entire image. These consist of the material of the ceiling,
lamps, corners, walls, doorways and so on. The downside with these is that there usually aren’t that
many of them. This can create problems with re-detecting known features and placing new features
compared to previously found. A positive feature about them is that they are stationary which makes
them ideal for guiding points for the robot localisation algorithm.
The ﬁrst step in the process is to create a starting point for the robot, by detecting the features the
camera can see from its origin point. The algorithm begins to search for any lamps in the image since
these are easy to detect, but also because they can cause problems when searching for other features.
This is done by converting the image ﬁrst to grayscale and then to binary with a threshold higher than
90% of the grayscale maximum of 255. Doing this will remove most of the image and only keep the
brightest areas, which we assume are either lamps or reﬂected light.
Because of noise in the image a 8-point connectivity algorithm is used to detect regions of interest and
eliminate areas that aren’t usable. Only clusters of pixels with a combined size above 7000 are considered
1
==PAGE:14==
as lamps, and the height and width of the area is used to ﬁnd the center of the cluster. This method
of ﬁnding the center of the lamp should work in most cases, since I assume that all the detected lamps
are either square or round. But there are limitations if the detected lamp are at the edges of the image.
Then the calculated center is shifted from the correct position.
To avoid this, the search area for the 8-point connectivity algorithm is limited to an area, that is half
the size of the most common lamp from the location of the test. This creates an area 72 pixels into the
image from all edges. Even though the search area for the connectivity algorithm is limited, it will still
ﬁnd connected pixels outside this area, which allows it to ﬁnd lamps that have completely entered the
image.
Another method have been tested that are able to ﬁnd circular lamps across the entire image. This
method was not used in the ﬁnal results because of a much higher computational time, uncertain results
and the necessity to be able to detect lamps at all times was not high enough.
After the detecting of the lamps the process continues to locate diﬀerent corners, using the Harris and
Stephen method and a function from the OpenCV library named goodFeaturestoTrack. When looking
at a ceiling diﬀerent corners can be found in many places. For example the transition between ceiling
and walls, the material of the ceiling or around objects on it like lamps, ventilation shafts and so on.
Corners can be deﬁned as the intersection between two lines and can be recognised by looking at the
intensity value in a local window. The OpenCV function ﬁnds the most prominent corners in the image
or in the speciﬁed region of interest and calculates the corner quality measure at every source image
pixel from the cornerHarris function also from OpenCV. The function is set to search for corners in the
image excluding the areas around the previously detected lamps. The reason for this is because of the
high intensity values around the lamps which will be detected as corners, but these are very random and
almost impossible to accurately re-detect. Instead the functions searches for a predetermined number
of corners in the rest of the image, so only the most obvious corners are use. The number of detected
corners are limited to reduce computational time for the algorithm, and to eliminate randomly found
corners that are not certain to be re-detected.
The detected features are then compared to the previously found, and when a feature have been
re-detected more than 3 times, it is considered a constant feature and not one randomly detected. De-
pending on if the distance to the ceiling is known or unknown, the world position is calculated using
diﬀerent methods. If the distance is known a simple conversion between the relationship of the internal
and external features is performed. Knowing the focal length in pixels, the pixel position and the dis-
tance to the feature the X- and Y- coordinates can be calculated.
If the distance is unknown then two diﬀerent methods are used using stereo vision. One method calcu-
lates the distance to the world position using the angles from the optical center to the feature in each
image plane. The X- and Y-coordinates can then be calculated using the previous method or from the
second method.
The other method calculates the intersection point between the two direction vectors from the cameras
Center of Projection (COP) to the features in the image planes. Using the parametric equations to
calculate all three coordinates.
2
==PAGE:15==
When at least two world positions, have been repeatedly found more than 3 times and is in the
same image. The translations of the robot from its origin is calculated. This is done by calculating
the translation between all the world positions found in the current image, and their previously found
coordinates. The result is n2 number of translations, where the most common will be the translation of
the robot.
Using this method gives an accuracy close to 10 mm of the odometry position, but it is dependent on
the accuracy of the results of the world features, which makes it sensitive to errors if the position of the
features are not accurate.

4.2
Homogeneous coordinates
When working with 2D or 3D coordinates, we are usually working with Euclidean geometry. This means
coordinates in X, Y and Z. But when working with camera images and coordinates it can sometime be
more useful to use projective geometry instead.
Projective geometry adds an extra dimension called W in addition to the X, Y and Z dimension, and

This also applies to 3D coordinates where the fourth dimension is the scaling.

4.5
Odometry
Odometry is the use of data from a robot to estimate its change of position over time. It is estimated
with data from the robots wheels or legs and it is quite sensitive to errors.
The robot used for testing is equipped with Ackermann steering. The intention of Ackermann steering
is to avoid the need for tyres to slip sideways when following the path around a curve. The geometrical
solution to this is for all wheels to have their axles arranged as a radii of a circle with a common centre
point.
As the rear wheels are ﬁxed, this centre point must be on this line as well requires that the inside front
wheel is turned, when steering, through a greater angle than the outside wheel.
Rather than the preceding ”turntable” steering, where both front wheels turned around a common pivot,
each wheel gained its own pivot, close to its own hub.
While more complex, this arrangement enhances controllability by avoiding large inputs from road surface
variations being applied to the end of a long lever arm, as well as greatly reducing the fore-and-aft travel
of the steered wheels.
A linkage between these hubs pivots the two wheels together, and by careful arrangement of the linkage
dimensions the Ackermann geometry could be approximated. This was achieved by making the linkage
not a simple parallelogram, but by making the length of the track rod shorter than that of the axle, so
that the steering arms of the hubs appeared to ”toe out”.
As the steering moved, the wheels turned according to Ackermann, with the inner wheel turning further.
If the track rod is place ahead of the axle, it should instead be longer in comparison, this preserving this
same ”toe out”.[2]

Another easily detectable feature on a ceiling are corners. These can appear at the transition from walls
to the ceiling, at square lamps, vents or in the material of the ceiling.
A corner can be deﬁned as the intersection of two line. It can also be deﬁned as a point where there are
two dominant and diﬀerent edge directions in a local neighbourhood of the point.
Most so-called corner detection methods, detect interest points in general, and in fact the term ”corner”
and ”interest point” are used more or less interchangeably.
If only corners are to be detected it is

corners.

5.1
Main Algorithm
The main algorithm for the localisation problem follows a number of steps to function. These steps
include initialization of diﬀerent variables and settings for the various elements used, and in ﬁgure 7.1
in Appendix B a simple ﬂowchart of the system can be seen.
The ﬁrst step is the initialisation of the diﬀerent start variables and ﬂags that will be used for the
program to work. This includes the setting of the cameras parameters, intensity and image parameters
and the initialisation of the odometry calculations. The program runs through the cameras buﬀer twice
to make certain that the ﬁrst image used for the calculations are correct, since the camera needs time
to adjust its parameters and focus.
After these ﬁrst steps, the program begins its main loop that controls when it is supposed to detect
features and run the diﬀerent localisation calculations. The ﬁrst time this loop is called, the program
takes an image of its starting position and detects the found features. Then the odometry position is
updated and the euclidian distance between the current position and the position of the last image taken
is calculated. The distance is then compared to a set threshold, which determines when the next set of
features calculations are to be performed. In the following test the threshold is set to 40 mm.
This threshold can be varied depending on the amount of time the localisation algorithm takes to com-
plete it calculations. If the algorithm is fast, then it is possible to have a lower threshold which will
increase the number of images of the same area which should yield a more precise result. When the
threshold have been reached the program takes the ﬁfth image in the buﬀer to make certain that it is
not the same image as before and begins to detect the features in the image.
The new image is converted into both grayscale and black and white in preparation for the feature
detection functions. A ﬂag on the previously detected features, either from the initialisation or previ-
ously calculations is reset to zero so it is possible to se which have been re-detected and which are new
features. The algorithm then starts to detect if there are any lamps by using the black and white image.
This process is called ﬁrst to eliminate the areas with and around the lamps from the region of interest
for the following corner detection to detect better features.
23
==PAGE:36==
The algorithm then proceeds to detect corners in the grayscale image, where the detected lamp areas
are removed. All of the newly detected features are compared to the previously stored features, and if
a feature is re-detected in either case, then the program stores the new data and check the number of
times it has been detected.
If This value exceeds a pre deﬁned threshold of 3 then we assume that it is an actual feature and not a
random detection. The algorithm then proceeds to calculate the world position of the feature using the
current data, and the previously stored for the same feature.
The same process of checking the number of re-detections is used with the results from the world fea-
tures calculations. If it exceeds its threshold of 3 then the world feature can be used for calculating the
position of the robot.
Using the found world positions, the algorithm compares the position of the currently detected world
features to the previously found features and calculate the translation between the points. This trans-
lation is the distance the robot has moved from its origin point.
At the end of the process, before the algorithm returns to the main loop, features that have not been
re-detected in the last couple of images are removed to reduce the amount of time and calculations the
algorithm takes.

5.3
Lamp detection
The lamps are found by looking at the connectivity of the remaining pixels in the binary image. At
ﬁrst a single pixel is found and it’s position stored. Then an 8-point connectivity algorithm is used to
evaluate the neighbouring pixels, and if they have a value of 255 they are stored along with the ﬁrst
pixel and given a new value so it is not detected again. Then the algorithm is repeated around the newly
found pixels and this continues until there are no neighbouring pixels with a value equal to 255. The
stored data is then evaluated to determine if it is a lamp.
Since the images are not completely free of noise, clusters of pixels with a combined size below 7000
are discarded. This value is quite high and have been chosen because the size of the lamps, which are
around 16 −18.000 pixels. But also because it can eliminate the ring around the lams.
To remove false rings, or other object that are larger than the 7.000 pixels but not a lamp, a box is
created around the maximum and minimum values of the found cluster. The percentage the cluster
represents of the box is found and if the value is below 70% then the cluster is discarded.
If an area is large enough to be considered a lamp further calculations can be applied to ﬁnd the
center of the lamp. Two diﬀerent approaches have been tested to see which give better results.
A simple way of calculating the center of an object in the shape of either a square or a circle is to take
half the height and width of the object as seen in ﬁgure 4.12. This is a simple and fast method though
it does have some problems, especially around the edges of an image.
Another method to ﬁnd the center of circular lamps is by using perpendicular bisectors of lines created
25
==PAGE:38==
from the edge of the found area. By taking three points on the edge of the circle, that are not to close
to each other or have the same x- or y-coordinates. It is possible to create two lines between the points
and then ﬁnd the intersection between their perpendicular bisectors. This point is the location of the
center of the circle, and an example can be seen in ﬁgure 4.13(a) in the theory section.
Repeating this process will create several center points and if their variance is below a speciﬁed threshold,
then the mean of the found intersection points can be used as the center of the circle. If the variance is
to high then it can be determined that the object is not a circle.
The positive eﬀect of the second method is that it can be used at the edges of the image and still give a
correct location for the center of round lamps as seen in ﬁgure 4.14 in the theory section. The negative
side to this method is that it is a lot slower than the ﬁrst, since it takes an average of 0.123 seconds
extra to be performed.
The intersection can be calculated right after the connectivity algorithm, where the needed information
is found, but this method needs several other steps to have the necessary information. This includes an
edge detection algorithm, relocating the newly found edges and then the calculations of the intersection
between the perpendicular bisectors, which takes several iterations.
Also some exceptions to the algorithm have to be taken into account. The combination of certain points
will calculate center points outside the circle or give impossible numbers for the slope values.
Because of this the simple and fast method of creating a square around the found pixel cluster is
used. The information needed is also already gathered through the connectivity algorithm, though cer-
tain precautions have to be taken using this method.
If the lamps are at the edge of the image then this method will give wrong center results. To compensate
for this the search area for ﬁnding connected pixels is limited to begin its search 72 pixels, which is half
the size of a lamp, away from the edges of the image in all four directions.
Though if a pixel is found at the edge of the search area, then the 8-point connectivity algorithm will
still locate connected pixels outside the search area. This means that it is not possible to locate partial
lamps at the edge of the image, but the error of wrong center points will be lowered.
Figure 5.4 shows diﬀerent examples of the lamp detection method using the mentioned approach,
and the black dots marks the calculated center of the lamps. Two lamps can be seen in the left image
and their calculated center points are located in the center of the lamps. In the right image it can be
seen that even though there are two lamps only one is detected and the center of the lamp is shifted

5.4
Corner detection
One of the most common feature on a ceiling are corners, which appear both at the edges between the
ceiling and walls, but also at diﬀerent object like decorations, lamps or the materials of which it is made.
To ﬁnd the corners a pre made function named goodFeaturesToTrack from the OpenCV (Open Source
Computer Vision) library is used. This function detect strong corners in an image using the Harris
corner detection algorithm.
The result of the algorithm can be changed by setting certain parameters, which aﬀect the number,
quality, distance between the found corners and area of interest. Most important is the quality level
which determines the minimum accepted quality of the Harris function response necessary to be used.
This parameter takes the highest found quality measure, multiplies with the minimum quality level and
any value below is discarded.
The maximum number of found corners also aﬀect the following calculations, as there is only a
limited amount of obvious corners in the image. If this parameter is to low there might not be enough
re-detectable features to perform decent localisation calculations, but if the number is to high then there
will be found several non re-detectable corners, which will add time and possible errors to the following
localisation calculations.
It is possible to set the minimum euclidean distance between the found corners. This will remove
corners close to each other so only the strongest is used and this will eliminate problems with re-detecting

Conclusion
Using the provided robot and camera it was possible to create a working prototype of a Vision based-
SLAM system. The algorithm is able to detect diﬀerent types of features on the ceiling, mapping them
and calculate the robots position accordingly. But there are some problems with the rotation of the
robot and re-detection of the features if the angle is to large. There are some precision errors when
calculating the position of the detected features, but it is still possible to calculate the position of the
robot within a couple of centimetres.
I would strongly suggest the use of a diﬀerent camera to this type of experiments. Information about
the hardware and speciﬁcations of the Microsoft Lifecam Studio is scarce and some of the provided
information was wrong. There also seems to be some errors in the construction of the camera which
aﬀects the calculations.
A camera where the user have a higher control of the diﬀerent settings and reliable data about it
speciﬁcations would be more suited.
On the tested ceiling there were only one type of lamp, but this type had a unique feature creating
some problems with the detection. The lamps are spread evenly across the room, which provides a scene
where there are often lamps in the image. These lamps are easy to ﬁnd and the method used to detect
these should work on most types of lamps, though some threshold values should be corrected according
to the size of the lamps. Another method was used to allow detection at the edges of the images. But
due to high computational time it was determined that the need was not high enough, since there were
plenty of other features to provide reliable information. If the ceiling was diﬀerent and the features
scarce, then the implementation of this method would be advised.
The used corner detection function provided good and fast detection of features and the possibility to
adjust the results. The ability to determine the number of detected corners, and only use the strongest,
reduced the computational time for the calculations of the feature positions, and limited the features so
only the most obvious corners was used. Being able to chose the region of interest, removed detections
of random corners near lamps.
The detected corner position was a few pixels oﬀwhen compared to the image, but this can easily be
adjusted. This is to be expected, since the function uses edge detection and the therefore the center will
be shifted.
Looking at images with diﬀerent intensity levels, it can be seen that even though the images are darker
when using the set intensity to detect lamps, it is still possible to detect the most obvious corners. There
can still occur problems if the lightning in the room is to bad or disturbed from the outside.
40
==PAGE:53==
During tests the diﬀerent methods used to determine the position of the features provides fairly
accurate results. The distance calculated using angles from the optical center of stereoscopic images is
within 1 cm of the actual value. And the resulting X- and Y-coordinates of the features is within 5 cm.
The position found using the intersection point between the directional vectors from the images, provides
decent results when calculating the X- and Y-coordinates is the average position error is around 18 cm,
but the distance value is far from correct. The method have a high susceptibility to errors since only a
small variations in the pixel values can give a large diﬀerence at the interception point. Problems also
occurs if the stereoscopic images are to close to each other, since the direction vectors will be very steep
and a small change in x or y- will give a large change in the distance.
Looking at the ﬁnal test result, a decent map of the environment is created and the position of the
robot is found within 1 −3 cm. The calculated feature positions are quite unstable in all directions
compared to previous test. Here the positions in the X- and Y- axis have an average error of 20 −30 cm
and the distance have and average error of 10 −25 cm.
I am quite happy with the precision of the robot, but would have liked to get the error of the world
features within 5 −10 cm of the correct position. There are still some problems with the algorithm and
improvements to both the precision and functionality can be made.
6.1
Future works
To reduce problems and increase accuracy of the measurements, a diﬀerent type of camera could be
used, instead of the Microsoft LifeCam Studio web camera.
The precision of the feature detection and following calculations should be improved with the imple-
mentation of mini pixels. This will change the pixel values from integers to ﬂoating points and allow
more precise values of the detected features. This which will increase the precision of the calculations
for the features world position, by increasing the accuracy of both angles and direction vectors.
The implementation of Singular Value Decomposition (SVD) could provide a method to calculate
the position of the world features and the position of the robot.
Using SVD it is possible to calculate the nearest approach between multiple lines and using this method
in combination with the direction vectors from a feature in subsequent images, might provide a better
position.
The position of the robot could also be found using the direction vectors from all of the currently detected
features to the robot.
