Abstract 
 
This thesis investigates the integration of real process data into a Moldex3D virtual 
model for a precision injection-molded snap-grommet component. Real machine 
curves including injection pressure, screw position, and injection-speed profiles 
were extracted from machine-interface images and used as boundary conditions for 
the filling and packing phases. The objective was to improve the correlation 
between simulation results and experimental measurements and to evaluate the 
influence of packing pressure and packing time on dimensional quantities. 
The results show that the simulation accurately captures the trends in part weight 
and radius, confirming that global shrinkage behavior is well reproduced when 
realistic process inputs are applied. Roundness, however, remains difficult to 
predict, showing an opposite trend to experimental data due to limitations in gate 
modelling, melt compressibility representation, and the use of a simplified single-
cavity geometry. 
A comparison with a previous experimental study1 and a previous simulation work2 
demonstrated consistent agreement on global shrinkage indicators, while 
highlighting persistent challenges in modelling local distortion. The thesis proposes 
several enhancements to improve the fidelity of future virtual models, including 
more accurate gate characterization, refined machine-dynamics representation, 
and the integration of cavity-specific effects. 
Overall, this work contributes to the development of more realistic digital tools for 
process optimization and dimensional-quality prediction in precision injection 
molding.

1  Introduction 
 
Injection molding is a dominant production method for manufacturing polymer 
components due to its ability to produce complex geometries with high throughput 
and repeatability. The process is widely adopted across industries such as 
automotive, medical devices, automotive or aerospace [1]. Yet, maintaining 
consistent part quality requires precise control of various process parameters such 
as injection speed, melt pressure, holding pressure, cooling conditions, and screw 
position because even small deviations can lead to defects such as shrinkage, 
warpage or sink marks [2]. These challenges highlight the strong dependence of 
final product quality on both material behavior and process stability. 
In modern manufacturing, there is increasing interest in digitalization and smart 
monitoring frameworks that integrate real-time and historical process data with 
physics-based models. This approach aligns with the broader paradigm of Industry 
4.0, where cyber-physical systems, sensor networks, and intelligent analytics 
converge to create more flexible and efficient production environments. Within this 
context, the concept of a Digital Twin (a virtual counterpart of the physical process 
that is continuously updated via sensor data and simulation) is being actively 
explored in injection molding. Digital twins enable advanced tasks such as fault 
detection, predictive analytics, virtual experimentation, and adaptive process 
control [3].  
Meanwhile, numerous studies have investigated how combining experiments and 
simulation can enhance process understanding and optimization. Numerical 
simulation tools (e.g., CoreTech Moldex3D®, Autodesk Moldflow®) are increasingly 
used to establish process windows, predict molding outcomes, and evaluate design 
alternatives, thereby reducing costly trial-and-error iterations in production [4]. In 
parallel, efforts have been made to integrate knowledge-based systems and 
machine learning approaches into injection molding process monitoring, with the 
aim of improving decision-making, supporting root-cause analysis, and enabling 
automated quality assurance [5]. 
Building on these developments, this thesis aims to contribute to the digitalization 
of injection molding by integrating image-based data extraction, process 
simulations, and data-driven analysis into a unified framework. This work seeks to 
improve process transparency, enable more robust defect detection, and provide 
actionable insights for adaptive control. The following section (1.1) describes the

1.1 Project background 
 
Injection molding is inherently complex: the process involves multiple interacting 
physical phenomena, such as melt flow, heat transfer, pressure propagation, 
polymer crystallization or solidification, and shrinkage. Over the years, research has 
sought to improve predictions of these phenomena and their influence on part 
quality [1]. 
A well-known limitation in simulation-based process optimization is the “reality gap” 
between idealized model inputs and actual process conditions. Many simulation 
studies assume ideal boundary conditions or simplistic injection profiles, which do 
not capture the real dynamics in a machine. In practice, measured process curves 
(pressure, screw position, cavity pressure vs. time) vary cycle-to-cycle, and 
capturing them is crucial to calibrate and validate simulations [6]. 
In contexts where direct sensor data from machines are limited or unavailable, 
image-based extraction of process curves (e.g. from screenshots, recorded graphs, 
or other visual outputs) offers a workaround. The idea is to trace out injection 
pressure versus time or injection speed versus time curves from images and feed 
them into simulations as boundary profiles thereby making the virtual process more 
representative of the real one. 
The next step is correlation and validation: by comparing simulation outputs (e.g. 
part geometry, predicted pressure, volumetric shrinkage) with experimental 
measurements (e.g. from prior projects such as Pietrobon’s) [7] and the data derived 
from images, one can assess how well the model captures reality. This closes the 
loop between theory, simulation, and experimental data. 
In short: 
• What is needed is to integrate image-extracted process curves. into 
simulation, validate results, and explore data-driven monitoring as a future 
extension. 
• This aligns with industrial and research trends toward smart injection 
molding and digital twins.

1.2 Objective of the project 
 
The aim of this thesis is to investigate and demonstrate methods for digitalizing the 
injection molding process by integrating image-based process data, numerical 
simulations, and experimental results. 
The specific objectives are: 
• Data extraction: To collect and extract process data (such as injection 
speed and pressure profiles) from available visual sources using image 
processing techniques. 
• Simulation integration: To apply these extracted data profiles in Moldex3D 
simulations, thereby enabling more realistic process analyses. 
• Correlation of results: To correlate process conditions, simulation 
outcomes, and experimental measurements (e.g., part dimensions, 
replication quality, or previous measurement results from related projects). 
• Contribution to digitalization: To evaluate how such methods can 
contribute to the broader digitalization of injection molding and the 
development of future quality monitoring strategies. 
By addressing these objectives, the thesis seeks to provide both practical insights 
into current simulation and monitoring challenges, and conceptual contributions 
towards the vision of digitalized manufacturing systems.

2 State of the Art 
 
Injection molding (IM) is one of the most versatile polymer processing technologies 
and remains the dominant method for mass production of plastic products. The 
process involves plasticizing polymer pellets, injecting the melt into a mold cavity 
under pressure, cooling the part to solidify. The ability to produce parts with 
complex geometries at high throughput with precision and relatively low cost makes 
IM indispensable in industries such as automotive, aerospace and consumer 
electronics [1]. 
 
2.1 Historical evolution 
 
Early work in the 19th century involved simple machines used to mold small items 
such as buttons and combs. In the 20th century, advances in polymer chemistry 
introduced materials like polyethylene, polypropylene and polystyrene, which were 
well suited to injection molding and helped the process expand. A key milestone 
came in 1946 with the first automatic injection molding machine, enabling 
automatic plastic injection into molds and greatly improving productivity. Since 
then, injection molding has become widely adopted, with ongoing developments in 
machine power, automation and mold design driven by stricter industrial and 
customer requirements [1]. 
Recent advancements in injection molding emphasize not only improvements in 
machine performance but also the digitalization and networking of the process. 
Concepts such as smart injection molding and Digital Twins integrate machine and 
in-mold sensors, real-time data acquisition and virtual models that mirror the 
physical process, enabling monitoring and control of each stage of the cycle [3], [5], 
[8].  
Knowledge-based and AI-assisted systems further exploit this data for fault 
detection, process optimization and predictive maintenance, thereby increasing 
robustness and reducing variability in production. In summary, injection molding is 
evolving into a highly controlled, data-driven and digitally connected technology 
that fits within Industry 4.0 cyber-physical production systems [3], [8].

2.2 Process phases 
 
The injection molding cycle can be systematically divided into filling, packing, 
holding, cooling, and ejection phases. Scientific molding practice emphasizes 
that these phases must be decoupled and individually optimized to minimize 
variability and ensure repeatability [9]. 
 
Figure 2.1  The phases of the injection molding cycle 
 
Injection phase 
During filling, molten polymer is injected into the cavity, displacing air and 
contacting the cooler mold surfaces. Frozen skin rapidly develops, restricting the 
effective flow channel. The filling speed strongly influences shear heating, viscosity, 
and the position of weld lines. Scientific molding recommends filling the cavity to 
approximately 95–98% of its volume during this phase to prevent overpacking from 
screw inertia or melt compressibility [9]. Insufficient control of this stage can result 
in short shots, jetting, or flow marks. 
Packing phase 
The packing phase begins immediately after filling, compensating for volumetric 
shrinkage as the polymer cools. Additional melt is forced into the cavity to ensure 
that the actual plastic volume equals the theoretical part volume. Optimized 
packing reduces sinks and voids, while inadequate packing causes visible shrinkage 
marks. Excessive packing, on the other hand, induces internal stress and may 
deform thin features [9]. 
Holding phase 
Once the gate begins to freeze, the holding phase maintains pressure to prevent 
==PAGE:20==
M.Sc. Thesis 
 
Page 20 of 160 
 
backflow of material. The transition point between packing and holding is critical, as 
insufficient holding time results in voids, while excessive holding extends cycle time 
and introduces molded-in stresses [9] 
Cooling phase 
Cooling represents the longest portion of the cycle and has a dominant effect on 
productivity and dimensional stability. Heat must be extracted uniformly through 
cooling channels to prevent thermal gradients that cause warpage or differential 
shrinkage. The rate of crystallization in semi-crystalline polymers also depends on 
cooling conditions, influencing part stiffness and dimensional stability. If ejection 
occurs prematurely, the part may warp, crack, or suffer from surface defects [9]. 
Screw recovery phase 
During the cooling stage of the molding cycle, the reciprocating screw begins to 
rotate, preparing material for the next injection. This process is known as the screw 
recovery phase. As the screw turns, it gradually moves backward while feeding 
molten polymer forward toward the screw tip, accumulating the required shot 
volume. The applied back pressure and rotational motion promote uniform melting 
and mixing of the polymer, ensuring consistent melt temperature and material 
homogeneity. Once the target volume is achieved, screw rotation stops, and the 
system remains idle until cooling is complete, after which the next injection cycle 
begins. This phase plays a critical role in maintaining melt quality, reducing cycle 
time, and ensuring process repeatability. [9]. 
Ejection phase 
After sufficient cooling, the mold opens and ejector pins or mechanisms remove the 
part. Poor ejection design can cause scratches, deformation, or sticking, especially 
in thin-walled or highly polished parts. Proper draft angles, surface coatings, and 
balanced ejection forces are essential to avoid damage and reduce cycle 
interruptions [9]. 
In industrial practice, in-cavity pressure and temperature sensors allow each phase 
to be monitored in real time, ensuring gate freeze-off, cooling uniformity, and 
ejection timing are optimized. Furthermore, advanced CAE tools simulate the entire 
cycle virtual optimization before tool manufacturing [4], [6]. 
In summary, understanding and controlling each cycle phase independently is 
fundamental to reducing variability and ensuring dimensional stability. This phase-
oriented approach underpins both scientific molding methodology and the digital

twin paradigm, where simulations and sensor data are synchronized to refine 
process control in real time [3], [8]. 
 
2.3 Process parameters and defects 
 
Injection molding quality is strongly linked to machine inputs, commonly 
summarized as the 11 + 2 key parameters: melt temperature, mold temperature, 
injection speed, injection pressure, packing pressure, packing time, holding 
pressure, holding time, screw rotation speed, back pressure, cooling time, plus 
optionally shot size and transfer position [9]. Each parameter influences different 
aspects of part quality and process robustness. 
 
Figure 2.2  The significant injection molding parameters[9] 
 
A robust injection-molding process is built on controlling a defined set of machine 
input parameters that directly influence melt preparation, cavity filling, shrinkage 
compensation, and part cooling. The process begins with establishing the correct 
barrel temperatures, which determine the thermal condition of the plastic as it 
melts. Although heater bands supply heat, the final melt temperature is also 
strongly influenced by shear heating from screw rotation. Once the molten material 
is prepared, its behavior inside the mold depends on the mold temperature, which 
governs the rate of heat transfer and therefore the cooling profile, crystallinity (for 
semi-crystalline plastics), surface finish, and dimensional stability. 
When the machine begins to inject material, the injection speed controls how 
quickly the screw moves forward to push the melt into the cavity. This speed must 
==PAGE:22==
M.Sc. Thesis 
 
Page 22 of 160 
 
be high enough to ensure that the melt front remains fluid until the cavity is filled. 
The machine automatically applies whatever injection pressure is needed to 
maintain the programmed speed. If viscosity varies, pressure adjusts; accordingly, 
therefore the machine must always have excess available pressure to maintain 
robust control. 
After the cavity is volumetrically filled, the compensation phase begins. The first 
part of this phase is governed by the packing pressure, which pushes additional 
melt into the cavity to offset volumetric shrinkage as the plastic cools. The duration 
of this action, the packing time, must last until the gate starts to freeze. Once the 
gate begins to solidify, the pressure must be reduced to the holding pressure, 
which prevents melt from flowing backward without over-packing the cavity. The 
holding time is maintained until the gate is fully frozen, ensuring dimensional 
consistency while avoiding excessive internal stress. 
Meanwhile, the screw must rotate to prepare the next shot. The screw rotation 
speed determines how quickly, and consistently new melt is generated, while 
balancing shear heating, melt homogeneity, and fiber integrity in filled systems. 
During screw recovery, back pressure is applied to compress the melt, eliminate 
trapped air, improve color mixing, and stabilize screw return, but it must be kept as 
low as possible to avoid unnecessary shear or material degradation. 
Once packing and holding have compensated for shrinkage, cooling becomes the 
dominant phase. The cooling time is the programmed duration for which the mold 
remains closed to bring the part down to ejection temperature. Real cooling time, 
however, actually begins as soon as the melt contacts the mold and includes fill, 
pack, hold, and cooling segments. 
Accurate shot preparation depends on the shot size, which is the screw-back 
distance required to accumulate the correct amount of melt, allowing for a 
deliberate and non-zero cushion at the end of hold. The point at which the machine 
switches from velocity-controlled injection to pressure-controlled compensation is 
the transfer position, typically set by screw position but sometimes by time or 
pressure. 
Two additional settings are sometimes used. Small and infrequently adjusted, 
decompression (suck-back) helps relieve melt pressure after screw recovery, 
preventing drool and improving part cosmetics. In some machines, pack speed can 
also be set to determine how quickly the screw moves during the packing portion of 
the compensation phase, though this feature is not always necessary. 
==PAGE:23==
M.Sc. Thesis 
 
Page 23 of 160 
 
Together, these 13 parameters form the complete scientific-molding input structure. 
When optimized, they establish a process that is robust, consistent, and capable of 
producing parts with minimal variation and predictable dimensional performance 
Scientific molding principles stress that a robust process is one that tolerates 
natural variations (e.g., minor changes in material viscosity) while still producing 
consistent quality. Consistency requires that only common-cause variations remain 
and that their statistical spread (standard deviation) is minimized. DOE studies 
further allow optimization of parameters to maximize robustness and minimize part-
to-part variability. 
Ultimately, while parts may be molded within specification limits even under non-
robust conditions, only a robust and consistent process ensures repeatability, 
efficiency, and long-term reliability of production [9]. 
 
2.4 Injection Molding and Metrology 
 
The growing demand for polymer components with tight dimensional tolerances 
and functional micro-structured surfaces has accelerated research in injection 
molding precision. While conventional injection molding emphasizes productivity 
and cost-efficiency, recent industrial needs focus increasingly on dimensional 
accuracy and process stability. That ensures that micro-features are replicated 
within specification requires a highly stable process supported by frequent 
metrological inspections and advanced process monitoring. [10] 
Metrology plays a fundamental role in validating product quality and process 
outcomes in injection molding, especially as dimensional tolerances continue to 
shrink. Geometrical metrology provides the link of manufacturing parameters with 
dimensional accuracy, enabling the control of both macro- and micro-scale 
features [11].  
In industrial practice, Coordinate Measuring Machines (CMMs) remain among the 
most widely used instruments due to their flexibility, high accuracy in the single-digit 
micrometer range, and ability to measure complex geometries through tactile or 
scanning probing systems.  Pietrobon emphasizes that CMMs are essential for 
measuring workpieces that combine complex surfaces, freeform elements, and 
tight tolerances characteristics typical of precision-molded components. 
Pietrobon’s thesis further demonstrates the centrality of coordinate metrology by 
==PAGE:24==
M.Sc. Thesis 
 
Page 24 of 160 
 
applying a CMM-based measurement framework within a Design of Experiments 
(DOE). The study shows how dimensional changes caused by holding pressure, 
injection speed, cavity number, and their interactions can only be detected reliably 
thanks to high-precision CMM measurements, where the measurement uncertainty 
often approaches the same order of magnitude as the process-induced variation 
itself. This underscores that metrology capability is a limiting factor in high-precision 
PIM investigations [7]. 
 
 
2.5 Digitalization and Simulation in Injection Molding 
 
The increasing complexity of polymer products and demand for faster development 
cycles have positioned Computer-Aided Engineering (CAE) as a cornerstone of 
modern injection molding. Simulation tools not only reduce reliance on costly 
physical trials but also support design optimization, process window identification, 
and defect prevention [1], [4]. 
 
2.5.1 Capabilities of CAE software 
 
Computer-aided engineering (CAE) software has become an essential tool for 
predicting, optimizing, and validating injection molding processes before physical 
trials are conducted. Modern CAE environments integrate multi-physics simulation 
thermal, mechanical, and flow behavior enabling engineers to virtually evaluate 
process performance and part quality with increasing precision. 
Recent studies demonstrate that CAE tools can reliably model the primary molding 
stages, including filling, packing, and cooling, allowing users to analyze how 
process parameters influence shrinkage, residual stresses, and dimensional 
accuracy. O’Leary et al. used Autodesk Moldflow to develop virtual process 
windows for high-impact polystyrene and showed that simulated shrinkage 
predictions could match experimental measurements within 1%, illustrating the 
maturity of CAE models for dimensional forecasting [4]. This capability supports 
robust process-window determination and reduces the need for extensive physical 
trial-and-error.

Similarly, the work of Knoll and Heim demonstrated a strong correlation between 
simulation outputs and experimental observations in terms of flow behavior, 
pressure evolution, and quality metrics such as warpage [6]. Their findings underline 
the reliability of CAE tools for evaluating the effects of material behavior, gate 
design, processing conditions, and mold geometry on the final part. 
Across these studies, key CAE capabilities relevant to injection molding include [4], 
[6]:  
• 
Filling, packing, and cooling simulation: predicting melt front 
advancement, cavity pressure, temperature fields, and solidification 
behavior [6]. 
• 
Dimensional accuracy and shrinkage prediction: enabling early 
assessment of part deformation and performance before tool manufacturing 
[6]. 
• 
Process-window development: using virtual parameter sweeps to identify 
stable operating regions (e.g., combinations of melt temperature, holding 
pressure, and injection speed) without the cost of physical DOE trials [4]. 
• 
Comparison of design alternatives: evaluating gate locations, runner 
layouts, and cooling configurations to reduce cycle time and improve 
uniformity [4], [6]. 
Overall, CAE software acts as a comprehensive virtual test environment, 
significantly accelerating mold development and enhancing the capability to 
achieve dimensional precision in molded components. 
 
2.5.2 Validation through reverse engineering and visualization 
 
A persistent challenge in the use of CAE tools is the “reality gap” between simulated 
and actual machine behavior. Differences in boundary conditions, material data 
accuracy, or tool geometry often result in discrepancies between predicted and 
measured outcomes [6]. Validation is therefore essential to ensure confidence in 
simulation-based decision-making and to establish CAE as a reliable component of 
digitalized workflows.

virtual models and building robust, data-driven monitoring strategies in injection

2.5.3 Process Monitoring, Digital Twins, and Data-Driven Approaches 
 
Effective monitoring of the injection molding (IM) process is central to achieving 
consistent part quality and closing the gap between simulations and real-world 
production. Traditional sensor-based monitoring, and advanced digitalization 
frameworks such as digital twins and machine learning represent complementary 
strategies that are shaping the future of process control [3]. 
 
2.5.3.1 Digital Twin Concept 
 
The Digital Twin (DT) framework represents the convergence of simulation and 
monitoring. A DT is a virtual counterpart of the physical molding process that is 
continuously updated with real machine data [3], [8]. In injection molding, DTs 
typically couple CAE-based models (filling, packing, cooling predictions) with live 
data streams from sensors or image-derived curves. 
DTs allow engineers to: 
• 
Detect deviations between predicted and actual process behavior. 
• 
Perform virtual trials of parameter adjustments before applying them on the 
shop floor. 
• 
Anticipate defects such as sinks, voids, or warpage based on real-time data-
model comparisons. 
The strength of DTs lies in their ability to evolve: they begin as simulation-driven 
models during mold design and progressively incorporate experimental and 
monitoring data during production, thus reducing the simulation–reality gap [3], [6], 
[8].

2.5.3.2 Applications of Digital Twins 
 
Digital twins enable a range of advanced applications in injection molding by 
integrating machine data, simulation outputs, and material information into a 
unified virtual model [3], [5]. This integration enhances analysis and supports data-
driven decision-making throughout the molding cycle. 
• Predictive analytics: 
Machine-learning and data-fusion methods embedded in DT frameworks allow 
forecasting of shrinkage, warpage, and dimensional accuracy, improving early 
detection of quality deviations [3], [5]. 
• Virtual Design of Experiments (vDOE): 
Simulations calibrated with machine and material data enable large parameter 
sweeps to be performed virtually, reducing the number of required physical trials 
and accelerating process optimization [4]. 
• Process optimization: 
DTs link real-time measurements with virtual predictions, allowing faster refinement 
of gate design, packing profiles, and cooling strategies, and improving agreement 
between simulation and experiment [2], [3]. 
• Industrial adoption: 
Recent studies demonstrate increasing implementation of DTs in manufacturing 
environments. Automotive suppliers use DT-driven optimization for dimensional 
stability, while medical manufacturers apply them for traceability and quality 
assurance [2], [3]. 
Overall, digital twins are evolving into practical tools that enhance accuracy, reduce 
trial-and-error, and support highly digitalized injection-molding workflows.

In many industrial and laboratory environments, historical or machine-recorded

3.6.3 Grid Detection (Code1) 
 
3.6.3.1 Overview 
 
The Python script Code10.py is designed to automatically detect vertical and 
horizontal dashed or solid lines within an image of a chart, such as a machine 
output graph. It uses the OpenCV and NumPy libraries to process the image, apply 
thresholding, and identify line patterns based on the continuity and segmentation of 
dark pixels. The script is structured into several functions, each performing a 
specific role in the detection process, and includes visualization capabilities to 
display the detected lines over the original image. 
The code begins by importing the necessary modules, including cv2 for image 
processing, NumPy for numerical operations, and typing for type annotations. A 
default image path is defined to serve as a demonstration input when the script is 
executed without command-line arguments. Two optional helper functions, 
is_rgb_match() and get_color_name(), are also included. The first function 
compares two RGB color tuples to determine whether they are similar within a 
specified tolerance, while the second function maps certain RGB color values to 
descriptive color names. These functions are retained for completeness, even 
though they are not directly used in the main detection procedures. 
 
3.6.3.2 Continuous Vertical Lines 
 
The first, detect_first_vertical_lines(), identifies continuous solid vertical lines within 
the image. The function reads the image, converts it to grayscale, and applies binary 
thresholding so that dark regions (potential lines) appear as white pixels and the 
background becomes black. It then scans through the columns of the binary image, 
typically starting from one-quarter of the image width, to search for columns that 
contain a long uninterrupted run of white pixels. If a column has a continuous 
==PAGE:42==

vertical streak of at least 300 pixels, it is classified as a vertical line. Optionally, the 
detected columns can be displayed visually as red vertical lines over the processed       
binary image using OpenCV’s imshow() function.

3.6.3.3 Vertical Dashed Lines 
 
The second function, detect_vertical_dashed_lines(), targets dashed vertical grid 
lines. After converting the image to grayscale and applying a fixed threshold, the 
algorithm examines each column to count the total number of white pixels and the 
number of distinct white segments separated by gaps. Columns that contain both a 
sufficient total proportion of white pixels and a high number of discrete segments 
are interpreted as vertically dashed lines.

vertical overlays are displayed both on the binary image and the original image. 
 
3.6.3.4 Horizontal Dashed Lines 
 
The third function, detect_horizontal_dashed_lines(), performs a similar analysis for 
horizontal dashed grid lines. It scans each row of the thresholded binary image, 
counting white pixels and short white segments corresponding to dashes. Rows 
meeting the defined thresholds for density and segmentation are classified as 
dashed lines. The function then computes the vertical distances between these 
lines and filters out small spacings that are likely to be noise. Optionally, the 
detected horizontal lines are drawn in green on both the binary and original images.

3.6.4 Legend Localization and Color Box Detection (Code2)

The Python script Code2 extends the image analysis framework developed in Code1

optical character recognition (OCR), and compute the horizontal scale factor of the 
plot, expressed as the ratio between physical x-axis units and image pixels. This 
enables the automatic retrieval of quantitative information from machine-generated 
plots, supporting subsequent computational or statistical analysis.

3.6.4.4 Detection of Legend Boxes

3.6.5.7 Determination of Grid and Axis Parameters 
 
The detection of grid and axis parameters is a crucial step in transforming the 
graphical data from the plot into numerical form. After retrieving the structural grid 
information from Code1.py and the legend data from Code2, Code3 computes both 
horizontal (x-axis) and vertical (y-axis) calibration factors that link pixel coordinates 
to real-world units. 
The set of horizontal dashed lines detected in the image represents the regular 
divisions of the y-axis. These lines are first filtered to remove faulty detections, such 
as upper titles or headers, which could interfere with calibration. The first and last 
valid horizontal dashed lines define the upper and lower boundaries of the active 
plotting region, corresponding respectively to the maximum and minimum 
displayed values along the y-axis. The vertical distance between these two grid 
lines, measured in pixels, represents the axis height in pixel space. 
To obtain the y-axis calibration factor, the script uses the numerical values extracted 
from the colored legend boxes located beneath the plot (using Code2). Each legend 
box contains two numeric values that represent the range of that parameter—for 
example, the minimum and maximum values of Injection pressure or Screw stroke 
within the graph. By sorting these two numbers, the lower value (v_bot) is assigned 
to the bottom of the axis and the upper value (v_top) to the top. 
The difference between these two values, (v_top – v_bot), represents the total 
physical range of that parameter displayed in the plot. Dividing this difference by the 
vertical axis height in pixels yields the y-axis unit-per-pixel ratio:

3.10 DOE for simulation  
 
To complement the experimental campaign, two simulation studies were 
conducted using Moldex3D in parallel with the physical Design of Experiments 
(DOE). The aim of these simulations was to replicate the molding conditions tested 
experimentally and to gain deeper insight into the effects of key process parameters 
on part quality and performance. This approach allowed for a direct comparison 
==PAGE:63==
M.Sc. Thesis 
 
Page 63 of 160 
 
between simulation outputs and experimental results, enabling both the validation 
and potential refinement of the numerical model. 
The first simulation campaign focused on the variation of packing pressure, with 
simulations performed at 500 bar and 600 bar while maintaining a constant holding 
time of 6 seconds. The second simulation campaign investigated the influence of 
packing time, ranging from 1 to 4 seconds, with the packing pressure fixed at 400 
bar. These setups were designed to analyze how changes in packing pressure and 
packing time influence the filling behavior, pressure distribution, weight and 
dimensional accuracy of the molded component. 
For each simulation run, Moldex3D automatically generated a comprehensive set of 
output data, including the, injection pressure and speed evolution curves for filling 
and packing phase, and the predicted weight. Additionally, the shrinkage and the 
warpage deformation fields were extracted to assess the dimensional stability of the 
part after cooling.  
The combination of simulations based on both nominal input data (from Renan’s 
work) and actual input data (from this study), together with the experimental DOEs, 
provides a robust framework for understanding the injection molding process. By 
comparing the simulation results with experimental findings, it becomes possible to 
evaluate the accuracy of the Moldex3D model and to identify potential 
discrepancies related to boundary conditions, material parameters, or processing 
assumptions. 
The following chapter presents and discuss the results obtained from these two 
simulation campaigns. Emphasis is placed on the influence of packing pressure and 
packing time on dimensional quality (Rmax and roundness) and part weight. 
Through this analysis, the predictive capabilities of the simulation tool are 
assessed, and its integration within the overall thesis workflow is demonstrated.

4.1 Image-Based Process Data Extraction Results 
 
This section presents the results obtained from applying the image-based data 
extraction pipeline described in Section 3.6. The purpose of these results is not to 
visualize the aggregated experimental curves (which are shown later in Section 4.2 
as part of the 3.7 methodology outcomes), but rather to evaluate the performance, 
accuracy, and robustness of the developed computer-vision pipeline itself. The 
results therefore focus on the intermediate digitalization steps like grid detection, 
legend and color-box recognition, OCR accuracy, axis calibration, curve extraction 
fidelity, and overall robustness across all experimental images. 
 
4.1.1 Grid Detection Accuracy 
 
The grid-detection module (Code1) successfully identified both horizontal and 
vertical dashed grid lines in all processed screenshots. For all images included in 
the dataset, continuous vertical boundaries and dashed grid lines were detected 
with no missed detections. 
The median horizontal spacing between dashed grid lines was consistent across all 
images, with a variability of less than ±0.7 pixels, confirming that the algorithm 
reliably detected the underlying coordinate system. Similarly, the vertical dashed 
line spacing showed a maximum deviation of ±0.9 pixels, demonstrating strong 
structural consistency. 
These results confirm that the grid-detection framework is sufficiently robust to 
support next operations such as axis calibration and accurate pixel-to-unit 
conversion.

4.1.2 Legend Localization and OCR Accuracy 
 
Legend localization was evaluated by comparing the detected color boxes against 
their ground-truth positions. The detection algorithm correctly identified all legend 
boxes corresponding to the three measured signals (injection pressure, screw 
stroke, and injection speed). In most cases, the detection error in the bounding-box 
position was less than ±2 pixels, which is negligible relative to the size of the legend 
region. 
The OCR system was validated by comparing the numerical values extracted from 
the legend boxes with the values printed on the machine screen. Across all images: 
• 
100% of the legend boxes were correctly interpreted with perfect OCR output 
• 
No images failed to produce usable OCR outputs 
 
4.1.3 Axis Calibration and Pixel-to-Unit Conversion 
 
Axis calibration was performed using the pixel spacing of the vertical dashed grid 
lines. For each image, the median spacing was compared against the expected 
spacing derived from the machine plot’s numerical axis labels. 
Across the entire dataset: 
• 
The pixel-to-time conversion factor differed from the expected value by less 
than 0.1 % on average 
• 
No images showed deviations larger than ±0.2 % 
This accuracy level ensures that the reconstructed time axis preserves the correct 
cycle duration and timing of machine events such as pressure peaks, screw 
movements, and velocity changes.

4.1.4 Curve Extraction Fidelity 
 
The fidelity of the curve extraction module (Code3) was evaluated using methods 
that do not require overlaying the extracted curves onto the original plots in pixel 
coordinates, since the machine-generated plots are visually scaled and cannot be 
directly compared on a point-by-point basis. 
Instead, validation was performed using internal consistency checks, shape-
preservation analysis, and time-alignment verification across multiple images 
belonging to the same experimental condition. These checks included: 
Color-separation accuracy 
• 
The extraction algorithm correctly identified and separated the three process 
curves based on LAB color thresholds. 
• 
100% of curves were correctly assigned to the corresponding parameter 
(injection speed, screw stroke, or injection pressure) 
• 
No color-mixing misclassifications were observed 
Time-axis alignment verification 
• 
Since all extracted curves include timestamps reconstructed from pixel-to-
unit conversion: 
• 
All curves belonging to the same run showed identical time ranges. 
• 
This confirms that the time scaling was applied consistently and accurately 
 
Detection of characteristic events 
Key machine events such as: 
• 
the start of filling 
• 
the peak injection speed 
• 
the drop in injection pressure during packing 
• 
the screw’s backward motion during recovery 
were consistently detected at the same time locations. This demonstrates that 
the extracted curves preserve the essential physical behavior of the molding 
cycle and indicates that the extraction is valid.

4.3.2 Explanation of the Early Pressure Drop in the 1 s and 2 s Packing-
Time Experiments 
 
In the experiments performed with packing pressures of 400 bar and packing times 
of 1 s and 2 s, the injection-pressure curves exhibit a pronounced pressure drop at 
approximately 1.3 s. At first this appears contradictory, since the machine was 
programmed to maintain the packing pressure of 1 s and 2 s respectively. However, 
examination of the corresponding injection-speed curves, together with typical 
injection-molding machine control logic, reveals why this behavior occurs. 
Although the controller nominally allows the user to specify a packing time, this 
parameter is not the sole condition determining the end of the holding stage. In 
most hydraulic injection molding machines, the transition from the holding phase to 
the next cycle phase (decompression and screw recovery) is triggered by whichever 
==PAGE:91==
M.Sc. Thesis 
 
Page 91 of 160 
 
of several conditions occurs first. These commonly include: the programmed 
packing time, a screw-position limit, a recovery-start command, or a cycle-
sequence timer. As a result, the effective duration of packing is determined by the 
earliest of these internal triggers, not necessarily the user-specified packing time 
alone. 
This interaction is clearly reflected in the recorded data. In both the 1 s and 2 s 
packing-time experiments, the pressure reaches the expected packing plateau of 
approximately 400 bars shortly after the velocity-to-pressure (V/P) switchover. The 
pressure remains stable at this level only until about 1.3 s, after which the hydraulic 
pressure collapses to nearly zero. At precisely the same time, the injection-speed 
signal shows a large negative spike, indicating a brief reverse movement of the 
screw. This combination of signals is characteristic of the machine entering the 
decompression (suck-back) phase. Immediately following decompression, the 
hydraulic pressure rises again to a higher plateau (around 520–540 bar), which 
corresponds not to packing pressure but to the backpressure used during screw 
recovery (plasticizing) for the next cycle. 
The critical observation is that this same transition occurs at the same moment 
(around 1.3 s) in both the 1 s and 2 s experiments. This demonstrates that in these 
cases the end of the packing stage is not governed by the packing-time setting but 
instead by another machine parameter that initiates plasticizing after a fixed delay 
or at a specific screw position. Consequently, even though the nominal packing 
time was set to 2 s, the actual holding pressure is maintained only until the internal 
recovery-start condition is reached, causing the early termination of the packing 
phase and the appearance of the sudden pressure drop. 
This explains why such a drop is absent in the experiments with 3 s and 4 s packing 
times. In those longer-packing-time tests, the internal recovery-start condition 
occurs later than the 2.9 s window shown in the plots, so the machine remains in 
the holding phase throughout the measured interval, resulting in a smooth and 
uninterrupted pressure plateau. Thus, the early pressure drop in the 1 s and 2 s 
experiments is not due to instability in the packing pressure itself but is simply the 
consequence of the machine’s control logic terminating the holding phase 
prematurely due to an overriding timing or position-based condition.

4.3.3 Explanation of differences in injection pressure profiles between 
the experiments 
 
A comparison across different packing pressures and packing times reveals 
systematic changes in the average process behavior. Although all experiments 
exhibit nearly identical injection-speed–versus-time curves, the injection-pressure 
profiles differ significantly between the test conditions. This difference arises from 
how the machine transitions between the filling (velocity-controlled) stage and the 
packing/holding (pressure-controlled) stage. 
During filling, the screw follows the same programmed velocity profile in every 
experiment; therefore, the speed curves appear similar. As the polymer melt flows 
into the cavity, flow resistance increases, causing the pressure to rise to a peak just 
before the velocity-to-pressure (V/P) switchover. Immediately after switchover, the 
screw decelerates and the pressure naturally drops. 
 
The behavior after this point depends primarily on the selected packing pressure: 
1. Experiments with high packing pressures (500 bar and 600 bar, 6 s): 
In these tests, the packing-pressure setpoint is relatively high compared to 
the pressure immediately after switchover. Consequently, once the machine 
enters the pressure-control stage, the controller attempts to raise the cavity 
pressure back up toward the setpoint. This results in a characteristic 
pressure profile showing: 
 
initial peak → drop at switchover → secondary rise (controller correction) 
→ gradual decline → plateau at the packing pressure 
 
The second peak occurs because the system actively increases pressure to 
achieve the high packing setpoint. 
 
2. Experiments with lower packing pressure (400 bar, 1 s, 2 s, 3 s and 4 s): 
In these tests, the packing-pressure setpoint is significantly lower than the 
pressure level immediately after the filling peak. Therefore, after switchover, 
==PAGE:93==
M.Sc. Thesis 
 
Page 93 of 160 
 
the pressure simply relaxes downward until it reaches the packing-pressure 
level, without requiring the controller to raise it again. The resulting profile is: 
 
initial peak → drop at switchover → smooth descent to a stable plateau 
 
No secondary rise appears because the system does not need to add 
additional pressure because its target is lower than the natural post-
switchover pressure. 
 
In summary, the differences in the pressure-versus-time curves are not caused by 
changes in the injection speed profile but by the relative magnitude of the packing-
pressure setpoint. High packing pressures require the machine to re-pressurize the 
melt after switchover, producing a secondary rise in the curve, whereas lower 
packing pressures result in a simple decay toward the setpoint.

In contrast, in the real injection molding machine, the screw continues to advance 
several millimeters beyond the theoretical “cavity full” point. The extra forward 
motion is caused by a combination of physical effects that are not reproduced in the 
simulation: 
 
1. Polymer melt compressibility: 
Under high injection pressure (500–700 bar), the melt behaves as a 
compressible fluid. The screw must advance further to compress the melt to 
the required packing pressure. This alone can account for several millimeters 
of screw displacement. 
 
2. Hydraulic system inertia and response lag: 
Hydraulic injection units cannot stop instantaneously. The screw 
deceleration occurs with a delay due to oil compressibility, valve dynamics, 
and mass inertia of the moving components. This leads to “over-travel,” 
where the screw moves slightly further even after the target speed or stroke is 
reached. 
 
3. Hydraulic intensification ratio: 
Because the machine uses a hydraulic cylinder, the force transmitted to the 
screw is amplified. As pressure rises sharply at the end of filling, this 
intensification contributes to additional melt compression and further screw 
travel. 
 
4. Machine control resolution: 
The physical machine cannot detect the exact cavity-full moment as 
precisely as the simulation. It reacts only after pressure feedback changes, 
introducing a short delay during which the screw continues to move forward. 
 
Combined, these effects explain why the experimental curve includes more screw 
displacement after the point where Moldex3D stops the simulation. In other words, 
the extra 9 mm corresponds exactly to the real physical compression and hydraulic 
delays that occur during the transition from filling to packing, while the simulation 
terminates filling ideally at the moment the cavity becomes full. The discrepancy 
does not indicate an error but rather reflects the mechanical and hydraulic 
characteristics of real injection molding systems.

naturally appear higher than the simulated values.

simulation manages the transition from the velocity-controlled filling phase to the 
pressure-controlled packing phase. In Moldex3D, this switch is triggered 
algorithmically based on cavity fill percentage, without replicating the smooth 
hydraulic response, damping, and pressure stabilization that occur in the real 
machine. Consequently, the simulation often produces oscillations or multiple 
pressure “bumps” as the solver converges toward the prescribed packing pressure.

slightly. The experimental curve again exhibits a dominant pressure peak followed 
by a small fluctuation before reaching its plateau. However, the corresponding 
simulation curve (red line) shows only one pressure bump, followed by a continuous 
increase toward the plateau, rather than the two bumps observed in the 500 bar 
case. This suggests that at the higher packing-pressure target, the simulation 
attempts to reach and stabilize the pressure level more rapidly. In the 500 bar case, 
the solver overshoots and re-adjusts, producing a second pressure bump to

maintain the required pressure. By contrast, in the 600 bar simulation, the solver 
drives the pressure upward more steadily in order to meet the higher target plateau, 
resulting in a single dominant bump rather than two. 
In summary, the number of pressure bumps before reaching the packing plateau 
differs between simulation and experiment due to the fundamental differences in 
how physical and numerical systems handle the dynamic transition between filling 
and packing. The simulation’s sharper and more discrete stage transition often 
leads to additional pressure oscillations that are not present, or are less 
pronounced, in the smoother real-world machine response. 
Overall, despite the expected differences arising from the hydraulic dynamics, 
compression effects, and Moldexe3D’s numerical treatment of filling-to-packing 
transition, the simulation successfully reproduces the characteristic form of the 
experimental response. The injection speed versus stroke agreement confirms the 
correctness of the boundary conditions, while the injection speed versus time and 
injection pressure versus time comparisons demonstrate that the Moldex3D model

4.6 Comparison of the results with Pietrobon and Mazza work  
 
The results obtained in this thesis can be positioned within the broader context of 
the two earlier studies on the same snap-grommet component: the experimental 
benchmark established by Pietrobon [7] and the simulation-driven analysis 
performed by Renan [12]. Although each study employed a different methodological 
perspective (Pietrobon relying exclusively on physical experiments and Renan using 
a simplified reverse-engineered Moldex3D model) the comparison reveals several 
consistent tendencies as well as important deviations. 
 
4.6.1 Integrated comparison of experiments and simulations 
 
The results obtained in this thesis can be positioned within the broader framework 
of two previous works on the same snap-grommet: the experimental benchmark by 
Pietrobon [7] and the simulation-based study by Renan [15]. Together, these three 
“assignments” provide a coherent picture of how packing pressure, packing time 
and machine-barrel modelling affect global shrinkage (weight and Rmax) and local 
distortion (roundness). 
 
 
==PAGE:126==
M.Sc. Thesis 
 
Page 126 of 160 
 
Weight and Rmax 
From an experimental perspective, Pietrobon showed that both increasing holding 
pressure and increasing holding time lead to higher part weight and larger 
diameters. Weight and a representative diameter were found to increase 
monotonically with packing intensity. 
Renan/Mazza’s simulations reproduced the same qualitative behavior: as packing 
pressure and packing time were increased, both predicted weight and Rmax grew, 
reflecting enhanced material compression and better compensation of volumetric 
shrinkage. This confirmed that the reverse-engineered Moldex3D model could 
correctly capture the macroscopic response of the component under stronger 
packing conditions. 
The present thesis, which uses real machine-measured profiles as simulation 
inputs, confirms and refines this picture. For both DOEs, the Moldex3D simulations 
with and without the barrel predict increasing weight and Rmax with higher packing 
pressure and longer packing time, in line with the experimental trends.  
In addition to these trend agreements, the absolute values of the predicted weight 
are also informative. When the barrel is included, the weight predictions obtained in 
this work lie slightly closer to Pietrobon’s experimental measurements than the 
corresponding results reported in Renan simulations, indicating an improvement in 
the weight prediction. For Rmax, the simulated magnitudes in this thesis and in 
Renan work are very similar, with both models predicting global dimensions that are 
comparably close to the experimental ones. Overall, all three studies agree that 
weight and Rmax are robustly predicted by the virtual models, provided that the 
main packing conditions are correctly represented. 
Roundness 
Experimentally, Pietrobon observed that increasing packing pressure reduces 
roundness (i.e. improves circularity) up to an optimal level, after which further 
pressure increases lead to a plateau in roundness. For holding time, roundness 
decreases with increasing packing time in one direction, while it increases in the 
other. Additionally, Pietrobon showed that cavity number has an influence on 
roundness comparable to that of packing pressure, revealing significant cavity-to-
cavity variation and underlining the strong sensitivity of this metric to local mold and 
process conditions. 
==PAGE:127==
M.Sc. Thesis 
 
Page 127 of 160 
 
Renan’s simulations, in contrast, reported an overall increase in roundness with 
both packing pressure and time, opposite to the experimental tendency of 
roundness decreasing (improving) with stronger packing.  
The present simulations show intermediate and more nuanced behavior: 
• 
Packing pressure 
For both “with barrel” and “without barrel” cases, roundness decreases with 
increasing packing pressure. This qualitatively aligns with the experimental 
observation that higher pressure makes the part more circular and mitigates 
circumferential distortions, even though the detailed shape of the curve and 
the saturation point do not fully coincide with Pietrobon’s measurements. 
• 
Packing time 
For holding time, the two machine-barrel models diverge: 
o Without barrel, roundness generally increases with longer packing 
time, suggesting that prolonged packing amplifies differential 
shrinkage when melt compressibility and realistic pressure damping 
are not taken into account. 
o With barrel, roundness decreases for one dimension and shows a 
milder, partially opposite trend for the other, better reflecting the 
direction-dependent behavior observed experimentally, where one 
diameter improves while another deteriorates as holding time 
increases. 
From a quantitative perspective, the magnitude of the roundness values predicted 
in this work for the “with barrel” simulations is close to those obtained in Renan’s 
study. In other words, while the detailed trends with packing pressure and time 
differ between the two simulation works and from the experiment, the numerical 
level of roundness is similar, but there is a discrepancy between experimental 
magnitude results. 
These mixed results highlight that roundness is much more sensitive than weight or 
Rmax to modelling assumptions: gate geometry, local cooling conditions, cavity-
specific effects and the representation of melt compressibility all play a critical role. 
The comparison may also be influenced by the fact that Pietrobon reported trends 
only for the first cavity, even though cavity number has a significant effect on the 
results. 
 
==PAGE:128==
M.Sc. Thesis 
 
Page 128 of 160 
 
4.6.2 Overall perspective 
 
The comparison shows that weight and Rmax are consistently and accurately 
reproduced by the virtual model, whereas local roundness remains more 
challenging to predict. This is consistent with previous findings and highlights the 
importance of: 
• direct measurement of gate and runner geometries 
• representation of cavity-specific thermal and flow effects 
• improved modelling of machine dynamics, including a realistic 
representation of the barrel 
• detailed thermal calibration across the mold, such as accurate modelling of 
the cooling channels 
Because roundness is governed by localized, direction-dependent mechanisms 
such as differential cooling, cavity imbalance and gate-freeze dynamics, it is 
inherently more sensitive to reverse-engineering assumptions than weight or Rmax. 
Nevertheless, all of these improvements are expected to enhance the overall 
predictive capability of the simulations, leading to more realistic results not only for 
roundness but also for global indicators such as weight and Rmax. 
Notably, the simulation with the barrel model agrees more closely with 
Pietrobon’s measurements, as it also exhibits opposite roundness trends for the 
two measured diameters when holding time increases. This suggests that including 
melt compressibility and realistic pressure decay at the nozzle is essential for 
reproducing the directional shrinkage behavior observed experimentally.

5 Conclusions and future work 
 
5.1 Conclusions  
 
This thesis set out to explore how real machine data can be integrated into injection 
molding simulations to improve the accuracy of virtual predictions. The specific 
research objectives were addressed as follows: 
1. Data Extraction: 
The goal of extracting process data from visual sources was successfully achieved. 
A computer vision workflow was developed to automatically retrieve injection 
speed, screw position, and injection pressure profiles from machine screenshots. 
The pipeline demonstrated robustness across all image qualities in the dataset, with 
no need for manual correction. The extracted curves were accurate in both shape 
and scale, requiring only minimal preprocessing (e.g., smoothing or scaling 
adjustments) before being used as direct input for simulations. 
2. Simulation Integration: 
The extracted profiles were effectively integrated into Moldex3D simulations as 
boundary conditions, replacing default machine settings with real process data. For 
example, the injection-speed profile was mapped into a polyline stroke–speed 
input, and the injection-pressure profile was used to validate the simulated cavity-
pressure evolution. This integration enabled the simulations to reflect real machine 
behavior, especially when the barrel model was included, thus enhancing the 
physical accuracy of the filling phase. 
3. Correlation of Results: 
The simulation outcomes showed strong agreement with experimental 
measurements for part weight and Rmax better than previous simulation with 
nominal data inputs. Roundness, however, remained the most challenging 
parameter to predict accurately. The simulations reproduced the direction-
dependent behavior observed experimentally namely, that one roundness 
dimension increases with packing time while the other decreases particularly when 
the machine-barrel model was activated. This agreement indicates that modeling 
melt compressibility and pressure decay at the nozzle is essential for capturing 
asymmetric and locally driven shrinkage effects. Nevertheless, quantitative 
discrepancies persist, primarily due to limitations in gate-geometry and barrel 
==PAGE:130==
M.Sc. Thesis 
 
Page 130 of 160 
 
reconstruction, simplified single-cavity modeling, and the absence of cavity-
specific thermal interactions. Since roundness is governed by highly localized 
mechanisms, even small inaccuracies in gate size or local cooling representation 
can significantly affect predictions. 
The use of real machine curves represents a significant advancement over earlier 
virtual models and reduces several sources of uncertainty related to boundary 
conditions. At the same time, it highlights the need for further refinement, including 
direct measurement of gate and runner dimensions, improved modeling of machine 
dynamics, and more detailed thermal calibration across the mold. 
 
5. Contribution to Digitalization: 
This work contributes to the digitalization of injection molding by bridging the gap 
between physical machine behavior and virtual simulations. The methodology 
supports the development of accurate digital twins and paves the way for 
simulation-assisted quality control. By enabling data-driven tuning of simulation 
inputs, it reduces uncertainty and improves the predictive value of simulation-
based process analysis. 
Overall, this thesis demonstrates that integrating real process data into Moldex3D 
constitutes a meaningful step toward more accurate digital representations of the 
injection-molding process. Combined with enhanced geometry characterization 
and more detailed simulation modelling, this approach provides a clear path for 
increasing the fidelity and predictive capability of virtual twins. By strengthening the 
connection between measured process behavior and simulated outcomes, the 
work contributes to the broader development of data-driven, simulation-assisted 
process monitoring and control in precision injection molding.

5.2 Future work 
 
5.2.1 Proposed enhancements to the computer-vision extraction 
framework 
 
This thesis showed that carefully engineered classical computer-vision techniques 
can accurately reconstruct quantitative process data from rasterized machine plots. 
The combination of run-length–based grid detection, CIELAB ΔE-based color 
segmentation, and OCR-driven automatic calibration provided a transparent, 
deterministic, and fully automated workflow for converting visual process 
documentation into numerical datasets. 
To further expand robustness, precision, and usability, the following extensions are 
recommended: 
Recommended improvements: 
• 
Automatic legend color calibration: Dynamically estimate curve and legend 
colors from each image to account for camera variation, screen brightness, 
or compression artifacts. 
• 
Adaptive thresholding and noise handling: Replace fixed thresholds with 
adaptive schemes (e.g., Sauvola or Otsu-based methods) to better handle 
varying lighting, low contrast, or degraded images. 
• 
Subpixel refinement and smoothing: Implement spline fitting or Kalman-
based filtering to produce smoother trajectories and increase temporal 
resolution, especially for rapidly varying signals. 
• 
Multi-curve separation in overlapping regions: Extend the current LAB 
distance approach with local shape priors to better distinguish overlapping 
curves in dense plot segments. 
• 
Automated batch-processing interface: Enable direct ingestion of folders of 
machine screenshots to fully eliminate remaining manual steps, making the 
system suitable for continuous or real-time data archiving. 
 
==PAGE:132==
M.Sc. Thesis 
 
Page 132 of 160 
 
5.2.2 Proposed enhancements to the Moldex3D virtual model 
 
The results of this thesis highlight several modelling improvements that would 
further increase the fidelity and predictive capability of the virtual environment. 
While the integration of real machine curves improved filling and packing realism, 
certain physical mechanisms remain difficult to reproduce with the current 
simplified model. 
Recommended improvements to geometry and mesh 
• 
Direct measurement of gate and runner geometry: Use mold CMM 
measurements or high-resolution 3D scanning rather than relying on gate 
vestiges, which tend to overestimate real gate size and distort freeze-off 
behavior. 
• 
Multi-cavity representation: Pietrobon’s measurements showed strong 
cavity-to-cavity variation. Modelling all six cavities or introducing cavity-
specific tolerances would allow the simulation to reflect the real dimensional 
scatter. 
Recommended improvements to machine behavior modelling 
• 
Enhanced screw-dynamics modelling: Incorporate machine-specific 
acceleration, hydraulic lag, and pressure-feedback effects. Although real 
injection-speed curves were used, Moldex3D currently simplifies early filling 
response, causing time shifts relative to the machine.

def _normalize_text(s: str) -> str: 
    s = (s or "") 
    s = s.replace('–', '-').replace('—', '-').replace('−', '-') 
    s = s.replace(',', '.') 
    s = re.sub(r'(\d)\s*\.\s*(\d)', r'\1.\2', s) 
    s = re.sub(r'(?<!\S)([+-])\s*(\d)', r'\1\2', s) 
    s = re.sub(r'(?<!\S)([+-])\s*\.\s*(\d)', r'\g<1>0.\2', s) 
    s = re.sub(r'(?<!\d)\.\s*(\d)', r'0.\1', s) 
    s = re.sub(r'[ \t]+', ' ', s) 
    return s 
 
 
def extract_two_floats_from_box_simple(bgr_crop) -> List[float]: 
    """OCR two numbers from a small legend box crop (BGR image).""" 
    pad = 4 
    crop = cv2.copyMakeBorder( 
        bgr_crop, pad, pad, pad, pad, 
        cv2.BORDER_CONSTANT, value=(255, 255, 255) 
    ) 
    gray = cv2.cvtColor(crop, cv2.COLOR_BGR2GRAY) 
    _, bw = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY + 
cv2.THRESH_OTSU) 
    big = cv2.resize(bw, None, fx=2.0, fy=2.0, 
interpolation=cv2.INTER_CUBIC) 
    config = r'--oem 3 --psm 6 -c tessedit_char_whitelist=0123456789.+-,' 
    raw1 = pytesseract.image_to_string(big, config=config) 
    inv  = cv2.bitwise_not(big) 
    raw2 = pytesseract.image_to_string(inv, config=config) 
    text = _normalize_text(raw1 + "\n" + raw2) 
    matches = re.findall(r'[+-]?(?:\d+\.\d+|\d+\.\d*|\.\d+|\d+)', text) 
    nums = [] 
    for s in matches: 
        if s.endswith('.'): 
            s += '0' 
        if s.startswith('-.') or s.startswith('+.'): 
            s = s[0] + '0' + s[1:] 
        try: 
            nums.append(float(s)) 
        except ValueError: 
            pass 
    return nums[:2]

def _robust_spacing_from_positions(pos: List[int]) -> Optional[float]: 
    """Median grid spacing from sorted positions with light IQR 
filtering.""" 
    if not pos or len(pos) < 2: 
        return None 
    diffs = np.diff(np.sort(np.asarray(pos, dtype=np.int32))) 
    if diffs.size == 0: 
        return None 
    q1, q3 = np.percentile(diffs, [25, 75]) 
    iqr = q3 - q1 
    lo, hi = q1 - 1.5 * iqr, q3 + 1.5 * iqr 
    keep = diffs[(diffs >= lo) & (diffs <= hi)] 
    use = keep if keep.size else diffs 
    return float(np.median(use))

def _robust_spacing_from_positions(pos): 
    """ 
    Median grid spacing from sorted line positions with light IQR filtering. 
    Returns None if < 2 positions. 
    """ 
    if not pos or len(pos) < 2: 
        return None 
    diffs = np.diff(np.sort(np.asarray(pos, dtype=np.int32))) 
    if diffs.size == 0: 
        return None 
    q1, q3 = np.percentile(diffs, [25, 75]) 
    iqr = q3 - q1 
    lo, hi = q1 - 1.5 * iqr, q3 + 1.5 * iqr 
    keep = diffs[(diffs >= lo) & (diffs <= hi)] 
    use = keep if keep.size else diffs 
    return float(np.median(use))
