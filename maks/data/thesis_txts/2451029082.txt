Summary
The art of fitting a curve to noisy data have always been of interest to scientist and
engineers alike. A natural condition is that the curve fits the data while its rough-
ness remain modest such that the curve represent the underlying process and not the
(rough) noise. As such the smoothing spline regression problem became a huge stud-
ied subject, as its solution is a tradeoff between not fitting the data and the roughness
of the curve.
There exist several efficient (linear time, linear storage) algorithms for calculating
the solution to the smoothing spline regression problem dating back to the 1960s
[REI67]. Most known methods are specifically engineered to efficiently handle the
structure of the various sparse matrices encountered when calculating the solution
the smoothing spline regression problem.
The method described in this thesis takes the approach of [AC19] in which it is
shown that the so-called spline kernel matrix is an extended generator representable
𝑝-semiseparable matrix. Using this the structure of the a diagonal plus kernel ma-
trix is shown to be an extended generator representable 𝑝-quasiseparable matrix. As
described in [AC19] the two separable matrix structures gives rise to efficient (linear
time, linear storage) linear algebra routines, resulting in algorithms for the smoothing
spline regression problem that that scales linear in time and storage.
Some advantages of this new approach is that it generalizes easily to higher order
splines as well as being elegant from an implementation point of view, as it is based
solely on two structured matrices. At the same time the new approach can be used to
show that the smoothing spline regression problem gives rise to scalable algorithms
for Gaussian processes.
During the completion of this thesis the derived algorithms were implemented in
the programming language Julia ([Bez+17]), as its capabilities of multiple dispatch as
well as operator overloading could be used to create both efficient as well as readable
code. The time complexity as well as memory usage of the algorithms were tested for
various sizes of data (𝑛) and order of splines (𝑝), in both one and two dimensions.

CHAPTER 1
Introduction
Fitting a curve through noisy data, while easily done by hand, is no trivial task to
model mathematically. As such there have been developed many methods that seeks
to solve such problem. The approach in this thesis is that of the smoothing spline,
whose objective is to minimize the total 𝑝th order derivative, while still approximating
the data. While the formulation of the problem at first glance might seem innocent, it
will become apparent that this is not the case, if the one aims to understand the theo-
retical foundations used when deriving the solution to the problem. Furthermore one
needs to be careful when computing the solution as a naive approach will require the
inversion of a dense matrix. The aim of this thesis is to briefly explain the theoretical
foundation of the smoothing spline regression problem as well as developing efficient
algorithms for calculating its solution. As a result of scope of this thesis is broad.
Firstly we need to explain the theoretical foundation and connect it to the smoothing
spline regression problem. Secondly we need to investigate the structure of extended
generator representable 𝑝-semiseparable (and 𝑝-quasiseparable) matrices and explain
how their structure can be used to efficiently (linear time, linear storage) calculate
the solution to the smoothing spline regression problem. Furthermore we need to
be able to efficiently evaluate the generalized likelihood function and its derivatives,
as this is used when setting the hyperparameters included in the model. Last but
not least a possible method for extending the earlier results from one-dimension to
two-dimensions is proposed. We now expand on the structure of this thesis.
Structure of the thesis
• The Smoothing Spline: We begin by characterizing the smoothing spline re-
gression problem followed by a description of the theoretical foundations needed
to solve the problem. Afterwards it is shown that the key (kernel) matrix when
calculating the solution to the smoothing spline regression problem is highly
structured and can therefore be stored in 𝑂(𝑝𝑛) memory.
• Semiseparable Matrices: In this chapter we further show that the structure
of the (kernel) matrix gives rise to efficient (linear time, linear storage) linear
algebra routines. Specifically algorithms for multiplication, Cholesky factoriza-
tion, linear systems solving as well as various traces is introduced.

• Gaussian Processes: This chapter seeks to connect the results from the two
earlier chapters. At the same time this chapter explains how the Gaussian pro-

• Tensor Product Splines: From the one-dimensional results it is natural to

2.2
Reproducing kernel Hilbert spaces
In this section a introduction to (real) reproducing kernel Hilbert spaces (RKHSs) is
given. Intuitively one can think of RKHSs as Hilbert spaces with restrictions such
that they become well behaved relative to other Hilbert spaces. We start by defining
the evaluation functional, which is essential to the definition of the RKHS.
Definition 2.1 (Evaluation functionals). Let 𝑉be a Hilbert space of real functions,
𝑓∶𝑋→ℝ. For a fixed 𝑥∈𝑋the map 𝛿𝑥(𝑓) = 𝑓(𝑥) is called the evaluation functional
at 𝑥.
As we will see later the Riesz representation theorem will become a key ingredient
when describing the solution to the smoothing spline regression problem. The proof
of the theorem can be seen in [Chr10, p. 70].
Theorem 2.1 (Riesz representation). In a Hilbert space 𝑉, all bounded (continuous)
linear functionals are of the form ⟨⋅, 𝑔⟩𝑉for some 𝑔∈𝑉.
We remind the reader that boundedness and continuity is equivalent in normed
spaces [Kre89, p. 104], as indicated above. We are now ready to define the reproduc-
ing kernel Hilbert space as in [MA14].
Definition 2.2 (Reproducing kernel Hilbert space). Let 𝑋by an arbitrary set and
denote by ℝ𝑋the vector space of all functions 𝑓∶𝑋→ℝequipped with pointwise
operations.
A subspace 𝑉⊂ℝ𝑋endowed with an inner product is a reproducing
kernel Hilbert space (RKHS) if 𝑉is complete and, for every 𝑥∈𝑋, the evaluation
functional 𝛿𝑥(𝑓) = 𝑓(𝑥) is bounded (continuous).
An interesting fact is that in the definition of a RKHS, the kernel itself is not
mentioned, but as we shall see is something that is a result of the above definition.
The kernel is defined as follows.
Definition 2.3 (The kernel). If 𝑉⊂ℝ𝑋is a RKHS then its kernel is the function
𝒦∶𝑋× 𝑋→ℝsatisfying ⟨𝑓, 𝒦(⋅, 𝑦)⟩= 𝑓(𝑦) for all 𝑓∈𝑉and 𝑦∈𝑋. Here 𝒦(⋅, 𝑦)
denotes the function 𝑔(𝑥) = 𝒦(𝑥, 𝑦) and is an element of 𝑉.
Theorem 2.2 (Uniqueness of the reproducing kernel). Given that the reproducing
kernel exist, then it is unique.
Proof. Assume that 𝑉have two reproducing kernels 𝒦1 and 𝒦2. Then ∀𝑓∈
𝑉, ∀𝑥∈𝑋we have that
⟨𝑓, 𝒦1(⋅, 𝑥) −𝒦2(⋅, 𝑥)⟩= ⟨𝑓, 𝒦1(⋅, 𝑥)⟩−⟨𝑓, 𝒦2(⋅, 𝑥)⟩= 𝑓(𝑥) −𝑓(𝑥) = 0.

CHAPTER 4
Gaussian Processes
In this chapter a short introduction to Gaussian processes is given. While Gaussian
processes can be used for both regression and classification, the introduction will
only cover that of regression. The chapter is largely based on [RW06], however as it
at times is missing key derivations also [Van+15] were consulted. A problem with
Gaussian processes is that due to matrix inversions they are typically not suitable
for large datasets. However as we will see in section 4.4, the fact that the spline
kernel is semiseparable enables the Gaussian processes representing the smoothing
spline regression problem to be scaled efficiently with respect to both computation
and memory usage.
4.1
Function-space view

Gaussian distribution to an infinite set of variables. The result is a distribution over
functions instead of points. Formally it can be stated as follows:
Definition 4.1. A Gaussian process is a collection of random variables, any finite
number of which have a joint Gaussian distribution.

CHAPTER 6
Implementation
The design of the implementation with respect to the extended generator repre-
sentable 𝑝-semiseparable (and 𝑝-quasiseparable) matrices is similar to that of the
AbstractOperators.jl package described in [Ant+18]. A big thanks to one of the
developers of said package, Niccolò Antonello, for his input and help in the initial
stage of the development. For the actual spline fitting using Gaussian processes in-
spiration was taken from the package GaussianProcesses.jl, which is described in
[Fai+18]. The following sections explain how the different structured matrices were
implemented in Julia and how they can be used to efficiently solve spline fitting
problems in one and two dimensions.
Remark.
Given that the structured matrices that we look at represent kernel
matrices, they are considered only in the symmetric case.

CHAPTER 8
Discussion
In this thesis algorithms for efficiently calculating the solution to the smoothing spline
regression problem based on the results found in [AC19] is explained in detail. All
algorithms are based on the structure of semi- and quasiseparable matrices which
have two useful properties: Firstly they can be stored in 𝑂(𝑝𝑛) memory and secondly
their structure can be used to perform all relevant linear algebra computations in
𝑂(𝑝2𝑛) operations. The two matrix structures were implemented in Julia. The im-
plementation was then used to numerically verify the expected memory usage as well
as run time for all relevant linear algebra routines. Afterwards the smoothing spline
regression problem was set into the context of Gaussian processes, which proofed use-
ful in two ways: It gave a natural way to set relevant hyperparameters through the
generalized likelihood function while also giving a way to quantify the uncertainty
at the predictions. In the end the implementation was used for its original purpose:
Calculating the solution to the smoothing spline regression problem.
While several of the provided linear-time algorithms can be seen as great improve-
ments to their standard counterparts, they all suffer from the same basic fact, namely
that they are recursive in nature. From a implementation point of view the recursive
nature is attractive as the resulting algorithms are simple and can be implemented
using very few lines of code. However from a computational point of view the recur-
siveness is not particular useful, as the resulting algorithms will become fully serial
and not suitable for modern-day parallel computing. While multiplication with the
semiseparable matrices can easily be parallelized using smaller blocks, similar things
can not be said about the Cholesky factorization. An idea to handle this issue is
to look into symmetric hierarchically semiseparable matrices, for which parallel algo-
rithms exist [Wan+16].
Setting the computational aspect aside the provided algorithms have another im-
portant, if not the most important, property that they scale linear in memory while
still representing matrix computations.
This is very crucial as even dealing with
relatively small data sizes would require large amount of storage.
For example if
𝑛= 40, 000 then storing the 40, 000× 40, 000 kernel matrix would have required close
to 12GB of RAM, hence would be problematic to handle for most modern laptops.
As suggested in the conclusion of [AC19] it was natural to look at possible gener-
==PAGE:82==
72

alizations to higher dimensions through the so-called tensor-product spline. The case
was limited to the two-dimensional case for which the data needs to be restricted to
a rectilinear grid. Given this restriction the data will naturally give rise to a struc-
tured kernel matrix equal to the Kronecker product of two (semiseparable) kernel
matrices. Ignoring the semiseparable structure the findings in [Ste+12] were used to
implement algorithms which where efficient in memory and reasonably efficient com-
putationally. However as stated previously it is believed that the structure of semi-
and quasiseparable matrices can be utilized to further speed up the algorithms such
that they become comparable to the one-dimensional algorithms in both memory and
computation. The reason for this belief is simply that there are algorithms stated in
[Kei11, p. 103], which have the desired run time and memory usage, that utilizes the
semi- and quasiseparable structure. Due the limited time frame of this project said
algorithms were not implemented.

CHAPTER 9
Conclusion
In this project the smoothing spline regression problem in 1-dimension was thoroughly
studied. In order to do so the concept of reproducing kernel Hilbert spaces were intro-
duced and used to sketch how the solution to the smoothing spline regression problem
can be computed. Furthermore the result from [AC19] that states that the kernel ma-
trix generated by the spline kernel is a so-called extended generator representable
𝑝-semiseparable matrix was reproduced. This result, while seeming insignificant, was
then shown to enable us to store the kernel matrix in 𝑂(𝑛𝑝) memory.
By exploring the structure of the extended generator representable 𝑝-semiseparable
(and 𝑝-quasiseparable) matrices it was found that the structure gave rise to efficient
(linear time, linear storage) algorithms for all linear algebra computations relevant
to the smoothing spline regression problem. Specifically we reproduced some results
from [AC19] such as the fact that the Cholesky factorization and its inverse have a
structure similar to the extended generator representable 𝑝-semiseparable matrices.
These results was then shown to be the cornerstone when efficiently calculate various
traces and determinants in linear time and storage.
Using the spline kernel as the kernel for a Gaussian process we were able to
show that the predictive mean of the Gaussian process was equivalent with the con-
vex quadratic program that needs to be solved when calculating the solution to the
smoothing spline regression problem. At the same time we saw how the Gaussian
process view of the smoothing spline regression problem could be used to set the
two hyperparameters included in the model. Specifically we saw how the extended
generator representable 𝑝-semiseparable structure of the kernel matrix could be used
to efficiently evaluate the generalized likelihood function and its derivatives. For the
actual optimization the package Optim was used [MR18].
Following this we took a top-down approach to the 2-dimensional spline fitting
problem by creating a kernel on the 2-dimensional space with a structure that could
be utilized to perform efficient calculations. While exploring the extensions to the
2-dimensional spline regression problem, it became apparent that the structure of ex-
tended generator representable 𝑝-semiseparable (𝑝-quasiseparable) matrices was not
enough to develop efficient algorithms. Instead we took the approach described in
[Ste+12], which only used the Kronecker structure of the 2-dimensional kernel. The

Afterwards we showed the implementation could be used to easily fit splines in one
