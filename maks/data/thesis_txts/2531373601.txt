Abstract
Autonomous vehicles and robots rely on various environment perception solutions in
order to navigate. The laser scanner, also known as LIDAR (Light Detection and Rang-
ing), is one such technology. In this project a 2-D laser scanner was used. It measures
distance and direction to points in a plane around the robot. From this information
objects can be recognized and in turn used for navigation. Before entering a zone where
known objects cannot be detected, it is important to have an accurate initial estimate of
the pose of the robot. This extends the range the robot can drive, using only odometry.
This study investigates whether the initial estimate of the pose of the robot can
be improved. This involves reducing the random error by averaging a series of single
measurements, which in turn improves precision. Reducing the systematic error improves
accuracy, which is performed by calibrating the laser scanner, based on a relationship
between the measured intensity and the error. A correction algorithm based on 3-D
interpolation was evaluated. From a series of measured points lying on a line (a straight
object), the pose of the robot can be found by linear regression. It was investigated
if the best estimate could be found by averaging distance measurements before linear
regression, or if the estimate improved by doing linear regression first, and afterwards
averaging the results of the linear regression. An experimental set-up was created where
the orientation of the laser scanner and the test object could be controlled, and the
distance between the two could be set and measured. Reference objects with different
grey levels were prepared and characterised with respect to reflection.
A useful experimental set-up consisting of targets and data logging software was
created. This allowed measurements to be performed. The only drawback of the experi-
mental set-up was that measurements were performed manually, which, compared to an
automated system, was time-consuming and less accurate. This limited the quality of
the measurements and the ability to test various cases quickly. The attempt to reduce
the systematic error by calibration was unsuccessful. This could have been caused by an
insufficient number of reference points, limited quality of the measurements and a rela-
tionship between the intensity and the error which was difficult to model. On the other
hand, it was shown that the random error of the measurement could be reduced signifi-
cantly by averaging a number of single measurements. Finally, a simulation showed that
the best estimate of the pose of the robot could be found by averaging measurements
first, then performing linear regression. In summary, the initial estimate of the pose of
the robot could be improved by averaging a series of single measurements, and by using
the correct method for linear regression.

CHAPTER 1
Introduction
1.1
Background
Autonomous vehicles and robots are in rapid development, and automotive and industrial
customers are very interested in improving environment perception solutions. The laser
scanner, also known as LIDAR (Light Detection and Ranging) is one technology used
for environmental perception.
Figure 1.1: GoPal pallet robot [6].
Laser scanners are used on mobile robots for navigation and mapping. An example
is the GoPal pallet robot, developed and manufactured by Robotize A/S (Figure 1.1).
The 2-D laser scanner measures distance and direction to points in a plane around the
robot. By comparing detected objects with a map, the pose (position and direction) of
the robot can be determined. On the other hand, if the pose of the robot is already
known, the surroundings can be mapped by registering the objects found.
It can be challenging for robots to navigate in surroundings where objects are not
static, for example in a factory where materials and products are are frequently moved
around. If they are placed in front of static objects with well-known positions, like walls,
pillars etc., then the static objects cannot be detected, and cannot help in the navigation.
In this situation, it is important to know the pose of the robot as exactly as possible
before entering an area without fixed objects. The better the start pose is known, the
farther the robot can drive using only odometry, without fixed objects for navigation.
The start heading of the robot is especially important, because position error increases
with the distance driven.
==PAGE:14==
2
1 Introduction
To determine the initial pose of the robot, it will be placed in an area where a
known object can be detected. It could be a straight object one meter long. From this
reference object the pose of the robot is determined. If it is possible to calibrate the
laser scanner so distance is known with better accuracy, then the pose of the robot can
also be determined more accurately. The heading of the robot is especially important.
1.2
Problem Formulation
The main question to be investigated is if it is possible to improve the initial estimate
of the pose of the robot (position and heading). In this project, a 2-D laser scanner will
be used to sense the environment.
The distance measurements from the laser scanner are assumed to have a systematic
and a random error. The systematic error is sought to be minimised by calibration. It
is assumed that the error of the measured distance depends on the amount of reflected
light from the scanned objects. According to Hokuyo, the manufacturer of the laser
scanner used in this project, it should be possible to calibrate the systematic error based
on the measured intensity [9]. The random error is sought be minimised by averaging
a number of measurements. It will be investigated how many single measurements are
needed to obtain a given precision.
The pose of the robot in relation to a straight object can be found by linear regression.
It will be investigated if the best estimate is found by averaging distance measurements
before linear regression, or if the estimate becomes more accurate by doing linear regres-
sion first and afterwards averaging the results of the linear regression.
To be able to perform measurements with the laser scanner, a test set-up will be
created.
The orientation of the laser scanner and the test object, and the distance
between the two, can be set and measured.
Reference objects with different grey levels, resulting in different reflectivity, will be
prepared. Measurements will be conducted to characterise the reflection of targets from
different angles. This will be done with a source emitting light at the same wavelength
as the laser used in the scanner.

Basic LIDAR functionality
Light detection and ranging (LIDAR), also called laser detection and ranging (LADAR)
is a sensing method that detects objects and maps their distances. It works by illumi-
nating the object with light and measuring the reflected return signal.
The common working principle of the lidar is based on measuring the time of flight
of light, from when it is emitted by sensor, then reflected by the object, then received by
the sensor. The light travels twice the distance to the object. When the speed of light
c = 2.998 × 108 m
s and the measured time delay is ∆t, then the distance from the sensor
to the object s is found as

general principles and the challenges in the design of a specific lidar.

a transmitted and received light pulse, see Figure 2.1. All lidars rely on a measure of

of systems is to detect the correct arrival time of the received pulse. Since the received
pulse varies in amplitude, it is not always well defined when it starts.
Figure 2.2: Wave from a AMCW lidar [1].
Amplitude modulated continous wave (AMCW) lidars measure the phase shift be-
tween the emitted wave and the reflected wave, see Figure 2.2.
Figure 2.3: Signal from a FMCW lidar [1].
With frequency modulated continous wave (FMCW) lidars, the transmitted wave is
modulated by a range of frequencies varying linearly by time (Figure 2.3). By mixing the
transmitted and received signals a difference frequency is generated which is proportional
to the distance.
When light from the laser scanner arrives at an opaque surface, it is reflected in both
a diffuse and a specular manner. The distance estimate is mainly based on the diffuse
component. It is suggested that the reflectivity of many materials, such as paper and
matte paint, can be approximated by Lambert’s cosine law, [7] cited in [2]. It states
that the amount of reflected light is proportional to the cosine of the angle between the
incident light and the surface normal. A Lambertian surface appears equally bright from
all viewing angles.

2.2
Error, Accuracy and Precision
In [14], the terms error, accuracy and precision are defined as follows:
The error is defined as the difference between the measured value and the true value.
Different sources of errors exist. Systematic errors can in theory be modelled, and are
therefore to some degree predictable. Random errors are not predictable, and can only
be described in a stochastic manner.
Accuracy is a measure of how well the measured value conforms with the true value.
Precision describes the closeness of the measured values. The precision does not say
anything about how accurate the measurements are. It is possible to measure the same
wrong value many times, which results in high precision but low accuracy.

3.1
Measurement Set-up Configuration
The purpose of the measurement set-up (Figure 3.1) is to make it possible to set and
measure the orientation of the scanner and the target, and also set and measure the
distance between the two. Due to space limitations of the room, all distance measure-
ments in this project are limited to 3 m. The measurement set-up is designed with the
specifications of the laser scanner in Table 3.1 in mind, but it is also intended to be a
general set-up which can be used to perform measurements with other laser scanners.

marked with a center and an angular scale. In this way the same measurement positions
and orientations can easily be repeated. In order to avoid multipath reflections, a stand
which elevates the laser scanner is prepared. In this way the laser spot is kept free of
the surface of the table. The center of the laser spot is aimed to approximately half the

When the targets are observed with the naked eye, the targets appear black, dark
grey, light grey and white, which indicate different reflectivity. It is also possible to get
an idea of how the light is reflected from the target whether it is diffuse or specular.
A Lambertian surface will exhibit diffuse reflection and appear equally bright when
observed from any angle. A non-Lambertian surface will exhibit diffuse reflection, with
a specular component. The extreme of non-Lambertian is a mirror, which only exhibits
specular reflection.
The laser diode in the laser scanner operates at a wavelength of 905 nm, which is in
the infrared spectrum. Infrared light is not visible to the naked eye, so it is necessary to
take other actions in order to characterise the targets.

+45° relative to the surface normal. The target is fixed to a rotational stand, which
makes it possible to control the angle between the surface normal and the axis to the
photo detector. The target is rotated within a range of -90° to +15°.

5.2
Performance of Correction Algorithm
The attempt to improve the systematic error by calibration was not successful. It is
not acceptable for the output of the calibration routine to have a bigger error than the
input.
Several factors could have had an impact on how well the correction algorithm per-
formed. The algorithm used was based on a lookup table and linear 3-D interpolation.
An alternative could have been a polynomial approach, where a curve is fitted to the
==PAGE:54==
42
5 Calibration
intensity-error data combined with 2-D interpolation.
The success of either method
depends on how well the interpolated values correspond with the real world.
The relationship between the intensity and the error is not a smooth curve, which is
very clear from Figure 4.6. When the value of the intensity is around 1000, the matter
becomes complicated. It is difficult to handle this situation with a look-up table, when
the data points are closely scattered in this way. It could be a better solution to fit a
polynomial to find a way through the scattered data points.
In order to avoid the scattered data around an intensity of 1000, the reference mea-
surements were only made with targets at an angle of 0°. It is very likely that the
reference points were simply not enough. Both the resolution of the intensity and also
the physical distance between the reference points in the plane could be insufficient.
It has not been investigated how the angle of the target affects the measured intensity
and the distance error. It was shown that the intensity and the error indeed varied with
the target angle. But it is not clear how the received pulse was changed by the target
angle. Due to the extension of the laser spot, an increased target angle will cause light
from one side of the laser spot to arrive earlier at the receiver, and light from the other
side to arrive later. In this way the received pulse is stretched in time, and might be
detected differently.
The internal data processing of the laser scanner is unknown to the author. Internal
corrections could counteract the external correction.
The quality of the measurements has been an issue throughout the project. It is
demanding to perform the measurements manually. It is very time-consuming, and the
needed accuracy in the placement and alignment of the laser scanner and the target
is very challenging. Since this is manual work, the accuracy is limited and collecting
enough measurements is not possible.
The repeatability of the measurements could influence the performance of the cor-
rection algorithm. This has not been investigated. A question related to this is whether
the sampling of distance and intensity happens at the same time on every rotation - is
the distance sampled in the same direction every time? Is the angle between the rays
evenly spaced? This will affect the accuracy of the distance measurement, if it doesn’t
happen in the expected direction.

CHAPTER 6
Future Work
A lot of time and work has been invested in conducting measurements. Despite the
effort, it was a big challenge to acquire enough data, and to maintain a high accuracy
of the measurements. It would be a great advantage to have an automated system that
could adjust the orientation of both the laser scanner and the target with high accuracy
(0.01°), and acquire measurement data. In this way a measurement series of an angular
sweep of both the laser scanner and the target could be recorded automatically.
In addition, it would be an advantage to have the ability to set the distance between
the laser scanner and the target. When both angle and distance can be set automatically,
measurements from a large amount of reference points in an area can be recorded. This
would give a high spatial resolution.
The data logging software should be extended with the ability to control the me-
chanical settings of the system. In this way, the operator only needs to configure the
measurement and press ’GO’.
In addition it could be considered to add a temperature logging device, so the ambient
temperature and the DUT temperature could be recorded.
With such a system, the path from an idea of a measurement to actually having the
acquired data would be much shorter.
Apart from generating high resolution spatial data for the calibration algorithm,
other subjects could be investigated too:
• Are the rays spaced equally with respect to angle?
• What is the angular variation of a singular ray?
• What is the exact angle of a specific ray?
• What is the temperature sensitivity of the laser scanner?
• What is the repeatability of distance measurements?

CHAPTER 7
Conclusion
The main purpose of this project was to investigate if it is possible to improve the
initial estimate of the pose of a robot. A 2-D laser scanner was used to perceive the
environment.
A test set-up was created which included targets and data logging software. The
set-up proved to be useful in the sense that measurements could be performed. The
data logging software was able to both capture measurement data and also record the
configuration of the set-up. In addition, the software provided a function that helped in
aligning the laser scanner by looking at a single ray. The only drawback of the test set-up
was that measurements were performed manually. Performing measurements manually
was very time-consuming and the accuracy of the orientation of the laser scanner was
limited, compared to an automated solution. It turned out to be much more laborious
than expected to perform measurements. The result was reduced ability to test various
cases. An automated solution with the ability to set the orientation of both the laser
scanner and the target precisely would have been preferable, but this was outside the
scope of the project.
Targets in four grey levels were prepared, and measurements were conducted in order
to characterise them. It was shown how the light surfaces were close to being Lambertian,
while the darker surfaces also contained a significant specular component. Surfaces with
a larger specular component reflect less light back to the laser scanner than Lambertian
surfaces, when they are not placed perpendicular to the measurement axis. This in turn
increases the measurement error. When the characterisation was done at DTU Risø, it
was not possible to measure the reflectivity of the targets, in terms of the percentage of
light reflected from the targets. This could have been used to evaluate whether the laser
scanner performed as stated in the data sheet, with respect to minimum reflectivity.
It was shown that the random error of the measurement could be reduced significantly
by averaging a number of samples. Averaging for a longer period gives better precision.
There is a trade-off between precision and averaging time.
An attempt was made to reduce the systematic error of the measurement by cali-
bration. The distance error was recorded as a function of intensity in reference points
on the scanned plane. A correction algorithm based on 3-D interpolation was evaluated
in Matlab. Unfortunately the algorithm did not improve the accuracy of the distance
measurement. Several factors could have had an impact on the performance of the algo-
rithm. The number of reference points could have been too small, resulting in a limited
==PAGE:58==
46
7 Conclusion
resolution. The quality of the measurements could have been reduced, in the sense that
the orientation of the laser scanner could have been inaccurate. The relationship be-
tween the intensity and the error did not follow a smooth curve. This is a challenge.
The repeatability of the measurements was not investigated, which could have helped
clarify to which degree a systematic error was really systematic.
When the pose of the robot was found by linear regression, it was simulated to de-
termine whether the best estimate could be found by averaging distance measurements
before linear regression, or if the estimate would be better when calculating linear re-
gression first and afterwards averaging the results of the linear regression. The best
estimate was found by averaging the measurements first, and then calculating the linear
regression. This is also the computationally least demanding procedure.
In summary, the three factors influencing the quality of the estimate of the initial pose
are: Accuracy, precision, and the method of linear regression. The attempt to improve
the accuracy by calibration of the systematic error was not successful. In contrast it was
shown that the precision could indeed be improved by reducing the random error. By
simulation it was also verified which method of linear regression should be applied, in
order not to reduce the accuracy.

The recorded data is stored as comma-separated values (csv) in an ASCII text file.
Storing numbers as ASCII text is not very efficient with respect to file size, but has the
advantage of being easy to code, human-readable to help with debugging, and capable
of being imported by many programs including Matlab.
The CSV file format is not fully standardized; one format is specified in RFC4180
[13]. The implementation used in UamLog follows the format described in RFC4180,
with the exception that the beginning of the file contains a list of variable names and
corresponding values on the form Variablename=Value (Table A.1).

which is followed by data records. The number of fields in the header and in all the
data records is the same. The field separator is ',' (comma), the text qualifier is '"'
(quotation mark) and the decimal separator is '.' (period). All lines end with CR + LF
(0x0D, 0x0A) characters. An example of a data file is seen in Figure A.4.

20

31 % Make sure that theta and rho are row vectors
32 theta = reshape(theta, [1,length(theta)]);
33 rho = reshape(rho, [1,length(rho)]);
34
35 % Calculate cartesian coordinates
36 x = rho .* cos(theta);
37 y = rho .* sin(theta);
38
39 n = length(x);
40
41 Sx = sum(x);
42 Sy = sum(y);
43
44 Sxx = x * x';
45 Sxy = x * y';
46 Syy = y * y';
47
48 alpha = 1/2 * atan2(2*sum(x)*sum(y) - 2*n*Sxy, Sx^2 - Sy^2 - n*Sxx + ...
49
n*Syy);
50
