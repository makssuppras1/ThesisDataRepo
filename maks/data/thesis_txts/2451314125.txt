ABSTRACT
1
Abstract
The project deals with a new type of number system, unums type III in posit mode (’posits’). The
project investigates whether posit numbers can outperform ﬂoating-point numbers, in terms of area
and speed in a digital system, which implements an application which requires computations with
high precision numbers.
A possible application of posits numbers is the training of neural networks. This process includes
large number of additions, multiplications and activation functions. The activation function used in
this project is the sigmoid function, as posits can compute a cheap approximation of this function.
Therefore, this project focuses on the measurements and benchmarking of the area and speed of
posit, and the ﬂoat implementations in those operations.
The project presents the posit RTL implementations of addition, multiplication and sigmoid function,
together with a downloaded existing posit RTL implementation of the same operations and uses
them to compare against ﬂoating-point implementations.
Our measurements found that for the sigmoid function, posits are 3200 % faster and 8400 % smaller
than ﬂoats, due to the build-in approximation of the sigmoid function by the unums. For addition
the area is 176 % larger for posits than for ﬂoats, while speed-wise posits are 377 % faster than
ﬂoats. For multiplication the area is 144 % larger for posits than for ﬂoats, while speed-wise posits
are 240 % faster than ﬂoats.
We then wanted to obtain an estimate for the impact in real scenarios. Therefore, we picked two
large neural networks described in publications and computed the area and the speed for posit and
ﬂoat implementations. The results for the neural network are that posits are 98.1 % faster but require
195.5 % more area than ﬂoats.
Based on the ﬁndings in this thesis, we recommend further investigations of the unums in to the
context of training neural networks, since we saw substantial performance gains. Furthermore, you
can convert one number system into the other, making it easier for the new number system to enter
in the market, since the majority of the systems work with ﬂoating-point.

INTRODUCTION
3
Introduction
In the emerging world of artiﬁcial intelligence and Machine Learning (ML), training of the neural
networks is a bottle neck due to the demand for high precision arithmetic operations. The computa-
tional resources for the training process are high and execution times long. Therefore, there is much
potential for improvement of the training process.
Figure 1 shows an abstract description of the training process of a neural network. The steps of the
training process are:
1. Normalize the input data so that all inputs are at a comparable range.
2. Feed the model in training with the normalized input data.
3. The model generates the output data. The main operations during the generation are: additions
and multiplications.
4. Compare the generated value with the labeled test data (expected results) using a cost function.
5. Adjust the parameters of the model with the cost function results.
Steps (3), (4) and (5) are repeated until the value of the cost function results are close to zero, or
the accuracy of the model is not improving anymore. Afterwards, the trained model can receive
new unknown input data and predict outcomes with certain accuracy. The generation of the model
output data and the adjustment of the various parameters of the model (shown in purple), constitute
the most computational demanding operations. Thus, these are the operations investigated in this
thesis (see scope chapter 4 for more information).

Representing and manipulating real numbers efﬁciently is required in many ﬁelds of science.
Floating-point arithmetic following the IEEE guideline is the most widely used way of representing
real numbers in modern computers. But since IEEE does not deﬁne a standard for ﬂoating-point
numbers it is possible to obtain different results in different systems. [20]
Floating-point is used when very small and very large real numbers need to be represented. A
ﬂoating-point system can be used to represent, with a ﬁxed number of digits, numbers of different
orders of magnitude.
The following equation depicts ﬂoating-point numbers:

β: Base of the number. It has to be equal or greater than 2.
Xs: Signiﬁcand part of the number. It needs to be smaller than β.
Xe: Exponent value. It scales the signiﬁcand part of the number.

Although the IEEE ﬂoating-point format is the most widely used way to represent real numbers in
modern computers,there are unsolved issues with this format:
• There are many combinations to express NaN that could have been used to express more
real numbers. For example in a 16-bit ﬂoating-point number system, 2046 of 65536 bit
combinations are NaN values.

of the numbers is NaN the result is going to be false even if the bit patterns are identical.

5.2
Unums (Universal NUMberS) Type I
The unum type I number format is an upwards compatible extension of IEEE 754 ﬂoating-point
format. The unum type I format was created as an alternative to ﬂoating-point numbers to address a
number format in which the results are more accurate. Unums type I support interval arithmetic
(using a so-called ubit). Furthermore the number of bits can change depending on the required
precision. This is achived by adding a tag called utag that contains information about the length of
the different ﬁelds.
The unum type I number format was not popular as it still has the same problems as ﬂoating-point
format (see appendix C for more information about unums type I). The following chapters describe
the differences with respect to ﬂoating-point format and the issues this number system has.

Differences compared to the IEEE 754 ﬂoating-point format
As described at the beggining of the chapter, this format was created to substitute the ﬂoating-point

Issues with unums type I format
Similarly as with the ﬂoating-point format, unums type I have some issues that are not solved:
• These numbers have the same issues as ﬂoats have, due to the fact that it is a superset of IEEE
754 ﬂoating-point format.
• The variable length of the ﬁelds demands extra management.
• In the cases where the result is rounded to the nearest exact number, the interval arithmetic it
is not needed and the ubit could be used as another fraction bits.
5.3
Unums Type II
The search for a better number system led to the invention of unums type II. One of the main
advancements of the format is the reduction of bit combinations for exceptions (NaN, ’negative
zero’, etc.). The unum type II format keeps the ubit ﬁeld that has been added by unum type I, but
instead of being compatible with the ﬂoat numbers it changes the way of expressing the number
values.
The unum type II number format was not popular because most of the operations rely on look-up
tables and it is based on interval arithmetic (see appendix D for more information about unums
type II). The following chapters describe the differences with respect to unum type I format and the
issues this number system has.
5.3.1
Differences compared to the unums type I format
This number system is a second version of unums and it has the following differences with respect
to unums type I:

• Unums type II is not compatible with ﬂoating-point numbers.
• Unum type II format does not have exponent and fraction ﬁelds.

ideal number system.
• Type II unums have many ideal mathematical properties (division and multiplication are

• These numbers have the same problems concerning interval arithmetic as unums type I have:

rounded to an exact value. In such a case it would be preferable to use the bit combinations
with ubit one that express open intervals could be used to represent more exact values.
5.4
Unums Type III
The unum type III number system is the third version of unums. Unums type III take the ideas of
the previous unums versions and improve them. These improvements will be explained later in
the chapter 5.4.3. One of the main innovations of unums type III numbers is that they have higher
accuracy than ﬂoats [13]. Before this the unums type III format is explained.
The unum type III number system has two modes of operation. These two modes resemble the
following two esthetics for calculation involving real numbers:
• Posit mode. Posit mode employs ﬂoat arithmetic. This mode of operation is non-rigorous,
but it is cheap, fast and ‘good enough’ for an established set of applications.
• Valid mode. Valid mode employs interval arithmetic. This operation mode is rigorous and
mathematical at the cost of greater execution time and storage.

5.5.1
Relation with machine learning and deﬁnition
Machine learning is a ﬁeld of study which researches how a computer program can learn without
being programmed. The program learns from the provided data inputs and creates a model that
predicts the outputs for new (unknown) inputs. Some of these models can be described using neural
networks.
A neural network consists of so-called neurons. There are several types of neurons, e.g. linear,
binary threshold, rectiﬁer, sigmoid, stochastic binary. All neurons receive inputs (x1, x2, x3) and
have weighs (w1, w2, w3). They also have a bias (w0). These parameters neurons produce an output
value that is forwarded to neurons in the next layer of a network. The different types of neurons use
different functions, so-called activation functions, to calculate the output. The following example
explains the sigmoid type neuron.
The neuron sums the bias and all the values of all the inputs multiplied by their weights. This sum is
the input of the activation function. The output of the activation function is the output of the neuron.
The activation function in this case is a sigmoid function. Figure 12 shows a schematic depiction of
a sigmoid neuron:

Activation functions are mathematical equations that determine how the output value for a neuron is
computed. They thereby affect the performance and accuracy of the whole neural network. Each
neuron in the network has an activation function. The activation function creates an output value,
usually in the range between 1 and 0 or -1 and 1. The activation function determines if the neuron
should be activated or not, based on how relevant each neuron’s input is for the model’s prediction
or not. There are different types of activation functions [1]:
• Binary step: Binary step is a threshold-based activation function. If the input is above or
below a certain threshold, the neuron is activated and sends the same signal to the next layer.
A binary step function can only propagate two possible values. That fact implies that the
classiﬁcation of the inputs into several categories is not as precise as with a function that has
more different output values.
• Linear: The output signal is proportional to the input.
One of the problems with linear functions is that it is not possible to use backpropagation to
train the model because the derivative of the function is a constant that has no relation to the
input. Therefore it is not possible to go back and understand which weights can provide a
better prediction. The backpropagation is explained later in the chapter.
The second problem is that all the layers of the neural network collapse into one as the last
layer will be a linear function of the ﬁrst layer.
• Non-linear: Non-linear activation functions allow the network to create complex mappings
between the inputs and the outputs. Non-linear activation functions solve both problems
linear activation functions have.

The objective of a neural network is to create a model which can make predictions for any given
input that matches the given task. This is the objective of machine learning, to generalize rather
than learn things from memory. Some machine learning algorithms require training to achive that
while others do not.
The training process in neural networks employs the so-called back propagation. Back propagation
is the process that adjusts the different weights and biases. There are different backpropagation
methods: Gradient descent, gradient descent with momentum, variable learning rate, conjugate
gradient and quasi-Newton [8].
The training process starts by feeding the neural network different inputs for which the output is
known. It then adjusts the weights and biases depending on the error between the correct output
and the obtained one (actual and predicted value). The error can be computed using different cost
functions. The result of a cost function is a single value. This value measures how far the prediction
is from the actual value. The objective is to have a cost function value close to zero.
There are different cost functions: Quadratic cost or mean squared error, cross-entropy cost,
exponential cost, Hellinger distance, Kullback–Leibler divergence, generalized Kullback–Leibler
divergence and Itakura–Saito distance are some of them (see [9] for more information).
project uses the quadratic cost:
1
2 ·(y(predicted) - y(actual))2
(16)
The cost function is minimized by applying the gradient descent algorithm. Gradient descent is an
optimization algorithm that attempts to ﬁnd a global minimum5 of a function. The gradient descent
method enables a model to learn the gradient or direction that the model should take to reduce
errors, or in other words how to tweak or correct the parameters to further reduce the cost function.

6.1
Hardware implementation of sigmoid function
There are different ways to implement the sigmoid function using hardware [27]: Look-up tables,
A-law approximation, Alippi and Storti-Gajani approximation, PLAN approximation and piecewise
second-order approximation.
The look-up tables approach does not introduce any error but it requires enough memory to store
the tables. Usually this memory is not available and approximations of the sigmoid functions are
used. These approximations introduce error with respect to the original sigmoid function but they
use less logic resources.
In this project the selected sigmoid approximations are:
• The posit sigmoid approximation. Posits numbers with ES parameter equal to 0 feature a
cheap approximation of the sigmoid value (see 5.4).
• The Alippi and Storti-Gajani approximation. This approximation is selected in the case
of ﬂoats numbers because the error introduced by this approximation is one of the lowest
together with the PLAN approximation.

7
Veriﬁcation
In the different parts of the process of creating a chip, there are different types of veriﬁcation.
First when the RTL design is created, the functionality needs to be veriﬁed through the functional
veriﬁcation. Apart from the funcional veriﬁcation, the timing and performance requirements can be
checked in this stage.
After that, the gate level veriﬁcation takes place. The gate level information of a design is the netlist
view of the circuit where it is possible to do static timing and IR drop8 analysis between other
checks.
Lastly, in the back end, it can be done the scan chain insertion. This test connects all the ﬂops in a
chain. This chain acts as a shift register and then the inputs vectors are applied to all the ﬂops of the
chain. The scan chain test is done in order to detect any manufacturing fault in the combinatorial
logic. Another test that is done is the memory Built-In Self Test (BIST), this test checks all the
possible addresses of a speciﬁc memory.
This chapter describes the functional veriﬁcation of the design. There is no requirements in terms
of timing and performance, that is the reason why there are no veriﬁcation of these terms. The
performance and timing values are compared in chapter 8 against the performance and timing of the
reference model implementation of posits, and the performance and timing of the ﬂoat numbers.
In order to do the functional veriﬁcation of an RTL we need to implement a test bench in a HDL. In
this test bench, the instantiation of the design to be veriﬁed is placed. Once the test bench is created
it is needed to do simulations with different inputs to see the outputs of the RTL. This simulations
are usually transaction based simulations.
7.1

The following veriﬁcation techniques describe the way in which the inputs of the tests are deﬁned:
7.1.1
Direct testing
Veriﬁcation engineers study the design and identify different veriﬁcation scenarios to cover func-
tionality. After that they implement different transactions to test for the predicted behavior.
Therefore, some corner cases can not be found by this method if the veriﬁcation engineers have not
thought about them. Even if the engineers have thought, in all the possible cases it is probable that

7.1.2
Constraint Random Veriﬁcation (CRV) [12] [19]
This concept basically allows the user to generate random test vectors. These vectors provide a way
of exercising the Device Under Test (DUT)9 with more combinations of inputs in less simulation
time.
The term constrained is used because the vectors are not always completely random. They need to
fulﬁll some constraints, for example testing a protocol some of the ﬁelds can have illegal values that
are not part of the veriﬁcation process.
Two of the key concepts of CRV are:
• Reference model [15] [7]: One of the key components in a CRV based methodology is the
reference model. In order to test the RTL design it is needed a reference model to compare
towards it. There are different types of reference models:
– Behavioural model: not concerned with its synthesisability and the constraints imposed
by writing an RTL model. It is not an RTL model so it can be written in different
programming languages like: C, MatLab, Java, Python, Transaction Level Modeling
(TLM) that are not synthesisable.
– RTL model: it is possible to have another RTL implementation of the same design and
use it as a reference model.
The most commonly used is the behavioural model as usually it is not possible to have another
RTL design and it is more time consuming to create another RTL just to verify another one.
• Validation: Once the output of the reference model and the RTL design are obtained, it is
necessary to compare them to determine the correctness of the design. In order to do that
different options are available:
– Waveform comparison: simulating the test bench and observing DUT behavior at the
waveform comparing the outputs bit by bit.
– Scoreboard comparison: The idea is to have a system in which the outputs of the
reference model and the RTL are automatically compared.
In general, the scoreboards can have two different queues, one for the reference model and
the other one for the RTL and the different outputs are inserted in the queues. Each time the
scoreboard ﬁnds a match between the queues, these elements are removed from the queues.

the one that has just been inserted.

tool focuses all the effort in optimized the area.
An optimization of the speed is achieved by selecting a big value for the maximum area and by
reducing the value of the period constraint until the constraint is violated. The minimum value of
period for which the constraint is met is the minimum period (maximum speed). With the high
value in the maximum area the tool focuses all the effort in optimized the period.
8.2
Results
Before showing the results, we need to explain some details of the ﬂoating-point values.
The periods of the ﬂoat addition and multiplication implementations are multiplied by 3 because
the result of the operation is only available after 3 clock cycles. The implementation is pipe-lined.
That means that 3 clock cycles are only needed to obtain the ﬁrst result. Afterwards one output is
received every clock cycle. The results for the ﬂoating-point implementation therefore indicate the
area and period required to conduct a single addition or multiplication operation. The period for
each following operation increases not by the value of 3 clock cycles but only by the value of a
single one.
Similarly, the ﬂoat implementation of the sigmoid operation is a pipe-lined design, too. In this case
the delay is 9 clock cycles.

Next steps
So far we have only compared area and speed.
The objective with this comparison is to see if
switching to posits for the training of a neural network can have any advantage over ﬂoats without
changing the algorithms. As stated in chapter 8, the posit implementation seems to be faster than
the ﬂoat implementation, however posit implementation is also occupying more area. As the posit
number system format is rather new (October 2017). Then the work done in this project is the
beginning of a much larger work.
Deﬁnitely one interesting next step is the comparison between ﬂoat and posit accuracy as posits are
more accurate. This comparison is not done in the project because it requires to train a real neural
network with both number systems and do the comparison of the performance afterwards.
Some additional possible next steps are:
• Comparison of the power consumption used for each of the number systems. In this thesis
as stated in chapter 4, only the speed and area of the posit and the ﬂoat implementations
are compared and the power consumption is another characteristic, that designers take into
account when designing a digital system.
• Optimization of posit implementations. The results of chapter 8 show that the posits are
bigger than ﬂoats. Therefore, one possible next step is to optimize the area of the posit
implementations to see if there is a possibility to reduce the difference.
• Fix the bugs described in chapter 7 in the downloaded implementation. Another possibility is
to ﬁx the downloaded implementation using as a reference model the design of the project.
• Creation of a converter to change from ﬂoats to posits and the other way around. With the
converter it would be possible to select the best number system depending on the operation to
have the maximum speed and the minimum area.
• Extension of the calculations of the neural network to a convolutional neural network to see if
the results are the same as for the fully connected networks.
• Comparison of the speed of the neural network using posits in a FPGA with the speed of the
neural network in a GPU. To check if posit implementation can also compete in speed with a
GPU.
• Creation of Single Instruction Multiple Data (SIMD) instructions with posits. To explore if
the nature of posits could provide better ways of accessing memory which contains the values
for the computation. This is one of the biggest challenges for GPUs to keep the data as close
as possible to the Arithmetic Logic Unit (ALU).

The round to nearest even method is based in three types of bits:
• Keep bit: This is the last bit that can be represented in the fraction bits.
• Trailing bits: There are two trailing bits and they are the next two bits after the keep bit.
• Sticky bit: It has information about all the rest of the bits after the trailing bits.
– If any of the bits after the trailing bits is 1, the value of the sticky bit is 1.
– If none of the bits after the trailing bits is 1, the value of the sticky bit is 0.

Unums (Universal NUMberS) Type I
The unum number format is an upwards compatible extension of IEEE 754 ﬂoating-point format.
The unum format was created as an alternative to ﬂoating-point numbers to address a number format
in which the results are more accurate. Unums type I support interval arithmetic (using a so-called
ubit, see bellow). Furthermore the number of bits can change depending on the required precision.
This is achived by adding a tag called utag that contains information about the length of the different
ﬁelds. These two concepts ubit and utag are explained in the following lines.
Unum number format has the same three ﬁelds as ﬂoating-point numbers (sign, exponent, and
fraction bit ﬁelds) plus one additional ﬁeld called utag. The ﬁrst three ﬁelds take their deﬁnition
from IEEE 754, but the exponent and fraction ﬁeld lengths vary from a single bit up to a maximum
set by the user using the additional ﬁeld utag.
The utag is placed at the end of the ﬂoat number. The utag has three ﬁelds:
• Ubit - Bit that indicates whether the number is an exact ﬂoat or lies in the open interval
between the adjacent ﬂoats.
• Exponent size - Deﬁnes the maximum length of the exponent ﬁeld - 1. Therefore in order to
obtain the maximum exponent size add one to the binary value of the exponent size ﬁeld.
• Fraction size - Deﬁnes the maximum length of the fraction ﬁeld - 1. It is needed to do the
same operation as with the maximum exponent size.
Unlike ﬂoats, the unum number format does not have different types of formats depending on
the number of exponent bits. But it is possible to have different dynamic ranges by changing the
exponent size in the utag. The exponent size affects the dynamic range in the same way as the
ﬂoat’s exponent size, i.e. a larger exponent size results in a larger exponent range and therefore a
larger dynamic range.

The search for a better number system led to the invention of unums type II. One of the main
advancements of the format is the reduction of bit combinations for exceptions (NaN, ’negative
zero’, etc.). The unum type II format keeps the ubit ﬁeld that has been added by unum type I but
instead of being compatible with the ﬂoat numbers it changes the way of expressing the number
values.
The unum type II number system has three ﬁelds: sign bit, normal bits and ubit. Sign bit and ubit
have the same meaning as in unum type I, normal bits will be explained later. Figure 44 represents
the format of the unums type II:

• There are three bit combinations reserved to represent the followings numbers:
– Zero: A unum type II represents zero if all the bits are zero.

normal bits is one and the rest of the bits are zeros.

sign bit is one and the rest of the bits zeros.

Methods
Vivado tool synthesizes RTL into an speciﬁc Xilinx FPGA model. Vivado enables developers to
synthesize their designs, perform timing analysis and examine RTL diagrams. Vivado is a design
environment for FPGA products from Xilinx and it can not be used with FPGA products from other
vendors. The FPGA of the project is an Artix-7 AC701 FPGA.
In order to measure the minimum area the following Vivado run strategies13 (see [29] for the
deﬁnition of the different run strategies used) were used to see which is the best for the different
operations:
• Flow AreaOptimized high: It is a synthesis run strategy.
• Area Explore: It is an implementation run strategy.
• Area ExploreSequential: It is an implementation run strategy.
• Area ExploreWithRemap: It is an implementation run strategy.
When the maximum speed is needed the Vivado run strategies that were used are the following:
• Flow PerfOptimized high: It is a synthesis run strategy.
• Performance Explore: It is an implementation run strategy.
• Performance NetDelay high: It is an implementation run strategy.
• Performance Retiming: It is an implementation run strategy.
• Performance WLBlockPlacement: It is an implementation run strategy.
• Performance WLBlockPlacementFanoutOpt: It is an implementation run strategy.
With Vivado tool two different results are obtained:
• Synthesis: During the synthesis the RTL code is converted into general logic gates, memories,
etc.
• Implementation: During the implementation the general components obtained in the synthesis
are translated into real components of the FPGA.

As it can be seen there are also sigmoid functions in the backward process but as it is the same
sigmoid function that has been computed in the forward process (sig(Z0), sig(Zh1) and sig(Zh2)) it
is not taken into account to increase the area or the period of the training. It is supposed that the
value previously computed is stored in a memory and there is no need to compute it again.
In the previous neural network only one neuron per layer is represented so the number of addition
and multiplication need to be scaled by the number of neuron per layer as follows:
• Eo depends on the number of outputs. The number of multiplications to compute Eo is 2 (2 *
no) and the number of additions is also 2 (2 * no).
• Wo, Eh2 and dWo depend on the number of neurons in the hidden layer 2. The number of
multiplications to compute them is 5 (5 * nh2) and the number of additions is 2 (2 * nh2).
• Wh2, Eh1 and dWh2 depend on the number of neurons in the hidden layer 1. The number of
multiplications to compute them is 5 (5 * nh1) and the number of additions is 2 (2 * nh1).
• Wh1 and dWh1 depend on the number of inputs. The number of multiplications to compute
them is 2 (2 * ni) and the number of additions is 1 (ni).
The following equations represent the number of additions and multiplications in the backwards
propagation:
n add (backward full network) = 2·no + 2·nh2 + 2·nh1 + ni
(27)
n mult (backward full network) = 2·no + 5·nh2 + 5·nh1 + 2·ni
(28)
The area of the neural network is computed by adding the number of operations in forward and
backward propagation and then multiplying the number of addition operations by the area of one
addition block. The same with the multiplication and sigmoid operations.
The speed is calculated in a different way as having one block per operation some of the additions,
multiplications and sigmoids can be performed at the same time. Therefore the period is not directly
the time needed to do an addition times the number of addition and the same with the rest of
operations. The formulas are:
• The multiplication and sigmoid functions in each of the layers are permormed all of them
at the same time. Therefore only one multiplication and sigmoid period is spent per hidden
layer and output layer.
