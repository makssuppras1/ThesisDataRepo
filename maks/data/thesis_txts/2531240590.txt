Abstract 
This is an exploratory study into clustered multiple linear regression with Voronoi-connectivity, 
approached through mathematical programming methods. A model is developed to solve the clustered 
multiple linear regression problem that ensures that the nodes within any given cluster are connected 
through Voronoi adjacency relationships, and where the user specifies the number of clusters. This 
model is decomposed via Dantzig Wolfe decomposition into a set-partitioning restricted master problem 
and a cluster-generating sub problem for speed concerns. Additional cuts, parameter bounds and an 
initialization procedure are all explored for further speeding up the execution. The final formulation of 
both the restricted master problem and sub problem are assessed to be 𝑁𝑃-hard. Computational testing 
of the model on publicly available regression data sets run in reasonable time and produce favorable 
results for one-dimensional and small two-dimensional point sets. Larger two-dimensional point sets 
time out at more than six hours. The results of higher-dimensionality point sets are inconclusive and 
require further investigation. Due to the method’s nascence, many directions of future work are 
outlined alongside potential complications for each.

2 Introduction & Motivation 
Linear regression is an age-old scientific technique hailing from the domain of statistics. The goal of 
performing a regression upon data is often tied to the prediction of future outcomes given one (or a series) 
of inputs, though it can also be used to describe the data. A plethora of knowledge exists concerning the 
subject, and statisticians have become apt at solving this problem. 
Various sources of data often lend themselves to natural groupings or clusters, whereby data points within 
a cluster are more similar to one another than they are to the data points in other clusters. This can 
manifest itself in a number of ways, and a simple example of this could be that the younger demographic 
of most populations tend to transit more often for their daily commute while older demographics tend to 
drive. This constitutes an age-based cluster pattern when trying to predict the most likely means of 
transport. The inherent challenge of clustering methods is just that: identifying the underlying patterns 
within the data set when groupings are less obvious. 
In combining the two aforementioned methods, cluster patterns could also be identified based on the 
prediction error for each point in a cluster to that cluster’s regressed line, plane or hyperplane. That is to 
say, the more predictable all data points are within a potential cluster, the more likely that this potential 
cluster pattern reflects reality. Data points that are more predictable within a cluster also imply that future 
predictions within this cluster are more likely to be accurate and reliable. 
As outlined in Bertsimas & Shioda (2007), improvements in solver technology over recent decades have 
largely made integer programming methods more competitive in solving statistics problems. It was, 
therefore, the goal of this exploratory study to investigate the practicality of integer programming as 
applied to clustered linear regression. This study also aimed to serve as a proof of concept for a Voronoi 
similarity criterion, by which the formed clusters must abide. The metrics on which this evaluation is 
principally hinged are the execution time and the accuracy of the results. 
2.1 Solution Goals and Properties 
Accordingly, this study aimed to develop an integer program that would accept a data set and an ideal 
number of clusters 𝐾̅ ∈ℤ+ as an input. The program would then output a solution that adheres to the 
following goals, returning: 
1. The actual number of resulting clusters. 
2. The map of point-cluster attributions. 
3. The total prediction error in each cluster. 
4. The regressive coefficients for each cluster. 
Such that the output solution depicts the following solution properties: 
1. Points within a cluster are similar to one another (clusters are well-formed). 
2. The sum of the prediction errors across all clusters is minimized.

Such a program would allow for a detailed characterization of the data set. It would simultaneously depict 
which points behave similarly to one another, while also establishing a framework for the reliable 
prediction of responses of future inputs. 
2.2 The Structure of the Paper 
In the following section (Section 3), the related works are presented. 
Section 4 presents the technical information that will set the basis for the initial model. Tenets of linear 
regression, the Voronoi Diagram and graph theory are all woven together for later use in the 
Methodology. 
The method is presented in Section 5, communicated as an evolution of the formulation throughout the 
study. It starts with the full-edge model, an integer program hinged upon the theories of Section 4, and 
meeting the output goals and properties outlined in Section 2.1. Given the slow solving time of the full-
edge model, Dantzig Wolfe decomposition with Gurobi callbacks was implemented in Section 5.2 to speed 
up the overall solving time. Further speed gains were garnered through more effective bounding of the 
big 𝑀, and through the implementation of tighter inequalities in both the callbacks and sub problem in

problem, followed by an investigation into the restricted master problem covering relaxation in Section 
5.4. 
The final model is presented in Section 6, tying in all changes introduced to the formulation in Section 5. 
The results of computational tests enacted upon a series of publicly available data sets are presented in 
Section 7. Section 8 concludes the main body of the paper with Section 9 outlining the future work to be 
done.

3 Related Work 
Several studies related to linear regression as approached through operations research methods have 
already been published or are in review. Bertsimas & Shioda (2007) developed a classification and 
regression package they termed CRIO for instances of clustered simple regression, in which their principal 
shortcomings were execution time and discontinuity at cluster boundaries. Bertsimas & King (2016) 
further introduce a mixed integer quadratic optimization model for determining the regressive 
coefficients for a linear regression model on a single set of points. The body of the mathematical model 
also considers many elements that are considered favorable in linear regression, such as sparsity, 
robustness, stability against outliers and statistical significance, among others. A mixed-integer program 
for robust cluster-wise linear regression was proposed by Ismail, Rashwan & Makary (2019) that 
simultaneously determines the number of clusters in the provided data set and detects outliers. The 
results of their numerical testing and simulation study proved favorable with individual runs time-capped 
at under an hour, despite the much larger set of variables required. Kenney, Chiaromonte & Felici (2019) 
developed a mixed-integer program for feature selection via the 𝐿0-norm, contrasted against methods 
such as 𝐿𝐴𝑆𝑆𝑂, which rely upon the 𝐿1-norm. The authors’ method cuts down on the processing time and 
improves the effectiveness for the sparsity constraints of MIP models which typically target regression. 
They achieve this through a bisection procedure, a cross-validation scheme, and whitening (collinearity 
pre-processing). Similarly, Willis & von Stosch (2017) propose a model that utilizes a slack variable 
constraint such that the 𝑀𝐼𝐿𝑃 is formulated as smooth, solved through a sequential linear programming 
approach. They suggest that the 𝐿0-norm is more natural for sparsity considerations than the 𝐿1-norm of 
𝐿𝐴𝑆𝑆𝑂, but that it carries its own difficulties due to non-convexity and its discontinuous nature. All of 
these papers were considered in the early phase of model development and each presented themes 
relevant to the framework. The work of Bertsimas & Shioda (2007) was the most influential, and the 
regression component of the models presented in later sections of this study stem from their 𝐶𝑅𝐼𝑂 
framework. Numerous extensions to the regression framework were presented in the other studies. 
However, these extensions were set aside and included in Section 9 on future work in favor of firstly 
producing a minimum viable product. 
Studies or methods concerning linear regression stemming from within the domain of statistics are also 
numerous and valuable for comparative purposes. Breiman & Cutler host a website in which they 
extensively detail classification and regression trees (𝐶𝐴𝑅𝑇), building to and further describing the 
method of random forests. In association with the Statistics Department of the University of Berkeley, the 
theory for the underpinnings of their website was also published (Breiman, 2001). Wolsey (1998) provides 
additional base resources concerning trees and forests, while Jungnickel (2013) offers resources on graph 
theory, also sections of which concern the nature and structure of trees and forests. Breiman & Cutler 
help to contextualize one of the principal manners in which statisticians would approach linear regression 
problems, while the theory covered by Wolsey and Jungnickel directly influenced the preliminary 
ruminations for the full-edge model of Section 5.1. 
Aurenhammer’s (1991) paper on Voronoi diagrams provides a rich base of knowledge concerning the 
fundamentals of the Voronoi diagram, including its properties, geometry and algorithmic applications. 
Referencing Aurenhammer’s work, Liebling & Pournin (2012) further describe the relationship between

the Voronoi diagram and the Delaunay triangulation, and also delve into their histories, geometries and 
applications. Leveraging the structure of the Voronoi diagram, Shibata, Kato & Wada (2003) attempt for 
a more expedient and memory-efficient nearest neighbor classifier. They opt for a method they termed 
Voronoi condensing to handle the computationally taxing demands of calculating the Voronoi diagram in

similarity measure. The authors’ trials with the Voronoi diagram shed light upon the decision to use the 
adjacency relationships as a basis for similarity. 
Where column generation is concerned, Bredström, Jörnsten, Rönnqvist & Bouchard (2014) detail the 
column generation procedure and further propose modifications to the algorithm such that generated 
columns are capable of producing optimal or near-optimal integer solutions. Their testing phase yielded 
feasible solutions for all test instances and optimal integer solutions in 80% of cases, all within reasonable 
time. Rönnberg & Larsson (2009) detail the integral simplex method for set partitioning problems as well 
as how to tailor column generation to such a method. The authors then extend these principles in an 
additional paper, elaborating upon tighter bounding criteria for the sub-problem, embedding the scheme 
into a branch and price framework, and other various additional measures to enhance the column 
generation algorithm (Rönnberg & Larsson, 2013). Tahir, Desaulniers & El Hallaoui (2019) combine the 
integral simplex using decomposition method (𝐼𝑆𝑈𝐷) with an integral column generation heuristic to solve 
set partitioning problems with large sets of variables. The authors show that for their tested instances, 
the algorithm they have developed can yield optimal or near-optimal solutions in run times of less than 
one hour. Ghoniem & Sherali (2008) also put forward a method to generate complementary columns 
within the column generation framework by leveraging a tailored overgeneration scheme. They also 
propose a bound on the sub-problem termination criteria to curb the long-tailed slow convergence 
typically present in column generation schemes. Kuo & Leung (2016) extensively study the polyhedral 
structure of the mixed set covering, packing and partitioning problems, and develop various classes of 
valid inequalities for reducing computation times. All these studies were invaluable in the understanding 
of the inner workings of the column generation heuristic, and crucial in deciphering the current study’s 
failings to produce acceptable gaps with larger data sets. Many of the concepts included in these studies 
are reported in Section 9 on future work to improve the column generation heuristic.

4.1 Linear Regression 
As previously mentioned, linear regression is a tool that pertains principally to the domain of statistics but 
that is widely used across both academia and industry. The basic tenets of linear regression will be 
discussed in the following sections. 
4.1.1 Simple Linear Regression 
Simple linear regression is a technique used to characterize the relationship between two continuous 
variables. The method necessitates a data set that includes one input dimension termed the explanatory, 
predictor or feature variable, and one output dimension termed the response or outcome variable. This 
is called a simple linear regression due to the single explanatory dimension. 
The goals of such a characterization are often varied, but generally include: 
1. The desire to predict future outputs when only the input is known. 
2. The assessment of the effects of the input on the output. 
3. A general description of the data 
For example, one might wonder how long it takes to cook an egg based on water temperature, or how 
much additional revenue should be expected for a firm for a given amount of capital invested in 
advertising. In these examples, the input dimensions are water temperature and invested capital, and the 
output dimensions are egg cooking time and additional revenue.

random errors with mean 0 and variation 𝜎2, or in notation: 𝛿𝑖 ~ 𝑁(0, 𝜎2). It should be noted that this 
equation is simply the standard equation of a line 𝑦= 𝑚𝑥+ 𝑏, but with some random error term 𝛿𝑖 added

4.1.2 Multiple Linear Regression 
Multiple linear regression is similar to simple linear regression in its general structure, except for now the 
data set can include any number of inputs despite still having only one output. The goals are much the 
same as with simple linear regression, save one is attempting to characterize a joint relationship between 
the collective inputs and the output. 
For the egg example, it could be hypothesized that the egg cooking time does not depend on just the 
water temperature, but maybe on the size of the egg and on the egg’s initial temperature as well (was it 
in the fridge or on the counter?). If all this information is known beforehand, how much time should be 
expected to cook the egg, or in other words, what is the predicted output?

The underlying assumption of linear regression, regardless as to simple or multiple, is that the output 
behaves linearly. This means that for every increment in some input dimension, the change in the output 
is the same regardless as to where on the value spectrum this increment in the input occurs. For the egg 
example, this would mean that it is assumed that increasing the temperature from 0°𝐶 to 10°𝐶 should 
have the same effect on cooking time of the egg as increasing the temperature from 100°𝐶 to 110°𝐶. 
This may not be true for egg cooking time, but it’s a reasonable assumption for many other circumstances 
or phenomena, such as travel speed and travel time. 
4.1.3 Clustered Regression 
In clustered regression, inputs are first grouped into clusters based on some criteria, and then a linear 
regression is applied to each group individually (simple or multiple regression). Accordingly, specific to the 
case under investigation, the theory concerning simple or multiple linear regression will hold within each 
cluster independently of the other clusters. This implies that different ranges of the input dimensions 
behave differently, but within each range, the effect is still linear. 
Consider the egg example of a simple linear regression again. If it was assumed that the cooking 
temperature ranges of [0°𝐶, 10°𝐶] and [100°𝐶, 110°𝐶] were two separate clusters, then this would mean 
that increasing the temperature from 0°𝐶 to 10°𝐶 will not necessarily have the same effect on the egg’s 
cooking time as increasing the temperature from 100°𝐶 to 110°𝐶. However, it is expected that increasing 
the temperature from 100°𝐶 to 103°𝐶 or from 105°𝐶 to 108°𝐶 would produce the exact same reduction 
to the egg’s cooking time since both ranges exist in the same cluster and both span a difference of 3°𝐶.

4.2 The Voronoi Diagram 
The Voronoi diagram is a geometric data structure that constitutes a tessellation of a given space into 
tributary areas or volumes. Voronoi tessellations can be used to closely model phenomena throughout 
==PAGE:14==
David James 
 
Page 14 of 66 
the natural world, such as biological structures (cell and bone architectures), geographical watersheds or 
the growth patterns of forests and non-overlapping root networks (such as mycelia). 
In the context of this study, the Voronoi diagram is used to assess the similarity between points by 
tessellating the explanatory space of the data set. Points in the resulting tessellation that share a 
boundary are assumed similar to one another given their proximity in explanatory space. 
The following sections describe all elements that pertain to the phase of Voronoi pre-processing where 
the current study is concerned, and then again relates it to the study itself. 
4.2.1 Voronoi Diagram Definition 
For some 𝐷̅-dimensional space with 𝐷̅ ≥2 and a series of points 𝑃≔{𝑝1, 𝑝2, … , 𝑝𝑁̅} existing within that 
dimensional space, a Voronoi diagram partitions this space into tributary hyper regions, where each hyper 
region is centered on a single input point. Any new point that exists within the hyper region of an existing 
point is closer (or more similar) to that existing point than to any other point in the space.

When all hyper regions are generated, the resulting diagram is termed the Voronoi Diagram, where each 
bounded hyper region exists as a convex polytope. However, the hyper regions that define the points 
which lay on the convex hull of 𝑃 will necessarily be unbounded, as for these points, there exist spaces 
arbitrarily far away but still closer to the point than to any other. Consequently, every Voronoi vertex is 
equidistant from at least three points in 𝑃, and any point existing along a Voronoi ridge (the separation 
hyperplanes) is equidistant from at least two points in 𝑃. 
The generalization of the Voronoi diagram to higher dimensions than two is referred to as the power 
diagram. Power diagrams incorporate weighted partitions rather than exact bisections, and therefore 
every 𝐷̅-dimensional Voronoi diagram can be cast as a power diagram with equal partition weights on all 
points. Conversely, not every 𝐷̅-dimensional power diagram is a 𝐷̅-dimensional Voronoi diagram. 
Voronoi diagrams cannot be created for one-dimensional point sets while still retaining all of their 
fundamental properties by their definition. Despite, when the explanatory space of a data set is one-
dimensional in this study, bisecting tributary segments are still defined for each point in the same manner 
as for the standard Voronoi diagram above. The properties that are not preserved in such one-dimensional 
cases are not of crucial importance to the similarity criterion, and are thus disregarded. 
For this study, the term “Voronoi diagram” will be used to collectively refer to any of the bisected 
segments in one-dimensional space, the classic Voronoi diagram in two-dimensional space, or the equally-
weighted power diagram in higher dimensional spaces. 
4.2.2 Delaunay Triangulation 
A Voronoi tessellation can be converted into an undirected graph by laying an edge between any two 
points that share a facet. This graph is defined in literature as the Delaunay Triangulation, corresponding 
to the dual graph of the Voronoi diagram. Given that the two are duals, the Delaunay Triangulation of 𝑁̅ 
points can be obtained in 𝑂(𝑁̅) time, assuming a pre-computed Voronoi diagram (Aurenhammer, 1991). 
The casting of the Voronoi diagram as an undirected network presents a means of graphically modeling 
the system of point adjacency and allows for the application of forest theory in the design of the 
optimization model itself (covered in Section 4.3).

4.2.3 Connectedness 
The concepts of Voronoi connectedness and separation in the context of cluster formation will thus be 
defined. 
When assessing a specific cluster 𝑐∈𝐶, a point 𝑝𝑖∈𝑃 is termed active in that cluster if it is selected to 
be included in cluster 𝑐, and inactive otherwise. Likewise, any edge 𝑒∈𝐸 in the Delaunay triangulation 
𝐷𝑒𝑙= (𝐸, 𝑃) is also termed active only if both of the edge’s terminal nodes are also active. 
Given these definitions, and given a point set 𝑃≔{𝑝1, 𝑝2, … , 𝑝𝑁̅} with 𝑝𝑖∈ℝ𝐷̅, then any subset of points 
𝑆⊆𝑃 is termed Voronoi-connected if an active path exists between all point pairings (𝑝𝑖, 𝑝𝑗) ∈𝑆. 
Otherwise, the subset of points is termed Voronoi-separated. 
As applied to the clustering and regression problem, the clustering of points is restricted such that only 
Voronoi-connected points may exist together within a cluster. The inherent assumption of this logic is that 
points that border or are close to one another are similar and should ideally be clustered together.

4.2.4 Bounding the Voronoi Diagram 
The logic behind the Voronoi measure of similarity is to cluster points which are similar. However, since 
the tributary regions of the points on the extremities of the point set extend towards infinity, sometimes 
points which do not share a Voronoi boundary within the immediate vicinity of the pointset will share a 
Voronoi boundary near infinity. In such a case, the two points are Voronoi-connected, forming a feasible 
subset from a technical perspective. However, the adjacency of the points occurring near infinity defeats 
the intention of similarity-based clustering. 
For this reason, a manually inputted scaling parameter 𝜑 was added to the code to assign a Voronoi 
bounding box to the resulting Voronoi diagram. The margins of this box are established as a percentage 
of the value range of each dimension and are within the control of the user through the scaling parameter 
(the default value is set to 10%). For a given dimension 𝑑∈𝐷, the maximum and minimum scaling bounds 
are defined as: 
𝑥𝑑,𝑚𝑖𝑛= min(𝑥𝑑) −𝜑⋅(max(𝑥𝑑) −min(𝑥𝑑)) 
𝑥𝑑,𝑚𝑎𝑥= max(𝑥𝑑) + 𝜑⋅(max(𝑥𝑑) −min(𝑥𝑑)) 
Any Voronoi adjacency relations that exist beyond the Voronoi bounding box are disregarded in the 
formation of the Delaunay triangulation. Further information on the Voronoi bounding procedure is 
presented in Appendix 11.4.4.2. 
4.2.5 Industry Comparison 
Most of the examined papers on linear regression, such as Bertsimas & Shioda (2007), Willis & von Stosch 
(2017) and Ismail, Rashwan & Makary (2019), focus on either sparsity of the regression coefficients, 
determining the optimal number of clusters, or eliminating outliers (or combinations thereof). The 
clustering procedure in these studies is executed based on the regressive error alone and they therefore 
all implicitly use predictability as the only measure of similarity for cluster formation.

Other clustering methods, such as 𝑘-means, take the Euclidean distance of the observations into account. 
However, unlike 𝑘-means, the Voronoi tessellation is not hindered by the initialization of cluster centroids, 
but rather, simply establishes a rule by which the clustering must abide (Lindsten, Ohlsson, & Ljung, 2011). 
Convex clustering cannot catch certain patterns (i.e., concentric rings), while the Voronoi restriction is a

4.2.6 Concerns 
There are several concerns related to the scaling of the method due to the Voronoi diagram. 
As the dimensionality of a data set increases, the time taken to compute the Voronoi diagram increases, 
as discussed by Shibata, Kato & Wada (2003). The Voronoi decomposition for 𝑁̅ samples existing in 𝐷̅-
dimensional space depicts a computational complexity of 𝑂(𝑁̅⌊𝐷̅
2+1⌋), and accordingly, there is an 
exponential increase in processing time with increasing spatial dimensions. This led the authors to adopt 
Voronoi condensing, which they also eventually removed due to speed concerns. 
Additionally, the power of the similarity metric can only decrease with increasing spatial dimension, as 
the increasing dimension implies additional hyper region facets and, accordingly, a chance that any two 
previously dissimilar points will now share this new facet.

4.3 Trees and Forests 
Tree and forest theory is a subsection of graph theory that usually involves finding the subgraph of a 
graph with specific properties. The relevant concepts will first be formally defined before describing 
them intuitively. The following information on trees and forests is all derived from Wolsey (1998). 
Given a graph 𝐺= (𝑉, 𝐸) with node set 𝑉 and edge set 𝐸, the following definitions are presented: 
1. A forest is a subgraph 𝐺′ = (𝑉, 𝐸′) which contains no cycles. 
2. A tree is a subgraph 𝐺′ = (𝑉, 𝐸′) which is a forest that is connected. That is to say, all nodes of 𝑉 
are connected by the edges of 𝐸′.

As a consequence of these definitions, a graph 𝐺= (𝑉, 𝐸) is a tree if and only if: 
1. It is a forest with exactly |𝑉| −1 edges 
2. It is a connected graph that spans 𝑉 
3. It contains a unique path between every pair of nodes in 𝑉 
4. The addition of an edge to 𝐸 creates a unique cycle

precisely |𝑉| −𝐾̅ edges present in the network to create a graph that’s a forest with 𝐾̅ disconnected trees

be the ideal number of clusters as specified by the user, then each of the trees in the resulting subgraph 
would represent a Voronoi-connected cluster.

5.2 Decomposed Model 
Preliminary run time tests of the full-edge model on small one- and two-dimensional data sets (≤15 
observations) resulted in ideal solutions, but in far too long a time. Loading the series of constraints (9) 
for every subset of observations 𝑠∈𝑆 alone became intractable. 
Given the length of time required for the execution of the full-edge model of the previous section, Dantzig-
Wolfe decomposition was implemented in an attempt to shorten the runtime.  
5.2.1 Column Generation via Dantzig-Wolfe Decomposition 
The decomposition method separates the problem into a restricted master problem (𝑅𝑀𝑃) and a sub 
problem (𝑆𝑃) that each communicates information to one another in an iterative process. The goal of 
such a method is to reduce the number of variables that are present in the problem framework and speed 
up the execution. 
To comprehend the functioning of the decomposition, it is assumed that the reader is familiar with the 
concept of the primal and dual representations of a linear model. 
In the decomposition framework, the sub problem generates individual clusters as columns that are sent 
to the restricted master problem. The restricted master problem then decides on which of the existing 
sets of potential clusters should be used and communicates dual value information to the sub problem to 
inform the next instance of cluster formation. The stopping criterion is met once the sub problem can no 
longer generate beneficial columns, and then the restricted master problem determines the final set of 
clusters to be used. 
The restricted master problem is termed “restricted” because it operates on a reduced set of columns 𝐶 
that grows iteratively with the running of the sub problem. Contrasted against the master problem with 
the full column set 𝒞 containing every possible cluster pattern, it can be said that 𝐶⊆𝒞 and it is expected 
that 𝐶̅ ≪𝒞̅. 
Additionally, column generation is a heuristic method that is not guaranteed to produce the optimal 
solution to the master problem. The algorithm is largely motivated by greed, motioning in the direction 
which looks most beneficial at every given iteration via the duals. It should be noted, however, that the 
heuristic has been successfully combined with other methods (such as branch and price) to great effect.

5.2.2.2 
Properties of the Set Partitioning Problem (SPP) 
The 𝑆𝑃𝑃 depicts the quasi-integrality property, meaning that for the polytope of feasible linear solutions, 
every edge of the convex hull of the integer points is also an edge of the linear polytope itself. The solution 
space can be visualized as a 𝐶̅-dimensional unit hypercube, i.e., the dimension of the number of clusters 
at every given iteration. The space may be visualized in such a way because the value of any 𝛾𝑐 cannot 
exceed one due to constraint (13) (Rönnberg & Larsson, 2013). 
Accordingly, the linear relaxation and integer program share the same extremities. Therefore, the linear 
relaxation and integer formulations also share the same optimal solution to the master problem, when 
operating on column set 𝒞. Given this property, the restricted master problem (𝑅𝑀𝑃) is always relaxed 
to its linear counterpart throughout the execution of the heuristic. This implies that any fractional 
solutions to the 𝑅𝑀𝑃 are guaranteed non-optimal to the overlying master problem due to the quasi-
integrality property. The linear relaxation to the restricted 𝑆𝑃𝑃 is also known to produce very tight lower 
bounds to the problem.

5.2.2.3 
SPP Column Generation Drawbacks 
Column generation under the 𝑆𝑃𝑃 framework suffers from several drawbacks, namely that it often 
produces fractional solutions (fractional 𝛾𝑐 allocations), and that it depicts a tailing-off effect. 
Fractional solutions to the linear relaxation of the restricted master problem often present themselves 
due to the restricted column set 𝐶. Accordingly, it is not possible to know apriori as to whether the final 
set of columns that are generated and sent to the restricted master problem encompass competitive 
variables or even collectively feasible sets of variables. It is entirely possible that no feasible subset of 
columns can be selected by the restricted master problem upon the termination of column generation, 
despite their individual quality (Ghoniem & Sherali, 2008). Various schemes have been proposed to 
mitigate this issue, including complementary column generation, stabilization techniques, and 
embeddings into branch and price frameworks. 
Additionally, column generation is often afflicted by an initial rapid decrease in the objective, but then a 
longer and more cumbersome convergence in later iterations (tailing-off). This issue can be mitigated 
through tighter stopping criteria on the sub problem.

5.2.3.6.2 Flooding & Cut-Off Procedure 
The flooding algorithm derives its name from the image its execution evokes. Upon arrival at an incumbent 
solution, the currently clustered set of nodes are sorted by their indices and the first node in this set is 
chosen as the root. The root node’s network of Voronoi-connected nodes is then flooded, moving from 
node to node in the current cluster through the active adjacency relationships. 
Once all active nodes in the root node’s network have been mapped out, if any additional active nodes 
remain in the clustered set but are not in the flooded set, then the flooded set and the remaining active 
nodes are Voronoi-separated.

5.2.4.1 
Initialization for the Decomposition 
Due to constraint (14) of the restricted master problem, at least 𝐾̅ columns must exist within the problem 
at every instance such that all points are covered. This implies that an initial feasible solution is required 
==PAGE:33==
David James 
 
Page 33 of 66 
from the onset of the execution of the model. An initialization procedure was developed to conform to 
this requirement. 
The adopted initialization is a sequentially generative algorithm that selects 𝐾̅ nodes to be the cluster 
roots, from which clusters are grown based on a nearest Voronoi neighbor search. An initialization is 
successful if the clusters number 𝐾̅, if they partition the point set of observations when considered 
together, and if the minimum individual cluster sizes are met. Once this is verified, their hyperplane 
regressions are determined, and their error values assessed before being sent to the restricted master 
problem. 
Note that the initialization method is algorithmic rather than optimization- or heuristic-based. This 
decision was made in an effort to narrow the project scope. 
5.2.4.1.1 General Procedure 
The initialization procedure partitions the data set into the required number of clusters 𝐾̅ by first sorting 
the observations based on their response values (creating a monotonically increasing spectrum), and then 
dividing these points into 𝐾̅ −1 quantiles. The 𝐾̅ points that demarcate the borders of the equally-sized 
ranges are selected as the cluster root nodes. 
To illustrate this process, if four clusters are needed and 20 points are present, then the algorithm would 
create 3 ranges of indices: [1, 6.66667], [6.66667, 13.33333] and [13.33333, 20]. These ranges are 
rounded to their nearest corresponding integers, resulting in [1, 7], [7, 13] and [13, 20]. Accordingly, the 
points at the indices 1, 7, 13 and 20 are selected as the root nodes for each of the four required clusters. 
Clusters are then formed by cycling through the root nodes and annexing the nearest unattributed 
Voronoi neighboring node to the cluster each time, as measured by the Euclidean distance to the root 
node. In this way, each cluster has the opportunity to annex a node during every cycle, promoting the 
formation of clusters of relatively equal sizes. This phase of the execution terminates once all nodes have 
been attributed to a cluster. 
The quantiles-based division attempts to space the cluster roots as far from one another as possible. The 
aim is to produce acceptable clusters that meet the dimensionality size requirements, regardless as to the 
regressed error of these clusters. Additionally, always comparing neighbor distances to these root nodes 
provides a stable and unchanging reference point that would tend to annex points to clusters in a circular 
(or spherical) fashion. This is an attempt to prevent any given cluster from extending out into some other 
cluster’s potential territory and cutting it off before it can assemble enough points to meet the 
dimensionality requirements for minimum cluster size.  
Once the clusters have been formed, the cluster sizes are evaluated for the minimum cluster size 
requirements, as dictated by the dimensionality. The initialization procedure is repeated if a given cluster 
does not meet the size requirements. However, the root node that initiated the restart will be selected 
first for node annexation to give that cluster the best chance of forming earlier.

5.2.4.2.1 Big M Constraints 
The big 𝑀 constraint modelling technique uses a parameter denoted by 𝑀 to represent a very large

if that point is not included in the cluster under consideration. Alternatively, the model may instead select 
inappropriate regression coefficients that will induce incorrect prediction errors for other points.

5.2.4.2.3 Bounding the Big 𝑀 
The method used to bound the big 𝑀 in this study relies on two conjectures that remain to be proven: 
5.2.4.2.3.1 
Conjecture 1 
The most extreme slopes that can be produced in any two-dimensional dataset (one response and one 
explanatory variable) will only ever exist between point pairings. Here, extreme is defined as low 
dependence on the explanatory variable. 
If a regressed line between three points produces an extreme slope, then there exists a paired subset of 
these points that can produce an even more extreme slope. 
5.2.4.2.3.2 
Conjecture 2 
The predicted response for each point is represented by a linear combination of explanatory variables, 
and therefore the actual prediction error can only be less than the sum of the individual absolute 
prediction errors of the problem when decomposed into a series of two-dimensional problems (each 
involving the response variable and a single explanatory variable).

5.3 Speeding Up the Execution 
While the numerical results were promising upon some preliminary testing of the decomposed model, 
execution times were still beyond reasonable levels. This then sparked an effort to identify the principal 
sinks of execution time, such that attempts could be made to improve these areas of the model. 
This section starts with a brief investigation into relaxing the restricted master problem. Following this 
relaxation, the iterative constraints and the sub problem formulation itself (standard cuts) were 
strengthened in an attempt to cut down on execution duration. 
5.3.1 Reformulating the Restricted Master Problem 
The set partitioning component of the restricted master problem is a widely studied problem within the 
domain of operations research. As such, the set partitioning problem is well documented and studied, and 
is not a direct source of lost time during execution so long as the column set 𝐶 remains small (limiting the 
number of binary variables). 
However, the additional constraint (14) necessarily requires that an initialization procedure be adopted 
such that the restricted master problem be feasible from the onset. While the initialization procedure of

initialization procedure also does not consider any information concerning the prediction errors of the 
cluster during the formation process.

5.3.3 Application of Standard Cuts to the Sub Problem 
One of the most significant time sinks by far exists within the sub problem, and results from the menial 
restrictions on what defines a feasible incumbent solution. There are no connectivity restrictions on 
incumbents because there are no Voronoi-connectivity constraints explicitly within the sub problem. 
Consequently, much time is spent in correcting problem-feasible but connectivity-infeasible solutions in 
the callbacks. 
The implementation of stronger incumbent cut-off constraints is one means of mending this, as per 
Section 5.3.2. Another method would be to simply limit the supply of “obviously bad” incumbent solutions 
which trigger callbacks and corrections in the first place. If the sub problem never generates bad 
incumbent solutions, they never need to be cut off in a callback, and the execution becomes more 
expedient. 
To achieve such a result then requires that additional constraints be added to the sub problem such that 
these evident connectivity-infeasible incumbents are not generated. It would be significantly difficult to 
restrict the space entirely to only connectivity-feasible solutions. In contrast, better connectivity-
infeasible incumbents may be encouraged through additional constraints that only limit the feasible 
space.

5.3.4 Dual Fixity 
One other more direct means of speeding up the sub problem is to fix the point with the most influential 
dual at every iteration of the sub problem. This would then force the sub problem to include this point in 
the formation of the cluster throughout its iteration. This would have numerous benefits, namely that at 
every iteration of the sub problem there is at least one point for which inclusion is certain. 
Each cluster having a guaranteed node would principally present implications in the callbacks, with a 
significant performance gain through largely limiting the set of desirable integer incumbent solutions (less 
time spent cutting off bad incumbents). 
5.3.4.1 
Exclusion from the Model 
While this idea largely cut down on the computation time, it was ultimately removed from the 
implementation as the fixity condition was skewing the column generation process for the Dantzig Wolfe 
decomposition. 
The column generation would eventually become fixated on a single problematic dual. This dual would 
then correspond to a specific point that would become the root for a series of columns, all of which could 
not be selected by the restricted master problem in tandem due to their all containing the same influential 
node (though fractional attributions of these columns were possible for the linear relaxation of the 𝑅𝑀𝑃).

The column generation would output a series of columns centered on the same fixed root node until such 
a time that any column involving this point, still corresponding to the most influential dual, would depict 
a non-negative reduced cost. At this point, the sub problem would reach its termination criteria, and the 
restricted master problem would be left with a series of mutually exclusive columns all centered on the 
same root point to choose from. In such cases, the master problem returns the unit cluster since no 
columns may be combined to partition the data set. 
The issue described above is also an issue rooted in the generation of complementary columns, which is 
described in the subsequent section. Depending on the format of the solution to the generation of 
complementary columns, dual fixity may again become an interesting avenue of study. 
5.4 Producing Complementary Columns 
In Section 5.3, the restricted master problem (𝑅𝑀𝑃) was reformulated and a series of both iterative and 
standard cuts were introduced to the sub problem formulation. With speed no longer presenting as grave 
of an issue, the quality of the results of the algorithm came into question. 
In many of the test trials of the algorithm, the initialized columns – be they the unit cluster (a simple 
regression in which all nodes are in a single cluster) or the columns of the initialization procedure – were 
often returned with a huge gap between the linear relaxation and the MIPIFY of the 𝑅𝑀𝑃. Upon further 
investigation, it was found that these columns were often being returned due to a lack of sets of 
complementary columns in the final column set 𝐶 of the 𝑅𝑀𝑃. Given no columns could meet the stringent 
partitioning constraint, the algorithm was returning the only feasible solution it had produced, regardless 
of the cost of this solution. 
Consequently, a means of ensuring complementary columns became necessary. There are several 
techniques in the literature to achieve exact results for column generation frameworks, such as 
embedding the column generation into a branch and bound framework or only allowing for integer pivots, 
among others. Alternative approaches include the over-generation of columns at each sub problem 
instance, or relaxing the 𝑆𝑃𝑃 to the set covering problem (𝑆𝐶𝑃), and then enacting repairs upon the 
resulting solution. Due to the time-intensive nature of the implementation for exact methods, column 
over-generation functionality was implemented for the sub problem, and a repair characterization of the 
𝑆𝐶𝑃 to the 𝑆𝑃𝑃 was developed in favor of these exact methods. 
5.4.1 Over-Generation 
Column over-generation is a method by which the sub problem is solved multiple times with the same 
dual information. Every column generated by the sub problem is both saved and cut-off for future repeats 
using the same duals at each iteration of the sub problem. 
Consequently, for some user-specified integer 𝐿, the sub problem produces the best 𝐿 columns at every 
iteration. These 𝐿 columns are sent to the restricted master problem (𝑅𝑀𝑃) together, the 𝑅𝑀𝑃 is solved, 
and then a new set of duals is communicated to the sub problem. Using this new set of duals, another 𝐿 
columns are generated, and the process continued until such a time that the initial column generated by 
the sub problem at any given iteration depicts a non-beneficial reduced cost. 
==PAGE:46==
David James 
 
Page 46 of 66 
It should be noted that if some column depicts a non-beneficial reduced cost at any point of the sub 
problem over-generation procedure, the procedure for the current iteration halts, and all columns 
previously saved by the sub problem for this iteration are communicated to the restricted master 
problem. The termination criteria for the column generation heuristic is only met when the first column 
generated by the sub problem proves non-beneficial.

that they become complementary.

Data sets 8, 25 and 26 begin exploring higher dimensional spaces, and despite completing their executions 
well within the six-hour time window, they were almost never solved to optimality. These instances are

producing significantly large gaps. 
The selected columns for the final solution also imply that all columns generated by the column generation 
heuristic were unused, and therefore the column generation heuristic yielded no benefit to the solving of 
these instances. As discussed in Section 5.4, the current status of the heuristic currently suffers from issues 
in the generation of complementary columns, a fact reflected in the final column sets chosen for these 
instances. Furthermore, as mentioned in Section 5.2.2.3, these results depict the worst-case scenario in 
which none of the generated columns can be combined in any feasible manner, despite their individual 
quality (Ghoniem & Sherali, 2008). These results highlight the importance of the column generation

The fast run times of the high-dimensional data sets were unexpected, and it is hypothesized, in 
conjunction with the fact that the linear relaxations of most of these instances reached the “no error” 
case, that these data sets constitute “simple cases” in these dimensional spaces. If the linear relaxation of 
the restricted master problem depicts an objective value of 0, then by the quasi-integrality property of 
the set partitioning problem, there must also exist some integer solution on the full column set 𝒞 of the 
master problem with an objective value of 0 as well. If such a case exists, then all the data points in the 
data set line up with the regressed hyperplane producing a “perfect” prediction. This does not reflect 
reality, as noise always exists in measurements. In these high-dimensional cases, it is conjectured that 
either some of the dimensions are spurious (sparsity must be considered), or that the point set is simply 
not large enough for a space of such dimensionality.

8 Discussion & Conclusion 
The results of the previous section are mixed, but hopeful. They imply a successful proof of concept for 
the Voronoi-connectivity model, and run very successfully for small data sets. Furthermore, the 
successful runs meet all the solution goals and properties outlined in Section 2.1, at the onset of the 
study. 
While the cuts of the current formulation have largely improved the speed of execution, further 
techniques for speed-ups will be required for practical use of the method on larger or more complex 
data sets. Additionally, the testing phase did not encompass a study on the effect of the over-generation 
count, a parameter kept stoically at a value of 3 due to time concerns. The effect of varying this 
parameter could also be interesting in evaluating the optimality conditions of the instances which 
already run well. 
The data sets used for higher dimensionality testing were deemed inappropriate as per the analysis of 
the results of Section 7.2.1.2. Larger sizes of point sets are required relative to the number of 
dimensions to handle the inherent dimensionality bound on cluster sizes for regressing hyperplanes. 
Given appropriate data sets, based on the attempted runs of data set 28, the computation of the 
Voronoi diagram may become the factor governing the success of the execution. 
In conclusion, this study has both investigated and proposed a mathematical formulation that solves the 
clustered multiple linear regression problem, using a Voronoi-connectivity clustering metric. The model 
execution proved successful, confirmed through visualizations, solidifying the proof of concept for the 
method. For posterity, computational tests upon the model suggest that further speed-ups to the model 
would be required for more practical application. The testing phase also depicts non-ideal gaps between 
the linear relaxation and the MIPIFY solutions, the reasons for which are rooted in complementary column 
generation. Additional weaknesses and avenues of further development are outlined in the subsequent 
section on future work.

9 Future Work 
9.1 Initialization Procedure 
While the current initialization procedure for the restricted master problem is not necessary, it 
contributes to a lowered initial upper bound. With a lower ceiling, the restricted master problem should 
theoretically require less calls to the sub problem and therefore, complete its execution more quickly. 
The current procedure is algorithmic, but even a simple heuristic initialization could have a drastic impact 
on the starting conditions of the model. 
9.2 The Big M 
The big 𝑀 modeling technique is one that is well known within the domain of linear programming. It is 
also known to be a non-ideal formulation unless it can be appropriately bound, the reasons for which are 
explained in Section 5.2.4.2.1. 
The current method of bounding the big 𝑀 parameter of constraints (16) and (17) in the sub problem 
are far from tight. As it stands, the established bound simply ensures that given a hypothetical worst-case 
scenario, the big 𝑀 is large enough for the technique to work as intended. It is not possible that such 
worst-case errors are even attenable due to the independent dimensional decomposition that was 
employed to arrive at these bounds. 
There is therefore a lot of optimization potential that can be carried out in this domain, either through 
the establishment of tighter bounds, or through the reformulation of the regression constraints such that 
the big 𝑀 modelling technique is not necessary. 
9.3 The Full-Edge Model 
The full-edge model of Section 5.1.3 was developed only to the level of functional use, but was not 
explored for further improvements. Accordingly, the model executes slowly, but could potentially be sped 
up if investigated. 
One simple area in which gains could be had is that of constraints (10) and (11):

9.4 Regressive Modelling 
The regression model that was used in this study is largely inspired by the model used by Bertsimas & 
Shioda (2007). This model was used and not largely explored for improvements, despite the vast 
==PAGE:63==
David James 
 
Page 63 of 66 
quantities of extensions that are available to it. The purpose for this was to evaluate the method at a basic 
level before extending the model to include other variables. 
Several key regression themes, such as robustness, sparsity, outliers, splines and continuity have not been 
considered in this basic model. Additionally, this study employs the use of the absolute prediction error, 
though the maximum or average prediction error metrics could have also been interesting avenues of 
investigation. 
9.5 Column Generation Improvements 
Currently, one of the more significant issues to the practical execution of the model is the column 
generation. As described in the Results section, the columns that are being generated are largely 
incompatible due to the stringent nature of the partitioning constraint. Accordingly, the column 
generation would need to be guided in a more informed way so as to produce sets of columns that are 
complementary. 
Despite the methods that are proliferate in literature, this study did not encompass sufficient duration for 
the additional implementation of said methods. Promising means of improving the column generation in 
this respect include embedding the column generation into a branch and bound framework as described 
by Rönnberg & Larsson (2013), various algorithms to ensure integer pivots such as that proposed by Tahir, 
Desaulniers & El Hallaoui (2019), or other techniques such as dual stabilization. 
As mentioned in Section 5.3.4 on Dual Fixity, depending on the improvements applied to the column 
generation heuristic, dual fixity may also be an interesting direction of exploration in speeding up the 
problem execution. 
9.6 Execution Speed 
As evidenced within the results of Section 7.2, the overall execution duration scales with increasing 
numbers of observations and dimensionality. Two particular domains in which time is lost is in the 
individual iterations of the sub problem and in the tailing-off effect of the column generation heuristic. 
The neighborhood constraints introduced in Section 5.3.3.2 restrict the feasible region of the solution 
space through cuts leveraging the neighborhood structures of each point. As previously mentioned, these 
constraints are hypothesized to not be scalable with increasing dimensionality of the point set due to their 
reliance on these neighborhoods. As the dimensionality of the point set increases, new Voronoi facet 
dimensions are introduced in which two previously non-neighboring points may now be similar (adjacent). 
Every additional dimension that is added to a point set can only increase the number of neighbors that 
any given point has. As a result, individual points are more likely to have larger sets of neighbors in higher 
dimensional data sets, and therefore the neighbor-constrained cuts are proportionally less effective in 
these higher dimensional spaces. 
The tailing-off effect of the column generation heuristic is known within literature but has not been 
addressed in this study. As described in Section 5.2.2.3, the tailing-off effect is characterized by a long 
phase of the heuristic in which the generated columns present marginal benefits to the current problem 
objective, but do not attain the iteration stopping criteria. Some means of curbing or eliminating this 
==PAGE:64==
David James 
 
Page 64 of 66 
phase of the column generation method would largely speed up the overall method by calling the sub 
problem less often.

11.2 Repair Mechanism 
The following sections detail the characteristics of the repair mechanism, including the multi-counted 
node attributional framework for characterizing infeasible nodes. 
11.2.1 Multi-Counted Attributions 
For each multi-counted node in each cluster, that node is given an attribution based on its multi-count 
situation in that cluster. The attributions are as follows, in order of severity: 
1. Clustered 
2. Bridge 
3. Redundant or Leaf 
These characterizations capture different situations of importance in which a multi-counted node may 
exist within a cluster. Each attribution, therefore, has a different repairing procedure. If a point meets 
the criteria for more than one of the attributions above, it is attributed the more severe attribution. 
11.2.1.1 Clustered Multi-Counted Nodes 
When multiple (more than or equal to 2) multi-counted nodes are adjacent to one another for some 
given cluster, then those multi-counted nodes are labelled with a “clustered” attribution in that given 
cluster. For example, given the following graph: 
 
It can be observed that all the numbered nodes are multi-counted nodes, with their color indicating the 
clusters that both lay claim to the point. Node 6, for example, is counted in both the red and blue 
clusters.

David James 
 
Page 7 of 23 
In this graph, in looking at the green cluster, nodes 1, 2, 3 and 4 are labelled with the “clustered” 
attribution because they are all multi-counted nodes, and they are adjacent to one another (they are 
clustered together). In observing the blue cluster, the cluster attribution is allocated to nodes 4 and 5 
because they are both in the blue cluster, they are both multi-counted, and they are adjacent to one 
another. 
Accordingly, point 4 would therefore receive a “cluster” attribution for both the green cluster and the 
blue cluster. 
11.2.1.2 Bridge 
Multi-counted nodes are allocated the bridge attribution if they are non-grouped multi-counted nodes 
that if removed from a given cluster, would render that cluster Voronoi-separated. Using the same 
graph as before: 
 
It can be observed that node 6 is a bridge node for both the red cluster and the blue cluster. If this node

node was removed from would become Voronoi-separated. 
Note that node 4 also meets the criteria for the bridge attribution for the blue cluster. However, given 
that the cluster attribution has a higher severity, node 4 is attributed the cluster classification. 
11.2.1.3 Leaf or Redundant Nodes 
These attributions occur when the multi-counted node is not a cluster and is not a bridge. Leaf 
attributions are for multi-counted nodes that are connected to only one other node in their cluster, and 
redundant attributions are for multi-counted nodes that are connected to more than one other node in 
their cluster, but that are not bridges.

instead classified as a bridge. 
To illustrate this, consider multi-counted node 4 once again. While it’s technically a redundant node to 
the blue cluster, because its removal would yield a cluster size of 2 which is less than the required 3 for 
the dimensional space, it’s instead attributed a bridge classification. 
11.2.2 Repair Algorithm 
The general procedure that was implemented was to iterate across all multi-counted nodes, and 
compare the attributions of each node across all clusters. Given the complexity and interconnected 
nature of cluster and bridge multi-counted nodes, no straightforward or simple solution could be 
implemented to easily repair these nodes in certain situations. Accordingly, the following was applied in 
order: 
1. Cut off all multi-counted nodal clusters 
2. Cut off all multi-counted nodes with more than one bridge attribution across all clusters 
3. Apply simple repairs to the resulting solution when no clusters or multi-bridges exist 
==PAGE:75==
David James 
 
Page 9 of 23 
The rationale behind the need to separate multi-counted node clusters from the onset is because when 
multi-counted nodes are grouped together, they can develop conditional relationships to one another. 
Consider the following graphic: 
 
Multi-counted nodes 1 and 2 are attributed cluster classifications for both the green and red clusters. 
When deciding on how to divide up these nodes one can notice: 
• 
If one of the nodes is attributed to the green cluster but the other isn’t, then the node that’s 
kept is a bridge node for the green cluster and the dropped one is redundant. 
• 
If they are both kept, then they are both redundant 
• 
If neither are kept, then they are both bridges 
Along a similar path of reasoning, if a multi-counted node is a bridge in two clusters, then this node 
cannot be cleanly attributed to one of the clusters without separating the other. It is for these reasons 
that cluster and bridge nodes are cut-off before attempting to make repairs. 
The algorithm would then take an integer solution to the 𝑅𝑀𝑃𝑐, and assess whether any multi-counted 
node clusters or multi-bridges exist. If one of these attributions exists, the algorithm cuts off the solution 
and reiterates the column generation procedure. This continues until such a time that no multi-counted 
clusters or multi-bridges exist, and then simple repairs are put into place. 
The full pseudocode for the repair algorithm is included in the Appendix.

When neither of the two above cases of complex attributions exists, the repairs become simple. If a

other clusters, then the cluster where the node is classified as a bridge keeps this node, and it is 
removed from all other clusters. 
In the event that a multi-counted node is considered a leaf or redundant across all clusters, then the 
cluster that yields the largest benefit from this multi-counted node keeps it, and the node is removed 
from all others. In this context, the largest benefit is defined as the largest difference in total cluster 
error between the case of the cluster with the point and the case of the cluster without it, across all 
clusters in the solution.

11.4.4.1 The Particularity of the Callback Function 
The reason the 𝑀𝑎𝑖𝑛. 𝑝𝑦 and 𝑆𝑃. 𝑝𝑦 files are a single combined file 𝑆𝑃_𝑀𝑎𝑖𝑛. 𝑝𝑦 is because the callback

To correct this and allow the callback function to have access to the parameters and methods of the 𝑆𝑃 
class that it needs, the 𝑆𝑃 object actively used in the main execution is declared a global variable from 
within the callback function (global s). The callback function then necessarily needs to be defined in the 
same file as the main execution for the global variable solution to function correctly.

11.4.4.2 Voronoi Bounding 
The Voronoi bounding in the initialization of the PointSet class has, in rare cases, failed. This is because 
the method of determining whether a Voronoi boundary between two points is “valid” depends on 
whether at least one of the terminal Voronoi vertices for that boundary is considered “in bounds” 
(through a dimensional decomposition). If a given node in the point set has no neighbors because all 
ridges were removed by the bounding box, then the bounding box increases in size and tries again. This 
loop will reoccur until every point in the point set has at least one neighbor.  
That is to say, for any two arbitrary nodes that have a ridge bisecting them, then for the terminal 
vertices of that ridge: 
1. If both Voronoi vertices are inside the bounding box, then the Voronoi ridge is also inside the 
bounding box and so the adjacency relation is accepted 
2. If one of the Voronoi vertices is outside of the bounding box but the other is inside, then the 
Voronoi ridge lays partially within the box, and so the adjacency relation is accepted 
3. If both of the Voronoi vertices lay outside of the box, then the adjacency relation is not accepted 
However, consider now the case that both Voronoi vertices lay outside of the box, but the ridge 
between them cuts a corner of the box. It is for this reason that a dimensional decomposition was 
introduced to assess the vertices. 
This mechanism is still flawed, however, because the SCIPY package that calculates the Voronoi diagram 
uses the −1 placeholder for vertices which exist near infinity. This means that for select nodes at the 
extremes of the Voronoi diagram, the Voronoi ridge will have only one Voronoi vertex, and the other 
will be a −1 signaling infinity. Because the −1 lends no information as to whether this is positive or 
negative infinity for each dimension, there is no way of knowing, given the single known vertex is out of 
bounds in all dimensions, as to whether the ridge re-crosses the bounding box, or if it is also out of 
bounds. 
As previously mentioned, to solve this problem, the size of the bounding box is simply increased until 
the singular known vertex becomes “in bounds”. 
The 𝑆𝐶𝐼𝑃𝑌 Python package largely eased the burden of implementation, but this problem has yet to be 
solved. It would perhaps be worth investigating coding up an algorithm that calculates the Voronoi 
diagram independently of any existing package, or devising another means of determining which 
Voronoi ridges are valid and which aren’t based on dimensional bounds.

11.4.4.4 Column Tester File 
This file is a simple function that, given a column, will return the regression information for that column 
as produced by the Gurobi solver. This function was implemented in an effort to remain constituent, as 
discrepancies were found in the error values between the Gurobi solver and some of the other various 
Python regression packages widely available. This is thought to be due to the absolute error 
minimization objective of the Gurobi model as compared to the minimized squared errors of standard 
regression packages.
