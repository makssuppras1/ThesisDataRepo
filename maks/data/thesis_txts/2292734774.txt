Summary (English)
The goal of this thesis is to make a game in Unity3D that can work with the
eye tracker The Eye Tribe. Articles will be analysed to make it possible to get
a glimpse of what already exists in the interaction between eye tracking and
computer games. Eye tracking will be studied to understand how it works, and
what information eye tracking can capture. The main idea behind the project
is to nd out how to improve the game, especially a horror game with the help
of eye tracking. The answer is that eye tracking can improve a game, but other
things can have an impact, e.g. the surrounding of the game.

Chapter 1
Introduction
New computer games are being developed each day. Most of them have been
seen before or look much like present games, and developers need new ideas to
get the customers attention. One way is to make the player immerse themselves
in the game. A perfect example is the Oculus Rift[1] which makes it feel like
the user is inside the game. The more the user can be in the game, the better.
Human-computer interaction can help make games more real to the player.
Studies show for example that using eye tracking in games, can make the game
better [2].
The eye tracker is a new human-computer interaction.
The big
question is how to use this new information and integrate it in a proper way
with games. Other studies show that the location of objects in games can be
relocated [13]. This will stimulate the gameplay for the user, by forcing them
to search the screen places, where they wouldn't look. The developer can with
eye tracking make use of the whole screen, without the user knowing it. Studies
have been made integrating eye tracking with games[2]. Most of them focuses
on active eye tracking where the user can control the game with the eyes, or
passive eye tracking, for analysis of the user. This project will focus on using all
kinds of eye tracking: active, passive, and adaptive eye tracking. However the
main subject is to use adaptive eye tracking to make a game t to the player.
The eye tracking will analyse the user and from this information evolve into a
better game for the player. Since people are dierent, there is no magic recipe
to make a perfect game for everyone. But with eye tracking the game can use
the information, and it might reach out to more people.

2.1
Eye Tracking
Eye tracking is basically storing information about where a person is looking.
Eye tracking is mostly used in marketing, where companies can get a insight
in what costumers are looking at on the website of the company. Eye tracking
data can be split up into three main parts. Active, passive and adaptive.
2.1.1
Active eye tracking
Active eye tracking is where the user controls something, lets say a game, with
the eyes. In a rst person shooter game, such as counter strike, the user can e.g.
control the cross-hair instead of using the mouse. Since the eyes moves faster
than the mouse, this can have a positive eect in games where a fast reaction
time is needed. In other scenarios such as home cooking, it might also be useful.
Imagine yourself baking a bread. Both your hands are used to knead the bread,
but you need to turn the page in the cookbook. This is where eye tracking
becomes useful. By focussing your gaze at a specic arrow or symbol, the page
turns itself.
==PAGE:16==
4
Analysis
2.1.2
Passive eye tracking
Passive eye tracking is where the data is collected, but where no further actions
are made. The data is only used for analysis, which can be used to improve
the specic matter. This could for example be for a website. Users look at
a website, but unfortunately the users never look on the "You Won!" button.
This can either be because people know its a scam, or because the position of
the button is poorly placed. If the most viewed area of interest is in the right
corner of the website, the company might relocate the button to that location.
2.1.3
Adaptive eye tracking
Adaptive eye tracking is where the environment being looked at changes based
on the eye tracking data. In relation to active eye tracking, the user is not always
aware of the change. Once again this can be used in games, such as counter
strike. It is obvious that with one enemy appearing on the screen, the user is
able to use all his attention towards the one enemy. But with more enemies
the attention is split. People have dierent threshold of how many things can
appear on the screen, without breaking the focus of the user. Using adaptive
eye tracking the game could manage this aspect. This way each user would get
the same amount of diculty, but where each user had dierent challenges.
2.1.4
Classication
To understand eye tracking, some classication of the eye movements need to
be declared.
Fixation: A xation is small stops during the eye movements. Imagine reading a
text, instead of reading the text in one smooth line, each word will be
processed. These are called xation.
Saccades: Saccades are the path from a xation to another. This is when the user
doesn't focus on an object, but moves from one to another.
Area of interest: The area of interest is based on the content. This is what users have been
looking at the most. This is used a lot in marketing and user experience,
by nding out what are good and bad product placement on e.g. websites.
Heatmaps can be used to dene area of interest. First picture in gure
2.1 shows the eye movement, while the second picture shows the heatmap.

2.2
The Eye Tracker
The eye tracker is the object which can extract data from the user. The user
looks e.g. on a computer screen, and the eye tracker collects the eye movements.
The eye tracker is build up from two components: a camera and an infrared light
source. The infrared light sends light towards the eye. The light will then be
reected in the eye, and the camera will then catch the reection. This way
the pupil position can be captured, and thus the eye xation points. Normal
light could be used as well, this could distract the user. Infrared light is not
visible to the human eye, and hence will not distract the user. The question in
which eye tracker should be used for the project is based on what the project is
about. There exist two main eye trackers. There are remote eye trackers which
are suitable for personal computers and laptops, and then there are wearable
eye trackers which can be used while the user moves around. For gathering
data for websites, games and such, the remote eye tracker is the best choice.
Since the remote eye tracker is stationary, it is the best eye tracker for most
data collection. The user can control the eye tracker in a better way than the
wearable eye tracker. However if the eye tracking should take place in a public
place such as a train station or a park, the wearable eye tracker is better. In
most cases a thumb rule is to go with the remote eye tracker as long as its
possible.
2.2.1
Sampling rate
The sampling rate is important for the eye tracker to extract valid data. A slow
sampling rate will cause the data to be missing some parts, as some important
information may be lost during the eye tracking. Since seccades are some of the
fastest movements possibble by the human body [3], a lot of data can be lost if
the eye tracker can't collect the data.
2.2.2
Accuracy and precision
Accuracy is how well the eye tracker hits the real location of the eye coordi-
nates, while the precision is how well each gaze point is captured. Sometimes
people think precision and accuracy as the same, but gure 2.5 shows a better
understanding of how accuracy and precision works with each other.

2.2.3
Head box size
The head box denes the freedom of the user. This indicates how much the user
can move his head without breaking the eye tracking. The best result of eye
tracking is when the user sits completely still, but this is not always possible.
Many computer gamers, have a tendency to move while they control a character
or object in the game. A good example is controlling a car. When the car turns,
the user has a tendency to lean to the turning direction. Another example is in
rst person shooter games, or horror adventure games. If the player will look
around a corner the player also have a tendency to lean, in order to have a
better view around the corner. However this does not have any eect on the
game what so ever. A wearable device will not have the data loss due to the
head box, as the head box obviously always follows the head of the user.
2.2.4
Monocular vs. binocular tracking
Monocular tracking only tracks one eye, while binocular tracking tracks both
eyes. In most cases monocular tracking is enough, since both eyes usually follow
the same path or look at the same area. However should one eye temporary be
unavailable, the other eye will still give data, using a binocular tracker. The
binocular tracking will therefore give a better chance of extracting all the eye
data.

Pupil illumination method
The pupil illumination method denes how the infrared light is emitted. Bright
pupil illumination is when the light source is placed in the same place as the
camera, while dark pupil illumination is when the light source is attached away
from the camera. Figure 2.6 shows the two methods.
Figure 2.6: Left picture indicates dark pupil illumination, while the right pic-
ture shows bright pupil illumination.
The two pictures are dierent in the way the corneal reection is. The dark pupil
illumination picture will reect the iris, while the bright pupil illumination will
reect the pupil.

3.1.3
Calibration
Before the eye tracker can work properly, it needs to know how the user sits in
front of the screen. Because all people are dierent, with dierent eyes, the eye
tracker needs to be calibrated. This is done by letting the user look at multiple
calibration points, which appears on the screen. Based on how many points will
appear the eye tracker can measure the eye coordinates on the screen. At least
four points should be shown in order to dene the screen, however for a better
calibration, more points are needed. The calibration used in this project uses
the one from The Eye Tribe tutorial, where 9 calibration points are shown.

The object is dened by a random number from 0-4, for each of the ve game
prefabs. This way the algorithm works like a breadth-rst search4, starting from
the initial point and working its way outwards. If one path ends in another open
path, both paths will be removed from the open list. This makes sure that the
maze wont continue a path which already has been visited3.4.
Figure 3.4: If two paths meet up, they will both be removed from the list.
The "Close" list is used when placing objects. A CornerPrefab cannot be placed
if the next position is closed3.5. If a path is not valid, the removed path will
be added to the list "Open" and will be handled later. Next time the path is
taken, a new random number will be chosen, which hopefully will t into the
maze. This can of course result in multiple runs on the same path. A better
solution for this would be to save the invalid selections, which would cause the
algorithm to never meet the same path more than four times.
