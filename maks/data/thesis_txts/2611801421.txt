Summary
Time Sensitive Networking is not a new technology, as it is an improvement of an ex-
isting networking technology — Ethernet. It is a set of Ethernet standards which are
contributing to the industrial networking. Ethernet servers have the responsibility
for ensuring reliable data transmission across networks. If you send a message across
an Ethernet network, you will be almost sure that it will arrive at its destination.
But the problem is, you cannot be sure precisely when it will arrive. It might have
delay to arrive destination because of several reasons. Normally these delays are not
very critical when it is just about opening a file or printing a document. Nonetheless
these delays are very crucial for applications that require Real-time communications,
such as autonomous-driving cars.
To implement a TSN network, three main aspects must be considered, namely
topology, routing and scheduling, these aspects are highly associated in a TSN net-
work.
In this thesis Routing problem in TSN networks is addressed and the other two
items are out of scope of the thesis.

CHAPTER 1
Introduction
By growing up the computer usage, the need for sharing data among the computers
appeared as an essential need. Thus computer network created for sharing data and
resources.
Nowadays computer networks are using for many different purposes such as printer
sharing, storage sharing, Internet, email and more. Delivering data reliably is the
concern of these networks, whereas they do not have deterministic time scheduling
for that. Hence a small delay or a small fault in synchronization does not lead to a
big issues in these networks.
Upon arrival of digitization and computer networks in industry, new characteristics
and requirements for the networks aroused. The industrial devices need to accom-
plish a task in a fraction of time, in the other word they need to be Real-time. Beside
the Real-time characteristic they have to be also reliable because wrong functionality
or inactivity of a device might leads to a big catastrophe. These characteristics in-
troduced two new concepts for the industrial networks, which are namely Real-time
systems and Safety-critical systems.
Real-time Systems: In the Real-time systems, it is not enough that the logical
computation results being correct, it is important that these results be provided in a
certain time. The maximum time that a process can wait for a message in order to
it happens in the system correctly, is considered as a Worst-Case Delay. Obviously,
the Worst-Case Delay should be smaller or equal to the message deadline.
Real-time systems can be considered in two groups: Soft Real-time system and Hard
Real-time system. In soft Real-time systems, if some signals or messages do not reach
the deadline and arrive later, it is still acceptable but it leads to lower quality of the
system. In hard Real-time systems if a signal or a message reaches after its deadline
the whole process will be considered as a failed process although the results being
correct. In this thesis we consider all systems as hard Real-time systems.
Safety-Critical Systems: Some process in a system has to be done always
correctly, and if it does not happen after its assigned plan, that could be leads to en-
danger human life, damage the environment or a big financial lost. A system, which

2.1.1
All-Pairs Shortest Path
The purpose of this algorithm is to create a table of vertices (V) that shows the
shortest distance between two vertices u,v ∈V. It will be a tabular form that the
junction of raw u and the column v shows the weight of shortest path from u to v. In
order to build this table Dijkstra’s algorithm can be applied if we have no negative
weights for the edges. Dijkstra’s algorithm provides the shortest path from specific
source vertex to all other vertices of graph, and if it runs |V| times, it will construct
the table of shortest paths between all pairs in the graph.
If negative value for the edges of graph is allowed then Dijkstra’s algorithm cannot be
used and instead we use another algorithm, which its name is Bellman-Ford algorithm.
The Problem of these two algorithms is their quickness. They are slow and are not
suitable for applying for big graphs with a lot of vertices. Therefore other algorithms
such as Floyd-Warshall algorithm is introduced.
Floyd-Warshall solve this problem by using dynamic programming, which tries to
solve the problem by making a sequence of decisions. It makes a new decision in each
stage. It is also faster than Bellman-Ford algorithm. Floyd-Warshall runs in O(V 3)
but Bellman-Ford runs in O(V 4).
For spars graphs Johnson’s algorithm can be used, which is even faster than Floyd-
Warshall algorithm. The input graph for this algorithm should not have a negative-
weight cycle. Johnson’s algorithm can find all the shortest paths for spars graphs in
O(V 2 Lg V + VE) time.

Each frame has a limited size of maximum 1500 bytes, which is known as Maximum
Transmission Unit (MTU). Larger messages split equally in MTU size except the last
frame, which would be smaller than MTU size because it contains the last remaining
part of the message. As a result, the TSN flows are composed of frames.
3.2
Switches
The construction of a Switch (also known as Bridge), in TSN network is included
input and output ports, known as ingress and egress ports, respectively. When a
frame arrives at the ingress port it will be send to - according to a specific schedule
- the respective egress port. Each egress port has a queue, where it saves the arrived
frames temporary and sends them respectively to the egress port, when it is idle.
3.3
Traffic types
Based on the criticality, three classes of traffic are introduced for the TSN. These
three traffic types are explained in the following. [Gav18]
Best Effort (BE): This is the basic class of Ethernet traffic that tries its best to
send messages, but there is no guarantee of time and delivery. It uses, to allocate
low-priority traffic to frames, in the scheduling process that takes place in the switch.
Time Trigger (TT): Time trigger traffic is based on deterministic time schedule.
This time schedule table is called Gate Control List (GCL), that determines the pre-
cise queue transmission times for the frames at the egress ports of switches in the
route to its destination. The schedule specifies, when exactly the associated gate to
the queue should be opened, and when it should be closed. Knowing that, there
are several devices in the TSN network, and their TT traffics need to be scheduled,
then having a global time is necessary in order to synchronize the clock of different
requests. This synchronization is fulfilled by the IEEE 802.1ASrev standard.
Audio-Video Bridging (AVB): It applies for the applications that need to fulfill
end-to-end latency, but it is not necessary for them to be Real-time like TT. It uses
for transferring multimedia contents, where Real-timing is not essential problem and
has lower priority than TT.

3.4
Scheduling
One of the most useful mechanism for scheduling the communication, in a TSN net-
work is Time-Aware Scheduler. It helps to prevent bottlenecks in data transmission,
and it contributes to comply latency and jitter by minimizing the queuing issues in
Ethernet switches. To use the Time-Aware Scheduler, the Ethernet switch must be
able to identify frames with different priorities.
There are some downsides to Time-Aware Scheduler as well, for example; they are
not plug and play and they should be configured in advance. They are not also good
choice for the applications that are not interested in the lower latency or jitter.
3.5
TSN Configuration Models [Bel18]
The first phase of building a TSN network is choosing the TSN model. There are
three TSN configuration model:
Centralized TSN: In this model the end devices announce their requirements to
a central user configuration (CUC). The CUC aggregates all information regarding
the end devices and their communication requirements. A second entity, called the
centralized network configuration (CNC), assembles all information from the network
infrastructure, such as the number of switches present, the physical and logical topol-
ogy, the TSN mechanisms, and the available bandwidth.
The advantage of centralized TSN is that it supports large network infrastructures
with dynamic device movement.
The disadvantage of the centralized model is that it requires dedicated infrastructure
that houses the CNC and CUC.
Decentralized TSN: In this model the end devices transmit their requirements
for sending and receiving TSN communication streams to the first TSN switch, which
is connected to, this switch receives the information, evaluates it, and distributes it
across the network through any other switches. Ultimately, all end devices are in-
formed about the data streams offered in the network and the requests to receive
certain streams. When an offer and a request in the network match, the end devices
and switches automatically establish the connection and process frames according to
the parameters agreed upon by the two end devices.
The advantages of decentralized model are the TSN switches require very little, and
the decentralized model is more plug-and-play.
The disadvantages of this model is, on larger networks, conflicts between the require-

Hybrid: In this model you can combine the centralized and decentralized TSN

The advantage of the hybrid model is that the end devices need to support only one

The application model is a set of Real-time streams, which are shown by a tuple of
attributes (va, vb, S, P, D, R), where va and vb represent respectively, source and
destination of the stream. The size of the each stream donated by S and it represents
in bytes (B). Streams are periodic, so P donates the period of strings in microseconds
(µs). The maximum time that can be taken for a stream to arrive to its destination
is deadline D of the stream, and it measures in microseconds (µs). It is important to
consider deadlines in streams scheduling. A message should not deliver later than its
deadline. The next attribute is redundancy level and it is shown by R. This attribute
differentiates critical streams and non-critical streams. For non-critical streams The
value of the redundancy level is considered to be 1, and for critical streams, it has
a value greater than 1. According to the level of critically redundancy level will be
larger. This indicates the number of routes expected for a stream, so that they have
the least overlap, preferably without overlap. Two routes have overlapping when they
have common vertices (switches) in their path. It is important for the critical streams
that have the minimum overlapping in their redundant routes.
There are two category of redundancy protocol; Seamless redundancy and non-seamless
redundancy.
In the Seamless redundancy, all network paths are used in parallel, so no disruption
occurs if one path fails. In non-seamless redundancy, the protocol recovers the fault
by switching from the primary path to the secondary path. In this thesis, it has been
assumed the Seamless redundancy protocol is used.

Problem
The problem that has been investigated in this thesis is an optimization problem.
We are interested to find the best Routing paths for TSN-networks. This problem
is always joint with topology and scheduling problem in TSN networks. We assumed
network topology has been given, and we do not investigate the scheduling problem,
since it is not in the scope of this thesis.
As an input, for this problem we have topology of the network, with the speed of
each link in the network. Moreover we have given a set of Real-time streams with
their attributes. These streams must be routed, meeting all the limitations in the
network. A subset of these Real-time streams, are critical-streams, which have the
redundancy-level that should be considered in the solution.

CHAPTER 6
Solution
This chapter presents a meta-heuristic solution to the problem described in Chapter
5. IIn this solution, the streams are assumed to be unicast and cannot be applied to
multicast or broadcast streams.
As input, network topology, links speed (bandwidth) and streams attributes, in-
cluding source, destination, size, deadline, period and redundancy for critical streams,
are given. So it has been tried to initiate a solution, which fulfills all the constrains
that are introduced by streams’ attributes and the links. In order to evaluate the re-
sults, which are obtained by a solution, a cost function has been designed, which will
be introduced later. Using the cost function, the lowest cost solutions are selected.
The represented method is based on Hill Climbing algorithm and combined with op-
timization algorithm Simulated Annealing (SA), to achieve better results.
This solution can be followed in five steps as below:
• Providing an initial solution
• Consider initial solution as the best solution so far and calculating the cost for
it
• Making a small change in this solution (for example a route for a stream) and
calculate again the cost for the new solution
• Investigate the improvement, by comparing the new result with the best result
so far, and select the better solution as the best result. In this step, it has been
tried to optimize the solution with applying optimization techniques, in order
to avoid lock in the local maximum and minimum in obtained results.
• By iterating the last two steps better results will be achieved
In the following, it has been attempted to explain the solution thoroughly, by intro-
ducing different parts of the solution.

6.3.2
Solution Optimization
Optimization is a process of trying to find the best solution among the available
solutions.
The heuristic method that is described above will guarantee an answer for our problem,
but this answer might not be the best answer since it compares the results with
neighbours and if a neighbour result is better, then it moves to the neighbour state
and iterate the solution until the optimum results be found. In this solution there
is a risk of trapping in a local optimum. Local optimum is a point in the search
that all its neighbours have the more worse state than this point. To avoid this risk
optimization algorithms help to investigate the other potential states in our solution,
which could be a better one, and in the best case finding the global optimum.
In order to optimize the suggested solution, Simulated Annealing (SA) as a search
optimization method is used to investigate the broader domain of possible answers.
6.3.2.1
Simulated Annealing
Simulated Annealing is a meta-heuristic optimization algorithm, which is trying to
find an approximate global optimum instead of finding a local optimum.
It considers also a neighbor’s state (si+1) of the current state (si), and by calculating a
probability decides between moving the system to state (si+1) or staying in state (si).
These probabilities, finally manage the system to move to the lower energy states.
Generally, this step is iterated until the system reaches a state that is acceptable, or
until the given time or iteration hits its limitation.

8.1
Conclusion
In this thesis, it has been tried to propose a solution to find the best paths for routing
streams in a TSN network. These routes must meet all the requirements of streams
and comply with network constraints.
The first method that tried, was Dijkstra algorithm, which finds the shortest path .
This solution dismissed because it does not take into account the level of redundancy
for critical streams and the bandwidth limitation of links. It also has the problem of
locking in the local optimums and does not provide the best results.
An improved algorithm was then used, which took into account the level of redun-
dancy, but was not enough to solve the problem, because there were still problems
with trapping in local optimums and considering the bandwidth of links.
Using Hill Climbing algorithm and applying the cost function helped to solve the
problem of, considering links bandwidth limitation, but it has still the problem of
locking in the local optimums. In order to overcome this challenge a meta-heuristic
search algorithm, Simulated Annealing (SA), has been used. SA does not focus just
on the neighbour’s result, and it does not improve the results regarding the local
neighbors. It tries to consider other area of solution domain and investigate for pos-
sibly better results. In this way, it has a big chance to overcome the trapping in local
optimums. Efficiency of this optimization algorithm (SA), depends on the number of
iteration and the adjustment of the initial coefficients.
The final method, which used SA to find the best result, was applied to the proposed
problem and was able to significantly improve the results. Based on the results that
are obtained by trying several test cases, it was found that this solution is sufficient.
It has particularly significant results for big and huge networks.

8.2 Future Work
67
8.2
Future Work
In this thesis, due to time constraints, it has been tried to focus on initial routing
problem for TSN networks, so it can be improved in some aspects such as Schedula-
bility, Fault tolerant and Multicasting.
Scheduability
In the thesis length of routes has been considered, and it has been tried to select
routes with shorter lengths, but it is not enough, if the stream’s transmission takes
longer time than the stream’s deadline.
It can be improved by taking into account the Worst Case Delay instead of route’s
length.
Fault tolerant
The solution, which is introduced in this thesis, uses all the links in order to find
the redundant routs with minimum overlapping. It can be developed to be able to
tolerate 1-link and 2-link failures instead of minimizing just the overlapping number.
This helps to introduce a solution with cheaper results.
Multicast
The proposed solution considers only the unicast streams, and it does not work for
multicast streams.
It can be developed in a way that can also manage multicast
streams.
