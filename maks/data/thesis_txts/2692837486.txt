Abstract
The security of the RSA crypto system relays upon the diﬃculty of integer factorization. Knowledge about the
currently best factorization methods is therefore crucial to ensure correct usage of RSA. The two fastest factorization
algorithms are the quadratic sieve (QS) and the general number ﬁeld sieve (NFS). In this thesis the mathematical
theory of the quadratic sieve will be explored along with the multiple polynomial quadratic sieve (MPQS) and the
multiple polynomial quadratic sieve with large prime variation (PMPQS). These algorithms will be implemented and
the practical improvements using the diﬀerent variations will be tested on numbers spanning from 19 to 79 digits.
Furthermore, theory and results from abstract algebra, which is necessary in order to understand the central concepts
of the general number ﬁeld sieve, will be presented and most of the results proven. At the end a comprehensive
description of the full number ﬁeld sieve algorithm will be given and the workings illustrated using a detailed example.
A simple implementation of the general number ﬁeld sieve will also be provided.
Keywords: Integer factorization, Quadratic sieve, Multiple polynomial quadratic sieve, Large prime variation of the
quadratic sieve, Algebraic number theory, Number ﬁelds, Algebraic integers, General number ﬁeld sieve

Introduction
The fundamental theorem of arithmetic states that every positive integer (except 1) can be written as a unique product
of prime numbers. However, the theorem does not mention anything about how this unique product of primes can be
found. So, for example, if the unique prime decomposition of
n = 7293469445285646172092483905177589838606665884410340391954917800303813280275279
was to be found, would the best strategy then simply be to start dividing n by the ﬁrst prime 2, then the next prime 3
etc. until a prime factor has been found? The answer is a deﬁnite no! In this thesis the mathematical theory of multiple
integer factorization algorithms of increasing complexity will be explored. The main focus will be on the subexponential
algorithms - the quadratic sieve and the number ﬁeld sieve - since these are the fastest factoring algorithms known today.
The theory will cover many diﬀerent versions of the quadratic sieve such as the multiple polynomial quadratic sieve and
the large prime variation and most of the abstract algebra (including proofs) needed to understand the number ﬁeld sieve.
Four diﬀerent versions of the quadratic sieve will be implemented and the theoretically stated practical improvements of
the diﬀerent versions will be tested. Also a simple version of the number ﬁeld sieve will be implemented with the purpose
of showing how the diﬀerent components of the algorithm work together to factor integers. All implementations will be
written in SageMath and will be handed in along with this thesis.

Motivation - RSA
RSA is a public key crypto system and is currently one of the most widely used crypto algorithms for secure communi-
cation. It works as follows [Knu17]:
Deﬁnition - RSA cryptosystem
Let φ denote Euler’s phi function where for k ≥2, φ(k) is the number of elements a ∈Zk such that gcd(a, k) = 1.
The diﬀerent components of RSA are then deﬁned in the following way:
Public key:
• n = p · q for diﬀerent odd primes p and q
• e ∈Zφ(n)
Private key:
• p and q
• d ∈Zφ(n) such that e · d ≡1 mod φ(n)
Encryption:
ENC(m) = me mod n for a message m ∈Zn
Decryption:
DEC(c) = cd mod n for a ciphertext c ∈Zn

Notice that if an attacker can factor the public value n into the primes p and q the whole system breaks down since the
enables the attacker to calculate φ(n) = (p −1) · (q −1) and from there derive d. This shows that it should be infeasible

Given n = p · q ﬁnd p and q where p and q are distinct odd prime numbers
Notice that the problem only considers the hardest case where only two primes are searched for since, if this version
of the problem can be solved, easier version of the problem can be solved as well. The motivation for looking at the
fastest factorization algorithms is therefore to get an understanding of what a possible attacker can do against the RSA
cryptosystem (according to factorization) and how the users of RSA should choose p and q to ensure the safety of the
system.

Early factorization algorithms
Before exploring the theory of the more complicated and theoretical dense factorization algorithms such as the QS and
the NFS the early and more basic factorization algorithms will be examined. These algorithms are rather simpler and
not nearly as eﬀective as the later algorithms but they should give the reader an idea of how the integer factorization
methods developed over time and how impressively eﬃcient the QS and the NFS are.
3.1
Naive factorization
The simplest way to try factoring an integer n = p · q is by brute force. This means simply dividing n with all the primes
less than √n until a factor is found.

Input: n = p · q for diﬀerent primes p and q where p ≤q

approximately π(√n) ≈

see [Pom10]. Hence this method is really ineﬃcient. To back up this claim let’s look at a concrete example.

Notice that the choice of s has not been speciﬁed in the algorithm. Of course one can use trial and error. However more

3.3
Pollard’s rho method (1975)
Let n = p · q for distinct primes p and q be an odd number. The strategy of Pollard’s rho method [Knu19] is very similar
to the p −1 method. The idea is to ﬁnd values x and x′ satisfying
x ≡x′
(mod p)
x ̸≡x′
(mod n)
If this is the case then p divides both x −x′ and n but q does not divide x −x′ since n = p · q does not divide x −x′.
Hence gcd(x−x′, n) = p. The question is now how x and x′ can be found. To do this let f : Zn −→Zn be some function
and deﬁne f i(x) = f(f i−1(x)) for i > 1 and f 1 = f. The idea is that since Zn is a ﬁnite set the function f deﬁnes an
eventually periodic sequence:
f(x), f 2(x), f 3(x), ..., f i(x), f i+1(x), ...., f i+j(x) = f i(x), f i+1(x)....
That is, this sequence will enter a cycle which means there exist i, j such that
f i(x) = f i+j(x)
and

Does the algorithm ﬁnd a non trivial factor of n?
The short answer to this question is yes. The reason for this answer is not trivial though. Assume that k +1 pairs (xi, ai)

Lemma 1 - Number of solutions to linear system in GF(2)
Let A be an m × n matrix in the ﬁnite ﬁeld GF(2). The number of solutions to A · ¯x = ¯b is then 2n−r where r
is the rank of A
Since we are working in GF(2), r is usually less than n and hence there are multiple solutions to such a linear system.
That means in step 6 of the algorithm there are 2k+1−r multiple choices for ¯s with r being the rank of MT . Now
remember that it is stated that each possible solution has probability 1
2 of ﬁnding a non trivial factor of n and the
probability of not ﬁnding a non trivial factor of n using each possible solution becomes
P(Dixon not succeeding) = 2−2k+1−r
However this calculation assumes that each of the diﬀerent possible solutions are independent from each other. This
is not the case since if gcd(X1 −Y1, n) = n and gcd(X2 −Y2, n) = n for two solutions (X1, Y1) and (X2, Y2) then
gcd(X1 · X2 −Y1 · Y2, n) is also n since X1 · X2 −Y1 · Y2 = (X1 −Y1) · X2 + Y1 · (X2 −Y2). Hence these solutions
are not independent. Instead assuming that only the basis elements in the kernel of MT are independent, only using
these solutions to try to factor n (which is actually how the attached implementations work) would result in the following
probability:
P(Dixon not succeeding) = 2−k+1−r
(4.9)
Depending on r this probability can become really small. Another way to make sure this probability becomes small is
to ﬁnd more than k + 1 B-smooth numbers. These arguments should hopefully convince the reader that the algorithm
ﬁnds a non trivial factor of n.

5
The quadratic sieve (1981)
Shortly after Dixon released his paper on how to use the method of equal squares to factor integers, the American number
theorist Carl Pomerance invented the quadratic sieve [Pom10] which improved upon Dixon’s method. In the quadratic
sieve the factorization strategy remains the exact same as in Dixon’s method. The part of Dixon’s algorithm that got
signiﬁcantly improved was the sieving procedure. The two main questions answered by Pomerance was:
1. Instead of choosing a random x and check if x2 mod n is B-smooth can this search be structured in a better way?
2. Can the number of divisions performed in the sieve be decreased?
Luckily the answer to both of these questions turned out to be yes and in this section it will be shown how the quadratic
sieve implements answers to these questions.
5.1
The sieving polynomial
Instead of choosing random values of x and check if x2 mod n is B-smooth, the quadratic sieve uses a sieving polynomial.
This polynomial is deﬁned as follows:
Q(x) = x2 −n
Then for all value of x it must hold that x2 ≡Q(x) (mod n). Hence for values xi, Q(xi) takes the role of ai in (4.4) and
the quadratic sieve searches for pairs (xi, Q(xi)) such that Q(xi) is B-smooth. To do this in the most eﬃcient way the
value of Q(x) should be as small as possible since the probability of a smaller number being B-smooth is larger than the
probability of a larger number being B-smooth because in general the factorbase only contains small primes. Therefore
let x0 = ⌊√n⌋+ 1. Then Q(x0) = x2

3. For the elements in S that surpass some well chosen bound Btest check if the corresponding element in the list L
is B-smooth by using trial division. If the i′th element in Llog turns out to be B-smooth save the corresponding
value of Q(i) and its factorization.

instead performs additions. Furthermore trial division is only performed on elements that have a high probability of being
B-smooth if the bound Btest is chosen well.
So how do one choose the value of Btest? This is a rather important question. If Btest is set too high more if not all
B-smooth numbers will be found but trial division may have to be applied to a lot of values that are not B-smooth.
Since trial division is a tedious operation this is unwanted. On the other hand if Btest is chosen too small some B-smooth
numbers (for example those that contains a lot of prime powers) might be missed which is bad since B-smooth numbers
are not easily stumbled upon. A very natural bound to choose as suggested in [Ger83] is the bound
Btest(xi) = log2(|Q(xi)|).
Since the powers of primes are not used to sieve for B-smooth numbers in practice some small constant can/should be
subtracted from this bound to ensure that only a small amount (if any) of B-smooth values are missed. Another bound
suggested in [Con97] is the bound
Btest(xi) = log2(2 · xi · √n)
This is due to the fact that Q(xi) ≈x2

error term from the rounding errors. Now other bounds have also been found but in practice it does not really matter
too much which bound is used but the bound might need to be tuned for n such that most of the B-smooth values are
found without checking too many numbers for B-smoothness.

found. Therefore M is a parameter that can be experimented with using the quadratic sieve but since it is always a
possibility just to expand the sieving array it does not matter that much. The second thing is the numbers that are
searched through to ﬁnd B-smooth numbers. Remember that Q(x) and the inputs used are constructed such that Q(x)

6.3
Large prime variation
As mentioned before the smoothness bound B is a parameter that needs to be tuned since the theoretical optimal value
presented in the complexity analysis above is usually too large. Choosing B too large result in a couple of computational
issues. The ﬁrst issue is more or less hidden. For a large number n the sieving array needs to be quite large and a full sieve
array cannot be stored with the memory of a computer. This is solved by splitting the sieving array into diﬀerent parts.
The hidden problem comes from the fact that if B is chosen large then there might be a prime exceeding the length of
each segment of the sieve array. That means it needs to be checked whether or not this large prime ”hits” something in
each segment of the sieve array which is not to be preferred. Another more obvious problem which is already mentioned
is that choosing B large means a lot of B-smooth numbers need to be found to construct the matrix M in the linear
algebra step of the sieve algorithms. It would take a lot of memory (even with sparse encoding) to store such a matrix yet
alone to ﬁnd the kernel of it. Hence B should be chosen carefully and not too large. The large prime variation [Pom10]
is an artiﬁcial method of increasing the smoothness bound B without actually increasing it. Now assume that u is a

• A1: Version 1 (regular QS) only searching for positive B-smooth values

sieving array should be large enough such that enough B-smooth numbers are found.

Number ﬁeld sieve (1988)
In 1988 J. Pollard, who already invented two of the algorithms presented in the thesis, came up with a third algorithm
called the ”special number ﬁeld sieve”. This is a special-purpose algorithm suitable for factoring numbers on the form
re + s for small r and s. This algorithm was quickly generalized to be able to factor arbitrary numbers not close to a
prime power. This algorithm (NFS) [Pom10] (sometimes also referred to as the general number ﬁeld sieve) is the fastest
known algorithm for factoring integers with more than 110 digits. This section will be structured as follows:
• Section 8.1 contains a high level overview of NFS
• Section 8.2 contains a short description of number ﬁelds
• Section 8.3 contains a description of algebraic integers and their properties
• Section 8.4 speciﬁes section 8.1
• Section 8.5 contains a description of the sieving procedure performed in NFS
• Section 8.6 contains a description of the linear algebra used in the NFS
• Section 8.7 covers four obstructions in the NFS
• Section 8.8 covers some of the computational problems encountered in the NFS
• Section 8.9 covers the full NFS algorithm along with a comprehensive example
With this in mind let’s dive into the theory of the NFS.
8.1
Main idea of the number ﬁeld sieve
Again let n = p · q be a product of two distinct odd primes. The main idea of the NFS algorithm is actually exactly the
same as in the quadratic sieve. Hence the idea is to construct the congruence

8.2
Number ﬁelds
In this section number ﬁelds [Mar18] and some properties of number ﬁelds will be examined. Let’s start with a deﬁnition:
Deﬁnition - Number ﬁeld
Two equivalent deﬁnitions of an algebraic number ﬁeld are given
1. A number ﬁeld is a subﬁeld of C with ﬁnite dimension as a vector space over Q
2. A number ﬁeld is ﬁnite degree ﬁeld extension of Q
Later mainly the second deﬁnition will be used. This deﬁnition contains the term ﬁeld extension. A lot of theory in
abstract algebra uses the notion of subgroups, subrings, subﬁelds etc. where a larger algebraic structure is divided into
smaller parts. For example Z can be viewed as a subset of Q. A ﬁeld extension is basically the opposite of this. Instead of
dividing a structure into smaller parts a ﬁeld extension enlarges a structure (which in this case is a ﬁeld) into something
bigger. Formally:
If K is a subﬁeld of L, then L is a ﬁeld extension of K denoted L/K

8.3.3
The ring of algebraic integers is a Dedekind domain
First of all what is a Dedekind domain? A Dedekind domain [Hun74] is deﬁned in the following way:
Deﬁnition - Dedekind domain
A Dedekind domain (or Dedekind ring) is an integral domain in which every nonzero ideal factors uniquely (up
to order) into a product of a ﬁnite number of prime ideals
It should be mentioned that there exist other equivalent deﬁnitions of a Dedekind domain but this is the deﬁnition that
is useful to consider in the NFS. To understand this deﬁnition fully, prime ideals need to be deﬁned as well:
Deﬁnition - Prime ideals
An ideal p of a ring R is prime if it satisﬁes:
1. For all a, b ∈R such that a · b ∈R either a ∈p or b ∈p.
2. p ⊊R
This is simply a generalization of the regular primes of the ring Z where if a prime p divides a · b for a, b ∈Z then either
p|a or p|b. With this in mind it is now possible to state a well known theorem:

This does not mean that the above idea of using F(a, b) as the sieving polynomial should be thrown away. Actually
F(a, b) will be used as the sieving polynomial but the factorbase won’t be the same as in the previous section or in the

that one ends up with a square in Z[α] some extra work needs to be done since Example 1 shows that just using the
norm of (a −b · α) is not suﬃcient. What exactly an algebraic factorbase is will be introduced later in this section.

Deﬁnition - Algebraic factorbase
An algebraic factorbase is the set of prime ideals of Z[α] with norm less than or equal to a smoothness bound B.
As in the quadratic sieve where some prime numbers were irrelevant to include in the factorbase, some prime ideals will
also be irrelevant to include in the algebraic factorbase. Actually it will only be ﬁrst degree prime ideals of Z[α] which
leads to the natural deﬁnition of a redeﬁned algebraic factorbase:
Deﬁnition - Redeﬁned algebraic factorbase
A redeﬁned algebraic factorbase is the set of ﬁrst degree prime ideals of Z[α] with norm less than or equal to a
smoothness bound B.

vectors is (0, 1, 1) modulo 2, which is good since (3 + i) · (2 + i) · (1 + i) is not a square in Z[i].
• On the other hand F(3, 1) · F(2, −1) · F(1, 1) = 100 which is a square in Z and (3 + i) · (2 −i) · (1 + i) =
8 + 6 · i = (3 + i)2 is a square in Z[i] which is then great.
• However, as mentioned, this is not a perfect method since i for example has the zero vector as it’s exponent
vector but i is not a square.
This example shows that even though Lemma 12 takes us a long way according to ﬁnding a square in Z[α] this is still
not a suﬃcient condition. In the next subsection this problem will be solved.

Final thoughts
Remember that the whole motivation for looking at the factorization problem was the practical use of the diﬃculty of
factoring integers in the RSA crypto system. So let’s see what the diﬀerent algorithms presented in this thesis tells us
about how p and q should be chosen for RSA:
• Naive factorization: This algorithm is ensured to work. However, if p and q are chosen large enough this method
is unusable due to its time complexity. Already when p and q are around 40-digit integers this method is too time
demanding.
• Pollard’s p-1 method: The algorithm only succeeds if the parameter s is chosen correctly. s is chosen correctly
if it is bigger than the largest prime power of p −1 but smaller than the largest prime power in q −1 (or the other
way around). However if p and q are chosen such that p −1 and q −1 has at least one very large prime power as
a factor then s becomes too big for the algorithm to terminate in a reasonable time.
• Pollard’s rho method: For a well chosen function f(x) the probability of the algorithm succeeding is close to
1. However due to the birthday paradox the complexity of the algorithm is O(n
1
4 ) and hence by choosing n large
enough (for example 79 digits) this method is too time consuming.
• The quadratic sieve: Here it starts to get interesting. Using the regular quadratic sieve to factor integers is
fast, but as it can be seen in section 7.2 this method starts to become relatively slow when n is around 40 digits.
However the multiple polynomial quadratic sieve - especially the version with the large prime variation - was still
very fast. Using this method a number n of 79 digits was factored in a little under 6 days on a regular Macbook.
Now ﬁrst of all the implementations used to factor this number is not fully optimized. Second of all it is not run
in parallel. Both of these things would make the algorithm more eﬃcient. Hence n needs to be quite large for the
quadratic sieve not to be eﬀective. How large will become clear later in the section.
• The number ﬁeld sieve: When the size of n increases the test values of the quadratic sieve starts to become very
large as well and the algorithm get less and less eﬃcient since the probability of encountering smooth numbers
reduces. Here the number ﬁeld sieve takes over and for numbers n of 110 digits or more this is the fastest known
factorization algorithm. Hence it is not possible to just choose n as a 110 digit number but in fact n should be
chosen much larger to ensure security of the RSA system.

9.1
Future work
In this thesis both the quadratic sieve and the number ﬁeld sieve have been explained theoretically and implemented.
However there are still further theory and optimizations that could be investigated for both algorithms.
9.1.1
Parameter tuning
In both the quadratic sieve and the number ﬁeld sieve there are a lot of parameters that can be tweaked and tuned.
Examples are:
• The smoothness bound B. In both the QS and the NFS algorithm this value has to be chosen. If B is chosen too
small not enough B-smooth values will be encountered. However choosing B too large would require too many
B-smooth values to be found. Hence some balanced choice of B should be found to make the algorithms work
optimally. Furthermore the theoretical value of B is in practice too large and hence a smaller value suﬃces.
• The number of test values M. This parameter is mostly important in the multiple polynomial quadratic sieve.
Choosing this parameter correctly could make the sieving procedure faster which in general would make the algorithm
be more eﬃcient.
• The bound Btest and the number subtracted from this. Remember that in both QS and NFS the sieving is
performed and then one only checks numbers larger than a certain bound weather or not this number is B-smooth.
This bound need to be chosen correctly and take into consideration that the sieving is not performed with prime
powers. If this bound is too large you might miss certain B-smooth numbers. However if the bound is too small
one have to check too many numbers for smoothness. Hence this bound should be chosen with care
• The polynomial f(x) and the degree d. This is for the the NFS only. Now how d and f(x) can impact the search
for B-smooth numbers in the NFS and the method presented in this thesis is not the only way of choosing d and
f(x). This is a very complicated topic and more information can be found for example in [Bai]. It is also mentioned
in [Pom10] that f(x) do in fact not need to be monic even though f(x) is constructed that way in the algorithm
presented in this thesis.

How these parameters are chosen can have a huge eﬀect on the practical running time of the diﬀerent algorithms. Hence
it is important to be able to choose these parameters correctly and a deeper analysis of how to do this and the diﬀerent

9.1.2
Free relations and partial relations in the NFS
Similar to the large prime variation of the quadratic sieve method it is possible to obtain free relations and partial
relations in the number ﬁeld sieve. The idea is that you can obtain free relations in the precomputation phase of the
algorithm if f(x) mod p has d roots. Also partial relations can be found which corresponds to the large prime variation
of the quadratic sieve. For more details see [Pom10]. It would make sense to try to look into these methods of gaining
more relations since big practical improvements could be seen using similar methods for the quadratic sieve and it would
therefore be a reasonable assumption that this will also lead to practical improvements of the NFS algorithm.
9.1.3
Optimization of implementations
The implementations of the QS and the NFS referred to in this thesis are not optimized to its fullest. Since the numbers
in the algorithms are very big it is crucial for a fully optimized version to ensure that all operations are performed eﬃciently.
Also all subroutines should be implemented as mentioned above. Another thing that need to be considered which is barely
mentioned in this thesis is how memory is used though the algorithms. Both the QS and the NFS in particular use a
large amount of memory and how the data is stored and handled throughout the algorithm can actually have an impact
on the performance of the algorithm.
