Abstract
The problem of automatic seizure detection is both an open and a heavily researched
field because of the immense relevance it has to a wide variety of applications. Machine
learning methods are dominant in this field where some applications utilize transfer learn­
ing. These algorithms have introduced sensitive seizure detection systems but these
results highly depend on both seizure type and modalities used for data collection. The
work carried out here has made use of machine learning methods to build an automatic
multi­modal seizure detection system in an out­patient setting. The process involved a
number of steps for choosing optimal methods taking into consideration the sensitivity
and false alarm rate (FAR) trade­off. The solution is tested first on in­patient data to be
used as a baseline. Next it is tested on out­patient data where different combination of
modalities is tested and sensitivity and FAR is evaluated based on a threshold of minimum
seizure duration (MSD). Finally, transfer learning was utilized to evaluate whether train­
ing on in­patient data can improve the learning of out­patient data and result in a more
robust personalized seizure detection system. Results show that utilizing multiple modal­
ities can improve sensitivity and FAR trade­off compared to uni­modal EEG. Out­patient
seizure detection yielded promising results although it imposes ambitious challenges that
allows for further investigation of the problem.

1
Introduction
In this project, the focus is set on achieving a wearable multi­modal automatic seizure
detection system in an out­patient setting using machine learning methods. The field of
seizure detection in out­patient setting refers to the method where a patient stays in a
home setting while data is collected using fully wearable devices over a period of time.
Epilepsy is a neurological disorder with complications that include death, injury and dis­
ability. Seizures are the primary symptoms of epilepsy and treatment aims to prevent
them [1]. It is essential to accurately identify seizure occurrence and moreover, determine
whether they have stopped once therapy is prescribed[2]. The method currently used to
assess the effectiveness of therapy is to ask the patients about occurrence of seizures and
ask patients to keep seizure diaries. However, studies have shown that documentation
of seizures by patient and caregivers are inaccurate and notably the seizures are under
reported [3]. In addition to detecting and counting seizures with a systematic alternative
to seizure diaries, home seizure detection can offer a real­time seizure alarming system
which can help patients and caregivers feel safer [4].
Automatic seizure detection systems can be very beneficial.Detecting seizures in an out­
patient setting creates the potential of real­time detection of seizures which introduces the
ability to use information about seizure occurrences to make better treatment decisions
and improve patient safety [2]. The task at hand is by nature twofold as it both touches
on seizure detection using wearable multi modalities and seizure detection in outpatient
setting.
This thesis is based on a previous study that utilized wearable Electroencephalogram
(EEG), Electrocardiogram (ECG) and an Accelerometry (ACM) for automatic seizure de­
tection performed in an epilepsy monitoring unit (EMU). That study will be hereafter re­
ferred to as the in­patient study. Machine learning methods will be used where a classifier
is trained to detect seizures in out­patient setting. Moreover, it will be investigated if using
transfer learning can improve the classification [5].
1.1
Objectives
The objectives of this thesis can be divided in four different parts
• Replicate in­patient results. Extract features from in­patient data and train a clas­
sifier to replicate results from in­patient study.
• Synchronize wearable modalities. Develop a solution where modalities in out­
patient setting are synchronized and valuable information can be extracted.
• Multi­modal out­patient seizure detection. Use data collected in out­patient set­
ting to train classifier for seizure detection where modalities should be evaluated
separately and in different combinations.
• Improve seizure detection using transfer learning. Use data collected from in­
patient setting to improve seizure detection in out­patient setting.
1.1.1
Replicate in­patient results
This study is based on the in­patient study that used a fully wearable behind­the­ear EEG,
ECG, and ACM with EMU validation. The results showed that combining modalities im­
proved sensitivity and lowered the FAR compared to uni­modal EEG. Multi­modal moni­

1.1.2
Synchronize Wearable Modalities
The modalities that will be used in this thesis are the same as those used in the in­patient

develop a method of synchronizing the devices so valuable information can be extracted
and used to train a classifier.
1.1.3
Out­patient seizure detection
This thesis aims to use fully wearable devices in an out­patient setting to train a classifier
for automatic seizure detection. However, acceptance of these algorithms is contingent
on a low false alarm rate and a high sensitivity. Moreover, there are limitations to using
wearable EEG in out­patient setting which include environmental and equipment asso­
ciated artifacts, electrodes can become loose and deteriorate with long term recordings.
Battery life and inaccuracy of the device sample rate can also affect the quality of the
data. Using multiple modalities may be able to overcome these limitations.
1.1.4
Improve Seizure Detection Using Transfer Learning
Transfer learning is a machine learning method in which knowledge obtained from a model
used in a source domain is reused as a foundation point for a target domain. Transfer
learning aims to improve the learning of the target using the knowledge from the source [6].
In an in­patient setting the patient is in sedentary setting in an EMU where interference
from external factors are minimal whereas in out­patient setting the patient can move
around freely which can lead to more disruption by external artifacts. In this thesis the in­
patient data is the source domain and out­patient data is the target domain. The objective
is to train a classifier on in­patient data to improve learning on out­patient data. Moreover,
can the classification improve further by combining out­patient and in­patient data. The
goal is to investigate improvement by varying the amount of out­patient data added to the
in­patient data used in the training set.
1.2
Literature Review
The following section covers the current background related to the topics of the thesis.
1.2.1
Wearable modalities
Long­term video­EEG in an epilepsy monitoring unit (EMU) is currently the best way to
diagnose epileptic activity. It is however, both time­consuming and expensive. [7] Wear­
able devices have been proposed to overcome the limitations of the video­EEG. Such
devices aim for continuous and less expensive patient monitoring [8]. Multiple different
modalities can be used for seizure detection and selection of these modalities often de­
pend on the manifestations of the seizure. In this out­patient study EEG, ECG and ACM
will be investigated. The most important modality in epilepsy diagnosis is EEG. It also
aids in the classification of seizure types and epilepsy syndromes [9]. EEG is a recording
of brain activity that can be used to detect seizures because seizure activities appear as
abnormal signal patterns on the EEG. ECG can be used to detect cardiac events, such
as tachycardia, which sometimes is associated with seizures. Moreover, heart rate vari­
ability features can be used to detect focal seizures or added to EEG­based detection to
improve accuracy [10]. A study using wearable ECG yielded a 90% sensitivity and 1/24 h

cardia (53% of the study population) [11]. ACM measures movement and can be used
to detect seizures that manifest as movements or seizures that disrupt normal movement
patterns [12].
1.2.2
Automatic Seizure Detection using machine learning
Machine learning (ML) algorithms are trained to detect epileptic seizure signal patterns au­
tomatically. Previous research has shown that using ML to automatically detect seizures
and classify seizure types based on EEG data is feasible. [13]. Seizure detection is a
routine process in epilepsy units that necessitates the manual review of highly trained
specialists. This process can be inefficient and time consuming, particularly for long­
term recordings [14]. Many machine learning methods have been used for automatic
seizure detection. Algorithms for automatic seizure detection serves two purposes. The
first is to aid in the offline manual review of data, in which case a high sensitivity is pre­
ferred. Second is to offer a real­time seizure alarming system where a high FAR may
cause unnecessary distress and frustration for patients and caregivers [15]. The process
involves training a classifier on either patient­independent or patient­specific data to clas­
sify seizures. Many studies have been conducted for automatic seizure detection. One
study used a behind­the­ear EEG along with a standard video­EEG for the classification
of seizures in patients with temporal lobe epilepsy (TLE). The study used both patient­
independent model and patient­specific model. The automated algorithm obtained 64.1%
sensitivity and 2.8/24 h FAR with the patient­independent model and the patient­specific
model achieved 69.1% sensitivity and 0.49/24 h FAR [16].
1.2.3
Wearable multi­modal seizure detection
Multi­modal monitoring, which combines measurements from two or more different modal­
ities, can be used to improve classification accuracy beyond what is possible with a sin­
gle modality [17]. The in­patient study showed that using multiple modalities improved the
sensitivity and FAR compared to uni­modal EEG. The study used a fully wearable behind­
the­ear EEG, ECG, and ACM in an epilepsy monitoring unit (EMU). A support vector ma­
chine was trained to detect focal seizures automatically by testing different multi­modal
combinations on three different patients. When compared to uni­modal EEG, the results
showed that combining EEG, ECG, and ACM improved sensitivity and FAR trade­off. The
classification of 47 focal tonic seizures in one patient revealed an 84% sensitivity and an
8/24 h FAR. It produced a sensitivity of 100% and a FAR of 13/24 h and 5/24 in two patients
respectively, each with nine focal nonmotor seizures. Wearable multi­modal monitoring
opens up new possibilities for personalization in seizure detection development and au­
tomatic seizure detection may benefit from multiple modalities versus uni­modal EEG [5].
1.2.4
Seizure detection in out­patient setting
There is a need for more reliable seizure detection in out­patient setting, both for diag­
nostic and therapeutic purposes. Out­patient seizure detection can provide tools that can
significantly improve the patients quality of life [2]. The field of automatic seizure detec­
tion in out­patient setting is relatively new where currently a non­EEG based study has
been conducted for seizure detection in out­patient setting. The study used a wrist­worn
devices to detect generalized tonic clonic seizure (GTCS) was performed in an outpatient
setting. The study uses motion and electrodermal activity sensors. Seizures where kept
with a seizure diary as well as an expert visually inspected the data. A classifier was first
trained using clinical data that yielded sensitivity of 95% and FAR of 6.1. Second the out­
patient data was added to the training which reduced the FAR to 2. Tuning the decision
threshold resulted in sensitivity of 93% and FAR 0.58 on 111 patients with GTCS [18].

1.2.5
Seizure detection using transfer learning
Transfer learning can be beneficial to solve the problem of serious shortage of training
data and improve the accuracy of classification [19]. Different methods have been used for
transfer learning for seizure detection where the source is based on patient­independent
data. To our knowledge no study has been performed using a patient­specific source to
improve learning on a patient­specific target where both source and target data come from
the same patient. However, there are some studies that have utilized transfer learning
for seizure detection by combining patient­independent data and patient­specific data for
the training. A study using ECG was performed utilizing transfer learning. The study
included a transfer learning approach that used preexisting data from multiple patients to
train a classifier and then compared it to adding 0­4 patient­specific seizures to see if a
more perzonalised classifier would give better results. The algorithm was tested for 24
patients with temporal lobe epilepsy, including 227 focal impaired awareness seizures.
The proposed, more personalized approach resulted in an overall sensitivity of 71% with
1.9 false detections per hour. This is an average decrease in false detection rate of 37%
compared to the reference patient­independent algorithm. The proposed transfer learning
approach adapted faster and more robustly to patient­specific characteristics as the ictal
heart rate changes are patient­dependent [20].

Methods
2.1
Data Collection
The out­patient data was collected over the course of five days from 17 different patients.
Each patient had wearable devices and no other monitoring methods. The devices in­
cluded behind­the­ear EEG, ECG and ACM. The EEG included two electrodes attached
behind each ear and a reference electrode in the cranial part of the columna sampling at
500 Hz. Single channel ECG with patch­electrodes was used with 500 Hz sampling rate
which also included the ACM sampling at 25 Hz. Four of the patients had been partici­
pants in the previous in­patient study and only one of them had enough recorded seizures
in out­patient data to be used for transfer learning. For those reasons, only this patient
will be used in this study.
2.2
Patient
The patient turned off the ECG device and took off the EEG electrodes during the day.
There were only recorded seizures during night 3 and 4. Therefore, the data that will be
used is two ECG recordings for nights 3 and 4 which will be referred to as ECG recording 3
and 4 and a single EEG recording spanning both nights. The patient had 44 seizures in in­
patient setting and 15 seizures in out­patient setting. The patient experiences focal motor
seizures with tonic posturing. Tonic seizures are characterized by the sudden onset of
stiffness or tension in the muscles and electromyographic activity is dramatically increased
[21].
2.3
Feature Selection
The features used in the in­patient study produced promising results. As a result, the same
features will be used for the out­patient data with minor changes to frequency domain
features. In the in­patient study, fewer power bands were used whereas in this study the
power bands will be divided into more bands with a narrower frequency range. For this
thesis, features were extracted from both in­patient and out­patient data using the same
methods.
2.4
Electroencephalogram Feature Extraction
The EEG recording included two channels and a third channel was made by subtracting
channel 1 from channel 2 to create a left­ to right­ear channel. Each channel was epoched
into a 2 second window with 50 % overlap. Each feature was then calculated for each
epoch. The features are both time domain and frequency domain features. The time
domain features were preprocessed using a 1­70 Hz band­pass filter and a 50 Hz band­
stop filter. A band­pass filter passes frequencies within that specific range and rejects
frequencies outside that range. The Band­stop filter attenuates the 50 Hz frequency to
a low level which was done to reduce the noise of power line interference. The features
can be seen in table 2.1.

Time domain features that were used are skewness, kurtosis, root mean square (RMS),
zero crossings and sample entropy. Skewness and kurtosis provide information on the
amplitude distribution of a time series. Specifically, skewness and kurtosis give an indica­
tion of the shape of the distribution. Skewness is a measure of asymmetry of a distribution
and kurtosis is a measure of weather the data is heavy­tailed or light­tailed relative to a
normal distribution. RMS measures the magnitude of the varying quantity and is a good
signal strength estimator in EEG frequency bands. Zero crossing is a point where the
sign of the amplitude changes, represented by an intercept of the axis. It is the number
of times the amplitude crosses 0 for each epoch. Entropy has been used as a measure
of the complexity or uncertainty of an EEG signal, where the more chaotic the signal is,
the higher the entropy is.

2.5
Electrocardiogram Feature Extraction
The features used from the ECG are Modified Cardiac Sympathetic Index (ModCSI) and
Modified Cardiac Sympathetic Index with Slope (ModCSISlope) for 100 previous R­R in­
tervals. The features are calculated based on R­R intervals which can be calculated from
R­peaks in the ECG data. The R­peaks are extracted using EDF browser where a times­
tamp of each R­peak is listed. The features could then be calculated as described in [11],
[22]. Since the features are based on R­R intervals the samples are not regular. The
samples need to be resampled to 1 Hz which was done using interpolation.

2.6
Accelerometer Feature Extraction
3­dimensional ACM is used, measuring 3 movement axes. Mean and standard deviation
(std) was calculated for each of the axis in non­overlapping 1 s window resulting in 6
features.
2.7
Electrographic Seizures
The EEG recording was visually inspected and all seizures were listed down using the
timestamps from the EEG. Total of 15 seizures were confirmed visually over two nights.
Since there are no other methods of monitoring, transient signs and/or symptoms can
not be confirmed. Therefore these seizures are classified as electrographic seizure [23],
[24]. A binary vector for the seizures was created where a seizure is marked as 1 and
non­seizure as 0 for each second.
2.8
Synchronize Features
The issue with using long­term recordings from different modalities is that the devices
can have differing sample rates. Because the timestamps of the recordings are based on
the sample rate, a time lag develops if the sample rate is not accurate. Furthermore, the
lag increases with the length of the recording. Seizure onset was listed using the EEG
timestamp and therefore, it is essential that ECG and ACM are aligned with the EEG so
the features are represented correctly. To account for this lag, the method first determines
how much lag exists between the recordings. Then, based on this lag, it is possible to
calculate how much the sample rate differs. After calculating the difference, the ECG
and ACM can be resampled to correct for this lag. Furthermore, R­peaks can be seen
in the EEG on occasions, so an R­peak comparison can be used to align the signals.
To compare the R­peaks, a band­pass filter must be applied to the EEG to make the
R­peaks more visible. Moreover, the filter needs to remove other artifacts so that cross­
correlation can be applied once the lag between the signals has been estimated using
R­peak comparison. The ECG and ACM features can then be shifted by the constant that
shows the maximum cross­correlation.
2.9
Classification
A two class classification using a support vector machine (SVM) with a radial kernel was
performed for the seizure detection as it has provided good results from the in­patient
study. The same hyper parameters were used as in the in­patient study. To account for
the class imbalance in the data set, different sampling techniques were tested. The model
allows for a detection delay of 60 seconds so if the model predicts a seizure 60 seconds
before or after the actual seizure then it counts as a true positive. Since all features are
now sampled at 1 Hz the model makes a prediction for 1 second at a time. It is known that
seizures are not 1 second long so the prediction is reversed if there is a single positive
or negative detection between opposite classes. If there is a seizure within 2 minutes
of another seizure, only the first seizure is counted and the other is discarded to avoid
multiple detections for the same seizure. The same classifier was used for all cases in
this thesis however, a different approach was used for the training process using cross
validation.
2.9.1
In­Patient and out­patient training
To avoid over­fitting a cross validation was performed where all features were centered
and scaled for the training set for each fold. Each fold was created by splitting the data
at the midpoint between each seizure. The same approach was used for in­patient and
out­patient classification where a leave­one­seizure­out cross validation was performed.

In each fold the classifier is trained on all seizures except one which is left out for testing.
The average from each fold is used as the final result for each combination of modalities.
2.9.2
Transfer Learning training
For Transfer learning, the classifier is first trained on all the in­patient data and tested on
all the out­patient data. Next the classifier is trained on the in­patient data and a section of
the out­patient data is added to the training set. This was done for three separate cases
where first cross validation was performed using 1 out­patient seizure in addition to in­
patient data in a 15 fold cross validation, then adding 3 out­patient seizures to training set
in a 5 fold cross validation and finally a 3 fold cross validation was performed adding 5
out­patient seizures in each fold.
2.10
Performance Metrics
Performance metrics are based on sensitivity and FAR. Sensitivity shows the percentage
of seizures that were accurately predicted whereas FAR tells us how many false positives
are predicted over a span of 24 hours. The desired result is having a high sensitivity and a
low FAR. However, there is a trade­off between sensitivity and FAR where the higher the
sensitivity, the higher the FAR. This trade­off is determined by a minimum seizure duration
(MSD). The MSD serves as a threshold which describes how long a seizure has to last in
order for the classifier to predict it as a seizure. The classifier was tested using MSD of 5
s up to 60 s with a 5 s increment. The results are then used to create a receiver operating
characteristic (ROC) curve which is a graphical plot that illustrates the sensitivity and FAR
as MSD is varied.

3.4
Imbalanced Dataset
The out­patient dataset is imbalanced where 99.6% of the data belongs to the non­seizure
class and 0.04% belongs to the seizure class. SVM is sensitive to class imbalance where
an SVM trained on an imbalanced dataset can produce suboptimal models that are bi­
ased toward the majority class and have low performance on the minority class. There
are multiple methods that can be used to counter this imbalance and the optimal solution
is often dataset dependant [25]. Because the synthetic minority oversampling technique
(SMOTE) was used in the in­patient study, it is already known that SMOTE can produce
good results for this type of dataset. When replicating the in­patient results it was dis­
covered that the computational time was too long where it took about 24 hours to finish
one out of the 44 folds in the cross­validation. As a solution a combination of SMOTE
and random undersampling was used. It is important to find the right balance between
SMOTE and undersampling since when undersampling you risk loosing important infor­
mation as the process throws away valid instances, which contain valuable information
[26]. However, SMOTE increases computational time. The balance between those two
was found by testing various parameters. The optimal results were discovered by com­
paring the in­patient results from the previous study to the replicated results and finding
the point where the result were most similar and minimizing computational time. As a re­
sult, the majority class was undersampled by half and the minority class was oversampled
with SMOTE to match, resulting in a balanced dataset and computational time decreased
from the estimated 44 days to 2 hours for in­patient data.

Conclusion
The process of manually reviewing EEG recordings can be a long and expensive process.
Moreover, efficacy decreases with longer recordings. To improve treatment of patients
with epilepsy it is crucial to get the correct overall image of how often the patient expe­
riences seizures. Automatic seizure detection in out­patient setting can both be used to
improve clinical management and help medical professionals by shortening the review
process and it can be used for real time seizure alarming system [15]. This study is the
initial investigation into whether automatic seizure detection in out­patient setting is feasi­
ble, whether using multiple modalities can improve sensitivity as well as lowering the FAR
and whether using transfer learning can improve the results further.
Visual comparison of the EEG features between in­patient and out­patient setting showed
that seizures presented similarly. The relative power features show that most of the EEG
activity comes from muscle activity. The movement manifestation of the seizure conceals
the brain activity in the behind­the­ear EEG. The patient experiences ictal tachycardia
which translates to distinct spikes in the ECG features. Moreover, ACM features show
low activity during a seizure and high activity right after the seizure. This indicates that
each modality contains unique information that can be used in combination for this seizure
type.
However, in order to combine these modalities they must be synchronized. Devices were
discovered to have differing sampling rates which can have significant impact on the result.
When using longer recordings with differing sample rates, a lag between the modalities
develops over time. The modalities were synchronized by comparing the R­peaks in the
EEG and ECG and then using cross­correlation to align the signals.
The biggest challenge in replicating the in­patient results came from computational time.
When using longer recordings, the data overload can become problematic, especially
when the class imbalance is high and sampling techniques are utilized. As a result, a
combination of SMOTE and undersampling was used, and the optimal balance between
the two was discovered by obtaining results that were comparable to those obtained in the
previous in­patient study and optimizing computational time. This reduced computational
time from an estimated 44 days to 2 hours.
The results from replicating the in­patient study confirmed that a better sensitivity and FAR
trade­off can be accomplished by combining modalities which resulted in 100% sensitivity
and 29/24 h for 5 s MSD and 86% sensitivity and 9/24 h FAR for 15 s MSD which is similar
to previous study that found all modalities resulted in 91% sensitivity and 20/24 h FAR with
15 s MSD and 25 s MSD yielded 84% sensitivity and 8/24 h FAR.
The results from the out­patient classification showed, when using all modalities, 93%
sensitivity and 6.7/24 h FAR with 5 s MSD and 86.7 % sensitivity and 0.84/24 h FAR for
10 s MSD. Multi­modal detection using all modalities resulted in lower FAR with similar
sensitivity compared to uni­modal EEG where EEG yielded 100% sensitivity and 21/24 h
FAR for 5 s MSD and 86% sensitivity and 6/24 h FAR for 10 s MSD.
The ability to collect data in an in­patient setting and use it to detect out­patient seizures
is contingent on modalities. Multi­modal EEG and ECG gives higher sensitivity and lower
FAR then uni­modal EEG. The ACM does not perform as well when transferred from one
domain to the other which is caused by different circumstances when data was collected.

The optimal sensitivity and FAR trade­off from out­patient classifier came when using all
modalities however, in the transfer learning the best trade­off was when using multi­modal
EEG and ECG. The transfer learning classification shows improvement when adding 1
out­patient seizure to the training set. The learning improves only marginally when adding
3 or 5 out­patient seizures to training set.
Based on these results, EEG and ECG data could be collected in in­patient setting and
used to train a classifier. Next, data would be collected in out­patient setting and tested
on the classifier. Initially the classifier would give 93% sensitivity and 49/24 FAR. 49
epochs for each day would have to be manually reviewed and marked as either seizure
or non­seizure. The classifier could then be retrained with this new out­patient data and
then again, more data would be collected. When 5 out­patient seizures have been added
in the training set and again tested on new out­patient data the classifier now has 93%
sensitivity and 20/24 h FAR. That leaves 20 epochs to be manually reviewed for each
day. This process could then be repeated multiple times, improving the review process
and resulting in a more robust personalized classifier. In that case a higher sensitivity and
FAR would be preferred as it would capture most seizures and the FAR would decrease
when retraining the classifier making the review process quicker each time.
The purpose of using transfer learning is to investigate if sensitivity and FAR trade­off
could be improved compared to only using out­patient data to train the classifier. The
results show that even though performance improves when adding more out­patient data
to the training set, the sensitivity is similar to the out­patient classifier and FAR is higher
for the transfer learning classifier. However, the transfer learning method only uses a
maximum of 5 out­patient seizures so it’s possible that adding more out­patient data to
the training set would ultimately lead to the transfer learning to out perform the out­patient
classifier.
This proof­of­concept study on out­patient seizure detection on one patient shows promis­
ing results where wearable ECG, ACM, and behind­the­ear EEG in combination can de­
tect ictal changes in focal tonic seizures. Utilizing multiple modalities can improve sensi­
tivity and FAR trade­off compared to uni­modal seizure detection. As this was only studied
for one patient, further investigation of other seizure types would be beneficial to deter­
mine if similar results are feasible for other patients.
