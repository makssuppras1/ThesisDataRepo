Abstract
In later years, increasingly complex prestressed concrete structures are constructed.
This development, creates a demand for calculation tools to handle these complex prob-
lems, not only accurately, but also e’¼āciently.
In this thesis, a program for e’¼ācient
calculation of anchor and deviation forces on prestressed concrete structures with arbi-
trary tendon geometry is developed in the programming language FORTRAN. Input in
the program is the topology matrix and nodal coordinates from a Finite Element model
consisting of linear eight-node volume elements. In addition, a spline function represent-
ing the tendons is speci’¼üed through coordinates at certain locations, and the slope of the
tendon at these locations. The output is a load vector containing equivalent nodal forces
due to prestressing. This load vector can in principal be imported to any commercial
Finite Element program.
To represent the tendons with a high degree of accuracy, while keeping the input
simple, a Hermite spline function is implemented. It was found that this kind of spline was
able to model the tendon in an accurate way. To determine intersection points between the
tendon and faces of the volume element mesh, plane equations are generated for selected
element faces. An alternative method for determining intersection points is introduced,
thus avoiding a traditional iterative approach commonly used. Isoparametric coordinates
for the intersection points relative to the intersected element are determined based on an
iterative Newton-Rapshon solution scheme. The iterative procedure is found to converge
rapidly. Shape functions are used to calculate equivalent nodal forces from the tendons
on the volume element model, through a simpli’¼üed approach where a point load on the
volume element face is interpolated to nodal equivalent forces. All tests performed thus
far, indicate that this is an e’¼ācient and practical way to represent anchor and deviation
forces within a desired degree of accuracy. The opportunity to calculate friction loss for
arbitrary tendon geometry is also implemented in the program. For regions with constant
friction loss, draw-in loss is also included.
Overall, as a tool for e’¼ācient calculation of anchor and deviation forces this program
performs well. Based on tests, it is proposed that an even simpler input for the tendon
description would be an improvement.

1
Introduction
1.1
Problem Description
In todayŌĆÖs society, the demand for complex structures are ever more pressing. Not only does
the structures grow in size and complexity, but aesthetics are also important. Prestressed
concrete has the advantage over non-prestressed concrete in that compressive stresses are
applied, reducing tensile stresses in the concrete and therefore reducing cracking. This implies
sti’¼Ćer members, and the ability to control de’¼éection. A larger part of the concrete is utilized,
and thus lighter and more slender members can be achieved. Not only does this make it
possible to construct structures which otherwise would have been impossible, but the slender
structural concrete members are usually also considered aesthetically pleasing.
Modeling of prestressed concrete structures has previously been a time-consuming task, es-
pecially when using solid elements for the concrete structure. Historically, including tendons
in the Finite element (FE) -model puts considerable restrictions on the mesh, as the interac-
tion between tendons and concrete has usually been highly mesh dependent. Thus, complex
concrete structures or complex tendon geometry makes the mesh generation troublesome.
Usually, changes to the geometry of a FE-model is inevitable during projects, and these
geometry changes will induce even more time consuming mesh modi’¼ücations.
1.2
Scope
In the design phase of a project, changes to the model may occur relatively often. After each
design iteration, it is vital to update the model with correct load cases in an e’¼ācient manner.
To facilitate this, the analyst should be able to model the concrete structure independently
of the tendons. After a concrete FE-model is established, a stand-alone program could be
executed in order to e’¼āciently calculate anchor and deviation forces, included tension losses
such as friction and draw-in loss from the tendons. This would enable the analyst to model
complex concrete structures with any general tendon geometry, independent of the mesh.
The aim of this thesis is to create a stand-alone program written in the programming language
FORTRAN. The purpose of the program is to e’¼āciently calculate deviation and anchor forces
from post-tensioned cables, when the concrete is modeled using volume elements. This is
something that is often achieved in the industry today by either very simpli’¼üed approaches,
or time consuming sophisticated methods.
The idea in this program is that by simulating each post-tensioned cable with splines, it is
possible to describe the tendon geometry by a high degree of accuracy. The spline/tendon will
1
==PAGE:14==
cross the concrete volume element boundaries in two given points, namely an entry and exit
face. At these points, anchor and deviation forces may be calculated. Although this thesis
is mainly concerned the the prestressing method of post-tensioning, prestressing is the term
that will be used throughout the thesis as a more general term, since many of the concepts
would also apply to pre-tensioned concrete.
The program will consist of several steps. First of all, a routine is needed to be developed in
order to e’¼āciently locate intersection points between the tendon and volume element faces.
This routine is based on the work started by (Elwi and Hrudey, 1989) and most recently
improved by (Markou and Papadrakakis, 2013). After ’¼ünding intersection points in global
coordinates, a routine for determining the isoparametric coordinates of the intersection points
relative to the considered element is implemented, based on the work by (Elwi and Hrudey,
1989) and later (Barzegar and Maddipuddi, 1994). After the isoparametric intersection points
on an element are found, the point load from the tendon at this location, is interpolated to
equivalent nodal forces through shape functions based on the isoparametric coordinates.
1.3
Overview
This thesis is separated into three main parts:
ŌĆó Section 2: Historical and theoretical background.
ŌĆó Section 3: Description of the implementation for the developed program.
ŌĆó Section 4: Demonstration of the program through three cases.
The ’¼ürst main part, Section 2, provides a brief overview of the development of prestressed
concrete as a concept. Also, a literature review of previous work on including reinforcement in
FE-models is performed. The focus of this review is on how the reinforcement is described in
the model, and related to the concrete elements. Especially, the algorithms used to ’¼ünd inter-
sections points between reinforcement and concrete elements are of interest. The theoretical
groundwork for the principals of prestressed concrete used in the thesis is also established.
The focus will be on anchor and deviation forces caused by the tendons, as well as tension
loss caused by friction and draw-in of wedges.
The second main part, Section 3, provides a theoretical description of the developed program
for this thesis. First, a general overview of the calculation steps is provided as a ’¼éowchart.
Then, the section is broken down into subsections, describing the main calculation steps in
the order they are actually needed in the code. The focus is on explaining the important
concepts that make up the code.
2

The third main part, Section 4, demonstrates the developed program. Three cases are con-
sidered. The ’¼ürst case, is a statically determinate beam, for comparison to hand-calculations.
Then the slightly more complicated case of a ’¼éat-slab is considered. In the last and most
complex case, a concrete cylinder of varying diameter and cross-section is used to demonstrate
the e’¼āciency of the program.
In the end, a discussion, recommendation for further work and a conclusion is included.

2.1
Historical Background
Prestressing of concrete is an idea that has been around for some time. The ’¼ürst patented
design for prestressed concrete was by the American Peter H. Jackson in 1888. He was the
’¼ürst to patent what can be recognized steel ties, threaded though the concrete, tensioned,

Unfortunately for Jackson, the technology and the materials at the time was not quite up
to the task, and although the concept was promising, it was not very e’¼ācient in practice.
The biggest challenge, was the steel used for the prestressing. The challenge was that over
a certain amount of time after the concrete was prestressed i.e. precompressed, it continued
to shorten due to creep and shrinkage, amounting to a total shortage of almost 1 part per
thousand (Collins and Mitchell, 1991). Because high-strength steel had not been invented,
and the normal strength steel which was used at this time could only be elongated 1.5 parts
4
==PAGE:17==
per thousand, about two thirds of the prestressing was lost, rendering the method unpractical.
The person behind practical prestressed concrete was Eugene Freyssinet (Collins and Mitchell,
1991). The breakthrough in practical prestressing was facilitated though the use of high-
strength steel wire for the prestressing. The key element was that high-strength steel could
be elongated to about 7 parts per thousand, meaning that a large potion of the applied
prestress remains, even after creep and shrinkage (Collins and Mitchell, 1991). In addition to
introducing the use of high strength steel, Freyssinet also stated the importance of using high-
strength concrete, to enable a higher level of stress in the concrete. FreyssinetŌĆÖs pioneering
within the prestressing technology, made it the useful construction technique it is today.
However, not much material or information was available relating to prestressing technology
and theory. Due to this, the technology was not as widely used, because engineers at the
time lacked the theoretical background in this area. All this changed when the ’¼ürst person to
write a complete text book on the subject, Gustave Magnel, in 1948 published the book titled
Le B┬┤eton pr┬┤econtraint. In this work, Magnel outlined the theoretical principals of prestressed
concrete and highlighted its strengths and usefulness. Magnel was excellent at explaining
complex concepts in a simple manner.
The following, is a short summary of the way he
outlined the strengths of prestressed concrete:
- If no reinforcement is present in the concrete cross-section, the member would fail
in tension after ’¼ürst cracking.
If reinforcing bars are included in the concrete, the
section may continue to resist tensile forces after the ’¼ürst crack until yielding of the
reinforcement crossing the crack. Inducing precompression from the prestressed steel,
’¼ürst cracking is postponed, utilizing the whole cross section for tensile capacity longer,
and thus increasing the sti’¼Ćness of the member. As prestressing eliminates or limits
cracking at service loads, and provides the ability to control de’¼éection, slender structures
are often the result. In corrosive environments, control of cracking through prestressing
can be highly advantageous.
Through the work of these pioneers and many others, prestressed concrete has become an
important and widely used construction method for concrete structures all over the world.
It is used in a wide range of structure types. Today, more than 50 percent of all bridges
are constructed using prestressed concrete (Collins and Mitchell, 1991), and in the o’¼Ćshore
sector, the CONDEEP foundations illustrated in Fig.2 are made possible partly due to the
prestressing technology. They also illustrate that prestressed structures can have a complex
geometry, and enormous scale.

tures. Over the years, several approaches for modeling reinforced concrete has been practiced.
In general, three approaches have been used for modeling of reinforcement in concrete. These
are the smeared, the discrete and the embedded approaches. Smeared representation of the
reinforcement contribution is generally suitable for homogeneously distributed reinforcement.
The discrete approach may be suitable for simple structures, where reinforcement are repre-
sented by truss elements. The truss nodes, and surrounding concrete nodes must be lined
up and connected. This makes for a rather cumbersome procedure for complex structure
geometries or if curved reinforcement or tendons are to be modeled. However, it is some of
the work done within the embedded approach that this thesis is built upon. In the embedded
approach, the reinforcement can be modeled rather independently of the mesh, making this
an attractive approach.
Some of the ’¼ürst attempts at modeling reinforced concrete via the embedded approach was
made by (Phillips and Zienkiewicz, 1976) and (Elwi and Murray, 1980). These approaches
made it possible to use a mesh which did not require rectilinear elements, thus being useful
for modeling curved structures.
The challenge with the embedded description, was that
the reinforcement coordinates had to be known in the isoparametric coordinate system of
the parent element in order to relate the two. The isoparametric coordinate space for an
element is normalized with the elements dimensions such that it has a domain from ŌłÆ1 to
+1. This was not a problem for regular rectilinear elements meshes, and reinforcement that
follow an element boundary. However, for arbitrary tendon geometry trough an element, the
isoparametric coordinates were not generally available.
A two-dimensional model for describing the geometric relations required for an embedded
’¼ünite element representation of generally curved reinforcing bars or tendons were ’¼ürst devel-
oped by (Elwi and Hrudey, 1989). In this paper, a routine for locating the reinforcement in
the isoparametric coordinate system is presented, and an isoparametric mapping algorithm
was developed. The concrete element, shown in Fig.4, was described using global coordinates.
The reinforcement is then described though something termed layer nodes shown in Fig.4.
The layer nodes, are found at points where the reinforcement crosses the concrete element
boundaries. Reinforcement locations within the element or between layer nodes, are then
found by interpolated of these. If the reinforcement is curved, additional layer nodes within
the concrete element is used to describe the reinforcement. Thus, both the reinforcement layer
nodes, and the concrete element are ’¼ürst described in global coordinates. However, Elwi and
Hrudey wanted to include the sti’¼Ćness contribution from the reinforcement, on the concrete
elements. For this, they used an incremental virtual fork formulation to derive the various
sti’¼Ćness terms. To perform the virtual work integration, they needed to determine the strain

in the concrete element for points on the reinforcement. These points, known in global coor-
dinates by the layer nodes, needed to be determined in associated isoparametric coordinates
within the concrete element, as the integration is performed over the isoparametric space.
Thus just as this thesis need the isoparametric coordinates of the intersection points within
an element in order to interpolate nodal forces, Elwi and Hrudey needed them for the sti’¼Ćness

Thus far in history, the embedded formulation has been restricted to 2D applications and
models. One of the reasons for this is that there was an inherent requirement that the global
coordinates of all intersection points or layer nodes in Elwi and HrudeyŌĆÖs formulation between
the reinforcement and the concrete elements had to be provided manually by the analyst. For
3D application this would be a overly time consuming task. In 1994, Elwi and HrudeyŌĆÖs
iterative formulation was expanded to 3D by (Barzegar and Maddipuddi, 1994). In addition
to extending the formulation, an autonomous procedure for calculating the intersection points
between the reinforcement, and volume elements were introduced, making the 3D formulation
useful in practice. In this approach, the reinforcement was described by straight line segments,
where the analyst only had to give the end-point coordinates P1 and P2, in Fig.5 as input.

these coordinates were within ┬▒1, then the point lies within the boundaries of the element.
If the condition is not satis’¼üed, the same procedure is performed on the next volume element
until the condition is met. This calculation required the above mentioned iterative procedure,

3.1
Introduction
A routine for modeling tendons, and calculating deviation forces will consist of several sub-
steps. In this section, the procedure developed for calculation of these forces is broken down
into sections, introduced chronologically in the order they are needed for the program to
run. A general overview of the procedure is outlined in the ’¼éowchart shown in Fig.17. The
procedure is roughly divided into two main parts. The ’¼ürst part, from start to the beginning
of the loop, consist of locating the ’¼ürst entry point of the tendon in the structure.
The
element at this location is determined and forces are applied to the elements entry and exit
face. After locating the entry and exit point of this ’¼ürst element, the information about the
exit face is used to determine the next element the tendon enters, leading to the second main
part. In the second main part, a loop over all remaining elements are performed. Intersection
points are e’¼āciently found by using information about the exit face of the previous element,
and loads are applied at the respective elements in each loop. The calculation steps making
up the procedure can be broken down into seven parts;
1. Generation of Splines: In order to describe the tendon geometry, the spline function
is called from a sub-routine. The input required from the analyst are the coordinates of
the cable at a chosen number of locations, and the slope of the tendon at these locations.
2. Generation of Planes: The global coordinates of the start-point on the tendon is
compared to the coordinates of all nodes of the volume elements. There may be eight
solid elements connected to the resulting node. The spline equation are therefore com-
puted against the equations for the six-element faces of all eight-elements. In the next
section, it is determined which of the elements are actually intersected.
3. Determining Intersection Points and Intersected Elements: By computing the
spline equation and the plane equations, it is ’¼ürst determined if the tendon is crossing
an in’¼ünitely large plane on the surface of each element face. For the faces having planes
crossed by the tendon, a ŌĆØvector-checkŌĆØ is used to determine if the tendon is passing
the plane inside or outside the boundaries of the elements. This should result in one
entry face and one exit face corresponding to one element.
4. Calculation of Isoparametric Intersection Coordinates: In this step, the isopara-
metric coordinates of the intersection points with regards to the intersected element is
calculated.
5. Calculation of Equivalent Nodal Forces: The force components of the tendon at
21
==PAGE:34==
the intersection points are calculated. Using the isoparametric coordinates, these force
components are interpolated to equivalent nodal forces on the volume element.
Once this procedure is performed for the ’¼ürst element, all other intersections are found based
on the information about the exit face of the previous element. The exit face of the previous
element, will always be the entry face of the next element.
This means that instead of
performing a search over all elements for the next intersection, known information is used
to continue directly to the next element by utilizing the topology matrix. The loop over the
remaining elements is therefor from step 3 - 5 in the above step-by-step description.

3.2
Generation of Splines
The tendon pro’¼üle needs to be described through a known function, such that this function
can be used together with the plane equations to ’¼ünd tendon-solid element intersections. In
general, the mathematical functions used to describe the lines and curves in a commercial
FE-program is not readily available. A routine for generating the tendon geometry is there-
fore created, such that the mathematical expression describing the tendon is known to the
user, and can be used for intersection calculations. The complete tendon pro’¼üle could in
principle be described from the start to the end-point by interpolation using a polynomial
function. However, the complete interpolation from the start to end-point would possibly
require a polynomial expression of a relatively high degree, and thus introducing both un-
necessary complexity and possibly numerical instability through increasing function oscilla-
tion (Kreyszig, 1999). When describing a tendon, two parameters are of special importance.
One, is the location/coordinates of the tendon at certain points, while the other is the slope
of the tendon at said points. It would therefore be convenient, to describe the tendon pro’¼üle
using only these two parameters as the input. Therefore, instead of describing the complete
tendon pro’¼üle, the pro’¼üle is divided into several sub-domains between points of special inter-
est where location and slope is provided. Spline functions can be used to interpolate values
between these points. Spline functions are typically lower order polynomial expressions, and
has their domains within each sub-interval and thus the possibly unfavorable function oscil-
lation is eliminated. Each spline, interpolating the tendon between two speci’¼üed points can
be combined in order to describe the complete tendon geometry. Thus, the complete tendon
is described through piecewise lower order polynomial interpolation (Kreyszig, 1999).

The concrete structure, in which the tendons are embedded, is modeled using eight-node linear
volume elements. As the load from the tendon are calculated and applied at each face of the
volume elements, it is important to establish the intersection points between the tendons, and
the faces of all the volume elements it passes though. For reference, Fig.22a illustrates such a
volume element. This is an eight-node hexahedron. A hexahedron is formally a polyhedron
with six faces, eight corners and twelve edges or sides.
In order calculate intersection points between the tendon and a volume element face, two
things are essentially needed. One is the equations for the faces/planes of the volume elements,

Discussion
5.1
E’¼āciency
The driving factor behind many of the decisions made in this thesis is e’¼āciency.
It was quickly realized that the crucial factor for the e’¼āciency of this program, is the method
used when determining tendon-volume element mesh intersections. In the development of
this program, several concepts were tested and considered in this regard. In the ’¼ürst design
iteration, all intersection points were determined by equating the tendon function against the
plane equations for all elements in the model, even after locating the ’¼ürst intersected element.
This proved to be a very time consuming procedure. Calculating all intersection points for
the GBS case above, needed a computational time of around half an hour as it needed to
constantly search through large arrays. Because of this, it was considered to implement a
similar geometrical constraint as used in (Markou and Papadrakakis, 2013) and discussed
brie’¼éy in Section 2.1. There, the constraint made sure that the search for tendon-volume
element intersections were only performed in the vicinity of the actual tendons, and thus
reducing the computational time greatly. However it was realized that an even more e’¼ācient
approach could be implemented by utilizing the topology matrix, and information about the
exit face of the previous element as done by (Barzegar and Maddipuddi, 1994) for intermediate
intersection points as described in Section 2.1. This was the ’¼ünal implemented procedure,
as described in Section 3.4. The result of this procedure, is that the computational time
needed for calculating the intersection points after the ’¼ürst intersection is found is no longer
dependent on the number of elements in the model, but rather the number of elements the
tendon intersects.
The Newton-Raphson iterative procedure for calculating isoparametric coordinates described
in Section 3.5, seems to converge rapidly as reported by both (Barzegar and Maddipuddi,
1994) and (Markou and Papadrakakis, 2013). During testing, the intersected volume elements
were given a skewness well beyond recommended values. The routine still found a solution
within a few hundred iterations, within a relative error of 10ŌłÆ9. Therefor, no modi’¼ücation
was considered for this method.
As described in Section 3.4, all possible intersection points with a plane parallel to each
element face is found, and then a procedure is executed to determine if the intersection
is actually within an element boundary or not. The traditional way of doing so, as done
in (Markou and Papadrakakis, 2013) is by checking if the isoparametric coordinates of the
intersection point lies within ┬▒1 for the considered element. However, as this requires the
above mentioned iterative method, the vector check described in Section 3.4 was implemented
75
==PAGE:88==
in this program. As mentioned above, the iterative procedure is usually relatively e’¼ācient,
but it is still considered preferable to avoid an iterative procedure if possible, making the
calculation simpler and more straight forward. For large models, the di’¼Ćerence in e’¼āciency
might also still be notable.
5.2
Accuracy
As described in Section 2.2.1, the anchor forces should be represented by the prestressing force,
and the angle at which the tendon is anchored to the structure. In the procedure described
in in Section 3.6, the anchor forces P1 and ŌłÆP2, using Fig.31a as an example are applied in
the direction of line though the ’¼ürst and last element. Thus, in the developed program, the
anchoring angle is not strictly the same as the angle at the entry point, but rather the angle
created by the line from the entry to the exit point of the ’¼ürst and last element. As the angle
at the anchor point, and the rate of angle change is usually small compared to the size of an
element, this di’¼Ćerence is usually insigni’¼ücant. However, the analyst should be aware of the
di’¼Ćerence.
As also shown in Section 2.2.1, the deviation forces are caused by the change in angle along the
tendon. In the developed program, the deviation forces originate from change in angle/change
in direction cosines between each element, which could be considered a discretized form of
the theory in Section 2.2.1.
The deviation force resulting from a curved tendon, should in theory be considered as a
line load along the tendon. The distributed force generated by the tendon, should therefor
be interpolated to equivalent nodal forces by integrating the force along the tendon. Such
an approach is slightly more complicated than the approach used in this thesis and requires
numerical integration of the force variation along the tendon as in (Galic et al. 2011) discussed
in Section 2.1. The advantage of this approach, would be that the force variation along the
tendon even within an element is taken into account. However, the curvature of the tendon
within each element is usually of such a small magnitude, that the straight line representation
in this thesis will usually be close to the real curve. One could imagine that as this approach
would take into account the force variation along the tendon within each element, it could
provide the same accuracy using less elements than in the chosen approach. However, as the
straight line representation will be close to the real curve within each element, this e’¼Ćect
should be small.

6
Further Work
In this section, suggestions for further work regarding the development of the program is con-
sidered. As the time provided for a master thesis is rather limited, considerable improvements
and additions could be made on the work done thus far.
An even more e’¼ācient procedure could be achieved by including a geometric constraint when
searching for the ’¼ürst intersection point. When searching for the ’¼ürst intersection point, the
node closest to the start of the tendon is sought, and the routine searches trough the whole list
of nodal coordinates in order to determine this. In the future, and option for only searching
in the vicinity of the starting point could be included for very large models.
At the ’¼ürst and last intersected elements, anchor forces are applied. The anchor forces are
calculated as equivalent forces on the four nodes at the entry face of the ’¼ürst element, and
exit face of the last element.
If the size of the element is small, and the anchor force is
large, this may create large stress concentrations in these locations. If a ŌĆØmore realisticŌĆØ force
distribution near the anchors is preferable, then an option to include anchor plates should be
considered. In practice, this would be done by including an option to distribute the equivalent
nodal forces on the nodes within a certain distance out from the nodes of the entry element
face.
At this stage, the friction loss is fully implemented for arbitrary tendon geometry. However,
only jacking/tensioning from one end of the tendon is considered. One can in practice tension
the cable at both ends on the structure, and thus reduce friction loss at the end of the
tendon. The option for calculating friction loss when the tendon is tensioned from both sides,
is something that should be implemented at a later stage.
The option to include draw-in loss should only be used if constant friction loss within the
a’¼Ćected area can be assumed. In further development of the program, an iterative approach
could be implemented in order to cope with nonlinear friction loss as shown in Appendix
A. Other forms of tension losses could also be included, such as steel relaxation, creep in

7
Conclusion
In this thesis a program for calculation of anchor and deviation forces on prestressed concrete
structures with arbitrary tendon geometry was developed.
To represent the tendons with a high degree of accuracy, while keeping the input simple, a
spline function is introduced in Section 3.2. It was decided that a cubic Hermite spline would
be suitable for this task. For normal tendon pro’¼üles, this kind of spline was able to model
the tendon in an accurate way.
To determine intersection points between the tendon and faces of the volume element mesh,
plane equations were generated for the element faces. A method for doing this through utiliza-
tion of the topology matrix, was described in Section 3.3. This method worked consistently,
as long as standard rules for node numbering is used for the topology matrix.
In Section 3.4, a description of how the tendon function and plane equations are equated
in order to ’¼ünd possible intersection points between the tendon and a plane lying on the
surface of an element face was provided. Then, to determine if the intersection lies within
an element boundary, a procedure using vectors along element boundaries, together with
the ŌĆØright-hand-ruleŌĆØ for normal vectors was introduced. This method proved e’¼ācient, and
provided an alternative to the traditional iterative procedure commonly used.
Isoparametric coordinates for the intersection points relative to the intersected element also
needed to be calculated. An iterative procedure for determining these coordinates were de-
scribed in Section 3.5. This method was found to converge rapidly.
After obtaining the isoparametric coordinates for the intersection points, they were in Section
3.6 used in the volume element shape functions to calculate equivalent nodal forces. All tests
performed thus far, indicates that this is an e’¼ācient and practical way to represent anchor
and deviation forces within a desired degree of accuracy.
Section 3.7, describes how friction loss was included for arbitrary tendon geometry, while
draw-in loss can be included for cases where constant friction loss can be assumed.
Overall, as a tool for e’¼ācient calculation of anchor and deviation forces this program performs
well. However, as this is a ŌĆØ’¼ürst editionŌĆØ of the program, certain bugs may still be present
and become apparent when applying it to a larger variety of cases. Also, further development
should include fully functional tension loss calculations for other relevant losses than friction
loss for arbitrary tendon geometry. Further simpli’¼ücation of the tendon description could also
be an improvement.
