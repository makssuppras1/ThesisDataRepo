Abstract - English
In this project, I investigate the possibility of using deep learning methods to improve the
localisation of ultrasounds scatterers as a part of a super-resolution ultrasound pipeline.
Super-resolution ultrasound is a method for creating images using ultrasound with a better
resolution than ordinarily allowed by the diffraction limit. It is dependent on accurately
identifying and localising individual scatterers in an ultrasound image.
To produce training data for deep learning models, I simulate super-resolution ultrasound
scans with known ground truth scatterer positions. Deep learning models are constructed
using an R-CNN-inspired architecture that uses deep learning in conjunction with a tradi-
tional peak detection method to improve the result of that method. The traditional peak
detection method functions as a proposal algorithm for the network which is then tasked
with discriminating between true and false detections.
The method is tested for the effect of variations in input size given to the network, and
the effect of noise in the data. Fully connected models are compared with convolutional
models, and models that consider multiple frames to make the detection are tested. The
best models found are trained on a range of signal-to-noise ratios, use a convolutional
architecture and use multiple image frames.
The method with the best-trained model achieves a Precision of 96% and a Recall of 55%
on simulated data which is an improvement over the traditional peak detection method.
And a Precision of 96% and a Recall of 28% on simulated data with artificial noise added
for an SNR of 50dB, which is also an improvement on the peak detection method. On
in-vivo rat kidney data, the model achieves an image that is very similar to the image from
the peak detection method.

1
Introduction
Ultrasound has a natural limit on its maximum achievable resolution from the wavelength
of the sound waves used. Super Resolution Ultrasound is a technique that is able to
overcome this resolution limit and generate images of structures that are smaller than
this limit. This is done by tracking the movement of scatterers over time with a series
of ultrasound images. These scatterers are either red blood cells (erythrocytes) or an
injected contrast agent moving along in the blood vessels. By tracking their movement
over time an image of fine vasculature can be gathered.
This project aims to improve a part of the super resolution pipeline used to generate these
images. Specifically the goal of this project is to develop deep learning based methods to
improve the detection and localisation of targets in ultrasound images for use in a super
resolution imaging pipeline. These methods are then integrated into the existing CFU
super resolution pipeline, and used to generate super resolution images.
The project uses a deep learning architecture inspired by the R-CNN object detection
method. The method relies on using an existing detection method to propose possible
detections, using which the deep learning network classifies the proposed detections as
either true or false. In theory, this limits the scope and required complexity of the deep
learning model, by offloading the complexity of general detection, allowing the model to
focus on learning the difficult nuances the excising methods are unable to take into con-
sideration.
The training of these models relies on simulated ultrasound data. With simulated data
it is possible to know the ground truth of the scatterer localisation and generate training
data for a deep learning network. Simulated data also allows the quantitative evaluation
of model performance. Though the final goal is to apply these models to in-vivo data.
This project started with the development of a basic prof a concept model, and then in-
volved testing different parameters in data and the model that can effect performance.
The project results are therefore presented as a series of experiments each considering
some of these parameters and evaluating there effect on model performance. At the end
the model that receives the best performance on training data is then applied to in-vivo
data from a rat kidney, and the resulting image compared to one generated using the
reference peak detection method.

Theory
2.1
Super Resolution Ultrasound
Medical ultrasound imaging is a widely used imaging technique that uses sound waves
to images tissue. It has many advantages over other types of imaging being cheap, non-
invasive, real time, and not using any ionizing radiation. It enables multiple types of imag-
ing including B-mode imaging and blood flow imaging. Two disadvantages of ultrasound
imaging are the limited imaging depth and the low resolution as compared with other
medical imaging modalities.
2.1.1
Diffraction limit
Ultrasound like all wave-based imaging is limited in the achievable resolution by the
diffraction limit of the waves. The diffraction limit is based on the wavelength of the waves.
And while imaging based on visible light have wave lengths of around half a micrometer
and x-ray imaging uses waves in the nanometer range, ultrasound in human tissue uses
waves on the order of half a millimetre. The diffraction limit as quantified by the Rayleigh
Criterion limits to theoretical achievable resolution. [1]
For ultrasound purposes this limit is about half a wavelength: λ/2 = c/2f0 where λ is the
wavelength, c the speed of sound, and f0 is the transducer center frequency. Scattered
that are much closer than this becomes impossible to separate. To the left in Figure 2.1
is shown a 2D representation of two scatterers right at this limit. This limits the spatial
resolution of a standard ultrasound image. [2]
Figure 2.1: Left: Two sources separated by half the wavelength that are barely distin-
guishable. Right: Super-localisation of these two sources from activation of each of them
separately. Figure from [2]
Super resolution ultrasound seeks to break this diffraction limit. Instead of imaging tis-
sue directly super resolution images are generated by tracking scatterers moving through
vessels over time. This fundamentally changes the limit to the spatial resolution. Now
the detections of close scatterers are separated in time. It is no longer about separating

overlapping peaks but is instead about accurately locating the individual peaks when they
appear. This theoretically allows a much higher resolution. To the right in Figure 2.1 is
shown a 2D representation of two close scatterers separated in time such that each wave
can be considered individually. [2, 1]
2.1.2
Super resolution imaging
As super resolution imaging does not image tissue directly, but instead images scatterers
moving over time the images generated are quite different in nature to a standard B-
mode image. A B-mode image captures the entirety of the tissue and has contrast from
interfaces where sound waves are reflected. Super resolution imaging instead tracks
individual scatterers moving through the bloodstream, it, therefore, generates an image
of just the position of the blood vessels. [3]
There are two general type of super resolution imaging. In SRI (Super Resolution Imaging)
a contrast agent is used for the imaging. This contrast agent consists of small air bubbles
that are injected into the blood stream of the vessel being scanned. As air has a very
different speed of sound relative to tissue this create a large contrast in the ultrasound
image. These contrast agents are very fragile and therefor severely limit the power of the
ultrasound that can be used, as to high acoustic pressure will burst the bubbles. Also the
mere injection of a contrast agent complicates the scanning procedure, increasing both
the work and the risk involved with performing the scan.
In SURE (SUper Resolution imaging with Erythrocytes) no contrast agent is used and in-
stead the erythrocytes (red blood cells) naturally moving through the vessels are tracked
instead.
They have the advantage of already being present abundantly in the blood
stream and being able to withstand higher acoustic pressure.
2.1.3
Super resolution pipeline
Figure 2.2: The super resolution pipeline. Figure from [3]
The super resolution pipeline as shown in Figure 2.2 is the process that allows the gen-
eration of super resolution images.
First data be acquired through an synthetic aperture ultrasound sequence scan. For the
in-vivo data used in this project all scans er performed on rat kidneys. Alternatively, data
is simulated. Next in the pipeline is the ultrasound beamforming. Here the raw ultrasound

data (RF-data) is turned into B-mode images. Then motion correction, which adjust for
any relative movement of the subject and the ultrasound probe. Echo cancelling then

CFU Visualize
CFU Visualize is a program developed internally at CFU that implements the super reso-

in many instances being replaced by the methods developed as part of this project.
2.1.4
Localisation of scatters
Figure 2.3: Input to the localisation step from simulated data
In Figure 2.3 is shown the input to the localization step of the super resolution pipeline.
The task of the localization step is to return a list of localization of points in the images
that represent the scatterers.
The current way to perform this localization is through peak detection on the envelope of
the image. This method first performs filtering on the image. A median filter to remove
noise, then a threshold is applied to the image to remove weak peaks. Finally, an esti-
mated point spread function based filter is applied to enhance the peaks, and the peaks
in the image are detected using a peak detection kernel.
2.2
Deep Learning
Deep Learning (also called Machine Learning) is a descriptor for a variety techniques and
methods that allow computers to learn to perform tasks without manually programming the
details of how to solve the task. For a computer to learn a model needs to be constructed
and then trained in reference to some evaluation metric measuring how well the computer
performs. What is not needed is for human programmers to understand or even conceive
of how the computer is going to perform each step in the process. This allows for some
truly unique results with Deep Learning models being able to beat humans, and computers
programmed without Deep Learning, at some very complex tasks.

2.2.1
Neural networks
Deep Learning networks are also referred to as neural networks, this because their con-
struction is conceived as analogous to the function of neurons in a brain. Each neuron
takes input from a other neurons, multiplies these inputs with a separate factor for each
then adds a bias, which then becomes that neurons output. These factors and biases are
referred to as the weights of the network and trick of deep learning is how these are deter-
mined. Real neurons in a brain are typically interconnected in complected ways, in a deep
learning networks the neurons instead exist in layers and commonly each layer in one is
connected to all neurons in the previous layer, such a layer is called a fully connected
layer. A representation of a simple example of this is shown in Figure 2.4.
Figure 2.4: An simple example of a neural network
Training of a neural network is done through the process of backpropagation, using train-
ing data consisting of inputs with a knows output. At each iteration training data is passed
through the network and the output is calculated. A loss function quantifies the differ-
ence between the actual and expected output, and the back propagation calculates the
gradients of the network, i.e. the direction of change for each parameter where a small
change would improve the result as defined by the loss function. Then a small change
is applied in the direction of the gradients to each parameter before the next iteration is
performed. Through repetition of this process a many dimensional process of gradient
descent is performed that should lead to a good network at the end of many iterations.
A good network being one that is capable of performing the classification shown through
the training data on not only the training data, but also new data.[4]
An import component of any neural network is the activation layers. An activation layer
applies a non-linear function to the activation of all the neurons in the preceding layer. This
is important to allow the network to learn. One way to see the importance is to consider
that a fully connected layer can be represented as matrix multiplication, and that a network
is then a series of matrix multiplications. Without a non-linear component all the layers
could be multiplied together resulting in a single matrix that of size input of the network by
output of the network.
Convolutional neural networks are another type of network that is great at many image
tasks. Here each layer consists of a convolution kernel and the output of the layer is
generated by convolving this kernel with the input data for the layer. The weights in this
case are the values in the kernel and a bias. This has many advantages when dealing

2.2.2
Object Detection
Object detection is a common deep learning application. With many networks excising
that are created for object detection. The goal for the Deep Learning model in this project
is to detect and localize several scatterers in an ultrasound image. This is in many ways
an object detection task. Though it differs from traditional object detection in a few ways.
Mainly traditional object detection methods usually aim at detection a wide variety of ob-
jects and to detect not only their location but also their size, often in terms of a bounding
box. For the scatterer localization task, we are only detecting one kind of object, the
location is simplified to a point, and the size of each point is relatively constant. [5]
Figure 2.5: Overview of the R-CNN architecture. Figure from [5]
One well widely used object detection methods is the Region-based Convolutional Neural
Networks (R-CNN). It uses a non-deep learning region proposal method to propose a
large number of potential regions, on the order of 2000 proposals. Then each proposed
region is cut out and a Convolutional Neural Network is used to classify it into object
categories and give a confidence score. Finally overlapping results of the same category
are discarded based on the confidence score. [5]
Figure 2.6: Overview of the Faster-RCNN architecture. Figure from [6]

Newer and faster variants of R-CNN exist. For example, Fast R-CNN which applies most
of the Convolutional Neural Network before cutting out the proposed regions. This im-
proves the speed by about an order of magnitude through saving a lot of time on repeated
calculations for pixels that are present in many proposed regions. Also, Faster R-CNN
which builds on Fast R-CNN replacing the region proposal method with a neural network.
This again improves performance by about an order of magnitude. [6]

3
Methods
To accomplish the goal of using a Deep Learning model to localise scatterers several
different components are needed. First, the simulations of ultrasound data are needed to
create training data for the model. For in-vivo data the true positions of the scatterers are
unknown, and any use as training data would then have to be based on another detection
method. Simulated data, on the other hand, allows full knowledge of the ground truth
necessary to train the models. The data needs to be processed to the SURE pipeline up
to the point before localisation is performed, and then the image at this step can be used
to train Deep Learning models.
After a model is trained the model needs to be integrated into the SURE pipeline such
that the model can be used in place of existing localisation methods. With the different
detection methods all available in the pipeline their performance needs to be evaluated.
This also needs to be done on simulated data as it again requires knowledge of the ground
truth positions.
3.1
Simulating data
Data simulation is performed using existing simulation scripts developed at CFU, the CFU
Sarus simulation program. The program simulates the movement of a number of scatters
in a digital phantom and uses Field II [7, 8] to simulate the acquisition of an ultrasound
image of the phantom at many time steps.
The phantom used for this project is a simple very rough approximation of a kidney. The
flow consists of 7 pairs of 2 lines next to each other in opposite directions at different
angles, these lines simulate blood vessels in the kidney. Each line is simulated as many
discrete points corresponding to either a contrast agent or erythrocytes depending on
whether we consider it an SRI or a SURE simulation. The locations of each of these
scatterers are saved along with the simulated ultrasound data, this forms the basis for the
ground truth for deep learning.
For the first experiments, there is no static tissue present in the simulation, for later ex-
periments static tissue is added on top. Static tissue is simulated as a large number of
randomly distributed points that are placed identically in each frame. In the pipeline, static
tissue is largely removed before the localisation step, though even when removed well its
presence in the raw data does affect the input to the localisation step.
In in-vivo data, there will also be some tissue movement i.e. natural movement of the tis-
sue scanned and the probe in relation to each other. In the pipeline, this tissue movement
is removed digitally prior to the localisation step. For simplicity tissue movement is not
simulated, as this project does not deal directly with motion correction.
3.2
Model architecture
Deep learning models generally work on a fixed input size and generate a fixed output
size. Having a fixed input size is not a huge obstacle as it would be easy for the purpose
of this project to choose a fixed image size to process and crop any oversized image to
match. Having a fixed output is much more difficult as the number of detections in a given
image can vary greatly. This is a similar problem as exists in general object detection
using deep learning. There I have for this project chosen to be inspired by the solutions
8

==PAGE:17==
that have been effectively implemented there. The most common approach is to separate
the detection process into a proposal process and a refinement process. Specifically, I
am implementing a process inspired by the R-CNN object detection model. Using a non
deep learning based proposal algorithm and then classifying each proposal individually
using a neural network.
The detection process starts by applying the proposal algorithm. The algorithm used is
the existing peak detection method, with the detection threshold set very low. Then the
neural network will take each proposal as an input separately, and classify them as either
a true or false detection. The list of true detections is then passed along to the next step of
the pipeline. Each proposal is a single point detected at a sub-pixel position by the peak
detection algorithm. A fixed-size region with the pixel containing the detected sub-pixel
position as the centre is the input to the network. Figure 3.1 shows an example of these
regions in an ultrasound image.
Figure 3.1: A section of a simulated ultrasound image as it reaches the localisation stage.
The yellow boxes show the regions that are extracted and passed to the deep learning
model for classification. Each is centred around a point generated by the proposal algo-
rithm. Each region is 31 pixels or 1.94λ vertically and 11 pixels or 2.75λ horizontally.
3.3
Model types
The deep learning model working as part of the described architecture now has a fixed
input of the fixed image region around the proposal, and a fixed output of a true or false
detection determination. For this many different model types could potentially be used.

For both models, the hidden layers are used with a leakyRelu activation and the output
layer with a softmax activation. The input and hidden layers are subjected to dropout
during training. Dropout layers randomly set a fraction of the outputs of each layer to 0
during training. The purpose of this is to help the model generalise. It forces it to train
different useful pathways and not really on a single initially promising one.
The models used are fairly small compared to traditional object detection models, with
e.g. implementations of ResNet ranging in size from 18 to 152 layer models. The main
reason for the use of small models in this project is that during the initial investigations,
these small models worked. Small models have the advantage of running faster, and in
training, they also typically converge faster. And it does make sense that smaller models
might be adequate for this task. A general object detection task has to learn many objects
over many photo parameters. For this task, we have a binary determination and use a
known constant imaging procedure.

3.4
Creating training data
The training data is created from the simulated data by passing the data for each frame
through the pipeline up until the point of localisation. At this point, the proposal algorithm
is run on each frame and for each proposal, the network input sections are created as
they would be when applying the model. These images are then saved to be used as
training data for the models. In this process, these sections are labelled as either true or
false detections based on a pairing with the ground truth.
For comparing the detections to the ground truth first the points in each needs to be clas-
sified. This is done using an algorithm that pairs each detection with the closest ground

Training Models
Each of the models trained in this project is trained under similar conditions. The models
were trained using the Adaptive moment estimation (ADAM) [4] optimiser as implemented
in MATLAB, with a learning rate of 10−3. For training the model the training data were
randomly split into a training and a validation data set, with 80% of the individual image

set. The training set is what is used to train the network. This is then further divided
into batches of equal size. At each iteration of the training, a single batch is used. The
validation set is used to monitor the performance of the network. At regular intervals in the
training, the network is tested on the validation set. Ideally, the network performs similarly
on both the test and validation set. A common occurrence in deep learning is that the
model starts to perform better on the test set. All the model training has been performed
using MATLABs Deep Learning Toolbox, with the code shown in Appendix A.1.
After the model has been trained the trained model is saved such that it can be used in

6
Discussion
Considering first the results on the simulated data. The Deep Learning models definitely
outperform the reference method on this data, when the data has a high SNR. Noise did
show to be a huge factor in model performance. Transferring to data with lower SNR is
challenging and definitely something that needs to be built into the model training. Adding
artificial noise to the training data did show to improve performance on lower SNR, espe-
cially when training on multiple SNR levels simultaneously. The results definitely show
that training a model across multiple SNR levels is not only feasible but likely preferable to
training a model on the specific SNR level of the data. This is good news for the possibility
of implementing the methods in practice. If the reverse were true many different models
would likely be needed for varying SNR levels, and estimating the true SNR of the in-vivo
data would become important. As it is it is likely feasible to train a single model over a
large SNR range.
Though training over multiple SNR levels in itself did not seem to be enough. Using
multiple adjacent frames in the model also had a slight improvement in generalisation to
lower data in itself. And combining adjacent frames and training on multiple noise levels,
in the end, proved very effective. Giving a model that performs well across the entire range
of noise levels it was trained on. The next step would in continuing this investigation would
be to incorporate even lower SNR levels in the training data.
The general technique applied of using the R-CNN-inspired architecture that tasks the
deep learning method with improving the results of another detection method instead of
performing the entire detection in deep learning looks to be very solid. It relies on also
applying the existing detection method as a proposal algorithm. This could be a hindrance
in achieving fast processing as the application of deep learning will always add to the time
taken using this architecture. Though it does mean that this overall architecture could
benefit from any improvements made to that method.
The other consequence of the current architecture is that it relies completely on the peak
detection method for sub-pixel localisation leaving it without a strong way to improve error.
Through all the experiments the error magnitude where rarely worth commenting on as it
stayed within a narrow range for the models that had good precision. It might be possible
to integrate some localisation improvements into the deep learning model. It would involve
adding a regression element to the model that re-estimates the peak position within the
image section provided to the model.
Considering then the translation to the in-vivo data. Ultimately there was a very small dif-
ference between the results achieved with the deep learning method and those achieved
with the standard peak detection method. Overall this is disappointing as the goal was to
achieve a better performance. Though in some sense it might be better than one could
expect from the results on simulated data. As the model only ended up having good per-
formance there on an SNR that is higher than the one seen throughout most of the in-vivo
data.
Ultimately the deep learning architecture seceded in outperforming the peak detection
methods on simulated data and achieved very similar performance on in-vivo data.

6.1
Possible improvements
There are several potential improvements to consider to push the method further. The
main task would be to make the model perform well at even lower SNRs. This might be
possible by simply providing more training data at even lower SNRs, and there are also
a lot of variations in the model design that could be considered. If this is not enough an
improved simulation that better mimics the in-vivo data is also worth considering.
When a model is achieved that performs well at lower SNRs there is a lot of hyperpa-
rameter tuning that can be done. Hyperparameters in machine learning are all the small
tuneable details of how the model is constructed and trained. The exact learning rate,
number of layers, size of layers, and batch size to name a few could all be modified
slightly which might generate better or worse results.
The speed of the model is not a huge hindrance. It does slow down the localisation step
increasing the total time it takes by a factor of 2 to 4. Though as part of the full pipeline,
this becomes a lot less significant as the total time is still dominated by data-loading,
beamforming, and motion correction. If a faster model is desired improvements could be
found by looking at the improvements on the original R-CNN model like Fast-RCNN and
Faster-RCNN. The main improvement there is to run the convolutional part of the network
on the full image and then extract sections for classification.
