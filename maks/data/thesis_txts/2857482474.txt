Abstract
The goal of this project is to design a hardware solution aimed at accelerating the rank-order filter, with a
particular emphasis on morphological operations. Different architectures for rank-order filters have been
analyzed, and 1D and 2D rank-order filters based on the systolic array architecture have been designed,
implemented, and evaluated. First, the filters have been extended in order to adjust the window size at
runtime, and thereafter, a binary mask has been applied to the kernel, to be able to change the shape of
the window.
To see how the filter will be applied in a real-world scenario and analyze the effective speed-up, the
filter has been integrated into a custom processor based on the RISC-V ISA. It was found that running
the processor with the accelerator compared to a typical software solution, which runs on the processor
without the accelerator, that the hardware solution, provided a significant performance improvement.
Speedups ranging from 30x to 40x were observed for the input samples used in the testing.

Chapter 1.
Introduction
The rank-order filter belongs to the class of nonlinear filters which opposed to linear filters can not
compute each output sample as a simple linear computation, thus making them computational harder to
compute. Therefore, it is optimal to use a hardware solution to accelerate the process of filtering large sig-
nals fast and efficient. Some advanced filters uses the rank-order filter to perform morphology operations.
This thesis focuses on the exploration, design, and implementation of rank-order filter architectures,
including solutions for one-dimensional (1D) and two-dimensional (2D) input signals, with a particu-
lar emphasis on morphological operations. It examines methods for optimizing filter performance and
making it more versatile, with extensions such as integrating runtime kernel adjustments and binary
masking, and evaluates hardware acceleration techniques. The goal is to enhance processing efficiency
while maintaining the flexibility required for modern computational tasks.

Chapter 2.
Theory
2.1
Nonlinear Filters
In digital signal processing, filters are used to manipulate the input signal, such that the output
signal has reduced or enhanced specific features. Digital filters can be used for a variety of tasks such
as removing unwanted noise, feature extraction, or restoration of signals. The filters can work on 1-
dimensional signals and can be extended to work on multi-dimensional signals. Digital filters are, for
example, used very often in digital imagery.
Ideally, a linear filter is preferred where the output value can be calculated with pure addition and
multiplication of the surrounding values and weights. However, linear filters can not handle all cases.
In these cases we need nonlinear filters. A famous nonlinear filter is the median filter. For every input
sample, the filter outputs the median value of the specific sample and k values around it, where k is a
given width, and a 1D filter will have the width N = 2k + 1. The filtered output can be expressed as:
yi = (k + 1)′th smallest of (xi−k, . . . , , xi, . . . , xi+k)
This filter is good at noise removal, and for images it is especially good at removing salt and pepper
noise. This project will explore the rank-order filter. The rank-order filter works similarly to the median
filter, but it can output any chosen r-th ranking, instead of just the middle rank.
yi = r′th smallest of (xi−k, . . . , , xi, . . . , xi+k)
Rank-order filters can be of recursive and non-recursive types. The recursive type additionally uses
the recent calculated ranked outputs, in the window for the next sample.
There are various architectures proposed for implementing rank-order filters in VLSI designs. Dif-
ferent architectures consists of Array architectures, stack filter architectures, and sorting network archi-
tectures.
2.2
Array Rank-order Filters
A systolic array is a network of connected data processing units called nodes, that each computes a
partial result in parallel. The input data flows through multiple of these nodes to be processed into the
final result.
Array rank-order filter architecture builds on a systolic array method. The idea behind the array
architecture is to save the value and rank of each input sample in a node, and the nodes are then
connected in a linear array. Every time a new sample enters the window, the new rank of the sample
gets computed in the node, and the value and newly computed rank is moved to the next node. Another
process compares the rank in each node with the sought rank, and outputs the sample value belonging
to the found rank for each step[2].
For this, we need a shift register that holds every sample in the filter window called S, which can be
seen as the window passing through the samples in the filter, where S0 is the oldest sample and SN−1
is the new sample, and N is the size of the window. For every sample, we also need to store their rank

in a register called RN−1 . . . R0. For every iteration, the sample and rank will go to the next register in
the line, but the rank needs to get updated in-between.
Figure 2.1: Single process for samples Sn−2 . . . S1
Figure 2.1 shows the process for sample SN−2 to S1. Sample 0 does not need the ’New rank logic’
block. Sample N −1, will get the rank for the new sample, which needs to be found by comparing the
value with sample values in the window.
The output Oi is found by identifying the sample that has the chosen rank r, and output the corresponding
value.
Figure 2.1 uses the rank of the sample that leaves the kernel to compute all of the other ranks.
Chakrabarti[2] also proposes to save the comparisons of the sample with all the previous samples that
came before it. Each node then knows if its sample is smaller or larger than the sample leaving the
kernel, and the node can decrease the rank as needed. This speeds up the computational path between
processes but requires more space.
This thesis will explore the array architecture in greater detail in Section 3.1.1, where we will design our
own filter with systolic array architecture.
2.3
Sorted Rank-order Filters
Another method is to sort the samples in the window and pick the r’th element of the sorted
values, where r is the sought rank. Charkrabarti proposes 2 sorting network architectures [1]. Both
of the architectures are inspired by Batcher’s Odd-Even Merge Sort, which is an algorithm that uses
merge sort that sorts the bottom ⌊N/2⌋and top ⌈N/2⌉elements in the window of size N, and thereafter
merges the two sorted sequences. The first proposed architecture sorts all N elements in the window
each time the window moves, using a sorting network. The sorting network consists of multiple smaller
pipelined sorting units. Figure 2.2 shows a sorting network for a rank-order filter with a kernel size 9.
The smaller sorting units consist of (I) sort-2 that sorts two inputs and uses one 2-element-comparator,
(II) sort-3a that sorts three inputs using three 2-element-comparators, and (III) sort-3b that merges two
sorted sequences together with sizes 2 and 1, through the use of two 2-element-comparators. The small
rectangles in Image 2.2 symbolize latches. The network uses multiple clock cycles to compute the rank
for the elements.

y1 = med{x1, x2, x3, x4, x5, x6, x7, x8, x9}

sort(S1) = merge(sort3(x5, x6, x7), sort2(x4, x5)

y2 sorted = merge(sort2(S1), merge(sort2(S2, sort2(x2, x10))

2.4
Stack-based Filters
Stack-based filters use an architecture that implements nonlinear filters through three key stages:
threshold decomposition, processing, and reconstruction[13]. In the first step of threshold decomposition,
the input gets decomposed into multiple bit signals. In the second step of processing, these signals are
then processed on bit-level using a boolean truth function that defines the nonlinear filter’s functionality.
The final reconstruction stage then constructs the processed bit-sequences back into values with the same
bit length as the input, which results in the filtered result.

2.5
Extensions
Multiple extensions have been proposed for the different kinds of architectures[3].
An important application is to extend the filters to work on higher dimensions - especially making them
5
==PAGE:13==
work with 2-D input signals, which is widely used in image processing and computer vision.
The array architecture follows an almost linear complexity in space, when the kernel size gets larger,
which will be explored in Section 3.5. This proves to be useful, as the kernel grows very large when
working with 2-D filters. The array architecture can be modified to take multiple new samples into the
kernel, and multiple samples will leave the kernel at the same time[2]. It then needs to compare all the
samples with all the samples leaving the kernel and those entering the kernel, to calculate the new rank.
The sorting architecture does not need much changing when converting it to 2-D, each 1-D element
gets replaced with entire rows or columns of 2r + 1 sorted elements. Merging two sorted lists in 1-D
corresponds to merging two sorted sequences of rows or columns in 2-D[1].
However, the number of comparators does not scale linearly with the size of the kernel, like the array
architecture does.
Sometimes it proves useful to add weights to positions in the kernel.
An example could be making
the samples around the center be more potent for the output. We could have the weights for a median
filter window of size 5 be W = {1, 2, 3, 2, 1} and a window full of the samples WIN = {1, 3, 2, 4, 5}. A
filter taking the r’th rank should then take the r’th element of the sorted sequence {1, 3, 3, 2, 2, 2, 4, 4, 5}.
In other words, sample WIN(n) will appear W(n) times in the sorted sequence.
Chakrabarti and Lucke[3], go through how to implement weighted rank-order filters, for the three archi-
tectures, for both recursive and non-recursive filters.
2.6
Morphology
Morphology operators are a class of image-filters used in image-processing applications[6]. Some of
the most popular morphology operators are dilation, erosion, opening, and closing.
The rank-order filters can be utilized to perform these operations. Choosing the maximum rank (100th
centile) results in a dilation. Instead, if the minimum rank is used (0th centile) an erosion is performed[9].
An opening operation involves performing an erosion followed by a dilation, whereas a closing operation
is achieved by applying a dilation first, followed by an erosion. To make the morphology operators more
robust against noise, a small centile c can be used instead of the minimum and (100 −c) instead of
maximum. Often the 5th and 95th centiles are used [6, 9].
The morphology operators are used to create filters for imagery. Kass et. Al [6] show an example,
where a closing operation followed by an opening operation using the 5th and 95th centiles, results in a
filter that makes photographs look like water color paintings.
Treece [9] proposes a filter called the bitonic filter that uses a combination of morphology and linear
operators.
The bitonic filter has noise removal capabilities similar to the Gaussian filter, while also
having better detail and edge preservation properties.
The shape of the window for the the rank-order filter used in the morphology, has impact on which
features are preserved. For 2D data, a circular shape is preferably used [9], however Treece also proposes
that if the shape does not stay constant, but instead varies locally across an image, it could offer perfor-
mance improvements [10].

Chapter 3.
1D Rank-order Filters
I have chosen to follow the array architecture to built the rank-order filters , because the size scales
linearly with kernel size which can get quite large when making the 2-D filter.
The stack architecture has the downside that it does not scale well with the bit size of the input.
3.1
1D Static Array Rank-order Filter
3.1.1
Design
For implementation, I had to design the logic for finding the next rank, how to output the right
value, and finding the rank for the next input entering the window.
When a new value enters the window, the value is either higher, lower, or the same for every sam-
ple in the window. If the new value is higher than sample Sx the rank Rx should stay the same, if it is
lower, the rank should increase by one. For new samples that have the same values as samples already
in the window, I have chosen to give the new sample the highest rank of them. This means Rx should
be unaffected, if Sx has has the same value as Snew.
For the oldest sample that leaves the window, we compare all the ranks. If R0 > Rx is the case, R0 has
no influence on Rx. If R0 < Rx then Rx should decrease by one rank.
I have made an example with a window size n = 7 showing the S and R registers at time 0, the next
input entering the window which is 5, and the registers after the shift at time 1.
At T0(Inew = 5):
S = {2, 1, 4, 5, 7, 8, 3}
R = {3, 1, 4, 5, 6, 7, 3}
At T1:
S = {5, 2, 1, 4, 5, 7, 8}
R = {5, 2, 1, 3, 4, 6, 7}
Here it is seen that the rank for values {1, 2} stay the same, since the incoming value and the leaving
rank is higher. Values {4, 5} get demoted by one since the leaving rank is lower. Values {7, 8} stay the
same, since the incoming value promotes and the leaving rank demotes by 1.
The new sample receives rank 5 since there are 5 samples with lower or the same value. This means that
the newest rank can be found by counting every sample in the window where Sx ≤Inew.
The chosen output can be made using a multiplexer which uses the ranks to select the right sample
output, by comparing every rank register with the sought rank, and setting the output to the corre-
sponding sample value. For example, if we have a filter of length 5 and want to find the median, the
value with rank 2 is selected, since the ranks go from 0 to 4.

3.1.2
Implementation
The non-recursive rank-order filter was implemented with a parameterized size in the hardware lan-
guage Verilog. The rank-order filter (ROF) can be compiled with a chosen window size, a word-length
for the sample values, and the rank that the output selector should choose, by changing the parameters
N, data bits and rank sel. The rank sel signal is in this first implementation been set with a parameter.
Ideally, this value is set in a register so it can be changed every time the filter is run. This will be the
case for future implementations in this thesis.
By default, it makes a filter size of 3, word length 8, and takes the median by setting rank sel to ⌊N/2⌋.
The sample registers are made as a single shift register of size N · data bits as the module shift reg.
The rank registers are implemented as a simple register of size ⌈log2(N)⌉as the module rank reg. The
preset value is set as a parameter in the module instantiation.
The logic for calculating the next rank (as depicted by the blue box in Figure 3.1) is made as its own
module rank logic. The module contains an output for the new rank and whether the new sample’s value
is greater or equal to the existing sample. The sample comparisons are sent to the adder module that
counts the number of inputs with the value 1, the result goes to the first rank register, as the rank of
the new sample. The module ranks contains all the instances of the rank-registers, rank-logic, and the
adder. By using a Verilog generate block it creates N rank-registers and N-1 rank-logic block, and chains
them together in sequence. The final module is the rank selector that generates N comparators, that
compares each rank with the given parameter rank sel and N and-gates. All the and-gates are fed into
an or-gate that outputs the right sample.
The main module is the rank order module, that contains one instance of the modules shift reg, ranks,
and rank selector. It has three inputs; clock, reset, and the next sample, as well as one output which is
the filtered output for the middle sample register.
I made a top module to interact with the filter on hardware, where the top module contains three
input buttons, a clock input, and four 7-bit outputs which is used to turn on four 7-segment displays.
The top module also contains a 255 word sized ROM and RAM. The ROM holds the input signal and
the output of the filter gets written to the RAM. Two of the input buttons are used to go through the
output RAM - by incrementing and decrementing the read address. The 7-segment displays show the
value of the read address and the corresponding value in RAM. The last button is hooked up to the reset
pin. All the buttons are getting debounced by a module that contains a shift register that makes sure
the button is held for 20 clock signals.

3.3
Masked Rank-order Filter
As discussed in Section 2.6 the shape of window changes which features are preserved when used
in morphology [9]. The masked-rank-order filter will use a mask with values one and zero, to enable or
disable neighboring pixels. For example this can be used to take the r’s rank of only the values values
to the left by applying the mask [11100] or ignore the immediate neighbors with a mask [10101]. For
1D-filters using a binary mask does not have many significant use cases, compared to 2D filters. Starting
by making a masked 1D rank-order filter, will make it easier to later adapt it into a 2D filter.
When working with the mask it is not only one sample that leaves and one sample that enters the
filter kernel, but instead many samples can be activated or deactivated. Using the same method as the
earlier filters would result in having all the samples compare with each other every cycle. Instead, the
rank-registers gets changed to not hold the rank but a relative state of the whole kernel for each sample.
Each of the rank registers will hold one bit for each sample in the kernel window in a state register Ri.
Each bit represents whether this sample is larger than the sample at the bits position. At the sample’s
own position it will always be one.

My synthesizer creates N adders as a linear chain, where the width of each adder increases by one
until it gets to ⌈log2(N +1)⌉. The time complexity grows linearly, as each adder introduces additional de-
lay. Implementing the counter as a binary tree structure reduces both the area and the time complexity.
As an example, having an input that is 8 bits wide, the tree structure uses 4·1-bit-adder + 2·2-bit-adder
+ 1·3-bit-adder, compared to the chain adder that uses 1·1-bit-adder + 1·2-bit-adder + 5·3-bit-adder.
Also, the time latency is now proportional to log2(N), as additions are performed in parallel at each stage.
Another optimization, which the synthesizer did by itself, was minimizing the needed registers to hold
the state of each sample. Instead of each state register holding a bit for each other sample in the kernel,
it holds the state only for the samples that come after it in the kernel. Since the registers that come
before it, already holds a comparison of the size, we do not need to store it again, but simply use the
previous comparison negated. So, the example given earlier in table3.3, the state registers will contain
the values:
S4: 10011, S3: 1111, S2: 101, S1: 11, S0: 1, and since the first bit is always referring to its own position
it can also be removed and instead always add 1 when counting the bit sum. The registers then contain:
S4: 0011, S3: 111, S2: 01, S1: 1.

Chapter 4.
2D Rank-order filter
The next challenge is to work on 2D signals, such as pictures, which is a very common use case for
rank-order filters. Some use cases with images are image smoothing, noise reduction and edge detection.
When working with 2D signals the input will be a H xW array of sample values. The 2D kernel is now
a window, of size RxC, and this implementation focuses on architectures for square kernels only with a
size of N xN, where N is an odd number, since square kernels with an odd side length are very common
sizes of filters. Furthermore, when implementing the masked-filter as a 2D filter, the size and shape of
the kernel can be chosen by the custom mask.
The kernel-window then sweeps the input array. For each sample when it is in the middle spot of the
kernel, the kernel outputs the new filtered value as the sample with the N ’th rank currently in the
window.
4.1
Design
While designing the filter, it was necessary to figure out how the input data should enter the kernel
and how the structural shape of the kernel should be. The structural shape is how each node should be
placed and how the data travels between them. Each node consists of the sample register and the rank
register, representing a cell in the kernel.
In Figure 4.1 I have drawn some different design ideas.
The first structure, ’One 1D Kernel’, is to simply use one of the 1D filters already implemented. The
benefit of this structure is that it is very simple to implement since the 1D filter has already been im-
plemented. It is easy to use different kind of filters by replacing the kernel, for example with either
the masked or non masked version. The cons are that it requires some more complex external logic for
reading the input in the right order and it can only take one input at a time.
The second structure, ’Multiple 1D kernels in parallel’, uses a 1D kernel for each row, this allows multiple
inputs in parallel, but it requires more comparators, since each input register have to compare with all
other samples. The implemented 1D-filters also have to be modified, since the next-rank-logic modules
usually only considers one sample entering or leaving.
The last structure, ’Multi-directional kernel’, has the ability for the data to travel in three directions. In
this way, the kernel can traverse the input array more efficiently, since it does not need to reset for each
line, but can move down though the array like a snake. This would reduce the amount of memory reads,
but it also makes the architecture very complex, by adding the need for up to 4 multiplexers for each
process, and the need to generate a new logic for calculating the next rank, depending on which way
the kernel is moving. Figure 4.1 shows the layout of the systolic array, for each of the three discussed
structures.

Chapter 5.
Integration with a Processor
5.1
RISC-V Processor
The processor, I have chosen to attach the filter accelerator to is a RISC-V processor. I have chosen
the RISC-V processor due to past experience with this processor and since RISC-V is an open standard
instruction set architecture, which means it is openly accessible and available to use by the public. It
also contains a small set of instructions, which makes it easier to change or implement custom assembly
code.
I implemented a 5-stage pipeline RISC-V processor with forwarding and hazard-detection, in Verilog.
The RISC-V processor supports all of the 32 integer base instructions, except for ecall and ebreak in its
usual function, however, ebreak is used to pause the processor.
5.2
Design of Integrated Filter
The filter takes many clock cycles to complete and therefore it will either stall the processor or
be implemented with an out-of-order-execution algorithm.
One way to integrate the filter is to use
the Tomasulo’s algorithm. Tomasulo’s algorithm introduces reservation stations for each execution unit,
where the corresponding instructions will be held until the execution unit is free and the needed operands
are ready. A common data bus connects all the reservation stations and registers to the output of the
functional units, such that the computed results can be accessed by the instructions waiting for the result
as an operant in the same clock cycle.
Using this algorithm, the filter can dynamically add memory load and store instructions to the reserva-
tion station containing memory instructions. This ensures that the input will be loaded into the kernel
when it is ready, and instructions that need the kernel’s output will wait for when the filter has written
back the output to the memory, while other independent instructions can run in parallel with the filter
running. The advantages of this method is that it ensures high throughput, since multiple operations
can be run in parallel and can be out of order, and it minimizes the time it needs to stall. Additionally,
by dynamically scheduling the memory access, it reduces data hazards.
Some of the challenges it brings, are the increased complexity of the architecture and the higher overhead
of needed resources.
Another method could be to integrate the filter as if it was a separate device that is connected to
the memory through a shared memory bus.
By using a DMA-controller (direct-memory-access) the
processor could give the DMA-controller the start address and size of the input samples in memory and
then the controller will take over the bus and load the needed memory chunks into the filter. Meanwhile,
the processor can continue running other independent instructions.
The processor still has to stall if it needs to access the memory bus while the DMA-controller has access.
One way to reduce the frequency of processor stalls is by giving the processor a cache. The more cache
hits the processor has, the less it relies on the memory bus. Additionally, depending on the bus width
37
==PAGE:45==
and the size of the sample data, the DMA can load multiple inputs into a buffer in the filter in one cycle.
While the filter is working on that chunk, the DMA-controller can give the CPU control of the bus.
The parameters are stored within the filter device and mapped to specific memory addresses, with the
mapping abstracted from the CPU.
One of the dangers is potential data hazard arising if the CPU needs the filter output for a later
operand. Then the CPU has to be stalled until the data has been written.
This method also increases throughput, but increases area overhead and complexity as a DMA-controller
has to be introduced.
The last method is to integrate the filter accelerator as a memory master that gains exclusive con-
trol of the memory interface during its operation. It will stall the processor while the filter is working.
The filter can then read and write the memory directly, and when it is done, it can stop its interrupt
signal, giving the access back to the CPU.
The CPU has to set the parameters before starting the filter. This can be achieved by utilizing free
opcodes, as previously discussed, or by mapping specific addresses to memory regions for parameters
and the mask.
This design will give the least complexity for the hardware, since we remove the need for a common data
bus or the complex modules that Tomasulo’s algorithm needs. However, it removes the ability to let the
processor continue doing operations in parallel with the kernel running.
For this project to prove the concept, the last method has been chosen for implementation and pro-
gramming on an FPGA. This implementation has been chosen because it has the simplest design and
therefore is suitable for a first implantation. Furthermore, this project leverages an FPGA, which uses
the embedded memory block on the chip as memory, and therefore the memory can be accessed in a
single cycle, resulting in no cache. Thus, with the DMA-controller the processor will likely still end up
stalling. The chosen architecture is a 5-stage pipelined RISC-V that only has one ALU, and all opera-
tions uses one cycle in each stage, and thus the Tomasulo’s algorithm is not optimal, but rather adds a
lot of complexity, for this architecture.
It has also been chosen to use dedicated memory addresses for containing the parameters.
The pa-
rameters, kernel size (N ), input-height (H ), input-width (W ) and rank R, has each been given a byte in
memory at a designated address that the filter has direct access to. One byte is used because the RISC-V
architecture uses byte accessible memory. The mask for the masked filter is also given an address range
in the memory. The needed space for the mask is ⌈Max N 2/2⌉bytes, where Max N is the maximum
kernel size that the filter can use.
5.3
Implementation of the Integrated Filter
A physical implementation has been implemented on an FPGA by combining the RISC-V processor
and the masked 2D-Filter from section 4. The changes for the RISC-V processor include modifying the
memory pipelining stage, such that it contains registers for filters parameters. The address-range for
the parameters starts at 0x80000000. The configuration parameters have been assigned the following
addresses:

==PAGE:46==

A memory controller checks the address, sets the memory output, and writes the address to either
the main memory or the parameter registers if the address is in the range, using a mux and a de-mux.
The Address controller and kernel are also placed inside the RISC-V module. The unused instruction
opcode 0b0001011 has been chosen to run the filter; Currently there are not any other functionality for
the rest of the instruction other than the opcode. The filter parameters have to be set before giving the
start instruction.
In the decode stage, the control-module sets the control signals. If it sees the matching opcode, it sets a
control signal in the pipeline register. The address controller gets the run signal in the execute-stage. It
takes one clock cycle for the address-handlers state machine to change to the run stage, so the filter will
run when the instruction is in the memory pipeline stage. When the Address-handler is in the running
state, it stalls the pipeline registers that precedes the memory stage.
Three multiplexers have been added to the main memory’s input signals (w en, address, and data in)
to select between the values from the address handler and those from the pipeline register, depending
on whether or not the filter is running. The address handler has been modified since Section 4, due to
the fact that there is only one memory module, instead of an input and output memory. Therefore, the
address handler now has one output for the memory address. The kernel cannot get the next sample
while it is writing the output to the memory.
Currently, the implementation assumes that the input data is put in memory starting at address 0,
and the filter will insert the output immediately after the input ends. This behavior can be changed
by using the rest of the given instruction. By using the R-instruction format [11], the Rs1 field could
contain the ID for a register holding the address where the input data starts, and the Rs2 could contain
the address where the filtered result should be stored.

is 50.14 MHz. with a masked kernel size of max size 5x5. The critical path is reported as the path going
from Address-Handlers’ write address to the main memory.
This is because the relationship between the memory and Address-Handler, is only half a clock cycle.
The Address-Handler have been implemented such that the kernel receives the input in the same cycle,
that the Adreess-Handler calculates the read address, and since the the memory activates on falling edge,
the Address-Handler only have half a cycle to calculate the read address.
A way to increase the slack is to phase shift the Address-Handler’s operations, such that there is a whole
cycle between the state change and time the memory needs the address.
5.6
Software Implementation
To compare the speedup from the accelerated processor to standard software solution running on
the RISC-V CPU, an implementation of a software median filter has been written in C and compiled
to RISC-V binary instructions. By running the compiled code in the RISC-V simulator Ripes [7], it is
possible to compare the number of cycles needed to run the software version of the filter and compare it
to the accelerator.
There are many algorithms used to create a rank-order filter.
The most straightforward method is
to keep the values in the kernel in a list and sort them, and then taking the n’th element to find the
n’th rank. However, this method has a high ’per-sample’ complexity generally of O(r2log r), but can be
reduced to O(r2) using bucket sort[8], where r is the radius of the kernel.
Huang’s two-dimensional median filtering algorithm [5] keeps the kernel values in a histogram. It works
similar to our implemented hardware 2D-filter, as the window moves from left to right. For every sample,
the histogram is updated by adding the next column to the right of the window and removing the left-
most column of the previous window. For every sample, r · 2 + 1 additions and subtractions are needed
to update the histogram, and 2 · (r · 2 + 1) comparisons are used for the median finding algorithm. It
uses a fast, constant time algorithm to find the median. The algorithm keeps track of the median for
the last window, and how many elements in the histogram are smaller than the previous median. The
algorithm then pivots around the last median going up or down the histogram, depending on how many
elements in the histogram are smaller than the last median. Huang’s algorithm reduces the ’per-sample’
complexity to linear time O(r) and is used by many public implementations[8].
Other median filter algorithms have been made to reduce the ’per-sample’ complexity. One example
is Weiss[12], who developed an algorithm that uses a complexity of O(logr) by using a hierarchy of
histograms.
Perreault and H´ebert [8] further proposed an algorithm that uses constant time. They use a histogram
for each column in the input and one for the samples in the kernel window. For every sample, the
41
==PAGE:49==
histogram belonging to the column immediately to the right of the kernel, moves one down by adding
the next sample in the column, and removing the top-most sample in that column. It then removes the
histogram for the leftmost column in the window and adds the histogram belonging to next column to
the kernel histogram. The complexity is O(1) since it only does one addition and one subtraction for
each sample, and since combining and subtracting a histogram from another histogram takes constant
time.
It has been chosen to compare the Accelerated Processor against Huang’s algorithm because it is widely
used and simple.
Even though Perreault and H´ebert’s method has the lowest complexity, it either requires a really large
kernel or more advanced optimizations to be efficient. For an 8-bit data input, 257 additions and sub-
tractions are needed per sample to subtract and add the histograms, whereas Huang’s simple method
uses r additions and subtractions. So for this algorithm to be effective, the processor must either use
vectorization using SIMD instructions, which our processor does not support, or the algorithm must be
made more complex by using multilevel histograms.
In this comparison, the kernels will be held small, so using a complex algorithm that scales on the size
of the kernel, will not be needed.
The algorithm is very similar to the pseudo code in Huang’s algorithm[5], but modified to also pad
with zero around the picture, such that the algorithm behaves the same way that the processor does.
The algorithm is described in pseudo code in Algorithm1 below.
5.7
Evaluation of Filter Kernels
The number of cycles that it takes the hardware accelerator to filter the input is only dependent on
the input size and kernel size. For each sample, it needs to read N samples to move the kernel one step
to the right, and at the start of every line it has to initialize the kernel, by reading r · N samples into
the kernel. Since it has to stall one cycle to write the result for each sample, each sample uses an extra
cycle. The filter always uses two extra cycles; one when it starts and one when it ends. The number of
cycles can be calculated with the formula:
Per Sample Reads + Initilize New Line + Write Back cycles + Constant Cycles =
H · W · N + (⌊N/2⌋∗N + 1) · H + H ∗·W + 2
The formula has also been checked by adding a counter in the hardware, that counts how many cycles
the kernel is running.
The algorithm was written in C and compiled to RISC-V instructions using the riscv64-linux-gnu-gcc
compiler as follows:
riscv64-linux-gnu-gcc -O1 -nostartfiles -fno-tree-loop-distribute-patterns -nostdlib -march=rv32i -mabi=ilp32
The O1 flag is used to optimize the code as much as possible. When using the O2 flag, the code would
not work and the processor would get stuck in an infinite loop and run forever. The flag -fno-tree-loop-
distribute-patterns is used in order to avoid calling mem-set and memcpy. It is worth noticing that
for modern processors these instructions can help reduce the runtime, since it is faster to initialize the
histogram for every row.

Chapter 6.
Conclusion
The project aimed to speed up the process of applying a rank-order filter to 1D and 2D signals,
with focus on morphology operations, by designing and implementing a hardware solution. Different
architectures for rank-order filters, have been explored and the systolic array architecture was selected
for further implementation. A hardware accelerator for a regular one-dimensional rank-order-filter was
designed and implemented. To make the accelerator more versatile a new version was implemented with
a dynamic window size. Different filter techniques use morphology operators, which can be made using
a rank-order filter, where shape of the window for these operation changes what features are preserved.
A new version of the hardware solution with this in mind was created that applied a given mask to
window. The implementations were analyzed based on their size and timing performance. The size of
the dynamic and static filter scaled with a complexity of O(N · log(N)), while the masked filter had
a size complexity of O(N 2), where N is the size of the window. The timing complexity of the static
and masked filter was O(log(N)), and O(N) for the dynamic filter, but a design using a complexity of
O(log(N)) was also discussed. The masked filter was further extended to work on two-dimensional input
signals. By integrating the accelerator into a custom processor, based on the RISC-V ISA, the running
time, using the accelerator, could be compared to the time it takes a typical software solution running
on the processor without the accelerator. It was found that the hardware solution showed a significant
speedup compared to the software solution, with calculated speedups of 30x to 40x.
