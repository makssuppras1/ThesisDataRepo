Abstract
In this work, an adaptive feedback controller is created. It is intended to solve some
known problems of PID controllers, which are the most extended ones in several ﬁelds.
These problems are related with lack of adaptation to changes experimented by the object
which is under control, the task it is performing or the environment in which it is immerse.
The tool which is used to develop the controller are Spiking Neural Networks. The
fundamental unit of these kind of networks - the neuron - has a behaviour which is similar
to the one found in biological neurons. The synapses - connections between neurons - in
the network are plastic, meaning that they can change and adapt and so the network can
learn.
The mechanism used for learning is Dopamine Modulated Spike-Timing-Dependant
Plasticity (DA-STDP), which is also based on biology and allows to provide an on-line
learning method - the network can learn while it is performing the control task and so it
does not have to be trained in advance.
The controller is tested in several experiments and it proves to be able to learn how
to perform the control task, to be able to adapt to changes and to have a reasonably good
performance.

CHAPTER 1
Introduction
In this work, an adaptive Feedback controller based on a Spiking Neural Network has
been developed. This technology has been chosen for this task because it offers several
advantages which are of interest within the scope of adaptive control, such as processing
speed, real-time functioning or biological ﬁdelity.
An Spiking Neural Network is used as a controller, so that there are no extra blocks
or complex designs in the control loop. This means that the network receives the error
in position of every joint and provides the torque needed to make the robot follow the
reference.
In Spiking Neural Networks, the behaviour of the neurons resembles that of the
biological neurons. The output of this neurons are always "spikes", which are fast changes
in the output voltage of the neurons - increasing and decreasing almost immediately -
between a rest level and a maximum level. If the input of the neuron is below a threshold,
the output is more or less ﬁxed around that rest level. If the input is above the threshold,
the neuron will "ﬁre spikes", and the frequency of these ﬁrings will be higher if the input
signal is higher, up to a maximum frequency.
The network is structured in an input layer (which receives the error signal) and an
output layer (which provides the torque) which are connected between them with synapses
that are plastic. Because of this plasticity, the synapses can change and so the network can
adapt - learn - to implement a relation between the input and the output which works as
the desired control function.
The kind of plasticity the synapses have is called Spike-Timing-Dependent Plasticity
(STDP). This mechanism regulates how the synapses change (getting weaker or stronger)
with base on the time difference between the moment an spike emitted at the beginning of
the synapse arrives to the end of it and the moment an spike is generated at that end.
Furthermore, dopamine is used as a modulator of the STDP mechanism. With two
extra neurons the concentration of dopamine in the network can be regulated. If there is no

dopamine at all, the mechanism is inactive. If there is dopamine, the changes can happen,
and these will be more signiﬁcant if the concentration is higher.
By using dopamine as a reward, and introducing it in the network only when the
behaviour of the robot is as expected, a reinforcement learning mechanism can be imple-
mented. This basically means that the network learns while the robot is performing the
control task and so the accuracy can improve along time.
The developed controller, based on these techniques, proves to be able to implement
the control function with a reasonable level of accuracy. By introducing more bio-inspired
blocks in the control loop, the performance could be better and closer to biology.

CHAPTER 2
Idea and Concept
The main purpose of control theory is to be able to handle an object in a way that it
follows a reference. This reference may be ﬁxed or changing. ([1])
Feedback controllers receive some input related with the output of the controlled object.
By comparing it with the desired output (that is, the reference) and sending some command
to the controlled object they make the output closer to the reference (or, ideally, equal to
that reference).
Within the ﬁeld of classical control theory, the controller which is more used is the
Proportional-Integrative-Derivative (PID) controller, because of its simplicity and good
performance for many tasks. It provides torque commands made by the addition of 3
terms: One proportional to the error, one proportional to the integral of the error and one
proportional to the derivative of the error. The structure of the controller is shown in ﬁgure
2.1
It uses 3 gains, one for each of the 3 terms (KP, KI and KD), that have to be set before
the control operation is carried and that when correctly tuned make the controller to be
accurate and responsive (in case the reference is changing). ([2])
PID control is broadly used in the industry due to its capability of precisely controlling
several kinds of robots, including complex ones. However, as it depends on static gains it
is not an adaptable way of controlling, that is, it cannot react to changes in the structure or
the dynamics of the object under control (without being re-tuned) nor avoid the effect of
perturbations in the environment where it is working.
Because of this lack of adaptability problem, there have been different developments
aiming to solve it and so to implement an adaptive controller, starting as soon as 30 years
ago ([3], [4]). Some of them add extra units to the control loop which provide additional
terms to the one provided by the PID, others tune the PID gains with some learning
component and others introduce a different structure which replaces a traditional PID.

More recently, adaptive schemes based on biology have been introduced. In these
implementations, some of the functions of the control loop (feedback control, error
prediction, etc) are executed by blocks that resemble the functioning of actual parts of the
nervous system.
There is a tool, called Spiking Neural Networks (SNN), which is of particular interest
in the ﬁeld of control because of its capabilities. They are also called "Third Generation
Neural Networks".
The ﬁrst generation networks contained neurons which were only "threshold gates", that
is, each neuron produces a ﬁxed output when the sum of its inputs is above a threshold.
Second generation networks introduced the concept of activation function, that is, the
output of each neuron is the result of applying a given function to the sum of the inputs
([5]).
The main difference between SNNs and the previous generation ones is that the neurons
in SNNs resemble the actual neurons in the nervous system, in what to concerns to their
behaviour - which was discovered by Hodgkin and Huxley in the year 1952 ([6]). Therefore
this networks are closer to biology than their predecessors.
Basically, each neuron "ﬁres" a spike whenever the sum of its inputs is above a threshold.
Firing an spike means that the output voltage of the neuron rises suddenly from a ﬁxed,
negative, "rest" level to a different ﬁxed, positive level and just after doing that it decreases
back to the previous rest level. After each spike, there is a rest-period in which the neuron
cannot ﬁre.
==PAGE:23==
5
The main advantages offered by SNNs are ([7]):
• They can process big amounts of information in a very fast time, thanks to the coding
of that information into spikes.
• Because of the way the information is processed, these networks are speciﬁcally
suited for their use with temporal information, and so they are suitable to be used in
real-time control tasks.
• Any function which can be abstracted with a second generation network can also be
abstracted with a SNN and, in general, using less neurons
• As it has been stated, they are a better model of the functioning of the nervous system
and so can be used to implement some structures, pathologies or other characteristics
found in the animal (or human) nervous system.
The model of Hodgkin and Huxley ([6]) is computationally too complex to simulate
large networks in real time. Because of that reason, several other models have been pro-
posed ([8]), which, even when not behaving exactly like the real neurons, are more efﬁcient.
The model which has a better relation between its biological realism and its computa-
tional cost is the Izhikevich neuron ([9]). Because of this reason, this is the model which
has been used in this work. Details about it can be found in Chapter 4, in section 4.4,
including the equations that deﬁne the behaviour of the neuron - 4.3.
The use of schemes which resemble not only the functioning of the neuron, but also
the structural organization of the nervous system, allows to have some more advantages
which are also of interest in the ﬁeld of control.
In order to accurately control an object, its inverse dynamic model has to be known
in advance. That dynamic model is used to obtain the effort required to send to each
joint of the controlled object to obtain a desired position (and maybe orientation too) of
the end effector of the object. The computation of this model is computationally costly.
Furthermore, the inverse dynamic model changes and so has to be recalculated every time
there is some physical change in the object (such as changes in its shape or its weight) or
in the trajectory it is following.
And that is not the single problem, because the dynamic model also depends on the external
inﬂuence in the object and of course it changes if that inﬂuence changes. If the environment
is noisy, this inﬂuence is constantly changing.
The use of neural models in control schemes allows them to make use of the ability of
learning and adapting to provide a solution to these problems. These schemes are able to
abstract the dynamic model of an object through biological inspired learning so it does not
==PAGE:24==
6

have to be calculated in advance. What is more, as these schemes are constantly learning,
they are able to adapt to changes in the dynamic model, caused either by a change in the
conditions of the own controlled object or in the environment. They also allow to deal
with noisy environments thanks to these same adaptation capabilities. ([10], [11], [12])
In addition to being able to adapt to new conditions, some of the neuro-inspired
schemes are able to store knowledge about how to control a particular object under some
particular conditions (a particular context). They can later recognize if the context is
similar to the one which was present when that knowledge was gained and retrieve it, thus
fastening the adaptation (compared with abstracting a completely new context). ([12])
2.1
Related Works
Now, a review of some previous works which research and use some of the technologies
which are used in this thesis is presented. In particular, other adaptive controllers, both
non-biological and bio-inspired are of interest.
So are other works which use the same learning technique as the one which will be used
in this work - DA-STDP - and developments that implement a control loop based on the
structure of the nervous system, with different blocks resembling different parts of it.
2.1.1
Adaptive feedback controllers
Since many years ago, several developments with the intention of having a more
adaptive controller than a PID have taken place. Some of them use some extra unit,
together with a PID, while others use a different kind of controller.
The controller presented in [3] and [4] is a combination of two parts: a PD feedback
and some dynamics compensation. With this, the dynamic model changes that might occur
during the operation can be handled. So, with a more complex design that a PID, but still
having a controller based on different kinds of feedback and gains - thus the controller is
not adapting itself - an adaptive control can be provided.
[13] presents a control loop in which a PID controller is used, together with a learning
unit in charge of performing the adaptation.
There are two approaches presented: First, this unit provides some extra torque which is
added to the one provided by the PID controller before being sent to the robot. The second
keeps this "corrective" addition, but furthermore it also tunes the gains of the PID.
Therefore, these controllers improve the performance of the PID, although they make the
control loop more complex as they add extra units.
[14] uses a controller based on a neural network (not a SNN) which resembles the
structure of a PID controller. These means it has 3 synaptic weights inside it which work as
==PAGE:25==
2.1. RELATED WORKS

the gains of a PID, but because of the properties of the neural networks these weights/gains
are able to be retuned on-line. Because of this, this controller is able to provide better
performance than a PID while keeping a simple control loop.
2.1.2
Use of SNN as controllers
The use of SNN as controllers has been successfully implemented in several works
and with several types of robots.
[15] uses a SNN with several populations organized in 2 layers to control a 4-degrees-
of-freedom arm. In the proposed structure, there is one input layer which has seven
populations of neurons:
• 4 of them are in charge of receiving the position of each of the joints, that is, all of
them together represent the current position of the robotic arm.
• 3 of them receive the desired direction of move of the end effector, understood as
the ﬁrst derivative of the difference between the current and the desired position (in
Cartesian coordinates).
The output layer contains 4 populations, where each one of them is in charge of
producing the desired move in each joint. Every input neuron is connected to every output
neuron, and the way the network has to learn is by modifying the strength of those synapses
with the mechanism known as Spike-Timing-Dependent Plasticity (STDP). This process
adjusts the strength of the synapse in two ways: Long-term Potentiation (LTP), which
makes the synapse stronger, and Long-term Depression (LTD), which makes the synapse
weaker ([16]).
The decision about how much and when to strengthen or weaken a synapse has been
implemented in several ways ([16]), but always depends on the time difference between
the arrival of a Post-Synaptic Potential (PSP) - that is, a spike coming from the neuron at
the beginning of the synapse - and the generation of a spike in the neuron at the end of the
synapse, also called pre-synaptic potential.
Figure 2.2 shows graphically how does STDP work. When ∆t is positive, that is, when the
PSP is generated after the arrival of the pre-synaptic potential, the synapse is strengthened,
and vice versa. The impact of the strengthening or weakening (∆w) is bigger if the time
difference is, in absolute value, smaller.
The SNN is able to learn how to control the arm, but to do that it has to learn in advance,
connecting the output generated by a standard controller to the output neurons until the
synapses between the input and output neurons are correctly established.

2.1.3
Reinforcement learning with DA-STDP
To be able to learn online, that is, while executing the control task, and not depending
on previous kinematic or dynamic models, it is interesting to use an algorithm which
enables reinforcement learning.
Reinforcement learning is a technique which consists in making the system learn to behave
in a proper way by rewarding it when it does so. The fact that a behaviour is rewarded - or
reinforced - makes it more likely to be repeated in the future and so in the long term the
global behaviour will be close to - or ideally, equal to - the expected one.
So, while executing the task and by rewarding the right behaviours, the global perfor-
mance gets better with time. There are some modiﬁcations of STDP that allow to do that,
being the most interesting, because of its biological similarities, the Dopamine Modulated
STDP.
This mechanism, proposed in ([17]), starts an "eligibility trace" whenever there are a
pre- and post-synaptic potential happening close in time in a neuron, instead of directly
changing the strength of the synapse. If there is some reward, dopamine is released and
then the strength of the synapse changes, considering the current value of the eligibility

trace, which is decaying over time.
The process is shown graphically in ﬁgure 2.3, which has been taken from [17].

[18] successfully uses this kind of learning to teach a mobile robot to develop a food-

[19] also use DA-STDP for several experiments with a robot in which they implement

4.1
Network structure
The Spiking Neural Network is made of different populations, structured in layers.
This network structure is replicated once for each joint. There are 3 layers in the network:

– The bigger group is connected through DA-STDP excitatory synapses to the

4.1.1
Synapses and connections
• The excitatory DA-STDP synapses are used to connect the ﬁrst group of input
neurons (the bigger one) with the two neurons of the output layer. These synapses
are plastic, what means that their weights change according to the STDP rule
explained in subsection 4.5.2, when dopamine is released.
– The weights of these synapses are randomly initialized around a mean of 50
mV, with a standard deviation of 100. This mean is intended to be low, with
respect to the maximum value, but to allow for some random move to start,
which is also why standard deviation is that high - if the weights differ more,
the output of the 2 neurons will also differ more and so there will be more
movement to reward and the learning will happen before.
– The weights are bounded between 0 (because they are excitatory) and 300 mV.

4.5
Learning mechanism
The learning is performed by changing the weights of the excitatory synapses between
the input and the output layer. The mechanism which controls this learning is DA-STDP,
governed by regulating the amount of a dopamine in the pool - through the ﬁring of
the neurons of the dopamine layer. Same as the structure of the network, the learning
mechanism is replicated - and so, independent - in each joint of the robot.
4.5.1
Dopamine
The network learns through Dopamine Modulated Spike-Timing-Dependent Plasticity
(DA-STDP), as described in [17]. This dopamine modulation allows to choose when to
enable or disable the learning, which happens because of STDP plasticity.
There are 2 neurons in charge of the dopamine concentration: When one of them ﬁres,
dopamine is added to the "dopamine pool" and when the other one ﬁres dopamine is
subtracted from that "dopamine pool". In addition, if there are no ﬁrings, the concentration
of dopamine naturally decreases. If there is dopamine in the pool, learning (changes in
synaptic weights) can happen. Furthermore, if there is more dopamine, the changes in the
weights will be more signiﬁcant - [17].
4.5.2
Spike-Timing-Dependent Plasticity
The STDP plasticity induces changes in the synaptic weights depending on the time
difference between the arrival of the pre-synaptic spike (the one generated by the neuron
at the beginning of the synapse) and the generation of the post-synaptic spike (the one
generated by the neuron at the end of the synapse).
There are several implementations of the mechanism that can be used as a form
of plasticity for learning. All of them have one key aspect in common: The fact that
the change in the synaptic weight depends on the time difference between the spikes
generated in both sides of the synapse (the beginning and the end). Several implementations
of STDP are summarized and compared in [16].
In this work, the implementation which will be used is Long Term Depression-biased
Hebbian STDP. There are two main reasons for this decision:
• Hebbian STDP a relatively basic form of plasticity for learning, is intuitive and
respects the Hebbian theory of learning.
In the year 1949, Donald Hebb introduced his theory of learning - [31], which can
be summarized in the sentence "Neurons that ﬁre together, wire together". More
speciﬁcally, it states that if one neuron ﬁrings constantly precede the ﬁrings of

4.5.3
Stability and competition
The system will be stable if the following conditions are met:
• The learning in the SNN is stable
• The SNN is able to learn (to implement) the required control function
The ﬁrst of those two conditions is the stability of the SNN. For the second one to
happen, there has to be competition in the SNN.
The ﬁrst of them, stability, happens only under some conditions. STDP is by deﬁnition an
unstable learning mechanism, but as it was proved in [33] and in [32], the average strength
of the weights will reach an stable point as long as the starting average strength is below

The competition inside a SNN means that different groups of neurons try to develop
opposed behaviours (for example, ones trying to execute a contract move and other trying
to execute an extend move) and so they "compete" for the control over the output neurons
(by developing stronger synapses). ([34])
Because of this competitive behaviour, the network is able to implement the different
functions it might have to, assuming that there exists a function like the one which is to be
implemented. In this case, it is sure that it exists, as it is also the function implemented by
a P controller.
The implementation of STDP which is used in this work - Hebbian STDP - is proven

4.5.4
Reward function
The dopamine is introduced in the pool through the ﬁring of the associated dopamine
neuron. The idea is that this neuron should be ﬁring when the behaviour developed by the
robot is considered adequate and so should be rewarded. Otherwise, it should not be ﬁring
and the other neuron - the one in charge of subtracting dopamine - should ﬁre to empty the
pool and so impede any learning of an incorrect behaviour.
As the main goal in a control task is to reduce the error, understood as the difference
between the reference and the actual state of the robot, the ﬁrst condition to produce a
reward in this job is that the error is reduced.
Furthermore, if more reward is given when the error is reduced more, that is, if there is
a relation between the error reduction and the dopamine introduction, the learning can be
performed faster ([35]). This relation is expressed with the equation 4.6a. It is also shown
in ﬁgure 4.5.

just the rate it takes when it is active) and the time constant - τ - which deﬁnes how fast
does the rate increase when the error reduction is greater.

4.6
List of all the parameters of the controller
In this section, a relation of all the parameters that can affect the performance of the
controller and have been tuned is presented. Each parameter and its inﬂuence are ﬁrst
explained and then, a summary table with the all the values used in this work is presented.
Please note that these are the parameters that have been tuned during the development
of the controller, and because of that they are used as the parameters that deﬁne this
controller. Nonetheless, there are more things which could be changed and which would
affect the performance (for example, the model of the neuron or the type of STDP).
4.6.1
Input encoding
• Standard deviation of the distribution: This has the effect of affecting how many
neurons are used to encode each value in the input layer. Looking at ﬁgure 4.3, it can
be understood that if the standard deviation increases, more neurons have a non-zero
rate. Ideally, the number of neurons used has to be small, to make the coding of
the input to be precise and distinguishable, but at the same time, and because of the
functioning of STDP, several neurons have to be ﬁring simultaneously.
In this work, the standard deviation is deﬁned by σ = 0.01 × N, where N is the
number of neurons used to encode the value. It is dependent on the number of
neurons because that way the Gaussian function will keep its shape for any given
number of them.
• Scaling of the rate obtained with RBF (or maximum rate): The rates obtained with
RBF are between 0 and 1. In order to be appropriate for the output decoding and the
network structure, these rates have to be scaled, because if they were too small they
would cause no output and therefore no move.
In this work, the value obtained with RBF is multiplied times 200 Hz.
4.6.2
Output decoding
• Effort per spike: This value has to be kept small, because if it were too big the
effort obtained - when multiplying it times the frequency of the output neuron - and
applied to the robot could also be too big and over the maximum value that can be
==PAGE:52==
34
CHAPTER 4. DETAILS OF THE IMPLEMENTATION
accepted by the robot, which could cause it to malfunction or even to get broken.
Furthermore, when the effort is kept as small as possible, the robot oscillates less
around the reference.
However, if the effort were not big enough the robot might not move as fast as
needed and so it would only be useful for slower trajectories.
The value used in this work has been 0.002 N.
• Duration of the effort: In practical terms, this is the time window in which the
spikes are counted and during which the effort is applied. Because of the ﬁrst of
those things, it has to be big enough to allow to several spikes to happen in that time
window. On the other hand, because of being the time during which effort is applied,
it has to be small enough to allow the effort which is sent to the robot to change
quickly and so to allow the move of the robot to be fast.
In this work, the value used is 20 ms.
4.6.3
Reward function and dopamine
• Maximum rate: This means the maximum rate that can be set to the neuron
in charge of introducing dopamine or the rate that is set to the one in charge of
subtracting dopamine - for better explanation check subsection 4.5.4. Together with
the parameters A+ and A−(explained below), it works as the learning rate.
This is an indicative of how fast is the network learning. If it were smaller the
network would take more time to learn and if it were too big the network would not
be able to learn.
The value for this parameter is 10 Hz.
• Time constant for the decay of the function: If this parameter, which deﬁnes how
fast does the rate for the neuron in charge of dopamine introduction grows with
the error reduction (check subsection 4.5.4), were too small, the reward would be
very similar to a binary one, thus losing the advantages of using an improvement-
dependant reward - being able to learn faster ([35]). However, if it were too small,
there would be few reward and so slower learning or even no learning at all.
The value used in this work is 0.5 s
4.6.4
Network structure
• Number of input neurons: Increasing the number of neurons implies that there are
more units to compute and so that the spending in computational power is greater.
Actually, for every input neuron, there are 3 things to calculate in each iteration: The
==PAGE:53==
4.6. LIST OF ALL THE PARAMETERS OF THE CONTROLLER
35
behaviour of the neuron itself and the one of each of the 2 synapses connecting it to
the output layer. So, in general the number of neurons should be kept as small as
possible to save computational power.
However, the model requires to have a certain complexity for it to be able to properly
implement the task it has to. Also, the input neurons represent positions in space (as
explained in section 4.2), so this representation in the space will be more precise if
there are more neurons, as the space will be divided in more, smaller, segments.
In this work, two groups of input neurons are used for each joint, one of 120 and
another of 30, so 150 neurons are used per joint.
• Number of inhibitory synapses: There is neurological evidence that the percentage
of synapses which are inhibitory in the nervous system is around 20-30%. Values in
that range are used in several developments, such as [18], [19] or [32].
The implementation in this work has consisted in splitting the input neurons in 2
groups, and connecting the smaller of them with ﬁxed, inhibitory synapses, while
connecting the bigger one with DA-STDP excitatory synapses, to the output layer.
As the percentage of neurons used for this purpose is 20%, so is the percentage of
synapses.
4.6.5
Synapses
• Weight limits of the STDP synapses: First of all, these are excitatory synapses,
meaning that they will always have a weight greater than or equal to 0, so that is
the lower bound. The upper limit is intended to be high enough so as to let the
weights evolve and differentiate between them. If it were too high, however, and
knowing there will be several weights taking high values, the output neurons would
be constantly saturated, and so ﬁring always at the same rate, and so no move would
be produced.
In this work, the maximum weight of these synapses is 300 mV.
• Initial weights distribution: The weights of the STDP synapses are randomly
initialized. For this purpose, a Gaussian probability distribution is used. The mean
of the distribution has to be smaller than the ﬁnal mean for the learning to be stable
(as explained in subsection 4.5.3), but if it is high there will be higher inputs arriving
to the output neurons at the beginning, so they will ﬁre more spikes and there will
be more movement of the robot to start with.
Also with the purpose of having more movement, the standard deviation is set to
be high. This means that the weights will be more different between them and so
the output of each of the 2 output neurons might differ more, leading into more
==PAGE:54==
36
CHAPTER 4. DETAILS OF THE IMPLEMENTATION
movement at the beginning. Having more and more different movement is interesting
because by rewarding the desired part of that initial movement the learning starts to
happen faster.
The distribution used in this worked is deﬁned by µ = 50 mV and σ = 100.
However, please note that the weights are bounded between 0 and 300 mV, so any
weight outside of those limits is changed to the closest of those.
• A+: It deﬁnes the maximum amount of strength added to a synapse in a single
potentiation. It could be understood - together with A−, as the learning rate. This
means that, as it deﬁnes how much does a weight change every time it does, it
deﬁnes how fast the learning is produced.
If it were too big, the learning would not be able to happen as the network would
not be able to converge on a valid set of weights to perform the control. If it were
smaller, the learning would take more time to happen.
The chosen value in this work has been 2 mV.
• A−: It deﬁnes the maximum amount of strength subtracted to a synapse in a single
depression. What has been said about A+ applies also here, but in addition, the
proportion between this 2 parameters is also relevant. Bear in mind that, as explained
in subsection 4.5.3, this value has to be greater than A+ for the learning to be stable
(For the Long Time Depression to be greater than the Long Time Potentiation).
Furthermore, the closer it gets to A+, more synapses are able to reach the maximum
weight, and the further it goes, less weights are potentiated and so less input neurons
are used "at a time". This means that if it were too much bigger than A+, few
neurons would be used and so the model would lack complexity and would not be
able to implement the needed function.
In this work, the value of A−has been set to 2.1 mV.
• Weights of inhibitory synapses: They are supposed to have a value which is
comparable to the weight of the excitatory synapses. Because of that, it is decided to
set all of them with the same value - but, of course, negative - as the starting mean
of the excitatory ones.
Because of this, all the inhibitory synapses have a weight of -50 mV.

CHAPTER 6
Conclusion and future work
Looking at the results from Chapter 5, it can be understood that:
• The controller is able to learn how to perform the control task, and to do so in a
short time period.
• The controller is able to keep the control task working when the trajectory changes,
either if the change happens in terms of frequency or amplitude, in one or both
joints.
• There is room for improvement in the precision of the controller. A comparison of
the precision of the developed controller with others is provided in section 6.2.
As a reminder, the test which has been used in all the analyses and comparisons
from now on is the one in section 5.8, as that is the one which contains several forms
of adaptation. All the comparisons and analyses are done with an average of several
repetitions of the experiments.

Because of this, it would be interesting to use a low-pass ﬁlter between the network and the

so the robot would follow the trajectory in an easier way - and probably more accurately.
6.3
Future work
The developed controller has some advantages with respect to P or PID controllers - not
having to tune the gains, resembling biology in its components, methods and behaviours -
but has the disadvantage of being less accurate (in average) than a P controller.
The ﬁrst matter to be faced, then, should be a improvement in the precision of the
controller. As it has just been suggested in subsection 6.2.3, ﬁltering the effort before
sending it to the robot and increasing the effort per spike could cause an improvement.
There is also a signiﬁcant difference between the accuracy in following the trajectory in
the two joints of the robot and this suggests that some of the parameters of the controller
might have to change between one joint and another. However, these would make the
controller to be less versatile if it wanted to be used with different robotic conﬁgurations
with more joints.
Another idea is to add some unit providing corrections to the torque generated by the SNN
controller.
==PAGE:78==
60
CHAPTER 6. CONCLUSION AND FUTURE WORK
In biological nervous systems, the unit in charge of adding corrections to make the
movement smoother and more precise is the cerebellum. This unit has also been extensively
researched and modelled, and so there are several models that could be used in conjunction
with the developed controller to make it more precise.
At the same time the introduction of this controller in control loops that have some other
parts which are based on biology, but not the controller, could make those control loops
closer to biology (by substituting a non-biological component - the previous controller -
by a bio-inspired controller - the newly developed controller).
As an example, the AFEL architecture, proposed in [23], is an architecture that could
get beneﬁted from the use of this controller, by becoming closer to biology, and that
could also be a more precise solution that the controller alone, because it incorporates a
cerebellum to provide corrections.
