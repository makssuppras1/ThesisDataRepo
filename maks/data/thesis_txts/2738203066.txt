Abstract
3D object detection is an increasingly important task to solve. Most of the research on this
topic is for fully-supervised methods, craving enormous amounts of data. 3D data annota-
tion is labour-intensive and expensive. Therefore this thesis focuses on weakly-supervised 3D
object detection, extinguishing the need for such data. By specifically investigating monocu-
lar methods, we leverage the high accessibility of single-camera systems over costly LiDAR
sensors or complex multi-camera setups. We create three methods using 2D box annotations:
A proposal-and-scoring method, a pseudo-ground-truth method, and a weak Cube R-CNN. The pro-
posal method generates 1000 cubes per object and scores them. The prediction of this method
is used as a pseudo ground truth in the Cube R-CNN framework [6]. To create a weak Cube R-
CNN, we modify the framework by replacing its 3D loss functions with ones based solely on 2D
annotations. Our methods rely heavily on external, strong generalised deep learning models
to infer spatial information in scenes. Experimental results show that all models perform com-
parably to an annotation time-equalised Cube R-CNN, whereof the pseudo ground truth method
achieves the highest accuracy. The results show the methodsвҖҷ ability to understand scenes in
3D, providing satisfactory visual results. Although not precise enough for centimetre accurate
measurements, the methods provide a solid foundation for further research.
The code for this thesis can be found on GitHub: https://github.com/luchsonice/3dod.

Chapter 1. Introduction
2D object detection has over the recent years been established as a вҖңstandardвҖқ task in computer
vision. The general motivation is to answer вҖңWhat objects are where?вҖқ From a high level, this
is solved by assigning labels to rectangular boxes around objects of interest.
In this thesis, we focus on 3D object detection, which answers the question вҖңWhat objects are
where and in what size and orientation?вҖқ, essentially finding 8 bounding corner points for
each instance in a frame. More precisely, these points are found by determining the centre of
an object, the distance of the centre to the camera, the length, width and height of the object
as well as its rotation. This dimensional information allows for an accurate understanding of
the spatial relationships between objects. The advantage of 3D- over 2D object detection is that
can detect partially occluded objects, a common issue for 2D object detectors. In this way, one
can fully reconstruct a scene in a 3D world.
However, the largest constraint holding back improvements in 3D object detection is the lack
of general annotated training data. Only huge corporations in the likes of Google have the
funds to use platforms, such as Amazon Mechanical Turk, to obtain the large amount of data
required. As an example; to annotate the cubes in the SUN-RGBD data set [45], the authors
hired and trained online workers. They spent 2,051 hours in total to label 10,335 images, which
is considered a tiny dataset. This means a single image takes roughly 12 minutes to 3D annotate,
compared to 65s for 2D annotation (вүҲ11x less).
Labelling images with 3D annotations is complex and severely more labour-intensive than 2D
annotations. This begs the question of whether it is possible to use training data that does
not contain the full information needed in a classical 3D object detection scenario and still
solve the problem accurately and more efficiently. For example, one could use widely available
2D annotations to generalise to 3D. There are many possible approaches, as the level of weak
supervision can be varied greatly.
Only using a weaker annotation reduces the annotation cost considerably. This makes it easier
to obtain larger amounts of data with the same effort. Or, conversely, the same amount of
annotated data can be obtained in a fraction of the time and cost. In some cases it helps with
handling ambiguity and noise, since annotation data with weak supervision labels is a simpler
labelling task and hence reduces the error in the labelling process. Additionally, different types
of weak labels can be used for cross-checking. In this context, we specify weak supervision as
anything that is less than 8 corner points for each instance.

Chapter 1. Introduction
Existing models have been made to solve the problem within the two major domains for 3D
object detection, which are indoor scenes and outdoor scenes. Indoor scenes mainly consist of
objects such as furniture [2, 3, 45], while outdoor scenes primarily contain cars, pedestrians, and
cyclists [7, 11]. Many private companies are creating solutions with the goal of developing self-
driving cars and thus most research has been done for this application. In the driving scenario,
it is typically assumed that all the objects in the scene rest on the same plane. Furthermore, a
LiDAR sensor is typically applied to gain accurate depth information. Consequently, these
methods lack generalisability across diverse scenarios. However, our focus is solely on the
camera-only scenario using a single camera, which is more challenging. This task is known
as monocular object detection. This is by nature an ill-proposed problem, as no information
about depth is given. This results in several ambiguities, one of which is the scale ambiguity.
Objects of different sizes and depths can appear with the same size on the image plane. Another
ambiguity is given due to the single perspective available. The appearance of instances can
change drastically with the viewpoint, making it difficult to make stable predictions. Inferring
spatial information from 2D images is the core difficulty of this task.
1.1
Goals & Objectives
We pursue several achievements in this thesis. The main goal is to build a weakly supervised
model that solves the task of 3D objection detection with no performance degradation in com-
parison to the accordingly fully-supervised model. Here, we make a differentiation based on
the annotation time, creating a fully-supervised baseline that is annotation time equalised to
our designed methods.
Our study consists of two approaches, firstly a traditional вҖңproposalвҖқ method inspired by early
R-CNN methods and secondly an end-to-end deep learning method based on the Cube R-CNN
model [6]. They both use 2D bounding boxes as input since it is a high level of weak supervision
and 2D annotated data is abundant. The proposal method transitions weakly supervised 2D
techniques to 3D. An ensemble of external methods and clever computation creates proposals,
while an assortment of scoring functions evaluates the proposals and picks one of those as the
final prediction. We do not only study whether prediction accuracy and annotation time scales
proportionally, but also whether the outcome of the proposal method can be used as ground
truth to the fully-supervised Cube R-CNN.
Furthermore, we address the challenges of monocular 3D object detection, including scale am-
biguity, by creating a weakly supervised version of Cube R-CNN. To make this work, we ex-
change the loss functions with self-designed loss functions relying on nothing more than 2D
information. A qualitative examination tells us whether it is advantageous to enhance models
in a multi-stage setup, either with a depth estimation model or other external models as an
extra input.
The analysis concludes that using the pseudo ground truths yields the strongest model, while
another model shows potential leading up to the possibility for further research.

Chapter 2. Related Work
In this section we provide an overview of the different fields influencing our work. We cover
the related literature within fully-supervised 2D and 3D object detection in sections 2.1 and
2.2. Section 2.3 covers a variety of weak supervision methods, both in 2D and 3D. Lastly, we
explain some of the core models employed in our method in section 2.4.
2.1
2D Object Detection
It is relevant to understand some of the key 2D object detectors, as these models are usually
extended to 3D or used as backbone models in 3D model structures.
2.1.1
CNN Architectures
A common goal with object detectors is to have them run in real-time. This is the major achieve-
ment of the Faster R-CNN method [41]. It is a key method and has inspired most of the newer
methods. It is a CNN-based two-stage method consisting of two parts: a region proposal net-
work (RPN) and a classification network. The core idea to excel running time is to share the
same CNN model (weights) across both of these two networks. The CNN is then trained jointly
with both the RPN and the classification part. The method works by passing an image through
the backbone to obtain feature maps. Then anchor boxes, typically 9, of different sizes are gen-
erated around anchor points, which are points evenly spaced on the image. One only wants to
consider the boxes where there is a high overlap with the ground truth boxes, as the positive an-
chor boxes contribute the most to optimise the model. A few negative boxes are left in to match
the number of positive boxes, the rest are discarded. At this point, the boxes (denoted regions
of interest, ROIs) must be classified. To make sure that the classification module sees images
of the same size, a ROI pooling technique is employed. During inference, non-maximum sup-
pression (NMS) is used to remove overlapping boxes.
2.1.2
Transformer Architectures
Vision transformers are already well established in 2D object detection [25]. Known as de-
tection transformers (DETR), see DETR [9], Deformable DETR [67] and DINO [61]. Yet, the
encoder and decoder only act as a вҖңneckвҖқ, see figure 2.1, which means they sit on top of a CNN
backbone and perform the sequence modelling on flattened image features from the CNN.
The improvements most of these methods have focused on are mostly to shorten the training

2.2
fully-supervised 3D Object Detection
As estimating cubes is rather challenging, several approaches exist. Many crave more data than
just images. As our focus is on monocular 3D object detection, we quickly glance over what
other fields exist and go into depth with monocular methods.
2.2.1
LiDAR
As accurate depth information greatly increases the performance of models [31, tab. 2], a
LiDAR sensor is sometimes implemented. The sensor returns a sparse point cloud giving in-
formation about the depth and shape of the object but only of the side facing the sensor, so one
gets a hollow, one-sided representation of objects.
As methods using LiDAR information are much more precise, it is desirable to get accurate
point clouds without the need for a sensor. DD3D [37] creates a LiDAR point cloud from a
depth map and applies LiDAR object detection methods. It is a clever trick to utilise existing
LiDAR-based methods. This has proven to increase performance.
Some methods only use LiDAR information during training to distil knowledge into the model,
but not during inference, see e.g. [22, 38].
2.2.2
Cube Proposals
A common method for 2D object detectors is to propose many candidate boxes and use a
strategy to keep only the most promising, this is for example what the methods Selective Search
[50, 68] and Region Proposal Networks (RPNs) [41] do. How to do region proposals effectively

in 3D is a huge issue. Naively extending the same ideas has some concerns about the viabil-
ity of the methods because the candidate search space is increased with one extra dimension,
which increases the number of cubes with different aspect ratios to scale from n2 to n3.
The method of leveraging an existing 2D object detector and extrapolating cubes from 2D boxes
is what Deep3DBox [34] has some simple ideas for and is dubbed YOLO3D. The interesting part
is the multiple re-implementations [12, 13] which improve on the original paper considerably.
The key observation they utilise to make the method work is that a cube fits tightly into a 2D
box, such that each side of the 2D box is touched by at least one of the cube corners. This is
already enough information to fix 4 of the 9 degrees of freedom defining a cube. Their way
of proposing cubes consists of some simplifications meaning that there are only 256 possible
configurations of 3D boxes inside each 2D box. This is further reduced to 64 for cars, where
pitch and roll are absent due to cars being only able to yaw rotate on the ground. This is a very
useful simplification that is only applicable in the car scenario, roads provide a very large, even
surface which is easy to locate, also because the camera is aligned to the road.
2.2.3
Explicit Depth Maps as Input
These kinds of methods take an extra piece of information as input in the form of a depth map,
which is an image where the colour-coding resembles the depth of the scenes. The depth map
can either be captured with a camera-based sensor [45] (Intel Realsense, Microsoft Kinect) or it
can be estimated from a LiDAR scan [17].
This additional depth information is lately used as the advantage a teacher has in a student-
teacher model. However, as [54] vividly displays, it can be unclear how exactly the model was
trained. The advantage of this setup is that it is possible to run inference on the model without
depth maps. EA-LSS [17] converts LiDAR information to a dense depth map which is used to
compute a loss with a depth map predicted by a CNN.
2.2.4
Depth Estimating Monocular Methods
CenterNet [64] is the foundation of many of the mentioned methods [26, 56, 63]. The model
predicts the centre of objects as a heatmap. This turns out to be a useful first step for many
methods. DLA-34 [60] is used as the backbone network. Many of the papers use DLA-34 as a
backbone, which mostly appears to be for fair comparison between the methods.
To model the different sizes of objects that are near and far, DEVIANT [23] plays around with
resizing and padding an image to simulate artificial depth translation to solve the issue of
depth. Perspective and the vanishing point are the motivations behind this method. For cars
that equals driving closer to the objects in front without changing the depth estimation, because
one can consider a road as a triangle ending in a single point infinitely far away. They propose
to replace vanilla convolution blocks with scale equivariant steerable convolution blocks which
are convolutions that are invariant concerning depth. This is an extension of the standard con-
volution block, which is by default invariant to 2D translation, but not depth translation. An-
other approach to this idea is the MonoDDE model [26] which performs several (20) depth

estimations for each target, combined with outlier estimation to make the model more robust.
It introduces extra heads for estimating the dimensions of objects. They model the problem
such that to locate an object one must solve 3 equations for x, y, and z coordinates respectively.
However, if you have the centre point you already have x and y so you only need to estimate
the z. The authors show that you can substitute out the z-coordinate by using the centre point
as a prior, which is typically very accurate (85% of the time only 1 pixel off). Then they model
the depth of each point as a normal distribution and find and remove the outlier points with
an algorithm similar to the EM algorithm.
The CenterNet model is utilised in a slightly different way in the MonoSAID [56] model. Fea-
tures of scale 1/8 and 1/4 are extracted from the model. Two sub-modules work on the dif-
ferent feature scales. The first module on the 1/8 scale captures the depth distribution of each
object, this is achieved with dilated convolutions of different dilation rates. Meanwhile, the
other module working on 1/4 scale learns the multi-scale features by using the feature map
and applying 1x1 convolutions to generate two new maps. These are multiplied together to
create a large attention map and fed into a 3D detection head which predicts, position, size,
and angle.
There is another approach which transforms the image to voxel (3D pixel) representation, CIE
[59]. At its core, a 2D backbone generates two feature maps, namely depth keys and image
features. An attention mechanism replaces point cloud estimates to create the 3D features
through the depth keys and image features.
2.2.5
Ensemble Methods
MonoFLEX [63] is another extension of the CenterNet method. They handle how objects whose
centres are inside or outside the image differently (decoupled representation). They use an
uncertainty guided ensemble to estimate depth. This means several keypoints (vertices of the
predicted cube) are predicted for each object. The depth of each keypoint is estimated by in-
dependent estimators by taking the physical placement of the points into account. This gives
a more robust estimate.
2.2.6
Depth Aware Transformers
This section covers a collection of methods based on 3D Detection transformers (DETR) [9].
MonoDTR [18] & MonoDETR [62] are DETRs for 3D monocular object detection. The tiny dif-
ference between them is that MonoDTR uses YOLOv3 for 2D box prediction, while MonoDETR
uses a ResNet. They make a depth encoder, which responds to the 3 last convolution layers of
different sizes of a ResNet50 model added together and then add another convolution layer
on top of the pooled layers. The same Resnet also makes a visual encoding. All these are fed
into a transformer structure, which has had its positional encoding swapped for a depth pos-
itional encoding. They require a loss that finds the optimal bipartite matching between boxes
both in 2D and 3D. See also Deformable or DINO DETR [61, 67] which are improved versions
concerning optimisation speed.

Another method that also constructs a depth map from the 3 last layers of the CNN backbone
is the MonoPGC method [55]. The method resembles the MonoDETR architecture a lot but
adds an intermediate output where a depth map is used as reference ground truth.
2.3
Weak Supervision
Since very few weakly-supervised methods have been tried for 3D object detection, we instead
give an overview of methods which are developed for 2D object detection. We hope that many
of the ideas can be transferred. Levels of weak supervision can be centre clicks [36], extreme
clicks [35] or image labels [65].
One model using segmentation annotations is MobilePose [16]. It uses synthetic data, pose,
and shape supervision. The first of two heads is a detection network based on mobilenetV2,
which outputs a heatmap in 2D. The second head is a regression head. They also use segment-
ation in 2D as weak supervision. The weak supervision is provided by a smaller amount of
synthetic data (similar to [10]), which is generated from placing known objects in AR, and also
by copy-pasting objects in 2D. The model is very light and can run in real-time on a phone. It
is implemented in the accessible Google Mediapipe framework.
2.3.1
Click Supervision
Various forms of click supervision have been proposed, namely: Center clicking and extreme
clicking [35, 36] in which annotations are placed at the centre or the extreme points of ob-
jects. When using 2 centre clicks one can estimate the size of an object by inferring the distance
between the points, as for smaller objects the points would be closer to each other than for a
larger object. This comes down to statistics; when a crowd-sourcing annotation platform is
employed it is common to label the same image more than once and aggregate the results over
multiple people. The point of using a centre click is that it is about 18x quicker than stand-
ard 2D box annotation. How the clicks are used: A box centre score is calculated and based
on that the most optimal box is selected as the candidate. Then a standard object detector is
employed.
Another method from [33] uses some of the same ideas but incorporates LIDAR data. The Cen-
ter click annotation is used as the centre point for a generic shape that can be individualised
to each object. The method proposes a 4m wide cylinder around each object and shrinks it to
fit a box, this works because they only detect cars which have roughly the same size. Further-
more, they still use around 25% precisely annotated data for training, the upside of this is that
performance is very close to the fully-supervised methods.

is both a 1D increase in input and output. The idea of weakmono3d [49] is to annotate with

Omni Supervision
The core idea of this method is that it can leverage different types of weak supervision as input

2.3.4
Image Level Annotation
This line of work focuses on using only the image level (class) annotation as the weak super-
vision. The reason why this works well is that you can leverage much larger image data sets
and strong pre-trained models in combination with region proposal methods. But they only
exist for 2D, therefore all these methods are 2D based. Weakly Supervised Deep Detection Net-
works [5] introduces a way to simultaneously perform region selection and classification. They
propose to branch off a pre-trained CNN in one of the later feature layers. Then proposal boxes
from a standard algorithm (Selective Search or Edge Boxes) are used in the model by using a
spatial pyramid pooling layer (ROI pooling across several image feature layers), which ensures
that the box proposals generate fixed-length outputs.
Other papers, [65], focus more on the large vocabulary detection problem, where many more
classes are present, which makes the problem of assigning the correct labels to each box in an
image more difficult. In their experimentation they found that the proposal network and box
regressors are not the performance bottleneck of current two-stage methods, hence why the
==PAGE:19==

9
focus is on classification. Their method applies image-level supervision to the proposal of the
largest size. Yet, it is a bit surprising that the model still learns to find small objects. The CLIP
model [40] is used as the classifier because of its ability to generalise well to many classes.
Other methods [19, 42, 48] solve roughly the same thing, but focus on different aspects. [42]
fixes 3 common problems in weakly supervised learning. 1) Instance Ambiguity, 2) Failing
to differentiate clustered instances and 3) Part Domination: When the models focus on the

[29, 39] use the inherent nature of 3D point clouds to make a 3D analogue to the Selective Search

inside the boxes together into вҖңsuper-pixelsвҖқ to try to figure out what should be grouped as

generates labels for 3D boxes that the student can learn. Furthermore, some assumptions about
the aspect ratios of cars are used to simplify the rotational placement of the cubes.
2.3.5
Point Cloud Supervision
Some papers we have been inspired by explain how they use point clouds [39] as the weak su-
pervision without 3D labels. Using a point cloud obtained from a high-quality LIDAR scanner
means it is easy to obtain accurate depth anywhere in an image. They then use a normalised
point cloud density as a guideway for proposing cubes. To improve rotation estimation, they
use a student-teacher setup where the teacher model is a viewpoint regression network pre-
trained on ImageNet and PASCAL VOC with image-level classification labels and viewpoint
labels.
2.3.6
Unsupervised Proposal Methods
These methods are based on classic computer vision methods on raw images focusing on cer-
tain features of images. This was how 2D object detection was done before supervised (deep
learning) methods were introduced, these methods are completely unsupervised. Selective
Search [50] (SS) is one such method which uses the concept of a hierarchical segmentation вҖңSu-
perpixelвҖқ approach. The method starts from an initial over-segmented image generated by a
graph segmentation algorithm. Then, segmentations are merged based on their similarity in
colour, texture, size, and shape to get segmentations of larger and larger sizes. Edge Boxes [68]
(EB) is a method to generate object proposals directly from edges. Similar to segmentations,
edges are a condensed representation of an image.

The point of this ROI pooling is that you can utilise a single global feature map (from a CNN) for
the entire image for efficiency. The problem with this approach is that for each object instance
on the image, one must crop the feature map corresponding to the size of each instance. This
yields different sized features which is undesirable for later linear layers. ROI pooling divides
each region proposal into smaller regions and applies max pooling to ensure an output of the
same shape. This is to overcome the fixed size limitations by homogenising the shape of all

are estimated from the camera view. Furthermore, if all annotations in an image are filtered

Cube Proposal
We introduce a method which, rather than relying on a cube head like the baseline does, see
section 2.4.3, relies on an alternative approach of generating proposals and evaluating them
afterwards. This consists of generating n cube proposals for each object.
A critical property of a proposal method is high recall and efficiency. In other words, it should
find as many of the actual objects in as few proposals as possible.

4.6
Learning to Predict Cubes
We can utilise the same heuristics as those developed for the proposal methods to create a
weakly supervised deep learning method to predict 3D cubes. For this, the heuristics must
be turned into a way to optimise the model, this entails transforming the heuristics into loss
functions.
The foundation for the weak Cube R-CNN is the Cube R-CNN architecture, but everything
related to the 3D ground truth is removed. This means we use the same 2D region proposal
network and 3D Cube head and train the whole model end-to-end.
The input to the cube head is the features from the 2D network which are pooled with ROI
Align V2, see section 2.4.4. The cube head predicts 13 parameters describing a cube:
вҖў [u, v] represents the projected 3D centre on the image plane relative to the 2D RoI
вҖў z вҲҲR+ is the objectвҖҷs centre depth in meters transformed from virtual depth zv
вҖў [w, h, l] вҲҲR3

вҖў p вҲҲR6 is the continuous 6D allocentric rotation
вҖў Вө вҲҲR+ is the predicted 3D uncertainty
[u, v] is a bounding box regression which can move the centre of the cube from the centre of
the box. This helps because the centre of the cube might not exactly be at the centre of the
box. Virtual depth serves to make the camera focal length similar between images taken with
different cameras of different focal lengths, this is important because the focal length impacts
the perceived size and depth of objects. However, it is not as important in our case since all
images are taken with very similar cameras. We predict the log-normalised dimensions simply
to avoid cubes of negative dimensions. The rotation is predicted in a special way. It is better
to predict 6d rotation than a full rotation matrix or Euler rotation. Directly cited from [66]:
вҖңWidely used representations such as quaternions and Euler angles are discontinuous and difficult for
neural networks to learn. [...] 3D rotations have continuous representations in 5D and 6D, which are
more suitable for learning. [...] continuous rotation representations outperform discontinuous ones for
several practical problems in graphics and vision, including a rotation estimator for 3D point cloudsвҖқ.
Allocentric rotation means that the rotation is local to the position of the object and not from
the POV of the camera. The uncertainty is handy to filter unconfident predictions.

This section covers the development process of the вҖңproposal methodвҖқ for proposing cubes.
The general idea is to consider the distributions of the various training parameters as our targets

cubes and later head on to more advanced approaches.
5.2.1
Center
Random proposals should be in a reasonable range. To ensure an even coverage we sample
uniformly from a specified range. As figure 5.2 shows, reasonable ranges are between plus-
minus 2 meters in the x-direction, plus-minus 1 meters in the y-direction and anything between
1 to 5 meters for the depth z.
Figure 5.2: Distribution of the centre for each location parameter of the training set.
The first approach, 2D patch, aims to improve the proposals by better estimating the x- and
y-coordinates of objects. Assuming the random ranges cover the image perfectly, the centre is
only allowed to be within the bounding box. We observe that centres are typically clustered
around the bounding box center. We therefore decrease the box by cropping the width and
height by a fourth on each side. The amount of reduction by three-quarters of its original

5.2.4
Evaluation of Proposal Methods
Now that we know how to propose cubes we need to figure out how many proposals it is worth
generating to achieve a satisfactory 3D IoU. We aim to balance a trade-off between performance
and speed, which is dependent on the number of proposals. To get an overview, detection rates
at the levels [0.25, 0.4, 0.6] of 3D IoU are illustrated in figure 5.15. Ideally, we want the graphs
to be as high as possible. In general, when the line tapers off we know that there exist enough
proposals, such that the search space is fully covered. Generating more proposals at this point
only gives a minor increase in performance.
One needs to decide on a desired level of 3D IoU and detection rate. We think achieving a
3D IoU of 0.4 in 75% of instances is advisable. The detection rate in the middle of figure 5.15
reveals that approximately 1000 proposals are needed with the full proposal method. If our
standards were lower and a 3D IoU of 0.25 would have been enough, the detection rates on the
left show us that around 400 proposals are enough, again using the full method. Conversely,
if we desire a higher 3D IoU, for example, 0.6, we run into the problem that the rates have not
tapered off. The fact that the rate does not level out at the high 3D IoU thresholds indicates
that it is worthwhile to propose even more cubes to get a higher detection rate. As such, it is
worthwhile exploring proposing > 10k cubes, however, this many proposals are at the usable
speed-wise limit of the method.
One noteworthy observation from figure 5.15 at thresholds [0.25, 0.4] is that the proposal method
2D patch outperforms 2D patch w/DA at roughly 200 and 700 proposals respectively. This is due
to DA not correctly identifying the depth of some objects, whereas 2D patch searches in a wide
range. This is also why 2D patch does not taper off, as the search range is so large that it still
after many proposals finds better cubes.
Generally, the relations between the proposal functions stay true across the different levels of
3D IoU. We can conclude that the method full performs best, so going forward, we select it and
propose 1000 cubes.

5.3
Scoring Functions
We have established how to create cubes and how many are needed. Currently, it is unclear
how we choose the best one out of all proposals. The idea is to create scoring functions that
resemble 3D IoU, in other words, a higher score should correspond to a higher 3D IoU.
The first function 2D IoU utilises the observation made in [34], see details in section 4.5.1, and
==PAGE:58==
48
Chapter 5. Experiments & Results
computes the IoU between the objectвҖҷs- and the bubeвҖҷs bounding box. A disadvantage is that
this relation from the observation does not hold the other way around. There are many ways
to achieve similar sized bounding boxes, as shown in figure 4.13a. Hence, there is no guarantee
that neither the dimensions nor the rotation are close to being correct. Actually, not even the
depth is necessarily close, as cubes can be predicted much closer or further away and still have
similar 2D bounding boxes.
A method that should mitigate this problem is the one described in section 4.5.2, denoted as
segment. It is based on our observation that the segmentation contour and the bube contour
align. Not only that, it also contains the objectвҖҷs segmentation fully. Furthermore, we observe
that the bube typically has quite an amount of excess area in comparison to the segmentation,
which does not need to be included. This means that it is more important for the bube to contain
the segmentation than the other way around. We, therefore, introduce the function described
in section 4.5.2 denoted mod. segment which values the intersection higher than the union. The
optimal exponent of the fifth power for the intersection was determined empirically through
trial and error. Theoretically, we would expect these scoring functions to capture the rotation
more precisely compared to the 2D IoU scoring function.
Note that to compute the segmentation mask of the bube one needs to compute the convex hull
of the 8 bube corners. We have implemented the Jarvis-March algorithm ВІ in PyTorch to speed
up the process.
We do not modify the IoU calculation as done in mod. segment. We propose the corner dist.
scoring function, which additionally requires finding the smallest enclosing rotated box around
the segmentation. For simplicity reasons, we measure distances between the bube corners and
the box, as described in 4.5.3, instead of calculating the IoU between the bube and this found
box.
The scoring functions so far have only exploited some form of projection. We therefore in-
troduce the dimensions scoring function. We directly compare the cubeвҖҷs dimensions to prior
knowledge, details are found in section 4.5.4. The disadvantage, here, is that it prefers mean-
sized cubes.
Another scoring function using the cube itself is the point cloud scoring function, see section
4.5.5. It places the cube inside the point cloud of the scene and counts how many points are
inside the cube. The aim is to simultaneously capture the depth, dimensions, and rotation of
the object.

much more reasonable. Focusing on the improvement of the proposals is advised. The mod.
segment scoring function is a valuable asset when we have better proposals.
5.4
Loss Functions
We complete our research on proposing and evaluating cubes and continue with learning cubes
by creating a weakly-supervised Cube R-CNN. This consists of replacing the loss functions that
require 3D annotated ground truths with loss functions working on 2D information only. Cube
R-CNN without its 3D components is essentially just Faster R-CNN.
We transfer the knowledge gained by creating the proposal method to create loss functions.
The losses LGIoU, Ldims, Lseg described in sections 4.6.2, 4.6.7, 4.6.6 are the siblings to 2D IoU,
dimensions, and segment scoring functions. In the proposal method, we use the ground normal
to propose cubes, here, we implement it in Lnormal from section 4.6.8 as ground truth rota-
tion.
Additionally, we introduce loss functions based on new ideas exclusive to this method. The
simplest is Lz, see section 4.6.3, which uses DAвҖҷs depth measurement as ground truth. We also
create another loss to determine how far off the depth prediction is, without relying on DA.
This loss is Lproj from section 4.6.4 and uses the fact that bubes of the same object at different
depths have different sizes. A loss function helping to find both dimensions and rotation is
Lrange. It makes use of SAM and DA models to find pseudo ground truth and compares it to
the difference in maximal and minimal depth of the cubesвҖҷ corner points. Details are described
in section 4.6.5.
We conduct an ablation study on a reduced dataset of 400 train and 400 test images to find the
best combination of loss functions. We had to reduce the dataset size for this experiment, as
we found the weak 3D loss functions to train at snail-pace. To further speed up the training,
we pre-train a 2D predictor on the full dataset until it is near convergence (58k iterations). The
2D predictor is shared as a starting point for all the experiments. Then, we activate the 3D
losses and train it further until the total 3D loss converges. We found this method worked
better than training the model from scratch because the 3D losses tend to be unstable when
the 2D network makes poor predictions. This leads to exploding gradients in many cases. We
train with a learning rate of 10вҲ’3 and use a learning rate decay of 10x once reaching a stable
convergence, see the loss curves in section A.4. The lr decay results in a jump in accuracy.

Chapter 6. Discussion
The experiments are conducted, we aim to reflect on the performance, highlight the defining
characteristics of the models and see how the models compare. Additionally, we explore the
limitations encountered during the study and suggest avenues for future work that could build
upon the foundations laid by this research.
6.1
Performance of External Models
Before going into detail with each method we want to emphasise the consequence of incorpor-
ating other models and techniques such as Depth Anything, Grounding DINO, and Segment
Anything. While they add value by providing or gaining additional information, our methods
also inherit all their errors. None of these methods work perfectly, even though we design our
methods with the assumption that they do. Our modelsвҖҷ performance is dependent on their
performance. In cases where these models return poor output, our methods fail catastrophic-
ally. These kinds of models can never become perfect, but they are actively being improved.
One could argue that the time-equalised comparison is unfair because our methods incorporate
these large models which have undoubtedly been trained on hundreds of millions of images.
The time used to annotate these large data sets should be added to the annotation time of our
models. However, we would argue that it is a pointless argument since these models would
have existed regardless of whether we made our models.
6.2
Discussion of Models
We found that the optimal way to propose cubes is to utilise the camera intrinsics and an objectвҖҷs
bounding box to determine the centre, priors for the dimensions and the ground normal for
rotation. Our design uses grids and search ranges, which effects are addressed now. Figure 5.16
shows that it is virtually impossible to create cubes with a 3D IoU greater than 0.8, which is
odd, considering the ground truth is within the search area over 90% of the time. This stresses
that grids in the search range combined with too large search ranges are major drawbacks of
this proposal method. A general observation made for parameters of the proposal method
was that narrowing down the range in which we search for the parameter always resulted in
less overall accuracy, but better good cubes. Considering rotation, we found that the 3D IoU
can be up to 0.031 lower when the ground truth is exactly in between two grid points. This is
acceptable. What is not acceptable is that just 2/36 (5.56%) of proposals are close (one grid step
==PAGE:76==
66
Chapter 6. Discussion
away) to the ground truth regarding the rotation. Conversely, 34/36 of proposals never get the
chance to be truly close to what they should be. And that is for one rotation axis only. This
also explains why more proposals yield better results, which we can see by the fact that it still
makes sense to propose more than 1000 cubes, see figure 5.15.
An advantage in the design of the proposal method lies in the mixture of instance in- and
dependent approaches. While the centre is heavily dependent on the instancesвҖҷ bounding box,
the dimension predictions are completely independent. And then there is the rotation, which
stands in between dependency-wise as it is image-dependent.
The method predicts fantastic boxes when the ground is identified correctly. If you were to use
this method in the real world, a possible way to use it could be to disallow inputs where the
ground is not visible, based on the output of Grounding DINO. We mostly decided to use the
plane RANSAC method, because we saw many of the methods in the related work for outdoor
driving scenarios use this approach. However, they used it in the driving scenario where the
ground is always visible, and as we found, it does not generalise perfectly in cases where the
ground is not visible.
The ability to score cubes proved to be a difficult task. The chosen cubes have only half the 3D
IoU of what is possible. Our focus lied on a balanced development between the proposal and
scoring functions. It is possible that focusing on one of them would have resulted in overall
better results. If we were able to propose cubes with an average best overlap of only 0.4 but
were excellent at evaluating them, it could yield better results. It can also work the other way
around. Due to this uncertainty, we went with a balanced approach. We base this discussion
on the difference in the performance of the average best overlap with and without the ground
truth, figure 5.18, and 5.17.
Priors are a strength when proposing cubes but become a disadvantage when scoring them.
The difficulty lies in evaluating a three-dimensional object with certain ground truths only in
2D. The only three-dimensional knowledge accessible to us is dimension priors and these are
instance independent. The importance of instance-dependent capabilities becomes clear when
looking at the ABO plot where the ground truth is included. Then, the dimensions scoring
function is the worst of our scoring functions.
The fact that the best way to evaluate cubes is an entanglement of three functions focusing on
bube size, object dimensions and rotation, makes the combined scoring function robust. Un-
reasonable cubes are excluded from being selected, and visually, the bubes often look okay.
The combined scoring function is robust because it has elements that are related to each of the
3 properties defining a cube. This result is appealing since it is the same 3 components that we
focused on when proposing a cube. Furthermore, these components are independent from the
location of objects images.
We have designed 8 weak loss functions, of which LGIoU, Lz, Ldim, Lrange, Lpose and Lnormal are
integrated into weak Cube R-CNN. Some of the loss functions have disadvantages, while some
have advantages. One of the challenges the dimension loss function faces is that the ground
truth does not have the lowest loss, more precisely, it can be non-zero and hence not a вҖңtrueвҖқ

loss function. That is why we built in the relaxation that it should only have an effect when the
predictions are more than one standard deviation from the mean dimension. A disadvantage
of the alignment loss is that it works on all the rotation axes, this is not necessarily desired
because some objects in a scene can have different yaw. To mitigate these issues, we employ
multiple loss functions. However, the compounding effect of these combined losses is difficult
to quantify, presenting an additional layer of complexity to our approach. Analysing the loss
curves in appendix A.4 we notice a strange effect. Many loss curves increase, even though
when run as simple experiments one by one the loss functions all decreased. This could be an
indication that the losses are fighting each other. An example that supports this hypothesis
is the already discussed image in figure 5.19f, where a correct fit to the chair to the ground
increases the segmentation loss. When this generally is the case, the segmentation loss can
therefore not decrease in the combined loss scenario. That individual losses increase does not
impact the correlation of higher accuracy and lower total loss.
Sometimes, specific loss functions fail, creating challenges in determining the best course of
action. SAM can fail to return a result, the predicted 2D center might fall outside the image
frame, thus outside the depth map, or the normal vector might be undetectable. We have
chosen to coerce these cases to work, or in the worst case scenario for the segmentation loss,
ignore a specific image completely. This approach ensures that the model continues to learn the
correct objective, despite introducing some noise into the loss function. Although we believe
this method does not introduce unintended side-effects, such as to negatively affect predictive
accuracy for images without a ground, we have not verified this assumption.
Among the list of advantages of weak Cube R-CNN is that it is much more consistent with its
predictions, especially compared to the proposal method. Additionally, it is also much faster
in inference. Furthermore, we believe weak Cube R-CNN has the most unrealised potential for
further experimentation. Currently, it does not scale well with the amount of data, this could
be remedied. We also base this opinion on the observation that the proposal method follows an
older paradigm in object detection, which has been superseded by end-to-end deep learning
methods. The amount of work put into the proposal method in this project has also been at
least twice the work put into weak Cube R-CNN.
A limitation of all of our proposed methods is that they cannot learn to predict anything outside
the image frame.This is supposed to be one of the areas where 3D object detection improves
upon 2D object detection. However, since our methods only use the 2D ground truths they
cannot overcome this limitation, because the 2D ground truths do not extend beyond the image
frame.

time-equalised, fully-supervised Cube R-CNN. This highlights the fact that deep neural net-

results. It also serves as a good starting point for other more advanced methods, such as the
pseudo ground truth method which performs best. Weak Cube R-CNN is not able to beat that
but still better than the proposal method. All our methods have difficulties predicting excellent
cubes, i.e. cubes with a 3D IoU over 0.5, as they are by far the worst in this category (seen by

6.3
Limitations
During the research, several limitations were identified. In the beginning, the lack of prior
research studies on this topic hindered our ability to build a robust theoretical framework and
necessitated reliance on ideas from 2D methods. Our development of the methods was further
constrained by the little and often poorly annotated data available. This small sample size
introduces significant bias in our methods, just as external models also introduce bias. The
main obstacle however is the speed at which our methods run, which reflects the weak nature
of the optimisation objective given by the weak ground truths. For example, each of the loss
ablations took around 12 hours to conduct, slowing development and limiting the number of
tests we could perform.
6.4
Further Work & Unaddressed Avenues
Throughout this project, we learned that we must think differently from normal deep learning
to make weakly supervised learning work efficiently. Weak supervision adds a layer of com-
plexity due to everything being a proxy to the real function. We think it was the right approach,
to begin with the proposal method as we gained lots of knowledge on the domain. We have
tried to develop 8 different loss functions, while papers usually focus on a single loss function.
It would have been better to focus on a few and improve them, even though that does not show
our abilities to come up with new ideas. Ideas for further work could be a direct 3D RPN,
bypassing the 2D RPN.
We were successful at investigating all the methods and techniques we planned. One of the
things we looked at, was proposing fewer cubes and then taking a certain amount of the best
cubes and creating variations of them. We found that a higher maximal 3D IoU can be achieved,
but our scoring functions were not able to differentiate between such small variations. Still, it
would be recommended to look into this possibility. On the topic of proposals, one could
adapt the idea of Lrange for predicting dimensions. Furthermore, we think that occlusion can
be utilised for scoring. Also, the generalisability to other data sets can be investigated. A larger
topic to look at would be to reduce the level of supervision further or to look at combinations
of different levels of weak supervision.
==PAGE:79==
Chapter 7. Conclusion
We set to out explore possibilities on how to design weakly-supervised methods that tackle the
task of 3D object detection. We managed successfully to create three methods with different
approaches. We have a proposal method inspired by the general ideas from Selective Search
and Edge Boxes, using 2D bounding boxes to generate 1000 candidate cubes. Then, a combin-
ation of 2D IoU, Dimensions and Corner scoring functions evaluates all candidates and chooses
the one it values highest.
Another method, pseudo ground truth, uses these predictions in a fully-supervised manner in
the Cube R-CNN framework. This method is an improvement to the proposal method and led
us to compose a set of weak loss functions which Cube R-CNN uses. The weak Cube R-CNN
does not live up to its promising potential.
Addressing the challenge of scale ambiguity in monocular 3D object detection, we found this
to be tightly intertwined with the ability to accurately estimate depth. Originally, we thought
that a depth estimation method or other external models could be an extension to improve
accuracy. In reality, we found it to be a necessity.
Measured by accuracy, the proposed methods beat the benchmark Cube R-CNN method in
a simulated scenario where the annotation is time equalised. Unfortunately, their accuracy is
only a bit more than half of the fully 3D supervised Cube R-CNNвҖҷs accuracy. This ratio is in
line with weakly-supervised methods within 2D object detection.
Rounding off, we have developed several weakly-supervised methods with 2D labels only and
an acceptable performance getting closer to achieving performance similar to fully-supervised
models.

A.6
RANSAC algorithm for finding planes in point clouds
This presents a generic algorithm for finding a plane in a point cloud. We use this to find
the ground in the point clouds. The function has one input: the points, and 2 parameters:
threshold, and maximum number of iterations. points are the points in the point cloud, given
as an N Г—3 array. threshold is the maximum distance at which we consider points to be inliers,
in metres. max iterations is the number of iterations to run the algorithm for. Note that we
do not have any logic to stop the algorithm before the max iteration count has been reached.
However, a way of doing this could for example to stop the algorithm if the number of inliers
> 1/4 total number of points.
To further speed up the algorithm, we sub-sample the point cloud to only use every 5th point
because we found that it would still provide the same result, just faster. Additionally, we run
all iterations in parallel on a GPU.
