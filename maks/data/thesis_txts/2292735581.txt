Summary
The ﬁrst part of the thesis is dedicated to the AKS primality test, the world ﬁrst
deterministic polynomial time primality test. We look into the theory behind
it and how it works. We implement it and verify that it runs in polynomial
time.
The second part of the thesis considers a probabilistic primality test
called the Miller-Rabin test. In particular we investigate how it can be made
into a deterministic algorithm that proves primality for all integers less than
some limit. We implement an algorithm that ﬁnds optimal bases for a such
a deterministic variant of Miller-Rabin.
The runtime of the implementation
is long and we do not succeed in improving known results on best bases for
Miller-Rabin.

Chapter 1
Primality Testing
A natural number greater than one is called a prime number if it has only two
positive divisors, namely itself and the number 1. A number that is not prime is
called composite. The fundamental theorem of arithmetic says that every integer
n ≥1 can be uniquely written as the product of distinct primes (if the order of
factors is disregarded) [Die04]. But how do we determine whether a number n
is prime or composite? The intuitive way is simply testing all numbers less than
n and see if any of them divide n. We note that if n has factor p ≥√n then it
must also have a factor q ≤√n. Therefore it is enough to check numbers up to
√n and see if they are a factor of n. This method is called trial division. The
problem with this simple approach is that it is very slow for large numbers. It is
O(√n), meaning that the number of steps required for trial division is bounded
above by c√n for some positive constant c [Kob94].
We would like to have an algorithm that determines primality of a number n
in polynomial time for the number of bits required to represent n (or polyloga-
rithmic time for n). This means that we want an algorithm A that can answer
deﬁnitly whether a number n is prime or not, where the number of operations
needed for the computations is bounded by c(log2 n)k for some constants c and
k. So we would like A to be O(logk
2 n), for some constant k.
Primality tests that do not depend on factorization (such as the trial division
does) have been known for a long time. They are based on some criteria that
==PAGE:10==

all prime numbers satisfy but do not necessarily contribute any knowledge to
the factorization of the number. One such criterion is Fermat’s Little Theorem
which we will discuss later. Since the 1970’s probabilistic polynomial time pri-
mality tests have been known. These kind of tests do not prove that a number
is prime but can declare a number to be prime with some small margin of error.
Two prominent examples are the Miller-Rabin test and Solovay-Strassen test
[Kob94]. Both of these tests include some randomness in the algorithm and
hence are not deterministic. It was not until 2002 that the ﬁrst deterministic
primality test that can be proved to run in polynomial time was published. It
is called AKS after its three authors, Agrawal, Kayal and Saxena [AKS04].
In this paper we will cover to some extent two topics in the vast ﬁeld of prime
number studies. In the next chapter we will discuss the AKS primality test in
detail and give a proof for its correctness. This chapter is a quite theoretical
presentation of AKS and how it works as a primality test. We will not go into
the complexity of the algorithm but there is plenty of literature on that subject,
for example [Die04], [Gra05] or the orignal paper [AKS04]. After the theory we
will discuss an implementation of the algorithm. In chapter 3 we will change
direction and consider the Miller-Rabin test. We will show how it can be made
into a deterministic test that proves primality for any number under a certain
limit. In this part we put more emphasis on the implementation rather than
theory. But before these topics are discussed we give a (very) brief overview of
the study of primes and then we present one of the most fundamental properties
that all prime numbers satisfy primes, namely Fermat’s Little Theorem.
1.1
Prime numbers
The earliest studies of prime numbers date back at least to the ancient Greek and
possibly even back to the ancient Egyptians. Euclid’s Elements (from around
300 BC) includes propositions of there being inﬁnitely many primes and that any
composite number can be factored into primes [Bal10]. After the ancient Greeks
there was not so much progress in the ﬁeld. Fermat achieved some interesting
results but it can be said that modern number theory started with Euler in the
18th century. Since that time prime numbers have captured the interest of some
of the worlds most brilliant minds, including Gauss and Legendre.
But are prime numbers of any interest other than academical? Until the 20th
century it seemed as the study of prime numbers was only for the pleasure
and interest of the researcher.
There is an interesting quote by the English
mathematician G.H. Hardy [Har40]:

...both Gauss and lesser mathematicians may be justiﬁed in

This remoteness of number theory from what Hardy calls ordinary human ac-
tivites is far from todays reality. In our normal daily lives nowadays we use
encrypted communication at countless occasions, whether it is surﬁng the web,
sending a text or using our credit card at the store. Already in the 1940’s so-
phisticated cryptographic machinery (such as the German Enigma) were seeing
the light of day. With the introduction of the computer, the science of cryptog-
raphy exploded in the second half of the 20th century, with results from number
theory setting the stage and prime numbers playing key roles in many protocols.
One of the ﬁrst public key cryptosystems to be published was the RSA in 1977
and it is still one of the most widely used protocols today. It uses the problem
of factoring and thus has prime numbers at its core. To set it up (securely) one
needs a key of a few thousand bits [BR11] and in order to generate this key
two large prime numbers satisfying certain conditions are required. So prime
number generation is integral for the RSA and in order to generate these primes
some sort of primality testing is needed.

Chapter 2
The AKS algorithm
For decades researchers had been trying to come up with a provable deterministic
polynomial time primality test but without success. In 2002 the AKS algorithm
was published and it came as a pleasant surprise to the scientiﬁc community.
This is because the general idea behind the algorithm is not so very complicated
and one does not need to use deep theorems from number theory to proof that it
is correct and runs in polynomial time for all inputs. In fact, the algorithm is the
result of two bachelor’s theses at the Indian Insitute of Technology in Kanpur,
India (IIT). The latter one was completed in 2002 by Kayal and Saxena. Agrawal
is a faculty member in the Computer Science Department at IIT [Gra05].

Roots of Unity and Cyclotomic Polynomials
We use cyclotomic polynomials and primitive roots of unity in the correctness
proof for AKS so we will give a brief discussion of these concepts here. For a
more thorough discussion and background we refer to [LN86].
Let K be a ﬁeld, the splitting ﬁeld of a polynomial f ∈K[X] over K is the
smallest ﬁeld F that contains K as a subﬁeld and such that f can be written as
a product of linear factors X −α ∈F[X]. Let n ∈N, then the splitting ﬁeld of
Xn −1 over K is called the n-th cyclotomic ﬁeld (over K), denoted K(n). The
roots of the polynomial Xn −1 in K(n) are the elements α ∈K(n) such that
αn −1 = 0 in K(n). These roots are called the n-th roots of unity and the set
of all n-th roots of unity is denoted by E(n).
Now, let K be of characteristic p where p is a prime and n ∈N not divisible
by p. Then we say a n-th root of unity is a primitive n-th root of unity over
K, if it is a generator for the cyclic subgroup E(n). The polynomial Qn whose
roots are precisely the primitive n-th roots of unity is called the n-th cyclotomic
polynomial over K. Furthermore, if ζ is a primitive n-th root of unity then ζ
can not be a root of unity for an integer m < n.
It can be shown that E(n) is indeed cyclic for a ﬁeld of prime characteristic.

Chapter 3
The Miller-Rabin primality
test
One of the most widely used primality testing algorithm is called the Miller-
Rabin primality test. For example, the FIPS standard (developed by the US
government) uses the Miller-Rabin primality test in its standard on prime num-
ber generation [KG13]. The test was proposed by Rabin in a paper dating back
to 1977 [Rab80] and is based on an idea that Miller used a few years earlier
for a primality test. Miller’s test is deterministic and can be proven to run in
polynomial time if the Extended Riemann Hypothesis holds true [Mil76]. Rabin
used the basic idea of Miller’s test to make a fast probabilistic primality test.
3.1
The algorithm
We will present the test in a slightly diﬀerent way from the original paper by
Rabin. The criterion used in the test stems from Fermat’s Little Theorem 1.1
and a fact about the square roots of 1 modulo a prime, namely
Theorem 3.1 If p is a prime and 1 ≤a < p and a2 = 1 mod p, then either
a = 1 or a = p −1.
==PAGE:30==
22
The Miller-Rabin primality test
Proof. Since p is a prime, we know that Zp is a ﬁeld and hence every nonzero
element has a multiplicative inverse. From this fact we can conclude that

So, for a prime p there are two square roots of 1 modulo p, they are 1 and −1
mod p. If n is any positive number and we ﬁnd a square root of 1 mod n that
is not 1 or −1 mod p, then certainly n is composite. The square roots of 1
modulo n that are diﬀerent from 1 and −1 mod p are called nontrivial.
Let us consider again an−1 mod n. If n > 2 is even, then it is clearly composite
so we assume that n is odd. Then we can write n −1 = 2ku, where u is an odd
number and k ≥1. Then it is possible to compute an−1 mod n in k + 1 steps,

then ai−1 is a square root of 1 and if this square root is nontrivial we conclude

(i) a0 = 1 mod n, or

We will refer to these conditions as the Miller-Rabin criteria or MR criteria.
Checking the MR criteria for a number of randomly chosen a ∈{2, 3, . . . , n−2}
is the idea behind the Miller-Rabin algorithm (Miller checked every a up to a
certain limit). If n does not satisfy the MR criteria for some a we say that a
is a witness to the compositeness of n. If n does satisfy the MR-criteria for
a particular value of a, we say that n is a strong probable prime to base a.
A composite number n that is a strong probable prime to a is called a strong
pseudoprime to base a, denoted by spsp(a). We say that n is spsp(b1, . . . , bt) if
n is a strong pseudorpime to bi for i = 1, . . . , t. There are many other diﬀerent
types of pseudoprimes. In a similar manner we call a composite number n a
(Fermat) pseudoprime if it passes the Fermat criterion for a base a and we
denote that with psp(a).

The algorithm
The original description of the algorithm is in Jaeschke’s paper [Jae93]. What
follows is a slightly altered version of that. The biggest change is that we do
not require the base elements to be consecutive primes. Let ν = (a1, . . . , aw)
denote the base elements in ascending order. The algorithm ﬁnds all strong
pseudoprimes less than or equal to the limit g to the base ν that have t diﬀerent
prime factors. There are two cases, if t > 2 and if t = 2.
Case 1. Assume that t > 2. We start by determining (t−1)-tuples (p1, . . . , pt−1),
with p1, . . . , pt−1 prime numbers in ascending order such that

We call such tuples feasible (t−1)-tuples. For each feasible (t−1)-tuple we choose
one of the base elements as b and compute η = lcm(op1(b), . . . , opt−1(b)). Now,
if gcd (η, p1 . . . pt−1) ̸= 1, then the tuple (p1, . . . , pt−1) is discarded. Otherwise,
we compute the multiplicative inverse c of p1 . . . pt−1 mod η. Then for each
prime y ≤g/(p1 . . . pt−1) with y = c mod η we test whether y · p1 . . . pt−1 is a
spsp(ν).

product we check if P is spsp(ν).

The implementation
Based on the description above, we write a program to ﬁnd the optimal base
element combination for a given number of base elements that are less than a
given limit. We note that in our implementation we restrict ourselves to prime
base elements but that is not necessary and some of the records on [Izy] have
composite base elements. By restricting to primes we do not actually perform
an exhaustive search for an optimal base but this allows us to set the search
limit higher. For a limit g, we have to run the algorithm for every t ≤log2 g
number of prime factors. We test the implementation by the same means as the
brute force in addition to making sure that our implementations are consistent
with each other.
When doing proﬁling on the code we note that a lot of time goes to primality
testing and originally this was done by doing a Miller-Rabin test. To decrease
the runtime of the program we create a ﬁle that contains information about
the primality of every number in the interval [0, 232 −1]. When running the
program, it reads the ﬁle into a bitset, eﬃciently reducing primality testing to
a lookup in a bitset. We chose to use bitset in order to use the least amount of
memory to store the list. The original prime list is generated by using the Sieve
of Eratosthenes [Rib04]. This resulted in a runtime decrease of a factor 10 for
the program.
Another thing we do to improve performance has to do with the fact that
Jaeschke’s algorithm ﬁnds all strong pseudoprimes less than some given limit.
This limit has to be carefully chosen. If we have a base given and we want to
use Jaeschke’s algorithm and if we choose the limit too large we will do unneces-
sary calculations and if it is too small we may not ﬁnd any strong pseudoprime.
Instead of having a constant limit for the search space we implement it to be dy-
namically increasing, meaning that the algorithm ﬁnds all strong pseudoprimes
within an interval. If non is found the search interval is shifted up and increased
by a certain factor L. We do the dynamic increase since there is some overhead
involved in shifting the search space, so we do not want to do "too many" shifts
in some sense.

4.1
Other primality tests
For cryptographical purposes there are standards that one can consult in order
to attain preferable security. The aforementioned FIPS standard gives, for ex-
ample, recommendations on how many rounds of Miller-Rabin are required for
diﬀerent purposes when using RSA [KG13]. It is also possible to combine two
diﬀerent tests to make a stronger one. The Baillie-PSW is a prominent example
of that. One variant is like this. It uses one round of Miller-Rabin with base 2
and if it passes that it uses a so called Lucas test. The Lucas test uses a diﬀerent
primality condition than Miller-Rabin. When using MR it turns out that if a
number is spsp(a) then the odds that it is also spsp(b) for a ̸= b increase. The
gain in using the Lucas test instead of multiple MR is that it seems the Lucas
pseudoprimes are diﬀerent kind of numbers to strong pseudoprimes (they seem
to fall into diﬀerent residue classes). So we don’t have the same kind of corre-
lation between Lucas and MR as we have between multiple MR tests [PSW80].
It turns out that no number of less than 64 bits passes the Baillie-PSW test
described above. There are probably inﬁnitely many composite numbers that
pass the test but non has been found. Some evidence even suggests that no
composite number of less than 10000 digits can pass the test [Wei].
The biggest drawback with MR is that it is not an actual primality test as it
can only prove compositeness and not primality. But there are other algorithms
that can prove primality. A lot was happening in this ﬁeld in the last quarter of
the 20th century. The ﬁrst unconditional primality proving algorithm, that was
not based on factoring, was published in the late 70’s by Adleman, Pomerance
and Rumley [APR83]. Unconditional here refers to the fact that it does not rely
on any unproven hypothesis such Miller’s test does with the Extended Riemann
Hypothesis. The test was simpliﬁed and made more practical by Lenstra and
Cohen and is often called the APR-CL test. It uses arithmetic in cyclotomic
ﬁelds and has a subexponential runtime (it can be said to be "nearly" polyno-
mial) as it will terminate within (log2 n)c log2 log2 log2 n for some constant c. In the
80’s elliptic curves started to become more apparent in the ﬁeld of cryptology
and one of the most powerful tests that is used in practice to prove primality is
based on elliptic curves. It was proposed by Atkin in the late 80’s and is referred
to as ECPP (elliptic curve primality proving) [AM93]. It is hard to determine
the runtime of the algorithm in a rigorous way but some arguments can be made
about the expected runtime and it agrees with implementations [Sch08]. One
of the big advantages of this test is that it can produce a certiﬁcate of primality
which can be easily veriﬁed. So once the certiﬁcate has been generated the cost
of verifying it is much lower.
