Abstract
Recent advances in computer vision research have made it possible to produce photoВӯ
realistic 3D volumetric reconstructions of complex scenes based on datasets of 2D imВӯ
ages Вӯ this includes the ability to manipulate specific objects in these scenes. Significant
progress has also been made in the field of visionВӯlanguage models, which have effecВӯ
tively been able to capture many different semantic relationships between text captions
and image data, and have paved the way for textВӯbased editing frameworks. This project
seeks to bridge the gap between these two areas of research, and proposes a novel
method which can perform manipulations of objects in photoВӯrealistic 3D scenes that are
semantically consistent with a given text prompt.
The code for the project is available at github.com/lucasalexsorensen/sgvop

Introduction
1.1
Background and motivation
In recent years, there has been some remarkable advances in the field of differentiable volВӯ
umetric rendering. This has made it possible to faithfully reconstruct complex 3D scenes
in a photoВӯrealistic manner. Specifically, a lot of work has been made based on Neural
Radiance Fields[1] (also called NeRFs). As such, many extensions of NeRFs have been
made already. Quite a few of these extensions revolve around improving the speed and
efficiency of NeRFs[2], but there are also extensions that deal with NeRF manipulation.
In [3], they present a novel method for disentangling foreground objects from their backВӯ
ground scenes, which allows for fineВӯgrained control of how the 3D scenes are composed.
Meanwhile, in a different area of research, there has also been made remarkable strides.
Namely, visionВӯlanguage models have seen significant improvements with the advent of
CLIP[4]. The concept of building models that connect textВӯimage pairs has proven to be
quite effective, and the models have been performing especially well on zeroВӯshot image
classification tasks, and play a significant role in modern textВӯconditional image generation
models (e.g. [5]).
The main idea of this project is to combine the aforementioned NeRF manipulation techВӯ
niques with CLIP models in order to manipulate objects in 3D volumetric scenes using
semantic prompts Вӯ to this end, the concept of semantically guided volumetric object placeВӯ
ment is established. Given a guiding text prompt, the framework should determine how to
perform the most appropriate NeRF manipulation. A concrete example could be to guide
a NeRF manipulation to properly relocate a disentangled coffee cup given the text prompt
вҖқa mug on a tableвҖқ. This is a powerful concept which could be essential for many virtual
reality applications.
1.2
Related work
ControlВӯNeRF[6]. By maintaining a discrete feature volume grid, this method is capable
of editing and mixing NeRF scenes. Concretely, this is done by manipulating the feaВӯ

semantically guided manipulations are not supported.
CLIPВӯNeRF[7].
This method works by using latent codes to independently represent

The technique is incapable of simply transforming objects in the context of scenes (e.g.
moving and rotating the object). Also, the paper only presents results from synthetic

guidance in order to manipulate NeRF scenes. The rendering loss now incorporates the
an вҖқobjectnessвҖқ probability for the points in 3D space, which guides the object disentangleВӯ
ment. The objectness probabilities are determined using the aforementioned pixelВӯlevel
annotations. Additionally, a CLIP loss term is added to assist in filling occluded parts of
the scene after extraction using a provided textual prompt. The method mostly works for
extracting objects, and does not appear to extend to object manipulation (i.e. scaling,
rotating, translating).

2.2
Semantic guidance
2.2.1
CLIP
Recent advances have been made in the field of visionВӯlanguage modeling Вӯ a very notable
one being OpenAIвҖҷs CLIP[4].
At its core, it contains an image encoder and a text encoder. The image encoder is based
on either a ResNet[11] or a Vision Transformer[12] architecture. The text encoder is based
on a Transformer[10] architecture. The encoders are configured to produce vectors of the
same dimensionality.
A fundamental concept in CLIP is the contrastive preВӯtraining process. Given a dataset of
imageВӯtext pairs, the encoders are jointly optimized such that they produce vectors that
align for the matching imageВӯtext pairs, and donвҖҷt align for the nonВӯmatching imageВӯtext
pairs. In other words, it seeks to maximize the cosine similarity of the embeddings for
matching pairs.
12
Semantically Guided Volumetric Object Placement
==PAGE:18==
Figure 2.13: Taken from [4]. CLIP contrastive preВӯtraining
The contrastive preВӯtraining is performed in batches of size N. As such, there are N2
pairs to consider for every batch Вӯ of which only N pairs are matches. This is illustrated
in figure 2.13, with the diagonal of the matrix indicating the matching pairs for which the
similarity should be maximized.
A benefit of preВӯtraining models in this contrastive fashion is the fact that no dataset laВӯ
bels are required. This makes it easier to gather huge datasets by scraping the internet,
where a lot of textВӯimage pairs can be found naturally (e.g. Instagram, reddit, etc.). HowВӯ
ever, despite being trained on a massive dataset, there might still be вҖқholesвҖқ in CLIPвҖҷs
representation space. For instance, it might not appropriately embed an image due to
lighting conditions, image noise, and other visual artifacts. As such, it might be important
to heavily augment or somehow regularize the signals obtained from CLIP.
After preВӯtraining, CLIP is capable of embedding instances of different modalities (i.e. text
and images) into the same space. This has led to a lot of additional research involving its
capabilities as a zeroВӯshot image classifier (zeroВӯshot meaning вҖқnot trained for the specific
taskвҖқ).

2.2.2
Shattered gradients
The shattered gradients problem occurs when working with deep neural network archiВӯ
tectures (such as the ones used in CLIP). The work done in [13] shows that the corВӯ
relation between gradients in standard feedforward networks decays exponentially with
network depth, which ultimately results in gradients that resemble white noise. On the
other hand, the gradients in networks that implement skipВӯconnections decay sublinearly,
which makes them more resilient to gradient shattering.
In [13], an example neural network can be constructed. It maps from R вҶ’R (i.e. scalars
to scalar), and contains N = 200 rectifier units per hidden layer. The gradient with respect
to the input xi is computed for 256 values of xi вҲҲ[вҲ’2, 2]. This has been done for multiple
network architectures (including ResNet, which has skipВӯconnections). The results can
be seen in figure 2.15.

Instead of tying everything together at once, the project has been divided into five increВӯ

1. 2D optimization experiments

4
Conclusion
4.1
Main conclusions
In summary, the results obtained in the previous subchapters indicate that it is possible to
perform semantically guided volumetric object manipulations using the chosen methodВӯ
ologies.
The experiments with the synthetic renderers (both 2D and 3D) demonstrate successes
and failures when optimizing a differentiable renderer with respect to CLIP loss. If the
optimization is only for a single parameter, there are viable alternatives to gradient deВӯ
scent. However, the benefit of using gradient descent becomes more pronounced when
optimizing for multiple parameters Вӯ especially since the alternatives begin to perform proВӯ
gressively much worse as the parameter count increases. Many attempts were made to
improve the stability of gradient descent, e.g. by augmenting the text prompts and renВӯ
dered images. This had a measurable, but not entirely significant impact, and as such,
the most deciding factor for the success of gradient descent remains to be the learning
rate and starting point.
The NeRF manipulations appear to work to a certain degree, although the extent of how
freely the object can be manipulated is limited by the bounds of the NeRF. As such, the
manipulations will likely work better for NeRFs of a more unbounded nature. Also, there
seems to be a few issues with the implementation Вӯ e.g. the вҖқtranslucentвҖқ reВӯinserted flowВӯ
ers and the skewed zВӯrotation. However, these issues seem to be mostly codeВӯrelated,
since the reВӯinsertion technique has been proven to work in [3].

4.2.1
Text prompt augmentation
A lot of work is currently being made in order to вҖқunlockвҖқ the full potential of CLIP. In that
regard, many are exploring the idea of prompt engineering, which essentially revolves
around rephrasing the text prompt to elicit different results from the model. Prompt engiВӯ
neering was explored a bit in 3.2, but it probably makes sense to perform these experiВӯ
ments in a more structured manner.
On CLIPвҖҷs official repository1, there is a notebook demonstrating how using various forВӯ
mulations involving the class name can improve the zeroВӯshot classification capabilities of
CLIP on ImageNet. For example, the formulations could be: вҖқa bad photo of a .вҖқ, вҖқa carВӯ
toon .вҖқ, вҖқa jpeg corrupted photo of the .вҖқ. These various formulations try to actuate some
of the context regarding the text captions for the images in the dataset. A lot of these
images are scraped from the internet, which is why they experiment with many prompts
related to this origin (e.g. вҖқa low resolution photo of .вҖқ, вҖқitap of .вҖқ (itap is a subreddit),
etc.). After compiling 80 of such templates, they proceeded to perform a sequential forВӯ
ward selection to determine which prompts performed the best on ImageNet. The search
terminated after ensembling 7 templates. These templates are tried out for the 3D cow
rendering example, producing the results seen in figure 4.1.

4.2.3
Hybrid optimization
Even after many tricks, gradient descent is still rather imperfect, and the quality of the
obtained solutions still depends a lot on the configuration (e.g. learning rate, schedule for
learning rate, starting points, augmentation hyperparameters, etc.). It might make sense
to draw inspiration from some of the gradientВӯfree global optimizers that were experiВӯ
mented with in section 3.1. Maybe trying an ensemble of configurations and performing
gradient descent multiple times could lead to finding better solutions.
4.2.4
NeRF improvements
There is definitely some room for improvement in regard to the torchВӯngp NeRF results.
It could be worthwhile to investigate why the disentanglement did not work as expected.
It might be important to uncover if the issue is codeВӯrelated, or if disentanglement is funВӯ
damentally incompatible with that type of NeRF.
As for the flower NeRF results, there seemed to be some visual artifacts upon reВӯinserting
the disentangled foreground object into the scene Вӯ even without performing any transforВӯ
mations. ItвҖҷs quite likely that the issue is codeВӯrelated, since the methodology has been
proven to work in [3]. In my opinion, it definitely makes a lot of sense to figure out the
reason behind this issue.
Also, given more time, it would have been very insightful to apply the disentanglement
and manipulation to several other NeRF datasets. Currently, it cannot be entirely ruled
out that the types of observed visual artifacts (e.g. the mosaic artifact) are specific to just
the flower scene. Finding a dataset which is less bounded allow for greater manipulations
in 3D space (e.g. rotate more freely, etc.).
4.2.5
ObjectВӯcentric transformations
The NeRF transformations (specifically rotations) experimented with were all centered
about the origin (i.e. [0,0,0]). If the goal is to rotate the object around itself, it requires
that the 3D position of the foreground object be known (or at least estimated). The idea
was to research how a вҖқcentreВӯofВӯmassвҖқ method (such as the one proposed in the end
of section 2.1.3) could be implemented. However, this was deemed outВӯofВӯscope since
there were issues with the regular NeRF transformations (i.e. visual artifacts appearing).
In the future, this could be an interesting direction to take, since it allows for more granular
manipulation of the object in the scene.
4.2.6
Complete entire pipeline
The main idea of this project was to combine CLIPВӯguided optimization and NeRF maВӯ
nipulations into a single pipeline. In practice, combining it all requires introduces a wide
range of technical challenges in regard to code and hardware (specifically VRAM), which
are beyond the scope of the project. Also, there are still existing issues which need to be
resolved first (e.g. visual artifacts from transformations).
