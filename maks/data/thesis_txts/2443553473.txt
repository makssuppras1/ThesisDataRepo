Abstract
This thesis describes the development of a device capable of monitoring a set of
paralleled AC inverters. The developed device is capable of monitoring real-time power
parameters of the parallel inverter system, as well as providing alarm functionality for a
set of predeﬁned parameters.
The ﬁrst two chapters is a general introduction, followed by a structured overview of
the requirements for the developed solution.
The third chapter describes the result of operating parallel inverters without proper
synchronization. It is shown through simulation how even small synchronization differ-
ences between parallel inverters will cause large differences in the load-sharing and power
generation for various load-types. This chapter also presents data from a real parallel
inverter setup, as well as from a test-bench setup.
The fourth chapter describes the considerations during the design-process of the
developed hardware. In this chapter a mathematical model for key sub-circuits is derived
to be used in the following chapter.
The ﬁfth chapter describes the implementation of data processing for fault detection.
In this chapter the CUSUM algorithm is proposed as a method for detecting the direction
and magnitude of circulating currents. The hardware model from the previous chapter is
then used to verify if the developed hardware is sufﬁcient to properly monitor the parallel
inverter setup. Finally some general power parameter calculations are presented, which
should be implemented in the embedded solution.
The sixth chapter displays the test results of the hardware and implemented algorithms.
In this chapter the test-bench setup is used to verify the developed hardware, while the
provided real-world data is used to test the implemented algorithms.
Finally a conclusion is made based on the presented results, as well as some suggestions
for future development of the project.

CHAPTER 1
Introduction
1.1
Background
Thiim A/S is a company situated in Soeborg specializing in electronics development
and commerce. The company has been developing customized electronics solutions for
almost 80 years, and has a large expertise in development of various types of monitoring
relays for industrial applications. In 2016, Thiim received a request from the company
Gutor Electronic, which is a sub-division of Schneider Electric. Gutor Electronic is one
of the world leading manufacturers of industrial Uninterruptible Power Supply (UPS)
systems. The request made by Gutor was for an ultra fast AC current direction detection
relay, capable of detecting the direction and magnitude of equalizing currents between two
parallel UPS systems.
1.2
The Project
The ﬁrst objective of this project is development of the required hardware for the
parallel inverter monitor. Since the developed product has to ﬁt into the existing range
of products developed by Thiim, some of the physical requirements for the product has
already been detailed. This includes the selection of casing and placement of external
connectors in the ﬁnal hardware. For this project, various circuits should be developed
for measuring currents and voltages, as well as provide suitable signal outputs for the
end-user.
The second objective for the project will be to develop an algorithm, which can quickly
and reliably detect the equalizing currents in parallel inverters. In order to determine the
optimal algorithm, several different approaches should be tested and compared for speed,
reliability and sensitivity.
The third objective for the project will be to develop the embedded software required

to implement the chosen detection algorithm. The software should handle all the online
measurements, data-processing and proper fault indication. Furthermore, the software
shall include a method to obtain the measurement results, as well as a method to perform
ﬁrmware upgrades. The possibility to perform ﬁrmware upgrades is crucial, as some minor
software bugs might be discovered down the road, which can then be ﬁxed in the ﬁeld.
The project is performed in cooperation with Thiim A/S, who will be using the ﬁndings
of this project as well as the developed product after the project is ﬁnished.
1.3
Project Delimitation
The project is 30 ECTS points and has a timeframe of 5 months spanning from August
1st to January 1st.
The primary focus of the project is to determine the necessary hardware features for
fulﬁlling the requirements set by the client and Thiim A/S. This will encompass simulations
of a simple parallel inverter system, as well as measurements on an actual parallel inverter
setup. The obtained knowledge will be used in designing a set of working prototype
hardware for doing measurements on the actual system.
The secondary focus of the project will be develop and test algorithms for fault
monitoring in parallel inverter systems. In this part of the project, at least one algorithm
must be developed and tested for each type of fault speciﬁed by the client. The developed
algorithm should be implemented in the embedded platform and tested on the parallel
UPS testbench used by the client for development, in order to verify functionality. The
developed algorithms should be modular and less hardware dependent to facilitate future
hardware updates.

CHAPTER 2
Project Requirements
In this chapter the various requirements for the developed solution will be detailed.
The requirements will be separated into functional requirements, which encompass re-
quirements for the functionality of the developed hardware and software to ensure a proper
detection. This will include which parameters to detect, as well as detection speed, reso-

The functional requirements has been divided into two separate groups. The ﬁrst
group is the requirements set by the client Gutor Electronic, which is considered the

The second group of requirements is set by the company Thiim A/S, which primarily
include requirements which will enhance the usability of the ﬁnal product as well as widen
the range of potential customers. These requirements are considered non-essential, but
should still be taken into account when designing the hardware.
2.1.1
Gutor Requirements
Based on the Gutor Requirements, the device should be able to detect the instantaneous
current direction between two parallel UPS systems, in case of circulating currents. The
device should furthermore be able to monitor the instantaneous load sharing between the
two UPS systems and sound an alarm in case the load sharing falls below a set threshold.

CHAPTER 3
Paralleling Inverters
This chapter will give an introduction to the various challenges faced when operating
power inverters in parallel. It will be investigated how two parallel inverters inﬂuence
eachother, as well as how this can be measured externally. The analysis will then be backed
by simulations on an optimal system as well as measurements from an experimental setup.
A distributed generation system consisting of several distributed micro generators
holds several advantages compared to conventional centralized power generation[5]. This
includes increased voltage support, reliability, price, and reduced environmental impact.
Distributed generators often consists of synchronous- or asynchronous generators, which
might be synchronized with the electrical grid, hence providing power directly to the grid.
Depending on the generation system this might be disadvantageous, as the power might
not be available to the grid when needed, and likewise might be abundantly available when
not required. To counteract this the power generator is often decoupled from the grid by
transforming the AC power into DC power, and then back into AC power utilizing a power
inverter. This provides several important advantages, as the generator is no longer required
to be synchronized to the grid and is even electrically shielded from the grid disturbances.
Furthermore, the DC power may be stored in batteries or capacitor banks for use when
needed by the grid. Examples of generators used for distributed power generation, which
rely on power inverters, are Uninterruptible Power Supply (UPS) systems, photovoltaic
(PV) solar systems, and some wind turbines.
3.1
Grid Synchronization
Whether the distributed generation system is connected to a wide area synchronous
grid or to a separate microgrid, synchronization of the inverters to the grid is of utmost
importance. Synchronization of the inverter will ensure proper grid support, without putting
additional strain on the grid and other distributed generators. The inverter synchronization
7
==PAGE:24==
8

is handled by the internal controller, which handles the controller synchronization and
power delivery. This can be done using different controller topologies, where some of the
most common are Droop Control and Average Power Control[6].
3.1.1
Islanded Operation
Operating inverters without direct connection to the main power grid is called islanded
operation. In this case, the distributed generators are the sole supply of power to the
microgrid, and increased control is therefore necessary, since there is no single source of
synchronization. This can be solved by employing a single master inverter to provide a
general waveform, frequency and voltage, and then adding a series of slave inverters to
provide increased power[7]. This setup has the disadvantage that the master inverter is no
longer redundant, and the entire grid might fail due to a breakdown of this inverter.
A commonly used solution to avoid having a single master in the distributed generation
system is to employ droop control for each connected generator[8, 7, 9, 10, 6]. This will
allow the generators to increase power delivery to the system in the event of a frequency
drop, at the expense of the frequency not always having the desired nominal value.
Another solution might be to employ a communications scheme between the individual
inverters, sharing the common goal and setpoint of the grid[11, 12]. This setup creates
increased requirements on the infrastructure of the setup, and with distributed generators,
a common communications network might not be practical. Furthermore, delays and
packet loss might lead to discrepancies between the inverters causing small currents to
ﬂow between those.

3.5
Test Summary
From the simulations and measurements above, it is clear to see that even slight
discrepancies in the voltage outputs of the two inverters have great impact on the provided
power. From the simulations it is seen that a small reactance in the transmission line
results in a phase-error of the produced current for both a phase discrepancy as well as
a voltage discrepancy. As expected, the produced current discrepancy is identical with
both an inductive and capacitive load, but with opposite current waveforms for the two
inverters. This is expected as the inherent phaseangle for the two loads is ≈±85°, and the
additional phaseshift caused by the discrepancy either lowers or raises the phaseangle of
the current. The additional phaseshift then either increases or lowers the resulting active
power, which is seen by the change in current magnitude.
Looking at the measurements from the test setup, it is seen that the output is much
more predictable. This is due to the large series resistance for each inverter, lowering the
effect of any reactance in the resistor or wires connecting the two inverters. A voltage
discrepancy is seen to directly affect the real power output, while having a negligible effect
on the phase angle. Simultaneously, it is seen that a phase discrepancy has an ampliﬁed
effect on the phase angle of the current, but a minor effect on the produced real power.
The increased phase angle of the currents is seen as a direct effect of the voltage phase
difference, causing a reactive load to be seen from each inverter caused by the load from the
opposite inverter. It is also seen that the current magnitude is increased for both inverters
without a signiﬁcant increase of load voltage, indicating a current ﬂowing between those
instead of to the load.

CHAPTER 4
Hardware Design
In order to have a device capable of measuring circulating currents between parallel
inverters, some hardware must be developed. This hardware should adhere to the previously

Measurement System
In order to measure the residual current as well as monitor the load sharing and power
generation of the inverters, a measurement system is required. The measurement system
will be able to measure the current ﬂowing from each inverter, as well as the voltage
produced in the interconnection point between the inverter outputs.
Since the inverters will be connected in parallel to a single interconnection point,
according to Kirchoff’s current law it will be sufﬁcient to measure the current on the
inverter outlets omitting the load current, as this can be calculated later. Furthermore, since
the device will be monitoring the load sharing between the inverters, it is necessary to
measure the voltage in the interconnection point. Given that this voltage is already fed to
the device, it will also be used to power the unit, as seen in Figure 4.1.
4.1.1
Current Measurement
To measure the current delivered by the inverters, a current measurement device has
been developed. First a suitable sensor for the envisioned application has been selected.
A signal conditioning circuit has then been developed and the produced signal is fed to a
high resolution analog-to-digital converter.
Sensor Selection
When selecting an appropriate sensor for measuring currents, several parameters
must be considered. One approach would be direct measurement, placing a small load
in series with the feed lines, measuring the voltage across this. This approach was
disregarded immediately, as this would place additional requirements to the isolation and
surge protection of the inputs. Furthermore it might interfere with the devices on which it
is measuring, and is therefore not a good solution.
As for indirect current sensors, there is a wide range of suitable technologies. Speciﬁ-
cally, two technologies have been considered for this device; Current Transformer (CT),
and Rogowski Coils.
Current Transformers traditionally consists of a circular piece of metal with a secondary
coil wrapped around it. The metal core is then placed around the current-carrying conductor
representing the primary coil. The current ﬂow in the primary conductor then induces
a magnetic ﬁeld which is concentrated in the magnetic core. This ﬁeld then induces a
current in the secondary coil proportional to the primary current divided by the number
of windings of the secondary coil. Metal core current transformers are available in a
wide selection of shapes and implementations. A common selection for clip-on current
==PAGE:43==
4.1. MEASUREMENT SYSTEM
27
transformers is a split-core transformer, where the transformer core can be separated in
two for installation without disconnecting the primary current conductor.
A traditional current transformer is available in a range of different accuracy classes[14].
These classes deﬁne the accuracy of the transformer at 100% to 120% of the rated output.
For lower outputs the rated accuracy of the current transformer degrades rapidly at 20%
and 5%. Another issue with traditional metal core current transformers is saturation of the
magnetic core. At some point the magnetic core in the current transformer will saturate
causing distortion in the secondary output. A third issue with the traditional current
transformer is the physical limitations. A traditional split-core current transformer might
not always be possible to install in tight spaces, as the solid metal core must ﬁt around the
primary conductor.
The speciﬁc traditional current transformer considered for this project is a YHDC
SCT036TS[15] capable of transforming 600A to 100mA. This transformer is not capa-
ble of handling the full range speciﬁed by the client, but might be sufﬁcient in some
applications.
A Rogowski Coil transformer[16] offers increased ﬂexibility compared to traditional
current transformers. The Rogowski Coil is an air-core transformer, wound around a
ﬂexible plastic tubing offering approximately the same ferromagnetic properties as air.
The Rogowski Coil works by having the pick-up coil magnetically coupled to the primary
current conductor, creating an instantaneous voltage at the coil output corresponding to
the mutual inductance between the primary conductor and the pick-up coil multiplied by
the derivative of the current ﬂowing in the primary conductor. This implies that the the
output voltage will be phase shiftet 90° relative to the primary current. Due to the lack of a
magnetic core, the Rogowski Coil can handle large currents with good linearity.
The speciﬁc Rogowski Coil considered for this project is a Meatrol MRC-36D[17],
which is a low-cost transformer capable of measuring up to 100kA with an output of
100mV/kA@50Hz and an accuracy of 0.5%. The bandwidth of the MRC-36D is 1Hz −
1MHz with a low temperature drift.

As for the RS-485 interface, the requirements are as follows. ESD Protection according

protection according to EN 61000-4-4 level 3 of 1kV peak transient for signal ports. Surge
protection according to EN 61000-4-5 level 2 of 1kV line-to-ground surge voltages. In
order to meet the speciﬁed isolation requirements, a Littelfuse SMBJ12CA[21] is added to
each input of the RS-485 interface. The SMBJ12CA is a 12V 600W TVS diode, which

at 30kV air- and 30kV contact discharge in accordance to the EN 61000-4-2 standard.
The diode clamps the input voltage at 19.9V at a peak current of 30.2A, satisfying a
peak transient of 1.5kA using a generator with output impedance of 50Ωas speciﬁed in
the EN 61000-4-4 standard. For the EN 61000-4-5 standard, a test scenario of coupling
to unshielded, unsymmetrical interconnection lines is assumed. This implies a testing
condition with an output impedance for the generator at 40Ω. Given the peak clamping
current of 30.2A, the TVS should be able to handle a test-condition of 1.2kV, making it
just reach the speciﬁed requirement.
Based on the above analysis, there is no guarantee that the developed circuitry will
pass the speciﬁed tests in the aforementioned standards. It has however been shown that
the theoretical properties of the speciﬁed components are likely sufﬁcient to pass the tests
to a satisfactory degree.

assumed to approximate a gaussian sequence with zero mean and the variance of the
measurement noise. Given that the circulating current is expected to be sinusoidal, a
two-sided CUSUM algorithm capable of detecting both positive and negative changes is

CHAPTER 6
System Test and Veriﬁcation
The system test and veriﬁcation will be performed by doing a series of tests, comparing
the developed hardware and algorithms to the expected results. Two different kinds of tests
will be performed.
The ﬁrst test will be on the experimental setup in an controlled environment. In
these tests a known voltage discrepancy will be applied to the system, and the developed
hardware will be used to monitor the resulting output. These results will be compared to
oscilloscope outputs, to verify the measurement capabilities of the hardware. A suitable
threshold will then be used in the detection algorithms, to verify that the expected fault-
magnitudes are detected within the speciﬁed time frame.
The second test will be on measurements from the reference parallel UPS setup
provided by the customer. Unfortunately it has not been possible to perform measurements
on the setup with the developed hardware. Given the high resolution from the provided
measurements, it is however assumed that a down-sampled set of data derived from the
provided measurements will be sufﬁcient to properly verify the fault-detection algorithms.
The down-sampling process has been performed using a model of the developed hardware,
as seen in the previous chapter, and is therefore assumed to reasonably represent the
expected output of the developed hardware.

CHAPTER 7
Conclusion
The use of distributed generators is a commonly used method of providing reliable
power to critical systems. The use of parallel UPS systems provide redundant power-
backup in case of power-outages, which is an important factor in ensuring the continuous
operation of critical loads. The analysis on parallel inverters have shown that even the
slightest asynchronicity between parallel inverters can cause large power-losses in the
system. This has been shown to be measurable by monitoring the circulating current
between the parallel inverters.
A piece of hardware has been developed with the minimum required amount of sensors
for monitoring such a system. Through analysis it has been shown that the developed
hardware is capable of providing sufﬁcient information about the operating state of the
system to accurately determine the power-ﬂow between the inverters and in the connected
load.
Two algorithms have been proposed and implemented in MATLAB to monitor the
desired fault-indicators as speciﬁed by the client. Through simulation it has been shown
that these algorithms are responsive and agile in detecting circulating currents within
the system, as well as determining the active and reactive load-sharing between the two
inverters.
Unfortunately it was not possible to coordinate a test-session for doing measurements
on the clients UPS setup using the developed hardware. A simulation has instead been
performed, down-sampling the provided high-resolution data from the client by using a
mathematical model of the developed hardware. This simulation showed that the expected
measurements will be of sufﬁcient resolution to properly reﬂect the state of the system,
although some aliasing was seen during the down-sampling.

CHAPTER 8
Future Work
The next step in the continuation of this project is to coordinate a test-session with the
client. This will provide an opportunity to present the preliminary ﬁndings to the client,
as well as ﬁnally verify if the developed hardware is sufﬁcient to monitor the required
parameters.
Some more tests should be performed using different algorithms for detecting the
circulating currents. Among others variance-based CUSUM detectors and Generalized
Log-likelihood Ratio tests should be explored to further improve the detection speed and
robustness of the circulating current detector.
The developed hardware should be updated to further improve resistance to noise.
Due to desired placement of the external connectors for the analog inputs, some of the
low-power signal traces became longer than necessary. A restructuring of the hardware
design could improve this. The maximum sampling speed of the device should also be
explored, as the over-sampling ratio of the ADC should be conﬁgured according to this to
further improve the usage of the internal antialiasing ﬁlter.
Finally an embedded realization of the implemented algorithms should be made, as
well as development of the software for the MODBUS RS-485 interface.
