Summary
This thesis introduce a continues-time state space model for a smart-house
suggested by the literature. The model is used for simulations of Economic Model
Predictive Control (EMPC) for control of smart-house energy system.
The
smart-house energy system consists of solar panels, electrical vehicle and stationary
battery that is connected to grid with an option of buying and selling electricity.
The constructed linear system for EMPC is investigated.
Smart-house energy
system is implemented in C/C++. The optimal input values for EMPC are found
by Cplex, Gurobi and Mosek optimizers.

CHAPTER 1
Introduction
This thesis utilize energy system based on renewable energy sources, the system is
constructed based on residential house with solar panels, stationary battery, electrical
vehicle and a heat pump. Figure 1.1. illustrates a house with energy sources from
solar panels and the grid, a stationary battery for energy storage, a EV battery and
a heat pump supplying heat to the house.
Economic model predictive controller
(EMPC) is propose for the system control and linear program optimizers will be used
to find optimal input values. Optimizers in question are Cplex, Gurobi and Mosek.
Moreover, the algorithm choice will be discused and LP problem will be analysed.

1.1
Motivation
In the resent years the increased emission of green-house gasses and high
consumption has let to environmental concern in the overall population.
At this
point the environmental concern has raised to a level where environmental activist’s
Greta Thunberg protest has grew into a global movement.
As generation of electricity is a major cause of industrial air pollution, the renewable
energy has a great potential in reducing the CO2 emission. As the issue is raised,
more and more societies wish to transition to sustainable energy and technology.
Distinct renewable energy source have gain attention in different countries, Norway
produces 99% of it’s electricity by hydropower plants, Denmark produces nearly
50% of it’s energy from wind turbines. Solar radiation is a well known renewable
energy source and it does not emit any greenhouse gases when producing electricity.
Multiple countries produce solar energy, like Germany, the UK, Italy, and India.
However, the process of transitioning to renewable energy sources will take time,
therefore more immediate solutions are looked at. In regards to that, Elon Musk
has shared his vision of ’Tesla Smart Home’ and is working on development of
advanced technology systems that adapt to the needs of the environment.[23]
The idea of ’Smart Home’ is to use individually produced renewable energy and
replace the fossil fueled car with electrical vehicles.
Electrical vehicles have been
around since 1873, but the spike of popularity was gain only in 21st century due to
environmental concerns. Electrical vehicles show great potential to reduce the CO2
emission created by vehicles. [19]

Linear programming
Linear Programming is an optimization technique for a system with linear objective
function and a set of linear constraints. Linear programming optimization problem
can be formulated as
min
x
c′x
(2.7a)
s. t. A′x ≤b
(2.7b)
The linear programming problem generally has a matrix A ∈Rm×n and column
vectors b ∈Rm and c ∈Rn. The task of linear programming is to find a vector
x ∈Rn, such that, Ax ≤b and c′x is minimum or maximum, depending on the
optimization problem.
If a solution can not be found, the problem is infeasible,
where {x ∈Rn : Ax ≤b} = ∅or the problem is unbounded , where ∀α ∈R there is
an x ∈Rn with Ax ≤b and c′x > α.
Solutions of LP problems:
• Feasible solution
• Optimal solution
• Degenerate solution
• Alternate optima
The solution of a Linear Program (LP) is called feasible, if solution x satisfies Ax ≤b.
The feasible solution is called the optimal solution if the objective function reaches
it’s maximum (or minimum) value, as presented in Figure 2.6.

• Theorem.
Strong Duality - If there exists an optimal solution to an LP
problem, then there exists an optimal one to its dual, and the respective
optimal values are equal. [17]

CHAPTER 8
Discussion
This thesis starts by developing model based control system for smart-house energy
system,
This
system
is
initially
implemented
in
Matlab.
The
Matlab
implementation is a starting point to get a better overview on system dynamics and
structure.
Matlab is a great tool to analyze data, develop algorithms and create
models.
Matlab has built-in math functions that enables one to explore multiple
approaches in a easy manner.
This is the reason why it was used when
experimenting with the system. In regards to that, the Matlab code was edited and
adjusted to many times, in consequence, the code it self at the end stage, became
unreliable. The author of this thesis regrets not using version control. However, the
smart-house energy system implemented in C/C++ plays a main role in the system
analysis. This implementation is carefully thought through and adjusted only when
real findings were found. The version control is well thought through.
There are multiple factors determining that a smart-house energy system is a better
fit for object oriented programming. The system is complex and computationally
heavy, therefore the performance plays an important role.
Matlab is a script
language, but C/C++ is compiled using the byte-code instructions that the
computer processor understands and executes directly, additionally, Matlab uses
multi-processes, but C/C++ is multi thread.
This makes the C and C++ faster
then other programming languages and it used the least memory.
Matlab has
dynamic memory allocation, in algebraic operations, it creates temporary variables,
in C/C++ memory can be allocated before hand.
Matlab most known advantages of tool-box and built-in math functions, but using
toolbox in Matlab is computationally expensive.
Regarding that, C/C++ can
matched the high-level syntax and functionality, as through time there have been
developed multiple open source libraries created to
match Matlab
easy-use
functions. In this thesis, Armadillo high quality linear algebra library is used for
transition to C/C++ code.
Performance and memory use are important in smart-house energy system, if one
wishes to create a software solution to the energy system.
During the experiments, it was established that in all cases the C/C++ language
implemented optimizers outperform Matlab optimizer.

8 Discussion
Moreover, object oriented programming (OOP) provides a clear modular structure.
In OOP templates for objects are created, that makes the code reusable and easier
to maintain.
Reusable code makes the software development faster.
The
smart-house energy system created in this thesis in regards to that, can be easily
adjusted into house specifications, additionally, building on top of the program is
fairly simple. For example, if one wishes to create a smart-city, by constructing an
additional class of city, one can add multiple house objects to it.
The smart-house energy system in this thesis consists of three linear models, heat
pump, EV and stationary batteries.
The EMPC used to control the system was
presented as LP with several layers of soft constraints. The derived LP problem was
analysed and the findings of the LP problem will be discussed in this section.
LP problem in the smart-house energy system optimizes Uk vector that provides the
optimal input values for heat pump and EV battery power supplied by stationary
battery.
In regards to stationary battery, the optimal electricity purchase/sell is
optimized based on electricity prices.
Stated optimization problem has a degenerate solution, as the optimized input
values can be zero in basic solution. Degeneracy is a phenomenon in LP that creates
complications if simplex method is used in optimization of LP problem, as it can
occur that the vertices of the convex polyhedron of basic solutions are not uniquely
defined. The results of this thesis show that there are multiple optimal solutions, as
the objective function values for open-loop simulations is 0.0077 with all optimal
solutions found by MOSEK, Cplex, Gurobi optimizers. This polyhedron also can be
caused by alternate optimal solution, that is, when the objective function line is
parallel to a constraint line.
The use of three optimizes for solving the LP problem has yield the discovery of
multiple optimal solutions. If the LP problem has multiple solutions it makes the
problem computationally expensive, that is another reason to argue for use of
C/C++ language in smart-house energy system.
Furthermore, the optimizers used in that C/C++ implementation are Cplex,
Gurobi and MOSEK, these optimizers are analysed based on optimal values and the
performance.
In the open-loop simulation analysis it has been found that the
optimal solution for electricity purchase and sell is the same with all the
implemented optimizers. This finding tells that the solution regarding this part of
LP problem is uniquely defined. The optimal input in regards to EV battery has
been found to be alternating between the optimizers.
This finding points to
degeneracy and/or alternate optima, it has been stated that the LP problem has a
degenerate solution, therefore attention is pointed to alternate optima.
==PAGE:77==
8 Discussion
63
To discuss alternate optimal solution, one has to look at the results from heat pump
input and EV battery input.
The Cplex and Gurobi optimizers find the same
optimal solution in open-loop simulation in regards to heat pump input power.
Moreover, the correlation is detected in closed-loop simulation, as when the multiple
optimal solution appears, which causes next LP problem to differ, it is in regards to
power transfer to EV battery and heat pump. This phenomenon points to alternate
optimal solution, as it is related to constraint regarding the maximum power
transfer from stationary battery to house devices.
Moreover, by examining results from open-loop simulation, the total energy stored
in stationary and EV batteries, was found to be the same. This finding points to
hypothesis that there is not a general difference between storing energy in EV
battery or stationary battery.
Furthermore, the open-loop simulation yields that the model is not valuing the
energy stored in batteries, therefore draining the batteries to the lower constraint.
This can be seen in the closed-loop simulation as the SOC of battery approaches the
constraint twice in the 5 day simulation time.
Due the exhausted time for this
thesis, the addition adjustments were not made.
The performance of optimizers implemented in C/C++ out-stand the Matlab
optimizer and between the optimizers implemented in C/C++ best performance is
shown by Cplex optimizer.
Additionally, the results point to similarities between
the Cplex and Gurobi optimizer, but Gurobi performance is significantly lower.
MOSEK optimizers, although uses the same algorithm, shows significantly different
solution.
The solvers are selected based on accessibility, can be used with academic licences,
speed of the solver and quality of documentation.
Armadillo is used as the library for linear algebra and scientific computing purposes.
Armadillo is selected as it can be directly used for development in C++ language, it
can automatically use OpenMP multi-threading, it is open source library and it’s
functionality is similar to Matlab.
This library has outstanding documentation.
Library can be found in [22].

CHAPTER 9
Conclusion
Smart-house energy system reduces electricity cost by adjusting to the electricity
price forecast, as well as, it increases the renewable energy used in individual
residencies.
The EMPC shows great potential as a control of energy system, the
outputs are kept within set constraints.
In regards to the software development, the OOP is the appropriate way to
implement Smart-house energy system, as it can be individually adjusted from
house to house and even to city.
For the optimization of LP problem the best performance is with Cplex optimizer,
hence, it is preferred in the energy system implementation.
Moreover, the LP optimization problem of EMPC has degenerate LP solution and it
has alternate optima.
These phenomenons are caused by EV battery and heat
pump subsystems.
Furthermore, when the LP problem is optimized, it does not
differentiate between the batteries, both batteries are treated as power storage.
However, the optimal solution for electricity purchase and sell is unique.
9.1
Future work
The LP problem formulation can be improved by adding additional constraints to
system to insure unique solution.
However, the author of this thesis has a
hypothesis that the issue of non-unique solutions would be eliminated if the
smart-house energy system is applied to smart-city energy system, due the grid
arising constraints. Furthermore, the objective function can be improved to count
for energy stored in the batteries.
On top of the smart-house energy system one can built application program interface
that sends information about the system to the smart-house owner and create energy
usage plan.
The sensor data can be integrated in the system and one can analyse how the LP
optimizer behaves, and if the optimizer speed is adequate for real time optimization.
