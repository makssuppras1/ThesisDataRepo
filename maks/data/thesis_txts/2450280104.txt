Cyber security companies such as FireEye have been gathering data and re-
searching the rising cyber-attacks carried out using PowerShell throughout
the past few years. PowerShell is used by cyber threat actors to among other
things install backdoors, execute malicious code and generally serves as a
way for malicious entities to carry out their cyber-crimes. The ﬂexibility and

I will develop a software which uses a model based on machine learning.
Cyber security companies have repositories available containing data on ex-
amples of malicious PowerShell activities that will be used for developing
the model, i.e. using the data to train and test the model. The ﬁrst step is
identifying the most suitable machine learning algorithms and techniques.
By studying each algorithm, an informed decision will be made on which
algorithm best ﬁts the software.
The outcome of the project is expected to be a working model that can detect
malicious PowerShell activity. There will also be an opportunity to integrate
the outcome of the model to a SIEM solution such as QRadar, utilizing the
model in this manner provides greater visibility when monitoring network-
s/endpoints.

the opportunity to get access to local registry, and capable of downloading
content from remote locations [Car16]. As PowerShell has increasing pop-

interesting for cyber threat actors. [Fan18]

researching the rising cyber-attacks carried out using PowerShell through-

by cyber threat actors to among other things install backdoors, execute ma-
licious code and generally serves as a way for malicious entities to carry out
their cyber-crimes. The ﬂexibility and capabilities of PowerShell has made

2.1
PowerShell
Knowing the concepts of PowerShell and how PowerShell interacts with the
user input is vital for this work. The main objective of this thesis is to de-
crease the amount of cyber-crimes carried out through PowerShell by adding
a detection mechanism to the tool. To do that, we must understand what
PowerShell is, what PowerShell is capable of and what makes it so relevant
to increase its security.
PowerShell is a scripting language and command-line shell developed by
Microsoft for automating and managing tasks.
PowerShell also provides
full access to Windows Management Instrumentation and Component Ob-
ject Model objects which are considered critical Windows system functions.
With these built-in functionalities, PowerShell becomes a powerful tool with
capabilities such as executing commands directly from memory, the opportu-
nity to get access to local registry, and capable of downloading content from
remote locations. [HKR18; Uga+19]
The functionality of PowerShell is also extended due to cmdlets which are
used to deal with objects. They are basically reusable scripts with a given
functionality. Here, the administrator or the crime-actor takes advantage of
PowerShell’s own designated functionality with cmdlets that are built into
the language, or simply by loading from external modules to further expand
the capabilities of PowerShell. There are ﬁve main design goals that charac-
terize the scripting language and they are the following: [ACW18; Uga+19]
Discoverability. PowerShell makes it easy to discover its features, to sim-
plify the development process. For example, if it is needed to know which
cmdlets can be used to interact with services, the following command can be

used to extract a list of all possible cmdlets that are related to services by sim-
ply using the cmdlet Get-Command and manipulating the argument with a
wildcard as the following:
Get-Command *-Service
Now the administrator has the possibility to dig deeper into each command
by using cmdlets like Get-Help, which corresponds to using the argument
-h or –help in other scripting languages. But what makes discoverability
an interesting thing is the cmdlet Get-Member. This can be used by piping
the output of a cmdlet into Get-Member. For instance, if it is desired to know
more about the Get-Service cmdlet, you will simply pipe the output into Get-

using the Sort-Object cmdlet, it is possible to sort the output of any cmdlet,
even without having knowledge of the internals of the cmdlet that is used
to generate the output.
This also means that developers do not have to
build sorting mechanisms into the custom cmdlets, as PowerShell provides a
framework that forces consistency.
Interactive and scripting environments. Powershell extends the capabili-
ties of the user by combining interactive shells and scripting environments.
This allows access to .NET libraries, Component Object Models (COM) and
command-line tools.
Object orientation. As PowerShell is based on objects, the output of the
commands are objects and not text. The outputs can be pipelined as inputs

2.1.1
Offensive use of PowerShell
The versatility of PowerShell and the freedom it provides for its users makes
it an ideal candidate for any purpose, whether the purpose is to carry out an
attack or defend against it. There are various reasons for why this tool can be
an ideal candidate for malware creators.
First, PowerShell is installed by default on all new windows machines. This
makes Windows machines an ideal target to execute malicious PowerShell
commands on, due to the high chance of the tool is already installed on the
machine. Cybercriminals can also take advantage of this functionality as they
often prefer preinstalled tools for various reasons, for instance, to speed up
the development or just for simply staying under the radar. [Wue16]
Furthermore, PowerShell has a growing community, which means a lot of
system administrators use and trust the framework, allowing malware cre-
ators to blend their attacks with the regular work of system administration.
Additionally, the fact that PowerShell commands may use external modules,
makes logging the code executed by PowerShell difﬁcult. When for instance
using SIEM solutions to monitor PowerShell, it is not necessarily possible
to see the content of the executed code if the user uses external commands.
Therefore, it makes it even harder to detect PowerShell based attacks due the

As PowerShell gives the opportunity to download remote content and to ex-
ecute payloads directly for memory, it is a perfect tool to carry out ﬁleless
attacks to evade detection by conventional anti-malware tools, as in other
words, making it stealthy. [Car16; Wue16]
2.1.2
PowerShell Code Obfuscation
Source code obfuscation is a programming technique used to perform mod-
iﬁcations on binary ﬁles or source code by obscuring it with random char-
acters and functions and at the same time maintaining the functionality. The
purpose of this is to secure the code from tampering and reverse engineering,
however, these techniques have been leveraged by cybercriminals to serve
their goals. The techniques are being used in malware by the attacker to ob-
scure the functionality of the application/functionality to avoid detection by
conventional anti-malware programs. [Wue16; YK19]
There are many obfuscation methods that are used by attackers to carry out
the attacks like the following [Wue16]:
• It is possible to write the commands in a mixed upper and lower letters
sequence as they are not case sensitive.
– Example. doWnLOaDﬁLe
• Some of the words in a command-line can be omitted and they will still
function as they would, for instance it is possible to remove ’Get’ from
several commands.
– Example. Command will function as if Get-Command was written.
==PAGE:22==

• Strings can be concatenated. This is very efﬁcient as well if an attacker
wants to avoid predeﬁned URLs.
– Example. ..DownloadString("ht"+’tp://’+"test.dk")
• Instead of using arguments like ’help’ or ’hidden’, these can be replaced
with their nummerical value.
– Example. -window 1 will function as if -window hidden was written.
• Strings can be encoded or decoded by using functions like Base64 UTF8.

2.2
Machine learning
Machine learning is a technique to let a machine increase its knowledge and
learn based on observations, experiences and analyzing different data pat-
terns based on a given data set, which also means this can happen without
needing explicitly to be programming. The programmer provides very clear
instructions for the machine learning model to follow, alongside a data set
that is used for training, i.e. it uses it to analyze and identify the data and
learn how to make decisions based on what is already observed and learned
from the data set. [Hag+]
When feeding a machine learning model with more knowledge/data it is
done by two steps. First, a data set is provided to the machine learning model
also known as the training set, this could contain examples on malicious and
non-malicious PowerShell commands alongside a classiﬁcation for each en-
try of which are malicious, and which are not. This training data enhances
the machines ability to identify and classify similar data.
The second step is to test the machine’s knowledge on another set of data,
also referred to as the test data. Here the machine tests its accuracy by try-
ing to identify and classify the new data set, but this time the entries are not
labelled to be malicious or not. Then the result of the correct (True Positives)
and the wrong (False Positives) classiﬁcations is used by the machine to ex-
pand its knowledge.
Machine learning can be classiﬁed and divided into three different categories
and they are the following [Hag+]:
• Supervised algorithms
==PAGE:29==
2.2. Machine learning
19
– In these types of algorithms, the training set is labelled. For in-
stance, if the training set includes PowerShell commands and the
purpose of the training is to teach the algorithm whether the com-
mand is malicious or not, each entry will have a label of what the
expected classiﬁcation/output would be.
• Unsupervised algorithms
– In these type of algorithms, in opposite to supervised algorithms
the training set is unlabelled. The algorithm ﬁnds trends in the
data, e.g. clusters, that can be used to classify new data.
• Reinforcement machine learning algorithms
– In these types of algorithms, the algorithms have to learn and un-
derstand the behavior within a speciﬁc context. Then based on the
decisions it takes, it will either be rewarded or punished for the
decision, and the result of this will enhance the machines ability to
determine which steps it should take in the future. Here the pro-
grammer has to make a decision to adjust the machine to either
focus on long-term or short-term rewards.
Referring to the research question of this thesis in section 1.1, "Can machine
learning be used to predict if a PowerShell command is malicious", the focus will
be on how to classify malicious PowerShell commands, i.e. which algorithms
ﬁts best into the question. As Machine Learning is a broad subject and there
exists many different algorithms used for different purposes, only the rele-
vant type of algorithm will be introduced in the following sections, namely
supervised algorithms.
Supervised machine learning algorithms can be further grouped into Regres-
sion and Classiﬁcation. Even though both types of algorithms have the same

Classiﬁcation in machine learning is a supervised approach to let the ma-
chine gain the ability to learn from the inputs it gets by observing, analyzing
and testing the data against other data sets.
There exist multiple different classiﬁcation methods, but only artiﬁcial neural
network will be brieﬂy introduced in this chapter.
2.2.1.1
Neural networks
Artiﬁcial neural network, also referred to as ANN, is a computational ap-
proach inspired by the structure of the biological neural network that forms
the brain. The computational unit of the brain is a neuron, and in the same
way as the human brain, ANN is built by interconnected neurons. However,
there exists different types of neurons, each with different properties. The
neurons that connects the network within the artiﬁcial neural network is in-
spired by the neurons from the brain, just in a simpliﬁed version. [Kar]
The neuron carries a function called the activation function, ﬁrst, it calculates
the sum of the multiplication between the neuron’s input (weight) and the
chosen bias (independent weight), then the activation function is applied to
the weighted sum and an output is then sent to the next neuron in the chain,
see ﬁgure 2.2.

The artiﬁcial neural network is organized into distinct layers of neurons.

from a neuron can become an input for another neuron, see ﬁgure 2.3. There

• Input layer
– The Input layer is the input/starting point of the neural network.
The number of neurons that should be placed in the input layer,

network.
• Hidden layer
– The Hidden layer is a layer in between the input layer and the
output layer. The neurons in the hidden layer take a weighted set
of inputs and perform an activation function operation to give the
output layer the inputs it needs.
• Output layer
– The Output layer contains the neurons that classiﬁes the network’s

3.1.1
Preprocessing
Preprocessing is divided into two phases, ﬁrst, the PowerShell commands
are run through a developed script that converts the command into some-
thing more readable. The second part is normalizing the data such that the
neural network can classify the input.
Before the data is sent through the neural network, it has to be processed by
letting the PowerShell commands being run through a script. In section 2.1.2,
a few obfuscation techniques where introduced. The idea of this phase is to
convert the PowerShell command to the most readable and understandable
version of itself. This means that we have to eliminate any obfuscation in the
command, decode any encoded commands and convert any short-ﬂags into
understandable ﬂags.
One of the reasons of this is that when the command is forwarded to the SIEM
solutions by an alarm, the alarm will contain both the PowerShell command
before it was processed and after it was processed. By that, the analyst will
have an easier time to understand the PowerShell command and the classiﬁ-
cation from the analyst, will be more accurate.
The steps of the preprocessing phase are the following:
1. Deobfuscate the PowerShell command.
• In this step, the script should look through different obfuscation
methods. For instance, it has to put strings together if they are
concatenated, remove any obfuscation symbols like carets, com-
mas, double quotes etc. Some ﬂags can have their arguments writ-
ten as a numeric value, which also makes it unclear to understand
what is happening. For instance, the ﬂag -W Hidden makes the
command run without showing any command-line window, this
==PAGE:39==
3.1. Architecture
29
can also be written as -W 1 which do the same. These numeric val-
ues should be replaced with the text that represent the action, i.e.,
-W 1 becomes -W Hidden and -W 0 becomes -W Normal.
2. Decode the Base64 commands if any.
• If there are any Base64 presented in the command, these have to
be decoded. The result of it can again be an obfuscated command,
therefore after we decode the command, there might be in need of
running through step 1 again.
3. Look for IOC entities.
• Now when the PowerShell has been transformed into a "clean"
form for text, the script should identify if there are any indica-

any speciﬁc IOC’s but for the entities, i.e., if there are any URL’s
or IP addresses presented in the command. The outcome of this
step will be a score represented by a number between zero and
one, that will be sent through the neural network to help the ma-

functioning as a weight.
When the above steps are completed, the "real" process of preprocessing
starts. This means that we have to convert the text into something that the

3.1.2
Neural network
In the following section the neural network for this solution will be described
based on the theory discussed in section 2.2.1.1.2. The choice of the type of
the neural network is determined by the application’s goal, there are differ-
ent primary types for artiﬁcial neural networks, Feedforward, Convolutional
and Recurrent. [Hag+; Bro18]
Each type of the neural networks has its own strengths for whatever the
application is designed for. For instance, the feedforward is a simple type
of network. The data only goes in one direction as they are formed as an
acyclic graph, commonly used for classiﬁcation, prediction of times series
and clustering. The convolutional neural network is specialized for images
even though it is structured as the feedforward network. But in addition to
the feedforward type, it contains some unique layers that helps the machine
to analyse small fractures of an image. Lastly there is the recurrent neural
network, this type is specialized for pattern recognition in different sequence
of data. Here the output relies both on the given input and the previous data
that has been processed through the time. This can for instance help to pre-
dict the next word in a given sentence. [Bro18]
As we are dealing with malicious PowerShell command detection and not
with images, the choices rely between the recurrent and feedforward neural
network types. The need of looking into previously processed PowerShell
commands is not existing for this solution. PowerShell commands can be
structured in various of different ways and still be functioning in an exact
same way. We do not want to let a classiﬁcation of a PowerShell command
be dependent on what earlier has been processed, therefore, this solution will
==PAGE:41==
3.1. Architecture

be built by developing a feedforward artiﬁcial neural network.
To build a neural network, there are some decisions that have to be taken
before we can start to implement it.
• How many neurons should the input layer contain of, i.e., a length of
the inputs to the model has to be deﬁned.
• How many hidden layers should be implemented in the model, some of
them can hold an element of memory (recurrent neural network), some
of them can be focusing on images (convolutional neural network) and
some of them are straight forward, parsing the weights to the next neu-
rons without any specialization (feedforward neural network).
• How many neurons should be placed in the output layer, i.e., how
many classiﬁcation possibilities should the model could give as an out-
put.
3.1.2.1
Output layer
Starting with the output-layer, the classiﬁcation of the PowerShell command
is either ’yes’ for malicious or ’no’ for non-malicious (binary classiﬁcation).
The model should output a probability score to deﬁne how conﬁdent the
model is with the classiﬁcation, see ﬁgure 3.2. For instance, if the score is 0.9,
it means that the model is 90 % conﬁdence about that the PowerShell com-
mand in question is malicious. Therefore, the activation function for the last
layer should be a sigmoid function (see section 2.2.1).
There is no need for multiple neurons in the output layer as we do not have
to classify the command into multiple classes. Simply one neuron will do the
job, a threshold will be set to say if that the probability score is over 0,5, i.e.
50 %, the command will be classiﬁed as malicious.

string can become into multiple words after decoding. Therefore, a length of
40 will be a reasonable size to have as a ﬁxed length as input. In this way, we
have to put 40 neurons in the input layer.

Chapter 4. Conclusion
4.1
Discussion
In section 3.2 we have two graphs showing the training and validation re-
sults of the developed model. At some point, the validation loss starts to
increase. This is due to the training set that we used to train the model, the
set contains data that is sufﬁcient, but when we input the training set to the
model we trim the input to a length of 40, while the actual data has a length
of 200 and thereafter. This means a lot of data is being let out and we just
started to confuse the model. However, when we use a data set where all
the data is around 40 of length or less, we will see a slightly difference in the
graphs. For instance, by tweaking the length of the model input to ﬁt the size
of the training set we will see results as the following:
FIGURE 4.1: Training and Validation accuracy
In ﬁgure 4.1, the model input was changed to a size of 200 to match the data
length in our training set. This shows us that in order to make the model
accurate in its prediction, the training set should be as accurate as possible.
Which also concludes that the model is working as it is designed, what it
needs is a more efﬁcient training set.
==PAGE:63==
4.1. Discussion
53
Another issue with this model it that it’s not able to translate the short-ﬂags
into actual words. As earlier described the words are getting transformed
into numerical values that represent a word in a dictionary. But here we face
problems in the words like NONI and NOP, as they are shortcuts for non-
interactive and no-proﬁle, and arguments like this can be written in a ton
of different ways due to the obfuscation techniques. A possible solution for
this is to use NLP (nature language processing) because of its different steps
to analyze the human language. For instance, the stemmer phase in NLP
stems tokens into semantically identical token, so no matter how the word
is written, it will always be converted to the "root" of the word, so we have
an identical structure and the training and validation of the model will be
more accurate. But as in this solution, we divided the preprocessing phase
into two. The ﬁrst is be able to handle these kind of issue when its fully im-
plemented, which will be further described in section 4.2.
The current solution now processes a malicious PowerShell command and
creates a ticket to an application that is integrated with the SIEM solution
QRadar. A better approach is to use the built in AQL functions that are
present in QRadar [Cen]. Even though they are developed for a different
reason, it is possible to design a function that runs a custom script and then
do any action if a condition is met. By this way, we do not have to install this
solution as a standalone solution on all the assets or to make a new server just
for this model, but it is going to be directly integrated with QRadar. Which
also means that whenever there are added more assets to the network or new
log-sources, the model will still take any PowerShell command trafﬁc, there
are no need to alter anything with the scalability. Having the script running
there will also give the analyst an opportunity to search after events that
happened within the same time frame as the command was run, to see if the
execution is related to something else.

Chapter 4. Conclusion
4.2
Future work
While this thesis succeeds covering the objectives, there are still more work
to be done to use the solution in our organization.
• Data
– The PowerShell command data from Deloitte’s Cyber Intelligence
team was not processed by the organization, which resulted in a
bad training set as among other things there were many identical
commands occurring again and again in the ﬁle. A new and bigger
data set should be gathered with at least of a 100.000 of entries,
such that they can be distributed as both training-data and test-
data.
• The model design
– The model developed is fairly simple (developed for the proof of
concept). A neuron should be added to only care about wherever
there exists any URL’s or IP’s. This property will have its own
weight added to the model. Also, as there must be experimented
with how many hidden layers there must be added in order to
achieve a higher accuracy, as the number of neurons in the hidden
layer is very small for this solution.
– The backpropagation phase has also not been discussed in this the-
sis. This will also further be investigated to let the neural network
learn from its mistakes (adjusting weights and biases).
• Implementation
– The provided implementation is just a ﬁrst step of the proof of
concept. First, in the preprocessing phase, we have to develop

malicious PowerShell commands.

out the thesis I have learned the basics and more about how machine learning
works, i.e. the algorithms, training, validation and the ease of implementa-

solutions. As well as I have learned how PowerShell obfuscation works and
how in general PowerShell is used today in the industry. I have also identi-
ﬁed what is needed to be able to detect malicious PowerShell commands,
here we discussed the different phases like preprocessing, normalization,
model creating and training.
