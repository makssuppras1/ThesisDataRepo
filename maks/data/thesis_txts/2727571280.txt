Abstract
The main objective of this thesis is to implement a monocular visual odometry al-
gorithm for the localization of unmanned aerial vehicles in confined spaces.
The
motivation for this project comes from the Inpectrone project which aims to com-
plete autonomous inspection of ballast tanks of ships using drones. The idea is to
use two monocular visual odometry methods, one for fast odometry estimates (SC-
SfmLearner) and one for slower but more accurate predictions for the correction of the
first one (Dense prediction transformer for depth estimation), and fuse them through
an Unscented Kalman Filter.
This thesis aims to create an algorithm that can run online on a drone and provide
high localization performance. The testing on KITTI shows that the slow method
cannot provide accurate enough estimates to correct the faster method while running
online.
The fast method provides estimates that fulfill the set requirements, but
without the correct scale. The fast method using the slower method for the scale
recovery provides better odometry estimates while also being able to run online. In
a faster system, the fusion provides good odometry results.
The methods cannot be validated on the drone data using the models that were
trained on the KITTI. The models that are used in the methods have to be retrained
with data collected from the drone. The testing shows that the models can learn how
to navigate in the ballast tanks, but more data are needed for the holistic integration
of the methods on the drone.

CHAPTER 1
Introduction
Robot localization is mandatory for autonomous mobile robots in order to complete
their tasks. This is typically achieved through Global Navigation Satellite Systems
(GNSS) because they provide reliable location data. However, in GNSS-denied areas
such as indoor spaces, GNSS signals may be obstructed or weakened making it diﬀicult
for robots to accurately determine their position.
A prime example of this can be seen in the Inspectrone project. This project aims
to do ballast tanks inspections using autonomous drones. Due to the nature of the
problem, localization of the drone using GNSS is not feasible because the satellite
signals cannot reach inside the tanks. Thus, other localization methods have to be
employed, such as visual odometry.
Visual Odometry (VO) is a necessary task for many real-world applications, such
as autonomous driving and robot navigation. This problem has been extensively stud-
ied in the past where geometric approaches are proposed. These rely on multi-view
geometry and require feature extraction from the camera frames that suffer because of
illumination differences, texture-less regions, blurred images and dynamic elements
in the environment. To overcome these hindrances, learning-based approaches are
proposed, which employ deep learning methods to improve performance.
The goal of this thesis project is to develop a Monocular Visual Odometry (MVO)
algorithm employing learning-based approaches, for localization of an autonomous
mobile robot. The MVO algorithm should be able to run accurately on real-world
scenarios. Furthermore, it should be able to run fast enough online on the hardware
of the vehicle that it is deployed on. The MVO algorithm is to be tested on KITTI
Vision benchmark suite [18], a dataset collected from ground vehicles that is used as a
universal benchmark of localization methods. Lastly, the developed MVO algorithm
is to be tested on data collected from a drone.
1.1
Objectives
The purpose of this thesis project is to develop an algorithm for monocular visual
odometry that can run online on a drone. The following objectives were identified:
• Perform research on state-of-the-art monocular visual odometry approaches
• Define the monocular visual odometry approach to be implemented

1.3
Related work
Two of the main problems in robot localization through visual odometry is the depth
estimation problem and the pose estimation problem.
They are usually resolved
by taking advantage of multiple views through cameras that have known relative
positions between them and their geometric constraints. Learning-based approaches
provide an alternative way to solve these problems.
These approaches can be classified in supervised and unsupervised. Supervised
methods require a large amount of labeled data to train the networks. This is problem-
atic because it is costly to collect the datasets and label them, so their applications are
limited. Unsupervised methods do not require labeled data to be trained. Usually
they are combinations of encoder-decoder architectures that utilize reconstruction

methods to synthesize views of other cameras using the relative position between
them and the original camera.
1.3.1
Deep learning for depth estimation
Supervised methods were the first to be introduced in pursuit of the solution of the
depth estimation problem. [11] proposed a multi-scale network for depth estimation
using a single image. The idea to use multi-scale features and structure of an image
was later used to increase the performance of depth estimation [57], [50], [28].
Since the Convolutional Neural Network (CNN) architecture cannot capture long
distance details well, [20] proposed a hybrid method that combined convolutional lay-
ers with Residual Network (ResNet) layers. The ResNet blocks are comprised from
Long Short-Term Memory (LSTM) units that can capture global feature representa-
tions. Later, LSTMs were used to capture spatial and temporal information to aid
the depth estimation process without pose supervision[9].
After the groundwork had been laid, scientists directed their attention to more
specific problems in depth estimation. [51] tackled the depth ambiguity problem by
using invariant surface normal and [22] proved the ambiguity between focal length and
monocular depth learning. Lastly, [56] proposed DFineNet, an end-to-end network
for depth map refinement of sparse and noisy data.
Although these supervised methods achieve good performance for the task at hand,
they require ground truth information that is diﬀicult to collect for many applications.
[16] was the first unsupervised method for depth estimation. It resembled an autoen-
coder architecture, where the encoder estimated the inverse depth map of the left
image from a stereo pair of images and the decoder reconstructed the left image by
synthesizing the right image using the predicted depth map.
Another method for synthesizing the target view uses Generative Adversarial Net-
work (GAN) architecture. [1] proposed MonoGAN, the first approach using a GAN
architecture for monocular depth estimation. The generator uses the image view to
produce its depth map and then the depth map to reconstruct the original image. The
discriminator then classifies the images as real or fake. In [10], the generator consists
of a DepthNet and a PoseNet used for depth and pose estimation respectively, which
are used to generate image pairs.
Geometry-based methods are the motivation and starting point for learning-based
approaches. [35] uses the epipolar constraints to enforce pixel correspondence. [30]
uses stereo image sequence trying to solve the scale ambiguity problem of monocular
visual odometry. [49] uses bilateral cyclic consistency in order to enforce left-right
disparities of stereo image pairs of images. [31] uses 2D photometric and 3D geometric
information trying to enforce 3D point cloud consistency across sequential frames.

1.3.2
Deep learning for pose estimation
PoseNet [27] was the first supervised method for the pose estimation problem. It
utilized an end-to-end CNN architecture to predict 6 degrees of freedom camera pose.
This was later extended in [26], which utilized multi-view geometry for the training
data to improve the performance of the network.
CNN and LSTM regressor were proposed to model the relationship of consecutive
images in [44]. The CNN is used for the feature extraction and the LSTM for the fea-
ture correlation between frames. MagicVO [24] uses a CNN and bidirectional LSTM
architecture, the first one to extract rich features from the images and the second
one to extract the geometric relations between them in order to get more accurate
results. ESPVO [47] extended this work by adding the uncertainty estimation of the
predicted poses.
On the other hand, unsupervised methods for pose estimation mostly use depth
and optical flow during the training process. [40] was the first to estimate both depth
and pose in an unsupervised manner. It uses optical flow to aid both estimation pro-
cesses. SfmNet [42] estimated depth, segmentation and camera pose from consecutive
images and D3VO [52] uses depth, pose and uncertainty estimation to increase the
performance.
Dynamic object and occluded areas in the images create problems in visual odom-
etry algorithms. [2] proposed an ephemerality mask that estimates the probability
that pixels belong to dynamic or static objects in the image. These masks are used
in inference time together with the depth maps as an input to a monocular visual
odometry pipeline that uses either sparse features or dense photometric matching.
Lastly, GeoNet [53] uses depth, optical flow and ego-motion from consecutive frames
in an isolated manner to extract geometrical relations and combine them in an image
reconstruction loss to predict dynamic and static objects.

2.1
Deep neural networks
In this section, basic deep neural networks that are used in the methodology will be
presented. This is to better understand the foundation blocks on which the more
complex networks were based on.
2.1.1
Long short-term memory
LSTM is a type of Recurrent Neural Network (RNN) architecture that is used to
capture dependencies over long sequences of data[19]. LSTMs were introduced to
address the vanishing gradient problem of traditional RNNs.
The vanishing gradient is a problem that occurs when training deep neural net-
works, especially multi-layer traditional feedforward neural networks and RNNs. It
is presented when the gradient of the loss function with respects to the weights of
the layers of the network approach zero as the gradients are propagated backwards
through the network. This is very apparent in RNNs where very small gradients can
be multiplied repeatedly through the recurrent connections.
In order to address this issue, each LSTM cell has three main components: the
input gate, the forget gate and the output gate. These regulate the flow of the infor-
mation through the network. The input gate determines which information is relevant
to store in memory. The forget gate decides whether to discard the information stored
in memory. Lastly, the output gate controls the flow of the retained information to
the next LSTM cell.
2.1.2
Residual network
ResNet is a deep CNN architecture used for computer vision tasks[21]. It was intro-
duced to alleviate the degradation problem that CNNs were facing.
==PAGE:18==
2.2 Algorithms
6
The degradation problem refers to the diminishing performance convolutional net-
works have as they become deeper. As the number of layers increases, the gradients
that are computed during the backpropagation step tend to approach zero. Conse-
quently, the updates received by the weights in the earlier layers are minimal, imped-
ing the learning of the network.
The key component of a ResNet is the inclusion of skip connections, or residual
connections. These enable the network to directly propagate the information from
one layer to a deeper one, bypassing a set of non linear transformation layers. The
identity mappings that are introduced learn the residuals of the input and output of
specific blocks, mitigating the aforementioned degradation problem.
Different types of ResNet exist. In this project ResNet18 and ResNet50 were used.
More about it on sections 3.1.3 and 3.2.1.4.
2.1.3
Transfomers
Transformers are a type of deep learning model architecture that is mainly used to
process sequential data[41]. They are mainly used in natural language processing
tasks such as machine translation, text generation, sentiment analysis and more.
The main feature of the transformers is the self-attention mechanism that captures
relationships between words in the input data. The self-attention mechanism lets each
word in the sequence attend to other words in order to determine their importance or
relevance with it. This mechanism encapsulates the conceptual dependency of each
word with the others. This is enhanced with the multi-head attention blocks which
use multiple heads to capture different linear projections of the embeddings. This
enables the model to learn simultaneously from different parts of the input.
Before inputting the sequential data in a transformer, the sequence has to be trans-
formed into numerical representations called word embeddings. These embeddings
hold information about all the words in the sequence, but not their positions. This is
why they are augmented with positional embeddings that convey their relative posi-
tion. The augmented embeddings pass through the multi-head attention blocks that
capture the relation between them, followed by fully connected feed-forward neural
networks that process each word position independently and captures their non linear
relationships.

Algorithms
In this section, algorithms that were used in the methodology will be presented.
2.2.1
Random sample consensus
Random Sample Consensus (RANSAC) is an iterative algorithm that can estimate
model parameters from noisy or outlier contaminated data[13]. It is commonly used
==PAGE:19==
2.2 Algorithms
7
in computer vision or geometric modeling of tasks, such as the MVO topic that this
thesis project is focusing on.
RANSAC operates under the assumption that part of a dataset can contain data
points that conform to a particular mathematical model. The steps the algorithm
follows are the following:
• Random sample selection: RANSAC randomly picks a ”minimal sample” of data
points that are required for determining the model parameters, also known as
”random sample”
• Model fitting: using this random sample, RANSAC fits a model to the data
points depending on the desired mathematical representation the underling
problem has
• Inlier classification: the remaining data points are classified as inliers or outliers
based on how compatible they are with the fitted model using a predefined error
metric
• Consensus set determination: the number of inliers, or ”consensus set”, that
agree with the estimated model and a predefined threshold is determined
• Iterative refinement: the aforementioned steps are repeated until a specific
amount of iterations have happened or the termination criterion is met and
the model with the largest consensus set is selected as the optimal model
• Model verification: after the iterations have concluded all the data points of
the consensus set of the optimal estimated model are used for further model
parameter refinement to increase the accuracy

The Unscented Kalman Filter (UKF)[45], is an algorithm estimate unknown values
based on the distributions of some various input data, disregarding their noise. It
is an extension of the Extended Kalman Filter (EKF)[12] that resolves the issue of
linearizing non-linear functions.
This issue is resolved through the unscented transformation. The unscented trans-
formation generates a set of points, that are called sigma points, that capture the
mean and covariance of the distribution of the input data. Then, this set is propa-

step. The predict step aims to predict the next state estimation based on the mean

the update step aims to refine the state estimate based it. The predict step is achieved

3.1.2
Feature detection and matching
The feature detection step consists of detecting potential important pixels in the
images. These pixels are called keypoints and should be easy to detect. This will aid
the feature matching algorithm to match them between consecutive frames.
As a feature detection algorithm, Features from Accelerated Segment Test (FAST)
[43] is used because of its low computation cost. It detects corners in an image choos-
ing potential corners, testing the intensity of the center pixel against all its adjacent
ones and marking them as potential keypoints if they have contrasting intensities.
These are further refined in order to reduce the false positives by applying additional
criteria to evaluate their quality.
Following the detection step, the keypoints between the consecutive frames are
compared using some similarity metrics [15].
The features are matched based on
the optical flow between each pair of consecutive frames using the Lucas-Kanade
method[7].
3.1.3
Dense prediction transformer for depth estimation
Vision transformers for dense prediction have been outperforming CNN-based state-
of-the-art approaches[36]. A vision transformer is used as the backbone of the DPT
model. The input image is chopped into pieces which are producing flattened repre-
sentations of features after they pass through a ResNet50 network. This embedding
is augmented with a positional embedding, the readout token. These embeddings are
passed through the sequence of the transformer layers. The output of the transformer
layers are reassembled into image like representations by the Reassemble blocks. The
Fusion blocks are then used to combine these representations and upscale them as
they pass through them. At last, the depth estimation is produced after the last
fusion block. This can also be seen on figure 3.1.

The method presented in 4.1 is computationally expensive due to the algorithms

3.2.2
PPnet for uncertainty estimation
PPnet captures the motion model of the robot that is being used.
It is a deep
learning model designed to predict the next pose and its uncertainty based on a
series of consecutive prior poses that are produced in equal time intervals. In this
context, this model is going to be used to produce the uncertainty of the next pose of
SC-SfmLearner model, based on the previous poses that were produced. It is assumed
that the previous poses have zero uncertainty. An illustration of PPNet can be seen
on 3.4.

PPNet receives 20 concatenated prior poses and it produces the next pose and its
uncertainty. This image was taken from [46].
3.2.2.1
Model overview
The problem is formulated as a multivariate time series regression problem. Knowing
a number of consecutive poses pi at times ti, where i ∈(1, 2, ..., n), n the number
of poses, ti −ti−1 = C and C a constant time interval, the next pose pn+1 and its
uncertainty is predicted. The number of prior poses used in this instance is n = 20.
To solve this problem, a combination of LSTM[19] followed by some linear layers
is used. The loss function is designed as per[17] in order to predict the uncertainty
of the next pose. Even though in [46] the uncertainty of each dimension is summed
to produce the uncertainty of the movement, in this approach each dimension uncer-
tainty is populating the diagonal of the 6 × 6 covariance matrix of the motion.

In this section, key components for the method are presented.
Pose centralization. The pose centralization is used to limit the error of the
input sequence. The central pose of the sequence is always a zero vector and the rest
are recomputed using the relative poses of the sequence. These new poses are the new
inputs of PPnet. This limits the travelled distance and the error to a fixed range.
Scale augmentation. The PPnet is prone to scale overfitting. This is solved by
multiplying the input poses of every sequence with a scale factor. This scale factor
changes between sequences in a predetermined range. This range is selected so that
the scale of the pose sequences produced by the SC-SfmLearner are the same as the
poses used to train PPnet.

predict the transformation between them.

CHAPTER 4
Evaluation
In the following chapter, the results of the experiments that were performed are be
presented. Firstly, the implemented methods were validated on the KITTI vision
benchmark suite, a popular benchmark for evaluation of computer vision tasks. Most
of the state-of-the-art approaches are already trained on this dataset, so validating
the methods with this and looking at their performance compared to the vanilla ap-
proaches is necessary. Then, the drone data collection procedure is explained followed
by the results of the methods on them.
All methods were turned into packages to be used in ROS[38]. This was done in
python. Furthermore, all the deep learning models were implemented in PyTorch[34].
A performance gain is to be expected if the proposed methods were implemented in
C++ and the models transformed in ONNX format but due to the time constraints
this was not done. More on 6.
4.1
KITTI benchmarks
In this section, all the validation tests and experiments that were done on the KITTI
vision benchmark suite are presented. The purpose of those is to validate that the
methods work reliably and fast enough to be integrated to the drone. Afterwards,
the integration investigation is shown together with the results that came from it. All
the tests were performed on a laptop with the specifications that are shown in A.1.
4.1.1
KITTI Vision benchmark suite
The KITTI vision benchmark suite provides a comprehensive dataset for the eval-
uation of a multitude of computer vision tasks[18]. The main focus of the dataset
is to aid the expansion of computer vision algorithms for autonomous driving. It
contains high resolution images from stereo cameras, 3D laser scans and GPS/IMU
measurements that were captured with moving ground vehicles in urban environ-
ments. Furthermore, it provides a standardized way of evaluating the performance
of the algorithms in their respective tasks.
For the purpose of this thesis project, the odometry dataset is used. It consists of
22 stereo video sequences, 11 with ground truth data, that can be used for the training
of the algorithms, and 11 that do not have ground truth data. Sequences 00−08 were
==PAGE:35==
4.1 KITTI benchmarks
23
used for training of the models and 09 for their evaluation. As this project focuses on
the MVO problem, only ”camera 2” data were used for the training and evaluation
of the methods.
For the evaluation of the algorithms, the KITTI vision benchmark suite is turned
into rosbags. This is done by using [29]. This is a repository that turns the dataset to
a rosbag, publishing at the correct times all the data, including the images that are
used for the implemented algorithms. Only sequence 09 is transformed into a rosbag
since it is the only one used for validation and evaluation of the methods.
The metrics used for the evaluation are fundamental for the comparison of different
MVO methods. The metrics used for the evaluation of the implemented methods, as
they are defined in the KITTI dataset, are listed below[15]:
• terr: average translation error, measured in percentage(%);
• rerr: average rotation error, measured in degrees per 100 meters(◦/100 m);
• ATE: absolute trajectory error, given in meters by the root mean squared error
(RMSE) between the estimated camera pose and the ground truth;
• RPE: relative pose error for rotation and translation, measured frame-to-frame
and given in degrees (◦) for the rotation and in meters (m) for the translation.
For the evaluation of the implemented methods, [54] is used. This is an evaluation
toolbox for the KITTI dataset. Given a specific sequence of the dataset, the ground
truth poses of that sequence and the predicted poses from a method for that sequence
it produces the evaluation metrics results. This toolbox also produces the translation
and rotation error graphs.
4.1.2
DPTVO
The primary purpose of the DPTVO algorithm is to provide accurate pose estimation
for the system it is applied to. The algorithms core objective is to be able to predict
the relative pose of the camera between two frames that are temporally distant to
each other. This capability is rather important because DPTVO operates in a slower
speed compared to other VO algorithms, requiring a longer execution time.
The
implementation for this thesis project was an adaptation from [14].
During extensive testing and evaluation, DPTVO achieved an average speed of 1.3
frames per second. This speed is relatively low compared to other VO algorithms, like
the SC-SfmLearner. This lack of speed is supposed to be compensated with accuracy
from DPTVO for sequences of frames that are temporally further apart.
On figure 4.1, the performance of DPTVO can be seen for different number of
skipped frames between every prediction. This tests were done in order to determine
firstly the limitations of the algorithm, secondly its robustness to temporal changes
during run time and lastly its usability for this specific dataset.

4.1.3
SC-SfmLearner
The primary purpose of SC-SfmLearner algorithm is to provide fast pose estimation
for the system it is applied to. The algorithms core objective is to be able to predict
the relative camera pose between two time frames based on the images taken at those
moments. It is supposed to provide a fast pose estimation even with small errors.
These errors are to be corrected with the slower and more accurate method, the
DPTVO algorithm. The implementation for this thesis project is an adaptation from
[4] and [25].
During extensive testing and evaluation, SC-SfmLearner manages to achieve an
average speed of 28.3 frames per second. This speed is high compared to other VO
algorithms. This means that this method can be used online on a drone providing
fast pose estimation, as it is intended. Something important that has to be noted is

4.1.4
DPTVO and SC-SfmLearner fusion with UKF
In this section, the results of the sensor fusion of DPTVO and SC-SfmLearner are
presented. The fusion of these two methods is achieved through the UKF algorithm
that was discussed on 2.2.2. The purpose of this fusion is to produce an accurate and
fast odometry estimation for the robot. Usually, the UKF node receives messages
from a multitude of sensors, including IMUs, GPS, Lidars and others, to produce an
accurate pose estimation. These can be integrated later but this analysis is out of
scope for this thesis project.
As discussed in 4.1.2, the DPTVO method is not accurate enough to run online
on the incoming images of the car. This is a major problem for the UKF algorithm,
since the prediction from the method will be very inaccurate and would basically
be disregarded due to the high covariance that would be produced from the relative
errors (3.1.6). For the purpose of this evaluation, it is assumed that the system that
the methods are tested on is much faster and can produce 10 predictions per second
for the DPTVO algorithm.
The SC-SfmLearner algorithm is integrated in the UKF algorithm, as it was dis-
cussed in 3.2. For the first 3 iterations, the predictions are compared with the pre-
dictions of the DPTVO algorithm in order to produce a correct scale estimation that
would later be used for the rest of the runtime of the algorithm. For the uncertainty
estimation, the PPNet is used as described in 3.2.2. This algorithm is fast enough to
==PAGE:42==
4.1 KITTI benchmarks
30
run online on the KITTI dataset, so even though we assume that the system is much
faster than it actually is, it does not impact this method.
Figure 4.7: Odometry results for the UKF algorithm.

truth data for sequence 09 of the KITTI dataset. As it can be seen, the algorithm
performs good with minimal drift on turns. In this approach, the odometry algorithm
corrects the drifting errors that can be observed on turns and still produce good pose
estimates without the need of further map correction.
The drift of the algorithm starts on frame 160 of sequence 09 of the KITTI dataset.

algorithms. Feature tracking failures, outliers in motion estimation and scene flow
ambiguity are just a few of the ways that the overexposed image can impede the
performance of the algorithm, that in turn produces poor results.
Another big issue with this image is that is contains dynamic features.

In this section, the training procedure and evaluation for SC-SfmLearner on the drone
image dataset is presented. The training was conducted as described in 3.2.1. The
evaluation of the model is presented as an ablation study on the depth estimations
that are predicted from DispResNet. Both DispResNet and PoseNet use the same en-
coder, so producing good depth estimations from DispResNet means that the PoseNet
produces good pose estimations. This is a given since the poses that PoseNet produces
is used to bring both depth maps on the same image pixel grid.
On figure 4.12 two example output depth maps from the pretrained DispResNet
are shown. Both examples show that the network cannot generalize on the scene
of the tank and the results it can produce are completely wrong. For this reason
it was chosen to retrain the model from the beginning instead of fine-tuning the
pretrained model. Also, in order to fine-tune the pretrained model, the drone image
dataset samples would have to be resized to the size of the KITTI dataset images
which would not work since the aspect ratio and the intrinsics of the two cameras are
completely different. Additionally, this transformation would add a computational
overhead to the procedure and make it slower.
In order to train SC-SfmLearner from the beginning, the model layer sizes had to
be adjusted to accommodate the size of the input images but also the requirements of
the models themselves. For that purpose the input images are resized from 720x1280
to 416x640. By resizing the images to this size, the aspect ratio is kept almost at the
same level, with a difference of 13.5%, and the requirements of the downscaling and
upscaling parts of the shared encoder and the decoders of the models, respectively, are
fulfilled. It is important to note that the intristics of the camera are scaled accordingly
so that the reprojections can be calculated correctly during the training process. The
change in the size of the decoder layers for DispResNet are shown of table 4.4. In the
PoseNet decoder all the channel factors were changed from 256 to 416.

4.2.4
Limitations
The data collection process proved diﬀicult. In order to collect the data, a real drone
had to be used in the replica ballast tank. The requirements to collect suﬀicient data
was to fly the drone in different speeds and orientations inside the ballast tank. This is
a challenging task because the drone creates turbulence in the tank during the flight,
which makes controlling it diﬀicult to the user. Thus, flying the drone precisely in
the tank requires experience and time, which was not possible during the conduction
of this thesis project.
The collected data that were used during the training of SC-SfmLearner were not
enough for the training or diverse. This was due to the aforementioned limitation in
the data collection part of the process. An attempt to resolve this issue is described
in 4.2.1, but as it can be inferred from 4.2.3.3, it was not enough.
Lastly, available time was a big issue during this thesis project. In order to train
the models, the HPC was used. Every experiment could run uninterrupted for only
24 hours. Models with ResNet18 backbones could train on TeslaV100 16GB GPU,
but models with ResNet50 backbones needed more VRAM, so they could only be
trained on nodes with TeslaV100 GPU with 32GB VRAM. This created a bottleneck
in the training process, since these experiments had to go into the HPC queues which,
depending on the time, can be very busy.

CHAPTER 5
Conclusion
The main objective of the thesis project was to implement a MVO localization system
utilizing geometry-based and learning-based techniques. This system had to be able
to run online on a drone utilizing only the onboard hardware.
The first method that was implemented in ROS was DPTVO. This method utilized
a dense prediction transformer network in order to predict the depth of a monocular
image, resolving the scale estimation shortcomings of the geometry based approaches.
By testing this method on the KITTI dataset, it was concluded that the performance
of the method was very good, achieving translation error of 1.496% and rotation
error 0.297 ◦/100m, better than all the other methods that were discussed. This
performance could not be maintained when the method was running online and it
produced underwhelming results. The method could achieve a speed of 1.3 frames
per second.
The second method that was implemented in ROS was SC-SfmLearner. This was
an end-to-end learning-based method that based on two consecutive frames produces
the relative pose of the camera. Testing of the method on the KITTI dataset showed
that it performs well, achieving translation error of 13.308% and rotation error 1.934
◦/100m with the custom scale factor and translation error of 10.829% and rotation
error 1.934 ◦/100m utilizing DPTVO for the scale recovery. This performance could
be maintained with a speed of 28.3 frames per second, which is fast enough to run
online on the drone.
The next part was to fuse the two methods with a UKF algorithm. UKF utilizes
the pose predictions and the uncertainties of these poses that the previous two meth-
ods produce and combines them, while simultaneously providing steady updates of
the poses based on the dynamics of the system. The performance of this algorithm
on the KITTI dataset was worse than DPTVO’s but better than SC-SfmLearner’s,
achieving translation error of 8.493% and rotation error 3.615 ◦/100m. These results
would only be possible in an ideal system that could run the DPTVO at at least 10
frames per second. Running this method online on the drone would not be possible
due to DPTVO’s speed.
The next objective was to transfer the methods to the drone and evaluate their
performance. The validation of the methods on the drone was unsuccessful, mainly
due to the networks that were trained on data collected from road vehicles.
Re-
training of SC-SfmLearner was attempted with the data that were collected from the
drone. The ablation study on the depth maps that the network produced showed
==PAGE:59==
5 Conclusion
47
that although the network could learn to create depth maps that represented reality,
they were not very detailed. A more diverse dataset is imperative in order to make
the method work on the drone.
To summarize, the project was successful. The system that was designed was
able to meet most of the requirements to run by itself and provide accurate localiza-
tion predictions utilizing only one camera. The method that was developed utilizing
DPTVO for the scale recovery of SC-SfmLearner shows promise as it can run online
and provide odometry estimates, but a new dataset and more training is necessary
to make this work on a drone reliably.

CHAPTER 6
Future Work
In this section, future work that can improve the methods will be discussed. The main
focus of this discussion is to identify limitations of the current implementations. The
subject of this thesis project was very broad and there was limited amount of time for
the research, implementation and changes that could be performed. The following
suggestions would improve the performance of the discussed methods and aid in
the development of more robust, accurate and fast approaches for the localization
problem.
6.1
Uncertainty estimation techniques refinement
The first major problem with the implementations that were discussed in 3 was the
uncertainty estimation methods that were used for both approaches. These two dif-
ferent approaches had different information that could be used from the uncertainty
estimation modules in order to produce the relative uncertainty of the motion.
In DPTVO, a closed form uncertainty estimation technique was used. The reason
for this was that closed form uncertainty estimation techniques are faster than itera-
tive ones. The implemented method produces fast results but it is majorly affected
from outliers that are produced during the feature detection and the depth estima-
tion steps. Furthermore, since the distribution of the errors is very much dependant
on the sampling number, which is not possible in this context, the uncertainty pre-
dictions can be very poor. A better approach would be to either adapt an iterative
approach for the uncertainty estimation of the motion or sample the motions during
the RANSAC step to produce a distribution from a bigger dataset that would be
invariant to outliers.
In SC-SfmLearner, PPNet was used to produce the uncertainty of the motion. In
[46], PPnet uses 20 prior poses in order to produce the next pose and its uncertainty.
This effectively means that the produced uncertainty, even though it depends on the
prior poses and follows the motion model of the robot, is not representative of the last
provided pose in the input sequence. Other techniques for predicting the uncertainty,
like leveraging the aleatoric uncertainty in order to calculate the photometric and
depth uncertainties [52] would be more appropriate.

6.2
Change UKF configuration to relative pose input
The UKF algorithm uses the provided poses and their uncertainties in order to correct
its predicted pose and produce poses utilizing the dynamics of the system. The input
poses that it can accept are absolute or relative. In the former case, which is the one
used in this thesis project, the implemented method has to calculate the relative pose
between two frames and the uncertainty of this motion, calculate the absolute pose
based on the outputs of the UKF algorithm and calculate the uncertainty based on
the uncertainty of the previous absolute pose and the relative pose of the motion, as
described in [6]. This uncertainty propagation can also be done internally in the UKF
implementation that was used [33] if the algorithm is provided with relative poses.
Due to the time limitations, this could not be done for this thesis project.
6.3
Speed performance enhancement
One critical aspect of the proposed system is to be able to run fast on the online plat-
form it was designed for. In order to increase the speed, the code should be transferred
to C++. C++ is a high performance, cross platform, object oriented programming
language that is faster than python. It is a compiled language, which means that
the code is translated into machine code before execution. This lets the code to be
optimized for the hardware it will run on, getting a speed boost in the process. Ad-
ditionally, since C++ is a statically-typed language, its compiler performs eﬀicient
memory management, whereas python is a dynamically-typed language, which comes
with some performance overhead.
Another way that the speed of the methods could be increased is by transferring
the models from the PyTorch framework to the ONNX framework. ONNX leverages
highly optimized backend libraries, like NVidia’s CUDA, that are written in C++
and are designed to utilize hardware features like GPU parallelism to accelerate com-
putations. On the other hand, PyTorch is a higher-level framework written on top of
C++ that does not exploit these low-level optimizations. Furthermore, ONNX was
designed to prioritize model exchange between different networks and performance,
while PyTorch was designed do be flexible, easy to use and experiment with. This
makes ONNX a better choice for doing model inference on an online platform.
6.4
Create a better dataset
Data engineering is a very important part of training a model. Training a model with
a small, unbalanced, not well-prepared or unsuited dataset for the task at hand will
only lead to models that do not perform well. Creating a better dataset is essential
for this thesis project.
The recommendation would be to create a replica of the
environment in a virtual environment, such as blender[8], and collect data from there.
This would eliminate the limitations in regards to battery life of the drone which
==PAGE:62==

50
leads to small flights and controlling the drone during the flights which is a diﬀicult
task. It would also aid in the collection of diverse data since flying the virtual drone
with different speeds and angles close and far from the walls would be easier, without
the fear of crushing and making damages to the real tank or the drone.
6.5
Re-train all models for better performance
The results that were presented in 4.2 were underwhelming. This was mainly due to
the fact that the images from the drone are much different than the ones from KITTI
and the models could not generalize to them. A main task to make this methods work
on the drone would be to retrain the models on a dataset created from the drone. An
attempt to retrain the was presented in 4.2.3 but it was not possible to produce a
high performance model, mainly due to the dataset that was used. So this is an
imperative step to make the methods work on the drone, but can only be completed
after creating a more suited dataset for this task.
