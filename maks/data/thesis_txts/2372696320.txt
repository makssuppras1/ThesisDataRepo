ABSTRACT 
With the advances in modern microcontroller technology, making them more powerful and cheaper, field 
oriented or vector control has become a widely spread solution for providing speed control of induction 
machines. It provides a higher accuracy and higher control flexibility compared to the Volts per Hertz speed 
control at the cost of requiring more sophisticated control algorithm which come with the according 
hardware requirements. 
The history of field oriented control already ranges back to the 1970s and has experienced a continuous 
development since then. The newest advances are mostly related to the sensorless control not requiring any 
speed measurement by reconstructing all necessary values from measured electrical properties. Using field 
oriented control over V-f control allows to adjust the flux level of the machine according to the load and 
thereby optimizing the efficiency depending on the operating point. 
In this project both a sensorless field oriented control algorithm and an encoder based control is developed 
and implemented on a test system using a NI cRIO system as controller. The developed control proves to 
be robust and accurate even in the presence of external disturbances as e.g. measurement errors.  
The initial scope of using the sensorless control to perform efficiency optimization could not be fulfilled 
due to issues encountered during the implementation of the control in the real system. The developed control 
algorithm however provides a good baseline for further projects focusing on the efficiency improvements.

INTRODUCTION 
Especially, but not only, since electrical cars have become more prominent and experienced a wider use the 
interest in reliable and accurate sensorless control solutions has increased. Since the beginning operation in 
the low speed region has always posed a problem for all solutions. Nowadays sensorless control algorithms 
however allow operation of the machine in almost the full speed range with a maximum speed error which 
is in the range of 1%. 
Most of the applications are relying on field oriented control where the control system is aligned to the 
magnetic flux of the machine allowing an independent control of flux and torque. Both, machine speed and 
flux position are estimated using measurements of stator currents and voltages and are than calculated using 
the equivalent circuit of the machine. 
One requirement for the accurate estimation is therefore the exact knowledge of the machine equivalent 
parameters which poses a problem since especially the resistances for instance depend on temperature and 
are therefore subject to change during the operation. To overcome this problem a variety of adaptive 
observer models have been developed which provide an online parameter correction during the operation. 
One particular advantage of field oriented control is that it allows an independent adjustment of the flux 
magnitude which allows to optimize the losses in the machine dependent of the operating point which are 
again dependent on the machine parameters.  
The initial idea behind this project was to provide control algorithm that combines sensorless control with 
the efficiency optimization. Since the adaptive observer model contains real-time estimates of machine 
parameters it can be used as an input to the calculations regarding loss reduction as well. 
The system is first tested using indirect field oriented control using an incremental speed encoder and 
current measurements to obtain flux and speed information. A sliding mode controller is used for the flux 
observer in the sensorless control method. One main advantage of the sliding mode control is that it is 
robust to external disturbances and does not require much optimization regarding controller gains. The first 
version implemented is not yet adaptive but adaptability can easily be obtained by extending the current 
system by a second observer. 
Due to issues encountered during the implementation of both indirect field oriented control and sensorless 
control the time was not sufficient to implement both and adaptive algorithm as well as an efficiency 
optimization. 
The main control system is programmed in LabVIEW and running on a NI cRIO device which only 
provides limited debugging possibilities and does not have an easy solution to emulate the control in 
connection to a model of the real machine. All controls are therefore tested in Simulink using a 
representation of the real system, including converter and induction machine. This methodology assures 
that the basic control concept is working before applying it to the real system. Additionally, occurring errors 
can be reconstructed which allows to identify the source and find possible solutions. 
The Simulink model however will not be specifically explained in this report since it basically contains the 
same mathematical equations as the LabVIEW system.

2.4 
DIRECT FIELD ORIENTED CONTROL 
In contrast to IFOC, direct field oriented control (DFO) uses either a direct measurement of the air-gap flux 
or measurements expressed in the stationary αβ-reference frame to obtain information about the 
instantaneous position of the rotor flux. Unlike the estimation in the dq-reference frame used in IFOC there 
is no need of the knowledge of initial position.  
Cost and required maintenance make a direct measurement of the flux unattractive and the most common 
methods use the measurements of stator current and voltages to estimate the flux position. The most 
common observers are the voltage model observer [1], current model observer [2] both introduced in the 
1970s by Blaschke and Becker respectively.  
Newer approaches include, apart from the flux observer, speed observers enabling the so called speed 
sensorless control. Some of which are based on Kalman filters [3] and Sliding Mode Observers [4],[5]. 
A general problem with all flux observer models is that they highly depend on an accurate estimation of the 
machine parameters which are also subject to variation during machine operation. Each error in the machine 
parameters directly translates to an error in the flux estimation which depending on error magnitude and 
parameter can have severe consequences on the performance of the control. 
Multiple adaptive schemes have been developed to cope with the problem of parameter variation. They rely 
on an online parameter adaptation which corrects the error introduced by the variation. Such schemes 
include the adaptive sliding mode also described e.g. in [5] and model reference adaption scheme (MRAS) 
as e.g. in [6]. 
In this project a sliding mode observer will be used to estimate flux position as well as the mechanical speed 
of the machine.

Where 𝑢0 is the gain of the sliding function. The behavior of a system containing a sliding mode controller 
can be divided into two distinguished phases: The reaching phase, where the sliding function tries to reach 
the sliding surface, and the actual sliding mode which is obtained once the system reached the sliding

The sliding function is basically a switching function and as such it is highly depending on the switching 
frequency. An infinite switching frequency guarantees that the system remains on the sliding surface at all 
times. In real implementation the switching frequency is, however, limited and between two consecutive 
switches the system leaves the sliding surface causing an error and a resulting control action. This control 
action is reflected as high frequency noise in the sliding function and increases at lower switching 
frequencies and with higher gains.

CRIO IMPLEMENTATION OF THE CONTROL 
A typical cRIO application comprises three different layers: 
The first part of the program is executed on the FPGA and typically contains the time critical code and code 
addressing the in- and outputs of the cRIO module. 
The second part is executed on the real-time system which is also part of the cRIO module itself and is 
connected internally to the FPGA using a fast bus connection. It can be used to collect data acquired by the 
FPGA to perform further analysis or just serve as a display and communication device. 
The third layer is a host or development computer connected to the cRIO by Ethernet. It can be used to 
store data which cannot be stored on the RT due to its limited capacity. When the cRIO is not equipped 
with any input or output devices it can also be used to replace the RT almost completely using the RT solely 
as an interconnector that passes through data between the FPGA and the host. 
 
 
Figure 4.1  
Structure of the NI cRIO 
The main control system is placed on the FPGA. Here measurements are acquired using the C series 
modules and processed by the various control loops based on the equations presented in section 0. When 
the appropriate output values are calculated they are passed on to the C series modules providing the 
respective output functionality. Selected measurements and process variables are collected and transferred 
to the RT for the purpose of observation and data logging. 
The RT processor reads the data provided by the FPGA and transfers it to the host computer and on the 
other hand receives data from the host. The data received from the host consists of all kind of user 
interactions, i.e. settings regarding the control system, control set points as well as system parameters. 
The user interface is located on the host computer. Here, all settings can be adjusted using a graphical 
interface and the system state can be observed using time plots of the measurements and control variables. 
Additionally, it provides the functionality of data logging by storing selected measurements in a text file 
for later analysis. 
All applications are created in LabVIEW which is a graphical programming environment providing 
basically all functionalities also known from text based programming languages as e.g. C. Portions of the 
code can be encapsulated in so called Virtual Instruments (VI) with specific in- and outputs which makes 
it reusable within a single application or cross application. A limited number of prebuilt VIs is already 
present in the standard LabVIEW environment and its extensions.

4.2.2 PWM 
The PWM loop is a single cycle timed loop using the pre-defined PWM VI from the motion control library 
of LabVIEW. It determines the switching pulses for the converter based on the duty cycle in each phase 
provided as an input to the VI. The PWM frequency is defined in periods per tick of the clock rate specified 
for the single-cycle timed loop. The switching signal is transferred to respective channel of the NI 9401 
module. 
 
 
Figure 4.3 PWM Loop 
Since the duty cycle of the PWM is aligned to the center of the PWM period it is guaranteed that at this 
point no switching occurs in any of the phases. That fact is made us of by providing a trigger signal that is 
set in the middle of the period and remains high for the rest of the period and which indicates that current 
can be measured without containing any switching noise. This signal is used to trigger the main loop which 
performs the measurement and all related calculations. It also assures that the main loop is automatically 
executed with the PWM frequency.

4.2.7 DEAD TIME COMPENSATION 
In the course of investigating the DC drift of the flux estimation it was observed that the command voltage 
output of the real system was by times almost by a factor of 2 larger in the real system compared to that of 
the Simulink model. The deviation is significantly larger in the low voltage, i.e. low speed, region. 
Comparing the command duty cycle, i.e. the output signals from the NI 9401 digital output module with 
the measured line to line voltage converter dead time is identified as the cause. It is observed that it contains 
a constant delay of 5 microseconds which is increased or decreased by an average of 2.4 microseconds 
depending on the load current direction. The effect can be tracked down to the influence of dead time which 
in no load manifests as an additional delay not affecting the output pulse width but in the presence of a load 
current prolongs or shortens the voltage pulse. This is caused by the instantaneous commutation of the load 
current to the diode which is antiparallel to the IGBT whose turn on is delayed by the dead time if the other 
IGBT in the converter leg is turned off. The dead time is then eliminated since current is already present 
although the IGBT is still turned off. For a positive load current this happens when the positive terminal is 
switched off, i.e. the pulse is shortened, and in case of a negative load current, at the turn off of the negative 
terminal, i.e. the pulse is prolonged. 
The effect is compensated by adding a compensation signal for each phase to the output of the space vector 
modulation 
Δ𝐷= 𝐷𝑇⋅ 𝑓𝑠𝑤⋅𝑠𝑖𝑔𝑛 (𝐼)   
(4.22) 
Where D is the duty cycle, DT is the dead time of the converter, fsw the switching frequency and I the load 
current in the respective phase. 
Due to the discrete sampling time it can happen that the load current crosses zero in between two 
consecutive samples and the application of the correction factor would actually have the opposite effect

The exchange of user inputs between the host and the RT is achieved using a messaging system. The data 
type of the stream is defined as a cluster containing two elements: The first element of the cluster contains

The message is used as a case selector containing the appropriate action connected to the message. The 
second element contains additional information as e.g. values. An example of such a message is a user input 
to the set point value of the control. The message would indicate that the user has changed the set point 
while the second element contains the new value of the input. When the appropriate case is selected the 
new value is obtained from the network stream and written to the associated variable. A list of all messages

5.1.1 OPTIMIZED SPEED MEASUREMENT ALGORITHM 
Initially the speed information is obtained using a loop with a sample time of 50 milliseconds and deriving 
the speed from the number of counts during that period. This is later replaced by a moving average filter 
included in the main loop and a resulting sampling time of 50 microseconds averaging the speed over 128 
samples. 
Increasing the number of samples allows a faster response of the system to dynamic changes of the speed 
and the speed controller gains can be increased without becoming unstable. Furthermore, it allows to add a 
LPF to the speed signal and still providing a faster response than the initial implementation which reduces 
oscillations in the steady state speed signal.

5.2.2 FLUX OBSERVER LOOP TIMING 
Another issue encountered during the implementation of the sensorless control is related to the performance 
of the cRIO. After implementing the sliding mode observer model in Simulink a clear correlation between 
the sample time and the performance of the control is observed. The faster the execution rate of the observer 
the better results. As mentioned in section 2.5 most of the issues connected to sliding mode control in 
general arise in discrete time where the control is discontinuous, ergo, the closer the control is to being 
continuous the better. This is quite obvious considering that at a constant number of iterations the control 
needs to converge the time elapsed is much shorter if the loop is executed at a higher rate. This impacts the 
equivalent control as the reaching phase becomes more dominant even though the signal passes a LPF. 
Increasing the gain of the control can solve this issue but causes a lot more chattering and lowering the 
cutoff frequency to filter the additional chattering prevents portions of the low frequency components from 
passing through again worsening the accuracy. 
Following these contemplations, a loop timing of 5 microseconds was initially chosen for the sliding mode 
observer. The result of the speed and flux estimation show that the flux estimation in Figure 5.15 yields a 
too low magnitude which consequently reflects in a too high speed estimation in Figure 5.14. It was 
impossible to reproduce the error on both Simulink and by debugging the VI on a development computer. 
Applying equal signals to the Simulink model of the observer and the LabVIEW VI always resulted in 
results with only minor deviations. 
Running a test with the same settings but executing the VI on the FPGA finally revealed that the error is 
connected to the execution on the FPGA. Since a difference in expected loop execution rate and the real 
rate affects the gains of both filters and integrators the execution rate is reduced to half of its initial value 
which provides correct results as demonstrated in section 5.2.1.

CONCLUSION 
A sliding mode sensorless control algorithm for induction machine is developed and tested in a Simulink 
simulation environment and in a real laboratory set up. The performance of the sensorless control is 
compared to the encoder based indirect field oriented control and proves to be adequate in terms of precision 
and speed.  
Implementation of the control algorithm in the laboratory environment reveals that although the theory is 
straight forward the implementation in a real system is quite a challenging task. The relationship between 
control performance and the accuracy of the estimated machine parameters is well known and has been 
dealt with by developing adaptive observers that are adjusted during operation but the results of this project 
have shown that the impact of other system parameters as converter characteristics and measurement errors 
can a significantly larger impact. The parameter estimation error is for instance negligible compared to the 
effect of converter dead time on the voltage estimation in the low speed region. 
Implementing compensation mechanisms and using improved integration methods however provides a 
control system that is both accurate and stable even in the presence of external disturbances. The control is 
tested up to a minimum speed of 5 % of the rated value where the steady state estimation error amounts to 
3%. A performance that can be considered accurate enough for most applications. Due to the limited time 
available for control parameter optimization it can be expected that the accuracy can still be improved 
without any substantial changes to the basic concept. 
Due to the issues encountered during the implementation of the sensorless control on the laboratory system 
an efficiency optimization based on the sensorless control could not be performed. The current state of the 
system however provides a solid base on which further projects can be built on. The system can easily be 
extended by an adaptive observer scheme by adding a second sliding mode current observer for parameter 
adaptation and an efficiency optimization algorithm can be developed that is based on the estimated 
parameters. Prerequisite is, however, the optimization of the code in terms of hardware requirements. 
In the current state the control is using more than 90% of the FPGA resources of the powerful NI cRIO 
9033 which shows the computational demand of the numerous calculations needed for the algorithm. The 
use of FPGA resources could clearly be optimized by converting the control to be fully based on fixed point 
numbers instead of floating point values.
