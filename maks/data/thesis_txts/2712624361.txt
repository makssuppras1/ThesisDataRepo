Abstract
Recently, mathematical running time analyses of evolutionary algorithms have investi­
gated different strategies for overcoming local optima on certain, carefully defined bench­
mark scenarios. This paper conducts a comprehensive experimental analysis guided by
state of the art theoretical results in order to understand and evaluate further these strate­
gies. The evolutionary algorithms and fitness functions studied in this thesis were selected
from the latest research papers in the field, and implemented in a versatile framework by
reusing pseudo­code and equations. All the data and results presented here were ob­
tained through this software that was programmed from scratch in Python. The focus
was made on the optimization of functions with fitness gaps of different nature, and evo­
lutionary algorithms derived from the RLS and (1+1) EA using stagnation­detection and
self­adjusting mechanisms, while comparing them to the cGA, an estimation of distribu­
tion algorithm. Advantages and disadvantages for each of these algorithms have been
discussed, and a great overview of their behavior and parametrization has been reached.
Finally, these results were used to give a first impression of the expected running times
of these algorithms on the MaxSAT problem.

Introduction
Since the beginning of computer science, algorithms have been used to solve increasingly
complex problems that humans were not able to unravel anymore. One type of problem
that caught the eye of researchers and that is still intensively studied is the understanding
of black­box systems. A black­box system is defined as a system on which there is no/little
information on its behavior, and that, given an input, will return an output. Testing is then
the only way to understand better its functioning. Even though this kind of problem is
mainly used in the field of computer science, it can also be applied to many different
areas. The main challenge when optimizing black­box systems is to do it in efficient time
and space complexity.
This thesis will focus on the optimization of problems called fitness functions. The particu­
larity of these functions is that the image of an element gives an estimation of its distance
from an optimal solution, that can be used for guiding the search of an algorithm. In this
report, a solution will be considered as an element that maximizes the fitness function. In
order to take advantage of this property, evolutionary algorithms (EAs) have been devel­
oped inspired by biological evolution , using mutation and crossover techniques.
The evolutionary algorithms that will be studied in this thesis were designed for optimizing
fitness functions by taking bit strings as input and returning an integer. It will be seen
that these algorithms are using little space memory, and therefore their analysis will be
based on their running times. The running times will be measured in number of fitness
calls as this is often the most onerous task, and will depend on n, the variable used for the
problem size (i.e. length of the bit string) throughout the report. In this report, the optimum
will be known beforehand for each fitness functions, in order to stop the algorithms when
it is reached and get the exact number of fitness calls made to obtain it. In practice, the
optimum of a fitness function might be unknown, and a stopping criteria should be defined
for the algorithm to end, which might only return an approximation of the solution.
At first, evolutionary algorithms were designed in a way so that they could improve through
the gradients of a fitness function until reaching the optimum. Many studies have then
been conducted to estimate the running times of evolutionary algorithms on specific func­
tions, which is made more complicated due to the fact that they are non­deterministic.
Even though it has been proven that these algorithms perform well on some simple func­
tions, limitations have been detected for fitness functions with local optima. This led re­
searchers to update the previously used algorithms by adding new mechanisms enabling
them to overcome fitness gaps.
The objective of this thesis will then be to expand the latest work on evolutionary algo­
rithms and their strategies for leaving local optima. At first, an overview of the state of the
art was done for selecting a variety of algorithms and interesting fitness functions for con­
ducting more analysis. The selected elements have then been implemented in a versatile
framework with a dedicated interface allowing a visualisation of the solution process and
the gathering of statistics on the runs of the algorithm. This framework has then been
used in order to test the implemented algorithms on benchmark fitness functions, and
has then been extended to MaxSAT, a combinatorial optimization problem involving local
optima. Finally, all the results obtained throughout the analyses will be used to argue the
possible advantages and disadvantages of each evolutionary algorithm.

Literature Review
In recent years, researchers have been interested in proving bounds on the running time
of evolutionary algorithms on different problems. This is highly challenging because of
their stochastic behavior, and has therefore led to a large number of studies. In order
to analyse the performances of this particular type of algorithms, many researchers in
the field have been using simplified problems such as finding the optimum of a function
taking bit strings of a fixed size as an input. At first, algorithms such as the Randomised
Local Search (RLS) and the (1+1) EA were designed to find the optimum of linear func­
tions. The advantage of such functions is that they have gradients towards the optimum,
a property that is used by these algorithms to find the solution. Though a polynomial time
of O(n log(n)) [3] can be expected to solve these simple functions instead of an expo­
nential time of O(2n) when checking all the possibilities, problems are often not as simple
as the ones stated before. In general, functions can have one or several local optima
which makes the two previous algorithms often unsuitable for finding the optimum. From
this point, modifications and improvements have been sought to overcome these gaps
between optima by adding self­adjustment and/or stagnation detection mechanisms to
the previous algorithms. Moreover, other leads and ideas have been explored to create
original evolutionary algorithms that could adapt better in many different scenarios.
This literature review was started by looking at the recent papers from Carsten Witt whom
has been conducting research in the field for more than 20 years, and it was followed
by going through the references cited in them as well as the papers from his peers. It
was then extended by searching for recent and popular articles in the field, that could add
insightful elements to it. The aim of this literature review is to gather relevant evolutionary
algorithms with interesting mechanisms to overcome local optima, and that could show
strong adaptability. At the same time, relevant fitness functions will be highlighted, as
well as the latest and tightest bounds on running times found in the literature for further
analysis. This collected data will be the starting point of the research made in this paper,
in order to have a standard benchmark to compare the experimental runtimes with, and for
selecting appropriate values for the parameters of the evolutionary algorithms and fitness
functions. The goal is to conduct experimentation to see if these promising evolutionary
algorithms and mechanisms could overcome local optima in an efficient and reliable way,
by evaluating their ability to cope with different challenges.

2.1.1
Running times of the RLS and the (1+1) EA on OneMax
The RLS [14] and the (1+1) EA [12] are two mutation­based algorithms, meaning they
generate new bit strings by modifying previous ones. The bit string that is transformed
will be called the parent, and the newly generated bit strings will be called offspring. Bit
strings will also be called individuals, and the population will then be the individuals stored
in the algorithm. These two evolutionary algorithms cited previously are similar, as they
both start by generating a random bit string which will be the parent and will be used to
create one offspring at every iteration. If this offspring is better than the parent, it replaces
the parent, and the process continues. The difference between these two evolutionary
algorithms is how the parent is transformed to generate the offspring.
The pseudo­code for the RLS is described in Algorithm 1 [14], and the offspring are gen­
erated by flipping s bits from the parent, s being the strength of the algorithm.

optimizing OneMax and other linear fitness functions, this choice gives many advantages.

impossible to flip the right number of bits to compute the only 1 bit string. If that happens,
the solution is then not reachable anymore, and this scenario can only be avoided if s = 1.

2.1.2
Main derivations of the (1+1) EA
In the previous section, the mutation­based (1+1) EA has been studied, in which the
population is only composed of one element and which only generates one bit string at a
time from it. The first modification that can be made on the (1+1) EA is to increase the
population size from 1 to µ, which gives the (µ+1) EA [1]. The pseudo­code for the (µ+1)
EA can be found in Algorithm 3 [1]. It can be noted that, as defined in [1], the strength of
the (µ+1) EA has automatically been set to 1. As the population has been increased, a
parent selection operator needs to be defined in order to determine which individuals from
the population will generate the next offspring. The generated offspring is then replacing
the worst individual in the population.
In this report, the two following parent selection operators will be studied:
• Uniform selection: Choose the parent uniformly at random from the population [1].
• Inverse K­tournament: Select K individuals uniformly at random, and choose the
loser (i.e. worst) of the tournament as parent [1].
The idea behind the (µ+1) EA is to expand the population in order to increase the diversity
of the individuals. The goal is to allow the algorithm to explore several paths in the fitness
function at the same time, in case some might only lead to local optima. The parent

2.1.3
Estimation of distribution algorithms
All the algorithms seen so far were mutation­based algorithms. Other kinds of algorithms
have been studied, such as estimation­of­distribution algorithms (EDAs). These algo­
rithms use a combination of implicit mutation and crossover. Crossover is the action of
merging two different individuals to create a new one. One of these algorithms is the
compact Genetic Algorithm (cGA) defined in Algorithm 6.
For the cGA, an estimation of the probability of each bit to be equal to 0 or 1 in the global
optimum is stored. All the probabilities are set to 1/2 at the beginning and are used to
generate 2 offspring (mutation). These two offspring will then be compared between each
other and the probabilities will be updated depending on the results (crossover).The prob­
abilities are increased/decreased by adding/subtracting 1/K.
The value of the parameter K has a great impact on the running time of the algorithm
on OneMax but also for other fitness functions. First, if the value of K is too small, the
probabilities might be updated too rapidly, creating wrong estimations. This could cause
the algorithm to get stuck in local optima if this were to happen for several bits causing
longer running times. That phenomenon will be called genetic drift. Secondly, if K is too
big, the probabilities take more time to get updated and longer running times are also
expected. Then, the value of K needs to be in a correct range in order to optimize the

2.2.3
The (µ +1) EA and its use of selective pressure
As explained in Subsection 2.1.2, the (µ+1) EA is increasing the size of the population
o augment its diversity. The idea behind this algorithm is to be able to explore several
gradients of a fitness function at the same time. In fact, it was seen that for the OneMax
fitness function, this algorithm would not be as efficient as the (1+1) EA as more than one
element has to climb the gradient. However, in more general problems, several gradients
leading to different local optima can be expected. In that case, the (1+1) EA will be more
likely to get stuck in a local optimum while the (µ+1) EA will have more chances to explore
several optima at the same time, implying it has more chances to reach the global one.
Yet, the behavior of the (µ + 1) EA depends on the parent selection operator that will
influence the selective pressure of the algorithm. With a highly selective operator, the
best elements of the population will be more likely to be selected for generating offspring
that will take over the rest of the population. On the contrary, using an operator with lower
selective pressure will allow less suited individuals to generate offspring as well, so that
diversity is kept longer, thus increasing the time for each individual to improve. That is
why the choice of the selection operator is fairly important regarding the expected running
time of the algorithm.

2.3
The complexity of analyzing evolutionary algorithms
It has been seen that many of these evolutionary algorithms have been analyzed in a
theoretical way on the OneMax fitness function. Indeed, consequent research has been
made on OneMax in order to get the tightest bound possible on the expected running
times of each of the algorithms. This gives a good overview of how each of the algorithms
is able to go up through a gradient, which is required for optimizing function. Furthermore,
studies have been extended to more complex fitness functions containing fitness gaps.
As for the OneMax, analyses have been made on different jump functions with fitness
gaps of different nature, due to their different start position, length or numbers. Even
though these analyses are also very useful for understanding better the behavior of each
algorithm, it also starts showing some limitations.
The first issue encountered is that evolutionary algorithms keep being improved, so that
Implementation and Evaluation of Strategies for Escaping Local Optima with Evolutionary Algorithms
19
==PAGE:28==
they can be efficient in many different scenarios. It has been seen for the (1+1) EA where
new mechanisms can be added such as stagnation­detection or/and self­adjusting, as well
as increasing the population or the number of generated offspring. All these modifications
lead to numerous updates of the bounds on the expected running times already found,
which might require heavy mathematical analysis. Furthermore, the addition of many dif­
ferent mechanisms, also makes the analysis way more complex, as it has been seen for
the SASD­(1,λ) EA where the algorithm can switch in­between two different phases. In
order to cope with that, some bounds on the expected running times need to be condi­
tioned to a certain event happening, not analyzing the full scope of the problem and how
some scenarios can affect the overall running times.
Another issue that is met is the difficulty of getting precise bounds on the expected running
times of the algorithms on complex functions. Due to the non­determinism characteristic
of evolutionary algorithms, it is not possible to give an exact number of fitness calls for
optimizing a selected fitness function. This property is then forcing to work on the expected
running times, which are often estimated using complex mathematical methods requiring
the use of asymptotic notations (o and O). In theory, these bounds are very practical to
see how the algorithm will perform when the problem size is increased. Nevertheless, in
practice, these bounds can lead to huge differences in running times. Indeed, depending
on the coefficient in front of the bound, running times could vary from hours to weeks.
Furthermore, it could also be interesting to have an insight into the behavior for smaller
values of n as this could be expected in practical problems.
In this chapter, it has also been seen that the parameter landscape of evolutionary algo­
rithms and fitness functions is very complex. Each of the algorithms seen previously is
dependent on the choice of some parameters beforehand. In that regard, theoretical work
has been very useful in order to give a magnitude for the optimal choices. Nevertheless,
these parameter choices can also be influenced by the fitness functions the algorithms
are studied on. Moreover, fitness functions are also dependent on the choice of their pa­
rameters, increasing or decreasing their complexity. All in all, the number of combinations
and possibilities is broad and it seems complicated to perform heavy mathematical proofs
on all of them.
However, the theorems demonstrated in the research in the field are still very useful. Even
though they are often computed on a specific function, the analyses can often be reused
on more complex and more practical problems. This allows to go from results on the
optimization of simple fitness functions to more complex problems. Many results here
could be applied on NP­Complete problems such as the MaxSAT, which are difficult for
both computers and humans to solve.
2.4
Summary of the literature review
To sum up, this chapter has given a broad overview of the state of the art of evolutionary
algorithms and the different techniques for leaving local optima. Furthermore, a selection
of fitness functions with different particularities has been presented to give an overview of
the challenges an evolutionary algorithm might face when optimizing a problem. On top
of that, for the selected fitness functions, the latest and tightest bounds on the expected
running times of these algorithms found in the literature were reported.
At first, simple algorithms such as the RLS and the (1+1) EA were defined, and their
capability of optimizing the OneMax function has been explained. It has been shown that
the expected running time for both these algorithms was Θ(n ln n). This has then been
set as a benchmark expected running time for other algorithms to optimize OneMax. It is

in a reasonable time. This has been one of the focus when choosing appropriate values
for the parameters of an algorithm, such as the population size or the number of offspring
generated. Nevertheless, even though many of the studied algorithms meet this condition,
it will be interesting to see which ones are indeed faster than the others on this task.
Then, more complex fitness functions with gaps and local optima were defined, such as

Software Design
Research in the field of evolutionary algorithms has been ongoing for years now and lots
of progress have been made in order to improve their general running times. Neverthe­
less, as seen in Chapter 2, their analysis requires deep mathematical knowledge, and it
is difficult to get tight bounds on their expected running times. In addition, bounds calcu­

problem. As this problem is NP­complete, this could also be a way for researchers to test
algorithm on other NP problems as, by definition, they can be translated to an instance of

4
Experimental analysis
As previously seen in Chapter 2, evolutionary algorithms have been intensely studied in
a theoretical manner to try to understand and predict their behavior depending on the
specific problem they are solving. This research focuses on different aspects of the evo­
lutionary algorithm, such as their efficiency for climbing up a gradient with the OneMax
function, or for instance, the ability of an algorithm to overcome gaps of different sizes
and complexities. So far, researchers have succeeded in proving very precise bounds on
the expected running times of these algorithms for optimizing fitness functions by using
advanced mathematical techniques. Some of these techniques were also made more
general, as for instance fixed­budget theory [9], so that they could be reused in different
situations. Furthermore, many papers verify their theoretical results through experimental
analyses.
The aim of the experimental analysis is to pursue previous research by generating new
benchmarks and trying to make new assumptions about the behaviors of each evolu­
tionary algorithm. In order to do that, already established results about ideal parameter
selection and expected running time bounds will be used. These benchmarks will also be
carried out by optimizing standard fitness functions with well­known characteristics. The
data generated and presented in this section has been obtained through the software im­
plementation described in Chapter 3 using the scripts and the HPC of DTU as explained
in Section 3.4.
A selection of algorithms had to be made in order to restrict the scope of the analysis.
The RLS algorithm will not be studied in this section as it is unable to overcome any gap
in the fitness functions. Instead, the (1+1) EA will be used as a standard benchmark
to be compared with. This will highlight the efficiency of the stagnation­detection and
self­adapting mechanisms present in the SD­(1+1) EA, SA­(1,λ) and the SASD­(1+λ) EA.
Observations will also be made on the impact of increasing the number of offspring, as
well as a comparison of the ”+” and ”, ” strategies. In addition, the two RLS derivations, the
SD­RLSr and the SD­RLSm will be taken into account as they have mechanisms allowing
them to overcome gaps without staying stuck forever in a local optimum. Furthermore, the
impact of increasing the population will be measured through the (µ + 1) EAs, where the
focus will be made on the inverse K­tournament parent selection operator as it allows less
selective pressure. The last selected algorithm is the cGA, so that comparison between
mutation­based and crossover­based evolutionary algorithms can be shown. This gives
a heterogeneous set of algorithms that should display the influence of each mechanism
in optimizing different fitness functions.
At first, these evolutionary algorithms will be compared on the basic OneMax function to
check their correctness and to ensure good choices of parameters. Secondly, their ability
to overcome fitness gaps will be studied as well as assumptions. Finally, these results
will be applied for the analysis of the optimization of the MaxSAT problem which is more
general and practical, and for which it is more difficult to study the theory.

4.3
Experimental analysis of the MaxSAT problem
After analyzing the different evolutionary algorithms and their average number of fitness
calls on the previous fitness functions, lots of information has been gathered. This is due
to the fact that the particularities of each fitness function were known and were used for
analyzing the algorithms. This gave a broad overview of the behavior of each algorithm,
highlighting their strengths and weaknesses. Now, the aim of this section is to apply the
results found to a black­box system, the MaxSAT problem. The problem is defined by a
set of clauses over n boolean variables and the objective is to return an assignment of
truth values to the variables which satisfy the maximum number of clauses [6]. In order
Implementation and Evaluation of Strategies for Escaping Local Optima with Evolutionary Algorithms
55
==PAGE:64==
to transform the MaxSAT to a fitness problem the previously algorithms can solve, each
variable will represent a bit in the bit string, and the fitness function will return the number
of satisfied clauses. The problem size is then equal to the number of variables n. k will be
the number of clauses in the problem. MaxSAT is an interesting problem due to the fact
that it is an NP­Complete problem [6], meaning that any problem in NP can be transformed
into an instance of MaxSAT. Therefore, any problems from NP can be optimized using the
evolutionary algorithms defined in this paper.
4.3.1
Scope of the experimental analysis on MaxSAT
In order to perform the analysis, it was decided to use the SATLIB library which can
be found at https://www.cs.ubc.ca/~hoos/SATLIB/benchm.html. The Uniform Random­
3­SAT set has been used for this analysis, using forced generation to make sure the
instances generated were satisfiable (i.e. the maximum of the fitness function is the num­
ber of clauses). Each of the problems inside that set were generated as follow:
• A clause is formed with 3 literals picked at random between the 2n possibilities (n
variables and their negation)
• If the clause has identical literals or is a tautology, it is discarded.
• If the clause is not compatible with the previously generated clauses (i.e. they can’t
all be satisfied with the same truth assignment), the clause is discarded. Else, the
clause is kept.
• The process is iterated again until k clauses are kept.
For the analysis in this report, 3 different subsets have been used for running the evolu­
tionary algorithms:
• uf20­91: 1000 instances of MaxSAT with 20 variables and 91 clauses. Each algo­
rithm was run 1000 times on each instance.
• uf50­218: The first 250 instances out of 1000 of MaxSAT with 50 variables and 218
clauses. Each algorithm was run 500 times on each instance.
• uf75­325: 100 instances of MaxSAT with 75 variables and clauses. Each algorithm
was run 500 times on each instance.
Finally, for running time purposes, a maximum number of fitness calls has been set to
1000000 for each run, after which the algorithm stops and returns the best individual it
has found in that amount of time.
4.3.2
Results of the experimental analysis of MaxSAT
At first, the graphs from Figure 4.22 have been computed. They represent the percentage
of runs for each algorithm that has been successful (i.e. found the optimum) in less than
a certain number of iterations.
At first glance, the results obtained from Figure 4.22 seem very difficult to analyze due to
the complexity of displaying the important information. Nevertheless, some observations
can be made.
The first thing that can be spotted from these graphs is that, for each evolutionary algo­
rithm and each set, there seems to be a great improvement in the percentage of successes
which is then followed by a diminished increase. The first assumption that will be made
there is that there are different scenarios for each of the mutation­based algorithm on the
optimization of the MaxSAT problem. The first expected scenario is that the algorithms
start in a position in which there is a simple gradient leading towards a global optimum,

similar to the NeedGlobalMut, though the global optima would be located in the other
direction, switching the probabilities of successes and failures.

enough data. This can only be considered as a lead for further research, as the three
sets used were generated in a similar way with similar ratios between the number of vari­

algorithms for the optimization of MaxSAT:
• SD­RLSm: It is interesting to see, that in the limit imposed by this analysis, no pos­

mechanism in comparison to the SD­RLSr.
• SA­(1,λ) EA: Another observation is the surprising result about the evolution of the

than 1000000 fitness calls. But in the end, the SA­(1,λ) EA was unable to perform
for larger values of n. Further research on the parameter λ could be conducted as
well as the impact of the ”comma” strategy.
• cGA: The cGA seems to perform correctly compared to the mutation­based algo­
rithm. Nevertheless, it is probable that genetic drift occurs in the MaxSAT problem.
Further analyses with different values of K could help understand the behavior of
the cGA in that context.
4.4
Summary of the experimental analyses
From the findings of the analyses, it has been seen that it was not possible to determine a
”better” algorithm as they all had advantages and disadvantages depending on the prob­
lem.
To start with, an analysis on the OneMax function was conducted to determine the ability
of each evolutionary algorithm to climb­up a gradient in order to improve the fitness until
a maximum is reached. The result showed that the algorithms based on the RLS were
faster in the optimization compared to the algorithm based on the (1+1) EA. As expected, it
58
Implementation and Evaluation of Strategies for Escaping Local Optima with Evolutionary Algorithms
==PAGE:67==
has been shown that an increase in the population was not helpful in that kind of situation.
This is the same for ”comma” strategies in which the parent is not kept, and that these
algorithms struggle to improve when there are more possibilities of decreasing the fitness
than increasing it.
Then, the Jump4, JumpOffset4 and JumpOffsetSpike8 fitness functions have been used
for further analysis, to show the properties of each algorithm in overcoming fitness gaps
of different nature. For more difficult fitness gaps where there are fewer possibilities of
improvement such as for the Jumpm function, the RLS have once again been more ef­
ficient due to a faster increase of their strength compared to the (1+1) EA based algo­
rithms. Nevertheless, when analyzing these same algorithms on easier gaps such as
for the JumpOffset4, the (1+1) EA were able to reduce their running times considerably,
while the SD­RLS were unable to take advantage of the situation. After that, the Hurdle4
function was used, combining jumps with different complexity. It was seen that overall,
the (1+1) EA based algorithms were doing better than the SD­RLSr, though the addition
of the memory radius mechanism in SD­RLSm could help reduce the running time con­
siderably in that case. Nevertheless, this mechanism hasn’t been particularly helpful for
the optimization of other fitness functions, and might even have some drawbacks.
All the previous analyses have brought a better understanding of the algorithms, enabling
the analysis of more complex functions such as NeedGlobalMut. By using a function with
different optimization scenarios, it has been seen that each algorithm has a probability of
exploring one or the other, leading to very different running times. For the NeedGlobalMut,
it has been seen that the (1+1) EA based algorithm was able to explore both scenarios
while the SD­RLS algorithms have only explored one. In that case, the (1+1) EA was
advantaged compared to the SD­RLS, due to the fact that the latter ones were going to­
wards a trap. Nevertheless, it is important to notice here that an increase in the population
with lower selective pressure has been the best improvement to the (1+1) EA for solving
this problem, as the probability of not exploring the correct scenario was considerably
reduced.
Finally, it was possible to tackle the MaxSAT problem. All the observations made pre­
viously were necessary to understand the behaviors of the algorithms as no information
on the instances was known in advance. The main assumption here was that there are
different scenarios for the mutation­based algorithms towards the global optimum, creat­
ing a strong disparity between the runs of an algorithm. The behaviors of the algorithms
have shown similarities to what has been seen in the NeedGlobalMut section, where the
(1+1) EA based are more flexible and can then solve more instances in shorter times.
Nevertheless, the use of stagnation­detection and self­adjusting have shown opposite
probabilities than for the NeedGlobalMut for going through the best scenario. Then, once
again, an increase in the population and decrease in the selective pressure has shown to
be the best answer to this problem. It would then be recommended to also increase the
population on the algorithms with self­adjusting and stagnation detection as better results
would be expected.
Regarding the cGA, its completely different behavior due to the implicit mutations and
crossovers has made its analysis more complex. As it was more difficult to compare
it with the other algorithms, more data had to be generated, especially regarding the
choice of the parameter λ. It has been assumed that for the optimization of the Jumpm
and the JumpOffsetm function, there are respectively a function f and g such that K =
f(n, m)√n ln n and K = g(n, m)√n ln n are optimal. Both f and g are expected to de­
crease when n is increased, and to increase when m is increased. Even though the cGA

could also lead to a better parameterization for the MaxSAT problem.

Future work
Throughout this paper, different leads have been suggested in order to pursue the work
done in this project. An overview of the main ideas will be summarised in this chapter.
One of the main issues that was encountered in this project is a too small quantity of
data. This led to a lack of understanding of some of the observed phenomena, as well
as a shortage of evidence to support some theories. This was mainly caused by the
complexity of generating and treating big amount of data with the current software. Many
improvements can still be made to the software to cope with this issue. The main problem
at the moment is that the interface can not be used for running consequent analyses. This
would require the possibility to start analyses directly on remote servers without having
to use scripts and terminal commands to start them. Furthermore, the scripts used for
getting all the results are yet separated from the interface, and they should be made
more accessible for the user. All these modifications will benefit from a cleaning and
restructuring of the code, especially from the point of view of the ActionInterface classes
and how the Parameter and Constraint classes are dealt with. Nevertheless, it is fairly
easy to implement more evolutionary algorithms and fitness functions to the software.
Improvement of the UX/UI should also be considered.
If further work were to be made on this paper, it would be necessary to stay up to date
on the latest paper published in the field of evolutionary algorithms. Due to the com­
plexity of the subject and the growing interest in its applications, researchers have been
continuously improving the evolutionary algorithms, as well as analyses to get more and
more tight and precise bounds on their expected running times. This is important in the
way that theoretical and experimental analyses need to be developed simultaneously, as
experimentation allows making conjectures and verification of the theoretical work.
Regarding the analyses made in this thesis, many would require further experimentation
to give more concrete results. This is especially true in order to find optimal choices for the
parameters of each algorithm. However, the data generated in Section 4.1 and Section 4.2
already gives a great understanding of the behavior of the evolutionary algorithms on the
tested fitness function. One might then try to explore new leads. It would be interesting
to develop new algorithms based on different techniques seen in this report, that could
perform better in all the problems that have been tackled during this thesis. Finally, more
experimentation is essential on the MaxSAT problem as only a preliminary study has been
made. A better understanding of the behavior of each of the evolutionary algorithms on
this problem could lead to improvement in the running times of many other problems due
to the NP­Completeness of MaxSAT.

6
Conclusion
All along this Master’s thesis, the work carried out has aimed to give a better understand­
ing of state of the art strategies on evolutionary algorithm for leaving local optima.
At first, a literature review was conducted in order to select the most recent evolution­
ary algorithms studied in the field for overcoming fitness gaps. This research has en­
abled a selection of interesting algorithms using different methods such as an increase of
the population, stagnation­detection and/or self­adjusting mechanism, or algorithms using
crossover transformations. At the same time, this process has given the opportunity to
find interesting fitness functions with fitness gaps of different nature for further analyses,
as well as theoretical lower and upper bounds on the expected running times of the evo­
lutionary algorithms. Even though there were many results for the analyses on OneMax,
there were fewer on the more specific functions chosen in this report, leaving room for
experimentation and hypotheses.
In order to pursue the theoretical work found in the literature review, a framework was de­
veloped, allowing any user to make its own experiments through a practical interface. The
evolutionary algorithms and fitness functions picked from the research papers have been
implemented, and functionalities such as displaying statistical results of an experimenta­
tion have been added to the software. Furthermore, for more demanding analyses, the
framework has the possibility of starting scripts that can be run on more powerful servers
such as the HPC at DTU. This made possible the computation of all the results presented
in this report.
A first preliminary analysis has been made on the well known OneMax fitness function.
This showed for each of the algorithm, their ability to go through straightforward gradi­
ents depending on the choice of its parameters. This allowed to see that the stagnation­
detection and self­adjusting mechanisms were not noxious for the optimization of simple
fitness functions. This also verified the correctness of the algorithms by comparing the
results obtained with the theory. After that, the analysis was pursued on the more com­
plex fitness functions, giving an interesting comparison between the SD­RLSr and the
(1+1) EA based algorithms using stagnation­detection and/or self­adjusting mechanisms.
It has been realised that on more difficult gaps, the SD­RLSr was performing better while
the others were faster for easier gaps, with more possibilities of improvement. Further­
more, the addition of a radius memory scheme on the SD­RLSr has shown little interest
in general due to the irregularity of the practical problems tested in this report. Another
observation that has been made was the fact that the ”, ” strategies revealed to be to un­
stable with the chosen parameters. In order to get a more consistent behavior, it would be
recommended to use an increase in the population size with a reduction of the selective
pressure. Indeed, even if the (µ+1) EA was not performing as good as other algorithms
on simple problems, it has been seen to be very consistent and efficient for problems
with several improvement scenarios. Finally, a first estimation of the nature of fitness
functions of MaxSAT problems has been made, while showing the best algorithms in its
optimization.
Further work should be conducted to extend the work made so far in this thesis. First,
more theoretical results could be found to understand better the experimental analyses.
Then, an improvement of the software could help generating more results, in order to give
a better insight on the behaviors of the selected algorithms depending on the problem.

Frames
This appendix is showing the frames of the 7 actions available in the framework at the
moment. For each action, a short description is given.
• Figure A.1: 1 ­ Display the graph of a fitness function. In this frame, the user can
choose a problem size and a fitness function with its parameters. By clicking on the
”Display graph” button, the image of the function is plotted in function of the norm of
the bit strings. In case the function is not only depending on the norm, the image of
the bit strings ”0000...”, ”1000...”, ”1100...”, etc. are used for the plot.
• Figure A.2: 2 ­ Display a run of an evolutionary algorithm on a fitness function.
In this frame, the user can choose a problem size, an evolutionary algorithm and
a fitness function with their parameters. By clicking on the ”Run” button, a scat­
tered plot of the improvements of the algorithm on its run on the fitness function, is
displayed.
• Figure A.3: 3 ­ Statistical analysis of an evolutionary algorithm on a fitness
function and fixed problem size. In this frame, the user can choose a problem
size, a number of iterations, an evolutionary algorithm and a fitness function with
their parameters. By clicking on the ”Run” button, statistical results of the runs are
displayed.
• Figure A.4: 4 ­ Statistical analysis of an evolutionary algorithm on a fitness
function for a range of problem sizes. In this frame, the user can choose a prob­
lem range, a number of iterations, an evolutionary algorithm and a fitness function
with their parameters. By clicking on the ”Run” button, a plot of the average number
of fitness calls in function of the problem size is displayed. Alongside, there is a box
plot of the number of fitness calls for each run in function of the problem size.
• Figure A.5: 5 ­ Comparison of several evolutionary algorithms on a fitness
function for a range of problem sizes. In this frame, the user can choose a prob­
lem range, a number of iterations, an evolutionary algorithm and a fitness function
with their parameters. By clicking on the ”New run” button, a plot of the average
number of fitness calls in function of the problem size is displayed. Clicking on the
”Add run” button will add the new run to the previous graph instead of generating a
new one.
• Figure A.6: 6 ­ Solve a MAX­SAT problem. In this frame, the user can choose a
problem range, a number of iterations, an evolutionary algorithm with its parameters
and a .cnf file. By clicking on the ”Run” button, a scattered plot of the improvements
of the algorithm on its run on the MAX­SAT problem is displayed.
• Figure A.7: 7­ Statistical analysis on MAX­SAT problems. In this frame, the user
can choose a problem range, a number of iterations, an evolutionary algorithm with
its parameters and a folder with .cnf files. By clicking on the ”Run” button, the evolu­
tionary algorithm will run once on every .cnf file in the folder and will return statistic
values. Furthermore, a plot with the number of fitness calls that were necessary for
each file is displayed.
