Summary
In this thesis we implement and analyze a set deep learning methods capable of solving the
Questions and Answers problem (Q&A) to a suitable degree for small real-world applications.
The models are derived from modifications of the BiDAF architecture, which formulates the
answer to a given query as a segment of text from an associated reading passage (context). As
initial modification, we include the ELMo contextual embeddings in the input of the model,
following the latest trends in the state-of-the-art of Natural Language Processing. The main
novelty in this thesis is the transformation of the layers of the model into Bayesian Neural
Networks, in which the point value weights of the original Neural Network are converted into
random variables. We compute an approximation of their posterior distribution using a novel
modification of the Variational Bayes algorithm (VB), which allows us to control the amount
of regularization that VB introduces. The extensive documentation of the models, together
with the Toy Examples and Background Theory in the Appendices provide this thesis with
an added educational value. Moreover, all the code developed is documented and publicly
available in the referenced repository, allowing the interested readers to replicate the results,
use the trained models and continue the research.

CHAPTER 1
Introduction
Machine Comprehension of Text (MCT) has been one of the main goals of Artificial Intelligence during
the last decades [4]. A system capable of understanding the meaning of written sentences would revolu-
tionize the information era we live in; having almost limitless applications including chat-bots, advisory
systems, and enhanced information retrieval. But how do we even define the “understanding” of text?
Can you describe how you understand this very sentence?
Our intuition tells us that a system capable of understanding text would require having general world-
class knowledge about the concept that each word represents, and the complex relationships between
them. Ultimately, we consider a machine to have human-level intelligence if its responses in a dialog
would be indistinguishable from those of a human, this constitutes the so called Turin test [44].
Such definition of MCT does not shed much light on how to develop systems capable of understanding
written language and therefore, simplifications of this task have been proposed.
Among them, we
encounter the Question and Answers (Q&A) task, where we consider that a machine comprehends a
reading passage if, for any question regarding the text, it can generate an answer that a majority of
humans would consider correct, or that at least it does not contain irrelevant information for answering
the question.
Historically, a major constraint to solve this simplified task has been the lack of large labeled datasets; a
situation which limited the types of models that could be trained automatically with data. Traditional
solutions for this kind of task had to rely on Natural Language Processing (NLP) pipelines that included
a lot of in-built human knowledge about linguistics to engineer hand-crafted features from which the
algorithm would ultimately learn. The famous IBM system Watson [14], is an example of such models.
From 2016, several increasingly large and challenging datasets on the Q&A problem have been published.
Their size and complexity offer a way to automatically evaluate the machine comprehension capabilities
of a system in a quantitative fashion. Among the most popular Q&A datasets that are publicly available,
we can find, in increasing order of complexity: CNN/Daily mail [6], SQuAD 1.1 [40] and TriviaQA [28].
In this thesis we train our models with the SQuAD 1.1 dataset, which contains samples that are
composed by triplets (passage, question, answers). The triplets are generated by crowdworkers from a
diverse selection of Wikipedia articles. Figure 1.1 shows an example passage from this dataset with 3
related questions and answers.
These relatively large datasets have recently enabled the development of end-to-end Neural Network
(NN) architectures which solve the Q&A task without the need to explicitly embed human expertise
about linguistics in their structure. This field is currently experiencing an explosion of research articles,
similar to what happened to the Machine Translation task in 2014, [8], where deep Encoder-Decoder ap-
proaches outperformed all previous translation techniques that humankind had handcrafted for decades.

Given the enormous potential of a deep learning system able to effectively solve the Q&A problem, the
biggest IT companies worldwide such as Google, IBM, Microsoft, and Alibaba, have been continuously
publishing their models over the last 2 years, consistently pushing the state-of-the art on a monthly
basis. The current accuracy of some architectures for datasets such as SQuAD 1.1 has already reached
human-like performance [20]. Due to the flourishing diversity of the new models, it is becoming harder
and harder to categorize the different approaches used by all of these big competitors, but in every
approach we can find the following 3 components:
• Word embeddings: The conversion of each word into a vectorial representation of usually
fixed size. Not so long ago, the most widely used transformation was the Bag of Words (BoW),
characterizing each word by its frequency in a given corpus. Since then, richer and richer word
embeddings have been formulated in order to increase the information that the vectors contain
about their associated words. In this line we can find famous NN-based architectures such as
Convolutional Neural Networks [31], word2vec [35] and GLoVe [37] embeddings.
The current
trend points towards “contextualized word embeddings” where the representation of each token
depends on the context in which it is used (surrounding words). This type of embeddings have
shown outstanding empirical performance, especially the ones based on deep learning architectures
such as ELMo [38], based on bi-LSTMs, and BERT [11], based on transformers.
• Fusion of the query and passage information: Where attention is the most commonly used
technique, with architectures such as BiDAF [42], Dynamic Coattention Networks [50], R-net
[19] and Reinforced Mnemonic Reader [26]. All these architectures compute some sort of metric
between each word in the passage and each word in the query, usually an element-wise product
of their dimensions followed by a linear transformation down to one dimension.
• Selection of the final answer: From successive transformations and fusions of all words in
the context and query, all methods end up with a query-aware representation of each word in
the context, over which they design a strategy to select the most likely answer. Most techniques
project each word representation into 1 dimension and apply the Softmax function to obtain two
probability distributions over the context words: p1, the probability of each word being the start
of the answer, and p2, the probability of each word being the end of the answer. From these
two distributions, some methods deterministically compute the answer as the most likely subspan
[42][38][11], and others include a last layer based on pointer networks [46] that learn to formulate
the final answer [26][50][19].
In October 2018, Google released BERT [11], a deep learning architecture based on bi-directional
transformers, which like ELMo, produces contextual word embeddings of the text given as input. With
the inclusion of an additional output layer and some task-specific fine tuning, BERT has been able to

obtain a new state-of-the art in 11 NLP tasks including Q&A, beating all previous architectures that
were specifically tailored for the problems.
BERT is based on deep transformers, an architecture developed by Google and published in the paper
“All you need is Attention” [45] back in 2017. This new promising model aims to completely replace the
recursion used in Recurrent Neural Networks by means of self-attention mechanisms that connect the
information among all the elements of the sequence with just one path (one vectorial transformation).
Also, while bi-RNNs learn the forward and backward representations of the words separately, BERT
learns both types of information in the same representation. All these improvements come with the
burden of a more complex training setup needed and a limitation on the scalability of the architecture
for very long sequences, as the self-attention representation grows with the square of the length of the
sequence.
Throughout the short history of deep learning, there has been many promising architectures that even
being well-founded, did not stand the test of empirical experiments. With the publication of BERT,
it seems that transformers have come to stay, and we can expect to see a lot of research in this area
during the upcoming months, trying to replace RNNs with attention mechanisms and aiming to solve
the scalability issues of self-attention by the formulation of hierarchical models. As a personal opinion,
the concept of applying 2D Convolutional Neural Networks over attention matrices used in [13] seems
very promising, as well as the possibility of combining all previous blocks into models that could jointly
learn image, audio and video in the same shared architecture [29].
There is no doubt that with the new rich contextual embeddings such as ELMo and BERT, the NLP
field is experiencing a similar revolution to what happened to computer vision with ImageNet [10]
in 2009. Although these embeddings lack a hierarchical structure that would allow them to properly
learn more abstract information such as discourse, coreference and entailment, they offer a powerful
representation that can be included in the input of most previous NLP processing pipelines and has
already become a “must-have” tool in every sequence-to-sequence text processing task.
1.1
Scope of this thesis
The principal objective of this thesis is to implement a modification of the deep learning architecture
Bi-Directional Attention Flow (BiDAF) [42] in which we substitute the original word representations by
the state-of-the-art ELMo embeddings [38] and validate its performance over the SQuAD 1.1 dataset
[40]. After achieving this goal, we validate the hyper-parameters of the network in order to obtain the
configuration with the best possible accuracy. In this regard, we perform experiments to quantize how
each hyper-parameter affects the learning, with especial attention to the ones controlling the regular-
ization, such as the dropout rate. Also, an extensive study of the error of the algorithms is included,
where we analyze the properties of the wrongly answered samples. This study helps determining the
situations where the algorithm performs reliably and those where it should not be used.
The main novelty in this thesis is the transformation of the model layers into Bayesian Neural Networks
(BNNs), where the original point valued weights are converted into hidden random variables, for which
we compute an approximation of their posterior distribution using the Variational Learning algorithm
known as Variational Bayes. When applied to neural networks, this algorithm has adopted different
names in the literature, such as Bayes by Backprop (BBP) [2][15] and Stochastc Variational Inference
(SVI) [24]. Figure 1.3 visually represents the difference between a normal Neural Network (NN) with
point values and a BNN where the weights are random variables. The interested reader can find a full
revision on the advances in Variational Inference in [54].

If the variational posterior distribution of the weights is properly found, this change of paradigm offers
numerous advantages, such as adding principled regularization in the learning process due to introduced
noise in the gradients and a penalty term in the loss function. The possibility of expressing each weight
as a random variable also allows us to asses its statistical significance and sample from its variational
posterior distribution, with which we can quickly generate Bayesian ensembles and compute a certainty
measure of the estimations given by the model.
Even though BNNs trained using Variational Bayes offer a principled way of dealing with uncertainty
in the training process, there is an almost complete lack of published real-world applications that use
such method, partially but not sufficiently explained by the fact that the training algorithm is relatively
new, published in 2015 for general NNs and adapted in 2016 for RNNs.
The very recent work in [47] (published a month before this thesis was concluded) could explain this
counter-intuitive situation, arguing that variational inference in neural networks is fragile, requiring
very fine tuned initializations and prior distribution parameters, as well as a more refined control of the
variance of the gradients derived in the updating rule.
In this line of thought, we propose two modifications of the original BBP algorithm in order to address
these issues in a simple manner. First, we normalize the hyper-parameters of the prior for each layer
according to its dimensionality, a similar approach used in the Xavier heuristics [18] for the initialization
of weights. Second, we add a new hyper-parameter ζ which multiplies the regularization term in the
VB loss function, allowing us to control the regularization level that best fits a given dataset.
These modifications and the advantages of BNNs are better explained through one of the toy examples
developed in this thesis, illustrated in Figure 1.3. In the caption we can observe, for different degrees of
regularization given by ζ, the resulting BNN trained using the BBP algorithm over a simple regression
problem, in particular:
• The lower half shows the variational posterior parameters obtained, characterizing each weight in
the network as an independent Gaussian random variable with mean µw and standard deviation
σw. We can observe how as we increase ζ, we add more regularization to the training algorithm.
As we increase the value of the parameter ζ, the final weights obtained have a smaller absolute
mean and a higher variance .
• The upper half shows the datapoints in the dataset and the predictions of 100 realizations of the
model, obtained by sampling the variational posterior distribution of the weights shown below.
From these realizations we can then compute the Bayesian ensemble estimation, shown in blue, and
a measure of the certainty of our estimations, given by the range of the 100 individual estimations.
As we can observe, the parameter ζ can properly control the lever of regularization in this toy
example.

CHAPTER 2
Deep Learning methods
implemented
The initial baseline of this thesis is a reimplementation of the Bi-Directional Attention Flow (BiDAF)
architecture [42].
The BiDAF model and several of its variations were the state-of-the-art for the
SQuAD 1.1 dataset at the time they were published, from June 2016 to March 2018. As of December
2018, the performance of the best BiDAF-based architecture is the model “BiDAF + Self Attention +
ELMo (ensemble)” developed by Allen Institute for Artificial Intelligence [20]. This model is described
in [38] and has as main modification, the inclusion of the ELMo contextual embeddings in the input of
the architecture. They also introduce several other changes such as changing the dropout to variational
dropout, replacing the LSTM layers by the simpler GRU layers, and including self attention mechanisms
on top of the already existing Attention Layer.
Over time, new models have surpassed the accuracy of BiDAF-based architectures, by December 2018,
the best performing models over the SQuAD 1.1 dataset are based on a new type of pre-trained general
language representations called BERT [11]. In short, the architecture of BERT differs from ELMo in
that the first one is composed by deep bidirectional transformers and ELMo is made of deep bidirectional
LSTMs. Also BERT is pre-trained using a more complex set of tasks than simply next word prediction,
and its parameters are fine tuned during the task-specific learning (Q&A in this case), as opposed to the
general use of ELMo embeddings, where their weights are kept locked during the task-specific learning.
Despite not being the current best performing architecture, the intuitively appealing structure of BiDAF
and its good documentation made it a good starting candidate for building Machine Comprehension
architectures that can later be improved upon. In this thesis we aim to use Variational Learning on the
weights of the networks to improve regularization, prune the network, and easily sample realizations of
the modules to create Bayesian ensemble models.
2.1
Bi-Directional Attention Flow (BiDAF)
The Bi-Directional Attention Flow (BiDAF) network [42] is a hierarchical deep learning architecture
designed for solving the Questions and Answers (Q&A) task, where the answer to the query is contained
in a span of the original context. The main characteristics of the BiDAF model are:
• It creates a representation of the passage and the query at different levels of granularity. The
BiDAF architecture includes representation of each word at character-level, word-level, and con-
textual embedding level.
• It uses several deep bi-LSTM networks throughout the model to capture the temporal relationships
between word representations at different stages.
• It uses a bi-directional attention flow to obtain a query-aware representation of the context. Two
different types of attentions are computed from the same similarity matrix S, Context-to-query
(C2Q) and Query-to-context (Q2C). These attentions are computed at each time step (word in the

context) as in [1] and are memory-less, that is, the attention at each time step does not directly
depend on the attention of the previous step.
Much like in Computer Vision where the information of an image is defined at different levels of
granularity: pixels, superpixels, convolutional filters, pooling...
It has been hypothesized that the
granularity of the embeddings used in the BiDAF allows further layers of the model to pay attention
to the relevant information linking the context and query.
The authors also hypothesize that the simplification of making the attention memory-less causes the
Attention Layer to focus on learning the word-to-word relation between the query and context tokens,
allowing the later Modeling Layer to focus on learning the interactions within the query-aware context
representation (the output of the Attention Layer). It also allows the attention at each time step to be
unaffected from incorrect attendances at previous time steps.
The original architecture of the BiDAF model can is illustrated in Figure 2.1, taken directly from the
original paper. This hierarchical multi-stage model consists of six layers that we individually analyze
in .
Figure 2.1: BiDAF architecture
The purpose of each individual layer can be summarized as follows:
• Character Embedding Layer transforms each word in the passage and query into a fixed-length
vector representation using character-level CNNs.
• Word Embedding Layer also transforms each word into another fixed-length vector represen-
tation using a pre-trained word embedding model. These 2 representations are then concatenated
and further transformed using a Highway network.
• Contextual Embedding Layer uses a deep Recurrent Neural Network to obtain a represen-
tation of each word in the passage and query that contains information about the surrounding
words.
• Attention Flow Layer merges the information of the query and passage vectors, generating a
query-aware representation of the passage.

• Modeling Layer also employs a Recurrent Neural Network to capture the relationships between
the query-aware vector representations of each word in the passage.
• Output Layer computes the answer span from the context word representations of Modeling
Layer.
In the following, we will further describe the content of each layer, including the Neural Network
architectures that they contain along with their purpose. We will also pay special attention to the
dimensionality of their parameters, inputs, and outputs.
For a more in-depth explanation on how

Let’s first establish some notation for expressing the original data, we denote a passage P as a set of
T words and we denote a query Q as a set of J words. Usually T is much bigger than J, since the
passages contain hundreds or thousands of words whereas the queries only contain a few words.
P = {x1, x2, ..., xT }
Q = {q1, q2..., qJ}
Each word in P is denoted as xt with t = 1, 2, ..., T each word in Q is denoted as qj with j = 1, 2, ..., J.
At the same time, every word is made of a set of characters which varies from 1 to 10 in most cases. As
an example we could have:
P = {Ingrid, really, loves, bacon, because, it, is, so, crispy, and, it, only, takes, 2,

Notice how each token word is separated by comas and how for example the exclamation sign “?”

how the passage and query are sequentially converted into increasingly abstract continuous vector
representations, which are finally used in the Output Layer to learn the most likely span of the context
that contains the answer.
2.1.1
Character, Word and Contextual Embedding Layers
The end goal of these 3 layers is to transform the string of characters that composes each individual
word (or token) in the passage and query into a high-dimensional continuous vector of fixed length
Dword. This representation models the information of each word at character and word level, as well
as the temporal relationships among words within the passage and query. As previously stated, this
granularity is hypothesized to allow the algorithm to focus its learning in whichever domains performs
best, much like the architectures found in computer vision.
In the Character embedding layer we map each word to a high-dimensional vector space by obtaining
a character level embedding of each word using Convolutional Neural Networks (CNN) as described in
[31]. In the original paper [42] 100 1D filters for CNN char embedding are used, each with a width of
5. To create this representation, the characters of each word are embedded into 1-dimensional vectors
of size 16, from a dictionary of 260 possible characters. The outputs of the CNN are max-pooled over
the entire width to obtain a fixed-size vector for each word of size 100. Therefore this layer transforms
the passage and query into the matrices XC ∈R100×T and QC ∈R100×J respectively.

2.1.2
Attention Layer
The attention flow layer is responsible for linking and fusing information from the context and the
query words. The end goal of this layer is to produce a query-aware representation of the words in the
passage, G, which further layers can use to learn the most probable span of the context that contains
the answer.

embeddings from the previous layer, H, and the attention vector at each time step. This differs from
previous approaches such as [49] [50] [51] where the attention flow layer is used to summarize the query
and context into single feature vectors.

2.2
Baseline model: BiDAF + ELMo
In our implementation of the BiDAF, we have replaced the Character Embedding Layer and the GLoVe
embeddings with the ELMo representations [38]. In short, the ELMo embeddings are obtained as a
linear combination of the hidden states of a deep bi-LSTM network whose parameters have been trained
using the next word prediction task for 10 epochs on the 1B Word Benchmark [5].
At the beginning of the ELMo architecture, each token word xt is transformed into a continuous context-
independent representation using convolutional neural networks over char embeddings. In particular,
ELMo uses 2048 channel char-ngram CNN followed by two highway layers [43] and a linear projection
down to 512 dimensions.
This continuous representation then goes through 2 bi-LSTM layers with a hidden state dimensionality of
512. Each of the l = 1, 2 bi-LSTM layers creates 2 hidden state representation of each word, −→h k,j ∈R512
from the forward LSTM and ←−h k,j ∈R512 from the backward LSTM. These 2 vectors are concatenated
into the hidden state vector of each layer ht,l = [−→h t,l; ←−h t,l] ∈R1024.
The second bi-LSTM has a residual connection from the first layer [30], meaning the hidden state of the
first layer is added to the hidden state of the second layer after it is computed. Residual connections
allow gradients to flow through the network directly, which has been found beneficial at reducing the
explosion or vanishing of gradients in deep architectures.
The weights of the initial embedding, Wx, and those of the bi-LSTMs, WLST Ms, are trained by means of
performing the next word prediction task over the 1B Word Benchmark. For this purpose, the forward
and backward hidden states of the last layer −→h t,2 and ←−h t,2 go through a Softmax function which maps
each of the two 512 dimensional states to a discrete distribution over the words in the vocabulary. We
denote the weights of the Softmax layer as Ws. Therefore we can obtain a word prediction both from
−→h t,2 and ←−h t,2 separately.

As most reimplementations, this baseline model is not an exact copy of the original paper. The main
difference consists on removing the Character Embedding Layer and replacing the word embeddings
from the Word Embedding Layer. The GLoVe embeddings [37] of the original paper are replaced by
the second layer of the ELMo embeddings [38].
The motivation for this replacement is the current trend in Language Modelling of using deep pre-trained
representations of the tokens. As described in [38], it has been observed that combining the GLoVe and
ELMo embeddings by means of concatenation does not improve performance, and consequently, the
GLoVe embeddings have been left out of this thesis.
Apart from replacing the word embeddings, the differences between the original implementation and
our baseline model are:
• Increased dimensionality of the architecture parameters: A trivial consequence of using ELMo
instead of GLoVe is that the dimensionality of Word Embedding Layer increases from 200 to
1024. This will significantly increase the dimensionality (and therefore number of weights) of the
Highway module in the Word Embedding Layer and the bi-LSTMs of the Context Embedding
Layer.
• Dropout is added to the ELMo embeddings. In the original paper, dropout was also added to the
GLoVe embeddings, in our architecture we will allow it to have a different dropout rate than the
rest of the layers.
• We use the Adam [32] optimizer instead of the AdaDelta optimizer used in the original paper [53].
This optimizer has been used in the ELMo paper so that is why we initially chose it over the
original one.

2.4
Bayesian transformation of the models
A mayor goal of this project is to include Variational Learning in the BiDAF model. As opposed to
traditional NNs where the weights of the network W are considered point values to be found in order
to minimize the loss function; in Variational Inference models, weights are considered random variables,
having a prior distribution W ∼p(W) before seeing any data, and a posterior distribution W ∼q(W|D)
after the learning process. Please refer to Figure 1.2 for a visualization of the difference between NNs
and BNNs and to 1.3 for the illustration of a toy example.
In Variational learning, we approximate the posterior distribution of W by means of a parametric family,
q(W|θ), where θ are the distribution parameters of these weights, which we estimate using the data D.
Like in the classic Back-Propagation algorithm, the parameters are iteratively tuned using the gradient
of a loss function F(θ, D) in a process known as Stochastic Variational Inference (SVI) [24], which when
applied to ANNs has adopted the name of Bayes by BackProp in [2] and [15].
This conversion of the point value weights into hidden random variables for which we approximate their
posterior distribution has 3 main goals:
• Regularization: In the original BiDAF paper, a dropout rate of 0.2 at each layer of the network
was found beneficial to avoid overfitting, nevertheless dropout has been found to not work well
with RNNs [52]. SVI adds regularization to the model in two different ways: 1) By adding a
regularizing term in the loss function penalizing complex models and 2) By adding noise in the
computation of the gradients. In the same way that dropout sets some weights to 0 at random in
each training iteration, in SVI we re-sample each weight from its variational posterior distribution,
increasing the noise of the gradients.
==PAGE:27==
2.4 Bayesian transformation of the models
17
• Pruning the network: Having the posterior variational distribution of each weight can also allow
us to know which weights are significant and which ones are not. For example, if the posterior
distribution is Gaussian, each weight having a posterior mean µws and a standard deviation σwd,
the ratio µws/σwd of each weight gives us its importance. This can be used to eliminate sets of
weights, reducing computational cost and overfitting. It can also give us clues on the ideal size of
the network and tell us which modules are the most important ones for the task.
• Building ensembles: Once the model is trained, we can use the learned parameters θ to sample
very quickly as many instances of the model as computational resources we have. In the original
BiDAF paper, the EM of a single model was 68% whereas the EM for an ensemble of 12 machines,
obtained as 12 different runs of the same model, increased to 73.3%. We hope that the randomness
in the posterior distribution will be sufficient to provide a Bayesian ensemble that works better
than the most likely set of weights given by the posterior.
• Computing the certainty of the estimations: We can use the variation in the different
instances of the model (sampled from the variational posterior) to compute a certainty measure
for each estimation given.
In this thesis we have implemented and tested out several schemes of Bayesian Inference. The details of
the mathematics and learning algorithms of VI can be found in Section B.2.3. A description on how the
different elements of the model were converted into Bayesian models can be found in the implementation
Section D.2.1.
The loss function used is a modification of the ELBo loss explained in Section A.3, where the negative
logarithm of the evidence is replaced by the loss function of the BiDAF:

2.4.3
Modification of the loss function
The authors of [16] attempted to show that dropout, the most widely used form regularization in deep
learning architectures, can be seen as an approximate form of VI where the variational posterior is a
mixture of trow Gaussians with very small variance, and where the mean of one of them is fixed to
0, and the other is the weight that we are tuning. In this approximation, the loss function does not
include the KL distance term between the variational posterior and the prior, it only contains the data
dependent term, equivalent to assuming that we have infinite data.
Although such interpretation suffers from several issues addressed by the same main author in a more
recent research study [25], the attempts of expressing dropout in a VI framework seem like the correct
path to justify the effectiveness of this “not so mathematically principled” regularization method, and
ultimately, aim to inspire more founded variations of dropout that could improve its usability.
The former analogy between dropout and an approximation of Variational Inference inspired us to
formulate a rather simple modification of the original BBP algorithm, where we add a new hyper-
parameter ζ, which multiplies the KL distance term between the variational posterior and the prior in
the loss function, giving us more control over the level of regularization that this term introduces. The
final loss function is therefore:
F(W, θ, D) = Eq(W (B)|θ)
[
LBiDAF (W, W (B), D) + ζ

Notice that for ζ = 1 we have the standard ELBo loss and for ζ = 0 we only have the data dependent
term, equivalent to the original loss function of the BiDAF. Notice that even when ζ is 0, As exemplified
in Figure 1.3, the value of ζ will allow us to weight the importance between fitting the dataset and
having a simple structure. Its value is to be found by validation as any other hyper-parameter.

Applying the same optimizer over the loss function FBiDAF (W, θ, D) for all parameters is the easiest
approach, but this differentiation between the different types of model parameters allows us to have
different updating rules for the non-Bayesian weights W and for the posterior variational parameters θ.
Furthermore we could also have different optimizers for different types of variational parameters, like
the means and standard deviations. Unfortunately such experiments are outside of the scope of this
thesis due to lack of time.
2.5
On the Randomness of the models
The models described and their implementation contain elements that can introduce uncertainty in the
estimations, even when the initialization random seeds and inputs to the same. The sources of these
randomness are:
• ELMo Embeddings: Due to the statefulness of the bi-LSTMs in the ELMo architecture, the ELMo
embeddings are not deterministic given an input sample to embed. The initial state of the bi-
LSTM is not reset after each sample, which will cause the same sentences to have slightly different
ELMo embeddings.
• The rest of the bi-LSTMs in the network are also statefull and therefore will affect the output in
the same way.
• The cuDNN optimizations used by PyTorch (and Tensorflow for that matter) make the GPU
computations non-deterministic [7].
In the code that trains the implemented architectures, the CUDA and CPU random seeds are chosen
“at random” from the time stamp obtained when the experiments are launched. These seeds are saved
in the configuration file of the model for later replicability of the experiment, but small differences can
still be expected due to the previously mentioned sources of randomness. Nevertheless, even setting
different random seeds, the variation of the accuracies of the trained models is usually contained within
0.4% of the mean accuracies so the models are robust enough to these non-deterministic sources.

In this Chapter we explore the content of the SQuAD 1.1 dataset [40] and we present the results achieved
by the different deep learning architectures implemented. Due to the prohibitive amount of time that
the models take to train, usually around 6-12 hours on a GPU Pascal Titan X, the number of possible
experiments that we could perform is limited. The following Sections represent the experimentation
paths selected in order to find the model with the best accuracy in the minimum number of trials.
Special attention is paid to the particular types of contexts and queries included in the dataset and
how their characteristics affect the accuracy of the models. A diverse set of visualizations is included
in order to help us detect possible patterns in the incorrectly answered samples, which could inspire
improvements in the architecture and allow us to differentiate the situations where the models perform
reliably from those where they should not be trusted.
As described by its creators, the Stanford Question Answering Dataset (SQuAD) is a reading compre-
hension dataset, consisting of questions posed by crowdworkers on a set of Wikipedia articles, where
the answer to every question is a segment of text, or span, from the corresponding reading passage
(Figure 1.1).
The SQuAD1.1 dataset contains over 100.000 samples partitioned into 2 sets, the training set (TR)
with 87599 instances (80%) and the development (or validation) set (DEV) with 10570 instances (20%).
Each sample contains a passage (context), a question (query) and at least 3 answers annotated by the
crowdworkers. The same passage can have up to 5 associated questions, where each triplet (passage,
question, answers) is treated as an independent sample. The entire dataset contains 23.215 paragraphs
from 536 relevant Wikipedia articles drawn at random. The articles cover a wide range of topics, from
musical celebrities to abstract concepts in order to remove any possible topic-related biased that could
be introduced.
The answers are noted by means of a span specified as a tuple of token indexes from the passage, being
“start index” the position of the first token contained in the answer, and “end index” the position of the
last one, therefore we have span = (start index, end index). The token corresponding to both indexes
is included in the answer, if both of them are the same, then the answer only contains one word. Every
instance includes a main answer and at least 2 other secondary answers, each of them assigned by a
different user. All of the answers are considered correct, being the secondary ones usually shorter since
the crowdworkers were asked to try to answer the questions with the minimum possible span.
Differences among the main and secondary answers often arise from the inclusion/exclusion of words
surrounding the essential information that characterizes the answer.

It was then that we performed experiments where we converted some of the modules of the BiDAF

• Is there a level of regularization (controlled with ζ) of the KL distance term in the loss function

• Can the VPPs give us clues on how the learning in taking place? Such as for example, what are
the variables that the network is paying attention to.
• How many weights of the network can we prune using statistical tests over the VPPs without
affecting its accuracy?
• Can dropout regularization be combined with the BBP algorithm?
Given the modularity of the implementation, where we can transparently mix Bayesian and non-

and non-Bayesian parameters. But of course the amount of possible scenarios grows exponentially with
the number of layers and thus, only a reduced set of experiments was carried out.

not harm the learning process.

• σ1 = 0.5 and σ2 = 0.5: We set both variances to be the same in order to avoid having to validate
another hyper-parameter.

means, which seemed a good trade-off heuristic for generating enough noise to add regularization
at learning time, but not enough to make it diverge.

3.2.4
Statistical pruning of the weights
Since we consider the posterior variational weights as independent random variables, we can perform
individual statistical tests on their parameters as a way to prune the network. We can for example
compute their ratio |µw|/σw and calculate the probability of obtaining an absolute mean |µw| bigger
than the one observed if we assume that the mean is actually 0. This is known in descriptive statistics
as the p-value of the null hypothesis H0, where in this case the null hypothesis is that µw = 0. Given the
big amount of samples that we have, we consider |µw|/σw to have converged to a Gaussian distribution,
instead of a Student’s t-distribution.
First, we carried a study of the distribution of the |µw|/σw ratio for the different layers of the final
sets of weights. Figure 3.29 shows this distributions for the architecture with ζ = 0.001, σ2 = 3, and
dropout rate 0.2. As we can observe:
• The highest quantiles are at 0 and they monotonically decrease with the ratio.
• WH has the smallest tail, meaning that, proportionally, its weights are less statistically significant
than the rest of the layers. This is consistent with the weights seen in Figure 3.26
• WS has the biggest tail, caused by the weights multiplying H ◦U seen before.

of the ensemble began to quickly decrease.

CHAPTER 4
Conclusion
During this thesis we managed to effectively develop and train deep learning methods that can solve the
Q&A problem with a high enough accuracy for their practical use in a limited environment. Our best
performing single model is a Bayesian modification of the ELMo + BiDAF architecture, achieving a F1
score of 82.2% and an EM accuracy of 73.4% over the SQuAD 1.1 dataset. An ensemble of 10 different
runs of this model improves the accuracies, reaching a maximum F1 score of 86.2% and a maximum
EM accuracy of 78.0%
The models implemented are the most reliable for the “Who”, “What”,“When” and “Where” types
of question, whereas estimated answers that are composed by clauses, such as the types “Why” and
“How”, should not be trusted in most situations. The system is designed for passages of around 200-300
words maximum with no adversarial examples that could confuse the network. We also found that the
architectures are biased towards giving answers located at the beginning or at the end of the passage.
The study performed shows that these limitations comes from the inherent properties of the SQuAD
1.1 dataset and the limited temporal relationships that the bi-LSTMs are capturing.
The inclusion of contextual embeddings significantly improves the accuracy with respect to the baseline
model. Just by replacing the initial embeddings of the BiDAF model (CNN + GLoVe) by the contextual
ELMo embeddings, the EM and F1 scores are increased by around 5%. In this line, the newly published
BERT embeddings would most likely improve the performance of the model. It is expected that almost
all future work will be based on the new architecture that BERT is based on, transformer.
There appears to be fundamental limitations in the learning capabilities of the BiDAF structure. Diverse
experimentation has shown that the bi-LSTMs are not effectively learning complex relationships among
the words, limiting the contextual information that the architecture used to give the final answer. Also,
the attention mechanisms of the BiDAF favors creating memory-less representations of the words in the
contextual layer, which might disregard useful information among the words. Both of these issues seem
to be improved in the BERT architecture, although since it recently came out, we could not include it
in the analysis and is left as future work.
We proposed a simple modification of the Bayes by Backprop algorithm that allows easier control of
the regularization caused by the KL term of the loss function. We tested this modification in a deep
learning architecture such as the BiDAF, a line of experimentation that is currently in exploration
phase. This lead to implementing the Bayes by Backprop algorithm over highway networks, with their
special peculiarities seen in Subsection 3.2.2. To the best of our knowledge, this is the first publication
of the BBP algorithm being applied to highway networks. In June 2018, Variational Inference has
been successfully applied to Convolutional Neural Networks for the first time [34], we hope that our
contribution will promote future research in applying the BBP algorithm to deep architectures.
Even though the Bayes by Backprop algorithm is conceptually appealing and works well for the toy
examples, it seems that so far, the training algorithms yield final configurations of variational posterior
parameters with limited usability. This Variational Learning algorithm is effective at including regular-
ization during training and provide principled approaches for weight pruning. But it fails to find final
configurations of variational posterior parameter from which we can create Bayesian ensembles.
Just as it took several years to solve the practicalities of training deep learning models, with many
(initially) empirical techniques such as (1) dropout and weight decay for improved regularization, (2)
Xavier heuristics for weight initialization, (3) training algorithm fixes such as gradient clipping to avoid
==PAGE:72==
62

gradient explosion, and (4) gated architectures such as Highways, LSTMs and residual networks to
avoid gradient vanishing; we expect future advances in the Variational Bayes algorithm such as the
ones described in [47] to finally release the full potential of BNNs.
We hope that the empirical study carried out in this thesis, the high level of detail in the explanation
of the methods and the developed VB library will raise awareness of such architectures and promote
more empirical studies that will eventually find the missing piece of the puzzle.
Finally we would like to point out the recent explosion in publications related to the application of
deep learning methods to the Q&A problem, partially enabled by the public access to increasingly big
and complex datasets. The field is advancing at a fast rate and reaching human-like performance for

case of the SQuAD 1.1 dataset.
This has caught the attention of the biggest IT companies world-wide (Google, IBM, Microsoft...)

As previously stated, the intrinsic distribution X contains all the information we could ever obtain from
our set of observations X, and more importantly, it quantifies its uncertainty, how much randomness
there is in X, how much the sampled datasets can vary. If there would be no uncertainty, we would
always sample the same dataset D, this would implicitly mean that the circumstances in which the
dataset was drawn were identical and that the relationships between the observed values are completely
deterministic.
In reality, every time we sample a dataset, it is different from all others because there are factors
influencing the observations which are different in each observation, like for example the noise of our
measuring device. One of the fundamental assumptions of Science is: Under the same circumstances,
the output of an experiment should always be the same. In any real experiment we are never able
to generate the exact same situation and therefore the observed variables will change slightly, if the
variation is small enough, the replicability of the experiment is accepted and a theory can be accepted
or rejected.
In the epistemic interpretation of statistics, uncertainty can be defined as the lack of knowledge we have
about an event. Assuming this statement as true, it could be the case that the Universe is actually
deterministic, a theory commonly known as Laplace’s Demon [33], however that is a discussion outside
of the scope this thesis. Even if the Universe as a whole is actually deterministic, the observation of a
subset of variables is not, there is still uncertainty in our observations because we don’t know all the
dynamics and initial conditions that generated them.
Therefore there will always be uncertainty in our dataset, and once there is uncertainty, it is impossible
to know with 100% confidence what is the real distribution X of our dataset, it would require an infinite
amount of samples, measurement devices with infinite precision... Once we get over these facts, the
key question here is, how do we obtain the best possible approximation of the distribution X from our
dataset D?
There are different approaches to accomplish this task. In the most general case, we simply don’t make
any assumption about the statistical distribution of the samples. Each sample Xi could come from a
completely different distribution Xi ∼Xi, Xi ̸= Xj, ∀i ̸= j and could be related to the other samples
Xj, i ̸= j in any allowed way by the laws of probability. In this case, X describes the joint probability
of the set of random vectors X1, X2, ..., XN in a way that cannot be factorized.
In this scenario, a single dataset D would only give us a single observation of each of the individual
random variables Xid. This will make it impossible for any learning method to generate any reliable

67
estimation of the distribution X. The number of samples needed to properly estimate the distribution
grows exponentially with the number of random variables Xid, and we only have one sample of the
entire joint distribution.

Sadly, not all phenomena in the world are jointly Gaussian, and this modeling process becomes an-
alytically intractable as the complexity of the distributions grow. Luckily for us, there exists many
algorithms that allows us to learn from data without such explicit assumptions on its statistical distri-
bution, as a matter of fact, many of them not even offer any explicit statistical framework. A family of
these methods are the parametric models.
A.1.2
Parametric models
Any learning method aims to find some pattern that occurs frequently in our dataset, differentiating
the signal from the noise. Statistical methods offer us a framework to quantify the reliability of the
patterns found, generally at the expense of a set of assumptions about the distribution that the data
follows. Any learning algorithm at its core makes assumptions on the properties of the dataset and
has limitations on the patterns that it is likely to identify. Usually, the more assumptions we make
about the data, the easier is to learn its distribution (we need less samples). On the other hand, if such
assumptions are not met by the data, the performance of the model will suffer, as the learnt patterns
do not actually represent the real data.
In parametric statistical models we aim to approximate the true probability density function of our
dataset fX (X) by a function f(X|W, M) with learnable parameters W that interact together according
to the model specifications M.
f(X|W, M) ≈fX (X)
During training, our goal is usually to find the parameters W that maximize the likelihood of the
observed data D, using the proposed model M. This ideal set of parameters W ∗
ML usually does not
have a closed form solution and gradient search based methods are commonly used to find good local
maxima.
W ∗

W {f(D|W, M)}
The model M is defined by all the choices taken while designing the learning algorithm. It will tell us
how the parameters of the model interact with each other in order to compute the likelihood function.
The model itself can have hyper-parameters H, defined before the training process which are not
modified during the learning process. As examples of choices of the model and hyper-parameters in
Neural Networks we have:
• The size of the network: For example the number of layers in deep architectures and the number
of neurons in each layer.
• The architecture of each layer: Dense Layer, Highway, Convolutional layer...
• The non-linearity of the activation functions: hyperbolic tangent, sigmoid, ReLu...
As we can see, there are many many choices to be made in the design of a model M. In practice, several
questions arise:
• How should we decide which one is the best model to use?
• How do we find the best possible set of parameters W ∗for the dataset, without falling into
over-fitting?
• How do we know which parameters are significant for the model and which ones are not?

A.1.3
Statistical Supervised Learning
So far we have aimed to build a complete statistical model of all the variables X in our dataset D, also
mentioning that learning the distribution function does not imply that we can easily draw samples form
the distribution, usually approximation sampling methods such as RS and MCMC are used.
In supervised learning models we are not so interested in finding the statistical relations among all
variables but rather the relationship between a set of input variables X, usually called the explanatory
variables, and a set of output variables Y , usually called the target variables. In a regression problem
the target variables are continuous and in a classification one, they are discrete. Our dataset is then
composed by a set of tuples (Xi, Yi), i = 1, 2, ..., N.

Ideally, for a given Xi we would like to know the likelihood/probability of any value that Yi could take.

approximated by the function f(X|θ, M), in this case we approximate the true conditional distribution

A.2
Bayesian Inference
So far, everything we have seen in this Section belongs to the descriptive statistics field, in which we
have a set of observations X and we aim to estimate their statistical properties by means of finding the
value of a set of parameters W, that interact together according to the model M.
Although these parameters W found are point values, its estimators are random variables, they have
uncertainty. This will always be true since the estimators are obtained as a deterministic function of
the dataset which are considered samples from random variables. If we know the distribution of our
learned parameters W, we can perform statistical tests over them as previously seen.
In a Bayesian statistics, we consider the parameters of our model, (the weights W in the case of NNs)
as hidden random variables, instead of just point values. The marginal distribution of the parameters
in the model is called the prior, characterized by its parametric density function p(W|ϕ, M) where ϕ
are now the distribution parameters of the prior over the hidden variables W.
W|ϕ, M ∼P(W)
The distribution parameters of the prior are not usually learnt from the data, but rather, their correct
values are found like any other hyper-parameter in the model. The prior distribution is our initial
guess of possible values of the weights before seeing any data. As we will see, it is commonly used
with the purpose of including our beliefs into the model, or more commonly in Neural Networks, as a
regularization term to give more probability to simpler structures of the weights.
Hidden random variables (or latent variables) are variables that cannot be observed directly, as opposed
to the observable variables X. Latent variables are estimated from the observable ones and can be used
for different purposes:
• Dimensionality reduction: In algorithms such as PCA or Variational Auto-encoders, we use latent
variables to model observable ones X in a lower dimensional space. [12].
• Clustering: In the Expectation Maximization algorithm [21], hidden variables are used to estimate
the cluster from which the samples came from.
• Regularization: For the purposes of this thesis, we model the weights of the network as random
variables in an attempt to improve regularization [15][2].
Treating our parameters W as random variables allows us to compute what is called their posterior
distribution f(W|X, ϕ, M) using Bayes’ Theorem. This distribution tells us the likelihood of the set
of hidden variables W for a given sample vector X. Each sample X can have a different associatied
posterior distribution.
f(W|X, ϕ, M) = f(X|W, ϕ, M)p(W|ϕ, M)
f(X|ϕ, M)

• The prior p(W|ϕ, M) tells us the marginal likelihood of any possible set of weights of the model.

favouring certain configurations and penalizing others. A common choice is to give high likelihood
to small values of W to constraint the size of the vectors, as we will see in later sections, this
generates regularization methods such as L1 and L2 regularization.

Artificial Neural Networks
In this Chapter we give an overview of Artificial Neural Networks (ANNs) from the point of view of
probabilistic parametric models.
We start by defining a neural network in terms of the previously
seen concepts in Chapter A, focusing on the classification task, using the Multilayer Perceptron (MLP)
architecture as basic example. Later on we review the most common techniques that are used for tunning
the parameters of the models. We derive the basic learning algorithms for the classical loss functions used
in classification problems and we include the Variational Bayes algorithm know as Bayes by Backprop.
The Chapter closes with the description of a set of ANN architectures that are implemented in this
thesis.
ANNs are a family of parametric models know for their high flexibility and applications in almost any
area of machine learning. Their high expressivity allows them to approximate any continuous function
to any degree of precision. This is known as the universal approximation theorem [9]. Even a NN with
a single hidden layer can be an universal function approximator, but the size of the network might need
to grow exponentially with the complexity of the task.
The way in which this type of models M combine their parameters W in order to learn from data
comes in many different flavors, and it is hard to describe them a broad enough description, but all
NNs incorporate in their structure a set projection units called neurons.
A neuron is modeled as a relatively simple system that applies the function o(·) to a continuous vector
X that it takes as input. The neuron computes a linear combination of the elements of its input vector
and, in most situations, it applies a non-linearity composed by adding a bias and applying a non-linear
function a(·).
o(X, W) = a(XT W + b) ∈R
where:
• W = [w1, w2, ..., wD]T ∈RD is the vector of weights of the linear projection and D is the dimen-
sionality of the input vector.
• b ∈R is the bias of the neuron.
• a(·) is usually a non-linear function such as tanh(), sigmoid(), ReLu()... known as the activation
function.
The number of neurons in a ANN can vary enormously, going from a handful to several millions depend-
ing on the difficulty of the task. In most architectures, sets of neurons are structured in parallel, taking
the same input vector X, forming the so-called layers which create a H-dimensional representation of
the input vector, where H is the number of neurons in the layer.
These layers are usually stacked in what are know nowadays as deep learning architectures. The input
vector of features X is propagated form layer to layer, creating increasingly complex transformations
of the input. As we will see, this gained expressiveness can reduce the number of neurons needed to
solve the task when compared with shallow NNs but it comes at a price of a more challenging training
process.
From now on, we will denote the set of all weights in the network as W, and the set of weights of
particular neurons will have corresponding subindexes.
These weights are usually iteratively tuned

RNNs are usually trained using a slight modification of the BP algorithm called the BackPropagation
Through Time (BPTT) algorithm. To derive the updating gradient formulas we need to repeatedly
apply the chain rule over the recurrent function gm(ht−1, Xt). Once again, the automatic differentiation

An RNN with a big enough hidden state dimensionality DH, ht ∈RDH, can approximate any sequence-
to-sequence mapping to any desirable degree of accuracy.
Basic RNNs such as Vanilla are Turing
complete and can therefore implement any algorithm, but they suffer from practical issues during
training. Due to the recurrent application of the non-linear function gm(), the information from previ-
ous time-instances becomes more and more distorted in each time-iteration, producing vanishing and
exploding gradients when applying the chain rule through the propagation of information.
The problem with normal RNNs such as the Vanilla RNN is that they are good at learning short-term
dependencies between the samples in the chain, but it hard for them to learn long term dependencies,
mainly due to the distortion of information and gradient vanishing. Many alternative RNN architectures
such as the Long-Short Term Memory (LSTM) RNNs have been proposed to solve this issues.
