Abstract
In this thesis we will present the three-dimensional bin packing problem (3D-
BPP) that consists of packing a given set of items into as few bins as possible.
The 3D-BPP is a well researched problem as it has various real world ap-
plications and is known to be NP-hard and dicult to solve in practice for
standard MIP solvers, especially for large instances.
In this thesis a heuristic approach based on the Danzig-Wolfe decomposition
is developed. To speed up the column generation process, several heuris-
tic algorithms are developed. A GRASP based rst-t decreasing heuristic
is developed to nd a promising initial solution for the column generation
process. The pricing problem in the column generation is a 3D knapsack
problem, which is also known to be NP-hard.
To speed up the pricing
problem two heuristics are developed, one simple and fast and another more
advanced and thus a little slower. The simple heuristic is based on a rst
t decreasing sorting of the items on dierent criteria and greedily packs the
items in an extreme point based manner. The more advanced heuristic is
based on the simple heuristic but makes use of a GRASP framework to solve
the pricing problem.
We use standard benchmark instances from the literature to compare the
state-of-the-art heuristics against our column generation based heuristic. The
tests showed that given a 60 second time limit our algorithm performs better
than the compared heuristics from the literature with a gap in the results
of respectively 1% and 0.65%. However our heuristic is outperformed when
a longer time limit is given with at gap of up to 0.16%. We will discuss
suggestions for improvements in the conclusion, chapter 8.

Introduction
In section 1.1 we will briey describe the bin packing problem in general.
A more detailed explanation of the three dimensional case will be given in
section 1.2, where possible versions and applications of the problem will also
be introduced. The aim of the thesis is presented in section 1.3 and the outline
of the thesis in section 1.4.
1.1
Introduction to the bin packing problem in
general
The bin packing problem(BPP) aims to pack a set of items into as few bins
as possible. Even though the problem can be well dened with a MIP model,
it is known to be NP-hard and current techniques are only able to solve
small instances of around 20 items for the three-dimensional case, but it is
easy for the one-dimensional case.
The applications of the bin packing problem can both be packing prob-
lems but also cutting problems, where the items are objects that needs to be
cut out from an element (the bin) this is applicable in all dimensions. The
one dimensional bin packing problem can be used for job scheduling as well.
The bins represent identical machines and the items represent jobs that needs
to be done, the jobs take a certain time. To simplify, all machines can do the
same jobs and only one at a time. We want to know how many machines we
need to do all the jobs within a certain time.
For the 2D- and 3D-BPP the items and bins are mostly considered to be
rectangular, but in some industries irregular items are needed to be cut from
rectangular materials. When solving the 2D and 3D cutting problems there
might be an extra constraint for having guillotine cuts. Guillotine cuts means
that the cuts must be made through the whole bin or the whole subpart of the
bin that have already been cut o. We have shown an example of guillotine

Another version of the BPP includes the possibility of varying bin sizes
with dierent costs. For the 3D-BPP this could for example be applied to
container shipping. If a company wants to ship goods in containers and both
40ft and 20ft containers are available. The shipping company might have
dierent charges depending on which containers is chosen.
A dierent version could be to have some kind of prot on the items and
the option to leave some items out of the packings, because it can be bene-
cial to do so. This relaxes the constraint that all items need to be packed.
This problem is closely related to the knapsack problem where each item has
a prot and you need to choose the subset of items with highest prot to
pack in a single bin.
There could also be some constraints on how the items are placed in the
bin. Trivella and Pisinger [18] includes a load balancing constraint within
the multi-dimensional BPP. This means that the average center of mass of
the packed items should be as close as possible to the center of the bin.
1.2
3D-BPP problem description
The three-dimensional bin packing problem (3D-BPP) that is considered in
this thesis aims to pack a set of items J = {1, ..., n} into as few bins as
possible. Each item j ∈J has a given width wj, depth dj and height hj. The
bins are identical with dimensions W, D and H. The amount of bins are
assumed to innite. Packing the bin are said to be orthogonal which means
that we pack the items such that their edge is parallel to the corresponding
bin edge. Furthermore we do not allow rotations of the items. It is assumed

that all input data are positive integers, but all the algorithms will also work
for real numbers. Furthermore it is assumed that wj ≤W, dj ≤D and
hj ≤H ∀j ∈J. The only restrictions that are considered is that all items
must be packed, such that they do not exceed the bin walls nor can the items
overlap. Thus we do not impose the guillotine constraint or any other special
constraints. We have some denitions.
Denition 1. An item j is a rectangular object that must be packed. It is
associated with a width, depth and height, respectively wj, dj and hj.
Denition 2. A bin b is a rectangular object that can contain items. It is
associated with a width, depth and height, respectively W, D and H.
Denition 3. An instance I contains information about the number of
items n, the dimensions of each item and the dimensions of the bins. All
bins have identical dimensions.
Denition 4. A packing is a description of which items are in a bin and
where they are placed.
Denition 5. A solution is a set of packings where all the constraints are
satised. The objective value of a solution is the number of bins used.
We can dene the 3D-BPP mathematically. For the placement of an item
j let (xj, yj, zj) dene the coordinates of the left-back-bottom corner of the
item and let the left-back-bottom corner of the bin be dened as (0, 0, 0).
The item must be within the bin boundaries W, D and H, so
0 ≤xj ≤W −wj
0 ≤yj ≤D −yj
0 ≤zj ≤H −zj

to overlap, i.e. the following cannot be true. Each line is true if an overlap

1.3
Aim of the thesis
The thesis aims to develop a new method for nding near-optimal solutions
to the 3D-BPP. This is done by combining the potential of column generation
with ecient extreme points based heuristics. We will devise a constructive
heuristic that quickly nds solutions within a reasonable gap from the lower
bound.
Then we will apply a heuristic column generation method which
uses a chain of heuristics for solving the pricing problem. We measure the
performance on the benchmarking instances from the literature and expect
the new algorithm to compete with the state-of-the-art heuristics on instances
with up to 200 items, being non-dominated by any of the existing ones.
1.4
Outline of the thesis
The thesis consists of eight chapters. Chapter 2 reviews the existing literature
in the bin packing eld.
Chapter 3 describes the lower bound presented
by Boschetti [1]. Chapter 4 describes the extreme points and introduces a
constructive heuristic. Chapter 5 introduces column generation and proposes
a way to combine column generation with heuristics approaches to obtain
promising results for large instances. Chapter 6 describes the nal algorithm
proposed for solving the 3D-BPP. Chapter 7 shows the results obtained by
the algorithm and compares to the state-of-the-art heuristics. Finally chapter
8 sums up the conclusion of the thesis and proposes methods for further
developing.

Literature review
The bin packing problem is a classic NP-hard combinatorial optimization
problem. It has been researched for more than 50 years and since it nds so
many practical uses it is one of the most researched optimization problems.
In this chapter we will look into the more recent research. In section 2.1 we
will describe the lower bounds that have been proposed. Section 2.2 shows how
column generation has been used for the 2D-BPP. In section 2.3 we will de-
scribe the exact method proposed in the literature with the use of corner points.
Section 2.4 introduces the extreme points and explains a GASP heuristic that
uses them. In section 2.5 the state-of-the-art heuristics TS2PACK and GLS
are described.
2.1
Lower bounds on the 3D-BPP
There are several lower bounds presented in the literature. The rst and most
obvious lower bound is the continuous lower bound L0 presented in [1,9,11].
Here it is simply the sum of volumes of each item that is considered and di-
vided by the volume of the bin and thus the running time is O(n). This lower
bound is poor and Martello et al. [11] prove that the asymptotic worst-case
performance of this bound is 1/8 even if rotation of any angle is allowed. This
means that the optimal solution value can be up to 8 times the L0 value.
Martello et al. [11] propose a new lower bound L1, where the problem is
reduced to three one-dimensional sub-problems, giving one lower bound per
dimension. The lower bound L1 is dened as the largest of these three lower
bounds and can be computed in O(n2) time. However L0 and L1 does not

2.2
Bounds from column generation
A MIP model for solving the 2D-BPP is presented by Onodera et al. [13],
but the extra dimension can be added so it also applies for the 3D case. The
MIP model presented by Onodera et al. [13] is dicult to solve for ordinary
MIP-solvers since it contains several symmetric solutions. Pisinger and Sig-
urd [16] present a set covering model instead. The LP relaxation of the set
covering model provides a fairly tight lower bound but it also needs a variable
for every feasible packing of a bin. Pisinger and Sigurd [16] propose the use
of delayed column generation, such that only a subset of feasible packings
are considered. The restricted master problem is the relaxed set covering
model and the initial packings are provided by a greedy heuristics. The pric-
ing problem is to nd a new packing with the most negative reduced cost.
By dening the prot of an item as the negative of the corresponding dual
variable, Pisinger and Sigurd [16] have converted the pricing problem into a
2-dimensional knapsack problem.
To solve the pricing problem they use a fast heuristic approach.
The
downside of this is that it does not guarantee the most negative reduced

2.3
Exact solution method for the 3D-BPP
Martello et al. [11] propose a combination of exact and approximation al-
gorithms for solving the 3D-BPP. Both use a method denoted as ONEBIN,
which given a subset of items can nd the best lling of a bin, i.e. where
the sum of volumes of items in the bin is maximized. ONEBIN starts the
lling from the left-back-bottom corner (0,0,0) when the bin is empty. The
new points where an item can be placed are called corner points and are
computed by an algorithm with time complexity O(n2).
The rst of the two algorithms proposed by Martello et al. [11] is an exact
algorithm. It uses a main branching tree that assigns items to bins without
specifying the positions of each item. The items are pre-sorted according to
nonincreasing volumes. At each decision node the next item is placed, in
turn, in all open bins. If the assignment of an item to a bin cannot be di-
rectly rejected by a lower bound. Then an approximation algorithm tries to
solve the sub-instance by applying two heuristics, the rst one uses a layer-
building principle derived from the 2-dimensional shelf approaches the other
repeatedly lls a single bin with the use of ONEBIN. If the sub-instance can
be lled into a single bin, the solution is accepted. If no solution is found
the optimal solution is found by the use of ONEBIN. If the item does not t
into any bin a new bin is opened.

2.4
Extreme points
Crainic et al. [2] proposed a new rule for placements of items in three di-
mensions called extreme points. Extreme points extends the corner points
concept. Corner points are the non-dominated locations where an item can
be placed into an existing packing. For the 2D case that is when the enve-
lope of the items in the bin changes from horizontal to vertical. The corner
points for 3D can be found by applying the 2D algorithm for each distinct
z-coordinate where a packed item "ends" [2, 11].
When placing an item
with its left-back-bottom corner at (xj, yj, zj) it generates a series of six po-
tential extreme points. The points are generated by projecting the points
(xj + wj, yj, zj), (xj, yj + dj, zj) and (xj, yj, zj + hj) on the orthogonal axes.
With these new extreme points dened, Crainic et al. [2] proposes the use of
rst t decreasing(FFD) and best t decreasing(BFD) constructive heuristics
for 1D-BPP converted into a 3-dimensional case and with the use of extreme
points. Both methods require an initial sorting of the items. In contrast to
the 1-dimensional problem there are many ways of sorting the items. In the
1-dimensional case, we only have volume, but in the 3-dimensional case there
is both width, depth, height and combinations.
The FFD assigns an item to the rst bin where it ts. BFD assigns an
item where the residual available volume is maximized. Both heuristics open
a new bin when an item cannot be assigned. For the 3-dimensional versions
the EP-FFD heuristic tries to place the item in all the extreme points of a
bin and if it does not exceed the bin boundaries or overlap another item the
item is placed and the extreme points are update. The EP-BFD is slightly
dierent. It considers all extreme points of all bins and the item is placed
where it is feasible and a merit function is maximized. The merit function
can be of dierent types according to how the best t is dened. The time
complexity of updating the extreme points is O(|Ij|) where Ij is the set of
items already in the bin. The time complexity of EP-FFD is O(n3) and for
EP-BFD it is O(n3 + nmax(n, O(UMF)) where O(UMF) is the time com-
plexity of the merit function, if O(UMF) ≤O(n2) then the time complexity
of EP-BFD is O(n3).
Crainic et al. [2] have tested an extreme point based and a corner point
based FFP and found that the extreme point based FFP performed bet-
ter. Crainic et al. [2] also tested dierent versions of EP-FFP where the

4.1.2
Extreme points
The extreme point(EP) idea is an improvement from the corner points con-
cept and was rst published by Crainic et al. [2]. The idea is the following.
When placing an item j with dimensions wj, dj and hj we place it with its
left-back-bottom corner at position (xj, yj, zj) and from there it generates
new potential points where the next items can be placed. These points are
the extreme points. The new points are computed by projecting the points
with coordinates (xj + wj, yj, zj), (xj, yj + dj, zj) and (xj, yj, zj + hj) on the
orthogonal axes of the bin. See Figure 4.2 for an illustration of the concept.
Every time a new item is placed in the bin the EPs must be updated. To
nd the new EPs we use the following algorithm.
==PAGE:32==

21
If the bin is empty the item is placed at (0, 0, 0) and the generated EPs
are (wj, 0, 0), (0, dj, 0) and (0, 0, hj). In any other case, the item is placed at
(xj, yj, zj) and the new EPs are generated by projecting:
 Point (xj + wj, yj, zj) in the direction of the y and z axis
 Point (xj, yj + dj, zj) in the direction of the x and z axis
 Point (xj, yj, zj + hj) in the direction of the x and y axis
Each point is projected on all items that lies between item j and the
wall of the bin in the direction. If there is more than one item, the closest
projection point is chosen. If there are no items, the point projected at the
bin wall is chosen.
When all projections have been made we have six extreme points. We check
the six extreme points for doublets and remove them. These new EPs must
be added to the list of EPs already present in the bin. Again we avoid adding
doublets and we sort the list of EPs in nondecreasing order of z, y, x.

Crainic et al. [4] claim that when the items are sorted, their constructive

are usually found in the middle portion of the item list and that a small
number of items should be swapped. Crainic et al. [4] introduces a scoring
system that the items are sorted according to.
In this constructive heuristic we want use a similar approach. We want to
swap items in the item list that are similar in the sorting value. We do this
by adding noise to the sorting values. The noise is set to ±10% of the sorting
value and thus items that are closer to each other have a higher probabil-

Exact method
In this chapter we introduce an exact method for solving the 3D-BPP. First
the MIP model is introduced in section 5.1. Then a set covering model is
introduced in section 5.2. We see that the set covering model can be solved
only for a small number of items and therefore we implement the column
generation method in section 5.3 and 5.4. We introduce dierent branch-and-
price approaches for obtaining integer solutions in section 5.5. It is discovered
that the exact MIP model for the pricing problem is time consuming, so in
section 5.6 and 5.7 we introduce heuristic methods for solving the pricing
problem.
5.1
MIP Model
The 3D-BPP can be formulated as a Mixed Integer Program (MIP). Assume
we have a set of n items J = {1, .., n}. Each item j has the width wj, depth
dj and height hj. We also have bins with width W, depth D and height H.
Assume an innite amount of bins is available. The goal is now to formulate
a model, such that all items are packed into the bins without overlapping or
rotating and by using as few bins as possible. The model is based on the
MIP model used by Pisinger and Sigurd [16] for the 2D-BPP and the MIP
model by Pisinger and Egeblad [14] for the 3D-KSP.
We start by introducing the decision variables. We dene the coordinates
of the left-back-bottom corner of an item j ∈J as positive integer vari-
ables (xj, yj, zj) while the left-back-bottom corner of the bin is (0, 0, 0). We
have another positive integer variable aj that indicates the bin number con-
taining item j. For each pair of items j, k ∈J where j < k we introduce

5.5
Branch-and-Price
In general column generation may not generate the set of columns needed
for the RMP to contain the optimal integer solution. Branch-and-Price com-
bines Branch-and-Bound with column generation, such that at each branch-
ing node we start column generation again [5]. Then for each node we will
nd either a fractional solution, an integer solution or an infeasible solution.
If we nd a fractional solution we will again have to explore the children of
that node, in general we will also add the columns generated in branching
node to its children. If we nd an integer solution this is an upper bound, if
it is equal to the lower bound we are done, if not we proceed from another
branch. We always keep track of the lowest upper bound we have found so
far. If the solution is infeasible we do not have to explore the children of the
branch node. There can be dierent ways of branching, we want to choose
a way such that the pricing problem does not change too much or gets too
complicated.
Mahvash et al. [10] suggest a depth rst branching heuristic that xes
fractional ωp to 1 if it is greater than a specic value. If no variable with
fractional value is greater than the specic value then the variable with high-
est fractional value is xed to 1. After xing a variable the column generation
procedure is started again and runs until there is little or no change in the
RMP solution. They claim that repeating this strategy leads to either integer
or infeasible solutions.
Another approach suggested by Mahvash et al. [10] is to choose two items
to branch on and force them to be in the same bin or in dierent bins. If item
j and k are forced to be packed in same bin we remove all packings where
they have been packed individually from the RMP and add a constraint to
the pricing problem that says sj = sk.
In the other branch all packings
where both j and k are present are removed from the RMP, and for the
==PAGE:51==
40
5.5. BRANCH-AND-PRICE
pricing problem the constraint sj + sk ≤1 is added. The two items j and k
is chosen from one of two selection rules:
1. Select the pair j and k with maximum volume from a packing with a
high fractional value
2. Select the pair j and k with maximum volume from a packing with
fractional value closest to 0.5
Again they chose a depth rst approach where pairs of items are forced
to be in the same packing. This algorithm is reported to be very time con-
suming.
In this report we have not implemented any branch-and-price method which
means that any feasible solution we get from the CG is an upper bound (UB)
and not necessarily an optimal solution. For more details on branch-and-price
for the 3D-BPP see Mahvash et al. [10].

Conclusion
In this chapter the results are discussed and the conclusions are summed up.
In addition section 8.1 gives some suggestions on how this work can be used
for further research.
We have implemented a GRASP EP-based constructive heuristic FFCH
that was able to nd solutions within a reasonable gap from the lower bound
LB. The tests showed that the best results were obtained when the items
were presorted according to non-increasing volume. It was also shown that
the FFCH often was not able to converge towards the lower bound LB, with
no signicant decrease in the gap between 5 seconds and 300 seconds. The
FFCH was then found suitable to be used as constructive heuristic.
We have shown that the MIP model is only able to solve the 3D-BPP to
optimality within a given time limit of 1000 seconds for a small amount of
items n ≤15. We then try a set covering model, which is also a MIP but
in general easier to solve to optimality. The set covering model needed a
set of all feasible packings, and we were not able to construct that within a
satisfying time limit, since the algorithm to do so was rather poor. The set
covering model was able to solve problems to optimality with up to n = 19
items and the set of all feasible packings generated heuristically.
In an attempt to solve larger instances we tried column generation, but as
the pricing problem is a 3D-KSP and thus hard to solve to optimality, we
try to nd pricing problem solutions heuristically.
The heuristic pricing problem approach is to rst use a simple and very
fast heuristic FP2. FP2 sorts the items according to some criteria depending
on the prot and then greedily adds items to a bin, when there can not be
added more items to the bin we see if the packing has negative reduced cost.

8.1. FURTHER WORK
If the simple heuristics fail to nd a packing with negative reduced cost, we
will use a more advanced heuristic. The advanced heuristic is a GRASP
based FP2 denoted GFP2. Like the constructive heuristic GFP2 adds noise
to the sorting values and iterates until a solution is obtained or the time
limit is reached. The overall procedure is denoted as Column Generation
with Heuristic Pricing, CG-HP. The CG-HP is tested against the state-of-
the-art heuristics TS2PACK and GLS on the benchmarking instances from
the literature. The tests from chapter 7 shows that CG-HP have a good
performance in the beginning, beating the state-of-the-art heuristics on the
60 second time limit with a 1% and 0.65% gap on the TS2PACK and GLS
respectively.
If a time limit of 150 seconds is given the gap between the three algorithms
gets smaller. TS2PACK is performing better than CG-HP and GLS with
gaps of around 0.2% and 0.3%. At a 1000 second time limit both TS2PACK
and GLS are performing better than CG-HP. CG-HP was able to pack all
the benchmarking instances in 731.0 bins in total, while TS2PACK and GLS
packed the instances in 729.8 and 730.2 respectively given the 1000 second
time limit.

Further work
There are several methods that can be researched to improve the CG-HP
algorithm. In this section we will discuss some of them.
We see from gure 7.1 that most work is done in the rst 500 seconds and in
the last 500 seconds there is a signicant decrease in the number of columns
generated. This time span may be utilized better. One possible way of doing
this could be to implement a more advanced heuristic for solving the pric-
ing problem, such that when GFP2 does not nd a solution within a given
time limit another heuristic approach will be executed. This heuristic may
be more time consuming but since we from gure 7.1 see a time window of
around 500 seconds, this might not be a problem. Another approach could
be to implement a dynamic noise, such that if no solution with negative re-
duced cost is found by GFP2 within a given time limit, the noise level will
increase, thus making it possible for new solutions to be explored.
Table 7.2 and gure 7.1 showed that good solutions were obtained in 60
seconds. It could be interesting to try a dierent strategy instead of letting
CG run for a full 1000 seconds; solve CG for 60 seconds, then solve the IP.
Remove the columns not used in the integer solution and run CG again for

TS2PACK or GLS, this might lead to better solutions.
It might also be advantageous to combine the CG-HP with a ALNS (adap-
tive large neighbourhood search). ALNS uses destroy and repair methods.
The destroy methods are used to remove a part of the solution, while the
repair methods are used to rebuild the whole feasible solution. ALNS chooses
probabilistically between the destroy and repair methods in each iteration.
Often a reward system for the performance of the individual destroy and re-
pair methods is used to change the probabilities. Possible ways to remove a
part of the solution can be seen in the list below, where g ∈{1, 2, . . . , m−1}.

 Remove a random subset of bins Bs ⊂B where LB(Bs) < |Bs|
