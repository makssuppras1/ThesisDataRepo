Abstract
Increased task difficulty can increase your cognitive load. When tasks get too
difficult they can cause an overload in the working memory. When that occurs,
it can happen that although the task was solved the participant did possibly
not learn anything from solving it. To prevent this, a system that detects the
rises in the cognitive load could be developed. This study is about conducting
an experiment to collect labelled data. Based on the data a classifier is trained.
During the experiment, participants solved tasks while having an eye tracking
and galvanic skin response (GSR) devices collecting data about their biomet-
rics. The experiment consists of N-back, calculation, and code example tasks,
as well as answering questions regarding the tasks and to collect more informa-
tion about the participants. There were 24 participants that participated in
this experiment. From the data collected, the pupil dilation, fixation duration
per minute, and SCR normalized, all showed statistical significance when com-
pared to the task difficulty, weighted TLX, the correct answers and how much
time they spent solving the task. The research question was answered by train-
ing a classifier. That classifier predicted the correct task difficulty 64.789% of
the time. This task difficulty was based on the Weighted NASA-TLX score
collected from the participants. That means that from this data, a classifier
can predict the task difficulty for the participant based on the Weighted TLX.

Preface
This thesis was prepared at DTU Compute in fulfillment of the requirements
for acquiring an M.Sc. in Engineering. The thesis deals with the problem
of source code reading by detecting the needs for intervention using an eye
tracker and GSR device. The thesis consists of designing and conducting an
experiment to get labeled data. This data is then analyzed and used to train
a classifier.

CHAPTER 1
Introduction
During almost every aspect of daily life, a person uses something that has
been programmed. The importance of programming is increasing every year.
Whether it is your phone, website, or application on your phone, it has a
development stage, and then it needs maintenance. When a developer has fin-
ished a product, it is most often only a version of the product. That product
will then need maintenance or updates, at least if it wants to follow modern
practices [1].
When keeping software up to date, it can occur that someone else, other than
the original developer, needs to update it. It can also happen that a software
developer quits their job, and a replacement needs to take over development.
The new developer has perhaps never worked or even seen the code before,
which can cause difficulties in understanding the code. It is a known problem
that software engineers change jobs quite often. There can be many reasons
why they leave their jobs, for example, a high demand for quality employees or
higher salary at other workplaces [2]. This is called job hopping; it is when em-
ployees are often relocating to new firms that have new and better products [3].
The results from the high rate of changes in employment can cause extra work
for others. That is, new employees have to take over the work of the previous
employee. It is not always easy to take over someone’s code and start adding
to it. The individual has to have an understanding of what the code is doing

and how it works. The understanding of the code is fundamental to start
building on it. It is difficult to jump straight to adding to the code, without
knowing how it works. It depends on the person who wrote it, how easy the
source code reading would be. Did they leave comments inside the code to
make it more clear, is there a lot of unnecessary code that is not relevant
for the program. This is only a fraction of the things that make source code
reading difficult. With an increase in the task difficulty, the cognitive load
increases until it reaches the maximum mental capacity [4]. Cognitive load is
how much of the working memory is being used. The working memory has
a limit of how much can be processed, and when this limit is reached, it can
have effects on what was learned during the task. This means, for example,
when an individual is solving a task that makes him reach his working memory
limits, even though he solved the task, he might not have learned anything
from it. That is because of an overload of his working memory capacity [5].
There are a lot of factors needed to create a system that detects when the user
has difficulties understanding what he is reading or doing. These intervention
systems will then hopefully prevent overload for the working memory. There
is a need for a machine learning classifier to detect whether the user needs
intervention or not, depending on the cognitive load. The classifier has to
be built on biometric data such as pupil dilation, eye tracking, galvanic skin
response (GSR), etc. In order to start training this classifier, the data for
that needs to be collected. The best way to measure and estimate the cogni-
tive load is to combine two different measurement techniques, subjective and
psychophysiological (objective). Subjective is, for example, a rating scale and
psychophysiological is the combination of body and mental measurements, for
example, heart rate and pupil diameter [6]. Therefore, the first step is to have
an experiment to collect the data needed to continue with further analysis and
to train the classifier.

1.1 Scope of the Thesis
3
1.1
Scope of the Thesis
This study is about collecting labelled data.
That is done by creating an
experiment with multiple tasks that participants need to do while collecting
their biometrics at the same time. There were three different tasks that the
participants had to perform, that have all been used before in other studies to
measure the participants’ cognitive load. The tasks that they performed were
N-Back, calculation, and code example tasks [4] [7]. It has been shown that
pupil dilation [8], fixation [9], and galvanic skin response [10] can indicate the
cognitive load of the participant. That is, when the value increases for these
factors, it can indicate that there is an increase in the participants’ cognitive
load [8] [9] [10]. Therefore, the focus was on collecting that data from the
devices used. Tobii Screen-based eye-tracker and their software Tobii Pro Lab
was used for eye tracking. Shimmer 3+ was used to collect the galvanic skin
response data. Then there were questionnaires about the participants and
NASA Task Load Index, which is a subjective method to get the participants
assessment of the workload [11]. By applying these multiple types of mea-
surements of the cognitive load together, the relationship between them can
be used to determine the cognitive load [12]. That is if the participant was
having difficulties with the task or even if he thought it was easy. From this
data, a data set table was created to train a classifier. As mentioned above
the goal was that the classifier would then detect the difficulty of the task that
the participant was doing.

Systems can be used for beneficial needs by adapting to their task difficulties.
That is possible by measuring and estimating the users’ cognitive load. Mea-
suring cognitive load would, therefore, allow interactive systems to respond to
users depending on the task difficulty. Task difficulty can then be measured
by detecting pupil diameter using eye trackers [13].
Lise Perkhofer and Othmar Lehner did two separate experiments in visu-
alizing information. They used multiple measurements to create one measure.
The measurements that they used where blinks, saccades, and fixations. They
said that to measure the cognitive load, the time to complete the task, and
the outcome from the task has to be analyzed. Furthermore, subjective mea-
surements like NASA Task Load Index (TLX) should also be used [14].
Subjective measures like NASA Task Load Index is known as NASA-TLX,
is categorized as a multidimensional rating scale that gives an assessment of
the cognitive load [4]. This is built on the assumptions that the participant
is capable of discussing his mental effort and the cognitive load after a task
[6]. NASA-TLX is a rating scale that is based on six different subscales. It
consists of rating and comparing these subscales and the outcome from it is a
number that can indicate the difficulty of the task [11].
Nyström et al. discussed what factors can have effects on the quality of the

Nourbakhsh et al. researched the cognitive load using galvanic skin response
(GSR). They measured both the average GSR and accumulative GSR. They
found out that the results from only using the standard GSR average and the
accumulative were not significant. However, when this data was normalized,
they produced significant results, especially concerning the accumulative GSR.
To normalize the data, task value was divided by the mean value for all the
tasks combined. This is done separately for each participant. This is also done
to eliminate the physical differences between participants. They proved that
when measuring the cognitive load using GSR, there is a difference in using
the normalized average GSR or the accumulative GSR. That is, normalized
accumulative GSR gives a better view of the participants’ cognitive load [23].
Bakker et al. indicated that after an event that is stressful, the GSR rises
and might not return to the same state as before the event occurred. That
probably happens if there is no relaxation between tasks or if similar events
occur again. This results in a constant increase in the level of the baseline.
The goal of their experiment was to detect a pattern in the stress level. Their
results were that it is most likely not possible to determine the stress level by
only having information about the GSR. However, by collecting other data
as well, it may give better information about the stress level. Therefore, it
might be helpful in seeing a pattern between the GSR data and other informa-
tion collected. This additional information could be the annotation of events,
questionnaires [10], and NASA-TLX [11].

Shimmer3 GSR
Our body contains sweat glands all over our bodies. Galvanic skin response
is the measuring of those glands by using electrodes. These electrodes need
to be located on a skin area that has as many sweat glands as possible to get
higher qualitative data. Apocrine and eccrine are the two types of glands that
are distributed over our skin. Eccrine glands can be found in most places on
the body, and they give information about the ”emotional activity.”. The loca-
tion for the electrodes depends on the structure of the experiment. The most
common places to use electrodes are on the shoulders, feet, fingers, and the
palms. The main reason why these places are the most used ones is because
of the high density of the sweat glands [30].
A Shimmer device to measure the galvanic skin response (GSR) was already
implemented into Tobii Pro Lab.
Tobii gives you the data in the form of
GSR, skin conductance response (SCR), and event-related skin conductance
response (ER SCR). GSR levels can fluctuate much during the tasks, and as
mentioned above, the accumulative GSR gives a better understanding of the

used to give information about the effects of the stimuli on the participant.
These rises in GSR levels are called peaks, and this measurement is called
event-related skin conductance response or (ER-SCR). When these peaks are
not related to an event, they are called skin conductance response (SCR) [31].
Tobii Pro Lab provides the SCR and ER-SCR Amplitude. The amplitude
indicates how intense that peak was [32].

can either press M = memory or N = No, depending on if they think that
the letter appeared three letters ago (if in a 3-Back test, otherwise depending

there should be one number between the same number for the participant to
say yes correctly.

Calculation Tasks
The calculation questions created for the experiment were designed similarly
to what Chen et al. did. That is, there were three different task difficulties
from easy, medium, and hard. Easy questions had single-digit numbers, the
medium had two-digit numbers, and then the hard questions had three digit
numbers. There was a slight difference in the mathematical signs that were
used because the examples they used, only included plus(+) and minus(-)
signs, but in my questions, I also included multiplication and division. The
questions can be seen in Appendix A.1.2. Therefore, some questions might
be more difficult than others because of the mathematical signs. For example,
423 * 523 + 210 is a much more difficult and demanding calculation than 748
+ 216 + 956. However, the assumption was made that numbers with more
digits are harder to solve compared to the fewer digit numbers. The differences
in the mathematical signs are not taken into consideration when making the
assumption of difficulty for the question. Each participant got two minutes to
solve the calculation question. After two minutes had passed, the next slide
turns up, and the participant gets a wrong answer.

each question. That means a question that will end up with the Weighted

considered as hard as it gets.

3.2.6
Experiment Design
One of the most frequently used methods for designing experiments was used
for this experiment. This method is called a Latin square design [35]. The goal
of Latin square design is to reduce the chance of systematic error. This de-
sign is originally made with t amount of treatments that are repeated t times.
Therefore, each treatment only appears once in each column [36]. Because of
that, no participant should ever get the same sequence of questions as some
other participant has gotten before. However, since the Latin square design
is t x t, the number of participants and questions are needed to be the same.
For our case that did not happen and, therefore, the complete Latin square
design was not followed, but it gave the idea of having a randomize factor
on what question appears each time. The task order was not randomized, so
every participant completed the tasks in the same order, but the questions
inside each task were randomized.
Tobii Pro Lab offers an easy way to create experiments.
For this experi-
ment, there was one timeline used, called ’Timeline1’. Inside this timeline, all
stimulus or images were placed in the order they were presented during the
experiment. Inside timeline one, the stimulus for the N-Back tasks, fixation
crosses, calculations tasks, and code examples can be found. The structure of
the stimulus can be seen in Figure 3.8. When a folder was created for each
task, there was a possibility to choose between making the stimulus inside it,
as shuffled, replacement, or sequential. Shuffle means that the objects inside
the folder are shuffled, so they do not necessarily come in the same order as
in the folder itself. Replacement means that only one folder inside that folder
is chosen, for example, in Figure 3.9 only one of the 1-Back tasks sequences is
chosen. Sequential means that the stimulus inside the folder are displayed in
the same order as presented in the folder.

Research Question: Can a classifier detect when the participants have dif-

Hypothesis (H1): The higher the task difficulty (easy, medium, hard) the
higher the Cognitive Load (both subjective (NASA TLX) and objective (vari-
ous measures)).

H1.2: If Task Difficulty increases, then the Pupil dilation increases as well.
H1.3: If Task Difficulty increases, then the Fixation Count increases as well.
H1.4: If Task Difficulty increases, then the Fixation Duration increases as well.
H1.5: If Task Difficulty increases, then the Normalized GSR increases as well.
H1.6: If Task Difficulty increases, then the ER SR Normalized increases as
well.
H1.7: If Task Difficulty increases, then the SCR Normalized increases as well.
H1.8: If Task Difficulty increases, then the Total time solving the task increases
as well.
H1.9: If Task Difficulty increases, then the amount of correct answers will
decrease.
Hypothesis (H1) is based on comparing the independent variable to the depen-
dent variables. That is comparing the estimated Task Difficulty with other

Hypothesis (H2): There is a significant relationship between subjective
Cognitive Load and objective Cognitive Load.
H2.1: If Weighted TLX increases, then the Pupil dilation increases as well.
H2.2: If Weighted TLX increases, then the Fixation Count increases as well.
H2.3: If Weighted TLX increases, then the Fixation Duration increases as well.
H2.4: If Weighted TLX increases, then the Normalized GSR increases as well.
H2.5: If Weighted TLX increases, then the ER SCR Normalized increases as

H2.6: If Weighted TLX increases, then the SCR Normalized increases as well.

3.2.13
Hypotheses Testing
Aggregation was needed to perform subjective analysis, that is, compare the
different variables between different task difficulties for each participant. The
aggregation works that an equal amount of easy, medium, and hard tasks
is taken. Then it takes the average of all easy, medium, and hard for each
participant. An equal amount of easy, medium, and hard task are used and
compared together individually for each participant. As discussed above, the
==PAGE:58==
40
3 Technologies
experiment consisted of three different types of tasks, and each question in-
side the tasks had different difficulties. To compare the difference between the
task difficulties for the same participant, we needed to conduct Repeated Mea-
sures ANOVA [41]. Repeated measures ANOVA was used for the first large
hypothesis and followed with a post-hoc paired sample t-test using Bonferroni
correction method. Bonferroni correction was used because there was a need
to have multiple comparisons. Bonferroni correction is often used when differ-
ent groups are being compared together and seeing if there is any relationship
between different variables [42].
For the second and third large hypotheses, I used ANCOVA, which is simi-
lar to the repeated measures ANOVA. ANCOVA can be used to determine
if there is a difference between the adjusted means. That is, compare the
individual difference between variables for each participant individually [43].
The aggregated data that was used for hypothesis H1 was also used to test
hypotheses H2 and H3.
The sub-hypotheses were both analyzed globally and locally. Globally means
the whole final data set after aggregation. Locally means that the N-Back,
calculation, and code examples tasks were aggregated separately creating three
additional data sets. The aggregation for the locally was done the same way
as mentioned above.

to model the noise of the training data [45].

6.2
Hypothesis
In this section, there are three large hypotheses checked that were made in
the development process of the experiment. To help us answer the there large
hypotheses, a few sub-hypotheses were made. All sub-hypotheses are both
checked for the whole data set and also each individual task, that is for N-
Back, calculation, and code example tasks. We will now refer to the test for
the whole data set as being tested globally, and the test for an individual task
is locally.
To do subjective analysis, that is, compare the different variables between dif-
ferent task difficulties for each participant, we needed to aggregate the data.
The aggregation works in that an equal amount of easy, medium, and hard
tasks are used and compared together individually for each participant. Re-
peated measured ANOVA was used for the first large hypothesis and followed
with a post-hoc paired sample t-test using Bonferroni correction method. For
the second and third large hypotheses, I used ANCOVA, which is similar to the
repeated measures ANOVA. If the p value was lower then 0.05, then we could
reject the null hypothesis. Otherwise, we reject the hypothesis. In hypothesis
H1, there was the need for a post hoc test to determine what condition the
hypothesis was rejected in.
6.2.1
Hypothesis (H1)
The hypothesis was: The higher the task difficulty (easy, medium, hard) the
higher the Cognitive Load (both subjective (NASA TLX) and objective (various
measures).
To help us answer this hypothesis, a few hypotheses were created. These sub-
hypotheses are comparisons between independent and dependent variables
described in subsection 3.2.7.

with a task.

To prevent

most frequent task difficulty. Out of the task difficulties, 40.602% of them
were considered as Medium and that makes the baseline 40.6206 %.

Limitations
One limitation of the experiment is that it is controlled, therefore, the par-
ticipants might be more nervous and aware of various factors compared in a
normal environment. This can affect the results but is difficult to avoid.
Most of the participants were students or previous students at the Technical
University of Denmark. There could have been participants with more variety
of backgrounds. However, the experiment consisted of code examples, so there
was a need for some programming knowledge.
There was a challenge finding people to participate in the experiment. Firstly,
it can be because the experiment took approximately two hours. Although
many individuals like to participate because of interest in the subject or even
just participating in an experiment, two hours is a lot of time for just helping
someone out. Therefore, the time factor could reduce the interest for indi-
viduals to participate. Furthermore, the experimental period was during the
end of the spring semester so many people had their schedule full due to final
projects and exams.
There are always going to be participants with a low gaze sample percentage,
but the factors that effect the data quality and the gaze percentage should
have been kept under a microscope throughout the whole experimental period.
There was sometimes a problem with participants complaining about being
tired and yawning before and during the experiment. During these cases, the
participant should have been asked to participate at another time when not
feeling tired. Although I noticed a participant being tired I did not want to
ask him to come at another time since it was challenging to find participants.
There were difficulties planning the experimental period since there is only
one computer that contains the Screen-based eye tracking device from Tobii
and the software Tobii Pro Lab. Tobii Pro Lab was only available on two

Conclusion
The goal of this study was to collect labelled data to detect intervention needs
while solving tasks. That is, when a participant has difficulties understand-
ing or solving the task. This is important because too much overload for the
working memory can result in that the participant might not have learned
anything from what he was doing, although, he managed to finish the task.
This is detected in an increase in cognitive load or working memory.
This was done by conducting an experiment where participants solved three
different tasks, N-Back, calculation, and code example tasks. While the par-
ticipants were solving the tasks, they had eye tracking, and GSR devices
measuring their biometrics. Between tasks, the participants needed to answer
questions following the NASA-TLX method.
As discussed in previous chapters, the goal of this study was reached. From
the measurements collected from the eye tracking device, fixation duration
per minute and the pupil dilation showed significant statistical results most
often. Furthermore, from the skin response measurements the SCR Normal-
ized showed significant statistical results most often. Therefore, these measure-
ments are considered the best indicators for the cognitive load. A classifier was
trained and it predicted the correct difficulty 64.789% of the time. The same
features that were important for the classifier were the ones which showed
statistical significant results in the hypotheses. In the future, during real-life
==PAGE:150==
132
8 Conclusion
situations, the measurements from the Shimmer 3+ will most likely not be
available. However, from what we have shown in this study, the data from
the eye tracking device should be enough to predict the cognitive load.
There were only 24 participants that participated in the experiment for this
study. It could be that a higher number of participants would have resulted
in more significant results. Like mentioned above, the pupil dilation, fixation
duration per minute, and SCR Normalized all showed significant results indi-
cating that they could be used to detect when there is a need for intervention.
The research question was answered by training a classifier which predicted
the correct task difficulty 64.789% of the time. That means that from this
data, a classifier can predict the task difficulty for the participant based on
the Weighted TLX.
