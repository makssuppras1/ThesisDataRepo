Summary
For over 30 years the problem of finding an optimal design for a reflector antenna, which
is a worst-case performance optimization problem, where the objective is to minimize
some error-measuring residual functions by changing the design variable, has been solved
by TICRA using the minmax (minimax) algorithm, where the objective is to minimize
the maximal residual, i.e. the worst-case performance. In the case of feed positioning
for a reflector antenna—a problem where the number of design variables is low—the
minmax algorithm is known to terminate in a bad local minimum, if the initial point for
the algorithm is poor, and we will investigate another way of solving this problem.
In this master thesis we propose the One-Sided Least-Squares (LSQ+) formulation,
where the objective is to minimize the sum of squares over positive residuals, as an
alternative problem to solve in worst-case optimization. The LSQ+ problem is a special
case of the well known Least-Squares problem, where the objective is to minimize the
sum of squares over all residuals.
Our method for solving the One-Sided Least-squares problem is trust-region based,
and we consider two different models for approximating the LSQ+ problem locally; a
convex model based on linearization of the residual functions, and a steepest descent
model based on linearization of the objective function. The convex model gives rise to a
quadratic program, which we discuss different approaches to solving, and we propose an
interior point method that exploits the structure in the problem when the dimensionality
of the input variable is low.
We explore the differences between the LSQ+ problem and the minmax problem,
and highlight the difference in the solutions obtained by solving the two problems re-
spectively.
Our results show that the LSQ+ algorithm does not converge to a bad local minimum
in the case of feed positioning, and instead it converges to a solution, which is close to
the minimum in the minmax formulation. We test the LSQ+ algorithm in combination
with the minmax algorithm on the problem of reflector shaping—a problem with a
large number of design variables—and find that in many cases the performance of this
combination is better than that of the minmax algorithm on its own, but we also find
that in some cases the outcome depends on a careful choice of the initial trust-region
radius.

Introduction
Optimization is ubiquitous in engineering applications. A common problem that arises
is the Least-Squares (LSQ) problem, where we seek to minimize the square of some
error-measuring residuals. This is for example used when trying to estimate parameters
for a model to fit real-world data. In this project we will consider a special case of the
LSQ problem.

The global goal can be used to change which residuals are positive, and hence contribut-
ing to the objective function. Choosing the global goal to a negative value will take the
smaller residuals out of consideration, and the objective function will resemble the one
from minmax more. Adding a global goal in the minmax formulation will not change it.

The solutions we get by solving the LSQ+ problem will in most cases be different
from the ones obtained when solving the minmax problem, and it depends on the appli-
cation which type of solutions are more enticing. There is probably a reason that the
minmax problem is solved in some applications, so solving the LSQ+ problem instead
might not yield the wanted solutions for these applications. If the solutions from min-
max are more desirable, the LSQ+ algorithm could still prove useful, as we will explore
using the LSQ+ solution as an initial guess for the minmax algorithm in Section 6.3.
In this project we will use a trust-region method which is an iterative algorithm.
When developing an algorithm there are a lot of considerations to make. Two of the
main properties of an algorithm are its robustness and its speed. Parameters in the
algorithm have to be chosen and the convergence is usually dependent upon these. We
will strive to make choices, which make the algorithm robust. Talking about the speed
of an algorithm is a two-way discussion. Firstly, it is good if few iterations are needed
to reach a minimum. Secondly, it is desirable that the computational cost (time) is
kept low in each iteration. It is case dependent, which one should be more in focus.
If the objective function is cheap to evaluate it is not a big issue that we use a lot of
iterations. On the other hand, if the objective function is expensive to evaluate, we
would rather spend more time in each iteration to make good decisions and keep the
number of iterations down. We will discuss what can be done to make the algorithm fast.
We will begin this report by introducing the trust-region method, which we will
use throughout the project, and present some theory regarding this in Chapter 2. In
Chapter 3 we describe different formulations of the problem and consider models for the
trust-region approach. In Chapter 4 we construe the algorithm developed in this project,
and look into some practicalities in regards to this. In Chapter 5 we will describe some
cases that TICRA has provided for testing the developed algorithm. In Chapter 6 we
present the results revolving around the cases. In Chapter 7 we discuss the results and
other aspects concerning this project, and in Chapter 8 we draw conclusions and sum
up the results.

Our general technique for solving nonlinear minimization problems in this project is the
trust-region method. We will describe the method for the unconstrained minimization
problem following Chapter 6 of [CGT00] and afterwards state the modifications neces-

solving problems of the form

To test the developed algorithm we have some problem instances, which we call cases.
First of all we have a self-created case, numbered 0, which is intended for visualization
of the algorithm in low dimensions. After that we have 4 cases provided by TICRA.
For case 0 we will provide the results in this chapter as it is intended to visualize the

Common for the four cases from TICRA is that the variable x ∈Rn describes some
part of the design of a geostationary satellite1. In Figure 5.1 we see an example design of
a reflector antenna with a single reflector and a single feed. All four cases are concerned
with optimizing some part of the design of an antenna like the one in the figure. In
case 1 we have an array of feeds instead of a single feed and we wish to optimize the
excitation coefficients for each feed. In case 2 we optimize the position of a single feed.
In cases 3 and 4 we optimize the shape of the reflector.
In Figure 5.2 we see an actual satellite being tested before launch. The white dots
on the reflectors are markers for measuring the geometry of the reflector, so disregard
these. The satellite has 5 reflectors, each with a feed, and looking closely at the bottom
right reflector, we can see that it has been shaped. Also on the top reflector we see some
extreme shaping to the right.
The objective in all cases is to get a good coverage at some stations in some area on
Earth. For each station a goal, gi, measured in dB, is set and they are collected in the
vector g ∈Rm. Furthermore, we have a function h : Rn →Rm that measures the gain
for a design x ∈Rn. We can then define the residuals as
r(x) = g −h(x).
In some cases we care more about some stations than others—if the satellite is transmit-
ting television, it might be sensible to favor larger cities—so for each goal we introduce
a weight, wi ∈R, to account for this. The residuals then become
ri(x) = wi(gi −fi(x)),
i = 1, 2, . . . , m.
(5.1)

CHAPTER 6
Results
There are several things to keep in mind when presenting the results.
We could go
through the cases presented in Chapter 5 chronologically but this seems unnecessarily
tedious.
Instead we have divided this chapter into sections that each explore some
property of the LSQ+ problem and algorithm. First, in Section 6.1, we will show some
solutions from the synthetic case 1 to see how the solutions look and to explore the role
of the global goal. Next, in Section 6.2, we will consider case 2, where the minmax-
algorithm runs into trouble, and see if the LSQ+ algorithm can do better. As discussed
in the introduction we will consider the LSQ+ algorithm both in its own right and as
a resource to finding a good initial guess for the minmax algorithm. In Section 6.3 we
will explore, whether we can use the LSQ+ algorithm in combination with the minmax-
algorithm to get better results than with the minmax-algorithm on its own. In Section
6.4 we will discuss the robustness of the algorithm with respect to the initial trust-region
radius, δ0.
We will try to introduce the means of visualizing the results as we go along, but for
a more thorough introduction see Appendix A.
Unless otherwise stated the algorithm that we use is the LSQ+ algorithm with the
convex model.
The global goal, G, acts as a parameter in the problem formulation and will naturally
change the solution. Therefore we will denote solutions as x∗(G) to be explicit about
the global goal, but for G = 0 we will omit the argument and use x∗= x∗(0).

of changing the global goal we run the LSQ+ algorithm with four different global goals

goals the algorithm stopped due to the trust-region radius becoming smaller than this.

algorithm tries to minimize these, while the residuals to the left of −53 are negative
when the goal is added, so the algorithm leaves these alone. In Figure 6.1(b) we see
for G = 55, that the width of the distribution is larger, so that the minimal residual is

6.3
Combining Algorithms
In this section we will explore if we can use the LSQ+ algorithm in combination with
the minmax algorithm to achieve better results. More specifically we will use the LSQ+
solution as an initial guess for the minmax algorithm. Before we present the results we
will mention that in practice evaluation of the objective function sometimes takes on
the order of hours, which makes it impractical to run the algorithm until it converges.
Instead the algorithm is terminated after a fixed number of iterations, which motivates
the approach we will take in investigating this matter. We will compare a number of
iterations using the minmax algorithm to the same number of iterations split between
the LSQ+ and minmax algorithms. We use the same total number of iterations even
though the LSQ+ algorithm takes more time computationally than the minmax algo-
rithm. To justify this keep in mind that the minmax algorithm we use is state of the
art (implemented in FORTRAN, MKL, etc.), while the LSQ+ algorithm has not been
tuned very much.

CHAPTER 7
Discussion and Outlook
In this chapter we will discuss the general properties of the LSQ+ problem and sum
up some of the practicalities and issues of the algorithm. Furthermore, we will look at
future work that could be done on this subject.
First of all the results that we have gotten with the LSQ+ algorithm are very satis-
fying but the algorithm is by no means as good as it can be.
We have seen that the convex model performs better than the steepest descent model
in terms of iteration and final convergence, but one of the drawbacks of the convex
model is the time it takes to compute a step. A way that we might bring this time
down is to identify active constraints in each iteration. By active constraint we mean
the residuals that are positive or have a chance of becoming positive by taking the step.
This would reduce the number of constraints on the convex model in (3.8). The problem
with neglecting negative residuals (inactive constraints) might be that the convex model
loses the final convergence, since it would make it somewhat similar to the steepest
descent model.
A concern about using the algorithm could be how to choose the global goal, G. We
have seen in Figure 6.4 that we obtain different solutions, when changing G, but only
to some extend. The results we have gotten for reflector shaping were with G = 0, so
it does not play a huge role, but it would be interesting to do more experimenting with
changing the global goal.
Regarding the alternative ratio ρσ we defined in Section 4.2.1 we have done some
testing and found that it does not make much of a difference for the convergence, since
σ is mostly close to 0 and therefore ρσ ≈ρ. One could consider other ratios; instead of
calculating the percentage it gets wrong, we could use something like

We saw in Section 6.4 that the convex model using the infinity norm as shape of

The same was the case for the steepest descent model with the infinity norm, but we did
not find the same sensitivity when using the 2-norm. We have tried to use Algorithm
17.2.1 from [CGT00] to choose the initial trust-region radius with no luck. The trouble
with this algorithm is that it takes an initial (arbitrary) trust-region radius but this
seemed to change the result—so one could argue that it is not arbitrary. Since there
did not seem to be the same sensitivity with the 2-norm it would be interesting to have

AMPL[Amp], CPLEX[Cpl], Gurobi[Gur] or MOSEK[Mos]. We could also modify the
interior point method to handle the cone-constraint.
Another concern is that the LSQ+ algorithm seems to stagnate after a number
iterations in cases 3 and 4 for the harder initial guesses.
To address this we have
proposed an early termination criteria when the decrease in the function is too small
over 10 iterations discussed in Section 4.3, since the algorithm does not accelerate much
after this point from our experience.
If it turns out that using the 2-norm as trust-region shape is more robust as our
findings suggest an interesting idea—if we want to avoid solving a SOCP and use our
existing methods—would be to combine the models; using the steepest descent model
followed by the convex model to get the final convergence.
Usually, when using a gradient method, as we do with the steepest descent model,
the gradient direction is a search direction and it is combined with a line search method
to yield a step. This is also something that could be investigated, but since line search
methods need more function calls we have chosen not to try this.
As a final note in this chapter, bear in mind that all our results revolve around some
specific cases, where some of them are very non-linear, which could also be the root of
the algorithmic issues we have seen.

CHAPTER 8
Conclusion
In this project we have formulated the One-Sided Least-Squares (LSQ+) problem (3.1)
and derived a trust-region method for solving this formulation, which we call the LSQ+
algorithm. We suggested two different models approximating the problem locally for the
trust-region algorithm; the convex model, which lead to a quadratic program to solve
for a step, and the steepest descent model, where the step is given analytically. The
convex model showed better convergence in terms of number of iterations, since it retains
information about negative residual that might become positive. The steepest descent
model had the advantage of being fast in terms of computational time and allowing us
to use the 2-norm as trust-region shape easily.
We tested the LSQ+ algorithm on cases revolving around the design of a reflector
antenna provided by TICRA, and compared the results to those of the minmax algo-
rithm (also provided by TICRA) solving the formulation (3.33). The case about feed
positioning had few input variables and many residuals. We found that the LSQ+ for-
mulation avoids some of the known obstacles that the minmax formulation has. This
is evident from Figure 6.2, where we saw that the minmax formulation had many local
minima, while the LSQ+ formulation appeared to be convex. Therefore we obtained the
satisfying result that the LSQ+ algorithm works well for feed positioning on a reflector
antenna.
After this we used the algorithm for some larger problems with many input variables
as well as residuals. We used the LSQ+ algorithm in combination with the minmax
algorithm (LSQ+ algorithm followed by minmax), and compared the results to those
of the minmax algorithm on its own. We found in Tables 6.1 and 6.3 that adding the
LSQ+ algorithm boosted the performance in most cases—some more than others—and
rarely gave a worse performance.
We have experienced a few algorithmic issues during the project. We have found that
the algorithm uses a lot of iterations with slow convergence in some cases. Perhaps some
parameters in the algorithm should be changed to help this, but most likely different
parameters are better for different cases. Besides the occasionally slow convergence, we
have seen that, when using the infinity-norm as trust-region shape, the LSQ+ algorithm
is sensitive to the initial choice of trust-region radius in some cases. The same issue did
not occur when using the 2-norm, which suggests that this is a more robust choice for
the algorithm.
To sum up we have found that solving the LSQ+ formulation is sensible, and that
it has advantages over the minmax formulation in application for feed positioning. In
reflector shaping we found that the LSQ+ algorithm is a good complement to the minmax
