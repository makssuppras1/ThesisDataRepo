Abstract
Mobile robots moving in an outside area is a complicated problem due to the changeable
weather and surroundings. Variable brightness and vegetation colors will make situation
awareness diﬀicult and obstruct further activities. This project focuses on developing a
vision-based approach for mobile robot perception in various weather situations, map-
ping and localizing in a forest-like path, and supporting autonomous navigation. With
the use of an RGB-D camera, the robot could recognize semantic information of sur-
roundings, measure relative movement, and localize according to a previously generated
semantic map.
To enable the robot to perceive in varied situations, a deep neural network was
trained to fulfill semantic segmentation utilizing camera input data acquired from dif-
ferent seasons. Meanwhile, visual odometry was constructed based on featured point
clouds from the camera. The robot was allowed to exploit the environment in advance
so that a semantic map can be generated offline based on the segmentation and posture
results from the former. Lastly, a particle filter was utilized for precise localization.
Through testing, the proposed solution could meet the requirement of this project.
Also, further possible navigation methods were mentioned at the end of the thesis.

CHAPTER 1
Introduction
There are many researches on the autonomous mobile robot (AMR) in recent decades
for its broad application prospects. The AMR could assist or even substitute humans
for working in dangerous, repeated, and demanding situations. For example, outdoor
application AMR could work in agriculture to fulfill crop-dusting and fruit picking; when
any disaster such as an earthquake happens, an AMR could also work as a deliveryman in
ruins. To make the robot achieve these purposes in real life, a sophisticated autonomous
system should be developed. Situation awareness, mapping, and precise localization
would be realized as the premise to support navigation. In addition, when navigating
outdoors, the changeable weather and rugged trails always cause uncertainties for robot
perception and localization. Specific problems are introduced in the situation of AMR
working outdoors:
Firstly, in terms of perception, the robot is expected to recognize surroundings and
extract useful information, such as drivable areas, for further activities. In order to
guarantee robustness, the perception result should not be impacted by the brightness
and appearance of the same object.
If the robot could have prior knowledge of the track it will drive on, the localization
and navigation strategy would be much easier to organized. So, another problem is how
to generate a bird-view map according to the information acquired from the sensor(s).
Generally, a map could be generated by recording the current posture of the robot and
merging processed data in adjacent frames. Also, the map should be combined with
semantic information to help the robot understand the path and obstacles.
Finally, the robot should be able to localize and navigate autonomously according
to the pre-generated map.
The project aims to implement reasonable solutions for enabling a mobile robot
to perceive, map, and localize at a chosen track outdoors. The track is a forest-like
running path around the DTU campus, with trees and grass on both sides, pedestrians,
and bicycles passing through.

In the light of the aim of this project, objectives are presented in this section. If they
are fulfilled, the robot will be able to perceive, localize, and be ready for navigation at

CHAPTER 2
Analysis
In this chapter, an analysis of the project is presented to propose specific methods for
solving the problems raised in chapter 1. A description of the test environment gives a
clear understanding when collecting and processing data from the sensor. A comparison
between two types of sensors explains why the vision-based method is applied in this
project. In order to determine what technologies are valid for solving problems proposed
in chapter 1, an overview of state-of-art is presented, following a specific problem analysis
and stating the solutions of this project.
2.1
Environment
In this project, a robot is expected to work outdoors. The running path around the
DTU Lyngby campus is chosen as the test track. The track is forest-like, with sturdy
trees, grass, and stones on both sides, and dustbins are also placed at most junctions.
Sometimes, pedestrians will pass by. The most prominent features in the track scene
are two: one is the differences between the color and texture of the road and roadsides;
another is the tall trees close to both edges of the road. These features could be used to
detect roads, determine the relative movement, and help precise localization. However,
with the change of seasons, vegetation will turn green, and the leaves will growing
up from trees. Though the position of trees and the road is permanent, the scene is
different. How the robot could extract the same information from different scenes is a
knotty problem that the perception system needs to solve.
2.2
Comparison between Camera and Lidar
In modern research, Lidar and camera are the two types of sensors commonly used to
perceive surroundings. Lidar, also called 3D laser scanning, could accurately measure
the distance between the neighboring objectives and itself in 3D by computing the laser
return time and the reflective intensity in terms of materials. If the number of lasers is
large enough, people can get a dense point cloud that depicts the surroundings. However,
a high-resolution Lidar is very expensive. If a low-resolution one is implemented in the
broad forest-like environment, it is hard to acquire enough detailed reflection of the
object. In addition, the roughness and the reflective intensity of the ground and trees
==PAGE:14==
4
2 Analysis
will be different with the change of seasons. Therefore, it is arduous to use Lidar to
extract object features in our project.
On the other side, cameras could provide visualized data that is more similar to what
humans see at a specific frame rate. An RGB camera could record the shape, color,
and even status of objects by reflecting natural light. Those visual data provided by
the RGB camera can support object detection and classification. However, data from a
single RGB sensor is 2-dimensional, where the distance between objects and the sensor is
unknown. Fortunately, the introduction of stereo camera and RGB-D camera could deal
with this problem. To be specific, the stereo depth camera could measure the distance
by comparing two images acquired by two IR sensors simultaneously, which is similar
to how human eyes perceive the depth. The depth information will be aligned with
colored pixels, resulting in the dense, colored 3D point cloud that could be generated.
The robot’s odometry could be computed with the point cloud, and the map could also
be built from it.
One of the drawbacks of the RGB-D camera is that the measurement accuracy is
lower than Lidar.
Further measurements will introduce more noise, while computer
vision algorithms could reduce measurement errors. Another is that the maximum mea-
surement range is limited, from 10m to 20m. However, this is enough for a mobile robot
with no need to drive at high speed. As a result, a camera that could give depth values
is ideal as the situation awareness tool to provide vision-based solutions in this project.
2.3
Related work
The autonomous mobile robot is a popular research topic in recent years. Many kinds of
research on AMR-related algorithms were published and inspire autonomously driving
technologies. An analysis of state of the art would be guidance of specific solutions in
this project.
When implementing a camera as the sensor in robot situation awareness, images
will provide preliminary information on perception. Image semantic segmentation, a
computer vision concept that segments one image into different labels based on the
type of object, is commonly utilized in robot perception systems. Segmentation is pixel-
leveled, and a labeled image always gives a robot a better understanding of the objects
around it. In order to fulfill semantic segmentation, a deep neural network is always
considered an eﬀicient tool for providing comprehensive segmentation results in a short
inference time after training.
Unlike object detection and classification, the neural network output for segmentation
has the same size as the input image. Therefore, the encoder-decoder structure is needed,
where the encoder contains layers for extracting features in the input. The decoder is for
upsampling the feature map exported from the encoder and recovering the size of the
map. [1],[2] and [3] are three representative encoder-decoder architectures for semantic
segmentation. [1] used AlexNet[4] as the encoder and upsample the last layer of the
==PAGE:15==
2.3 Related work
5
encoder by transposed deconvolution until getting a result with the same resolution as
the input. [2] developed a different upsample method in the decoder by using pooling
indices computed in the max-pooling step of the corresponding encoder to perform non-
linear upsampling. With this novel decoding way, the number of training arguments is
reduced, and the architecture is more eﬀicient and has higher accuracy than the FCN [1].
U-Net [3] is another well-known architecture that is firstly designed for medical image
segmentation. The shape of the U-Net is symmetric, with skip-connections between the
shallow layers and deep layers. U-Net is also a light and eﬀicient network.
In terms of the RGB-D image segmentation, [5] aligned the RGB image with the
corresponding depth image as a four-channel input. The author developed a modified
AlexNet to demonstrate that the neural network with RGB-D image performs better
than an RGB image input, especially when two objects overlap. In addition, [6] was
developed based on SegNet[2]. It suggested an idea that develops two brunches of input
respectively for RGB and depth and fuses the encoders, as using four-channel RGB-D
would not fully exploit the structure of the scene encoded by the depth channel. The
test accuracy (i.e., IOU) was very competitive compared with other state-of-art networks,
and it also had a short inference time. FuseNet could be considered appropriate in this
project.
Other deep neural network architectures were also developed for fusing RGB and
depth features. [7] summarized the ways to fuse the RGB and depth brunches in com-
monly used networks and develop the RDFNet by extending RefineNet[8] and get higher
accuracy than FuseNet for indoor segmentation. However, the network structure is a
bit complicated. As the outdoor environment in this project does not contain as many
objects as indoor cases, it is tolerable for lower accuracy.
When a robot navigates in the environment, its odometry is commonly obtained from
a wheel encoder. This project assumes the wheel encoder is not available; therefore, a
visual odometry needs to be developed based on the RGB-D camera. VO is a cheap way
for acquiring real-time odometry from image sequences. [9] reviewed modern sensors
for robot localization and analyzed the type, algorithms, and challenges of the VO. A
simple real-time VO could be developed through feature detection, matching, and point
cloud registration, where ICP[10] are always used in point cloud alignment. [11] came
up with an improved ICP method that adds semantic information into the optimizer.
Once the odometry is available, the 2D map could be generated from it [12]. A more
robust and precise way for mapping both in 2D and 3D is using VSLAM. For example,
[13] and [14] have generated 3D semantic maps by using positions computed by ORB-
SLAM2 and ElasticFusion. The generated maps were complex, but the SLAM ensured
the accuracy.

2.4
Problem Analysis
In this section, possible solutions are discussed to fulfill the objectives mentioned in
section 1.1. The solutions are derived from state-of-art works and should be able to deal
with image segmentation, VO, 2D mapping, and robot localization.
Image semantic segmentation aims to help robots detect roads, different kinds of
obstacles, and other symbols nearby. The scene in the DTU running path is not complex,
and the type of objectives that need to be recognized in this project is not many. Apart
from this, the eﬀiciency of the segmentation system is vital for real-time use. In this
case, the FuseNet [6] is implemented to segment the RGB and depth images acquired
by the camera. For training the neural network, a series of data should be collected
previously. During training, the arguments of the FuseNet should be tuned to make the
network fit the local environment better.
The FuseNet is open-resourced, implemented in Python 3.7 with the library PyTorch.
The loss and images are visualized through Visdom during training. In order to acceler-
ate the training procedure, NVIDIA GPUs from the DTU HPC department are utilized.
At the same time, RGB and depth images will be sent to the visual odometry for
computing the relative positions of the robot.
Real-time visual odometry could be
developed in 3 steps: firstly, extract interest points in two neighboring RGB images
and define the corresponding points; secondly, based on the depth image, generate the
featured 3d point clouds; finally, compute the homography matrix by registering adjacent
point clouds via ICP. The relative displacement and the orientation of the robot could
be fetched from the homography matrix.
After having the segmentation result and the relative positions of the robot, it is
time to produce a 2D map. A single frame of bird-view map with required objects is
shown by segmentation, denoising, clustering, and projection. Single frame maps will be
assembled according to the relative positions. In this project, we assume that the robot
is allowed to explore the environment in advance, which means the complete map could
be generated offline. So a SLAM is not imperative in this case.
In the parts mentioned above, much work on point clouds needs to be done. Open3d
is a powerful library in Python with great well-built functions for point cloud processing,
such as point cloud coloring and DBSCAN clustering. OpenCV in Python is another
sound library with many computer vision functions implemented. To realize vision-based
algorithms such as visual odometry, OpenCV is indispensable.
It is much easier for a robot to localize and navigate if the track map is available.
Global localization could be achieved by GPS, which, in some cases, has limited accuracy
or is not available all the time. In the last part, an observer would be implemented to
improve localization performance by combining GPS, VO, and map data.

2.5
Modules
To make the solution discussed in section 2.4 clearer, a flow chart of each module is
shown. The chart in Figure 2.1 explains how they connect and what their aims are.
Figure 2.1: The flow chart for proposed modules.
• Image acquisition: to acquire RGB images and depth images from the RGB-D cam-
era in a certain frame rate. The images are simultaneously sent to segmentation
and visual odometry.
• Semantic segmentation: to use a deep neural network to segment the acquired
images into different semantic objects.
• Visual Odometry: to find the relative positions of the robot in different frames.
• 2D Semantic mapping: to build a 2D semantic map of the track, prepare for

4.2.1.1
Convolutional Layer
The convolutional layer is the most commonly used block in this architecture.
The
characteristic of the convolutional layer is that it applies several filters (kernels) to the
elements of the input repeatedly and orderly to extract the feature of the input. Different
filters could extract different features in the same image.
Figure 4.3: An illustration of the convolutional layer.
Figure 4.3 illustrates the computing procedure of one convolutional layer. The filter
moves from left to right, from top to bottom, until it reaches the last unit of the image.
The input image is 2-dimensional with a different number of channels. For example, the
RGB image has three channels for each color component, and the depth image only has
one, so the filter implemented on the input image is also 2-dimensional with the same
number of channels as the input. At each computing time, the processor will compute
the dot product between the filter and the filtered area of the input image and sum the
results in all image channels together as the output. The number of filters implemented
in one convolutional block determines the dimensions of the output of that block.
One advantage of the convolutional layer compared with a fully connected layer is
that it could analyze an image with a small number of arguments and correctly enhance
the features by implementing filters. It is a very eﬀicient image processing method in
deep learning.

4.2.1.2
Pooling and Unpooling Layers
In the encoder, the pooling layer is always used to reduce the dimension of the next layer.
With a smaller size, the computing burden is significantly reduced, and the smaller num-
ber of arguments guarantees the universality of the neural network. The max-pooling
method is commonly used in this case, which means the max-pooling layer’s output only
keeps the maximum values and their locations in each pooling area. It enables the neural
network to only focus on the primary features instead of other insignificant factors such
as the local location of the object in a specific image.
Similarly, in the decoder, to recover the resolution of the output image, the unpooling
layer is used to reconstruct the pixels compressed by the max-pooling in the encoder. In
Fusenet, the max-unpooling layer is implemented, which adds the maximum features at
the maximum locations and fills zero at other units.
Figure 4.4: An illustration of the max pooling and max unpooling [17].

was firstly used in this project is stochastic gradient descent (SGD) with momentum.
The momentum could be computed by
mt = γmt−1 + ηgt

With the introduction of the momentum, parameter update direction is determined

weights with little change in the gradient direction to accelerate their update and slows
down the weights with a significant change in the gradient direction.
SGD uses the same learning rate to update all weights in the neural network. How-
ever, the frequency of different semantics appearing in the training data is different,
which leads to weights updating in different frequencies.

CHAPTER 5
Visual Odometry
5.1
Introduction
In this chapter, simple visual odometry will be developed to enable the robot to find its
relative pose only based on the data from the visual sensor. With the help of odometry,
it is easy to determine where the robot is in a specific coordinate system. In addition,
odometry is very useful when mapping.
The RGB-D camera D455 can support the development of visual odometry. The
depth camera could provide both 2-dimensional images and a 3-dimensional point cloud
of surroundings. To acquire input data of the visual odometry, the camera would take
pictures and generate point clouds at a specific frame rate during driving. By comparing
the difference between point clouds in adjacent frames, the relative movement and the
robot’s orientation could be known through computer vision algorithms.
The robot’s relative pose could be calculated from the transformation matrix between
two frames. Epipolar geometry(2D to 2D), PnP(3D to 2D), ICP(3D to 3D) are three
commonly used algorithms in visual odometry for transformation matrix calculating.
Since the RGB-D camera could provide 3D point clouds of each frame, the ICP is the
most suitable method. To accelerate the classic ICP method, the feature extraction
algorithms and FLANN are implemented before the ICP computing. In the following
sections, an introduction of ICP and improved ICP method will be presented. Finally,
a module test is shown to prove that the visual odometry is able to compute the pose
of the robot in real-time.
5.2
Iterative Closest Point
The initial purpose of the Iterative closest point (ICP) algorithm is to find a transforma-
tion matrix so that the source point cloud could be converted to the coordinate system
of the target point cloud, and fulfill the point cloud registration. The transformation
matrix contains the rotation and the translation information on how the source point
moves to the target coordinate. In the visual odometry case, if we consider the source as
the point cloud captured in the current frame, and the target as the point cloud recorded
last frame, the transformation matrix computed by the ICP algorithm would tell the
robot’s movement between two frames.

To compute the rotation and translation between two frames, the specific problem
that the ICP dealing with could be described as:
Given two point clouds ( the source S and the target T), match the nearest point pairs

∀i, Ti = RSi + t
(5.5)

Specifically, the main steps of the ICP algorithm are:
Assume the source point cloud is S, the target point cloud is T
• For each point Si in S, search and match the nearest neighbour ti in T
• Compute the transformation matrix that minimizes the loss function by SVD
• Convert the S with the transformation matrix computed in the last step, to get
the new point cloud S’
• Calculate the distance between S’ and T. Stop if the distance is less than threshold.
Otherwise, set S’ as the new source and go back to step 1.
The ICP algorithm is implemented in Python library Open3d, which asks two point
clouds as the input, and output the transformation matrix. However, as the function
manages to search and match all points, the computation time is pretty high. There-
fore, the feature extraction algorithm and the advanced searching method should be
implemented in visual odometry for real-time use.
5.3
Improved ICP
In order to deal with the time-consuming problem, the feature extraction algorithm
needs to be implemented so that only the representative points in the point clouds will be
matched. This could significantly reduce the number of points in processing. In addition,
Fast Library for Approximate Nearest Neighbors (FLANN) will be implemented instead
of the Brute-force searching method to help reduce the time complexity.
The commonly used feature extraction algorithms such as SIFT, SURF, and ORB can
be implemented through OpenCV. They are used to extract the representative points
==PAGE:46==
36
5 Visual Odometry
from an RGB or grayscale image. [19] has compared the performance of these three
algorithms, where the SURF is most robust and with a competitive computing time.
This is the reason why SURF is used for extracting features in this project.
In FLANN, the K-Nearest Neighbor (KNN) matching with the KD-tree index is
applied to search the nearest neighbors of the features in adjacent images.
KNN is
the method that searching K neighbors having the nearest Euler distance with the
feature, while KD-tree is a data structure often used in high-dimensional spaces. In
the visual odometry, each feature point output from the SURF has a descriptor with
64 dimensions, so it is expected to use a KD-tree data structure to help reduce the
computation complexity.
A KD-tree could be developed by following steps:
• Select the one dimension which has the largest variance in the K-dimensional data
set, and select the median point in this dimension as a node.
• Divide the dataset as two sub-sets by the value of the node.
• Move to next dimension
• Repeat the former steps until all sub-sets can no longer be divided.
When searching for the nearest neighbor, instead of comparing the Euler distance
between all features, the KD-tree index divided the features by their values in a different
dimension so that the searching process would be conducted in different sub-sets. As a
result, the number of searched features will be reduced sharply. If there are N features in
the searched set, the time complexity will be reduced from O(n) to O(n1−1
k +m) (where
m is the number of required neighbor, and k is the dimension) by the introduction of
KD tree.
The feature extraction and the matching process are conducted on 2-dimensional
images. To continue work on ICP, the 2-dimensional features need to be aligned with
the depth information to convert the imaging features to the featured point cloud.
There will be hundreds of features matched, and the quality of those matches deter-
mines the accuracy of the transformation result. Filters will be implemented to filtrate
extracted points based on the strengths, depth value, and ration test[20]. The flow chart
of the whole visual odometry system is in Figure 5.2.
The image used to extract features is grayscale instead of RGB. This is because the
color impacts feature edges extraction, and the grayscale could be seen as the intensity
and easy to find the edges.

5.5
Error Analysis
The uncertainty in each estimated step will be accumulated and cause a drift in the
final. Especially when there is a computing mistake, the pose in the next frame will
be estimated based on the wrong transformation. This simple visual odometry cannot
correct the wrong transformations. In application, a well-developed SLAM commonly
introduces a loop closure to optimize the accumulation of errors from visual odometry.
There are two leading causes of the error through the test: one is from the envi-
ronment, another is the blurred image input. For the one cause, the brightness change
==PAGE:56==
46
5 Visual Odometry
requires specific arguments adjustment, and the strong wind may change the position
of potential interest points in neighboring frames. For another cause, the blurred image
introduces a huge diﬀiculty in describing interest points. As a result, the number of
matches could not support an accurate ICP calculation.
Figure 5.16: Feature matching on a blurred image. The left image is blurred .
Following are solutions to help reduce the accumulation errors. The first two methods
are aimed to solve the blurred images, and the last one is the optimization algorithm
that commonly used in SLAM.
• limit the distance between two neighboring frames around 0.5m. For example,
increase the frame rate from 2 FPS to 5 FPS or slow down the speed of the
robot,i.e. 0.8m/s,
• add a tripod head to reduce the impact of camera shake;
• implement loop closure algorithm in future work.

6.4
Localization
One of the applications of the semantic map is to localize the robot. Assume the GPS
is available for the initial global position estimation with an accuracy of 10 meters. It
is expected to prove the real-time situation awareness, and the map could enable the
localization preciser. In this section, the map in Figure 6.9 will be applied. Consider the
center of the trees as the landmark. The robot could recognize where it is by calculating
the distance with each landmark in FOV on the map and comparing it with the landmark
measuring in the current frame.
Different types of observers could be used to localize the robot. Compared with other
filters, the particle filter does not require modeling and linearization and is very simple
to implement. Therefore, a particle filter is implemented to demonstrate the localization
task.
==PAGE:64==
54
6 Mapping and Localization
The particle filter utilizes the randomness of a large number of particles to approx-
imate the actual problem. In the beginning, hundreds of particles will be introduced
around the initial position acquired from GPS. Then, the possibility of each particle
being the final estimated position would be computed and contribute to the final com-
putation. Following is an introduction of the steps to implement the particle filter for
localization.
• Initialization: Randomly generate 200 particles around the initial position ac-
quired from GPS
Prediction: Get the relative movement of the robot from visual odometry and
move all particles based on that.
• Update: Measure the distance between the landmarks in FOV with each particle,
and compare it with the trees in real-time captured point cloud. The particle with
the most similar distribution of trees in FOV will be given the highest weight.
• Resample: Generate new particles around the particles with high weights for
next frame.
• Estimation: Calculate the weighted average position of the particles as the final
estimation.
Figure 6.11 is an example that shows the initialization of the particle filter. In this
figure, the black points are particles randomly generated in a particular area, and the
blue point is the average position of particles. The light blue triangle represents the field
of view of the estimated particle. In the beginning, the filter has not found a particle
that perceives similarly to the image captured in the real world, so the weights of each
particle are similar, and the distribution is sparse.

CHAPTER 7
Future work
In this chapter, expected future work is presented based on the solution and test result
of this project.
• Improve robustness of visual odometry: From Chapter 5, it is noticed that
the visual odometry is vulnerable to the change of brightness and blurred images,
and the uncertainty accumulation from each frame is unavoidable. Loop closure
is an effective method to improve the robustness by recording the keyframe. The
robot could eliminate the accumulation error by introducing the loop closure by
recognizing the scene it has been before. This makes it possible to build a map of
a larger size.
• Add more semantics in the map: In this project, the trees and path are used
as the required semantics when generating the map. This is because the path
is indispensable when navigation, and trees are the most common object in the
environment and could be well detected. However, the neural network can segment
other objects such as stone and dustbin. In the future, it is expected to introduce
these new objects in the map as other landmarks to help localization.
• Apply map in navigation: The robot is capable of detecting the driveable area
when having the map in its brain. With the help of the map, the robot is expected
to:
– Detect a line to follow: the map records the shape of the road, from which
the robot could find the road edges and calculate the middle line to follow
and foresee if a turn is coming.
– Path planning: A well-built map shows the relationship between routes. It
can implement a graph planner to calculate the shortest route to the goal
point.
– obstacle avoidance: from the image segmentation module, the people passing
by could be recognized. It is possible to ask the robot to stop when it is very
close to the human.
• Implement code in C++: Implement the code from Python to C++ to reduce
the computation time.

Conclusion
Modules are implemented in this project to achieve the goal of allowing the robot to
observe, map, and localize itself in a forest-like outdoor environment.
To begin, the neural network ”Fusenet” is used as an image segmentation module

robot’s posture in the camera coordinate system. The visual odometry can be imple-
mented in real-time and is valid in a short travel. There is a discussion about how
the environment affects visual odometry. The visual odometry is subject to changes in
brightness, according to the investigation. Then, a potential solutions were also pro-
posed.

offline semantic point cloud map. Finally, a particle filter is utilized to show how the map
might assist the robot in locating itself based on tree landmarks. Finally, a discussion
