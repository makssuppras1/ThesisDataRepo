Abstract
Growing climate and environmental concerns are raising awareness about energy waste
and non-sustainable processes. The development of renewable energy supplies partially
addresses these matters and entails more distributed energy systems as well as more
uncertain power system operations. At the same time, energy consumers are becom-
ing more active with the expansion of individual electricity or heat storage capacities
together with generation devices like solar panels. Energy systems are more and more
sophisticated to drive, and smart grid strategies are needed.
Today, more and more available data provides relevant information about energy sys-
tem’s behavior and is valuable in the development of artificial intelligence and machine
learning methods to support energy system management. However, in order to put to
use the most classical supervised algorithms, labeled data sets are needed but are not
always available (here consisting in tagged interesting events for temporal energy system
data). As manual labeling is a very expensive and consuming task, the aim of this thesis
is to develop an unsupervised event labeling method.
In this report, a system is described by a set of characteristics and explanatory variables.
The labeling framework focuses on the detection of external events, associated to an
atypical evolution of the system with respect to the explanatory features.
The framework relies on two different methods. First, to detect the temporal component
of an event, a predictive method is used to model the reference behavior of the system and
detect atypical events. Second, to cluster potential different events’ types, a Conditional
Variational Auto-Encoder (CVAE) is used to learn a conditional representation of the
data. The methods are tested on a baseline system, the French notional load, in order
to evaluate their performance regarding the labeling problem. As the labeling task can
be complex, assumptions about the type of events one is looking for are made. In the
baseline system, the focus is put on daily atypical events.
Predictive models with a low modeling error and a good generalization performance seem
efficient to detect important external events but also more subtle ones. Feed-Forward
Neural Networks architecture comes out to be a pertinent model choice. On the baseline
system, the CVAE is able to learn a condensate representation of the data. By choosing
the appropriate latent space dimension and avoiding overpruning, it manages to learn
features that characterized the external event.

Promising results demonstrate that these two methods can be combined to cluster the
different categories of external event, and therefore achieve unsupervised event labeling.

CHAPTER 1
Introduction
This thesis aims at developing an unsupervised machine learning method to label events
in energy systems. The project was carried out within the research and development
group of the French Transmission System Operator RTE.
1.1
Energy system and machine learning
An energy system is a system that either involves the supply, the transfer, the end-use
of energy, or a combination of these elements. It can be of different scales and various
complexity, going from a national electricity grid or an off-shore wind farm to a single
home power set-up. Even if many types of energy can be considered, it often involves
electricity. Such systems have always been developed and improved, but are subjects to
new challenges since recent years.
Indeed, under the impulse of the European Union, renewable energy penetration is in-
creasing in the different European countries, making the generation mix more distributed
and less predictable. Along with this, consumers are progressively becoming active ac-
tors of the electrical grid: organized as communities they can constitute flexible and
reliable smart-grid systems.
From the energy generation, transport or consumption perspective, energy systems are
more and more sophisticated to drive, and therefore require smart managing strategies.
This transformation is going along with the digital revolution and the development of
efficient communication and data acquisition devices. By 2020, 200 millions electricity
smart meters will be installed in Europe, which represents 72 % of the consumers [1].
More and more data is available and provides relevant information about energy system’s
behavior.
This data can be valued to develop artificial intelligence and machine learning methods.
In many different fields such as computer vision or natural language processing, new
algorithms achieved incredible performance, even beating human on very particular tasks
(at game of Go or image classification for example). Applied to energy systems, it can
be used in order to improve system’s management strategies and support the energy
transition.

1.2
On the importance of labeled data
To be efficient, machine learning algorithms need data of good quality to be trained
on, and in important quantity if recent deep-learning algorithms are to be used. Today,
supervised learning is the most common type of learning, when the models are trained
on labeled data. The model has to map input data X to a given output Y, and can be
trained on a set of value pairs (Xtrain, Ytrain). With a deep and representative labeled
data set, supervised machine learning can be very accurate. One famous example is the
ImageNet data base [2]: presented in 2009 by a group of researchers, it contains around
3 millions images of more than 5000 different categories (flower, animals, cars, etc).
The machine learning community is competing in a yearly contest to build the better
classification model, and performances have significantly been improved year after year.
The best architectures (like the resNet [3]) are now used for many different applications.
However, most of the time, only unlabeled data are available. In an energy system,
smart-meters will register characteristic metrics, but not the reasons behind their evo-
lution. A pre-processing stage is needed to label the information if one wants to use
supervised learning. Usually it involves one or more human through an expensive and
time consuming task [4]. ImageNet was constituted thanks to the Amazon Mechanical
Turk service, which connects firms that want to label data with a work force of people
paid to do so [5]. But this kind of service cannot be used all the time. On very specific
tasks, non-expert people will not be able to label data correctly. Furthermore, confiden-
tiality issues can be a brake to these solutions. Even if few experts are available, manual
labeling is not always optimal when a lot of data needs to be processed. Some efforts
have been made to assist users to label data more efficiently [6], but the process is still
very human-dependant.
Therefore, unsupervised methods are needed to label data in an automatic manner. With
these techniques, there is no confidentiality issue and important amount of data can be
rapidly processed. Even if the label estimates are not perfect, it still allows to narrow
the research and simplify the labeling task for a potential expert.
Energy systems are associated to temporal processes. Labeling a data set corresponds
to tagging interesting events that could have occurred during a given period. This label
could then be used to train supervised model that would improve the management of
the system to design more efficient strategies and identify critical situations for example.

Aim of the master thesis
This thesis aims at developing an unsupervised method to label particular events in
energy systems.

CHAPTER 2
Event labeling
Event labeling can be viewed as the process of marking events in an unlabeled data set.
Even if it is a simple process to understand, it can be a very challenging task. First
because one has no prior knowledge about the events that have to be characterized (it is
an unsupervised learning problem) and also because the system associated to this data
set can be very sophisticated.
At the beginning of this chapter, the key concepts related to event labeling are first
described. A definition of a ’system’ and how its behavior can be described by a couple
of multivariate time series (X, Y) and a set of hidden latent states H is provided. A
formal definition of an event is also given.
Then, a mathematical modeling of the system adapted to the labeling problem is pro-
posed. The three main assumptions taken to decrease the problem complexity are de-
tailed.
It will be shown that under these hypothesis, the labeling event problem is
equivalent to a specific clustering task.
In the last part, the particularities of this clustering problem are discussed with regards
to the previous studies made in similar fields. The details of the two-step framework
developed in this thesis will be addressed at the end.
In all this chapter, the hypothetical system of the electrical load of an office building is
taken as an example to illustrate the different concepts.

interaction of the individual system elements that delineate the system-level results,
producing a desired behavior. Even if the latter description provides a global definition,
it remains very abstract. Therefore, an adapted mathematical representation of a system

2.2.2
Event labeling as a particular clustering problem
As an unsupervised technique, the labeling problem turns into a clustering task. The
particularity of it comes from the definition of the system and the external events one
tries to detect. In a classic clustering problem one will try to detect the class c of an
observation given a feature vector x. However on the labeling framework, the system is
described by a set of features (X, Y). The hidden states of S and therefore the external
events are conditioned by the evolution of y with respect to x (in Sexample the raise of
the building load in a cold day is not interesting). Any labeling model has to consider
this dependency.

2.3
Methodology
2.3.1
Literature review
In the literature, with regards to the previous modeling of the labeling task, one of
the closest field corresponds to Data Stream mining. Many real applications, such as
network traffic monitoring, credit card fraud detection, and web click stream, generate
continuously arriving data. Since classification of arriving data can help decision making,
many algorithms have been developed in recent years ([9] or [10] for example). If the
methods are more and more complex to deal with ’concept-drift’ challenge or ’novelty
class detection’, the basic idea consists of training a classifier based on historical data
to predict if an event is occurring, and if so, what is its class.
There are however three main differences with the labeling problem:
• In data stream, the ’time direction’ has to be taken into consideration: a new
observation can only be compared with the past ones, while to label an historic,
one has access to a system description over a given period, and an observation can
be compared with the past ones as well as the future ones.
• In data stream an event is classified according the evolution of the entire stream
of data x. The notion of system to which is associated a set of characteristics and
explanatory variables (y, x) is not present.
• As it is well described in [11], classification of data stream is a supervised task. It
is assumed that after an observation arrives, it will be labeled after a given time
period (thanks to expert or other means). Therefore the classifiers can be updated
regularly using most recent labeled data. As explained in section 2.2.1, for the
labeling problem one has no information about the state of the system, and the
associated classification task is therefore totally unsupervised.
This last point is the most important difference, as it makes it impossible to adapt the
algorithm developed for data stream mining to the labeling problem. In fact, very few
studies have be found in the literature about unsupervised classification of data stream,
or about a framework that could be applied to the labeling problem as described in this
report.
2.3.2
Framework
This thesis presents the basics of a two-stage framework to address the labeling problem.
Theoretically, event labeling is equivalent to cluster the system’s states to estimate the
categorical and temporal characteristics of an event at the same time. It will be explained
==PAGE:20==
12
2 Event labeling
later in the report (chapter 4) that in practice, many challenge are associated to this
task.
In the studied framework the characterization of event’s components is split, and the
theoretical clustering task is divided into two sub-problems:
• First problem: estimation of the external events’ period (no matter the class).
Many relevant studies exist in the literature that could be gathered under the field
of atypical event detection.
• Second problem: learning characteristics features of the system that can be used
to compare and characterize different potential external events. Particular types
of model can be used to learn a conditional representation of the data.
The first model provides an estimate of the system’s external events’ period. Then, the
representation learns thanks to the second model can be used to compare the different
estimated external events and cluster them into different categories.

CHAPTER 3
Data set presentation
To test and develop a labeling event framework, a baseline data set (X, Y) is needed.
The best data set would be one with complete information associated to a system where
one knows exactly the different types of exogenous events that occurred and when. The
machine learning community has always been trying to generate generic data set for
specific application that are used for benchmarks. The MNIST (Modified National Insti-
tute of Standards and Technology database) handwritten digits [12] is one of the most
famous, mostly used in computer vision to test classification and clustering algorithms.
Some labeled data sets have been widely used in the field of atypical event detection
to test algorithms, such as the ECG data set used in [13], or the Numenta Machine
Temperature data set used in [14]. They contain labeled atypical events, but the main
issue concerning our labeling problem is that they are not a couple of (X, Y) sets from
which one can detect what has been defined as external events
This is why all the models developed in the thesis have been tested on a data set of
interest for the problem. Provided by RTE it is associated to the system that will be
called the ’French national load’ (French national electric consumption).
In this chapter, the characteristics of the different features available are first described.
Then, it is explained how validation methods can be implemented to test labeling models.
3.1
National load behavior
3.1.1
Behavior and atypical events
National electric load is driven by consumer behavior. This behavior is influenced by
many factors that could be social or environmental. For example, changes in the daily
load shape and load level during the year are influenced by weather conditions including
temperature, wind speed, cloud cover, humidity, precipitation and daylight hours. How-
ever, weekly cycles are determined by workdays and holidays, and are therefore, purely
social constructions [15].
On a yearly time scale it is a highly seasonal system, and as collective behavior is
influenced by many external factors, atypical events sometimes occur. One example

would be peak load due to a large group of consumers to turn on their air conditioners
as a consequence of a hot weather. However, as many different factors impact the end
use electricity consumption, detecting this type of abnormal events is a difficult task [15]

3.1.2
TSO and national load
The Transmission System Operator (TSO) is in charge of the security and the reliability
of the energy transmission from the producers to the consumers. It has to ensure that
the production match the demand every time because electricity storage capacity is
for now very limited. Hence, a good knowledge of the load behavior is a priority for
this entity. In recent years, energy systems have become more and more stressed due
to the increasing renewable penetration and the lack of resources to improve the grid.
Thus, accurate load forecasts are essential for the TSO to support an efficient system
management.
At RTE, a whole division is in charge of the load forecasts and has a recognized expertise
in this domain. As they are delivering continuous day-ahead predictions, they developed
very standardized and efficient processes. In coordination with the dispatchers working
at the National Control Center (Centre National d’Exploitation du Système - CNES)
they publish daily ’exploitation reports’, that among other things analyze the forecast
performance of the day before.
These reports are of interest for this thesis as they
contain information about potential atypical events that are interpreted as causes of
wrong forecasts.

4.1.1
Literature Review
As explained in [19], many different categories of problem are gathered under the name
of outlier detection in temporal data, either one is dealing with:
• Time series or multidimensional data (Sensor records versus sequences of text for
example),
• Point detection or window detection,
• Fixed or variable time interval,
• Supervised or unsupervised logic.
The categories of interest for the labeling problem are related to outlier detection in a
given time series or in a data stream, where the purpose is to find ’particular elements (or
time points) within the time series as outliers, or subsequence outliers’. Other categories
where the purpose is to find outlier (a whole time serie) in a time series database, or
where there is a spatial dimension, are irrelevant for the labeling problem.
The most popular methodology to detect points outlier is to use a predictive model. The
outlier score for a given observation is computed as the variation with its corresponding
prediction. Many types of models have been tested, using regression models, multilayer
perceptron [20], or ARIMA model [21]. The model is supposed to capture the normal
behavior of the system, and it is expected that a high prediction error corresponds to an
atypical phenomenon. Supervised or unsupervised approaches are both possible. In a
supervised approach, a training set is used to determine the threshold value that separate
inliner from outlier based on the outlier score of an observation [22]. In an unsupervised
approach this threshold is set based on expert rules [23].
Detecting subsequences of time series as outlier is a more complicated task. It is for-
malized in [13] as: ’given a time series T, the subsequence D of length n beginning at
position l is said to be the discord (or outlier) of T if D has the largest distance to its
nearest non-overlapping match’. The brute force solution considers all possible subse-
quences s ∈S of length n in T and computes the distance of each s with each other
non-overlapping s′ ∈S. It is a very heavy process and many techniques detailed in [19]
have been developed to improve the computation. Even though most of the time the
size of the subsequence is taken as a parameter and anomalies have a fixed length. All
of theses methods are totally unsupervised.

External events that have to be characterized are associated to an abnormal evolution of
y with respect to x. This is the main difference with most of the problems the methods
presented in the last section addressed.

4.2.1
Influence of the model window size in the filtering
performance
As explained in section 4.1.1, predictive models are most of the time used as ’point
outlier’. Based on the predictive residuals (equation (4.6)), an atypical score can be
assigned to each observation and an event is associated to a continuous sequence of
atypical points.
The model window size concept is introduced and refers to the number of time stamps
used to build the atypical score. Usually the atypical score is constructed by comparing
the individual residuals with each other (many different metrics can be used, practical
details are provided in the next chapter): the model window size is equal to one.
With a perfect model, the lower the time step of the time series the better the estimation
of the event’s time window. But in reality, a very low time step (compared to the average
event time window scale) brings challenging issues. In fact, fitting a predictive model
that is accurate from the labeling perspective for all the time stamps (i.e equation (4.9)
respected all the time) is idealistic. Hence, one true external event can be divided in a
succession of estimated events (a post-processing stage would be needed to gather close
estimated events together) and the probability to have isolated false positive can be
high.
Therefore it can be useful to use a model window size greater than one. Sequences of
residuals are compared against each other and the atypical score is assigned uniformly
to a sequence of time stamp. It decreases the risk of false positive and to have time
stamps of a true external event identified as normal.
Of course it impacts the filtering performance i.e the ratio of events that can be detected.
With high model window size it is possible to miss events of small duration. A compro-
mise have to be found between the filtering performance and the difficulty of identifying
true external events when choosing the model window size.
For the French National Load System the time step of the time series is equal to 30

The use of predictive model to detect atypical events has been widely studied. [22] and

• a model is fitted to forecast the value of a signal y based on feature x (that can

Feed Forward Neural Network
Neural network have been loosely inspired by the functioning of the brain. Referring
to the neurons that have millions of connections between them, the idea is to create a
network of artificial neurons. These neurons (or units) are in fact constituted of inputs
from previous neurons, an activation function, and the output to the next neurons. It
can be represented with a directed graph (Figure 5.3).
Figure 5.3: Example of Feed-Forward Neural Network with two hidden layers
Formally, a FFNN defines a mapping y = f(x; θ) and learns the values of parameters
θ that result in the best function approximation. The parameters θ are all the weights
(W1, W2, WO) and biases (BE, B1, B2) of the network.
Details about the Forward Propagation, the Back Propagation of the information and
the algorithms used to train the models are provided in Annex C.
This type of model is very flexible and has improved state of the art performance in
many machine learning application. Indeed they are able to learn hierarchical represen-
tations from very granular and redundant data, such as pixels, with low need for feature
engineering. It has many hyper-parameters and can be difficult to train however, but if
well fitted it can be accurate in predictive tasks.

5.3.1
Cross validation
Training a predictive model is a supervised task. To fit the machine learning model the
data set needs to be split into two sets: a train set and a test set. The first one is used to
determine the optimal parameters of a model (the weights in the FFNN case). The test
set is used to evaluate the performance on observations never seen by the model and to
tune hyper parameters (the number of hidden layers for the FFNN model for example).
Splitting the data set in two parts (holdout method) is the simplest form of validation.
But more advanced and data efficient methods, referred to as cross-validation are most
of the time used to estimate the general performance of a model on all the data set obser-
vations. k-fold-cross-validation and Leave-one-out-cross-validation are most commonly
used. In the first one the original sample is randomly partitioned into k equal sized
subsamples. Of the k subsamples, a single subsample is retained as the validation data
for testing the model, and the remaining k −1 subsamples are used as training data.
The cross-validation process is then repeated k times, with each of the k subsamples
used exactly once as the validation data. The k results can then be averaged to produce
a single estimation. In Leave-one-out cross-validation the testing subsample is just one
observation and the training all the other ones. The cross-validation process is repeated
for all the data set points.
The French national load system is described by 5 years of data. A 5-fold-like cross-
validation method will be used to evaluate the predictive performance of the models,
selecting the different calendar years of data as subsamples (Figure 5.4). It is not exactly
5-fold-cross-validation as the subsamples are not generated randomly. This choice stems
for two reasons:
• The national load has an important yearly pattern.
• The models fitted on the subsamples will then be used to detect events. As the
detection of daily events is based on the predictive residuals, it is important that
the daily residuals come from the same model.

Event representation
In the previous chapter a method was developed to detect periods corresponding to
potential external events. If the temporal component of an event eh
t,d can be characterized
thanks to this method, one still needs to evaluate its type h.
In this chapter, it is first explained how features extraction model can be use to provide
a condensate representation of the system’s data. In the labeling problem this represen-
tation could further be used as a baseline to cluster the external event categories.
As explained in chapter 2, external events are related to an atypical evolution of charac-
teristic variables of the system y with respect to explanatory variables x. The represen-
tation learn by the model needs to take into account this dependency. In the second part
of the chapter, details about the Auto-Encoder models are illustrating why a Conditional
Variational Auto-Encoder architecture is relevant to perform this task.
The third part presents practical details about the CVAE, and explains how to fit it on
the National load system to obtain an interesting data representation with regards to S.
In the last part this representation is studied, to evaluate if the model managed to learn
relevant features to characterize external events.
6.1
Feature extraction in the labeling problem
6.1.1
Motivation
It has been demonstrated that with accurate models, the predictive method is efficient
to detect external events associated to a system S. However, as explained in chapter 2,
this is only one sub problem of the labeling problematic. An event eh
t,d is characterized
by a temporal component but also a categorical one: h. Other methods need to be
developed to tackle the second sub-problem: detecting the class of the different external
events.
In the National French Load System atypical scores have been assigned to each day. One
possible method to classify the days that are most likely to be external events would
be to compare the daily predictive residuals profiles of the most atypical days between
each other, and try to cluster them. Some attempts were tried in this direction but with
==PAGE:62==
54
6 Event representation
no success. The main difficulty is that the predictive residuals are very noisy signals
(equation (4.8), where the system answer to an external event is mixed with the system
inner noise and the modeling error. Furthermore, this method is based on the predictive
model results: if the predictive model is not accurate enough, the event clustering will
also perform poorly.
Instead another type of framework was investigated. The main idea is to use a machine
learning model to learn representative features about observations of the system. If the
model manages to encode characteristics of the different observations (daily features
for the National Load System) one can hope that external events will have particular
signature compared to the reference behavior of the system. Then the results of the
predictive method and this representation can be combined together to detect the type
of an event (further developed in the next chapter).
Another advantage of using two parallel independent methods is that this modularity
allows for automatic verification tools to detect if one or the other is performing poorly.
6.1.2
Feature extraction methods
Feature extraction is widely used in machine learning. The main principle is to derive
properties intended to be informative and non-redundant from a set of measured data.
The learnt features can then be used in many applications. By reducing the dimension
of the data it can improve the learning performance of a supervised model. It can also
be used to improve classification or clustering algorithm by using lower representative
dimensional space.
The second point is the most interesting with regard to the labeling application. If a
model can learn a reduced set or representative features to characterize the different
days of the data set, it can be used to deduce the type of the events detected thanks to
the predictive method.
Many features extraction methods exist. In image recognition for example, or natural
language processing [37] particular techniques have been developed. One of the most
common method is the Principal Component Analysis (PCA) [38] that finds the direction
of the data space that maximizes the variance (the directions where the data is most
spread out). It is used as a pre-processing tool in many machine learning problems.
But as for the event period detection sub-problem, particularities from the labeling task
must be taken into consideration. A system is described by a set of representative Y and
descriptive variables X and one wants the representative features to take into account
this dependency.
I.e a representation of y conditioned by x is expected.
From the
perspective of the National Load system, what is wanted is a latent space in which days
related to external events are gathered together by types and can be isolated.

One recent type of model that can learn such conditional representation is an Auto-
Encoder using neural networks. In the next section a presentation of the architecture of
the models is provided and the usefulness in the labeling context is discussed.
6.2
Auto-Encoder architectures
An auto-encoder, is an artificial neural network used to learn a distributed representation
(encoding) for a set of data. It is trained to encode the input x into some representation
z so that the input can be reconstructed from that representation. Hence, the target
output of the auto-encoder is the auto-encoder input itself.
If there is one linear hidden layer and the mean squared error criterion is used to train
the network, then the k hidden units learn to project the input in the span of the first
k principal components of the data [39]. If the hidden layer is non-linear, the auto-
encoder behaves differently from Principal Component Analysis (PCA), with the ability
to capture multi-modal aspects of the input distribution [40].
In the recent years more sophisticated architecture called Variational Auto-Encoder
(VAE) and Conditional Variational Auto-Encoder have proven to be very efficient in
learning hidden structure of the data and generalize. The different architectures are
presented in the next three sections.
6.2.1
Simple auto-encoder
A simple auto-encoder is just a feed-forward neural network were the input and output
layers have the same dimension, with one or more hidden layer connecting them, and
with the purpose of reconstructing the same input.
It consists of two parts: the encoder and the decoder. In the simple case, where there
is only one hidden layer, the encoder stage takes the input x and maps it to the latent
space z:
z = g1(W1x + b1)
(6.1)
Where W1 and b1 are the weights and biases of the first layer, g1 the corresponding
activation function. The vector z is referred as the latent variables or the code of x.
Then the decoder maps the latent variables to the reconstruction ˆx of x:
ˆx = g2(W2z + b2)
(6.2)

As they are neural networks, Auto-Encoder are trained by minimizing a loss called the
reconstruction loss (log-likelihood or MSE for example):
L(x, ˆx) = ∥x −ˆx∥2
(6.3)
More hidden layers can be added but usually the encoder and decoder will have symmet-
rical structure and the latent space corresponds to the layer with the lowest dimension.

6.2.2
Variational Auto-Encoder
The Variational Auto-Encoder is a generative model: instead of just learning a function
representing the data (like the simple Auto-Encoder), it learns the parameters of a
probability distribution representing the data. Since it is able to model the data, one
can sample from this probability distribution and generate new outputs.
The theory behind the VAE is explained in [43]. Here are main ideas:
==PAGE:65==
6.2 Auto-Encoder architectures
57
Vae loss
Let P(z) be the probability distribution of the latent variables. The idea of VAE is
to infer P(z) using P(z|x), i.e to make the latent variable likely under the data under
study. The main problem is that one has to infer the distribution P(z|x) as it has no
information about it. This is done using an inferring distribution Q(z|x) easy to evaluate
(like Gaussian).
In practice Q(z|x) and P(x|z) are approximated by an encoder Qθ(z|x) and a decoder
Pϕ(x|z) were θ and ϕ are the respective weights and biased. The maths are not detailed,
but as explained in [43] the purpose is to minimize the difference between Q(z|x) and
P(x|z) using KL-divergence, which can be done by minimizing the following loss (for
one sample):
li(θ, ϕ; xi) = −E[logPϕ(xi|z)] + DKL(Qθ(z|xi)||P(z))
(6.4)
Where the first term is the reconstruction loss, or expected negative log-likelihood of the
i-th datapoint. It encourages the decoder to learn to reconstruct the data. The second
term is a regularizer that measure the KL divergence between the inference model and
the true posterior P(z). It constrains the inference model to follow a given distribution.
In the VAE the prior is chosen as P(z) ∼N(0, 1) (To easily sample once the model
is trained) and therefore Q(z|x) to be Gaussian with parameters µ(x) and Σ(x). The
advantage of this modeling is that the KL term in equation (6.4) can be calculated in
close form:
DKL(Qθ(z|xi)||P(z)) = DKL(N(µ(xi), Σ(xi))||N(0, 1)) = 1

Where k is the dimension of the latent space. The encoder is not learning a function
mapping x to z (like in the simple Auto-Encoder), but the parameters µ and Σ of the
Gaussian distribution.
Reparametrization trick
Once µ and Σ are obtained from the encoder output one could sample z from the
corresponding Gaussian distribution. However, it raises a problem: the encoder and
decoder are trained using gradient descent but the sampling operation does not have a
gradient. The so-called reparametrization tricks is proposed in [43] to tackle this problem.
The idea is to put the non-differenciable sampling operation out of the network, to let
back-propagation works. z is calculated as:

6.2.3
Conditional Variational Auto-Encoder
The Conditional Variational Auto-encoder (CVAE) model was presented in [44] only one
year after the VAE (in 2014). Again, all the theory is detailed in [44] and only the main
principles are presented in this section.
With the CVAE it is possible to take into consideration a class variable c during the
generating process. It proposes ’a probabilistic model that describes the data as being
generated by a latent class variable c in addition to a continuous latent variable z’. The
inference model represented by the encoder learn a representation of the latent space
conditioned by x and y: Qθ(z|y, x). The generative model represented by the decoder
generate a signal based on z and y: Pϕ(x|y, z).
The loss of the CVAE is similar to the loss of the VAE (6.4) with c taken into consider-
ation:

li(θ, ϕ; xi, ci) = −E[logPϕ(xi|z, ci)] + DKL(Qθ(z|xi, ci)||P(z|ci))
(6.7)
It is assumed that the real latent variables is distributed under some conditional distri-
bution P(z|c). Like for the VAE, with the Gaussian assumption of the prior and of the
inference model the, KL-loss can be calculated in close form (similar to equation (6.5)).
The reparametrization trick is also used.

to a system modeling: S = (τ, H, X, Y) (be careful with the notation as x does not
have the same meaning than in the Auto-Encoder formulation), with a CVAE one could
learn representative features of the characterized system’s variables y conditioned by the
describing variables x.

System Data set
Daily profile
The system characteristic time series Y is the national load. Therefore the output of the

output is then a 48 dimensional vector.
Calendar conditions

and the weekend day or not condition still well characterized the different types of days
inside a week (see Figure 3.3).

6.3.4
Over-pruning
As noted in [45], the VAE structure can lead to simplified representation of the data
which failed to the entire network modeling capacity. In [46] this behavior is defined as
over-pruning, when a significant number of stochastic factors fail to learn anything and
become inactive. To understand this phenomenon one must consider the structure of the
loss which is the sum of two terms (equation 6.5). The first term (reconstruction-loss)
==PAGE:72==
64
6 Event representation
enforces proper reconstruction when the second term (kl-loss) captures the divergence
between the inference model Q(z|x) and the Gaussian prior p(z) for each component
independently. Therefore, one easy way to minimize the sum of these two terms for
the model is to have a lot of components that follow exactly the prior (zero mean and
unitary variance for all samples), and few highly non-Gaussian components to help
reconstruction. In this case the latent space representation (which is of interest for the
labeling problem) makes less sense.
Over-pruning issues were encountered when fitting VAE and CVAE on the National
Load Consumption. The solution proposed by [46] to solve the problem is to add a
weighting factor λ to decrease the influence of the kl term (λ between 0 and 1):
LV AE = Lrecon + λ · LKL
(6.8)
6.3.5
CVAE architecture and optimizer
On Figure 6.7 below is represented the symbolic architecture of the CVAE used on
S. The encoder is composed of 2 hidden layers with respectively 24 and 12 units, and
the decoder structure is the symmetric. Different dimension of the latent space were
tried and its impact on the data representation learn by the CVAE is studied in the
next section. A similar structure was used on daily load data in [47] and achieved great
performance. Small modifications were tried, by adding more hidden layers or modifying
the number of units of the encoder and decoder but provided similar results.
The activation functions are ReLU functions except: between the last layer of the en-
coder and the latent space, and for the temperature embedding where a linear function
is needed, for the output where sigmoid activation is used.
Each Auto-Encoder is trained using Adam optimizer with a learning rate of 0.001, on
150 epochs and a mini batch size of 20.

interesting as the model manages to learn features that can be used to classify events.

from the labeling perspective (i.e the potential external events). The prediction method

CHAPTER 8
Conclusion and future
works
This chapter presents the conclusions that can be drawn out of this master thesis work,
as well as some directions for possible future works.
8.1
Conclusion
This thesis has introduced the event labeling problem. A representation of a system and
the associated notion of external events, associated to changes in the system’s behavior
conditioned by explanatory variables were defined. As explained in the introduction,
this problem is of particular importance for energy systems to build more performing
supervised machine learning models and achieve efficient managing strategies.
As this task can be of high complexity, the focus was put on systems respecting three
main assumptions: the external event scale, event’s rarity and disjoint event hypothesis.
A particular system, the French National Load was used as a strong baseline to test and
evaluate different methods. The concept of Model window size was also introduced, and
the study focuses on methods with a 24-hour window size where daily external events
are to be detected.
A two-step unsupervised framework was developed to separately estimate the temporal
and categorical components of external events.
It was demonstrated that the predictive method can be very efficient to model the
reference behavior of a system and to detect external atypical events. The detection per-
formance mostly depends on the accuracy of the predictive model and its generalization
behavior. Simple models, like Linear regression are able to detect important external
events which are associated to an important reaction of the system (like the holidays in
the National load system), but not more subtle ones. The lower the modeling error is,
the more events can be detected. Feed Forward Neural Networks provided good predic-
tive and generalization performance and have proven to be accurate to detect important
external events, but also less evident ones. A particular attention is needed regarding
overfitting issues, as it could lead to bad system’s behavior modeling and approximate

events’ detection.
To deal with the event’s categorical component, Auto-Encoder machine learning models
were used to learn representative features about system’s observations (about days in
the National load system). The purpose is to further use these characteristics features
as a baseline to compare potential external events and cluster them into different types.
As these events are associated to an atypical behavior of the system with respect to
explanatory variables, a conditional representation of the data is needed. The Condi-
tional Variational Auto-Encoder (CVAE) architecture is designed for this purpose and
was tested. It was demonstrated that it can learn a pertinent condensate representation
of the data with regards to event labeling. On the baseline system, it managed to learn
holidays (known external events) pertinent characteristics. However, it can be difficult
to fit and it was emphasized that finding the optimal latent space dimension is necessary
to obtain a good representation, and also that overpruning effect needs to be avoided.
This representation can be used to cluster the categories of the estimated external events
detected by the predictive method. A basic methodology combining the two methods was
presented and provided encouraging preliminary results. Using successively a predictive
model and a CVAE representation proved to be a relevant framework regarding the
event labeling problem.
8.2
Future works
The framework was developed on a very particular and simplified event labeling problem
by studying potential daily events in the French national load system. As event labeling
is a complex task, it allowed to find relevant models and to prove that they could perform
on this task.
Many directions can be considered to go deeper into the framework and estimate its true
potential.
Preliminary results were presented on the last stage of the framework when combining
the two methods. More investigations are needed to find efficient algorithms to perform
clustering of the external events’ categories.
A model window size of 24 hours was used in all the study. It would be interesting to
evaluate the impact of this parameter on the labeling models. In the French national
load system, one can try to detect events of smaller periods (few hours) by decreasing
its size.
Furthermore the characteristic time series of the studies system was mono-
variate. Investigating how the framework would perform on multivariate time series is
also necessary.
The baseline system was simple, with few different types of variables. The framework
needs to be tested on other systems (more complex) to evaluate its scalability.

Finally, another potential investigation direction is the use of the labeling framework
in an iterative manner. Once different types of external events have been detected, the
information could be inserted in the explanatory variables of the system. The process
is repeated and new events are detected. The advantage of this method is that it could
allow to progressively estimate subtle external events that were not detected at the first
iteration, potentially masked by more important events.

APPENDIX A
Overfitting
Generalization refers to how well a machine learning model perform on specific examples
not seen by the model when it was learning. A good model needs to generalize well from
the training data to any data from the problem domain, to be able to perform good
prediction or classification on new samples.
Overfitting
A model is overfitting when it fits to the training set too well. It happens when the
model learns the details but also the noise of the training set and is then not able to
generalize on new data.
Figure A.1 displays the decision boundary learn by two different classifiers. The model
corresponding to the green line is overfitting when the model corresponding to the black
line is not. The green line exactly follows the training data but is too dependent to the

Formally, a FFNN defines a mapping y = f(x; θ) and learns the values of parameters
θ that result in the best function approximation. The parameters θ are all the weights
(W1, W2, WO) and biases (BE, B1, B2) of the network.
Back propagation
FFNN are trained to minimize a loss function, using gradient descent. The aim is to
derive the loss function L(x, y, θ) with respect to the parameters θ. It is done using back-
propagation, a method that takes into account the particular structure of the model.
The loss error is propagated through the network, starting from the last layer until
reaching the input layer. For a layer Li, the formula to update the weights within this
layer requires the weights in the next layer Li+1 (at its right). Therefore, the back-
propagation allows to compute the new weights iteratively.
Stochastic gradient descent
In standard gradient descent algorithm the weight of the model are updated as:
θ = θ −η∇θE[L(θ)]
(C.9)
Where η is the learning rate and the expectation is approximated by evaluating the
cost and gradient over the full training set. In Stochastic Gradient Descent (SGD) the
gradient of the parameters is calculated using only a single or a few training examples:
θ = θ −η∇θL(xi, yi, θ)
(C.10)
Where (xi, yi) is a pair of observation from the training set.
Generally each parameter update in SGD is computed with respect to a minibatch
of samples instead of a single example. First because it reduces the variance in the
parameter update and can lead to more stable convergence. Second because it allows
faster training using parallel computation and optimized matrix operations.
Gradient descent with momentum
Other optimization techniques can be used to train a machine learning model. One
example is SGD with momentum, that adds a momentum term to the weight updating
formula. It enables the optimization algorithm to get out of local minimum, if it gets
stuck. Another example is Adam optimization algorithm. . While SGD has a fixed
learning rate for all the parameters, Adam computes adaptative learning rate for each
parameter, given the first and second moment of the gradient of the loss at each iteration.
It requires little parameter tuning and can deal with large number of parameters in a
computationally efficient way
