Introduction
Since First Order Logic allows nested quantifiers, there exist dependency patters between
these quantified variables. However, sometimes, we want to express the dependencies
in different ways.
To solve this problem, several frameworks have been proposed: branching quantifier
logic by Henkin [1], independenceВӯfriendly logic by Hintikka [2] and dependence logic by
VГӨГӨnГӨnen [3].
In the latter, Jouko VГӨГӨnГӨnen, proposed an extension of first order logic introducing
the dependence atoms. These dependence atoms are expressions with the structure
=(x1, . . . , xnвҲ’1, xn), where the value of xn is determined by the values of x1, . . . , xnвҲ’1
given a set of assignments. This is an interesting way of solving the previous problem, as
this determination of variables does not depend on the different quantifiers. See also [4].
Dependence logic allows expressing the determination of a variable by a set of variables.
The analysis of dependencies between variables may be of great importance when workВӯ
ing with databases, as the latter are a useful tool to understand relationships between
variables.

This basic chess example is one direct application of dependence logic. There are many
situations that we can model with a database and study the dependencies of its variables
with dependence logic.
However, the complexity of the algorithms for checking dependence logic formulas may
exceed the resources available.
The purpose of this master thesis is to study the complexity of different algorithms for
checking dependence logic formulas, defining the computational time and memory storВӯ
age.
Dependence logic introduces a new kind of atomic formula, atomic dependence formula,
so the first part of this thesis analyses this kind of formulas isolated. Specifically, two
different algorithms. The first one checks whether a dataset is the type of a given atomic
dependence formula or notНҫ and the second finds all the atomic dependence formulas
given a dataset.
Although, dependence logic is an extension of first order logic, the atomic dependence
logic formula can be used in different fragments of first order logic. This thesis analyses
the introduction of the atomic dependence formula in three different branches of logic with
different degree of complexity. The first one is propositional logic, the second, modal logic,
and finally, first order logic.
For each one, it is proposed an algorithm to check whether a dataset is the type of a given
dependence formula or not. Then the complexity of the algorithm is studied. Depending
on the complexity, some applications are suggested.
In the end, all the results are shown, comparing the complexity of the different algorithms
and their applications.

Atomic Dependence Logic Formula
3.1
Introduction
According to eq. (E3), given an atomic dependence logic formula П•, a team X will be the
type of П• if and only if for all the assignments with the same value for all dependence
arguments, the value of the dependent variable is the same.
This dependence between variables is not stated, that means any combination is valid
if there is no assignment in the team that contradicts it. This condition causes the main
difficulty to check whether a formula П• describes the type of a team X, as it cannot be
validated for each assignment one by one, on the contrary, every assignment needs to
be validated against the rest of the assignments of the team.
Therefore, a way to check an Atomic Dependence Logic formula is taking all different
combinations of values for the dependence variables and compare if for all assignments
the value of the dependent variable is the same.

Modal dependence logic is defined as a set of worlds with relationships between them.

For relationship m, the term rm

Conclusion and Further Work
Dependence Logic expands First Order Logic expressing deterministic relation between
variables thanks to the evaluation of the formulas in teams instead of in single assignВӯ
ments. This makes Dependence Logic a great framework when working with given dataВӯ
bases. However, this evaluation over teams makes modelВӯchecking more challenging.
In this master thesis, several algorithms has been proposed to solve this problem (for
propositional, first order and modal logic) based on the DВӯDescriptive formula. However,
the complexity of these algorithm grows exponentially with the size of the dataset, the
number of dependence variables, the number of different values for the variables, and
the number of possible worlds. For that reason, sometimes is impossible to use them
because of computing limitations, but fixing some of these variables it will be possible to
use them for practical purposes.
This work could be followed by implementing all the algorithms. Also, in the thesis has
been proposed several algorithms to order different objects, when implementing them, it
would be possible to use more efficient ordering algorithms depending on the programВӯ
ming language, reducing the complexity of them.
Another interesting area for further exploration would be to find practical uses. In this
thesis, have been used several examples based on chess, this is a small demonstration
about how powerful this framework is, allowing to express easily very complex sentences.
One of the most interesting applications would be related with the implementation of AIs
applied to games. With a fixed number of variables, it is possible to analyze the time
required to check a formula, in the case that the time is reasonable, the game could be
solved by logical reasoning, if not, the AI could use heuristics models.
A different area for further research could be to analyze other frameworks describing
dependence, such as branching quantifier logic [1] or independence friendly logic [2] and
establish if these algorithms can be extended to them.
