Abstract
The autonomous robotics field is getting more and more attractive with each coming year.
These
autonomous vehicles lessen the burden of tedious tasks that can be automized, tasks such as moving
stuff in a warehouse to driving from place to place in a car.
The project explores the integration
of LiDAR and camera technologies for autonomous tasks in real-time mobile robots.
The project
aims to develop and implement real-time lane detection, human detection and avoidance, parking spot
detection, and autonomous navigation in an indoor environment. The methods used in the project are
image processing for lane detection, deep-learning model YOLO for human detection and avoidance and
for parking spot detection, and lastly SLAM for indoor navigation. The project was made open-ended,
which means this is the building block towards a more robust autonomous system. The project ended
with successfully achieving lane detection on a mobile robot and the Roadrunner, successfully detecting
a human and avoidance in static situations, parking spots were detected with 60-80% certainty and
indoor navigation was successfully achieved by the use of SLAM and QR-codes.

CHAPTER 1
Introduction
The transition toward autonomous living is becoming more apparent in an advancing world. One field
that is showing accelerated growth is autonomous robotics, and one of the more pressing matters is the
autonomy part of the car industry, the need for self-driving cars. Human error is one of the leading
causes of traﬀic accidents. Autonomous vehicles can potentially reduce the number of accidents caused
by human error, such as distracted or impaired driving.
According to recent studies, autonomous
vehicles have been found to be safer than manual-driven cars. [Par23]
This chapter will briefly introduce the thesis, outlining the background and objectives of the project.
At the end, an overview of the chapters will be provided.
1.1
Background
This master thesis will use a mobile robot with LiDAR and a camera in real-time and will utilize these
sensors for different goals. The goals will be presented in the objective section of this chapter.
1.1.1
Desciption of Shell challenges
The Roadrunner will engage in 3 different autonomous tasks, autonomous driving on a track, au-
tonomous parking maneuverability, and obstacle avoidance through gates. This project will try to
solve autonomous driving on a track and parking detection and maneuverability in Gazebo
and on a mobile robot in real-time. Gazebo is a real-time simulation environment and will be
explained further in the analysis section.

2.3
Object Detection using vision
The detection of objects in autonomous vehicles provides more autonomy for the vehicle and the safety
of both the driver of the vehicle and pedestrians. The estimation of the position of an object is a very
well-known problem within the autonomous world. This problem has been tackled with both LiDAR
and camera over the years and in some cases with both. In recent years deep learning and the usage of
only cameras, e.g. as seen in a Tesla, has been the front runner for the solution to this problem within
the autonomy of the car industry.
2.3.1
Deep Learning Approaches
The usage of deep learning for object detection is becoming better and better, Convolutional Neural
Networks (CNNs) have become the de facto standard for human detection in images and videos. Re-
searchers have proposed architectures like Faster R-CNN and YOLO (You Only Look Once) which
have shown significant improvements in accuracy and real-time performance. YOLO is the front-runner
when it comes to object detection, this is because of its fast detection rate. [PPG23] [DPG18]
2.3.2
Parking detection 2D LiDAR
A LiDAR-based parking spot search algorithm for autonomous vehicles was developed in [Ima+21]. The
algorithm uses a 2D LiDAR sensor to detect parking spots in real-time and has been tested on a dataset
collected from a mobile robot equipped with a LiDAR sensor. The results showed that the method can
effectively detect parking spots under various conditions.

2.4
Summary
While there are multiple approaches to developing lane detection for autonomous vehicles, the use of
camera-only methods leveraging image processing has gained traction due to its cost-effectiveness and
is the least computationally heavy with good accuracy. Unlike the 3D LiDAR method, which is both
computationally intensive and expensive, camera-based approaches are more economical. Therefore
going forward, the solution for lane detection in this project will be solved using a camera
only with image processing methods.
From these related works, object detection can be done with both a LiDAR and a camera, however,
LiDARs are a lot more expensive than cameras, so if the object can be detected with almost the same
accuracy then it would be economically best to choose the camera over the LiDAR. Therefore going
forward, the solution for Human detection and parking detection will be done using a
camera only in this project.

3.2
Image Processing
Image processing is commonly used in computer vision for line following, lane detection, object detection,
etc. The process of image processing starts with an RGB (Red, Green, Blue) image and this is then
converted to either a grayscale image or some other format of image intensity depending on what the
image processing is used for. Image processing used for lane detection usually converts the image into
grayscale since the lanes mainly consist of white lines. Grayscale is the best way to differentiate white
color from other colors, as it captures only intensity information, which makes it easier to distinguish
between white (high intensity) and other colors (varying intensities).
3.2.1
RGB to Grayscale Conversion
The image processing is done by transforming the image from its original RGB color space to something
in 2 dimensions, like grayscale. The reason behind this is that color images are usually depicted as three-
dimensional arrays, each dimension representing one of the RGB color channels and this can be seen in
figure 3.11.
Figure 3.1: The figure represents how a normal colored image consists of 3 main colors RGB (Red,
Green, Blue) and how an image in those 3 colors if combined will be equal to a ”normal” colored image.
If the image is processed in RGB, the complexity of the following image-processing tasks will increase.
So by switching the image to grayscale, it would be possible to streamline it down to a two-dimensional

3.2.2
Edge Detection using Canny Algorithm
Edge detection in the image is done by a process called Canny in OpenCV. Canny looks for a jump in
pixel intensity and thereby can detect the edges in the image. This means that Canny needs an image
that is in a 2D matrix and focuses on intensity, which means that the image needs to be in an HSV(hue,
saturation, value), Grayscale format, etc.
The Canny algorithm is made of 5 steps and these steps can be seen below [Sah23]:
1. Noise reduction: The first step is to reduce the noise in the image, so the focusing objects will
have a clear edge compared to the background. This is done by applying a Gaussian blur.
2. Gradient calculation: This step calculates the gradient of the image to detect edge intensity
and direction. Sobel filters are applied in both horizontal and vertical directions to calculate the
derivatives Ix and Iy.
3. Non-maximum suppression: The aim here is to thin out the edges. The algorithm scans
through the gradient intensity matrix to find pixels with maximum value in their edge directions.
If a pixel is not a local maximum, it is suppressed.
4. Double threshold: Pixels are categorized into three types: strong, weak, and non-relevant based
on their intensity. Strong pixels have intensities higher than a high threshold, while weak pixels

the low threshold.
5. Edge Tracking by Hysteresis: Weak pixels are converted to strong pixels if they are connected
to strong pixels.
This step ensures that true edges are highlighted while discarding weak or
spurious edges.

3.3
Kalman Filter
The Kalman filter is a mathematical algorithm used in navigation and computer vision for predicting
the next result and reducing the error. The Kalman filter is used in computer vision for tracking objects
and analyzing the motion of objects. The Kalman filter is a recursive estimator, which means that it
can make an estimation based only on the current data and the previous one.
The Kalman filter operates in two steps based on two main states: the prediction state and the
update state.
3.3.1
Prediction
The prediction state is the state of the system using the previous input and the measured input, using
these two inputs the prediction state predicts the future state. The equations involved in this step are:
xnew = F × xold
Pnew = F × Pold × F T + G × GT × acceleration_variance
x is the state estimate, P is the covariance matrix representing the uncertainty of the estimate, F
is the state transition matrix, and G is the process noise matrix.
==PAGE:23==
3.4 Object Detection
15
3.3.2
Update
The update state incorporates the new measurement to reduce the error. This step is crucial as it helps
in bounding the uncertainty of the new measurement. The equations for the update step are:
y = z −H × x_new
S = H × P_new × HT + R
K = P × HT × S−1
xnew = x + K × y
Pnew = (I −K × H) × P
Here, z is the measurement, H is the measurement function, R is the measurement noise covariance,
and K is the Kalman gain which determines how much the prediction should be adjusted based on the
measurement.
These equations are used in a closed loop and done until there is no new measurement, the illustration
of the closed loop can be seen in Figure 3.5.
Figure 3.5: The figure illustrates how the Kalman filter is a closed loop. The Kalman filter is first
initialized, then it predicts the state ahead, and the update block uses the prediction as a way to reduce
the error in the new measurement.
3.4
Object Detection
Deep learning methods are often used to identify and locate objects in computer vision. Object detection
in computer vision is categorized into two types: one-stage detectors and two-stage detectors. Two-stage
detectors like RCNN, are slower and more complex to implement, however, these detectors are generally
more accurate for complex tasks and better at detecting multiple objects in close proximity. One-stage
detectors like YOLO, are faster because they perform detection in a single ”shot” through the network,
they are easier to implement and they still have high accuracy. To summaries:
• One-stage detectors are faster, which makes them suitable for real-time applications, however
generally less accurate.

• Two-stage detectors are slower, which makes them unsuitable for real-time applications, however,
they are more accurate and suitable for applications that require high accuracy in non-real-time

3.4.1
YOLO
YOLO is a one-stage object detection algorithm and was first presented by Joseph Redmon et al.
[Red+16] The YOLO algorithm is used with two other files, the configuration file and the weight file. A
configuration file that provides the algorithm with the necessary parameters for the model to function.
The configuration file is a text-based file that provides the settings for the network like batch size,
learning rate, momentum, etc. The weight file contains the trained parameters for the neural network
model.
The algorithm applies a single neural network to an entire image, which it subdivides into S × S
cells. Within each cell, the network will predict if there is an object or not. For each predicted object
the network will output the probability of a class and an offset value for the B bounding boxes and
secondly, it computes a confidence score for these predicted bounding boxes. There will be a lot of
bounding boxes as can be seen in Figure 3.6, so therefore a threshold is set. This threshold will discard
all bounding boxes below the threshold and finally output the final detections as can be seen also in
Figure 3.6. [Red+16]
Figure 3.6: The figure shows how the deep learning algorithm splits the image into a S × S grid and
then it proceeds to divide into bounding boxes and class probability map.[Cha17]
3.5
SLAM
SLAM stands for Simultaneous Localization and Mapping. SLAM is a method used mostly by
indoor robots and autonomous vehicles. This method allows you to build a map in an unknown while
simultaneously keeping track of the robot’s/vehicle’s own location within that environment. SLAM is
very important for some of the robots we our daily life, eg. a robotic vacuum cleaner.
3.5.1
Components of SLAM
1. Localization: The localization determines where a robot is currently located in the surrounding
environment. This is often achieved using sensors like GPS, IMUs (Inertial Measurement Units),
and cameras. These sensors are used to monitor the robot’s motion and observe the environment.
==PAGE:25==
3.6 Mobile robot and hardware
17
2. Mapping: The mapping is the creation of a representation of the environment. This can be a
2D grid map, 3D point cloud, or other representations. Sensors like LiDAR (Light Detection and
Ranging) and cameras are often used for this purpose.
3. Loop Closure: The loop closure ensures when a robot revisits a previously explored area, that it
doesn’t mess up the previous work. This helps in correcting any drifts or errors that might have
accumulated in the map over time.
3.5.2
Challenges
SLAM usually uses sensors like LiDARs, which can be problematic when used outdoors, as the light from
the sun can interfere with the sensor. For these reasons, SLAM is often used in an indoor environment
for autonomous robotic purposes.
3.5.3
Applications
Some of the applications include:
• Robotics: Indoor environments like homes or factories.
• Drones: Drones are using SLAM as a way to navigate when GPS is not an option.

Field of Movement: The Ackermann Steering Principle
The RosBot Mini operates based on the Ackermann steering principle, a geometric arrangement that is
similar to that of the Roadrunner’s steering or a car’s steering.
In Ackermann steering geometry, the inner and outer wheels rotate at different angles during a turn,
ensuring that all wheels move along paths with a common center point. This is represented in figure
3.8.
Figure 3.8: The figure shows the steering of an Ackermann vehicle and how the steering of a turn
revolves around a center point outside of the vehicle.
However, it’s important to note that the Ackermann steering is limited and this steering cannot
execute sharp turns, unlike omnidirectional robots that can move in any direction. The RosBot Mini,
like any vehicle using Ackermann steering, is limited to forward and backward motion.
Therefore, while planning the movements or paths for the RosBot Mini, these restrictions must be
taken into account. For instance, tight spaces that require lateral movement or complex maneuvers
may pose challenges. However, in open spaces or paths with suﬀicient room for turns, the RosBot is
capable of navigation, with the added advantage of predictable and stable movement patterns due to
the Ackermann steering.

4.2
Lane Detection
This section presents the image processing approach for detecting lane lines on the road. Lane detection
is a critical component of autonomous vehicle navigation. Accurate lane detection ensures safe vehicle
navigation, acting as the ”eyes” of the vehicle. The primary goal of the image processing is to distin-
guish the lane line from the road. The design of the lane detection algorithm involves using Canny to
extract the features from the image and using Hough Transformation for line detection, Kalman filters
for smoothing and prediction, and a simple waypoint generation trajectory.
The lane detection process is described in the article: ”Building a lane detection system”1. And then a
Kalman filter is applied to the detected lines in the image, this part is not part of the article, and it is
added to perceive a more accurate detection of the lane. The process follows these steps:
1. Image Preprocessing: The first step in the process involves converting the input image to gray-
scale format. This is done because the white color, which the lane lines are, is easier to distinguish
from other colors in a gray-scale format rather than RGB or another image format.
Figure 4.1: This figure shows the original image cropped on the left side and after preprocessing it
and converting the RGB image to a gray-scale image, the result of this can be seen on the right side in
the figure.
2. Gaussian Smoothing: To reduce the image noise and detail, the image is smoothed using a
Gaussian filter. This process involves convolving the image with a Gaussian function, this will blur
the image and smooth out rough edges. The Gaussian filter works by giving higher importance to
1https://medium.com/analytics-vidhya/building-a-lane-detection-system-f7a727c6694

24

points close to the center of the window (the pixel being processed) and less importance to points
farther away. This is reflected in the Gaussian kernel, as shown in Figure 4.2.
Figure 4.2: The figure shows the output of using a Gaussian filter to smoothen an image.
3. Canny Edge Detection: The Canny Edge detection algorithm is applied to the smoothed image.
This algorithm works by detecting areas of the image with sharp changes in pixel intensity, which
are likely to correspond to edges, or in this case, lane lines.
Figure 4.3: This figure illustrates the Canny function with the values 50 and 200 has been applied
to a blurred gray-scale image. This means that only intensities that change from 0-50 to 200-255 are
considered edges.

unwanted feature in the top corners and also a little bit in the middle, this is why the next step
is important in lane detection, which is designating a region of interest.
4. Region of Interest Extraction: Not all edges detected by the Canny algorithm are relevant
for lane detection. Therefore, a ROI (region of interest) is defined and only the lane edges are
considered in the following steps. Typically, this region is defined as the lower portion of the
image where lanes are expected to be, and as the lanes are expected to move further away from
the camera, a Trapezoid shape is chosen for the ROI.

their slope. As the lane moves further away from the camera, into the horizon, this results in
the left line of the lane having a positive slope and the right line having a negative. When this
is done, the Kalman filter is initialized and updated with the new measurements from each line.
The algorithm will draw a waypoint in the middle of the detected lane, indicating the center of
the lane.

To analyze why this delay occurs, a test is conducted using the same repository in the Roadrunner
workspace and testing it on the real-time simulation environment Gazebo, to see if the same delay
occurs. If the same delay occurs, this would mean that there might be something wrong with the ROS-
YOLO repo. After installing the repository to the Roadrunner’s workspace and testing it out, the same
delay occurred of about 10-15s. As the same delay occurred, a thorough look by searching the internet
for answers provided an answer that the detection uses the CPU and not the GPU. the architecture of
GPUs is more powerful than CPUs for specific types of computations, and this applies to this case of
object detection with a deep learning model. [Clo23] [Mun]

CHAPTER 6
Discussion and Future Work
6.1
Introduction
This chapter will discuss the various challenges that occurred during the testing of lane detection, human
detection, and parking detection. And then solutions, workarounds, and future work will be presented
as to how someone could proceed with this project.
6.2
Challenges during the testing of lane detection
As the RosBot proceeds along its path, it meets the curve of the lane, as shown in Figure 6.1. Due to
the camera’s position at the front of the RosBot and its close narrow field of view, and the usage of a
Kalman filter that can predict only a straight road/line, the RosBot does not turn in a curve until it
reaches it. This results in the RosBot not initiating a turn until it loses sight of one of the lane lines
and until the predicted road tilts with curvature.
Figure 6.1: The lane detection performed on the image received from the camera right before the lane
is curving.
A notable issue that arises when the RosBot loses sight of one of the lane lines is the possibility of
an overshoot. This could lead to a delay in the robot’s turn, causing it to cross over the lane boundary.
When this occurs, the RosBot’s camera might only capture the right lane line, and due to the viewing
angle, the algorithm could potentially misidentify it as the left line. A variety of potential solutions to
this challenge will be discussed and explored in the future work section.
==PAGE:62==
54
6 Discussion and Future Work
6.3
Challenges during the testing of Human detection and
avoidance
1. Detection Delay: The initial tests with ROS-YOLO for human detection on the mobile robot
showed a significant delay between the detection and the actual camera feedback. When a person
changed position, the detection system took around 10-15 seconds to update the person’s new
position, which is not feasible for real-time human avoidance.
2. Computation Power: The delay in detection was attributed to the computation power being
set on the CPU and not the GPU. This was discovered after the same delay occurred when the
deep learning algorithm was tested on Gazebo on a computer with powerful GPU.
3. Noise from LiDAR: During testing, some noise was detected from the LiDAR. This noise was
considered an external error, and it occurred even when another mobile robot was used for testing.
4. Human Avoidance Algorithm: The human avoidance algorithm was designed to detect a
human, determine if the human is on the right or left side of the robot, and then shift the desired
trajectory to the opposite side with the width of the human. This method, while simple, may not
be the most eﬀicient for maneuvering around a human, as this method will only be applicable to
a static scenario.
6.4
Future Work
6.4.1
Lane Detection
One of the challenges of lane detection is the curves, due to the way the algorithm is detecting the lane
and the way it makes the waypoint, the robot will not see the turn until it is at it. Due to the limitation
of the turning of the anchorman-steering, the robot will sometimes have a small overshoot. A solution
and a future work to be tested could be:
1. Instead of drawing only one waypoint, one could detect a series of waypoints and make an approx.
polynomial and this could determine the steering of the robot’s angular Z. An idea of how this
could be like can be seen in figure 6.2.
Figure 6.2: The figure presents the idea of having multiple waypoints and this can later on be used
in combination with an approx. polynomial.

6.4 Future Work

6.4.2
Human Detection
1. GPU Computation: The delay in human detection was resolved by shifting the computation
power from the CPU to the GPU. So a possible future work to make the detection work on the
mobile robot is to apply to the mobile robot with a GPU or send the information to an external
computer with ROS and then do the detection and send the information about the detection
back to the robot. The last one might only be eﬀicient in a testing environment as delays due to
sending the data over the internet can occur.
2. Human Avoidance Algorithm: The human avoidance algorithm was designed to avoid a
human in a static environment. A possible future work would be to apply the Kalman filter to
the detected human and this could predict where the human was going with a certain velocity.
This would also make the vehicle more robust in choosing which way it should avoid hitting the
human.
6.4.3
Parking Detection
6.4.3.1
2D detection to 3D
The parking detection only detects the parking spots in a 2D position, so possible future work would
be to combine the current work with a stereo image. The stereo camera will be used to find the depth
of the object. Combining the depth with the 2D position would then give a 3D position of the parking
spot in relation to the camera/vehicle. This representation is also illustrated in figure 6.31.
Figure 6.3: The image represents an overview of how depth estimation is done using a stereo camera.
On the right side of the image, a figure can be seen, LRSd. L is a point on the left camera of the stereo
camera, R is the same point on the right camera, S is the point where the point where lines of the two
points intersect, and d is the depth of the point calculated by triangular methods.
After knowing the 3D location of the parking spot in relation to the vehicle, a path planner could
be designed to reach the desired location.

CHAPTER 7
Conclusion
This thesis had three main objectives: lane detection, human detection, and parking detection and at
the end this was done successfully with room for improvement, resulting in significant advancements in
robotics and autonomous vehicle technology.
Lane detection was achieved by combining lines from the Hough transform and a Kalman filter. The
lane detection performed great results and completed the track designed in ASTA with a robust ap-
proach. Continuing the lane detection to the simulation of the Roadrunner, the lane detection was also
able to perform with great results and completed the London track designed in Gazebo. From this, it
can be concluded that lane detection has been achieved for real-time tasks.
Human detection was achieved by using the deep learning algorithm YOLO and avoidance was achieved
by knowing the 2D position of the human and manipulating the desired trajectory to follow the lane
and on the mobile robot it was achieved through the use of the 2D LiDAR.
Parking detection was done by successfully training a custom deep-learning model to detect parking
spots for both the Roadrunner and the mobile robot. The parking spots were accurately detected most
of the time with a 60-70% accuracy. This enables the robot or car to identify parking spots with high
accuracy.
A LiDAR was used to measure distances, mapping, and obstacle avoidance by the use of SLAM. The
robot achieved the tests that were put and this was somewhat successful.
Overall the objectives were successfully achieved and this is a step forward in the field of autonomous
systems and robotics. The project is ready to build upon and achieve even better lane detection, human
detection and avoidance, and parking detection for maneuverability.
