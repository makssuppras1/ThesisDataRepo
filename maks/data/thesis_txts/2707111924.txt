Abstract
Object detection for images and videos have had a large growth in recent years, and it
is seen implemented in various industries. It has now been developed object detection
models that are able to perform complex detections with real­time performance. Object
detection models are able to locate pedestrians crossing the street at real­time speed,
and detecting cancer cells from biopsies. Object detection are able assist fields like au­
tonomous driving, health care, assistive robots and video surveillance.
This thesis will implement, test and evaluate one of the state of the art object detection
models, You Only Look Once (YOLO). Two different versions of the model will be tested
and compared, YOLO v4, and a simplified model, YOLO v4­tiny. The network is trained
on four objects using the COCO dataset and evaluated in real­time through video. The
result is a detector with an accuracy of 63.54% for YOLO v4 and 35.33 % for YOLO v4
tiny when run on a testset. The model is tested by evaluating the performance of the
detector with different distance, brightness, distractions and speed. It is found that YOLO
v4 performs better in real­time with better predictions, except when the speed is high,
where the compressed tiny model performs better.

Introduction
Deep learning have in recent years revolutionized industries like robotics, medical tech­
nology, manufacturing, retail and cybersecurity. It has now become difficult for companies
to stay relevant without implementing some sort of deep learning system. Deep learning
consists of artificial neural networks that are inspired by the network of the human brain.
By feeding a computer a large amount of data samples and identifying what that data
contains, the computer will be able to learn patterns and make predictions. One of the
reasons for the recent growth in deep learning, is the availability to data. Data drives the
machine learning system. The more data the deep learning system receives, the better it
learns to make correct predictions.
The developments within deep learning have led to tremendous improvements in com­
puter vision techniques. Object detection is an area within computer vision and artificial
intelligence that has been receiving special attention in the recent years. Object detection
is a procedure to identify and locate objects in an image. It has a numerous of applications
in many different fields, such as face detection, autonomous driving, surveillance, military
and medical devices. Object detection and deep learning have the possibility to make a
huge impact on the real world. The advanced in object recognition proved in 2020 that
Google’s AI is able to detect breast cancer from mammograms better than radiologists.
[1] This is just one example of how object detection is set to transform many industries.
This thesis will implement, test and evaluate the performance of one of the state of the
art object detection algorithm called You Only Look Once (YOLO).
You Only Look Once(YOLO) is inspired by how the humans can detect and identify objects
in the blink of an eye. The algorithm of YOLO is simple; A single convolutional neural
network predicts multiple bounding boxes and the class probabilities for each box. This
basic algorithm makes YOLO extremely fast, making it one of the best algorithms for
real­time object detection. YOLO is more than 1000x faster than the R­CNN detection
algorithm, which requires thousands of networks to make prediction on a a single image.
[2] A study published May 2021 compared three major object detection algorithms, and
found that YOLO had the best overall performance in terms of speed and accuracy.
YOLO has large­scale applicability with many use cases, particularly for autonomous driv­
ing, vehicle detection and intelligent video analytics. The YOLO algorithm can be imple­
mented for detecting the use of mobile phone while driving. A study from 2021 found that
the YOLO is the best algorithm for this case, with a 96% accuracy.[3] Another use case
is animal detection, where animals in the wildlife is counted to help the fields of ecology
such as wildlife biology and conservation biology.
This thesis will implement YOLO version 4(YOLO v4) and the compressed version, YOLO
v4­tiny. They will be trained on four objects, where two are very small and two are larger.
In this way it is possible to compare the performance on small vs. big objects, as one
of YOLO’s weaknesses is detecting smaller objects. The objects will be trained using a
large set of images from the COCO dataset. The final weight will be run on a testset of
1000 images and the precision will be evaluated. Then, the object detector will be tested
in real­time through video in different scenarios.

1.1
Problem statement
The purpose of this thesis is to implement state­of­the art object detection algorithms,
YOLO v4 and Tiny­YOLO v4, and evaluate the performance in terms of speed and accu­
racy. The thesis aims to find out if the YOLO v4 and YOLO v4­tiny detectors has sufficient
performance to be used in real­time in scenarios with different distances, distractions,
brightness and in fast videos where the object is displayed in only a few glimpses. It will
also be evaluated if the detector is better at detecting large objects, as in a real­time envi­
ronment, like autonomous driving, it is most important to be able to see large objects like
traffic signs and pedestrians.
These different factors is important to investigate since implementing a object detector in
a real­time environment, for instance for autonomous vehicles, there will be a variation
in brightness, object distances and distractions covering the object. Being able to detect
an object fast is also a critical factor for real­time performance. For instance in a critical
environment like autonomous driving, an animal might suddenly run in front of the vehicle.
In a scenario like this it is extremely important to have a object detector that is fast, such
that the vehicle have time to stop in time. From reading the theory behind YOLO, it is
expected that YOLO v4 will have higher accuracy than YOLO v4­tiny, but that YOLO v4­
tiny will be faster than YOLO v4.

2
Background
2.1
Object detection
Object detection consists of identifying an instance of a class to which the object belongs
to and predicting the location of the object in the image by a bounding box. Detecting one
single class from an image is known as single class object detection, whereas detecting
multiple objects in an image is known as multi class object detection. The state­of­the­art
results obtained using deep learning for object detection have proven great performance.
[4]
2.1.1
Object detection using deep learning
Deep learning is a rapidly developing research area within machine learning and artificial
intelligence. Deep learning is developed by drawing inspiration from the biological activity
of the human brain. Deep learning makes it possible to input raw data without extract­
ing any features beforehand. Deep learning is based on learning to automatically detect
these features by going through a training process of adjusting the weights in a neural
network. When training a neural network, an input vector is presented to the network,
which produces an output vector. The error between the input and output vector is calcu­
lated. Based on this error, the network weights are adjusted to attempt match the input
with the output. During training, the network should gradually converge to values such
that the input vector matches the target output vector.
When using deep learning for an image, the pixel intensity values are given as input to
the deep learning system. The pixel values are fed to a number of hidden layers that
extract features from the image. These layers are built in a way where simple features
are extracted first, like corners and contours, and in the higher levels these features are
combined to to get more abstract object classifications for the next layer. The output layer
then calculates the probabilities that the image belongs to one of several classes.
2.2
Convolutional Neural Networks
Convectional Neural Network is a specific type of feed­forward neural networks. The al­
gorithm trains on a large set of images where the label are determined beforehand by
humans. The algorithm learns to identify objects in an image by associating character­
istics of an object with the appropriate label. When later presenting an image containing
unlabeled objects that the algorithm has learned, a correctly trained should usually assign
the correct label. Convolutional neural networks resembles how humans classify images,
by recognizing specific elements anywhere in the image that determines each specific ob­
ject class. Convolutional neural networks have become the the state­of­the­art for object
recognition and used for many different AI tasks.
The structure of convolutional neural networks consists of two hidden layers, convolu­
tion layers and pooling layers. Convolution layers look for instances of small patterns in
the image, while pooling layers downsamples the image dimension in order to speed up
the computation process. The neural network architectures uses many convolution and
pooling layers. The following describes the these two layers.[5]
2.2.1
Convolutional layers
The convolution layer consists of a large set of convolution filters. Each of these are a
template that determined is a local feature is found in an image. A convolution filter is

based on a simple calculation, called a convolution, which consists of multiplying matrix
elements and then summing the results. This gives the convoluted image, which locates
parts in the original image that has similarities to the convolution filter. Convolution filters
are arrays of size l1 x l2, where l1 and l2 are positive integers that may not be equal.
In a convolution layer, many different filters are used to find different patterns and shapes
in an image. In convolution neural networks this filters are learned for the specific detec­
tion task. The filter weights can be pictured as the parameters when transferring from a
input layer to a hidden layer, where there is one hidden component for every pixel in the
convolved image.
Because the input image is in colour, it has three channels represented by a three dimen­
sional feature array. Each of these channels consists of a two dimensional feature map,
with one map for each of the colours, red. green and blue. A convolution filter will also
have three channels, one for each colour. The results of the three convoltions are added
together and creates a two­dimensional output feature map. [6]
2.2.2
Pooling layers
The pooling layer is used to compress the size of feature maps. The map is reduced by
a factor called stride or Downsampling factor. For instance, downsampling by a stride of
s=2 on a 10­dimensional vector means we have to pick every alternating value, starting
from index 0. This will result in a 5­dimensional vector. Pooling works in the same way
as this, where it evaluates the elements between alternating pixels.
There are several methods for performing pooling, where two of the most commonly used
methods are Max pooling and Average pooling. Max pooling reduces the dimension of
the vector by choosing the largest value in the local neighbourhood of the current element,
where the size of the local neighbourhood is determined by d. Average pooling is similar
to max pooling, but instead calculates the average in every local neighbourhood of the
current element. [5]
2.2.3
Fully connected layers
Fully connected layers is the last part of the network and where the input is the output from
the final pooling or convolutional layer. The input is flattened, meaning it is transformed
from a matrix and into a vector, before being fed into the next fully connected layers, where
weights are applied to make correct predictions. The output of the fully connected layer
gives the final probabilities for each class. The output is in many cases combined with a
Softmax activation function, which normalizes the network output to be between 0 and 1
in order to calculate the probabilities and classify the different objects.
2.2.4
CNN architecture
Each filter in a convolution layer outputs a two­dimensional feature map. The number of
convolution filters in a convolution layer is equal to the number of elements at a specific
hidden layer in a neural network. Figure 2.1 shows an example of an CNN architecture
for an image classification task.

Figure 2.1: Example of a CNN architecture. From [7]
The input to the network is a three­dimensional feature map of a color image. Each color
channel consists of a two­dimensional pixel feature map. Every convolution filter gener­
ates a new channel, where each is a two­dimensional feature map. This first convolution
step generates a feature map with six channels. The next step is a max­pool layer, which
reduces the size of each feature map by a factor of two in each dimension. This convolu­
tion and pool process is repeated for two more layers. Since the dimension of the feature
map is reduced after each pool layer, the number of filters in the next layer is usually
increased to compensate for this. When the convolution and pool process is completed
the feature maps will be reduced to only a few pixels in each dimension.
In the last step the pixels in the feature maps are flattened. This means that the pixels in
the feature map is converted to a vector and fed into a fully connected layer. The output
layer is a Softmax activation function that gives prediction for each label. [8]
2.2.5
Training of CNN
When training a CNN, a batch of images is fed to the network, where the softmax function
outputs the probability estimation. In the next step, the loss function calculates the error
between the predefined labels and the estimated labels. The goal is to minimise the loss
function. The loss function is the sent to an optimizer that calculates how the weights and
biases of the network needs to be adjusted to reduce the loss. A complete iteration of this
algorithm for the entire dataset is called an epoch. [9]
2.3
State of the art object detection methods
There has been a huge progress in the last ten years on object detection due to the
rise of convolutional neural networks. There are now many available object detection
algorithms to choose from, and which architecture to go with depends on the type of
application. Object detection algorithms have a trade­off between speed and accuracy,
and it can be difficult to pick one as the best in terms of both speed and accuracy. Some
are very accurate, but requires a high computation time, and others are faster, but gives
a less accurate detection. For real­time applications like autonomous navigation, speed
is important. Implementation of object detection into embedded devices will also require
an architecture with a low computational requirement.
Object detection models can be separated into single­stage detectors and two­stage de­
tectors. [10] Two­stage detectors use a Region Proposal Network to create regions of

interests in the first step and then sends the region proposals to the next stage which is
object classification and refinement of the bounding box prediction. Theses models have
higher accuracy, but are usually slower. Single­stage detectors skips the region proposal
stage and generates classification and bounding box prediction at once. Single­stage
detectors are less accurate, but are much faster than two­stage detectors. The next two
subsections will describe two of the most popular state of the art object detection algo­
rithms besides YOLO.
2.3.1
Faster R­CNN
R­CNN stands for region­based Convolutional Neural Networks. Faster R­CNN is a two­
stage detector. [3] It was published in 2015. This algorithm uses region proposals for
object segmentation and R­CNN for object detection. Faster R­CNN is not as fast as
other real­time detection algorithm, and operates at only 7 frames per second.
The original R­CNN uses a Selective Search Algorithm, where region proposal are ex­
tracted from the input image. Then, the region proposals goes thought the CNN compo­
nent and extracts distinct features as a vector output. The extracted features are lastly
fed into a support vector machine to produce final candidate proposals. This model had
many weaknesses. The training of R­CNN is very time consuming, and no where near
obtaining real­time performance. In addition, the Selective Search Algorithm used in R­
CNN is a fixed algorithm, and therefore machine learning cannot be used, which can lead
to incorrect candidate region proposals being produced.
In faster R­CNN the time­consuming Selective Search Algorithm is removed from the
model and instead uses Region Proposal Network (RPN). RPN divides the regions into a
n x x grid. It needs to run less number of times as compared to Selective Search. After
the image has been fed to the RPN, the predicted regions are reshaped using a Region
of interest (ROI) pooling layer. The ROI pooling layer is a type of max­poolinng layer for
reshaping the image into a fixed size, so the image can be fed into the fully­connected
layer, as this layer expects the same input size. The output from the ROI pooling layer is
then used to classify the different proposed regions within the image.
2.3.2
Single Shot Detector (SSD)
Single Shot Detector removes the Region Proposal network used in faster R­CNN which
significantly speeds up the detection algorithm. [11] SSD is made up of a network that
generates standard sized bounding boxes and for each time an object occures in on of
those boxes, a score is generated. After the score is generated, a non­maximum sup­
pression is used to calculate the final detection results. SSD consists of a backbone with
of a standard architecture of convolutions layers, which is called the base network. After
these layers an auxiliary structure which consists of convolutional feature layers. These
feature layers gradually decreases in size to make it possible to predict detect ions at
different scales.
SSD also uses convulutional filters that generates a fixed number of predictions. For every
future layer, the compoents for producing prediciton variables for a possible detection is
a kernel that creates a confidence score for each class.
SSD uses different default boxes with each feature map cell, which creates multiple fea­
ture maps at the end of the network. At each feature map cell, the offsets are predicted
relative to the fixed box shapes in the cell, and the class score for each class is calculated.
The fixed boxes are similair to the anchor boxes used in Faster R­CNN, but in the SDD
model the boxes are applied to several feature maps of different resolutions. [3]

2.4
You only look once (YOLO)
The original YOLO model was released in 2015. Since then, several versions of the model
has been released in the attempt to improve the performance. There are currently five
versions of YOLO that has been released. These are YOLO v1, YOLO v2, YOLO v3
and YOLO v4 and YOLO v5. There is also a compressed version of YOLO, called Tiny­
YOLO. The following sections will briefly describe the main features of the different YOLO
versions.
2.4.1
YOLO v1
The network of YOLO uses takes features from the entire image to predict bounding
boxes. Using the YOLO algorithm, you only look once at an image to predict what objects
are there and where the object is located in the image. This is done by using a single
convolutional network to predict bounding boxes. It consists of an input layer, a convo­
lutional layer, a pooling layer, a fully connected layer, and an output layer. The network
uses features from the entire image to predict every bounding box. It also predicts all
bounding boxes for all classes for an image at the same time.
Figure 2.2: YOLO model. The YOLO algorithm divides the image into an SXS grid. From
[12]
The algorithm of YOLO divides the input image into an SxS grid. Each grid cell predicts
if there are any bounding boxes and the confidence scores for each box. Each grid cell
makes a prediction for each class that has been trained on the network. Only one set
of class probabilities are predicted per cell, regardless of how many bounding boxes are
predicted.[12]
YOLO v1 has some weaknesses. If the dimensions of the objects in the image are different
from the dimensions of the object used in the training data, the model has difficulties with
identifying and localizing the objects. It also performs poorly if there is a group of small
objects, because each grid is only able to detect one object.
YOLO is based on the neural network framework, Darknet. The network consists of 24
convolutional layers followed and 2 fully connected layers. [13] [yolo] [12]

2.4.2
YOLO v2
The second version of YOLO is named YOLO9000. [14] YOLO 9000 has improved from
YOLO v1 by bettering the recall and localization while still maintaining a high precision.
YOLO v2 is based on the Darknet­19 architecture. Darknet­19 consists of 19 convolutional
layers, 5 maxpooling layers and a softmax layer for predicting the class label.
Batch normalization has been implemented, improving the mean average precision(mAP)
by 2%. Batch normalization is a method for training deep neural networks that standard­
izes the inputs to a network. This will stabilize the training and speed up the training
process.
Another change from YOLOv1 to YOLOv2 is increased input size, from 224x224 to 448x448.
This high resolution gives an increase in mAP by almost 4%. An additional feature added
to YOLOv2 is anchor boxes. These anchor boxes are used for predicting bounding boxes.
The anchor boxes are designed for a specific dataset by using k­means clustering. The
theory behind anchor boxes is explained further in section 3.4.1.
One of the issues with YOLOv1 is the detection of small objects in the image. The YOLOv2
architecture has improved this by dividing the image into 13x13 grid cells, which is smaller
than the grid cells used in YOLOv1. [14] This makes the network better at detecting
smaller objects.
2.4.3
YOLO v3
YOLO v3 is the third version of the YOLO series. In YOLO v3 the accuracy has seen clear
improvements compared to the previous version. YOLO v3 uses Darknet­53 network for
feature extractor which has 53 convolutional layers. [15]
YOLO v3 calculates the prediction score for each predicted bounding box using logis­
tic regression. The model uses logistic classifiers for every class instead of a softmax
function used in YOLO v2. [15] This formulation helps when using datasets with many
overlapping labels. Using softmax imposes that each box has exactly one class, which
is not often the case. For instance, a network trained to detect a woman and a human
will only predict one of the classes and not both when using softmax. Logistic regression
makes it possible to label multiple classes on one object.
YOLOv3 makes predictions similar to Feature Pyramid Network (FPN). FPN is a method
where the feature map gradually decreases in spatial dimension, and after this the fea­
ture map expands again and is combined together with previous feature maps with cor­
responding sizes. This procedure is repeated, and each combined feature map is fed
to a separate detection head. The FPN method makes it possible for YOLO V3 to learn
objects at different sizes, where for instance a 19x19 detection block has a broader con­
text and a lower resolution compared with the other detection blocks, so it specializes in
detecting large objects, whereas a larger block, for instance of size 76x76 specializes in
detecting small objects. [16] [15]
2.4.4
YOLO v4
The YOLO v4 update has a significant increase in both precision and speed. [17] YOLOv4
improves YOLOv3’s average precision (AP) and frames per second (FPS) by 10% and
12%, respectively. YOLO v4 uses CSPDarknet53 as a Backbone, and employs the same
anchor based detection as in YOLO v3. YOLO v4 is still not optimized for all scenarios, for
instance it still struggles to detect smaller objects with a high accuracy. Figure 2.3 shows
a comparison between YOLO v4, YOLO v3 and other state of the art object detectors.

Bag of Freebies
YOLO v4 implements a method called ”Bag of Freebies”. [17] This method does not add
any inference time in production, but only changes the training strategy, hence it is called
”Bag of Freebies”. Bag of Freebies performs data augmentation to increase the variation
of the input image, so that the object detection system is able to detect objects in different
environments. Some of the augmentation methods used are adjusting the brightness,
contrast, hue and saturation, and cropping, flipping and rotating the image. [12]
Bag of Specials
Another term used in YOLO v4 is ”Bag of Specials”. [17] This is a set of modules that has
a small impact on inference time, but in return gives an increase in performance. This
method implements a new activation function, Mish, that provides better accuracy and
lower loss.
Bag of Spceials uses DIoU NMS to remove multiple predicted bounding boxes for the
same class. The network may predict multiple bounding boxes for a single object, and it
would be useful to out only one box.
Bag of specials also includes DropBlock regularization. In DropBlock, some parts of the
image are removed from the first layer. DropBlock is a process where the network is
forced to learn features that it may not otherwise rely upon. For instance, it could be a cat
with the head hidden behind some object. In this way the network is trained to identify the
cat from its tail and torso alone.

Choosing object classes and image quantity
Four object classes are chosen to be trained by the model, a fork, a cup, a TV and a chair.

In order to achieve a high accuracy for the object detector a large dataset is required. The

testing the model.

4.4.1
Anchor boxes
YOLO uses anchor boxes to make predict bounding boxes. The bounding boxes are
predefined with a specific height and width. These boxes are designed to capture the ratio
of the object classes that will be trained, and are initial bounding box guesses. They are
calculated based on the object sizes in the training dataset. During detection, thousands
of anchor boxes are spread across the image and the network predicts the Intersection
over Union (IoU), background and offset for each anchor box. These calculations are
used refine each anchor box. Several anchor boxes are usually defined. YOLO v4 uses
9 anchor boxes, while YOLO v4­tiny uses 6 anchor boxes. Using anchor boxes makes
it possible to detect several objects at once, objects of different scales and overlapping
objects.
The configuration file for YOLO has already predefined anchor boxes included. Calcu­
lating custom anchor boxes based on the dataset we are using instead of the standard
anchors has the possibility of achieving better predictions. The Darknet repository has a
script that automatically computes the anchor boxes based on your data set and the size
of the image input. It was attempted to train the network with customized anchor boxes by
running the Darknet script. This did not give any noticeable improvement in mAP or loss
for either of the two YOLO models, and it was therefore decided to stick with the standard
anchors from the configuration script.

5
Discussion
5.1
Test set result
The test set results shows that the mean Average Precision (mAP) is significantly higher
for the YOLO v4 model, with a mAP of 63.54%, than for the tiny model, with a mAP of
35.33%. However the Frames Per Seconds (FPS) is much higher for the tiny model, with
a FPS of 260 as opposed
It is also seen from the test set that the fork and the chair has a lower AP than the cup
and the TV in both the models. A possible explanation to this is that the objects with a
more complex geometry are harder to train than simple shaped objects like a cup and a
TV. Especially chairs that are clustered together around a table can be difficult to predict
correctly. Even though the chair is a bigger object and expected to get a higher AP be­
cause of the size, it only receives a AP of 54.26% and 23.51% for the YOLO v4 and the
tiny model, respectively. The cup, which is a very small object, therefore has a higher AP
than the chair. The TV gets the highest AP of 86.07% and 59.32 % for the YOLO v4 and
the tiny model, respectively. This is expected as a TV has a simple geometry and is of a
large size.
5.2
Real­time implementation testing
When testing the detector in real­time the detector has short comings in several of the
cases, and struggles to detect the objects at many instances. The distance test performs
poorly for both models, where it stops detecting the cup and the fork at only a small step
away, and the detection for the chair is mistaken for being a TV. As mentioned, the chair
has a difficult shape and it also comes in many different models. It is therefore possible
that this type of chair was not included in the training set, and so the detector has not
been able to learn to predict this exact type of chair.
Another reason to why the detector is not working well at a distance could be because
the camera used may not be of good enough quality and it sometimes struggles to focus
on the object, making them appear a bit blurry. In addition, when holding the phone in the
hand to film, it might contribute to the frame moving a little bit because of the instability of
the hand while holding the phone, which might make the detection not as good as if the
phone was standing still on a tripod.
The real­time testing of the detector with distractions faces the same problems in both
models. The main issue is that the detector makes incorrect predictions by detecting the
object that are not a cup to be a cup, because of the similarity in shape. For fixing this
issue it could be possible to remove any similar object to the trained objects from the
environment, or make sure to train any object in the environment that might the mistaken
for one of the trained objects.
There is a clear difference between the models when testing the speed. It is seen that
YOLO v4 has a lower speed performance than the tiny model, and the v4 model is not
able to keep up with the objects when it moves fast around in the frame, which the tiny
model is able to do well.
The performance is for both models are reduced in dark lightning, where none of the two
models can detect the three objects in the frame at the same time. For implementing
Object detection using deep learning
37
==PAGE:48==
YOLO v4 into a real application, it might be a good idea to make sure that the lightning is
good, as YOLO requires that to make good detections.
The last real­time testing looks at investigating the larger objects. Both of the detectors
performs surprisingly poorly in this video. The YOLO v4 model is able to detect the TV but
the chairs around the table is not detected. This is strange as the examples pictures from
the test data showed that the detector was able to detect chairs around a table well. Yet
again this might be because the type of chairs in the video are not similar to the ones used
in the training set, and the detector has not learn to recognize it. From this it may seem
like a simple geometry might be more important for getting a better prediction, rather than
a larger object.
It is important to note that these results are not particularly surprising, as YOLO, does
not have the highest precision, but instead has a good trade­off between precision and
speed. Also, designing an object detector that performs perfectly, is very difficult, as there
are so many different scenarios and factor that affects it.
5.3
Test set results vs. real­time implementation test
When comparing the test set results with the real­time video testing, it is seen that the test
set results will somehow give a sense of how the detector will perform in real­time, were it
can be seen that the mAP for YOLO v4 is higher and will give a more accurate detection,
as confirmed from the real­time testing. YOLO v4­tiny also has a much higher FPS, and
this is confirmed in real­time when testing the speed, where it is seen that the tiny model
can track the object better when it moves around in the frame at a high speed.
5.4
Other possible improvements
Other possible improvements that could have been done is to use even more images for
training. The more images that are used to train the network, the better the detector will
get. However when using Google Colab for training, the training process would take end
up taking days. For training is also possible to decrease the learning rate, this can give a
better result, but also slows down the training process, which is already very long.
There is also a possibility that the predefined bounding boxes in the COCO dataset has
some annotations errors, but since 10 000 images is used for this project it would take
incredibly long to go thorough every single image and check that the bounding boxes are
correct.
If a higher precision is desired, it is also possible to increase the image size input for the
network, but this will like previously mentioned in section 4.4, reduce the speed of the
detector. If the model is being trained for an application that does not required a very high
speed, it could be a possibility to increase the image size.
5.5
Possible application areas
The results give an idea of possible application areas for YOLO v4 and YOLO v4­tiny.
If both accuracy and speed is desired, YOLO v4 is a good choice. To make sure the
detector has a high performance it might be a good idea to train objects of more simple
geometry and avoid using an environment with many untrained objects in the scene, as
this might confuse the detector and give false predictions on the untrained objects. Video
surveillance of people or counting animals in the wildlife could be possible real life im­
plementation. They are also not high risk tasks, like autonomous driving, where YOLO
v4 might still have too low precision to be implemented for such a task. The results also

6
Conclusion
In thesis, we have implemented one of the state of the art object detection algorithm,
YOLO, and evaluted the performance of the detector through a test set of images and
video. A detailed description is given on how the data set is prepared, how the model is
trained, the theory of CNNs and an overlook on state of the art object detection models.
The results shows that YOLO v4 has the best trade­off between speed and accuracy as
compared to YOLO v4­tiny. YOLO v4 is able to make detections in real­time, but has
several weak points, for instances like increased distance, low brightness, and distracting
environments. It also struggles to detect object with a more complicated geometry. These
issues are the same for YOLO v4­tiny. From the real­time tests it is seen that YOLO v4­tiny
is superior when it comes to speed, and so the tiny model is the best model to implement
if speed is the most important factor. The results also confirmes the prediction that YOLO
v4 has a higher accuracy than YOLO v4­tiny.
Possible ways to improve the detections is using a larger dataset, adjusting the configu­
rations parameters, like reducing learning rate, checking that the annotations are correct
for the training set and make sure to train all ojects in the enviroment that are similair to
the trained ones to avoid it being mistaken by the detector.
