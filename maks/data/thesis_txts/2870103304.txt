Abstract - In this paper, a reservoir computer of the
type Echo State Network(ESN) is being used to solve a
regression task in the form of denoising of audio speech
signals, and a classification task in the form of sound source
localization(SSL) of binaural speech signals. The results
of the ESN is compared to the results of a Convolutional
Neural Network(CNN) performing the same tasks. The
results show that the ESN is able to perform denoising on
par with the CNN, and that the CNN is more accurate in
performing SSL. Furthermore, it is shown that the ESN is
training faster than the CNN, especially for denoising, and
that the CNN has a faster inference time for all cases. In
this paper, it is demonstrated that it is possible for an ESN
to perform denoising and SSL on binaural noisy speech
signals simultaneously, and that the training and inference
of such is faster than performing the tasks individually.
I. INTRODUCTION
In the field of audio engineering, signal processing is a
necessary and important aspect of the development of usable
audio products. The field of machine learning has made a
considerable impact, as a tool for solving signal processing
problems. With regards to machine learning models for signal
processing tasks, the Convolutional Neural Network (CNN)
has been proven a viable and powerful algorithm. Especially
in the deep learning-based approach of image processing,
the CNN has proven itself a key [1]. Furthermore, the
CNN is considered an affirmative solution for noise removal
problems. In recent years, reservoir computing has shown
promise as an alternative to other classical machine learning
models. Reservoir computing require a small dataset and
therefore require minimal computational resources[2]. This
type of machine learning could have a great potential, in a
time where traditional machine learning models require more
data, and therefore more computational resources to optimize
and improve its results. In this paper, the machine learning
models will be considered as a black-box model, were the
aim is to apply machine learning for signal processing tasks
and analyze the results. It is not the aim to optimize or
analyze the deeper levels of the machine learning models.
A CNN is a feed forward neural network that consist
of several hidden layers. The type of CNN typically used
for denoising of images are Autoencoders[3][4]. The aim
of an Autoencoder is to understand and interpret visual
data, by reducing the dimensional space of the data(encoder),
Fig. 1.
Basic concept of an Autoencoder[6]. An Autoencoder is a type of
CNN. The Autoencoder consist of an encoder which performs dimensional
reduction to represent the input data as code to extract its features, and a
decoder which reconstructs the data from the code.
extracting features of the data, and reconstructing the modified
data(decoder)[5]. The basic concept of an Autoencoder is
shown on figure 1. In this paper, an Autoencoder is used
for denoising. For classification, no input data needs to be
reconstructed, therefore the CNN of choice have no decoding
part.
Reservoir computing is a type of recurrent neural network,
which hidden layer consist of a fixed, non-linear dynamic
system called a reservoir[7]. The input layer is connected to
the reservoir and the reservoir is connected to the output layer.
Contrary to a CNN, only the output weights of the network
are trained. This minimizes training and inference resources
and run time of the network, as well as the size of the dataset
needed. One of the more common reservoir computers is the
Echo State Network(ESN), which is the reservoir computer
of choice in this paper. The hidden layer of an ESN is fixed
and sparsely connected, and its connectivity and weights
of the neurons are assigned randomly. Only the weights of
the output are modified during training to produce a desired
output. A visual presentation of the general concept of an
ESN can be seen on figure 2.
One of the metrics used to evaluate the output of the
models is speech quality. Speech quality is fundamentally a
subjective attribute. One objective metric for speech quality is
the Standard Perceptual Evaluation of Speech Quality(PESQ).

The machine learning algorithm of the CNN and ESN is
deterministic in nature, where initial conditions are set at
random. To recreate the results of a machine learning model,
the initial conditions should be identical. This is done by
setting the same ”random seed” value for each experiment.
In this paper, the significance of choosing different random
seed values will be investigated.
In this paper, a CNN and an ESN will be applied for signal
processing task in the form of denoising and SSL, with the
objective of investigating the potential of reservoir computing
as an alternative solution for signal processing tasks.
II. METHODS
For this project, a readily made CNN architecture previously
used for denoising of images was used[4], and a CNN ar-
chitecture previously used for classification of CIFAR images
was used[10]. A readily made standard ESN for regression
and classification was used from[8]. First a regression task
were performed in the form of denoising of audio signals,
using the CNN and ESN designed for regression. Secondly
a classification task were performed in the form of sound
source localization (SSL), using the CNN and ESN designed
for classification. Different metrics was chosen to evaluate the
output of the models. Using the readily made ESN’s, an ESN
capable of performing denoising and SSL of noisy speech
signals simultaneously was constructed. The software used for
this project was Python. All code developed during this project
is publicly available in the git repository[11].
DENOISING OF SPEECH SIGNALS
A. Data construction
For the denoising task, the training and test dataset were
created by speech signals superimposed with noise. The
data used was from the database Librispeech. LibirSpeech
is an open source collection of approximately 1000 hours
of English speech audio, sampled at 16 kHz with a 16 bit
depth[12]. The length of each audio segment was fixed with a
duration of 2.048 seconds. A segment duration of more than
two seconds was chosen so the results and the effect of the
CNN and ESN could be heard. 2.048 seconds corresponds
to 215 number of samples with a sampling rate of 16 kHz.
215 is a power of two and will therefore allow for perfect
reconstruction of the signal, after it being Fourier transformed.
In[4], where a CNN was used to denoising of images, a total
number of 144 images were used for training the CNN. A
total number of 1021 audio segments were therefore deemed
sufficient for the regression task in this paper.
All datasets were initially normalized to be between zero
and one, by dividing the signal with the maximum value of
the signal. The labels required for the supervised learning
approach were the identical signals without added noise. To
investigate how the ML models performed on different types
of noise, a dataset with added white noise and a dataset with
added pink noise were created. Data was created at different
types of speech-to-noise ratio(SNR). The noise was added so
that each noisy sample had a fixed SNR. A dataset was made
for SNR’s in the interval [-10, 10] dB in steps of 2 dB. A
dataset of 215 samples used for testing, was constructed the
same way as the training dataset.
B. Data pre-processing
To utilize an autoencoder for sound processing, the audio
signals were converted into a spectrogram using Short-Time-
Fourier-Transform(STFT). As speech can be considered sta-
tionary in the time period of approximately 15-35 ms[13], an
FFT size of 512 with a 50% overlap was chosen, to get a STFT
frame of 32 ms. A Hamming window was chosen to be able
to perform perfect reconstruction of the modified signal. A
segment of 215 samples, yielded a spectrogram of dimension
(257, 129). Due to the nature of the image convolution, only
dimensions of equal length was accepted by the CNN. The last
time bin and the highest frequency bin were therefore removed
from the spectrograms of the CNN training and testing sets.
The magnitude of the spectrogram was converted to dB to
enhance the spectral features of speech. The spectrogram was
scaled between zero and one and reshaped to (256, 128, 1)
to account for grayscale. The phase remained unchanged and
saved for the reconstruction of the audio. A flowchart of the
pre-process can be seen on figure 3.
C. CNN model
The CNN used for denoising was based on[4], previously
used for denoising of images. The training dataset was splitted
in two, where 15% of the training data was used as validation
data. Based on pilot data, the epoch and batch size were
chosen as listed in table I. The weights with the smallest
validation error during training was used in the model. The
CNN architecture is summarized in table II.

L. Data construction and pre-process
The data construction and pre-process of the data was
performed as a mixture of the previous data construction and
pre-process methods. 5999 segments of LibriSpeech data of
2.048 seconds were superimposed with white noise, with a
fixed SNR of 10 dB. The signals were made binaural using
the same method as described previously. The direction of the
source was saved as labels for training. For this experiment,
only four directions in the azimuth were used. The four
directions are listen in table IV. The data was then converted
to spectrograms in the same manner as in the denoising and
SSL task, as displayed on figure 3. Unlike the SSL data pre-
process, the spectrograms of the left and right channel were not
subtracted from each other, as this would have made the signal
impossible to reconstruct, thereby defeating the purpose of
noise removal. Instead, the two spectrograms were appended
along the horizontal axis, resulting in a matrix of dimension
(257, 260). This process was done both on the data containing
noise, and on the data without noise to be used as labels
for training. A dataset of 455 samples used for testing, was
constructed the same way as the training dataset.
M. ESN for regression and classification
To construct the combined model, the ESN used for
classification was modified. The classification method uses
a for-loop to extract the last internal state of the reservoir
per single spectrogram. The readout is then trained only
on the last state of the reservoir, obtaining a single output
label per spectrogram. The regression part of the task was
designed using the same method, using the same for-loop as
the classification task. Two different ridge regression readouts
were created. One to fit the classification labels, and one to
fit the regression labels. Both readouts were fitted with the
internal states extracted from the same reservoir, resulting in
two outputs, a spectrogram and a direction of the sound source.
The hyperparameters of the ridges were set as described in
table III and VII for regression and classification respectively.
The hyperparameters of the reservoir were initially set as per
the regression task listed in table III.
To evaluate the results, the output from the combined ESN
were compared to the output of a pure regression ESN and a
pure classification ESN, trained and tested on the same data,
with the hyperparameters listed in table III and VII.

III. RESULTS
DENOISING OF SPEECH SIGNALS
A. SNR and PESQ output
The CNN and ESN were tested on different input SNR’s
with pink and white noise asynchronous. This was tested
on five different random seed values. The SNR and speech
quality of the output, for white and pink noise, with respect
to input SNR, for the different random seed values, can be
seen on figure 4 and 5.
The results shows that the output of the CNN, for both
white and pink noise, is fluctuating with respect to input SNR.
This is the case for both output SNR and speech quality. No
obvious pattern of these fluctuations can be observed. There
is a general tendency for the output SNR and speech quality
to decrease as the SNR is decreasing. This is seen more
clearly in the case of pink noise. One would expect to see a
correlation between output SNR and speech quality, as high
SNR should increase speech quality. This type of correlation
can be observed in the data, but is not always the case. It is
clear that the choice of random seed is of large significance
as the behavior of the plot vary considerably for each random
seed. For the CNN the general trend of the white and pink
noise are the same.
The output from the ESN is much more monotonic in
nature. No fluctuations is observed with respect to input SNR.
For output SNR of the ESN, the curve is relative flat from
zero to 10 dB SNR, and is decreasing from zero to -10 dB
SNR. The speech quality is gradually increasing as the input
SNR is increasing. This means that even though the SNR of
the output remains the same for the higher input SNR’s, the
signal is still modified in same way which is worsening the
speech quality. For the ESN the general trend of the white
and pink noise are the same. The white noise data does have
a higher SNR output than the pink noise in the higher input
SNR range. It is clear that the output of the ESN is relative
robust with respect to choice of random seed.
In every instance of the results, the output SNR is larger
than the input SNR. For the ESN, the output SNR is better
than the SNR of the reference for input SNR of -4 dB and
higher. In every instance the speech quality is better or equal
to the speech quality of the input.
A visual example of the denoising of a speech signal of the
CNN and the ESN can be seen on figure 18.
B. Training and inference time
To get an estimate of the complexity of the two models,
the training and testing time were measured. No considerable
difference is observed between white and pink noise run times.
Based on the average, it is seen that the ESN is training faster
than the CNN. For white noise data, the ESN is faster than
the CNN by a factor of 156. The CNN has a faster inference
time by a factor of 2.25.

IV. DISCUSSION
The results of the denoising task seen on figure 4 and 5,
shows that the output SNR of the ESN is higher than the SNR
of the reference audio, for input SNR of four dB and higher.
One could argue the that the goal of the ML algorithms are
not to get the highest SNR output possible, but to get as
close to the reference as possible. If so, the performance on
the pink noise data is better than the performance on the
white noise data. White noise have more energy in the higher
frequencies compared to pink noise. This means that a higher
amount of noise exist in the same frequency range as the
speech for the pink noise data. The noise that lies outside
this range might be easier to remove for the ESN, without
removing the speech, resulting in a higher SNR for white
noise data.
The method for calculating the SNR of the output is a
coarse estimate. To investigate the validity of the method,
the SNR of the noisy signal with known SNR was estimated
using this method. On figure 15 it can be seen that the
estimated SNR was between 1.5 and 2.5 dB above the
expected SNR. The error could occur when the data is being
superimposed with the noise or when the SNR is estimated.
If the error exist as a product of the SNR estimation, 1.5 to
2.5 dB could rightfully be subtracted from the output SNR
at figure 4 and 5, as well as the reference. As the error is
systematic, the results are viable for comparison which is the
objective of this paper.
The PESQ of the output of the denoised speech signals
is better than the PESQ score of the input, in all instances
for white and pink noise. It should however be noted that
the improvement of the speech quality is low, relative to the
maximum PESQ score of 4.5. The best speech quality of
the output is around 2 PESQ, which is not considered good.
Furthermore, speech quality is fundamentally a subjective
attribute, and PESQ is does not necessarily describe the
speech quality of the output.
The training and inference time for denoising vary little
with respect to input SNR’s. This can be seen on figure 12
and 13. The low variance between input SNR’s makes the
average time a viable metric for comparing the training and
inference times of the CNN and ESN.
The SSL accuracy on table VIII and the confusion matrices
on figure 7, clearly shows that both models performs better
SSL when there are fewer angles of directions to be classified.
This makes sense as the difference between the spectrograms
gets smaller. The accuracy of the ESN and the CNN shows
that the CNN is more accurate than the ESN. The ”X-like”
pattern seen on the confusion matrices indicates that the
ESN are having issues distinguishing between angles that
are mirrored over the coronal plane, E.g. 45 and 315◦. The
”X-like” characteristics are similar in the three cases for
the ESN. The curve is flat in the bottom left corner, due
to many false positives between 10 and 90 degrees, with
less false positives in the top right corner between 270 and
335 degrees of predicted values. A possible reason for this
could be because of asymmetry in the HRIR’s. This could
be investigated by flipping the left and right channel before
subtracting them from each other. If the asymmetry comes
from the HRIR, it would be expected that the asymmetry in
the ”X-like” pattern would be flipped. In order to optimize the
results of the ESN, this phenomenon should be investigated
further. The confusion characteristics are different between
the CNN and the ESN. This is most visible in the case of 50
angles of direction were the accuracy of both models are the
lowest. The confusion matrix of the CNN have the ”X-like”
feature to a lesser extend than the ESN. It also show vertical
lines along the diagonal of the matrix. This indicates that
the CNN also have some problem distinguishing between

complexity does not change significantly with respect to the
kind of noise or the number of different classes.

higher accuracy than speech only signals.
When the data was constructed for the speech/noise data,
the noise was superimposed with the speech, before the
data was convolved with the HRIR. It is fair to assume that
this will not always be the case in an actual system. In for
example a communication system, the noise could be present
in an equal amount on both channels. For training a model
to solve that specific case, the noise should be added to the
binaural training data after the convolution.
The results of the combined ESN shows that there is great
potential in reservoir computing. As a reservoir computer
model only trains on the output, that means that only one
reservoir is necessary for n number of outputs. Contrary
to, for example an autoencoder, where the model trains for
one task. As can be seen on figure 11, both the training
and inference times are lower than the sum of the training
and inference time of the regression and classification only
models. If more tasks is to be handled, the gain of a combined
ESN would probably be larger.
When comparing the CNN and ESN, it is worth noting
that the CNN architecture is developed for a specific
task, E.g. removing of noise or image classification, and
its hyperparameters optimized for the specific task. This
is not the case for the ESN. Except for the number of
neurons, the structure and choice of hyperparameters are the
standard parameters for regression and classification from
[8]. Hyperparameter optimization for each specific task could
possibly improve the quality of the output of the ESN. In this
paper, this is the case for both the regression, classification
and the combined ESN. Even if hyperparameter optimization
does not improve the output of the ESN significantly, it still
offer a model with a very simple structure, making it really
easy to work with.
Times were used as a course estimate of the computational
complexity of the two models. Calculating the actual time
complexity requires other methods, such as ”Big O” notation,
where every operation should be counted. How one would
calculate time complexity of a reservoir computer should be
investigated in order to accurately describe the complexity of
the model. It should also be mentioned that the parameters
of the CNN and neurons of the ESN, are not the same thing,
and can therefore not be compared.
It is true that the ESN was superior in training time com-

V. CONCLUSION
An ESN is capable of performing regression in the form
of denoising of speech signals, thereby improving SNR and
speech quality. It is capable of doing so on a par with a
conventional CNN. The output of the ESN was monotonic
with respect to SNR of input signals and was relative robust
with respect to random seed.
The ESN is capable of performing classification task in the
form of sound source localization on binaural speech and/or
noise signals. The accuracy of the ESN was lower than the
accuracy of the CNN when performing classification. This
was the case for all number of directions, when classifying
the direction of both speech and noise signals.
The ESN had a faster training time than the CNN in all
cases. The CNN had a faster inference time in all cases. The
training of the ESN in the regression task was much faster
than the CNN, but not a lot faster in the classification task. The
time gained training using an ESN is therefore task dependent.
The ESN has the ability to connect several readouts to the
same reservoir and to fit each of the readouts individually.
This was demonstrated when an ESN performed denoising
of
binaural
speech/noise
signals,
while
simultaneously
performing SSL of the source of the audio. This attribute is
not shared with a CNN, which is constructed for one specific
objective. The results of the combined model was not as
accurate as an ESN performing the same task individually.
Both the training and inference time was faster than the sum
of the individual regression and classification ESN. This
showed that it possible to solve several different tasks using
only one reservoir.
The ESN offer a simple machine learning framework, which
can offer minimal training time. The ESN and reservoir com-
puting has great potential to be an alternative to conventional
machine learning models for signal processing tasks.
VI. FUTURE WORK
As previously described, the chosen hyperparameters of the
ESN are taken from [8], expect for the number of neurons.
This means that practically no form of hyperparameter
optimization
has
been
done
in
this
paper.
To
further
investigate the the full potential of an ESN, hyperparameter
optimization should be performed.
It is reasonable to doubt the accuracy of the method for
calculating the SNR. The measured SNR worked as a metric
for comparison, but can not be trusted to be used as an
absolute metric. To accurately measure the SNR of the output
of the models, the SNR method used in this paper should be
optimized or another method should be used.
It would be interesting to explore the possibility of using
an ESN to perform DSP tasks in real time, for example
to remove noise from a signal with minimal delay, making
it applicable for real life communication systems. A good
place to start would be to minimize the length of the speech
segment, used for training an testing. The length of each
segment in is this paper was 2.048 seconds long. It would be
interesting to investigate how the model(s) would behave on
segment lengths of around 100 ms. A segment length which
allowed for perfect reconstruction should be chosen.
As described, the ESN does not perform with the same
amount of accuracy as the CNN when is comes to SSL. In
order to better understand why this is the case, the ”X-like”
shape of the ESN results visualized in the confusion matrices
should be investigated. This might lead to realizations which
could be used to optimize the ESN.
It was show that it was possible to have an ESN perform
regression and classification simultaneously. When comparing
the output of the combined model to the output of the
regression and classification only models, it was clear that
the the regression and classification models performed better
than the combined model. It should be investigated why this
phenomenon occur in order to optimize the results.
It should be further investigated how an ESN with more
than two readouts would perform. How would mare readouts
affect the results, and how many readouts would it be possible
to connect to the single reservoir. Furthermore, how would
this affect the training and inference times.
The python code of the project was uploaded to a git
repository [11]. The code is however not suitable for the public
use. The repository should be streamlined and prepared for
public use.
