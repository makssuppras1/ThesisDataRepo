Summary (English)
The goal of the thesis is to develop and implement a model predictive controller to
regulate the sewage ο§²ow in a local area of a drainage network. Speciο§¹cally an area
in Northern Aarhus was selected for a case study. An underground storage basin de-
noted the Trojborg basin, built to detain water temporarily during heavy rain events
is the core in the control objectives along with the actuators surrounding it.
The end goal for this project is to devise a control strategy that with minimal load
onto the surroundings and the natural environment in the Bay of Aarhus next to the
harbour area, can regulate sewage streams in the network during draughts and rain,
so the system always run with maximum available capacity.
DHIs MIKE Urban software was deployed as a proxy for the real system during ex-
periments.
The solution was a setup with to prioritised layers of control. First the problem is
solved with a horizon covering the full event, ο§¹nding the minimal sequence of dis-
charge to the surrounding network, while avoiding sewer overο§²ow to the bay. The
lower controller has a detailed dynamical description of the network area and pre-
dictively regulates the system, realising the planned discharge. This controller has
information ot edit control strategy online, given a situation where there is a risk for
overο§²ow.
The implemented model is based on the linear reservoir idea, which has been doc-
umented in other studies. In this work several extensions are dealt with to improve
predictive accuracy in varying conditions.
The motivation is in part that techniques for identiο§¹cation and design should be scal-
able. This has inο§²uenced the step-response experiments and lead to the development
of a design model for simulation to provide a closed test environment.

Chapter 1
Introduction
Overο§²ow risk in drainage networks and environmental impact is a serious challenge,
that cities around the world are faced with. Controlling the water loads is not a sim-
ple task as city-wide networks are complex and often, sensors and measurements are
scarce relative to the complexity.
In the city of Aarhus, Denmark, as a means to mitigate or avoid overο§²ow, several un-
derground storage basins are installed across the city. By controllable oriο§¹ces, pumps
and static weirs, the water load can be diverted to and between storage basins, form-
ing an active network to detain rain and sewage, in cases where it might otherwise
overο§²ow. A monitoring and control system prioritises the utilisation of each storage
basin, and diverts water to areas with lesser overο§²ow cost or risk parameters [MΟΗl14].
Models for prediction that utilises runoο§Ä- and ο§²ow forecasts, to control overο§²ow in
drainage networks to bay areas or ο§²ooding to street-level is a well covered topic,
[Dur14] [MHP+14] [MP01] [OM07] [VG14]. A typical approach among these appli-
cations, is to describe the network capacity by grouping catchments and areas, from
which the exiting ο§²ow is a ο§¹rst order model of the detained volume. This was for
instance the case in the city of Barcelona, Spain, where a model predictive controller
was implemented to control the drainage network by this formulation [OM07]. This
type of model is reported as a linear reservoir or virtual tank, but has been extended
to cover a number of specialisations, such as time-delay and non-linear switching
functions [Dur14] [MP01].
==PAGE:16==
4
Introduction
The purpose of this study is to identify a model of this type and implement it in a
model predictive controller for a local area of the drainage network in Aarhus. The
emphasis will be application of the linear reservoir description inside a small area
where each compartment will be as little as a few hundred cubic meters. The model
quality is the ability to predict ο§²ow at speciο§¹c measurement points, with temporal
variations in water load and parameters. The inquiry is if a ο§¹rst order model is a
suitable description for the system, possibly with a time delay.
Commonly the referenced works introduce a number of reservoirs or storage capaci-
ties to describe the network as compartments of volume. The linear reservoir model
is based on an assumption that a drainage network can be divided in a number of sec-
tions, each one represented as a single reservoir with the volume of the pipe network.
The spatial extension of a section could be that it covers a single connected catchment,
that there is a bottleneck in the network or some constraint that must be represented,
or that pipe connections with other sections demand a boundary. The linear reservoir
can have multiple input sources, but only one output to the surrounding network,
which is constrained by the linearised model [OM07] [Dur14].
Alternatively the ο§²ow leaving a reservoir can be considered as controllable, as a de-
gree of freedom, rather than constrained. Then the ο§²ow between the compartments
are the consequence of measurements or another predictions, or they represent de-
sired values that should be realised to meet some objective [HF17] [VG14].
The application of this approach has been in city-wide networks, including Aarhus
[MΟΗl14], where the DORA framework [VG14] was implemented. Here and in the
Barcelona study [OM07], the motivation is to regulate the water ο§²ow between com-
partments and storage basins such that overο§²ows in the drainage network can be
controlled or avoided.
1.0.1
About procedure
The focus for this report is to devise a model and a closed loop procedure that accu-
rately predicts the dynamics in a local area, making it suitable for a closed loop imple-
mentation. In this work the controller will be assessed in terms of two key properties;
disturbance rejection and reference tracking [Ole12]. The objectives for this controller
is to avoid overο§²ow in the associated storage basin, while protection the downstream
sections from heavy discharges. Provided that this spatially local model regulate sys-
tem well, we will extend the implementation to a layered setup [MMTS16].
A separate controller, a planning mpc, using a dynamically simple model computes an
optimal desired ο§²ow trajectory for the lower level controller, providing a time-varying
acceptable discharge to the downstream section. The challenge for the controller de-
sign is that measurements in the simulation system will be scarce, and the control
strategy should be formulated with this in mind. That the model is imperfect or some
states are unmeasurable is however, a well known circumstance in regulation theory.

1.1
Objectives
The main objective in this thesis is to develop and test a controller, for a section of
drainage network, that predicts the outcome, prioritising multiple objectives with a
quadratic cost function. The controller should be formulated and only require infor-
mation, in a way that makes it readily scalable to a broader (if not city-wide) context.
For all purposes of this work, MIKE Urban will be deployed in place of a real sys-
tem. This opportunity is taken to investigate the design model in areas that might not
be possible in a real setting. The goal however is to apply a series of techniques for
identiο§¹cation, that scales well to a real system.
1. Design a suitable state space model for predictions. Formulate a non-linear
model, using derivatives to obtain better predictions.
2. Develop a test environment, that allow for constraint and tuning tests away
from the βÄôrealβÄô system.
3. Identify model parameters, as well as signiο§¹cant and secondary dynamics, by
means of open loop experiments in the βÄôrealβÄô system. Validate design model
assumptions.
4. Test controller properties in controlled settings with MIKE Urban.
5. Run simulations with historical rain events, investigating controller perfor-
mance, with only enhancement over a real system, being perfect forecasts, and
an otherwise inert network.

6.6
Chapter summary
In this chapter multiple outcomes of the rain event was presented. Among the most
important results, it was shown that the planning mpc allocating extra volume in the
outgoing discharge is immensely important for the capability in a small catchment like
this to avoid an overο§²ow. We found that anticipating an overο§²ow sooner would lead
to a better mitigation of events. The simulations were biased in this regard, since the
initial state information would be drawn from the historical data, and therefore vary.
The results do show that the operating controller can follow a reference trajectory
minutely. This supports the assumption that a viable control strategy is one where,
the planning mpc oversees several operational mpcs and prioritise discharges. This
way, during draughts, one mpc at a time can be ordered to empty its tank by raisin
the reference. This will reduce the required horizon, which here was 10 hours to
anticipate and drain the tank to avoid overο§²ow.

Chapter 7
Conclusion
In conclusion two model predictive controllers have been designed and implemented,
meeting their requirements. The controllers are formulated in a hierarchy where the
upper controller minimises overο§²ow, computing the optimal reference for the lower
controller, that regulates discharge to the downstream sections, while locally trim-
ming the control strategy if overο§²ow is imminent.
First order models with a volume representation were used in the design model. It was
necessary to extend the model formulation with extra dynamics, mainly time delay,
model for pump lag and a pulse model under certain conditions. With these additions
in the operational mpc predictions were adequate to provide stable control.
The parameters for these linear reservoirs were estimated by experiment to good re-
sult. It has been shown that using on-line parameter estimates from a non-linear ο§²ow
model performs satisfactory, eο§Äectively substituting step-response experiments with
a simple, physically based design model. Predictions could be further improved ο§¹tting
parameters loocally, when measurements were available. The design model used in
the mpc was also formulated as a simulation model, as a tool in initial screening for
constraints and weights.
The hierarchical control setup was able to deliver the minimal load to the receiving
downstream, while averting a waste water overο§²ow in the storage basin.
It was not tested how the planning mpc would perform in a situation where it should
prioritise water load between multiple network sections with individual operating
mpcs.

qualitative test - If control is possible - is important. As the scope is broadened and
several sections must be prioritised, the performance under forecasts with uncertainty
gain more relevance.
