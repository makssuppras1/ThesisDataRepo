Abstract
Industrial robots are part of almost all modern factories. Their use is extensive in
various areas of industry. Industrial robots nowadays manipulate objects of a huge variety,
in different environments and in general knowledge about both of them is available. The
aim of the current thesis is to investigate the ability of an industrial robotic system to
operate within an unknown environment manipulating unknown objects. The goal of the
system is to place each of the objects belonging to a speciﬁc set, into the corresponding
slot of a placing box. The developed system relies on 3D sensing and feature extraction
by exploiting computer vision. Most industrial robots are taking advantage of the a priori
knowledge they hold about the objects they are manipulating and their environment. On
the current thesis no prior knowledge is available, neither for the objects nor for the placing
box, thus the system relies solely on information extracted by itself during the procedure.

CHAPTER 1
Introduction
1.1
Motivation
The level of adaptability in manufacturing is low given the progress of the research
in robotics. The level of diversity of the objects that exist in manufacturing is extremely
high, hence a system which operates using prior knowledge is not applicable in different
environments with unknown objects. Acquiring data for both environment and objects in
such a scenario is a time consuming task.
The idea behind the current thesis was inspired by the STAMINA (Sustainable and
reliable robotics for part handling in manufacturing automation) project. It is a European
Union funded project in which universities such as Aalborg University (AAU, Denmark),
University of Edinburgh (UEDIN, United Kingdon), and University of Bonn (UBO,
Germany) participate. Part handling across the various assembly stages in the automotive
industry is the only task with automation levels below 30%1. This is outcome of the
high variety of the production lines and of the diversity of the parts used in industry. An
automated part handling task will affect the automotive industry and will make room for
development of mobile robots capable of operating in different environments.
Key partner in STAMINA project is PSA Peugeot Citroën. The whole project is based
on a concept called Kitting. For each car during the production line a kitting box is placed
on the chassis. This box contains only the parts that need to be assembled on that speciﬁc
car, thus each box contains different parts. At that point the robotic system developed
under STAMINA contributes. The robot will get information about the parts that are
required for each one kitting box, it will identify the requested objects and it will place
them into speciﬁc slots inside the kitting box. These holes inside the kitting box are square
or rectangular shaped and their dimensions are around 5-8 cm enlarged comparing to the
corresponding objects (ﬁg. 1.1).

1.2
Problem statement
Industrial robots are working in pre-speciﬁed structured environments and are manipu-
lating a priori known objects. The problem that this project will examine is the ability
of an industrial robotic system to operate within an uncertain environment (e.g. un-
known location of placing box) and manipulate unknown objects. The objects ﬁt in a
speciﬁc place inside the placing box which is unknown as well. Correct correspondence
between the objects and the holes is required before starting manipulating the former.
Afterwards, the robot picks the object with no knowledge about grasping poses and moves
to the matching hole. Before releasing it, extracting information about the exact position
and orientation of the placing box slot is a necessity.
1.3
Limitations
On the current thesis the development and the testing of the algorithm was performed
on a single placing box (ﬁg. 1.2(a)) and a speciﬁc set of seven objects (ﬁg. 1.2(b)). The
placing box contains holes that are shaped exactly as the objects and each one’s dimensions
are enlarged by 4-5 mm in total comparing to the corresponding object (ﬁg. 1.3). The
objects designed by keeping in a mind a scale. Two are simple shapes, two are slightly
more complex but similar to each other, and three are more detailed and complicated. Of
course there is a huge variety of objects that are used in industry, although this thesis
explores the capabilities of placing this set of objects.

CHAPTER 2
Related Work
Previous work regarding object detection has been conducted using different methods.
Divvala et al. (2009) conducted an empirical study on object detection. Using a standard
dataset and top-performing local appearance, they evaluate numerous sources of context.
Context is also used by Xiong and Huber (2010) in order to create semantic 3D models.
These 3D models contain information about the geometry and the identity of a part of a
facility (ﬂoors, walls). Using data from a 3D laser scanner (point clouds) they classify
planes discovered in the environment. Also, Koppula et al. (2011) label semantically
objects in indoor scenes using 3D point clouds. Their graphical model, contains infor-
mation, such as, visual appearance, shape, geometric relationships. Furthermore, Fisher
and Hanrahan (2010) developed an algorithm that can search a scene and distinguish the
asked object among the others using geometric cues and spatial relationships. Robust
real-time object detection performed by Viola and Jones (2001). They introduced a new
image representation that allows rapid feature detection. Also, their learning algorithm is
capable on detecting a number of crucial features on the images. Their developed algorithm
utilises classiﬁers that allow quick and robust background extraction, in order to focus
only on image’s part that contains useful for detection features. Another comparative study
about object detection performed by Sapna Varshney et al. (2009). They tested different
techniques for image segmentation, such as, edge-based, KMeans clustering, thresholding
and region-based. Moreover, using still images, 3D geometric properties can be derived
that will allow easier object detection. Saxena et al. (2008) estimate depth from a single
still image. They collect monocular images of outdoor environment alongside with their
corresponding depth maps (ground truth). Applying supervised machine learning make an
estimation of the depth using still images.
Object matching in two-dimensional images has been an important issue in computer
vision. Persoon and Fu (1977) discriminate objects using Fourier descriptor. Based on
a stochastic labelling procedure, Bhanu and Faugeras (1984) developed an algorithm

that matches two-dimensional objects. Their technique was also tested on objects that
partially occlude to each other, creating another object. Even in that case, their algorithm
matched, these objects in a way that they can not be matched to different actual objects.
Jain et al. (1996) performed object matching using Bayesian scheme that is based on the

Work on object matching similar to the task the current thesis attempts to deal with, is the
one presented by Flusser (1995). This article present the work on matching two sets of
objects, which may differ in translation, rotation and scale. Aiming in accurate matching,
local information (set of invariant features) and object-to-object distances on the plane are
used. Also, matching likelihood coefﬁcients are introduced to indicate the correspondence
between objects. Work on shape matching and later object recognition was conducted by
Berg et al. (2005). Using geometric blur point descriptors and geometric distortion between
the corresponding feature points, they calculate the aligning transformation that results
in solid shape matching. Gavrila and Philomin (1999) utilised object matching for smart
vehicles. In order to capture objects, template hierarchy was the method of choice. Despite
the ofﬂine hierarchies generation, the matching is conducted online using a coarse-to-ﬁne
approach. Object matching using locally afﬁne-invariant constrain conducted by Li et al.
(2010). The idea behind their work is that each point can be represented by an afﬁne
combination of its neighbour points. Jiang and Yu (2009) proposed a linear formulation
that ﬁnds feature points correspondences and the geometric transformations.
Regarding grasping objects, a lot of previous important work has been conducted.
Bicchi and Kumar (2000) survey the work that has been done on robotic grasping. They
present the theoretical background and also assess the more modern work on this ﬁeld. A
combination of human input and automatic grasping technique is introduced by Ciocarlie
and Allen (2008). They created a system that is equipped with an automated grasp planner
capable of shaping the artiﬁcial hand accordingly to the shape of the object that is aiming
to grasp, letting the user to complete successfully the task. An other approach is introduced
by Miller et al. (2003). They simpliﬁed grasping task by simplifying the objects and
modelling them into sets of primitive shapes, such as cylinders, boxes, spheres. As a result,
simpler objects and sets of rules allow the calculation of grasping poses. Grasping objects
in conjunction with supervised machine learning introduced by Saxena et al. (2006). Their
learning algorithm does not require 3D model of the object. The training is performed on
synthetic images set.
Object grasping in cluttered environment presented by Berenson et al. (2007). Their
algorithm performs fast grasp planning in complex environments where there are obstacles
alongside with the object that is about to be grasped. Also work on grasping novel objects
on cluttered planes with no use of machine learning concerning grasping points was
performed by Rao et al. (2010). They simplify this task by using depth information,

although supervised localization method was utilized in order to select graspable segments.
Hsiao et al. (2009) introduced a method for grasping objects using optical proximity
sensors, located inside the ﬁngertips of the gripper. This system could be supplementary
to existing grasping algorithms. A combination of different object representations is
conducted by Brook et al. (2011). Instead of using one representation of the object
in order to plan the grasping, all the available representations are combined and the
extracted information is used to plan grasping accordingly. Moreover, efﬁcient grasping
was presented by Jiang et al. (2011). Their technique derives information from RGB-D

suitable for grasping is excluded and the remaining is tested with advanced features until
the best one is detected. Inspired by human actions is the work conducted by Dogar and
Srinivasa (2011). They used a library of actions inspired of actions that humans perform
while grasping objects in cluttered environment (i.e. rearranging clutters).
Regarding object placing, Schuster et al. (2010) developed an algorithm to detect
clutter-free planes were objects can safely be placed. Orientation is also essential, thus Fu
et al. (2008) based on geometrical features of the objects they reduced the dimensionality
of the orientation to a set of possible orientations that are suitable for an object. Their
algorithm focused on deriving the upright orientation for proper placing on ﬂat areas. Also,
Saxena et al. (2009), focused on deriving object orientation. Their algorithm could extract
object’s orientation from a single image. Glover et al. (2012) calculate the pose of an
object using sets of local features on partial point clouds. Interaction between human
and robot that places objects was introduced by Edsinger and Kemp (2006). The human
passes an object to the robot that afterwards places it on a shelf. Placing task utilizes force
control that leads to a gentle execution release on the shelf. A different approach using a
rule based planning was presented by Sugie et al. (1995). This technique was tested on
object placing tasks on ﬂat surfaces. Toussaint et al. (2010) integrated planning, control,
reasoning for placing objects located on ﬂat surfaces into stacks. Similar work performed
also by Coffman Jr et al. (1996) on the bin-packing problem.

3.1.1
Robot Operating System
The Robot Operating System (ROS) is a framework suitable for creating robot software.
It consists of powerful libraries and tools that let people around the world create complex
robotic applications for a wide variety of platforms. The main goals of ROS can be
described in a few words, peer-to-peer, tool based, multi lingual, thin, free and open source
(Quigley et al. 2009). The ﬁrst release was in 2007 and since then its development is fast.
The main principles behind ROS implementation are nodes, messages, topics and
services. A node performs computations and communicate peer-to-peer with other nodes
by making use of messages, topics and services. They can receive, send and modify
complex messages. One of ROS’s aims is to be modular, hence nodes are modules
that can be transferred easily and be used in conjunction with other nodes. Also, the
concept of nodes is convenient regarding system visualization at runtime. The graph
architecture that is utilised to visualize nodes lets the user understand easily the way nodes
are communicating to each other.
As it was mentioned above, nodes use messages and topics to communicate to each
other. Messaged are well deﬁned data structures that contain essential information. Nodes
publish messages into topics. These messages can be as simple as a string or a number, or
they can be more complex by containing a huge amount of information. Then, other nodes
can subscribe to topics in order to gain access to that information. Nodes can publish
and/or subscribe to multiple topics. An example on how nodes communicate to each other
is visible in the image below (ﬁg. 3.1). This is a small and simple example of viewing the
scene that the camera points at. There is a node that operates the camera (/wrist_camera)
and publishes a message (RGB image) in a topic (/wrist_camera/rgb/image_raw). Af-

This topic-based communication is simple and ﬂexible for exchanging information but
it is not convenient when synchronised data transaction is needed. For that exact reason,
services are necessary. Services are deﬁned by a name and the type of messages that they
transmit (response) and receive (request). A service is running in the background and
is waiting for a node to call it. When this is the case, the service receives a request and
performs the appropriate computations. In the end, it responds by transmitting data and it
is ready again for an other call. It is obvious that this is different than publishing and/or
subscribing to a topic due to the fact that the service performs computations only when it
is asked to and not constantly.
3.1.2
Point Cloud Library
The Point Cloud Library (PCL) is a C++ library suitable for 2D/3D image and cloud
processing. It contains a variety of algorithms about ﬁltering, feature extraction and
estimation, segmentation, surface reconstruction, registration and many more (Rusu and
Cousins 2011). Modularity is one of its advantages due to the fact that it consists of
many smaller libraries that can be compiled separately, hence it is suitable for all different
machines regardless of the computational power they have. It is a library written by keeping
in mind performance and efﬁciency, and utilizes Streaming SIMD Extensions (SSE)
optimization in data structures. Also, a strong advantage of PCL is its full compatibility
with ROS.
The main principle behind PCL library is the 3D point cloud processing, hence the
name. The point cloud is the depiction in three dimensions of a scene that a sensor points
at. An RGB image is a two dimensional one, but the point cloud adds the third dimension
that an RGB image lacks, depth. In case an RGB-D sensor (for more information about
RGB-D sensors see section 3.2.3) is used, PCL library combines the RGB and the depth
image that sensor generates and as a result, in every pixel depth information is added.
PCL’s design philosophy is based on Perception Processing Graphs (PPG). The idea
behind this philosophy is that applications that process point clouds consist of blocks that
are parametrised accordingly to achieve their goals. In simpler words, there are blocks that

Moveit!
Moveit! is software suitable for manipulation, motion planning, trajectory generation

3.2.1
Robotic manipulators
A robotic manipulator is a programmable mechanical arm that mimics the functionality
of a humanoid arm. It consists of links that are connected by joints that allow rotational
(revolute joints) or linear motion (prismatic joints). Kinematics describe the relation
between the different parts of the robot. At the end of a robotic manipulator there is the
end effector, which is a tool suitable for a speciﬁc task (spinning, welding, gripping) or a
hand, similar to a human hand. In any moment, the pose (position and orientation) of the
end effector or the gripper can be obtained through the kinematic chain.
Robotic manipulators nowadays are used more and more into different types of tasks
and in many ﬁelds. In industry they are already well implemented in a huge variety of
areas. Their need is increased lately and they are already a fundamental part of modern
factories. Their precision, speed and reliability made them a must in industry. On the other
hand, robotic manipulators are so precise nowadays that people started using them even in
surgeries or in other tasks that are vital and precision considered to be extremely important.
There are modern and sophisticated arms that are much more precise than human hand.
There are many different types of robotic manipulators each one with its own advan-
tages and drawbacks. The nature of a task is crucial in order to use the appropriate type of
robotic arm. The main types are the following (Craig 2005):
• Cartecian: Industrial manipulator mostly used for picking and placing tasks, as-
sembly operations and welding. It performs linear movements in its axis that have
ﬁxed angles to each other (ﬁg. 3.3(a)).
• Articulated: It is also called anthropomorphic manipulator. In most cases it consists
of two “shoulders” and an “elbow”.
• SCARA: This type of manipulator typically has three parallel revolute joints (that
allow rotational motion) and a prismatic one that allows translation of the end
effector.
• Cylindrical: It consists mainly of rotational joints. Its working area forms a
cylindrical coordinate frame.
• Spherical: That type of robotic manipulators has two revolute joints and a prismatic
one. Welding and handling tools are two of the most common tasks that they are
used for.

single end effector (ﬁg. 3.3(b)). Their movement is very precise but in a rather

weak edge or no edge. If pixel’s gradient value is greater than the high threshold then
it is consider as strong edge. If it is between the low and the high threshold value
then it is categorized as weak edge. Otherwise (smaller than low threshold), it is
suppressed. The deﬁnition of the these two vales (high and low) is totally dependent
on the nature of the task and completely empirical. Conducting tests will reveal a
good and robust pair for the case.
• Final strong edges determination. After applying double threshold on the detected
edges most of the noise has been eliminated, but still there are edges considered as
weak that may derive from noisy data or from real edges. In case a weak edge pixel
belongs to a strong edge pixel, it will be connected to it, hence the eight neighbour
pixels around it are examined. As long as, one strong edge pixel is located around a
weak one, the latter is now consider also as strong.

Point Cloud Filtering
Before extracting edges using point cloud, ﬁltering is needed. The interesting part of

decreases the data that need to be processed.

where the ﬁrst obstacle is detected plus a threshold (15 cm was used on this setup).

5.2.3
Shape matching using OpenCV
Moments in general are widely used to describe an image, a shape. An image moment
is the weighted average of pixel intensities and they are used to describe the detected
objects. Example of what properties can be found using image moments are the centroid,
orientation, area. There are different types of moments, such as, the raw moments and
the central moments, which are translation invariant and can provide information about
centroid and orientation. Moreover, Hu moments are the most commonly used moments
that are invariant in translation, rotation and scale. Also, the last Hu moment (seventh) is
skew invariant and can help to distinguish mirror shapes.
Direct shape matching uses Hu invariants to compare shapes. This technique is utilised
on the current thesis for object-hole matching as it is implemented in OpenCV. The results
are quite well especially comparing with the previous descriptor matching methods. As it
is already stated, Hu moments are scale invariant. This is not always helpful when trying
to distinguish between similar objects that are different in size. Therefore, an additional
ﬁlter regarding the size of the detected contour area was added to the matching process.

Grasping & Placing
5.3.1
Grasping
After matching the objects with the appropriate box-holes, they need to be grasped by
the arm and placed accordingly. When the RGB-D sensor is looking at an object vertically
the precision is greater. The position of the objects are known, therefore the arm moves
above the selected one (using the centre of mass of the detected contour) that will be
picked at a certain moment, to get a more precise measurement of the pose (position and
orientation) of it.
Grasping an object is a quite tricky process. It is not considered as an easy process
because of the difﬁculty of recognising a good pose that will allow a solid grasping.
Without a good grasping pose the object will be moved while the gripper tries to pick it up,
hence it will not be at the position and will not have the orientation that the robot thinks
the object has. This will of course affect the whole process due to the fact that the placing
afterwards will not be successful.
There is no prior knowledge about the objects, so there are no predeﬁned grasping
poses. When the RGB-D sensor get a more precise measurement of the object it also
calculates its two-dimensional orientation on the plane. In order to acquire that information
principal component analysis (PCA) is performed on the detected contour of the object
located on the centre of the image (the object that the arm aims to grasp).
Principal component analysis (PCA) is a statistical tool that converts a set of data
(observations) into a set of linearly uncorrelated variables (Abdi and Williams 2010).
Principal components are orthogonal because they are the eigenvector of a symmetric
==PAGE:55==
5.3. GRASPING & PLACING
39
covariance matrix. The ﬁrst principal component has the largest variance, and each of the
following component has the highest variance as long as it is orthogonal to the preceding
component. For example, if PCA is performed on a set of two-dimensional points, the
output will consist of two vectors (eigenvectors) which are the principle components. The
size of each eigenvector is related to an eigenvalue which denotes how much the data vary
along a principle component.
After acquiring information about the centre of mass and the orientation of the object,
the arm grasps it and releases it again. In most of the cases after grasping, the object moves
and rotates. The data about the centre of mass and the orientation are rarely perfect, but
even if they are, the ﬁngers of the gripper can slide into an object’s cavity or on a non
vertical side and as a result to change its pose.
A second measurement above the object is performed (using the exact same pose
that was utilized to acquire the ﬁrst measurement) and extracts only the centre of mass,
before the gripper grasps it again by employing the pose of the ﬁrst grasping attempt.
This strategy decreases the possibility of moving the object once again and measures the
misplacement that occurred after the ﬁrst grasping attempt. The change on the position
(centre of mass misplacement) and not on the rotation is important at this point.
In the following ﬁgures (ﬁg. 5.16) the output of PCA algorithm applied on the detected
contours of the object before and after the ﬁrst grasping attempt is displayed. It is visible
that one axis (the smaller one) has opposite direction even though it is the same object
slightly rotated. At that point this has no effect due to the fact that PCA is used just once
before grasping the object for the ﬁrst place, in order to get the orientation that the gripper
will use so to grasp the object. The change on the orientation before and after grasping for
the ﬁrst time, does not matter at that point.

well. The most common problem that was faced is the difference on the coordinate frame

Iterative Closest Point (ICP) is an algorithm that aims to the minimization of the differ-
ence between two point clouds. It is commonly used in reconstruction processes of two or
three-dimensional surfaces using data derived from different scans. This reconstruction
gives robots the opportunity for optimized path planning. ICP uses a reference point cloud,
on which no transformation is applied and a second one that is the source. On the source
one, iteratively different transformations are applied (for both translation and rotation)
until the difference between the two point clouds is minimized (Besl and McKay 1992).
When this is accomplished, the ﬁnal transformation that aligns the two point clouds is
the output. On the other hand, ICP is prone to errors due to local minimums. In case
ICP reaches a local minimum it “locks” the current transformation and outputs it. Thus,
an optional step before exploiting ICP is to apply a transformation to the source point
cloud even if it is not precise. This optional step can decrease the possibility for the ICP to
“lock” in a local minimum. In order to obtain a better view on the Iterative Closest Point
algorithm, its steps are summarized in the following:
• Correlation for each point of the source point cloud to the closest one of the reference
point cloud.
• Calculation of the transformation (translation and rotation) that best aligns each
point of the source point cloud to the corresponding one on the reference point cloud
as it was deﬁned on the ﬁrst step and simultaneously minimization of the error. This
calculation is performed by making use of the mean squared error (MSE) function,
which estimates the average of the square of the difference between each one pixel
(p1i) and its corresponding one (p2i).

CHAPTER 7
Conclusions
Going back to the problem statement, the aim of the current thesis is to test the
ability of an industrial robotic system to manipulate unknown objects within an unknown
environment. The developed system indeed fulﬁlled its purpose, by placing objects using a
robotic arm and computer vision, with an acceptable success rate. Even though there was
no prior knowledge about the objects, it detected and matched them to the corresponding
box-hole with a good success rate. Several methods were tested in all parts of the developed
system in order to conclude to the one that performs better on current thesis’s needs. Of
course there is room for improvement that can lead to a more robust, stable and successful
system.
Regarding edge detection, Canny edge detection algorithm was the selected one due to
its robustness throughout the development of the robotic system. Applying it on the depth
image instead of the RGB one, was a decision made by keeping in mind that the system is
aiming industry, thus robustness here also, is excessively important.
Shape matching was another stepping stone of the developed system. Shape match-
ing using OpenCV performed well, resulting in good success rate during testing. The
developed algorithms that aimed to provide higher accuracy regarding matching did not
accomplished their goals and were outperformed by OpenCV. Furthermore, some of the
designed objects proved to be small and much detailed. As a result the combination of
high details, small dimensions and the low ability of the sensor to depict these details, in
some cases led to mismatches, despite the dynamic of the shape matching algorithm.
Apart from difﬁculties related to the matching task, the low ability of the used RGB-
D sensor to distinguish all the details around objects’ contour led occasionally to not
precise enough estimation of the centre of mass, hence increased misplacements. Also
misplacements occurred due to erroneous placing orientation that derive from the drawback
of the ICP algorithm to “lock” in local minimums. Nonetheless, ICP performed better than
the tested PCA algorithm.

On the other hand, PCA performed well concerning the grasping task. It provided
information that allowed picking objects even though no prior knowledge about them was
available. However, the designed objects proved to be relative small for the bulky gripper
that was used, hence grasping in some occasions was problematic.
Finally, it is notable the fact that the planner is responsible for several failures and
cancellations during the testing process. It does not affect the results because it did not
considered as failure of the developed algorithm but it is an important issue that lowers the
robustness of the system.

CHAPTER 8
Discussion and further
development
In order to improve the developed algorithm there are changes that should be conducted.
Dealing with the planner issues would require checking of the estimated trajectory to be
performed before even executing it. This check could either be performed on the time
that is needed for the trajectory to be executed, or on the total distance that the trajectory
indicates. Also, carefully applying constrains to each joint separately will deteriorate the
issues, but it will not solve it.
Larger objects while using the current gripper will make grasping easier and more
solid. In case the system has to manipulate small objects, a replacement of the gripper with
one designed for smaller objects would deﬁnitely help. Of course a more generic solution
would be the use of a tool changer. Information extracted by the RGB-D sensor provide
enough evidence for each object’s size in order to decide regarding the proper gripper that
is more suitable. Thus switching end effector during the process will conclude in a more
robust system. One gripper can not be perfect for all objects.
Regarding grasping, an additional algorithm that estimates grasping poses that lead to
solid grasping will make much of a difference. This additional algorithm is necessary, in
order to have a system which does not require prior knowledge. Also an iterative process
of grasping the object and releasing it again until the misplacement is within a threshold,
is an alternative. This will increase the possibilities for not moving the object during the
last grasping attempt before it moves above the box-hole.
Concerning wrong orientation during placing an object, an addition to the ICP algo-
rithm could improve its performance. As it was mentioned (see more in section 5.3.2) the
ICP can “lock” on a local minimum and calculate wrong rotation due to that. Hence if an
initial rotation is performed that is not precise but close to the correct angle, then the ICP
will skip local minimums and will calculate the exact rotation. A solution on the initial but
51
==PAGE:68==
52
CHAPTER 8. DISCUSSION AND FURTHER DEVELOPMENT
not precise rotation could be the PCA algorithm.
Another technique for placing the object could use force sensing, the so called guarded
motion. Even if the placing pose is not perfectly calculated, the arm could slide the
object around the estimated position and using different orientation that are close to the
estimated one while having access to the forces that the robot senses. When these forces
are minimized the object has the proper pose for placing. However, this would require a
solid placing box that is rigidly attached on the working plane.
Furthermore, placing could be improved by using visual servoing (vision based robot
control). Either image-based (IBVS) or pose-based (PBVS) visual servoing could be
employed. The contrast between these tow methods is that the second one takes into
consideration also the depth information and not only information in the 2D plane. This
technique would transform the developed system from an open control loop to a close
loop one, by extracting information from a vision sensor to control robot’s motion. Using
the current eye-in-hand conﬁguration, the sensor could observe the placing target while
the robot is moving towards it and extract information that will allow re-estimation or
correction of the trajectory, thus more accurate placing.
Lastly, a more precise sensor will deﬁnitely boost the accuracy of the centre of mass
that is calculated for both objects and holes. Taking into account the fact that this system is
aiming industry, testing also other sensors is a necessity. Parts that are used in industry are
mostly metallic, hence reﬂective. This will decrease signiﬁcantly current sensor’s potential.
Alternative sensors should be checked.
