Abstract
The demand for quality battery powered loudspeakers is increasing. To accommodate for
this demand high-eﬃcient audio systems with integrated audio ampliﬁers are required. To
supply such integrated audio ampliﬁers matched power supplies are necessary which can
be achieved with a boost converter. The boost converter can enhance the ﬂexibility of
the ampliﬁer to improve the eﬃciency and cost of the end-application. The focus of this
project is the control of a boost converter which employs the ﬂying capacitor topology to
achieve stable output voltage required by the audio ampliﬁer.
The dynamics of the converter have been modelled using averaging and linearisation
to obtain a linear time-invariant model from a switching system. The loading proper-
ties of an ideal ampliﬁer have been modelled with a constant power load. The system
with constant power load has been analysed both with pole placement approach and an
impedance matching approach. A controller has been designed using a multi-controller
PID-based structure that achieves reference tracking and disturbance rejection for the
converter loaded by a resistive load. A PID-based control structure has been analysed
theoretically for the instability problem caused by a constant power load, and it has been
found that a derivative controller can achieve stability. The theoretical results have been
veriﬁed in simulation and experimentally for a case study. The open-loop results showed
increased dampened dynamics compared the idealised models but were otherwise found
to validate the models applied in the design of the controller. The problems related to
constant power load were experimentally shown, and the solution of a derivative con-
troller was tested which was found to solve the problem in practice, but with limitation
in bandwidth.
The controller solution applies a conventional controller structure and
has low requirements for sampling frequency. Further research into the performance and
possibilities of the architecture is therefore needed.

Abbreviations
• Pulse Width Modulator(PWM): Circuit that generates the duty cycle from a value
to a modulated signal with variable length using a carrier waveform.
• Single Input Single Output system(SISO): General system description for a system
with only a single input and a single output. Such a system are suited for conven-
tional control theory and can be described using transfer functions.
• Multi Input Multi Output system(MIMO): General system description for a sys-
tem with multiple inputs and multiple outputs. Such system cannot be adequately
described by a single transfer function and is usually modelled with state space
description.
• Right Half Plane(RHP): Referring to the right plane of the complex plane, which
corresponds to a positive real part for the system eigenvalues.
• Field-Programmable Gate Array(FPGA): An integrated circuit that can be conﬁg-
ured using a hardware description language to implement most digital circuits.
• Very High Speed Integrated Circuit Hardware Description Language(VHDL): A com-
monly used hardware description language.
• Analogue-to-Digital converter(ADC): A circuit that converts a voltage or current to
a digital value in binary form.
• Proportional Integral Derivative controller(PID): Commonly used control architec-
ture described in section 2.3.3.
• Simulation Program with Integrated Circuit Emphasis(SPICE): Class of software
circuit simulation tools.
• Matlab: Mathematical software tool.
• Simulink: Simulation tool which extents Matlab.

Chapter 1
Introduction
1.1
Motivation
In recent years, music has become more and more accessible to consumers through the in-
creasing amount of audio streaming services like Itunes and Spotify along with increasing
use of smartphones. However, the loudspeaker properties of smartphones are generally
known to have low quality and insuﬃcient volume.
This tread leads to an increasing
demand for portable and high-quality loudspeaker systems [1]. The main limiting factor
in portable loudspeaker devices is battery power. Therefore high-eﬃciency audio systems
are desirable. To accommodate this demand, Inﬁneon has developed several generations
of integrated audio ampliﬁers using modern ampliﬁer technology. The technology gives
signiﬁcant advantages in both eﬃciency, audio quality and distortion[2]. To implement
an audio ampliﬁer into an audio system, the device needs power either from a matched
power source or from an external power converter. Requiring a matched power source lim-
its the ampliﬁers application possibilities and implementing an external power converter
can increase the cost of the system as well as reduce the total eﬃciency. It is, therefore,
necessary to implement an internal power converter into the integrated audio ampliﬁer,
such that a broader range of power sources can be applied directly to the audio ampliﬁer
while maintaining excellent eﬃciency and low cost for each end application.
The main task of an integrated converter for an audio ampliﬁer is to deliver the nec-
essary input voltage for the ampliﬁer. The typical structure for such a converter is simple
to implement, but it has high requirements for the components. These requirements mean
that more extensive and more expensive components are needed for each chip. The in-
tegrated ampliﬁer is being designed for large-scale production, so there is much to gain
from a cheaper and smaller design. Therefore, an alternative architecture for the converter
structure is made using a ﬂying capacitor topology, which has advantages compared to the
conventional converter structure[3][4]. The principle is in simple terms, to add additional
electrical components and distribute the power among an increased amount of components
compared to the conventional topology, such that the requirements to the individual com-
ponents are smaller[5]. The task of designing and controlling the converter is much more
complicated when a ﬂying capacitor topology is selected compared to the conventional
converter topology. Furthermore, the converter has to supply an audio ampliﬁer, which is
more challenging than other systems, as the ampliﬁer’s demand for power is dictated by
the audio signal provided by the end consumer.
The main focus of this project is to design a controller that can regulate a converter

with a ﬂying capacitor topology to the desired output, such that the converter can be im-
plemented in the next-generation audio ampliﬁers. The controller has to work in several
use-cases and has to make the system stable in practice. The goal of this project is to
improve eﬃciency and reduce costs in the end applications, which is high-quality portable
loudspeaker systems.
1.2
Problem
The challenge of controlling a boost converter is to ensure the necessary constant output
voltage while keeping the converter in a stable state. The ﬂying capacitor boost converter
has an additional capacitor which also has to be balanced by the controller. The converter
is an electrical system with fast dynamics compared to a mechanical system, and it is
operated by switching between diﬀerent switching states which is a challenge to model.
The designed controller has to regulate the system under several input/output conditions
using reference tracking with low errors in steady state. The converter is made to supply an
audio ampliﬁer which induces challenging load properties both to model and to control.
To verify that the designed solution works in practice, an implementation of the boost
converter and controller has to be made.
The performance of the solution is veriﬁed
through practical experiments with the implementation on a veriﬁcation board.
1.3
Method
This project will include both a theoretical and practical approach to veriﬁcation. For the
theoretical part, a mixed methods approach where the dynamics of the ﬂying converter

2.1
Audio Systems
The standard way of reproducing music is through a loudspeaker system. The common
loudspeaker system consists of a source, an ampliﬁer, and a loudspeaker. The source will
supply a small amplitude signal containing the full sound information. The ampliﬁer will
amplify the audio signal such that it is strong enough to drive the loudspeaker.
The
loudspeaker converts the ampliﬁed audio signal to hearable sound waves. In order for the
ampliﬁer to amplify the signal, it needs to be supplied with a high voltage. The high
voltage can be delivered from an external source, but to make the end application more
versatile, a solution is investigated where the voltage is generated by a power converter,
which converts the voltage from a source with a smaller voltage.
2.2
Boost Power Converter
In an electrical system it is often required to have a speciﬁc voltage for a circuit to function.
However, basic electrical sources such as power grids or batteries might not supply the
necessary voltages. Therefore circuits for conversion of voltages are often applied. These
are called power converters, and if the converter raises the voltage, it is called a boost
converter. This project is about making a boost converter for an audio ampliﬁer, such
that diﬀerent batteries can power the ampliﬁer. The basic way that a converter works is
by charging and discharging inductors and capacitors such that the output keeps close to
a desired level. The basic concepts of the conventional boost converter will be described
in this section to give an introduction to the boost converters. The conventional converter
circuit is shown in Figure 2.2.1.
Figure 2.2.1: Diagram of a conventional boost converter.

The circuit consists of two switches, a power source, an inductor, a capacitor, and the
load. The source could, for example, be a battery and the load is the system that the
converter supplies. The converter charges the inductor with energy by decoupling it from
the capacitor and the load while connecting it to ground. The connection cause a voltage
diﬀerence over the inductor which charges it. When the inductor has reached a desired
charge level, the circuit will use the switches to connect the inductor to the capacitor and
the load instead of ground. Then the inductor will discharge the build-up energy to the
capacitor and load, which increases the voltage on the output. Before the inductor is fully
discharged the switches change back to the ﬁrst state, where the inductor will be charged
again. While the inductor is charging, the capacitor will transfer energy to the output.
By charging and discharging with high frequency neither the inductor nor the capacitor
will be fully charged or discharged. Correspondingly, the voltage over the output will also
achieve some value equal to the capacitors charge level. Furthermore, the charge level
depends on the periods used to charge and discharge the components. A control system
controls these periods. This project is about the design of such a control system. However,
the boost converter in consideration does not follow the standard structure. To improve
eﬃciency in the audio system a more advanced circuit is used, where a ﬂying capacitor
topology is applied. The ﬂying capacitor will be described in section 2.2.2.
2.2.1
Pulse Width Modulation
To increase the output voltage of a circuit, the boost converter shift between charging
energy into energy stores and discharging the energy into the output. To change states
between the diﬀerent functions switches are applied. The switches can either be on or oﬀ.
Their states are controlled by a signal, which is called: the switches gate signal. In terms
of ensuring that the switch functions are as ideal as possible and that the gate voltage
is correct, is the task of an analogue designer, which not be considered in this project.
Therefore, the switches gate signal can be considered as a binary signal that either turns
the switch on or oﬀ.
However, most controllers will not output a binary signal, but
a decimal value that describes how long each period should be. This signal is called the
duty cycle, and it will be described in details later. For now, the task of converting a signal
for period length to binary time signal is considered. In other words, the assignment of
making pulses with variable duration based on an input value, which is commonly known
as pulse width modulation(PWM). A common way to implement PWM is to compare the
input value to a saw wave or a triangular wave. If the input value is lower than the wave,
then the output will be a binary one or else it will be a binary zero. For example, if the
waves are assumed to go between 0 and 1 in amplitude, then 0.5 input value will result in
a pulse with a length of 50% of the saw wave period. Thus, the frequency of the saw wave
determines the rate of the pulses, this frequency is called the switching frequency, as it will
also be the operating frequency of the switches. The switching frequency is an important
design parameter for a power converter. In some articles [4][6], half of the switches are
replaced with diodes which achieves the same basic operation of the boost converter as
the switches otherwise is operated in pairs. The reason switches are applied in this project
is to avoid voltage drops in the diodes which is often more signiﬁcant than in switches.
However, this should not have any eﬀect on the system in relation to the control problem.
2.2.2
Flying Capacitor Topology
The ﬂying capacitor topology is in a multilevel class of power converters. Which means
that it uses an increased amount of components compared to the conventional converter

topology in order to achieve some advantages. In this project, the topology is chosen to be
a 3-level ﬂying capacitor boost converter. Compared to the conventional converter shown
in Figure 2.2.1 it contains two more switches and an additional capacitor. The capacitor
is added in the middle and works as extra internal energy storage. The topology can be

The additional capacitor is called the ﬂying capacitor, as it is not connected to ground,
input or output but is ”ﬂying” between them. One advantage of this converter topology
is that the switches in the circuit are exposed to reduced voltage stress. Voltage stress
describes the amount of voltage the switch needs to handle. If a switch needs to handle
higher voltages, it will generally have a higher energy loss. Therefore, the lower voltage
stress is a desired property. The ﬂying capacitor topology is beneﬁcial when high power
density is desired such as in integrated circuits, because of the reduced voltage stress[7].
Furthermore, the inductor current ripple can be reduced in the ﬂying capacitor topology.
Current ripple is the variations in the current over a single switching period, where the
inductor will both charge and discharge. Reduced current ripple allows for a decrease in
inductance of the inductor, which can also reduce size, losses, and cost of the circuit. Be-
sides from voltage stress and current ripple the ﬂying capacitor topology should give lower
harmonic distortion compared to the conventional converter topology [3][8]. However, the
addition capacitor also presents challenges which will be described later.

values. Furthermore, the cost of a single chip is expected to be low which further limits
the constraints on component values. The most problematic component limitation is the

the capacitors are expected to be limited to maximum 20µF as this is small for a boost

2.3.1
Transfer Functions
The most important information about a system based on control theory is the relationship
between system inputs and system outputs. For simple single-input-single-output(SISO)
linear time-invariant systems the input/output relations can be described by mathematics
in a transfer function. A transfer function is in the Laplace domain, which means they
depend on the complex variable s.
The transfer function can be calculated based on
physical knowledge of the system or derived based on input/output data from the system.
However, in both cases, the description will not be perfect, and it is therefore called a
model of the system. The circuit in considerations in this project is modelled before it
is implemented, and so it is not possible to get measurement data from the system for
modelling. Therefore, the models of the system will be based on physical laws and circuit
analysis. The problem with transfer function is that they only model the relation between
one input and one output, which makes them less useful for MIMO systems. Especially,
when the inputs and outputs are interconnected.
2.3.2
State Space Models
A commonly used mathematical representation of a system applied for control theory is
the state space representation. In this representation, the system is modelled by inputs,
outputs, and internal states.
The change in the state can be described by a system
dynamics matrix times the state vector and an input dynamics matrix times the input
vector. The output can be found by an output dynamics matrix time the states and the
feedthrough matrix times the input. However, the feedthrough matrix is often only zeros.
The state space model structure is shown in Equation 2.3.1.

on a state space model a transfer function can be generated from each input to each

2.3.3
PID Controllers
The most commonly used controller structure is the PID controller structure, even though
some of the elements might not be used in all applications. PID stands for proportional,
integral and derivative which are the elements of the controller structure. Based on the
system output and the desired value called the reference an error is calculated. The error
is fed to each component of the controller. The proportional controller multiplies the error
signal by a constant factor, which is called the controller gain. The integral integrates the
error and then multiply the result by another controller gain. The derivative calculates

the derivative and multiplies by a third gain. The sum of the results are then sent to the
system as input. The design parameters are the gains of each controller element. A PID
controller can be build in both ideal and parallel form, but mainly the parallel form will
be considered for implementation in this thesis. Based on the system, the eﬀects of each
controller element can aﬀect the resulting system diﬀerently. A system supported by a
controller is called a closed-loop system, as the controller would make a loop around the
system. It is crucial that the closed-loop system is stable. Otherwise, the system output
will grow unbounded. In practice, an unstable system is bounded by physical limitations,
but it will cause something to break or another undesired result.
2.3.4
Control System Properties
When evaluating the design and performance for a control system several parameters are
used. The section will describe the most important quantities applied in this project.
• Operating point: The control problem depends on the conditions of the system,
therefore the system can be characterised in diﬀerent modes of being. A characteri-
sation of the system in single mode is an operating point, which is especially useful
when approximating the behaviours of the system.
• Steady state: When a system is left untouched it will either immerse to a ﬁnal value
or grow unbounded. The term steady state describes the theoretical state of the
system when an inﬁnitely long time has passed.
• Settling time: The parameter settling time is used to describe the time it takes for
a system to settle to a value close to the ﬁnal value it will have in the steady state.
This parameter is used to determine the speed of a closed-loops reference tracking
capabilities.
• Steady state error: The a system is augmented by a controller that should regulate
the system output to a reference level if often deviate by some amount in the steady
state. The error between the desired and the resulting output value for the system
in the steady state is the steady state error.
• Natural frequency: Some systems exhibit oscillatory behaviours when operated. The
frequency of the oscillation is called the natural frequency of the system which is an
important parameter when modelling the system.
• Dampening: A eﬀect that reduces the oscillations usually cause by resistance in the
system.

3.1
Control of Switching System
A switching system like the boost converter is a challenge to control since conventional
control theory normally only applies to linear systems and switching will cause the system
to change dynamics in a piece-wise linear manner. To solve this problem, diﬀerent methods
can be used. The methods will be outlined below:
3.1.1
Average State Space Models
This method use averaging and linearisation of the system to convert the system model
from a non-linear time-variant model to a linear time-invariant model, such that the
conventional control methods can be applied to the system. Firstly, the system can be
described in a non-linear and time-variant switched state-space model, which can not be
used for control since it is a non-linear and time-variant model. For a two-state system
the state space will look the following:
˙x = [A1δ(t) + A2(1 −δ(t))]x + [B1δ(t) + B2(1 −δ(t))]u
y = [C1δ(t) + C2(1 −δ(t))]x + [D1δ(t) + D2(1 −δ(t))]u
Then the state-space model is integrated over the switching period, and the average val-
ues for the states are chosen as new state variables, such that the model becomes time-
invariant.
˙˜x = [A1δ1 + A2(1 −δ1)]˜x + [B1δ1 + B2(1 −δ1)]˜u
˜y = [C1δ1 + C2(1 −δ1)]˜x + [D1δ1 + D2(1 −δ1)]˜u
Where tilde marks the average value of the variable. Once averaging has been applied to
the model, linearisation can convert the non-linear model to a linear model. Note that
after the linearisation of the system only the dynamics of the fundamental harmonics will
remain[3]. Both as a consequence and an advantage of the averaging method, the control
architecture is limited to a single switching strategy and will have a constant switching
frequency.
The result can be a controller which is sensitive to massive ripple eﬀects,
but the advantage is that since only the average of the states is necessary, it requires

3.1.2
Sliding Mode Control
Sliding mode control is a method where the non-linear system with switching states is
controlled using a discontinuous control strategy. The discontinuous control strategy con-
sists of two or more continuous control states that each lead the system states in diﬀerent
directions. For a boost converter, the control states will be the states of the switches in the
system, which will give charging or discharging dynamics for the components. The goal
is that the system will slide along the desired path by having control dynamics on each
side of the path that leads the system back on the path. Thus a sliding mode controller
will directly change the state instead of controlling the averaged duty cycle in a constant
switching strategy. The advantage is a fast controller which can react to ripple and high-
frequency disturbances. Especially when dealing with a constant power load, which will
be further described in section 4.4, the sliding mode controller is a common choice [10][11].
However, this puts large requirements on the sampling frequency and precision of the mea-
surement system, as even the steady state of the system will be depending on detecting
a change of output within a switching period. Furthermore, this will often require a vari-
able switching period which might be more complicated to analyse for switching losses.
This method is not applied in this project, mainly because of the added complexity when
working with a ﬂying capacitor. The ﬂying capacitor topology has the additional state of
the extra capacitor which means that the sliding mode controller will be a MIMO system.
In a MIMO system the sliding surface becomes a complex sliding area, which requires a
far more complex controller solution both for design and implementation, especially as the
input to the system it coupled pulse width signal instead of the actual choice of states.
3.1.3
Active Damping Methods
Another possible solution to achieve stability for the system is by adding a resistive load in
parallel with the load or in series with the inductor to apply active damping[12], however
this will have a negative eﬀect on the eﬃciency. Similar results are achieved by adding a
controlled current consuming component close to the load [13]. Improved eﬃciency can be
achieved with a storage system or a large capacitor [14], but it requires additional circuits,
and the size of a large capacitor is not well suited for integrated circuits.
3.2

In order to create control loop that can react to changes in the system and perform
reference tracking it is necessary to measure essential states in the system. There are
two commonly used methods for implementing such a measurement system. Note that in
extension to both methods it is necessary to measure the ﬂying capacitor voltage as will
be further described in section 4.2.5.

3.2.1
Voltage Mode Control
This method measures only the output voltage of the converter. Even though the inductor
current is not measured it can be assumed stable when the output is stable as the two
states will depend on each other. It has the advantage that the system output is directly
regulated which means that any undesired variations are always visible to the controller,
and the measurement system will only require a voltage measurement.
Voltage mode
control is chosen as measurement for this project and implemented in the ﬂying capacitor
boost implementation.
3.2.2
Current Mode Control
Current mode control measures only the current in the inductor of the converter. The
same dependency between the output voltage and the inductor current as in the voltage
mode control will apply. Hence the output voltage will follow the inductor current with
some bandwidth and gain.
The advantage is that the inductor current is much more
sensitive to disturbances than the output voltage and a solution based on the current
mode control will, consequently, be more sensitive to a disturbance on the input of the
converter. However, the ﬂying capacitor boost converter should be balanced to ensure
the underlying assumption of a direct link between the inductor current and the output
voltage.
3.2.3
Combination
If both the inductor current and the output voltage is measured the controller will have full
state information along with a mathematical redundancy. This might have an advantage if
a state feedback controller is applied and gives the possibility for a fault tolerant controller
architecture. However, it will need all of the requirements from the two above mentioned
measurement systems. For a PID based controller architecture, this will not be useful as
the controller can only apply one of the measurements.
3.3
Linear Controller Architecture
Once a linear time-invariant model of the system is achieved, any conventional control
method can be applied. In this project, a PID based controller is designed and imple-
mented, but the controller could also have been designed as a linear state feedback con-
troller. The result of an optimal PID based controller and an optimal linear state feedback
controller should be very close, as the PID based controller applies all the measurement
information provided and since the system is semi-decoupled. It could be interesting to
investigate if the linear state feedback controller has any advantages compared to the
PID-based, but this is not included in this thesis.

But in this project, the main simulation tool is MATLAB Simulink since it is an eﬃcient
tool within controller design. All simulation has been done with the ODE45 solver as
it is assumed to be the most conservative with a reasonable simulation time. A simula-

secondary tool Cadence is applied for circuit simulations.

4.2
Average State Space Model
To apply conventional control theory methods a linear time-invariant model is needed.
Often a non-linear time-invariant model is the initial model and then by linearisation using
ﬁrst-order Taylor approximation a linear model can be acquired. However, the switching
dynamics in the converter causes the converter to have linear time-variant properties, as
described in section 3. Therefore, only a piece-wise linear model can be constructed, where
only the dynamics in each state can be modelled. This piece-wise linear model is named
the switching model. To deal with the challenging piece-wise dynamics an average state
space modelling was chosen in section 3. With average state space methods, the switching
model is converted to a non-linear time-invariant model by averaging over a switching
period. That model is called the average model. The average model is then linearised to
get a state space model, which is both linear and time-invariant. That model is called
==PAGE:35==
4.2. AVERAGE STATE SPACE MODEL
19
the average state space model and will be the primary model used for the design of a
controller. Each model is derived and presented in the sections in this chapter.

4.2.2
Switching Strategy
To get from the time-variant switching model to the time-invariant average model the
variables has to be averaged across the general switching period. To accomplish this, it is
necessary to identify the steady state behaviour of the switches. The switching model in
section 4.2.1 speciﬁes the behaviours in each state, but the task of choosing which states
are active in which sequences remains. A sequence of states utilised during each switching
period is called the switching strategy or switching rules. One of the advantages of the
ﬂying capacitor topology is that there is a way to choose switching strategy such that the
system, in theory, becomes self-stabilising [15][16]. How the switching strategy is chosen
can be described in two ways: the intuitive way and the mathematical way.
The intuitive way
The intuitive way looks at the general behaviours in each state
and choose a combination that balances the states of the system. To simplify this task,
simpliﬁed dynamics of each state are shown in Table 4.1. For the output to achieve the
desired level the output stage, consisting of the output capacitor and the load, have to be
charged and discharged. The output is charged in state B and D, but discharged in state
A and C, so the strategy ought contain a combination of these states. In state B and C,
the ﬂying capacitor is used for energy transport, wherein states A and D only the inductor
is applied. It is important to note, that the capacitor nor the inductor can discharge to the
output if they have not been charged in any previous periods. Therefore, a valid strategy
must at least contain either state B and state C or state A and state D. If state B and
state C are not used, it can be seen from the sub-circuits in Figure 4.1.2 that the ﬂying
capacitor will not be used at all. In this case, the converter functions as a conventional
converter, and thereby does not give the desired properties of the ﬂying capacitor topology.
So, in order for the converter to act as a ﬂying capacitor converter, it must to include the
ﬂying capacitor by using the states B and C. If the switching strategy is chosen such
that only state B and state C are utilised with 50% period for each, the ﬂying capacitor
voltage is balanced and an output voltage of 2 times the input will be achieved. With
lower output voltage, the output stage will discharge in shorter periods and charge over
extended periods and thus utilising the B and D states further. However, if the B state
is used more than the C state, the voltage balance in the ﬂying capacitor will be lost, as
it will discharge longer than it charges. Therefore, for lower output voltage the switching
strategy should include state B, C and D, such that the periods of state B and C remains
in balance. The opposite applies for higher output voltages. The transport components
will need to charge longer, and the output needs to discharge longer. Therefore the states
A and C should be used more. To keep the balance in the ﬂying capacitor, the state A, B
and C have to be applied. These requirements result in a switching strategy where state

An eigenvalue analysis can determine the stability of a state space system.

eigenmode of the system is marginally stable. A positive real part of an eigenvalue indicates
that the system is unstable. Furthermore, if the eigenvalues have a non-zero imaginary

the state. When a state is marginally stable, it is on the very edge of being unstable, and
the smallest change of dynamics could cause it to become unstable. Such a state needs to
be controlled to achieve robust closed-loop dynamics than will work in practice. Therefore,
the state cannot be allowed to be uncontrollable and additional actuation is required.

Control Design
The system has been modelled to achieve both a linear time-invariant state-space model
and an impedance-based model for the cascaded converter problem based on Middlebrook’s
extra element theorem. This section will use the models to design a controller for the
system. The controller should primarily stabilise the system, but also achieve fast reference
tracking while reducing propagation of noise as much as possible. To simplify the design
process, a controller has been designed for the boost converter loaded with a resistive load
and secondly with a constant power load. Stability of the system with a constant power
load is analysed for both the state space model and the impedance-based problem. Lastly,
problems related to realising the controller are outlined.
5.1
Multi Controller Solution
The system is a MIMO system since the system has two output measurements and two
inputs. In such case conventional PID-based control theory would usually not be applica-
ble, but as presented in an article by Hung-Chi Chen, Che-Yu Lu and Wei-Hsiang Lien,
the dynamics of the system and the general requirements for reference tracking allow for a
combination of PID-based controllers to control the system [8]. General requirements for
ﬂying capacitor voltage are half the output voltage, which is a natural steady state with
a resistive load. Note that this study assumes that the system is loaded with a resistive
load, which is a simpler case than the ampliﬁer load.
Based on the system dynamics
shown in section 4.3 the ﬂying capacitor voltage and output voltage states are indepen-
dent and that one input aﬀects only the output voltage while the other eﬀects both states.
Therefore, it can be deduced that the output voltage can be controlled with the ﬁrst input
independently of the ﬂying capacitor voltages. However, to control the ﬂying capacitor
the second input has to be used, which will aﬀect the output voltage. This eﬀect on the
output voltage control can be seen as a disturbance, which can be rejected with the right
controller architecture to achieve zero steady-state error. Therefore, the control problem
can be decoupled to two control problems: One control problem with the goal of con-
trolling the ﬂying capacitor using the period imbalance, which will be called the voltage
balancing control problem and one where the goal is to control the output voltage using
the duty cycle, which will be called the voltage regulating control problem. These control
problems are both SISO problems which can be addressed with conventional PID-based
control. However, it is important to note that this control problem’s independence only
exists in the stationary point. Therefore, a requirement for a solution based on this ar-
chitecture is that both states can be stabilised to the stationary point. Furthermore, the
model will be less accurate the further the state moves from the stationary point. To

Integral Controller Design
Based on the knowledge that the integral term dominates
the settling time and achieve zero steady state error a purely integral controller is designed.
To simplify the design, a dominant pole approximation can be made on the closed-loop
system. Dominant pole approximation is a method where the system is described by the
pole that dominates the response to a step change, which means the slowest poles in the
system. It is a good approximation when there is a single pole or pole pair closer to the
imaginary axis than the rest and it is especially useful when considering the settling time
of the system. A pole is considered dominant if it is not cancelled by a zero. However,
this is not the case for the voltage regulation problem. For the voltage regulating problem
with an integral controller, the pole made by the controller is the dominant pole compared
to the open-loop systems poles for a broad range of low gain values. Therefore, for low
integral gains, the system can be approximated by dominant pole approximation to a
ﬁrst order system with only the integral controller pole and thus the settling time can be
determined for that system. The approximation will still have the RHP zero which will
aﬀect the settling time. However, for this approximation, the RHP zero is assumed so
far from the imaginary axis that it has little eﬀect on the system. The resulting coarse
approximation is a ﬁrst order system with no zero which can be used to determine the
approximated settling time of the system. It should be noted that this approximation is
limited to a single steady state point and only when the integrator gain is low enough for

Limited control eﬀort
The period imbalance is made as a skew in the period where
the ﬂying capacitor is active. How signiﬁcant those periods are is a limiting factor as the
skew in periods cannot exceed the period size. This means that at 50% duty cycle the
maximum period imbalance is also 50%. However if when the duty cycle goes towards 0%
or 100% the maximum period imbalance also goes towards zero. A duty cycle of 100%
will never be used as that theoretically would result in inﬁnite large output voltage, which
cannot be achieved in practice. A duty cycle close to zero is possible, but this is considered

a special case as the system will be almost inactive, and the ﬂying capacitor disturbances
can be assumed to be neglectable. This knowledge about common operations mean there
will always be some possible actuation in the period imbalance in normal operation, but
it is a limiting factor for the proportional gain not to saturate the imbalance for common
disturbances.
It should be noted that the eﬀect of the imbalance disturbance on the
other control loop also depends on the current duty cycle. The duty cycle to the output
voltage is a non-linear function, which is diﬀerently aﬀected by a percentage change. The
disturbance aﬀect most when close to 0% or 100% which should also be considered when
limiting the control eﬀort of the voltage balancing loop. However, it is assumed that this
is not the main limiting factor of the gain.

5.4
Control with Constant Power Load
As described in section 5.2.3, a boost converter with a resistive load can be controlled using
integral control, as the system is open-loop stable and simply needs reference tracking
properties. However, this is not suﬃcient when working with a converter with a constant
power load because of the instabilities caused by the load dynamics.
As described in
section 4.4 the ampliﬁer acts as a constant power load which for the linearised model can
be approximated with negative resistance. Note that the approximated negative resistance
will be considered as RCPL = −R in this section to avoid hidden signs. A study claim
conventional control is not possible for boost converters loaded by purely constant power
loads [25]. However, the arguments are that the converter is unstable in open-loop, which
does not theoretically mean that conventional control inapplicable, unless none of the
controller elements have stabilising properties for the system. Therefore, the following
section investigates the system with a constant power load, both in open-loop and with
conventional PID control. Some studies model the boost converter with some parasitic
resistances or a parallel resistive load [19][26], but this is found to have a dampening and
stabilising eﬀect in section 4.6. The analysis in this thesis is therefore made with ideal
components and only a single purely constant power load, to access only the worst case
problems of the constant power load.

5.4.4
Integral Derivative Control
In the section 5.4.3 it was concluded, a derivative controller exists that stabilise the boost
converter loaded with a constant power load. However, there are several problems related
to derivative controllers. One the of problems is that a derivative controller does not oﬀer
anything in relation to reference tracking. Even though the closed-loop with a derivative
controller would be stable, it would not follow the reference. Either a proportional con-
troller or an integral controller is needed to add reference tracking properties. It is known
from section 5.2.2 that the proportional controller has a very destabilising eﬀect on the
system compared to the steady state error in reference tracking. Therefore, an integral
controller similar to the one for a resistive load is chosen for reference tracking capabili-
ties. From Figure 5.4.1 it is known that the integral controller has a destabilising eﬀect on
the closed-loop system, so the integral gain should be chosen as a trade-oﬀbetween the
speed of reference tracking and stability of the solution. The integral gain which achieves
a speciﬁc settling time depends on the choice of derivative gain as both controllers aﬀect
the placement of the poles. Based on the root locus the derivative gain that results in
the most stable system can be determined. However, there are other factors than stability
that might inﬂuence the choice of derivative gain, such as noise propagation and controller
realisation.

5.5
Realisation of Controller
A derivative controller is designed for the system, which stabilises it even with a constant
power load. A problem with a derivative controller is that an ideal derivative controller
cannot be implemented in practice or even in MATLAB Simulink. The reason for this is
that an ideal derivative controller would require knowledge about the instantaneous change
of error, which means calculating inﬁnitely fast or knowing the future of the error. To
realise the controller, it is necessary to limit calculation speed or bandwidth to a resonable
value. A commonly used mathematical realisation is a ﬁrst order derivative ﬁlter:
Contd =
Kds
Tfs + 1
(5.5.1)
The resulting controller will act as a derivative controller up to the ﬁlter frequency. The
ﬁlter should be made such that the cut-oﬀfrequency is higher than the highest frequency
for the system dynamics, as the controller thereby will act as an ideal controller for the sys-
tem. The main eﬀect of the derivative controller is that it suppresses the resonance peak.
The ﬁlter cut-oﬀfrequency should therefore be chosen to be higher than the resonance
frequency. The maximum cut-oﬀof the ﬁlter is in the implementation of a controller re-
lated to the frequency of the digital controller as the derivative cannot be calculated faster
than the sampling time. This will be described further in the implementation section in
6.2.2. A property of the derivative ﬁlter is that it reduces the high frequency gain of the
controller, which removes some of the high frequency noise propagation.

6.2.2
Digital Controller Implementation
A PID controller takes one input and gives one output, but it contains three controller
elements. The proportional controller multiplies the current error with a gain. A digital
integrator accumulates error and multiplies the accumulated by a factor since this corre-
sponds to integrating in discrete time. The derivative controller is more challenging as it
cannot be implemented in practice since that would require knowing the next output value
of the system. Therefore, it is approximated by the change in output from the previous
output to the current. This approximation degrades the performance of the derivative con-
troller but is necessary for implementation [29]. The approximation becomes worse with
low sampling frequency as the interval between data point becomes large. However, if the
sampling frequency is high, this approximation is assumed to be suﬃciently good. The
implementation in VHDL is made such that it contains one synchronous process and the
purely combinatoric. This gives minimum delay in the implementation as the controller
does not require several states to calculate the output value. Furthermore, it is crucial
that the derivative contributions are only calculated once per sample. The synchronous
process loads new input saves the old input and accumulates errors for the integrator.
The combinatoric part calculates the contribution from each controller element and sums

Measurements and Results
The designed controller has been tested both through simulation and in practice for veriﬁ-
cation of the methods, models and solutions. The controller has been implemented on the

7.1
Model Veriﬁcation
In section 4 the boost converter is modelled and analysed with a resistive load. The open-
loop duty-cycle to output voltage is found to be stable without a controller. The ﬂying
capacitor is found to be marginally stable, but in practice, some non-linearities will cause
the ﬂying capacitor to be slightly self balancing with a resistive load. The circuit therefore
can be tested without a controller to verify the models of the open-loop system with a resis-
tive load. An experiment is carried out, by varying the duty cycle to the open-loop circuit
and measuring the transient response for the output voltage with an oscilloscope. In the
ﬁrst attempt this experiment did not give suﬃciently correlated results, and the possible
losses in the system were therefore investigated with a thermal analysis, which is described
in Appendix C. However, with the second veriﬁcation board, the experiment gave bet-
ter results which is assumed to be because of board improvements and improvements to
measuring method. The results from Matlab, Cadence and experimental measurements
from the second board can then be compared.
All the results from both experiment,
simulation and modelling are shown in Figure 7.1.1. Several important conclusions can
be made from the plot. Focusing on the experimental measurements (blue graph in Fig-
ure 7.1.1), it can be seen that the system does react to the step change, it exhibits damped
oscillatory behaviour, and the signal does contain some fast ripple corresponding to the
switching frequency. When compared to the remaining graph the experimental results is
more damped, which is assumed to be because of unmodelled losses in the system. The

The cadence model is made with a circuit simulator, and it includes a complex model of
switches, dead time delays and parasitic components, but it does not include magnetic
core losses in the inductor. It can be seen that the cadence simulation response also has
the damped oscillatory dynamics which corresponds to the experimental results but with

7.2.4
Ampliﬁer Load
In section 5.4.1, it was concluded that the open-loop boost converter becomes unstable
when loaded with an audio ampliﬁer that draws power. A closed-loop controller is designed
and implemented which based on theoretical conclusion and simulations should stabilise
the system, even if the ampliﬁer acts as a purely constant power load. To verify that this
achieved in practice, the closed-loop controller is implemented on the boost converter on
the veriﬁcation board and tested by loading it with an audio ampliﬁer. Experiments are
made with a ramping signal with the ampliﬁer supplying a purely resistive load.
Experiment parameters
The closed-loop ampliﬁer experiment is done with a MA12070P
class D audio ampliﬁer from Inﬁneon Technologies, which is implemented into an evalua-
tion board made for general testing and evaluation of the chip. The board is made with
large decoupling capacitors, which were reduced to 0.2µF to reduced oscillatory reﬂec-
tions with the connector cables. A Raspberry Pi controlled and provided the test audio
signal for the audio ampliﬁer was. The test signal was chosen as a 1KHz sinus wave with
ramping amplitude over 4 seconds.
Open-Loop Stability
In the theoretical part is was concluded that the open-loop sys-
tem should be unstable with a constant power load, but with losses it achieves a small
margin of stability. However, as the power is increased the constant power load dynamics
of the audio ampliﬁer should become dominant and cause instability. This causes the
voltage drop which can be seen in the open-loop results in Figure 7.2.5 at about 2.6s.
At about 3s the ramping signals amplitude becomes higher than the maximum ampliﬁer
threshold, and it starts to perform clipping. When the ampliﬁer starts clipping the signal
the peak power does not increase further, and the open-loop voltage does not drop further.
In the Figure 7.2.6, it can be seen that the open-loop output voltage varies with the power
demand of the ampliﬁer as thus it is found to be open-loop unstable and the theoretical
conclusion is conﬁrmed.

Closed-Loop Performance
Based on the experimental results in Figure 7.2.5 and
Figure 7.2.6 it can be seen that the closed-loop system regulates the boost converter
output voltage.
At about 3sec In Figure 7.2.5 the output voltage starts to vary with
levels up to 18V which is undesired. The problem occurs when the ampliﬁer starts to clip
the signal amplitudes which can be seen in Figure 7.2.6. The problem is assumed to be
caused by the fast non-linear dynamics that occurs when the ampliﬁer leaves the clipping
range. The power region where the ampliﬁer starts clipping should be avoided for common
operations, but the experiment still shows that the stability of the closed-loop solution
degrades if the constant power load vary with fast dynamics. Beside from the variations
that occur during clipping the variation in the closed-loop output voltage is small and
without voltage drops. It can be seen Figure 7.2.6 that the system is resistant to the
variation of the ampliﬁer power consumption. However, the steady state output voltage
is about 15.5V at 2sec and almost 16V at 2sec. The reference might have been chosen a
little too high but it does not explain the change of steady state voltage. The variation is
assumed to be because of ripple sampling. When the power consumption is high the ripple
will be larger, and thus the error from sampling to average can be worse. Based on this
closed-loop experiment it is concluded that implemented control loop can stabilise and
control the output voltage of the boost converter loaded by an audio ampliﬁer. However,
the closed-loop is also found to have deviations from the desired output if the constant
power load changes power demand with fast dynamics.

Sources of Error
The section will describe some of the challenges and error that occurred during implemen-

7.3.1
Measurement Errors in Closed-Loop
A closed-loop experiment with the ﬁrst veriﬁcation board revealed a problem with the
measurement system. The measurement system consists of two diﬀerential ADCs which
are implemented with discrete components on the veriﬁcation board. The measurements
are inputted to the FPGA, where the controller is implemented. The closed-loop system is
tested by making a step change in the reference signal similar to the experiment described
in section 7.2.1.
The resulting transient responses show big problems which measure-
ments errors. Through repeated experiments with diﬀerent input voltages and various
references, it was found that the errors occurred more frequently at some combinations of
input/output relations than others. The cause of this problem is assumed to be electrical
disturbances caused by switching in the core boost converter and an oversensitive ADC. If
the disturbances in the switches synchronise with the sampling in the ADC, it might cause
severe errors in the measurements. The eﬀect could be reduced if the board had a good
ground plane or a robust ADC implementation, but both of these elements had problems
on the ﬁrst veriﬁcation board. The two step responses shown in Figure 7.3.1 are made
at an input/output relation where the measurement error only occur rarely. One step
response is plotted where the error exists, which can show the magnitude of the errors. It
can be seen the actual output voltage is about 10.4V with the measurement at about 0.5V ,
which is a colossal error. The voltage in the ﬂying capacitor is included in Figure 7.3.1
to show that the error does not aﬀect the balance in the ﬂying capacitor voltage. This
problem is assumed signiﬁcantly reduced on the second board by improving the ground
plane and applying buﬀer ampliﬁers to secure the correct input impedance for the ADCs.

7.3.2
Decoupled Ground Planes
The setup consists of an FPGA, The boost converter and an audio ampliﬁer which are
each on an individual board with an independent ground plane. The coupling of ground
planes between the board is limited to wires.
It was found that a problem similar to
the measurement errors described in section 7.3.1 occur when the ground planes are not
properly connected. To improve this the FPGA board and the veriﬁcation board was con-
nected with two ground wires and a copper knot in the corner. However, the ground plane
will still not be as coupled as in an integrated chip, which might cause small measurement
errors in the closed-loop experiments.
7.3.3
Practical Measurements
The measurement made in this project has been mainly focused on the average behaviour
of the signals. Therefore, it was assumed suﬃcient to ground measurement probes at the
side of the board. However, for some measurement, it could cause a distance from the
measurement point to the grounding point of about half the veriﬁcation board. This can
cause the measurement to pick up some disturbance from the board which is not present
in the desired signal. However, it should be noted these measurement errors should have
little eﬀect on the average values of the signal, but mainly cause a little more switching
noise to be picked up.

7.3.4
Wire Inductance and Input Capacitors
The through closed-loop test with the audio ampliﬁer it was found that the wire from
the boost converter to the audio ampliﬁer and input capacitors on the ampliﬁer board
causes problems to the solution. The long wire between the two boards will act like an
inductor which will combine to a serial oscillator with the input capacitors at the audio
ampliﬁer. This will add an additional pole pair to the closed-loop and thus an additional
resonance peak. The closed-loop has not been designed with the extra dynamics in mind,
and the solution might be unstable for this scenario. The problem will be diﬀerent for an
integrated solution as the boost converter, and the audio ampliﬁer will be combined in the
same chip, and the long wires will therefore not exist. To reduce the eﬀect of the problem
smaller wires was selected and rolled together and the input capacitors were reduced in
size.

Conclusion
This section will summarise the project with the most important ﬁndings. The section

8.1
Conclusion
To improve the eﬃciency and compatibility of an integrated audio ampliﬁer chip, a ﬂying
capacitor boost converter has been analysed with a focus on the control system. The goal
was to secure a stable output voltage from the converter even with the load properties of
an audio ampliﬁer and with the additional state of the ﬂying capacitor. The challenge of
modelling switching dynamics was handled using averaging and linearisation. The result-
ing model was found to be close to reality based on simulation and experimental results.
The open-loop system was stable with a resistive load.
However, the ﬂying capacitor
voltage was marginally stable and should in practice be controlled using an imbalance in
switching strategy. The load properties of an ideal audio ampliﬁer were investigated, and
it was found to act as a constant power load. The open-loop system was unstable with a
constant power load. To enforce reference tracking on the system load by a resistive load a
purely integral controller was designed to give the best balance between settling time and
stability. For the ﬂying capacitor balancing a purely proportional controller was designed
which achieves zero steady-state error and small disturbances on the other control loop.
For the system loaded by a constant power load only a derivative controller can achieve
stability for a speciﬁc interval of gains that depend on the system. In this case, an integral
derivative controller was designed for the closed-loop to achieve stability and reference
tracking. The designed controllers were implemented in a test setup, which included a
digital controller and digital pulse width modulation.
Based on theory, simulation and experimental validation a controller has successfully been
designed and implemented which achieved constant output voltage for a ﬂying capacitor
boost loaded by an audio ampliﬁer. Limitations in implementation and non-ideal com-
ponents aﬀected the practical solution. Furthermore, it was found that the solution was
limited in bandwidth and deviations in the output voltage occurred when the ampliﬁer
changed power demand with fast dynamics.
The applied controller solution has the advantages both in the simplicity of implementation
and in low requirements for sampling frequency, and is therefore a worthy candidate for
further research.

8.2
Future Work
This section will discuss some options for future work based on research and ﬁndings from
this project.
Non-ideal modelling and analysis
Analysis of losses and non-ideal components was
outside the scope of this project, including the calculations and considerations concerning
eﬃciency. Only considerations to the overall eﬀect of losses have been included. However,
to get a high quality validation of the controller design, it was necessary to acquire an
accurate model for the converter on the veriﬁcation board. Therefore a future study of
losses and non-idealities in the system could be beneﬁcial. Such a study could furthermore
result in a strategy for choosing switching frequency and reference signals to minimise
losses in the converter as well as the audio ampliﬁer.
Other Controller Structures
In this thesis, a PID-based multi-controller solution was
tested and veriﬁed. The arguments for choosing this architecture was presented in section
3 of the project. However, there are several other possible controller architectures, which
could prove not only functional but maybe also an improvement over the PID-based con-
troller. It could therefore be interesting to investigate the optimal control architecture
based on a comparison of controllers. Especially, the architecture of a sliding mode con-
troller based solution as this has been mentioned in several articles as a solution to the
problem of constant power load instability[32][33].
Improved Implementation
The digital implementation of the controller and the PWM
generator was in this project limited to a minimum for veriﬁcation, which caused limi-
tations in the closed-loop system. The issue of digital PWM generation is a broad topic
and could be improved in several ways. The quantisation problem described in section
6.2.3 could be improved, but it could also be beneﬁcial to make a measurement system
and PWM scheme that secure asynchronous sampling of output value and the switching
in the system. The beneﬁt could be avoidance of switching noise in the measurements. It
is however challenging as the controller changes the timing in the system.
Full Veriﬁcation
The veriﬁcation in this project was based on a case study in selected
scenarios. The results therefore demonstrate that the speciﬁc system works in the par-
ticular cases, which supports the hypothesis that the solution is fully functional. But to
fully verify the solution a more thorough veriﬁcation process could be beneﬁcial. A gain
and phase measurement of the system which sweeps the behaviour of the system for all
frequencies could give a precise and in-depth measure of the closed-loop performance.

Appendix C
Thermal Loss Measurements
Through open-loop experiments, it has been found that the step response of the boost
converter implemented on the ﬁrst board diﬀers from the step responses of the simula-
tions and mathematical models, even when the models and simulation are augmented with
losses. The deviation is assumed to be caused by another source of loss not modelled in the
system. To further investigate for possible losses a thermal camera has been applied. Most
losses are converted to heat dissipation, which means that by sensing the temperature of
the components a picture of the losses can be achieved. This measurement is very impre-
cise as diﬀerent components will have diﬀerent heat capacity, size and diﬀerent cooling.
This means that diﬀerent components with the same losses might operate with diﬀerent
temperature, but the thermal camera might still give a pointer for further investigation.

Inductor Loss
The result of this experiment can be seen in Figure C.1.1c and it shows
far higher temperature in the inductor compared to the constant duty cycle measurement.
This could mean that the relatively large damping seen in the step response could be
caused mainly by a lossy eﬀect in the inductor that only occurs when the duty cycle is
changed. Inductor losses are often split into two part the conduction loss and the magnetic
core loss. The conduction loss is caused by resistive eﬀects in the wiring of the inductor
coil. The conduction loss is included in the models in section 4.6. The magnetic core loss
in the inductor occur especially when the inductor is charged and discharged, but this is
normally low compared to the conduction loss. Another source of heating in the inductor
could be current ripple which is close to zero with 50% duty cycle, but exist when the
duty cycle is either 40% or 60%. This means that even though the inductor heats up more
with variable duty cycle it is not necessarily because of unexpected losses but could be
because of losses caused by ripple.
Switching Loss
Another observation based on Figure C.1 is that the switches have
diﬀerent temperatures even though they are the same components and should have the
same voltage stress when 50% duty cycle is applied. This is very unexpected and very
undesired as one of the main advantages of the ﬂying converter topology reduced voltage
stress on the individual transistor. The problem is assumed to be caused by the gate
driver system because large sudden changes in one switch might cause another switch to
temporally open because of parasitic capacitance between the drain and gate. To deal
with this problem and verify that this is indeed the reason the new veriﬁcation board is
made with an improved gate driver system.

Propertional Controller Design
The most common methods for design of PID-based controllers is to use heuristic methods,
that ﬁnds critical values based on open-loop step responses, and use them to calculate the
gains, which are reasonable for most systems. However, the voltage regulation problem
has an RHP zero. An article [24] concludes, that the Ziegler-Nichols method or other
heuristic methods are not a good for the design of a controller for boost converter as the
RHP zero causes the critical values to be hard to determine properly. Moreover, even with
a modiﬁed method for obtaining the critical values, the resulting controller should give a
less preferable result compared to other methods [24]. Therefore the problem is initially
approached with a gain and phase margin design method. The Bode plot for the transfer
function is shown in Figure D.0.1 using the reasonable component values described in
section 2.4.
