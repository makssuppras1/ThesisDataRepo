Abstract
A model for simulating nonlinear multidirectional waves was developed. The
model work on a periodic grid in order to emulate an open ocean.
The model is based on Boussinesq-type velocity formulation and fully non-
linear free surface equations.
Spatial derivatives within the model were computed using a spectral method
based on Fast Fourrier Transforms.
The simulations is times stepped trough a fourth order Runge-Kutte scheme.
The model had problems with instability, two causes was identiпђБed to be
the source of the instability.
One came from aliasing due to nonlinear terms in the governing equations.
This was solved by implementing OrszagвАЩs de-aliaing procedure, which sets
all velocity of high wavenumber waves to 0 in the nonlinear terms of the
governig equations.
The second cause for instability was due to inaccuracy of the iterative solv-
ing procedure used to solve the instaneous velocity пђБeld within each time
steps. This was corrected by implementing the Orzag procedure at lower
wavenumbers than the method normally requires.
The пђБnal model was tested with stream function solution waves and was
able to propagate a wave with a steepness of H/L=11% for 50 wave periods
without showing any sign of instability. A 12% was successfully propagated
for 49 periods, but at that time the simulation became inaccurate when
compared to a target solution.

Chapter 1
Introduction
The motivation behind the project was to make an improved model for the
calculation of multidirectional irregular waves.
The model should, when completed, be used for the simulation of highly-
nonlinear irregular waves. such a model should be able to predict some of
the extreme wave cases, which have been observed in North-Sea.
The theoretical background for the model was derived in [Madsen et al., 2002].
In here a Boussinesq-type formulation is derived, the Idea behind a Boussinesq-
type formulation is to eliminate the vertical coordinate while maintaining
an accurate representation of the physics.
[Madsen et al., 2002] derived three sets of velocity formulations. Originally
all formulations should have been implemented and tested, but in the пђБnal
thesis only one velocity formulations presented - the so-called One-step-5th
order polynomial.
The numerical solving procedure used for solving the equations presented
by [Madsen et al., 2002], is given in [Fuhrman et al., 2004]. This thesis fol-
lows this procedure closely, with the exemption of three things: The пђБnite-
diпђАerence schemes used for computing spatial derivatives is replaced with a
spectral method. A new de-aliasing procedure is implemented compared to
[Fuhrman et al., 2004]. And lastly, the model presented in this thesis works
on a periodic grid, which is a necessity when using the spectral method im-
plemented in the model.
A similar project, to the one presented in this thesis, have already been
presented at DTU [Andersen, 2016]. His thesis uses the same sets of equa-
tions and the same solving procedure.
The пђБnal model developed by [Andersen, 2016] was able to propagate waves
of up to H/L = 10%, but were unstable for waves of higher nonlinearity.
This project tries to individuate the cause of this instability in the solving

Scope of work
The actual of scope of this work has been to implement the Boussinesq-type
equation from [Madsen et al., 2002] and applying the solving procedure from
[Fuhrman et al., 2004] using spectral methods.
After the solving procedure have been implemented, causes for instability
should be individuated and solutions to these problems should be imple-
mented.
Once the model is done it should be used to propagate as steep a non-linear
waves as possible, without any instability in the model.
Structure of thesis
The individual chapters of the thesis covers:
Chapter 2: Derivation of the non-linear equation used in the model to de-
scribe the пђВow. The derivation starts from the governing equation of an
irrotational free surface пђВow, where the пђВuid is considered in-compressible
and inviscid .
Chapter 3: Presentation of the solving procedure for the equation and all
the numerical methods within this procedure.
The methods include:
Spectral diпђАerentiation for computing spatial derivatives.
Fourth order Runge-Kutta for time stepping the equations.
De-alias which is a procedure implemented to avoid that the model becomes
unstable because of aliasing.
Iterative solving which is the method used for computing the instantaneous
velocity пђБelds.
Special emphasis is put into the parts regarding de-aliasing and Iterative
solving. This is because these proved to be the cause of instabilities in the
preliminary tests of the model.
The chapter ends with a brief discussion of the methods introduced and
the complication with regards to instability introduced with them.
Chapter 4: All nonlinear parts of the model are switched oпђАand the linear

and comparing the simulations to known analysis of the equation governing
the model.
Chapter 5: The full nonlinear model is tested with nonlinear initial con-

The initial conditions tested are stream function solutions at varying steep-
ness. These cases provide an excellent test scenario because they are known
stable solutions to the equations, which are being time stepped by the model.
And so the wave elevations and velocities should remain stable over time.
Chapter 6: Conclusion from the report based on the tests performed. This
chapter recaps the performance of the model. Points out what has been
found to be the cause of instability in the model, and how these instabilities
can be controlled.

limitations of the numerical methods used. This is then followed by sugges-
tions to further work.

This chapter provided an overview of the numerical solving procedure used

The time stepping scheme used for the model is fourth order Runge-Kutta.

De-aliasing of the model is necessary because of the modelвАЩs nonlinear terms.

bers than low tolerances.

aпђАected on the initial wave.

solver by more than 1-3 mвИТ1. De-aliasing will, however, set all frequency
components of the velocities over the пђБlter value to 0, which must be as-
sumed to be better than overestimating them by many factors as was the
case or the unпђБltered case.
The main conclusion from the section regarding the iterative solver is: Use
as low a tolerance as possible. And check that the de-aliasing пђБlters kick in
before the gmres solution starts to overestimate the velocities.
Throughout this work a tolerance setting of e-14 will be used, as this is the
minimum practical value possible.

Conclusion on Nonlinear Test
The accuracy of ЋЬw after the пђБrst time-iteration have a clear eпђАect on long-
term simulations. If ЋЬw is overestimated the simulation will quickly become
unstable.
The de-aliasing procedure is correct as energy is not growing in the de-
aliased k-region. This goes for all the test cases, even the ones which are
unstable. As long as ЋЬw is only mildly inaccurate, all the instability remains
in the unпђБltered region.
The theoretical limit for the de-aliasing suggested by Orzag, which is nmin =
3, works when using nx=32 for simulations of waves up to steepness of 10%.
While the same limit does not work for any wave cases when using nx=64,
it is believed that this is due to the limitations imposed by gmres.
The reason that waves steeper than 10% are not simulated correctly by
the model using nx=32, is that at this number of discrete points too much
of the wave energy is computed linearly by the model.
Varying the tolerance of gmres and repeating a simulation for a wave of
H/L = %7 revealed that the rate of growth of the amplitude at high wave
number was independent of the tolerance setting.
This is an interesting
пђБnding but does not tell us that using too large tolerance is not causing the
instability. Most likely there exists a tolerance level that for a given wave

Chapter 6
Conclusion
Within this thesis, a numerical model solving Bousinesq-type equation of
free surface пђВow is developed.
The пђБnished model was able to accurately propagate nonlinear waves with
a steepness of up to 11%H/L.
For a wave of H/L = 12% the model propagated the wave for 49 periods
before it became inaccurate and the simulations were terminated. Is it not
known if this simulation could be carried on, and is in fact only inaccurate
and not unstable.
Steeper wave cases were not tested.
The developed model had problems with instability.
This thesis individuates two causes for instability in the model: Aliasing
and gmres Error.
The пђБrst instability cause, aliasing, occurs because nonlinear terms of the
model create oscillations in the пђВow, which is beyond the maximum resolv-
able wavenumber of the model.
This problem was solved by introducing OrszagвАЩs de-aliasing method. For
waves of low nonlinearity, for which we mean waves with a Height to length
ration of 7 вИТ10% the method removed all instability from the model.
The second cause for instability in the model was the error introduced by
the iterative solving gmres.
This was handled by applying the Orszag пђБlter at lower wavenumbers than
the method actually requires.
By applying the пђБlter at lower wavenumbers the error introduced by gmres,
would be controlled as all velocity amplitudes of waves over the пђБlter limit
would be set to 0. If the пђБlter was not implemented the solutions produces
by gmres, would exaggerate the amplitude of the high-frequency waves with
many factors and the simulation would blow up.

The number of tests on the пђБnished model was limited due to time limita-

Similar mod-

This could supply a good benchmark for
evaluating the performance of the model.
A test case should be devised to make it possible to make a clear to distin-

insuпђГcient accuracy on the gmres solution to ЋЬw or instability caused by пђБl-

This would point out where the model should be improved.
The model developed with its пђБlter was able to propagate a deep water wave
of H/L12% for 49 T, at this point the simulation became inaccurate. This
simulation should be carried on to see if the simulation is just inaccurate or
unstable.
Results shown in the report shows that an 11% can be propagated for 50
T by a model with 32 grid points. The simulation became inaccurate, but
they were stable. This supports the idea, that a H/L 12% can inaccurately
be propagated indeпђБnitely by a model using 64 grid points per wavelength.
Trough out this work the same пђБltering procedure is used for handling both
classical aliasing caused by non-linear terms in the model and the error in-
troduced by the gmres.

CHAPTER 7. DISCUSSION AND FUTURE WORK
It is believed by the author of this report, that there is no reason for apply-
ing the same пђБltering procedure.
Separate methods may be included for handling the two types of instabil-
ity. This could enhance the model making it able to propagate even steeper
waves.
Only step пђБlters with an all or nothing scheme have been tested, gradually
decreasing the amplitudes of the elevation and velocities at higher wavenum-
bers should also be tested.
This thesis has not focused on computational time or memory storage re-
quirements. As the model is intended to be used on large domains, this is
two key aspect, that has to be minimized in order to have a good working
model for practical purposes. If work is resumed with this model, an eпђАort
should be put into investigating how the model can be streamlined in order
to reduce the strain on a computational system.
With regards to time stepping. This thesis has only brieпђВy discussed the
stability limit for the linear model under the assumption that the spectral
derivatives were exact representations of the actual derivatives.
A thorough stability-analysis of the complete non-linear system including
the implications which arises from the use of spectral derivatives should be
performed.
[Trefethen, 1996] have in his unpublished book given some limits to the max-
imum time step allowed with respect to the grid spacing used, when using
spectral derivatives.
His limits are for other time-stepping schemes than the one used in this the-
sis and for much simpler systems. But they suggest much lower time-step
limits and should, therefore, be studied.
Implementation of other velocity пђБelds. A two-step method is brieпђВy tested
within one of the appendices of this report. Only the solution to the in-
stantaneous velocity пђБeld is tested. The results show that with regards to
estimating the vertical velocity at the surface, the two-step method is better
than the one-step method.
A full implementation of the two-step method should be made in the model.
This should be done using the same de-aliasing procedure as implemented
on the one-step method.
Changing the velocity пђБelds so iterative solving is only used from the still
water level to the free surface is expected to improve the models accuracy.
This would mean that higher k-values could be used as пђБltering limit. Which
then should make it possible to propagate waves of higher steepness.
