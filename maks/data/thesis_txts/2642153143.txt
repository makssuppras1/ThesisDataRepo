Abstract
There is a rapidly increasing demand for smart cameras in
the industry, however the available devices are too expensive,
have closed source software running inside and do not contain
deep learning capabilities in most of the cases.
The scope of this project is to experiment with various state-
of-the-art deep neural networks and see their inference capa-
bility, deployed on diﬀerent embedded devices.
The goal is to ﬁnd an eﬃcient network-inference engine-
hardware combination that can achieve a suﬃciently fast and
accurate object detection on multiple custom datasets with
transfer learning. This would pave the way towards edge com-
puting custom smart cameras with extended machine learning
abilities, focused on object detection.

Introduction
The ﬁrst chapter gives a general introduction to the topic of this thesis. It reviews the application of a
smart camera and formulates the motivation to consider alternative devices. This leads to the problem
statement together with the requirements and limitations of the project. Finally, the structure of the
whole report is presented.
1.1
Smart cameras
We are living in the world of Industry 4.0 which is a digital transformation of industrial processes
to create ”smart factories”. These autonomous systems function by utilizing data, machine learning,
and the Internet of Things (IoT). They often employ the power and scalability of cloud computing
platforms. However, sending a large amount of data to the cloud can become problematic regarding
speed, bandwidth, security, and cost of transmission or storage. This brings edge computing into the
picture. An edge agent is a solution for the aforementioned problems that hosts and controls local
tasks autonomously [10].
Computer vision is an important ﬁeld within the industry, requiring the use of such systems. Thus,
it often relies on smart cameras to perform part inspection and gauging or robot manipulation. These
smart cameras are embedded devices with a built-in camera sensor and a small computer capable of
running the manufacturer image library. They sense the environment in real-time without the need for
cloud computation. Therefore, they are considered edge computing platforms. However, the available
devices have diﬀerent challenges when it comes to a company or engineer using them as a part of their
system.
1. The major problem is the lack of generic deep learning capabilities. The ﬁrst models that
include neural networks appeared in April 2020 [11].
2. The second diﬃculty is the closed software structure. It is impossible at the moment to
deploy custom deep neural models or to extend, boost the performance beyond the provided
default features.
3. Finally, the third issue is the price. As an example, a Cognex camera with conventional com-
puter vision capabilities costs 75.000 DKK that is around 10 times overhead for a camera without
deep learning and with closed-source software (Cognex is a leading provider of vision systems).
1.2
Motivation
The above challenges increase the motivation to make deep learning available on low-cost embedded
devices and provide an alternative for smart cameras in real-time industrial computer vision. For this
thesis, object detection is chosen from the computer vision domain because it has numerous industrial
applications and academic research to provide a diverse and state-of-the-art baseline. Furthermore, it
requires complex and intensive computations.
UN goals
The United Nations Sustainable Development Goals (SDG) add an extra inspiration for the project.
Goal eight deﬁnes the need for decent work for all. Decent work requires safety, which can be
provided in various forms with the help of object detection systems running on embedded devices.
Goal nine raises the voice for sustainable industrialization. Developing better and more reliable
edge devices transforms applications to run locally and save energy instead of sending the computations
to the cloud servers.

Requirements
An embedded deep learning system has multiple attributes to consider:
• Price: It is often required to install multiple clones of the detection system that leads to the
need for low-price hardware.
• Speed: The system has to work close to real-time in various scenarios, therefore the inference
time, measured in frame per second (FPS), is also important.
• Accuracy: There is a diﬀerent need in terms of recall and precision depending on speciﬁc use
cases. The more domain-speciﬁc metric in object detection is the average precision (AP), which
combines these two. AP is always an important factor in industrial object detection applications.
• Reliability: If an industrial machine stops, it causes a large extra cost for the company that
runs it. Therefore, the system has to be robust and fault-tolerant.
It is important to note that regarding speed, the focus of the thesis is only the inference time on
embedded hardware. This means that the training occurs on either a desktop or in a cloud-based
computing platform. The procedure speeds up by utilizing GPU resources in both cases. Similar to
the training, diﬀerent optimization strategies are implemented on the aforementioned platforms. Only
the converted and optimized models are used on the embedded devices to run the inference on the
deployed networks.

2.1
Deep learning in computer vision
Before going into more details about object detection, it is essential to understand the basic concept
of deep learning and how it is utilized within the computer vision domain.
2.1.1
Deep Learning
The original intention of deep learning was to recreate the human brain and complete diﬀerent cal-
culations copying the neural connections. Deep learning is a subﬁeld within the domain of artiﬁcial
intelligence (AI) and machine learning (ML). AI is a general term for any technique that enables
machines to mimic human behavior. This means a computer that can sense, adapt, reason, and solve
problems similar to the brain’s cognitive functions.
ML is a subﬁeld of AI and it is dated back around four decades from today.
The basic idea
in ML is to apply statistical algorithms. It takes data and learns from it by recognizing patterns
and making predictions from them. It has various methods that can be grouped into three major
categories: supervised, unsupervised, and reinforcement learning. When a task is supervised, the
algorithm tries to predict quantities based on another set of quantities. It needs labeled data that
provides direct feedback. The most important supervised techniques are [9]:
• classiﬁcation
• regression
Unsupervised learning means the lack of ”ground-truth” labeled data, therefore the model needs to
learn from hidden structures. The most important unsupervised techniques are [9]:
• clustering
• density estimation
• anomaly detection
• dimensionality reduction
• association mining
The third big class is reinforcement learning. The core idea is to have an agent controlled by a
computer. The decisions are made based on feedback information from the environment and by a
reward system. One decision process consists of the following steps: agent observes the state of the
world; agent chooses an action; agent receives a reward. The goal is to learn from series of actions to
maximize the rewards over time [9].
There are certain tasks that the previously mentioned machine learning models cannot solve. Deep
learning is a subset of ML to reach a higher accuracy and solve complex problems with artiﬁcial neural
networks (ANN). The learning is done by optimizing the weights at each layer. Deep learning does
not need the feature extraction preprocessing step as it is able to learn it throughout the training.
The phrase end-to-end is used to describe this behavior. It means that the model can predict the
output directly from the input. The power of neural networks can mainly be utilized if a large dataset
is present to learn from. For a smaller amount of training data, traditional machine learning is often

Neural networks can be divided into two types: feedforward networks and recurrent networks
[12]. The ﬁrst type is called feedforward because there is no feedback connection, in opposition to
the second, where the output of one layer can be fed back to itself or previous layers. In the case
of a feedforward network, the output is calculated by forwarding the input through all layers. At
each layer, the signal to the next layer is calculated as a linear combination of the input and weights
together with the bias nodes, furthermore, an activation function is applied on the result of the weight
calculation to introduce nonlinearity in the system. The so-called gradient descent is used to learn
the parameters and biases. They are updated by back-propagating the error through the network.
CNNs
Convolutional Neural Network (CNN) is used for processing data that has a grid-like topology, such as
images or time-series data. The name comes from the linear mathematical operation called convolution
[12]. An example of a CNN can be seen in Figure 3, the network is discussed in more detail in Section
2.3. CNNs have a hierarchical structure and are able to learn a diﬀerent level of abstraction hidden

in the data. The convolution is always done between an input feature map from the previous layer

and the convolutional kernel that is also called weight or ﬁlter. An activation function is used after
the convolution to add nonlinearity, similarly to the feedforward networks. These two operations are
represented with purple in Figure 3. The blue color indicates another important operation called
pooling, that does the upsampling or downsampling of the feature map. The downsampling and

Convolutional layers are powerful in extracting hidden information, which is done in an unsuper-
vised manner. However, in almost every application, it is combined with a fully connected layer at the
end of the network to introduce the supervised aspect of the learning [2]. There is usually a special
activation function at the end of the network that converts the ﬁnal features of the network into a
meaningful output. In the case of image-related networks, this layer is usually a softmax function that
predicts a normalized probability for each of the available classes that the dataset contains.
The revolution of CNNs has come for two main reasons: First, the better availability of Graph-
ics Processing Units (GPUs), providing the high computational needs of the network. Second, the
increasing number and size of labeled datasets that can be used for training.
2.1.2
Computer vision applications
Computer vision is one of the ﬁelds that can take advantage of the utilization of CNNs since the
fundamental goal is to extract useful information from images or videos.
The major recognition
problems can be observed in Figure 4 and grouped as the following [3]:
• Classiﬁcation: predicts an object category for a given image.
• Object detection: predicts the location of the objects represented by bounding boxes on top
of classifying them.
• Semantic segmentation: predicts the classes on a pixel level, therefore each pixel is assigned
to a speciﬁc category label (which can also be the background).
• Instance segmentation: it combines object detection and semantic segmentation, therefore
it localizes each object independently on the pixel level, even if a category is represented by
multiple instances on the image.
Classiﬁcation with localization used to be distinguished as an independent ﬁeld before object
detection networks became popular. The deﬁciency, compared to object detection, is that it can only
localize one class per image [13]. It is also worth to mention object tracking as an independent
problem today. Many of the tracking networks use the knowledge from object detection networks.
However, this extends the process for a series of images to continuously track the initially captured
object as its location is changed from frame to frame in the video [14].
The list of deep learning applications in computer vision is continuously extending with more
and newer interesting research ﬁelds, such as style transfer, which takes the semantic content of an
image and renders it in diﬀerent styles that are taken from another image [15]. Colorization converts
gray-scale images into a colored version using three channels [16]. Image reconstruction means the
restoration of old photos or videos [17]. Super-resolution is a technique to recover ﬁner details on
an image by upscaling it [18]. There are also great eﬀorts made in the synthetic content generation
using general adversarial networks [19].
From the listed applications, this thesis investigates object detection which is explained in more
detail in Section 10.

2.2.1
Mobile networks and edge devices
In Section 1.1, some of the relevant phrases are introduced, such as edge devices or IoT. Edge devices
will be continued to be discussed regarding computer vision tasks.
It is important to distinguish training and inference with respect to deep learning applications.
Training neural networks requires high computational power, but usually it has to be done once, or
in some cases where the data is updated continuously it needs to be done on a reoccuring basis. On
the other hand, inference is a lower compute-intensive process that has to be executed repeatedly at
a high frequency. Thus, for training it is common to utilize the beneﬁts of cloud computing, however,
for inference, switching to local devices has emerged as an alternative, thanks to the improvement in
hardware eﬃciency.
Edge devices are made of microcontrollers and sensors. They are the fundament of IoT that are
usually connected to a network. Gordon E. Moore noted in 1965, that “the number of transistors in
a dense integrated circuit doubles approximately every two years”. This prediction was so accurate
that it became Moore’s law. This phenomenon of increasing computational capacity with decreasing
device-size and low power consumption is the driving force of edge devices1. An additional beneﬁt is
the reduced price from the industrial perspective.
The computationally stronger, cheaper, and smaller edge devices took the attention of the machine
learning community, and the so-called Tiny Machine Learning (TinyML) became a research area.
The goal is to create fast and eﬃcient models that can solve tasks on embedded devices. However, there
is a trade-oﬀbetween the speed and accuracy of deep learning networks that makes it a challenging
task to develop networks that are designed for edge devices.

The second important architecture is the Graphics Processing Unit (GPU). It is able to ac-

Application-Speciﬁc Integrated Circuit (ASIC) is the most eﬃcient device as it has a custom

a real market need.
It is important to mention a special ASIC, the Tensor Processing Unit (TPU), which is
developed by Google.
It is optimized to accelerate inference on neural networks with low energy
consumption. It uses matrix multiplication units to achieve a 15 to 30 times faster speed than a
contemporary GPU or CPU, based on their research paper [21]. First, it was deployed in data centers
where lower energy consumption has a great priority because of cost and environmental perspectives.
Another interesting device is the Digital Signal Processor (DSP) that is developed to process
sequential data such as audio or video. It is similar to FPGAs regarding the signal processing aspect.
However, the DSP is a microcontroller whereas the FPGA is a re-programmable silicon chip. DSP
can solve more complex tasks, however, it has a lower sampling rate than FPGA. Based on these
attributes, it could be placed between GPU and FPGAs in Figure 5.

Structure
First, the structure of deep learning object detectors is discussed because it makes the understanding of
the historical evolution easier. Figure 7 shows all relevant detector parts. It is taken from the YOLOv4
paper [4] because it is one of the latest published papers. Therefore, it is capable of representing the
state-of-the-art knowledge of a detector network structure.
Figure 7: Object detector structure [4]
As it is shown in Figure 7, the important object detection network subparts can be deﬁned as the
following three blocks:
1. Backbone: First, the image tensor goes through the backbone that is basically an image classi-
ﬁcation CNN that extracts features on diﬀerent depth.
2. Neck: The neck is an intermediate block in the network that collects and shares feature maps
between diﬀerent stages of the network. It can be a bi-directional information ﬂow.
3. Head: The head is responsible for the ﬁnal prediction. It can be done in two ways [3].
(a) Sparse prediction has two steps: ﬁrst, using a proposal generator to create a sparse set
of proposals and extracting features from each proposal; second, classifying the proposed
regions by predicting the category of each of them.
(b) Dense prediction is introduced later and is used in the one-stage detectors which is a one-
step process. It makes the categorical prediction from the feature maps.
The backbone of a detection network is a crucial part, hence the history of object detection starts
with the evolution of classiﬁcation networks in the next section. After that, it reviews the important
two-stage, then one-stage detectors. Mobile detectors are particularly important within the one-stage
detectors’ group for this thesis. Therefore, there is an additional historical introduction that focuses
on them at the end of the next section.

Traditional methods
The need for detecting objects in images had already been presented before the deep neural networks
appeared. These traditional approaches mostly depend on the sliding window technique. It examines
the entire image by taking subareas, then it looks for hand-crafted features inside the window and
classiﬁes them according to the extracted features. One of the challenges is the required prior knowl-
edge about the features to be extracted. These methods are relevant because some important features
that have already appeared here will be essential later on for the deep learning models. In the next
bullet points, the most important traditional methods are presented.
Viola-Jones Detector: This is one of the earliest successful methods and it uses the Haar
features (that consists of dark and light regions) accelerated by the technique of Integral Images
(intermediate representation of an image by summing up given pixel neighbors) then it carries out
the classiﬁcation by diﬀerent cascaded classiﬁers (i.e. multi-stage classiﬁers) that have been trained
by Adaptive Boosting. It was created for facial recognition by P.Viola and M.Jones in 2001. [26]
Histograms of Oriented Gradient (HOG) detectors: HOG was developed to detect humans by
using features from locally normalized histograms of gradient orientations. It is similar to the Scale-
Invariant Feature Transform (SIFT) descriptor, used in an overlapping grid arrangement to detect
persons. Compared to the previous Haar feature-based method, the false positive rate is decreased
by a factor of ten. For the classiﬁcation, it also uses linear Supported Vector Machine (SVM). The
method was published by N.Dalal and B.Triggs in 2005.
[27] Deformable Part-based Model
detectors (DPM): The next milestone was achieved by the DPM detector, which was proposed by
P.Felzenszwalb in 2007 [28] and ﬁne-tuned later by R.Girshick. The method was the VOC winner in
object detection for three continuous years between 2007 and 2009. The key of the DPM technique is
based on the part-based model that investigates the pictorial structure and combines it with machine
learning and modern features. Thus, the algorithm learns to decompose an object into diﬀerent sub
parts, connected by spring-like links, and compose it back during inference. Firstly, Felzenszwalb
introduced the ”star model” and later Girshick developed the ”mixture model” on top of that as the
decomposition technique which can deal with a larger variety of objects. A special ﬁltering enables
modeling the visual appearance at diﬀerent scales where the subparts are learned by part ﬁlters
(represented as latent variables).
Girshick introduced the phenomenon of ”hard negative mining”, which helps to reduce false posi-
tives, furthermore ”bounding box regression”, that represents the prediction of the localized boxes.
It can already be observed, that machine learning and various important techniques were already
in use even before the deep learning-based methods (i.e., adaptive boosting, SVMs, hard negative
mining, bounding box regression).

Deep neural models
Unlike the hand-crafted features from the previously described traditional methods, convolutional
neural networks can learn more general and high-level features from the images. This new technology
has started to revolutionize computer vision areas. The ﬁrst breakthrough was accomplished in clas-
siﬁcation. The object localization and detection needed approximately a year to develop meaningful
models.
Figure 10 can be seen as the zoomed version of the blue area from Figure 9, therefore the deep
learning era with a more comprehensive glance.

Classiﬁcation
First, the major achievements in classiﬁcation are discussed.
Other than the chronological order,
it is particularly important to investigate prior to object detection because the feature extraction
from classiﬁcation neural networks creates the backbone of the detection models. The classiﬁcation
milestones are indicated at the bottom range in Figure 10 with brown color.
The aforementioned AlexNet was trained and designed by a research group at University of
Toronto, led by Alex Krizhevsky [29]. The ﬁrst version of their work was made for the ImageNet
classiﬁcation contest in 2010. They won the challenge in 2012 with a further developed variant of
their original architecture. Compared to the later classiﬁcation models, it is considered as a simple
structure with ﬁve convolutional layers with ReLU activation and three fully connected layers. Yet, it
managed to outperform the previous years’ results by a high margin. The next year, in 2013, ZFNet
won the contest as a further developed variant of AlexNet, developed by M.Zeiler and R.Fergus [30].
The major diﬀerence is the reduction in ﬁlter size to capture more pixel information. Furthermore,
they introduced a novel way to make the visualization of convolutional networks via DeConv Net.
From Figure 10, it can be observed that the next year (2014) brings two networks on the time line.
The winner network of the ImageNet challenge, out of these two, is GoogLeNet from C.Szegedy and
W.Liu[31]. The major achievement of their architecture is the so-called Inception Module which is a
repeated subpart of the network. It has multiple kernel ﬁlter sizes operating at the same level, instead
of the usual pure sequential stacking. The other network—that has not won the competition, however,
managed to create an even more enhanced variant of AlexNet—is called VGG which is the abbrevia-
tion of the developing team called Visual Geometry Group led by K.Simonyan and A.Zisserman from
the University of Oxford. They further reduced the ﬁlter size to 3x3 (11x11 - AlexNet, 7x7 - ZFNet)
to achieve faster training with fewer hyper-parameters.
The next big milestone is ResNet from the Microsoft Research team that has won the challenge
in 2015. The name comes from the residual network which contains residual blocks to introduce a new

From 2018, the traditional ILSVRC challenge has stopped. The next major milestone—that is
considered as one of the most state-of-the-art classiﬁcation networks up until today—is EﬃcientNet,

a new compound scaling method that describes how the the previously described scaling dimensions
should correlate. This eﬀective scaling makes EﬃcientNet a very strong backbone of later object detec-

(or EﬃcientNet-L2) is third with 88.4% [35] and FixEﬃcientNet-L2 is second with 88.5% [36].
Two-stage detectors
The next review steps back in time again and goes through the history and milestones of two-stage
detectors. It is the middle zone in Figure 9, colored by green. The naming is originated because of
the two stages that are:
1. proposal generation, so ﬁnding the areas on the image that might contain objects and
2. prediction, which is the classiﬁcation of the proposed areas of potential objects.
As AlexNet brought feature extraction to a new level, object detection networks also started
to evolve at an unexpected speed.
The ﬁrst detection network that is the starting point of deep
learning-based object detection is R-CNN, therefore Regional-based Convolutional Neural Network
developed by R.Girshick (whose name has already come up during the discussion of the ﬁne-tuned
HOG detector related to the traditional methods) [37]. The novel network relies on three major points.
First it generates the region proposals with selective search. Secondly, it extracts features. Finally, it
classiﬁes the areas using the extracted features.
15
==PAGE:25==
s182596 - Bertalan Kovacs
The next important network is SPPNet, Spatial Pyramid Pooling Network from K.He [38]. Its
major improvement is the SPP layer that is inserted before the fully connected layer. It enables the
network to take any image size at the input, yet compute a ﬁxed-length representation from it. Before
introducing this extra layer, the overlapping proposals led to a long computational time because most
of the features were computed several times. By having the SPP layer before the fully connected layer,
the repeated computations are not needed anymore thanks to the ﬁxed length. This yields up to 20
times faster inference time while keeping the same accuracy.
After speeding up the detection time, the next network aimed to cut oﬀon the multistage training
process as well as to keep increasing the accuracy of the network. To accomplish these goals, R.Girshick
introduced Fast RCNN in 2015 [39] where the detector and the bounding box regressor can be trained
at the same time. It is the ﬁrst end-to-end training pipeline. Another change, compared to the SSP
layer, is the Region of Interest (ROI) pooling layer before the fully connected layers. This is similar
to the SPP layer, except that it takes a single scale and it divides the feature vector into a ﬁxed size
grid of subwindows. Therefore, it can be considered as a single level of the SPP pyramid.
The most common and well-known two-stage model was introduced in 2016 by S.Ren as Faster
RCNN [40]. The major addition compared to Fast RCNN is the Region Proposal Network (RPN)
as part of the detection pipeline.
RPN is inspired by MultiBox (which will be discussed as the
forerunner of one-stage detectors) and it follows the sliding window approach to accomplish bounding
box regression and box conﬁdence scoring by a neural network instead of using the traditional methods
such as Selective Search.
An important improvement—that is not a detector network itself, however, it boosts the perfor-
mance of detectors—is the Feature Pyramid Network (FPN) introduced by T-Y.Lin in 2017 [41].
Before FPN, the detection has been done purely in the head, however important features can also be
extracted from deeper layers to help the detection decision. Therefore, T-Y.Lin builds a top-down
architecture with lateral connections to strengthen the semantic information from deeper layers with
spatially strong features from shallow layers.
Since 2017, the scope within two-stage detectors stayed at ﬁne-tuning the existing base models or
shifted towards segmentation problems. In segmentation, the most important architectures are Mask
R-CNN or Mask Scoring R-CNN, however, this computer vision domain is not the scope of the thesis.
One-stage detectors
The next and most related ﬁeld to this thesis, within the object detection domain, is the one-stage
detection algorithm that is represented by the upper zone in Figure 10 with the color red. As the
name implies, the two steps—that can be seen in the previous two-stage detectors—are reduced to
one. Thus, the one-stage detectors are classifying each region of interest and label them as either
objects or background with a uniﬁed detection pipeline.
Before the attention of the researchers turned towards the one-stage approach from the two-stage
method, the aforementioned OverFeat had already been implemented by Sermanet in 2013 [42] even
before the RNN has been introduced. OverFeat is a fully convolutional network with a single forward
pass. This enabled a fast inference, however, due to the lack of advanced hardware, the training took
a large amount of time which led to a less accurate system than RCNN at that time because of the
lack of powerful hardware.
Few years later, in 2016, J.Redmon publishes his ﬁrst famous model and paper: the ﬁrst You Only
Look Once YOLO network [43]. This is the starting point of one of the most famous object detection
network family in terms of speed and real-time inference. Following the one-stage paradigm, it divides
the image into regions and predicts the bounding boxes together with the classes at the same time.
Even though it achieved a much faster running time, the accuracy stayed much lower compared to

the two-stage models, especially regarding smaller objects. He does the implementation on Darknet
which is a C and Compute Uniﬁed Device Architecture (CUDA) based open source neural network
framework.
The next interesting milestone comes with SSD, the Single Shot Multi-Box Detector from W.Liu
in 2016 [44]. Similarly to YOLO, it also aims for fast and real-time detection, however it also develops
new techniques to capture and recognize smaller objects by using ”multireference and multiresolution
detection”. It also uses the RPN from Faster R-CNN to classify objects inside the prior boxes by a
convolutional network. It generates the anchors with multiple aspect ratios and scales on the image
grid cells, unlike the YOLO which sets anchor priors manually.
The next fascinating feature comes with RetinaNet [45]. T.-Y.Lin discovers that the two-stage
detectors’ accuracy could be approached.
His main idea is that the high foreground-background
imbalance is the source of many objects to be missed. Therefore, he introduces a new loss called focal
loss which is a modiﬁed version of the cross entropy loss which was used before. This new loss function
resolves the class imbalance problem.
After a year of further research, Redmon publishes an enhanced version of the YOLO network
called YOLOv2 [46]. It uses batch normalization and takes the improvement from SSD regarding
the anchor priors, supplemented with k-means clustering from the training data. Redmon also creates
another version (i.e., YOLO9000) for a more general dataset with 9000 classes.
A year later, he
publishes his last paper on YOLOv3 [47] which achieves better results regarding both speed and
accuracy than any other network developed up to this point of time. It has been used for two years
all around the industry, due to its great performance. Two years later, after publishing his model,
J.Redmon has decided to stop his work in February 2020 within the object detection ﬁeld with the
following tweet message: ”I stopped doing CV research because I saw the impact my work was having.
I loved the work but the military applications and privacy concerns eventually became impossible to
ignore.” [CV: computer vision]6
The next network opens up a new perspective of one-stage models by questioning the dominant
role of the anchor box. H.Law proposes CornerNet as the ﬁrst anchor-free object detection network
in 2018 [48]. Anchor based methods try to ﬁt an object inside the anchor, while the proposed anchor-
free method wants to predict the key points of the bounding box. The name of the network comes
from the fact that it detects objects as a pair of corners. It increases the accuracy compared to the
previous models, however it also reduces the speed. CenterNet is an improved version of CornerNet
by K.Duan. He tries to ﬁx incorrect bounding boxes with a triplet of keypoints instead of pairs by
adding an extra one in the center [49]. It further increases both the accuracy and inference time,
questioning the eﬃciency of anchor-free techniques.
The timeline arrives within the last one year. The next large milestone is EﬃcientDet from
Google Brain [50], which became open-source in March 2020. It is based on EﬃcientNet which was
introduced in the classiﬁcation section. Besides the novel scaling method, EﬃcientDet introduces the
weighted bidirectional feature pyramid network (BiFPN) that allows feature fusion to ﬂow backward
and forward as well. It becomes the backbone of leader networks of object detection challenges until
today. Currently, pretrained EﬃcientDet networks can be downloaded in seven diﬀerent sizes from
D0 to D7.
The next very recent network is YOLOv4 from Alexey Bochkovskiy, who released his work in
April 2020 as a continuation of Redmond’s work [4]. He keeps using the Darknet framework by forking
the original YOLOv3 repository. He implements the state-of-the-art bag of freebies (BoF) and the
bag of specials (BoS). BoF is increasing the accuracy by keeping the same inference. It can be realized
by diﬀerent data augmentation strategies to diversify and maximize the information of the limited

Evaluation metrics
To validate the performance of the algorithms on any of the described datasets, it is crucial to establish
a standardized way of comparing their results. While for classiﬁcation it is more clear how to interpret
the output classes compared to the ground truth, in object detection the evaluation procedure gets
more complex.
First, the terms that are used to evaluate the performance of classiﬁers are summarized because
the basic terms in the object detection metrics are the same. To begin with, the confusion matrix is
presented which corresponds to the model’s correct and incorrect predictions. It is shown in Figure
17 for classiﬁcation problems. It is possible to derive further performance indicators from these four

sponding confusion matrix [9]
measures (TP, FN, FP and TN). For classiﬁcation, precision and recall are usually used to evaluate
the classiﬁer. They are deﬁned as the following equations:

Precision =

Recall is also called as true positive rate or sensitivity because it shows how well the model predicts

relevant items. Precision is sometimes called positive predictive value as it shows how well the model

trivial labeling of every prediction as positive would result a recall of 1.0, however it would reduce
the precision. There are further combinations of these metrics that are not discussed i.e. F1 score,
accuracy, error rate, false positive rate and true positive rate, etc. [9].
Intersection over Union
From the perspective of object detection evaluation, the next important term is the Intersection over
Union (IoU). It converts the bounding box location prediction into a binary classiﬁcation problem for
a given object with the help of a chosen threshold that is usually deﬁned as Ω. Figure 18 shows an
IoU calculation which resulted IoU = 0.52 in this example. This means that for a strict threshold e.g.
Ω= 0.9 it would be considered as FP, however in case of less strict lower threshold e.g. Ω= 0.5 it
would be accepted as a TP.

Average Precision
After precision and recall are deﬁned in the object detection context, the next important term is the
Average Precision (AP). AP is calculated independently for each object class from the precision-
recall curve. The process of drawing the curve is the following [2]:
1. There is given: a class, an Ωthreshold value for the IoU, number of images with annotations,
furthermore the predicted bounding boxes of the speciﬁed class.
2. The predictions are sorted by the conﬁdences from the detector’s classiﬁer for a given bounding
box.
3. The IoU Ωdetermines if the prediction is a TP or FP, moreover the FN can also be calculated
from the known labels that are not the predicted TP. (The TN for object detection is not a
useful metrics as it would mean background predictions from the detector.)
4. Recall and precision can now be calculated with TP, FP and FN (TN = 0) which gives the ﬁrst
point on the curve.
5. The third and fourth points are repeated until all the predictions are evaluated. It results in a
jagged curve after the calculated points are connected (the precision can move in both directions
while the recall increases monotonically).
AP is deﬁned as the area-under-curve (AUC). For easier computation, AUC is calculated from the
smoothed curve instead of the original zigzag pattern. Graphically, at each recall level, the precision
value is replaced by the maximum precision value for any higher recall level. The interpolated precision
value in a mathematical form follows as

The currently used challenge metric is usually the so-called COCO metric that can be written
as mAP, COCO mAP or mAP@.5:.05:.95. It uses ten IoU values from Ω= 0.5 to Ω= 0.95,

There is also a variant with Ω= 0.75 that is called strict metric and noted as
mAPIoU=.75 or mAP@.75.
The COCO dataset contains various object sizes, therefore another measure is introduced to eval-
uate the performance of a network regarding the size of the detected objects. They deﬁne APsmall
for objects that have area under 322 pixels, APmedium for objects that have area between 322 and
962 pixels and APlarge for objects that have area above 322 pixels.

2.4
Optimization techniques
It is demonstrated throughout the previous sections, that there is an important trade-oﬀbetween the
eﬃciency and eﬀectiveness of a network. While academic research prioritizes the accuracy, the industry
also gives a high importance to the speed. Diﬀerent applications have diﬀerent limitations based on
the hardware, framework, or the requirements that it needs to fulﬁll. Instead of designing a new
network architecture, another way to change the model performance is optimizing the already trained
network or slightly modifying the network structure. Edge devices are a usual example for having
limited computational power and memory, these steps can improve the performance signiﬁcantly.
Optimization techniques have several beneﬁts, usually to the detriment of a small accuracy drop.
It can decrease the inference time by reducing the computations by some means. This can also
decrease the power consumption and reduce the model size, which is an important factor for, e.g.,
mobile phones where the users have to download the application into a device with limited memory.
Another advantage is hardware compatibility, which means that some hardware can only run
speciﬁc computations requiring a particular model format in addition.
The four DNN optimization techniques, based on [65], are:
1. Parameter pruning and quantization
2. Low-rank factorization
3. Transferred convolutional ﬁlters
4. Knowledge distillation
All methods are applicable for both convolutional layers and fully connected layers as well, despite
transferred convolutional ﬁlters that can only be applied in the convolutional layers.
Pruning and quantization methods optimize the neural network by exploring the redundant
parameters that are not sensitive to the performance and try to remove them. Low-rank factoriza-
tion approaches the optimization problem from a matrix point of view. It uses matrix decomposition
to determine which are the informative parameters of the DNN. Transferred convolutional ﬁlter
is a special way of doing convolution and it reduces the parameter space. Finally, knowledge distil-
lation is the method of creating a distilled network that is more compact, yet can reproduce the same
output that the larger network gives. It is also called teacher-student training scheme, since there is
a heavy and powerful network guiding a lightweight and optimized one [3].
Challenges
All methods have certain disadvantages. Quantization can reduce the accuracy to a large extent,
especially binarization, which is the most extreme variant of network quantization. Pruning can be a
challenging method as it requires manual setup and ﬁne-tuning that makes it cumbersome for many
applications. In addition, it is mainly used to decrease the model size, however, it usually cannot im-
prove the eﬃciency. Low-rank approximation is a complex and computationally expensive approach
that also requires retraining the whole model. Transferred convolutional ﬁlter has a competitive per-
formance in ﬂat networks, however it does not perform suﬃciently in deep-layer structures. Knowledge
distillation can only be used with softmax as a loss function, furthermore it has a weaker performance
compared to any of the previous methods.
These methods are orthogonal, therefore it is possible to combine them to maximize the gain [65].
In the ﬁeld of object detection, one could use the low-rank based method to compress the CNN-based
backbone and pruning to optimize the fully connected layer of the head.
2.5
Transfer learning
It was presented in the previous sections (2.3.3, 2.3.4) that object detection networks are always trained
and tested on a given evaluation set with well-deﬁned metrics. This is currently the MS-COCO, and
recently the Open Image datasets with COCO mAP metrics. However, in real-life and in industrial
application, the network often has to recognize new classes from new data that requires fresh training.
25
==PAGE:35==
s182596 - Bertalan Kovacs
Training object detection networks is a complex area. Firstly, the state-of-the-art models are designed
for large datasets, therefore retraining them on smaller sets can easily yield to poor performance due
to a quick overﬁtting of the weights. Furthermore, the training requires high computations and careful
choice of hyper-parameters.
Transfer learning is a great method of overcoming these presented challenges. In general, it is
applicable when the network has to perform diﬀerent tasks, where many of the descriptive factors in
one problem are relevant to the factors that have to be learned in the second, third, etc. problem [12]
(Chapter 15.2). In object detection, it means ”transferring” the learned data representation from a
large dataset to a smaller, diﬀerent dataset by generalizing the shared features. These features are
usually low-level representations (i.e., edges, visual shapes, geometric changes, or changes in lighting)
[12]. Two extreme forms of transfer learning are one-shot learning and zero-shot learning. The former
occurs when there is only one labeled example given while the latter has no labeled example at all.
Zero-shot learning is only possible because of the additional information that has been extracted during
training. It is a popular research ﬁeld within machine perception and within all major computer vision
domains (classiﬁcation, object detection, segmentation).

The speed and accuracy requirements have a dynamic deﬁnition in this thesis. One of the major

has suﬃcient support from the community or the providers. It makes it possible to use the research and
development time to spend on numerous models. This is particularly substantial regarding the frame-

Hardware
The hardware, framework and inference engine selection has less priority in this thesis than choosing

4.1.2
Conversion to TensorFlow Lite
Obtaining the inference graph
The previous test is made with TensorFlow. The next challenge is how to get working TFLite models
from the TensorFlow frozen graph representation. The YOLO models have an additional complexity.
The Tiny-YOLO v4 weights are in the Darknet structure, furthermore the YOLO v5s model is trained
in PyTorch. Figure 24 is created to visualize the pipeline of the diﬀerent conversion processes.
Figure 24: TFLite conversion pipeline
In general, pretrained models have two important attributes:
1. Structural deﬁnition of the model: It deﬁnes the diﬀerent layers and activation functions for the
head and the backbone. Additionally, it speciﬁes the number of classes that are crucial for the
last layer to make predictions.
2. Weights: It stores the weight and bias values of the last training.
The two required items can be considered as a skeleton (structural deﬁnition) supplemented by the
weight data. It can also contain further information about the training process.
In the Darknet framework, there is only one ﬁle (.weights format) including both the aforemen-
tioned information. In PyTorch, the training process outputs two ﬁles: A conﬁguration ﬁle (.yaml
format), furthermore a weight ﬁle (.pt extension). TensorFlow has two ways of representing the model.
The ﬁrst is similar to the PyTorch structure. There is a graph deﬁnition ”protobuf” ﬁle (.pb or .pbtxt
format) containing the network structure as well as the description of the computations needed for the
training. In addition, there is a checkpoint ﬁle (.ckpt) capturing the exact value of all parameters
used by the model55. The other representation is similar to Darknet-way of representing a model. It
55Source: https://www.tensorﬂow.org/guide/checkpoint
39
==PAGE:49==

is called Saved Model format including a serialized description of the computation deﬁned by the
model, in addition to the parameter values. It is considered as a complete TensorFlow program56.
During training, the values of the weights are continuously updated due to back-propagation.
These dynamic variables have to be changed to static to run inference. The freeze graph operation
eﬀectively freezes the weights of a TensorFlow model by converting checkpoint variables into constant
operations. There is usually a last step before running TensorFlow model inference which is optimizing
the frozen graph for inference. It makes the running time shorter as it removes the training nodes that
are not needed for inference. Both the Frozen Graph and the Frozen Inference Graph are protobuf
ﬁles, however the latter has higher speed and needs less storage space. Resuming training is a usual
process of further improving an already trained network. It is possible from the aforementioned model
representations, except the Frozen Inference Graph. It does not contain information on the training
computations that disables unfreezing its weights. This is represented by the characteristics of the
arrows in Figure 24 (i.e., unidirectional, bidirectional). The frozen inference graph representations
were used already for the previous test with diﬀerent hardware architectures (Figure 23).
There is an implementation found online which covers the whole pipeline to get the last TensorFlow
representation from the Darknet model57. The same pipeline is implemented in diﬀerent repositories
to convert PyTorch models into TensorFlow graph structure58. In Section 3.2.2 it is mentioned that
PyTorch is a leading framework for research purposes. Therefore, there are also implementations that
convert the TensorFlow models into meaningful network representations for PyTorch. In contrary,
Darknet is only used for the YOLO models, except YOLO v5, therefore it is a unidirectional conversion.
Obtaining the TFLite model
The last step is converting the protobuf inference graph formats into TFLite representation for all
models.
The TensorFlow Lite library serves two major purposes: Providing conversion tools and
running TFLite inference59.
The purpose of the conversion is to accelerate the running time as well as reducing the model
size. The astonishing advantage of the process is the unaﬀected accuracy (without using quantization,
which is discussed in Section 4.2).
The conversion pipeline is theoretically a simple procedure, however there were numerous issues
and challenges which were addressed during the implementation of some of the selected models.
It is important to note that TensorFlow 2.0 was released in September 201960. Furthermore, the last
stable release is the TF 2.4. Despite it having been released a year ago, several functionalities do not
work for particular tasks. The solution sometimes is to downgrade the environment to TensorFlow
1.15 or to use the developer version tf-nightly. The latter contains the most up-to-date version of
TensorFlow with the latest features developed until the previous night.
After altering between diﬀerent library versions, all model conversions are succeeded. MobileNet
SSD and MobileDet models are converted by the functions from the Object Detection API of Ten-
sorFlow 1.1561.
The EﬃcientDet D0 conversion is done by the functions from the oﬃcial Google
repository62. Finally, the Tiny-YOLO v4 and YOLO v5s TFLite ﬁles are obtained from repositories
of public implementation63,64.

Running the converted models requires a special TFLite interpreter. It uses custom memory allocation
and static graph ordering to guarantee low latency and minimal load. Running TFLite inference follows

4.2
Optimization - quantization
One of the reasons of choosing TFLite is conﬁrmed in the previous Section 4.1 by demonstrating the
fastest running time on the Raspberry Pi among frameworks. The second reason is that only TFLite
models are compatible with the Edge TPU. However, running a model on the Edge TPU requires a
particular representation that supports a speciﬁc set of neural network operations and architectures.
The weights in the TFLite model have to be converted into an 8-bit ﬁxed format to obtain the
Edge TPU model. This process is called quantization and was introduced in Section 2.4 as a way of
optimizing neural networks to reduce inference time and model size without signiﬁcant accuracy loss.
The aim of this subsection is not only to obtain the right model format for the Edge TPU but also to
get familiar with quantization as an optimization technique.
After the brief literature review, further papers could be examined [66] within the ﬁeld of quan-
tization. However, the TensorFlow descriptions are taken as the theoretical ground of the described
techniques due to the practical nature of the topic. Furthermore, TensorFlow also published a white
paper [67] with several acknowledged authors that gives academic credit for the theoretical contents
of the examined TensorFlow pages for this topic74,75.
There are three major optimization techniques used: model pruning, clustering, and quantization.
Model pruning is a popular technique of removing parts of the network that have minor impact on
the output prediction to reduce latency. Clustering is a technique of reducing unique weight values
by grouping them into a predeﬁned number of clusters at each layer. Sharing the centroid values
within a cluster is a way to reduce the number of parameters. Finally, quantization reduces the
precision of the parameters to reduce the required computation.
Quantization is the only implemented technique among the three methods for two reasons. First, it
is needed because of the Edge TPU deployment. Furthermore, pruning has not been implemented for
TFLite with object detection models and clustering is mainly supported with classiﬁcation networks.

4.2.1
Quantization techniques
There are two major ways of performing quantization on a neural network:
1. Quantization Aware Training: It emulates inference-time quantization by inserting fake-quantization
nodes during the full-precision training procedure. This way, the model learns robust parameters
against the loss of precision invoked with quantization.
2. Post Training Quantization: Quantizing already-trained ﬂoat models to accelerate inference and
reduce model size.
Quantization aware training will be discussed in more detail in Chapter 5. The important note is
that the TensorFlow 1 model zoo contains pretrained models which were trained using this technique
(they have quantization or edgetpu in the ﬁlename).

The second type is the ﬂoat 16 quantization which reduces the precision of all weight, activation and
input/output parameters from 32 bits to 16 bits. It also reduces the model size to half of the original.
GPU mobile devices are the only beneﬁciary of the ﬂoat 16 representation. CPU-based architectures
can also run it, however they de-quantize the parameters, therefore the only beneﬁt is the model
compression for CPUs without any change in the latency.
The third option is dynamic range optimization. It reduces the weight precision to integer 8-bit
precision by leaving the activation and input/output (I/O) parameters at 32-bits. It results in a model
four times smaller than the original size. It can only be used for CPU-inference. Even though the
activations are 32-bits, dynamic-range operators are introduced to perform computations with 8-bit
weights by dynamically quantize the activations based on their range projected to 8-bits only during
the inference time.
The fourth method, integer 8 quantization with ﬂoat fallback, is a similar concept to the
dynamic range technique with the only diﬀerence of having already converted activations to 8-bits
integer format before inference.
The last and most important is the full integer technique that quantizes to 8-bits the I/O,
activations, and weights as well. This enables deploying models on the Edge TPU and DSP devices,
since both work on an 8-bit based architecture.
Representative dataset
The weight quantization follows a static approach to approximate the integer 8-bit precision values.
However, reducing the activation and I/O precision to 8-bit requires a more complex calibration. The
goal is to estimate the range of all tensor values in the original model (except weights and bias). The
calibration requires a representative dataset which is a subset of the training and validation data.
The representative dataset is only required for the full integer and integer quantization with ﬂoat
fallback, since these are the only techniques which quantize the activations.

5.1
Dataset
It is often said that the key to deep learning and machine learning is data. Having suﬃcient quantity
and quality of training data is often the bottleneck of achieving the desired results due to the cum-
bersome work of annotation. The scope of this thesis is not solving a speciﬁc dataset related problem,
however, to ﬁnd state-of-the-art devices and models to run eﬃcient and eﬀective object detection on
the edge. Transfer learning aims to study the transferability and co-adaptation phenomena among the
ﬁve selected models. The mAP accuracy is given for each model trained on the MS-COCO collection,
however, it has to be revealed how eﬀective they perform on other datasets.
The following considerations are taken into account while selecting the new datasets:
1. Transfer learning should be carried out on preferably two datasets to conﬁrm observations on
transferability and challenge the networks by diﬀerent learning tasks.
2. The images should already be annotated not to spend an unnecessarily long time on the prepro-
cessing stage.
3. One dataset should contain more, the other less similar features among the objects in the images.
4. The images should be publicly available for usage.
5. The number of images should be at least between 1000 −2000 to avoid overﬁtting of the object
detection networks.
6. Datasets should have industrial relevance.
7. Potential applications should require edge device(s).
Based on the requirements listed above, two datasets are selected from the database of Northeastern

The ﬁrst dataset contains workers in workplace settings which require a hard hat. The images were

The dataset contains 7041 images which is an adequate number for transfer learning. It has only

• Helmet: 19747 (73.0%)
• Head: 6677 (24.70%)
• Person: 615 (2.3%)

learning it. The average annotation per image is 3.84 on average.

that most of the images are wide and not square images. This brings additional complexity to feature
extraction since all networks have a square input, which requires reshaping of the images distorting
of the objects.

The dataset is chosen because of the similar characteristics of a head and helmet to the COCO dataset
which the pretrained models were trained on. The expectation for all ﬁve networks is to be able to
transfer the already known features. Moreover, the number of images and the low number of classes
provide a large amount of information for the ﬁve networks to learn from.
Safety is a priority in all jobs where human life is exposed to danger.
Wearing a helmet at
construction sites can become galling for the workers as well as they can forget to wear it after breaks
during their daily routine. Unsafe behavior must be detected and alerted.
The dataset can provide a suitable base to develop an automatic hard hat usage detector which
can give an alert to the leader of the construction site to notify the involved people. Resolve the
problem with an Object detection network is an applicable solution. Multiple edge devices can be
placed across the construction site to capture workers. It is a ﬂexible system which is important for
the dynamic environment. Furthermore, it is a cost-friendly way of reinforcing the phenomenon of
”smart construction sites”.

Dataset splits
Before describing the individual data preparation processes, the data splits are introduced. Training
a neural network requires special attention regarding data division.
The training and evaluation
procedure uses three types of data: train, validation, and test split. Train split represents the major
part of the images which are used to ﬁnd the weights that give the best model ﬁt via the loss function.
Validation set is used to provide an unbiased evaluation of the model parameters during training.
The network relies on the validation performance to ﬁne-tune the hyperparameters (i.e., the learning
rate) of the training repeatedly after a deﬁned number of epochs. Thus, the network does not learn
from this data but only sees it. Test split is needed to get an unbiased evaluation of the ﬁnal model.
It is important to have it besides the validation set because the network does not learn from the
validation set, however, it still has a bias due to the conﬁguration steps during training. Contrary to
validation, the test images are only seen after the complete training is ﬁnished.
58
==PAGE:68==
s182596 - Bertalan Kovacs
There are common ratios used for the three sets. The selected one for this thesis is 70% train,
15% validation, and 15% test.
Testing on diﬀerent splits helps to generalize the bias coming from the speciﬁc test sets, which is
introduced by containing less variance in the subsets compared to the whole dataset. Cross valida-
tion and K-fold validation are the most relevant machine learning evaluation tools for this purpose.
However, given the time and hardware limitations (which will be introduced in Section 5.3), only a
subset of the models are tested on three diﬀerent splits of the Hard Hat Workers dataset. The mean
and standard deviation are calculated on the selected models to see how close the training results are
to each other.
TFRecords
TFRecords are a particular data representation for the TensorFlow framework. It stores a sequence
of binary records for faster and more eﬃcient serialization of structured training data. TensorFlow
provides inner functions that can be used for conversion within the Object Detection API. The con-
version script is written in TF 1 because the functions are not fully ported to TF 2 yet. A repository
was found which can be used for a large part of the conversion90. The following steps are needed to
obtain the TFRecord representation from the initial XML and JPG ﬁles:
1. Split the downloaded raw data accordingly to the deﬁned 70% −15% −15% ratios and save
indices for the latter data formats.
2. Create separate folders for the three sets and move the image and annotation ﬁles respectively.
3. Create the label ﬁle based on the classes.
4. Iterate through the images and the matching annotation information. Append all attributes to
a tf.train.Example TensorFlow object.
5. Convert the objects to binary records.
The same process is repeated two more times with shuﬄed indices to obtain the aforementioned
three versions (each containing the three further splits). The diﬀerent splits are made by shuﬄed
indices, therefore overlapping is possible between diﬀerent test data. This should further reduce the
variance between the three tests.
Darknet and Pytorch format
Even though PyTorch and Darknet are diﬀerent frameworks, the data structure they require is sim-
ilar. Both start with the conversion of XML representations to a text ﬁle structure where each ﬁle
belongs to an image and contains the needed information of the bounding box annotations. After the
conversion, the ﬁles are separated into three sub-folders based on the same indexing which is used for
the TFRecords. A label ﬁle is also created for both frameworks containing the classes.
After the data preprocessing, all ﬁles are zipped and uploaded to Google Drive where the training
codes can download the appropriate ﬁles. All formats contain the 70% −15% −15% splits.

Model retraining
After the data is ready, the next important step is to set up the training pipeline for each model,
which can load various datasets and complete transfer learning by using the pretrained models.
Transfer learning can be done in two ways91:

1. Last layers-only retraining: Only the last few layers are retrained. These layers are responsible
for the classiﬁcation of the objects inside the predicted bounding boxes. The rest of the layers
are frozen keeping the pretrained weights. The method is useful when the training time is a
crucial factor. Meaningful results can only be obtained by using a small dataset with similar
features to the original data on which the model was trained.
2. Full model retraining: The whole model is retrained updating all layers. The pretrained model
provides the initial knowledge to the network. It gives higher accuracy than the last layers-only
method by the cost of the training time. It requires more data samples to avoid overﬁtting.
In this section, the second option, the full model retraining is implemented for both datasets with
all selected models.
Diﬀerent techniques and hyperparameters are inﬂuencing the progress and result of training.
Learning rate controls how large the model response is for a given error estimation during the
model weight update. Dropout zeros out randomly selected weight nodes during training to make
the model more robust. Optimizer is referring to the algorithm which determines the optimization
of the gradient descent. Various methods exist, e.g., Momentum, Adagrad, Adam, RMSprop, etc...
This thesis does not aim to experiment with these techniques having numerous models and two
datasets.
The diﬀerent training conﬁgurations are sought to be completed with the most similar
settings (besides hardware and network-speciﬁc limitations).
Quantization aware training
Quantization aware training was mentioned in Section 4.2, moreover, two quantization aware pre-
trained model were used in the ﬁnal implementation in Section 4.3. Since these were the only working
pretrained models deployed on the Edge TPU, quantization aware training is implemented on the
MobileNet SSD and MobileDet models.
Full integer models use the 8-bit representation. Quantization aware training emulates the inference-
time quantization during the training procedure92 . The precision has to be deﬁned in the TensorFlow
training conﬁguration ﬁle by adding the following ﬁeld93:
graph_rewriter {
quantization {
delay: 0
weight_bits: 8
activation_bits: 8
}
}
The ﬁrst parameter speciﬁes the delay, which is the number of steps the quantization does not take
eﬀect. The other two parameters guarantee the full integer compatible model format.

5.2.1
Batch size selection
In the neural network terminology, it is important to distinguish epoch, iteration, and batch size. An
epoch is the unit of passing all training samples through the network. Bath size is the number of
images that are propagated forward and backward in one step. Higher batch size requires more RAM.
Finally, iteration shows the number of passes needed to complete one epoch.
As an example, with a batch size of 32, it takes 40 iterations to complete one epoch on the 1260
training images of the Surface Defect dataset (the last batch contains fewer images).
The upper limit of the batch size is determined by the available RAM size. The higher the value
is, the faster the training procedure will ﬁnish. However, a too large batch size can also reduce the
accuracy. Smaller batch size tends to converge faster, however, it brings more noise in the training
process. Large value results in a slower but more stable converging process.
Based on some prior experiments and online examples, the batch size is chosen as 32 for all model
training, except the EﬃcientDet D0, where only a batch size of 16 is compatible with the available
RAM.

described by Equation 4.
epochs · num images = iterations · batch size
(4)
Both sides give the total number of images passing the network during training. The left side demon-
strates it by multiplying the epoch number with the number of images. One epoch means one full pass
of all images, therefore the expression gives the number of all passed images throughout the whole
training. The right side gives the same value by multiplying the number of iterations with the batch
size. Iteration is deﬁned as the number of times a whole batch is passed. Multiplying it with the
batch size also gives back the number of all passed images. The blue values can be easily calculated
given the Equation 4.

6.2.1
Requirement prioritization
Diﬀerent applications have various prioritizations among the requirements in industrial applications.
The following list presents some of the combinations with the recommended hardware and network.
• Training time + price: Tiny-YOLO v4. Because of the quick training time, the Raspberry
Pi running the Tiny-YOLO v4 is a great candidate for prototyping and experimenting.
• Accuracy + price: YOLO v5s. Without high-speed demand, the Raspberry Pi is a perfect
candidate to run accurate object detection on custom datasets. The choice of the detection
conﬁdence threshold is an important factor to inﬂuence the number of FPs and FNs.
• Inference time:
MobileNet v2 SSD. If an application requires high inference speed, the
MobileNet-based network is a great choice running on the Edge TPU.
It is noted that the tested combinations cannot provide a promising candidate for tasks where
speed and accuracy are both equally important. However, in many scenarios, this is the desired goal.
This takes the discussion back to the hardware selection. The Raspberry Pi with the Edge TPU
was selected to accelerate neural networks at a cheap price. It included the assumption of being able
to deploy all ﬁve networks, which was only fulﬁlled for the two mobile models.
Thus, due to the unmappable operations regarding the YOLO models, switching to a GPU based
device and continue carrying out further tests seems reasonable to enable acceleration for the YOLO
models. The other option is to wait for the Google developers to include new operations which allow
a more successful mapping process.
Another reason for solving the YOLO model acceleration is the appeared YOLObile and Scaled-
YOLO v4 models. Their appearance proves the evolving research direction of the YOLO model family
which requires GPU based devices or a more ﬂexible Edge TPU device variant.

Conclusion
7.1
Conclusions
The aim of the thesis was to make deep learning available on low-cost edge devices as an alternative
for smart cameras.
Chapter 1 deﬁned the detailed problem statement together with the requirements and limitations
of the thesis.
Chapter 2 provided thorough research, particularly focusing on the history of object detection
networks to locate and understand the state-of-the-art models. It also gave a brief introduction to
edge devices and the potential hardware solutions which are able to deploy deep neural networks.
Based on the previous ﬁndings, Chapter 3 dived into the edge hardware market and the available
framework options which can be used for training and for inference on embedded devices. It also
zoomed in to the group of most recent and most state-of-the-art mobile object detection networks to
assess their capabilities. Finally, the most promising combinations of the system components were
selected which became a Raspberry Pi 4 with an Edge TPU accelerator running TFLite. The selected
networks were MobileNet v2 SSD, MobileDet, EﬃcientDet D0, Tiny-YOLO v4, and YOLO v5s.
In Chapter 4 all models were converted to a TFLite format and deployed on the Raspberry Pi
to measure their inference time capabilities. After testing multiple inference engines, the TensorFlow
Lite framework proved to provide the fastest execution time. It was also observed that MobileNet v2
SSD and MobileDet achieved the fastest running time among all models.
Later the quantization techniques were implemented on the models (except the EﬃcientDet D0)
to prepare the deployment on the Edge TPU which requires full-integer models. It was concluded
from the test that quantization completed the anticipated size reduction, however, it did not decrease
latency.
Edge TPU compatible formats were compiled from the four full-integer models. The neural network
computations were successfully mapped to the Edge TPU in the case of the two mobile networks,
while the YOLO models have a high number of unmappable operations.
With the four compiled models, the second round of inference tests was completed. The two models
with a signiﬁcant acceleration were the MobileNet v2 SSD and MobileDet networks.
The highest running speed of MobileNet v2 SSDLite reached 65 FPS on the Edge TPU. The
achieved speed clearly demonstrated the power and topicality of the selected system components to
run object detection on embedded devices with low latency.
Chapter 5 prepared two datasets to retrain all networks and compare their ﬂexibility in terms of
learning custom data samples. All networks accomplished the training process on two datasets: the
Hard Hat Workers and the Surface Defect databases.
The most accurate model performance was achieved by the YOLO v5s model. It was concluded that
only the mobile networks accomplished better results on the data with smaller-size textural features
against the COCO-similar dataset. Generally, the transfer learning gave satisfying results when good
annotations were present. It proved the ﬂexible learning capability of all models.
Finally, Chapter 6 merged the performance results based on the diﬀerent tests and elaborated on
various attributes to provide hardware-model alternatives for diﬀerent computer vision requirements.
It found Tiny-YOLO v4 as the best model for fast and iterative training. It concluded the mobile
models having the highest eﬃciency. Finally, it stated YOLO v5s as the most eﬀective network with
an acceptable speed and dynamic input size.
Five state-of-the-art models are deployed and validated via transfer learning on two custom
datasets. Suggested alternatives are made to replace the expensive and closed software structured

smart cameras, therefore both major goals are satisﬁed.

Future work
Further work can be done in various ﬁelds to establish deeper investigations.

• Models: Scaled-YOLO v4 and YOLObile show promising results and were released in December

– Test the accuracy of quantized models and benchmark how it decreases.

berry Pi.

– Investigate multiple transfer learning techniques with layer freezing, etc...
