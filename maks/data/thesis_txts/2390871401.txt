Abstract
We present a novel concept for robust shape matching and classiﬁcation method for indoor
3D point cloud data. Based on this concept we designed several deep learning architectures
and tested them on diﬀerent point cloud data sets to compare their performance. The
results show that a state-of-the-art level accuracy can be achieved on a noise-free data set.
Also our architectures can exhibit higher robustness for a highly occluded data set.
The key idea of this concept is to feed statistical properties extracted from 3D point cloud
into a neural network which has a novel architecture. Our experimental results show that
feeding sampled point pairs from 3D point cloud followed by a neural network with a
”symmetrization function”, or feeding a hand-crafted statistical descriptor based on point
pairs sampled from a 3D point cloud followed by a convolutional neural network, can solve
the permutation problem of the input data, while oﬀering higher robustness for highly
corrupted data sets.
We adopt the statistical random sampling-based descriptor [Che16], as the input into the
neural network. This serves as a symmetrization function to address the ordering problem
of unordered point sets. Our experimental results show that this method can oﬀer more
robustness to noise and occlusion as compared to PointNet [QSM+17].
For feeding sampled point pairs into a neural network, we mainly adopt PointNet archi-
tecture, which projects raw 3D points into a high dimensional space by a shared weights
neural network and uses a max-pooling operation to select maximum response along each
dimension among all points to represent the entire 3D point cloud. Finally a fully con-
nected neural network is attached as a classiﬁer. Instead of consuming raw 3D points,
we concatenate sampled point pairs from raw data as 6D points to feed into PointNet.
This can achieve a similar accuracy on a data set which has lower noise level compared to
PointNet, but higher accuracy on a data set with higher levels of noise and occlusion.

Chapter 1
Introduction
In recent years, as 3D range scanners became more and more popular, the acquisition of
3D point cloud data became much easier. This provides a rich source of information about
an indoor environment. However, the 3D point cloud data does not have meaningful labels
that are understandable for humans. In order to make use of these data, algorithms for
3D shape matching and classiﬁcation have been investigated over these years.
1.1
Motivation
Previously, hand-crafted descriptors have been thoroughly investigated for 3D point cloud
data. A descriptor is a vital element for similarity estimation, which can be implemented on
shape matching and classiﬁcation, for instance assigning a class label afterwards. However
the hand-crafted descriptor is not guaranteed to take the most representative information
into account while maintaining the computational eﬃciency, leading to an unsatisfying
result.
Recent works focus on deep learning strategies to extract abstract features from 3D point
cloud data and classify objects at the same time. These publications showed that this strat-
egy has a potential to achieve even better than the state-of-the-art performance. However
there are signiﬁcant challenges as compared to 2D images. In particular, the 3D point
cloud data is inherently unordered, which makes it harder to apply convolution and verify
its symmetry. This causes a huge problem for the neural networks, as directly feeding un-
sorted data does not exhibit satisfactory performance. In order to deal with this challenge,
diﬀerent strategies have been investigated, such as training additional neural network to
sort the input data into a canonical form, voxelization, taking multi-view 2D projections
(images) and converting the problem into 2D dimension, or deﬁning a ”symmetrization
function”, whose output is invariant to the ordering.
Furthermore, ﬁnding the optimal hyperparameters in deep learning is another diﬃculty, as

there is limited understanding about what exactly happens in the network. In this sense
the controllability of the performance from deep learning strategy is worse than the one of
traditional hand-crafted techniques.
Our motivation is to investigate a new strategy to balance the advantages of hand-crafted
and neural network strategies, and more importantly, to be more robust to noise and
occlusion, which are common in 3D point cloud data obtained with 3D range scanners.
1.2
Overview
We ﬁrst review and evaluate several hand-crafted 3D point cloud descriptors from Stan-
ford [ASZS15] data set and compare them by testing their receiver operating characteristic
(ROC) curve. After selecting the most promising descriptor as the symmetrization func-
tion, we feed it into several designed neural networks and test the accuracies on given
data sets. The experimental results show that a statistical feature descriptor [Che16] is
suitable to be the hand-crafted symmetrization function to solve the permutation problem
of the input data and has more potential to achieve better classiﬁcation and matching
performance when combined with a neural network.
We also propose a second strategy of neural network inspired by PointNet [QSM+17], but
with modiﬁcation that combines advantages from statistical properties: randomly sampled
point pairs. Experimental results indicate that feeding sampled point pairs as the input
into PointNet and using our statistical feature descriptor based neural networks can achieve
similar accuracy of PointNet on ModelNet40 [WSK+15] data set that contains clean mesh
models with no noise or occlusion, but shows higher robustness than PointNet in highly
noisy and occluded Stanford data set.
1.3
Thesis outline
This thesis is organized as follows:
Related work is discussed in Chapter 2. In Chapter 3, methodology for shape matching and
classiﬁcation is reviewed. Experimental evaluations are presented in Chapter 4. Finally
we conclude with the summary of this thesis.

Chapter 2
Related Work
In this chapter, we will discuss diﬀerent strategies to deal with 3D point cloud classiﬁcation
problem. They can be divided into two diﬀerent directions in general. The ﬁrst one is the
traditional hand-crafted descriptors’ representation while the second one is based on neural
networks.
2.1
Hand-crafted descriptors
Unlike 2D images, 3D point clouds often have various sizes, point densities as well as
diﬀerent amount of points. Furthermore, though two objects are identical to each other,
they could be in diﬀerent orientations. These are some common challenges in 3D point
cloud representation. Descriptors are introduced to overcome these diﬃculties as signatures
for 3D point clouds. Usually descriptors have relatively lower dimensions so that similarity
measurements between objects can be calculated based on them. Descriptors can be divided
into local and global ones, depending on whether they are computed on the entire object
or only on its part.
2.1.1
Local descriptors
Usually local descriptors are extracted from a certain neighborhood of an individual point.
This only covers local information of a keypoint, and all local descriptors from an object
aggregate the global information.
Point Feature Histogram [RBMB08] is a local descriptor based on relative relationships
of all point pair combinations. To represent the relative relationship of a point pair, Ps
and Pt, the quadruplet (α, φ, θ, d) is used, which are 3 angles calculated from their normal
vectors ns and nt, and the euclidean distance between two points.

Hence for all point pairs, the quadruplets are binned into a 125-dimensional histogram,
which is the ﬁnal representation of the PFH descriptor. As described above, the quadru-
plets are calculated based on the surface normal vector of each point, thus the result is
highly dependant on the quality of the surface normal estimation.
Fast Point Feature Histogram (FPFH) [RBB09] is a modiﬁcation of PFH. It aims to reduce
the complexity of PFH, due to the fact that PFH has to iterate through all point pair
combinations. Instead, FPFH does not iterate through all point pairs, but only the query
point and its neighbors. This can greatly reduce the overall complexity so that it has the
possibility to be implemented in real-time applications. On the other hand, it might miss
some important point pairs that capture geometry around the query point.

2.1.2
Global descriptors
A global descriptor represents the entire information of a given point cloud, thus pre-
segmentation is needed before global descriptor calculation.
3D HOG [DAGT15] is a global descriptor, which simply extends the HOG descriptor for
2D image into 3D. It voxelizes the point cloud into a voxel grid and then calculates a
statistical histogram of oriented gradients. Though the rotation invariance is obtained by
the orientated direction, the robustness to noise and occlusion in the point cloud data is
an unsolved issue.
Figure 2.3: The 3D Voxel HOG visualisation of an example object, sticks on the surface
of the object indicate the local gradients [DAGT15].
The Ensemble of Shape Functions (ESF) [WV11] voxelizes the point cloud as well. This
randomly sampled based descriptor is represented by 3 diﬀerent shape functions. These
shape functions cover the distance, angle and area information as well as visibility between
point pairs described as ”In”, ”Out” or ”Mix”. Though it is powerful, the disadvantage is
that ESF is not scale invariant. Furthermore, the criteria of judging ”In”, ”Out” or ”Mix”
may not be robust in the scanned point clouds since the boundary points are subject to
high level of noise and occlusion.
Another global descriptor proposed [Che16] has similar idea of PFH. It is a statistical
descriptor containing the euclidean distances of sampled point pairs, the normal diﬀerences,
the plane angle diﬀerences and the ratio of visibility binned into a 20 ∗5 ∗4 ∗3 histogram.
This descriptor is rotation invariant but scale variant, and it can capture the correlations
between various features, thus enhancing the descriptive power.

2.2
Deep learning strategies
Deep learning strategy has been widely investigated in 2D image domain and has shown
outstanding performance as compared to traditional computer vision methods. In recent
years people also started designing deep learning architectures to recognize and classify
3D point cloud data. However, 3D data poses new challenges. One of the major issue
is that, unlike 2D images which are aligned pixel by pixel in sequence, 3D point cloud
has a data permutation problem: diﬀerent orderings of 3D data do not change the object
at all and result in the same representation. This poses a huge problem for the neural
networks, as feeding data in a diﬀerent sequence can confuse the network and result in a
poor performance.
The ﬁrst strategy is simply extending 2D CNN into 3D, but it requires voxelization of
the data points. This is called volumetric data. VoxNet [MS15] is based on this idea.
However, voxelization is constrained by the resolution of the object because of the large
computational consumption on 3D CNN. Also the sparseness of the data set can cost 3D
CNN even more unnecessary computation.
==PAGE:12==

Figure 2.5: The VoxNet Architecture [MS15].
Multiview CNN [SMKLM15] investigated methods to render 3D point clouds into 2D
images, for example multiview projection, and then apply 2D CNN for object classiﬁcation.
Though these result in high classiﬁcation performance, rendered images from multiple views
only partially capture the entire geometry of an object. Furthermore, viewpoint selection
criteria is diﬃcult to deﬁne as well.
Figure 2.6: The Multiview CNN pipeline [SMKLM15].
Some other eﬀorts focus on directly consuming unsorted point cloud, and several strate-
gies are investigated. One of them is to sort the point cloud into a canonical form by
==PAGE:13==

an additional neural network. Another one is to train an RNN to learn diﬀerent input
permutations of the data. Others include a method to deﬁne a symmetrization function,
whose output is invariant to input permutation, to ensemble all information from the raw
data. PointNet [QSM+17] approaches this problem by deriving an abstract symmetrization
function using a dedicated neural network and a max-pooling operation.
Figure 2.7: The Pointnet architecture [QSM+17].
PointNet++ [RYSG17] is an advanced version of PointNet, which makes use of the capacity
of PointNet that can aggregate the information of a set of points into a single ”point
vector” to capture local structures of the data. Instead of directly feeding all 3D points
into a PointNet, it ﬁrst partitions point cloud into local patches by iterative farthest point
sampling and nearest neighbor classiﬁcation and then these local patches are fed into a
PointNet individually to extract the representative ”point vector” to encode local details.
And ﬁnally all these representative ”point vectors” are fed into a ﬁnal PointNet to aggregate
information.
Figure 2.8: The Pointnet++ architecture [RYSG17].

Chapter 3
Methodology
Our goal is to ﬁnd an architecture that can achieve classiﬁcation accuracy as high as
possible, not only on ideal point cloud data sets without noise, but also on data sets
subject to high level of noise and occlusion. We investigate neural network based scenarios
that are able to capture statistical properties of the data set as input.
We proposed
two directions: our proposed descriptor [Che16] plus convolutional neural networks and
replacing raw points by sampled point pairs as input to PointNet [QSM+17]. We expect
these methods can oﬀer more robustness to noise and occlusions.
3.1
Convolutional neural network scenarios
Though PointNet [QSM+17] shows high classiﬁcation accuracy on Modelnet40 [WSK+15]
data set, one of its main disadvantage, as claimed by the authors, is that PointNet can not
capture local features of the point cloud. It generally learns the ”critical points” of the
object. However a realistic point cloud data usually has much noise and corruption. This
means, local features are more valuable to the classiﬁcation task.

On the other hand, convolutional neural network is widely used in 2D image classiﬁcation
tasks. Its sliding convolutional window captures information from local receptive ﬁeld,
thus resulting in a great success in 2D domain. Inspired by this, we adopt convolutional
neural network for our 3D point cloud classiﬁcation.
Unlike VoxelNet [MS15], which suﬀers from resolution and computational insuﬃciency
of 3D convolutions, as discussed in the introduction chapter, our key idea is to capture
statistical properties of diﬀerent shapes. This is presumably able to capture local features
and also is more robust to noise and occlusion.
3.1.1
Symmetric function selection
Due to the fact that 3D point cloud data is inherently unordered, we have to convert our
input data into a form that invariant to the raw data permutation. This is due to the fact
that the permutation of points in the point cloud still gives the same geometrical shape.
A straight forward idea is to adopt a histogram based descriptor, which should be able to
capture statistical properties as well. We also abandoned PFH [RBMB08] because it is too
slow.
We list out several candidates: 3D HOG [DAGT15], ESF [WV11] and a sampling based
statistical feature descriptor [Che16] as introduced in previous chapter.
We plot the confusion matrix of 3D HOG descriptor on a selected part of Stanford data set
[ASZS15] and the result indicates that it is diﬃcult for 3D HOG to distinguish diﬀerent
objects with diﬀerent orientations.

So we focus on ESF descriptor and the sampling based statistical feature descriptor [Che16].
We test their L1 norm confusion matrix and ROC curves on Stanford data set [ASZS15]
to verify whether they are suitable for this task.
The results indicate that our proposed descriptor [Che16] has higher potential when it
is followed by a neural network classiﬁer. Thus we select our proposed descriptor as the
symmetric function in our convolutional neural network architectures.
This proposed descriptor, as introduced in previous chapter, contains the euclidean dis-
tances of the sampled point pairs , the normal diﬀerences, the plane angle diﬀerences and
the ratio of visibility binned into a 4-dimensional histogram with a total number of 1200
bins.
3.1.2
Convolutional neural network architectures
After the selection of symmetrization function, we design diﬀerent neural network scenarios
for the further process of our proposed feature [Che16], and output the scores of each class.

reshaping to 2D is unavoidable. Thus we reshape the histogram (1200 bins) into a 30 by
40 2D image and then assign 3 layers of convolutional neural network, followed by a 2 by
2 max pooling layer and fully connected layer with dropout.

Chapter 4
Experimental Results
In this chapter, we verify our designed scenarios on two diﬀerent point cloud data sets:
ModelNet40 [WSK+15] and Stanford [ASZS15] data set. By analyzing the experimental
results, the scenarios will be further discussed.
4.1
ModelNet40 data set
ModelNet40 [WSK+15] data set is provided by Princeton ModelNet project that aims to
oﬀer comprehensive clean collection of 3D CAD models for objects. The CAD models
have been divided into training and testing splits, containing 40 categories of objects. All
the CAD models are completely cleaned and saved in Object File Format (.oﬀ) ﬁle. This
format is used to represent the geometry of a model by specifying the polygons of the
model’s surface. It states the number of vertices, the number of faces, and the number of
edges. The vertices are listed with x, y, z coordinates, written one per line. After the list
of vertices, the faces are listed, with one face per line. For each face, the number of vertices
is speciﬁed.

4.2
Stanford data set
Though ModelNet40 is a common data set for purposes of algorithm veriﬁcation, it is a
set of cleaned mesh models with much of manual post-processing to generate point clouds
thus has no noise and corruption. However a scanned point cloud usually contains plenty
of noise and occlusion, thus we need a more realistic data set to test the robustness of our
designed approaches.
The Stanford data set [ASZS15] is collected in 6 large-scale indoor areas within 3 diﬀerent
buildings of classrooms and oﬃces, using the structured-light sensors to capture RGB
and depth images during a 360-degree rotation at each scanning location. After that, 3D
textured meshes are reconstructed and then point clouds are sampled. Finally the point
clouds of each category of objects are segmented from the entire point cloud and their
labels are annotated manually.

we attach our importance on robustness to noise and corruption. Though Stanford data
set [ASZS15] states the oﬃcial folds for training and testing, we found the training samples

4.4
Experimental results of PointNet’s variations:
scenarios 6 and 7 on ModelNet40 data set
As we discussed above, our proposed descriptor itself does not provide strong enough
information of those decisive small features of objects (low possibility to be sampled),
making it diﬃcult to generalize.
As introduced before, scenario 6 can be regarded as ”taking one step back”: to substitute
the histogram computational step of the proposed descriptor by a max pooling operation.
To implement this idea, we mainly adopt PointNet architecture, but instead of using raw
points as input we concatenate f1 (euclidean distance), f2 (normal angle diﬀerence), f3
(plane angle diﬀerence) and f4 (ratio of visibility) together extracted from each sampled
point-pair as 4D abstract points.
Experiments indicate the following: scenario 6 does not converge with 0.5 dropout rate.
For 0 dropout, scenario 6 reaches 100% training accuracy but only has 19% testing accu-
racy. Thus the PointNet architecture is not able to learn from these complicated ”abstract
points”.
Figure 4.8: A typical divergence of scenario 6 with 0.5 dropout.

architecture but instead of feeding raw points as input, it concatenates the sampled point
pairs as 6D points as input. Also we have to adjust the output matrix of the ﬁrst T-net
in PointNet to a 6 by 6 matrix which is initialized as an identity matrix. We tested the
performance of scenario 7 without T-nets as well.

enlarged resolution results in no diﬀerence on the performance, which indicates that these

We also noticed that our 4D convolution-based scenario 4 performs better than 2D

Figure 4.12: A proposed descriptor from a ”chair” object.
In Figure 4.12, we visualize a proposed descriptor extracted from a ”chair” object and
notice that there are four obvious patterns (marked out in red circles).
Though they
appear in diﬀerent slices, since they have more or less the same position in their slices,
actually in 4D space they are clustered together. This indicates sampled point pairs from
a ”chair” have mainly the same kind of statistical feature.
We visualize the responses of nodes in the ﬁrst 4D convolutional layer. From the visual-
ization we found out that the ﬁrst 4D convolutional layer has the tendency to generate
diﬀerent perturbations of the input descriptor. This kind of learned perturbation does
not only perform simple gaussian smoothing but more importantly, it has ampliﬁed the
responses in diﬀerent regions and has suppressed some other regions as well. Each node
obtains a certain level of ampliﬁcation or suppression. This special perturbation beneﬁts
the generalization of our network, as the ﬁrst 4D convolution has the probability to ”zoom
in” those small decisive features while ”zooming out” some other redundant responses.

Chapter 5
Conclusion
In this thesis, we proposed a new conceptual strategy of dealing with 3D point cloud classi-
ﬁcation task: extracting statistical properties of an object as the input to neural networks.
Based on this idea we designed several architectures and tested their performances on two
diﬀerent data sets: ModelNet40 (a data set without noise) and Stanford (a data set with
high level of noise and occlusion).
From the experimental results we can draw a conclusion that feeding statistical features
of 3D point cloud as the input into a neural network potentially oﬀers more robustness to
noise and occlusion compared to PointNet. Especially, a scenario based on our proposed
descriptor and 4D convolutions can achieve good experimental results on ModelNet40 data
set and has even better performance than PointNet on Stanford data set. Also a scenario
that mainly adopts PointNet architecture but obtains concatenated sampled point-pairs
as the input can achieve similar accuracy on ModelNet40 data set, but higher accuracy in
Stanford data set as well.
We acknowledged that the performances of our proposed descriptor based scenarios have
the potential to be enhanced as those statistical features adopted in this descriptor are not
the best representative ones for a 3D point cloud.
